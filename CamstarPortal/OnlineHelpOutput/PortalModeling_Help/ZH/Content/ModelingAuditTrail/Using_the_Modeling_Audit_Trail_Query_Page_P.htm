使用审核追踪页

使用“审核追踪”页，可以查看现有和已删除建模对象的审核追踪。对象实例的审核跟踪即使在该对象被删除后依然存在。因此，您依然可以在对象被删除后查看该对象的审核跟踪。在“审核追踪”页上，可以查看以下对象类型：

• 命名数据对象
• 已修订对象
• 子实体

使用建模审核追踪页

在“CDO 类型”字段中单击以展开“已命名数据对象”、“已修订对象”或“子实体”，选择合适的建模对象。页面上的字段将根据所选的 CDO 类型相应变化。

有关使用“审核追踪”页查看不同 CDO 类型对象的审核追踪的信息，请参考以下主题：

• "查看命名数据对象的审核跟踪"
• “查看已修订对象的审核追踪”
• “查看建模对象子实体的审核追踪”

在选择 CDO 类型和建模对象之后，应用程序将激活“查看审核”按钮。单击此按钮可显示所选建模对象的审核追踪。

Note: 如果希望应用程序在建模审核追踪中以您的首选语言显示所有更新，则在定义建模对象实例之前设置该首选语言。

建模审核追踪对象类型

在“审核追踪”页的“CDO 类型”字段中选择的对象类型决定了在该页面上显示的其他字段。

下表定义了根据所选对象类型而显示的不同字段。

如果选择此类型 ...

将显示此字段

命名数据对象

名称

此字段对应于所选的特定 NDO。例如，如果您选择“员工”作为对象类型名称，则显示的“对象名称”字段为“员工”。

已修订对象

对象名称，修订版本

这些字段对应于所选的特定 RO。例如，如果您选择“产品”作为对象类型名称，则显示的“对象名称”字段为“产品”。“对象修订版本”字段也会和“产品”关联。

子实体

“父 CDO 类型”和“对象名称”

这些字段对应于所选的特定子实体。例如，当选择“培训记录”作为“对象类型名称”时，则显示“父类型 CDO”字段。选择父类型 CDO（例如“员工”）时，将显示对应于所选父类型的“对象名称”字段。在该示例中，显示的“对象名称”为“员工”。
