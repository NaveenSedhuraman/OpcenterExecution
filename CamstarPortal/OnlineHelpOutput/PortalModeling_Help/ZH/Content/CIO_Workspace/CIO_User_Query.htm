定义 CIO 用户查询

“CIO 用户查询”对象可用于为 CIO 出站消息传递创建用户查询。

定义 CIO 用户查询

“CIO 用户查询”对象上的“用户查询参数”表格包括参数值表达式的列。参数值表达式是可以在运行时计算参数值的表达式。参数值表达式的示例是：

Container.MfgOrder?Container.MfgOrder.Name:Container.Name

这称为三元表达式。本示例中的表达式解析为制造订单名称（如果它存在）。否则，它将解析为容器名称。

还可以在“参数值表达式”字段中使用对文本变量有效的任何表达式。有关文本变量的信息，请参考“定义文本变量”。

在使用用户查询测试弹出窗口时

单击“保存并测试用户查询”时，将显示“用户查询测试”弹出窗口。弹出窗口中将显示所有必需的参数字段。填充参数字段时，将显示包含结果的表格。如果不需要参数，应用程序将测试查询并提供结果。如果查询语法无效且表格中未显示任何结果，则会显示错误消息。

如何定义 CIO 用户查询

按照以下步骤定义 CIO 用户查询：

1. 打开 CIO 用户查询页。随即“建模”页中将显示“CIO 用户查询”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在 CIO 用户查询字段中输入用户查询的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 有关测试需要参数的查询的信息，请参考“如何测试需要参数的 CIO 用户查询”。

或

有关测试不需要参数的查询的信息，请参考“如何测试不需要参数的 CIO 用户查询”。

6. 单击保存。 应用程序保存建模对象并显示成功消息。

如何测试需要参数的 CIO 用户查询

按照以下步骤测试需要参数的 CIO 用户查询：

1. 执行“如何定义 CIO 用户查询”过程。

或

选择现有的 CIO 用户查询实例。

2. 在需要参数的查询文本字段中输入 SQL 语句。
3. 在“用户查询参数”表格中输入用户查询参数名称和数据类型。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存并测试查询以测试查询。将显示“用户查询测试”弹出窗口，同时在弹出窗口顶部显示必需的参数。
6. 输入查询返回的任何参数。
7. 单击测试查询。与输入的查询和参数匹配的任何结果都会显示在“查询结果”表格中。
8. 单击关闭。“用户查询测试”弹出窗口关闭。

如何测试不需要参数的 CIO 用户查询

按照以下步骤测试不需要参数的 CIO 用户查询：

1. 执行“如何定义 CIO 用户查询”过程。

或

选择现有的 CIO 用户查询实例。

2. 在不需要参数的查询文本字段中输入 SQL 语句。
3. 单击保存并测试查询。将出现“用户查询文本”弹出窗口，结果显示在“查询结果”表格中。
4. 单击关闭。“用户查询测试”弹出窗口关闭。

如何将参数添加到 CIO 用户查询

按照以下步骤将参数添加到 CIO 用户查询：

1. 执行“如何定义 CIO 用户查询”过程。

或

选择现有的 CIO 用户查询实例。

2. 在“用户查询参数”表格中单击添加新行。 将显示新行。
3. 在用户查询参数名称字段中输入参数的名称。该名称将显示为数据输入字段的标题，用户在该字段中键入所需的输入。
4. 从列表中选择数据类型。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 重复步骤 2-5，根据需要添加其他参数。
7. 单击保存。 应用程序保存建模对象并显示成功消息。
