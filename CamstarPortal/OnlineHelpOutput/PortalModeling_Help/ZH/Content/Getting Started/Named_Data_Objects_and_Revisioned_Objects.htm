命名数据对象和修订对象

Opcenter EX MDD 和 Opcenter EX CR 应用程序有两种类型的对象：已命名数据对象和已修订对象。这两种类型都在“建模”菜单中。

在定义已命名数据对象和已修订对象时

不可在对象名中使用无效字符。例如，不可使用撇号。有关信息，请参考以下特定于产品的相应系统设计师用户指南：

• Opcenter Execution Core Designer 用户指南
• Opcenter Execution Medical Device and Diagnostics Designer 用户指南
Note: 如果希望应用程序在建模审核追踪中以您的首选语言显示所有更新，则在定义建模对象实例之前设置该首选语言。

命名数据对象

已命名数据对象 (NDO) 是应用程序内由唯一名称（如“企业”或“作业”）标识的一类对象。NDO 实例是永久性的，即有关它们的信息会被写入数据库。

修订对象

已修订对象 (RO) 是具有名称和修订版本的对象，例如，“规范”或“产品”。已修订对象支持记录修订版本功能，这样就可以将一个版本指定为默认版本。与 NDO 一样，RO 也是永久性的。

已命名数据对象维护页和已修订对象维护页的另一个区别就是页面上的按钮不同。除了“关闭”、“保存”、“复制”和“删除”按钮以外，所有已修订对象还有两个其他按钮：“新增修订版本”和“复制修订版本”。有关使用这些按钮的信息，请参考“修订管理”。
