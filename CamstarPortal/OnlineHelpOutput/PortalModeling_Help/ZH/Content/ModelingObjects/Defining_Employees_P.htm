定义员工

员工是指在系统中执行如下活动的用户：维护建模定义、进行查询、管理安全管理和/或执行车间事务。系统管理员使用安全管理功能向员工授予特权，以使他们能够访问这些活动。

员工必须具有 Active Directory 帐户，且员工姓名必须与 Active Directory 登录 ID 匹配。有关用户名的确切语法，请与 Active Directory 系统管理员联系。

可以定义不具有帐户的员工，但这些员工无法登录。可以将它们作为执行容器事务或资源事务的员工加以输入。例如，具有 Active Directory 帐户（和相应权限）的主管可以登录到应用程序并访问起始页面。记录在起始页面上的员工姓名可以是主管或其他员工（实际执行启动容器这一工作的任何人）。此员工不一定具有 Active Directory 帐户。

为每个员工设置会话值可以对执行事务的人员限制查看。员工只能查看他或她被允许查看的内容，并具有一个用于搜索的较短值列表。这就限制了搜索时间并减少了犯错的机会。

安全模型

Opcenter EX MDD 和 Opcenter EX CR 的安全模型以基于角色的访问控制模型为基础。系统管理员为员工指派角色成员资格，以控制员工对整个系统或系统某些部分的访问权。为员工指定主要组织和角色成员资格时，需要为该员工设置安全权限。用户必须在组织中通过角色传播直接或间接地具有角色才能执行任何操作。

可以在“角色”页上定义角色。可以在“员工”页或“角色”页上指派角色成员资格。但是，只能在“角色”页上为该角色指派权限。

产品的安全指南包含有关安全管理的章节。该章讨论权限类型和模式、身份验证和授权过程的工作方式以及验证用户权限的方式。请根据具体产品参考《Opcenter Execution Medical Device and Diagnostics 安全指南》或《Opcenter Execution Core 安全指南》。

默认员工

默认情况下，Opcenter EX MDD 和 Opcenter EX CR 提供了三种员工定义：

• 管理员
• CamstarAdmin
• InSiteAdmin

最初登录到应用程序时，使用这些员工姓名之一。

最初实施 Opcenter EX MDD 或 Opcenter EX CR 或者转换为或更新到新版本并升级默认实例数据时，默认 Portal 菜单会自动指派给 Administrator、CamstarAdmin 和 InSiteAdmin 员工。可以使用“员工”页上的“门户菜单定义”字段为其他员工指派门户菜单。

默认角色

Siemens 还为默认企业组织提供多个预定义的角色。系统为默认员工指派全部或部分默认角色中的成员资格。还为具有 Change Management 许可的用户提供其他默认角色。

有关如何安全使用员工和角色的信息，请参考《Opcenter Execution Medical Device and Diagnostics 安全指南》或《Opcenter Execution Core 安全指南》。

定义员工时

至少一个默认员工必须是有效的 Active Directory 用户，这样首次登录到门户时才能成功。

如果是首次定义员工，确保您自己以管理员身份创建员工定义。可以使用 Windows 登录名作为员工姓名。如果要为其他员工创建定义，可以使用该员工的 Windows 登录名。

应用程序中的员工定义的使用方式如下：

• 作为已登录到 Portal 的员工，默认组织是您的主要组织。只有您在其中具有直接角色或传播角色的组织在门户页面上可供选择。
• 在应用程序中执行记录搜索时，可以输入所有者作为搜索准则的一部分。所有者是与要搜索的质量记录关联的员工。

注意以下事项：

• “员工”包含以下可选建模定义：“组织”、“字典”、“电子签名角色组”、“工厂”、“培训计划”、“作业”和“工作中心”。
• “员工”在所有容器事务和所有资源事务中都是可选字段。

在定义上报收件人时

上报收件人是在员工具有即将开始、到期或逾期的工作时将收到电子邮件通知的人员。例如，可以在工作到期时向员工的经理发送电子邮件通知。可以使用“上报收件人”表格中的“员工”字段定义上报收件人。

“更改管理设置”建模对象可用于设置要在审批者或协同者具有未完成的任务时发送给上报收件人的提醒通知。如果选择此类型的提醒通知，应用程序将向为员工指定的上报收件人发送提醒。有关设置审批提醒和协同提醒的信息，请参考“定义更改管理设置”。

如果未在“上报收件人”表格中选择上报收件人，则应用程序不会发送电子邮件通知。确保每个上报收件人的“员工”建模对象实例中的“电子邮件地址”字段都完整，否则收件人不会收到电子邮件通知。

文档管理

可以指定用户名和密码，以用于访问存储在外部文档存储库中的文档。“文档”建模对象上的“身份验证类型”字段必须设为“基本”才能使用用户名字段和密码字段。有关信息，请参考“在访问外部存储库中的文件时”。

在为员工指派过滤器标记时

可以使用“过滤器标记指派”部分来控制员工可以使用过滤器标记功能的方式，以及允许员工的过滤器标记选择在不同的会话之间持续存在。员工可以选择使用“设置过滤器标记”弹出窗口来进一步优化过滤器标记选择。通过单击主导航栏中的“设置”，然后单击浮出菜单中的“设置过滤器标记”，可访问“设置过滤器标记”弹出窗口。

Important: 身为管理员，必须先将自己的过滤器标记指派设为“所有实例”，然后再为其他员工指派过滤器标记。这可确保所有已标记的对象实例不会受到非法访问。

“过滤器标记指派”部分中有三个单选按钮，用于控制员工如何使用过滤器标记功能。

• 所有实例

选择“所有实例”时，无论过滤器标记处于哪种状态，员工都可以查看所有实例。保存页面后，将隐藏“指派的过滤器标记”表格，并移除以前指派给该员工的任何过滤器标记。将禁用“设置过滤器标记”弹出窗口。

• 已标记和未标记

选择“标记的和未标记的”时，员工可以查看具有指派给他们的过滤器标记的实例，以及未标记的实例。这是新员工对象实例的默认选择。

• 仅已标记

选择“仅标记的”时，员工仅可以查看与指派给他们的过滤器标记关联的实例。

使用“指派的过滤器标记”表格指派过滤器标记时，在默认情况下，可以从全局标题访问的“设置过滤器标记”弹出窗口中选择这些过滤器标记。员工可以使用此弹出窗口取消选择任何该会话不需要的过滤器标记。应用程序随后将取消选择“指派的过滤器标记”表格中的相应过滤器标记。这可使员工的选择在不同的会话之间持续存在。

如果管理员创建或更新某个实例的过滤器标记，员工将无权访问该实例，除非将新的或更新的过滤器标记添加到该员工的“指派的过滤器标记”表格或者将该员工设为可以查看所有实例。此外，如果员工向未指派给他们的建模对象指派过滤器标记，则他们尝试的任何更改都可能无法反映所查看的更改；建议员工在尝试将过滤器标记指派给建模对象之前，首先确保管理员已将这些过滤器标记指派给该员工。

Note: 从表格中删除过滤器标记时，必须取消选择所有过滤器标记，然后仅选择要删除的标记。删除标记后，可以重新选择要指派给员工的过滤器标记。

允许员工签入至多个工作区域
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

可以使用“员工”页来允许员工签入至多个工作区域，方法是选中“详细信息”部分中的“允许签入至多个工作区域”复选框。员工必须逐个签入至工作区域。有关签入工作区域的信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

允许员工指派给多个团队
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

可以使用“员工”页来允许员工指派给多个团队，方法是选中“详细信息”部分中的“允许多个团队”复选框。

如何定义员工

按照以下步骤定义员工：

1. 打开员工页。随即“建模”页中将显示“员工”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在员工字段中输入员工的 Windows 登录名。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。

如何为员工指派一个或多个角色

按照以下步骤为员工指派一个或多个角色。

1. 执行“如何定义员工”中所述的过程。

或

选择现有员工实例。

2. 单击“角色”表格中的添加新行。 将显示新行。
3. 选择角色。
4. 选择该角色对其有效的组织。
5. 如果需要，选中传播复选框，以将角色传播到指定组织的子组织。
6. 重复步骤 2-5 为此员工添加其他角色。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何为员工指派上报收件人

按照以下步骤为员工指派上报收件人：

1. 执行“如何定义员工”中所述的过程。

或

选择现有员工实例。

2. 单击“上报收件人”表格中的添加新行。 将显示新行。
3. 从员工列表中选择要接收上报电子邮件通知的员工。
4. 重复步骤 2-3 添加其他上报收件人。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何为员工指派过滤器标记

Important: 身为管理员，必须先将自己的过滤器标记指派设为“所有实例”，然后再为其他员工指派过滤器标记。这可确保所有已标记的对象实例不会受到非法访问

按照以下步骤将过滤器标记指派给员工：

1. 执行“如何定义员工”中所述的过程。

或

选择现有员工实例。

2. 选择过滤器标记指派类型对应的单选按钮。
• 如果选择所有实例，将隐藏“指派的过滤器标记”表格。跳到步骤 4。
• 如果选择标记的和未标记的或仅标记的，则继续执行步骤 3。
3. 在“指派的过滤器标记”表格中选择要指派给员工的过滤器标记。
4. 单击保存。 应用程序保存建模对象并显示成功消息。
