定义用户查询组

用户查询组是一组用户查询。用户查询组可用于根据业务需求缩小可用的用户查询数量。例如，可以定义一个用户查询组，该组仅包含在 Portal Shop Floor 应用程序中执行容器搜索时使用的用户查询。用户查询组可以包括单个用户查询和其他预定义的用户查询组。

在定义用户查询组时

可以将多个条目或组添加到各自的表格。

在定义用户查询组之前，必须定义一个或多个用户查询。

使用“解决条目”按钮可以显示在此组和所有嵌套组的条目列表中指定的所有值的列表。

如何定义用户查询组

按照以下步骤定义用户查询组：

1. 打开用户查询组页。随即“建模”页中将显示“用户查询组”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在用户查询组字段中输入此组的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将条目添加到用户查询组

按照以下步骤将条目添加到用户查询组：

1. 执行“如何定义用户查询组”过程。

或

选择现有的用户查询组实例。

2. 按如下所示向组中添加条目：
a. 单击“条目”表格底部的添加新行。 将显示新行。
b. 在新行中单击以显示条目列表，然后选择一个添加到组中。所选条目即添加到表格。
c. 重复步骤 a–b，将另一条目添加到表格。
3. 按如下所示向组中添加子组：
a. 单击“组”表格底部的添加新行。 将显示新行。
b. 在新行中单击以显示先前定义的组的列表，然后选择一个（作为子组）添加到组中。应用程序将选定的组添加到表格中。
c. 重复步骤 a–b，将另一组添加到表格。
4. 单击保存。 应用程序显示成功消息，指出建模对象已上传。
