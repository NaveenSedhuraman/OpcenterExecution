用户数据收集定义和数据点

用户数据收集定义可用于建立数据点，并将这些数据点与电子流程的任务列表相关联。数据点表示生产过程中的一个点，此时可以或必须收集数据。定义了数据点后，可以将多个数据点归入用户数据收集定义中，然后将该用户数据收集定义指派给电子流程。

定义用户数据收集

定义用户数据收集定义时，可以在行和列布局或表格布局中加入值数据点和对象数据点。应用程序会在运行时动态渲染指定布局中的数据点。有关信息，请参考“定义用户数据收集定义”。

在 添加值数据点

将值数据点添加到用户数据收集时，可以执行以下操作：

• 通过输入静态值或表达式来定义上限和下限
• 指示是否允许限制替代
• 选择将数据点值映射到容器属性

可以使用表达式来定义上限、下限或全部两个限制。使用表达式时，在运行时确定限制。

使用 表达式定义上限和下限时，要使用应用程序的统一表达式语法。表达式最多可包含 255 个字符，可能包括：

• 字段表达式;例如，Container.CurrentStatus.Spec.UserDefinedInteger
• 字段表达式和基本算术的组合; 例如，(100-Container.CurrentStatus.Spec.UserDefinedInteger) *0.2
Note: 输入表达式时，不会验证语法是否正确。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。

在定义值数据点的限制时

限制仅适用于值数据点，但可用于行和列布局以及迭代表格布局。“用户数据收集定义”页上的“显示限制”复选框决定应用程序是否显示在数据收集期间为值数据点定义的限制。处于选中状态时（默认设置），应用程序会显示这些数据点的限制。

定义限制后，“允许替代限制”复选框会确定当输入的值超出限制时向用户显示的颜色，以及某项电子流程数据收集任务是通过还是失败。无论电子流程任务通过还是失败，应用程序始终接受用户输入的任何值，即使该值超出限制。

在添加对象数据点时

对象数据点实质上与值数据点具有相同的用途 - 其提供在生产流程中的特定点收集数据的方法。对象数据点引用特定对象。例如，可使用对象数据点建立原因代码，从而记录预定义检验状态结果。这些结果包含诸如污点和表面划痕等情况。在车间内，操作者只需从选择值列表中选择合适的结果就可以完成检验任务。

Note: 限制不适用于对象数据点。

如何定义用户数据收集

按照以下步骤定义用户数据收集：

1. 打开用户数据收集定义页。随即“建模”页中将显示“用户数据收集定义”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在用户数据收集定义字段中输入此定义的唯一名称。
4. 在修订版本字段中输入此定义的修订版本标识。
5. 如果它是记录的修订版本，则选中记录修订版本复选框。
6. 从列表中选择数据点布局的类型：迭代表格或行列位置。
7. 使用下面适当的流程将数据点添加到数据点表格：
• “如何添加值数据点”
• “如何添加对象数据点”
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何添加值数据点

按照以下步骤将值数据点添加到用户数据收集：

1. 执行“如何定义用户数据收集”流程。

或

选择现有用户数据收集定义实例。

2. 单击“数据点”表格中的添加新行。将显示“数据点类型”菜单。
3. 选择值数据点。将显示“值数据点详细信息”弹出窗口。
4. 在数据点名称字段中输入名称。
5. 输入数字值，以指示应用程序在数据收集中显示数据点的行位置。
6. 输入数字值，以指示必须将值数据点放在数据收集中的列位置。
7. 从列表中选择数据类型。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击确定。 应用程序关闭弹出窗口，并在“数据点”表格中显示数据点。
10. 重复步骤 2-9，添加其他值数据点。

如何添加对象数据点

按照以下步骤将对象数据点添加到用户数据收集：

1. 执行“如何定义用户数据收集”流程。

或

选择现有用户数据收集定义实例。

2. 单击“数据点”表格中的添加新行。将显示“数据点类型”菜单。
3. 选择对象数据点。将显示“对象数据点详细信息”弹出窗口。
4. 在数据点名称字段中输入名称。
5. 输入数字值，以指示应用程序在数据收集中显示数据点的行位置。
6. 输入数字值，以指示必须将值数据点放在数据收集中的列位置。
7. 从列表中选择对象类型。
8. 从列表中选择选择值类型。根据您的选择，出现增加的字段。
9. 参照此表填写其他字段。
