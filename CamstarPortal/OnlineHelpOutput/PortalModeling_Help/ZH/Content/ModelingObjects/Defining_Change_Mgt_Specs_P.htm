定义更改管理规范

更改管理规范（规范）定义了在更改管理工作流程的工步中执行的活动。当定义更改管理工作流程时，应用程序要求至少参考一个更改管理规范。一个规范可以供不同的工作流程多次参考。更改管理规范是规范建模对象的子类。

默认更改管理规范

默认情况下，Siemens 提供多个更改管理规范。这些规范供 Siemens 提供的更改管理工作流程参考。可以在自己的自定义更改管理工作流程中使用提供的更改管理规范。此外，还可以自定义提供的规范以满足业务流程需求。

在定义更改管理规范时

可以执行以下操作：

• 指定当将包移动到更改管理工步时使用的包状态。
• 指定要在包信息标头中显示的自定义图像。此图像指示更改管理包所在的规范（工步）。自定义图像必须：
• 位于 C:\Program Files (x86)\Camstar\Camstar Portal\ 文件夹中
• 以 *. png 格式保存
• 大小调整为 20 x 20 像素
• 为规范中允许的事务指派角色。使用角色建模对象来定义角色。有关信息，请参考“定义角色”。有关更改管理默认角色、权限和允许的事务的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。
• 指定在包重新输入规范时是否必须重置更改管理包的审批流程。重置审批流程将取消待处理或完成的审批，并要求审批者在重新路由以审批时重新审批更改管理包。应用程序将所有捕获审批移动到历史。
Note: 可以在“审批包”页的“审批状态”选项卡上查看已取消审批的历史。
• 指定当包处于工作流程中的工步时，“更新包”页上是否显示“审批者”选项卡。
• 指定在此规范中更改管理包是否需要审批。应用程序使用此复选框进行更改控制实施，以确定包是否在审批工作流程中。有关更改控制实施的信息，请参考《Opcenter Execution Medical Device and Diagnostics Change Management 用户指南》或《Opcenter Execution Core Change Management 用户指南》。
Note: 使用主数据目录建模对象可指定建模对象的更改控制实施。有关设置更改控制实施的信息，请参考“定义主数据目录”。
• 指定当包输入此规范时，是否必须锁定更改管理包中的建模实例。

在锁定更改包建模对象实例时

如果选中“锁定此规范的更改包中的建模对象”复选框，则会在更改管理包移动到此规范时锁定包中的所有建模实例。

Note: 不能修改或删除锁定的更改包建模对象实例。当尝试修改或删除锁定的更改包建模对象实例时，应用程序将显示警告消息。

应用程序在以下情况下解锁实例：

• 更改包移动到未选中此复选框的规范中，并且
• 引用这些实例的所有包都位于未选中此复选框的规范中。

如何定义更改管理规范

按照以下步骤定义更改管理规范：

1. 打开更改管理规范页。随即“建模”页中将显示“更改管理规范”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在更改管理规范字段中输入此规范的名称。
4. 在修订版本字段中输入此对象的修订版本。
5. 选择包状态。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将角色指派给允许的事务

按照以下步骤将角色指派给允许的事务：

1. 执行“如何定义更改管理规范”过程。

或

选择现有的更改管理规范实例。

2. 展开事务部分。
3. 单击添加新行。表格中便添加了一个新行。
4. 从列表中选择要指派给允许的事务的角色。
5. 重复步骤 3-4 以指派其他行。
6. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
