定义工厂

工厂是任何具备财务和报告职能的独立分部、部门或组。一个企业可以包含一个或多个工厂。

它通常代表一栋物理制造建筑，但也可以代表公司本身或公司的一个大型分部。

工厂可以包含对企业以及工厂内位置的参考。

在定义工厂时

“SMTP 传输”字段使用简单邮件传输协议 (SMTP) 发送电子邮件。这样当 Opcenter EX MDD 和 Opcenter EX CR 配置为以 Windows 服务来运行时便能够发送电子邮件。

创建事件时，Portal 用户可以将工厂指定为发现区域。

工厂包含可选建模定义“企业”、“制造日历”、“调度规则”、“培训要求组”、“打印队列”、“容器编号规则”、“SMTP 传输”和“更改管理设置”。

如果与 Teamcenter 集成，工厂包含可选建模定义“PLM 建模设置”。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics 安装指南》或《Opcenter Execution Core 安装指南》。

工厂是：

• 可以在定义员工、企业、制造订单、资源和装运目的地时指定的可选字段。
• 大多数 Opcenter EX MDD 或 Opcenter EX CR 事务中的可选字段。
Note: 如果工厂与容器定义中的字段相关联，则它可以与 WIP 消息相关。有关信息，请参考“定义 WIP 消息”以及《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

可以选择要从导出中排除的任何对象引用。如果要排除该对象，则必须在“员工”级别指派该对象。必须注销然后重新登录，在“员工”页上对工厂指派所做的更改才会生效。

有关导出流程的信息，请参考“导出建模数据”。有关更改管理包的信息，请参考《Opcenter Execution Medical Device and Diagnostics Change Management 用户指南》或《Opcenter Execution Core Change Management 用户指南》。

添加位置

位置定义工厂内的一个物理位置。它可以像定义整个工厂的单一位置那样属于较高级别，或者像保存区内每个料仓这样的单个位置那样具体。

每个容器都包含一个位置属性，此属性用来标识该容器的物理位置并且生成报告（例如“按位置的 WIP”）。位置不是在 Opcenter EX MDD 或 Opcenter EX CR 中单独定义的，它们在工厂内定义。

位置是大多数 Opcenter EX MDD 或 Opcenter EX CR 事务中的可选字段。可以在定义工厂、资源组和装运目的地时指定位置。

添加常规消息

可以添加常规消息，当员工登录时显示这些消息，以通知他们即将发生的事件，如计划维护、待处理审核或任何其他相关通知。

添加常规消息的选项显示在“工厂”页的“常规消息”部分中，并通过选中“显示消息”复选框来启用。提供了一个用于创建这些消息的富文本编辑器。如果选中“显示消息”复选框而不在文本框中输入消息，则应用程序将在保存对象时生成错误。有关使用该文本编辑器的信息，请参考“定义任务列表和相关任务”。

在用户设置生产线指派后，常规消息弹出窗口将显示在用户的默认主页中心；或者，如果将应用程序配置为绕过生产线指派页，则在登录后显示。当用户单击“关闭”按钮时，常规消息弹出将关闭。

为工厂事务中的文档附加或拆离添加电子签名要求

可以设置要求，规定用户必须提供电子签名才能将文档附加到在工厂内执行的事务或从这些事务中拆离文档。文档可以包括特殊指示、测试结果、图纸或照片。有关文档附件的信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

为文档附加或拆离设置电子签名要求的选项显示在“事务”部分的“电子签名事务映射”中。或者，可以选择要求所有事务均使用电子签名。

为工厂事务中的制造批次和采样批次添加电子签名要求

可以设置要求，规定用户必须为在工厂内执行的事务中的制造批次和采样批次提供电子签名。设置这些要求的选项显示在“事务”部分的“电子签名事务映射”中。或者，可以选择要求所有事务均使用电子签名。

可以在“创建采样批次”或“更新采样批次”车间页上提交采样批次的签名。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

在工厂页上定义团队追踪事务
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

可以使用“工厂”页定义哪些事务需要跟踪员工在给定作业、规范、工作中心、工作单元或工作站执行的工作。通过在工厂级别定义事务，可以为所有规范设置跟踪要求，而无需修改工作流程中的每个规范。但是，在规范级别设置的任何事务均优先于在工厂级别进行的选择。

为工厂指派默认库存位置
Note: 下述功能仅在安装了 Industry Solutions 工作区时可用。有关信息，请参考“Industry Solutions 工作区”。

可以将默认库存位置指派给工厂。还可以将默认库存位置指派给产品和产品系列。

在指派默认库存位置时，以下规则适用：

• 在产品上指派的默认库存位置优先于在产品系列上指派的默认库存位置。
• 在产品系列上指派的默认库存位置优先于在工厂上指派的默认库存位置。
• 如果在工厂定义了默认库存位置，将某个容器移动到定义为库存点的作业时，应用程序会自动将默认库存位置指派给该容器。

有关信息，请参考“为产品指派默认库存位置”和“为产品系列指派默认库存位置”。

从物料队列中启用物料消耗
Note: 下述功能仅在安装了 Industry Solutions 工作区时可用。有关信息，请参考“Industry Solutions 工作区”。

在“物料队列”部分的“消耗物料事务”字段中选择事务后，会自动将物料消耗加载到物料队列。可用事务包括：

• 开始
• “移入”(MoveIn)
• 移动

通过在“验证物料队列事务”字段中选择一个或多个事务，可以验证是否将正确的物料加载到特定容器的物料队列中。如果选择了事务但未选中“验证物料队列数量”复选框，则应用程序将返回一条警告消息。如果选择了事务并选中“验证物料队列数量”复选框，但数量不足，则事务将失败。可以选择“发出差别原因”，以防在自动分发组件期间发出部分数量时出现失败情况。

在工厂页上启用 Opcenter Execution Core Scheduling
Note: 仅当安装有 Opcenter Execution Core Scheduling 时，下述功能才可用。有关信息，请参考 "Opcenter Execution Core Scheduling"。

必须使用“工厂”页上的“启用 Preactor 接口”复选框来启用 Opcenter Execution Core Scheduling，才能激活从 Opcenter EX MDD 或 Opcenter EX CR 到 Siemens Preactor Advanced Planning and Scheduling (Preactor) 应用程序的信息传递。

在工厂上指派 OEE 设置
Note: 下述功能仅在安装了 Industry Solutions 工作区时可用。有关信息，请参考“Industry Solutions 工作区”。

如果要设置应用程序对所选工厂的设备综合效率 (OEE) 进行计算的时间跨度，则在 OEE 部分中选中“OEE 设置”复选框。还可以在资源和资源系列上指派 OEE 设置。

以下规则在指派 OEE 设置时适用：

• 在“资源”建模对象上指派的 OEE 设置优先于在“资源系列”建模对象上指派的 OEE 设置。
• 在“资源系列”建模对象上指派的 OEE 设置优先于在“工厂”建模对象上指派的 OEE 设置。

有关将 OEE 设置指派给资源的信息，请参考“在资源上指派 OEE 设置”。有关将 OEE 设置指派给资源系列的信息，请参考“在资源系列上指派 OEE 设置”。

如何定义工厂

按照以下步骤定义工厂：

1. 打开工厂页。“工厂”页显示在“建模”页中。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在工厂字段中输入此工厂的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。
6. 请参考“如何添加位置”过程向此工厂中添加位置。
7. 有关设置电子签名要求的信息，请参考“如何为工厂事务中的文档附加或拆离添加电子签名要求”。
8. 有关设置电子签名要求的信息，请参考“如何为工厂事务中的制造批次和采样批次添加电子签名要求”。

如何添加位置

按照以下步骤向工厂中添加位置：

1. 执行“如何定义工厂”过程。

或

选择现有工厂实例。

2. 单击“位置”表格中的添加新行。 将显示新行。

如何添加常规消息

按照以下步骤添加当用户登录 Portal 时显示的常规消息：

1. 执行“如何定义工厂”过程。

或

选择现有工厂实例。

2. 如果需要，展开常规消息部分。
3. 选中显示消息复选框以启用常规消息。
4. 在富文本编辑器中输入消息。
5. 将任何所需的样式或格式设置应用于消息。
6. 单击保存。 应用程序保存建模对象并显示成功消息。

如何为工厂事务中的文档附加或拆离添加电子签名要求

按照以下步骤为工厂事务中的文档附加或拆离添加电子签名要求：

1. 执行“如何定义工厂”过程。

或

选择现有工厂实例。

2. 如果需要，展开事务部分。
3. 单击“电子签名事务映射”表格中的添加新行。 将显示新行。
4. 如果要设置文档附加要求，选择“事务”字段中的附加文档。

或

如果要设置文档拆离要求，选择拆离文档。

5. 如果要将电子签名要求应用于所有事务，选中全部事务字段中的复选框。
6. 从电子签名要求字段中选择电子签名要求。有关信息，请参考“定义电子签名要求”。
7. 单击保存。 应用程序保存建模对象并显示成功消息。
