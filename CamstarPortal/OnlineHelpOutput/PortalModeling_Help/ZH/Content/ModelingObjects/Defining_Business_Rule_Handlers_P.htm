定义业务规则处理程序

“业务规则处理程序”确定了业务规则执行的操作。“业务规则处理程序”会被指派给业务规则。有关业务规则的信息，请参考“定义业务规则”。

利用“业务规则处理程序”建模对象可执行以下任一操作：

• 定义在业务规则中实施业务逻辑的脚本
• 标识应用程序要调用的现有服务

利用脚本处理程序可以指定条件，并且在该条件为真时应用程序将执行的操作。利用服务处理程序可以标识应用程序要调用的现有服务。

Opcenter EX MDD 或 Opcenter EX CR 提供以下默认业务规则处理程序：

• BRH_ACTIVATION - 管理与目标系统上的更改包相关联的建模实例的激活状态。
• BRH_DEPLOY - 在用户请求部署包时管理更改包的部署状态。
• BRH_DEPLOYSTATUS - 管理从导出完成直到包已交付时段内，源系统上更改包的状态。
• BRH_NOTIFICATIONS - 脚本将更改管理包审批和协同工作的到期日期，与“更改管理设置”建模对象中“包审批提醒”表格和“内容协作提醒”表格中的信息进行比较。它将确定是否需要发送电子邮件提醒。有关信息，请参考“定义更改管理设置”。

有关从一个系统导出建模对象，然后将这些对象导入另一系统的信息，请参考《Opcenter Execution Medical Device and Diagnostics Change Management 用户指南》或《Opcenter Execution Core Change Management 用户指南》。

在定义业务规则脚本处理程序时

脚本是一组语句，可以选择包含一个或多个具有默认值的参数，这些参数可以通过业务规则使用表达式设置。应用程序在运行时使用为参数指定的默认值替代脚本参数。

脚本使用“统一表达式”语法配置。使用此语法可以创建简单或复杂的表达式，可以根据业务规则的指派位置计算这些表达式。

Important: 要在业务规则处理程序中添加、移除或更新参数，还需要打开并保存每个引用了该业务规则处理程序的业务规则。否则，在执行关联业务规则时可能出错。有关信息，请参考“在查看现有业务规则时”。
Note: 有关统一表达式语法的信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。

创建业务规则时，只有语法已通过验证的脚本处理程序才可供选择。

在定义业务规则服务处理程序时

定义服务处理程序时，使用应用程序可以添加其在运行时将执行的其他参数。注意，在此处添加参数不会修改实际的服务。需要使用 Designer 来修改服务本身。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。

“业务规则处理程序”是“业务规则”中的可选“建模”定义。

如何定义业务规则脚本处理程序

按照以下步骤定义“业务规则脚本处理程序”：

1. 打开业务规则处理程序页。随即“建模”页中将显示“业务规则处理程序”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在业务规则处理程序字段中输入此业务规则处理程序的名称。
4. 从处理程序类型字段中选择脚本。应用程序将更新“业务规则处理程序”页以显示必要的脚本字段。
5. 在脚本字段中输入要执行的脚本。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何定义业务规则服务处理程序

按照以下步骤定义“业务规则服务处理程序”：

1. 打开业务规则处理程序页。随即“建模”页中将显示“业务规则处理程序”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在业务规则处理程序字段中输入此业务规则处理程序的名称。
4. 从处理程序类型字段中选择服务。应用程序将更新“业务规则处理程序”页以显示必要的服务字段。
5. 从服务字段中选择要执行的服务。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将参数添加到业务规则处理程序

按照以下步骤向业务规则脚本处理程序添加参数：

1. 执行“如何定义业务规则脚本处理程序”流程。

或

执行“如何定义业务规则服务处理程序”流程。

或

选择现有的业务规则处理程序实例。

2. 单击“参数”表格上的添加新行。 将显示新行。
3. 在名称字段中输入上方脚本中参数的名称。
4. 从列表中选择数据类型。
5. 如果从数据类型字段中选择了对象，则选择对象类型。
6. 在默认值字段中输入参数的值。
7. 重复步骤 2-6，添加其他参数。
8. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
