定义服务器目录系统

服务器目录系统将标识可从中发送或接收数据包的 Opcenter EX MDD 或 Opcenter EX CR 中的服务器。可以为贵组织中的本地服务器和所有可访问（目标）服务器定义服务器目录系统。

在定义服务器目录系统时

“服务器目录系统”页可用于定义两种类型的服务器位置：

• 本地系统 - 必需的 Opcenter EX MDD 或 Opcenter EX CR 目标服务器
• 非 Camstar 服务器 - 用于当前未运行 Opcenter EX MDD 或 Opcenter EX CR 的处理的其他服务器

必须确保源服务器和目标服务器包含有效的 SSL 证书。定义“IP 地址”字段时，必须使用证书上列出的服务器的完全限定域名。

本地系统

只能将一个服务器定义为本地服务器。本地服务器是要发送到目标服务器的任何数据包所源自的服务器。在“服务器目录系统”页上选中“是本地系统”复选框可将该服务器设为本地服务器。必须通过选中“是本地系统”来定义至少一个本地服务器。

如果在已将另一个实例定义为本地服务器时，尝试将某个实例另存为本地服务器，则应用程序将显示一条错误消息。应用程序要求先取消选中现有本地服务器实例上的“是本地系统”复选框，然后才能将其他服务器指定为本地服务器。

应用程序将使用“服务器详细信息”部分中的字段在发送或接收数据包时比较本地和目标服务器的属性。当属性不匹配时，应用程序会显示一条警告消息，但不会妨碍完成事务。例如，如果为本地服务器输入的版本为 6，而为目标服务器输入的版本为 5.8，则应用程序将显示一条警告消息。

当主机服务器和目标服务器位于不同的域上时，指定 Active Directory 域可使应用程序能够从主机服务器映射到目标服务器上的 Temp 文件夹。如果没有为目标系统指定域，应用程序将尝试使用主机服务器的域从主机服务器映射到目标服务器。

不是 Camstar 服务器

可以在未运行 Opcenter EX MDD 或 Opcenter EX CR 的服务器上定义其他处理文件夹位置。当前未运行某产品版本的服务器不可用于在“激活影响”页上进行查看。

如何定义服务器目录系统

按照以下步骤定义服务器目录系统：

1. 打开服务器目录系统页。随即“建模”页中将显示“服务器目录系统”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在服务器目录系统字段中输入服务器的名称。
4. 在 MDB 版本字段中输入安装在服务器上的 MDB 的版本。
5. 在 Camstar 版本字段中输入服务器上安装的 Opcenter EX MDD 或 Opcenter EX CR 的版本。
6. 在用户名字段中输入有权访问此服务器的用户名。
7. 在密码字段中输入用户名的密码。
8. 在 IP 地址字段中输入与 SSL 证书匹配的服务器的完全限定域名。
9. 在端口字段中输入端口。
10. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
11. 单击保存。 应用程序保存建模对象并显示成功消息。
