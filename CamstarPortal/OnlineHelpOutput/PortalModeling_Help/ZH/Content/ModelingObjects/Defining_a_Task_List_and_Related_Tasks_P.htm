定义任务列表和相关任务

任务列表可定义一系列要执行的规范任务。可以指定是否必须按照顺序执行任务。任务列表的一个示例为一组与电子流程用户数据收集相关的产品计量任务。

了解任务

任务指的是要求在容器的一个规范中完成的工作单位。可将任务顺序指派给电子流程，再将电子流程指派给规范。电子流程执行任务列表的顺序，而任务列表执行任务的顺序。必须先保存任务，然后才能将其设为下一个任务的必要条件任务。任务是合格还是不合格，取决于收集的数据或车间操作员的决策。

任务项类型

可以在添加任务项时选择任务项类型。

任务项类型

描述

指示

任何常规的任务类型。

计算

计算由应用程序在执行任务时根据用户常量、参数和运算符输入执行。

事务

此项类型定义完成事务的任务；例如，任务 1 是“移入”任务，任务 2 是“移动”任务。

称重分发

此任务用于定义过程中一个组件将被分发的点，并提供称重所需的任何规则。

注：仅当安装 Batch Processing 时，才会显示此任务项类型。有关信息，请参考 "Batch Processing"。
Note: 用户常量和计算也在建模中定义。有关信息，请参考“定义用户常量”和“定义计算”。

任务类型

任务项的任务类型对于执行生产线清理的工作站非常重要。任务项可以是以下任一任务类型。

任务类型

描述

开始流程

表示工作站中流程的开始。“开始流程”任务在容器级别或制造订单级别将工作站状态设为“处理中”。“开始流程”任务对于执行生产线清理的工作站是必需的，并且必须是任务列表中的第一个任务。事务任务不能是“开始流程”任务。

注：“开始流程”任务的最小迭代值必须大于 0。

正常

表示任务可发生在执行生产线清理的工作站的“开始流程”任务和“结束流程”任务之间。任务列表可以包含不执行生产线清理的工作站的所有正常任务。正常任务必须运行与“开始流程”任务相同的任务列表、电子流程和规范工步内的相同容器或制造订单。否则，它将失败。

结束流程

表示工作站中流程的结束。“结束流程”任务可重置工作站状态。“结束流程”任务指示工作站可用于处理下一个容器或制造订单。“结束流程”任务对于执行生产线清理的工作站是必需的，并且必须是任务列表中的最后一个任务。

注：“结束流程”任务的最小迭代值必须大于 0。

指示类型

描述

确认

此指示类型在任务完成时要求确认。

数据收集

数据收集类型任务，通常收集制造车间的参数和用户数据。根据结果，决定后续任务。

通过/失败

此指示类型要求通过或失败通知。

了解执行模式

按照创建任务列表时必须完成的顺序输入各个任务。如果不要求按照特定顺序执行任务，则可使用任意顺序输入任务。任务列表定义了任务的执行模式。

通过在“任务列表”页中的“执行模式”字段选择“顺序”或“非顺序”，便可指定任务列表的执行模式：

• 顺序（默认）- 每一个任务列表按照电子流程建立的顺序显示。只有当任务列表的所有任务可选时才可跳过该任务列表。
• 非顺序 - 应用程序不要求按照电子流程定义的顺序执行任务列表。同样，电子流程定义任务列表的执行模式。

定义任务列表和相关任务

在定义任务列表之前，可能需要定义以下实体（视要求而定）：

• 计算
• 文档集
• 电子签名要求
• 流程计时器
• 培训要求组
• 用户常量
• 用户数据收集定义

任务列表是电子流程中必需的建模定义。

任务列表使用以下可选建模定义：资源、物料箱、资源组、物料箱组、电子签名要求、培训要求组和用户数据收集定义。

不可在对象名中使用无效字符。例如，不可使用撇号。

Note: 有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。

文本编辑器

Siemens 提供了一个文本编辑器，可用于为任务列表或相关任务中的指示设置格式。该文本编辑器支持熟悉的文本格式设置功能，例如，字体大小和颜色、字体样式、文本对齐、项目符号列表、高亮显示、插入图像等。

Note: Siemens 当前不建议在任务或任务列表指示中创建编号列表，因为它们在“指示”字段内渲染不正确。

Siemens 建议高亮显示现有指示，然后将格式设置应用于所选文本。如果未选择任何格式设置属性，应用程序将应用默认字体大小、样式和颜色。应用程序会向车间用户显示应用的格式设置。

可以在“设计”或 HTML 模式下使用该文本编辑器。在“设计”模式下复制文本时，不会复制隐藏的标记。因此，粘贴复制的文本时，可能不会保留格式设置。但是，如果切换到 HTML 视图，则可以复制隐藏的标记。

“指示”字段的字符数上限为 4,000，包括嵌入的 HTML 代码。从其他应用程序（例如 Microsoft Word）复制文本并将其粘贴到此字段中时，除了实际文本外，还会将隐藏的 HTML 代码粘贴到字段中。因此，即使文本包含的字符少于 4,000 个，也可能超出限制。

Note: Google Chrome 不支持文本编辑器的“复制”和“粘贴”按钮。使用 Google Chrome 时，可以使用键盘快捷方式进行复制和粘贴（Ctrl+C 和 Ctrl+V）。

设计模式

设计模式是一个 WYSIWYG（所见即所得）文本编辑器，可在车间用户查看时显示指示。

HTML 模式

单击 HTML 按钮可显示应用于指示的源代码。由于 WYSIWYG 编辑器可能会自动插入标记代码（标记），因此如果需要，在 HTML 模式下查看指示可以验证和修改此代码。

使用文本编辑器插入图像

通过单击文本编辑器上的“插入/编辑图像”按钮，可以在指示中插入图像。然后，通过在“插入/编辑图像”弹出窗口上输入图像的路径上传图像。

必须按以下形式输入图像的路径：

• 相对 URL，例如 "Themes/MyImages/Image.png"

或

• 绝对 URL，例如 "https://localhost/Themes/MyImages/Image.png"
Note: 大型图像可能不适合“电子流程”页上指示部分的可见区域。Siemens 建议您编辑图像的尺寸，以使其适合首选设备上的可见区域。

在指派流程计时器时

流程计时器可由任何任务项类型弹出窗口（“指示”、“计算”、“事务”）上的任务启动或结束，但只能使用“标准移动”和“移入”事务。

Note: 如果尝试保存的任务列表包含计时器或事务并非“移动（标准）”或“移入”的事务任务，则应用程序将显示一条错误消息。

计时器可以：

• 由任务启动并由任务结束
• 由任务启动并由事务结束（基于规范建模）
• 由事务启动（基于规范建模）并由任务结束
• 由事务启动并由事务结束（均基于规范建模）
Note: 流程计时器仅取决于“移动（标准）”或“移入”事务，而不考虑可对任务执行的迭代次数。执行任务后，计时器就会停止。

单个任务的开始计时器和结束计时器之间没有关联。例如，一个表格可能定义了多个计时器，而另一个表格可能没有定义任何计时器。

有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

在为指示任务定义数据收集任务时

定义数据收集任务时，必须从“指示类型”字段中选择“数据收集”，并从“数据收集定义”字段中选择预定义的数据收集定义。如果将数据收集定义与该任务关联，但从“指示类型”字段中选择除“数据收集”之外的其他值，则应用程序不会在“电子流程”页上显示数据收集字段。

在定义计算任务时

在定义计算任务时，应用程序要求关联数据点和计算参数的数据类型相同。如果数据类型不同，则应用程序将显示一条错误消息。数据点的数据类型在指定的数据收集定义中设置，而计算参数的数据类型在指定的计算定义中设置。

必须选择数据点的修订版本。展开列表中的数据点可选择特定的修订版本。

在定义事务任务时

事务任务可以是正常任务或结束流程任务。“开始流程任务”选项在事务任务的“任务类型”字段中不可用。此外，“电子签名要求”字段对于事务任务不可用。

在定义中间批次时
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

您可以使用“批处理”部分中的字段定义中间批次。您可以输入一个表达式来计算中间批次的数量。您可以为中间批次设置度量单位。

添加到中间批次的所有组件必须具有相同的度量单位。如果组件的度量单位不相同，应用程序将转换组件的度量单位，使其与批次的度量单位相匹配。转换在发出组件之前进行。有关信息，请参考“在定义度量单位的转换信息时”。

在定义称重分发任务时
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

可以定义一个称重分发任务，用于定义过程中一个组件将被分发的点，并提供称重所需的任何规则。

可以将表达式定义为任务的一部分，以计算所需数量和相关的最小和最大允许数量。这些计算值优先于产品或物料列表上的值。有关信息，请参考“定义表达式以调整产品上的组件数量”、“定义制造订单上允许的组件数量”、“定义物料清单上允许的组件数量”和“定义 ERP 物料清单上允许的组件数量”。

可以使用“测重器组”字段在称重分发任务上指派一个测重器组。还可以在规范、产品系列和产品页上指派测重器组。

在指派测重器组时，以下规则适用：

• 在产品系列上指派的测重器组优先于在规范上指派的测重器组。
• 在产品上指派的测重器组优先于在产品系列上指派的测重器组。
• 在称重分发任务上指派的测重器组优先于在所有其他对象上指派的测重器组。

请参考“在规范上指派测重器组”、“在产品系列上指派测重器组”和“为组件产品指派测重器组”。

您还可以定义以下用于称重组件的规则：

• 允许手动输入重量
• 手动称重时要求电子签名
• 当替代重量公差违规时要求电子签名
• 仅允许手动输入重量
• 按物料发出量增加批处理容器数量

称重发布任务可以是正常任务或结束流程任务。“开始流程任务”选项在称重发布任务的“任务类型”字段中不可用。

如何定义任务列表

按照以下步骤定义任务列表：

1. 打开任务列表页。随即“建模”页中将显示“任务列表”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在任务列表字段中输入此任务列表的名称。
4. 在修订版本字段中输入此任务列表的修订版本。
5. 从列表中选择执行模式。
6. 使用下面的适当过程至少向“任务”表格添加一个任务：
• “如何将指示任务添加到任务列表”
• “如何将计算任务添加到任务列表”
• “如何将事务任务添加到任务列表”
7. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
8. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将指示任务添加到任务列表

按照以下步骤将指示任务添加到任务列表：

1. 执行“如何定义任务列表”过程。

或

选择一个现有“任务列表”实例。

2. 展开“任务详细信息”部分。
3. 单击“任务”表格上的添加新行。将显示“选择任务项类型”菜单。
4. 选择指示。将显示“指示任务详细信息”弹出窗口。
5. 在任务字段中输入任务名称。
6. 从列表中选择任务类型。
7. 从列表中选择指示类型。
8. 在最小迭代字段中输入一个数字，表示可以执行任务的最小次数。
9. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
10. 单击确定。 弹出窗口关闭，任务将显示在“任务”表格中。
11. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将计算任务添加到任务列表

按照以下步骤将计算任务添加到任务列表：
