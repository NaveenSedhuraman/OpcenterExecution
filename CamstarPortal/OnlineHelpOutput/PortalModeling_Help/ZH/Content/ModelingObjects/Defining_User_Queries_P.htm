定义用户查询

用户查询是作为实例数据存储的预定义查询。请注意，当查询定义被定义为实例数据时，查询本身可以用于从实例数据表或元数据表中检索信息。

“用户查询参数”网格作为持久列表被实施，以便参数顺序由相应的持久列表自动维护。

用户查询要求

在 Opcenter EX MDD 和 Opcenter EX CR 中编写查询时，需要熟悉应用程序的对象模型、了解 SQL 语法，并具有编写 SQL 查询的经验。您还必须理解最终用户的需求，以便查询将在用户界面上显示正确类型的信息。

当查询被创建和测试后，必须由自定义应用程序来调用。可以通过 Camstar 服务 API 调用 (WSQuery) 或通过自定义执行来完成此操作。

在定义用户查询时

确保您有合理数量的容器信息以便有效地测试用户查询。这意味着信息模型必须配置为具有各种属性的产品、用户代码、工作流程和容器。

如果要构建要在容器搜索中使用的查询，则查询必须在选择准则中使用容器 ID。否则，当从“用户查询”选项卡执行时，查询将不会返回任何结果。

如何定义用户查询

按照以下步骤定义用户查询：

1. 打开用户查询页。随即“建模”页中将显示“用户查询”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在用户查询字段中输入查询的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 有关测试需要参数的查询的信息，请参考“如何测试需要参数的查询”。

或

有关测试不需要参数的查询的信息，请参考“如何测试不需要参数的查询”。

6. 单击“用户查询”页上的保存。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何测试需要参数的查询

按照以下步骤测试需要参数的查询：

1. 执行“如何定义用户查询”过程。

或

选择现有的用户查询实例。

2. 在需要参数的查询文本字段中输入 SQL 语句。
3. 在“用户查询参数”表格中输入用户查询参数名称和数据类型。
4. 单击保存并测试查询。将显示“用户查询文本”弹出窗口，同时在弹出窗口顶部显示必需的参数。
5. 输入查询所需的任何参数。
6. 单击测试查询。与输入的查询和参数匹配的任何结果都会显示在“查询结果”表格中。
7. 单击关闭。“用户查询测试”弹出窗口关闭。

如何测试不需要参数的查询

按照以下步骤测试不需要参数的查询：

1. 执行“如何定义用户查询”过程。

或

选择现有的用户查询实例。

2. 在不需要参数的查询文本字段中输入 SQL 语句。
3. 单击保存并测试查询。将出现“用户查询文本”弹出窗口，结果显示在“查询结果”表格中。
4. 单击关闭。“用户查询测试”弹出窗口关闭。

如何将参数添加到用户查询

按照以下步骤将参数添加到用户查询：

1. 执行“如何定义用户查询”过程。

或

选择现有的用户查询实例。

2. 在“用户查询参数”表格中单击添加新行。 将显示新行。
3. 在用户查询参数名称字段中输入参数的名称。该名称将显示为数据输入字段的标题，用户在该字段中键入所需的输入。
4. 从数据类型字段中选择参数的数据类型。
5. 重复步骤 2-4，根据需要添加其他参数。
6. 单击“用户查询”页上的保存。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
