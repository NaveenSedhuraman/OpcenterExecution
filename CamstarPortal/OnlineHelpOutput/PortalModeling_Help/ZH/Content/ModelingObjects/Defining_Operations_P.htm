定义作业

作业是库存和生产活动被跟踪的制造点或处理点。工作流程工步的规范将引用作业。

作业定义描述了一些项，例如，原因代码、处理规则以及作业过程中物料移动的被允许事务。相反，规范定义作业中执行的处理。

一些作业信息的示例包括：

• 对记录损失、额外、返工以及装运信息的用户代码的引用
• 作业的可选标志，如库存、运输中或服务点外
• 关于允许的事务和不允许的事务的标志
• 有关是否使用了队列中状态的指示，如果使用了则需要“移入”

Portal 用户可以在创建事件时指定作业，以标识检测到事件的制造或处理点。

在定义作业时

作业包含：

• 必需的建模定义“容器级别”（作为“产出率报表层次”输入）。
• 可选的建模定义“调度规则”、“装运目的地”、“培训要求组”、“打印队列”、“工作中心”以及“用户代码组”（各种原因代码组）。

“作业”是：

• 定义“工序”中的必需字段。
• 定义“员工”中的可选字段。
• 事务“移入”、“标准移动”、“非标准移动”、“生产量”、“转移”、“返工”、“转移返工”、“搁置”、“释放”、“容器维护”、“更改数量”、“容器缺陷”、“组件缺陷”、“组件分发”以及“组件移除”中的可选字段。

如果未选中“使用队列”，则“排队容器”表格可能不会显示在“作业视图”页上。这由“行分配”中选择的“作业”以及选择的容器所决定。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

使用“部署详细信息”部分以选择与数量更改和缺陷原因相关联的用户代码组的名称。

仅当您的组织购买了 Nonconformance Management 模块时，“拒绝进入的不合格容器”复选框才适用。选中此复选框可使应用程序阻止用户将具有打开的不合格品的容器移动到作业中。

Note: 如果作业与容器定义中的字段关联，则作业可以具有相关的 WIP 消息。有关信息，请参考“定义 WIP 消息”以及《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

库存点

选中时表示此作业是库存持有点。若选中，此作业为库存点。库存作业可以是工作流程或单独作业中的一个工序。

可选

服务点外

选中时表示此作业是服务点外，例如转包商。若选中，此作业为服务点外。此信息用于分类和报告与作业相关的信息。

如何定义作业

按照以下步骤定义作业：

1. 打开作业页。随即“建模”页中将显示“作业”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在作业字段中输入作业名称。
4. 输入生产量报表层次。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 单击保存。 应用程序保存建模对象并显示成功消息。

如何指定此作业中的不允许事务

按照以下步骤指定此作业中的不允许事务：

1. 执行“如何定义作业”过程。

或

选择现有作业实例。

2. 展开详细信息部分。
3. 单击“不允许的事务”表格上的添加新行。 将显示新行。
4. 在事务类型字段中选择条目的事务。
5. 如有需要，重复步骤 3-4 来添加其他不允许的事务。
6. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
