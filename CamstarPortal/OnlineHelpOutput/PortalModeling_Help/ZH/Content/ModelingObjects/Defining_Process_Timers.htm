定义流程计时器

流程计时器是一个已修订数据对象，它通过定义在容器上发起处理时在满足最小时间之前或在经过待执行流程的最大允许时间之后要实施的操作，来控制和监控制造流程。

在下列情况下，可以触发容器的流程计时器：

• 在工作流程中的指定工步对容器执行移动或移入。

或

• 在电子流程中对容器执行一项任务。

有关使用流程计时器的信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

Note: 流程计时器只取决于“移动（标准）”和“移入”事务。应用程序不考虑可以在电子流程任务上执行的这些事务的数量。执行任务时流程计时器停止。

在定义流程计时器时

可以定义当计时器在最小时间之前结束、在最大时间后结束，或这两种情况下发生的操作。

可以指定允许或不允许执行结束事务和处置操作，也可以指定在获得电子签名的情况下不执行处置操作而允许结束事务。

如果指定不允许执行结束事务和处置操作，则计时器将在达到最小或最大时间时继续运行。对于小于最小时间的情况，在计时器达到最小时间时允许结束事务。对于大于最大时间的情况，如果已超过最大时间，则必须手动操作容器（执行任何事务），并使用“流程计时器搜索”页上的“停止计时器”按钮手动停止计时器。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

单击“保存”，应用程序将验证四个计时器字段中的数据，以确保：

• 最小时间大于最小警告时间。
• 最大时间大于最大警告时间。
• 最大时间大于最小时间。
• 如果验证失败，则会发生错误。

流程计时器包含可选的建模定义“流程计时器类型”。

如何定义流程计时器

按照以下步骤定义流程计时器：

1. 打开流程计时器页。随即“建模”页中将显示“流程计时器”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在流程计时器字段中输入此流程计时器的名称。
4. 在计时器下选择所需行为的单选按钮。默认选择“仅最小时间”。
5. 按照计时器格式 ([d.]hh:mm:ss) 输入所需时间。
Note: 根据在计时器下的选择这些字段将标记为必填或禁用。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。

如果选择的是 ...

操作步骤 ...

仅最小时间
a. 完成“处置详细信息”下所需的部分：在计时器结束 < 最小时间时。
b. 转到步骤 7。

仅最大时间
a. 完成“处置详细信息”下所需的部分：在计时器结束 > 最大时间时。
b. 转到步骤 7。

a. 完成两者下所需的部分。
b. 转到步骤 7。
7. 单击保存。 应用程序保存建模对象并显示成功消息。
