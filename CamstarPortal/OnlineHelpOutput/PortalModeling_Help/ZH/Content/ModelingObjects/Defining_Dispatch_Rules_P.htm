定义调度规则

调度规则定义显示给车间用户的调度列表。随后，车间用户在工厂中启动或移动容器时，将以这些列表作为参考基础。

调度规则执行以下任务：

• 显示必须执行的工作的列表
• 可以选择实施执行工作的顺序

调度详细信息表格

此表格列出受调度规则影响的事务以及每个事务的详细信息。

可选

调度类型

受此调度规则影响的容器事务的类型：

• 容器移入 - 移入事务的调度类型。
• 容器移出 - 移出事务的调度类型（标准移动和非标准移动）。
• 电子流程 - 在“电子流程”页上对容器进行排序的调度类型。
• 订单 - 基于制造订单的“启动”事务的调度类型。
• 批订单 - “启动批”事务的调度类型。

可选

实施调度

此复选框表示应用程序将在车间服务上实施规则。

启动服务示例：

如果选中此复选框，车间用户只能根据列表顶部的制造订单创建容器。

如果未选中此复选框，调度列表仍将根据优先级对订单进行排序，但可以按列表中的任何顺序启动容器。

可选

查询名称 (Query Name)

用于填充调度表格的查询列表。列表中的项基于查询类型的值。

如果“查询类型”为“Designer 查询”，则列表显示 Siemens-提供的查询名称。

如果“查询类型”为“用户查询”，则列表显示用户创建的查询名称。

可选

如何定义调度规则

按照以下步骤定义调度规则：

1. 打开调度规则页。随即“建模”页中将显示“调度规则”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在调度规则字段中输入名称。
4. 从查询类型列表中选择一个值，如下所示：
• 如果要使用 Designer 定义的默认查询来填充调度列表，则选择 Designer 查询。
• 如果要使用在建模中定义的查询来填充调度列表，则选择用户查询。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 单击保存。 应用程序保存建模对象并显示成功消息。

如何添加调度详细信息

按照以下步骤添加调度详细信息：

1. 执行“定义调度规则”过程。

或

选择现有调度规则实例。

2. 单击添加新行按钮。 将显示新行。
3. 从调度类型字段中选择下列项之一：
• ContainerMoveIn（如果要显示移入事务的容器调度列表）。
• ContainerMoveOut（如果要显示移出事务的容器的调度列表）。
• 电子流程（如果要在“电子流程”页上显示容器的调度列表）。
• 订单（如果要显示“启动”事务的制造订单的调度列表）。
• 批订单（如果要显示“启动批”事务的制造订单的调度列表）。
4. 如果要实施调度规则，则选中实施调度复选框。选中该复选框表示必须在调度列表的顶部项上执行车间服务。
5. 从查询名称列表选择查询。此查询用于填充调度列表。
6. 根据此调度规则的需要，重复步骤 2-5 添加其他调度类型。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
