定义更改管理设置

通过“更改管理设置”可配置用于更改管理的各种全局设置。这些设置包括：

• 要传递的最大何处使用实例数 - 当包所有者或协同者将内容指派给包时，应用程序可以传递给“何处使用”函数的最大建模对象实例数。
• 包创建模板 - 用于为从 Teamcenter 发送到 Opcenter EX MDD 或 Opcenter EX CR 的项和修订版本创建包的模板。
• 通知事件 - 当更改管理包发生某些事件时向员工发送的通知。这些事件包括：
• 更改管理审批指派 - 当路由包以进行审批时，通知审批更改管理包所必需的员工。
• 更改管理审批委托 - 通知指派的员工和目标员工，告知其已重新指派包审批任务。
• 更改管理协同指派 - 通知协同者，他们可以将内容指派给更改管理包。
• 更改管理协同委托 - 通知指派的员工和目标员工，告知其已重新指派包协同任务。
• 更改管理所有者协同指派完成 - 当协同者完成向包指派内容时，通知包所有者。
• 包审批提醒 - 当包审批者拥有即将到期、到期或逾期审批时，发送给所有者、审批者或审批者上报收件人的提醒通知。
• 内容协同提醒 - 当内容协同者拥有即将到期、到期或逾期协同工作时，发送给所有者、协同者或协同者上报收件人的提醒通知。
Note: 有关更改管理电子邮件通知的信息，请参考《Opcenter Execution Medical Device and Diagnostics Change Management 用户指南》或《Opcenter Execution Core Change Management 用户指南》。

在定义更改管理设置时

“最大何处使用实例数”默认为 1。如果输入的值小于 1，则应用程序将显示一条警告消息。

Note: 增加“最大何处使用实例数”会影响性能。建议的值为 5。应用程序会接受大于 5 的值，但这可能会影响性能。

Opcenter EX MDD 或 Opcenter EX CR 在为从 Teamcenter 发送到 Opcenter EX MDD 或 Opcenter EX CR 的项和修订版本创建包时将使用包创建模板。包中包括含有工艺清单 (BOP) 和物料清单 (BOM) 信息的产品和修订版本。如果在与启动包的员工关联的工厂上设置“更改管理设置”字段，则指定模板时，将使用预定义的包详细信息、更改管理工作流程和部署详细信息来填充“创建包”页。有关信息，请参考“定义包创建模板”和“定义工厂”。

Note: 此功能仅在与 Teamcenter 集成时可用。有关信息，请参考《Opcenter Execution Core Teamcenter 集成指南》。

定义更改管理设置时，应用程序不要求添加电子邮件通知。但是，如果想要应用程序在包发生事件时向员工发送通知，则必须配置电子邮件通知。添加电子邮件通知时，应用程序会要求选择通知类型和电子邮件消息。使用电子邮件消息建模对象来定义电子邮件消息。

只能从“更改管理设置”页的“通知类型”选择列表中选择更改管理的通知类型。不能从“组织”页的“通知类型”列表中选择这些通知类型。

系统管理员可以将一个员工的任务重新指派给另一员工。例如，如果包审批或协同工作到期时最初受指派执行该工作的员工将要休假，则管理员可以将该工作委托给其他员工。选择“更改管理审批委托”或“更改管理协同委托”通知类型，然后选择重新指派工作时要发送给受指派员工和目标员工的相应电子邮件消息。有关委托任务的信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

“更改管理设置”为可选字段，可在定义“工厂”建模对象实例时进行设置。定义员工实例时，“工厂”为可选字段。因此，要将员工链接到更改管理设置，可通过将“员工”页上的“工厂”字段设为与更改管理设置关联的工厂。

必须定义用于电子邮件通知的 SMTP 传输。“工厂”建模对象实例必须指明为电子邮件通知定义的 SMTP 传输。确保“员工”建模对象实例指明与 SMTP 传输关联的工厂。有关更改管理电子邮件通知的信息，请参考《Opcenter Execution Medical Device and Diagnostics Change Management 用户指南》或《Opcenter Execution Core Change Management 用户指南》。

在定义提醒通知时

当包审批者或内容协同者拥有即将到期、到期或逾期工作时，如果想要向特定员工发送电子邮件消息，可设置提醒通知。应用程序将根据审批和协同任务的到期日期发送提醒通知。

到期日期由包所有者在指派工作时指定的指派日期、“持续时间”和“期限”确定。审核人和协同者需要在特定的时间范围内完成工作。如果包所有者在指派工作时设置了“持续时间”和“期限”字段，则到期日期由这两个字段确定。如果包所有者未设置“持续时间”和“期限”，则到期日期为指派日期和时间。如果未设置“持续时间”和“期限”，任务将在指派时到期。

必须指定电子邮件消息、收件人和希望应用程序发送消息的日子（相对于到期日期）。

使用电子邮件消息建模对象来定义电子邮件消息。要设置动态消息，可以在电子邮件中使用“文本变量”建模对象。例如，可使用正文文本“包 <包名称> 具有未完成审批”为包所有者定义电子邮件消息。有关信息，请参考“定义电子邮件消息”和“定义文本变量”。

可以指定以下员工接收审批提醒通知：

• 所有者 - 通知包的所有者。
• 审批者 - 通知具有未完成任务的所有审批者。
• 审批者上报收件人 - 通知具有未完成任务的任何审批者的上报收件人。

可以指定以下员工接收内容协同提醒通知：

• 所有者 - 通知包的所有者。
• 协同者 - 通知具有未完成任务的所有协同者。
• 协同者上报收件人 - 通知具有未完成任务的任何协同者的上报收件人。
Note: 确保完成每个收件人的“员工”建模对象实例中的“电子邮件地址”字段，否则收件人无法接收电子邮件通知。
Note: 使用“员工”建模对象来定义上报收件人。例如，如果协同者的工作逾期，则可以向内容协同者的经理发送电子邮件通知。如果没有在审批者或协同者的“员工”建模对象实例的“上报收件人”表格中选择上报收件人，则应用程序不会向其发送电子邮件通知。

必须指定发送通知时距到期日期之前或之后的天数，或者可以在到期日期当天发送通知。例如，要在审批或协同工作到期前三天发送通知，则在“到期前天数”中输入 3。如果要在工作逾期一天后发送通知，则在“逾期天数”中输入 1。要在到期日期当天发送通知，在“到期前天数”或“逾期天数”中输入 0。

不能在“包审批提醒”或“内容协同提醒”表格的同一行中同时指定“到期前天数”和“逾期天数”。如果在“逾期天数”字段中输入值，而同一行中的“到期前天数”字段已存在值，则应用程序将清除“到期前天数”字段中的值。同样，如果在“到期前天数”字段中输入值，而同一行中的“逾期天数”字段已存在值，则应用程序将清除“逾期天数”字段中的值。

如果选中“每天重复”复选框，则应用程序将每天发送提醒通知，直到工作完成。根据指定的收件人类型，将发生以下情况：

• 在所有审批或协同工作完成之前，包所有者将一直接收提醒通知。
• 审批者或协同者在完成审批或协同工作之前将一直接收提醒通知。
• 审批者或协同者完成审批或协同工作之前，审批者或协同者的上报收件人将一直接收提醒通知。

应用程序将根据计划的业务规则发送提醒通知。Siemens 提供一个名为 SBR_NOTIFICATIONS 的默认计划业务规则。此规则从凌晨 4:00 开始每隔 24 小时运行一次。计划的业务规则每次运行时且提醒符合资格时都将发送提醒。例如，系统管理员可以将提醒通知设为在协同工作到期前一天（星期五）上午 9:30 发送给协同者。应用程序将在星期四凌晨 4:00 运行默认的计划业务规则，并将提醒通知发送给协同者。

系统管理员还可以定义其他计划的业务规则。例如，管理员可以将计划的业务规则定义为，从中午 12:00 开始每隔六个小时发送这些电子邮件消息。则应用程序将在星期四的中午 12:00、早上 6:00、中午 12:00 和下午 6:00 运行计划的业务规则，并且每次都向协同者发送提醒通知。有关信息，请参考“定义计划的业务规则”。

如何定义更改管理设置

按照以下步骤定义“更改管理设置”：

1. 打开更改管理设置页。随即“建模”页中将显示“更改管理设置”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在更改管理设置字段中输入更改管理设置的名称。
4. 在最大何处使用实例数字段中输入最大何处使用实例数。
Note: 应用程序默认值为允许的最小何处使用实例数。建议的“最大何处使用实例数”值为 5。当然，应用程序会接受大于 5 的值，但这可能会影响性能。
5. 要添加通知事件，可执行“如何添加通知事件”过程。
6. 要添加包审批提醒，可执行“如何添加包审批提醒”过程。
7. 要添加内容协同提醒，可执行“如何添加内容协同提醒”过程。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击保存。 应用程序保存建模对象并显示成功消息。

如何添加通知事件

按照以下步骤添加通知事件：

1. 执行“如何定义更改管理设置”过程。

或

选择现有“更改管理设置”实例。

2. 如果需要，展开电子邮件通知部分。
3. 单击“通知事件”表格中的添加新行。 将显示新行。
4. 选择通知类型。
5. 选择电子邮件消息。
6. 重复步骤 2-5，添加其他通知类型。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何添加包审批提醒

按照以下步骤添加包审批提醒：

1. 执行“如何定义更改管理设置”过程。

或

选择现有“更改管理设置”实例。

2. 如果需要，展开电子邮件通知部分。
3. 单击“添加包审批提醒”表格中的添加新行。 将显示新行。
4. 选择电子邮件消息。
5. 选择收件人。
6. 在到期前天数字段中输入大于等于 0 的整数值。

或

在逾期天数字段中输入大于等于 0 的整数值。

7. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
8. 重复步骤 3-7，添加其他包审批提醒。
9. 单击保存。 应用程序保存建模对象并显示成功消息。

如何添加内容协同提醒

按照以下步骤添加内容协同提醒：

1. 执行“如何定义更改管理设置”过程。

或

选择现有“更改管理设置”实例。

2. 如果需要，展开电子邮件通知部分。
3. 单击“内容协同提醒”表格中的添加新行。 将显示新行。
4. 选择电子邮件消息。
5. 选择收件人。
6. 在到期前天数字段中输入大于等于 0 的整数值。

或

在逾期天数字段中输入大于等于 0 的整数值。

7. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
8. 重复步骤 3-7，添加其他内容协同提醒。
9. 单击保存。 应用程序保存建模对象并显示成功消息。
