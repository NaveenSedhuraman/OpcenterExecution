定义产品系列

产品系列是共享公共属性（例如工作流程、培训需求和启动数量）的一组产品。为产品系列定义的属性应用于系列中的每个产品。

Note: 产品定义中指定的任何属性替代产品的关联产品系列定义中指定的属性。

产品系列通过定义一组产品而不是每个产品的通用数据来简化产品信息维护。每个产品都可以属于一个产品系列，但一件产品只能指派给一个系列。可以在定义产品时将产品指派给产品系列。有关将产品指派给产品系列的信息，请参考“定义产品”。

产品系列组是与对象组不同的概念。对资源和用户代码等对象进行分组，以进行验证和报告。此外，一个对象可以属于多个组，且对象组可能包含其他对象组。

优先级规则

以下优先级规则用于产品和产品系列数据：

• 为产品系列定义的数据可应用于系列中的所有产品。
• 对产品系列数据的更改适用于系列中所有的产品。
• 与产品系列数据相比，在产品级指定的数据优先级更高。

在定义产品系列时

产品系列包含可选的建模定义：

• 文档集
• 培训要求组
• UOM
• 工作流程

“产品系列” 是产品定义中的可选字段。

定义用于 Opcenter Execution Core Scheduling 的产品系列
Note: 仅当安装有 Opcenter Execution Core Scheduling 时，下述功能才可用。有关信息，请参考 "Opcenter Execution Core Scheduling"。

与 ERP 路线关联的工作流程必须指派给制造订单的产品或产品的产品系列。有关信息，请参考“定义产品”。

为产品系列指派默认库存位置
Note: 下述功能仅在安装了 Industry Solutions 工作区时可用。有关信息，请参考“Industry Solutions 工作区”。

可以将默认库存位置指派给产品系列。还可以将默认库存位置指派给产品和工厂。

在指派默认库存位置时，以下规则适用：

• 在产品上指派的默认库存位置优先于在产品系列上指派的默认库存位置。
• 在产品系列上指派的默认库存位置优先于在工厂上指派的默认库存位置。
• 如果在产品系列上定义了默认库存位置，则当容器移动到定义为库存点的作业时，应用程序会自动将默认库存位置指派给容器。

有关信息，请参考“为产品指派默认库存位置”和“为工厂指派默认库存位置”。

在产品系列上指派测重器组
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

可以在产品系列上指派测重器组。还可以在规范、产品和称重分发任务上指派测重器组。

在指派测重器组时，以下规则适用：

• 在产品系列上指派的测重器组优先于在规范上指派的测重器组。
• 在产品上指派的测重器组优先于在产品系列上指派的测重器组。
• 在称重分发任务上指派的测重器组优先于在所有其他对象上指派的测重器组。

有关信息，请参考“在规范上指派测重器组”，“为组件产品指派测重器组”和“在定义称重分发任务时”。

定义产品系列的 UOM 转换信息
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

“UOM 转换”表格包含允许为产品系列定义数量转换信息的字段。可以定义：

• 将产品系列的任何分发量的度量单位转换为另一度量单位的转换系数。
• 转换数量的取整规则。
• 计算的转换数量中允许的小数位数。最大值为 4。

在定义产品系列的 UOM 转换信息时，以下规则适用：

• 必须填充“取整规则”和“小数位数”字段以进行取整。
• 在产品上定义的转换信息优先于在产品系列上定义的转换信息。
• 在产品系列上定义的转换信息优先于在 UOM 上定义的转换信息。

有关信息，请参考“定义产品的 UOM 转换信息”和“Medical Device 工作区”。

如何定义产品系列

按照以下步骤定义产品系列：

1. 打开产品系列页。随即“建模”页中将显示“产品系列”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在产品系列字段中输入产品系列的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。
