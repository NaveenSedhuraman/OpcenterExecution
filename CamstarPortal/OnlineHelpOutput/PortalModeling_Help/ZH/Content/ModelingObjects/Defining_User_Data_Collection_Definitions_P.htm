定义用户数据收集定义

用户数据集定义包含一组数据收集参数。这些数据收集参数在活动完成期间呈现给用户。

用户数据收集定义由使用它的活动模板引用。

每个数据收集参数都包含一组数据点。数据点表示业务流程中可以或必须收集数据的点。一旦定义完成，多个数据点可在一个用户数据收集定义中分组。

用户数据收集定义有助于捕获作为活动一部分的数据，可以选择按照数据点定义中所定义的上限和下限进行验证。

例如，可以指定值数据点（比如布尔值）。在这种情况下，上部和下部数据点应相同，并且该值之外的任何内容都将导致收集任务失败，该失败会记录在历史中。

或者，可以指定对象数据点。对象数据点有助于在引用流程模型中的特定活动时捕获数据。

此应用程序允许您：

• 使用“值数据点详细信息”弹出窗口的值数据点。
• 使用“对象数据点详细信息”弹出窗口的对象数据点。

两者都可从“用户数据收集定义”页访问。

定义用户数据收集

该应用程序为基于用户的数据收集提供了两种基本布局：

• 行和列（单一字段）
• 迭代表格

可以在“用户数据收集定义”页的“数据点布局”字段中指定布局。

用户数据收集定义包含可选的建模定义：SPC 图表组。

行列布局

行和列布局提供传统的基于字段的页面。

迭代表格

迭代表格布局提供由多行多列构成的表格。可以使用表格右上角的“方向”按钮在垂直视图和水平视图之间切换以更改信息显示方向。

在添加对象数据点时

对象数据点实质上与值数据点具有相同的用途 - 其提供在生产流程中的特定点收集数据的方法。对象数据点引用特定对象。例如，可使用对象数据点建立原因代码，从而记录预定义检验状态结果。这些结果包含诸如污点和表面划痕等情况。操作员只需从选择值列表中选择适当的结果即可完成车间的检查任务。

无法创建具有相同行位置和列位置的两个数据点。

Note: 限制不适用于对象数据点。

添加 值数据点

将值数据点添加到用户数据收集时，可以执行以下操作：

• 通过输入静态值或表达式来定义上限和下限
• 指示是否允许限制替代
• 选择将数据点值映射到容器属性

可以使用表达式来定义上限、下限或全部两个限制。使用表达式时，在运行时确定限制。

使用表达式来定义上限和下限时，应使用 Opcenter EX MDD 或 Opcenter EX CR 统一表达式语法。表达式最多可包含 255 个字符，可能包括：

• 字段表达式;例如，Container.CurrentStatus.Spec.UserDefinedInteger
• 字段表达式和基本算术的组合; 例如，(100-Container.CurrentStatus.Spec.UserDefinedInteger) *0.2
Note: 输入表达式时，不会验证语法是否正确。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。

允许 值数据点的限制替代

将值数据点添加到用户数据收集定义时，可以选择允许用户输入上限和下限之外的值。选中“允许限制替代”复选框会导致在用户输入超出限制的值时，应用程序将限制字段的底纹变为橙色。应用程序允许用户执行数据收集。

清除“允许限制替代”复选框会导致在用户输入超出限制的值时，应用程序将限制字段的底纹变为红色。

Note: 此功能适用于行和列布局以及迭代表格。

启用数据收集工具提示

可以在收集数据时使用工具提示。只需将光标悬停在任务数据输入字段上即可显示数据点值：度量单位和下限与上限。此功能适用于行和列布局以及迭代表格。

如何定义用户数据收集定义

按照以下步骤定义用户数据收集定义：

1. 打开用户数据收集定义页。随即“建模”页中将显示“用户数据收集定义”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在用户数据收集定义字段中输入此定义的唯一名称。
4. 在修订版本字段中输入此定义的修订版本标识。
5. 如果它是记录的修订版本，则选中记录修订版本复选框。
6. 从列表中选择数据点布局的类型：迭代表格或行列位置。
7. 请参考一下的过程之一来添加选定的数据点类型：
• “如何添加值数据点”
• “如何添加对象数据点”
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击保存。 应用程序保存建模对象并显示成功消息。

如何添加值数据点

按照以下步骤将值数据点添加到用户数据收集定义中：

1. 执行“如何定义用户数据收集定义”过程。

或

选择现有用户数据收集定义实例。

2. 展开详细信息部分。
3. 单击“数据点”表格中的添加新行，然后在“数据点类型”弹出窗口中选择值数据点。将显示“值数据点详细信息”弹出窗口。
4. 在数据点名称字段中输入此值数据点的唯一名称。
5. 输入数字值来表示值数据点必须放在任务中的行位置。
6. 输入数字值来表示值数据点必须放在任务中的列位置。
7. 从列表中选择数据类型。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击确定。 “值数据点”弹出窗口关闭，值数据点显示在“用户数据收集”页上的“数据点”表格中。

如何添加对象数据点

按照以下步骤将对象数据点添加到用户数据收集定义中：

1. 执行“如何定义用户数据收集定义”过程。

或

选择现有用户数据收集定义实例。

2. 展开详细信息部分。
3. 单击“数据点”表格中的添加新行，然后在“数据点类型”弹出窗口中选择对象数据点。将显示“对象数据点详细信息”弹出窗口。
4. 在数据点名称字段中输入此对象数据点的唯一名称。
5. 输入数字值来表示对象数据点必须放在任务中的行位置。
6. 输入数字值来表示对象数据点必须放在任务中的列位置。
7. 从对象类型列表中选择与此对象数据点关联的特定对象。
8. 从选择值类型列表中选择要对对象类型使用的值类型。
9. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
10. 单击确定。 “值数据点”弹出窗口关闭，值数据点显示在“用户数据收集”页上的“数据点”表格中。
