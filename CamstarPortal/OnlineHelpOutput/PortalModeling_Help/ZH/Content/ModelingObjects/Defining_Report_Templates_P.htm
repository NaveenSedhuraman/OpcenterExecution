定义报告模板

此“报告模板”会指定用于智能报告和图表的 RPT 文件。有关图表和报告的信息，请参考《Opcenter Execution Core Intelligence 参考指南》。

在定义报告模板时

这些规则在定义报告时适用：

• 报告模板的每个实例都包含一个“查找文件”字段，以查找要保存或上传的文件。
• 可以使用“上传文件到数据库”复选框表示将特定文件上传到数据库的时间。通常，在最初定义报告模板时会选中此复选框。更新现有实例时将该复选框留空，可在无需上传文件的情况下进行更改。
• 将文件上传到数据库并保存“报告模板”实例后，应用程序将启用“查看文档”按钮。单击“查看文档”查看或下载文档。
• “存储的文件名”字段是一个只读字段，单击“保存”后将显示文件名。
• RPT 文件的下载/上传时间将因文件大小而异。

如何定义报告模板

按照以下步骤定义“报告模板”：

1. 打开报告模板页。随即“建模”页中将显示“报告模板”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在报告模板字段输入报告模板名称。
4. 在修订版本字段输入此模板的修订版本。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将文件添加到报告模板

按照以下步骤将文件添加到报告模板：

1. 执行“如何定义报告模板”过程。

或

选择现有的“报告模板”实例。

2. 单击“查找文件”字段旁边的浏览按钮。将出现一个用于浏览文件的弹出窗口。
3. 浏览某个文件，然后单击打开。弹出窗口将关闭，文件名将显示在“查找文件”字段。
4. 选择将文件上传到数据库。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
