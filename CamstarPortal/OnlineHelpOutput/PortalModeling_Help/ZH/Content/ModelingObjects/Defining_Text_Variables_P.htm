定义文本变量

文本变量将合并到消息字符串中并用于在运行时派生值。它们由 WIP 消息、历史查询等对象使用。

每个文本变量定义都包括一个名称和一个表达式，并且描述了如何处理无法成功计算表达式的情况。此名称是嵌入静态字符串中的值，作为其前缀的字符将其识别为文本变量。在启用的字符串中引用文本变量时，必须在文本变量实例名称前面使用 # 符号（例如，#CONTAINER）。

例如，具有名称 CONTAINER 的文本变量可以包含在消息“#CONTAINER 已被搁置”中。 表达式是运行时将要计算的字符串，以得出替换字符串的运行时版本内的标识的值。在此示例中，标识将被替换为容器的名称。

有效表达式字符串的组成内容根据参考文本变量的消息类型而变化。某些表达式（因此文本变量）可被所有的消息类型共享。例如，一个文本变量可用来为当前 SessionValue 合并值。其他表达式只有在特定消息类型的上下文中才有意义。例如，一个 WIP 消息可能包括当前在机器上运行的产品的一个变量，在这台机器上“移入”事务正显示 WIP 消息。有关使用带有 WIP 消息的文本变量的信息，请参考“定义 WIP 消息”。

Note: 仅当配置了用于替换字符串字段的 Designer 设置时，才计算字符串字段中的文本变量。有关信息，请参考《Designer 用户指南》。

如何定义文本变量

按照以下步骤定义文本变量：

1. 打开文本变量页。随即“建模”页中将显示“文本变量”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在文本变量字段中输入文本变量的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。
