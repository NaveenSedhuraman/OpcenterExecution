定义资源组

资源组是一个代表一组资源的命名对象组子类。对象组用于提供对象列表，例如资源、产品或特定类型的用户代码的列表。这些列表通常用于确认和选择。例如，作业包含对 LossCodeGroup 的引用。此列表中的条目定义容器在作业时执行工作的有效损失（原因）代码。

对象组实例包括特定对象列表和对象组列表。任何给定对象组的对象列表只限于特定类型（或对象列表的任何派生类型）。同样，对象组内的对象组列表只限于相同类型（或对象组列表的任何派生类型）。

以下是已命名对象组的子类示例：

• 资源组
• 装运目的地组
• 用户代码组

“物料箱组”是“资源组”的子类。

有关此建模对象的信息，请参考“定义物料箱组”。

在定义资源组时

在定义“准备”、“规范”和“工作中心”时，“资源组”是可选字段。

可以将多个条目或组添加到各自的表格。

使用“解决条目”按钮可以显示在此组和所有嵌套组的条目列表中指定的所有值的列表。

定义用于 Opcenter Execution Core Scheduling 的资源组
Note: 仅当安装有 Opcenter Execution Core Scheduling 时，下述功能才可用。有关信息，请参考 "Opcenter Execution Core Scheduling"。

在 Opcenter EX MDD 或 Opcenter EX CR 中创建新资源组或更新现有资源会触发应用程序将资源组信息发送到 Preactor。

如何定义资源组

按照以下步骤定义资源组：

1. 打开资源组页。随即“建模”页中将显示“资源组”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在资源组名称字段输入资源组名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 按如下所示向组中添加条目：
a. 单击“条目”表格底部的添加新行。 将显示新行。
b. 在新行中单击以显示条目列表，然后选择一个添加到组中。所选条目即添加到表格。
c. 重复步骤 a–b，将另一条目添加到表格。
6. 按如下所示向组中添加子组：
a. 单击“组”表格底部的添加新行。 将显示新行。
b. 在新行中单击以显示先前定义的组的列表，然后选择一个（作为子组）添加到组中。所选组即添加到表格。
c. 重复步骤 a–b，将另一组添加到表格。
7. 单击保存。 应用程序保存建模对象并显示成功消息。
