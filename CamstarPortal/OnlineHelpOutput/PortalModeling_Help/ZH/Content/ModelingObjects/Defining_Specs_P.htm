定义工艺

规范定义了在工作流程或主配方的某个工步中执行的活动，包括详细的计划和处理参数信息。规范将引用许多其他建模组件，包括“作业”、“准备”、“数据收集定义”以及“资源”。

定义工作流程和主配方时，应用程序将要求引用至少一个规范。不同的工作流程和主配方可以多次引用同一个规范。

Portal 用户可以在创建事件时指定规范。

定义工艺时

工艺包含：

• 必需的建模定义“作业”。
• 可选的建模定义“业务规则”、“数据收集定义”、“文档集”、“电子流程”、“电子签名要求”、“打印机标签定义”、“配方”、“资源组”、“测重器组”、“设置”以及“培训要求组”。
Note: “测重器组”字段仅在安装了 Batch Processing 时出现。有关信息，请参考 "Batch Processing"。

“规范”是定义“工作流程”和“主配方”的必需字段。

Note: 如果规范定义与容器定义中的字段相关联，则它可以具有相关的 WIP 消息。

在指定电子签名和电子流程时

“电子签名事务映射”中指派的电子签名仅适用于容器事务。如果将电子流程与此规范相关联，并且要捕获电子流程任务的电子签名，则必须分别为指示和计算任务指派电子签名。指派给规范的电子签名将适用于电子流程中的事务任务。有关定义指示和计算任务的信息，请参考“定义任务列表和相关任务”。

向规范添加文档或从规范拆离文档时添加电子签名的要求

可以设置用户在向规范添加文档或从规范拆离文档时添加电子签名的要求。文档可以包括特殊指示、测试结果、图纸或照片。有关文档附件的信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

为文档附加或拆离设置电子签名要求的选项显示在“事务”部分的“电子签名事务映射”中。或者，可以选择要求所有事务均使用电子签名。

收集有关规范的批次采样数据时添加电子签名的要求

可以设置用户在收集有关规范的批次采样数据时添加电子签名的要求。设置这些要求的选项显示在“事务”部分的“电子签名事务映射”中。

可以提交用于在“收集批次采样数据”车间页上收集批次采样数据的签名。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

添加流程计时器

可以添加流程计时器来控制和监视制造过程。可以定义事务在满足最小时间或最大时间已过之前尝试进行处理时应用程序要执行的操作。

用于在规范中设置流程计时器的选项显示在“事务”部分的“开始计时器”和“结束计时器事务映射”中。应用程序要求将“移动”或“移入”事务指派给设置的所有流程计时器。将“移动”事务指派给某个计时器会导致应用程序在容器移出设置计时器的规范时触发该计时器。将“移入”事务指派给某个计时器会导致应用程序在容器移入设置计时器的规范中时触发该计时器。

有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

Note: 每个流程计时器都应该有一个开始和结束计时器，但也可以只有其中一个计时器而没有另一个计时器。
Note: 流程计时器仅依赖于“移动”（标准）或“移入”事务。应用程序不会考虑可以对电子流程任务执行的这些事务的数量。执行任务时流程计时器停止。

将业务规则添加到规范

可以将业务规则添加到规范，以在发生与容器事务关联的事件时执行业务规则逻辑。在“事务”部分的“业务规则事务映射”表格中添加业务规则。只能选择指派了容器用法类型的业务规则。有关信息，请参考“业务规则页字段定义”。

“业务规则事务映射”表格包含三列：业务规则、事件和事务。选择业务规则以及将执行该业务规则的容器事务。此外，还要选择用于确定何时在容器事务中执行业务规则的事件。可用的事件包括：

• 执行用户
• 导出信息初始化用户
• 预执行用户
• 预验证用户
• 提交后处理用户
• 验证用户

规范页上的到期日期计算
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

当预定义的事务在已标识的规范下成功完成时，可以使用“规范”页定义自动计算容器的到期日期所需的字段。这些字段如下所示：

• 到期日期事务
• 到期周期单位
• 到期周期

仅当产品未定义到期日期时，才能根据规范信息计算容器的到期日期。

如果容器的到期日期早于计算出的日期，则除非选中“始终替代到期日期”复选框，否则不会更新到期日期。

如果将“到期周期单位”字段设为“月份”，并且到期日期计算出无效的日期，则到期日期将调整为该月的最后一天。例如，如果当前日期为 1 月 31 日，并且到期周期为 3 个月，则计算的到期日期为 4 月 31 日。日期将调整为 4 月 30 日。

规范页上的制造日期计算
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

可以在“规范”上选择“制造日期事务”，以便自动将制造日期指派给容器。这样就可将事务日期和时间指派给容器上的“制造日期”字段。如果在产品上定义了制造日期信息，则以产品数据为准。

在规范页上定义 UDI
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

应用程序的 Unique Device Identification (UDI) 功能可用于验证是否已为容器指派了 UDI。如果选择了规范上的“验证 UDI”，则在 UDI 字段具有值之前，容器将无法移动。

在规范上指派测重器组
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

可以使用“测重器组”字段在规范上指派测重器组。还可以在产品系列、产品和称重分发任务上指派测重器组。

在指派测重器组时，以下规则适用：

• 在产品系列任务上指派的测重器组优先于在规范上指派的测重器组。
• 在产品上指派的测重器组优先于在产品系列上指派的测重器组。
• 在称重分发任务上指派的测重器组优先于在所有其他对象上指派的测重器组。

有关信息，请参考“在产品系列上指派测重器组”、“为组件产品指派测重器组”和“在定义称重分发任务时”。

在规范页上定义团队追踪事务
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

可以使用“规范”页来定义需要追踪员工在给定作业、规范、工作中心、工作单元或工作站所执行的工作的事务。通过在工厂级别定义事务，可以为所有规范设置跟踪要求，而无需修改工作流程中的每个规范。但是，在规范级别设置的任何事务均优先于在工厂级别进行的选择。

如何定义规范

按照以下步骤定义规范：

1. 打开规范页。随即“建模”页中将显示“规范”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在规范字段中输入此规范的名称。
4. 在修订版本字段中输入此对象的修订版本。
5. 从作业列表中选择要与规范关联的作业。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何添加数据收集信息

按照以下步骤添加数据收集信息：

1. 执行“如何定义规范”步骤。

或

选择现有规范实例。

2. 展开事务部分。
3. 单击“数据收集事务映射”表格上的添加新行。 将显示新行。
4. 从要使用的事务字段中选择要为指派数据收集定义的事务。
5. 选择数据收集定义，以确定在事务处理过程中向用户显示的数据收集参数组。
6. 重复步骤 3-5，为此规范添加其他数据收集定义引用。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何添加流程计时器

按照以下步骤添加流程计时器：

1. 执行“如何定义规范”步骤。

或

选择现有规范实例。

2. 展开事务部分。
3. 单击“开始计时器事务映射”表格上的添加新行。 将显示新行。
4. 从计时器字段中选择一个现有计时器。
5. 从开始事务字段中选择“移动”或“移入”。
6. 单击“结束计时器事务映射”表格上的添加新行。 将显示新行。
7. 从计时器字段中选择一个现有计时器。
8. 从结束事务字段中选择移动或移入。
9. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
Note: 每个流程计时器都应该有一个开始和结束计时器，但也可以只有其中一个计时器而没有另一个计时器。
