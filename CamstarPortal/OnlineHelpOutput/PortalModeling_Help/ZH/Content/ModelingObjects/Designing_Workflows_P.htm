设计工作流程

在“工作流程”页上的“工作流程图”部分，可以查看、设计和修改工作流程。可以通过将规范或工作流程的修订版本拖动到图上来添加工步。通过选择工步上的路径图标并拖动到工作流程中的下一工步来添加路径。

在定义工作流程时

应用程序使用规范名称作为工作流程工步和新添加工步的路径的默认名称。例如，如果添加了“装运”规范作为工步，则工步和工步路径的名称都为“装运”。重命名工步和路径有助于区分工步和规范以及明确路径。可以将“装运”规范工步重命名为正在装运，并将路径重命名为目标装运。

将先前已定义的工作流程添加到当前工作流程时，同样的原则也适用。添加的工作流程将成为子工作流程，并根据添加的工作流程命名。

必须展开规范或工作流程，然后选择特定修订版本，以添加到工作流程图中。

工作流程包含必需的建模定义（规范）。 

工作流程包含可选的建模定义 ERP 路线。

此外，工作流程在开始、移动（路径和工步）、非标准移动、非标准移动（多个）和返工（工步）事务中为可选。

定义用于 Opcenter Execution Core Scheduling 的工作流程
Note: 仅当安装有 Opcenter Execution Core Scheduling 时，下述功能才可用。有关信息，请参考 "Opcenter Execution Core Scheduling"。

必须将 ERP 路线指派给工作流程，并将 ERP 路线工步指派给对应的工作流程工步。

每个工作流程工步不一定有对应的路线工步。路线工步只能在特定工作流程中使用一次。

在向工作流程添加工步时

工步名称在工作流程中必须是唯一的。默认情况下，应用程序会将所引用的规范或子工作流程的名称指派给添加到工作流程的工步。如果添加了引用同一规范或工作流程的第二个工步，则应用程序将为该工步指派数值扩展名。例如，默认情况下，将名为“可视化检查”的规范添加到工作流程中会导致工步被命名为可视化检查。第二次将“可视化检查”规范添加到工作流程中会导致工步被命名为可视化检查_1。

可随时更改工步名称。选择工步，然后单击“修改元素”按钮以查看“工步详细信息”弹出窗口。有关查看和修改工步信息的详情，请参考“使用工步详细信息”。

用于描述规范工步与工作流程工步的图标不同。此外，无论第一工步是规范还是子工作流程，第一工步的图标都与连续工步的图标不同。

可以通过单击并拖动工步来重新定位工作流程中的工步。通向和从该工步开始的任何路径都将随工步一起重新定位。

Important: 默认情况下，Management Studio 中的“返回的最大行数”配置设置设为 15,000。如果具有非常大的工作流程，则可以将该设置修改为大于工作流程中的工步数。有关信息，请参考《系统管理指南》。

在工步之间添加路径时

路径名称在工作流程中必须唯一。应用程序将使用“目标工步”的名称作为路径的默认名称。例如，添加从“打包”工步到“装运”工步的路径会生成名为“装运”的路径。无论路径类型为默认、备选、返工还是循环，应用程序都将使用“目标工步”这一名称。

路径名称仅在“路径详细信息”弹出窗口中可见。选择路径，然后单击“修改元素”按钮可查看“路径详细信息”弹出窗口。有关查看和修改路径信息的详情，请参考“使用路径详细信息”。

工步图标左侧显示三个路径图标：默认（主要）、备选和返工。使用这些图标可指定从该工步到另一工步的路径类型。只能添加从某工步开始的一个默认路径，但可以添加从某工步到其他工步的多个备选路径和返工路径。还可以将这些路径中的任何一个循环回当前工步。单击路径图标并将鼠标拖动到工步可绘制路径。

Note: 针对每个工步只能绘制一个循环路径。

应用程序：

• 在“来源工步”和“目标工步”之间只允许有一个路径。如果要更改两个工步之间的路径类型，则必须删除路径后再重新绘制，或者修改路径详细信息。
• 必须先绘制默认路径，再绘制备选路径。如果在绘制默认路径之前绘制备选路径，应用程序会自动将备选路径转换为默认路径。
• 仅允许有一个来自“来源工步”的默认（主要）路径。应用程序会自动将后续的任何默认路径转换为备选路径。如果要将默认路径重定向到另一工步，则必须删除现有默认路径后再重新绘制到另一工步，或修改路径详细信息。
Note: 单击“重置”不会重置工作流程画布。必须手动删除已添加的任何工步或路径。

在删除工作流程或工作流程元素时

删除工作流有两种方式：

• 删除工作流程及其所有修订版本。
• 删除工作流程修订版本但保存工作流程实例。

当存在多个工作流程修订版本时，无法删除记录修订版本。必须首先将指定记录的修订版本指派为不同的修订版本。

如果删除原始的第一工步，则必须指派另一个工步作为第一工步。如果删除第一工步并且未指定另一工步作为第一工步，则在保存工作流程时应用程序将显示错误消息。

如果多个工步中有一个工步包含路径选择器，则无法删除它们。必须先删除路径，单击“保存”，然后删除其他路径。无法删除包含路径的工步，除非先删除路径。

试图查看正在使用或已使用过的任何已删除规范或工作流程的历史可能导致错误。如果从工作流程中删除其所在的工步，则容器将失去其当前状态。

删除路径不会影响其链接的工步。但是，删除默认路径时，“来源工步”上将显示默认路径图标，用于添加新的默认路径。

在 Opcenter Execution Semiconductor (Opcenter EX SM) 中使用工作流程
Note: 仅当安装有 Opcenter EX SM 时，下述功能才可用。

应用程序为具有大量工步的工作流程执行递增计划。递增计划包括多次计划批次，每次包含总工作流程工步的一个子集。这些子集逐个进行计划，直到所有工步完成 WIP 处理。

示例解释

1. 应用程序根据“要处理的最大计划工步数”值 20，计划批次的前二十个工作流程工步。
2. 当要完成的剩余工步数等于“更新 Preactor 工步计数”值（该值为 2）时，应用程序发送下一组工步。此组包含两个未完成的工步和后续 18 个工步。
3. 继续执行过程，直到工作流程中的所有工步完成 WIP 处理为止。

如何定义工作流程

按照以下步骤定义“工作流程”：

1. 打开工作流程页。随即“建模”页中将显示“工作流程”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在工作流程字段中输入工作流程的名称。
4. 在修订版本字段中输入此定义的修订版本标识。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考“”主题。
6. 单击保存。 应用程序保存建模对象并显示成功消息。
7. 完成以下过程，以便在必要时向工作流程图添加工步和路径：
• “如何将工步添加到工作流程”
• “如何在工作流程工步之间添加路径”

如何将工步添加到工作流程

按照以下步骤将工步添加到工作流程：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 展开要添加的规范或工作流程作为查看所有修订版本的工步。
3. 选择要作为工步添加的修订版本，并将其拖动到工作流程图中。引用所选规范或工作流程的工步将出现在工作流程图中。该工步将显示在被放置的位置。
4. 重复步骤 2-3 添加其他工步。
5. 单击保存。 应用程序保存建模对象并显示成功消息。

如何在工作流程工步之间添加路径

按照以下步骤在工作流程工步之间添加路径：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 是否要在两个工步之间添加默认路径、备选路径或返工路径？

如果要添加 ...

操作步骤 ...

默认路径
a. 单击“来源工步”上的默认路径图标，然后将其拖至“目标工步”。两个工步之间会显示绿色实线。
b. 重复步骤 a，将其他默认路径从“来源工步”添加到“目标工步”。
c. 转到步骤 3。

备选路径
a. 单击“来源工步”上的备选路径图标，然后将其拖至“目标工步”。两个工步之间会显示绿色虚线。
b. 重复步骤 a，将其他备选路径从“来源工步”添加到“目标工步”。
c. 转到步骤 3。

返工路径
a. 单击“来源工步”上的返工路径图标，然后将其拖至“目标工步”。两个工步之间会显示红色虚线，并在“来源工步”中隐藏返工路径图标。
b. 重复步骤 a，将其他返工路径从“来源工步”添加到其他“目标工步”。
c. 转到步骤 3。
3. 完成“如何定义路径选择器和返工路径选择器”过程，以便在必要时使用为各个路径使用路径选择器。
4. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何将循环路径添加到工作流程工步

按照以下步骤向工作流程工步添加循环路径：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 单击相应的路径图标（默认、备选或返工）并将其拖动到当前工步的中心。将显示一条回环至同一工步的路径。
3. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何将工步插入现有工作流程

按照以下步骤将工步插入现有工作流程：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 选择两个工步之间要插入工步的路径。路径将高亮显示。
3. 单击移除项。路径已移除。
4. 展开要添加的规范或工作流程作为查看所有修订版本的工步。
5. 选择要插入的修订版本，并将其拖动到工作流程图中。该工步将显示在被放置的位置。
6. 选择“来源工步”上的适当路径图标，然后将其拖至新插入工步。路径已添加。
7. 选择新插入工步上的相应路径图标，然后将其拖至下一相应工步。
8. 为要从新插入的工步绘制的任何其他路径重复步骤 7。
9. 如果需要，绘制从现有工步到新插入工步的相应路径。
10. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何修改工步信息

按照以下步骤修改工步信息：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 选择要修改的工步。“修改元素”按钮将显示在“工作流程图”工具栏上。
3. 单击修改元素。 将显示“工步详细信息”弹出窗口。
4. 如果需要，修改必填字段：
• 工步名称
• 规范
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考“工步详细信息弹出窗口字段定义”主题。
Note: 有关使用路径选择器的信息，请参考“使用路径选择器”。
6. 单击确定。 “工步详细信息”弹出窗口关闭。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何修改路径信息

按照以下步骤修改路径信息：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 选择要修改的路径。“修改元素”按钮将显示在“工作流程图”工具栏上。
3. 单击修改元素。 将显示返工路径的“返工工步详细信息”弹出窗口和所有其他路径的“工步详细信息”弹出窗口。
4. 如果需要，修改返工路径名称或路径名称。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考“路径详细信息弹出窗口字段定义”和“了解工作流”主题。
6. 单击确定。 弹出窗口关闭。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何删除工作流程工步或路径

按照以下步骤删除工作流程工步或路径：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 选择要删除的元素。
3. 单击移除项。删除所选工步时，将移除该工步以及绘制到该工步或从该工步绘制的所有路径。删除所选路径时，只移除该路径。
4. 如果需要，添加新路径。有关信息，请参考“如何在工作流程工步之间添加路径”。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
