定义更改管理工作流程

更改管理工作流程是用于指导包完成更改管理过程的工步序列。更改管理工作流程可以包含多个元素：

• 更改管理工步
• 更改管理路径
• 更改管理子工作流程

更改管理工作流程与容器工作流程中的元素类似。有关工作流程元素的信息，请参考“了解工作流”。有关更改管理工作流程的具体信息，请参考《Opcenter Execution Medical Device and Diagnostics Change Management 用户指南》或《Opcenter Execution Core Change Management 用户指南》。

可在“更改管理工作流程”页的“更改管理工作流程图”部分中查看、设计和修改更改管理工作流程。要添加工步，可通过将更改管理规范或更改管理工作流程的修订版本拖至“更改管理工作流程图”。要添加路径，可选择工步上的路径图标，然后拖至工作流程中的下一工步。有关路径的信息，请参考“使用路径选择器”。

默认更改管理工作流程

Siemens 默认提供三个更改管理工作流程，可用于立即开始部署更改管理包。

• 无需审批 - 用于从源系统部署更改管理包，无需对包进行审批。
• Camstar - 需要先对更改管理包进行审批，然后才能从源系统进行部署。选择此选项将启用“更新包”页上的“审批者”选项卡，用于查看审批模板和指派的审批者。
• PLM - 更改管理包必须有产品生命周期管理 (PLM) 或外部审批决策，才能从源系统进行部署。

如果要与 Teamcenter 集成，Siemens 将提供其他默认工作流程。

• MOMPLM - 用于在 Opcenter EX MDD 或 Opcenter EX CR 中处理 Teamcenter 更改管理包。有关在 Opcenter EX MDD 或 Opcenter EX CR 中安装 MOM 工作区以及处理 Teamcenter 更改管理包的信息，请参考《Opcenter Execution Core Teamcenter 集成指南》。

每个默认工作流程都包含 Siemens 提供的更改管理规范。可使用或自定义提供的更改管理规范和工作流程，或者自行创建以支持组织的业务流程模型。有关定义更改管理规范的信息，请参考“定义更改管理规范”。

版本 6、软件更新 12 中的默认工作流程更新

Siemens 已将默认更改管理工作流程中的“已关闭”和“已作废”工步替换为版本 6、软件更新 12 中的“关闭”和“作废”事务。版本 6、软件更新 12 中的默认工作流程不再包含“已关闭”和“已作废”工步。

安装 SU12 或更高版本的软件更新不会影响现有的默认工作流程。可以继续使用“已关闭”和“已作废”工步以及“更改包工步”功能来关闭和作废包。或者，可以从默认工作流程中移除“已关闭”和“已作废”工步，然后使用“关闭”和“作废”包事务。有关关闭和无效事务的具体信息，请参考《Opcenter Execution Medical Device and Diagnostics Change Management 用户指南》或《Opcenter Execution Core Change Management 用户指南》。

在定义更改管理工作流程时

应用程序将使用更改管理规范的名称作为工作流程工步和新添加工步的路径的默认名称。例如，如果之前添加了 Deploy 更改管理规范作为工步，则工步名称为 Deploy 并且工步路径为 Deploy。重命名工步和路径有助于区分工步和规范以及明确路径。可以将 Deploy 规范工步重命名为 Deploying 并将路径重命名为 To Deploying。

同样的原则也适用于将之前定义的更改管理工作流程添加到用户自己的工作流程。添加的工作流程将成为子工作流程，并根据添加的工作流程命名。

必须展开更改管理规范或更改管理工作流程，然后选择要添加到“更改管理工作流程”图的特定修订版本。

“更改管理工作流程”包含必需的建模定义更改管理规范。 

WIP 消息与更改管理建模对象无关。

在向更改管理工作流程添加工步时

工步名称在更改管理工作流程中必须是唯一的。默认情况下，应用程序将引用的规范或子工作流程的名称指派给添加到更改管理工作流程的工步。默认情况下，应用程序将添加到工作流程默认设置的第一个工步标记为“第一工步”。如果添加了引用同一规范或工作流程的第二个工步，则应用程序将为该工步指派数值扩展名。例如，向工作流程添加名为 Deploying 的规范时，系统将默认生成名为 Deploying 的工步。第二次将 Deploying 规范添加到工作流程时，系统将生成名为 Deploying_1 的工步。

可随时更改工步名称。选择工步，然后单击“修改元素”按钮即可查看“更改管理工步详细信息”弹出窗口。有关查看和修改工步信息的信息，请参考“如何修改工步信息”。

用于描述规范工步与工作流程工步的图标不同。此外，无论第一工步是规范还是子工作流程，第一工步的图标都与连续工步的图标不同。

可通过在更改管理工作流程中单击并拖动工步来重新定位。通向和从该工步开始的任何路径都将随工步一起重新定位。

在工步之间添加路径时

路径名称在更改管理工作流程中必须是唯一的。应用程序将使用“目标工步”的名称作为路径的默认名称。例如，将基于“起草”工步的路径添加到部署未完成工步时，系统会生成名为部署未完成的路径。无论路径类型为默认、备选还是循环，应用程序都将使用“目标工步”的名称。

在删除更改管理工作流程或工作流程元素时

删除更改管理工作流程的方法有两种：

• 删除更改管理工作流程及其所有修订版本
• 删除修订版本，但保留更改管理工作流程实例。

如果存在多个更改管理工作流程修订版本，则无法删除记录的修订版本。必须首先将指定记录的修订版本指派为不同的修订版本。

可以删除标记为工作流程中第一个工步的工步，但是将无法保存，除非重新指派其他工步作为第一个工步。

如果多个工步中有一个工步包含路径选择器，则无法删除它们。必须先删除路径，单击“保存”，然后删除其他路径。无法删除包含路径的工步，除非先删除路径。

试图查看正在使用或已使用过的任何已删除规范或工作流程的历史可能导致错误。如果将包所在的工步从工作流程中删除，则包将失去其当前状态。

删除路径不会影响所链接的工步。但是，删除默认路径时，“来源工步”上将显示默认路径图标，用于添加新的默认路径。

如何定义更改管理工作流程

按照以下步骤定义更改管理工作流程：

1. 打开更改管理工作流程页。随即“建模”页中将显示“更改管理工作流程”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在更改管理工作流程字段中输入更改管理工作流程的名称。
4. 在修订版本字段中输入此定义的修订版本标识。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 单击保存。 应用程序保存建模对象并显示成功消息。
7. 如果需要，完成以下过程，将工步和路径添加到更改管理工作流程图：
• “如何将工步添加到更改管理工作流程”
• “如何在工作流程工步之间添加路径”

如何将工步添加到更改管理工作流程

按照以下步骤向更改管理工作流程添加工步：

1. 执行“如何定义更改管理工作流程" 过程.

或

选择现有更改管理工作流程实例。

2. 展开要作为工步添加的更改管理规范或更改管理工作流程，以查看所有修订版本。
3. 选择要添加为工步的修订版本，然后将其拖至“更改管理工作流程图”。引用所选规范或工作流程的工步将显示在“更改管理工作流程图”中。该工步将显示在被放置的位置。
4. 重复步骤 2-3 添加其他工步。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何在工作流程工步之间添加路径

按照以下步骤在工作流程工步之间添加路径：

1. 执行“如何定义更改管理工作流程" 过程.

或

选择现有更改管理工作流程实例。

2. 是否要在两个工步之间添加默认路径或备选路径？

如果要添加 ...

操作步骤 ...

默认路径
a. 单击“来源工步”上的默认路径图标，然后将其拖至“目标工步”。应用程序将在两个工步之间添加一条绿色实线。
b. 重复步骤 a，将其他默认路径从“来源工步”添加到“目标工步”。
c. 转到步骤 3。

备选路径
a. 单击“来源工步”上的备选路径图标，然后将其拖至“目标工步”。应用程序将在两个工步之间添加一条绿色虚线。

3. 对于要添加的每个其他路径重复步骤 2。
4. 根据需要为每个路径完成“如何定义路径选择器”过程。

如何将工步插入现有更改管理工作流程

按照以下步骤将工步插入现有更改管理工作流程：

1. 执行“如何定义更改管理工作流程" 过程.

或

选择现有更改管理工作流程实例。

2. 选择两个工步之间要插入工步的路径。应用程序将突出显示该路径。
3. 单击移除项。应用程序将移除该路径。
4. 展开要作为工步插入的更改管理规范或更改管理工作流程以查看所有修订版本。
5. 选择要插入的修订版本，然后将其拖至更改管理工作流程图。该工步将显示在被放置的位置。
6. 选择“来源工步”上的适当路径图标，然后将其拖至新插入工步。应用程序将添加该路径。
7. 选择新插入工步上的相应路径图标，然后将其拖至下一相应工步。
8. 为要从新插入的工步绘制的任何其他路径重复步骤 7。
9. 如果需要，绘制从现有工步到新插入工步的相应路径。
10. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何删除路径选择器语句

按照以下步骤删除路径选择器语句：

1. 执行“如何定义更改管理工作流程" 过程.

或

选择现有更改管理工作流程实例。

2. 选择要为其删除路径选择器语句的工步。应用程序将在“更改管理工作流程图”工具栏上显示“修改元素”按钮。
3. 单击修改元素。 将显示“更改管理工步详细信息”弹出窗口。
4. 在“路径选择器”表格中选择要删除的路径选择器语句。
5. 单击表格中的删除所选行。应用程序即会从该表格中删除选定行。
6. 单击确定。 将关闭“更改管理工步详细信息”弹出窗口。
7. 单击保存。 应用程序保存建模对象并显示成功消息。
