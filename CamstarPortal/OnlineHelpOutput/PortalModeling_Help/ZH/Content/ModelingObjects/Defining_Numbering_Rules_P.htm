定义编号规则

编号规则是一种可配置的编号方案，它为质量记录和容器指派唯一的追踪编号。通常，组织会为这些实体定义和指派编号方案。应用程序应用适当的编号方案，并将下一个可用的 ID 或编号指派给相应的质量记录或容器。

在为质量记录定义编号规则时

Siemens 建议您为每种类型的质量记录（事件或不合格品）定义唯一的编号规则，从而每个记录类型都具有唯一的标识。应用程序使用在组织建模对象上指定的编号规则向质量记录指派标识。

编号规则是组织建模定义中的一个可选字段。

有关记录事件和不合格品的信息，请参考《Opcenter Execution Medical Device and Diagnostics Event Management 用户指南》或《Opcenter Execution Core Event Management 用户指南》，以及《Opcenter Execution Medical Device and Diagnostics Nonconformance Management 用户指南》或《Opcenter Execution Core Nonconformance Management 用户指南》。

在为容器定义编号规则时

每个容器都必须具有唯一的容器名称。可以手动生成容器名称，也可以通过指派给建模对象的编号规则自动生成。可以使用“编号规则”建模页面来创建编号规则。可以通过从以下一个或多个建模对象上的“编号规则”可选字段中选择编号规则来进行指派：

• 容器级别
• 制造订单
• 产品
• 产品系列
• 工厂
Note: 上面的列表反映了选择首选项的顺序。例如，如果同时在容器级别和产品级别配置了编号规则，应用程序将选择容器级别的编号规则。

车间用户可以选择在启动父容器和子容器、拆分容器和拆分数量时自动生成容器名称。有关自动生成容器名称的信息，请参考《Opcenter Execution Medical Device and Diagnostics Shop Floor 用户指南》或《Opcenter Execution Core Shop Floor 用户指南》。

在选择编号规则类型时

在定义编号规则时，必须选择编号规则类型。有两种类型，每种类型都有优缺点。两种编号规则类型为：

• 高容量/跳过
• 低容量/不跳过

保存实例后，不能更改编号规则类型。单击“保存”按钮时，应用程序将禁用“编号规则类型”字段。通过单击“复制”按钮创建新实例时，“编号规则类型”字段将保持禁用状态。

高容量/跳过功能

在为高容量项目创建编号规则时，如果不考虑严格遵守顺序编号，则使用“高容量/跳过”编号规则类型。

优点

“高容量/跳过”编号规则类型的优点包括：

• 允许跳过序列号
• 用于对编号规则的高容量访问
• 最大程度降低访问冲突的可能性
• 可以修改上次指派的序列号

缺点

“高容量/跳过”编号规则类型的缺点包括：

• 无法保证顺序序列号
• 在需要顺序序列号时不推荐使用

推荐使用

“高容量/跳过”编号规则类型最适用于对编号的高容量请求，建议用于物料容器自动编号。

低容量/不跳过功能

在为低容量项目创建编号规则时，如果严格遵守顺序编号非常重要，则使用“低容量/不跳过”编号规则类型。

优点

“低容量/不跳过”编号规则类型的优点包括：

• 不允许跳过序列号
• 可以修改上次指派的序列号

缺点

“低容量/不跳过”编号规则类型的缺点包括：

• 存在访问冲突的可能性
• 对于编号规则中的自动编号高容量请求不建议使用

推荐使用

“低容量/不跳过”编号规则类型最适用于对自动编号的低容量请求，建议用于质量记录（事件）。

前缀和后缀格式要求

数字不得多于 30 个字符，包括任何前缀或后缀和序列号。生成的数字按如下方式计算：

编号 = 前缀（如有）+ 序列号（必需）+ 后缀（如有）

例如：EVENT-ORG15-00000001-2014

“前缀”和“后缀”字段可用于文本或统一表达式语法条目。

两者都需要非常特定的格式，例如：

• 文本前缀可以定义为 "ABCCorp_" 或 "_Smith"。需要在条目两边加双引号。
• 统一表达式语法可用于创建动态前缀或后缀，例如：
• 前缀 = "(<Organization.Name>"
• 后缀 ="_"+StrSubstring(String(TruncateTime(GMTToLocal(SetSystemDateGMT()))),(StrLength(String(TruncateTime(GMTToLocal(SetSystemDateGMT()))))-4),4)

这将生成以下方案：Event (Corporate00001_2014 started on 02/06/2014 3:21:00 PM。

有关统一表达式语法的信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。

基于规则的序列号与基于前缀的序列号

所有编号规则都包含一个序列号。默认情况下，每次使用某个规则来生成新值时，应用程序都会递增该规则的“上次指派的序列号”值。通过在“编号规则”建模页面上选中“使用基于前缀的上次指派的序列号”复选框，可以将此基于规则的序列号替代为基于前缀的序列号。

基于前缀的序列号适用于使用统一表达式配置的编号规则，因为这些编号规则会动态生成前缀。编号规则首次生成唯一前缀时，应用程序会指派起始序列号值。如果编号规则随后生成同一个前缀，则在生成容器名称时，应用程序会递增该前缀的上次指派的序列号。

Important: 理论上来说，任何编号规则都可能会生成与另一个编号规则相同的前缀。无论基于前缀的序列号由哪个规则生成，应用程序都会递增该序列号。这使得可以使用统一表达式为多个制造订单和产品等配置一个规则，以在前缀中动态标识产品和制造订单名称等信息。

如何使用默认序列号或基于规则的序列号定义编号规则

按照以下步骤使用默认的基于规则的序列号定义编号规则：

1. 打开编号规则页。随即“建模”页中将显示“编号规则”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在编号规则字段中输入编号规则的名称。
4. 选择编号规则类型。
5. 在上次指派的序列号字段中输入值。
6. 在序列号长度字段中输入值。
7. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
8. 单击保存。 应用程序保存建模对象并显示成功消息。

如何使用基于前缀的序列号定义编号规则

按照以下步骤使用基于前缀的序列号定义编号规则：

1. 打开编号规则页。随即“建模”页中将显示“编号规则”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在编号规则字段中输入编号规则的名称。
4. 选择编号规则类型。
5. 在前缀字段中输入统一表达式。
6. 选中使用基于前缀的上次指派的序列号复选框。应用程序将“前缀”设为必填字段，并禁用“上次指派的序列号”字段。
7. 在序列号长度字段中输入值。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击保存。 应用程序保存建模对象并显示成功消息。
