定义用户标签

用户标签是应用程序用户界面上的文本。标签包括字段标题和系统消息等。Siemens 提供默认系统标签。这些标签由 Opcenter EX MDD 和 Opcenter EX CR 中的应用程序使用。

与大多数应用程序配置一样，可以在不更改原始定义的情况下修改系统标签。还可以创建自己的标签（用户标签）作为实例数据，并通过门户方便地执行此操作。

可以将标签名称或标签 ID 传递给应用程序的服务或 XML 客户端。

该应用程序包括“用户标签”建模对象的一个子类：消息类别。有关“消息类别”建模对象的信息，请参考“定义消息类别”。

在转换系统和用户标签时

如果要转换系统和用户标签，可以定义“字典”建模对象来组织转换。字典是指派给员工的自定义标签的集合。

当具有指定字典的员工登录 Portal 时，呈现给该员工的用户界面将显示字典中包含的相应标签。如果员工没有指定的字典，则会显示默认标签。

Note: 有关“字典”建模对象的信息，请参考“定义字典”。

特定语言标签和特定行业标签

可以创建属于以下组的标签：

• 属于词汇表的特定语言标签。

在此组中，可以创建特定语言标签，也可调用 Siemens 标签用以转换。

• 特定行业的术语特定标签。

例如，应用程序所称的容器在工厂中可被称为批次或分选；员工在其他地方可被称为制造同事。特定术语的标签有助于多分部业务的确立，这些分部中的行业术语不能应用于所有分部。

定义了必需的标签后，可以将其添加到字典中。有关信息，请参考“定义字典”。

如何定义用户标签

按照以下步骤定义用户标签：

1. 打开用户标签页。随即“建模”页中将显示“用户标签”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在用户标签字段中输入用户标签的名称。
4. 输入标签值，此值将显示在用户界面上。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 单击保存。 为标签生成一个标签 ID。
