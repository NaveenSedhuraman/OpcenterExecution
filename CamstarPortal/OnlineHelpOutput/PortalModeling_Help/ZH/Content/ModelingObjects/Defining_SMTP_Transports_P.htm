定义 SMTP 传输

SMTP 传输是数据传输的一个子类。应用程序使用简单邮件传输协议 (SMTP) 发送电子邮件。这样当应用程序配置为以 Windows 服务来运行时便能够发送电子邮件。SMTP 虚拟主机是 Windows Server 上 IIS 6.0 组件服务的一部分。

Note: 要链接到 Exchange Mail 服务器，必须输入用户名、密码和 URL（与 SSL 证书匹配的主机服务器的名称）。“用户名”(User Name) 和“密码” 要求由您的电子邮件服务器决定。

在定义 SMTP 传输时

“SMTP 传输”是以下建模定义中的可选字段：

• 企业
• 工厂
• 组织
• 位置
Note: 如果未收到预期的电子邮件，则可能需要将 SMTP 服务器配置为允许匿名用户连接到 SMTP 服务器。

如何定义 SMTP 传输

按照以下步骤定义 SMTP 传输：

1. 打开 SMTP 传输页。随即“建模”页中将显示“SMTP 传输”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在 SMTP 传输字段中输入 SMTP 传输的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。
