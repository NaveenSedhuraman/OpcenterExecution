定义打印队列

“打印队列”是标识将要用于打印容器或不合格品标签的打印机网络位置的对象。

在定义打印队列时

“打印队列”页包含将模板写入文件而不是将其发送到打印机的选项。如果您选择该选项，则模板信息将会根据您指定的目录和基本文件名写入 .txt 文件。该文本文件本身具有以下预定义的命名约定：

<base file name>_<name of transaction>_<date stamp>_<time stamp>_<thread ID>_ <process ID>_<label count>.txt

例如：假设打印文件字段包含条目 server2\myfile 并且将要打印“开始”事务的一个标签。存储在 server2 上的结果文件可以按如下方式命名：

myfile_Start_20061114_225850513_2332_1944_1.txt

定义打印队列时，可以指定标签编码类型。可能的选择包括 ASCII、UTF-16 BE、UTF-16 LE 和 UTF-8。选择与要向其发送输出的打印机匹配的编码类型。

选择“输出到文件”选项将导致应用程序将模板编码到文件。如果模板文件是 ANSI 格式，则选择“输出到文件”选项将导致应用程序使用“标签编码”字段中选定的格式对模板文件进行编码。如果模板文件不是 ANSI 格式，则应用程序会使用 UTF-16 LE 格式对文件进行编码。

定义打印队列时，注意以下事项：

• 不可在对象名中使用无效字符。例如，不可使用撇号。
Note: 有关有效字符和保留字的信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。
• “打印队列”是“资源”和“作业”定义中的可选字段。

如何定义打印队列

按照以下步骤定义打印队列：

1. 打开打印队列页。随即“建模”页中将显示“打印队列”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在打印队列字段输入此打印队列的名称。
4. 在打印队列路径字段输入想在其上打印容器标签的网络打印机位置的路径。
5. 是否想将模板数据写入文件而不是打印模板数据？
