定义角色

角色定义具有一组固有权限的工作职能，这些权限用于授权访问门户的全部或部分内容。门户管理员将角色指派给员工，以授权他们访问执行其工作所需应用程序的某些部分。例如，可以定义“制造班次主管”角色，并授予该角色启动容器的权限。

了解角色

角色由权限和权限访问模式的集合组成。权限是用于执行操作的授权；例如，用于启动容器或发起请求的权限。也可以将权限视为事务或页。

访问模式指定指派的权限的授权级别。例如，如果某个用户需要执行“开始”事务，则该用户必须与包含“开始”事务的权限且具有为该权限指派的“执行”访问模式的角色关联。默认情况下，在指派权限时，应用程序会指派所有访问模式。

角色仅对在制造领域内将其指派给的组织有效。如果某个实体没有组织关联（如事务），则在将角色指派给用户时不必指定组织。此外，仅当管理员将角色传播到子组织时，角色才对这些子组织有效。

Note: 系统不会自动将应用程序的扩展（如新页）的权限添加到默认服务。必须手动为这些角色授予权限。

在定义角色时

角色包含可选的建模定义：

• 员工
• 组织

“角色”是以下建模定义中的必填字段：

• 审批模板 - 在添加审批者时
• 电子签名要求* - 在添加电子签名要求时
• 归类规范 - 在添加归类项时

“角色”是以下建模定义中的可选字段：

• 更改管理规范*
• 电子邮件通讯组
• 员工
• 电子签名角色组
• 组织

*星号表示对象需要单独许可的模块。

角色类型用于指定正在定义的角色的类型：更改管理、质量或车间。例如，可以将默认角色“包创建者”的角色类型设为“更改管理”并将“默认质量”的角色类型设为“质量”。

Note: 如果要按角色类型过滤角色选择列表，可以在 Designer 中更新查询。通过全球技术支持中心 (GTAC) 与产品支持联系，以获取相关信息。

在指派权限时

从“权限类型过滤器”字段中选择某个权限类型时，会更新“指派的权限”表格和“可用权限”表格，以仅显示指定权限类型的权限。应用程序会在当前实例中维护为权限类型指派的所有权限，并在用户查看之前选择的权限集时显示这些指派的权限和可用权限。仅当用户单击“保存”时，应用程序才会将权限保存到角色定义。

例如，选择“容器”权限类型时，会更新“可用权限”表格，以显示与容器相关的事务。如果将某些权限指派给角色，然后选择一种不同的权限类型，则应用程序会更新“可用权限”表格，以仅显示与该权限类型相关的权限。应用程序会从“指派的权限”表格中清除所有与容器相关的权限，但不会将其删除。当用户重新选择“容器”权限类型时，应用程序会显示所有与容器相关的指派。

可以单独选择要指派的一个或多个可用权限，也可以一次选择要指派的所有可用权限。选中“可用权限”表格的标题行的复选框时，会自动选择所有显示的权限。“指派的权限”表格的功能与之类似，使用户可以单独移除指派的权限，也可以一次移除所有指派的权限。

Note: 无论 Portal Studio 中的全局表格设置为何，“指派的权限”表格都会以不分页的形式显示所有可用权限。有关信息，请参考《Opcenter Execution Core Portal Studio 用户指南》或《Opcenter Execution Medical Device and Diagnostics Portal Studio 用户指南》。

在管理角色时

可以在两个建模页（即“角色”和“员工”）上管理角色成员资格。如果在“角色”页上对角色成员资格进行更改，则应用程序会在“员工”页上为相关员工显示更新后的角色成员资格信息。同样，如果在“员工”页上更新角色成员资格信息，则在“角色”页上显示该角色时，应用程序会显示更新后的信息。

角色和电子签名

在为车间和质量记录事务实施电子签名时，角色是必需的建模对象。电子签名等效于手写签名，表示对特定事务的绩效负有责任。可以在制造审核追踪中查看容器或质量记录的已记录电子签名。有关实施电子签名的信息，请参考“用于建模的电子签名简介”。

Note: 电子签名功能是单独许可的模块。

如何定义角色

按照以下步骤定义角色及其权限：

1. 打开角色页。随即“建模”页中将显示“角色”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在角色字段中输入名称。
4. 从权限类型过滤器字段中选择权限类型。“可用权限”表格将会更新，以显示指定权限类型的权限。
5. 使用以下其中一种方法来选择要指派的权限：
• 选中“可用权限”表格的标题行中的权限名称复选框，以选择所有权限。
• 选中各个权限对应的复选框。
6. 单击添加以指派已选权限。系统会将这些权限添加到“指派的权限”表格，并且默认情况下会选择所有访问模式。
7. 重复步骤 4-6，为此角色添加其他权限。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将角色指派给员工

按照以下步骤将角色指派给员工：

1. 执行“如何定义角色”流程。

或

选择现有“角色”实例。

2. 在员工角色成员资格表格中单击添加。 将显示新行。
3. 选择员工。
4. 如果需要，选择角色将对员工有效的组织。
5. 如果希望应用程序使角色对指定组织的所有子组织有效，选中传播复选框。
6. 重复步骤 2-5，将此角色指派给其他员工。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
