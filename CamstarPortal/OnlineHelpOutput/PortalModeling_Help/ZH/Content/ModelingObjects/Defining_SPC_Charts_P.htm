定义 SPC 图表

“SPC 图表”建模对象包含显示单个 SPC 图表所需的信息。它描述了图表类型、Statit Software 宏、Statit Software 规则文件、失控操作和流程 (OCAP)，以及绘制的变量。

Siemens 提供了四个默认 SPC 图表供用户使用：

• 移动极差
• 极差图
• 试用 I-图
• X 图

在定义 SPC 图表时

必须为要显示的每个图表创建一个具有相应图表生成宏的 SPC 图表。

确保图表宏和图表属性的路径有效。例如，如果 SPC 安装在非默认安装目录的目录中，则需要更改路径。

使用 SPC 图表的“失控操作”部分来确定在违规时触发的操作。

SPC 图表是“SPC 图表组”中的可选建模定义。

SPC 图表包含以下可选建模定义：文档、电子邮件消息、搁置原因和资源状态代码。

有关设置和配置 SPC 的信息，请参考《Opcenter Execution Core Statistical Process Control 参考指南》。

如何定义新的 SPC 图表

按照以下步骤定义新的 SPC 图表：

1. 打开 SPC 图表页。随即“建模”页中将显示“SPC 图表”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在SPC 图表、图表类型、图表变量、生成图表的宏和 SPC 图表显示选项中输入内容。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。
