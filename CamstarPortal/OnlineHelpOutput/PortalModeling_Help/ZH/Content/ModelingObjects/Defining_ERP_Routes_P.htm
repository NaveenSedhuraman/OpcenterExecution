定义 ERP 路线

ERP 路线工步是最接近 Opcenter EX MDD 或 Opcenter EX CR 工作流程的 ERP 概念。Opcenter EX MDD 或 Opcenter EX CR 工作流程和制造订单中都会引用 ERP 路线。工作流程引用 ERP 路线，以使各个工作流程工步可以引用 ERP 路线工步。这通常对 ERP 里程碑或库存点执行。制造订单引用 ERP 路线，以使物料列表项可以引用在其中分发组件的 ERP 路线工步。

在定义 ERP 路线时

ERP 路线是组件分发事务的必需对象，用于将制造订单物料列表项链接到工作流程。

ERP 路线用于将 Opcenter EX MDD 或 Opcenter EX CR 事务的“移动”和“组件分发”与发生事务的 ERP 工步相关联。通常在比 Opcenter EX MDD 或 Opcenter EX CR 工作流程中的工步更汇总的级别定义 ERP 路线中的工步。

ERP 路线包含可选建模定义“产品”。

ERP 路线是“制造订单”定义中的可选字段。

定义用于 Opcenter Execution Core Scheduling 的 ERP 路线
Note: 仅当安装有 Opcenter Execution Core Scheduling 时，下述功能才可用。有关信息，请参考 "Opcenter Execution Core Scheduling"。

必须为处理制造订单或产品的每个 ERP 作业提供处理时间。可以使用“ERP 路线”页上的“路线工步”表格，采用以下方法定义产出率：

• 每单元小时数
• 每小时单元数
• 每批次时间

只能选择一个选项来定义产出率。

必须为工作流程指派 ERP 路线。然后可以将该工作流程指派给产品或产品系列。

为路线工步建立准备时间
Note: 仅当安装有 Opcenter Execution Core Scheduling 时，下述功能才可用。有关信息，请参考 "Opcenter Execution Core Scheduling"。

可以定义在路线工步中准备处理资源所需的时间量。例如，需要在生产间隔中清理加工设备。清理过程花费三十分钟完成。可以在“准备时间”字段中指示此时间。

如何定义 ERP 路线

按照以下步骤定义 ERP 路线：

1. 打开 ERP 路线页。“ERP 路线”页显示在“建模”页中。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在 ERP 路线字段中输入 ERP 路线的名称。
4. 在修订版本字段中输入修订版本。
5. 要添加路线工步，请参考“如何将路线工步添加到 ERP 路线”中所述的过程。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将路线工步添加到 ERP 路线

按照以下步骤将路线工步添加到 ERP 路线：

1. 执行“如何定义 ERP 路线”中所述的过程。

或

选择现有 ERP 路线实例。

2. 展开详细信息部分。
3. 单击“路线工步”表格上的添加新行。 将显示新行。
4. 在路线工步名称字段中输入该路线工步的名称。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 重复步骤 3-5 添加其他路线工步。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
