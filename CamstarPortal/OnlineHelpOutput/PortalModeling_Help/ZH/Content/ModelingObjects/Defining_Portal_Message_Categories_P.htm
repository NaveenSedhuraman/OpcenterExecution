定义门户消息类别

Siemens 提供了具有 Portal 安装的单个“门户消息类别”建模对象实例。通过“门户消息类别”建模对象，可以自定义以下内容：

• 在“任务中心”和“消息中心”显示的消息类别
• 在消息类别中显示的通知类型
• 指派给消息类别的图标
Note: 图标指派仅在使用经典主题时才可用。
• 质量项的到期日期指示器
Note: 有关任务中心和消息中心的信息，请参考以下内容：
• Opcenter Execution Medical Device and Diagnostics Event Management 用户指南或 Opcenter Execution Core Event Management 用户指南
• Opcenter Execution Medical Device and Diagnostics Nonconformance Management 用户指南或 Opcenter Execution Core Nonconformance Management 用户指南
• 可通过质量记录页面获得的联机帮助

门户只能存在一个消息类别列表。不能删除或复制此建模对象实例。

在编辑门户消息类别实例时

“门户消息类别”包含所需的建模对象，消息类别。Siemens 提供了四个在默认情况下指派给此实例的消息类别，但您可以定义要添加的其他消息类别。有关定义消息类别的信息，请参考“定义消息类别”。

Important: 向“任务中心”添加不超过六个“消息类别”以获得最佳性能。

可以与消息类别关联的通知类型会在安装过程中随应用程序一起提供，且无法更改。可以将一个或多个通知类型与消息类别关联。应用程序要求您至少将一个通知类型与消息类别关联，然后应用程序才会在“消息中心”或“任务中心”显示消息类别。

在管理消息类别时

“消息类别定义”表格显示“任务中心”和“消息中心”中可用的消息类别。“消息类别定义”表格中的消息类别的顺序决定了它们将在“任务中心”和“消息中心”显示的顺序。使用表格执行以下操作：

• 添加或移除消息类别
• 更改消息类别显示的位置
• 更改消息类别的顺序
• 自定义每个消息类别的设置，包括：
• 更改与消息类别关联的图标
Note: 图标指派仅在使用经典主题时才可用。
• 将通知类型添加到/移除出消息类别

定义黄色最小和最大范围

“门户消息类别”页上的黄色最小和最大范围字段确定到期日期指示器（绿色圆圈、橙色圆圈和红色圆圈）何时出现在“任务中心”和“消息中心”项旁边。下面的信息在定义到期日期指示器的最小和最大范围时可能会有帮助。

到期日期范围指示器字段的规则

确定到期日期范围指示器的规则包括：

• 黄色最小和最大范围字段中的值必须是整数。
• “黄色最小范围（天）”必须小于“黄色最大范围（天）”。
• “黄色最小范围”是橙色指示器要显示的到期日期的最小天数。默认值为 0。如果活动的日期差异小于此天数，则显示红色圆圈。
• “黄色最大范围”是橙色指示器要显示的到期日期的最大天数。默认值为 2。如果活动的日期差异大于此天数，则显示绿色圆圈。
• 在到期日期结束之前，应用程序不会显示任何红色指示器。

如何计算优先级指示器

此信息解释如何计算优先级指示器。假设：

• 黄色最小范围 = n
• 黄色最大范围 = m
• m => n

据此：

• 红色：（到期日期 - 今天）< n
• 橙色：n <=（到期日期 - 今天）<= m
• 绿色：m <（到期日期 - 今天）

如何选择门户消息类别

按照以下步骤选择门户消息类别：

1. 打开门户消息类别页。随即“建模”页中将显示“门户消息类别”页。
2. 选择门户消息类别实例。将显示实例的字段和值。

如何编辑消息类别

按照以下步骤编辑“门户消息类别”列表中的“消息类别”：

1. 执行“如何选择门户消息类别”过程。

或

选择现有门户消息类别实例。

2. 单击编辑所选行编辑现有消息类别的属性。将显示“编辑”弹出窗口。
3. 执行以下一项或多项操作以修改消息类别设置：
• 从消息类别列表中选择类别，以更改现有类别。
• 选中或清除显示在消息中心或显示在任务中心复选框。
• 单击“通知类型”表格上的添加新行按钮，选择要与消息类别关联的通知。 将显示新行。
• 选择一个行，然后单击“通知类型”表格上的删除所选行，从消息类别中移除通知类型。
4. 执行下述中的一项操作：

单击确定。 “门户消息映射详细信息”弹出窗口关闭。

或

单击关闭以进行关闭而不保存。“编辑”弹出窗口关闭。

5. 单击保存。 应用程序保存更改，并显示一条消息，表明“门户消息类别”已更新。

如何将类别添加到消息类别定义表格

按照以下步骤将新类别添加到“消息类别定义”表格中：

1. 执行“如何选择门户消息类别”过程。

或

选择现有门户消息类别实例。

2. 单击“消息类别定义”表格上的添加新行按钮。将显示“添加”弹出窗口。
3. 从列表中选择消息类别。
4. 如果希望此类消息出现在“消息中心”显示中，选择显示在消息中心复选框。
5. 如果希望此类消息出现在“任务中心”显示中，选择显示在任务中心复选框。
6. 单击“通知类型”表格上的添加新行，然后从通知类型列表中选择通知类型。
7. 重复步骤 7 以关联其他通知类型。
8. 单击确定。 “添加”弹出窗口将关闭，新消息类别将显示在“消息类别定义”表格中。
9. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何从列表中删除消息类别定义

按照以下步骤从列表中删除消息类别：

1. 执行“如何选择门户消息类别”过程。

或

选择现有“门户消息类别”实例。

2. 在“消息类别定义”表格中选择一行。
3. 单击表格页脚中的删除所选行以移除所选消息类别。
4. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何更改任务中心和消息中心的排序顺序

按照以下步骤更改“任务中心”“消息中心”的排序顺序：

1. 执行“如何选择门户消息类别”过程。

或

选择现有门户消息类别实例。

2. 在“消息类别定义”表格中选择一行。
3. 使用表格页脚中的“移动行”箭头，将所选行移动到列表中的适当位置。
4. 重复此过程，直到类别以适当的顺序显示。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何定义到期日期指示器的范围

按照以下步骤定义消息到期日期指示器的范围：

1. 执行“如何选择门户消息类别”过程。

或

选择现有门户消息类别实例。

2. 展开到期日期范围部分。
3. 在黄色最小范围（天）字段中输入数字。
4. 在黄色最大范围（天）字段中输入数字。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
