定义汇总表和视图

“汇总表定义”建模对象用于配置视图和汇总表。视图和汇总表将以合乎逻辑的汇总方式组织数据。Siemens 将使用这些定义来进行业务智能 (BI) 报告。

汇总表

汇总表是被永久存储并在运行时读取的数据的相邻字段的集合。它可以保持不变、添加内容或进行完整刷新。

可以手动执行汇总表，也可将其配置为自动运行。如果指定为手动执行汇总表，则应用程序将禁用之前为汇总表定义的任何重复模式设置。

视图

视图是一种描述数据库中信息的方式。通常会将视图视为虚拟表，实际上它本身并不存储信息，而是将从一个或多个现有表中提取信息。

虽然视图不是永久的，但是可以通过将其准则存储在查询中来重复访问视图。

必须手动执行视图。

配置汇总表和视图

在配置汇总表定义时，记住以下原则：

• 只有数据库管理员才能从数据库中移除视图或汇总表。
• 虽然定义在默认情况下使用 DataStore，但是可以将其重新配置为指向事务数据库或同时指向这两者。然而，指向事务数据库或这两个数据库可能会大幅降低系统性能。
• 大多数汇总表的默认重复模式是每 24 小时运行一次，从午夜（零点）开始。csiProductionDashboard 和 csiYield 汇总表的默认重复模式是每四小时运行一次，从午夜（零点）开始。
• 重复计划的选择是递增的。因此，在进行选择时务必要谨慎。例如，如果最初选择的是星期一和星期六，汇总表将仅在星期一和星期六运行。如果随后决定将月份的第 1 天添加为“每月特定天”，则汇总表将仅在月份的第一天为星期一或星期六的日期运行。
• 测试 SQL 查询时，仅返回 100 行以供审核。
• 在“视图”或“表格名称”字段中输入的视图或表格名称必须唯一。
• 可以使用“建模审核追踪”来查看此建模对象的历史。

创建查询

创建使用 LastRunDate 参数的查询时，必须指示它是针对 SQL 数据库还是 Oracle 数据库运行。

数据库

值

SQL

@LastRunDate

Oracle

:LastRunDate

示例

此示例显示针对 SQL 数据库运行的查询。

SELECT productid,count(*) CNT

FROM container

WHERE productid IS NOT NULL

AND (originalstartdate >= @LastRunDate OR @LastRunDate IS NULL)

GROUP BY productid

Important: 每次修改 SQL 语句时，都必须选中“强制刷新”复选框，然后单击“保存并手动执行”按钮以提交更改。此操作同时适用于表和视图。

执行汇总表

执行汇总表时，记住以下原则：

• 如果决定手动执行以前计划的汇总表，则必须选中“手动执行”复选框。这会导致“计划”部分消失。然后，必须单击“保存并手动执行”按钮来执行该表。
• 如果决定重新计划该表，则取消选中“手动执行”复选框。这将导致“计划”部分与原始设置一起重新显示。
• 单击“保存并手动执行”按钮不会像“保存并测试查询”按钮一样验证查询。

如何定义汇总表

按照以下步骤定义汇总表：

1. 打开汇总表定义页。随即“建模”页中将显示“汇总表定义”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在汇总表定义字段中输入名称。
4. 在查询文本字段中输入查询文本。
5. 展开计划部分。
6. 选择汇总表。该页将显示特定于汇总表的复选框和重复模式字段。
7. 在视图或表格名称字段中输入汇总表的名称。
8. 是要手动执行汇总表查询还是将其配置为自动运行？

如果您想完成以下操作 ...

操作步骤 ...

手动执行汇总表查询
a. 选中手动执行复选框。将隐藏重复模式字段。
b. 转到步骤 9。

将汇总表查询配置为自动运行
a. 选中启用重复复选框。
b. 使用重复模式和重复范围选项来配置运行次数。
c. 转到步骤 9。
9. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
10. 单击保存。 应用程序保存建模对象并显示成功消息。

如何定义视图

按照以下步骤定义视图：

1. 打开汇总表定义页。随即“建模”页中将显示“汇总表定义”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在汇总表定义字段中输入视图的名称。
4. 在查询文本字段中输入查询文本。
5. 展开计划部分。
6. 选择视图。将隐藏重复模式信息。
7. 在视图或表格名称字段中输入名称。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击保存。 应用程序保存建模对象并显示成功消息。

如何测试查询

按照以下步骤测试汇总表定义查询：

1. 执行下述中的一项操作：
• 执行“如何定义汇总表”过程。
• 执行“如何定义视图”过程。
• 选择现有的汇总表定义实例。
2. 单击保存并测试查询。将显示“用户查询维护测试”弹出窗口，并显示查询的状态消息或结果。
3. 单击关闭以关闭“用户查询维护测试”弹出窗口。
4. 如果需要，再次修改和测试查询，直到实现所需的结果。
Important: 如果汇总表查询需要很长时间才能运行，则它可能会因为传入事务在 SQL Server 中遇到表锁定错误。可以修改汇总表 SQL，以在发生此情况时不锁定查询。

如何手动执行汇总表或视图

按照以下步骤手动执行汇总表或视图：

1. 执行下述中的一项操作：
• 执行“如何定义汇总表”过程。
• 执行“如何定义视图”过程。
• 选择现有的汇总表定义实例。
2. 展开计划部分。
3. 单击保存并手动执行。应用程序保存建模对象并显示成功消息。
