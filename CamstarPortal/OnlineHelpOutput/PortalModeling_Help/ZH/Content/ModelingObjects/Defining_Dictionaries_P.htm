定义字典

使用 Opcenter EX MDD 和 Opcenter EX CR 可为应用程序创建自定义或转换的用户标签和系统标签。应用程序中的标签是一些消息和文本，如显示的字段标题、错误消息、事务完成消息和字符串选择值等。系统使用字典来对自定义标签和转换的标签进行组织并将其指派给员工。

可以按以下两种类型的字典对自定义或转换的标签进行分组：

• 语言 - 存储用于特定语言的转换的标签。
• 术语 - 存储用于行业特定语言的自定义标签。例如，Opcenter EX MDD 和 Opcenter EX 使用词语“容器”来表示工厂中生产的产品的离散集合，并使用“员工”来代表人员。您可能想在您的业务中使用不用的词，例如，用“分选”代替“容器”，用“同事”代替“员工”。

应用程序根据与登录的员工关联的字典显示标签。如果员工没有指定的字典，则会显示默认标签和用户定义的标签。有关为员工指派字典的信息，请参考“定义员工”。

应用程序允许为员工同时指派语言字典和术语字典。如果标签同时存在于术语字典和语言字典中，则术语字典中的标签优先于语言字典中的字典。

字典标签在应用程序服务器和门户应用程序缓存中都会进行缓存。Opcenter EX MDD 和 Opcenter EX CR 字典标签存储在数据库中，并缓存在门户的 IIS 缓存中。必须回收 IIS 应用程序池才能刷新缓存。更新字典建模实例时，将更新字典标签缓存。

保存字典建模实例时，如果保存包含更改，标签缓存将自动即时刷新。

在员工定义中指定这些字典类型或类别，将字典分配给员工。

可以选择在 Management Studio 中使用自定义 CSV 文件加载字典标签。有关加载自定义字典的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

字典包含可选建模定义“用户标签”。

如何定义字典

按照以下步骤定义字典：

1. 打开字典页。随即“建模”页中将显示“字典”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在字典字段中输入字典的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何编辑字典中的标签

按照以下步骤编辑字典中的标签：

1. 执行“如何定义字典”过程。

或

选择现有字典实例。

2. 编辑此字典中的标签，如下所示：

如果您想完成以下操作 ...

操作步骤 ...

编辑 Opcenter EX MDD 或 Opcenter EX CR 标签
a. 从“类别过滤器”列表中选择带有前缀 csi 的类别过滤器，将搜索范围缩小到 Opcenter EX MDD 或 Opcenter EX CR 标签。还可以在标签名称过滤器中输入部分用户标签名称，或在标签值过滤器中键入部分标签值。
b. 转到步骤 3。

编辑您创建的用户标签
a. 在标签名称过滤器中键入用户标签名称或在标签值过滤器中键入标签值，将搜索范围缩小到包含输入文本的用户标签。请参考“如何定义用户标签”。
b. 转到步骤 3。
3. 单击搜索。“搜索结果”表格显示所需要的标签。
4. 在表格中选择一个标签，然后单击编辑所选行。将显示“编辑”弹出窗口。
5. 在标签文本字段中输入标签的转换。
6. 单击确定。 “编辑”弹出窗口关闭，新标签文本显示在“搜索结果”表格中。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
