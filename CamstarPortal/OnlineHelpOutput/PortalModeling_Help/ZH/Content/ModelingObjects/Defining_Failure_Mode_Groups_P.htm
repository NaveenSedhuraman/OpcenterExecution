定义失败模式组

失败模式组建模对象可用于定义一组失败模式。它使用户可以对表示实际失败原因的失败模式进行分组，简化了选择失败模式的过程。

可以在“组织”页上将失败模式组与事件分类和子分类相关联。

在定义失败模式组时

可以将多个条目或组添加到各自的表格。

使用“解决条目”按钮可以显示在此组和所有嵌套组的条目列表中指定的所有值的列表。

失败模式组包含可选建模定义“失败模式”。

如何定义失败模式组

按照以下步骤定义失败模式组：

1. 打开失败模式组页。随即“建模”页中将显示“失败模式组”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在失败模式组字段中输入此失败模式组的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将条目添加到失败模式组

按照以下步骤将条目添加到失败模式组：

1. 执行“如何定义失败模式组”中所述的过程。

或

选择现有失败操作类型组实例。

2. 单击“条目”表格中的添加新行。 将显示新行。
3. 单击空行，然后从可用失败模式的列表中选择失败模式。
4. 对要添加到该组的每种失败模式重复步骤 2-3。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何将组添加到失败模式组

按照以下步骤将其他失败模式组添加到失败模式组：

1. 执行“如何定义失败模式组”中所述的过程。

或

选择现有失败操作类型组实例。

2. 单击“组”表格中的添加新行。 将显示新行。
3. 单击空行，然后从可用组的列表中选择失败模式组。
4. 对要添加到该组的每个失败模式组重复步骤 2-3。
5. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
