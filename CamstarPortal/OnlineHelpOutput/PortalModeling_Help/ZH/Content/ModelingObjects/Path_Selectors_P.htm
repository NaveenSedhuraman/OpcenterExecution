使用路径选择器

路径从一个工步指向下一个工步。多数情况下，工作流程工步间的容器可能有多条路径。路径选择器基于指定的条件提供这些备选路径。这意味着路径选择器是默认路径的替代形式，在沿着默认路径继续前进之前，应用程序将检查路径选择器。

每个路径选择器语句包含一个表达式，针对与容器相关联的值来判断此表达式的值。路径选择器基于具有布尔返回值的表达式。如果表达式为真，则沿着指定的备选路径移动容器。

需记住的要点

• 表达式的处理要区分大小写。
• 忽略所有的空格符。
• 字符串必须在双引号内。
• 通过二进制数据类型处理带小数点的数。
• 处理顺序是基于运算符优先级的。
• 支持括号。它们能用于替代运算符优先级。
• 算术表达式语义取决于操作数类型。整数运算应用于整数操作数。如果运算既有整数操作数又有带小数点的操作数，则应用小数点算术。例如，1/4 的计算结果为 0，而 1/4.（注意 1/4 后的小数点）判断为 .25。

路径选择器语句示例

以下是路径选择器语句的示例以及应用程序判断它们的方式：

• Container.CurrentStatus.ReworkStatus[0].ReworkReason = "SomeReworkCode"
这意味着转向容器表的当前状态、查找返工状态 0、返工状态和返工原因，并与所提供的值 (SomeReworkCode) 相比较。
• Not (Container.Product.Name == "SomeProductName")
容器的产品名称不等于“SomeProductName”
• 1 + 2 * 3 == 7
二乘以三等于六（乘法比加法具有更高优先级），再加一，结果等于七。
• (1 + 2) * 3 ! = 8
一加二等于三（即使加法比乘法具有较低的优先级，也要先算第一步，因为加法是括在括号里的），结果（三）再乘以三。结果不等于八。
• 1 < 2
一小于二。
• Container.ChildCount > 0 or Container.Qty > 0
此容器的子容器数大于零，或容器数量大于零。

如果第一个条件 (Container.ChildCount > 0) 的计算结果为 true，则整个表达式的计算结果为 true，因为它已满足其中至少一条件为 True 的要求。如果第一个条件判断为假，则判断第二个条件。如第二个条件为真，则整个表达式为真且容器转向指定的路径。如果第二个条件为假，则整个表达式为假，并判断下一个路径选择器语句。如果这是最后一个语句，则容器继续在默认路径上执行。

• Container.ChildCount > 0 and Container.Qty > 0
容器的子容器数大于零且容器数量大于零。

如果第一个条件 (Container.ChildCount > 0) 的计算结果为 false，则整个表达式的计算结果为 false，因为它不满足两个条件都为 True 的要求。如果第一个条件判断为假，则判断第二个条件。如第二个条件判断为真，则整个表达式为真且容器转向指定的路径。如果第二个条件为假，则整个表达式为假，并判断下一个路径选择器语句。如果这是最后一个语句，则容器继续在默认路径上执行。

• Container.Product.Name="SomeProductName" ? Container.Customer.Name : Container.Owner.Name= "SomeOwnerName"
如果此容器的产品名称等于 SomeProductName，则检查此容器是否有关联的客户姓名；如果此容器的产品名称不是 SomeProductName，则检查此容器的所有者名称是否为 SomeOwnerName。（只要问号后的表达式判断为真，容器就继续在路径选择器语句中定义的路径上执行。如果表达式判断为假，则判断下一个路径选择器语句。如果这是最后一个路径选择器语句，则容器继续在默认路径上执行。）

判断顺序

路径选择器语句由“标准移动”事务判断，返工路径选择器由“返工”事务判断。应用程序按照路径选择器语句在选择器语句表格中出现的顺序对这些语句进行判断。通过使用上下箭头或拖放行来排列路径选择器语句的顺序。

当语句判为真时，此判断结束，容器将沿着指定的“要使用的路径”移动。当语句的计算结果为 false 时，应用程序将执行以下操作之一：

• 计算下一个选择器语句。
• 如果没有任何其他选择器语句需要计算，则沿着工步的默认路径移动容器。（没有任何语句的计算结果为 true。）

如果当前日期超出有效日期范围或状态为不活动，应用程序将不会判断路径选择器语句。

如何定义路径选择器和返工路径选择器

按照以下步骤定义路径和返工路径选择器：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 选择要为其定义路径选择器语句的工步。
3. 单击修改元素。 将显示“工步详细信息”弹出窗口。
4. 要添加路径选择器语句还是返工路径选择器语句？

如果要添加 ...

操作步骤 ...

路径选择器语句

转到路径选择器表格。继续执行步骤 5。

返工路径选择器语句

转到返工路径选择器表格。继续执行步骤 5。
5. 单击添加新行。 将显示新行。
6. 在表达式字段中输入计算语句。
7. 当语句计算为 True 时，选择要使用的路径。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考“路径详细信息弹出窗口字段定义”和“了解工作流”表。
9. 重复步骤 4-8，添加其他路径选择器语句。
10. 通过使用上下箭头或拖放行来排列路径选择器语句的顺序。
11. 单击确定。 “工步详细信息”弹出窗口关闭。
12. 单击保存。 应用程序保存建模对象并显示成功消息。

如何修改路径或返工路径选择器语句

按照以下步骤修改路径或返工路径选择器语句：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 选择要为其修改路径选择器语句的工步。
3. 单击修改元素。 将显示“工步详细信息”弹出窗口。
4. 在“路径选择器”或“返工路径选择器”表格中选择要修改的路径选择器语句。
5. 根据需要修改语句。有关字段的信息，请参考“工步详细信息弹出窗口字段定义”表。
6. 通过使用上下箭头或拖放行来排列路径选择器语句的顺序。
7. 单击确定。 “工步详细信息”弹出窗口关闭。
8. 单击保存。 应用程序保存建模对象并显示成功消息。

如何删除路径或返工路径选择器语句

按照以下步骤删除路径或返工路径选择器语句：

1. 执行“如何定义工作流程”过程。

或

选择一个现有工作流程实例。

2. 选择要为其删除路径选择器语句的工步。
3. 单击修改元素。 将显示“工步详细信息”弹出窗口。
4. 在“路径选择器”或“返工路径选择器”表格中选择要删除的路径选择器语句。
5. 单击删除所选行。随即删除所选行。
6. 单击确定。 “工步详细信息”弹出窗口关闭。
7. 单击保存。 应用程序保存建模对象并显示成功消息。
