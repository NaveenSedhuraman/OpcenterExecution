定义产品

产品是工厂或工厂外的供应商生产的物料。产品可以是成品，配件和组件。每个产品定义包含两类基本信息：描述产品的数据以及提供默认处理信息的数据，该默认处理信息在启动新产品容器时使用。

在产品定义中提供以下数据（如果您的业务规则有此要求）后，无需在每次开始此产品容器时输入这些数据：

• 工作流程
• 开始数量
• 次要开始数量
• 计量单位
• 次要计量单位
• 客户
• 测重器组
Note: “测重器组”字段仅在安装了 Batch Processing 时出现。有关信息，请参考 "Batch Processing"。

这些信息（除“客户”外）将替代在产品系列级上所作的任何设置。（未在产品系列级别设置客户。）

此外，还可以为产品指派自定义的动态属性，应用程序将在启动容器时指派这些用户定义属性。在容器本身中存储信息后，您和其他用户不必在容器的历史中搜索信息。

Portal 用户可以针对产品创建事件。

将产品指派给产品系列

产品可能属于产品系列，用于对共享通用属性（如工作流程、培训要求和启动数量）的产品进行分组。产品系列通过定义一组产品的通用数据来简化产品信息维护。为产品系列定义的属性应用于系列中的每个产品。

产品可以从指派的产品系列继承下列属性：

• 工作流程
• 文档集
• 开始数量（和开始次要数量）
• 开始 UOM（和开始次要 UOM）
• 标准成本
• 当前成本

在“产品”页上为其中某一个属性选择值后，将替代关联的产品系列定义上指定的值。

定义用于 Opcenter Execution Core Scheduling 的产品

为 Opcenter Execution Core Scheduling 定义制造订单时，“产品”是制造订单建模对象上的必填字段。在定义制造订单之前必须先定义产品。

与 ERP 路线关联的工作流程必须指派给制造订单的产品或产品的产品系列。有关信息，请参考“定义用于 Opcenter Execution Core Scheduling 的产品系列”。

产品页上的到期日期计算
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

按照标识的规范成功完成预定义的事务时，可以使用“产品”页定义自动计算容器到期日期所需的字段。这些字段如下所示：

• 到期日期事务
• 到期日期规范
• 到期周期单位
• 到期周期

产品的到期日期信息优先于规范的到期日期信息。

如果为容器定义的到期日期早于计算日期，除非选中“始终替代到期日期”复选框，否则不会更新到期日期。

如果将“到期周期单位”字段设为“月份”，并且到期日期计算出无效的日期，则到期日期将调整为该月的最后一天。例如，如果当前日期为 1 月 31 日，并且到期周期为 3 个月，则计算的到期日期为 4 月 31 日。日期将调整为 4 月 30 日。

产品页上的制造日期计算
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

按照标识的规范成功完成预定义的事务时，可以使用“产品”页定义自动计算容器制造日期所需的字段。这些字段如下所示：

• 制造日期规范
• 制造日期事务

产品上的制造日期信息优先于规范上的制造日期信息。

按照标识的规范成功完成指定事务时，将在处理过程中向容器指派当前日期和时间。

在产品页上定义 UDI
Note: 仅当安装 Medical Device 时下述功能才可用。有关信息，请参考“Medical Device 工作区”。

使用 Opcenter EX MDD 和 Opcenter EX CR 的 UDI 功能可为产品定义以下内容：

• 设备标识符
• 生产标识
• UDI 表达式
• UDI 事务

可以为产品指派多个设备标识。在生产过程中，如果为产品定义了多个设备标识，则用户可以选择要指派给容器的特定设备标识。如果只为产品定义了一个设备标识，则应用程序会自动将该值指派给容器。

为产品指派默认库存位置
Note: 下述功能仅在安装了 Industry Solutions 工作区时可用。有关信息，请参考“Industry Solutions 工作区”。

可以为产品指派默认库存位置。还可以为产品系列和工厂指派默认库存位置。

在指派默认库存位置时，以下规则适用：

• 在产品上指派的默认库存位置优先于在产品系列上指派的默认库存位置。
• 在产品系列上指派的默认库存位置优先于在工厂上指派的默认库存位置。
• 如果在产品上定义了默认库存位置，则当容器移动到定义为库存点的作业时，应用程序会自动为容器指派默认库存位置。

有关信息，请参考“为产品系列指派默认库存位置”和“为工厂指派默认库存位置”。

为组件产品指派测重器组
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

可以使用“测重器组”字段指派测重器组，以定义允许称量组件产品的测重器。还可以将测重器组指派给规范、产品系列和称重分发任务。

在指派测重器组时，以下规则适用：

• 在产品系列上指派的测重器组优先于在规范上指派的测重器组。
• 在产品上指派的测重器组优先于在产品系列上指派的测重器组。
• 在称重分发任务上指派的测重器组优先于在所有其他对象上指派的测重器组。

有关信息，请参考“在规范上指派测重器组”、“在产品系列上指派测重器组”和“在定义称重分发任务时”。

定义表达式以调整产品上的组件数量
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

通过在产品上定义表达式，可以在批次处理过程中调整组件数量。例如，假定批次处理所需的组件产品的数量取决于效力。产品的效力未设置，但因组件容器而异。在 Opcenter EX MDD 或 Opcenter EX CR 中创建组件容器时，将根据来料检验记录其效力。可以定义表达式，计算处理时的所需数量以及最小和最大公差，以说明组件容器之间的效力差异。可以使用产品上的表达式来调整以下值：

• 批次处理所需的组件数量
• 批次处理允许的最小组件数量
• 批次处理允许的最大组件数量

还可以在“任务列表”上的称重分发任务上定义这些表达式。在称重分发任务上定义的表达式优先于在产品上定义的表达式。有关信息，请参考“在定义称重分发任务时”。

定义表达式时使用统一表达式语法。表达式最多可包含 4000 个字符，可能包括：

• 变量；例如，(?InQty)
• 用户定义的常量；例如，Constant::Pi

有关统一表达式语法的信息，请参考《Opcenter Execution Medical Device and Diagnostics Designer 用户指南》或《Opcenter Execution Core Designer 用户指南》。

将成品定义为受控批次产品
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

通过“受控批次”复选框，可以将成品标识为批处理产品。选中“受控批次”复选框表示产品物料清单中每个组件的物料数量是成品批次所需的总数量。组件物料数量不是每个单元所需的数量。

定义产品的 UOM 转换信息
Note: 仅当安装 Batch Processing 时下述功能才可用。有关信息，请参考 "Batch Processing"。

“UOM 转换”表格包含允许定义产品数量转换信息的字段。可以定义：

• 将产品的任何分发量的度量单位转换为另一度量单位的转换系数。
• 转换数量的取整规则。
• 计算的转换数量中允许的小数位数。最大值为 4。

在定义产品的 UOM 转换信息时，以下规则适用：

• 必须填充“取整规则”和“小数位数”字段以进行取整。
• 在产品上定义的转换信息优先于在产品系列上定义的转换信息。
• 在产品系列上定义的转换信息优先于在 UOM 上定义的转换信息。

有关信息，请参考“定义产品系列的 UOM 转换信息”和“定义度量单位”。

如何定义产品

按照以下步骤定义产品：

1. 打开产品页。随即“建模”页中将显示“产品”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在产品字段中输入产品名称。
4. 在修订版本字段中输入此产品的修订版本。
5. 选择产品类型。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将产品添加到产品系列

按照以下步骤将产品添加到产品系列：

1. 执行“如何定义产品”过程。

或

选择现有产品实例。

2. 从产品系列字段中选择产品系列。
3. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何为物料清单组件产品选择有效替代品

按照以下步骤为物料清单组件产品选择有效替代品：

1. 执行“如何定义产品”过程。

或

选择现有产品实例。

2. 展开产品结构部分。
3. 单击“有效替代品”表格上的添加新行。 将显示新行。
4. 选择产品或产品修订版本。应用程序将显示选择。
5. 如果它是记录修订版本，则选中此复选框。
6. 重复步骤 3-5 指定其他替代品。
7. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
