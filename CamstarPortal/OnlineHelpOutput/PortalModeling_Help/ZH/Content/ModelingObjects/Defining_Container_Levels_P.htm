定义容器级别

容器级别是分配给容器的跟踪标志。它也称作生产量层次。常用的层次名称是宗和批。定义层次是为了满足每家公司特定的需求。某些特定行业的层次比如卷、仓以及盒。

Opcenter EX MDD 和 Opcenter EX CR 提供多层级在制品(WIP) 追踪功能，用于收集并报告详细到任何层级的状态系谱和参数信息。Opcenter EX MDD 和 Opcenter EX CR 使用常规术语容器来标识工作单元。您可以决定与作业相匹配的特定术语，例如，批、宗、盘、罐或序列号。由于应用程序支持多个和无限制的容器级别（容器中的容器），因此可以在任何细节级别定义和追踪产品。

很多公司也在更大的父级批次中跟踪较小级别的批次。在 Opcenter EX MDD 和 Opcenter EX CR 中，这通过多层级容器追踪来完成。父级与子级容器通过“关联”或“启动 - 两级”事务相互关联。“启动 - 两级”创建容器，而“关联”建立父级-子级关系。一组批次可以具有一个使用宗层次的父级批次和相关联的使用盘层次的子级批次。批次组也可以有两个以上的层次。

此外，还可以将自定义动态属性指派给容器级别。在容器本身中存储信息后，您和其他用户不必在容器的历史中搜索信息。当启动容器时，应用程序指派用户定义的属性。

指定要评估的 WIP 消息

可以为特定建模对象定义处理中 (WIP) 消息，以便当具有特定属性的材料容器到达某个处理点时，该消息会显示给用户。例如，可以定义当容器到达启动规范以及用户执行移入事务时显示的 WIP 消息。

必须执行以下操作才能配置 WIP 消息：

• 在建模对象上定义 WIP 消息。有关定义和配置 WIP 消息的信息，请参考“定义 WIP 消息”。
• 将希望为其评估消息的建模对象添加到容器级别建模对象（具体为“WIP 消息评估条目”表格）中。
Note: 执行这些任务的顺序无关紧要。可以先定义 WIP 消息，或者先将建模对象添加到容器级别建模对象。

下面是“WIP 消息评估条目”表格的有效值：

• BOM，表示物料清单
• CurrentStatus。
ReworkStatus.ReworkReason，表示返工原因
• 客户
• HoldReason，表示搁置原因
• Level，表示容器级别
• MfgOrder，表示制造订单
• 作业
• Operation.WorkCenter，表示工作中心
• 所有者
• Priority，表示优先级代码
• 产品
• Product.ERPBOM，表示 ERP 物料清单
• Product.ProductFamily，表示产品系列
• Product.ProductType - 对于产品类型
• 资源
• 销售订单，表示销售订单
• ShippingReason，表示装运原因
• 规格
• Spec.Setup，表示准备
• Spec.TxnMap.DataCollectionDef，表示数据收集定义
• StartReason，表示开始原因
• UOM
• 工作流程

对于要为其配置 WIP 消息的所有容器事务，必须将级别添加到“WIP 消息评估条目”表格。

从特定作业中排除 WIP 消息评估

当配置 WIP 消息评估时，可以指定一个或多个要排除其 WIP 消息评估的作业。例如，如果在移动事务中为小部件产品配置了 WIP 消息，则可以排除包装作业，以便用户在包装作业中针对小部件产品容器执行移动事务时不显示 WIP 消息。

了解用户定义的属性

用户定义的属性是一组动态属性，用于存储有关易于访问的容器的信息。例如，可能希望将实验室结果存储在容器上，以避免不得不通过容器的历史进行搜索这种情况。属性可能与容器颜色一样简单。用户定义的属性对每个容器是唯一的，在指派时不需要数据库更新。

当容器启动时指派用户定义的属性

当启动容器时，应用程序将已定义的属性从以下位置复制到容器：

• 容器级别
• 产品
• 制造订单

应用程序最初从容器级别复制，然后从产品复制，最后从制造订单复制。如果列表中包含同名的属性，则只要数据类型相同，应用程序就会用新值覆盖现有的属性值。如果数据类型不相同，应用程序将显示错误消息。

示例:

假定以下对象已关联这些属性：

批次层次

产品

制造订单

型号 = AAA

颜色 = 红色

年份 = 2015

颜色 = 蓝色

应用程序首先处理容器级别，因此属性列表包含：

• 型号 = AAA

应用程序接下来处理产品。产品有两个新属性，因此应用程序将它们追加到属性列表中：

• 型号 = AAA
• 颜色 = 红色
• 年份 = 2015

应用程序最后处理制造订单。制造订单包含与容器属性（具体为颜色属性）同名的属性。应用程序使用制造订单颜色属性值覆盖现有的颜色属性值：

• 型号 = AAA
• 颜色 = 蓝色
• 年份 = 2015

应用程序将最终属性列表作为容器启动历史的一部分，存储在容器中。可以在“制造审核追踪”中查看容器启动时指派的属性。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Audit Trail 用户指南》或《Opcenter Execution Core Audit Trail 用户指南》。

如何定义容器级别

按照以下步骤定义容器级别：

1. 打开容器级别页。随即“建模”页中将显示“容器级别”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在容器级别字段中输入此容器级别的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 请参考“如何添加父或子容器级别”以添加父或子容器级别。
6. 请参考“如何将 WIP 消息搜索准则添加到容器级别”以添加 WIP 消息搜索准则。
7. 要添加用户定义的属性，请参考“如何将用户定义的属性添加到容器级别”。
8. 单击保存。 应用程序保存建模对象并显示成功消息。

如何添加父或子容器级别

按照以下步骤添加父或子容器级别：

1. 执行“如何定义容器级别”过程。

或

选择现有容器级别实例。

2. 单击“父层次”表格或“子层次”表格上的添加新行。 将显示新行。
3. 单击此行显示层次列表。
4. 选择级别。
5. 重复步骤 2-4 添加其他级别。
6. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何将 WIP 消息搜索准则添加到容器级别

按照以下步骤将 WIP 消息搜索准则添加到容器级别：

1. 执行“如何定义容器级别”过程。

或

选择现有容器级别实例。

2. 展开 WIP 消息配置部分。
3. 单击“WIP 消息评估条目”表格上的添加新行。 将显示新行。
4. 输入 WIP 消息标签。
5. 单击“排除的作业”表格上的添加新行。 将显示新行。
6. 单击此行显示作业列表。
7. 单击不需要包含 WIP 消息的作业。
8. 重复步骤 3-7 添加其他准则。
9. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何将用户定义的属性添加到容器级别

按照以下步骤将用户定义的属性添加到容器级别：

1. 执行“如何定义容器级别”过程。

或

选择现有容器级别实例。

2. 展开用户属性部分。
3. 单击“用户属性列表”表格上的添加新行。 将显示新行。
4. 在属性名称字段中输入属性的名称。
5. 从类型列表中选择属性类型。
6. 在值字段中输入静态值或表达式。
7. 当值为表达式时选中是表达式复选框，以确保正确分析值。
8. 重复步骤 3-7，添加其他属性。
9. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
