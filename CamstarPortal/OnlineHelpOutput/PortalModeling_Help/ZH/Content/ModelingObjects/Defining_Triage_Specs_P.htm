定义归类规范

“归类规范”建模对象用于定义默认处理值以对事件执行归类。它定义在手动归类期间要使用的默认值，以及在自动归类期间要应用的业务规则。

应用程序将根据事件所属的组织确定要使用的归类规范。定义该组织时，将归类规范与组织和类别相关联。

有关组织的信息，请参考“定义组织”。

在定义归类规范时

归类规范包含以下可选的建模定义：“角色”、“所有者”、“检查表模板”和“业务规则”。

将业务规则添加到归类规范时，只有“用法”类型为质量对象的业务规则才可供选择。

Note: 必须使用“建模”中的“业务规则”页来添加或修改业务规则。有关信息，请参考“定义业务规则”。

如何定义归类规范

按照以下步骤定义归类规范：

1. 打开归类规范页。随即“建模”页中将显示“归类规范”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在归类规范字段中输入此任务列表的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将手动归类项添加到归类规范中

按照以下步骤将手动归类项添加到“归类规范”中：

1. 执行“如何定义归类规范”过程。

或

选择现有的“归类规范”实例。

2. 找到“归类项”表格。
3. 单击添加新行。 将显示新行。
4. 从目标类别列表中选择一个上报类别。
5. 从列表中选择角色。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 重复步骤 3-6 以添加其他手动归类项。
8. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何移除手动归类项

按照以下步骤从“归类规范”中移除手动归类项：

1. 执行“如何定义归类规范”过程。

或

选择现有的“归类规范”实例。

2. 找到“归类项”表格。
3. 选择要删除的行。
4. 单击删除所选行。该行将从表格中移除。
5. 重复步骤 3-4 以移除其他项。
6. 单击保存。 应用程序显示成功消息，指出建模对象已更新。

如何将业务规则添加到归类规范

按照以下步骤将自动归类的业务规则添加到“归类规范”中：

1. 执行“如何定义归类规范”过程。

或

选择现有的“归类规范”实例。

2. 找到“业务规则”表格。
3. 单击添加新行。 将显示新行。
4. 从列表中选择业务规则。
5. 重复步骤 3-4 以添加其他业务规则。
6. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
