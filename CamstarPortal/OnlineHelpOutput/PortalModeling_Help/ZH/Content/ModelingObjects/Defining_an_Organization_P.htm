定义组织

组织是具备处理和报告职能的独立业务实体。组织与企业或工厂无关联。

在定义组织时

组织包含以下可选建模定义：

• 检查表模板
• 分类
• 电子邮件通讯组
• 电子邮件消息
• 失败模式组
• 编号规则
• 所有者
• 组织
• 角色
• SMTP 传输
• 子分类
• 归类规范
• 用户界面首选项

如果指定这些可选建模定义，则需要某些字段来定义它们。

组织是可在定义“员工”和“角色”时指定的可选字段。

虽然在创建或更新组织时编号规则字段并非必填项，但是必须先为该组织创建相应类别的记录，然后才能创建事件或不合格品。

在记录生产事件之前，必须将“主要组织”指派给“员工”。此外，还必须将主要组织的已登录用户的角色和所有者指派给事件分类规范映射表格中的相应分类和子分类。

安全处理准则

需要组织来执行 Opcenter EX MDD 和 Opcenter EX CR 安全管理。至少需要一个组织才能设置安全管理。Opcenter EX MDD 和 Opcenter EX CR 使用企业的默认组织进行安装。

通过员工与在组织上下文中有效的角色（工作职能）的关联为员工指派权限，这些关联决定了员工的系统特权。

用户必须在组织中通过角色传播直接或间接地具有角色才能执行任何操作。

有关默认员工登录的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》，它们在安装 Opcenter EX MDD 或 Opcenter EX CR 时加载。

质量记录处理准则

通过定义具有质量记录处理职能的组织，可以指定组织的处理准则，包括：

• 用于创建质量记录的分类/子分类映射
• 用于在“制造审核追踪”中显示质量记录信息的用户界面首选项
• 特定类别的质量记录处理准则，例如，归类规范、类别、角色和所有者
• 质量事务的电子签名要求
• 质量记录的编号规则映射
• SMTP 传输定义
• 电子邮件通知信息
• 与一般或生产事件关联的容器上不允许的事务
• 执行时将受影响容器添加到质量记录的事务

创建质量记录（事件）时，Opcenter EX MDD 或 Opcenter EX CR 使用该记录的组织和类别（类型）来确定用于处理该记录的准则。例如，如果质量记录是属于企业组织的事件，则应用程序将查看组织定义来确定用于处理质量记录的归类规范和为质量记录生成唯一 ID 时要使用的编号方案。

通知可用于标识当应用程序中发生特定事件/操作时要发送的电子邮件。它们可以提醒收件人所需的操作或提供重要的状态更新。例如，可以指定当某个事件的所有权发生变化时要发送给新所有者的电子邮件。

还可以使用组织对象指定需要电子签名的质量事务。

可以为与一般或生产事件关联的容器指定要禁止的事务。还可以指定在对具有受影响物料的相关容器执行事务时，应用程序将受影响容器自动添加到质量事件。

接下来的几个主题提供了有关应用程序如何使用组织对象的详细信息。

角色和所有者指派

质量记录处理要求先将角色和所有者指派给事件，然后用户才能提交事件。如果在提交时没有为事件指派角色和所有者，则会显示一条错误消息，指示未指派所有者和角色并阻止用户提交事件。

应用程序提供了多次指派角色和所有者的机会：

• 自动化业务规则（如果与用来处理事件的归类规范关联，则在归类期间自动应用）
• 事件分类规范映射（定义为组织建模对象的一部分）
• 类别处理映射（定义为组织建模对象的一部分）

指派角色和所有者的第一次机会发生在通过归类自动执行业务规则期间。如果缺少其中一个或两个值，应用程序会先检查“事件分类规范映射”。（优先于类别处理映射。）如果在此处指定了角色和所有者，则应用程序将使用这些值。否则，应用程序会使用为“类别处理映射”指定的角色和所有者值。

Note: 如果在自动执行业务规则期间，仅设置了其中一个值，则应用程序将使用在“事件分类规范映射”或“类别处理映射”中指定的值覆盖该值（如果适用）。例如，如果在归类期间设置了角色值，但所有者值为空，则应用程序会检查在“事件分类规范映射”中指定的角色和所有者值。如果“事件分类规范映射”包含角色和所有者值，则应用程序将使用在“事件分类规范映射”中指定的角色值覆盖在归类期间设置的角色值。

事件和不合格品需要“事件分类规范映射”。

门户页面和页面流在 Portal Studio 中定义。

可以使用“分类”和“子分类”建模对象为组织创建分类和子分类。

在定义类别处理映射时

质量记录处理使用类别处理映射将以下内容指派给特定类别中的质量记录：

• 归类规范
• 角色和所有者（如果先前未指派）

定义类别处理映射时，类别本身是主键且必须唯一。例如，对于每种类别的事件，只能创建一个类别处理映射。

在配置电子签名要求时

通常，在指定要为其收集电子签名的质量事务时，只需将该质量服务添加到组织建模对象上的电子签名事务映射即可。例如，如果希望在用户取消对质量记录解决的审批时收集电子签名，则可以将“取消审批表”质量服务添加到“电子签名事务映射”中。

事件创建除外。如果希望事件记录页流使用电子签名，则不仅要在电子签名事务映射中指定 CreateEvent 服务，而且还必须添加 UpdateEvent 服务。

如果不将更新服务添加到电子签名事务映射中，则在执行事件记录页面流时，用户可以绕过电子签名要求。

指派审批表电子签名要求

为指派审批表配置电子签名时，该签名仅与质量记录的解决审批指派/路由相关。

在定义事务默认设置时

可以为与一般或生产事件关联的容器指定要禁止的事务。单击“生产事件不允许的事务”或“一般事件不允许的事务”表格中的“添加新行”将显示一个列表，可用于选择容器事务。该列表包括每个容器事务，例如“更改数量”、“关闭”、“搁置”和“移动”。

应用程序可以将受影响容器自动添加到质量事件。

可以指定希望应用程序为其将容器自动添加到质量事件的容器事务。如果对具有受影响物料的相关容器执行在该表格中指定的任何事务，应用程序会将受影响容器自动添加到质量事件。

“将受影响容器添加到事件”表格中提供了以下默认事务：

• 关联
• 合并
• 组件分发
• 组件移除
• 拆分

如果对具有受影响物料的相关容器执行在该表格中选择的任何事务，应用程序会将受影响容器自动添加到质量事件。

Note: 如果选择“合并”，在执行“合并容器”或“合并数量”事务时，应用程序会将受影响容器添加到质量事件。同样，“拆分”适用于“拆分容器”和“拆分容器”事务。

有关将受影响容器自动添加到质量事件的信息，请参考《Opcenter Execution Medical Device and Diagnostics Event Management 用户指南》或《Opcenter Execution Core Event Management 用户指南》，以及《Opcenter Execution Medical Device and Diagnostics Nonconformance Management 用户指南》或《Opcenter Execution Core Nonconformance Management 用户指南》。

如何定义组织

按照以下步骤定义组织：

1. 打开组织页。随即“建模”页中将显示“组织”页。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在组织字段中输入此组织的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。 应用程序保存建模对象并显示成功消息。

如何将事件分类规范添加到组织

按照以下步骤将事件分类规范添加到组织：

1. 执行“如何定义组织”过程。

或

选择现有组织实例。

2. 找到“事件分类规范映射”表格。
3. 单击添加新行。 将显示新行。
4. 为事件选择相应的分类。
5. 选择要与分类关联的子分类。
6. 选择要与事件分类和子分类关联的失败模式组。
7. 选择要与使用该事件分类和子分类创建的质量记录关联的默认角色。
8. 选择要与使用该事件分类和子分类创建的质量记录关联的默认所有者。除非先选择角色，否则不能选择所有者。
9. 或者，选择检查表模板。
10. 单击保存。 应用程序显示成功消息，指出建模对象已更新。
11. 重复步骤 3-10，将另一个事件分类规范添加到此组织。

如何从组织中移除事件分类规范
