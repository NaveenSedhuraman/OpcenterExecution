导入建模数据

Opcenter EX MDD 和 Opcenter EX CR 中的 Portal 提供了将导出的建模对象实例从一个安装上传和导入到另一安装的功能。可以通过指定不希望包含在导入中的选定实例来执行部分导入，而不是选择整个导出文件来导入。

Note: Microsoft HTTP 类的默认设置可能不足以进行文件导入。建议导入文件的大小不超过 6 MB。可以导入大于 6 MB 的文件，但性能可能会受到影响。有关导入大于 6 MB 的文件的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。
Note: 查看 CamsarAppServer 应用程序池的 Private Memory Usage 应用程序池设置。导入 1000 个或更多实例（包括参考）时，可能需要增加该值。有关配置应用程序池内存上限的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

在导入建模数据时

应用程序按名称导入建模对象实例。当导入文件包含与现有实例同名的实例时，导入流程可以覆盖应用程序中的现有实例。可以在开始导入流程之前启用替代模式，以防止实例被覆盖。

导入将一直运行，直到完成导入流程。可以修改 Portal Studio 中的 Portal 设置以启用超时，这样可以在达到时间限制时中止导入流程。有关设置超时的信息，请参考《Opcenter Execution Medical Device and Diagnostics Portal Studio 用户指南》或《Opcenter Execution Core Portal Studio 用户指南》。

在“执行控制台”页上的 Portal 替代模式设置中，可以指定应用程序中的建模对象实例是否可以被指定导入文件中的建模对象实例覆盖以及如何覆盖。导入流程为导入提供了三个替代选项：

• 中止 - 在应用程序中检测到与导入文件中的建模对象实例同名的建模对象实例时，导入流程将中止。
• 覆盖 - 导入流程从导入文件导入建模对象实例，并覆盖应用程序中同名的任何实例。
• 跳过 - 导入流程从输入文件导入建模对象实例，但跳过与应用程序中现有实例同名的实例。
Note: 如果未选择替代方法，则应用程序默认使用“跳过”替代方法。

应用程序在“实例选择”页的“对象类型名称”表格中显示可从指定的 XML 文件导入的所有建模对象实例。可以删除不想导入的实例。在“对象类型名称”表格中选择实例，然后单击“删除所选行”按钮。

导入建模对象定义时，如果未先导入它们依赖的建模对象定义，则会发生错误。例如，如果一个对象定义了填充另一个对象页上必需字段的实例，则该对象依赖于另一个对象。必须先导入建模对象所依赖的其他对象，然后在单独的导入流程中导入该对象。

例如，若想导入审批模板和审批决定列表的实例，那么应先导入审批决定列表实例，因为在导入审批模板实例的过程中要求系统中存在审批决定列表实例。然后，应在单独的导入流程中导入审批模板实例。

Note: 应用程序不会导入过滤器标记关联。例如，如果导出包含过滤器标记关联的“产品”建模对象实例，则导入实例时不会包含过滤器标记关联。可以导出和导入过滤器标记实例。

可以使用 Portal Studio 自定义导入页上显示的标签。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Portal Studio 用户指南》或《Opcenter Execution Core Portal Studio 用户指南》。

导入失败

该完成消息表示实例导入失败。如果实例已存在于应用程序中并且在导入期间阻止覆盖，则导入实例将失败。

手动导入建模数据的页面流

手动导入建模数据的页面流包含 4 个页面：

• 数据传输类型，可在该页中选择手动导入
• 实例选择，可在该页中选择要导入的 xml 文件和要导入的特定建模对象实例
• 执行控制台，可在该页中指定替代模式（如果有），然后开始导入
• 执行汇总，可在该页中查看导入的汇总和错误信息

如何手动导入建模数据

按照以下步骤手动导入建模数据：

1. 在导出/导入菜单中打开导出/导入页。将显示“导出/导入数据传输类型”页。
2. 按以下过程完成页面流上的必需字段：
• “如何选择数据传输类型”
• “如何选择要导入的建模对象实例”
• “如何开始导入”
3. 单击完成以完成导入流程。应用程序关闭“导出/导入”页并显示“执行汇总”页，可以在该页中查看导入的汇总以及发生的任何错误。
