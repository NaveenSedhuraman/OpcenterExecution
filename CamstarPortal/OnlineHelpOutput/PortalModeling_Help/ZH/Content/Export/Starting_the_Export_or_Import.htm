开始导出或导入流程

“执行控制台”页是手动数据导出和导入页面流中的最后一页。可以在此页面上开始导出或导入流程。

执行导出

单击“开始导出”按钮时，导出将开始。在生成 XML 文件的过程中，应用程序将在“导出/导入控制台”中显示导出状态。生成 XML 文件后，应用程序将启用“下载导出文件”按钮。可以从此页面下载生成的 XML 文件。

单击“下载导出文件”按钮可显示一个弹出窗口，使您可以查看或保存导出文件。只有在选择“保存”选项时，应用程序才会保存导出文件。打开或保存导出文件或取消打开或保存功能后，应用程序将禁用“下载导出文件”按钮。

单击“完成”按钮可显示“执行汇总”页，可以在其中查看导出汇总和错误信息。

执行导入

可以在此页面上指定应用程序是否可以将现有建模实例覆盖为导入文件，以及开始导入流程。默认情况下，应用程序使用“跳过替代”方法来覆盖建模对象实例。可以选中“存在时导入”复选框，并根据业务需求在“已修改则替代模式”字段中指定其他替代模式（例如，“覆盖”或“中止流程并回滚”）。

如果选择“中止流程并回滚”替代模式，则当遇到重复实例或发生服务器错误时，将停止导入流程并回滚所有已提交的数据和正在等待导入的数据。

单击“开始导入”按钮时，导入流程将开始。从 XML 文件上传选定的建模对象实例时，应用程序将在此页面显示导入状态。

如何开始导出

按照以下步骤开始导出建模数据：

1. 完成“如何选择要导出的建模对象实例”中所述的过程以访问“导出/导入控制台”页。
2. 单击开始导出以开始生成导出数据包。应用程序将生成导出文件，并在“导出/导入控制台”中显示进度。
3. 要查看还是下载导出文件？

如果 ...

操作步骤 ...

是
a. 单击下载导出文件。 应用程序会提示打开或保存该文件，或者它会自动下载该文件。（Internet 浏览器将确定应用程序是否显示弹出窗口、应用程序的外观以及“打开”或“保存”选项的措辞。）
b. 选择要执行的作业，并完成任何使用特定应用程序打开文件的步骤或将文件保存到计算机的其他步骤。
c. 转到步骤 4。

否

转到步骤 4。
4. 单击完成以完成导出流程。应用程序关闭“导出/导入”页，并显示“执行汇总”页，您可以在其中查看导出的汇总以及发生的任何错误。

如何开始导入

按照以下步骤开始导入建模数据：

1. 完成“如何选择要导入的建模对象实例”中所述的过程以访问“导出/导入控制台”页。
2. 是否要允许导入流程覆盖现有 Portal Modeling 对象实例？

如果 ...

操作步骤 ...

是
a. 单击存在时导入。应用程序将启用“已修改则替代模式”字段。
b. 从已修改则替代模式字段中选择替代模式。
c. 转到步骤 3。

否

转到步骤 3。
3. 单击开始导入。导入流程将开始，并且应用程序会在“导出/导入控制台”中显示进度。
4. 单击完成以完成导入流程。应用程序关闭“导出/导入”页，并显示“执行汇总”页，可以在其中查看导入的汇总以及发生的任何错误。
