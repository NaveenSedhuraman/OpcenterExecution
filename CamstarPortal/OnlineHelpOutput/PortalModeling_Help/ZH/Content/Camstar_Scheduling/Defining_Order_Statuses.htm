定义订单状态

订单状态指示制造订单的订单状态。可以使用订单状态对象指派“打开”或“关闭”订单状态。

在定义订单状态时

订单状态是制造订单建模定义中的可选字段。但是，必须将状态为打开的订单状态指派给制造订单实例，才能使用 Opcenter Execution Core Scheduling。有关信息，请参考《Opcenter Execution Core Scheduling 用户指南》。

已安装 Opcenter Execution Electronics 时，“是确认的”复选框可用，用于确认生产的订单状态。有关信息，请参考《Opcenter Execution Electronics 用户指南》。

有关所有建模对象共有的字段的信息，请参考“建模页上的公共字段”。

定义用于 Opcenter Execution Semiconductor 的订单状态
Note: 仅当安装有 Opcenter Execution Semiconductor 时，下述功能才可用。

Opcenter Execution Semiconductor 要求：

• 使用 Preactor 指派给每个计划批次的工作订单。可以在使用“批次开始”或“批次创建”页时或使用“计划”页之一输入初始计划数据时，将工作订单指派给批次。
• 具有指派的订单状态的工作订单。必须在“订单状态”为“打开”且选中“可用”复选框的情况下配置订单状态。

如何定义订单状态

按照以下步骤定义订单状态：

1. 打开订单状态页。“订单状态”页显示在“建模”页中。
2. 单击新建。 显示用于定义新实例的空白字段。
3. 在订单状态字段中输入此订单状态的名称。
4. 根据您的业务需求输入可选信息。
5. 单击保存。 应用程序保存建模对象并显示成功消息。
