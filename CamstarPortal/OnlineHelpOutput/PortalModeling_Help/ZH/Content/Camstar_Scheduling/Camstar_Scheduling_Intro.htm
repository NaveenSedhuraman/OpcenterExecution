Opcenter Execution Core Scheduling

Note: 仅当安装有 Opcenter Execution Core Scheduling 时，下述功能才可用。有关信息，请参考《Opcenter Execution Core Scheduling 用户指南》。

Opcenter Execution Core Scheduling 提供了一种机制，用于使用 Preactor Advance Scheduling 应用程序来计划 Opcenter EX MDD 和 Opcenter EX CR 制造订单。

通过 Opcenter Execution Core Scheduling，组织可以在 Opcenter EX MDD 或 Opcenter EX CR 上为产品和制造订单定义企业资源规划 (ERP) 路线数据，然后在 Siemens Preactor Advanced Planning and Scheduling (Preactor) 应用程序中使用该数据为制造订单创建生产计划。

要启用 Opcenter Execution Core Scheduling，必须在该应用程序中执行以下任务：

• 运行软件随附的 CIO 建模脚本来为 Preactor 配置 CIO 适配器
• 为工厂启用接口
• 提供必需的 ERP 路线信息，例如处理时间
• 将 ERP 路线和路线工步指派给关联的工作流程和工作流程工步
• 将工作流程指派给关联的产品或产品系列
• 对于有效产品及其处于开放状态的订单创建制造订单

当计划人员从 Preactor 发布计划时，有关计划订单的信息会发送到要传回 Opcenter EX MDD 或 Opcenter EX CR 应用程序的暂存表。可以定义另一个 Opcenter Connect MOM 数据库适配器来调用存储程序，以将计划的订单信息以调度列表的形式返回给应用程序。
