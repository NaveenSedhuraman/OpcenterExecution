Electronic Signatures for Modeling Introduction

An electronic signature (ESignature or ESig) is an electronically stored record that serves as the equivalent of a handwritten signature. The Electronic Signatures module provides the ability to capture electronic signatures for changes made to modeling objects. The transactions will fail without the required electronic signatures.

For example, you can define an electronic signature requirement that automatically deploys an Electronic Signature page whenever someone defines a new employee. When you establish this requirement, a modeler cannot create an Employee instance until a signer with appropriate authority has provided a signature approving the transaction.

Each electronic signature is captured and stored in transaction history. Each entry contains the signature requirements and how they were met, along with the local time and GMT. Consequently, you can generate a complete audit trail from this data.

Electronic Signatures for Modeling includes:

• Capturing Modeling Sequence
• Modeling Electronic Signature Information
• Assigning an Electronic Signature Requirement
• Viewing Electronic Signature Details
• Changing an Existing Electronic Signature Assignment
• Removing an Existing Electronic Signature Assignment
• Assigning an Electronic Signature Requirement to Lock/Unlock
• Entering an Electronic Signature for a Modeling Object
• Viewing the Audit Trail for an Electronic Signature
