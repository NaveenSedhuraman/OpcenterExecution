Entering an Electronic Signature for a Modeling Object

A pop-up appears prompting you to provide the appropriate signatures if electronic signatures are required when you create, modify, or delete a modeling object. The modeling object change cannot be completed until these signatures are provided and authenticated.

Signature requirements can be set up for single or multiple signatures of various authority levels. The authority level and number of signatures required depends on how the required signatures were defined.

When Entering an Electronic Signature

The Cosign Reason field is populated only if a Cosigner signature is entered.

Clicking the Add Signature button validates that the entered signer and cosigner information meet the transaction’s electronic signature requirements. The application ensures these conditions are met:

• The signer or cosigner is a valid employee and has provided a valid password
• The role or cosigner role defined in the electronic signature requirement is valid, meaning the role is:
• Associated with the signer or cosigner
• Part of the Esig role group assigned to the signer or cosigner

If the signatures are valid, the application adds the information to the row selected in the child grid, selects the next row automatically, and places the cursor in the Signer field for the next signature.

When all the signatures have been captured for the selected signature requirement, the application:

• Changes the background color of the completed requirement to green
• Selects the next requirement in the grid, if applicable
Important: If you need to modify a signature, you must delete the signature from the requirement and then re-add it with the new information.Reset button removes all signatures from the grid.All signatures for each signature requirement have been added.

Invalid Information Entry

The application displays an error message on the ESignature pop-up if you entered an incorrect ID or password. You can either correct the information and add the signatures or close the ESignature pop-up. The transaction is not completed If you close the pop-up without making corrections.

When Removing Signatures

The application provides two buttons to remove signatures from the Electronic Signature Entry grid: Clear Selected Signatures and Reset. Clicking the:

• Clear Selected Signatures button removes all selected signatures from the currently selected signature requirement.
• Reset button removes all signatures from the grid.

When Submitting Electronic Signatures

When you submit a captured electronic signature, the application performs a final verification to ensure:

• All signatures for each signature requirement have been added.
• The same person did not sign multiple times. (However, cosigners can fulfill multiple signatures, as long as that cosigner has the appropriate roles.)

The ESignature pop-up closes and transaction processing proceeds if the requirements are met. If not, the application leaves the pop-up open and displays an error message.

ESignature Pop-Up

You can enter, view, and maintain signature authorizations for the current service using the ESignature pop-up. The Electronic Signature Entry grid displays the electronic signature requirements defined for the current service.

How to Enter Electronic Signatures for Modeling Objects

Follow these steps to enter an electronic signature for a modeling object:

1. Open the modeling object and instance you want to update. The instance and instance information appears.
2. Create a new instance of the object, modify an existing instance, or delete an existing instance that has an electronic signature requirement.
3. Click Save. The ESignature pop-up appears. The application selects the first (child) row for the first unfulfilled signature requirement by default.
4. Select a different signature requirement if necessary.
5. Enter the Signer and Signer Password.
6. Enter the Cosigner, Cosigner Password, and Cosigner Reason, if a cosigner is designated.
7. Enter comments in the Signer/Cosigner Comments field if necessary.
8. Click Add Signature. The application validates and adds the signature information to the requirement selected in the Electronic Signature Entry grid. If additional signatures are required, the application adds the next child row automatically and places the cursor in the Signer field for the next signature.
9. Repeat steps 4-8 to add additional signatures for the selected requirement. When all signatures for a requirement have been captured, the application changes the background color of the parent row to green and selects a blank signature row in the last unfulfilled signature requirement.
10. Click Submit Signatures. The application closes the ESignatures pop-up and displays a status message indicating the modeling object was saved.
