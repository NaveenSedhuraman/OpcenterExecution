Assigning an Electronic Signature Requirement

The Modeling ESig page, which is available from the Modeling menu, enables you to assign an electronic signature requirement to four different actions on a modeling object service.

Action

Requires an electronic signature when you . . .

Create

Create a new instance of a modeling object.

Update

Update an existing instance of a modeling object.

Delete

Delete an existing instance of a modeling object.

Lock/Unlock

Lock or unlock an existing instance of a modeling object.

When you assign an electronic signature requirement, the application applies the assignment to the selected service, and displays the assignments in the Resolved Signature Requirements section.

When Assigning an Electronic Signature Requirement

Electronic signatures must have been created previously using ESig Requirements in Modeling before you can assign an electronic signature requirement. Refer to "Defining Electronic Signature Requirements" for information on defining electronic signatures.

An electronic signature cannot be assigned to a service that is used to manage the electronic signature-to-maintenance service relationship (for example, MaintenanceESigReq). You can also restrict access to performing that maintenance service using the role-based access control security available in the application.

How to Assign an Electronic Signature Requirement

Follow these steps to assign an Electronic Signature Requirement:

1. Open the Modeling ESig page. The Modeling ESig page appears.
2. Select the modeling object service from the list in the Services grid.
3. Select an electronic signature requirement from the appropriate action list Create, Update, Delete, or Lock/Unlock in the Signature Requirement Assignment grid to assign a requirement to the service.
4. Click Update. The electronic signature requirement is assigned to the service.
