Removing an Existing Electronic Signature Assignment

You can remove an electronic signature requirement assigned to a service. Use the Modeling ESig page to remove the assignment. Refer to "Changing an Existing Electronic Signature Assignment" if you want to change the assignment instead of removing it.

When Removing an Existing Electronic Signature Assignment

Setting the assignment to None creates an override that indicates this service does not require any signature, even if the parent has one attached to it. An alternate method for removing the assignment for a service is to set the assignment to Inherit. Inherit indicates the specific service does not have direct assignment, but may still require a signature based on its parent's assignment.

How to Remove an Existing Electronic Signature Assignment

Follow these steps to remove an existing electronic signature assignment:

1. Open the Modeling ESig page. The Modeling ESig page appears.
2. Select the modeling object service with the electronic signature assignment that you want to remove from the list in the Services section.
3. Select the electronic signature requirement that contains the assignment from the applicable action list (Create, Update, Delete, or Lock/Unlock) in the Signature Requirement Assignment section.
4. Select None from the list to override any assignments to the service.

Or

Select Inherit from the list to inherit the requirement assignment of its parent.

5. Click Update. The electronic signature requirement is no longer assigned to the service.
