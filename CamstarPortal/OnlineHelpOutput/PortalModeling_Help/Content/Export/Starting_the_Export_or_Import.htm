Starting the Export or Import Process

The Execution Console page is the final page in both the manual data export and import pageflows. You start the export or import process on this page.

When Performing the Export

The export begins when you click the Start Export button. The application displays the status of the export in the Export/Import Console as it builds the XML file and enables the Download Export File button when the XML file is built. You can download the resulting XML file from this page.

Clicking the Download Export File button displays a pop-up enabling you to view or save the export file. The application saves the export file only when you select the Save option. The application disables the Download Export File button after you open or save the export file or cancel the open or save function.

Clicking the Finish button displays the Execution Summary page where you can view the export summary and error information.

When Performing the Import

You can specify whether or not the application can overwrite existing modeling instances with the import and start the import process on this page. The application uses the Skip override method for overwriting modeling object instances by default. You can select the Import if Exists check box and specify a different overriding mode such as Overwrite or Abort Process and Roll Back, from the Overriding Mode if Modified field according to your business needs.

Selecting Abort Process and Roll Back as the overriding mode stops the import process and rolls back all committed and pending imported data if a duplicate instance is encountered or a server error occurs.

The import process begins when you click the Start Import button. The application displays the status of the import on this page as it uploads the selected modeling object instances from the XML file.

How to Start the Export

Follow these steps to start the export of Modeling data:

1. Complete the "How to Select the Modeling Object Instances to Export" procedure to access the Export/Import Console page.
2. Click Start Export to begin building the export data package. The application builds the export file and displays the progress in the Export/Import Console.
3. Do you want to view or download the export file?

If . . .

Then . . .

Yes
a. Click Download Export File. The application prompts you to open or save the file, or it automatically downloads the file. (Your Internet browser determines whether the application displays a pop-up, its appearance, and the wording of the Open or Save options.)
b. Select the action you want to perform and complete any additional steps for opening the file with a specific application or saving the file to your computer.
c. Go to step 4.

No

Go to step 4.
4. Click Finish to complete the export process. The application closes the Export/Import page and displays the Execution Summary page where you can view a summary of the export and any errors that occurred.

How to Start the Import

Follow these steps to start the import of Modeling data:

1. Complete the "How to Select the Modeling Object Instances to Import" procedure to access the Export/Import Console page.
2. Do you want to allow the import process to overwrite existing Portal modeling object instances?

If . . .

Then . . .

Yes
a. Click Import if Exists. The application enables the Overriding Mode if Modified field.
b. Select an override mode from the Overriding Mode if Modified field.
c. Go to step 3.

No

Go to step 3.
3. Click Start Import. The import process begins and the application displays the progress in the Export/Import Console.
4. Click Finish to complete the import process. The application closes the Export/Import page and displays the Execution Summary page where you can view a summary of the import and any errors that occurred.
