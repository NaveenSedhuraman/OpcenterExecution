Selecting Modeling Object Instances

The Instance Selection page is the second page in the both the manual data export and import pageflows. This page provides the ability for you to do one of the following:

• Select the modeling object instances to export when performing a manual export
• Specify the XML file to import and to specify the specific modeling object instances to import when performing a manual import

When Selecting Modeling Object Instances to Export

The application displays all instances defined for a selected modeling object. You can select the Select All check box in the header row to select all displayed instances automatically or you can select the check boxes for individual instances.

Clicking the Assign button adds the selected instances to the Object Type Name grid and updates Object And Instance Counts grid. Entries in the grid appear in alphabetical order by modeling object name. You must click the Expand icon in the modeling object row to view the specific instances assigned for export.

The Object And Instance Counts grid indicates the following:

• Number of modeling objects available for selection
• Total number of modeling object instances available for selection
• Number of modeling objects assigned for export
• Total number of modeling object instances assigned for export
• Number of modeling objects referenced by the selected modeling objects
• Total number of referenced modeling object instances assigned for export

Example:

You select to assign the Batch, Rack, and Tray instances of the Container Level modeling object for export. Clicking the Assign button displays Container Level in the Object Type Name grid and updates the Object And Instance Counts grid to indicate one Selected modeling object and three instances. You can expand the Container Level row in the Object Type Name grid to view the Batch, Rack, and Tray instances.

When Selecting Modeling Object Instances to Import

The application displays all instances contained in the file selected to import. You can delete the instances you do not want to import. Clicking the Delete selected row button on the Object Type Name grid deletes the selected instances. Select the select all check box in the header row to select all displayed instances automatically or select the check boxes for individual instances. You must expand the modeling objects to see the instances for each object.

The Object And Instance Counts grid indicates:

• Number of modeling objects available for import
• Total number of modeling object instances available for import
• Number of modeling objects assigned for import
• Total number of modeling object instances assigned for import

The Browse button opens a File Upload dialog. The file location default is your desktop. Siemens recommends that you copy exported files to a location that is easy to access when performing imports.

How to Select the Modeling Object Instances to Export

Follow these steps to select modeling object instances to export:

1. Complete the "How to Select the Data Transfer Type" procedure to access the Instance Selection page.
2. Select a modeling object in Available Objects. The application displays all the instances defined for that modeling object.
3. Click the Select All check box in Available Instances to select all the instances.

Or

Select the check box next to each instance.

4. Click Assign. The application adds the object and its selected instances to the Object Type Name grid and updates Object And Instance Counts.
Note: Click the expand icon in the Object Type Name grid to view all instances assigned for that modeling object.
5. Repeat steps 2-4 to specify additional instances.
6. Click Next to move to the next page in the pageflow. The Export/Import Add References page appears.

How to Select the Modeling Object Instances to Import

Follow these steps to select the modeling object instances to import:

1. Complete the "How to Select the Data Transfer Type" procedure to access the Instance Selection page.
2. Click the File Upload icon and use the pop-up displayed to select an XML file to import. The application displays the modeling object instances contained in the selected file in the Object Type Name grid and the name of the selected file in the Select Content to Include field.
Note: Click the expand icon in the Object Type Name grid to view all instances assigned for that modeling object.
3. Expand the object and select any instances you do not want to import and click the Delete selected row button on the Object Type Name grid. The application removes those instances from the grid.
4. Click Next to move to the next page in the pageflow. The Export/Import Execution Console page appears.
