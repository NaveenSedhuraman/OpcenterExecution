Exporting Modeling Data

The Portal provides the ability for you to export modeling object instance information to an XML file. You can generate export data for specific modeling object instances or for all instances. Additionally, you can include all references (dependencies) for the selected instances.

Before Exporting Modeling Data

Some modeling object instances available for selection on the Instance Selection page should not be exported and then imported to a target system. Some modeling objects may cause conflicts when exporting and then importing to a target system.

Note: Only modeling objects for which instances have been defined are available for selection in the Available Objects grid. The application does not display any modeling objects without defined instances.

Modeling Objects That Should Not Be Exported

Siemens recommends you do not export these modeling object instances for import on a target system.

Note: Adding modeling objects to the Modeling Objects to Exclude grid on the Factory modeling object ensures those instances do not appear for selection during export. Refer to "Defining a Factory" for information.

Modeling Object

Reason Not to Export

Action

Action modeling object instances contain references to pages. Pages may contain different IDs from one system to another.

Data Collection Def

Data collection definitions require Designer and metadata updates.

Note: This does not apply to user data collection definitions.

Factory

A factory modeling object instance may contain circular references.

Mfg Calendar

Overriding an existing calendar may have unintended consequences on history. Refer to "Defining Manufacturing Calendars" for an alternative method to load a calendar from Excel.

Portal Menu Definition

Instance IDs are not likely to match from one system to another unless the database has been cloned. There is a potential to orphan menu items.

Resource

A resource modeling instance may contain circular references.

Role

Instance IDs are not likely to match from one system to another unless the database has been cloned.

UI Page Flow

You should export and import UI Page Flows using Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information.

UI Virtual Page

You should export and import UI Virtual Pages using Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information.

Web Part

You should export and import Web Parts using Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information.

Modeling Objects That May Cause Conflicts

Siemens recommends you carefully consider exporting these modeling object instances for import on a target system.

Modeling Object

Reason Not to Export

Employee

An employee modeling object instance may contain circular references. Instances may also reference role IDs that do not exist in the target system.

History View

The manufacturing audit trail display may change unexpectedly on the target system. Custom changes on the target system may be overwritten.

Modeling ESig Requirement

A Modeling ESig requirement instance may contain circular references. The users, roles, and electronic signatures may not be consistent from source to target.

Numbering Rule

Conflicts may occur with numbering rule instances when the number on the source system is different from the number on the target system.

Organization

An organization modeling object instance may contain circular references.

Revisioned Objects (ROs)

The application sets the first revision included in the import of revisioned objects as the Revision of Record when:

• No Revision of Record exists on the target system for an imported RO, and
• The export does not include a Revision of Record for the RO.

Scheduled Business Rule

Importing a scheduled business rule exported from another system will update the start date on the target system. Additionally, you may need to restart the Camstar Deployment Server and the Camstar Notification Server if the scheduled business rule instance being updated or created on the target system affects deployment or notifications.

Deprecated Modeling Objects You Can Export and Import

Existing customers who use modeling object instances that have been deprecated in later releases can still export and import these modeling object instances:

• FTP Transport
• MSMQ Transport
• Scale
• Scale Code
• Scale Group
• Scale Status Reason
• Web Service Transport

When Exporting Modeling Data
Note: The default settings for the Microsoft HTTP class may be insufficient for file imports. A maximum of 6 MB is recommended for file imports. Files larger than 6 MB can be imported, with a default maximum size limit of 30 MB, but performance could be affected. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on importing files larger than 6 MB.
Note: Review the Private Memory Usage application pool setting for the CamsarAppServer application pool. You may need to increase the value when exporting 1000 or more instances (including references). Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on configuring the maximum memory limit for the application pool.

You can export both named data object (NDO) instances and revisioned object (RO) instances.

Note: Settings in the Designer application determine whether modeling object instances are available for export. Non-exposed, abstract modeling objects are not available. Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information.
Note: The application does not import filter tag associations. For example, if you export a Product modeling object instance that contains a filter tag association, the filter tag association is not included when you import the instance. You can export and import Filter Tag instances.

You can customize the labels that appear on the export pages using Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information.

Clicking the Start Export button on the Execution Console page (the last page of the pageflow) starts the export. Clicking the Finish button on the Execution Console page closes the Export/Import pageflow and displays the Execution Summary page.

Exports run until the export process is complete. You can modify the Portal settings in Portal Studio to enable a timeout that aborts the export process when a time limit is reached. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information on setting a timeout.

Clicking the Download Export File button displays a pop-up enabling you to view or save the export file. Clicking the Finish button displays the Execution Summary page where you can view and print the export summary and error information. The application saves the export file only when you select the Save option. The application disables the Download Export File button after you open or save the export file or cancel the open or save function.

Pageflow for Manually Exporting Modeling Data

The pageflow for manually exporting Modeling data consists of five pages:

• Data Transfer Type, where you select the manual export
• Instance Selection, where you select the modeling object instances to export
• Add References, where you may opt to include all modeling object instances referenced in the previously selected modeling object instances
• Execution Console, where you start the export and then download the XML file if necessary
• Execution Summary, where you can view summary and error information for the export

How to Manually Export Modeling Data

Follow these steps to manually export Modeling data:

1. Open the Export/Import page from the Export/Import menu. The Export/Import Data Transfer Type page appears.
2. Complete the required fields on the pageflow using these procedures:
• "How to Select the Data Transfer Type"
• "How to Select the Modeling Object Instances to Export"
• "How to Add Referenced Modeling Object Instances"
• "How to Start the Export"
3. Click Finish to complete the export process. The application closes the Export/Import page and displays the Execution Summary page where you can view a summary of the export and any errors that occurred.
