Adding Referenced Modeling Object Instances

The Add References page is the third page in the manual Modeling data export pageflow. This page allows you to include modeling objects that are referenced by the modeling objects selected on the Instance Selection page and display all objects in a hierarchical view.

When Including References

When you choose to include references, the application adds all instances of all modeling objects referenced by the modeling objects selected on the previous page. Clicking the Include References button adds all the referenced instances to the Object Type Name grid and updates the Object And Instance Counts grid. The application maintains the alphabetical order in this grid also, regardless of whether the added instance is a reference.

Any objects excluded from the reference search are listed in the Object References Not Included field. This list includes any objects selected for exclusion on the Factory page. Refer to "Defining a Factory" for information.

Adding references is optional when performing an export. Objects are still exported and imported in the correct order if you choose not to include references. You must assign modeling object instances in the correct order or add references only when the Change Mgt Save option is set to the On position. Refer to the Change Management User Guide for information on the Change Mgt Save option.

Note: Selecting Add References may automatically add modeling objects to the Object Type Name grid that should not be exported. Siemens recommends you manually remove these instances from the grid to avoid conflicts. Refer to "Before Exporting Modeling Data" for information.

Example:

The Container Level modeling object may contain the optional modeling objects Numbering Rule and UI Preference. If you choose to include references for the selected instances, the application adds the Numbering Rule and UI Preference modeling objects, if referenced by any of the container level instances, to the Object Type Name grid. The application also updates the Object and Instance Counts grid to display the number of referenced modeling objects and the total number of instances of referenced modeling objects.

When Viewing Referenced Instances

Clicking the Hierarchy button displays the Hierarchy pop-up. The Hierarchy pop-up is a read only pop-up that displays an indented tree view of the selected modeling objects and all referenced objects associated with the selected instances. The Hierarchy pop-up only displays the assigned and referenced object instances that are listed in the Object Type Name grid.

Note: The Hierarchy pop-up does not display the Filter Tags modeling object even if it is attached to the selected instance. Filter Tags is a comma delimited list that is not attached to the object as a reference.

How to Add Referenced Modeling Object Instances

Follow these steps to add referenced modeling object instances:

1. Complete the "How to Select the Modeling Object Instances to Export" procedure to access the Add References page.
2. Click Include References to add all modeling object instances referenced by the instances you assigned on the Instance Selection page (unless they were excluded on the Factory page). The application adds all the referenced instances to the Object Type Name grid and updates the Object And Instance Counts grid.
Note: Click the expand icon in the Object Type Name grid to view all instances assigned for that modeling object.
3. Click Hierarchy to see a hierarchical view of the object instances. The application displays the Hierarchy pop-up.
4. Expand the object and select any instances you do not want to export and click Delete selected row on the Object Type Name grid. The application removes those instances from the grid.
5. Click Next to move to the next page in the pageflow. The Import/Export Execution Console page appears.
