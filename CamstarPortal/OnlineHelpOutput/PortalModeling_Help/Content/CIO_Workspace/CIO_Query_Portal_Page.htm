Defining CIO Query Portal Pages

[US30047 V8 RH]

You can use the CIO Query Portal Page object to define a sequence of queries to return results to the CIO Query Portal Page Inquiry. Refer to the Portal Modeling User Guide and "Defining CIO User Queries" for information on defining user queries.

When Defining CIO Query Portal Pages

You must use the CIO User Query page to define CIO user queries before you can define fields on the CIO Query Portal Page object.

You can define a maximum of four queries in the Query Portal Page Queries grid. The application automatically assigns a sequence order to the queries listed in the Query Portal Page Queries grid.

The first query listed in the Query Portal Page Queries grid should be a summary-type query. This query does not require parameters. An example of a summary-type query is a query to display CIO log entry rows for today.

The second query must have parameters using the same name as rows in the first query. The application populates the results of the second query based on the row selection in the first query. The application populates the results of the third query based on the row selection in the second query. The application populates the fourth query based on the row selection in the third query. Refer to "Performing CIO Query Portal Page Inquiries" for information on performing a CIO Query Portal Page Inquiry.

How to Define a CIO Query Portal Page

Follow these steps to define a CIO Query Portal Page:

1. Open the CIO Query Portal Page object. The CIO Query Portal Page object appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the query portal page in the CIO Query Portal Page field.
4. Click Add new row on the Query Portal Page Queries grid. A new row appears.
5. Enter a name for the first query in the Query Name field.
6. Select a User Query from the list.
7. Repeat steps 4-6 to add queries.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Save. The application saves the modeling object and displays a success message.
