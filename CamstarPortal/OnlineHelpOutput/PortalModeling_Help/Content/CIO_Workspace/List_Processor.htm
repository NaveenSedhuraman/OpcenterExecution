Defining CIO List Processors

There are two options for list processing in an outbound message definition. These are:

• CIO List Template / List Expression
• CIO List Processor

The CIO List Template / List Expression is the simpler option if all of the following are true:

• There is one list to be output in the service.
• The list does not contain another list.
• The list is a SubentityList (not a list of objects).

A CIO List Processor is used with CIO outbound messaging. A CIO list processor conditionally evaluates each item in a list and builds an XML nodeset based on the list items. CIO list processors can contain other CIO list processors to further define the resulting XML nodeset returned.

The CIO list processor option provides a mechanism to process N number of lists in a service. These lists can be subentity or object lists. The CIO List Expression and CIO List Template provide basic list processing capability. The CIO List Processor provides advanced list processing capability.

Before Using a CIO List Processor

You must define a text variable and include it in a CIO template. The text variable expression must be in the form:

 

Findlistitem(CIOListProcessResults,Name,”<ListName>”).CIOListResult

<ListName> is the name of the list being processed. Refer to "Defining CIO Templates" for information.

CIOListProcessResults is the named subentity owned by the Service CDO and has a string field named CIOListResult. Each nodeset that is processed is appended to this subentity. This nodeset’s outermost element name is the list processor name, unless overridden by the NodesetName.

When Defining a CIO List Processor

List item value expressions must use CLF::ListItem.field to refer to fields in the list item. You can refer to the parent list item using CLF::ListItemParent.field if the list is nested.

Using the CIO List Processors Grid

A CIO list processor appends string output to the CIOListResults string field of the Service CDO's CIOListProcessResults subentity when evaluating each list item. A CIO list processor can optionally reference one or more CIO list processors in the CIO List Processors grid. These list processors are evaluated in the order they appear in the grid. Their output is appended to the parent CIO List Processor results.

How to Define a CIO List Processor

Follow these steps to define a CIO List Processor:

1. Open the CIO List Processor page. The CIO List Processor page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the list processor in the CIO List Processor field.
4. Enter the revision designation in the Revision field.
5. Enter the list expression in the List Expression field.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.
