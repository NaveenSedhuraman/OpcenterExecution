Creating CIO Outbound Message Definitions

The CIO Outbound Message Definition object enables you to create the message definition for CIO outbound messaging by selecting the service to generate an outbound message.

When Creating a CIO Outbound Message Definition

These rules apply when creating a CIO outbound message definition:

• Outbound message definitions are named for the services that trigger them.
• Outbound messages must be assigned a message type.
• Outbound messages must have an associated channel adapter or channel adapter expression.

Using an Outbound Message as a Response Message

You can create an outbound message definition to use an outbound message as a response message. The application does not create a response message when the MIOCamstarSubmit activity is used in a workflow. The application generates an outbound message in place of the response message. The application sets the Request ID of this outbound message to the Request ID of the response message.

Only one outbound message can be generated as a response message. The application ignores any subsequent outbound message definitions marked as IsResponse.

When Creating a CIO Outbound Message Definition for Synchronous Outbound Request and Response Messages

You must select the following on the Add CIO Outbound Msg Def pop-up:

• Is Enabled check box
• Is Synchronous check box
• Request Message in the Message Type field

How to Create a CIO Outbound Message Definition

Follow these steps to create a CIO Outbound Message Definition:

1. Open the CIO Outbound Msg Def page. The CIO Outbound Msg Def page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Select a Service Type Name from the list.
4. Click Add new row in the Outbound Definition grid to add an input row to the grid. The Add pop-up appears.
5. Select a CIO Sync Timing from the list.
6. Enter a name for the message type in the Message Type Name field.
7. Select a CIO Template from the list.
8. Enter a revision designation in the Revision field.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click OK. The Add pop-up closes.
11. Click Save. The application saves the modeling object and displays a success message.
