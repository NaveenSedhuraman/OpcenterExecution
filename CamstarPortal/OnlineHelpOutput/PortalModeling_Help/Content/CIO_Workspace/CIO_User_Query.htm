Defining CIO User Queries

The CIO User Query object allows you to create user queries for CIO outbound messaging.

When Defining CIO User Queries

The User Query Parameter grid on the CIO User Query object includes a column for the parameter value expression. The parameter value expression is an expression which can evaluate to a parameter value at run time. An example of a parameter value expression is:

Container.MfgOrder?Container.MfgOrder.Name:Container.Name

This is called a ternary expression. The expression in this example resolves to the Mfg Order name if it exists. Otherwise, It resolves to the Container Name.

You can also use any expression valid for a text variable in the Parameter Value Expression field. Refer to "Defining Text Variables" for information on text variables.

When Using the User Query Test Pop-up

The User Query Test pop-up appears when you click Save and Test User Query. Any required parameter fields are displayed in the pop-up. A grid with results appears when the parameter fields are populated. The application tests the query and provides results if no parameters are required. An error message will be displayed if the query syntax is invalid and no results are displayed in the grid.

How to Define a CIO User Query

Follow these steps to define a CIO User Query:

1. Open the CIO User Query page. The CIO User Query page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the user query in the CIO User Query field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Refer to "How to Test a CIO User Query that Requires a Parameter" for information on testing a query that requires a parameter.

Or

Refer to "How to Test a CIO User Query that Does Not Require a Parameter" for information on testing a query that does not require a parameter.

6. Click Save. The application saves the modeling object and displays a success message.

How to Test a CIO User Query that Requires a Parameter

Follow these steps to test a CIO User Query that requires a parameter:

1. Perform the "How to Define a CIO User Query" procedure.

Or

Select an existing CIO User Query instance.

2. Enter a SQL statement in the Query Text field that requires a parameter.
3. Enter the User Query Parameter Name and Data Type in the User Query Parameter grid.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save and Test Query to test the query. The User Query Test pop-up appears with the required parameters at the top of the pop-up.
6. Enter any parameters returned by the query.
7. Click Test Query. Any results matching the query and parameters entered are displayed in the Query Results grid.
8. Click Close. The User Query Test pop-up closes.

How to Test a CIO User Query that Does Not Require a Parameter

Follow these steps to test a CIO User Query that does not require a parameter:

1. Perform the "How to Define a CIO User Query" procedure.

Or

Select an existing CIO User Query instance.

2. Enter a SQL statement in the Query Text field that does not require a parameter.
3. Click Save and Test Query. The User Query Text pop-up appears with results displayed in the Query Results grid.
4. Click Close. The User Query Test pop-up closes.

How to Add Parameters to a CIO User Query

Follow these steps to add parameters to a CIO User Query:

1. Perform the "How to Define a CIO User Query" procedure.

Or

Select an existing CIO User Query instance.

2. Click Add new row in the User Query Parameter grid. A new row appears.
3. Enter a name for the parameter in the User Query Parameter Name field. The name will appear as the caption for the data entry field in which the user types the required input.
4. Select a Data Type from the list.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Repeat steps 2-5 to add additional parameters as needed.
7. Click Save. The application saves the modeling object and displays a success message.
