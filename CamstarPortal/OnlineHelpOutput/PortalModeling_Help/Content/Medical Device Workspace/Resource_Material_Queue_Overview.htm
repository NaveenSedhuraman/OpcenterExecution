Resource Material Queue Overview

Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to the Opcenter Execution Medical Device and Diagnostics Industry Solutions User Guide or the Opcenter Execution Core Industry Solutions User Guide for information.

Resource Material Queue (RMQ) functionality enables you to track inventory quantity at an inventory location, and to distribute inventory to a material queue on a production line. You can associate the material queue with a resource so that inventory is automatically consumed as a container associated with the material queue moves through the resource. This functionality consists of the following:

• Inventory Location Management
• Material Queues
• Material Consumption

Inventory Location Management

An inventory location represents a physical holding place for materials. Examples of inventory locations include shelves or storage rooms.

Materials can be stored in an inventory location as containers, lots, or products. A lot typically represents a vendor identification that is not tracked on a workflow. Lots are required for lot and stock component issue types.

Products are used when there is no associated container or lot. Products are used for the Quantity Only component issue type. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information on component issue types.

Inventory location management enables you to do the following:

• Track material quantity assigned to an inventory location
• Manually assign materials to an inventory location
• Automatically assign materials to an inventory location
• Transfer materials from one inventory location to another
• Remove materials from an inventory location

You can automatically put containers into an inventory location by defining a default inventory location for factories, product families, and products. When the container is moved to an operation defined as an inventory point, the container is automatically assigned to the inventory location. Refer to "Assigning a Default Inventory Location to a Factory," "Assigning a Default Inventory Location to a Product Family," and "Assigning a Default Inventory Location to a Product" for information.

Material Queues Overview

A material queue represents a collection of materials used to produce a product. You can allocate materials to a material queue from an inventory location which reduces the quantity available in the inventory location. You can allocate materials from an inventory location to one or more material queues. Material queues can then be associated to resources for automatic material consumption.

You can associate one or more material queues with a resource. However, only one material queue can be active for the resource at a given time. As containers are processed through the resource, the materials in the active material queue are automatically consumed and the quantity remaining in the material queue is reduced accordingly.

You can remove leftover materials from a material queue. Materials that are removed are automatically returned to the inventory location they were allocated from.

Material Consumption Overview

The material consumption functionality enables you to automatically issue material loaded on a resource as Work in Progress (WIP) is processed. The transaction to consume the materials can be defined on the Factory modeling object. You can also optionally validate that the material queue quantities are sufficient to produce the container before it begins processing.

Material Queue Kitting Overview

You can use the material queue functionality to to add and remove products or components to and from material queues representing kits as needed for production orders. Material queue kits can be created and assigned to a container, a manufacturing order, or a work area. Loading the materials from an inventory location reduces the quantity available from that location.

Material queue kit functionality enables you to:

• Create a material queue kit with components that are issued using any component issue type.
• Validate the correct quantity and product are issued based on the container or manufacturing orders material list.
• Specify the order the components should be used in the kit.
• Specify the removal strategy for materials.
• Specify if materials are automatically or manually consumed. Components consumed when processing the container are validated and consumed from the material queue kit.
• Specify which consumed quantities are reported back to an enterprise resource planning (ERP) system when an ERP is configured. Refer to the Opcenter Connect MOM User Guide for information on configuring an Opcenter Connect MOM database adapter.
• Unload Items from a material queue kit. The unloaded items are automatically returned to the inventory location.
