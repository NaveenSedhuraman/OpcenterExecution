Defining Production Identifiers and UDI Expressions

You can define both the production identifier and unique device identifier directly on a container by manually entering the information when you start the container from the Start page or the Order Dispatch page. You can use the Container Maintenance page to update these fields after the container is started. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information on assigning production identifiers and UDI directly to containers.

You can use the UDI Expression object to create expressions for the UDI and PI. These expressions are used to automatically create and assign the UDI and PI to the container at run time when configured on the product. These expressions can use any values that can be referenced from the Move or Move In transactions. You must create a different expression for each PI and UDI expression you use in your implementation.

For example, use this variable expression in the Variable Expression field to create and assign a UDI that contains the device identifier, manufacturing date, and container name:

 

((Container.DeviceIdentifier!=Null? "(01)" + Container.DeviceIdentifier.Name:Null) + (DateConvert

(Container.ManufacturingDate,"yyMMdd")!=Null? "(11)" + DateConvert

(Container.ManufacturingDate,"yyMMdd"):Null) + (Container.Name!=Null? "(21)" + Container.Name:Null))

Usage

Defines the type of variable. Options include:

• Alarm Message

• Instruction Text
• WIP Message

Note: UDI expressions are always general.

Optional

Unresolved Value

Value to use in the event the expression cannot be successfully evaluated. This can be an empty string (to ignore that the expression cannot be evaluated) or some other value (for example, "ERROR!") to make it clear that the text of the resulting message is not complete or accurate.

How to Define a UDI Expression

Follow these steps to define a UDI Expression:

1. Open the UDI Expression page. The UDI Expression page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the UDI expression in the UDI Expression field.
4. Enter a variable expression in the Variable Expression field.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.
