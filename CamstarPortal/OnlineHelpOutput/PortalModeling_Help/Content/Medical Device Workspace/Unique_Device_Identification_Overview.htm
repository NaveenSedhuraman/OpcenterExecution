Unique Device Identification Overview

Note: The functionality described below is available only when Medical Device is configured. Refer to the Opcenter Execution Medical Device and Diagnostics Medical Device User Guide or the Opcenter Execution Core Medical Device User Guide for information.

UDI is a requirement by the Food and Drug Administration (FDA) to uniquely identify medical devices, combination products that contain devices, and devices licensed under the Public Health Service Act. UDI helps the FDA in tracking device production and facilitating device recalls, thereby improving product quality and patient safety.

The UDI functionality provides a mechanism to define and assign UDI to a container. You can use the Spec and Product modeling pages to define UDI information that will propagate to a container.

You can also add UDI information directly to a container from the following transaction pages:

• Start
• Order Dispatch
• Container Maintenance

The Move and Move In transactions can assign UDI to a container automatically if specific information is defined on the spec and product. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide and the Opcenter Execution Core Shop Floor User Guide for information on assigning UDI on the Move and Move In transactions.
