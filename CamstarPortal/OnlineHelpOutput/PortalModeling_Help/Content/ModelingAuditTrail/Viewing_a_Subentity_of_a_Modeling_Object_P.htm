Viewing the Audit Trail for a Subentity of a Modeling Object

A Subentity is an object that is owned by another object. A subentity object belongs to one and only one parent object.

When Viewing Subentities of Modeling Objects

The fields in the Object Key Values section are dynamic and are based on the subentity selected and whether that subentity has more than one parent CDO type available. The application populates the Parent CDO Type field and displays the subentity object field automatically when only one parent CDO type exists for the selected subentity. You must select a parent CDO type when there are multiple parent CDO types available before the application displays the subentity object field.

Selecting a subentity object displays a grid populated with instances of that subentity. Clicking the View Audit button displays the Modeling Audit Trail pop-up for the selected instance. The application activates the View Audit button after you select a revision of an instance.

How to View a Modeling Audit Trail for a Subentity

Follow these steps to view an audit trail for a Subentity:

1. Open the Audit Trail page. The Audit Trail page appears.
2. Select a Subentity from the CDO Type field. The application displays the Parent CDO Type field. The application may display the subentity object field also if the Parent CDO Type field was populated automatically.
3. Select the Parent CDO Type if necessary. The Object Instance Id field appears.
4. Select the subentity object for which you want to view the defined instances. The application displays a grid containing all the instances associated with the selected object.
5. Select an item from the grid.
6. Click View Audit. The Audit Trail pop-up appears.
7. Click any arrow to expand the audit trail and see more details.
