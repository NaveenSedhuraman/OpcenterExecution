Viewing the Audit Trail for a Revisioned Object

A Revisioned Object (RO) is an object that has a unique name and a revision. Product is an example of an RO.

When Viewing Revisioned Objects

The fields in the Object Key Values section are dynamic and are based on the RO selected. For example, if you select Product from the CDO Type field, the application displays a Product field and a revision field.

The application requires you to select a particular revision of the RO instance to view. You must expand the object instance and then select the revision. You cannot select the instance and then enter the revision manually.

The application activates the View Audit button after you select a revision of an instance. Clicking the View Audit button displays the Modeling Audit Trail pop-up for the selected instance.

How to View a Modeling Audit Trail for a Revisioned Object

Follow these steps to view an audit trail for a Revisioned Object:

1. Open the Audit Trail page. The Audit Trail page appears.
2. Select a Revisioned Object from the CDO Type field. The object name and object revision fields appear under Object Key Values.
3. Select the instance and revision of the instance for which you want to view the audit trail. The application activates the View Audit button.
4. Click View Audit. The Modeling Audit Trail pop-up appears.
5. Click any arrow to expand the audit trail and see more details.
