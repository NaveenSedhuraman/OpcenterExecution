Viewing the Modeling Audit Trail Pop-Up

Modeling provides the ability to view the audit trail of existing modeling objects. You can access the Audit Trail pop-up from two locations:

• The modeling page for an existing instance of an object
• The Audit Trail page accessed from the Modeling menu

Accessing the Audit Trail Pop-Up from a Modeling Page

Modeling pages include an Audit Trail button when you select an existing instance of an object, such as Factory. The Audit Trail button is not available on the modeling object page for a new instance.

For example, if you click New on the Factory page, the application displays blank fields for you to add a new Factory instance. This page does not have an Audit Trail button because there is not yet an audit history for this object. However, an audit history is created for the instance when you save the new object, and the Audit Trail button appears on the page. This is also true when you create a new instance by copying an existing instance.

Clicking the Audit Trail button displays the audit trail of the object in the Modeling Audit Trail pop-up.

Accessing the Audit Trail Pop-Up from the Audit Trail Page

You can access the Audit Trail pop-up for a modeling object from the Audit Trail page. Select Audit Trail from the Modeling menu to view the Audit Trail page. The Audit Trail page allows you to search for Revisioned Objects, Named Data Objects, and Subentities based on criteria specific to the CDO type. You can then click the View Audit button to display the Modeling Audit Trail pop-up for that instance.

Refer to "Using the Audit Trail Page" for information.

When Viewing the Modeling Audit Trail Pop-Up

The Modeling Audit Trail pop-up initially provides a collapsed, or summary, view of the audit trail information.

The collapsed view of the audit trail does not provide information on subentities or lists. You must use the expanded view of the audit trail to drill down into subentity and list details. Click on the arrow icon to expand a line item to view its details, subentities, and lists.

The application displays the most recent change at the top of the list and the oldest change at the bottom of the list.

If you are using Change Management, the Package Name field allows you to limit the results to instances assigned to a specific change package. If no results are available for the specified package, the application displays a message indicated that no results met your search criteria.

Note: A change package must exist before adding or updating an instance for the package name to be updated. If the change is made prior to the package creation, the modeling object audit trail is not updated with the package name.

The Return Results between fields allow you to change the date range of results displayed. You can select dates from the calendar or enter them manually. If no results fall within your specified range, the application displays a message indicating that no results met your search criteria.

Note: The number of rows displayed by default is a configurable setting in Portal Studio.
Important: The application may time out when trying to view a modeling audit trail with a large number of records. You can increase the number of milliseconds in the time out setting in the XMLClientConfig.xml file. The XMLClientConfig.xml file is located in C:\Program Files (x86)\Camstar\InSite Utility by default.

When Printing the Modeling Audit Trail

You can print the modeling audit trail from the pop-up by pressing Ctrl+P on your keyboard. The audit trail will print exactly as it is displayed. You must expand the sections you want to view before printing the modeling audit trail.

How to View the Audit Trail Pop-Up from a Modeling Page

Follow these steps to view the Audit Trail pop-up from a Modeling page:

1. Open a modeling object, such as Factory. The selected modeling object page appears within the Modeling page.
2. Select an existing modeling object instance for which you want to view the audit trail. The application displays the information for that instance.
3. Click Audit Trail. The Modeling Audit Trail pop-up appears in the collapsed view.
4. Do you want to limit the results to a specific change package?

If . . .

Then . . .

Yes
a. Enter the package name in the Package Name field.
b. Click Search. The application returns all results associated with the specific package. If no results are associated with the package, the application displays an error message.
c. Go to step 5.

No

Go to step 5.
5. Do you want to limit the results to a specific date range?

If . . .

Then . . .

Yes
a. Enter the start date in the first Return Results between field or select the date from the calendar.
b. Enter the end date in the second Return Results between field or select the date from the calendar.
c. Click Search. The application returns all results that fell in the specified date range. If no results fell within the range, the application displays an error message.
d. Go to step 6.

No

Go to step 6.
6. Click any arrow to expand the audit trail and see more details.
7. Click Close to close the Modeling Audit Trail pop-up.

How to Print a Modeling Audit Trail

Follow these steps to print an audit trail:

1. Perform the "How to View the Audit Trail Pop-Up from a Modeling Page" procedure.
2. Press Ctrl-P. The Print pop-up appears.
3. Select the appropriate printer and settings.
4. Click OK. The application prints the Audit Trail page and Modeling Audit Trail pop-up exactly as they are displayed.
