Using the Audit Trail Page

You can view the audit trail for existing and deleted modeling objects using the Audit Trail page. The audit trail for an object instance exists even after the object has been deleted. Therefore, you can still view the audit trail for an object after the object has been deleted. The Audit Trail page enables you to view these object types:

• Named Data Object
• Revisioned Object
• Subentity

When Using the Modeling Audit Trail Page

Click in the CDO Type field and expand Named Data Object, Revisioned Object, or Subentity to select the appropriate modeling object. The fields on the page change depending on the CDO type selected.

Refer to these topics for information on using the Audit Trail page to view audit trails for the different CDO types:

• "Viewing the Audit Trail for a Named Data Object"
• "Viewing the Audit Trail for a Revisioned Object"
• "Viewing the Audit Trail for a Subentity of a Modeling Object"

The application activates the View Audit button after you have selected a CDO type and modeling object. Clicking this button displays the Audit Trail for the modeling object you have selected.

Note: Set your preferred language before defining modeling object instances if you want the application to display all updates in your preferred language in the Modeling Audit Trail.

Modeling Audit Trail Object Types

The object type you select in the CDO Type field on the Audit Trail page determines what other fields are displayed on the page.

This table defines the different fields that are displayed, depending on the object type selected.

If you select this type . . .

This field is displayed

Named Data Object

Name

This field corresponds to the particular NDO that is selected. For example, if you select an Employee as the Object Type Name, the Object Name field displayed is Employee.

Revisioned Object

Object Name, Revision

These fields corresponds to the particular RO that is selected. For example, if you select a Product as the Object Type Name, the Object Name field displayed is Product. The Object Revision field would also be associated with the Product.

Subentity

Parent CDO Type and Object Name

These fields corresponds to the particular subentity that is selected. For example, when you select Training Record as the Object Type Name, the Parent Type CDO field is displayed. When you select a parent type CDO, such as Employee, the Object Name field displayed corresponds to the parent type selected. In this example, the Object Name displayed is Employee.
