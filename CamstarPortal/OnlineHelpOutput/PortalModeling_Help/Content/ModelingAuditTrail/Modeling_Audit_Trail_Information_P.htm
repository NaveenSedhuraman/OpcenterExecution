Modeling Audit Trail Information

Modeling Audit Trail captures common audit data and detailed audit data. Common audit data is recorded regardless of the specifics of the object changes while detailed data includes the history of all persistent fields for the modeling object.

Common Audit Data

This common audit data information is recorded:

• User who performed the transaction
• Transaction date (local and GMT)
• Alternate key information of the object (such as Name for named objects, or Name and Revision for revisioned objects)
• Type of action executed (create, copy, edit, and so on)
• Comments

Detailed Audit Data

Detailed data information recorded for the audit trail includes the name of the attribute or field, the old value of the attribute and the new value of the attribute.

This table provides a list of type-specific audit data.

Audit Data Type

Information Recorded

Simple (Trivial) fields such as string, Boolean, and numeric

Old value and new value of the field.

Object references

Display name and alternate reference information for object reference fields (fields of type object where ownership = none).

Compositions
(for example, Operation.SchedulingDetail)

Subentity object name and the field, old value and new value of strongly owned objects.

List fields

Entry for each list item that was changed, including old and new values of the list item. Also indicates whether the update was an insert, change, or delete.

Trivial lists (lists that are not made up of objects)

Entire old and new image list.
