Disabling an Audit Trail

Opcenter EX MDD and Opcenter EX CR provide the ability to disable the generation of an audit trail at the maintenance transaction level. You must add the No Audit Trail CDOUsage to the modeling maintenance transaction’s CDOUsage mask in Designer. This disables any future collection of audit trail information.

Note: Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information on logging on to and using the Designer.

When Disabling an Audit Trail

Maintenance service objects are the only objects that support the modeling audit trail. The No Audit Trail attribute in the CDO Usages is not available for other types of objects and, by default, is not selectable. You must update your database to pick up any Designer changes. You must also use Management Studio to apply your Designer changes to the databases.

How to Disable an Audit Trail

Follow these steps to disable an audit trail for a modeling object:

1. Log on to Designer.
2. Click CDOs. Designer displays the CDOs and CDO detail tabs.
3. Select the CDO for which you want to disable the audit trail.
4. Click the General Tab for the selected CDO.
5. Click the Details button. The CDO Usage pop-up appears.
6. Select No Audit Trail.
7. Click OK. The audit trail is disabled for the selected CDO.
