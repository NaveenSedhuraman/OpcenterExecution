Defining Collaborator Templates

A Collaborator Template identifies employees who can collaborate with you on assigning content to a package. The collaborator template enables you to predefine collaborators and their instructions. The application populates the Collaborators grid and the Instructions for All Collaborators field on the Update Package page when you select a collaborator template. You do not have to enter all the information in the grid manually each time if you select a template.

Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information on enabling collaboration and updating a package.

When Defining Collaborator Templates

Note the following:

• You must specify a level for the collaboration work associated with each collaborator. Level determines whether the work occurs in parallel, sequentially, or both.
• You must select the roles assigned responsibility for the collaboration work.
• You must select the names of employees responsible for the collaboration work. Available employees are dependent on role.
• You can define general instructions for all collaborators and specific instructions for a single collaborator.
• You can define substitute collaborators to indicate whether employees in the same role as the collaborator can perform the collaboration work.

When Defining Role Substitution

You must perform the following tasks to enable substitute employees to perform collaboration work for the specified collaborator:

• Add the substitute collaborators (employees) and their organizations to the specified collaborator's Role modeling object. Substitute collaborators must belong to the same role as the specified collaborator. Refer to "Defining Roles" for information on updating the Role modeling object.
• When defining the collaborator template, select Role as the Substitute Option for the specified collaborator. Selecting Role enables a substitute to perform the collaboration work in place of the specified collaborator.
• When adding the substitute collaborators to the collaborator template, you must add them at the same level that you added the specified collaborator.

Collaborator Template Page Field Definitions

This table defines the fields unique to the Collaborator Template page and Add Collaborators pop-up.

Refer to "Common Fields on Modeling Pages" for information on the fields common to all modeling objects.

Field

Definition

Type

General

Instructions for All Collaborators

Instructions that apply to all collaborators associated with this template. You can enter a maximum of 4000 characters.

Optional

Collaborators grid

Grid listing all collaborator roles and corresponding collaborators for this template. The fields in this grid appear on the Add and Edit pop-ups.

Optional

Level

Level of collaboration associated with the collaborator. You can enter numeric values only. The grid is sorted by this column in ascending order.

Level defines the collaborator routing sequence. The application can route the collaboration work and e-mail notifications to collaborators in parallel, sequentially, or as a combination of parallel and sequentially.

Parallel indicates collaborators with the same level number can assign content at the same time. Serial indicates that collaborators with a lower level number must finish assigning content before collaborators with higher level numbers can begin.

Required

Role

Role assigned responsibility for the collaboration work. When selected, the Name list populates.

Required

Name

Employee assigned responsibility for the collaboration work. The names available depend on your selection in the Role field. The list updates if you change the setting in the Role field.

Required

Substitute Option

Value indicating whether employees other than the one specified in the Name column can perform the collaboration work:

• Role indicates any user in the same role, and defined at the same level in the Collaborators grid, can perform the collaboration work.
• None indicates only the specified user can perform the collaboration work.

Optional

Duration

Number representing the total time allowed for collaboration work.

Optional

Period

Unit of measure (Days, Hours, Weeks) for the duration in which the collaboration work should be completed.

Optional

Instruction for Collaborator

Specific instructions from the package owner to the collaborator. You can enter a maximum of 4000 characters.

Optional

Add Collaborators Pop-Up

The title of the pop-up used to add or edit collaborators in the Collaborators grid on the Collaborator Template page changes dynamically based on the action you are performing. Clicking Add new row on the Collaborators grid displays the Add pop-up. Selecting an existing collaborator in the Collaborators grid and clicking Edit selected row displays the Edit pop-up.

How to Define a Collaborator Template

Follow these steps to define a Collaborator Template:

1. Open the Collaborator Template page. The Collaborator Template page appears within the Modeling page.
2. Enter a name for this collaborator template in the Collaborator Template field.
3. Enter instructions for all collaborators in the Instructions for All Collaborators field.
4. Add at least one collaborator to the Collaborators grid by completing the "How to Add Collaborators to a Collaborator Template" procedure.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.

How to Add Collaborators to a Collaborator Template

Follow these steps to add collaborators to a Collaborator Template:

1. Perform the “How to Define a Collaborator Template” procedure.

Or

Select an existing Collaborator Template instance.

2. Click Add new row in the Collaborators grid. The Add pop-up appears.
3. Enter a numeric collaboration level in the Level field.
4. Select a Role.
5. Select a Name.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click OK. The application closes the Add pop-up and displays the collaborator information in the Collaborators grid.
8. Repeat steps 2-7 to add additional collaborators.
9. Click Save. The application displays a success message indicating the modeling object was updated.
