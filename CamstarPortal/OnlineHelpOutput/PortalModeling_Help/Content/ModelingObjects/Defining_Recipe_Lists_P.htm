Defining Recipe Lists

A Recipe List, a revisionable object, is a procedure to complete a step in a master recipe. A recipe list consists of one or more tasks necessary to complete the step. The application requires you to specify at least one recipe list when defining a master recipe. An example of a recipe list is a group of product-measuring data collection tasks related to the Master Recipe. Refer to "Defining Master Recipes" for information.

Understanding Tasks

A task is a unit of work required to be completed on a material container at a specification. You group tasks logically into Recipe Lists and assign one or more Recipe Lists to a Master Recipe. The recipe list enforces the defined task sequence when the recipe list is marked as sequential. Whether a task passes or fails depends on either data collected or the decision of the shop floor operator.

Task Item Types

You can choose the task item type when adding a task item.

Task Item Type

Description

Instruction

Any general type of task.

Computation

The application performs a calculation based on user constants, parameters, and operator input when the task is performed.

Recipe Issue Item

A task to issue a specified amount of product. You can also use this task to incorporate rules on how and what to issue.

Note: User constants and computations are also defined in Modeling. Refer to "Defining User Constants" and "Defining Computations" for information.

Task Types

The task type of a task item is important for those workstations that enforce line clearance. A task item can be one of the following task types.

Task Type

Description

Start Process

Denotes the beginning of processing at a workstation. A Start Process task sets the workstation status to “In Process” at either the Container level or the Manufacturing Order level. A Start Process task is required and must be the first task in the task list for workstations that enforce line clearance.

Note: The minimum iterations value for a Start process task must be greater than 0.

Regular

Regular tasks can occur between a Start Process task and an End Process task for workstations enforcing line clearance. A task list can consist of all Regular tasks for workstations that do not enforce line clearance.

A Regular task must be run against the same container or manufacturing order within the same task list, master recipe, and spec step as the Start Process task. Otherwise, it will fail.

End Process

Denotes the end of processing at a workstation. An End Process task resets the workstation status. The End Process task indicates the workstation is available to process the next container or manufacturing order. An End Process task is required and must be the last task in the task list for workstations that enforce line clearance.

Note: The minimum iterations value for an End Process task must be greater than 0.

Instruction Types

An instruction task can be one of the following instruction types.

Instruction Type

Description

Acknowledgment

Requires acknowledgment of task completion.

Data Collection

Collect parametric and user data from the manufacturing floor. Subsequent tasks can be dependent on results.

Pass/Fail

Requires notification of Pass or Fail.

Understanding Execution Mode

Enter the individual tasks in the order in which they must be completed when you create a recipe list. If the tasks do not have to be executed in a specific order, you can enter them in any sequence. The recipe list defines the execution mode for the tasks.

You indicate the execution mode for a task list by selecting Sequential or Non Sequential in the Execution Mode field on the Recipe List page:

• The default execution mode is Sequential. In this case, each recipe list is presented in the order established through a master recipe. A recipe list can only be skipped if all its tasks are optional.
• If the execution mode is Non Sequential, the application does not require the recipe list to be performed in a sequence defined through a master recipe. Likewise, the master recipe defines the execution mode for recipe lists.

Text Editor

Opcenter EX MDD and Opcenter EX CR provide a text editor that allows you to format Instructions in either Recipe Lists or Recipe List Tasks. Refer to "Defining a Task List and Related Tasks" for information.

When Defining Data Collection Tasks for Instruction Tasks

You must select Data Collection from the Instruction Type field and select a predefined data collection definition from the Data Collection Def field when you define a data collection task. If you associate a data collection definition with the task, but select a value other than Data Collection from the Instruction Type field, the application will not display the data collection fields on the EProcedure page.

When Defining Computation Tasks

The application requires the data type of the associated data point and computation parameter to be the same when defining a computation task. The application displays an error message if the data types are different. Data point data types are set in the specified data collection definition and computation parameter data types are set in the specified computation definition.

How to Define a Recipe List

Follow these steps to define a Recipe List:

1. Open the Recipe List page. The Recipe List page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the recipe list in the Recipe List field.
4. Enter the revision of this recipe list in the Revision field.
5. Select an Execution Mode for this recipe list.
6. Add at least one task to the Tasks grid using the following procedures:
• "How to Add an Instruction Task to a Recipe List"
• "How to Add a Computation Task to a Recipe List"
• "How to Add a Recipe Issue Item to a Recipe List"
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Save. The application saves the modeling object and displays a success message.

How to Add an Instruction Task to a Recipe List

Follow these steps to add instruction tasks to a recipe list:

1. Perform the "How to Define a Recipe List" procedure.

Or

Select an existing Recipe List instance.

2. Expand the Task Details section.
3. Click Add new row on the Tasks grid. The Select Task Item Type menu appears.
4. Select Instruction. The Instruction Task Details pop-up appears.
5. Enter a name for the task in the Task field.
6. Select Batch or Container from the Task Usage field.
7. Select a value from the Task Type field.
8. Select an instruction type from the Instruction Type field.
9. Enter a number in the Min Iterations field to indicate the minimum number of times the task can be executed.
10. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
11. Click OK. The application closes the pop-up and displays the task in the Tasks grid.
12. Click Save. The application saves the modeling object and displays a success message.

How to Add a Computation Task to a Recipe List

Follow these steps to add a computation task to a recipe list:

1. Perform the "How to Define a Recipe List" procedure.

Or

Select an existing Recipe List instance.

2. Expand the Task Details section.
3. Click Add new row on the Tasks grid. The Select Task Item Type menu appears.
4. Select Computation. The Computation Task Details pop-up appears.
5. Enter a name for the task in the Task field.
6. Select Batch or Container from the Task Usage field.
7. Select a value from the Task Type field.
8. Enter a number in the Min Iterations field to indicate the minimum number of times the task can be executed.
9. Select a computation from the Computation field. The application displays the input variables defined for the computation selected.
10. Map each variable to the appropriate data point by selecting the data point from the Data Point lists in the Computation Param Map grid.
11. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
12. Click OK. The application closes the pop-up and displays the task in the Tasks grid.
13. Click Save. The application saves the modeling object and displays a success message.

How to Add a Recipe Issue Item to a Recipe List

Follow these steps to add a recipe issue item to a recipe list:

1. Perform the "How to Define a Recipe List" procedure.

Or

Select an existing Recipe List instance.

2. Expand the Task Details section.
3. Click Add new row on the Tasks grid. The Select Task Item Type menu appears.
4. Select Recipe Issue Item. The Recipe Issue Item Task Details pop-up appears.
5. Enter a name for the task in the Task field.
6. Enter a number in the Min Iterations field to indicate the minimum number of times the task can be executed.
7. Select a Product.
8. Select an Issue Control.
9. Enter the quantity of product you expect to be dispensed at the recipe instruction for all of the target containers in the Qty Required Nominal field.
10. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
11. Click OK. The application closes the pop-up and displays the task in the Tasks grid.
12. Click Save. The application saves the modeling object and displays a success message.
