Defining Manufacturing Calendars

A Manufacturing Calendar translates the timestamp for a transaction into a specific manufacturing date and shift.

When Defining a Manufacturing Calendar

Siemens provides two ways to add shifts to a manufacturing calendar. You can import an Excel file with shift information or you can add each shift manually. You will enter every shift for every day when adding shift information manually.

The application requires you to export your calendar, add shifts manually, and re-import the calendar when adding new shifts to an existing calendar.

The application does not require you to enter shift start and end dates. However, the application requires you to enter both dates if you enter one date. For example, if you enter a shift start date and time, the application requires you to enter a shift end date and time.

The Fiscal columns in the Calendar Shift grid enable you to clarify your fiscal year for reporting purposes. For example, if your manufacturing calendar runs from April to March, but your shift calendar starts on January 2, 2013, you would enter the fiscal values as follows for your January 2nd entries:

• Fiscal Year = 2012
• Fiscal Quarter = 4
• Fiscal Month = 10
• Fiscal Week = 40

Mfg Calendar contains the optional modeling definitions Shift and Team.

Mfg Calendar is an optional field in the Factory and Work Center modeling definitions.

When Importing a Shift Calendar

The application imports Excel files in the *.xlsx format only. Earlier versions of Excel files are not supported.

Siemens provides a sample manufacturing calendar that you can customize to meet your needs. The ManufacturingCalendar.xlsx file contains a worksheet with instructions describing how to customize the calendar worksheet. The ManufacturingCalendar.xlsx file is located here by default:

C:\Program Files (x86)\Camstar\Camstar Portal

You must define the shifts and teams specified in the Excel file before importing. The application will not allow you to save the manufacturing calendar if the shifts and teams have not been defined in Modeling.

When Defining a Manufacturing Calendar for OEE
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can use the Non-scheduled Time field on the Calendar Shift grid to enter the time resources are not scheduled during a shift. Non-scheduled time cannot exceed the shift's duration.

How to Define a Manufacturing Calendar

Follow these steps to define a Manufacturing Calendar:

1. Open the Mfg Calendar page. The Mfg Calendar page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name in the Mfg Calendar field.
4. Refer to one of these procedures to load shift information:
• "How to Add Shifts Manually"
• "How to Load Shift Information from an Excel File"
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.

How to Add Shifts Manually

Follow these steps to add Shifts manually:

1. Perform the "How to Define a Manufacturing Calendar" procedure.

Or

Select an existing Manufacturing Calendar instance.

2. Click Add new row on the Calendar Shift grid. A new row appears.
3. Select a Shift and Team if importing shift information from Excel.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Repeat steps 2-4 for every shift on every day your site operates.
6. Click Save. The application displays a success message indicating the modeling object was updated.

How to Load Shift Information from an Excel File

Follow these steps to load shift information from an Excel file:

1. Perform the "How to Define a Manufacturing Calendar" procedure.

Or

Select an existing Manufacturing Calendar instance.

2. Click the File Upload icon to navigate to the Excel file.
3. Click Add Shift. The shift information is imported into the manufacturing calendar.
4. Click Save. The application displays a success message indicating the modeling object was updated.
