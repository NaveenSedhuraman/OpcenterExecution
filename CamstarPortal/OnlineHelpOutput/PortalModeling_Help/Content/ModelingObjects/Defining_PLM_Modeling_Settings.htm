Defining PLM Modeling Settings

The PLM Modeling Settings modeling object enables you to configure global settings in the MOM workspace that are required to download a Teamcenter package in to Opcenter EX MDD or Opcenter EX CR. These settings include:

• Product Type
• Thruput Reporting Level

The application may fail to download a Teamcenter package that contains objects that require these settings if they have not been configured before the package is downloaded by Opcenter EX MDD or Opcenter EX CR.

PLM Modeling Settings page fields are populated by instances defined in Modeling. You must define Product Type and Container Level (Thruput Reporting Level) instances in Modeling before configuring PLM Modeling Settings.

PLM Modeling Settings is an optional field on the Factory modeling page. You must configure PLM Modeling Settings on the factory associated with the logged-in employee to apply the settings to downloaded Teamcenter packages. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining Factory and Employee modeling instances.

PLM Modeling Settings Page Field Definitions

This table defines the fields on the PLM Modeling Settings page.

Field

Definition

Type

General

Product Type

Product Type instances that are defined in Modeling. Product Type user codes are used to differentiate between types of product such as WIP, Raw Material, and Finished Goods.

Note: Product Type is a required field on the Product modeling page.

Optional

Thruput Reporting Level

Container Level instances defined in Modeling. The application uses Thruput Reporting Level as a container level for calculation and reporting of thruput.

Note: Thruput Reporting Level is a required field on the Operation modeling page.

Optional

How to Define PLM Modeling Settings

Follow these steps to define PLM Modeling Settings:

1. Open the PLM Modeling Settings page. The PLM Modeling Settings page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the PLM modeling settings in the PLM Modeling Settings field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the PLM Modeling Settings page.
5. Click Save. The application saves the modeling object and displays a success message.
