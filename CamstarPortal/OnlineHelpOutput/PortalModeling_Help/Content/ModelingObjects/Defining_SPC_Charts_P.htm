Defining SPC Charts

The SPC Chart modeling object contains information required to display a single SPC chart. It describes the type of chart, the Statit Software macro, the Statit Software Rules file, Out of Control Actions and Procedures (OCAPs), and the variable being plotted.

Siemens provides four default SPC Charts for your use:

• Moving Range
• Range Chart
• Trial I-Chart
• X Chart
When Defining an Inline SPC Chart

You must install and configure Inline SPC on your server before using this page. Otherwise, you will not be able to populate all required fields or save the page.

You must create one SPC chart with the appropriate SPC Chart Type and SPC Query for each chart you want to display.

Use the Out of Control Actions section of the SPC chart to establish the actions triggered upon violation.

SPC Chart is an optional Modeling definition in SPC Chart Group.

SPC Chart contains the following optional Modeling definitions: Document, E-mail Message, Hold Reason, and Resource Status Code.

Refer to the Opcenter Execution Core Inline Statistical Process Control Reference Guide for information on setting up and configuring SPC.

 

When Defining a Legacy SPC Chart

You must create one SPC chart with the appropriate chart-generating macro for each chart you want to display.

Ensure the path for the chart macros and chart properties are valid. For example, if SPC is installed in a directory other than the default installation directory, you will need to change the paths.

Use the Out of Control Actions section of the SPC chart to establish the actions triggered upon violation.

SPC Chart is an optional Modeling definition in SPC Chart Group.

SPC Chart contains the following optional Modeling definitions: Document, E-mail Message, Hold Reason, and Resource Status Code.

Refer to the Opcenter Execution Core Statistical Process Control Reference Guide for information on setting up and configuring SPC.

Options to determine when the chart is displayed.

Required

Default height (in pixels) you want set for the chart.

Chart Width

Default width (in pixels) you want set for the chart.

Out of Control Actions

Trigger OCAPs (Out of Control Actions and Procedures)

Check box indicating whether to allow the chart to trigger the Out-of-Control Actions and Procedures when a violation occurs. If the check box is not selected, the Out-of-Control Actions and Procedures will not be triggered even if a violation occurs.

Optional

Put Container on Hold - reason code

Hold reason that will be assigned to the container. If a violation occurs, the container will be placed on hold with this as the reason for the hold.

Optional

Set Resource Status

Status reason that will be assigned to the resource when a violation occurs.

Optional

Document to View

Action setting the reference to the document to be viewed when an OCAP is triggered. Document instances can be created by modeling of the Document object. SPC does not support documents sets. Only individual documents can be viewed.

Optional

Require Annotation

Check box indicating whether an annotation will be required from the user when a violations occurs. The annotation will apply to the data point that caused the violation. The SPC Chart display pop-up cannot be closed until the data point has been annotated. If the user attempts to close the pop-up, a message will be displayed to inform the user that an annotation is required.

Optional

List of e-mail address (separate with commas)

E-mail addresses to which a notification is sent when a violation occurs. Separate each e-mail address with a comma.

The e-mail can include DCD name, variable name and value, chart details (such as CL, UCL, LCL), violation, and details of any Out-of-Control-Actions that were taken if you configure text variables. The server must have IE 9 or higher installed. IE must have its e-mail settings defaulted to the network e-mail server.

Note: Further e-mail option configuration is now possible in the application's App.Config file.

Optional

E-mail Message

List of messages defined in the E-mail Message modeling object.

Optional

Record against custom SPC data source

Check box indicating whether annotations or violations are recorded against a custom data source you provided rather than the application data collection. Ensure this box is selected if you are running charts on data outside of the Opcenter EX MDD or the Opcenter EX CR DataStore database.

Optional

Parameters

User Parameter (1-10)

User-defined values entered in a ParamName=Value format. You can enter multiple values in a field, separated by commas.

Optional

How to Define a New SPC Chart

Follow these steps to define a new SPC Chart:

1. Open the SPC Chart page. The SPC Chart page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the SPC Chart, Chart Type, Variable to chart, Chart Generating Macro, and SPC Chart Display Options.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.

How to Define a New Inline SPC Chart

Follow these steps to define a new SPC Chart:

1. Log on to Opcenter Execution Medical Device and Diagnostics.

Or

Log on to Opcenter Execution Core.

2. Select Modeling from the Modeling menu. The Modeling page appears.
3. Click SPC Chart. The SPC Chart page appears within the Modeling page.
4. Click New. Blank fields appear for you to define a new instance.
5. Enter the SPC Chart Name.
6. Select the Inline SPC checkbox to define an Inline SPC chart.
7. Select a Chart Type.
8. Select a Variable to chart.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information about the optional fields.
10. Click Save. The application saves the modeling object and displays a success message.
