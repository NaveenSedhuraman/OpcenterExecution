Defining Resource Status Models

A Resource Status Model identifies two things:

• The initial resource status code group that will be associated with an individual resource on the resource setup transaction. This means that any resource status code within the selected group is available for assignment to the resource when it is set up initially.
• The valid resource status code group that defines the allowed statuses to which a resource with a another status can transition. For example, a resource with the status code In Repair can only be assigned to a status that belongs to the status code group Stand By. In this way, the transition of a resource status can be restricted based on the defined model.

When Defining a Resource Status Model

You must define one or more resource status code groups before you can define a resource status model. Resource status model contains the required modeling definition: Resource Status Code Group.

Resource status model contains these optional modeling definitions:

• Carrier Status Code
• Resource Status Code
Note: The application does not require status codes when defining resource status models. However, at least one resource or carrier must be defined before you can select a value for the From Status Code in the Resource Status Diagram.

Resource status model is an optional field in these modeling definitions:

• Carrier
• Resource
• Resource Type

How to Define a Resource Status Model

Follow these steps to define a Resource Status Model:

1. Open the Resource Status Model page. The Resource Status Model page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the model in the Resource Status Model field.
4. Select an Initial Status Code Group.
5. Click Add new row in the Resource Status Diagram grid to add a blank row.
6. Select a From Status Code.
7. Select a To Status Code Group.
8. Repeat steps 5-7 to add more associations.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click Save. The application saves the modeling object and displays a success message.
