Defining Scheduled Business Rules

The Scheduled Business Rule modeling object is used to define a set of run options that you can apply to a business rule to determine when the application will execute that rule. You can specify whether the execution should occur as a one time event or on a recurring basis for an extended period of time.

Opcenter EX MDD and Opcenter EX CR run scheduled business rules as a service separate from any user-entered event. Because the schedule runs independently of any user service, a context is identified from which to reference parameter values within the business rule.

Opcenter EX MDD and Opcenter EX CR provide these default scheduled business rules:

• SBR_ACTIVATION - This scheduled business rule links to the BR_ACTIVATION business rule. It runs every 5 minutes.
• SBR_DEPLOY - This scheduled business rule links to the BR_DEPLOY business rule. It runs every 5 minutes.
• SBR_DEPLOYSTATUS - This scheduled business rule links to the BR_DEPLOYSTATUS business rule. It runs every 5 minutes.
• SBR_NOTIFICATIONS - This scheduled business rule links to the BR_NOTIFICATIONS business rule. It runs every 24 hours starting at 4:00 a.m.
Note: Start Date is populated with the installation date and time for each of these default scheduled business rules. Refer to "Defining Business Rules" for information on each of the default business rules.

When Defining Scheduled Business Rules

You can define scheduled business rules manually using the Scheduled Business Rule modeling object or automatically by executing a business rule with a script that creates a new scheduled business rule. The application enforces business rule usage at the point when business rules are assigned to other objects.

You select whether you want to use basic or advanced scheduling options when defining a rule using the Scheduled Business Rule modeling object. The advanced options provide additional flexibility in scheduling when the rule executes. Refer to "When Using the Basic Option for Defining Scheduled Business Rules" and "When Using the Advanced Option for Defining Scheduled Business Rules" for information.

You can specify a range of recurrence (start date and time, number of iterations, and end date and time) for the scheduled business rule whether you use the basic or advanced options.

The application deletes scheduled business rules from the database after the end date has expired. The scheduled business rule instance will not appear in the Instances list.

Start Dates and Times

You must specify a starting time that is later than the current time when specifying the current date as the start date for the schedule. For example, if the current time is 9:30 a.m., you must enter a time later than 9:30 a.m. for the application to run the schedule without errors.

Iterations

The Iterations field enables you to indicate the number of times that the application should run the rule. The application calculates an end date automatically. The End Date field takes precedence if you specify values for both Iterations and End Date. The application schedules the business rule indefinitely if you leave Iterations and End Date blank.

End Dates and Times

The application defaults the time to 00:00:00 (meaning that the rule ends as soon as the date changes) if you specify an end date but do not enter a time. If you have specified a number of iterations such that the application should execute an iteration on the end date, the application will not execute it—the iteration scheduled for the end date would occur after 00:00:00.

When Using the Basic Option for Defining Scheduled Business Rules

You must define a recurrence pattern (recurrence and frequency) when you define a scheduled business rule using the basic option.

Recurrence and Frequency

The following recurrence intervals are available when defining scheduled business rules:

• Once - Schedules run once at the scheduled date and time.
• Seconds - Schedules have a fixed number of seconds between execution.
• Minutes - Schedules have a fixed number of minutes between execution.
• Hourly - Schedules have a fixed number of hours between execution.
• Daily - Schedules have a fixed number of days between execution.
• Weekly - Schedules execute on a specific day of the week.
• Monthly - Schedules execute on a specific day of the month.
• Yearly - Schedules execute on a month and/or a day during the month.

Additional fields appear on the page if you select Weekly, Monthly, or Yearly in the Recurrence field. You must specify:

• Day of Week for the schedule to execute if you select Weekly.
• Either a numeric Day of Month or select the Is Last Day of Month check box if you select Monthly. (These two fields are mutually exclusive.)
• Month of Year and either complete the Day of Month field or select the Is Last Day of Month check box if you select Yearly.

All of the recurrence intervals support a frequency, which identifies a pattern for how often the requirement repeats: 1 indicates that the application execute the business rule on every scheduled recurrence; 2 indicates every other scheduled recurrence, and so forth.

Example 1

Specify Weekly with a Frequency of 2 and Wednesday as the Day of Week to schedule a business rule to run every other Wednesday.

Example 2

Specify Minutes with a Frequency of 20 to schedule a business rule to run every 20 minutes.

When Using the Advanced Option for Defining Scheduled Business Rules

An hourly pattern (hourly frequency and starting hour) must be specified when you define a scheduled business rule using the advanced options. You can also specify the specific days of the week, specific numerical days of the months, and specific months on which to execute the rule.

Hourly Pattern

The default values for Every <n> hours starting at hour <n> are 24 and 0. This means the rule executes every 24 hours starting at 12:00 a.m. on the start date. Valid values for hourly frequency are integers from 1-24. Valid values for starting hour are integers from 0-23.

Note: Ensure that you enter feasible values for the hourly pattern. For example, the rule will execute only at 5:00 a.m. if you specify Every 22 hours starting at hour 5 because 27:00 is not a valid time.

Specific Days of Week

The Specific Days of Week check box enables you to indicate whether you want to select the specific days of the week on which to execute the scheduled business rule. Additional fields for each of the seven days of the week appear on the page if you select the check box.

Specific Days of Month

The Specific Days of Month check box enables you to indicate whether you want to specify the day of the month on which to execute the scheduled business rule initially and the interval of days to execute the rule after that. The additional fields Every <n> days starting at day <n> appear on the page if you select the check box. The default value for each of these fields is 1. This means the rule executes every 1 day starting on day 1 of the start date. Valid values for these fields are integers from 1-31.

Note: Ensure that you enter feasible values for the Specific Days of Month pattern. For example, the rule will execute only on day 10 if you specify Every 25 days starting at day 10 because 35 is not a valid day.

Specific Months

The Specific Months check box enables you to indicate whether you want to select the specific months in which to execute the scheduled business rule. Additional fields for each of the 12 months of the year appear on the page if you select the check box.

Example 1

Specify the following to schedule a business rule that runs every 12 hours starting at 6:00 a.m. on Tuesdays and Thursdays:

• 12 and 6 for Every <n> hours starting at hour <n>
• Tuesday and Thursday for Specific Days of Week

The rule runs at 6:00 a.m. and 6:00 p.m. on Tuesdays and Thursdays.

Example 2

Specify the following to schedule a business rule that runs every 6 hours starting at 12:00 a.m. on the 15th day of the month and repeats every 5 days:

• 6 and 0 for Every <n> hours starting at hour <n>
• 5 and 15 for Every <n> days starting at day <n>

The rule runs at 12:00 a.m., 6:00 a.m., 12:00 p.m., and 6:00 p.m. on the 15th, 20th, 25th, and 30th day of the month.

Example 3

Specify the following to schedule a business rule that runs every 4 hours starting at 12:00 p.m. on the 1st day of the month and repeats every 10 days during the months of January and July:

• 4 and 12 for Every <n> hours starting at hour <n>
• 10 and 1 for Every <n> days starting at day <n>
• January and July for Specific Months

The rule runs at 12:00 p.m., 4:00 p.m., and 8:00 p.m. on the 1st, 11th, 21st, and 31st of January and July.

Defining Multiple Schedules

You may need to define multiple schedules for a business rule depending on your business needs.

Example 1

Your organization requires that the application execute a specific business rule at the end of each week and at the end of each month. Specify the following to define two scheduled business rules using the basic options: one weekly and one monthly:

• Weekly for Recurrence, 1 for Frequency, and Friday for Day of Week
• Monthly for Recurrence, 1 for Frequency, and Is Last Day of Month

Example 2

Your organization requires that the application execute a specific business rule twice daily: once at start of day and again at the end of day. The start of day occurs at 7:00 a.m. and the end of day occurs at 6:00 p.m. You can define one scheduled business rule using the advanced options. The application executes the rule in the morning and again in the evening for each business day.

Specify the following to configure the scheduled business rule:

• 11 and 7 for Every <n> hours starting at hour <n>, and Monday, Tuesday, Wednesday, Thursday, and Friday for Specific Days of Week

When Saving Scheduled Business Rules

If you try to save a new scheduled business rule with the same name as an existing one, the application indicates that a scheduled business rule by that name exists already. The application prompts you to enter a new schedule name.

The basic and advanced options are mutually exclusive. The application processes only one set of options. The values you enter for the basic and advanced options remain on the page until you save the Scheduled Business Rule instance. For example, you can specify basic options, select Advanced, specify advanced options, and then select Basic to view the values you specified previously. The options for both are present. The application saves only the values for the currently selected option (either Basic or Advanced) when you save the instance. The application clears the values for the other options.

The application makes these validations for the basic options before saving the scheduled business rule.

The application makes these validations for the advanced options before saving the scheduled business rule:

• The application saves the page with the default values for the required fields (Every <n> hours starting at hour <n>) if you do not enter values for them.
• The application clears the Specific Days of Week and Specific Months check boxes if you select either of these options, but then do not select any of the check boxes for the individual days or months.
• The application clears the values from the fields in the advanced sections (Specific Days of Week, Specific Days of Month, Specific Months) if you clear the check mark from the advanced section before saving. The section is not used.
• The application resets the values for Every <n> hours starting at hour <n> and Every <n> days starting at day <n> to their default values if you enter invalid values.

When the System Executes a Scheduled Business Rule

Opcenter EX MDD and Opcenter EX CR provide a scheduler service that periodically checks for scheduled business rules that need to be run. The frequency at which the scheduler service checks for schedules depends on the Polling Interval for Business Rules Scheduler setting in Management Studio. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information.

Note: Although you can define run frequencies to the second, the scheduler polling interval will still respect the Polling Interval for Business Rules setting in Management Studio.

Validation During Execution

Validation occurs during business rule execution to ensure the following:

• The parameters on the business rule match the parameters on the attached script handlers.
• The script on the script handler is syntactically valid.

Service History

The application records these details as part of the service history for each scheduled business rule run:

• Schedule name
• Business rule
• Completion status (Success or Failure)
• Run date (including time)
• Completion message
• Context
• Context value

How to Define a Scheduled Business Rule

Follow these steps to define a Scheduled Business Rule:

1. Open the Scheduled Business Rule page. The Scheduled Business Rule page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this scheduled business rule in the Scheduled Business Rule field.
4. Select the context on which the business rule will be evaluated from the Usage list.
5. Select the business rule to schedule from the Business Rule list.
6. Select the context value on which the business rule will be evaluated from the Context Value list.
7. Do you want to use the basic options or the advanced options?

If you want to use the . . .

Then . . .

Basic options
a. Select a value from the Recurrence list to specify when the application should run the business rule. Additional fields appear on the page if you select Monthly, Weekly, or Yearly.
b. Enter a value in the Frequency field to specify the interval at which the application should run the business rule.
c. Depending on your selection in the Recurrence list, enter a value or select the appropriate entry in the Day of Week, Day of Month, Is Last Day of Month, or Month of Year fields.
d. Go to step 8.

Advanced options

a. Select Advanced. The application hides the basic options and displays the advanced options.
b. Enter values for Every <n> hours starting at hour <n> if you want to specify settings other than the defaults.
c. Select one or more advanced options (Specific Days of Week, Specific Days of Month, Specific Months) if needed.
d. Depending on your selections in step c, enter values or select the appropriate check boxes in the advanced sections.
e. Go to step 8.
8. Enter (or select from the calendar control) the first date and time that you want the application to execute the business rule in the Start Date field.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click Save. The application saves the modeling object and displays a success message.
