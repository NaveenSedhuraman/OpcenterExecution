Defining Sampling Plans

The Sampling Plan page is used to organize the sampling activity that needs to occur for a given product.

When created a sampling plan is assigned to a product. Refer to "Assigning Sampling Plans to Products" for information.

Sampling Rate

A sampling rate must be defined for either the entire sample plan or each sample test. The sampling rate is a number that specifies the sampling frequency as every N containers, for example N=1 is every container, N=2 is every other container, N=3 is every third container, and so on. You can override the sample rate defined on a sampling plan to enforce inspection of all containers by selecting the Inspect All Containers check box on the Product page. Refer to "Assigning Sampling Plans to Products" for information.

You can change the sample rate by updating the sample rate counter on the Current Sampling Status Update page. You can override the sample rate for a sampling lot by defining a sample rate on the Create Sampling Lot page. You can change the sample rate for a lot by entering a new sample rate on the Update Sampling Lot page. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Shop Floor User Guide or the Opcenter Execution Core Portal Shop Floor User Guide for information.

Maximum Duration

Defining a maximum duration ensures that only containers produced within the maximum duration time frame are included when determining if sampling is required based on the sampling rate. If the maximum duration is exceeded by all relevant containers, the sampling counter is reset to prevent large time lapses from affecting the frequency of sampling.

For example, every 10th container is being inspected, but production stops at container 5. Production starts again one month later. Since the maximum duration is set at 15 days, the container count is now outdated, so it resets back to 1 and starts again. The 10th container is inspected, instead of the 5th, which would have been inspected had the counter not restarted. This process avoids comparing containers that are too far apart chronologically.

When Defining Sampling Plans

These rules apply when modeling the Sampling Plan:

• A value in the Details grid overrides the default value defined in the Processing section.
• The default sampling test values apply globally to every sample test unless preempted by the values specific to a particular sample test in the Details grid.
• Either the default field in the Processing section or the same-named field in the Details grid may be populated for these optional fields:
• Spec
• AQL Level
• Resource
• Vendor Item
• Sample Rate
• Inspection Level
• Switching Rule

AQL contains these optional modeling definitions:

• AQL Level
• Inspection Level
• Resource
• Spec
• Switching Rule

AQL contains the required modeling definition Sample Test.

If a sample plan is updated after the sampling requirements are determined and written to the container, the updated requirements are not immediately enacted. In other words, containers or lots in process do not pick up any revision updates. The next container or lot, however, does get these changes.

How to Define a Sampling Plan

Follow these steps to define a Sampling Plan:

1. Click Sampling Plan. The Sampling Plan page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the name for the product sampling plan in the Sampling Plan field.
4. Enter the revision of this sampling plan in the Revision field.
5. Perform the "How to Define Sampling Plan Details" procedure.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.

How to Define Sampling Plan Details

Follow these steps to define Sampling Plan details:

1. Perform the "How to Define a Sampling Plan" procedure.
2. Expand the Details section.
3. Click Add new row in the Sampling Plan Details grid. A new row appears.
4. Select a sample test from the list in the Sample Test field to make it part of this sampling plan.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Repeat steps 3-5 to define more sampling plan details.
7. Click OK. The Add pop-up closes and the application displays the Sampling Plan object.
8. Click Save. The application saves the modeling object and displays a success message.
