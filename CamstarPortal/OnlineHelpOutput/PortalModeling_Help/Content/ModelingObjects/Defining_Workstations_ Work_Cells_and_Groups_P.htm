Defining Workstations, Work Cells, and Groups

The following procedures enable you to define work cells, work cell groups, workstations, and workstation groups. The basic purpose of defining these entities is to provide detailed tracking of equipment without necessitating Move transactions.

Additionally, work cells, workstations, and their respective groups:

• Facilitate the grouping and assigning of tasks within the production environment.
• Accommodate line clearance.

There are many ways to set up work cells, workstations, and groups. These procedures explain one method.

Defining Work Cells, Workstations, and Groups

You define work cells, workstations and groups using the Resource, Resource Group, and Spec modeling objects. Depending on how you define these resources, you can restrict processing to a single container or quantity.

Note: As you establish your work cells, remember that a given work cell is always the parent to the workstations within it.

For a given work center, you may want to establish a resource group that contains all the resource groups for the child operations of the work center. This enables the Work Cell and Work Center fields on the Line Assignment page to display all values available for the work center.

Setting Up Workstations, Work Cells, and Groups

After defining workstations, work cells, and groups, you must set the availability of these resources and resource groups to Up on the Resource Setup page. Setting the availability to Up ensures the resource will appear in relevant resource selection values lists.

Additionally, the Resource Setup page allows you to specify a status code and status reason for resource history tracking and reporting purposes. For example, when a resource fails, you can perform a Resource Setup transaction to change the availability to Down, the Status to Maintenance, and the Reason to Repair. When the repair is completed, you can perform another Resource Setup transaction to return the availability to Up and the Status to In Production. You now have a record of when a failure occurred, the reason for the failure, and how long it took to repair.

Note: Resource status codes and status codes are defined in Modeling. A Resource Status Code provides a description of the current availability for a resource. A Resource Status Reason is used to provide an added level of detail describing the reason for the current status of a Resource.

Establishing Line Clearance at the Container Level

You can establish line clearance for a work cell or a workstation at the container level. Line clearance at the container level ensures that only a single container is processed at a given resource. You specify the clearance level in the Resource definition.

Line clearance at the container level is enforced when the shop floor user performs a Move or a Move In transaction on a container. The container must be at a workflow step with an electronic procedure assigned to a spec. The application prevents a second container from moving to the work cell or workstation until the first container has cleared.

Establishing Line Clearance at the Manufacturing Order Level

You can establish line clearance for a work cell or a workstation at the manufacturing order level. This feature permits containers associated with the same manufacturing order to be worked on within that particular work cell or at that particular workstation.

Line clearance at the manufacturing order level is enforced when the shop floor user performs a Move or a Move In transaction on a container. The container must be at a workflow step with an electronic procedure assigned to a spec. It is enforced when the shop floor user executes a Start Task in a task list.

The shop floor user chooses Work Cell or Work Station from the Resource list based on selection values provided through the Spec definition. The transaction fails if the work cell or workstation is already processing another manufacturing order. The transaction completes if the work cell or workstation is not processing another manufacturing order. As soon as the shop floor user moves the first container out of the step, it clears the work cell and a container associated with a different manufacturing order can be moved to and into the work cell and processed using electronic procedures.

Line clearance at the manufacturing order level can also be enforced when an electronic procedure is performed. In this case, a Move or Move In is not required. Rather, a Start Process task is performed to set the clearance and an End Process task closes it.

When Establishing Line Clearance

Following are the principles that govern line clearance. If clearance is set to:

• Container for a work cell then only one container can be processed within all workstations in the work cell at a time. A new container cannot move into the work cell until all activity at all workstations has completed for the current container.
• Container for a workstation then only one container can be processed at a workstation at a time.
• Manufacturing Order for a work cell then only containers that reference the same manufacturing order can be processed within all workstations in the work cell at a time. A new container from a different manufacturing order cannot move into the work cell until all activity at all workstations has completed for the current manufacturing order.
• Manufacturing Order for a workstation then only containers from one Manufacturing Order can be processed at a workstation at a time.

Resource Group Page

A Resource Group is a named object group subclass that represents a group of resources. An object group is used to provide a list of objects, such as a list of resources, products, or user codes of a particular type. These lists are typically used for validation and selection. For example, an Operation includes a reference to a LossCodeGroup. The entries in this list define the valid loss (reason) codes for work performed while a container is at that Operation.

An instance of an object group includes a list of specific objects and a list of object groups. This list of objects for any given object group is restricted to a specific type (or any of its derived types). Likewise, the list of object groups within the object group is restricted to the same type (or any of its derived types).

How to Define a Workstation

Follow these steps to define a Workstation:

1. Open the Resource page. The Resource page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the workstation in the Resource field.
4. Select a Factory.
5. Select a Clearance Level.
6. Select a Parent Resource (one of the resources created previously).
7. Enter optional information according to your business requirements. Refer to the "Resource Page Field Definitions" for information on this page.
8. Click Save. The application saves the modeling object and displays a success message.

How to Define a Workstation Group

Follow these steps to define a Workstation Group:

1. Open the Resource Group page. The Resource Group page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the workstation group in the Resource Group field.
4. Click Add new row in the Entries grid to add a workstation to this group. A new row appears.
5. Select a workstation from the list. This should be one of the workstations you created in the previous procedure.
6. Repeat steps 4-5 for each workstation you want to add to this workstation group.
7. Enter optional information according to your business requirements. Refer to the "" for information on this page.
8. Click Save. The application saves the modeling object and displays a success message.
