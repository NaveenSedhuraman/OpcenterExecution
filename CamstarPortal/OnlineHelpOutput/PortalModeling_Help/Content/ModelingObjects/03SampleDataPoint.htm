Defining Sample Data Points

The Sample Data Point object enables you to define the data points that are required as part of a sample test. The data point definition includes the data type and limits. The Sample Data Point object is only used when the sampling test is measured.

When Defining Sample Data Points

This rule applies when modeling Sample Data Points:

• Either the Upper Limit field or the Lower Limit field is required.

Sample Data Point contains the optional modeling definition UOM.

Sample Data Point is an optional field in the definition of Sample Test.

How to Define a Sample Data Point

Follow these steps to define a Sample Data Point:

1. Open the Sample Data Point page. The Sample Data Point page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the name of the data point instance in the Sample Data Point field.
4. Select a Data Type from the list.
5. Enter a value in the Upper Limit or the Lower Limit field, or enter a value in both to set upper or lower failure points.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.
