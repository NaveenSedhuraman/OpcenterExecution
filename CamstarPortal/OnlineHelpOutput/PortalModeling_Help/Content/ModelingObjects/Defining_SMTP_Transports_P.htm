Defining SMTP Transports

SMTP Transport is a subclass of Data Transport. The application uses Simple Mail Transport Protocol (SMTP) to send e-mail. This enables the application to send e-mail while configured to run as a Windows service. SMTP virtual host is part of the IIS 6.0 Component Services on Windows Server.

Note: If you want to link to your exchange mail server, you must enter a User Name, Password, and URL (name of the host server that matches the SSL certificate). The User Name and Password requirements are based on your e-mail server.

When Defining an SMTP Transport

SMTP Transport is an optional field in the following modeling definitions:

• Enterprise
• Factory
• Organization
• Site
Note: The SMTP server may need to be configured to allow anonymous users to connect to the SMTP server if the expected emails are not being received.

How to Define an SMTP Transport

Follow these steps to define an SMTP Transport:

1. Open the SMTP Transport page. The SMTP Transport page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the name of the SMTP transport in the SMTP Transport field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
