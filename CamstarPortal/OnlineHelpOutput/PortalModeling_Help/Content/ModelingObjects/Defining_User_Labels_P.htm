Defining User Labels

A User Label is text on the application's user interface. Labels include field captions, system messages, and so on. Siemens provides default system labels. These labels are used by applications in Opcenter EX MDD and Opcenter EX CR.

As with most of the application's configurations, you can take advantage of modifying the system labels without changing the original definitions. You can also create your own labels (User Labels) as instance data and do this conveniently through the Portal.

You can pass either the Label Name or Label ID to the application's services or the XML Client.

The application includes one subclass of the User Label modeling object: Message Category. Refer to "Defining Message Categories" for information on the Message Category modeling object.

When Translating System and User Labels

You can define Dictionary modeling objects to organize translations if you want to translate system and user labels. Dictionaries are a collection of customized labels that are assigned to employees.

When an employee with a designated Dictionary logs in to the Portal, the user interface presented to that employee displays the appropriate labels contained in the Dictionary. The default labels appear if an employee has no designated Dictionary.

Note: Refer to "Defining Dictionaries" for information on the Dictionary modeling object.

Language and Industry-Specific Labels

You can create labels that fall into these groups:

• Language-specific labels that pertain to vocabulary.

In this group, you can create language-specific labels and also call up Siemens labels for translation.

• Terminology-specific labels that are industry-specific.

For example, what the application terms a container can be called Lot or Bin in your factory; the Employee can be a Manufacturing Associate somewhere else. Terminology-specific labels are useful for business establishments with multiple divisions where industry terms cannot be applied across all divisions.

After you define the required labels, you can add them to dictionaries. Refer to "Defining Dictionaries" for information.

How to Define a User Label

Follow these steps to define a User Label:

1. Open the User Label page. The User Label page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the user label in the User Label field.
4. Enter the Label Value that will be displayed on the user interface.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. A label ID is generated for the label.
