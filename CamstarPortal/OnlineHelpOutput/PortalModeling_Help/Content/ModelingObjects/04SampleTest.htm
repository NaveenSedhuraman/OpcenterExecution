Defining Sample Tests

The Sample Test page allows you to create sample tests, which are then grouped to create sample plans. The two types of sampling tests are Counted and Measured.

Both sampling test types allow you to decrease the container by sample size (used for destructive testing), decrease the container by reject count, allow a move even if the container fails sampling, and automatically create a production event in the case of a failure.

If a sample test is updated after the sampling requirements are determined and written to the container or lot, these requirements are not enacted. Containers or lots in process do not pick up any revision updates. The next container or lot, however, does get these changes.

Understanding Sample Test Types

The two types of sampling tests are Counted and Measured.

Counted Sample Tests

A counted sample test allows you to record the number of samples tested and the number of rejected samples. Rejected samples are recorded by manually entering a count next to the appropriate loss reason. The list of loss reasons available when collecting sample data is determined from the AQL reject reason group. The AQL Reject Reason Group is required for a counted test. If rejects are to be scrapped, you have two options:

• Scrap the rejects using the associated loss reason selected on the Collect Sampling Data page by selecting the Scrap Counted Rejects By Reason check box.
• Scrap the rejects using the default reason from the Scrap Rejects Default Reason field.

Measured Sample Tests

A measured sample test enables you to enter one or more sample data point values that are then evaluated against limits. The sample is rejected if any of the values falls outside the defined limits. A sample test must have one or more sample data points selected if it is a measured sample test. If rejects are to be scrapped, select a loss reason from the Scrap Rejects Default Reason field.

When Defining Sample Tests

These rules apply when modeling the Sample Test:

• The AQL Reject Reasons Group field is required if the Sample Type is Counted.
• At least one Sample Data Point is required if the Sample Type is Measured. You cannot use Sample Data Points when the Sample Type is Counted.
• The following production event definition fields must be populated if a production event is to be created when a test fails. Refer to "Defining an Organization" for information. Leave these fields blanks if a production event is not to be created when a test fails.
• Default Failure Mode
• Classification
• Subclassification
• Default Production Event Description

Sample Test contains these optional modeling definitions:

• Failure Mode
• Classification
• Subclassification
• Loss Reason
• Loss Reason Group

The Scrap Rejects Default Reason and Decrease by Sample Size lists are both populated by Loss Reason, while the AQL Reject Reasons Group list is populated by Loss Reason Group.

Sample Test is a required field in the definition of Sampling Plan.

How to Define a Measured Sample Test

Follow these steps to define a measured Sample Test:

1. Open the Sample Test page. The Sample Test page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the sample test name in the Sample Test field.
4. Enter the revision of this sample test in the Revision field.
5. Select Measured from the Sample Type list.
6. Expand the Details section.
7. Click Add new row in the Sample Data Points grid. A new row appears.
8. Select either the ROR or a specific revision of the instance.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click Save. The application saves the modeling object and displays a success message.

How to Define a Counted Sample Test

Follow these steps to define a counted Sample Test:

1. Open the Sample Test page. The Sample Test page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the sample test name in the Sample Test field.
4. Enter the revision of this sample test in the Revision field.
5. Select Counted from the Sample Type list.
6. Select the loss reason group to be used for this sample test from the AQL Reject Reason Group list.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Save. The application saves the modeling object and displays a success message.

How to Add a Production Event to a Sample Test

Follow these steps to add a production event to a Sample Test:

1. Perform the "How to Define a Measured Sample Test procedure.

Or

Perform the "How to Define a Counted Sample Test" procedure.

Or

Select an existing Sample Test instance.

2. Expand the Production Event section.
3. Select a failure mode from the Default Failure Mode list.
4. Select a classification from the Classification list.
5. Select a subclassification from the Subclassification list.
6. Enter a description of the production event in the Default Production Event Description field.
7. Click Save. The application displays a success message indicating the modeling object was updated.
