Defining Text Variables

Text Variables are incorporated into message strings and are used to derive a value at run time. They are used by objects such as WIP messages, history inquiries, and so on.

Each Text Variable definition includes a name and an expression and describes how to treat the condition where the expression cannot be successfully evaluated. The name is the value embedded in the static string prefixed by a character that identifies it as a text variable. You must use the # symbol in front of the text variable instance name (for example, #CONTAINER) when referencing a text variable in an enabled string.

For example, a text variable with the name CONTAINER could be included in a message "#CONTAINER has been put on hold." The expression is a string to be evaluated at run time to produce the value that replaces the identifier within the run-time version of the string. In this example, the identifier would be replaced by the name of the container.

What constitutes a valid expression string varies based on the type of message referencing the text variable. Certain expressions (and therefore text variables) can be shared by all message types. For example, a text variable may be used to incorporate values for the current SessionValues. Other expressions will only make sense in the context of a particular type of message. For example, a WIP message may include a variable for the product currently running on the machine where the Move-In transaction is presenting the WIP message. Refer to "Defining WIP Messages" for information on using text variables with WIP messages.

Note: Text variables are only evaluated in string fields for which Designer settings are configured for their substitution. Refer to the Designer User Guide for information.

Text Variable Page Field Definitions

This table defines the fields unique to the Text Variable page.

Refer to "Common Fields on Modeling Pages" for information on the fields common to all modeling objects.

Field

Definition

Type

General

Expression

String to be evaluated to produce a value at run-time.

Optional

Usage

Defines the type of variable. For example:

• General (can be used in any text)
• WIP Message
• Alarm Message
• Instruction Text

Optional

Unresolved Value

Value to use in the event the expression cannot be successfully evaluated. This can be an empty string (to ignore the fact that the expression cannot be evaluated) or some other value (for example, "ERROR!") to make it clear that the text (of the resulting message) is not complete or accurate.

How to Define a Text Variable

Follow these steps to define a Text Variable:

1. Open the Text Variable page. The Text Variable page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the text variable in the Text Variable field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
