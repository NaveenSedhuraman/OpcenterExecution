Processing Modes

Electronic procedures accommodate both task-driven processing and workstation-driven processing.

• In task-driven processing, the shop floor operator does not select a specific workstation in the Line Assignment fields. In this case, the operator can view and complete the tasks of a given procedure from multiple workstations on the shop floor.
• In workstation-driven processing, the shop floor operator selects a specific workstation in the Line Assignment fields. In this case, the operator can only view and complete specific tasks filtered at that workstation.

Workstation-Driven Mode

When a shop floor operator has selected a particular workstation in the Line Assignment fields, the Electronic Procedures feature works in a very specific manner.

• The Electronic Procedure page filters tasks based on the workstation selected in the Line Assignment field.
• The Electronic Procedure page disables the existing Workstation field underneath the task grid.
• The shop floor operator must return to the Line Assignment fields and set a different workstation, if he or she wants to see or perform tasks for a different workstation. Subsequently, the operator returns to the Electronic Procedure page which refreshes and filters by the new workstation set in the Line Assignment fields.
