Defining Business Rule Handlers

A Business Rule Handler determines the action performed by a business rule. Business rule handlers are assigned to a business rule. Refer to "Defining Business Rules" for information on business rules.

The Business Rule Handler modeling object allows you to do one of the following:

• Define a script to implement business logic in a business rule
• Identify an existing service for the application to call

A script handler allows you to specify a condition and actions for the application to execute when that condition is true. A service handler allows you to identify an existing service for the application to call.

Opcenter EX MDD or Opcenter EX CR provides these default business rule handlers:

• BRH_ACTIVATION - Manages the activation status of the modeling instances associated with a change package on the target system.
• BRH_DEPLOY - Manages the deployment status for the change package when a user requests a package to be deployed.
• BRH_DEPLOYSTATUS - Manages the status of a change package on the source system from the time the export is completed until the package is delivered.
• BRH_NOTIFICATIONS - The script compares due dates for change management package approvals and collaboration work with information in the Package Approval Reminders grid and Content Collaboration Reminders grid in the Change Mgt Settings modeling object. It determines whether e-mail reminders need to be sent. Refer to "Defining Change Mgt Settings" for information.

Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information on exporting modeling objects from one system and then importing those objects on another system.

When Defining Business Rule Script Handlers

The script is a set of statements that optionally includes one or more parameters with default values that can be set by business rules using expressions. The application uses the default values specified for parameters to override script parameters at run time.

Configure the script using the Unified Expression syntax. This syntax allows you to create simple or complex expressions that can be evaluated based on where the business rule is assigned.

Important: Adding, removing, or updating parameters within a business rule handler requires that you also open and save each business rule that references that business rule handler. Otherwise, errors may occur when the associated business rules are executed. Refer to "When Viewing Existing Business Rules" for information.
Note: Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information on the Unified Expression Syntax.

Only those script handlers for which the syntax has been validated will be available for selection when creating a business rule.

When Defining Business Rule Service Handlers

When defining a service handler, the application allows you to add additional parameters that the application will execute at run time. Note that by adding parameters here, you are not modifying the actual service. You would need to use Designer to modify the service itself. Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information.

How to Define a Business Rule Script Handler

Follow these steps to define a Business Rule Script Handler:

1. Open the Business Rule Handler page. The Business Rule Handler page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this business rule handler in the Business Rule Handler field.
4. Select Script from the Handler Type field. The application updates the Business Rule Handler page to display the necessary script fields.
5. Enter the script to be executed in the Script field.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.

How to Define a Business Rule Service Handler

Follow these steps to define a Business Rule Service Handler:

1. Open the Business Rule Handler page. The Business Rule Handler page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this business rule handler in the Business Rule Handler field.
4. Select Service from the Handler Type field. The application updates the Business Rule Handler page to display the necessary service fields.
5. Select the service to execute from the Service field.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.

How to Add Parameters to a Business Rule Handler

Follow these steps to add parameters to a business rule script handler:

1. Perform the "How to Define a Business Rule Script Handler" procedure.

Or

Perform the "How to Define a Business Rule Service Handler" procedure.

Or

Select an existing Business Rule Handler instance.

2. Click Add new row on the Parameters grid. A new row appears.
3. Enter the name of the parameter from the script above in the Name field.
4. Select the Data Type from the list.
5. Select the Object Type if you selected Object from the Data Type field.
6. Enter the value for the parameter in the Default Value field.
7. Repeat steps 2-6 to add additional parameters.
8. Click Save. The application displays a success message indicating the modeling object was updated.
