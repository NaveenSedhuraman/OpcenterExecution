Defining Resource Types

The Resource Type identifies a type of equipment, such as Regular Maintenance Equipment, that can be associated with a Resource Status Model.

When Defining a Resource Type

Resource Type is an optional field in the Resource modeling definition.

Resource Type contains the optional modeling definitions Resource Status Model and UI Preference.

Resource Type Page Field Definitions

This table defines the fields unique to the Resource Type page.

Refer to "Common Fields on Modeling Pages" for information on the fields common to all modeling objects.

Field

Definition

Type

General

Resource Status Model

Resource status model associated with this resource type.

Optional

UI Preference

UI preference to associate with this resource type. A UI preference determines the information to display for a resource in the Status tab on the Resource Audit Trail page. UI preferences are defined in Modeling.

Optional

How to Define a Resource Type

Follow these steps to define a Resource Type:

1. Open the Resource Type page. The Resource Type page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the resource type in the Resource Type field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
