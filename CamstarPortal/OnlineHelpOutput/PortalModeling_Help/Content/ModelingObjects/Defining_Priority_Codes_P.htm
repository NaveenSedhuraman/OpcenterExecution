Defining Priority Codes

A Priority Code defines an indicator used to assign a processing priority to a container. Each priority code includes a description and a relative priority value. The application uses the relative priority value to sequence containers for dispatching (independent of the code name or description). Priority codes are only one of the algorithms used to determine a numeric value for scheduling and dispatching.

When Defining Priority Codes

Do not confuse the Priority Code modeling object with the Priority Level modeling object. Priority codes are assigned to containers while priority levels are assigned to events and quality records. Refer to "Defining Priority Levels" for information on defining event and quality record priority indicators.

Priority codes can be used as a search parameter on the Container Search page.

Do not use a space as the first or last character in the package priority code name.

How to Define a Priority Code

Follow these steps to define a Priority Code:

1. Open the Priority Code page. The Priority Code page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this priority code in the Priority Code field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
