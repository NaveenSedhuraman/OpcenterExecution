Defining User Data Collection Definitions

A User Data Collection Definition contains a set of data collection parameters. These data collection parameters are presented to users during activity completion.

User Data Collection Def is referenced by the activity template that uses it.

Each data collection parameter contains a set of data points. A data point represents a point in a business process where data can or must be collected. Once defined, multiple data points can be grouped within a user data collection definition.

User data collection definitions facilitate the capture of data as part of an activity, which can optionally be validated against the upper and lower limits as defined in the data point definition.

For example, you can specify a value data point such as a Boolean value. In this case, the upper and lower data points should be identical, and anything outside that value would result in a collection task failure which is recorded in History.

Alternately, you can specify an object data point. An object data point facilitates the capture of data while referencing a specific activity in your process model.

The application enables you to add:

• Value data points by using the Value Data Point Detail pop-up.
• Object data points by using the Object Data Point Detail pop-up.

Both are accessed from the User Data Collection Def page.

When Defining a User Data Collection

The application provides two basic layouts for user-based data collection:

• Row and column (individual fields)
• Iteration grid

You specify the layout in the Data Point Layout field on the User Data Collection Def page.

User Data Collection Def contains the optional Modeling definition: SPC Chart Group.

Row and Column Layout

The row and column layout offers the traditional field-based page.

Iteration Grid

The iteration grid layout offers a grid made up of multiple rows and columns. You can change the orientation of information displayed between vertical and horizontal views using the Direction button in the top-right corner of the grid.

When Adding Object Data Points

An object data point essentially serves the same purpose as a value data point—it offers a means of collecting data at a particular point in the production process. An object data point references a particular object. For example, you could use an object data point to establish a Reason Code to record pre-defined inspection status results. These results could include such conditions as contamination and surface scratches. The operator would complete the inspection task on the shop floor by simply choosing the appropriate result from a selection value list.

You cannot create two data points with the same row position and column position.

Note: Limits do not apply to object data points.

When Adding Value Data Points

When adding value data points to a user data collection, you can do the following:

• Define upper and lower limits by entering static values or an expression
• Indicate whether limit overrides are allowed
• Opt to map the data point value to a container attribute

You can use an expression to define the upper limit, lower limit, or both limits. The limits are determined at runtime when an expression is used.

Use the Opcenter EX MDD or Opcenter EX CR Unified Expression syntax when defining upper and lower limits with an expression. Expressions can contain a maximum of 255 characters and may include:

• Field expressions; for example, Container.CurrentStatus.Spec.UserDefinedInteger
• A combination of a field expression and elementary arithmetic; for example, (100-Container.CurrentStatus.Spec.UserDefinedInteger) *0.2
Note: There is no validation of correct syntax when entering expressions. Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information.

Allowing Limit Overrides for Value Data Points

When adding a value data point to the user data collection definition, you have the option of enabling a user to enter values outside of the lower and upper limits. Selecting the Limit Override Allowed check box results in the application shading the limit field in orange when a user enters a value outside of a limit. The application allows the user to execute the data collection.

Clearing the Limit Override Allowed check box results in the application shading the limit field in red when a user enters a value outside of a limit.

Note: This feature is available with the row and column layout as well as with the iteration grid.

Enabling Data Collection Tool Tips

You can use tool tips when collecting data. Simply hover the cursor over the task data entry fields to display the data point values: UOM and Lower and Upper Limits. This feature is available with the row and column layout as well as with the iteration grid.

How to Define a User Data Collection Definition

Follow these steps to define a User Data Collection Definition:

1. Open the User Data Collection Def page. The User Data Collection Def page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the unique name of this definition in the User Data Collection Def field.
4. Enter the revision identifier for this definition in the Revision field.
5. Select the Revision of Record check box if this is the revision of record.
6. Select the type of Data Point Layout from the list: Iteration Grid or Row Column Position.
7. Refer to one of the procedures below to add the selected type of data point:
• "How to Add a Value Data Point"
• "How to Add an Object Data Point"
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Save. The application saves the modeling object and displays a success message.

How to Add a Value Data Point

Follow these steps to add a value data point to the User Data Collection Definition:

1. Perform the “How to Define a User Data Collection Definition” procedure.

Or

Select an existing User Data Collection Def instance.

2. Expand the Details section.
3. Click Add new row in the Data Points grid and select Value Data Point in the Data Point Type pop-up. The Value Data Point Detail pop-up appears.
4. Enter a unique name for this value data point in the Data Point Name field.
5. Enter a numeric value to indicate the Row Position that the value data point must be placed within the task.
6. Enter a numeric value to indicate the Column Position that the value data point must be placed within the task.
7. Select the Data Type from the list.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click OK. The Value Data Point pop-up closes and the value data point appears in the Data Points grid on User Data Collection page.

How to Add an Object Data Point

Follow these steps to add an object data point to the User Data Collection Definition:

1. Perform the “How to Define a User Data Collection Definition” procedure.

Or

Select an existing User Data Collection Def instance.

2. Expand the Details section.
3. Click Add new row in the Data Points grid and select Object Data Point in the Data Point Type pop-up. The Object Data Point Detail pop-up appears.
4. Enter a unique name for this object data point in the Data Point Name field.
5. Enter a numeric value to indicate the Row Position that the object data point must be placed within the task.
6. Enter a numeric value to indicate the Column Position that the object data point must be placed within the task.
7. Select the particular object associated with this object data point from the Object Type list.
8. Select the value type to be used against the object type from the Selection Value Type list.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click OK. The Value Data Point pop-up closes and the value data point appears in the Data Points grid on User Data Collection page.
