Viewing Modeling Object Instance References

You can view all modeling object instances referenced by the selected instance. You can also view the modeling object instances that reference the instance selected on the Modeling page.

Note: The application does not display Filter Tags modeling object instances even if one is attached to the selected instance. Filter Tags is a comma delimited list that is not attached to the object as a reference.

When Viewing Referenced Instances

The Hierarchy view displays the selected modeling object instance and all of the instances it references. Clicking the Hierarchy button on the modeling object instance's page displays the read-only Hierarchy pop-up. The Hierarchy pop-up displays an indented tree view of the instances referenced by the selected object instance. The selected instance appears in bold, black font. The referenced instances appear in normal, black font.

The Hierarchy view includes indirect references also. For example, the application displays the Specs referenced by the Workflow that is referenced by the selected Product instance.

Note: The application may time out when trying to view a modeling audit trail with a large number of records. You can increase the number of milliseconds in the time out setting in the XMLClientConfig.xml file. The XMLClientConfig.xml file is located in C:\Program Files (x86)\Camstar\InSite Utility by default.

When Viewing Referencing Instances

The Where Used view displays the modeling object instances that directly reference the selected instance. Clicking the Where Used button on the modeling object instance's page displays the read-only Where Used pop-up. The Where Used pop-up displays all modeling objects and modeling object instances that reference the selected instance.

The Where Used pop-up does not display instances that reference the instance that references the selected instance. For example, the application does not display the Product that references the Workflows that reference the selected Spec instance. The application displays only the BOMs and Workflows that reference the selected Spec instance.

How to View Instances Referenced by the Selected Instance

Follow these steps to view instances that reference the selected instance:

1. Select a modeling object.
2. Select an instance.
3. Click Hierarchy. The Hierarchy pop-up appears.
4. Click Close to exit the pop-up.

How to View Instances that Reference the Selected Instance

Follow these steps to view instances that reference the selected instance:

1. Select Modeling from the Modeling menu. The Modeling page appears.
2. Select a Modeling object. The object's page appears within the Modeling page.
3. Select an existing instance. The instance and instance information appears.
4. Click Where Used. The Where Used pop-up appears.
5. Click Close to exit the pop-up.
