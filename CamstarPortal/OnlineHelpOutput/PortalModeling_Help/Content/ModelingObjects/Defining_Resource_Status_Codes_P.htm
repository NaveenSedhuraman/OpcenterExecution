Defining Resource Status Codes

A Resource Status Code describes the status that can be assigned to a resource, both initially at setup and later for each status change. Some examples could be Available, In Use, and Unavailable.

When Defining a Resource Status Code

Resource Status Code is an optional field in the Resource Status Code Group and Resource Status Model modeling definitions.

Resource Status Code is a required field in the Resource Setup transaction.

When Defining a Resource Status Code for OEE
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

The application must summarize resource status code data for each shift to perform OEE calculations. The application summarizes the data at the end of each shift using a stored procedure to calculate the time the resource was in each status.

You can use the OEE Loss Category field on the Resource Status Code page to indicate the type of loss for each scheduled downtime. Options include:

• Availability - Indicates the time in this resource status code counts as availability loss for a resource. The application includes the time in the Scheduled Downtime calculation.
• Performance - Indicates the time in this resource status code counts as performance loss for a resource.
• Schedule - Indicates the time in this resource status code counts as schedule loss for a resource. The application includes the time in the Unscheduled Downtime calculation.

How to Define a Resource Status Code

Follow these steps to define a Resource Status Code:

1. Open the Resource Status Code page. The Resource Status Code page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the status code in the Resource Status Code field.
4. Select the Availability associated with this status code.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.
