Electronic Procedures Definitions

These definitions are provided to help you understand electronic procedure functionality.

Electronic Procedures

An Electronic Procedure consists of one or more pre-defined task lists—the steps necessary to complete a spec. When you assign an electronic procedure to a spec, you cannot move a container to the next spec in the workflow until all required tasks in the electronic procedure have been completed. Additionally, electronic procedures provide an electronic record that an individual task was completed.

Note: A required task is a task for which the Min Iteration value is greater than zero. Refer to "Defining a Task List and Related Tasks" for information.

Electronic procedures facilitate the movement of quantities across a collection of workstations without requiring a Move transaction. Significantly, this feature accommodates processing in which quantities are processed at specific workstations within a chosen work cell and not processed at workstations outside the work cell.

Workstations

A Workstation is the resource you use to complete a task list. You can assign a workstation to a task list to specify that the work be completed on that resource. A workstation divides a work cell into multiple stations and should belong to one work cell only.

Workstation Groups

A Workstation Group identifies multiple workstations that perform the identical or equivalent processing for a task list. The workstations in a workstation group should be assigned to different work cells.

Work Cells

A Work Cell is a location where all the tasks assigned to an electronic procedure are performed. When a container begins processing within a work cell, all subsequent tasks for the container must be executed at that work cell. An entire electronic procedure is processed within a work cell.

While a work cell definition is not required, it is highly recommended. The concept of the work cell offers the flexibility of an additional boundary that can be used like any other resource at the time a container is moved in at a spec step. The work cell can also provide line clearance enforcement if you have configured this feature.

Work Cell Groups

A Work Cell Group identifies multiple work cells that perform the same type of processing. This enables the shop floor operator to choose which work cell will be used to process a container. It also ensures that all processing stays within the selected work cell.

Tasks and Task Lists

A Task is a unit of work required to be completed on a container at a specification. You group tasks logically into Task Lists and assign one or more task lists to an electronic procedure. The electronic procedure enforces the defined task sequence when the task list is marked as sequential. Whether a task passes or fails depends on either data collected or the decision of the shop floor operator.

Data Collection

Data points are points in the production process where data can or must be collected. You can define data points in a data collection definition and assign that definition to a task in a task list when you need to collect data for the task. Data collection definitions are defined in Modeling.

Data points facilitate defined upper and lower limits. The layout of the data points can be either row and column positions or an iteration grid.

Data Point Collection Definition

When the shop floor operator collects data for a task, there are two methods:

• Data collection used with electronic procedures

Data collection definitions facilitate the capture of parametric and user data from the shop floor. Optionally, the application can validate the captured data against the upper and lower limits defined in the data collection definition.

For example, you can specify a value data point such as a Boolean value. In this case, the upper and lower data points should be identical. Therefore, anything outside that value would result in a collection task failure which is recorded in History. In addition, a History record is created for each data point collected.

Alternately, you can specify an object data point. An object data point facilitates the capture of parametric data while referencing a specific object in your factory model.

• Data collection wizard

This is the standard data collection that is defined through Designer or the wizard. This data collection can be used for data collection on a task. Refer to the Data Collection User Guide for setup instructions.

Line Clearance

Line clearance provides processing exclusivity. You can configure electronic procedures to enforce line clearance at workstations or work cells at the container level or manufacturing order level. Line clearance ensures that only a single container or a specific quantity of the container is processed at a given resource. You specify the clearance level in the Resource definition.
