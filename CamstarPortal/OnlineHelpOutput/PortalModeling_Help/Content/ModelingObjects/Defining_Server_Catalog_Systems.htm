Defining Server Catalog Systems

A Server Catalog System identifies a server within Opcenter EX MDD or Opcenter EX CR from which you can send or receive data packages. You can define a server catalog system for your local server and all accessible (target) servers in your organization.

When Defining Server Catalog Systems

The Server Catalog System page enables you to define two types of server locations:

• Local System - The required Opcenter EX MDD or Opcenter EX CR target server
• Is Not a Camstar Server - Additional servers for processing that are not currently running Opcenter EX MDD or Opcenter EX CR

You must ensure that the source server and target servers contain a valid SSL certificate. You must use the server's fully qualified domain name listed on the certificate when defining the IP Address field.

Local System

You can define only one server as the local server. The local server is the server from which any data package you want to push to target servers originates. Selecting the Is Local System check box on the Server Catalog System page sets that server as the local server. You must define at least one local server with Is Local System selected.

The application displays an error message if you try to save an instance as a local server when another instance has been defined as the local server already. The application requires you to first clear the Is Local System check box on the existing local server instance before you can specify a different server as the local server.

The application uses the fields in the Server Details section to compare the attributes (properties) of the local and target servers when you are sending or receiving data packages. The application displays a warning message when the attributes do not match, but it does not prevent you from completing the transaction. For example, the application displays a warning message if the version entered for the local server is 6 and the version entered for the target server is 5.8.

Specifying an Active Directory domain enables the application to map from the host server to the Temp folder on a target server when the two servers are on different domains. The application attempts to map from the host server to the target server using the host server's domain if you do not specify a domain for the target system.

Is Not a Camstar Server

You can define additional processing folder locations on servers which are not running Opcenter EX MDD or Opcenter EX CR. Servers that are not currently running a version of the product are not available for review on the Activation Impact page.

How to Define a Server Catalog System

Follow these steps to define a Server Catalog System:

1. Open the Server Catalog System page. The Server Catalog System page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the server in the Server Catalog System field.
4. Enter the version of the MDB installed on the server in the MDB Version field.
5. Enter the version of Opcenter EX MDD or Opcenter EX CR installed on the server in the Camstar Version field.
6. Enter the user name authorized to access this server in the User Name field.
7. Enter the password for the user name in the Password field.
8. Enter the server's fully qualified domain name that matches the SSL certificate in the IP Address field.
9. Enter the port in the Port field.
10. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
11. Click Save. The application saves the modeling object and displays a success message.
