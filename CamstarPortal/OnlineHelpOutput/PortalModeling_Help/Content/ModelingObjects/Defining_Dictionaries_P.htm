Defining Dictionaries

Opcenter EX MDD and Opcenter EX CR enable you to create custom or translated user and system labels for the application. Labels in the application are messages and text, such as displayed field captions, error messages, transaction completion messages, string selection values, and so on. Custom and translated labels are organized and assigned to employees by using dictionaries.

You can group custom or translated labels under two types of dictionaries:

• Language - stores translated labels for specific languages.
• Terminology - stores custom labels for industry-specific languages. For example, Opcenter EX MDD and Opcenter EX CR use the word Container to represent a discrete collection of product produced in a factory, and uses Employee to represent a person. You might want to use different words, such as Bin instead of Container and Associate instead of Employee, for your business.

The application displays labels based on the logged-in employee's associated dictionaries. The default labels and user-defined labels appear if an employee has no designated dictionary. Refer to "Defining Employees" for information on assigning dictionaries to an employee.

The application allows you to assign both Language and Terminology dictionaries to an employee. Labels in the terminology dictionary are prioritized over labels in the language dictionary if labels exist in both dictionaries.

Dictionary labels are cached in both Application Server and Portal application caches. Opcenetr EX MDD and Opcenter EX CR dictionary labels are stored within the database and cached in Portal's IIS cache. You must recycle the IIS application pool to refresh the cache. The dictionary label cache is updated when a dictionary modeling instance is updated.

On saving of a dictionary Modeling instance, the label cache is automatically refreshed out of the box if the save includes changes.

You specify these dictionary types, or categories in the employee definition for assigning a dictionary to an employee.

You can optionally load dictionary labels using custom CSV files in Management Studio. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on loading custom dictionaries.

How to Define a Dictionary

Follow these steps to define a Dictionary:

1. Open the Dictionary page. The Dictionary page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the dictionary in the Dictionary field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application displays a success message indicating the modeling object was updated.
