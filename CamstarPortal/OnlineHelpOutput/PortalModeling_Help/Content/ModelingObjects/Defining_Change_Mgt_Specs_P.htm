Defining Change Mgt Specs

A Change Mgt Spec (specification) defines the activities carried out at a step in a change management workflow. The application requires you to reference at least one change management spec when you define change management workflows. One spec can be referenced multiple times by different workflows. Change Mgt Spec is a subclass of the Spec modeling object.

You can use the provided change management specs in your own custom change management workflows. Additionally, you can customize the provided specs to meet your business process needs.

When Defining a Change Mgt Spec

You can:

• Specify the package status used when a package is moved to a change management step.
• Specify a custom image to display in the package information header. This image indicates the spec (step) the change management package is in. The custom image must be:
• Located in the C:\Program Files (x86)\Camstar\Camstar Portal\ folder
• Saved in the *.png format
• Sized to 20 x 20 pixels
• Assign a role or roles for transactions allowed at the spec. You define roles using the Role modeling object. Refer to "Defining Roles" for information. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on the change management default roles, permissions, and allowable transactions.
• Specify whether the approval process for the change management package must be reset when the package re-enters the spec. Resetting the approval process cancels pending or completed approvals and requires approvers to re-approve the change management package when re-routed for approval. The application moves any capture approvals to history.
Note: You can view a history of canceled approvals on the Approval Status tab on the Approve Package page.
• Specify whether the Approvers tab appears on the Update Package page when the package is at that step in the workflow.
• Specify whether a change management package requires an approval at this spec. The application uses this check box for change control enforcement to determine if the package is on an approval workflow. Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information on change control enforcement.
Note: You use the Master Data Catalog modeling object to specify change control enforcement for modeling objects. Refer to "Defining a Master Data Catalog" for information on setting change control enforcement.
• Specify whether modeling instances in the change management package must be locked when the package enters this spec.

When Locking Change Package Modeling Object Instances

Selecting the Lock modeling instances in a Change Package at this Spec check box locks all modeling instances in the change management package when the package is moved to this spec.

Note: You cannot modify or delete a locked change package modeling object instance. The application displays a warning message when you try to modify or delete a locked change package modeling object instance.

The application unlocks the instances when:

• The change package is moved to a spec that does not have this check box selected, and
• All packages that reference these instances are at a spec that does not have this check box selected.

How to Define a Change Mgt Spec

Follow these steps to define a Change Mgt Spec:

1. Open the Change Mgt Spec page. The Change Mgt Spec page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this spec in the Change Mgt Spec field.
4. Enter the revision of this object in the Revision field.
5. Select a Package Status.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.

How to Assign a Role or Roles to Allowable Transactions

Follow these steps to assign a role or roles to allowable transactions:

1. Perform the "How to Define a Change Mgt Spec" procedure.

Or

Select an existing Change Mgt Spec instance.

2. Expand the Transactions section.
3. Click Add new row. A new row is added to the grid.
4. Select a role or roles from the list to assign to the allowable transactions.
5. Repeat steps 3-4 to assign additional rows.
6. Click Save. The application displays a success message indicating the modeling object was updated.
