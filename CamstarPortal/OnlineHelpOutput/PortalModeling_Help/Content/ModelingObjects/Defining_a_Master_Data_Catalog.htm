Defining a Master Data Catalog

A Master Data Catalog provides the ability to enforce change control on all or a subset of modeling objects. The application uses the master data catalog to ensure modeling object instances are assigned to change management packages on the appropriate workflows. Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information on change management and enforcing change control.

When Defining a Master Data Catalog

You can define only one master data catalog. The application prevents creation of more than one master data catalog by preventing:

• Package association - The Pkg Assoc toolbar button is not available for the master data catalog object.
• Addition of instances - Master data catalog is not available for selection in the Objects grid.
• Change Mgt Save - The application displays an error message when Change Mgt Save is set to On while saving the master data catalog.

You can specify the modeling objects for which change control should be enforced.

The master data catalog provides two types of change control enforcement:

• Enforcement for instances in a package on any workflow - The application allows you to assign a modeling object instance to a package associated with any change management workflow.
• Enforcement for instances in a package on a workflow with an approval step - The application allows you to assign a modeling object instance to a package associated with a change management workflow containing an approval step.

You can select only one type of enforcement.

The Objects grid displays all modeling objects by default. You can filter the Objects grid to display only those for which change control is enforced or only those for which change control has not been enforced.

The application displays an error message when you save a modeling object instance for which change control has been enforced and:

• You have not associated the instance with a package, or
• You assigned the instance to the wrong workflow (for example, the modeling object instance must be assigned to a workflow containing an approval step and you assigned it to a non-approval workflow).

How to Define a Master Data Catalog

Follow these steps to define a Master Data Catalog:

1. Open the Master Data Catalog page. The Master Data Catalog page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this master data catalog in the Master Data Catalog field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
