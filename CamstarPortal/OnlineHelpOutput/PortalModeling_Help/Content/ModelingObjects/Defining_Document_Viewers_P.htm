Defining Document Viewers

A Document Viewer specifies an application used to view documents defined within Opcenter EX MDD or Opcenter EX CR. A document viewer definition contains the location of the application and a command line string that launches the program to display the document. For example, you can define document viewers that use Word, Excel, or Portable Document Format (PDF) to view various documents.

Note: The default implementation of Opcenter EX MDD or Opcenter EX CR does not use this object; however, the object is available for your use in your custom implementation.

Document Viewer Page Field Definitions

This table defines the fields unique to the Document Viewer page.

Refer to "Common Fields on Modeling Pages" for information on the fields common to all modeling objects.

Field

Definition

Type

General

Current Directory

Directory from which the viewer application is launched.

Optional

Command Line

String with the name of the application to launch and any required arguments. The string includes an identifier that is replaced with a Document identifier when the command string is constructed at run time.

For example, the command string AcroRd32.exe%1 launches the Acrobat® Reader® application and replaces %1 with a Document identifier.

Optional

How to Define a Document Viewer

Follow these steps to define a Document Viewer:

1. Open the Document Viewer page. The Document Viewer page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the name of the document viewer in the Document Viewer field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
