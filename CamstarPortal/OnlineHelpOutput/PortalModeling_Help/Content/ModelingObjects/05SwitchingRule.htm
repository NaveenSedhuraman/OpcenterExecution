Defining Switching Rules

The Switching Rule page allows you to define the rules used to decide whether to move a sampling plan from one inspection level to another. Switching rules enable a sampling plan to change to a reduced or tightened inspection plan based on the number of passed or failed containers in a sequence of consecutive containers. A sampling plan enables defining switching rules for either the entire plan or for each sample test. A switching rule is evaluated based on X passes or fails per Y containers or on an expression.

Switching rules do not apply to lots.

When Defining Switching Rules

These rules apply when modeling a Switching Rule:

• The Pass (or Fail) Container Count, Last Container Set Count, and Passed/Failed fields are required if the Expression field is not populated.
• The Expression field is required if the Pass (or Fail) Container Count, Last Container Set Count, and Pass/Fail fields are not populated.

When the switching rule criteria is met, a switching rule inspection plan change can be automatically triggered or require manual intervention after an e-mail notification has been sent to a decision maker.

Similar to sampling plans, switching rules can have a maximum duration. As part of the sample plan, a maximum duration may be defined for the switching rule. Therefore, only containers processed within the maximum duration time frame are included when evaluating switching rules.

Switching Rule contains these optional modeling definitions:

• E-mail Message
• E-mail Distribution

Switching Rule contains the required modeling definition Inspection Level.

Switching Rule is an optional field in the definition of Sampling Plan.

How to Define a Switching Rule

Follow these steps to define a Switching Rule:

1. Open the Switching Rule page. The Switching Rule page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the name for the switching rule in the Switching Rule field.
4. Enter the revision of this switching rule in the Revision field.
5. Perform the "How to Define Switching Rule Details" procedure.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.

How to Define Switching Rule Details

Follow these steps to define Switching Rule details:

1. Perform the "How to Define a Switching Rule" procedure.
2. Expand the Details section.
3. Click Add new row in the Switching Rule Details grid. A new row appears.
4. Select the inspection level that is the starting level for this switching rule from the list in the From Inspection Level field.
5. Select the inspection level to be switched to when the switching rule criteria are met from the list in the To Inspection Level field.
6. Define the switching rule conditions by defining the components or an expression as follows.

If you are defining the conditions
using . . .

Then . . .

The components
a. Enter in the Pass (or Fail) Container Count field the X value of the expression If X out of Y containers pass/fail, then engage the switching rule. Pass versus Fail is designated in Step c.
b. Enter in the Last Container Set Count field the Y value of the expression If X out of Y containers pass/fail, then engage the switching rule. Pass versus Fail is designated in Step c.
c. Select Passed or Failed from the list in the Passed/Failed field depending on whether Step a and b count passed or failed sample tests.
d. Go to step 7.

An expression
a. Enter in the Expression field the unified expression that can replace the previous steps in determining if the switching rule is triggered.
b. Go to step 7.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Repeat steps 3-7 to define more switching rule details.
9. Click OK. The Add pop-up closes and the application displays the Switching Rule page.
10. Click Save. The application saves the modeling object and displays a success message.
