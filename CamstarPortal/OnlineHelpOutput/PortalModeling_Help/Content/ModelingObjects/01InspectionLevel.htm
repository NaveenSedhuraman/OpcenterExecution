Defining Inspection Levels

The Inspection Level modeling page establishes the sample size code based on a minimum and maximum container or lot size. Examples of inspection levels include Normal and Tightened. The sample size code is used to determine the sample size and allowable reject limit which is defined as part of the AQL level.

For container sampling, the application uses a container's current quantity by default when determining the sample size and allowable rejects for a sample test. You can specify that the application use the original container quantity instead.

For lot sampling, the application uses the lot's quantity by default when determining the sample size and allowable rejects for a sample test. You can specify that the application use the current quantity of all containers assigned to the lot if the lot quantity is exceeded.

When Defining the Inspection Levels

This rule applies when modeling the Inspection Level:

• The Inspection Level page must include at least one item in the Lot Size Details grid.

Inspection Level is:

• A required field in the definition of Switching Rule (From Inspection Level and To Inspection Level).
• An optional field in the definition of Sampling Plan.

How to Define an Inspection Level

Follow these steps to define an Inspection Level:

1. Open the Inspection Level page. The Inspection Level page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the inspection level in the Inspection Level field.
4. Click Add new row in the Lot Size Details grid. A new row appears.
5. Enter the minimum container or lot quantity for the associated sample size code in the Min. Qty field.
6. Enter the maximum container or lot quantity for the associated sample size code in the Max. Qty field.
7. Enter the Sample Size Code to be used to determine the sample size and allowable rejects for this inspection level.
8. Repeat steps 4-7 for each lot size.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click Save. The application saves the modeling object and displays a success message.
