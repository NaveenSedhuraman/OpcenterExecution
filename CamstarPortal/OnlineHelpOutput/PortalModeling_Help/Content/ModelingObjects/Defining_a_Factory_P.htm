Defining a Factory

A Factory is any division, department, or group that is separated for accounting and reporting purposes. An enterprise can contain one or more factories.

It often represents a physical manufacturing building, but can also represent the company itself, or a large division within the company.

A factory can contain references to an enterprise and to locations within the factory.

When Defining a Factory

The SMTP Transport field uses Simple Mail Transport Protocol (SMTP) to send e-mail. This enables Opcenter EX MDD and Opcenter EX CR to send e-mail while configured to run as a Windows service.

A Portal user can specify a factory as the discovery area when creating an event.

Factory contains the optional modeling definitions Enterprise, Mfg Calendar, Dispatch Rule, Training Requirement Group, Print Queue, Container Numbering Rule, SMTP Transport, and Change Mgt Settings.

Factory contains the optional modeling definition PLM Modeling Settings if you are integrating with Teamcenter. Refer to the Opcenter Execution Medical Device and Diagnostics Installation Guide or the Opcenter Execution Core Installation Guide for information.

Factory is:

• An optional field you can specify when defining Employee, Enterprise, MfgOrder, Resource, and Shipment Destination.
• An optional field in most Opcenter EX MDD or Opcenter EX CR transactions.
Note: A factory can have related WIP Messages if it is associated with a field from a container definition. Refer to "Defining WIP Messages" and the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information.

You can select any object references you want to exclude from an export. The object must be assigned at the Employee level if it is to be excluded. You must log out and back in before a change made to the factory assignment on the Employee page takes effect.

Refer to "Exporting Modeling Data" for information on the export process. Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information on change management packages.

Adding Locations

Location defines a physical location within a factory. This can be as high-level as a single location that defines the entire factory or as specific as individual locations for each bin within a holding area.

Every container includes a location attribute, used to identify the physical location of the container and produce reports such as WIP by Location. Locations are not defined separately in Opcenter EX MDD or Opcenter EX CR; they are defined within Factory.

Location is an optional field in most Opcenter EX MDD or Opcenter EX CR transactions. You can specify a location when defining Factory, Resource Group, and Shipment Destination.

Adding General Messages

You can add general messages that display when an employee logs on to notify them of upcoming events such as scheduled maintenance, a pending audit, or any other relevant notifications.

The option to add general messages appears in the General Message section of the Factory page and is enabled by selecting the Display Message check box. A rich text editor is provided for creating the messages. The application generates an error when saving the object if the Display Message check box is selected without entering a message in the text box. Refer to "Defining a Task List and Related Tasks" for information on using the text editor.

The general message pop-up displays in the center of the user's default home page after the user sets the line assignment, or displays following log on if the application is configured to bypass the line assignment page. The general message pop-up is dismissed when the user clicks the Close button.

Adding Electronic Signature Requirements for Attaching or Detaching Documents on Factory Transactions

You can set requirements for users to provide electronic signatures to attach documents to or detach documents from transactions performed within a Factory. Documents can include special instructions, test results, drawings, or photos. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information on document attachment.

The option to set electronic signature requirements for document attachment or detachment appears in the Electronic Signature Txn Map in the Transactions section. You can optionally choose to require all transactions to have electronic signatures.

Adding Electronic Signature Requirements for Manufacturing Lots and Sampling Lots on Factory Transactions

You can set requirements for users to provide electronic signatures for manufacturing lots and sampling lots on transactions performed within a factory. The option to set these requirements appears in the Electronic Signature Txn Map in the Transactions section. You can optionally choose to require all transactions to have electronic signatures.

You can submit the signatures for a sampling lot on the Create Sampling Lot or Update Sampling Lot shop floor pages. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information.

Defining Team Tracking Transactions on the Factory Page
Note: The functionality described below is available only when Medical Device is installed. Refer to "Medical Device Workspace" for information.

You can use the Factory page to define the transactions that require tracking of work performed by employees at a given operation, spec, work center, work cell, or workstation. Defining transactions at the factory level allows you to set tracking requirements for all specs instead of modifying each spec in a workflow. However, any transactions set at the spec level take precedence over selections made at the factory level.

Assigning a Default Inventory Location to a Factory
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can assign a default inventory location to a factory. You can also assign a default inventory location to a product and a product family.

These rules apply when assigning default inventory locations:

• The default inventory location assigned on the product takes precedence over the default inventory location assigned on the product family.
• The default inventory location assigned on the product family takes precedence over the default inventory location assigned on the factory.
• The application automatically assigns the default inventory location to a container when the container is moved to an operation defined as an inventory point if a default inventory location is defined on the factory.

Refer to "Assigning a Default Inventory Location to a Product" and "Assigning a Default Inventory Location to a Product Family" for information.

Enabling Material Consumption from Material Queues
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

The automatic consumption of material loaded onto a material queue occurs when a transaction is selected in the Consume Material Txns field in the Material Queues section. The available transactions include:

• Start
• MoveIn
• Move

You can verify that the correct material is loaded onto a material queue for a specific container by selecting one or more transactions in the Validate Material Queue Txns field. The application returns a warning message if you select a transaction but do not select the Validate Material Queue Qty check box. The transaction fails if the quantity is not sufficient if you select a transaction and select the Validate Material Queue Qty check box. You can select an Issue Difference Reason to prevent a failure from occurring when partial quantities are issued during the automatic component issue.

Enabling Opcenter Execution Core Scheduling on the Factory Page
Note: The functionality described below is available only when Opcenter Execution Core Scheduling is installed. Refer to "Opcenter Execution Core Scheduling" for information.

You must enable Opcenter Execution Core Scheduling using the Preactor Interface Enabled check box on the Factory page to activate the transfer of information from Opcenter EX MDD or Opcenter EX CR to the Siemens Preactor Advanced Planning and Scheduling (Preactor) application.

Assigning OEE Settings on the Factory
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

Select the OEE Settings check box in the OEE section if you want to set the time span for which the application calculates Overall Equipment Effectiveness (OEE) for the selected factory. You can also assign OEE Settings on the resource and resource family.

These rules apply when assigning OEE Settings:

• The OEE Settings assigned on the Resource modeling object take precedence over the OEE Settings assigned on the Resource Family modeling object.
• The OEE Settings assigned on the Resource Family modeling object take precedence over the OEE Settings assigned on the Factory modeling object.

Refer to "When Assigning OEE Settings on the Resource" for information on assigning OEE Settings to a resource. Refer to "When Assigning OEE Settings on the Resource Family" for information on assigning OEE Settings to a resource family.

How to Define a Factory

Follow these steps to define a Factory:

1. Open the Factory page. The Factory page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this factory in the Factory field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
6. Refer to the "How to Add a Location" procedure to add locations to this factory.
7. Refer to "How to Add Electronic Signature Requirements for Attaching or Detaching Documents on Factory Transactions" for information on setting electronic signature requirements.
8. Refer to "How to Add Electronic Signature Requirements for Manufacturing Lots and Sampling Lots on Factory Transactions" for information on setting electronic signature requirements.

How to Add a Location

Follow these steps to add a location to a Factory:

1. Perform the "How to Define a Factory" procedure.

Or

Select an existing Factory instance.

2. Click Add new row in the Locations grid. A new row appears.
3. Enter the Location Name and a Description in the appropriate fields in the grid.
4. Select Active or Inactive from the Status list.
5. Click Save. The application saves the modeling object and displays a success message.

How to Add a General Message

Follow these steps to add a general message that displays when a user logs on to the Portal:

1. Perform the "How to Define a Factory" procedure.

Or

Select an existing Factory instance.

2. Expand the General Message section if necessary.
3. Select the Display Message check box to enable general messages.
4. Enter your message in the rich text editor.
5. Apply any desired styles or formatting to your message.
6. Click Save. The application saves the modeling object and displays a success message.

How to Add Electronic Signature Requirements for Attaching or Detaching Documents on Factory Transactions

Follow these steps to add electronic signature requirements for attaching or detaching documents on Factory transactions:

1. Perform the "How to Define a Factory" procedure.

Or

Select an existing Factory instance.

2. Expand the Transactions section if necessary.
3. Click Add new row in the Electronic Signature Txn Map grid. A new row appears.
4. Select Attach Document in the Transaction field if you want to set requirements for attaching documents.

Or

Select Detach Document if you want to set requirements for detaching documents.

5. Select the check box in the All Txns field if you want to apply the electronic signature requirements to all transactions.
6. Select the electronic signature requirement from the ESig Requirement field. Refer to "Defining Electronic Signature Requirements" for information.
7. Click Save. The application saves the modeling object and displays a success message.

How to Add Electronic Signature Requirements for Manufacturing Lots and Sampling Lots on Factory Transactions

Follow these steps to add electronic signature requirements for manufacturing lots and sampling lots on Factory transactions:

1. Perform the "How to Define a Factory" procedure.

Or

Select an existing Factory instance.

2. Expand the Transactions section if necessary.
3. Click Add new row in the Electronic Signature Txn Map grid. A new row appears.
4. Select MfgLot or SamplingLot in the Transaction field.
5. Select the check box in the All Txns field if you want to apply the electronic signature requirements to all transactions.
6. Select the electronic signature requirement from the ESig Requirement field. Refer to "Defining Electronic Signature Requirements" for information.
7. Click Save. The application saves the modeling object and displays a success message.
