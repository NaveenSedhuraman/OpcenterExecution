Defining Package Creation Templates

A Package Creation Template identifies the details, change management workflow, and deployment details for a package. It also identifies approval and collaborator templates assigned to a package. The package creation template page enables you to define templates that are available for use on the Create Package page. Selecting a template populates the Create Package page with the predefined package details, deployment details, and applicable approver and collaboration templates. The application then uses the selected template to populate the Update Package page using the specified approval and collaborator templates to populate the Approvers and Collaborators tabs. You do not have to enter all of the information manually each time you create a package.

Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information on enabling collaboration and creating a package.

When Defining Package Creation Templates

Note the following:

• Approval Template becomes active if you select a change management workflow with Assign Approvals configured in the change management spec on the first step of workflow. Refer to "Defining Change Management Workflows" for information about configuring workflows.
• Collaborator Template becomes active only if you select Owner Plus Collaborators as the Content Collaborators setting.

Approvers and Collaborators

 

 

Approval Template

Approval templates you can assign to the change management package. Approval templates identify the users who need to approve change management packages. Additionally, the approval template specifies a list of possible decisions (the associated approval decision list) for those users.

Note: Approval templates are defined in Modeling.

Optional

Content Collaborators

Value indicating whether only the owner or the owner and collaborators can assign content to the package:

• Owner Only (the default value) indicates only the user with ownership permissions can assign content to the package.
• Owner Plus Collaborators indicates the user with ownership permissions and collaborators designated by the owner can assign content to the package. When selected, Collaborator Template is activated.

Optional

Collaborator Template

List of collaborator templates defined in Modeling. The application populates the Collaborators grid and the Instructions for All Collaborators field on the package with the information from the template.

Note: Collaborator templates are defined in Modeling.

Optional

How to Define a Package Creation Template

Follow these steps to define a Package Creation Template:

1. Open the Package Creation Template page. The Package Creation Template page appears within the Modeling page.
2. Enter a name for this package creation template in the Package Creation Template field.
3. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
4. Click Save. The application saves the modeling object and displays a success message.
