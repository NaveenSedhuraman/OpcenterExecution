Defining Carrier Status Codes

You can create standardized Carrier Status Codes that can be assigned and used in shop floor transactions. The use of standardized status codes can help your operators make more informed decisions on the shop floor.

When Defining Carrier Status Codes

Carrier status codes must be unique. These are examples of carrier status codes:

• Used
• Clean
• Used but available

Carrier Status Code is an optional field in the Resource Status Code Group modeling definition. Carrier status codes are referenced by the Resource Status modeling definition, but only if you specify a resource status code group containing carrier status codes.

Carrier Status Code contains the optional Modeling definition: Resource Status Reason Group.

How to Define a Carrier Status Code

Follow these steps to define a Carrier Status Code:

1. Open the Carrier Status Code page. The Carrier Status Code page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the carrier status code in the Carrier Status Code field.
4. Select the Availability to be associated with this status code.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.
