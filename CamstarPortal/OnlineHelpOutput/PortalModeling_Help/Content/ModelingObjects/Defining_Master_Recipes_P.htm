Defining Master Recipes

A Master Recipe, based on the S88 recipe model, is a revisionable object that includes all of the information necessary to produce a batch of material. A master recipe is a collection of one or more procedures (recipe lists) that you assign to a batch when you start the batch.

Before Defining Master Recipes

Before defining a master recipe, you must define one or more of the following:

• Recipe lists
• Specs

Additionally, you may need to define the following entities depending on your requirements:

• Product
• UOM
• Workflow

When Defining Master Recipes

The application requires that you specify at least one recipe list when creating a master recipe.

When adding a recipe list to the master recipe, the application requires that you associate the recipe list with a spec.

You can add the same recipe list to the master recipe list multiple times, but you must associate it with a different spec each time. The application will not allow you to save a master recipe with multiple copies of the same recipe list at the same spec.

Note: Be sure to specify nominal and limit values for the master recipe. The application does not require these values to save a master recipe, but it does require them when starting a batch.

Recipe List Execution Order

When you add a recipe list to a master recipe, the sequence number determines execution order while the subsequence number determines the display order. When associating multiple recipe lists to the same spec, the application requires unique sequence and subsequence combinations to ensure a valid display order.

Note: Recipe lists at a spec can be run concurrently with the same sequence value.

Recipe Qty fields

Values used to validate the quantity when starting a batch:

• Recipe Qty Nominal: The desired amount of product to be produced from the recipe.
• Recipe Qty Lower Limit: The lower limit for the amount of product produced from the recipe.
• Recipe Qty Upper Limit: The upper limit for the amount of product produced from the recipe.

The value you enter in these fields can be static text or an expression. If you enter an expression, the application will calculate the quantity at run time. You can enter a maximum of 255 characters.

How to Define a Master Recipe

Follow these steps to define a Master Recipe:

1. Open the Master Recipe page. The Master Recipe page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the recipe in the Master Recipe field.
4. Enter the revision of this object in the Revision field.
5. Select Sequential or Non Sequential from the Execution Mode list.
6. Add at least one recipe using the "How to Add a Recipe List to a Master Recipe" procedure.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Save. The application saves the modeling object and displays a success message.

How to Add a Recipe List to a Master Recipe

Follow these steps to add a Recipe List to a Master Recipe:

1. Perform the "How to Define a Master Recipe" procedure.

Or

Select an existing Master Recipe instance.

2. Expand the Recipe Procedure section.
3. Click Add new row on the Recipe Procedures grid. A new row appears.
4. Select a recipe list in the Recipe List field.
5. Select a spec in the Spec field.
6. Enter a sequence number for the recipe list in the Sequence field.
7. Enter a subsequence number for the recipe list in the Sub Sequence field.
8. Repeat steps 3-7 to add additional recipe lists.
9. Click Save. The application displays a success message indicating the modeling object was updated.
