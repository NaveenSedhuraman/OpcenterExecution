Defining a Task List and Related Tasks

A Task List provides a method for defining a list of tasks to be executed against a spec. You specify whether the tasks must be performed in sequential order. An example of a task list is a group of product-measuring tasks related to the electronic procedure User Data Collection.

Understanding Tasks

A Task is a unit of work required to be completed for a container at a spec. You can assign a sequence of tasks to an electronic procedure which can be assigned to a spec. The electronic procedure enforces the sequence of task lists while the task list enforces the sequence of tasks. A task must be saved before it can be set as the next task's prerequisite task. Whether a task passes or fails depends on either data collected or the decision of the shop floor operator.

Task Item Types

You can choose the task item type when adding a task item.

Description

Computation

Calculation performed by the application based on user constants, parameters, and operator input when the task is performed.

Task Types

The task type of a task item is important for those workstations that enforce line clearance. A task item can be one of the following task types.

Task Type

Description

Start Process

Denotes the beginning of processing at a workstation. A Start Process task sets the workstation status to "In Process" at either the container level or the manufacturing order level. A Start Process task is required for workstations that enforce line clearance, and must be the first task in the task list. A transaction task cannot be a Start Process task.

Note: The minimum iterations value for a Start process task must be greater than 0.

Regular

Denotes tasks that can occur in between a Start Process task and an End Process task for workstations enforcing line clearance. A task list can consist of all Regular tasks for workstations that do not enforce line clearance. A Regular task must be run against the same container or manufacturing order within the same task list, electronic procedure, and spec step as the Start Process task. Otherwise, it will fail.

End Process

Denotes the end of processing at a workstation. An End Process task resets the workstation status. The End Process task indicates the workstation is available to process the next container or manufacturing order. End Process task is required for workstations that enforce line clearance, and must be the last task in the task list.

Note: The minimum iterations value for an End Process task must be greater than 0.

Understanding Execution Mode

Enter the individual tasks in the order in which they must be completed when you create a task list. If the tasks do not have to be executed in a specific order, you can enter them in any sequence. The task list defines the execution mode for the tasks.

You indicate the execution mode for a task list by selecting Sequential or Non Sequential in the Execution Mode field on the Task List page:

• Sequential (default) - Each task list is presented in the order established through an electronic procedure. A task list can only be skipped if all its tasks are optional.
• Non Sequential - The application does not require the task list to be performed in a sequence defined through an electronic procedure. Likewise, the electronic procedure defines the execution mode for task lists.

Text Editor

Siemens provides a text editor that enables you to format instructions in either task lists or related tasks. The text editor supports familiar text formatting features such as font size and color, font style, text alignment, bulleted lists, highlighting, inserting images, etc.

Note: Siemens does not currently recommend creating numbered lists in task or task list instructions because they do not render properly within the Instructions field.

Siemens recommends highlighting the existing instructions and then applying formatting to the selected text. If you do not select any formatting attributes, the application applies the default font size, style, and color. The application displays the applied formatting to the shop floor user.

You can use the text editor in either Design or HTML mode. When you copy text in Design mode, you are not copying the hidden tags. Therefore, the formatting may not be retained when you paste the text that was copied. You can copy the hidden tags, however; if you switch to HTML view.

There is a limit of 4,000 characters for the Instruction field, including embedded HTML code. When you copy text from another application (for example, Microsoft Word) and paste it into this field, hidden HTML code is pasted into the field in addition to the actual text. Therefore, you may exceed the limit even though the text contains less than 4,000 characters.

Note: Google Chrome does not support the text editor's Copy and Paste buttons. You can use the keyboard shortcuts for copying and pasting (Ctrl+C and Ctrl+V) when using Google Chrome.

Design Mode

Design mode is a WYSIWYG (what you see is what you get) text editor that displays the instructions as the shop floor user sees them.

HTML Mode

Clicking the HTML button displays the source code applied to the instructions. Because a WYSIWYG editor may insert markup code (tags) automatically, viewing the instructions in HTML mode allows you to validate and modify that code if necessary.

Inserting Images Using the Text Editor

You can insert images in instructions by clicking the Insert/edit image button on the text editor. Images are then uploaded by entering a path to an image on the Insert/edit image pop-up.

Paths to an image must be entered as either:

• A relative URL, such as "Themes/MyImages/Image.png"

Or

• An absolute URL, such as "https://localhost/Themes/MyImages/Image.png"
Note: Large images may not fit within the visible area of the instructions section on the EProcedure page. Siemens recommends that you edit the dimensions of your image to fit within the visible area on your preferred device.

When Assigning Process Timers

Process Timers can be started or ended by tasks on any of the task item type pop-ups (Instruction, Computation, Transaction), but only using the MoveStd and Move In transactions.

Note: The application displays an error message if you try to save a task list that contains a transaction task with timers and a transaction other than Move (Standard) or Move In.

Timers may be:

• Started by a task and ended by a task
• Started by a task and ended by a transaction (modeled on a spec)
• Started by a transaction (modeled on a spec) and ended by a task
• Started by a transaction and ended by a transaction (both modeled on a spec)
Note: Process Timers are only dependent on the Move (Standard) or Move In transactions and do not take into account the number of iterations that could be performed on a task. Once the task is executed the timer stops.

There is no association between start timers and end timers on an individual task. For example, one grid may have multiple timers defined and the other may have none.

Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information.

When Defining Data Collection Tasks for Instruction Tasks

You must select Data Collection from the Instruction Type field and select a predefined data collection definition from the Data Collection Def field when you define a data collection task. If you associate a data collection definition with the task, but select a value other than Data Collection from the Instruction Type field, the application will not display the data collection fields on the EProcedure page.

When Defining Computation Tasks

The application requires the data type of the associated data point and computation parameter to be the same when defining a computation task. The application displays an error message if the data types are different. Data point data types are set in the specified data collection definition and computation parameter data types are set in the specified computation definition.

You must select a revision of the Data Point. Expand the Data Point in the list to select a particular revision.

When Defining Transaction Tasks

A transaction task can be either a Regular task or an End Process task. The Start Process Task option is not available in the Task Type field for a transaction task. Additionally, the Electronic Signature Requirement field is not available for transaction tasks.

When Defining an Intermediate Batch
Note: The functionality described below is available only when Batch Processing is installed. Refer to "Batch Processing" for information.

You can define an intermediate batch using the fields in the Batch Processing section. You can enter an expression to calculate the quantity of an intermediate batch. You can set a unit of measure for the intermediate batch.

All components added to the intermediate batch must have the same unit of measure. The application converts the units of measure for components to match the unit of measure for the batch if the components do not have the same unit of measure. The conversion occurs before the components are issued. Refer to "When Defining Conversion Information for Units of Measure" for information.

When Defining Weigh Issue Tasks
Note: The functionality described below is available only when Batch Processing is installed. Refer to "Batch Processing" for information.

You can define a weigh issue task to define the point in the process a component is to be issued and to provide any rules required for weighing.

You can define expressions as part of the task to calculate the required quantity and associated minimum and maximum allowed quantities. These calculated values take precedence over values derived from the product or from the material list. Refer to "Defining Expressions to Adjust Component Quantities on the Product," "Defining Allowed Component Quantities on the Manufacturing Order," "Defining Allowed Component Quantities on the BOM," and "Defining Allowed Component Quantities on the ERP BOM" for information.

You can assign a scale group on the weigh issue task using the Scale Group field. You can also assign a scale group on the spec, product family, and product.

These rules apply when assigning scale groups:

• The scale group assigned on the product family takes precedence over a scale group assigned on the spec.
• The scale group assigned on the product takes precedence over the scale group assigned on the product family.
• The scale group assigned on the weigh issue task takes precedence over the scale group assigned on all other objects.

Refer to "Assigning a Scale Group on the Spec," "Assigning a Scale Group on the Product Family," and "Assigning a Scale Group to a Component Product."

You can also define the following rules for weighing the component:

• Allowing manual weight entry
• Requiring an electronic signature with manual weighing
• Requiring an electronic signature when weight tolerances violations are overridden
• Allowing manual weight entry only
• Increasing the batch container’s quantity by the material issued quantity

A weigh issue task can be either a Regular Task or an End Process Task. The Start Process Task option is not available in the Task Type field for a weigh issue task.

How to Define a Task List

Follow these steps to define a Task List:

1. Open the Task List page. The Task List page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this task list in the Task List field.
4. Enter the revision of this task list in the Revision field.
5. Select an Execution Mode from the list.
6. Add at least one task to the Tasks grid using the appropriate procedures below:
• "How to Add an Instruction Task to a Task List"
• "How to Add a Computation Task to a Task List"
• "How to Add a Transaction Task to a Task List"
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Save. The application saves the modeling object and displays a success message.

How to Add an Instruction Task to a Task List

Follow these steps to add instruction tasks to a Task List:

1. Perform the "How to Define a Task List" procedure.

Or

Select an existing Task List instance.

2. Expand the Task Details section.
3. Click Add new row on the Tasks grid. The Select Task Item Type menu appears.
4. Select Instruction. The Instruction Task Details pop-up appears.
5. Enter a name for the task in the Task field.
6. Select a Task Type from the list.
7. Select a Instruction Type from the list.
8. Enter a number in the Min Iterations field to indicate the minimum number of times the task can be executed.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click OK. The pop-up closes and the task displays in the Tasks grid.
11. Click Save. The application saves the modeling object and displays a success message.
