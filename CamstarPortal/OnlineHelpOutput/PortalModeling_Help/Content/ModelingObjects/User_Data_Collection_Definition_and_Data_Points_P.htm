User Data Collection Definition and Data Points

The User Data Collection definition provides a method for establishing data points that can then be associated with a task list for an electronic procedure. A data point represents a point in the production process where data can or must be collected. When defined, multiple data points can be grouped within a user data collection definition that can then be assigned to an electronic procedure.

When Defining a User Data Collection

When defining user data collection definitions, you can include both value data points and object data points in either a row and column or grid layout. The application dynamically renders the data points in the specified layout at runtime. Refer to "Defining User Data Collection Definitions" for information.

When Adding Value Data Points

When adding value data points to a user data collection, you can do the following:

• Define upper and lower limits by entering static values or an expression
• Indicate whether limit overrides are allowed
• Opt to map the data point value to a container attribute

You can use an expression to define the upper limit, lower limit, or both limits. The limits are determined at runtime when an expression is used.

Use the application's Unified Expression syntax when defining upper and lower limits with an expression. Expressions can contain a maximum of 255 characters and may include:

• Field expressions; for example, Container.CurrentStatus.Spec.UserDefinedInteger
• A combination of a field expression and elementary arithmetic; for example, (100-Container.CurrentStatus.Spec.UserDefinedInteger) *0.2
Note: There is no validation of correct syntax when entering expressions. Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information.

When Defining Limits for Value Data Points

Limits are applicable to value data points only, but are available with both the row and column and iteration grid layouts. The Display Limits check box on the User Data Collection Def page determines whether the application displays the limits defined for the value data points during a data collection. When selected (the default setting), the application displays limits for those data points.

When limits are defined, the Limit Override Allowed check box determines the color displayed to a user when an entered value is outside a limit and whether to pass or fail an EProcedure data collection task. Regardless of whether the EProcedure task passes or fails, the application always accepts any value you enter, even when that value falls outside of a limit.

When Adding Object Data Points

An object data point essentially serves the same purpose as a value data point—it offers a means of collecting data at a particular point in the production process. An object data point references a particular object. For example, you could use an object data point to establish a Reason Code to record pre-defined inspection status results. These results could include such conditions as contamination and surface scratches. On the shop floor, the operator would complete the inspection task by simply choosing the appropriate result from a selection value list.

Note: Limits do not apply to object data points.

How to Define a User Data Collection

Follow these steps to define a User Data Collection:

1. Open the User Data Collection Def page. The User Data Collection Def page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the unique name of this definition in the User Data Collection Def field.
4. Enter the revision identifier for this definition the Revision field.
5. Select the Revision of Record check box if this is the revision of record.
6. Select the type of Data Point Layout from the list: Iteration Grid or Row Column Position.
7. Add a data point to the Data Points grid using the appropriate procedure below:
• "How to Add a Value Data Point"
• "How to Add an Object Data Point"
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Save. The application displays a success message indicating the modeling object was updated.

How to Add a Value Data Point

Follow these steps to add a value data point to a User Data Collection:

1. Perform the "How to Define a User Data Collection" procedure.

Or

Select an existing User Data Collection Def instance.

2. Click Add new row in the Data Points grid. The Data Point Type menu appears.
3. Select Value Data Point. The Value Data Point Detail pop-up appears.
4. Enter a name in the Data Point Name field.
5. Enter a numeric value to indicate the Row Position where the application will display the data point in the data collection.
6. Enter a numeric value to indicate the Column Position that the value data point must be placed within the data collection.
7. Select the Data Type from the list.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click OK. The application closes the pop-up and displays the data point in the Data Points grid.
10. Repeat steps 2-9 to add additional value data points.

How to Add an Object Data Point

Follow these steps to add an object data point to a User Data Collection:

1. Perform the "How to Define a User Data Collection" procedure.

3. Select Object Data Point. The Object Data Point Detail pop-up appears.

5. Enter a numeric value to indicate the Row Position where the application will display the data point in the data collection.
6. Enter a numeric value to indicate the Column Position that the value data point must be placed within the data collection.
