Defining Object Groups

Object Groups are used to create groups of modeling objects such as user codes.

There are two types of object groups:

• Named Object Groups
• Revisioned Object Groups

Named object groups are used to group together user codes or other named data objects, while revisioned object groups are used to group together revisioned objects.

You can also add other groups to a group.

These object groups can then be used for selection in other modeling objects. For example, the Operation object uses Component Defect Reasons. However, you cannot select component defect reasons you created without first adding them to the Component Defect Reason Group. When you add the component defect reasons to a group, that group is available in the Component Defect Reason list on the Operation.

Refer to "Named Data Objects and Revisioned Objects" for information on these items.

When Defining an Object Group

Each group is a logical collection of similar entries and subgroups. When you select a specific group, the application limits your choices to only those entries and groups of that type. You can add multiple entries or groups to their respective grids.

All object group pages contain the Resolved Entries button. Clicking the Resolved Entries button displays a list of all of the values specified in the entries list for the group and all of the nested groups.

You must select an entry type from the Entry Type list if defining a Named Object Group instance or User Code Group instance at the main level.

How to Define an Object Group

Follow these steps to define an Object Group:

1. Open an object group modeling page. The selected object group page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the object group in the <Object Group> field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.

How to Define a Named Object Group Instance or User Code Group Instance

Follow these steps to define a Named Object Group instance or User Code Group instance:

1. Perform the "How to Define an Object Group" procedure.

Or

Select an existing Object Group instance.

2. Select an entry type from the Entry Type list.
3. Add entries to the group as follows:
a. Click Add new row at the bottom of the Entries grid. A new row appears.
b. Click in the new row to display a list of entries, and then select one to add to the group. The selected entry is added to the grid.
c. Repeat steps a–b to add additional entries to the grid.
Note: You may need to perform a save after adding entries to the Entries grid before attempting to add groups to the Groups grid.
4. Add a subgroup to the group as follows:
a. Click Add new row at the bottom of the Groups grid. A new row appears.
b. Click in the new row to display a list of previously defined groups, and then select one to add (as a subgroup) to the group. The application adds the selected group to the grid.
c. Repeat steps a–b to add additional groups to the grid.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.
