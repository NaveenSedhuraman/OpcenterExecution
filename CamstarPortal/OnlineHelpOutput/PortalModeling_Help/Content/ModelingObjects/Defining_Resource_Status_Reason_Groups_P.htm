Defining Resource Status Reason Groups

A Resource Status Reason Group contains a logical grouping of similar resource status reasons, previously defined. For example, the status reasons Calibrated and Repaired could be included in a group named Equipment Ready Group.

When Defining a Resource Status Reason Group

Resource Status Reason Group is an optional field when defining Resource Status Code and Carrier Status Code.

You can add multiple entries or groups to their respective grids.

The Resolved Entries button enables you to display a list of all of the values specified in the entries list for this group and all of the nested groups.

How to Define a Resource Status Reason Group

Follow these steps to define a Resource Status Reason Group:

1. Open the Resource Status Reason Group page. The Resource Status Reason Group page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this resource status reason group in the Resource Status Reason Group field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Add entries to the group as follows:
a. Click Add new row at the bottom of the Entries grid. A new row appears.
b. Click in the new row to display a list of entries, and then select one to add to the group. The selected entry is added to the grid.
c. Repeat steps a–b to add another entry to the grid.
6. Add a subgroup to the group as follows:
a. Click Add new row at the bottom of the Groups grid. A new row appears.
b. Click in the new row to display a list of previously defined groups, and then select one to add (as a subgroup) to the group. The selected group is added to the grid.
c. Repeat steps a–b to add another group to the grid.
7. Click Save. The application saves the modeling object and displays a success message.
