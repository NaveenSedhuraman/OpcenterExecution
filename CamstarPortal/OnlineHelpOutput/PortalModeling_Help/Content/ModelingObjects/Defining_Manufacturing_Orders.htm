Defining Manufacturing Orders

A Manufacturing Order is a request to manufacture a product. Manufacturing orders are typically originated to fulfill a sales order or an inventory requirement and are often a link between an MES and ERP system.

A manufacturing order can contain a list of product containers. The optional relationship of a container to a manufacturing order is established and maintained in the container's data record. This information is used to provide WIP status by manufacturing order and WIP updates to the ERP system.

Additionally, you can assign custom, dynamic attributes to a manufacturing order and the application will assign these user-defined attributes when you start a container. Storing information on the container itself prevents you and other users from having to search through the container’s history for the information.

When Defining a Manufacturing Order

Mfg Order contains these optional modeling definitions:

• ERP BOM
• Document
• Order Status
• Order Type
• Priority
• Product
• Route Step
• Container Numbering Rule
• UOM

Mfg Order is an optional field in most Container transactions.

Your entries in the Priority and Planned Start Date fields determine the sort order of manufacturing orders in the Order Dispatch transaction.

When Attaching Documents

You can attach documents to and then detach or view them from a manufacturing order using the Mfg Order modeling page. You must save the manufacturing order instance before you can attach a document to it.

Attachments are optional; however, the application requires you to complete certain fields when attaching files before you start the container.

The maximum size for an attachment is 30 MB.

Note: The document attachment functionality that is available on the Mfg Order modeling page can be added to any modeling object. This is done through Portal Studio by including the document attachment web part on the modeling page. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information.

When Viewing Attachments

Your browser may block the file from being downloaded until you take additional steps if you are accessing the file remotely. For example, Internet Explorer may block the file from being downloaded until you click Download File in the IE information bar.

When you are attaching an existing document, a document file must be attached to the document instance selected, or you will receive an error message when you click View Document on the pop-up.

You must also have the correct software installed to view the file. For example, you must have Microsoft Excel installed to view the file if the file is an Excel spreadsheet.

When you add a document that requires a revision, the application automatically creates a new Document instance.

Defining Allowed Component Quantities on the Manufacturing Order
Note: The functionality described below is available only when Batch Processing is installed. Refer to "Batch Processing" for information.

You can define the required component quantity and the minimum and maximum allowed quantities using the Material List grid. You can also define component quantities using the Material List grid on the BOM or ERP BOM. The material list defined for the manufacturing order takes precedence over the material list defined for the BOM or the ERP BOM. Refer to "Defining Allowed Component Quantities on the BOM" and "Defining Allowed Component Quantities on the ERP BOM" for information.

Use the Minimum Allowed Quantity and the Maximum Allowed Quantity fields to define the allowed component quantities. These fields appear in the pop-ups displayed when you click the Add new row button to add a material to the Material List grid.

When Defining Manufacturing Orders for Use with Opcenter Execution Core Scheduling
Note: The functionality described below is available only when Opcenter Execution Core Scheduling is installed. Refer to "Opcenter Execution Core Scheduling" for information.

Opcenter Execution Core Scheduling uses information provided in the manufacturing order instance to schedule orders. The ERP route assigned to the manufacturing order must match one of these fields when using Opcenter Execution Core Scheduling:

• The ERP route assigned to the workflow of the product
• The ERP route assigned to the workflow for the product family

Use the Order Status field on the Mfg Order page to assign an order status to a manufacturing order. You can only schedule open manufacturing orders in Opcenter Execution Core Scheduling.

When Releasing Manufacturing Orders for Use with Opcenter Execution Core Scheduling
Note: The functionality described below is available only when Opcenter Execution Core Scheduling is installed. Refer to "Opcenter Execution Core Scheduling" for information.

Releasing a manufacturing order triggers the interface to send the order data from Opcenter EX MDD or Opcenter EX CR to Preactor. A manufacturing order is released whenever a new order is saved or an existing order is updated when these conditions are met:

• The manufacturing order has a product, quantity, and order status.
• The manufacturing order's product or product family has an assigned workflow.
• The order status has an Open state.

Manufacturing orders are placed in a Preactor staging table as they are released. Staged orders can be updated by modifying and saving the manufacturing order. Orders can be removed from the staging table by changing the Order Status to a closed state and then saving the manufacturing order. Orders are continually added to the staging table as they are released. Importing orders into Preactor replaces the current order list in Preactor with the orders currently in the staging table.

How to Define a Manufacturing Order

Follow these steps to define a Manufacturing Order:

1. Open the Mfg Order page. The Mfg Order page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this task list in the Mfg Order field.
4. Refer to “How to Add a Material List to a Manufacturing Order” to add a material list.
5. Refer to “How to Add User-Defined Attributes to a Container Level” to add user-defined attributes.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.
8. Refer to one of the following document attachment procedures to attach documents:
• "How to Attach a New Document that Will Not Be Reused to a Manufacturing Order"
• "How to Attach a New Document that Will Be Reused to a Manufacturing Order"
• "How to Attach an Existing Document to a Manufacturing Order"
9. Refer to "How to Detach a Document from a Manufacturing Order" to detach documents.
10. Refer to "How to View a Document Attached to a Manufacturing Order" to view attached documents.

How to Add a Material List to a Manufacturing Order

Follow these steps to add a material list to a Manufacturing Order:

1. Perform the “How to Define a Manufacturing Order” procedure.

Or

Select an existing Manufacturing Order instance.

2. Expand the Materials section.
3. Click Add new row on the Material List grid. The Add pop-up appears.
4. Select a Product from the list.
5. Select an Issue Control from the list.
6. Enter a value in the Quantity Required field.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click OK. The Add pop-up closes and the Material List Item appears in the Material List grid on the Mfg Order page.
9. Click Save. The application displays a success message indicating the modeling object was updated.

How to Add User-Defined Attributes to a Manufacturing Order

Follow these steps to add user-defined attributes to a Manufacturing Order:

1. Perform the “How to Define a Manufacturing Order” procedure.

Or

Select an existing Manufacturing Order instance.

2. Expand the User Attributes section.
3. Click Add new row on the User Attribute List grid. A new row appears.
4. Enter a name for the attribute in the Attribute Name field.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Select the Is Expression check box only if you selected String as the Type.
7. Repeat steps 3-6 to add additional attributes.
8. Click Save. The application displays a success message indicating the modeling object was updated.

How to Attach a New Document that Will Not Be Reused to a Manufacturing Order

Follow these steps to attach a new document that will not be reused to a Manufacturing Order:

1. Perform the “How to Define a Manufacturing Order” procedure.

How to Attach a New Document that Will Be Reused to a Manufacturing Order

Follow these steps to attach a new document that will be reused to a Manufacturing Order:

How to Detach a Document from a Manufacturing Order

Follow these steps to detach a document from a Manufacturing Order:

1. Open the Mfg Order page. The Mfg Order page appears within the Modeling page.
2. Select an existing Manufacturing Order instance.
3. Expand the Attached Documents section if necessary.
4. Select an item in the Attached Document Details grid.
5. Click Detach. The Detach Document pop-up appears with the Instance Name and Attached Document fields populated.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Detach. The application displays a message indicating that the document has been detached successfully.
Note: The document instance that was created when you added a new revisioned document is not deleted when you detach the document.
