Defining ERP Routes

An ERP Route is the closest ERP concept to an Opcenter EX MDD or Opcenter EX CR workflow. ERP routes are referenced in Opcenter EX MDD or Opcenter EX CR workflows and in manufacturing orders. The workflow references the ERP route so that individual workflow steps can reference ERP route steps. This is often done for ERP milestones or inventory points. The manufacturing order references the ERP route so that material list items reference the ERP route steps where the components are issued.

When Defining an ERP Route

ERP route is a required object to link the manufacturing order material list items to the workflow for the component issue transaction.

ERP route is used to relate the Opcenter EX MDD or Opcenter EX CR transactions Move and Component Issue to the ERP step where the transaction occurred. Steps in an ERP route are usually defined at a more summarized level than is true for steps in an Opcenter EX MDD or Opcenter EX CR workflow.

ERP route contains the optional modeling definition Product.

ERP route is an optional field in the definition of Mfg Order.

When Defining ERP Routes for Use with Opcenter Execution Core Scheduling
Note: The functionality described below is available only when Opcenter Execution Core Scheduling is installed. Refer to "Opcenter Execution Core Scheduling" for information.

You must provide the processing time for each ERP operation where a manufacturing order or product is processed. You can use the Route Step grid on the ERP Route page to define the run rate in these ways:

• Hours per unit
• Units per hour
• Time per batch

You can only select one option to define the run rate.

You must assign an ERP route to a workflow. You can then assign the workflow to the product or the product family.

When Establishing Setup Times for a Route Step
Note: The functionality described below is available only when Opcenter Execution Core Scheduling is installed. Refer to "Opcenter Execution Core Scheduling" for information.

You can define the amount of time required to set up a resource for processing at a route step. For example, you have a processing machine that must be cleaned between runs. The cleaning process takes thirty minutes to complete. You can indicate this time in the Setup Time field.

How to Define an ERP Route

Follow these steps to define an ERP Route:

1. Open the ERP Route page. The ERP Route page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the ERP route in the ERP Route field.
4. Enter a revision in the Revision field.
5. Refer to the "How to Add Route Steps to an ERP Route" procedure to add route steps.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Save. The application saves the modeling object and displays a success message.

How to Add Route Steps to an ERP Route

Follow these steps to add route steps to an ERP Route:

1. Perform the "How to Define an ERP Route" procedure.

Or

Select an existing ERP Route instance.

2. Expand the Details section.
3. Click Add new row on the Route Steps grid. A new row appears.
4. Enter the name of this route step in the Route Step Name field.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Repeat steps 3-5 to add additional route steps.
7. Click Save. The application displays a success message indicating the modeling object was updated.
