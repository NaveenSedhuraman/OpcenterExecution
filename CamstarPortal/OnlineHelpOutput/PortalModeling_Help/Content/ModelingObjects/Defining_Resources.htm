Defining Resources

A Resource identifies a machine, a piece of equipment, or any other nonmaterial entity within a factory. The common use and most concrete example of a resource is a machine. Resources are associated with a single factory to define their physical location.

You can enter the resource used at each step in a workflow. This information can then be incorporated into the analysis of a container's throughput, cycle times, and yield loss.

You can assign resources to resource groups, a type of object group to provide logical grouping. Resource groups are used to provide selection lists of valid resources and to validate user input.

Resource is a static modeling entity. Each resource can have a resource status entry to track the current status for each status category defined. The application is delivered with three specific status categories:

• Production is used to determine availability for production.
• Preventive Maintenance is used to track the current preventive maintenance status for a resource.
• Log is used to provide a mechanism for logging free-form comments and optionally assigning predefined status and reason codes. Customers can define additional status categories to concurrently track resource status in multiple user-defined ways.

When Defining a Resource

If implementing resource states, a resource type and/or a resource status model is associated with the resource on the Resource Setup transaction.

Note the following:

• Resource contains the optional modeling definitions: Factory, Location, Print Queue, Maintenance Class, Training Requirement Group, Resource Type, and Resource Status Model.
• Resource is:
• An optional field in the definition of Resource Group.
• A required field in the transactions Resource Setup, Collect Resource Data, and Resource Thruput.
• An optional field in the transactions Move In, Thruput, Move Standard, Move NonStandard, Transfer, Rework, Transfer Rework, Collect Data, Container Maintenance, and Change Quantity.
Note: A Resource definition can have related WIP Messages if it is associated with a field from a Container definition. Refer to "Defining WIP Messages" and the Opcenter Execution Core Shop Floor User Guide or the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide for information, depending upon your product.

Two fields on the Resource page are specific to setup for the resource:

• Resource Type
• Resource Status Model

A Portal user can specify a resource when creating an event.

When Defining a Resource for OEE
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

Select the Include in OEE Data check box if you want the application to accumulate OEE summary data for the resource.

You can use the Ideal Cycle Times grid to define the ideal cycle time for a product or product family. These rules apply when defining an ideal cycle time:

• The ideal cycle time defined on the Resource page takes precedence over the ideal cycle time defined on the Resource Family page.
• You must specify a product or a product family for each ideal cycle time defined in the Ideal Cycle Times grid.
• You can specify a product or product family only once in the Ideal Cycle Times grid. The product or product family must be unique for each ideal cycle time.
• The ideal cycle time defined for a product takes precedence over the ideal cycle time defined for a product family.

When Assigning OEE Settings on the Resource
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

These rules apply when assigning OEE Settings:

• The OEE Settings assigned on the Resource modeling object take precedence over the OEE Settings assigned on the Resource Family modeling object.
• The OEE Settings assigned on the Resource Family modeling object take precedence over the OEE Settings assigned on the Factory modeling object.

Refer to "When Assigning OEE Settings on the Resource Family" for information on assigning OEE Settings to a resource family. Refer to "Assigning OEE Settings on the Factory" for information on assigning OEE Settings to a factory.

When Defining Downtime Schedules for a Resource
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can use the Downtime Schedule grid to define downtime schedules for the selected resource. The application populates the Resolved Downtime Schedule grid with information from the Downtime Schedule grid.

These rules apply when defining downtime schedules for a resource:

• The downtime schedule defined on the Resource page takes precedence over the resource downtime schedule define on the Resource Family page. Refer to "When Defining Downtime Schedules for a Resource Family" for information on defining downtime schedules for a resource family.
• You can define scheduled or non-scheduled downtime for a resource.

When Defining a Resource for Use with Scheduling
Note: The functionality described below is available only when Opcenter EX CR Scheduling is installed. Refer to "Opcenter Execution Core Scheduling" for information.

Creating a new resource or updating an existing resource in the application triggers the application to send the resource information to Preactor. Preactor schedules manufacturing orders to a specific resource.

Validating Material Queues for a Resource
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can select the Validate Queue check box in the Material Queues section to require the application to validate the material being loaded onto the material queue associated with a resource against the bill of materials (BOM) for the product being manufactured.

How to Define a Resource

Follow these steps to define a Resource:

1. Open the Resource page. The Resource page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the resource in the Resource field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
