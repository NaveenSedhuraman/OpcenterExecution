Defining Resource Families

A Resource Family provides a way to categorize resources based on their attributes and according to your business requirements. You can use a resource family as a filter for various shop floor transactions.

Carrier Family is a subclass of Resource Family.

Refer to "Defining Carrier Families" for information on this modeling object.

When Defining a Resource Family

Examples of resource families include the following:

• Centrifuges
• Testers
• Handlers

Resource Family is an optional field in the Resource modeling object.

Resource Family contains the optional modeling definition UI Preference.

When Defining a Resource Family for OEE
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can use the Ideal Cycle Time field on the Ideal Cycle Times grid to define the ideal cycle time for resources in the same resource family. These rules apply when defining an ideal cycle time:

• The ideal cycle time defined on the Resource page takes precedence over the ideal cycle time defined on the Resource Family page.
• You must specify a product or a product family for each ideal cycle time defined in the Ideal Cycle Times grid.
• You can specify a product or product family only once in the Ideal Cycle Times grid. The product or product family must be unique for each ideal cycle time.
• The ideal cycle time defined for a product takes precedence over the ideal cycle time defined for a product family.

When Assigning OEE Settings on the Resource Family
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

These rules apply when assigning OEE Settings:

• The OEE Settings assigned on the Resource modeling object take precedence over the OEE Settings assigned on the Resource Family modeling object.
• The OEE Settings assigned on the Resource Family modeling object take precedence over the OEE Settings assigned on the Factory modeling object.

Refer to "When Assigning OEE Settings on the Resource" for information on assigning OEE Settings on the resource. Refer to "Assigning OEE Settings on the Factory" for information on assigning OEE Settings on the factory.

When Defining Downtime Schedules for a Resource Family
Note: The functionality described below is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can use the Downtime Schedule grid to define downtime schedules for the selected resource family. The application populates the Resolved Downtime Schedule grid on the Resource page with information from the Downtime Schedule grid.

This rule applies when defining downtime schedules for a resource family:

• The downtime schedule defined on the Resource page takes precedence over the resource downtime schedule define on the Resource Family page. Refer to "When Defining Downtime Schedules for a Resource" for information on defining downtime schedules for a resource.

How to Define a Resource Family

Follow these steps to define a Resource Family:

1. Open the Resource Family page. The Resource Family page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the resource family in the Resource Family field.
4. Enter optional information according to your business requirements. Refer to "Common Fields on Modeling Pages" for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
