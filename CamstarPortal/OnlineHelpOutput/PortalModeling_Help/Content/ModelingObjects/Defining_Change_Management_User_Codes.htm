Defining Change Mgt User Codes

Change mgt user codes are lists of codes specific to the requirements of the Change Management application. User codes can provide valuable tracking and reporting information. User-defined codes enable you to specify a list of allowable values for a specific field. You can use change management user codes to filter your searches for packages on the Package Search page. Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information.

When Defining Change Mgt User Codes

Most user codes consist of three attributes: name, description, and notes. Some user codes include more attributes for additional functionality or validation.

The name for each user code must be unique within its type. This value is used as an alternate key for look-up, data entry, and validation.

You can also create new user code types as needed.

You cannot use a space as the first or last character in the Name field when defining a change management user code.

Refer to "Common Fields on Modeling Pages" for information on the fields common to all modeling objects.

Change Mgt User Codes

This table describes the change mgt user codes provided by default with Change Management.

User Code

Description

Package Creation Reason

Reason for the creation of a change management package.

Package Creation Reason is a required field in the Create Package and Update Package transactions.

Package Creation Reason is an optional field in the Package Search transaction.

Package Priority Code

Indicator used to assign processing priority to a change management package.

The Relative Priority field enables you to associate a numerical value with the priority code to indicate its priority relative to other priority codes. For example, a relative priority for a "High" priority code could be "1," while a relative priority for a "Low" priority could then be "3."

Package Type

Indicator used to identify a type of change management package.

When Defining Package Priority Codes

Do not confuse the Package Priority Code modeling object with the Priority Level or Priority Code modeling object. Priority levels are assigned to events and quality records. Priority codes are assigned to containers, while package priority codes are assigned to change management packages. Refer to "Defining Priority Levels" for information on defining event and quality record priority indicators. Refer to "Defining Priority Codes" for information on defining container priority indicators.

How to Define a Change Mgt User Code

Follow these steps to define a change mgt user code:

1. Open the page for the user code you want to define. The modeling object instance page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the user code in the user code's Name field.
4. Enter optional information according to your business requirements. Refer to "Common Fields on Modeling Pages" for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
