Defining Process Timers

A Process Timer is a revisioned data object that controls and monitors the manufacturing process by defining actions to take if processing is initiated on a container before a minimum time is met or after the maximum allowable time for the process to execute has passed.

Process timers can be triggered for containers when:

• A Move or Move In is executed against a container at a specified step in a workflow.

Or

• A task is executed against a container in an electronic procedure.

Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information on using process timers.

Note: Process timers are only dependent on the Move (Standard) and Move In transactions. The application does not take into account the number of these transactions that can be performed on an EProcedure task. The process timer stops when the task is executed.

When Defining Process Timers

You can define actions to occur when a timer ends before a minimum time, when it ends after a maximum time, or both.

You can specify to either allow or not allow end transactions and disposition actions to execute, or you can specify to allow the end transaction without disposition actions if an electronic signature is obtained.

The timer continues running when the Min or Max Time is reached if you specify to not allow end transactions and disposition actions to execute. For less than minimum, the end transaction is allowed when the Min Time has been reached. For greater than maximum, you must manually manipulate the container (execute any transactions) and manually stop the timer using the Stop Timer button on the Process Timer Search page if the Max Time has elapsed. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information.

Clicking Save causes the application to validate data in the four timer fields to ensure:

• Min Time is greater than Min Warning Time.
• Max Time is greater than Max Warning Time.
• Max Time is greater than Min Time.
• An error occurs if the validation fails.

Process Timer contains the optional modeling definition Process Timer Type.

How to Define a Process Timer

Follow these steps to define a process timer:

1. Open the Process Timer page. The Process Timer page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this process timer in the Process Timer field.
4. Select the radio button under Timer for the behavior you want. The default is Min only.
5. Enter the desired times under Timer Format ([d.]hh:mm:ss).
Note: The fields will be marked as required or disabled based on your selection under Timer.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.

If you selected . . .

Then . . .

Min Only
a. Complete the desired sections under Disposition Details: When Timer Ends < Min.
b. Go to step 7.

Max Only
a. Complete the desired sections under Disposition Details: When Timer Ends > Max.
b. Go to step 7.

Min and Max
a. Complete the desired sections under both.
b. Go to step 7.
7. Click Save. The application saves the modeling object and displays a success message.
