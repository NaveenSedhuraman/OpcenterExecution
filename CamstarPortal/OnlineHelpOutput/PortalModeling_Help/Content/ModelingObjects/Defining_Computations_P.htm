Defining Computations

The Computation modeling object allows you to create and maintain process computation definitions for use with EProcedure tasks. You can use the user constants defined in Modeling when defining these computations.

When Defining Computations

Use the Opcenter EX MDD or Opcenter EX CR Unified Expression syntax when defining a computation expression. Expressions can contain a maximum of 4000 characters and may include:

• Variables; for example, (?InQty)
• User-defined constants; for example, Constant::Pi
Note: Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information on the Unified Expression syntax.

If your computation expression contains variables, the application adds the variables to the computation variable table when you tab out of the Computation Expression field or save the computation. The default data type for each variable is Integer. The variables in the expression must match those in the table. If you delete or modify a variable in the expression, you must delete or modify the variable in the table manually. Similarly, if you modify or delete the variable in the table, you must update the expression manually.

You can instruct the application to map computation results to a container when the computation is performed on the container. The application stores the computation results as a user-defined attribute. You can view, edit, and delete user-defined attributes on the Container Attribute Maintenance page. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information on the Container Attribute Maintenance page.

When Defining Limits

You can set an upper and/or a lower limit for the calculated result when defining a computation. You can use an expression to define the upper limit, lower limit, or both limits. The limits are determined at runtime when an expression is used.

Note: There is no validation of correct syntax when entering expressions. Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information on the Opcenter EX MDD or Opcenter EX CR Unified Expression syntax.

The application always accepts any value the user enters even when that value falls outside of a limit. The Display Limits and Limit Override Allowed check boxes determine whether the limits are displayed to the user at task runtime, the color to display to a user when a value is outside a limit, and whether to pass or fail the EProcedure computation task.

How to Define a Computation

Follow these steps to define a Computation:

1. Open the Computation page. The Computation page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter the name of the computation in the Computation field.
4. Enter a name for the calculated value in the Result Name field.
5. Select a Result Data Type from the list.
6. Enter the formula or expression for calculating the result in the Computation Expression field.
7. Click Save or tab out of the Computation Expression field. Any variables in the expression are added to the Computation Specification Parameter grid and assigned a default data type of Integer.
8. Select a row in the Computer Parameter Specification grid to enable the Data Type field.
9. Select a different Data Type for each variable if necessary.
10. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
11. Click Save. The application saves the modeling object and displays a success message.
