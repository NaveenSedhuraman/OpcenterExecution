Setting Up Carriers

Use the Resource Setup transaction to record changes to a Carrier setup or to indicate the availability of a Carrier. A Carrier’s availability determines whether an employee can use it to perform shop floor transactions.

Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information on resource setup and availability. Use the procedures on this page to simply set up the Carrier.

When Defining Carrier Setup

Setup information includes a reference to a specific Carrier setup definition and the Carrier being processed.

Availability information includes the Carrier’s availability (Up or Down), Status (such as Dirty, Clean, Dirty but usable), and a Reason (such as Dirty because of acid use). The application uses the availability information to create a history of Carrier use and availability details.

For example, when a Carrier is unavailable, you can perform a Resource Setup transaction to change the availability to Down, the Status to Dirty, and the Reason to Dirty because of acid use. When the Carrier is cleaned and available once again, you can perform another Resource Setup transaction to change the availability to Up and the Status to Clean. You now have a record of when the Carrier was unavailable, the reason it was unavailable, and the amount of time before it was available again.

How to Set Up a Carrier Resource

Follow these steps to set up a Carrier:

1. Open the Resource Setup page from the Resource menu. The Resource Setup page appears.
2. Select the appropriate Carrier Group from the Resource Group list to limit the list of Resources displayed in the Resource field.
3. Select the carrier to set up from the Resource field.
4. Select a resource status code from the Resource Status Code field.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click the Setup button to collect the data you entered about the setup.
7. Click Save. The application displays a success message indicating the modeling object was updated.
