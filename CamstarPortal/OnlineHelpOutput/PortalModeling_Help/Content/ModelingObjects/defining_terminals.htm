Defining Terminals

A terminal is a shop floor computer uniquely identified by an IP address. The Terminal page enables you to define and configure a terminal with terminal-based line assignment settings. Terminal-based line assignment settings override employee-based line assignment settings while an employee is logged onto the configured terminal.

An employee who logs onto the terminal will not be able to change any of the terminal-based line assignment settings unless the factory is configured with Allow Terminal Line Assignment Override selected. Changes made using this option only apply for the duration of the employee's current session on the terminal. Refer to "Defining a Factory."

How to Define a Terminal

Follow these steps to define a terminal:

1. Open the Terminal page. The Terminal page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the terminal in the Terminal field.
4. Enter the IP Address of the terminal.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.
