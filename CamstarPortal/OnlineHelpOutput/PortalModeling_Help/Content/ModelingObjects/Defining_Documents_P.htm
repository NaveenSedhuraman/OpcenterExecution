Defining Documents

A Document instance contains the information required to view an external document. Each definition includes the document's file name and revision.

You can browse for and upload any type of file and associate it with the Document object, and you can enter any version number for the file. If a file attached to the object instance has been saved in the database, the name of the file is displayed in the Stored File Name field.

You can also download and save a file to the database that was uploaded initially as a part of the object instance.

Most modeling entities include a reference to a document. Documents are used in this context to provide extended (unstructured) information, such as product drawings, scanned images, relating existing manufacturing specifications, or operating procedures. Each instance of a document (CDO) includes an identifier, such as a file name, for the external document.

Note: Recipe is a subclass of Document. Refer to "Defining Recipes" for information on the Recipe modeling object.

Requirements

You must have software on your computer or server for the type of document you are trying to open.

HTTP browsing is restricted to the Portal server's virtual directories, so the document must be in a directory such as the Camstar Portal folder or one of its subfolders. (You cannot use a path such as C:\My Documents to define the location of the document.)

The default path used for URL browsing is based on the URL used to log in for the session. Therefore, you may need to edit the URL for the document. For example, if you logged in with https://localhost/CamstarPortal/default.htm#/login, you will not see the fully qualified domain name in the Locate File field after you select a value in the View Mode field. You should either log in with the fully qualified domain name or edit the URL to use the fully qualified domain name to ensure users across other domains can access the document.

Default browser security settings prevent documents on unsecured websites from opening by default. You must change your settings to allow unsecured content.

Note: You are logged out of the application when you change this setting in the browser.

When Defining a Document

These rules apply when defining documents:

• Each instance of a document includes a file location in the Locate File field, except when defining a hyperlink to an external web address.
• Your View Mode selection determines whether the application uploads the file to the database, links to a file, or links to an external web address. Select Local File and click the File Upload icon to navigate to and upload the specified file to the database.
• The Stored File Name field is a read-only field that displays the name of the local file uploaded when you clicked Save. The Stored File Name field remains empty when you link to a file (the View Mode is HTTP File).
• The download/upload time of files varies by file size.
• Documents added as Reusable Documents using the Attachment Document function are visible on the Document page.

Document is:

• A required field in the definition of Document Set.
• An optional field in the definition of Spec (in Recipe File) and WIP Message.
Note: A Document definition can have related WIP Messages if it is associated with a field from a container definition. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information.

When Accessing Files in an External Repository

You can access documents stored in an external repository from within the Portal. You can set a user name and password on the Employee modeling object if the external repository requires a different user name and password than is used to access the Portal. You must set Authentication Type to Basic to use this feature.

• View Mode determines whether the application uploads the file to the database, links to a file, or links to an external web address. Select HTTP File to link to the specified file in the external repository.
• The application displays the Authentication Type pick list only if View Mode is set to HTTP File.
• You must set Authentication Type to Basic on the Document modeling object to use the user name and password fields set on the Employee modeling object. The application passes the user name and password entered on the Employee modeling object to the external document repository for authentication if you select Basic.
• The user name and password entered on the Employee modeling object must match the user name and password required for accessing the repository.
• The application passes the user’s credentials to the repository when the Authentication Type is set to Basic if a user name and password are not specified on the Employee modeling object.
When Linking to Electronic Work Instruction Files

You can link to EWI files to enable users with access to Teamcenter to view EWI information in a separate pop-up.

• View Mode determines whether the application uploads the file to the database, links to a file, or links to an external web address. Select URL to link to an EWI file.
• The application displays the URL field only when the View Mode is set to URL. Selecting URL for the View Mode also hides the Authentication type, Locate File, Stored File Name, and File Version fields.
• EWI files are viewed in a separate pop-up. The pop-up prompts you for your Teamcenter user name and password before displaying the EWI file. Your Teamcenter session will remain open until your browser is closed.
• The URL field does not accept URLs containing any parameters that configure work instructions, such as product view, final assembly, document viewer settings, or any login credentials when defining a link to an EWI file.

How to Define a Document

Follow these steps to define a Document:

1. Open the Document page. The Document page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the document in the Document field.
4. Enter the revision of this document in the Revision field.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object, enables the View Document button, and displays a success message.

How to Add a File to a Document Definition

Follow these steps to add a file to a document definition:

1. Perform the "How to Define a Document" procedure.
2. Do you want to upload a file to the database, hyperlink to a file, or hyperlink to an external web address or EWI file?

If you want to . . .

Then . . .

Upload a file to the database
a. Select Local File from the View Mode field, if not already selected. The Browse button appears.
b. Click the File Upload icon next to the Locate File field. A File Upload pop-up appears.
c. Navigate to and select a file, and then click Open. The File Upload pop-up closes and the file name appears in the Locate File field.
d. Click Save. The application displays a success message and the file name appears in the Stored File Name field.

Hyperlink to a file
a. Select HTTP File from the View Mode field. The Locate File field displays an arrow.
b. Select Basic or None from the Authentication Type field.
c. Select a file from the Locate Field field. The hyperlink to the file appears in the Locate File field.
d. Click Save. The application displays a success message indicating the modeling object was updated.

Hyperlink to an external web address or an EWI file.
a. Select URL from the View Mode field. The URL field appears.
b. Enter a hyperlink to an external web address or an EWI file in the URL field.
c. Click Save. The application displays a success message indicating the modeling object was updated.
