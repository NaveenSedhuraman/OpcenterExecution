Defining UI Preferences

A User Interface (UI) Preference enables you to specify the information to display for a container, carrier, or event record in the Status tab on the Manufacturing Audit Trail page. It also enables you to specify the information to display for a resource record in the Status tab on the Resource Audit Trail page. The UI preference also determines the order in which the application displays the information on the Status tab.

When Defining a UI Preference

You cannot use invalid characters in the object name. For example, you cannot use an apostrophe. Refer to the Opcenter Execution Medical Device and Diagnostics Designer User Guide or the Opcenter Execution Core Designer User Guide for information on valid characters.

When Specifying Fields to Display

Use these formats for field names:

• Container.FieldName when specifying the fields to display for container and carrier records
• QualityObject.FieldName when specifying fields to display for event records
• Resource.FieldName when specifying fields to display for resource records

Associating UI Preferences with Record Types

UI Preference is an optional field on the following modeling objects:

• Container Level
• Organization
• Resource Type
• Resource Family

You must associate UI preferences with the record types before any information appears in the Status tab on either the Manufacturing Audit Trail or Resource Audit Trail pages. You associate the UI preference for container and carrier records in the UI Display Configuration field on the Container Level modeling object. You can associate a different UI preference with each defined container level. Refer to "Defining Container Levels" for information on container levels.

You associate the UI preference for event records in the UI Preference Map on the Organization modeling object. You can associate a different UI preference for each event record type. Refer to "Defining an Organization" for information on associating UI preferences.

Additionally, you must set the organization as the primary organization for the employee recording the event. The Status tab appears for event records only if a UI preference has been specified on the organization and that organization has been set as the event recording employee's primary organization. Refer to "Defining Employees" for information on specifying a primary organization.

You associate the UI preference for resource records in the UI Preference field on either the Resource Type or Resource Family modeling objects, or both. Resource Type and Resource Family are optional fields on the Resource modeling object and both may specify a UI preference. The application uses the UI preference specified with Resource Type first. It uses the UI preference defined with the Resource Family if one is not defined with Resource Type. You can associate a different UI preference with each defined resource. Refer to "Defining Resource Types" and "Defining Resource Families" for information.

How to Define a UI Preference

Follow these steps to define a UI Preference:

1. Open the UI Preference page. The UI Preference page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the UI preference in the UI Preference field.
4. Click Add new row in the Fields to Display grid. A new row appears.
5. Enter the name of the field you want to display in the UI Pref Field Def Name column.
6. Repeat steps 4-5 to specify additional fields.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Save. The application saves the modeling object and displays a success message.

How to Specify the UI Preference to Use for Container and Carrier Records

Follow these steps to specify the UI preference to use for container and carrier records:

1. Perform the "How to Define a Container Level" procedure.

Or

Select an existing Container Level instance.

2. Enter or select a UI preference in the UI Display Configuration field.
3. Repeat this procedure for each Container Level.
4. Click Save. The application saves your changes, and uses this UI preference when displaying information on the Manufacturing Audit Trail page's Status tab for the specified container or carrier record.

How to Specify the UI Preference to Use for Event Records

Follow these steps to specify the UI preference to use for event records:

1. Perform the "How to Define an Organization" procedure.

Or

Select an existing Organization instance.

2. Click Add new row in the UI Preference Map grid. A new row appears.
3. Enter or select the appropriate Record Type.
4. Enter or select the UI Preference to associate with the record type.
5. Click Save. The application saves your changes, and uses this UI preference when displaying information on the Manufacturing Audit Trail page's Status tab for the specified quality record.
