Defining Document Sets

A Document Set is a collection of one or more documents. Document sets are referenced by other modeling entities and provide extended information such as drawings, scanned images, procedures, and recipes. For example, a Setup definition can reference a document set that contains an engineering drawing and setup instructions.

Each instance of a document (CDO) includes an identifier for the external document (such as a file name or a URL) and a reference to the program used to present the Document.

When Defining a Document Set

You must add a minimum of one document to a document set. The application prevents you from saving a document set if you have not added any documents.

Select a document in the Documents grid and click the View Document button to view the selected document. The application enables the View Document button when you select a file.

Viewing the document helps ensure you are adding the correct document to the document set. You should ensure the following if the document does not display in the browser:

• The URL is defined correctly for the Document object.
Important: The application will block http URLs by default. Refer to your browser settings for information on enabling unsecure content in a secure site.
• You must have the software on your computer or server for the type of document you are trying to open.
• EWI documents open in a separate pop-up that prompts you for your Teamcenter user name and password. You must have access to Teamcenter to view EWI files. Refer to "Defining Documents" for information on defining EWI documents.

Note the following:

• Document Set contains the required modeling definition Document, which includes Recipes.
• Document Set is an optional field in the definition of Setup and Spec.
• A Documents pop-up appears when you click Add new row on the Documents grid.

How to Define a Document Set

Follow these steps to define a Document Set:

1. Open the Document Set page. The Document Set page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for the document set in the Document Set field.
4. Click Add new row in the Documents grid. The Documents pop-up appears.
5. Enter the name of the document entry in the Document Entry Name field.
6. Select a Document or Recipe from the Document field.
7. Click OK. The Documents pop-up closes and the specified information appears in the Documents grid.
8. Repeat steps 4-7 to add additional documents.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click Save. The application saves the modeling object and displays a success message.
