Opcenter Execution Core Scheduling

Note: The functionality described below is available only when Opcenter Execution Core Scheduling is installed. Refer to the Opcenter Execution Core Scheduling User Guide for information.

Opcenter Execution Core Scheduling provides a mechanism to schedule Opcenter EX MDD and Opcenter EX CR manufacturing orders using the Preactor Advance Scheduling application.

Opcenter Execution Core Scheduling enables an organization to define Enterprise Resource Planning (ERP) routing data for products and manufacturing orders on Opcenter EX MDD or Opcenter EX CR and then use that data in the Siemens Preactor Advanced Planning and Scheduling (Preactor) application to create production schedules for manufacturing orders.

You must perform these tasks in the application to enable Opcenter Execution Core Scheduling:

• Run the provided CIO modeling scripts to configure the CIO adapters for Preactor
• Enable the interface for the factory
• Provide required ERP route information such as processing time
• Assign the ERP route and route steps to associated workflows and workflow steps
• Assign the workflow to the associated product or product family
• Create a manufacturing order with a valid product and an order status whose state is open

Information on the scheduled orders is sent to a staging table to be transmitted back to the Opcenter EX MDD or Opcenter EX CR application when the planner releases the schedule from Preactor. You can define another Opcenter Connect MOM database adapter to invoke a stored procedure to return the scheduled order information to the application in the form of a dispatch list.
