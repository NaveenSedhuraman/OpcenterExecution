Defining Order Statuses

An Order Status indicates the order state of a manufacturing order. You can use the Order Status object to assign an order state of open or closed.

When Defining Order Statuses

Order status is an optional field in the Mfg Order modeling definition. However, you must assign an Order Status with an open state to a Mfg Order instance to work with Opcenter Execution Core Scheduling. Refer to the Opcenter Execution Core Scheduling User Guide for information.

The Is Confirmed check box is available when Opcenter Execution Electronics is installed and enables you to confirm the order status for production. Refer to the Opcenter Execution Electronics User Guide for information.

Refer to “Common Fields on Modeling Pages” for information on the fields common to all modeling objects.

When Defining Order Statuses for Use with Opcenter Execution Semiconductor
Note: The functionality described below is available only when Opcenter Execution Semiconductor is installed.

Opcenter Execution Semiconductor requires:

• A work order to be assigned to each lot scheduled using Preactor. You can assign work order to a lot when using the Lot Start or Lot Create page or when entering preliminary scheduling data using one of the Scheduling pages.
• The work order to have an assigned Order Status. You must configure the Order Status with an Order State equal to "Open" and the "Is Available" check box selected.

How to Define an Order Status

Follow these steps to define an Order Status:

1. Open the Order Status page. The Order Status page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this order status in the Order Status field.
4. Enter optional information according to your business requirements.
5. Click Save. The application saves the modeling object and displays a success message.
