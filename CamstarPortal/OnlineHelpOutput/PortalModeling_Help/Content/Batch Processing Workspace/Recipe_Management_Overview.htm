Recipe Management Overview

Note: The functionality described below is available only when Batch Processing is installed. Refer to the Opcenter Execution Medical Device and Diagnostics Batch Processing User Guide or the Opcenter Execution Core Batch Processing User Guide for information.

Sometimes recipes require adjustment to the quantities defined in the BOM. Reasons for adjustment include changes in the exact amount of components previously added, differences in the potency of raw materials, or different batch sizes. Batch processing functionality enables you to adjust material list quantities by providing expressions to calculate the target quantity and the required minimum and maximum quantities during recipe processing. You can define these expressions on the product or the recipe’s weigh issue task item.

Quantities resolved from expressions defined on the component product take precedence over the quantities defined on the material list. Quantities resolved from expressions defined on the weigh issue task take precedence over quantities resolved from expressions defined on the product. Refer to "Defining Expressions to Adjust Component Quantities on the Product" for information on defining quantity expressions on the product. Refer to "When Defining Weigh Issue Tasks" for information on defining quantity expressions on the weigh issue task.

Batch processing functionality also enables you to define intermediate batches that must be processed before being added to the main batch. You can define expressions on the Task List modeling object to calculate the quantity of the intermediate batch. Refer to "When Defining an Intermediate Batch" for information on defining intermediate batches.
