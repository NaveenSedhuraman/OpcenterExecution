Defining Scales

A Scale is a resource that provides the setup information for a scale used for processing batches. This information includes:

• Working range
• Precision
• Scale vendor information
• Unit of measure (UOM)

You can assign scales to specs, products, and product families as part of a scale group. Scale groups are used to create logical groupings of scales to provide selection lists of valid scales to the user. Refer to "Defining Scale Groups" for information.

You can maintain the status and state of a scale using the Resource Setup page. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information on the Resource Setup page.

When Defining Scales

You can use:

• The Minimum Capacity and Maximum Capacity fields to indicate the working range of weight the scale can measure.
• The Total Digits (Precision) and Decimal Places (Scale) fields to indicate the precision of the scale's measurement.
• The Readability field to indicate the smallest increment of the scale's display.

When Integrating Scales

Opcenter Connect MOM and CIO functionality are both used to integrate scales. Any scale can be used with Batch Processing. Refer to the Opcenter Connect MOM User Guide for information on Opcenter Connect MOM. Refer to "CIO Workspace" for information on CIO modeling objects.

How to Define a Scale

Follow these steps to define a Scale:

1. Open the Scale page. The Scale page appears within the Modeling page.
2. Click New. Blank fields appear for you to define a new instance.
3. Enter a name for this scale in the Scale field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves the modeling object and displays a success message.
