Understanding Deleted Object References

Deleting a modeling object removes instance information from the database, except for its history, which can be viewed in the modeling audit trail.

If a deleted object is referenced by another modeling object, that reference is now invalid as it refers to an instance that is no longer available. You cannot save modeling objects with fields containing deleted references. You must either set the reference to a different modeling object or clear the deleted object reference before you can save it.

Deleted Object Example

When you select a modeling object on a maintenance page, the application displays the deleted object’s instance ID instead of its name.

For example, if you delete the Role Type that is referenced by a Role object, the deleted role type reference on the Role maintenance page displays its instance ID.

Lists do not display deleted object references.

Caution About Deleting Objects

When granting permission to delete objects, verify that employees with this permission understand the implications of object deletion.

• Deleted objects (Specifications, Operations, etc.) that are referenced by workflows will cause WIP containers on those workflows to encounter processing problems across the workflow.
• Deleted objects (Employee, Workflow, Workflow steps, etc.) that are referenced in history objects will cause reporting inconsistencies and errors on the Mfg Audit Trail page. These inconsistencies can cause significant issues for regulated customers' traceability.
