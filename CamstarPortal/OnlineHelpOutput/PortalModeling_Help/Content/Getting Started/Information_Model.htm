Information Model

The Manufacturing information model contains the data objects that describe and control product-processing activities. You use Modeling pages in the application to construct a unique model for each of your manufacturing facilities.

Familiarizing yourself with these modeling objects and their requirements will help you plan and create your company’s information model.

The information model for each of your manufacturing facilities is composed of these components:

• A physical model
• A process model
• An execution model

It is expected that you will revisit each of these components during the modeling process until your model is complete. There is great flexibility in how these components can be used in your specific model.

Physical Model

The physical model represents physical components of your manufacturing facility such as:

• Enterprise
• Factory
• Organization
• Employee
• Role
• Resource Status Reason
• Resource Status Reason Group
• Resource Status Code
• Resource Status Code Group
• Resource Status Model
• Resource Type
• Resource
• Resource Family
• Resource Group
• Carrier
• Carrier Family
• Carrier Group
• Carrier Status Code
• Carrier Status Reason
• Document Viewer
• Document
• Recipe
• Document Set
• Setup
• Work Center

Process Model

The process model represents the control portion of your information model. It contains objects that represent your manufacturing process flow such as:

• Container Level
• Operation
• Spec
• Product Family
• Product
• Bill of Process
• Bill of Materials (BOM)
• ERP BOM
• ERP Route
• Recipe List
• Master Recipe
• Workflow
• Process Timers
Note: Refer to "Defining Workflows" for information on defining workflows.

Execution Model

The execution component is the aspect of your information model in which all the activity occurs, based on the configurations of your physical and process models. It tracks the status and history of products and resources on the shop floor and provides a real-time view of production activity. The tracking functions are implemented using the Portal Shop Floor.

Execution includes the following key components:

• Tracking and control
• Real-time WIP status and genealogy
• Real-time machine status and history

These objects can be modeled to facilitate execution:

• Customer
• Vendor
• Sales Order
• Mfg Order
• Data Collection Definition
• User Code
• E-mail Notification
• SMTP Transport
• Text Variable
