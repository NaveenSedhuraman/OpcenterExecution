Named Data Objects and Revisioned Objects

The Opcenter EX MDD and Opcenter EX CR applications have two types of objects: Named Data Objects and Revisioned Objects. Both types are available on the Modeling menu.

When Defining a Named Data Object and Revisioned Object

You cannot use invalid characters in the object name. For example, you cannot use an apostrophe. Refer to the appropriate product-specific system Designer user guide below for information:

• Opcenter Execution Core Designer User Guide
• Opcenter Execution Medical Device and Diagnostics Designer User Guide
Note: Set your preferred language before defining modeling object instances if you want the application to display all updates in your preferred language in the Modeling Audit Trail.

Named Data Objects

A named data object (NDO) is a class of objects within the application that is identified by a unique name, such as Enterprise or Operation. NDO instances are persistent; that is, information about them is written to the database.

Revisioned Objects

A revisioned object (RO) is an object that has a name and a revision; for example, Spec or Product. Revisioned objects support a revision of record feature that enables you to specify one version as the default version. Like NDOs, ROs are persistent.

Another difference between the named data object maintenance page and that of the revisioned object is the buttons on the page. In addition to the Close, Save, Copy, and Delete buttons, all revisioned objects have two additional buttons: New Rev and Copy Rev. Refer to "Revision Management" for information on using these buttons.
