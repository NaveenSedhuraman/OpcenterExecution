Modeling Page

The Modeling page provides access to all of the modeling objects. However, you may only see a subset of the available modeling pages depending on how the permissions are set for your employee definition. The page allows you to select a modeling object and open that object's page within the Modeling page.

This image shows an example of the Modeling page.

Change Mgt Save Option

An additional option labeled Change Mgt Save appears on the Modeling page if you are using Change Management and have created at least one change management package. You can create and update modeling object instances and assign them to existing, active packages using this option. Refer to the product-specific Change Management guide below for information:

• Opcenter Execution Core Change Management User Guide
• Opcenter Execution Medical Device and Diagnostics Change Management User Guide

Searching for a Modeling Object

Enter the first few characters or the entire modeling object name in the filter field to search for a specific modeling object. Refer to "Filtering the List of Instances" for information on how to use the filter feature.

Opening a Modeling Page

Modeling objects are accessed through the Modeling page. Refer to "Accessing Modeling Pages" for information on selecting and accessing a modeling page.

Common Fields on Modeling Pages

Modeling object pages contain fields common to other modeling objects. These fields appear in the header section of the page and at the bottom of the page.

This table defines fields that appear on the Modeling pages.

Field

Definition

Type

<modeling object>

Unique name for the modeling object. Refer to each modeling object topic for specific information.

Required

Filter Tags

Check box enabling you to assign one or more filter tags to the modeling object.

Optional

Revision Locking icon

Icon indicating the lock status of this revision. When selected, this icon locks this revisioned object.

Note: This icon appears for revisioned objects only.

Optional

Revision

Revision, or version, of the definition. Both the name and revision are required to identify the definition.

Note: This field appears for revisioned objects only.

Required

Is Revision of Record

Check box indicating whether this revision is the revision of record. The application requires one revision to be defined as the revision of record, and only one is allowed.

Note: This field appears for revisioned objects only.

Optional

Description

Short description of the modeling object. You can enter a maximum of 255 characters.

Optional

Notes

Relevant comments about the modeling object. You can enter a maximum of 2000 characters.

Optional

Change Mgt

Note: This tab appears only if you are using Change Management and have created at least one change management package.
