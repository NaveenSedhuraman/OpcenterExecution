Platform Architecture

The application's platform architecture is unique in the way it uses configuration data, not source code, to define application objects and the object’s behavior. The two basic components of this architecture are:

• Configurable Data Objects
• Configurable Logic Flows

Configurable Data Objects

Configurable Data Objects (CDOs) define the data elements (or fields) of each object and reference CLFs to implement the object’s behavior. All of the specific information about the CDOs and CLFs are stored in data objects. The application's architecture allows you to extend the application and modify it without changing the source code.

A CDO is a user-defined data structure that falls into two categories:

• Service
• Data

Client applications interact with Service CDOs by setting the value of certain fields and then executing the Service. The Client application retrieves the result of the Service through other fields within the Service.

Data CDOs are the application objects that represent attributes such as:

• Product definitions
• Work centers
• Machines
• Material being produced

Configurable Logic Flows

Configurable Logic Flows (CLFs) link a set of prebuilt components (Executions) to perform a specific function. CLFs are the mechanism the Portal uses to implement user-defined methods. All application logic is provided through the use of CLFs that execute functions on the Application Server.

The application provides a comprehensive library of executions to perform tasks such as calculations, searching data structures, querying the database, and validating data.
