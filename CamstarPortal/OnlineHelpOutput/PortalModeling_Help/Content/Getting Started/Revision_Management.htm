Revision Management

A revisioned object is used for objects that have a name and a revision. Revisions enable you to maintain multiple versions of the same modeling definition, with one revision designated as the revision of record. The latest revision is designated by default as the revision of record until you designate another. Other revisions may be active and available for use, but the revision of record is the revision currently designated as the default in Modeling and on the shop floor.

Revision of Record

The revision of record feature reduces model maintenance and ensures the use of correct revisions on the shop floor.

For example, when you build a workflow, you create steps by referencing specifications (or other workflows). When you reference a specification, you must select either a specific revision or the revision of record. If you:

• Reference a specific revision, the workflow always uses that revision even if new revisions are created.
• Reference the revision of record, the workflow always uses the revision of record, regardless of the actual revision number.

The advantage is that when you reference the revision of record, you do not need to update the workflows that use that Spec every time a new revision of the specification is approved for use.

Only one revision can be designated as the revision of record, but you can change the designation at any time and make another revision the revision of record.

When Defining a New Revision

You can define a revision of an existing revisioned object instance using either of two buttons:

• The New Rev button creates a copy of the revision with all fields blank except the name field. Use New Rev if most attributes will be unique to the new revision.
• The Copy Rev button creates a copy of the revision with all the information from the original revision except the revision number.

The first instance of a revisioned object to be created is the revision of record by default. Each subsequent revision defined is by default the new revision of record.

Revisioned Objects in the Instance Selection List

The instance selection list displays in expanded view so that you can see the instances and their associated revisions.

Clicking on a revisioned object name displays the maintenance page for the revision of record of that instance. Clicking the name of a specific revision displays the maintenance page for that revision.

The revision of record is indicated by a checkmark displayed in a box after the name of the revision.

How to Define a New Revision

Follow these steps to define a new revision:

1. Click the page that contains the revisioned object for which you want to define a new revision. The instance selection list appears.
2. Click the instance name. A page appears displaying the selected instance.
3. Click New Rev. The page for the new revision appears.
4. Enter any required information such as Operation, Product Type, etc.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application updates the revision and displays a success message.

How to Delete an Instance or Revision

Follow these steps to delete an instance or revision:

1. Click the page that contains the revisioned object for which you want to delete an instance or revision. The instance selection list appears.
2. Click a specific instance or revision.
3. Do one of the following:

If you want to delete . . .

Then . . .

All revisions
a. Select the ROR of the modeling object. The application displays the Delete All button.
b. Click Delete All. The Delete dialog box appears.
c. Go to step 4.

A specific revision
a. Select a specific revision of the modeling object. The application displays the Delete Rev button.
b. Click Delete Rev. The Delete dialog box appears.
c. Go to step 4.
4. Click Delete in the Delete dialog box. The selected revision or all revisions are deleted, depending on the option you selected in step 3.
