Locking Change Package Modeling Object Instances

Siemens provides two types of modeling object instance locking:

• Revision locking
• Change package modeling locking

You cannot modify or delete a locked change package modeling object instance. The application displays an error message when you try to modify or delete a locked instance.

Revision Locking

Revision locking is available by default. Revision locking provides the ability to lock or unlock an instance of a revisioned object (RO). Clicking the lock icon on the instance's Modeling page locks and unlocks the instance. Refer to "Revision Locking" for information.

Change Package Modeling Locking

Change package modeling locking is available with Change Management. Change package modeling locking provides the ability to lock modeling object instances included in a change management package based on the package's current spec (step). Both RO and NDO instances can be locked. A setting on the change management spec determines whether the change package modeling object instances are locked or unlocked when the change management package enters that spec (step).

You can assign modeling object instances to multiple change management packages concurrently regardless of whether they are locked or unlocked in other packages. The modeling object instances remain locked when associated with a change management package until all packages with which they are associated are closed, voided, or returned to draft step.

Note: The default draft step provided by Siemens unlocks the associated modeling object instances. You can configure this step (spec) and other specs to meet your business needs.

Refer to "Defining Change Mgt Specs" for information on setting the property to lock modeling object instances when the change management package enters that spec. Refer to the appropriate product-specific change management guide below for information on change management packages:

• Opcenter Execution Core Change Management User Guide
• Opcenter Execution Medical Device and Diagnostics Change Management User Guide

Revision and Change Package Modeling Locking

RO instances can be locked using both revision and modeling locking. NDO instances can be locked using change package modeling locking.

You can include a locked RO in a change management package. An RO locked on the instance page remains locked regardless of whether it is locked or unlocked in a change management package. The lock on the Modeling page is not affected by change management.

The error message displayed when you try to modify or delete a locked RO indicates where the instance is locked. The error message specifies the object instance is locked either because of an association with a change management package or because it is locked (on the Modeling page).
