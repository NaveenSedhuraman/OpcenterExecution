Revision Locking

Revision locking provides the ability to dynamically lock or unlock an instance of a revisioned object. It is controlled with the lock icon.

Security and Role-based Access Control

Security administration for Opcenter EX MDD and Opcenter EX Core is based on a role-based access control model. Security administration is used to set up user permissions, which affect who can use the lock/unlock feature.

Refer to these topics for information on the objects used to set up role-based access control:

• Defining an Organization
• Defining Roles
• Defining Employees

When Locking and Unlocking Instances

When locking and unlocking an instance of a revisioned object, remember:

• You do not have to save an instance to lock or unlock it.
• After you lock an instance, you can still access its fields; however, you will not be able to save it.
• You cannot delete a locked instance.

The application displays an error message when you try save a locked instance.

When Copying Locked Instances

When copying a locked instance of a revisioned object, you use these two buttons.

This table defines the buttons for copying a locked instance of a revisioned object.

Button

Click this button to . . .

Copy

Create a copy of the locked instance with all of its original information except the name; this new instance is unlocked.

The first instance of a revisioned object is the revision of record (ROR) by default. Each subsequent revision created becomes the new ROR by default.

Copy Rev

Create a copy of the locked instance with all of its original information except the revision; this new instance is unlocked.

The first instance of a revisioned object is the ROR by default. Each subsequent revision created becomes the new ROR by default.

How to Lock an Instance

Follow these steps to lock an instance:

1. Click the page that contains the revisioned object with the instance you want to lock. The instance selection list appears.
2. Click an instance name. A page appears displaying the selected instance and the open lock icon.
3. Click the lock icon. The instance is locked and the closed lock icon displays.
Note: An error message appears if you are not authorized to lock instances.

How to Unlock an Instance

Follow these steps to unlock an instance:

1. Click the page that contains the revisioned object with the instance you want to unlock. The instance selection list appears.
2. Click an instance name. A page appears displaying the selected instance and the closed lock icon.
3. Click the lock icon. The instance is unlocked and the open lock icon displays.
Note: An error message appears if you are not authorized to unlock instances.

How to Copy a Locked Instance

Follow these steps to copy a locked instance:

1. Click the page that contains the revisioned object with the instance you want to copy. The instance selection list appears.
2. Click an instance name. A page appears displaying the selected instance and the closed lock icon.
3. Click Copy. The Copy pop-up appears.
4. Enter a Target Instance Name and a New Revision Name, then click Copy. The copied instance appears displaying the new name, revision, and the open lock icon.
Note: The copied instance is unlocked, but the original instance remains locked.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.

How to Copy a Revision of a Locked Instance

Follow these steps to copy a revision of a locked instance:

1. Click the page that contains the revisioned object with the instance/revision you want to copy. The instance selection list appears.
2. Click an instance name. A page appears displaying the selected instance and the closed lock icon.
3. Click Copy Rev. The Copy pop-up appears with a field in which to enter a new revision name.
4. Enter a New Revision Name and click Copy. The copied revision instance appears displaying the original name, new revision, and the open lock icon.
Note: The copied instance revision is unlocked, but the original instance remains locked.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Save. The application saves the modeling object and displays a success message.
