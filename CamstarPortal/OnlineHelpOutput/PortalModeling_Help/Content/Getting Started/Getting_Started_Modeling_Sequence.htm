Modeling Sequence

You must create some components in a specific sequence because modeling definitions are interrelated.

Required and Optional Modeling Definitions

Each modeling definition may be dependent upon the predefinition of another. For each definition, you may have one, both, or none of the following:

• Required Definitions
You must complete some definitions before you create other definitions because they are required fields within the other definitions. For example, you must define a specification before defining a workflow because the specification is a required field for a Workflow definition.
• Optional Definitions
The optional definition is an optional field within another definition. For example, you may want to define an Enterprise before you define a Factory because Enterprise is a field within the Factory definition. However, since the Enterprise field is optional, it is not required that you define Enterprise ahead of time. Therefore, Enterprise is an optional definition for Factory.

Physical Modeling Sequence

You define the physical model first. The main components of the physical model are Enterprise, Factory, Location, and Resource. Other objects are Resource Group, Setup, Work Center, Document Set, Document, and Document Viewer.

With the exception of Document, these objects can be defined in any sequence because none are required to define the others, but the modeling sequence indicates that the object below is a field in the one above it. Document has to be defined before Document Set because it is a required field in Document Set.

Process Modeling Sequence

The second phase of modeling is process modeling. Process modeling can be divided into the following components:

• Workflow modeling
• Product modeling

Workflow Modeling Sequence

Workflows are the fundamental components of your information model. A workflow defines a sequence of steps that are used to manufacture a product. Each step in a workflow references a Specification.

Product Modeling Sequence

Products are the materials produced in a factory or by outside suppliers for a factory. Products can be end-items, subassemblies, and components. You can group products into product families for easier maintenance. A BOM (bill of materials) contains the components and quantities required to manufacture a Product (end-item or subassembly).

Execution Modeling Sequence

Execution modeling is the final modeling phase. Execution modeling includes the following components:

• Tracking and control
• Real-time WIP status and genealogy
• Real-time machine status and history

Tracking and Control

You track and control products on the shop floor by using containers. Each container has a unique identifier used for traceability and the performance of shop floor transactions. Containers use tracking designations called levels. You can also track multiple levels of related containers, allowing a single transaction to be applied to a container grouping.

Container levels are defined in Modeling, but containers are started through the Start page in Portal Shop Floor.

WIP Status and Genealogy

The application tracks real-time work in process (WIP) status and genealogy information for each container on the shop floor. WIP status information includes quantities, position in the workflow, dates, and status. Genealogy information allows the tracing of activities such as movements of containers, combines, splits, and changes to container values.

Using the Portal you can view WIP information such as container status and container history.

WIP Messaging can be used to perform the following tasks:

• Display a reminder or special processing instructions to the person doing the work.
• Notify an engineer that a container has reached a specific location.
• Place a container on hold when it reaches a specific point.
• Require an operator or supervisor to acknowledge that the special instructions provided by the WIP Message have been reviewed.

Machine Status and History

The application tracks real-time status and history of resources, including availability, status, time at current and previous statuses, maintenance and calibration data, and setup information. It supplies a comprehensive set of execution transactions that manage the flow of work through your operations and provide a real-time view of production activity. These transactions are accessible through the Portal Shop Floor.
