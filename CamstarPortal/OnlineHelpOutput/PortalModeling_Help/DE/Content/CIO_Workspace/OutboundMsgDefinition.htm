Erstellen von Definitionen für ausgehende CIO-Nachrichten

Das Objekt "Definition ausgehender CIO-Nachrichten" ermöglicht Ihnen das Erstellen von Nachrichtendefinitionen für ausgehende CIO-Nachrichten, indem der Service zum Generieren einer ausgehenden Nachricht ausgewählt wird.

Beim Erstellen einer Definition für ausgehende CIO-Nachrichten

Beim Erstellen der Definition ausgehender CIO-Nachrichten gelten die folgenden Regeln:

• Definitionen ausgehender Nachrichten werden nach den Diensten benannt, von denen sie ausgelöst werden.
• Ausgehenden Nachrichten muss ein Nachrichtentyp zugewiesen sein.
• Ausgehenden Nachrichten muss ein Kanaladapter oder Kanaladapterausdruck zugewiesen sein.

Verwenden einer ausgehenden Nachricht als Antwortnachricht

Sie können eine Definition für ausgehende Nachrichten erstellen, um eine ausgehende Nachricht als Antwortnachricht zu verwenden. Die Anwendung erstellt keine Antwortnachricht, wenn die MIOCamstarSubmit-Aktivität in einem Workflow verwendet wird. Die Anwendung erstellt eine ausgehende Nachricht anstelle der Antwortnachricht. Die Anwendung legt die Anforderungs-ID dieser ausgehenden Nachricht auf die Anforderungs-ID der Antwortnachricht fest.

Nur eine ausgehende Nachricht kann als Antwortnachricht generiert werden. Die Anwendung ignoriert alle nachfolgenden Definitionen für ausgehende Nachrichten, die als IsResponse gekennzeichnet sind.

Beim Erstellen einer Definition für ausgehende CIO-Nachrichten für synchrone ausgehende Anforderungs- und Antwortnachrichten

Sie müssen im Popup "Hinzufügen", das über die Seite "CIO - Definition der ausgehenden Nachricht" aufgerufen wird, folgende Optionen auswählen:

• Kontrollkästchen "Ist aktiviert"
• Kontrollkästchen "Ist synchron"
• Anforderungsnachricht im Feld "Nachrichtentyp"

Vorgehensweise zum Erstellen einer Definition für ausgehende CIO-Nachrichten

Gehen Sie folgendermaßen vor, um eine Definition für ausgehende CIO-Nachrichten zu erstellen:

1. Öffnen Sie die Seite CIO - Definition der ausgehenden Nachricht. Die Seite "CIO - Definition der ausgehenden Nachricht" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Wählen Sie einen Servicetypnamen in der Liste aus.
4. Klicken Sie auf Neue Zeile hinzufügen in der Tabelle Ausgangsdefinition, um zu der Tabelle eine Eingabezeile hinzuzufügen. Das Popup "Hinzufügen" wird angezeigt.
5. Wählen Sie ein CIO-Sync-Timing in der Liste aus.
6. Geben Sie in das Feld Name des Nachrichtentyps einen Namen für den Nachrichtentyp ein.
7. Wählen Sie eine CIO-Vorlage in der Liste aus.
8. Geben Sie im Feld Revision eine Revisionsbezeichnung ein.
9. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
10. Klicken Sie auf OK. Das Popup "Hinzufügen" wird geschlossen.
11. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
