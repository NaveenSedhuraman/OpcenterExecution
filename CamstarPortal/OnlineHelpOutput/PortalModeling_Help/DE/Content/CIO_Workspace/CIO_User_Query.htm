Definieren von CIO-Benutzerabfragen

Mit dem Objekt "CIO-Benutzerabfrage" können Sie Benutzerabfragen für ausgehende CIO-Nachrichten erstellen.

Beim Definieren von CIO-Benutzerabfragen

Die Tabelle "Benutzerabfrageparameter" für das Objekt "CIO-Benutzerabfrage" enthält eine Spalte für den Parameterwertausdruck. Der Parameterwertausdruck kann zur Laufzeit in einen Parameterwert aufgelöst werden. Beispiel für einen Parameterwertausdruck:

Container.MfgOrder?Container.MfgOrder.Name:Container.Name

Dies wird als ternärer Ausdruck bezeichnet. Der Ausdruck in diesem Beispiel wird zu dem Namen des Produktionsauftrags aufgelöst, sofern vorhanden. Andernfalls wird er zu dem Containernamen aufgelöst.

Sie können im Feld "Parameterwertausdruck" auch jeden Ausdruck verwenden, der für eine Textvariable gültig ist. Weitere Informationen zu Textvariablen finden Sie unter "Definieren von Textvariablen".

Beim Verwenden des Popups "Benutzerabfragetest"

Das Popup "Benutzerabfragetest" wird angezeigt, wenn Sie auf "Abfrage speichern und testen" klicken. Alle erforderlichen Parameterfelder werden in diesem Popup angezeigt. Eine Tabelle mit Ergebnissen wird angezeigt, wenn die Parameterfelder gefüllt werden. Die Anwendung testet die Abfrage und liefert Ergebnisse, wenn keine Parameter erforderlich sind. Eine Fehlermeldung wird angezeigt, wenn die Abfragesyntax unzulässig ist. In diesem Fall werden keine Ergebnisse in der Tabelle angezeigt.

Vorgehensweise zum Definieren einer CIO-Benutzerabfrage

Gehen Sie folgendermaßen vor, um eine CIO-Benutzerabfrage zu definieren:

1. Öffnen Sie die Seite CIO-Benutzerabfrage. Die Seite "CIO-Benutzerabfrage" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie im Feld CIO-Benutzerabfrage einen Namen für die Benutzerabfrage ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Weitere Informationen zum Testen von Abfragen, die einen Parameter erfordern, finden Sie unter "Vorgehensweise zum Testen einer CIO-Benutzerabfrage, die einen Parameter erfordert".

Oder

Weitere Informationen zum Testen von Abfragen, die keinen Parameter erfordern, finden Sie unter "Vorgehensweise zum Testen einer CIO-Benutzerabfrage, die keinen Parameter erfordert".

6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Testen einer CIO-Benutzerabfrage, die einen Parameter erfordert

Gehen Sie folgendermaßen vor, um eine CIO-Benutzerabfrage zu testen, die einen Parameter erfordert:

1. Führen Sie die Schritte in "Vorgehensweise zum Definieren einer CIO-Benutzerabfrage" durch.

Oder

Wählen Sie eine vorhandene Instanz einer CIO-Benutzerabfrage aus.

2. Geben Sie in das Feld Abfragetext eine SQL-Anweisung ein, die einen Parameter erfordert.
3. Geben Sie den Name für Benutzerabfrageparameter und den Datentyp in die Tabelle "Benutzerabfrageparameter" ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Abfrage speichern und testen, um die Abfrage zu testen. Das Popup "Benutzerabfragetest" wird angezeigt. Oben im Popup werden die erforderlichen Parametern angezeigt.
6. Geben Sie alle Parameter ein, die von der Abfrage zurückgegeben werden.
7. Klicken Sie auf Abfrage testen. Alle Ergebnisse, die der Abfrage und den eingegebenen Parametern entsprechen, werden in der Tabelle "Abfrageergebnisse" angezeigt.
8. Klicken Sie auf Schließen. Das Popup "Benutzerabfragetest" wird geschlossen.

Vorgehensweise zum Testen einer CIO-Benutzerabfrage, die keinen Parameter erfordert

Gehen Sie folgendermaßen vor, um eine CIO-Benutzerabfrage zu testen, die keinen Parameter erfordert:

1. Führen Sie die Schritte in "Vorgehensweise zum Definieren einer CIO-Benutzerabfrage" durch.

Oder

Wählen Sie eine vorhandene Instanz einer CIO-Benutzerabfrage aus.

2. Geben Sie in das Feld Abfragetext eine SQL-Anweisung ein, die keinen Parameter erfordert.
3. Klicken Sie auf Abfrage speichern und testen. Das Popup "Text der Benutzerabfrage" wird mit den Ergebnissen in der Tabelle "Abfrageergebnisse" angezeigt.
4. Klicken Sie auf Schließen. Das Popup "Benutzerabfragetest" wird geschlossen.

Vorgehensweise zum Hinzufügen von Parametern zu einer CIO-Benutzerabfrage

Gehen Sie folgendermaßen vor, um einer CIO-Benutzerabfrage Parameter hinzuzufügen:

1. Führen Sie die Schritte in "Vorgehensweise zum Definieren einer CIO-Benutzerabfrage" durch.

Oder

Wählen Sie eine vorhandene Instanz einer CIO-Benutzerabfrage aus.

2. Klicken Sie in der Tabelle "Benutzerabfrageparameter" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Geben Sie den Namen des Parameters in das Feld Name für Benutzerabfrageparameter ein. Der Name wird als Überschrift des Dateneingabefelds, in das der Benutzer den erforderlichen Wert eingibt, angezeigt.
4. Wählen Sie einen Datentyp in der Liste aus.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Wiederholen Sie Schritte 2 bis 5, um nach Bedarf zusätzliche Parameter hinzuzufügen.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
