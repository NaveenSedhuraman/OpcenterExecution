Definieren von CIO-Listenbearbeitern

Für Listenbearbeiter in einer Definition ausgehender Nachrichten sind zwei Optionen verfügbar. Diese lauten wie folgt:

• CIO-Listenvorlage/Listenausdruck
• CIO-Listenbearbeiter

"CIO-Listenvorlage/Listenausdruck" stellt die einfachere Option dar, wenn alle folgenden Aussagen wahr sind:

• Von dem Dienst wird eine Liste ausgegeben.
• Die Liste enthält keine andere Liste.
• Die Liste enthält Unterobjekte (keine Objekte).

CIO-Listenbearbeiter werden mit ausgehenden CIO-Nachrichten verwendet. Ein CIO-Listenbearbeiter wertet alle Elemente in einer Liste bedingt aus und erstellt einen XML-Knotensatz basierend auf den Listenelementen. CIO-Listenbearbeiter können andere CIO-Listenbearbeiter enthalten, um den zurückgegebenen XML-Knotensatz weiter zu definieren.

Die CIO-Listenbearbeiteroption bietet einen Mechanismus für die Verarbeitung von N Listen in einem Service. Hierbei kann es sich um Unterobjekt- und Objektlisten handeln. Die Optionen "CIO-Listenausdruck" und "CIO-Listenvorlage" bieten grundlegende Funktionen zur Listenbearbeitung. Die Option "CIO-Listenbearbeiter" bietet erweiterte Funktionen zur Listenbearbeitung.

Vor Verwendung eines CIO-Listenbearbeiters

Sie müssen eine Textvariable definieren und in eine CIO-Vorlage einschließen. Der Textvariablenausdruck muss dabei das folgende Format aufweisen:

 

Findlistitem(CIOListProcessResults,Name,”<ListName>”).CIOListResult

<ListName> ist der Name der Liste, die bearbeitet wird. Weitere Informationen finden Sie unter "Definieren von CIO-Vorlagen".

CIOListProcessResults ist das benannte Unterelement, das dem Service-CDO gehört und das Zeichenfolgefeld "CIOListResult" aufweist. Jeder Knotensatz, der bearbeitet wird, wird an dieses Unterelement angehängt. Der äußerste Elementname dieses Knotensatzes entspricht dem Namen des Listenbearbeiters, solange nicht durch NodesetName überschrieben.

Beim Definieren eines CIO-Listenbearbeiters

Für die Ausdrücke von Listenelementwerten muss "CLF::ListItem.field" verwendet werden, um auf Felder im Listenelement zu verweisen. Sie können mit "CLF::ListItemParent.field" auf das übergeordnete Listenelement verweisen, wenn die Liste verschachtelt ist.

*Sie müssen ein Präfix oder Suffix definieren, wenn ein entsprechendes Suffix oder Präfix definiert ist. Andernfalls gibt die Anwendung beim Schreiben der XML-Nachricht einen Fehler zurück. Die Anwendung führt die XML-Validierung erst nach der Bearbeitung der Nachricht durch. Sie müssen gewährleisten, dass das Präfix und Suffix gültige, passende XML-Elementkonstrukte sind. Die separaten Präfix- und Suffixfelder unterstützen Attribute im Präfix. Sie können einen bzw. mehrere Attributwerte und einen Elementwert in demselben Listenbearbeiterdetail definieren.

Beachten Sie das Leerzeichen vor "Type" und "Qty" in der Spalte "Präfix".

Verwenden der Tabelle "CIO-Listenbearbeiter"

Ein CIO-Listenbearbeiter hängt bei der Auswertung der einzelnen Listenelemente eine Zeichenfolgenausgabe an das CIOListResults-Zeichenfolgenfeld des CIOListProcessResults-Unterelements des Service-CDO an. Ein CIO-Listenbearbeiter kann optional einen oder mehrere CIO-Listenbearbeiter in der Tabelle "CIO-Listenbearbeiter" referenzieren. Diese Listenbearbeiter werden in der Reihenfolge ausgewertet, in der sie in der Tabelle aufgeführt sind. Deren Ausgabe ist an die übergeordneten Ergebnisse der CIO-Listenbearbeiter angehängt.

Vorgehensweise zum Definieren eines CIO-Listenbearbeiters

Gehen Sie folgendermaßen vor, um einen CIO-Listenbearbeiter zu definieren:

1. Öffnen Sie die Seite CIO-Listenbearbeiter. Die Seite "CIO-Listenbearbeiter" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie in das Feld CIO-Listenbearbeiter einen Namen für den Listenbearbeiter ein.
4. Geben Sie in das Feld Revision die Revisionsbezeichnung ein.
5. Geben Sie in das Feld Listenausdruck den Listenausdruck ein.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
