Sperrung von Instanzen eines Änderungspaket-Modellierungsobjekts

Siemens bietet zwei Möglichkeiten an, Modellierungsobjekt-Instanzen zu sperren:

• Sperrung von Revisionen
• Sperrung von Änderungspaket-Modellierungsobjekten

Sie können eine gesperrte Modellierungsobjektinstanz eines Änderungspakets weder ändern noch löschen. Die Anwendung zeigt eine Fehlermeldung an, wenn Sie versuchen, eine gesperrte Instanz zu ändern oder zu löschen.

Sperrung von Revisionen

Die Sperrung von Revisionen ist standardmäßig verfügbar. Die Sperrung von Revisionen ermöglicht die Sperrung oder Entsperrung von Instanzen eines revisionierten Objekts (Revisioned Object, RO). Wenn Sie auf das Symbol "Sperren" auf der Seite "Modellierung" der Instanz klicken, wird diese gesperrt bzw. entsperrt. Weitere Informationen finden Sie unter "Sperrung von Revisionen".

Sperrung von Änderungspaket-Modellierungsobjekten

Die Sperrung von Änderungspaket-Modellierungsobjekten ist über die Änderungsverwaltung verfügbar. Die Sperrung von Änderungspaket-Modellierungsobjekten ermöglicht die Sperrung von Modellierungsobjekt-Instanzen, die in einem Änderungsverwaltungspaket enthalten sind, abhängig von der aktuellen Spezifikation (Schritt) des Pakets. Sowohl RO-Instanzen als auch NDO-Instanzen können gesperrt werden. Eine Einstellung in der Änderungsverwaltungsspezifikation bestimmt, ob die Instanzen des Modellierungsobjekts "Änderungspaket" gesperrt oder entsperrt werden, wenn das Änderungsverwaltungspaket diese Spezifikation (diesen Schritt) erreicht.

Sie können Modellierungsobjekt-Instanzen gleichzeitig mehreren Änderungsverwaltungspaketen zuweisen, unabhängig davon, ob diese in anderen Paketen gesperrt oder nicht gesperrt sind. Wenn die Modellierungsobjekt-Instanzen einem Änderungsverwaltungspaket zugewiesen sind, bleiben sie gesperrt, bis alle Pakete, denen sie zugewiesen sind, geschlossen, ungültig gemacht oder an den Entwurfsschritt zurückgegeben wurden.

Hinweis: Die standardmäßigen, von Siemens zur Verfügung gestellten Entwurfsschritte entsperren die verknüpften Modellierungsobjekt-Instanzen. Sie können diesen Schritt (Spezifikation) und andere Spezifikationen entsprechend Ihren geschäftlichen Anforderungen konfigurieren.

Weitere Informationen zur Einstellung der Sperrung von Modellierungsobjekt-Instanzen, wenn das Änderungsverwaltungspaket diese Spezifikation erreicht, finden Sie unter "Definieren von Änderungsverwaltungsspezifikationen". Informationen zu Änderungsverwaltungspaketen finden Sie im entsprechenden produktspezifischen Handbuch zur Änderungsverwaltung:

• Opcenter Execution Core Change Management-Benutzerhandbuch
• Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch

Sperrung von Revisionen und Änderungspaket-Modellierungsobjekten

RO-Instanzen können über die Revisions- oder die Modellierungsobjektsperrung gesperrt werden. NDO-Instanzen können über die Sperrung von Änderungspaket-Modellierungsobjekten gesperrt werden.

Sie können ein gesperrtes RO in ein Änderungsverwaltungspaket aufnehmen. Wenn ein RO auf der Instanzseite gesperrt ist, bleibt es gesperrt, und zwar unabhängig davon, ob es in einem Änderungsverwaltungspaket gesperrt ist oder nicht. Die Sperrung auf der Seite "Modellierung" ist unabhängig von der Änderungsverwaltung.

Wenn Sie versuchen, ein gesperrtes RO zu ändern oder zu löschen, wird eine Fehlermeldung angezeigt, aus der hervorgeht, wo die Instanz gesperrt ist. Die Fehlermeldung gibt an, ob die Objektinstanz aufgrund der Zuweisung zu einem Änderungsverwaltungspaket gesperrt ist oder ob sie auf der Seite "Modellierung" gesperrt ist.
