Sperrung von Revisionen

Die Funktion zur Sperrung von Revisionen bietet Ihnen die Möglichkeit, eine Instanz eines revisionierten Objekts dynamisch zu sperren oder zu entsperren. Sie wird über das Sperrsymbol gesteuert.

Sicherheit und Zugriffssteuerung auf Grundlage von Rollen

Die Sicherheitsverwaltung für Opcenter EX MDD und Opcenter EX CR basiert auf einem rollenbasierten Zugriffssteuerungsmodell. Mit Hilfe der Sicherheitsverwaltung können die Benutzerberechtigungen vergeben werden, die festlegen, wer die Funktion "Sperren/Entsperren" verwenden darf.

Weitere Informationen zu den Objekten, die für die rollenbasierte Zugriffssteuerung verwendet werden, finden Sie in den folgenden Themen:

• Definieren einer Organisation
• Definieren von Rollen
• Definieren von Mitarbeitern

Beim Sperren und Entsperren von Instanzen

Beim Sperren und Entsperren einer Instanz eines revisionierten Objekts ist Folgendes zu beachten:

• Sie müssen eine Instanz nicht speichern, um sie zu sperren oder zu entsperren.
• Auch auf die Felder einer gesperrten Instanz können Sie nach wie vor zugreifen. Allerdings kann eine gesperrte Instanz nicht gespeichert werden.
• Eine gesperrte Instanz kann nicht gelöscht werden.

Die Anwendung zeigt eine Fehlermeldung an, wenn Sie versuchen, eine gesperrte Instanz zu speichern.

Beim Kopieren gesperrter Instanzen

Beim Kopieren einer gesperrten Instanz eines revisionierten Objekts verwenden Sie diese beiden Schaltflächen.

In dieser Tabelle sind die Schaltflächen zum Kopieren einer gesperrten Instanz eines revisionierten Objekts gespeichert.

Schaltfläche

Funktion der Schaltfläche . . .

Kopieren

Erstellen einer Kopie der gesperrten Instanz mit allen Informationen des Originals außer dem Namen. Die neue Instanz ist nicht gesperrt.

Die erste Instanz eines revisionierten Objekts ist standardmäßig die Standardrevision. Jede Revision, die anschließend erstellt wird, wird automatisch zur neuen Standardrevision.

Revision kopieren

Erstellen einer Kopie der gesperrten Instanz mit allen Informationen des Originals außer der Revision. Die neue Instanz ist nicht gesperrt.

Die erste Instanz eines revisionierten Objekts ist standardmäßig die Standardrevision. Jede Revision, die anschließend erstellt wird, wird automatisch zur neuen Standardrevision.

Vorgehensweise zum Sperren einer Instanz

Gehen Sie folgendermaßen vor, um eine Instanz zu sperren:

1. Klicken Sie auf die Seite, die das revisionierte Objekt mit der Instanz, die Sie sperren möchten, enthält. Die Liste "Instanzauswahl" wird angezeigt.
2. Klicken Sie auf einen Instanznamen. Eine Seite mit der ausgewählte Instanz und einem Symbol mit offenem Schloss wird angezeigt.
3. Klicken Sie auf das Symbol Sperren . Die Instanz ist gesperrt und ein Symbol mit geschlossenem Schloss wird angezeigt.
Hinweis: Eine Fehlermeldung wird angezeigt, falls Sie nicht über die Berechtigung verfügen, Instanzen zu sperren.

Vorgehensweise zum Entsperren einer Instanz

Gehen Sie folgendermaßen vor, um eine Instanz zu entsperren:

1. Klicken Sie auf die Seite, die das revisionierte Objekt mit der Instanz enthält, die Sie entsperren möchten. Die Liste "Instanzauswahl" wird angezeigt.
2. Klicken Sie auf einen Instanznamen. Eine Seite mit der ausgewählte Instanz und einem Symbol mit geschlossenem Schloss wird angezeigt.
3. Klicken Sie auf das Symbol Sperren . Die Instanz wird entsperrt und ein Symbol mit offenem Schloss wird angezeigt.
Hinweis: Eine Fehlermeldung wird angezeigt, falls Sie nicht über die Berechtigung verfügen, Instanzen zu entsperren.

Vorgehensweise zum Kopieren einer gesperrten Instanz

Gehen Sie folgendermaßen vor, um eine gesperrte Instanz zu kopieren:

1. Klicken Sie auf die Seite, die das revisionierte Objekt mit der Instanz, die Sie kopieren möchten, enthält. Die Liste "Instanzauswahl" wird angezeigt.
2. Klicken Sie auf einen Instanznamen. Eine Seite mit der ausgewählte Instanz und einem Symbol mit geschlossenem Schloss wird angezeigt.
3. Klicken Sie auf Kopieren. Das Popup "Kopieren" wird angezeigt.
4. Geben Sie einen Zielinstanznamen sowie den Neuen Revisionsnamen ein und klicken Sie anschließend auf Kopieren. Die kopierte Instanz wird mit dem neuen Namen, der Revision und einem Symbol mit offenem Schloss angezeigt.
Hinweis: Die kopierte Instanz ist nicht gesperrt, aber die Originalinstanz bleibt gesperrt.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So wird eine Revision einer gesperrten Instanz kopiert

Gehen Sie folgendermaßen vor, um eine Revision einer gesperrten Instanz zu kopieren:

1. Klicken Sie auf die Seite, die das revisionierte Objekt mit der Instanz/Revision, die Sie kopieren möchten, enthält. Die Liste "Instanzauswahl" wird angezeigt.
2. Klicken Sie auf einen Instanznamen. Eine Seite mit der ausgewählte Instanz und einem Symbol mit geschlossenem Schloss wird angezeigt.
3. Klicken Sie auf Revision kopieren. Das Popup "Kopieren" wird mit einem Feld, in das Sie einen neuen Revisionsnamen eingeben können, angezeigt.
4. Geben Sie einen Namen in das Feld Neuer Revisionsname ein und klicken Sie auf Kopieren. Die kopierte Instanz der Revision wird mit dem Originalnamen, der neuen Revision und einem Symbol mit offenem Schloss angezeigt.
Hinweis: Die kopierte Instanz der Revision ist nicht gesperrt, aber die Originalinstanz bleibt gesperrt.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
