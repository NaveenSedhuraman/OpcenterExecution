Named Data Objects und revisionierte Objekte

Die Opcenter EX MDD- und Opcenter EX CR-Anwendungen verfügen über zwei Objekttypen: benannte Datenobjekte und revisionierte Objekte. Beide Typen sind im Menü "Modellierung" verfügbar.

Beim Definieren von Named Data Objects und revisionierten Objekten

Der Objektname darf keine ungültigen Zeichen enthalten. Z. B. darf kein Apostroph verwendet werden. Weitere Informationen finden Sie unten im entsprechenden produktspezifischen Systemdesigner-Benutzerhandbuch:

• Opcenter Execution Core Designer-Benutzerhandbuch
• Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch
Hinweis: Legen Sie Ihre bevorzugte Sprache fest, bevor Sie Modellierungsobjektinstanzen definieren, wenn die Anwendung alle Aktualisierungen im Modellierungsprotokoll in der bevorzugten Sprache anzeigen soll.

Named Data Objects

Ein benanntes Datenobjekt (Named Data Object, NDO) ist eine Klasse von Objekten in der Anwendung, die durch einen eindeutigen Namen identifiziert wird, z. B. "Unternehmen" oder "Operation". NDO-Instanzen sind persistent, d. h., dass die zugehörigen Informationen in die Datenbank geschrieben werden.

Revisionierte Objekte

Ein revisioniertes Objekt (RO) ist ein Objekt, das einen Namen und eine Revision hat; z. B. "Spez." oder "Produkt". Revisionierte Objekte verfügen über eine Standardrevision, die es Ihnen ermöglicht, eine Version als Standardversion festzulegen. ROs sind ebenso wie NDOs persistent.

Ein weiterer Unterschied zwischen der Wartungsseite für Named Data Objects und der für revisionierte Objekte sind die Schaltflächen auf der Seite. Neben den Schaltflächen "Schließen", "Speichern", "Kopieren" und "Löschen" haben alle revisionierten Objekte zwei zusätzliche Schaltflächen: "Neue Revision" und "Revision kopieren". Weitere Informationen zur Verwendung dieser Schaltflächen finden Sie unter "Revisionsverwaltung".
