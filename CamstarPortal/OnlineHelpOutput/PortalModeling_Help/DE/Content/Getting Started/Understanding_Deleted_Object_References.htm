Grundlegendes zu gelöschten Objektreferenzen

Beim Löschen eines Modellierungsobjekts werden die Instanzinformationen aus der Datenbank entfernt. Davon ausgenommen ist die Historie des Objekts, die im Modellierungsprotokoll eingesehen werden kann.

Wenn ein gelöschtes Objekt von einem anderen Modellierungsobjekt referenziert wird, ist diese Referenz jetzt ungültig, weil sie auf eine Instanz verweist, die nicht mehr vorhanden ist. Modellierungsobjekte mit Feldern, die gelöschte Referenzen enthalten, können nicht gespeichert werden. Sie müssen entweder ein anderes Modellierungsobjekt als Referenz angeben oder die gelöschte Objektreferenz entfernen, damit Sie das Modellierungsobjekt speichern können.

Beispiel für ein gelöschtes Objekt

Wenn Sie ein Modellierungsobjekt auf einer Wartungsseite auswählen, zeigt die Anwendung die Instanz-ID des gelöschten Objekts anstelle des Namens an.

Wenn Sie z. B. den Rollentyp löschen, der von einem Rollenobjekt referenziert wird, wird auf der Seite "Rollenwartung" die Instanz-ID des gelöschten, referenzierten Rollentyps angezeigt.

Referenzen auf gelöschte Objekte werden in Listen nicht angezeigt.

Vorsicht beim Löschen von Objekten

Wenn Sie die Berechtigung zum Löschung von Objekten erteilen, gewährleisten Sie, dass sich die berechtigten Mitarbeiter über die Auswirkungen einer solchen Löschung im Klaren sind.

• Gelöschte Objekte (Spezifikationen, Operationen usw.), die von Arbeitsplänen referenziert werden, können dazu führen, dass bei den in diesen Arbeitsplänen enthaltenen Umlaufbestands-Containern Probleme bei der Verarbeitung im Arbeitsplan auftreten.
• Gelöschte Objekte (Mitarbeiter, Spezifikationen, Arbeitsplanschritte usw.), die in Historienobjekten referenziert werden, verursachen auf der Seite "Produktionsprotokoll" Inkonsistenzen und Fehler bei der Berichterstellung. Diese Inkonsistenzen können zu erheblichen Problemen bei der Nachverfolgbarkeit regulierter Kunden führen.
