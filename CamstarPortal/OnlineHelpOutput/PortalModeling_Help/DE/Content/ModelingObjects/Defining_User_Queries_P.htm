Definieren von Benutzerabfragen

Eine Benutzerabfrage ist eine vordefinierte Abfrage, die als Instanzdaten gespeichert vorliegt. Beachten Sie, dass die Abfragedefinition als Instanzdaten definiert ist, während die Abfrage selbst verwendet werden kann, um Informationen aus Instanzdatentabellen oder Metadatentabellen abzurufen.

Die Tabelle "Benutzerabfrageparameter" ist als persistente Liste implementiert, sodass die Parameterreihenfolge automatisch mit Informationen aus der entsprechenden persistenten Listentabelle aktualisiert wird.

Anforderungen an Benutzerabfragen

Das Schreiben von Abfragen in Opcenter EX MDD oder Opcenter EX CR erfordert, dass Sie mit dem Objektmodell der Anwendung vertraut sind, die SQL-Syntax verstehen und über Erfahrung beim Schreiben von SQL-Abfragen verfügen. Sie müssen auch die Anforderungen der Endbenutzer kennen, damit mit Ihren Abfragen die richtigen Informationen ermittelt und auf der Benutzeroberfläche angezeigt werden.

Nachdem die Abfragen erstellt und getestet wurden, müssen diese von der benutzerdefinierten Anwendung aufgerufen werden. Dies erreichen Sie mit API-Aufrufen in Camstar Services (WSQuery) oder über eine benutzerdefinierte Ausführung.

Beim Definieren von Benutzerabfragen

Gewährleisten Sie, dass Ihnen genügend Containerinformationen zur Verfügung stehen, damit Sie die Benutzerabfrage effektiv testen können. Dazu muss das Informationsmodell mit Produkten, Nutzercodes, Arbeitsplänen und Containern mit verschiedenen Attributen konfiguriert sein.

Die Suchkriterien der Abfrage müssen eine Container-ID umfassen, wenn Sie eine Abfrage für die Containersuche erstellen. Andernfalls liefert die Abfrage keine Ergebnisse, wenn Sie von der Registerkarte "Benutzerabfrage" aus gestartet wird.

Beim Verwenden des Popups "Benutzerabfragetest"

Das Popup "Benutzerabfragetest" wird angezeigt, wenn Sie auf "Abfrage speichern und testen" klicken. Alle erforderlichen Parameterfelder werden in diesem Popup angezeigt. Eine Tabelle mit Ergebnissen wird angezeigt, wenn die Parameterfelder gefüllt werden. Die Anwendung testet die Abfrage und liefert Ergebnisse, wenn keine Parameter erforderlich sind. Eine Fehlermeldung wird angezeigt, wenn die Abfragesyntax unzulässig ist. In diesem Fall werden keine Ergebnisse in der Tabelle angezeigt.

So werden Benutzerabfragen definiert

Gehen Sie folgendermaßen vor, um Benutzerabfragen zu definieren:

1. Öffnen Sie die Seite Benutzerabfrage. Die Seite "Benutzerabfrage" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Abfrage in das Feld Benutzerabfrage ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Weitere Informationen zum Testen von Abfragen, die einen Parameter erfordern, finden Sie unter "So wird eine Abfrage getestet, die einen Parameter erfordert".

Oder

Weitere Informationen zum Testen von Abfragen, die keinen Parameter erfordern, finden Sie unter "So wird eine Abfrage getestet, die keinen Parameter erfordert".

6. Klicken Sie auf Speichern auf der Seite "Benutzerabfrage".
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So wird eine Abfrage getestet, die einen Parameter erfordert

Gehen Sie folgendermaßen vor, um eine Abfrage zu testen, die einen Parameter erfordert:

1. Führen Sie die Schritte unter "So werden Benutzerabfragen definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Benutzerabfrage aus.

2. Geben Sie in das Feld Abfragetext eine SQL-Anweisung ein, die einen Parameter erfordert.
3. Geben Sie den Name für Benutzerabfrageparameter und den Datentyp in die Tabelle "Benutzerabfrageparameter" ein.
4. Klicken Sie auf Abfrage speichern und testen. Das Popup "Text der Benutzerabfrage" wird angezeigt. Oben im Popup werden die erforderlichen Parametern angezeigt.
5. Geben Sie alle Parameter ein, die die Abfrage erfordert.
6. Klicken Sie auf Abfrage testen. Alle Ergebnisse, die der Abfrage und den eingegebenen Parametern entsprechen, werden in der Tabelle "Abfrageergebnisse" angezeigt.
7. Klicken Sie auf Schließen. Das Popup "Benutzerabfragetest" wird geschlossen.

So wird eine Abfrage getestet, die keinen Parameter erfordert

Gehen Sie folgendermaßen vor, um eine Abfrage zu testen, die keinen Parameter erfordert:

1. Führen Sie die Schritte unter "So werden Benutzerabfragen definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Benutzerabfrage aus.

2. Geben Sie in das Feld Abfragetext eine SQL-Anweisung ein, die keinen Parameter erfordert.
3. Klicken Sie auf Abfrage speichern und testen. Das Popup "Text der Benutzerabfrage" wird mit den Ergebnissen in der Tabelle "Abfrageergebnisse" angezeigt.
4. Klicken Sie auf Schließen. Das Popup "Benutzerabfragetest" wird geschlossen.

So werden einer Benutzerabfrage Parameter hinzugefügt

Gehen Sie folgendermaßen vor, um einer Benutzerabfrage Parameter hinzuzufügen:

1. Führen Sie die Schritte unter "So werden Benutzerabfragen definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Benutzerabfrage aus.

2. Klicken Sie in der Tabelle "Benutzerabfrageparameter" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Geben Sie den Namen des Parameters in das Feld Name für Benutzerabfrageparameter ein. Der Name wird als Überschrift des Dateneingabefelds, in das der Benutzer den erforderlichen Wert eingibt, angezeigt.
4. Wählen Sie den Datentyp des Parameters aus dem Feld Datentyp aus.
5. Wiederholen Sie Schritte 2 bis 4, um nach Bedarf zusätzliche Parameter hinzuzufügen.
6. Klicken Sie auf Speichern auf der Seite "Benutzerabfrage".
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
