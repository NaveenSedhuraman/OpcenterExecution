Definieren einer Organisation

Eine Organisation ist eine Wirtschaftseinheit, die für Verarbeitungs- und Berichtszwecke abgetrennt wurde. Eine Organisation ist weder einem Unternehmen noch einer Fabrik zugeordnet.

Beim Definieren einer Organisation

Eine Organisation enthält die folgenden, optionalen Modellierungsdefinitionen:

• Checklistenvorlage
• Klassifikation
• E-Mail-Verteiler
• E-Mail-Nachricht
• Fehlermodusgruppe
• Nummerierungsregel
• Eigentümer
• Organisation
• Rolle
• SMTP-Transport
• Unterklassifikation
• Einteilungsspez.
• UI-Einstellungen

Wenn Sie diese optionalen Modellierungsdefinitionen angeben, werden bestimmte Felder benötigt, um sie zu definieren.

"Organisation" ist ein optionales Feld, das Sie bei der Definition von "Mitarbeiter" und "Rolle" angeben können.

Die Felder für Nummerierungsregeln sind nicht erforderlich, wenn Sie eine Organisation erstellen oder aktualisieren möchten. Sie müssen jedoch einen Datensatz der entsprechenden Kategorie für die Organisation anlegen, bevor Sie ein Ereignis oder eine Abweichung erstellen können.

Eine "Primäre Organisation" muss dem Mitarbeiter zugewiesen werden, bevor Sie ein Produktionsereignis aufzeichnen können. Darüber hinaus muss die Organisation für den angemeldeten Benutzer eine Rolle und einen Eigentümer haben, die in der Tabelle "Zuordnungen von Ereignisklassifizierungsspez." der entsprechenden Klassifikation und Unterklassifikation zugewiesen sind.

Kriterien zur Sicherheitsverarbeitung

Organisationen sind für die Opcenter EX MDD- und Opcenter EX CR-Sicherheitsverwaltung erforderlich. Mindestens eine Organisation ist erforderlich, um die Sicherheitsadministration einzurichten. Opcenter EX MDD und Opcenter EX CR werden mit der Standardorganisation "Unternehmen" installiert.

Durch die Zuweisung eines Mitarbeiters zu einer Rolle (Arbeitsfunktion), die im Kontext der Organisation gültig ist, werden ihm Rechte zugewiesen. Diese Zuweisungen bestimmen die Systemberechtigungen des Mitarbeiters.

Ein Benutzer muss, um irgendeine Aktion ausführen zu können, eine Rolle in der Organisation haben, entweder direkt oder über Rollenweitergabe indirekt.

Informationen zu Standard-Mitarbeiteranmeldungen, die geladen werden, wenn Opcenter EX MDD oder Opcenter EX CR installiert ist, finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration und in der Opcenter Execution Core-Anleitung zur Systemadministration.

Kriterien zur Verarbeitung von Qualitätsdatensätzen

Indem Sie Organisationen zur Verarbeitung von Qualitätsdatensätzen definieren, können Sie die Verarbeitungskriterien einer Organisation festlegen, unter anderem:

• Zuordnungen von Klassifikationen/Unterklassifikationen zum Erstellen von Qualitätsdatensätzen
• UI-Einstellungen für das Anzeigen von Informationen aus Qualitätsdatensätzen im Produktionsprotokoll
• Kategoriespezifische Kriterien zur Verarbeitung von Qualitätsdatensätzen, z. B. Einteilungsspezifikation, Kategorie, Rolle und Eigentümer
• Anforderungen zur elektronischen Unterschrift für Qualitätstransaktionen
• Nummerierungsregelzuordnungen für Qualitätsdatensätze
• SMTP-Transport-Definitionen
• E-Mail-Benachrichtigungsinformationen
• Verweigerte Containertransaktionen für generische Ereignisse oder Produktionsereignisse
• Transaktionen, bei deren Ausführung betroffene Container zu Qualitätsdatensätzen hinzugefügt werden

Wenn Sie einen Qualitätsdatensatz (Ereignis) erstellen, verwendet Opcenter EX MDD oder Opcenter EX CR die Organisation und die Kategorie (den Typ) des Datensatzes, um die Kriterien für die Verarbeitung des Datensatzes zu bestimmen. Wenn beispielsweise der Qualitätsdatensatz ein Ereignis ist, das zu der Corporate-Organisation gehört, ermittelt die Anwendung aus der Definition der Organisation die verwendete Einteilungsspez., um einen Qualitätsdatensatz zu verarbeiten. Außerdem wird das verwendete Nummerierungsschema ermittelt, um eine eindeutige ID für den Qualitätsdatensatz zu erzeugen.

Mithilfe von Benachrichtigungen werden die E-Mails ermittelt, die zu senden sind, wenn in der Anwendung bestimmte Ereignisse/Aktionen auftreten. Sie können verwendet werden, um einen Empfänger über eine erforderliche Aktion zu unterrichten oder eine wichtige Statusaktualisierung zur Verfügung zu stellen. Z. B. können Sie angeben, dass eine E-Mail an den neuen Eigentümer geschickt wird, wenn sich der Eigentümer eines Ereignisses ändert.

Sie können das Organisation-Objekt auch benutzen, um die Qualitätstransaktionen festzulegen, die eine elektronische Unterschrift erfordern.

Sie können die Transaktionen angeben, die für Container für generische Ereignisse oder Produktionsereignisse verweigert werden sollen. Sie können auch angeben, dass die Anwendung betroffene Container automatisch zu einem Qualitätsereignis hinzufügt, wenn eine Transaktion für einen zugehörigen Container mit betroffenem Material ausgeführt wird.

Die nächsten Themen enthalten detailliertere Informationen dazu, wie die Anwendung das Organisation-Objekt benutzt.

Rollen- und Eigentümerzuweisung

Die Verarbeitung von Qualitätsdatensätzen erfordert, dass einem Ereignis eine Rolle und ein Eigentümer zugeordnet werden, bevor der Benutzer das Ereignis abschicken kann. Falls dem Ereignis zum Zeitpunkt des Abschickens keine Rolle und kein Eigentümer zugeordnet sind, wird eine Fehlermeldung angezeigt, die den Benutzer am Abschicken hindert und angibt, dass dem Ereignis kein Eigentümer und keine Rolle zugewiesen sind.

Die Anwendung bietet mehrere Möglichkeiten zur Zuweisung von Rolle und Eigentümer:

• Automatisierte Geschäftsregeln (die während der Auswahl automatisch angewandt werden, wenn sie der Einteilungsspezifikation zugewiesen sind, die verwendet wird, um das Ereignis zu verarbeiten)
• Zuordnung von Ereignisklassifizierungsspez. (definiert als Teil des Modellierungsobjekts "Organisation")
• Kategorieverarbeitungszuordnung (definiert als Teil des Modellierungsobjekts "Organisation")

Die erste Möglichkeit für die Zuweisung von Rolle und Eigentümer ergibt sich während der automatischen Ausführung von Geschäftsregeln durch Einteilung. Falls ein Wert oder beide Werte fehlen, prüft die Anwendung zuerst die "Zuordnung von Ereignisklassifizierungsspez.". (Sie hat eine höhere Priorität als die Kategorieverarbeitungszuordnung.) Wenn Rolle und Eigentümer hier angegeben werden, verwendet die Anwendung diese Werte. Wenn nicht, verwendet die Anwendung die Werte für Rolle und Eigentümer, die für die "Kategorieverarbeitungszuordnung" angegeben sind.

Hinweis: Wenn bei der automatischen Ausführung von Geschäftsregeln nur einer der beiden Werte festgelegt ist, überschreibt die Anwendung diesen Wert mit dem zutreffenden Wert, der in der "Zuordnung von Ereignisklassifizierungsspez." bzw. in der "Kategorieverarbeitungszuordnung" angegeben ist. Wenn der Rollenwert z. B. bei der Einteilung festgelegt wurde und der Eigentümerwert leer ist, prüft die Anwendung den Rollen- und den Eigentümerwert, die in der "Zuordnung von Ereignisklassifizierungsspez." angegeben sind. Wenn die "Zuordnung von Ereignisklassifizierungsspez." einen Rollen- und einen Eigentümerwert enthält, überschreibt die Anwendung den Rollenwert, der bei der Einteilung festgelegt wurde, mit demjenigen, der in der "Zuordnung von Ereignisklassifizierungsspez." angegeben ist.

Die "Zuordnung von Ereignisklassifizierungsspez." ist für Ereignisse und Abweichungen erforderlich.

Portalseiten und Seitenfolgen werden in Portal Studio definiert.

Mithilfe der Modellierungsobjekte "Klassifikation" und "Unterklassifikation" können Sie Klassifikationen und Unterklassifikationen für Ihre Organisation erstellen.

Beim Definieren von Kategorieverarbeitungszuordnungen

Bei der Verarbeitung von Qualitätsdatensätzen werden Kategorieverarbeitungszuordnungen verwendet, um Folgendes zu Qualitätsdatensätzen in einer bestimmten Kategorie zuzuweisen:

• Einteilungsspezifikationen
• Rolle und Eigentümer (falls nicht bereits zugewiesen)

Beim Definieren einer Kategorieverarbeitungszuordnung ist die Kategorie selbst der primäre Schlüssel und muss eindeutig sein. Z. B. können Sie nur eine Kategorieverarbeitungszuordnung mit der Kategorie Ereignis erstellen.

Beim Konfigurieren von Anforderungen zur elektronischen Unterschrift

Wenn Sie die Qualitätstransaktionen festlegen, für die Sie elektronische Unterschriften erfassen möchten, müssen Sie nur diesen Qualitätsdienst zur "Elektr. Unterschrift: Transaktionszuordnung" des Modellierungsobjekts der Organisation hinzufügen. Wenn Sie z. B. eine elektronische Unterschrift erfassen möchten, wenn ein Benutzer die Genehmigung für eine Qualitätsdatensatzlösung abbricht, müssen Sie den Qualitätsservice "Genehmigungsabbruchformular" zur "Elektr. Unterschrift: Transaktionszuordnung" hinzufügen.

Ereigniserstellung ist die Ausnahme. Wenn Sie möchten, dass für Seitenfolgen einer Ereignisaufzeichnung elektronische Unterschriften erforderlich sind, müssen Sie nicht nur den Service für Ereigniserstellung (CreateEvent-Service) in der "Elektr. Unterschrift: Transaktionszuordnung" angeben, sondern auch den Service für Ereignisaktualisierung (UpdateEvent-Service) hinzufügen.

Wenn Sie in "Elektr. Unterschrift: Transaktionszuordnung" keine Services für Aktualisierung hinzufügen, kann der Benutzer die Anforderung zur elektronischen Unterschrift umgehen, wenn er die Seitenfolge einer Ereignisaufzeichnung ausführt.

Anforderung zur elektronischen Unterschrift für "Genehmigungsblatt zuweisen"

Wenn für "Genehmigungsblatt zuweisen" eine elektronische Unterschrift konfiguriert ist, wird diese nur auf die Zuweisung/Weiterleitung von Lösungsgenehmigungen für Qualitätsdatensätze angewandt.

Beim Definieren von Transaktionsstandardwerten

Sie können die Transaktionen angeben, die für Container für generische Ereignisse oder Produktionsereignisse verweigert werden sollen. Wenn Sie in der Tabelle "Produktionsereignis - Verweigerte Transaktionen" oder "Generisches Ereignis - Verweigerte Transaktionen" auf "Neue Zeile hinzufügen" klicken, wird eine Liste angezeigt, in der Sie eine Containertransaktion auswählen können. Die Liste umfasst alle Containertransaktionen, z. B. "Mengenänderung", "Schließen", "Anhalten" und "Abmelden".

Die Anwendung kann betroffene Container automatisch zu einem Qualitätsereignis hinzufügen.

Sie können Containertransaktionen angeben, für die die Anwendung automatisch Container zu einem Qualitätsereignis hinzufügen soll. Die Anwendung fügt betroffene Container automatisch zu einem Qualitätsereignis hinzu, wenn eine der in der Tabelle angegebenen Transaktionen für einen zugehörigen Container mit betroffenem Material ausgeführt wird.

Diese Standardtransaktionen sind in der Tabelle "Betroffene Container zu Ereignissen hinzufügen" verfügbar:

• Verbinden (Associate)
• Vereinigen
• Komponentenverbrauch
• Komponente entfernen
• Aufteilen

Die Anwendung fügt betroffene Container automatisch zu einem Qualitätsereignis hinzu, wenn eine der in der Tabelle ausgewählten Transaktionen für einen zugehörigen Container mit betroffenem Material ausgeführt wird.

Hinweis: Die Anwendung fügt die betroffenen Container zu einem Qualitätsereignis hinzu, wenn Sie bei Auswahl von "Vereinigen" die Transaktionen "Container vereinigen" oder "Mengen vereinigen" ausführen. Gleiches gilt für "Aufteilen" und die Transaktionen "Container aufteilen" und "Menge aufteilen".

Informationen zum automatischen Hinzufügen betroffener Container zu Qualitätsereignissen finden Sie im Opcenter Execution Medical Device and Diagnostics Event Management-Benutzerhandbuch und im Opcenter Execution Core Event Management-Benutzerhandbuch sowie im Opcenter Execution Medical Device and Diagnostics Nonconformance Management-Benutzerhandbuch und im Opcenter Execution Core Nonconformance Management-Benutzerhandbuch.

Liste der Einteilungsspezifikationen, die Sie dem Qualitätsdatensatz zuweisen können. Die Anwendung verwendet diese Einteilungsspezifikation, wenn Qualitätsdatensätze des im Feld "Kategorie" angegebenen Typs verarbeitet werden.

Optional

Tabelle "Lösungsgenehmigung"

In dieser Tabelle können Sie Lösungsaktionen Genehmigungsvorlagen zuordnen und angeben, ob eine Genehmigung erforderlich ist, um diese Lösung auszuführen.

Optional

Lösungsaktion

Liste der Lösungen für den Qualitätsdatensatz. Die Lösungen, die hier zur Auswahl stehen, hängen vom Typ des Qualitätsdatensatzes ab. Wenn Sie im Feld "Kategorie" z. B. "Ereignis" ausgewählt haben, ist "Schließen - keine Aktionen erforderlich" die einzige Lösungsaktion, die zur Verfügung steht.

Erforderlich, wenn eine Lösungsgenehmigung definiert wird

Genehmigungsvorlage

Liste der Genehmigungsvorlagen, die einer Lösungsaktion zugeordnet werden können. Die Anwendung verwendet die Genehmigungsvorlage, um die Benutzer zu bestimmen, die die Lösung genehmigen müssen, und diesen Benutzern eine Liste der gültigen Entscheidungen anzuzeigen.

Genehmigungsvorlagen werden in der Modellierung definiert.

Genehmigung erforderlich

Kontrollkästchen, das angibt, ob die Anwendung den Qualitätsdatensatz nur dann auflöst, wenn alle in der Genehmigungsvorlage angegebenen Benutzer die Auflösung genehmigen.

Vorgehensweise zum Hinzufügen einer Ereignisklassifizierungsspezifikation zu einer Organisation

Gehen Sie folgendermaßen vor, um eine Ereignisklassifizierungsspezifikation zu einer Organisation hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Organisation" aus.

Oder

Wählen Sie eine vorhandene Organisationsinstanz aus.

2. Navigieren Sie zur Tabelle "Zuordnung von Ereignisklassifizierungsspez.".
