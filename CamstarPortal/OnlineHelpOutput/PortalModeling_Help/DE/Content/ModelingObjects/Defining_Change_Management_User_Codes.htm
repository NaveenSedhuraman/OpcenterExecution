Definieren von Nutzercodes für die Änderungsverwaltung

Nutzercodes für die Änderungsverwaltung sind Listen mit Codes für die jeweiligen Anforderungen der Anwendung für die Änderungsverwaltung. Nutzercodes können wertvolle Verfolgungs- und Berichtsinformationen enthalten. Benutzerdefinierte Codes ermöglichen es Ihnen, eine Liste von zulässigen Werten für ein bestimmtes Feld anzugeben. Mithilfe der Nutzercodes für die Änderungsverwaltung können Sie Suchvorgänge für Pakete auf der Seite "Paketsuche" filtern. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch oder im Opcenter Execution Core Change Management-Benutzerhandbuch.

Beim Definieren von Nutzercodes für die Änderungsverwaltung

Die meisten Nutzercodes bestehen aus drei Attributen: Bezeichnung, Beschreibung und Anmerkungen. Einige Nutzercodes enthalten weitere Attribute für zusätzliche Funktionen oder zur Überprüfung.

Die Bezeichnung für jeden Nutzercode muss innerhalb des Typs eindeutig sein. Der Wert wird als alternativer Schlüssel für Suche, Dateneingabe und Überprüfung verwendet.

Sie können auch nach Belieben neue Nutzercodetypen erstellen.

Beim Definieren von Nutzercodes für die Änderungsverwaltung dürfen Sie im Feld "Name" kein Leerzeichen als erstes oder letztes Zeichen eingeben.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Nutzercodes für die Änderungsverwaltung

Diese Tabelle beschreibt die Nutzercodes für die Änderungsverwaltung, die standardmäßig in Change Management verfügbar sind.

Nutzercode

Beschreibung

Paketerstellungsgrund

Der Grund für die Erstellung eines Änderungsverwaltungspakets.

"Paketerstellungsgrund" ist ein erforderliches Feld in den Transaktionen "Paket erstellen" und "Paket aktualisieren".

"Paketerstellungsgrund" ist ein optionales Feld in der Transaktion "Paketsuche".

Paketprioritätscode

Indikator, über den einem Änderungsverwaltungspaket eine Verarbeitungspriorität zugewiesen wird.

Über das Feld "Relative Priorität" können Sie dem Prioritätscode einen numerischen Wert zuordnen, um die Priorität relativ zu anderen Prioritätscodes anzugeben. Die relative Priorität eines "hohen" Prioritätscodes könnte z. B. "1" lauten, während die relative Priorität einer "niedrigen" Priorität entsprechend bei "3" läge.

Pakettyp

Verwendeter Indikator, um den Typ eines Änderungsverwaltungspakets anzugeben.

Beim Definieren von Paketprioritätscodes

Verwechseln Sie das Modellierungsobjekt "Paketprioritätscode" nicht mit dem Modellierungsobjekt "Prioritätsstufe" oder "Prioritätscode". Prioritätsstufen werden Ereignissen und Qualitätsdatensätzen zugewiesen. Prioritätscodes werden Containern, Paketprioritätscodes hingegen Änderungsverwaltungspaketen zugewiesen. Weitere Informationen zum Definieren von Prioritätsindikatoren für Ereignisse und Qualitätsdatensätze finden Sie unter "Definieren von Prioritätsstufen". Weitere Informationen zum Definieren von Prioritätsindikatoren für Container finden Sie unter "Definieren von Prioritätscodes".

Vorgehensweise zum Definieren von Nutzercodes für die Änderungsverwaltung

Gehen Sie folgendermaßen vor, um einen Nutzercode für die Änderungsverwaltung zu definieren:

1. Öffnen Sie die Seite für den Nutzercode, den Sie definieren möchten. Die Seite der Modellierungsobjekt-Instanz wird in der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für den Nutzercode in das Feld Name des Nutzercodes ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Unter "Gemeinsame Felder auf Modellierungsseiten" finden Sie Informationen zu den optionalen Feldern.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
