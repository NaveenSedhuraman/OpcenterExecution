Definieren von Ressourcen

Eine Ressource steht für eine Maschine, ein Ausrüstungsstück oder eine andere, immaterielle Einheit in einer Fabrik. Das häufigste und konkreteste Beispiel für eine Ressource ist eine Maschine. Zur Festlegung ihres physischen Standorts sind Ressourcen einer einzelnen Fabrik zugeordnet.

Sie können für jeden Schritt des Arbeitsplans die verwendete Ressource eingeben. Diese Information kann dann in die Analyse des Durchsatzes, der Zykluszeiten und des Ausbeuteverlustes eines Containers einbezogen werden.

Sie können Ressourcen Ressourcengruppen zuweisen, einem Typ von Objektgruppe, der eine logische Gruppierung ermöglicht. Ressourcengruppen werden verwendet, um Auswahllisten gültiger Ressourcen zur Verfügung zu stellen und Benutzereingaben zu überprüfen.

"Ressource" ist ein statisches Modellierungselement. Jede Ressource kann einen Ressourcenstatuseintrag haben, um den aktuellen Status jeder festgelegten Statuskategorie zu verfolgen. Bei Auslieferung umfasst die Anwendung drei spezielle Statuskategorien:

• "Produktion" wird verwendet, um die Verfügbarkeit für die Produktion festzustellen.
• "Vorbeugende Wartung" wird verwendet, um den aktuellen Status der vorbeugenden Wartung für eine Ressource zu überwachen.
• "Protokoll" wird als Mechanismus für die Protokollierung frei formulierbarer Kommentare und optional für die Zuweisung vordefinierter Status- und Ursachencodes verwendet. Kunden können zusätzliche Statuskategorien definieren, um den Ressourcenstatus gleichzeitig mit mehreren benutzerdefinierten Abläufen zu verfolgen.

Beim Definieren einer Ressource

Wenn Sie Ressourcenstatus implementieren, wird der Ressource in der Transaktion "Ressourceneinrichtung" ein Ressourcentyp und/oder ein Ressourcenstatusmodell zugewiesen.

Beachten Sie Folgendes:

• Eine Ressource enthält die optionalen Modellierungsdefinitionen: "Fabrik", "Standort", "Druckwarteschlange", "Wartungsklasse", "Trainingsanforderungsgruppe", "Ressourcentyp" und "Ressourcenstatusmodell".
• Eine Ressource ist:
• Ein optionales Feld in der Definition der Ressourcengruppe.
• Ein erforderliches Feld in den Transaktionen "Ressourceneinrichtung", "Ressourcendaten sammeln" und "Ressourcendurchsatz".
• Ein optionales Feld in den Transaktionen "Anmelden", "Durchsatz", "Abmelden - standardmäßig", "Abmelden - nicht standardmäßig", "Transfer", "Nacharbeit", "Transfer zur Nacharbeit", "Daten sammeln", "Containerwartung" und "Mengenänderung".
Hinweis: Einer Ressourcendefinition können Umlaufbestandsnachrichten zugewiesen sein, wenn sie mit einem Feld einer Containerdefinition verknüpft ist. Weitere Informationen finden Sie in Abhängigkeit von Ihrem Produkt unter "Definieren von Umlaufbestandsnachrichten" sowie im Opcenter Execution Core Shop Floor-Benutzerhandbuch und im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch.

Zwei Felder der Seite "Ressource" sind spezifisch für das Einrichten der Ressource:

• Ressourcentyp
• Ressourcenstatusmodell

Ein Portal-Benutzer kann eine Ressource angeben, wenn er ein Ereignis erstellt.

Beim Definieren einer Ressource für OEE
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Aktivieren Sie das Kontrollkästchen "In GAE-Daten einschließen", wenn die Anwendung GAE-Übersichtsdaten für die Ressource akkumulieren soll.

Sie können die Tabelle "Ideale Zykluszeit" verwenden, um die ideale Zykluszeit für ein Produkt oder eine Produktfamilie zu definieren. Beim Definieren einer idealen Zykluszeit gelten folgende Regeln:

• Die ideale Zykluszeit, die auf der Seite "Ressource" definiert wurde, hat Vorrang vor der idealen Zykluszeit, die auf der Seite "Ressourcenfamilie" definiert wurde.
• Sie müssen für jede ideale Zykluszeit, die in der Tabelle "Ideale Zykluszeiten" definiert wird, ein Produkt oder eine Produktfamilie angeben.
• Sie können ein Produkt oder eine Produktfamilie nur einmal in der Tabelle "Ideale Zykluszeiten" angeben. Das Produkt oder die Produktfamilie muss für jede ideale Zykluszeit eindeutig sein.
• Die ideale Zykluszeit, die für ein Produkt definiert wird, hat Vorrang vor der idealen Zykluszeit, die für eine Produktfamilie definiert ist.

Beim Zuweisen von OEE-Einstellungen für die Ressource
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Die folgenden Regeln gelten beim Zuweisen von OEE-Einstellungen:

• Die OEE-Einstellungen, die dem Modellierungsobjekt "Ressource" zugewiesen wurden, haben Vorrang vor den OEE-Einstellungen, die dem Modellierungsobjekt "Ressourcenfamilie" zugewiesen wurden.
• Die OEE-Einstellungen, die dem Modellierungsobjekt "Ressourcenfamilie" zugewiesen wurden, haben Vorrang vor den OEE-Einstellungen, die dem Modellierungsobjekt "Fabrik" zugewiesen wurden.

Weitere Informationen zum Zuweisen von OEE-Einstellungen zu einer Ressourcenfamilie finden Sie unter "Beim Zuweisen von OEE-Einstellungen für die Ressourcenfamilie". Weitere Informationen zum Zuweisen von OEE-Einstellungen zu einer Fabrik finden Sie unter "Zuweisen von OEE-Einstellungen für die Fabrik".

Beim Definieren von Zeitplänen für Ausfallzeit für eine Ressource
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

In der Tabelle "Zeitplan für Ausfallzeit" (Downtime Schedule) können Sie Zeitpläne für Ausfallzeit für die ausgewählte Ressource definieren. Die Anwendung füllt die Tabelle "Beschlossener Downtime-Zeitplan" auf der Seite "Ressource" mit Informationen aus der Tabelle "Zeitplan für Ausfallzeit" (Downtime Schedule).

Diese Regeln gelten beim Definieren von Zeitplänen für Ausfallzeit für eine Ressource:

• Der auf der Seite "Ressource" definierte Zeitplan für Ausfallzeit hat Vorrang vor der auf der Seite "Ressourcenfamilie" definierte Zeitplan für Ausfallzeit. Informationen zum Definieren von Zeitplänen für Ausfallzeit für eine Ressourcenfamilie finden Sie unter "Beim Definieren von Zeitplänen für Ausfallzeit für eine Ressourcenfamilie".
• Sie können geplante oder nicht geplante Ausfallzeit für eine Ressource definieren.

Beim Definieren einer Ressource für die Verwendung mit Scheduling
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter EX CR Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Beim Erstellen einer neuen Ressource oder beim Aktualisieren einer vorhandenen Ressource in der Anwendung sendet die Anwendung Ressourceninformationen an Preactor. Preactor plant Produktionsaufträge für eine bestimmte Ressource.

Prüfen von Materialwarteschlangen für eine Ressource
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Sie können das Kontrollkästchen "Warteschlange prüfen" im Abschnitt "Materialwarteschlangen" aktivieren, damit die Anwendung das Material, das in die Materialwarteschlange für eine Ressource geladen wird, anhand der Stückliste für das hergestellte Produkt überprüft.

Vorgehensweise zum Definieren einer Ressource

Gehen Sie folgendermaßen vor, um eine Ressource zu definieren:

1. Öffnen Sie die Seite Ressource. Die Seite "Ressource" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Ressource in das Feld Ressource ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
