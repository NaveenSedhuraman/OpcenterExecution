Trainingsanforderungen und -einträge

Trainingsanforderungen und Trainingseinträge sind die Schlüsselkomponenten des Trainings- und Zertifizierungsprozesses für Bediener.

Trainingsanforderungen

Wenn Sie eine Trainingsanforderung erstellen, definieren Sie die folgenden Informationen:

• Das Gültigkeitsdatum, um anzugeben, ob die Trainingsanforderung aktiv ist oder zu einem späteren Zeitpunkt aktiv wird
• Eine Liste der zugewiesenen Trainer, die berechtigt sind, die Trainingseinträge der Benutzer zu verwalten
• Verweis auf eine externe Datei, die Standardarbeitsvorgänge und sonstige Details enthalten kann, die von Ihrem Unternehmen benötigt werden

Sie organisieren die Trainingsanforderungen in Gruppen, die von unterschiedlichen Modellierungsobjekten referenziert werden, z. B. Unternehmen, Fabrik, Produkte, Produktfamilien, Arbeitsbereiche, Operationen, Spezifikationen und Ressourcen.

Entsprechend Ihren Geschäftsregeln können Sie mehrere Revisionen einer Trainingsanforderung definieren.

Trainingseinträge

Anhand von Trainingseinträgen lässt sich der Status eines bestimmten Benutzers in Bezug auf eine Trainingsanforderung verfolgen. Siemens empfiehlt, dass Sie Benutzern einen Trainingseintrag für jede Trainingsanforderung zuweisen. Trainingseinträge werden von den Trainern erstellt und aktualisiert, die in der Trainingsanforderung festgelegt sind.

Trainingseinträge enthalten diese Informationen:

• Trainingseintragsstatus, um zu kennzeichnen, ob der Besitzer des Eintrags berechtigt ist, eine Transaktion auszuführen, und ob eine elektronische Unterschrift erforderlich ist
• Verweis auf eine bestimmte Revision einer Trainingsanforderung (nicht die Standardrevision)
• Ablaufdatum auf Grundlage der Trainingsanforderung oder festgelegt vom Administrator der Trainingseinträge
• Verweise auf mehrere Trainingseinträge der einzelnen Benutzer

Ablaufdaten von Trainingseinträgen

Sie geben in der aktiven Trainingsanforderung Ablaufdaten an, die zunächst von der zugehörigen Trainingsanforderung verwendet werden. Eine Trainingsanforderung kann aber auch ein eigenes Ablaufdatum haben. Eine Trainingsanforderung gilt als "Aktiv", solange ihr Ablaufdatum noch nicht überschritten ist.

Der Trainingseintrag ermittelt das anfängliche Ablaufdatum anhand der folgenden Regeln:

• Wenn in der Trainingsanforderung nur das Ablaufdatum angegeben wurde, wird dieses Datum im Trainingseintrag unverändert verwendet.
• Wenn nur die Ablaufperiode (in Tagen) in der Trainingsanforderung angegeben ist, wird das Ablaufdatum bestimmt, indem die Anzahl der Tage zu dem Datum addiert wird, an dem der Eintrag erstellt wurde (das Transaktionsdatum).
• Wenn Ablaufdatum und -periode in der Trainingsanforderung angegeben sind, verwendet die Anwendungslogik das frühere Datum.
• Der Administrator der Trainingseinträge kann das Ablaufdatum jederzeit überschreiben.
Aktivieren von elektronischen Unterschriften für die Trainingseintragsverwaltung

Auf der Seite "Modellierung elektr. Unterschrift" können Sie eine Anforderung zur elektronischen Unterschrift für drei verschiedene Aktionen eines Trainingseintragsverwaltungsservices zuweisen:

• Erstellen (Create)
• Löschen (Delete)
• Aktualisieren (Update)

Services für die Trainingseintragswartung sind unter dem Service "Unterobjektwartung" auf der Seite "Modellierung elektr. Unterschrift" verfügbar, die im Menü "Modellierung" aufgerufen werden kann. Weitere Informationen zum Zuweisen elektronischer Unterschriften für Trainingseintragsverwaltungsaufgaben auf der Seite "Modellierung elektr. Unterschrift" finden Sie unter "Informationen zu Elektronische Unterschriften – Modellierung".

Erzwingen von Trainingsanforderungen in Fertigungstransaktionen

Sie definieren Trainingsanforderungen für Ihre Informationsmodellierungsobjekte, auf die die Anwendungspools, Services und Container von Opcenter EX MDD oder Opcenter EX CR verweisen. Eine Abmeldetransaktion wird z. B. einer Fabrik und deren Unternehmen, ein Container einer Ressource, einer Operation, einem Produkt und dessen Produktfamilie zugewiesen usw.

Transaktionen mit Trainingsanforderungen

Diese Transaktionen referenzieren die Trainingseinträge des Benutzers, um zu bestimmen, ob die Trainingsanforderungen erfüllt sind:

• ChangeQty
• Komponentenverbrauch
• Komponente entfernen
• Anmelden
• Abmelden - standardmäßig
• Aufgabe ausführen
• Nacharbeit
• Start

Validierungsprozess

Wenn ein Benutzer eine Transaktion an einem Container ausführt, überprüft die Anwendung Folgendes:

1. Alle Trainingsanforderungen, die den Modellierungsobjekten zugeordnet sind, wurden kumuliert.
2. Die Trainingseinträge des Benutzers wurden gelesen und anhand der kumulierten Trainingsanforderungen geprüft, um zu sehen, welche Anforderungen aktiv sind.
3. Die Trainingseinträge werden in dieser Reihenfolge geprüft:
a. Das Trainingsdatum muss innerhalb des Bereichs der Gültigkeitsdaten der Trainingsanforderung liegen.
b. Aus der Berechtigung muss hervorgehen, dass der Benutzer die Transaktion ausführen darf.
c. Das Transaktionsdatum muss vor dem Ablaufdatum des Trainingseintrags liegen.

Wenn die oben genannten Bedingungen erfüllt sind, wird die Transaktion erfolgreich beendet. Andernfalls schlägt sie fehl und eine entsprechende Fehlermeldung wird angezeigt.

Aktive Trainingsanforderungen und Gültigkeitsdaten

Trainingsanforderungen können je nach den Gültigkeitsdaten der Anforderung als aktiv oder inaktiv betrachtet werden. Nur aktive Trainingsanforderungen werden überprüft, wenn Benutzer Transaktionen ausführen. Die folgenden Feldwerte werden verwendet, um den aktiven Status einer Trainingsanforderung zu ermitteln:

• Gültig ab
• Gültig bis

Diese Regeln bestimmen, ob eine Trainingsanforderung noch aktiv ist:

• Wenn Sie sowohl das Feld "Gültig ab" als auch das Feld "Gültig bis" ausfüllen, muss das Datum, an dem der Benutzer die Transaktion ausführt, zwischen diesen beiden Datumsangaben liegen.
• Wenn Sie nur für "Gültig ab" ein Datum eingeben, muss das Transaktionsdatum dasselbe oder ein späteres Datum sein.
• Wenn Sie nur für "Gültig bis" ein Datum eingeben, muss das Transaktionsdatum dasselbe oder ein früheres Datum sein.
• Wenn Sie beide Felder leer lassen, läuft die Trainingsanforderung nie ab.

Weitere Überlegungen

Wenn Sie diese Felder bei der Definition der Trainingsanforderung festlegen, müssen Sie die Modellierungsobjekte berücksichtigen, die diese Anforderungen verwenden. Wenn sich die Anforderung z. B. auf den Betrieb einer bestimmten Ausrüstung (Ressource) bezieht, soll die Trainingsanforderung vermutlich immer aktiv sein, damit gewährleistet ist, dass die Benutzer für Verwendung der Ausrüstung geschult sind. Wenn die Ausrüstung durch ein neues Modell ersetzt wird, erstellen Sie eine neue Revision der Trainingsanforderung, die ein bestimmtes "Gültig ab"-Datum hat und zeitlich auf die Installation und die Bereitstellung der neuen Ausrüstung abgestimmt ist. Trainingseinträge werden dann zusätzlich aktualisiert, damit sie die neue Revision der Trainingsanforderungen referenzieren.
