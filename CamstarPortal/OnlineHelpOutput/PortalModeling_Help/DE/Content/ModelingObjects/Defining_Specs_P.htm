Definieren von Spezifikationen

Eine Spezifikation legt die Tätigkeiten fest, die an einem Arbeitsschritt eines Arbeitsplans oder Stammrezepts durchgeführt werden, und enthält genaue Planungs- und Prozess-Parameterinformationen. Spezifikationen referenzieren zahlreiche andere Modellierungskomponenten einschließlich "Operation", "Einrichtung", "Datensammlungsdefinition" und "Ressource".

Wenn Sie Arbeitspläne und Stammrezepte definieren, erfordert die Anwendung, dass Sie mindestens eine Spezifikation referenzieren. Eine Spezifikation kann mehrfach von verschiedenen Arbeitsplänen und Stammrezepten referenziert werden.

Ein Portal-Benutzer kann eine Spezifikation angeben, wenn er ein Ereignis erstellt.

Beim Definieren einer Spezifikation

Die Spezifikation beinhaltet:

• Die erforderlichen Modellierungsdefinition "Operation".
• Die optionalen Modellierungsdefinitionen "Geschäftsregel", "Datensammlungsdefinition, "Dokumentsatz", "Automatischer Ablauf", "Anforderung zur elektronischen Unterschrift", "Drucketikettendefinition", "Rezept", " Ressourcengruppe", "Waagengruppe", "Einrichtung" und "Trainingsanforderungsgruppe.
Hinweis: Das Feld "Waagengruppe" wird nur angezeigt, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

"Spez." ist ein erforderliches Feld in der Definition von Arbeitsplan und Stammrezept.

Hinweis: Für eine Spezifikationsdefinition können zugehörige Umlaufbestandsnachrichten vorliegen, wenn sie mit einem Feld einer Containerdefinition verknüpft ist.

Beim Zuweisen von elektronischen Unterschriften und automatischen Abläufen

Elektronische Unterschriften, die in der Transaktionszuordnung für die elektr. Unterschrift zugewiesen werden, gelten nur für Containertransaktionen. Wenn Sie dieser Spezifikation einen automatischen Ablauf zuweisen und Sie elektronische Unterschriften für die Aufgaben des elektronischen Ablaufs erfassen möchten, müssen Sie die elektronischen Unterschriften für Anweisungs- und Berechnungsaufgaben separat zuweisen. Die der Spezifikation zugewiesene elektronische Unterschrift ist für Transaktionsaufgaben im Rahmen der automatischen Abläufe gültig. Weitere Informationen zur Definition von Anweisungs- und Berechnungsaufgaben finden Sie unter "Definieren einer Aufgabenliste und zugehöriger Aufgaben".

Hinzufügen von Anforderungen zur elektronischen Unterschrift zum Anhängen oder Abtrennen von Dokumenten an bzw. von Spezifikationen.

Sie können die Einstellungen so ändern, dass Benutzer elektronische Unterschriften leisten müssen, wenn sie Dokumente an eine Spezifikation anhängen oder von ihr abtrennen möchten. Mögliche Dokumente sind besondere Anweisungen, Testergebnisse, Zeichnungen oder Fotos. Informationen zum Anhängen von Dokumenten finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Die Option, Anforderungen zur elektronischen Unterschrift für das Anhängen oder Abtrennen von Dokumenten festzulegen, ist im Abschnitt "Transaktionen" in "Elektr. Unterschrift: Transaktionszuordnung" enthalten. Optional können Sie festlegen, dass für alle Transaktionen elektronische Unterschriften erforderlich sind.

Hinzufügen von Anforderungen zur elektronischen Unterschrift für das Sammeln von Los-Stichprobendaten zu Spezifikationen

Sie können die Anforderung festlegen, dass die Benutzer elektronische Unterschriften für die Sammlung von Los-Stichprobendaten für Spezifikationen leisten müssen. Die Option zum Festlegen dieser Anforderungen befindet sich im Abschnitt "Transaktionen" in "Elektr. Unterschrift: Transaktionszuordnung".

Sie können die Unterschriften für die Sammlung von Los-Stichprobendaten auf der Fertigungsseite "Los-Stichprobendaten sammeln" leisten. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch oder im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Hinzufügen von Prozesstimern

Sie können Prozesstimer hinzufügen, um den Fertigungsprozess zu steuern und zu überwachen. Sie können Aktionen definieren, die von der Anwendung ausgeführt werden, wenn versucht wird, eine Transaktion vor Erreichen einer Mindestzeit oder vor Überschreiten einer maximalen Zeit auszuführen.

Die Option zum Setzen von Prozess-Timern in einer Spec erscheint in den Transaktionszuordnungen "Start-Timer" und "Ende-Timer" im Abschnitt "Transaktionen". Jedem Prozesstimer, den Sie setzen, müssen Sie entweder die Transaktion "Abmelden" oder die Transaktion "Anmelden" zuweisen. Wenn Sie einem Timer die Transaktion "Abmelden" zuweisen, löst die Anwendung den Timer aus, sobald ein Container die Spezifikation erreicht, für die der Timer gesetzt wurde. Wenn Sie einem Timer die Transaktion "Anmelden" zuweisen, löst die Anwendung den Timer aus, sobald ein Container die Spezifikation erreicht, für die der Timer gesetzt wurde.

Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch oder im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Hinweis: Jeder Prozesstimer muss einen Start- und einen Endtimer aufweisen. Es gibt jedoch keine Logik, die das alleinige Auftreten nur eines der beiden Timer verhindert.
Hinweis: Prozesstimer sind nur von den Transaktionen "Abmelden - standardmäßig" und "Anmelden" abhängig. Die Anwendung berücksichtigt nicht die Anzahl dieser Transaktionen, die für eine Aufgabe im automatischen Ablauf durchgeführt werden können. Der Prozesstimer hält an, nachdem die Aufgabe ausgeführt wurde.

Hinzufügen von Geschäftsregeln zu einer Spezifikation

Sie können Geschäftsregeln zu einer Spezifikation hinzufügen, sodass die Logik der Geschäftsregel ausgeführt wird, wenn ein mit einer Containertransaktion verknüpftes Ereignis auftritt. Fügen Sie die Geschäftsregeln in der Tabelle "Geschäftsregel: Transaktionszuordnung" im Abschnitt "Transaktionen" hinzu. Es stehen nur Geschäftsregeln zur Auswahl, denen der Verwendungstyp "Container" zugewiesen ist. Weitere Informationen finden Sie unter "Felddefinitionen der Seite "Geschäftsregel"".

Die Tabelle "Geschäftsregel: Transaktionszuordnung" enthält drei Spalten: "Geschäftsregel", "Ereignis" und "Transaktion". Wählen Sie die Geschäftsregel und die Containertransaktion aus, von der sie ausgeführt wird. Wählen Sie außerdem das Ereignis aus, das bestimmt, wann die Geschäftsregel in der Containertransaktion ausgeführt wird. Die folgenden Ereignissen stehen zur Verfügung:

• Benutzer ausführen (Execute User)
• Benutzer initialisieren Export-Info (Export Info Initialize User)
• Benutzer vorab ausführen (Pre Execute User)
• Benutzer vorab überprüfen (Pre Validate User)
• Nach Übergabe von Benutzer verarbeiten (Process After Commit User)
• Benutzer überprüfen (Validate User)

Berechnen des Ablaufdatums auf der Seite "Spez."
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Auf der Seite "Spez." können Sie die Felder definieren, die zur automatischen Berechnung eines Auflaufdatums für einen Container erforderlich sind, wenn eine vordefinierte Transaktion für die angegebene Spezifikation erfolgreich ist. Es handelt sich um folgende Felder:

• Ablaufdatumtransaktion
• Maßeinheit für Auflaufperiode
• Ablaufperiode

Das Ablaufdatum des Containers wird nur dann auf Grundlage der Spezifikationsinformationen berechnet, wenn für das Produkt kein Ablaufdatum definiert ist.

Wenn der Container bereits ein Ablaufdatum hat, das vor dem berechneten Datum liegt, wird das Ablaufdatum nicht aktualisiert, es sei denn, das Kontrollkästchen "Ablaufdatum immer überschreiben" ist aktiviert.

Wenn als "Maßeinheit für Auflaufperiode" der Wert "Monate" festgelegt ist und die Berechnung des Ablaufdatums ein ungültiges Datum ergibt, wird das Ablaufdatum angepasst und auf den letzten Tag des Monats festgelegt. Beispiel: Wenn das aktuelle Datum der 31. Januar ist und die Ablaufperiode drei Monate beträgt, ist das berechnete Ablaufdatum der 31. April. Das Datum wird in diesem Fall auf den 30. April festgelegt.

Berechnen des Produktionsdatums auf der Seite "Spez."
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Sie können in der Spezifikation eine Produktionsdatumtransaktion auswählen, damit dem Container automatisch ein Produktionsdatum zugewiesen wird. Dadurch werden das Datum und die Uhrzeit der Transaktion dem Feld "Produktionsdatum" für den Container zugewiesen. Wenn für das Produkt ein Produktionsdatum definiert ist, hat dieses Datum Vorrang.

Definieren einer UDI auf der Seite "Spez."
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Mit der UDI-Funktionalität (UDI = Unique Device Identification, eindeutige Gerätekennung) der Anwendung können Sie überprüfen, ob einem Container eine UDI zugewiesen wurde. Wenn die Option "UDI überprüfen" in der Spezifikation ausgewählt ist, kann ein Container nur bewegt werden, wenn das Feld "UDI" einen Wert enthält.

Zuweisen einer Waagengruppe für die Spezifikation
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Sie können über das Feld "Waagengruppe" eine Waagengruppe für die Spezifikation zuweisen. Sie können eine Waagengruppe auch für Produktfamilie, Produkt und Verbrauchswiegeaufgabe zuweisen.

Die folgenden Regeln gelten beim Zuweisen von Waagengruppen:

• Die für die Produktfamilie zugewiesene Waagengruppe hat Vorrang vor der Waagengruppe, die der Spezifikation zugewiesen ist.
• Die für das Produkt zugewiesene Waagengruppe hat Vorrang vor der Waagengruppe, die der Produktfamilie zugewiesen ist.
• Die für die Verbrauchswiegeaufgabe zugewiesene Waagengruppe hat Vorrang vor der Waagengruppe, die allen anderen Objekten zugewiesen ist.

Weitere Informationen finden Sie unter "Zuweisen einer Waagengruppe für die Produktfamilie", "Zuweisen einer Waagengruppe zu einem Komponentenprodukt" und "Beim Definieren von Verbrauchswiegeaufgaben".

Definieren von Transaktionen mit Teamnachverfolgung auf der Seite "Spez."
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Auf der Seite "Spez." können Sie die Transaktionen definieren, bei denen die Arbeit verfolgt werden muss, die von Mitarbeitern für bestimmte Operationen, Spezifikationen, Arbeitsbereiche, Arbeitszellen oder Arbeitsplätze geleistet wird. Durch Definieren von Transaktionen auf Fabrikebene können Sie die Verfolgungsanforderungen für alle Spezifikationen festlegen, anstatt jede Spezifikation in einem Arbeitsplan zu ändern. Alle Transaktionen, die auf Spezifikationsebene festgelegt werden, haben jedoch Vorrang vor der Auswahl auf Fabrikebene.

Produktwandlungsanforderung

Gibt die Anforderung an, die für die Produktwandlung erfüllt sein muss. Gültige Werte sind:

• NULL – Leere Zeichenfolge, die angibt, dass keine Wandlung erforderlich ist und das Service-Flag "Prozessdurchsatzanforderung" auf "wahr" gesetzt ist.
• MANUAL – Die Wandlung muss manuell vor einer Abmeldetransaktion ausgeführt werden, während das Service-Flag "Prozessdurchsatzanforderung" auf "wahr" gesetzt ist.
• AUTO – Die Wandlung wird automatisch während einer Abmeldetransaktion ausgeführt, während das Service-Flag "Prozessdurchsatzanforderung" auf "wahr" gesetzt ist.

Durchsatzanforderung

Gibt die Anforderung für den Durchsatz an, wenn für einen Container eine Abmelden-Transaktion ausgeführt wird. Gültige Werte sind:

• OPTIONAL - Durchsatz ist nicht erforderlich.
• AUTO - Durchsatz wird vor dem Abmelden automatisch ausgelöst. Der Durchsatz erfolgt immer für die gesamte Menge.
• MANUALPARTIAL - Durchsatz ist erforderlich, muss jedoch nicht für die gesamte Menge des Containers erfolgen. Während des Abmeldens wird ein Fehler ausgegeben, wenn kein Mengendurchsatz erkannt wird.
• MANUALFULL - Durchsatz ist erforderlich und muss die gesamte im Container enthaltene Menge umfassen. Während des Abmeldens wird ein Fehler ausgegeben, wenn der Durchsatz nicht die gesamte Menge umfasst.

Das Materialrücknahmedatum des Containers wird nur auf das Datum der Abmelden-Transaktion gesetzt, wenn es Null lautet. Andernfalls wird die Differenz zwischen dem Materialrücknahmedatum und dem Datum der Abmelden-Transaktion auf den Materialverfügbarkeitsoffset aktualisiert. Dieser Offset wird verwendet, um einen vorübergehenden Stillstand der Verfügbarkeit darzustellen.

Optional

Ist Materialverbrauch

Kontrollkästchen, über das ein Flag festgelegt wird, das verwendet wird, wenn die Ressourcenkomponenten-Einrichtung oder der Komponentenverbrauch abgewickelt wird. Die Transaktion ist nicht zulässig, wenn eine Tauanforderung verfügbar ist und der Container die Taudauer noch erreichen muss. Analog dazu schlägt die Transaktion fehl, wenn eine Verfügbarkeitsanforderung vorhanden ist und der Container die Verfügbarkeitsdauer überschritten hat.

Optional

Ist Materialrücklauf

Kontrollkästchen, über das ein Flag festgelegt wird, das verwendet wird, wenn ein Container zu einer Spezifikation angemeldet wird. Beachten Sie, dass das Attribut der Zielspez. für die Logik verwendet wird. Anhand der maximalen Materialrückläufe für das Container-Produkt wird überprüft, ob eine Anmeldung zu einer derartigen Spez. zulässig ist. Sobald das Los den Wert übersteigt (nur im Falle von > 0), ist es nicht zulässig, den Container zu der Spez. anzumelden. Wenn dagegen nicht verstoßen wird, wird die Materialgesamtverfügbarkeit angesammelt, wenn "Materialsammlung Verfügbarkeit" für das Produkt auf "wahr" gesetzt und ein Wert für das Materialrücknahmedatum angegeben ist. Dann werden die Materialgesamtrückläufe des Containers um 1 erhöht, das Materialrücknamedatum annulliert und der Materialverfügbarkeitsoffset auf 0 gesetzt.

Optional

Autom. schließen

Kontrollkästchen, das bestimmt, ob durch die Abmelden-Transaktion ein Container geschlossen wird, wenn er zur Zielspezifikation abgemeldet wird.

Optional

Eltern-Container schließen

Wird innerhalb der vorhandenen Abtrennen-Logik verwendet, um die Eltern-Container (Panels) zu schließen, nachdem die Kind-Container (PCBs) freigegeben oder abgetrennt wurden. Weitere Informationen finden Sie unter "Autom. abtrennen".

Optional

Autom. abtrennen

Kontrollkästchen, über das eine Elementierung aufgehoben werden kann. Die Elementierung von PCBs wird in einem auf dieser Spezifikation basierenden Schritt aufgehoben und die PCBs werden vom Eltern-Panel abgetrennt, wenn das Kontrollkästchen "Autom. abtrennen" aktiviert ist.

Optional

Automatisch in Fenster ausgeben

Kontrollkästchen, mit dem konfiguriert wird, wie die Anwendung automatischen Komponentenverbrauch für elementierte Produkte verarbeitet. Standardmäßig ist dieses Kontrollkästchen deaktiviert, d. h., dass Komponenten, die für automatischen Verbrauch konfiguriert sind, für jede PCB, aber nicht für das Panel verbraucht werden. Ist das Kontrollkästchen aktiviert, werden Komponenten, die für automatisches Verbrauchen konfiguriert sind, für das Panel, aber nicht für die PCBs verbraucht.

Hinweis: Ist das Kontrollkästchen "Automatisch in Fenster ausgeben" (automatisch für Panel verbrauchen) aktiviert, multipliziert die Anwendung die Komponentenmenge mit der Anzahl der PCBs, für die die Komponente verbraucht wird, um die für das Panel angewendete Menge zu ermitteln.

Optional

Autom. Komponentenverbrauch

Kontrollkästchen, das bestimmt, ob von der Anwendung automatisch die Komponentenverbrauch-Transaktion während einer Abmelden-Transaktion ausgelöst wird, wenn eine Ressource zur Verfügung gestellt wird und mindestens ein Verbrauchsdetail erkannt wird.

Optional

Produktionsauftrags in Produktions-Client neu zuweisen

Wenn dieses Kontrollkästchen aktiviert ist, zeigt der Produktions-Client ein Popup an, in dem der Bediener die derzeit eingegebenen oder eingescannten Container einem anderen Produktionsauftrag zuweisen muss.

Optional

Physische Adresse zuweisen

Kontrollkästchen, das bestimmt, ob die Anwendung physische Adressen automatisch zuweisen soll, wenn ein Container basierend auf dieser Spezifikation aus dem Arbeitsplanschritt abgemeldet wird.

Optional

Physische Adresse prüfen

Kontrollkästchen, das bestimmt, ob die Anwendung überprüft, ob alle physischen Adressen zugewiesen wurden, um zu verhindern, dass der Container vom Schritt abgemeldet wird, wenn diese nicht zugewiesen sind.

Optional

Iterationsgrenzenprüfung (Iteration Limit Check)

 

 

Iterationsgrenzen aktivieren (Enable Iteration Limits)

Kontrollkästchen, das angibt, ob die Anwendung verhindert, dass ein Container einen Schritt basierend auf dieser Spezifikation durchläuft, wenn dadurch der konfigurierte Grenzwert überschritten werden würde.

Optional

Maximale Anzahl der Male, die derselbe Container basierend auf dieser Spezifikation einen Schritt durchlaufen kann.

Hinweis: Dieses Feld wird von der Anwendung nur verwendet, wenn das Kontrollkästchen "Iterationsgrenzen aktivieren (Enable Iteration Limits)" aktiviert ist.

Optional

Aktion, die für einen Container auszuführen ist, wenn der Iterationsgrenzwert überschritten wird. Beispiele für Aktionen schließen "Anhalten" und "Ausschuss" ein.

Der Anhaltegrund, der standardmäßig verwendet wird, wenn ein Container angehalten wird, da der Iterationsgrenzwert überschritten wurde.

Optional

Der Ausschussgrund, der standardmäßig verwendet wird, wenn ein Container als Ausschuss festgelegt wird, da der Iterationsgrenzwert überschritten wurde.

• Kein: Es erfolgt kein Löschen und das Kontrollkästchen wird ignoriert.

Vorgehensweise zum Definieren einer Spezifikation

Gehen Sie folgendermaßen vor, um eine Spezifikation zu definieren:

1. Öffnen Sie die Seite Spez. Die Seite "Spez." wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Spezifikation in das Feld Spez. ein.
4. Geben Sie die Revision dieses Objekts in das Feld Revision ein.
5. Wählen Sie die Operation, die mit der Spezifikation verbunden werden soll, aus der Liste Operation aus.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Datensammlungsinformationen

Gehen Sie folgendermaßen vor, um Datensammlungsinformationen hinzuzufügen:

Oder

Wählen Sie eine vorhandene Spezifikationsinstanz aus.

2. Erweitern Sie den Abschnitt Transaktionen.
3. Klicken Sie in der Tabelle "Zuordnung von Datensammlungstransaktionen" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
