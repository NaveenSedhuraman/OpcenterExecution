Definieren von Anforderungen zur elektronischen Unterschrift

Anforderungen zur elektronischen Unterschrift ermöglichen es Ihnen, verschiedene elektronische Unterschriftskomponenten in einer einzelnen Definition zu gruppieren, die dann einer Fertigungstransaktion zugeordnet werden kann.

Wenn Sie z. B. eine elektronische Unterschrift anfordern möchten, um zu überprüfen, ob Schachteln korrekt befüllt und versiegelt wurden, können Sie eine Anforderung namens "Schachtelüberprüfung" erstellen, in der diese Komponenten definiert sind.

Rolle:

Die Person, die die Unterschrift leisten soll

Bedeutung:

Zweck der Unterschrift

Anzahl:

Die Anzahl erforderlicher Unterschriften

Mitunterzeichnerrolle:

Falls zutreffend

Verifizierungsmethode:

Die Methode zur Überprüfung der Authentizität der Unterschrift dieser Person

Auf Grundlage dieses Beispiels könnte die Schachtelüberprüfungsanforderung aus Folgendem bestehen.

Rolle:

Schachtelüberprüfer

Bedeutung:

Prüfer

Anzahl:

1

Mitunterzeichnerrolle:

Bereichsleiter

Verifizierungsmethode:

Passwortprüfung

Diese Anforderung ermöglicht es dem Karton-Überprüfer, zu bestätigen, dass ein Karton korrekt befüllt und versiegelt wurde. Dazu leistet er eine Unterschrift und authentifiziert diese durch Eingabe seines Kennworts. Wenn die Unterschrift des Karton-Überprüfers aus irgendeinem Grund nicht verifiziert werden kann, kann der Bereichsleiter eine Mitunterschrift leisten, um die Unterschrift des Karton-Überprüfers zu bestätigen.

Beim Definieren von Anforderungen zur elektronischen Unterschrift

"Elektr. Unterschrift: Anforderung" wird von "Spez." referenziert.

"Elektr. Unterschrift: Anforderung" referenziert die "Elektr. Unterschrift: Rollengruppe" und die "Elektr. Unterschrift: Bedeutung".

Wenn Sie eine Anforderung zur elektronischen Unterschrift erstellen, die bei Fertigungstransaktionen verwendet werden soll, müssen Sie die Anforderung im Fenster "Spez." einer zugehörigen Operation zuweisen.

Mitarbeiter müssen über alle zu einer Anforderung zugewiesenen Rollen eindeutig sein. Die Anwendung ermöglicht es einem Mitarbeiter, nur einmal pro Anforderung zu unterschreiben, unabhängig von der Anzahl von Bedeutungen, die der Anforderung zugeordnet sind.

Vorgehensweise zum Definieren einer Anforderung zur elektronischen Unterschrift

Gehen Sie folgendermaßen vor, um eine Anforderung zur elektronischen Unterschrift zu definieren:

1. Öffnen Sie die Seite Elektr. Unterschrift: Anforderung. Die Seite "Elektr. Unterschrift: Anforderung" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Anforderung zur elektronischen Unterschrift in das Feld Elektr. Unterschrift: Anforderung ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von elektronischen Unterschriften zu einer Anforderung zur elektronischen Unterschrift

Gehen Sie folgendermaßen vor, um eine elektronische Unterschrift zu einer Anforderung zur elektronischen Unterschrift hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Anforderung zur elektronischen Unterschrift" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Anforderung zur elektronischen Unterschrift aus.

2. Klicken Sie in der Tabelle "Elektronische Unterschriften erforderlich" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Geben Sie Informationen in die folgenden erforderlichen Felder ein:
• Rolle
• Bedeutung
• Anzahl
• Verifizierungsmethode
4. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
