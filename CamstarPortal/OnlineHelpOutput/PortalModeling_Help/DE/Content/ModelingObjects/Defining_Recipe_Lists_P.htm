Definieren von Rezeptlisten

Eine Rezeptliste, ein revisionierbares Objekt, ist ein Ablauf zur Ausführung eines Schritts in einem Stammrezept. Eine Rezeptliste besteht aus einer oder mehreren Aufgaben, die zur Ausführung eines Schritts erforderlich sind. Sie müssen mindestens eine Rezeptliste angeben, wenn Sie ein Stammrezept definieren. Ein Beispiel einer Rezeptliste ist eine Gruppe von Aufgaben zur Erfassung von Produktmessdaten im Zusammenhang mit dem Stammrezept. Weitere Informationen finden Sie unter "Definieren von Stammrezepten".

Grundlegendes zu Aufgaben

Eine Aufgabe ist eine Arbeitseinheit, die an einer Spezifikation für einen Materialcontainer ausgeführt werden muss. Sie fassen Aufgaben in logischen Rezeptlisten zusammen und verknüpfen eine oder mehrere dieser Rezeptlisten mit einem Stammrezept. Die Rezeptliste setzt die definierte Reihenfolge der Aufgaben durch, wenn sie als "Sequenziell" gekennzeichnet ist. Ob eine Aufgabe als erfolgreich ausgeführt bewertet wird oder nicht, hängt von den erfassten Daten oder der Entscheidung des Bedieners in der Fertigung ab.

Aufgabenelementtypen

Sie können den Aufgabenelementtyp auswählen, wenn Sie ein Aufgabenelement hinzufügen.

Aufgabenelementtyp

Beschreibung

Anweisung

Jede allgemeine Art von Aufgabe.

Berechnung

Die Anwendung führt auf der Grundlage von Benutzerkonstanten, Parametern und den Eingaben des Bedieners eine Berechnung durch, wenn die Aufgabe ausgeführt wird.

Rezeptverbrauchselement

Eine Aufgabe zum Verbrauchen einer bestimmten Menge eines Produkts. Sie können mit dieser Aufgabe auch Regeln festlegen, die bestimmen, was verbraucht wird und wie.

Hinweis: Benutzerkonstanten und Berechnungen werden ebenfalls in der Modellierung festgelegt. Weitere Informationen finden Sie unter "Definieren von Benutzerkonstanten" und "Definieren von Berechnungen".

Aufgabentypen

Der Aufgabentyp eines Aufgabenelements ist wichtig für diejenigen Arbeitsplätze, die Linienfreigabe (Line Clearance) erzwingen. Ein Aufgabenelement kann einen der folgenden Aufgabentypen haben.

Aufgabentyp

Beschreibung

Prozess starten

Kennzeichnet den Verarbeitungsbeginn an einem Arbeitsplatz. Eine Prozess starten-Aufgabe ändert den Status des Arbeitsplatzes in "In Bearbeitung", und zwar entweder auf dem Containerlevel oder auf der Produktionsauftragsebene. Eine Prozess starten-Aufgabe ist erforderlich und muss bei Arbeitsplätzen, die Linienfreigabe erzwingen, die erste Aufgabe in der Aufgabenliste sein.

Hinweis: Der minimale Wiederholungswert für eine Prozess starten-Aufgabe muss größer als 0 sein.

Normal

Normale Aufgaben können zwischen einer Prozess starten-Aufgabe und einer Prozess beenden-Aufgabe an Arbeitsplätzen, die Linienfreigabe erzwingen, auftreten. Eine Aufgabenliste kann alle normalen Aufgaben für Arbeitsplätze enthalten, die keine Linienfreigabe erzwingen.

Eine normale Aufgabe muss für denselben Container oder Produktionsauftrag innerhalb derselben Aufgabenliste, desselben Stammrezepts und desselben Spezifikationsschritts ausgeführt werden wie die Prozess starten-Aufgabe. Andernfalls schlägt sie fehl.

Prozess beenden

Kennzeichnet das Ende eines Prozesses an einem Arbeitsplatz. Eine Prozess beenden-Aufgabe setzt den Arbeitsplatzstatus zurück. Die Prozess beenden-Aufgabe zeigt an, dass der Arbeitsplatz für die Bearbeitung des nächsten Containers oder Produktionsauftrags bereit ist. Eine Prozess beenden-Aufgabe ist für Arbeitsplätze, die Linienfreigabe erzwingen, erforderlich und muss die letzte Aufgabe in der Aufgabenliste sein.

Hinweis: Der minimale Wiederholungswert für eine Prozess beenden-Aufgabe muss größer als 0 sein

Grundlegendes zum Ausführungsmodus

Geben Sie die einzelnen Aufgaben in der Reihenfolge ein, in der sie durchgeführt werden müssen, wenn Sie eine Rezeptliste erstellen. Wenn die Aufgaben nicht in einer bestimmten Reihenfolge erledigt werden müssen, können Sie diese in beliebiger Reihenfolge eingeben. Die Rezeptliste definiert den Ausführungsmodus der Aufgaben.

Sie geben den Ausführungsmodus einer Aufgabenliste an, indem Sie im Feld "Ausführungsmodus" auf der Seite "Rezeptliste" die Option "Sequenziell" oder "Nicht sequenziell" auswählen:

• Der Standardmodus für die Ausführung ist "Sequenziell". In diesem Fall wird jede Rezeptliste in der Reihenfolge angezeigt, die im Stammrezept angegeben ist. Eine Rezeptliste kann nur übergangen werden, wenn alle Aufgaben optional sind.
• Wenn der Ausführungsmodus "Nicht sequenziell" festgelegt wird, muss die Rezeptliste nicht in einer im Stammrezept festgelegten Reihenfolge ausgeführt werden. Ebenso gibt das Stammrezept den Ausführungsmodus für Rezeptlisten an.

Bevor Sie eine Rezeptliste definieren

Bevor Sie eine Rezeptliste definieren und Aufgaben hinzufügen, müssen Sie je nach Bedarf u. U. die folgenden Elemente definieren:

• Behälter
• Berechnungen
• Containerlevel
• Dokumentensätze
• Anforderungen zur elektronischen Unterschrift
• Produkt
• Trainingsanforderungsgruppen
• Maßeinheit
• Benutzerkonstanten
• Benutzerdatensammlungsdefinitionen
• Arbeitsplatz
• Arbeitsplatzgruppe
Hinweis: Arbeitsplätze werden mit Hilfe des Modellierungsobjekts "Ressource" und Arbeitsplatzgruppen mit Hilfe des Modellierungsobjekts "Ressourcengruppe" definiert.

Opcenter EX MDD und Opcenter EX CR bieten einen Text-Editor, mit dem Sie Anweisungen entweder in Rezeptlisten oder in Rezeptlistenaufgaben formatieren können. Weitere Informationen finden Sie unter "Definieren einer Aufgabenliste und zugehöriger Aufgaben".

Beim Definieren von Datensammlungsaufgaben für Anweisungsaufgaben

Sie müssen "Datensammlung" im Feld "Anweisungstyp" auswählen und dann eine vordefinierte Datensammlungsdefinition aus dem Feld "Datensammlungsdefinition" auswählen, wenn Sie eine Datensammlungsaufgabe definieren möchten. Wenn Sie eine Datensammlungsdefinition mit der Aufgabe verknüpfen, aber im Feld "Anweisungstyp" einen anderen Wert als "Datensammlung" auswählen, zeigt die Anwendung auf der Seite "Automatischer Ablauf" keine Datensammlungsfelder an.

Beim Definieren von Berechnungsaufgaben

Die Anwendung verlangt, dass die Datentypen des verknüpften Datenpunkts und Berechnungsparameters identisch sind, wenn Sie eine Berechnungsaufgabe festlegen. Wenn sich die Datentypen unterscheiden, wird eine Fehlermeldung angezeigt. Datenpunktdatentypen werden in der angegebenen Datensammlungsdefinition und Berechnungsparameter-Datentypen in der angegebenen Berechnungsdefinition festgelegt.
