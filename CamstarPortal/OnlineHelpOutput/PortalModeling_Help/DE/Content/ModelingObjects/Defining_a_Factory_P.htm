Definieren einer Fabrik

Eine Fabrik ist eine beliebige Division, Abteilung oder Gruppe, die in ihrer Buchhaltung und in ihrem Berichtswesen eigenständig ist. Ein Unternehmen kann eine oder mehrere Fabriken enthalten.

Oft stellt sie ein Fabrikgebäude dar, aber sie kann auch für das Unternehmen selbst oder für eine große Abteilung innerhalb des Unternehmens stehen.

Eine Fabrik kann Referenzen auf ein Unternehmen und auf Orte innerhalb der Fabrik enthalten.

Beim Definieren einer Fabrik

Das Feld "SMTP-Transport" verwendet das Simple Mail Transport Protocol (SMTP), um E-Mails zu verschicken. Dadurch können Opcenter EX MDD und Opcenter EX CR E-Mails versenden, wenn die Anwendung für die Ausführung als Windows-Service konfiguriert ist.

Ein Portal-Benutzer kann eine Fabrik als Entdeckungsbereich festlegen, wenn er ein Ereignis erstellt.

Die Fabrik enthält die optionalen Modellierungsdefinitionen "Unternehmen", "Produktionskalender", "Erledigungsregel", "Trainingsanforderungsgruppe", "Druckwarteschlange", "Nummerierungsregel für Container", "SMTP-Transport" und "Änderungsmanagementeinstellungen".

Bei einer Integration mit Teamcenter enthält die Fabrik die optionale Modellierungsdefinition "PLM-Modellierungseinstellungen". Weitere Informationen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Installationsanleitung oder in der Opcenter Execution Core-Installationsanleitung.

Eine Fabrik ist:

• Ein optionales Feld, das Sie bei der Definition eines Mitarbeiters, eines Unternehmens, eines Produktionsauftrags, einer Ressource und eines Versandziels ausfüllen können.
• Ein optionales Feld in den meisten Opcenter EX MDD- oder Opcenter EX CR-Transaktionen.
Hinweis: Eine Fabrik kann zugehörige Umlaufbestandsnachrichten haben, wenn sie mit einem Feld aus einer Containerdefinition verbunden ist. Weitere Informationen finden Sie unter "Definieren von Umlaufbestandsnachrichten" sowie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Sie können Objektreferenzen auswählen, die Sie aus einem Export ausschließen möchten. Das Objekt muss auf der Mitarbeiterebene zugewiesen sein, wenn es ausgeschlossen werden soll. Sie müssen sich ab- und wieder anmelden, damit die Änderungen an der Fabrikzuweisung, die Sie auf der Seite "Mitarbeiter" durchgeführt haben, wirksam werden.

Weitere Informationen zum Exportprozess finden Sie unter "Exportieren von Modellierungsdaten". Informationen zu Änderungsverwaltungspaketen finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.

Hinzufügen von Orten

Ein Ort definiert einen physischen Ort innerhalb einer Fabrik. Diese Definition kann abstrakt sein, etwa ein einzelner Ort, der die gesamte Fabrik definiert, oder konkret, etwa individuelle Orte für jeden einzelnen Behälter innerhalb eines Lagerbereichs.

Jeder Container hat ein Standort-Attribut, mit dem Sie den physischen Ort (Standort) des Containers angeben und Berichte, wie z. B. "Umlaufbestand nach Standort", erstellen können. Standorte werden in Opcenter EX MDD und Opcenter EX CR nicht separat, sondern innerhalb der Fabrik definiert.

"Standort" ist ein optionales Feld in den meisten Opcenter EX MDD- und Opcenter EX CR-Transaktionen. Sie können bei der Definition einer Fabrik, einer Ressourcengruppe und eines Versandziels einen Ort angeben.

Hinzufügen allgemeiner Meldungen

Sie können allgemeine Meldungen hinzufügen, die Mitarbeitern bei der Anmeldung angezeigt werden und sie über kommende Ereignisse wie geplante Wartungen oder eine ausstehende Prüfung informieren oder sonstige Mitteilungen enthalten.

Die Option zum Hinzufügen allgemeiner Meldungen wird auf der Seite "Fabrik" im Abschnitt "Allgemeine Meldung" angezeigt und über das Kontrollkästchen "Meldung anzeigen" aktiviert. Zum Erstellen von Meldungen steht ein Rich-Text-Editor zur Verfügung. Die Anwendung erzeugt beim Speichern des Objekts einen Fehler, wenn das Kontrollkästchen "Meldung anzeigen" aktiviert ist, im Textfeld jedoch keine Meldung eingegeben wird. Informationen zur Verwendung des Text-Editors finden Sie unter "Definieren einer Aufgabenliste und zugehöriger Aufgaben".

Das Popup "Allgemeine Meldung" wird in der Mitte der Standard-Homepage des Benutzers angezeigt, nachdem dieser die Linienzuordnung eingestellt hat. Falls die Anwendung so konfiguriert ist, dass die Seite "Linienzuordnung" übersprungen wird, dann wird die Meldung nach dem Anmelden angezeigt. Das Popup "Allgemeine Meldung" wird geschlossen, wenn der Benutzer auf die Schaltfläche "Schließen" klickt.

Hinzufügen von Anforderungen zur elektronischen Unterschrift zum Anhängen oder Abtrennen von Dokumenten bei Fabrik-Transaktionen

Sie können festlegen, dass Benutzer elektronische Unterschriften leisten müssen, um Dokumente an Transaktionen anzuhängen oder von Transaktionen abzutrennen, die in einer Fabrik ausgeführt werden. Mögliche Dokumente sind besondere Anweisungen, Testergebnisse, Zeichnungen oder Fotos. Informationen zum Anhängen von Dokumenten finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Die Option, Anforderungen zur elektronischen Unterschrift für das Anhängen oder Abtrennen von Dokumenten festzulegen, ist im Abschnitt "Transaktionen" in "Elektr. Unterschrift: Transaktionszuordnung" enthalten. Optional können Sie festlegen, dass für alle Transaktionen elektronische Unterschriften erforderlich sind.

Hinzufügen von Anforderungen zur elektronischen Unterschrift für Herstellungslose und Stichprobenlose bei Fabrik-Transaktionen

Sie können festlegen, dass Benutzer für Herstellungslose und Stichprobenlose bei Transaktionen, die in einer Fabrik ausgeführt werden, elektronische Unterschriften leisten müssen. Die Option zum Festlegen dieser Anforderungen befindet sich im Abschnitt "Transaktionen" in "Elektr. Unterschrift: Transaktionszuordnung". Optional können Sie festlegen, dass für alle Transaktionen elektronische Unterschriften erforderlich sind.

Sie können die Unterschriften für ein Stichprobenlos auf den Fertigungsseiten "Stichprobenlos erstellen" oder "Stichprobenlos aktualisieren" leisten. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch oder im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Definieren von Transaktionen mit Teamnachverfolgung auf der Seite "Fabrik"
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Auf der Seite "Fabrik" können Sie die Transaktionen definieren, bei denen die Arbeit verfolgt werden muss, die von Mitarbeitern für bestimmte Operationen, Spezifikationen, Arbeitsbereiche, Arbeitszellen oder Arbeitsplätze geleistet wird. Durch Definieren von Transaktionen auf Fabrikebene können Sie die Verfolgungsanforderungen für alle Spezifikationen festlegen, anstatt jede Spezifikation in einem Arbeitsplan zu ändern. Alle Transaktionen, die auf Spezifikationsebene festgelegt werden, haben jedoch Vorrang vor der Auswahl auf Fabrikebene.

Zuweisen eines Standardlagerorts für eine Fabrik
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Sie können einer Fabrik einen Standardlagerort zuweisen. Sie können auch einem Produkt und einer Produktfamilie einen Standardlagerort zuweisen.

Die folgenden Regeln gelten beim Zuweisen von Standardlagerorten:

• Der Standardlagerort, der für das Produkt zugewiesen ist, hat Vorrang vor dem Standardlagerort, der für die Produktfamilie zugewiesen ist.
• Der Standardlagerort, der für die Produktfamilie zugewiesen ist, hat Vorrang vor dem Standardlagerort, der für die Fabrik zugewiesen ist.
• Die Anwendung weist einem Container den Standardlagerort automatisch zu, wenn der Container in eine Operation bewegt wird, die als Lagerort definiert ist, sofern für die Fabrik ein Standardlagerort definiert wurde.

Weitere Informationen finden Sie unter "Zuweisen eines Standardlagerorts für ein Produkt" und "Zuweisen eines Standardlagerorts für eine Produktfamilie".

Ermöglichen von Materialverbrauch aus Materialwarteschlangen
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Das in eine Materialwarteschlange geladene Material wird automatisch verbraucht, wenn eine Transaktion im Feld "Material-verbrauchen-Transaktionen" im Abschnitt "Materialwarteschlangen" ausgewählt ist. Folgende Transaktionen sind verfügbar:

• Start
• Anmelden
• Abmelden

Sie können überprüfen, ob für einen bestimmten Container das richtige Material in eine Materialwarteschlange geladen ist, indem Sie im Feld "Materialwarteschlange-prüfen-Transaktionen" mindestens eine Transaktion auswählen. Die Anwendung gibt eine Warnmeldung aus, wenn Sie eine Transaktion auswählen, aber nicht das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktivieren. Die Transaktion schlägt fehl, falls die Menge nicht ausreichend ist, wenn Sie eine Transaktion auswählen und das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktiviert ist. Sie können einen "Verbrauchsabweichungsgrund" auswählen, um zu verhindern, dass ein Fehler auftritt, wenn beim automatischen Komponentenverbrauch Teilmengen verbraucht werden.

Aktivieren von Opcenter Execution Core Scheduling auf der Seite "Fabrik"
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Sie müssen Opcenter Execution Core Scheduling aktivieren, indem Sie das Kontrollkästchen "Preactor-Schnittstelle aktiviert" auf der Seite "Fabrik" aktivieren, um die Übertragung von Informationen aus Opcenter EX MDD oder Opcenter EX CR an die Siemens-Anwendung Preactor Advanced Planning and Scheduling (Preactor) zu ermöglichen.

Zuweisen von OEE-Einstellungen für die Fabrik
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Wählen Sie im Abschnitt "OEE" das Kontrollkästchen "OEE-Einstellungen" aus, wenn Sie die Zeitspanne festlegen möchten, für die die Anwendung Overall Equipment Effectiveness (OEE, Gesamtanlageneffektivität) für die ausgewählte Fabrik berechnen soll. Sie können auch OEE-Einstellungen für die Ressource und die Ressourcenfamilie zuweisen.

Die folgenden Regeln gelten beim Zuweisen von OEE-Einstellungen:

• Die OEE-Einstellungen, die dem Modellierungsobjekt "Ressource" zugewiesen wurden, haben Vorrang vor den OEE-Einstellungen, die dem Modellierungsobjekt "Ressourcenfamilie" zugewiesen wurden.
• Die OEE-Einstellungen, die dem Modellierungsobjekt "Ressourcenfamilie" zugewiesen wurden, haben Vorrang vor den OEE-Einstellungen, die dem Modellierungsobjekt "Fabrik" zugewiesen wurden.

Weitere Informationen zum Zuweisen von OEE-Einstellungen zu einer Ressource finden Sie unter "Beim Zuweisen von OEE-Einstellungen für die Ressource". Weitere Informationen zum Zuweisen von OEE-Einstellungen zu einer Ressourcenfamilie finden Sie unter "Beim Zuweisen von OEE-Einstellungen für die Ressourcenfamilie".

Vorgehensweise zum Definieren einer Fabrik

Gehen Sie folgendermaßen vor, um eine Fabrik zu definieren:

1. Öffnen Sie die Seite Fabrik. Die Seite "Fabrik" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Fabrik in das Feld Fabrik ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
6. Weitere Informationen zum Hinzufügen von Orten zu dieser Fabrik finden Sie unter "Vorgehensweise zum Hinzufügen eines Orts".
7. Weitere Informationen zum Festlegen von Anforderung zur elektronischen Unterschrift finden Sie unter "Vorgehensweise zum Hinzufügen von Anforderungen zur elektronischen Unterschrift zum Anhängen oder Abtrennen von Dokumenten bei Fabrik-Transaktionen".
8. Weitere Informationen zum Festlegen von Anforderungen zur elektronischen Unterschrift finden Sie unter "Vorgehensweise zum Hinzufügen von Anforderungen zur elektronischen Unterschrift für Herstellungslose und Stichprobenlose bei Fabrik-Transaktionen".

Vorgehensweise zum Hinzufügen eines Orts

Gehen Sie folgendermaßen vor, um einer Fabrik einen Ort hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Fabrik" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Fabrik aus.

2. Klicken Sie in der Tabelle "Orte" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Geben Sie den Ortsnamen und eine Beschreibung in die entsprechenden Felder der Tabelle ein.
4. Wählen Sie Aktiv oder Inaktiv in der Liste Status aus.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Anforderungen zur elektronischen Unterschrift zum Anhängen oder Abtrennen von Dokumenten bei Fabrik-Transaktionen

Gehen Sie folgendermaßen vor, um Anforderungen zur elektronischen Unterschrift zum Anhängen oder Abtrennen von Dokumenten zu Fabrik-Transaktionen hinzuzufügen:

Oder
