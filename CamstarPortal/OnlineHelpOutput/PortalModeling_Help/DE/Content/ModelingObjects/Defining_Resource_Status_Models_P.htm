Definieren von Ressourcenstatusmodellen

Ein Ressourcenstatusmodell gibt zwei Dinge an:

• Die anfängliche Ressourcenstatuscodegruppe, die einer einzelnen Ressource bei der Transaktion zur Ressourceneinrichtung zugeordnet wird. Dies bedeutet, dass jeder Ressourcenstatuscode in der ausgewählten Gruppe der Ressource zugewiesen werden kann, wenn diese erstmals eingerichtet wird.
• Die gültige Ressourcenstatuscodegruppe, in der die zulässigen Status definiert sind, die eine Ressource mit einem anderen Status annehmen kann. Eine Ressource mit dem Statuscode In Reparatur (In Repair) kann z. B. nur einem Status zugewiesen werden, der zu der Statuscodegruppe In Bereitschaft (Stand By) gehört. Auf diese Weise kann der Übergang des Ressourcenstatus auf Grundlage des definierten Modells eingeschränkt werden.

Beim Definieren eines Ressourcenstatusmodells

Sie müssen zunächst eine oder mehrere Ressourcenstatuscodegruppen definieren, bevor Sie ein Ressourcenstatusmodell definieren können. Das Ressourcenstatusmodell enthält die erforderliche Modellierungsdefinition, die Ressourcenstatuscodegruppe.

Das Ressourcenstatusmodell enthält folgende optionalen Modellierungsdefinitionen:

• Behälterstatuscode
• Ressourcenstatuscode
Hinweis: Bei der Definition von Ressourcenstatusmodellen sind in der Anwendung keine Statuscodes erforderlich. Es muss jedoch mindestens eine Ressource oder ein Behälter definiert werden, bevor Sie im Feld "Von-Statuscode" im Ressourcenstatusdiagramm eine Wert auswählen können.

"Ressourcenstatusmodell" ist ein optionales Feld in diesen Modellierungsdefinitionen:

• Behälter
• Ressource
• Ressourcentyp

Vorgehensweise zum Definieren eines Ressourcenstatusmodells

Gehen Sie folgendermaßen vor, um ein Ressourcenstatusmodell zu definieren:

1. Öffnen Sie die Seite Ressourcenstatusmodell. Die Seite "Ressourcenstatusmodell" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für das Modell in das Feld Ressourcenstatusmodell ein.
4. Wählen Sie eine Anfängliche Statuscodegruppe aus.
5. Klicken Sie auf Neue Zeile hinzufügen in der Tabelle Ressourcenstatusdiagramm, um eine leere Zeile hinzuzufügen.
6. Wählen Sie einen Von-Statuscode aus.
7. Wählen Sie eine Zu-Statuscodegruppe aus.
8. Wiederholen Sie die Schritte 5 bis 7, um weitere Zuweisungen hinzuzufügen.
9. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
10. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
