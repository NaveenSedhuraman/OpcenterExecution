Definieren von Trainingsanforderungen

Die Trainingsanforderungen werden zusammen mit den Trainingseinträgen des Benutzers geprüft, wenn der Benutzer eine Fertigungstransaktion startet. In den Trainingsanforderungen ist der Trainings- und Zertifizierungsprozess für Bediener genau definiert.

Wichtig: Eine Trainingseintrag referenziert eine bestimmte Revision einer Trainingsanforderung – nicht die Standardrevision.

Beim Definieren von Trainingsanforderungen

Gewährleisten Sie, dass Sie die folgenden Modellierungsobjekte definiert haben, bevor Sie die Trainingsanforderungen definieren:

• Die als Trainer vorgesehenen Mitarbeiter
• Dokumentsätze, die sich auf eine oder mehrere externe Dateien beziehen

Zur Definition der Trainingsanforderungen müssen Sie folgendermaßen vorgehen:

• Geben Sie Ablaufdaten oder -Zeiten an, damit bei der Überprüfung der Anforderungen festgestellt werden kann, ob sie noch gültig sind.
• Weisen Sie dieser Anforderung Trainer zu.
• Verknüpfen Sie die Anforderung mit einer externen Datei, die, falls erforderlich, referenziert werden kann.

Beachten Sie Folgendes:

• Eine Trainingsanforderung enthält die optionale Modellierungsdefinition "Dokument" und die erforderliche Modellierungsdefinition "Mitarbeiter". (Mitarbeiter werden als Trainer hinzugefügt.)
• Eine Trainingsanforderung wird für die Modellierungsdefinition "Trainingsanforderungsgruppe" und beim Hinzufügen eines Trainingseintrags in der Fertigung benötigt.
• In den folgenden Modellierungsdefinitionen ist die Trainingsanforderung optional:
• Mitarbeiter, bei der Zuordnung eines Trainingsplans
• Trainingsplan

Bei Verwendung einer Trainingsanforderung

Wenn Sie ein Datum auswählen, hängt das Format des Datums von der in Portal Studio oder Ihrem Browser festgelegten Spracheinstellung ab.

So wird eine Trainingsanforderung definiert

Gehen Sie folgendermaßen vor, um eine Trainingsanforderung zu definieren:

1. Öffnen Sie die Seite Trainingsanforderung. Die Seite "Trainingsanforderung" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Anforderung in das Feld Trainingsanforderung ein.
4. Geben Sie die Revision dieser Anforderung in das Feld Revision ein.
5. Klicken Sie in der Tabelle "Trainer" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
6. Wählen Sie einen Trainer aus.
7. Wiederholen Sie die Schritte 5 bis 6, um zusätzliche Trainer hinzuzufügen.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
