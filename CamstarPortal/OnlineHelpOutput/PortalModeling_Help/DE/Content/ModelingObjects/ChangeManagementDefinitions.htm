Definitionen zur Verwaltung von Änderungen

Change Management ist eine Anwendung zur Änderungsverwaltung in Fertigungsprozessen, die Herstellern dabei hilft, den Lebenszyklus von Änderungen des Produkts und der Prozessdefinitionen zu verwalten. Ein Unternehmen kann z. B. Instanzen von Modellierungsobjekten von seinem Testsystem ins Produktionssystem übertragen.

Sie können Geschäftsprozesse zur Änderungsverwaltung in Ihrem Informationsmodell definieren, um diese Anwendung für Ihre Änderungsverwaltungs-Transaktionen zu verwenden. Informationen zum Durchführen von Transaktionen zur Änderungsverwaltung finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.

Die folgenden Kapitel enthalten Informationen zu Definitionen zur Verwaltung von Änderungen:

• Definieren von Serverkatalogsystemen
• Definieren von Einstellungen für die Änderungsverwaltung
• Definieren von Änderungsverwaltungsspezifikationen
• Definieren von Arbeitsplänen zur Änderungsverwaltung
• Definieren von Nutzercodes für die Änderungsverwaltung
• Definieren eines Stammdatenkatalogs
• Definieren von Projektmitarbeitervorlagen
• Definieren von Paketerstellungsvorlagen
