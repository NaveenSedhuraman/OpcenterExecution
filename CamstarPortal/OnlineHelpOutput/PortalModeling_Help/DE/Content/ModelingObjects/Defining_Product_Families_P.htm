Definieren von Produktfamilien

Eine Produktfamilie ist eine Gruppe von Produkten, die gemeinsame Attribute haben, etwa Arbeitsplan, Trainingsanforderungen und Startmengen. Die für eine Produktfamilie definierten Attribute gelten für jedes Produkt der Produktfamilie.

Hinweis: Die in der Produktdefinition festgelegten Attribute überschreiben die Attribute, die in der Produktfamilien-Definition des Produkts definiert wurden.

Produktfamiliengruppen vereinfachen die Pflege der Produktinformationen, da sie gemeinsame Daten für eine Gruppe von Produkten festlegen, statt dass Daten für jedes Produkt separat definiert werden. Jedes Produkt kann zu einer Produktfamilie gehören, aber ein Produkt kann immer nur zu einer Produktfamilie gehören. Sie können ein Produkt einer Produktfamilie zuweisen, wenn Sie das Produkt definieren. Weitere Informationen zum Zuweisen eines Produkts zu einer Produktfamilie finden Sie unter "Definieren von Produkten".

Eine Produktfamiliengruppe ist nicht dasselbe wie eine Objektgruppe. Die Gruppierung von Objekten wie Ressourcen und Nutzercodes erfolgt zur Überprüfung und zur Erstellung von Berichten. Darüber hinaus kann ein Objekt mehreren Gruppen angehören und Objektgruppen können andere Objektgruppen enthalten.

Vorrangregeln

Diese Vorrangregeln werden für Produkt- und Produktfamiliendaten verwendet:

• Die Daten, die für eine Produktfamilie festgelegt werden, gelten für alle Produkte der Familie.
• Änderungen an Daten einer Produktfamilie gelten für alle Produkte dieser Familie.
• Daten, die auf der Produktebene festgelegt werden, haben Vorrang vor Daten der Produktfamilie.

Beim Definieren einer Produktfamilie

"Produktfamilie" enthält die folgenden optionalen Modellierungsdefinitionen:

• Dokumentsatz
• Trainingsanforderungsgruppe
• Maßeinheit
• Arbeitsplan

"Produktfamilie" ist ein optionales Feld in der Produktdefinition.

Zuweisen eines Standardlagerorts für eine Produktfamilie
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Sie können einer Produktfamilie einen Standardlagerort zuweisen. Sie können auch einem Produkt und einer Fabrik einen Standardlagerort zuweisen.

Die folgenden Regeln gelten beim Zuweisen von Standardlagerorten:

• Der Standardlagerort, der für das Produkt zugewiesen ist, hat Vorrang vor dem Standardlagerort, der für die Produktfamilie zugewiesen ist.
• Der Standardlagerort, der für die Produktfamilie zugewiesen ist, hat Vorrang vor dem Standardlagerort, der für die Fabrik zugewiesen ist.
• Die Anwendung weist einem Container den Standardlagerort automatisch zu, wenn der Container in eine Operation bewegt wird, die als Lagerort definiert ist, sofern für die Produktfamilie ein Standardlagerort definiert wurde.

Weitere Informationen finden Sie unter "Zuweisen eines Standardlagerorts für ein Produkt" und "Zuweisen eines Standardlagerorts für eine Fabrik".

Zuweisen einer Waagengruppe für die Produktfamilie
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Sie können eine Waagengruppe für die Produktfamilie zuweisen. Sie können eine Waagengruppe auch für Spezifikation, Produkt und Verbrauchswiegeaufgabe zuweisen.

Die folgenden Regeln gelten beim Zuweisen von Waagengruppen:

• Die für die Produktfamilie zugewiesene Waagengruppe hat Vorrang vor einer Waagengruppe, die der Spezifikation zugewiesen ist.
• Die für das Produkt zugewiesene Waagengruppe hat Vorrang vor einer Waagengruppe, die der Produktfamilie zugewiesen ist.
• Die für die Verbrauchswiegeaufgabe zugewiesene Waagengruppe hat Vorrang vor der Waagengruppe, die allen anderen Objekten zugewiesen ist.

Weitere Informationen finden Sie unter "Zuweisen einer Waagengruppe für die Spezifikation", "Zuweisen einer Waagengruppe zu einem Komponentenprodukt" und "Beim Definieren von Verbrauchswiegeaufgaben".

Definieren der Maßeinheitenkonvertierungsinformationen für die Produktfamilie
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Die Tabelle "Maßeinheitenkonvertierung" enthält Felder, mit denen Sie die Mengenkonvertierungsinformationen für Produktfamilien definieren können. Sie können Folgendes definieren:

• Ein Faktor für die Konvertierung einer Verbrauchsmenge der Produktfamilie von einer Maßeinheit in eine andere.
• Eine Regel für die Rundung von konvertierten Mengen.
• Die Anzahl der erlaubten Dezimalstellen für die berechnete konvertierte Menge. Der Maximalwert ist 4.

Die folgenden Regeln gelten beim Definieren von Maßeinheitsumrechnungsinformationen für Produktfamilien:

• Damit eine Rundung stattfindet müssen die Felder "Rundungsregel" und "Dezimalstellen" ausgefüllt werden.
• Konvertierungsinformationen, die für das Produkt definiert sind, haben Vorrang vor Konvertierungsinformationen, die für die Produktfamilie definiert sind.
• Konvertierungsinformationen, die für die Produktfamilie definiert sind, haben Vorrang vor Konvertierungsinformationen, die für die Maßeinheit definiert sind.

Weitere Informationen finden Sie unter "Definieren der Maßeinheitenkonvertierungsinformationen für das Produkt" und "Arbeitsbereich "Medical Device"".

Vorgehensweise zum Definieren einer Produktfamilie

Gehen Sie folgendermaßen vor, um eine Produktfamilie zu definieren:

1. Öffnen Sie die Seite Produktfamilie. Die Seite "Produktfamilie" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Produktfamilie in das Feld Produktfamilie ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
