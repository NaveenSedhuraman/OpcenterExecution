Definieren von Containerlevels

Ein Containerlevel ist die Nachverfolgungskennzeichnung, die einem Container zugeordnet wird. Sie wird auch als Durchsatzlevel bezeichnet. Häufig verwendete Levelnamen sind "Batch" und "Los". Levels werden den Anforderungen jedes Unternehmens entsprechend definiert. Beispiele für industriespezifische Levels sind etwa "Rolle", "Behälter" und "Kassette".

Opcenter EX MDD und Opcenter EX CR bieten Funktionen für die mehrstufige Nachverfolgung des Umlaufbestands (Work in Process, WIP), mit denen Sie Statusgenealogie- und parametrische Informationen in jeder Detailtiefe sammeln und melden können. Opcenter EX MDD und Opcenter EX CR verwenden den generischen Begriffscontainer, um eine Arbeitseinheit zu bestimmen. Sie können die spezielle Terminologie festlegen, die Ihren Operationen entspricht; z. B. "Los", "Batch", "Tablett", "Tank" oder "Seriennummer". Da die Anwendung unbegrenzt mehrere Containerlevel, das heißt, Container innerhalb von Containern unterstützt, können Sie Produkte beliebig detailliert definieren und nachverfolgen.

Viele Unternehmen verfolgen auch kleinere Level innerhalb des größeren Eltern-Containers nach. In Opcenter EX MDD und Opcenter EX CR wird dies über eine mehrstufige Containerverfolgung erreicht. Eltern-Level- und Kind-Level-Container sind durch die Transaktion "Verbinden" oder "Starten - zwei Level" miteinander verbunden. "Starten - zwei Level" erstellt die Container und "Verbinden" legt die Eltern-Kind-Beziehung fest. Eine Gruppe von Containern kann einen Eltern-Container auf dem Batchlevel und zugeordnete Kind-Container auf Fachleveln (Tablettlevel) haben. Containergruppen können mehr als zwei Levels haben.

Darüber hinaus können Sie einem Containerlevel benutzerdefinierte, dynamische Attribute zuweisen. Wenn Sie Informationen am Container selbst speichern, ist es nicht erforderlich, dass Sie oder andere Benutzer die Historie des Containers nach Informationen durchsuchen. Die Anwendung ordnet die benutzerdefinierten Attribute zu, wenn Sie einen Container starten.

Angeben von auszuwertenden Umlaufbestandsnachrichten

Umlaufbestandsnachrichten (WIP-Nachrichten) können für bestimmte Modellierungsobjekte definiert werden, sodass sie dem Benutzer angezeigt werden, wenn ein Container mit seinem Material mit speziellen Attributen einen bestimmten Bearbeitungspunkt erreicht. Sie können z. B. definieren, dass eine Umlaufbestandsnachricht angezeigt wird, wenn ein Container die Spezifikation "Start" erreicht und ein Benutzer eine "Anmelden"-Transaktion ausführt.

Um Umlaufbestandsnachrichten zu konfigurieren, müssen Sie Folgendes tun:

• Definieren Sie die Umlaufbestandsnachricht im Modellierungsobjekt. Weitere Informationen zum Definieren und Konfigurieren von Umlaufbestandsnachrichten finden Sie unter "Definieren von Umlaufbestandsnachrichten".
• Fügen Sie das Modellierungsobjekt, für das die Nachricht ausgewertet werden soll, dem Modellierungsobjekt "Containerlevel" hinzu, genauer gesagt der Tabelle "Umlaufbestandsnachricht-Bewertungseingaben".
Hinweis: Die Reihenfolge, in der Sie diese Aufgaben ausführen, spielt keine Rolle. Sie können die Umlaufbestandsnachricht zuerst definieren oder zuerst das Modellierungsobjekt zum Containerlevel-Modellierungsobjekt hinzufügen.

Im Folgenden sind die gültigen Werte für die Tabelle "Umlaufbestandsnachricht-Bewertungseingaben" aufgeführt:

• BOM für Bill of Materials (Stückliste)
• CurrentStatus.
ReworkStatus.ReworkReason für den Nacharbeitsgrund
• Customer
• HoldReason für den Anhaltegrund
• Level für den Containerlevel
• MfgOrder für Produktionsauftrag
• Operation
• Operation.WorkCenter für Arbeitsbereich
• Owner
• Priority für Prioritätscode
• Product
• Product.ERPBOM für ERP-Stückliste
• Product.ProductFamily für Produktfamilie
• Product.ProductTyp für Produkttyp
• Resource
• SalesOrder für Verkaufsauftrag
• ShippingReason für den Versandgrund
• Spec
• Spec.Setup für Einrichtung
• Spec.TxnMap.DataCollectionDef für Datensammlungsdefinition
• StartReason für Startgrund
• UOM
• Workflow

Sie müssen Level zur Tabelle "Umlaufbestandsnachricht-Bewertungseingaben" für alle Containertransaktionen hinzufügen, für die Sie Umlaufbestandsnachrichten konfigurieren möchten.

Ausschließen der Umlaufbestandsnachrichtenauswertung aus bestimmten Operationen

Wenn Sie die Auswertung einer Umlaufbestandsnachricht konfigurieren, können Sie eine oder mehrere Operationen festlegen, die Sie aus der Umlaufbestandsnachrichtenauswertung ausschließen möchten. Wenn Sie z. B. eine Umlaufbestandsnachricht für das Produkt Widget in der "Abmelden"-Transaktion konfiguriert haben, können Sie die Operation "Verpackung" ausschließen. Somit wird die Umlaufbestandsnachricht nicht angezeigt, wenn der Benutzer die "Abmelden"-Transaktion für einen Container des Produkts Widget bei der Verpackungsoperation ausführt.

Grundlegendes zu benutzerdefinierten Attributen

Benutzerdefinierte Attribute sind ein dynamischer Satz von Eigenschaften, die es Ihnen ermöglichen, Informationen zu einem Container leicht zugänglich zu speichern. Sie könnten z. B. Laborergebnisse zu dem Container selbst speichern, um nicht die Containerhistorie durchsuchen zu müssen. Ein Attribut kann etwas Einfaches sein, z. B. die Farbe des Containers. Benutzerdefinierte Attribute können für jeden Container einzigartig sein und erfordern kein Aktualisieren der Datenbank, wenn sie zugewiesen werden.

Zuweisen von benutzerdefinierten Attributen beim Containerstart

Wenn Sie einen Container starten, kopiert die Anwendung die definierten Attribute aus folgenden Stellen in den Container:

• Containerlevel
• Produkt
• Produktionsauftrag

Die Anwendung kopiert zuerst aus dem Containerlevel, anschließend aus dem Produkt und dann aus dem Produktionsauftrag. Wenn die Liste Attribute mit demselben Namen beinhaltet, überschreibt die Anwendung die vorhandenen Attributwerte mit den neuen Werten, solange der Datentyp derselbe ist. Wenn die Datentypen nicht dieselben sind, zeigt die Anwendung eine Fehlermeldung an.

Beispiel:

Angenommen, die folgenden Objekte haben diese zugewiesenen Attribute:

Containerlevel

Produkt

Produktionsauftrag

Modell = AAA

Farbe = Rot

Jahr = 2015

Farbe = Blau

Die Anwendung verarbeitet zuerst "Containerlevel", also enthält die Attributliste:

• Modell = AAA

Die Anwendung verarbeitet danach "Produkt". "Produkt" hat zwei neue Attribute, also fügt die Anwendung diese der Attributliste hinzu:

• Modell = AAA
• Farbe = Rot
• Jahr = 2015

Die Anwendung verarbeitet zuletzt "Produktionsauftrag". "Produktionsauftrag" enthält ein Attribut mit demselben Namen wie ein Attribut des Containers, genauer gesagt das Attribut "Farbe". Die Anwendung überschreibt den vorhandenen Wert des Attributs "Farbe" durch den Wert des Attributs "Farbe" von "Produktionsauftrag":

• Modell = AAA
• Farbe = Blau
• Jahr = 2015

Die Anwendung speichert die fertige Attributliste des Containers als Teil der Starthistorie des Containers. Sie können im Produktionsprotokoll die Attribute anzeigen, die bei einem Containerstart zugewiesen wurden. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Audit Trail-Benutzerhandbuch und im Opcenter Execution Core Audit Trail-Benutzerhandbuch.

Beim Definieren eines Containerlevels

"Container" ist:

• Ein erforderliches Feld in der Definition von "Operation" (als "Durchsatzberichtsebene" eingegeben), in der Transaktion "Start" und in der Transaktion "Starten - zwei Level".
• Ein optionales Feld in der Transaktion "Containerwartung".

Vorgehensweise zum Definieren eines Containerlevels

Gehen Sie folgendermaßen vor, um einen Containerlevel zu definieren:

1. Öffnen Sie die Seite Containerlevel. Die Seite "Containerlevel" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diesen Containerlevel in das Feld Containerlevel ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Weitere Informationen zum Hinzufügen eines Eltern- oder Kind-Containerlevels finden Sie unter "So wird ein Eltern- oder Kind-Containerlevel hinzugefügt".
6. Weitere Informationen zum Hinzufügen von Suchkriterien für Umlaufbestandsnachrichten finden Sie unter "So werden Suchkriterien für Umlaufbestandsnachrichten einem Containerlevel hinzugefügt".
7. Weitere Informationen zum Hinzufügen benutzerdefinierter Attribute finden Sie unter "So werden benutzerdefinierte Attribute einem Containerlevel hinzugefügt".
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So wird ein Eltern- oder Kind-Containerlevel hinzugefügt

Gehen Sie folgendermaßen vor, um ein Eltern- oder Kind-Containerlevel hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Containerlevels" aus.

Oder

Wählen Sie eine vorhandene Containerlevel-Instanz aus.

2. Klicken Sie in der Tabelle "Eltern-Level" oder der Tabelle "Kind-Level" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Klicken Sie in die Zeile, um eine Liste von Leveln anzuzeigen.
4. Wählen Sie einen Level aus.
5. Wiederholen Sie die Schritte 2 bis 4, um weitere Levels hinzuzufügen.
6. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

So werden Suchkriterien für Umlaufbestandsnachrichten einem Containerlevel hinzugefügt

Gehen Sie folgendermaßen vor, um Suchkriterien für Umlaufbestandsnachrichten einem Containerlevel hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Containerlevels" aus.

Oder

Wählen Sie eine vorhandene Containerlevel-Instanz aus.

2. Erweitern Sie den Abschnitt Umlaufbestandsnachrichten-Konfiguration.
3. Klicken Sie in der Tabelle "Umlaufbestandsnachricht-Bewertungseingaben" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Geben Sie eine Beschriftung für die Umlaufbestandsnachricht ein.
5. Klicken Sie in der Tabelle "Ausgeschlossene Operationen" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
6. Klicken Sie in die Zeile, um eine Liste von Operationen anzuzeigen.
7. Klicken Sie auf die Operation, für die "Umlaufbestandsnachricht" nicht einbezogen werden soll.
8. Wiederholen Sie die Schritte 3 bis 7, um weitere Kriterien hinzuzufügen.
9. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

So werden benutzerdefinierte Attribute einem Containerlevel hinzugefügt

Gehen Sie folgendermaßen vor, um einem Containerlevel benutzerdefinierte Attribute hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Containerlevels" aus.

Oder

7. Aktivieren Sie das Kontrollkästchen Ist Ausdruck, wenn der Wert ein Ausdruck ist, um sicherzustellen, dass der Wert korrekt interpretiert wird.
8. Wiederholen Sie die Schritte 3 bis 7, um weitere Attribute hinzuzufügen.
9. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
