Definieren von Nummerierungsregeln

Eine Nummerierungsregel ist ein konfigurierbares Nummerierungsschema, das Qualitätsdatensätzen und Containern eindeutige Nachverfolgungsnummern zuweist. In der Regel definiert eine Organisation ein Nummerierungsschema und weist das Schema diesen Objekten zu. Die Anwendung wendet das entsprechende Nummerierungsschema an und weist den entsprechenden Qualitätsdatensatz oder Container die nächste verfügbare ID oder Nummer zu.

Beim Definieren von Nummerierungsregeln für Qualitätsdatensätze

Siemens empfiehlt, für jeden Qualitätsdatensatztyp eine eindeutige Nummerierungsregel zu definieren, damit jeder Datensatztyp (Ereignis oder Abweichung) einen eindeutigen Identifikator aufweist. Die Anwendung verwendet die im Modellierungsobjekt "Organisation" festgelegten Nummerierungsregeln, um Qualitätsdatensätzen Identifikatoren zuzuweisen.

"Nummerierungsregel" ist ein optionales Feld in der Modellierungsdefinition "Organisation".

Informationen zum Aufzeichnen von Ereignissen und Abweichungen finden Sie im Opcenter Execution Medical Device and Diagnostics Event Management-Benutzerhandbuch und im Opcenter Execution Core Event Management-Benutzerhandbuch sowie im Opcenter Execution Medical Device and Diagnostics Nonconformance Management-Benutzerhandbuch und im Opcenter Execution Core Nonconformance Management-Benutzerhandbuch.

Beim Definieren von Nummerierungsregeln für Container

Jeder Container muss einen eindeutigen Containernamen aufweisen. Containernamen können manuell oder automatisch über Nummerierungsregeln erstellt werden, die Modellierungsobjekten zugewiesen sind. Sie erstellen eine Nummerierungsregel mithilfe der Modellierungsseite "Nummerierungsregel". Sie weisen eine Nummerierungsregel zu, indem Sie sie aus dem optionalen Feld "Nummerierungsregel" für eines oder mehrere dieser Objekte auswählen:

• Containerlevel
• Produktionsauftrag
• Produkt
• Produktfamilie
• Fabrik
Hinweis: Die obige Liste gibt die Reihenfolge der Auswahlvoreinstellungen an. Die Anwendung wählt z. B. die Nummerierungsregel auf Containerebene aus, wenn sowohl die Containerebene als auch das Produkt mit einer Nummerierungsregel konfiguriert sind.

Benutzer in der Fertigung haben die Möglichkeit, beim Starten von Eltern- und Kind-Containern, Aufteilen von Containern und Aufteilen von Mengen automatisch Containernamen zu generieren. Informationen zum automatischen Generieren von Containernamen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Beim Auswählen eines Nummerierungsregeltyps

Sie müssen einen Nummerierungsregeltyp auswählen, wenn Sie eine Nummerierungsregel definieren. Es gibt zwei Typen und beide haben ihre Vor- und Nachteile. Die zwei Typen von Nummerierungsregeln sind:

• Hohes Volumen / Überspringen
• Geringes Volumen / Nicht überspringen

Sie können den Typ der jeweiligen Nummerierungsregel nicht mehr ändern, nachdem Sie die Instanz gespeichert haben. Die Anwendung deaktiviert das Feld "Typ der Nummerierungsregel", wenn Sie auf die Schaltfläche "Speichern" klicken. Das Feld "Typ der Nummerierungsregel" bleibt deaktiviert, wenn Sie durch Klicken auf die Schaltfläche "Kopieren" eine neue Instanz erstellen.

Merkmale von "Hohes Volumen / Überspringen"

Verwenden Sie den Nummerierungsregeltyp "Hohes Volumen/Überspringen", wenn Sie Nummerierungsregeln für häufige ("Hohes Volumen") Elemente erstellen, bei denen striktes Einhalten von sequenziellen Nummern keine Wichtigkeit hat.

Vorteile

Die Vorteile des Nummerierungsregeltyps "Hohes Volumen/Überspringen" umfassen:

• Er ermöglicht das Überspringen von Sequenznummern.
• Er ist für häufige Zugriffe auf die Nummerierungsregel vorgesehen.
• Die Möglichkeit von Zugriffskollisionen ist minimiert.
• Die letzte zugeordnete Sequenznummer kann geändert werden.

Nachteile

Die Nachteile des Nummerierungsregeltyps "Hohes Volumen/Überspringen" umfassen:

• Er kann keine sequenziellen Sequenznummern gewährleisten.
• Er sollte nicht verwendet werden, wenn sequenzielle Sequenznummern erforderlich sind.

Empfohlene Verwendung

Der Nummerierungsregeltyp "Hohes Volumen/Überspringen" eignet sich am besten für häufige ("Hohes Volumen") Anforderungen für Nummern und wird für die automatische Nummerierung von Materialcontainern empfohlen.

Merkmale von "Geringes Volumen / Nicht überspringen"

Verwenden Sie den Nummerierungsregeltyp "Geringes Volumen/Nicht überspringen", wenn Sie Nummerierungsregeln für seltene ("Geringes Volumen") Elemente erstellen, bei denen striktes Einhalten von sequenziellen Nummern unverzichtbar ist.

Vorteile

Die Vorteile des Nummerierungsregeltyps "Geringes Volumen/Nicht überspringen" umfassen:

• Er verhindert das Überspringen von Sequenznummern.
• Die letzte zugeordnete Sequenznummer kann geändert werden.

Nachteile

Die Nachteile des Nummerierungsregeltyps "Geringes Volumen/Nicht überspringen" umfassen:

• Es besteht die Möglichkeit von Zugriffskollisionen.
• Er wird nicht für häufige Anforderungen für automatische Nummern aus der Nummerierungsregel empfohlen.

Empfohlene Verwendung

Der Nummerierungsregeltyp "Geringes Volumen/Nicht überspringen" eignet sich am besten für seltene ("Geringes Volumen") Anforderungen für automatische Nummern und wird für Qualitätsdatensätze (Ereignisse) empfohlen.

Anforderungen an das Präfix- und Suffixformat

Nummern dürfen maximal 30 Zeichen lang sein, einschließlich des jeweiligen Präfixes (Vorsilbe) und Suffixes (Nachsilbe) sowie der Sequenznummer. Die generierte Nummer wird folgendermaßen berechnet:

Nummer = Präfix (falls vorhanden) + Sequenznummer (erforderlich) + Suffix (falls vorhanden)

Z. B.: EREIGNIS-ABT15-00000001-2014

Die Felder "Vorsilbe" und "Nachsilbe" können entweder für literale Einträge oder für Einträge mit Syntax für einheitliche Ausdrücke verwendet werden.

Beide erfordern sehr spezielle Formate, z. B.:

• Ein literales Präfix kann z. B. als "ABCUnt_" oder "_Schmidt" definiert sein. Der jeweilige Eintrag muss in doppelten Anführungszeichen stehen.
• Die Syntax für einheitliche Ausdrücke kann genutzt werden, um ein dynamisches Präfix oder Suffix zu erzeugen, etwa:
• Präfix = "(<Organisation.Name>"
• Suffix ="_"+StrSubstring(String(TruncateTime(GMTToLocal(SetSystemDateGMT()))),(StrLength(String(TruncateTime(GMTToLocal(SetSystemDateGMT()))))-4),4)

Dies führt zu folgendem Schema: Ereignis (Corporate00001_2014 am 02/06/2014 3:21:00 PM gestartet).

Weitere Informationen zur Syntax für einheitliche Ausdrücke (Unified Expressions) finden Sie im Opcenter Execution Medical Device und Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Regelbasierte vs. präfixbasierte Sequenzen

Alle Nummerierungsregeln enthalten eine Sequenz. Standardmäßig erhöht die Anwendung den zuletzt zugewiesenen Sequenzwert einer Regel immer dann, wenn diese Regel zum Erzeugen eines neuen Werts verwendet wird. Sie können diese regelbasierte Sequenzierung mit präfixbasierter Sequenzierung außer Kraft setzen, indem Sie das Kontrollkästchen "Präfix basierend auf zuletzt zugewiesener Reihenfolge verwenden" auf der Modellierungsseite "Nummerierungsregel" aktivieren.

Die präfixbasierte Sequenzierung ist für Nummerierungsregeln sinnvoll, die mit einheitlichen Ausdrücken konfiguriert wurden, die Präfixe dynamisch erzeugen. Die Anwendung weist immer dann einen Anfangssequenzwert zu, wenn eine Nummerierungsregel zum ersten Mal ein eindeutiges Präfix erzeugt. Wenn eine Nummerierungsregel später dasselbe Präfix erzeugt, erhöht die Anwendung beim Erzeugen des Containernamens die zuletzt zugewiesene Sequenz für dieses Präfix.

Wichtig: Eine beliebige Nummerierungsregel könnte theoretisch dasselbe Präfix erzeugen wie eine andere Nummerierungsregel. Die Anwendung erhöht eine präfixbasierte Sequenz unabhängig von der Regel, durch die sie erzeugt wurde. Dadurch kann eine Regel für mehrere Produktionsaufträge, Produkte usw. konfiguriert werden, wobei ein einheitlicher Ausdruck Informationen wie Produktname und Produktionsauftragsname im Präfix dynamisch identifiziert.

Vorgehensweise zum Definieren einer Nummerierungsregel mithilfe der Standard- oder regelbasierten Sequenzierung

Gehen Sie folgendermaßen vor, um eine Nummerierungsregel mithilfe der regelbasierten Standardsequenzierung zu definieren:

1. Öffnen Sie die Seite Nummerierungsregel. Die Seite "Nummerierungsregel" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Nummerierungsregel in das Feld Nummerierungsregel ein.
4. Wählen Sie einen Typ der Nummerierungsregel aus.
5. Geben Sie einen Wert in das Feld Letzte zugeordnete Sequenz ein.
6. Geben Sie einen Wert in das Feld Sequenzlänge ein.
7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Definieren einer Nummerierungsregel mithilfe der präfixbasierten Sequenzierung

Gehen Sie folgendermaßen vor, um eine Nummerierungsregel mithilfe der präfixbasierten Sequenzierung zu definieren:

1. Öffnen Sie die Seite Nummerierungsregel. Die Seite "Nummerierungsregel" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Nummerierungsregel in das Feld Nummerierungsregel ein.
4. Wählen Sie einen Typ der Nummerierungsregel aus.
5. Geben Sie einen vereinheitlichten Ausdruck in das Feld Präfix ein.
6. Aktivieren Sie das Kontrollkästchen Zugewiesene präfixbasierte Sequenz verwenden. Die Anwendung legt "Präfix" als erforderliches Feld fest und deaktiviert das Feld "Letzte zugeordnete Sequenz".
7. Geben Sie einen Wert in das Feld Sequenzlänge ein.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
