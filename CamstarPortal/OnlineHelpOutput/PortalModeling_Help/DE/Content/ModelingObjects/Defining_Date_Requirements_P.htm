Definieren von Datumsanforderungen

Eine Datumsanforderung (Terminanforderung) definiert den Wartungsplan einer Ressource auf Grundlage eines einzelnen Kalenderdatums. Die Anforderungsdefinition enthält auch die Toleranz- und Warnbereiche (in Tagen) und die entsprechenden E-Mail-Verteilerlisten für E-Mail-Benachrichtigungen zu ausstehenden, fälligen und überfälligen Wartungsereignissen. Sie haben auch die Möglichkeit, einen Dokumentsatz und eine Datensammlungsdefinition mit der Anforderung zu verbinden.

Beim Definieren von Datumsanforderungen

Eine Datumsanforderung beinhaltet die erforderliche Modellierungsdefinition "Wartungsgrund" und folgende optionale Modellierungsdefinitionen: Dokumentsatz, Datensammlungsdefinition, E-Mail-Benachrichtigung, Anforderung für elektr. Unterschrift und Maßeinheit.

Löst die "Ausstehend"-E-Mail aus.

Toleranzfrist

Gibt an, ab dem wievielten Tag vor dem geplanten Datum die Wartungsanforderung als ausstehend für die entsprechende Ressource gilt.

Versendet die "Überfällig"-E-Mail und die Ressource steht für die weitere Bearbeitung nicht mehr zur Verfügung.

Wenn der Standardwert 0 zulässig ist, wird die "Überfällig"- E-Mail am geplanten Datum versandt. Die Ressource ist ab sofort nicht mehr für die weitere Verarbeitung verfügbar.

Optional

Anforderung zur elektronischen Unterschrift

Vordefinierte Anforderungen zur elektronischen Unterschrift. Eine Anforderung zur elektronischen Unterschrift definiert die elektronische Unterschrift, die für die Transaktion einzuholen ist. Wenn eine Anforderung zur elektronischen Unterschrift angegeben ist, können Sie die Transaktion erst durchführen, wenn alle Unterschriften vorliegen.

Benachrichtigungen

Tabelle "Ausstehend - E-Mail-Empfänger"

In dieser Tabelle werden die Empfänger angegeben, die eine E-Mail-Benachrichtigung über eine ausstehende Wartung erhalten. Weitere Informationen finden Sie im Abschnitt zum Feld "Warnungsfrist" unter "Definieren von Wiederholungsterminanforderungen".

Ausstehend - E-Mail-Text

Der Text der E-Mail, die versandt wird, wenn die Wartung ausstehend ist. Sie müssen einen Text eingeben, wenn Sie im Feld "Ausstehend - E-Mail-Empfänger" eine Angabe machen. Es können Textvariablen verwendet werden.

Tabelle "Fällig - E-Mail-Empfänger"

Fällig - E-Mail-Text

Der Text der E-Mail, die versandt wird, wenn die Wartung fällig ist (aber noch in der Toleranzfrist liegt). Sie müssen einen Text eingeben, wenn Sie im Feld "Fällig - E-Mail-Empfänger" eine Angabe machen. Es können Textvariablen verwendet werden.

Optional

Tabelle "Überfällig - E-Mail-Empfänger"

In dieser Tabelle werden die Empfänger angegeben, die eine E-Mail-Benachrichtigung erhalten, wenn die Wartung überfällig ist. Weitere Informationen finden Sie im Abschnitt zum Feld "Toleranzfrist" unter "Definieren von Wiederholungsterminanforderungen".

Optional

Überfällig - E-Mail-Text

Der Text der E-Mail, die versandt wird, wenn die Wartung überfällig ist. Sie müssen Text eingeben, wenn Sie im Feld "Überfällig - E-Mail-Empfänger" eine Angabe machen. Es können Textvariablen verwendet werden.

Vorgehensweise zum Definieren einer Datumsanforderung

Gehen Sie folgendermaßen vor, um eine Datumsanforderung zu definieren:

1. Öffnen Sie die Seite Datumsanforderung. Die Seite "Datumsanforderung" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Anforderung in das Feld Datumsanforderung ein und geben Sie den Revisionsidentifikator für diese Definition in das Feld Revision ein.
4. Erweitern Sie den Abschnitt Ablaufplanungsdetails.
5. Wählen Sie den Grund, der mit dieser Anforderung verknüpft ist, im Feld Wartungsgrund aus.
6. Geben Sie das Wartungsdatum für diese Anforderung in das Feld Geplantes Datum ein (oder wählen Sie es im Kalender aus).
7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden Datumsanforderungs-Benachrichtigungen hinzugefügt

Gehen Sie folgendermaßen vor, um eine Datumsanforderungs-Benachrichtigung hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Datumsanforderung" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Datumsanforderung aus.

2. Expandieren Sie den Abschnitt Benachrichtigungen.
3. Klicken Sie in der Tabelle "Ausstehend - E-Mail-Empfänger", "Fällig - E-Mail-Empfänger" oder "Überfällig - E-Mail-Empfänger" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie eine E-Mail-Benachrichtigung aus der Auswahlliste in der neu hinzugefügten Zeile aus. Die Benachrichtigung wird der Tabelle hinzugefügt.
5. Geben Sie den Text der E-Mail in das entsprechende Feld ein: Ausstehend - E-Mail-Text, Fällig - E-Mail-Text oder Überfällig - E-Mail-Text. Sie haben auch die Möglichkeit, Textvariablen zu verwenden.
6. Wiederholen Sie die Schritte 3 bis 5, um die Benachrichtigungen für weitere Empfänger einzurichten.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
