Definieren von Berichtsvorlagen

Eine Berichtsvorlage wird verwendet, um die RPT-Datei für Ihre Informationsberichte und -grafiken anzugeben. Informationen zu Diagrammen und Berichten finden Sie im Opcenter Execution Core Intelligence-Referenzhandbuch.

Beim Definieren von Berichtsvorlagen

Diese Regeln gelten, wenn Sie Berichte definieren:

• Jede Instanz einer Berichtsvorlage enthält das Feld "Speicherort der Datei", das zum Suchen einer zu speichernden oder hochzuladenden Datei dient.
• Über das Kontrollkästchen "Datei in die Datenbank hochladen" können Sie bestimmen, wann die angegebene Datei in die Datenbank hochgeladen werden soll. Normalerweise aktivieren Sie dieses Kontrollkästchen, wenn Sie eine Berichtsvorlage erstmals definieren. Wenn Sie das Kontrollkästchen bei der Aktualisierung vorhandener Instanzen leer lassen, können Sie Änderungen vornehmen, ohne die Datei hochzuladen.
• Die Anwendung aktiviert die Schaltfläche "Dokument anzeigen", nachdem eine Datei in die Datenbank hochgeladen und die Instanz der Berichtsvorlage gespeichert wurde. Klicken Sie auf "Dokument anzeigen", um das Dokument anzuzeigen oder herunterzuladen.
• Das Feld "Name der gespeicherten Datei" ist schreibgeschützt und zeigt den Dateinamen an, nachdem Sie auf "Speichern" geklickt haben.
• Die Zeit, die für das Herunterladen/Hochladen einer RPT-Datei benötigt wird, hängt von der Größe der Datei ab.

Vorgehensweise zum Definieren einer Berichtsvorlage

Gehen Sie folgendermaßen vor, um eine Berichtsvorlage zu definieren:

1. Öffnen Sie die Seite Berichtsvorlage. Die Seite "Berichtsvorlage" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Berichtsvorlage in das Feld Berichtsvorlage ein.
4. Geben Sie die Revision dieser Vorlage in das Feld Revision ein.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So wird eine Datei einer Berichtsvorlage hinzugefügt

Gehen Sie folgendermaßen vor, um einer Berichtsvorlage eine Datei hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Berichtsvorlage" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Berichtsvorlage aus.

2. Klicken Sie auf Durchsuchen neben dem Feld "Speicherort der Datei". Es wird ein Popup angezeigt, in dem Sie eine Datei suchen können.
3. Suchen Sie nach einer Datei und klicken Sie dann auf Öffnen. Das Popup wird geschlossen und der Dateiname wird im "Speicherort der Datei" angezeigt.
4. Aktivieren Sie Datei in die Datenbank hochladen.
5. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
