Definieren von Einstellungen für die Änderungsverwaltung

Mit den Einstellungen für die Änderungsverwaltung (Änderungsmanagementeinstellungen) können Sie verschiedene globale Einstellungen für die Änderungsverwaltung konfigurieren. Zu diesen Einstellungen zählen:

• Zu übergebende "Max. Anzahl 'Wo genutzt'-Instanzen" – Die maximale Anzahl von Modellierungsobjektinstanzen, die die Anwendung an die Funktion "Wo genutzt?" übergeben kann, wenn der Paketeigentümer oder Projektmitarbeiter einem Paket Inhalt zuweist.
• Paketerstellungsvorlage: Die zum Erstellen eines Pakets für ein Element und einen Änderungsstand verwendet wird, der von Teamcenter an Opcenter EX MDD oder Opcenter EX CR gesendet wurde.
• Benachrichtigungsereignisse – Benachrichtigungen, die an Mitarbeiter gesendet werden, wenn für ein Änderungsverwaltungspaket bestimmte Ereignisse auftreten. Zu diesen Ereignissen zählen:
• Änderungsmanagement: Genehmigungszuweisung – Benachrichtigt Mitarbeiter, die ein Änderungsverwaltungspaket genehmigen müssen, wenn das Paket zur Genehmigung weitergeleitet wird.
• Änderungsmanagement: Genehmigungsdelegierung – Benachrichtigt zugewiesene Mitarbeiter und Zielmitarbeiter darüber, dass eine Paketgenehmigungsaufgabe neu zugewiesen wurde.
• Änderungsmanagement: Zusammenarbeitszuweisung – Benachrichtigt Projektmitarbeiter darüber, dass sie einem Änderungsverwaltungspaket Inhalt zuweisen können.
• Änderungsmanagement: Zusammenarbeitsdelegierung – Benachrichtigt zugewiesene Mitarbeiter und Zielmitarbeiter darüber, dass eine Paketzusammenarbeitsaufgabe neu zugewiesen wurde.
• Änderungsmanagement: Eigentümer: Zusammenarbeitszuweisung beendet – Benachrichtigt Paketeigentümer, wenn Projektmitarbeiter die Zuweisung von Inhalt zu einem Paket beendet haben.
• Paketgenehmigungserinnerungen – Erinnerungsbenachrichtigungen, die an den Eigentümer, die Genehmiger oder die Eskalationsempfänger für Genehmiger verschickt werden, wenn für einen Paketgenehmiger bald fällige, fällige oder überfällige Genehmigungen vorliegen.
• Inhaltszusammenarbeitserinnerungen – Erinnerungsbenachrichtigungen, die an den Eigentümer, die Projektmitarbeiter oder die Eskalationsempfänger für Projektmitarbeiter verschickt werden, wenn für einen Inhaltsprojektmitabeiter bald fällige, fällige oder überfällige Zusammenarbeit vorliegt.
Hinweis: Informationen zu E-Mail-Benachrichtigungen für die Änderungsverwaltung finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.

Beim Definieren von Einstellungen für die Änderungsverwaltung

Der Standardwert für "Max. Anzahl 'Wo genutzt'-Instanzen" ist 1. Wenn Sie einen Wert kleiner als 1 eingeben, wird eine Warnmeldung angezeigt.

Hinweis: Ein höherer Wert für die maximale Anzahl von "Wo genutzt-Instanzen" beeinträchtigt die Leistung. Die empfohlene Anzahl ist 5. Die Anwendung akzeptiert Werte, die größer als 5 sind, dies kann jedoch die Leistung beeinträchtigen.

Opcenter EX MDD oder Opcenter EX CR verwendet die Paketerstellungsvorlage, wenn es ein Paket für ein Element und einen Änderungsstand erstellt, die von Teamcenter an Opcenter EX MDD oder Opcenter EX CR gesendet wurden. Das Paket enthält das Produkt und die Revision, die die Informationen der bedingten Prozessanweisungen (BOP) und Stücklisten (BOM) enthält. Beim Angeben einer Vorlage werden die vordefinierten Paketdetails, der Arbeitsplan für die Änderungsverwaltung und die Bereitstellungsdetails in die Seite "Paket erstellen" eingetragen, wenn Sie das Feld "Änderungsmanagementeinstellungen" für die Fabrik des Mitarbeiters, der das Paket startet, festlegen. Weitere Informationen finden Sie unter "Definieren von Paketerstellungsvorlagen" und "Definieren einer Fabrik".

Hinweis: Diese Funktionalität ist nur bei einer Integration mit Teamcenter verfügbar. Weitere Informationen finden Sie im Opcenter Execution Core Teamcenter Integration-Handbuch.

Bei der Definition von Einstellungen für die Änderungsverwaltung verlangt die Anwendung nicht, dass Sie eine E-Mail-Benachrichtigung hinzufügen. Sie müssen jedoch E-Mail-Benachrichtigungen konfigurieren, falls Benachrichtigungen an Mitarbeiter gesendet werden sollen, wenn Ereignisse für ein Paket auftreten. Beim Hinzufügen einer E-Mail-Benachrichtigung müssen Sie sowohl einen Benachrichtigungstyp als auch eine E-Mail-Nachricht auswählen. Sie definieren E-Mail-Nachrichten mithilfe des Modellierungsobjekts "E-Mail-Nachricht".

Der Benachrichtigungstyp für die Änderungsverwaltung kann nur in der Auswahlliste "Benachrichtigungstyp" auf der Seite "Änderungsmanagementeinstellungen" ausgewählt werden. Sie können diese Benachrichtigungstypen nicht in der Liste "Benachrichtigungstyp" auf der Seite "Organisation" auswählen.

Ein Systemadministrator kann die einem Mitarbeiter zugewiesene Aufgabe einem anderen Mitarbeiter zuweisen. Der Administrator kann z. B. eine Paketgenehmigung- oder Zusammenarbeitsaufgabe an einen anderen Mitarbeiter delegieren, wenn der Mitarbeiter, dem die Aufgabe ursprünglich zugewiesen wurde, zum Fälligkeitszeitpunkt der Aufgabe im Urlaub ist. Wählen Sie die Benachrichtigungstypen "Änderungsmanagement: Genehmigungsdelegierung" oder "Änderungsmanagement: Zusammenarbeitsdelegierung" und dann die entsprechenden E-Mail-Nachrichten aus, die an die zugewiesenen Mitarbeiter und Zielmitarbeiter gesendet werden sollen, wenn die Arbeit neu zugewiesen wird. Informationen zum Delegieren von Aufgaben finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

"Änderungsmanagementeinstellungen" ist ein optionales Feld, das Sie festlegen können, wenn Sie eine Instanz eines "Fabrik"-Modellierungsobjekts definieren. "Fabrik" ist ein optionales Feld bei der Definition von Mitarbeiterinstanzen. Deshalb können Sie einen Mitarbeiter mit Einstellungen für die Änderungsverwaltung verknüpfen, indem Sie das Feld "Fabrik" auf der Seite "Mitarbeiter" auf die Fabrik einstellen, die den Einstellungen für die Änderungsverwaltung zugeordnet ist.

Sie müssen den SMTP-Transport definieren, der für E-Mail-Benachrichtigungen verwendet werden soll. In der Instanz des "Fabrik"-Modellierungsobjekts muss der für E-Mail-Benachrichtigungen definierte SMTP-Transport angegeben sein. Gewährleisten Sie, dass in den Instanzen des "Mitarbeiter"-Modellierungsobjekts die Fabrik angegeben ist, der der SMTP-Transport zugeordnet ist. Informationen zu E-Mail-Benachrichtigungen für die Änderungsverwaltung finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.

Beim Definieren von Erinnerungsbenachrichtigungen

Sie können Erinnerungsbenachrichtigungen einrichten, um E-Mail-Nachrichten an bestimmte Mitarbeiter zu senden, wenn für einen Paketgenehmiger oder Inhaltsprojektmitarbeiter bald fällige, fällige oder überfällige Arbeit vorliegt. Die Anwendung versendet Erinnerungsbenachrichtigungen für Genehmigungs- und Zusammenarbeitsaufgaben auf der Grundlage von deren Fälligkeitsdatum.

Das Fälligkeitsdatum wird durch das Zuweisungsdatum, die Dauer und den Zeitraum bestimmt, die der Paketeigentümer angegeben hat, als die Arbeit zugewiesen wurde. Von den Genehmigern und Projektmitarbeitern wird erwartet, dass sie die Arbeit im festgelegten Zeitrahmen abschließen. Das Fälligkeitsdatum wird durch die Dauer und den Zeitraum bestimmt, wenn der Paketeigentümer diese Felder beim Zuweisen der Arbeit festlegt. Das Fälligkeitsdatum entspricht dem Datum und Zeitpunkt der Zuweisung, wenn der Paketeigentümer Dauer und Zeitraum nicht festlegt. Eine Aufgabe ist sofort bei ihrer Zuweisung fällig, wenn Dauer und Zeitraum nicht festgelegt werden.

Sie müssen die E-Mail-Nachricht, den Empfänger und die Tage (im Verhältnis zum Fälligkeitsdatum) angeben, an denen die Anwendung die Nachrichten versenden soll.

Sie definieren E-Mail-Nachrichten mithilfe des Modellierungsobjekts "E-Mail-Nachricht". Sie können das Modellierungsobjekt "Textvariable" in der E-Mail-Nachricht verwenden, um dynamische Nachrichten einzurichten. Sie können z. B. eine E-Mail-Nachricht für den Paketeigentümer mit dem Text "Die Genehmigungen für das Paket <Paketname> sind unvollständig." definieren. Weitere Informationen finden Sie unter "Definieren von E-Mail-Nachrichten" und "Definieren von Textvariablen".

Sie können angeben, dass die folgenden Mitarbeiter die Erinnerungsbenachrichtigungen für Genehmigungen erhalten:

• Eigentümer – Benachrichtigt den Eigentümer des Pakets.
• Genehmiger – Benachrichtigt jeden Genehmiger, für den eine unvollständige Aufgabe vorliegt.
• Eskalationsempfänger für Genehmiger – Benachrichtigt die Eskalationsempfänger für jeden Genehmiger, für den eine unvollständige Aufgabe vorliegt.

Sie können festlegen, dass die folgenden Mitarbeiter die Erinnerungsbenachrichtigungen für Inhalts-Zusammenarbeit erhalten:

• Eigentümer – Benachrichtigt den Eigentümer des Pakets.
• Projektmitarbeiter – Benachrichtigt jeden Projektmitarbeiter, für den eine unvollständige Aufgabe vorliegt.
• Eskalationsempfänger für Projektmitarbeiter – Benachrichtigt die Eskalationsempfänger für jeden Projektmitarbeiter, für den eine unvollständige Aufgabe vorliegt.
Hinweis: Gewährleisten Sie, dass das Feld "E-Mail-Adresse" in der Instanz des "Mitarbeiter"-Modellierungsobjekts jedes Empfängers vollständig ist. Andernfalls erhalten die Empfänger keine E-Mail-Benachrichtigungen.
Hinweis: Sie können die Eskalationsempfänger über das Modellierungsobjekt "Mitarbeiter" definieren. Sie können z. B. eine E-Mail-Benachrichtigung an den Vorgesetzten eines Inhaltsprojektmitarbeiters senden, wenn die Arbeit des Projektmitarbeiters überfällig ist. Die Anwendung sendet keine E-Mail-Benachrichtigungen an Eskalationsempfänger, wenn Sie diese nicht in der Tabelle "Eskalationsempfänger" in der Instanz des "Mitarbeiter"-Modellierungsobjekts des Genehmigers oder Projektmitarbeiters auswählen.

Sie müssen die Anzahl der Tage vor oder nach dem Fälligkeitsdatum angeben, an dem die Benachrichtigungen versendet werden sollen, oder Sie können die Benachrichtigung am Fälligkeitsdatum versenden. Geben Sie z. B. 3 für "Fällig in (Tage)" ein, um die Benachrichtigung drei Tage vor Fälligkeit der Genehmigung oder Zusammenarbeit zu versenden. Geben Sie 1 für "Überfällig (Tage)" ein, um die Benachrichtigung einen Tag nach Fälligkeit der Arbeit zu versenden. Geben Sie entweder für "Fällig in (Tage)" oder für "Überfällig (Tage)" eine 0 ein, um die Benachrichtigung am Fälligkeitsdatum zu versenden.

Sie können nicht sowohl "Fällig in (Tage)" als auch "Überfällig (Tage)" für dieselbe Zeile in der Tabelle "Paketgenehmigungserinnerungen" oder "Inhaltszusammenarbeitserinnerungen" angeben. Wenn Sie einen Wert in das Feld "Überfällig (Tage)" eingeben und in dem Feld "Fällig in (Tage)" bereits ein Wert in derselben Zeile vorhanden ist, wird der Wert aus dem Feld "Fällig in (Tage)" gelöscht. Wenn Sie analog dazu einen Wert im Feld "Fällig in (Tage)" eingeben und in dem Feld "Überfällig (Tage)" in derselben Zeile bereits ein Wert vorhanden ist, wird der Wert im Feld "Überfällig (Tage)" gelöscht.

Die Anwendung sendet tägliche Erinnerungsbenachrichtigungen, bis die Arbeit abgeschlossen ist, wenn Sie das Kontrollkästchen "Täglich wiederholen" aktivieren. Abhängig von den angegebenen Empfängertypen tritt Folgendes ein:

• Der Paketeigentümer erhält Erinnerungsbenachrichtigungen, bis alle Genehmigungen oder sämtliche Zusammenarbeit abgeschlossen ist bzw. sind.
• Der Genehmiger oder Projektmitarbeiter erhält Erinnerungsbenachrichtigungen, bis er die Genehmigung oder Zusammenarbeit abschließt.
• Die Eskalationsempfänger für den Genehmiger oder Projektmitarbeiter erhalten Erinnerungsbenachrichtigungen, bis der Genehmiger oder Projektmitarbeiter die Genehmigung oder Zusammenarbeit abschließt.

Die Anwendung sendet Erinnerungsbenachrichtigungen gemäß einer geplanten Geschäftsregel. Siemens stellt eine standardmäßige geplante Geschäftsregel namens SBR_NOTIFICATIONS bereit. Diese Regel wird alle 24 Stunden, beginnend um 4:00 Uhr ausgeführt. Die Erinnerungen werden, soweit zutreffend, bei jeder Ausführung der geplanten Geschäftsregel ausgeführt. Ein Systemadministrator kann z. B. festlegen, dass einen Tag vor Fälligkeit der Zusammenarbeitsaufgabe am Freitag um 9:30 Uhr eine Erinnerungsbenachrichtigung an den Projektmitarbeiter gesendet wird. Die Anwendung führt die geplante Geschäftsregel am Donnerstag um 4:00 Uhr aus und sendet die Erinnerungsbenachrichtigung an den Projektmitarbeiter.

Ein Systemadministrator kann auch weitere geplante Geschäftsregeln definieren. Z. B. kann er eine geplante Geschäftsregel mit dem Inhalt definieren, dass diese E-Mail-Nachrichten alle sechs Stunden, beginnend um 00:00 Uhr gesendet werden. Die Anwendung führt die geplante Geschäftsregel am Donnerstag um 00:00 Uhr, 6.00 Uhr, 12:00 Uhr und 18:00 Uhr aus und sendet jedes Mal die Erinnerungsbenachrichtigung an den Projektmitarbeiter. Weitere Informationen finden Sie unter "Definieren von geplanten Geschäftsregeln".

Vorgehensweise zum Definieren von Einstellungen für die Änderungsverwaltung

Gehen Sie folgendermaßen vor, um Einstellungen für die Änderungsverwaltung zu definieren:

1. Öffnen Sie die Seite Änderungsmanagementeinstellungen. Die Seite "Änderungsmanagementeinstellungen" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Einstellungen für die Änderungsverwaltung in das Feld Änderungsmanagementeinstellungen ein.
4. Geben Sie einen Wert für die maximale Anzahl von "Wo genutzt?"-Instanzen in das Feld Max. Anzahl 'Wo genutzt'-Instanzen ein.
Hinweis: Die Anwendung verwendet als Standardwert die zulässige minimale Anzahl von "Wo genutzt-Instanzen". Der empfohlene Wert für "Max. Anzahl 'Wo genutzt'-Instanzen" ist 5. Die Anwendung akzeptiert zwar Werte, die größer als 5 sind, dies kann jedoch die Leistung beeinträchtigen.
5. Führen Sie die Schritte unter "Vorgehensweise zum Hinzufügen von Benachrichtigungsereignissen" aus, wenn Sie Benachrichtigungsereignisse hinzufügen möchten.
6. Führen Sie die Schritte unter "Vorgehensweise zum Hinzufügen von Erinnerungen für Paketgenehmigungen" aus, wenn Sie Erinnerungen für Paketgenehmigungen hinzufügen möchten.
7. Führen Sie die Schritte unter "So werden Erinnerungen für Inhaltszusammenarbeit hinzugefügt" aus, wenn Sie Erinnerungen für Inhaltszusammenarbeit hinzufügen möchten.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Benachrichtigungsereignissen

Gehen Sie folgendermaßen vor, um Benachrichtigungsereignisse hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren von Einstellungen für die Änderungsverwaltung" aus.

Oder

Wählen Sie eine vorhandene Instanz von Einstellungen für die Änderungsverwaltung (Änderungsmanagementeinstellungen) aus.

2. Erweitern Sie ggf. den Abschnitt E-Mail-Benachrichtigungen.
3. Klicken Sie in der Tabelle "Benachrichtigungsereignisse" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie einen Benachrichtigungstyp aus.
5. Wählen Sie eine E-Mail-Nachricht aus.
6. Wiederholen Sie die Schritte 2 bis 5, um weitere Benachrichtigungstypen hinzuzufügen.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Erinnerungen für Paketgenehmigungen

Gehen Sie folgendermaßen vor, um Erinnerungen für Paketgenehmigungen hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren von Einstellungen für die Änderungsverwaltung" aus.

3. Klicken Sie in der Tabelle "Paketgenehmigungserinnerungen" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.

5. Wählen Sie einen Empfänger aus.
6. Geben Sie einen Integer-Wert größer oder gleich 0 in das Feld Fällig in (Tage) ein.

Oder

Geben Sie einen Integer-Wert größer oder gleich 0 in das Feld Überfällig (Tage) ein.

7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Wiederholen Sie die Schritte 3 bis 7, um weitere Erinnerungen für Paketgenehmigungen hinzuzufügen.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden Erinnerungen für Inhaltszusammenarbeit hinzugefügt

Gehen Sie folgendermaßen vor, um Erinnerungen für Inhaltszusammenarbeit hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren von Einstellungen für die Änderungsverwaltung" aus.

Oder

Wählen Sie eine vorhandene Instanz von Einstellungen für die Änderungsverwaltung (Änderungsmanagementeinstellungen) aus.

2. Erweitern Sie ggf. den Abschnitt E-Mail-Benachrichtigungen.
3. Klicken Sie in der Tabelle "Inhaltszusammenarbeitserinnerung" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie eine E-Mail-Nachricht aus.
5. Wählen Sie einen Empfänger aus.
6. Geben Sie einen Integer-Wert größer oder gleich 0 in das Feld Fällig in (Tage) ein.

Oder

Geben Sie einen Integer-Wert größer oder gleich 0 in das Feld Überfällig (Tage) ein.

7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Wiederholen Sie die Schritte 3 bis 7, um weitere Erinnerungen für Inhaltszusammenarbeit hinzuzufügen.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
