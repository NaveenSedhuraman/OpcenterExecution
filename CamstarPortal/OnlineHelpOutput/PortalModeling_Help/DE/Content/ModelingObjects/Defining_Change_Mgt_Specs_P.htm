Definieren von Änderungsverwaltungsspezifikationen

Eine Änderungsverwaltungsspezifikation (Änderungsmanagement-Spez.) definiert die Aktivitäten, die an einem Schritt in einem Arbeitsplan zur Änderungsverwaltung (Änderungsmanagement-Arbeitsplan) ausgeführt werden. Die Anwendung erfordert, dass Sie mindestens eine Änderungsverwaltungsspezifikation referenzieren, wenn Sie einen Änderungsmanagement-Arbeitsplan definieren. Eine Spezifikation kann mehrfach von verschiedenen Arbeitsplänen referenziert werden. "Änderungsmanagement-Spez." ist eine Unterklasse des Modellierungsobjekts "Spez.".

Standardmäßige Änderungsverwaltungsspezifikationen

Siemens stellt standardmäßig mehrere Änderungsverwaltungsspezifikationen bereit. Diese Spezifikationen werden von den von Siemens bereitgestellten Änderungsmanagement-Arbeitsplänen referenziert. Sie können die bereitgestellten Änderungsverwaltungsspezifikationen in Ihren eigenen, benutzerdefinierten Änderungsmanagement-Arbeitsplänen verwenden. Darüber hinaus können Sie die bereitgestellten Spezifikationen an Ihre Geschäftsanforderungen anpassen.

Diese Tabelle beschreibt die von Siemens bereitgestellten standardmäßigen Änderungsverwaltungsspezifikationen (Änderungsmanagement-Spezifikationen).

Spez.

Beschreibung

Standard-Bilddatei

Entwurf (Draft)

Erster Schritt im "Ohne Genehmigung"-Arbeitsplan für ein neues Paket. Die zulässige Transaktionsrolle, die der Spezifikation "Entwurf" zugeordnet ist, ist "DraftPermissions".

Benutzer mit entsprechenden Berechtigungen können die folgenden Aktionen für ein Paket im Schritt "Entwurf" ausführen:

• Kernattribute des Pakets bearbeiten
• Dem Paket Inhalte zuweisen
• Pakete zum nächsten Schritt bewegen
• Das Paket aktualisieren
• Das Paket bereitstellen
• Nach dem Paket suchen
• Den Paketschritt ändern
• Die Paketdetails anzeigen

Entwurf Camstar (Draft Camstar)

Erster Schritt im Camstar-Arbeitsplan für ein neues Paket. Die zulässige Transaktionsrolle, die der Spezifikation "Entwurf Camstar" zugeordnet ist, ist "DraftCamstarPermissions".

Benutzer mit den entsprechenden Berechtigungen können dieselben Aktionen für Pakete in diesem Schritt ausführen wie für die Pakete im Schritt "Entwurf". Darüber hinaus können diese Benutzer Genehmiger zum Paket hinzufügen und die Aktion "Route für Genehmigung" für Pakete ausführen.

Bereitstellung ausstehend (Pending Deployment)

Gibt an, dass das Paket an alle konfigurierten Ziele bereitgestellt werden kann. Die zulässige Transaktionsrolle, die der Spezifikation "Bereitstellung ausstehend" zugewiesen ist, ist "PDPermissions".

• Den Bereitstellungsstatus des Pakets anzeigen
• Die Paketdetails anzeigen
• Nach dem Paket suchen
• Den Paketschritt ändern

Bereitstellung abgeschlossen (Deployment Complete)

Gibt an, dass das Paket erfolgreich an allen konfigurierten Zielen bereitgestellt wurde. Die zulässige Transaktionsrolle, die der Spezifikation "Bereitstellung abgeschlossen" zugewiesen ist, ist "DEPCPermissions".

Benutzer mit den entsprechenden Berechtigungen können die folgenden Aktionen für Pakete im Schritt "Bereitstellung abgeschlossen" ausführen:

• Den Bereitstellungsstatus des Pakets anzeigen
• Die Paketdetails anzeigen

• Den Paketschritt ändern

Bereitstellung unvollständig (Deployment Incomplete)

Benutzer mit den entsprechenden Berechtigungen können dieselben Aktionen für Pakete in diesem Schritt ausführen wie für die Pakete im Schritt "Bereitstellung abgeschlossen".

Weitere Informationen finden Sie im Opcenter Execution Core Teamcenter Integration-Handbuch.

Download wird durchgeführt

Beim Definieren einer Änderungsverwaltungsspezifikation

Sie können:

• Den Paketstatus angeben, der verwendet wird, wenn ein Paket zu einem Änderungsverwaltungsschritt (Änderungsmanagement-Schritt) bewegt wird.
• Ein benutzerdefiniertes Bild festlegen, das im Paketinformationskopf angezeigt werden soll. Aus diesem Bild geht die Spezifikation (Schritt) hervor, in der sich das Änderungsverwaltungspaket befindet. Das benutzerdefinierte Bild muss:
• Sich im Ordner "C:\Programme (x86)\Camstar\Camstar Portal\" befinden
• Im *.png-Format gespeichert sein
• Eine Größe von 20 x 20 Pixel haben
• Eine oder mehrere Rollen für Transaktionen zuweisen, die in der Spezifikation zulässig sind. Sie definieren Rollen mit dem Modellierungsobjekt "Rolle". Weitere Informationen finden Sie unter "Definieren von Rollen". Informationen zu den Standardrollen für die Änderungsverwaltung, zu Berechtigungen und zu zulässigen Transaktionen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration und in der Opcenter Execution Core-Anleitung zur Systemadministration.
• Angeben, ob der Genehmigungsprozess für das Änderungsverwaltungspaket zurückgesetzt werden muss, wenn das Paket erneut in die Spezifikation kommt. Wenn Sie den Genehmigungsprozess zurücksetzen, werden ausstehende oder abgeschlossene Genehmigungen abgebrochen und müssen Genehmiger das Änderungsverwaltungspaket erneut genehmigen, wenn es erneut über die Genehmigungsroute gesendet wird. Die Anwendung speichert sämtliche erfassten Genehmigungen im Verlauf.
Hinweis: Einen Verlauf abgebrochener Genehmigungen können Sie auf der Registerkarte "Genehmigungsstatus" auf der Seite "Paket genehmigen" anzeigen.
• Angeben, ob die Registerkarte "Genehmiger" auf der Seite "Aktualisierungspaket" angezeigt werden soll, wenn das Paket an diesem Schritt im Arbeitsplan ist.
• Angeben, ob ein Änderungsverwaltungspaket bei dieser Spezifikation eine Genehmigung erfordert. Die Anwendung verwendet dieses Kontrollkästchen für Erzwingung der Änderungssteuerung, um zu bestimmen, ob sich das Paket in einem Genehmigungsarbeitsplan befindet. Informationen zur Erzwingung der Änderungssteuerung finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.
Hinweis: Sie können das Modellierungsobjekt "Stammdatenkatalog" verwenden, um die Erzwingung der Änderungssteuerung für Modellierungsobjekte festzulegen. Weitere Informationen dazu, wie Sie die Erzwingung der Änderungssteuerung festlegen, finden Sie unter "Definieren eines Stammdatenkatalogs.
• Angeben, ob Modellierungsinstanzen im Änderungsverwaltungspaket gesperrt werden müssen, wenn das Paket in diese Spezifikation gelangt.

Beim Sperren von Modellierungsobjektinstanzen von Änderungspaketen

Durch Auswählen des Kontrollkästchens "Modellierungsinstanzen in einem Änderungspaket bei dieser Spez. sperren" werden sämtliche Modellierungsinstanzen im Änderungsverwaltungspaket gesperrt, wenn das Paket an diese Spezifikation übergeben wird.

Hinweis: Sie können eine gesperrte Modellierungsobjektinstanz eines Änderungspakets weder ändern noch löschen. Die Anwendung zeigt eine Warnmeldung an, wenn Sie versuchen, eine gesperrte Modellierungsobjektinstanz eines Änderungspakets zu ändern oder zu löschen.

Die Anwendung entsperrt die Instanzen, wenn:

• Das Änderungspaket zu einer Spezifikation bewegt wird, für die dieses Kontrollkästchen nicht aktiviert ist und
• Alle Pakete, die diese Instanzen referenzieren, in einer Spezifikation sind, für die dieses Kontrollkästchen nicht aktiviert ist.

Bei Eingang in diese Spez. Genehmigungen zurücksetzen

Dieses Kontrollkästchen gibt an, ob die Anwendung ausstehende oder abgeschlossene Genehmigungen abbricht, wenn ein Änderungsverwaltungspaket erneut zu dieser Spezifikation gelangt. Die Anwendung zeichnet die abgebrochene Genehmigung im Genehmigungsverlauf auf und setzt den Genehmigungsprozess zurück, wenn dieses Kontrollkästchen aktiviert ist. Alle angegebenen Genehmiger müssen das Änderungsverwaltungspaket erneut genehmigen, wenn es erneut über die Genehmigungsroute gesendet wird.

Anmerkung: Dieses Kontrollkästchen ist standardmäßig für die Spezifikationen "Entwurf Camstar" und "Entwurf PLM" aktiviert.

Genehmiger zuweisen

Kontrollkästchen, das angibt, ob die Anwendung die Registerkarte "Genehmiger" auf der Seite "Paket aktualisieren" anzeigt.

Anmerkung: Dieses Kontrollkästchen ist für die Spezifikation "Entwurf Camstar" standardmäßig aktiviert.

Optional

Erfordert Genehmigung

Dieses Kontrollkästchen gibt an, ob ein Paket bei dieser Spezifikation (bei diesem Schritt) eine Genehmigung erfordert. Die Anwendung bestimmt anhand dieses Kontrollkästchens, ob das Paket, dem Sie eine Modellierungsobjektinstanz hinzufügen, sich im richtigen Arbeitsplan für die Erzwingung der Änderungssteuerung befindet.

Anmerkung: Dieses Kontrollkästchen ist standardmäßig deaktiviert.

Optional

Modellierungsinstanzen in einem Änderungspaket bei dieser Spez. sperren

Kontrollkästchen, das angibt, dass Modellierungsinstanzen gesperrt werden, wenn das Änderungspaket zu dieser Spezifikation bewegt wird.

Anmerkung: Dieses Kontrollkästchen ist standardmäßig für folgende Spezifikationen aktiviert:

• Bereitstellung abgeschlossen
• Bereitstellung unvollständig
• Genehmigung ausstehend Camstar
• Genehmigung ausstehend PLM
• Bereitstellung ausstehend (Pending Deployment)
• Abgelehnt (Rejected)

Optional

So wird eine Änderungsverwaltungsspezifikation definiert

Gehen Sie folgendermaßen vor, um eine Änderungsverwaltungsspezifikation zu definieren:

1. Öffnen Sie die Seite Änderungsmanagement-Spez. Die Seite "Änderungsmanagement-Spez." wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Spezifikation in das Feld Änderungsmanagement-Spez. ein.
4. Geben Sie die Revision dieses Objekts in das Feld Revision ein.
5. Wählen Sie einen Paketstatus aus.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden eine oder mehrere Rollen zu erlaubten Transaktionen zugewiesen

Gehen Sie folgendermaßen vor, um erlaubten Transaktionen eine oder mehrere Rollen zuzuweisen:

1. Führen Sie die Schritte unter "So wird eine Änderungsverwaltungsspezifikation definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Änderungsverwaltungsspezifikation (Änderungsmanagement-Spez.) aus.

2. Erweitern Sie den Abschnitt Transaktionen.
3. Klicken Sie auf Neue Zeile hinzufügen. Eine neue Zeile wird der Tabelle hinzugefügt.
4. Wählen Sie eine oder mehrere Rollen aus der Liste aus, die den verfügbaren Transaktionen zugewiesen werden sollen.
5. Wiederholen Sie die Schritte 3 bis 4, um weitere Zeilen zuzuordnen.
6. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
