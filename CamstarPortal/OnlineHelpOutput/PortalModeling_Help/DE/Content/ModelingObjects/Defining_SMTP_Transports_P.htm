Definieren von SMTP-Transporten

SMTP-Transport ist eine Unterklasse des Datentransports. Die Anwendung verwendet das Simple Mail Transport Protocol (SMTP), um E-Mails zu verschicken. Dadurch kann die Anwendung E-Mails versenden, wenn sie für die Ausführung als Windows-Service konfiguriert ist. Der virtuelle SMTP-Host ist Teil des IIS 6.0 Component Service auf Windows-Servern.

Hinweis: Wenn Sie einen Link zu Ihrem Exchange-Mailserver erstellen möchten, müssen Sie einen Benutzernamen, ein Kennwort und eine URL (Name des Hostservers, der mit dem SSL-Zertifikat übereinstimmt) eingeben. Für Nutzername und Kennwort gelten die Anforderungen des E-Mail-Servers.

Beim Definieren eines SMTP-Transports

"SMTP-Transport" ist ein optionales Feld in den folgenden Modellierungsdefinitionen:

• Unternehmen
• Fabrik
• Organisation
• Standort
Hinweis: Sollten die erwarteten E-Mails nicht ankommen, muss der SMTP-Server eventuell so konfiguriert werden, das anonyme Benutzer eine Verbindung zum SMTP-Server herstellen können.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Vorgehensweise zum Definieren eines SMTP-Transports

Gehen Sie wie folgt vor, um einen SMTP-Transport zu definieren:

1. Öffnen Sie die Seite SMTP-Transport. Die Seite "SMTP-Transport" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen des SMTP-Transports in das Feld SMTP-Transport ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
