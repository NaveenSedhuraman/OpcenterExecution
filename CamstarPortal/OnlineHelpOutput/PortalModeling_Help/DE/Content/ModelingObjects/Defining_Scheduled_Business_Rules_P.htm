Definieren von geplanten Geschäftsregeln

Mit dem Modellierungsobjekt "Geplante Geschäftsregel" wird ein Satz von Laufzeitoptionen festgelegt, die Sie einer Geschäftsregel zuweisen können, um zu bestimmen, wann die Anwendung diese Regel ausführt. Sie können bestimmen, ob die Ausführung einmalig oder über einen längeren Zeitraum regelmäßig ausgeführt wird.

Opcenter EX MDD und Opcenter EX CR führen geplante Geschäftsregeln als Service aus, der von jedem vom Benutzer eingegebenen Ereignis getrennt ist. Da der Zeitplan unabhängig von Benutzer-Services ausgeführt wird, wird ein Kontext bestimmt, aus dem Parameterwerte in der Geschäftsregel referenziert werden.

Opcenter EX MDD und Opcenter EX CR bieten die folgenden standardmäßig geplanten Geschäftsregeln:

• SBR_ACTIVATION – Diese geplante Geschäftsregel ist mit der Geschäftsregel BR_ACTIVATION verknüpft. Sie wird alle 5 Minuten ausgeführt.
• SBR_DEPLOY – Diese geplante Geschäftsregel ist mit der Geschäftsregel BR_DEPLOY verknüpft. Sie wird alle 5 Minuten ausgeführt.
• SBR_DEPLOYSTATUS – Diese geplante Geschäftsregel ist mit der Geschäftsregel BR_DEPLOYSTATUS verknüpft. Sie wird alle 5 Minuten ausgeführt.
• SBR_NOTIFICATIONS – Diese geplante Geschäftsregel ist mit der Geschäftsregel BR_NOTIFICATIONS verknüpft. Sie wird alle 24 Stunden beginnend um 4:00 Uhr ausgeführt.
Hinweis: Als Startdatum jeder dieser standardmäßigen geplanten Geschäftsregeln werden Installationsdatum und -uhrzeit verwendet. Weitere Informationen zu jeder dieser Standardgeschäftsregeln finden Sie unter "Definieren von Geschäftsregeln".

Beim Definieren von geplanten Geschäftsregeln

Sie können geplante Geschäftsregeln manuell definieren, indem Sie das Modellierungsobjekt "Geplante Geschäftsregel" verwenden, oder auch automatisch, indem Sie eine Geschäftsregel mit einem Skript ausführen, das eine neue geplante Geschäftsregel erstellt. Die Anwendung erzwingt die Verwendung von Geschäftsregeln, sobald diese anderen Objekten zugewiesen werden.

Sie wählen aus, ob Sie grundlegende oder erweiterte Optionen für die Ablaufplanung verwenden möchten, wenn Sie eine Regel mit dem Modellierungsobjekt "Geplante Geschäftsregel" definieren. Die erweiterten Optionen bieten zusätzliche Flexibilität für die Ablaufplanung, wenn die Regel ausgeführt wird. Weitere Informationen finden Sie unter "Beim Verwenden der grundlegenden Optionen zum Definieren von geplanten Geschäftsregeln" und "Beim Verwenden der erweiterten Optionen zum Definieren von geplanten Geschäftsregeln".

Sie können einen Wiederholungsbereich (Startdatum und -zeit, Anzahl der Iterationen und Enddatum und -zeit) für die geplante Geschäftsregel festlegen, und zwar unabhängig davon, ob Sie die grundlegenden oder die erweiterten Optionen verwenden.

Die Anwendung löscht die geplanten Geschäftsregeln aus der Datenbank, nachdem das Enddatum abgelaufen ist. Die Instanz der geplanten Geschäftsregel wird nicht in der Liste "Instanzen" angezeigt.

Startdaten und -zeiten

Sie müssen eine Startzeit festlegen, die nach dem aktuellen Zeitpunkt liegt, wenn Sie als Startdatum für den Zeitplan das aktuelle Datum festlegen. Wenn es momentan z. B. 9:30 Uhr vormittags ist, müssen Sie eine spätere Uhrzeit als 9:30 Uhr eingeben, damit die Anwendung den Zeitplan fehlerfrei ausführt.

Iterationen

Im Feld "Iterationen" können Sie die Anzahl der Wiederholungen für die Ausführung der Regel angeben. Die Anwendung berechnet automatisch ein Enddatum. Das Feld "Enddatum" hat Vorrang, wenn Sie sowohl einen Wert für "Iterationen" als auch für "Enddatum" angeben. Die Planung der Geschäftsregel läuft unbegrenzt weiter, wenn Sie die Felder "Iterationen" und "Enddatum" leer lassen.

Enddaten und -zeiten

Die Anwendung legt als Standardzeit 00:00:00 fest (d. h. die Regel wird beendet, sobald sich das Datum ändert), wenn Sie ein Enddatum festlegen, aber keine Zeit eingeben. Wenn Sie eine Anzahl von Iterationen angegeben haben, die dazu führt, dass die Ausführung einer Iteration für das Enddatum geplant wird, so wird diese Iteration nicht ausgeführt, da sie nach 00:00:00 Uhr erfolgen würde.

Beim Verwenden der grundlegenden Optionen zum Definieren von geplanten Geschäftsregeln

Sie müssen ein Wiederholungsmuster (Wiederholung und Häufigkeit) definieren, wenn Sie eine geplante Geschäftsregel mit den grundlegenden Optionen definieren.

Wiederholung und Häufigkeit

Die folgenden Wiederholungsintervalle stehen beim Definieren von geplanten Geschäftsregeln zur Verfügung:

• Einmalig – Die Pläne werden einmalig zum festgelegten Datum und zur festgelegten Uhrzeit ausgeführt.
• Sekunden – Zwischen den Ausführungen der Pläne liegt eine festgelegte Anzahl von Sekunden.
• Minuten – Zwischen den Ausführungen der Pläne liegt eine festgelegte Anzahl von Minuten.
• Stündlich – Zwischen den Ausführungen der Pläne liegt eine festgelegte Anzahl von Stunden.
• Täglich – Zwischen den Ausführungen der Pläne liegt eine festgelegte Anzahl von Tagen.
• Wöchentlich – Die Pläne werden an einem bestimmten Wochentag ausgeführt.
• Monatlich – Die Pläne werden an einem bestimmten Tag des Monats ausgeführt.
• Jährlich – Die Pläne werden in einem bestimmten Monat und/oder an einem Tag des Monats ausgeführt.

Wenn Sie im Feld "Wiederholung" "Wöchentlich", "Monatlich" oder "Jährlich" auswählen, werden auf der Seite zusätzliche Felder angezeigt. Sie müssen Folgendes festlegen:

• Den "Wochentag" für den auszuführenden Zeitplan, wenn Sie "Wöchentlich" auswählen.
• Einen numerischen Wert für den "Tag des Monats", wenn Sie "Monatlich" auswählen. Alternativ können Sie das Kontrollkästchen "Ist letzter Monatstag" aktivieren. (Diese beiden Felder schließen sich gegenseitig aus.)
• Den "Monat des Jahres", wenn Sie "Jährlich" auswählen. Zusätzlich müssen Sie das Feld "Tag des Monats" ausfüllen oder das Kontrollkästchen "Ist letzter Monatstag" aktivieren.

Allen Wiederholungsintervallen kann eine bestimmte Häufigkeit zugewiesen werden, aus der sich ein Muster dafür ergibt, wie oft die Anforderung wiederholt wird: 1 bedeutet, dass die Geschäftsregel bei jeder geplanten Wiederholung ausgeführt wird; 2 bedeutet, dass die Geschäftsregel jedes zweite Mal ausgeführt wird und so weiter.

Beispiel 1

Legen Sie "Wöchentlich" mit der Häufigkeit 2 und Mittwoch als "Wochentag" fest, um eine geplante Geschäftsregel einzurichten, die an jedem zweiten Mittwoch ausgeführt wird.

Beispiel 2

Legen Sie "Minuten" mit der Häufigkeit 20 fest, um eine geplante Geschäftsregel einzurichten, die alle 20 Minuten ausgeführt wird.

Beim Verwenden der erweiterten Optionen zum Definieren von geplanten Geschäftsregeln

Ein stündliches Muster (stündliche Häufigkeit und Startstunde) muss festgelegt werden, wenn Sie eine geplante Geschäftsregel mit den erweiterten Optionen definieren. Sie können auch bestimmte Tage einer Woche, bestimmte numerische Werte für Tage des Monats und bestimmte Monate festlegen, an denen die Regel ausgeführt wird.

Stündliches Muster

Die Standardwerte für "Alle <n> Stunden beginnend mit Stunde <n>" sind 24 und 0. Das bedeutet, dass die Regel alle 24 Stunden, beginnend um 00:00 Uhr des Startdatums ausgeführt wird. Gültige Werte für die stündliche Häufigkeit sind ganze Zahlen zwischen 1 und 24. Gültige Werte für die Startstunde sind ganze Zahlen zwischen 0 und 23.

Hinweis: Stellen Sie sicher, dass Sie realistische Werte für das stündliche Muster eingeben. Eine Regel wird z. B. nur um 5:00 Uhr ausgeführt, wenn Sie "Jede 22 Stunde(n), beginnend mit der Stunde 5" festlegen, da 27:00 Uhr keine gültige Zeit darstellt.

Bestimmte Tage einer Woche

Das Kontrollkästchen "Bestimmte Tage einer Woche" ermöglicht es Ihnen anzugeben, ob Sie die Wochentage auswählen möchten, an denen die geplante Geschäftsregel ausgeführt werden soll. Auf der Seite werden zusätzliche Felder für jeden der sieben Wochentage angezeigt, wenn Sie das Kontrollkästchen aktivieren.

Bestimmte Tage eines Monats

Das Kontrollkästchen "Bestimmte Tage eines Monats" ermöglicht es Ihnen anzugeben, ob Sie den Tag des Monats festlegen möchten, an dem die geplante Geschäftsregel erstmalig ausgeführt werden soll, sowie das Intervall von Tagen für die anschließende Ausführung der Regel. Die zusätzlichen Felder "Alle <n> Tage beginnend bei Tag <n>" werden auf der Seite angezeigt, wenn Sie das Kontrollkästchen aktivieren. Der Standardwert für jedes dieser Felder ist 1. Das bedeutet, dass die Regel jeden 1 Tag ausgeführt wird, beginnend mit dem Tag 1 des Startdatums. Gültige Werte für diese Felder sind ganze Zahlen zwischen 1 und 31.

Hinweis: Stellen Sie sicher, dass Sie realistische Werte für das Muster "Bestimmte Tage eines Monats" eingeben. Eine Regel wird z. B. nur am Tag 10 ausgeführt, wenn Sie "Jede 25 Tag(e), beginnend mit dem Tag 10" festlegen, da 35 keinen gültigen Tag darstellt.

Bestimmte Monate

Das Kontrollkästchen "Bestimmte Monate" ermöglicht es Ihnen anzugeben, ob Sie die Monate auswählen möchten, in denen die geplante Geschäftsregel ausgeführt werden soll. Auf der Seite werden zusätzliche Felder für jeden der zwölf Monate des Jahres angezeigt, wenn Sie das Kontrollkästchen aktivieren.

Beispiel 1

Legen Sie Folgendes fest, um eine geplante Geschäftsregel einzurichten, die alle 12 Stunden ausgeführt wird, und zwar beginnend um 6:00 Uhr an Dienstagen und Donnerstagen:

• 12 und 6 für "Alle <n> Stunden beginnend mit Stunde <n>"
• Dienstag und Donnerstag für "Bestimmte Tage einer Woche"

Die Regel wird ausgeführt um 6:00 Uhr und um 18:00 Uhr an Dienstagen und Donnerstagen.

Beispiel 2

Legen Sie Folgendes fest, um eine geplante Geschäftsregel einzurichten, die alle 6 Stunden ausgeführt wird, und zwar beginnend um 0:00 Uhr am 15. Tag des Monats mit einer Wiederholung alle 5 Tage:

• 6 und 0 für "Alle <n> Stunden beginnend mit Stunde <n>"
• 5 und 15 für "Alle <n> Tage beginnend bei Tag <n>"

Die Regel wird ausgeführt um 0:00 Uhr, 6:00 Uhr, 12:00 Uhr und 18:00 Uhr am 15., 20., 25. und 30. Tag des Monats.

Beispiel 3

Legen Sie Folgendes fest, um eine geplante Geschäftsregel einzurichten, die alle 4 Stunden ausgeführt wird, und zwar beginnend um 12:00 Uhr am 1. Tag des Monats mit einer Wiederholung alle 10 Tage während der Monate Januar und Juli:

• 4 und 12 für "Alle <n> Stunden beginnend mit Stunde <n>"
• 10 und 1 für "Alle <n> Tage beginnend bei Tag <n>"
• Januar und Juli für "Bestimmte Monate"

Die Regel wird ausgeführt um 12:00 Uhr, 16:00 Uhr und 20:00 Uhr am 1., 11., 21. und 31. Januar und Juli.

Definieren von mehreren Zeitplänen

Entsprechend Ihren geschäftlichen Anforderungen müssen Sie gegebenenfalls mehrere Zeitpläne für eine Geschäftsregel definieren.

Beispiel 1

Ihre Organisation verlangt, dass eine bestimmte Geschäftsregel am Ende jeder Woche und am Ende jedes Monats ausgeführt wird. Legen Sie Folgendes fest, um zwei geplante Geschäftsregeln mit den grundlegenden Optionen einzurichten, einmal wöchentlich und einmal monatlich:

• "Wöchentlich" als "Wiederholung", "1" für die "Häufigkeit" und "Freitag" als "Wochentag"
• "Monatlich" als "Wiederholung", "1" für die "Häufigkeit" und "Ist letzter Monatstag"

Beispiel 2

Ihre Organisation verlangt, dass eine bestimmte Geschäftsregel zweimal täglich ausgeführt wird: einmal am Anfang des Tages und einmal am Ende des Tages. Anfang des Tages ist 7:00 Uhr und Ende des Tages ist 18:00 Uhr. Über die erweiterten Optionen können Sie eine geplante Geschäftsregel definieren. Die Anwendung führt die Regel am Morgen und erneut am Abend jedes Geschäftstages aus.

Legen Sie Folgendes fest, um die geplante Geschäftsregel zu konfigurieren:

• 11 und 7 für "Alle <n> Stunden beginnend mit Stunde <n>" und Montag, Dienstag, Mittwoch, Donnerstag und Freitag als "Bestimmte Tage einer Woche"

Beim Speichern von geplanten Geschäftsregeln

Wenn Sie versuchen, eine neue geplante Geschäftsregel unter demselben Namen zu speichern wie eine bereits vorhandene, so teilt Ihnen die Anwendung mit, dass eine geplante Geschäftsregel mit diesem Namen bereits vorhanden ist. Die Anwendung fordert Sie auf, einen neuen Namen für den Zeitplan einzugeben.

Die grundlegenden und erweiterten Optionen schließen sich gegenseitig aus. Die Anwendung verarbeitet nur einen Satz von Optionen. Die Werte, die Sie für die grundlegenden und erweiterten Optionen eingeben, bleiben auf der Seite, bis Sie die Instanz von "Geplante Geschäftsregel" speichern. Sie können z. B. grundlegende Optionen festlegen, "Erweitert" auswählen, erweiterte Optionen festlegen und anschließend "Grundlegend" auswählen, um die Werte anzuzeigen, die Sie zuvor festgelegt haben. Für beide sind die Optionen vorhanden. Die Anwendung speichert nur die Werte für die aktuell ausgewählte Option (entweder "Grundlegend" oder "Erweitert"), wenn Sie die Instanz speichern. Die Anwendung löscht die Werte für die anderen Optionen.

Die Anwendung führt die folgenden Validierungen für die grundlegenden Optionen durch, bevor die geplante Geschäftsregel gespeichert wird.

Die Anwendung führt vor dem Speichern der geplanten Geschäftsregel die folgenden Validierungen für die erweiterten Optionen durch:

• Die Anwendung speichert die Seite mit den Standardwerten für die erforderlichen Felder (Alle <n> Stunden beginnend mit Stunde <n>), wenn Sie keine Werte dafür eingegeben.
• Die Anwendung deaktiviert die Kontrollkästchen "Bestimmte Tage einer Woche" und "Bestimmte Monate", wenn Sie eine dieser beiden Optionen auswählen, dann aber keines der Kontrollkästchen für die einzelnen Tage oder Monate aktivieren.
• Die Anwendung löscht die Werte aus den Feldern in den erweiterten Abschnitten ("Bestimmte Tage einer Woche", "Bestimmte Tage eines Monats", "Bestimmte Monate"), wenn Sie das Kontrollkästchen des erweiterten Abschnitts vor dem Speichern deaktivieren. Der Abschnitt wird nicht verwendet.
• Die Anwendung setzt die Werte für "Alle <n> Stunden beginnend mit Stunde <n>" und "Alle <n> Tage beginnend bei Tag <n>" auf die jeweiligen Standardwerte zurück, wenn Sie ungültige Werte eingeben.

Beim Ausführen einer geplanten Geschäftsregel durch das System

Opcenter EX MDD und Opcenter EX CR bieten einen Scheduler-Service der regelmäßig nach geplanten Geschäftsregeln sucht, die ausgeführt werden müssen. Die Häufigkeit, mit der der Scheduler-Service nach Plänen sucht, hängt von der Einstellung "Abfrage-Intervall für den Geschäftsregel-Scheduler" in Management Studio ab. Weitere Informationen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Hinweis: Obwohl Sie Ausführungshäufigkeiten auf die Sekunde genau festlegen können, beachtet das Abfrageintervall des Schedulers dennoch die Einstellung zum Abfrageintervall für Geschäftsregeln in Management Studio.

Überprüfung während der Ausführung

Die Überprüfung findet während der Ausführung der Geschäftsregeln statt, um Folgendes sicherzustellen:

• Die Parameter der Geschäftsregel entsprechen den Parametern der angehängten Skript-Handler.
• Das Skript im Script-Handler ist syntaktisch korrekt.

Service-Historie

Die Anwendung zeichnet die folgenden Details als Bestandteil der Service-Historie für jede Ausführung einer geplanten Geschäftsregel auf:

• Planname
• Geschäftsregel
• Ausführungsstatus (Erfolg oder Misserfolg)
• Ausführungsdatum (einschließlich Zeit)
• Beendigungsnachricht
• Kontext
• Kontextwert

Vorgehensweise zum Definieren einer geplanten Geschäftsregel

Gehen Sie folgendermaßen vor, um eine geplante Geschäftsregel zu definieren:

1. Öffnen Sie die Seite Geplante Geschäftsregel. Die Seite "Geplante Geschäftsregel" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese geplante Geschäftsregel in das Feld Geplante Geschäftsregel ein.
4. Wählen Sie den Kontext für die Bewertung der Geschäftsregel aus der Liste Verwendung aus.
5. Wählen Sie die zu planende Geschäftsregel aus der Liste Geschäftsregel aus.
6. Wählen Sie den Kontextwert für die Bewertung der Geschäftsregel aus der Liste Kontextwert aus.
7. Möchten Sie die grundlegenden Optionen oder die erweiterten Optionen verwenden?

Grundlegende Optionen
a. Wählen Sie einen Wert aus der Liste Wiederholung aus, um festzulegen, wann die Anwendung die Geschäftsregel ausführen soll. Wenn Sie "Monatlich", "Wöchentlich" oder "Jährlich" auswählen, werden auf der Seite zusätzliche Felder angezeigt.
b. Geben Sie einen Wert in das Feld Häufigkeit ein, um festzulegen, in welchen Intervallen die Geschäftsregel ausgeführt werden soll.
c. Je nach Ihrer Auswahl in der Liste Wiederholung müssen Sie einen Wert eingeben oder den entsprechenden Eintrag in den Feldern Wochentag, Tag des Monats, Ist letzter Monatstag oder Monat des Jahres auswählen.
d. Gehen Sie zu Schritt 8.

Erweiterte Optionen

a. Aktivieren Sie Erweitert. Die Anwendung blendet die grundlegenden Optionen aus und zeigt die erweiterten Optionen an.
b. Geben Sie Werte für "Alle <n> Stunden beginnend mit Stunde <n>" ein, wenn Sie von den Standardwerten abweichende Einstellungen festlegen möchten.
c. Wählen Sie bei Bedarf eine oder mehrere erweiterte Optionen (Bestimmte Tage einer Woche, Bestimmte Tage eines Monats, Bestimmte Monate) aus.
d. Geben Sie gemäß Ihrer Auswahl in Schritt c Werte ein oder aktivieren Sie die entsprechenden Kontrollkästchen in den erweiterten Abschnitten.
e. Gehen Sie zu Schritt 8.
8. Geben Sie in das Feld Startdatum den Zeitpunkt (Datum und Uhrzeit) ein, zu dem die Geschäftsregel das erste Mal ausgeführt werden soll. Sie können das Datum und die Uhrzeit auch im Kalender auswählen.
9. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
10. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
