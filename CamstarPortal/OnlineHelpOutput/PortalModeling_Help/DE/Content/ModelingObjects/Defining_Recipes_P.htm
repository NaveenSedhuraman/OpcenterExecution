Definieren von Rezepten

Ein Rezept ist ein Dokumenttyp. Eine Rezeptdefinition besteht aus dem Namen und dem Speicherort der Rezeptdatei sowie aus optionalen Parametern.

In einem Rezept werden in der Regel die Ressourceneinstellungen, z. B. Takt und Temperatur einer Maschine, für die Produktion definiert. Sobald ein Rezept definiert wurde, wird es normalerweise, aber nicht immer, von der Ressource selbst angefertigt.

Anforderungen

Ihr Browser muss über das entsprechende Plugin zum Anzeigen des ausgewählten Rezeptdokuments verfügen (z. B. eine PDF-Datei).

Beim Definieren eines Rezepts

Diese Regeln gelten, wenn Sie Rezepte definieren:

• Ein Rezept darf nicht den gleichen Namen wie ein Dokument haben. Wenn Sie z. B. ein Dokument namens "Test" erstellen, können Sie kein Rezept mit dem gleichen Namen erstellen.
• Über das Kontrollkästchen "Datei in die Datenbank hochladen" können Sie bestimmen, wann die angegebene Datei in die Datenbank hochgeladen werden soll. In der Regel aktivieren Sie dieses Kontrollkästchen, wenn Sie ein Rezept erstmalig definieren. Wenn Sie das Kontrollkästchen bei der Aktualisierung vorhandener Instanzen leer lassen, können Sie Änderungen vornehmen, ohne das Dokument hochladen zu müssen.
• Das Feld "Name der gespeicherten Datei" ist schreibgeschützt und zeigt den Dateinamen einer lokalen Datei an, wenn Sie diese hochladen und auf "Speichern" klicken.
• Die Zeit, die für das Herunterladen/Hochladen von Dateien benötigt wird, hängt von der Größe der Datei ab.
• Alle Werte, die in die Tabelle "Parameter" eingegeben werden, können in einem Rezeptplan oder einer Rezeptmatrix überschrieben werden.

Das Rezept ist:

• Ein optionales Feld in der Definition einer Spezifikation (in der Rezeptdatei) und Umlaufbestandsnachricht.
Hinweis: Für eine Rezeptdefinition können zugehörige Umlaufbestandsnachrichten vorliegen, wenn sie mit einem Feld einer Containerdefinition verknüpft ist. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch oder im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

So wird ein Rezept definiert

Gehen Sie folgendermaßen vor, um ein Rezept zu definieren:

1. Öffnen Sie die Seite Rezept. Die Seite "Rezept" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für dieses Rezept in das Feld Rezept ein.
4. Geben Sie die Revision dieses Rezepts in das Feld Revision ein.
5. Klicken Sie auf das Symbol Datei hochladen neben dem Feld Speicherort der Datei. Das Dialogfeld "Datei hochladen" wird angezeigt.
6. Suchen Sie die Datei und klicken Sie dann auf Öffnen. Das Dialogfeld "Datei hochladen" wird geschlossen und der Dateiname wird im Feld "Datei hochladen" angezeigt.
7. Aktivieren Sie das Kontrollkästchen Datei in die Datenbank hochladen, wenn diese Datei beim Speichern in die Datenbank hochgeladen werden soll.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt, aktiviert die Schaltfläche "Dokument anzeigen" und zeigt eine Bestätigungsmeldung an.
