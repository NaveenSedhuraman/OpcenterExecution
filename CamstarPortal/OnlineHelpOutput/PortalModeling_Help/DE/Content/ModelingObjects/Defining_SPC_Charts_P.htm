Definieren von SPC-Charts

Das Modellierungsobjekt "SPC-Chart " enthält Informationen, die erforderlich sind, um ein einzelnes SPC-Chart anzuzeigen. In dem Objekt werden der Charttyp, das Statit Software-Makro, die Statit Software-Regeldatei, Außer-Kontrolle-Aktionen und -Prozeduren (OCAPs) und die zu plottende Variable beschrieben.

Siemens bietet vier Standard-SPC-Charts an, die Sie verwenden können:

• Moving Range
• Range Chart
• Trial I-Chart
• X Chart

Beim Definieren eines SPC-Charts

Für jedes Chart, das angezeigt werden soll, müssen Sie mit dem entsprechenden Makro zur Charterstellung ein SPC-Chart erstellen.

Gewährleisten Sie, dass die Pfade zu den Chartmakros und den Charteigenschaften korrekt sind. Wenn SPC z. B. in einem anderen Verzeichnis als dem Standardverzeichnis installiert wird, müssen Sie die Pfadangaben ändern.

Verwenden Sie den Abschnitt "Außer Kontrolle-Aktionen" des SPC-Charts, um die bei einer Regelverletzung ausgelösten Aktionen festzulegen.

SPC-Chart ist eine optionale Modellierungsdefinition in der SPC-Chartgruppe.

Ein SPC-Chart enthält die folgenden optionalen Modellierungsdefinitionen: "Dokument", "E-Mail-Nachricht", "Anhaltegrund" und "Ressourcenstatuscode".

Informationen zum Einrichten und Konfigurieren von SPC finden Sie im Opcenter Execution Core Statistical Process Control-Referenzhandbuch.

Außer Kontrolle-Aktionen

Kontrollkästchen, das angibt, ob das Chart bei einer Regelverletzung Außer Kontrolle-Aktionen und -Prozeduren (OCAPs) auslösen kann. Wenn dieses Kontrollkästchen nicht aktiviert ist, werden keine OCAPs ausgelöst, auch wenn es zu einer Regelverletzung kommt.

Container anhalten – Ursachencode

Der Anhaltegrund, der dem Container zugewiesen wird. Im Fall einer Regelverletzung wird der Container mit diesem Anhaltegrund angehalten.

Optional

Ressourcenstatus setzen

Der Statusgrund, der der Ressource zugewiesen wird, wenn eine Regelverletzung auftritt.

Optional

Anzuzeigendes Dokument

Aktion, die festlegt, dass die Referenz zum Dokument angezeigt werden soll, wenn eine OCAP ausgelöst wird. Dokumentinstanzen können erstellt werden, indem das Dokumentenobjekt modelliert wird. SPC unterstützt keine Dokumentsätze. Nur einzelne Dokumente können angezeigt werden.

Optional

Anmerkung erforderlich

Kontrollkästchen, das angibt, ob eine Anmerkung des Benutzers erforderlich ist, wenn eine Regelverletzung auftritt. Die Anmerkung gilt für den Datenpunkt, der die Verletzung verursacht hat. Das SPC-Chart-Anzeigepopup kann erst geschlossen werden, wenn der Datenpunkt mit einer Anmerkung versehen wurde. Wenn der Benutzer versucht, das Popup zu schließen, wird der Benutzer in einer Meldung informiert, dass eine Anmerkung erforderlich ist.

Optional

E-Mail Verteilerliste (kommagetrennt)

E-Mail-Adressen, an die eine Benachrichtigung gesendet wird, wenn eine Verletzung auftritt. Trennen Sie die E-Mail-Adressen mit einem Komma.

Die E-Mail kann DCD-Namen, Variablennamen- und Werte, Chart-Details (wie CL, UCL, LCL), Verletzung und Details zu durchgeführten Außer Kontrolle-Aktionen enthalten, wenn Sie Textvariablen konfigurieren. Auf dem Server muss IE 9 oder höher installiert sein. Die E-Mail-Einstellungen von IE müssen standardmäßig für den E-Mail-Server des Netzwerks eingestellt sein.

Hinweis: Eine weitere Konfiguration der E-Mail-Optionen ist jetzt mithilfe der Datei App.Config möglich.

Optional

E-Mail-Nachricht

Liste der im Modellierungsobjekt "E-Mail-Nachricht" definierten Nachrichten.

Optional

Für angepasste SPC-Datenquelle aufzeichnen

Kontrollkästchen, das angibt, ob Anmerkungen oder Verletzungen für eine von Ihnen bereitgestellte benutzerdefinierte Datenquelle statt für die Anwendungsdatensammlung aufgezeichnet werden. Vergewissern Sie sich, dass dieses Kontrollkästchen aktiviert ist, wenn Sie Diagramme mit Daten außerhalb der Opcenter EX MDD- oder Opcenter EX CR DataStore-Datenbank ausführen.

Optional

Parameter

Nutzerparameter (1-10)

Benutzerdefinierte Werte, die im Format ParamName=Wert eingegeben wurden. In eine Feld können mehrere, durch Kommas getrennte Werte eingegeben werden.

Optional

Vorgehensweise zum Definieren eines neues SPC-Charts

Gehen Sie folgendermaßen vor, um ein neues SPC-Chart zu definieren:

1. Öffnen Sie die Seite SPC-Chart. Die Seite "SPC-Chart" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie SPC-Chart, Charttyp, Auszuwertende Variable, Chartgenerierungsmakro und Anzeigeoptionen für SPC-Chart ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
