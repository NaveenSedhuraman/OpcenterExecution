Definieren von Projektmitarbeitervorlagen

Eine Projektmitarbeitervorlage identifiziert die Mitarbeiter, die bei der Zuweisung von Inhalt zu einem Paket mit Ihnen zusammenarbeiten können. Mit der Projektmitarbeitervorlage können Sie Projektmitarbeiter und deren Anweisungen vordefinieren. Wenn Sie eine Projektmitarbeitervorlage auswählen, füllt die Anwendung die Tabelle "Projektmitarbeiter" und das Feld "Anweisungen für alle Projektmitarbeiter" auf der Seite "Aktualisierungspaket" aus. Wenn Sie eine Vorlage auswählen, ist es nicht erforderlich, alle Informationen jedes Mal manuell in die Tabelle einzugeben.

Informationen zum Aktivieren der Zusammenarbeit und zum Aktualisieren eines Pakets finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.

Beim Definieren von Projektmitarbeitervorlagen

Beachten Sie Folgendes:

• Sie müssen ein Level für die Zusammenarbeit angeben, die jedem Projektmitarbeiter zugewiesen ist. Das Level bestimmt, ob die Arbeit gleichzeitig, sequenziell oder gleichzeitig und sequenziell erfolgt.
• Sie müssen die Rollen auswählen, denen die Verantwortung für die Zusammenarbeit zugewiesen ist.
• Sie müssen die Namen der Mitarbeiter auswählen, die für die Zusammenarbeit verantwortlich sind. Die Rolle bestimmt, welche Mitarbeiter verfügbar sind.
• Sie können allgemeine Anweisungen für alle Projektmitarbeiter und spezielle Anweisungen für einen einzelnen Projektmitarbeiter definieren.
• Sie können Ersatzprojektmitarbeiter definieren, um zu bestimmen, ob Mitarbeiter, die dieselbe Rolle haben wie der Projektmitarbeiter, die Zusammenarbeit ausführen dürfen.

Beim Definieren von Rollenersatz

Sie müssen folgende Aufgaben ausführen, damit ersetzende Mitarbeiter die Zusammenarbeit anstelle des angegebenen Projektmitarbeiters ausführen können:

• Fügen Sie die Ersatzprojektmitarbeiter (Mitarbeiter) und deren Organisationen dem "Rolle"-Modellierungsobjekt des angegebenen Projektmitarbeiters hinzu. Ersatzprojektmitarbeiter müssen zur selben Rolle gehören wie der angegebene Projektmitarbeiter. Weitere Informationen zur Aktualisierung des Modellierungsobjekts "Rolle" finden Sie unter "Definieren von Rollen".
• Wählen Sie "Rolle" als "Ersetzungsoption" für den angegebenen Projektmitarbeiter aus, wenn Sie die Projektmitarbeitervorlage definieren. Wenn Sie "Rolle" auswählen, kann ein Ersatzprojektmitarbeiter die Zusammenarbeit anstelle des angegebenen Projektmitarbeiters ausführen.
• Wenn Sie der Projektmitarbeitervorlage Ersatzprojektmitarbeiter hinzufügen, müssen Sie diese zur selben Ebene hinzufügen, zu der Sie den angegebenen Projektmitarbeiter hinzugefügt haben.

Felddefinitionen der Seite "Projektmitarbeitervorlage"

Diese Tabelle definiert die Felder, die nur auf der Seite "Projektmitarbeitervorlage" und im Popup "Projektmitarbeiter hinzufügen" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Allgemein

Anweisungen für alle Projektmitarbeiter

Anweisungen, die für alle Projektmitarbeiter gelten, die dieser Vorlage zugewiesen sind. Sie können maximal 4000 Zeichen eingeben.

Optional

Tabelle "Projektmitarbeiter"

Tabelle mit allen Projektmitarbeiterrollen und entsprechenden Projektmitarbeitern für diese Vorlage. Die Felder in dieser Tabelle werden in den Popups "Hinzufügen" und "Bearbeiten" angezeigt.

Optional

Level

Das Zusammenarbeitslevel, das dem Genehmiger zugewiesen ist. Sie können nur numerische Werte eingeben. Die Tabelle wird nach dieser Spalte in aufsteigender Reihenfolge sortiert.

Das Level definiert die Weiterleitungsreihenfolge für Projektmitarbeiter. Die Anwendung kann die Zusammenarbeit und E-Mail-Benachrichtigungen parallel, der Reihe nach (seriell) oder in einer Kombination aus beiden an Projektmitarbeiter weiterleiten.

Parallel bedeutet, dass Projektmitarbeiter mit derselben Levelnummer Inhalte gleichzeitig zuweisen können. Seriell bedeutet, dass Projektmitarbeiter mit einer niedrigeren Levelnummer die Zuweisung von Inhalten erst beenden müssen, bevor Projektmitarbeiter mit höheren Levelnummern beginnen können.

Erforderlich

Rolle

Rolle, der die Verantwortung für die Zusammenarbeit zugewiesen ist. Wenn diese Option gewählt ist, wird die Liste "Name" gefüllt.

Erforderlich

Name

Mitarbeiter, dem die Verantwortung für die Zusammenarbeit zugewiesen ist. Die verfügbaren Namen hängen von Ihrer Auswahl im Feld "Rolle" ab. Die Liste wird aktualisiert, wenn Sie die Einstellung im Feld "Rolle" ändern.

Erforderlich

Ersetzungsoption

Ein Wert der angibt, ob Mitarbeiter die Zusammenarbeit ausführen dürfen, die nicht in der Spalte "Name" aufgelistet sind:

• Rolle bedeutet, dass jeder Benutzer mit derselben Rolle, der auf demselben Level in der Tabelle "Projektmitarbeiter" definiert ist, die Zusammenarbeit ausführen kann.
• Kein bedeutet, dass nur der angegebene Benutzer die Zusammenarbeit ausführen darf.

Optional

Dauer

Eine Zahl, die die Gesamtzeit angibt, die für Zusammenarbeit zulässig ist.

Optional

Zeitraum

Maßeinheit (Tage, Stunden, Wochen) für die zulässige Dauer der Zusammenarbeit bis zu deren Abschluss.

Optional

Anweisung für Projektmitarbeiter

Spezielle Anweisungen vom Paketeigentümer für den Projektmitarbeiter. Sie können maximal 4000 Zeichen eingeben.

Optional

Popup "Projektmitarbeiter hinzufügen"

Der Titel des Popups, das zum Hinzufügen oder Bearbeiten von Projektmitarbeitern in der Tabelle "Projektmitarbeiter" auf der Seite "Projektmitarbeitervorlage" verwendet wird, ändert sich dynamisch, in Abhängigkeit von der Aktion, die Sie gerade ausführen. Durch Klicken auf "Neue Zeile hinzufügen" in der Tabelle "Projektmitarbeiter" wird das Popup "Hinzufügen" angezeigt. Wenn Sie einen vorhandenen Projektmitarbeiter in der Tabelle "Projektmitarbeiter" auswählen und auf "Ausgewählte Zeile bearbeiten" klicken, wird das Popup "Bearbeiten" angezeigt.

Vorgehensweise zum Definieren einer Projektmitarbeitervorlage

Gehen Sie folgendermaßen vor, um eine Projektmitarbeitervorlage zu definieren:

1. Öffnen Sie die Seite Projektmitarbeitervorlage. Die Seite "Projektmitarbeitervorlage" wird innerhalb der Seite "Modellierung" angezeigt.
2. Geben Sie einen Namen für diese Projektmitarbeitervorlage in das Feld Projektmitarbeitervorlage ein.
3. Geben Sie Anweisungen für alle Projektmitarbeiter in das Feld Anweisungen für alle Projektmitarbeiter ein.
4. Fügen Sie der Tabelle Projektmitarbeiter mindestens einen Projektmitarbeiter hinzu, indem Sie die Schritte unter "Vorgehensweise zum Hinzufügen von Projektmitarbeitern zu einer Projektmitarbeitervorlage" ausführen.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Projektmitarbeitern zu einer Projektmitarbeitervorlage

Gehen Sie folgendermaßen vor, um einer Projektmitarbeitervorlage Projektmitarbeiter hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Projektmitarbeitervorlage" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Projektmitarbeitervorlage aus.

2. Klicken Sie in der Tabelle "Projektmitarbeiter" auf Neue Zeile hinzufügen. Das Popup "Hinzufügen" wird angezeigt.
3. Geben Sie ein numerisches Zusammenarbeitslevel in das Feld Level ein.
4. Wählen Sie eine Rolle aus.
5. Wählen Sie einen Namen aus.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf OK. Die Anwendung schließt das Popup "Hinzufügen" und zeigt die Projektmitarbeiterinformationen in der Tabelle "Projektmitarbeiter" an.
8. Wiederholen Sie die Schritte 2 bis 7, um weitere Projektmitarbeiter hinzuzufügen:
9. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
