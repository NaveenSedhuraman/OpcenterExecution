Definieren von Berechnungen

Das Modellierungsobjekt "Berechnung" ermöglicht es Ihnen, Prozessberechnungsdefinitionen, zu erstellen und zu verwalten, die für Aufgaben automatischer Abläufe verwendet werden sollen. Sie können die in der Modellierung definierten Benutzerkonstanten verwenden, wenn Sie diese Berechnungen definieren.

Beim Definieren von Berechnungen

Verwenden Sie die Opcenter EX MDD- oder Opcenter EX CR-Syntax für einheitliche Ausdrücke (Unified Expressions), wenn Sie einen Berechnungsausdruck definieren. Ausdrücke können bis zu 4.000 Zeichen umfassen und Folgendes beinhalten:

• Variablen; z. B. (?InMenge)
• Benutzerdefinierte Konstanten; z. B. Constant::Pi
Hinweis: Weitere Informationen zur Syntax für einheitliche Ausdrücke (Unified Expressions) finden Sie im Opcenter Execution Medical Device und Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Falls Ihr Berechnungsausdruck Variablen enthält, fügt die Anwendung die Variablen zur Tabelle mit den Berechnungsvariablen hinzu, wenn Sie das Feld "Berechnungsausdruck" verlassen oder die Berechnung speichern. Der Standarddatentyp für jede Variable ist "Integer". Die Variablen im Ausdruck müssen mit denen in der Tabelle übereinstimmen. Wenn Sie eine Variable im Ausdruck löschen oder ändern, müssen Sie die Variable in der Tabelle manuell löschen oder ändern. Ebenso müssen Sie den Ausdruck manuell aktualisieren, wenn Sie die Variable in der Tabelle ändern oder löschen.

Sie können die Anwendung anweisen, Berechnungsergebnisse einem Container zuzuordnen, wenn die Berechnung für den Container ausgeführt wird. Die Anwendung speichert die Berechnungsergebnisse als benutzerdefiniertes Attribut. Sie können benutzerdefinierte Attribute auf der Seite "Containerattributwartung" anzeigen, bearbeiten und löschen. Informationen zur Verwendung der Seite "Containerattributwartung" finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Beim Definieren von Limits (Grenzen)

Sie können ein oberes und/oder unteres Limit für das Berechnungsergebnis festlegen, wenn Sie eine Berechnung definieren. Sie können einen Ausdruck verwenden, um das obere Limit, das untere Limit oder beide Limits zu definieren. Die Limits werden zur Laufzeit bestimmt, wenn ein Ausdruck verwendet wird.

Hinweis: Bei der Eingabe von Ausdrücken wird keine Validierung der richtigen Syntax vorgenommen. Informationen zur Opcenter EX MDD- oder Opcenter EX CR-Syntax für einheitliche Ausdrücke (Unified Expressions) finden Sie im Opcenter Execution Medical Device und Diagnostics Designer-Benutzerhandbuch und im Opcenter Execution Core Designer-Benutzerhandbuch.

Die Anwendung akzeptiert alle Werte, die der Nutzer eingibt, selbst wenn der Wert außerhalb eines Limits liegt. Die Kontrollkästchen "Limits anzeigen" und "Limitüberschreitung erlaubt" bestimmen, ob die Limits dem Benutzer zur Laufzeit angezeigt werden. Ebenso bestimmen sie die Farbe, die angezeigt wird, wenn ein Wert außerhalb der Limits liegt, und ob die Berechnungsaufgabe des automatischen Ablaufs als "Bestanden" oder "Fehlschlagen" gewertet werden soll.

Limits anzeigen

Wenn dieses Kontrollkästchen aktiviert ist, zeigt die Anwendung zur Laufzeit die obere und untere Grenze an. Wenn der berechnete Wert außerhalb des oberen oder unteren Limits liegt, zeigt die Anwendung einen visuellen Hinweis an, um Sie zu warnen, dass der Wert außerhalb eines Limits liegt.

Limitüberschreitung erlaubt

Wenn dieses Kontrollkästchen aktiviert ist, werden Aufgaben automatischer Abläufe auch dann als bestanden bewertet, wenn ein berechneter Wert außerhalb der angegebenen Grenzen liegt.

Optional

Oberes Limit

Der erlaubte Höchstwert, bei dessen Überschreitung die Anwendung den Benutzer visuell warnt, dass der Wert außerhalb des Limits liegt. Der Wert kann ein statischer alphanumerischer Wert oder ein Ausdruck sein. Die Anwendung zeigt diese Grenze zur Laufzeit an, falls Sie das Kontrollkästchen "Limits anzeigen" aktiviert haben.

Hinweis: Die Ausführung einer Berechnungsaufgabe eines automatischen Ablaufs schlägt fehl, wenn das Kontrollkästchen "Limitüberschreitung erlaubt" nicht aktiviert ist und der berechnete Wert über dieses Limit steigt.

Optional

Unteres Limit

Der erlaubte Mindestwert, bei dessen Unterschreitung die Anwendung den Benutzer visuell warnt, dass der Wert außerhalb des Limits liegt. Der Wert kann ein statischer alphanumerischer Wert oder ein Ausdruck sein. Die Anwendung zeigt diese Grenze zur Laufzeit an, falls Sie das Kontrollkästchen "Limits anzeigen" aktiviert haben.

Hinweis: Wenn Sie eine Berechnungsaufgabe eines automatischen Ablaufs ausführen, schlägt die Aufgabe fehl, wenn das Kontrollkästchen "Limitüberschreitung erlaubt" nicht aktiviert ist und der berechnete Wert unter diese Grenze fällt.

Optional

Zu Containerattribut zuordnen

Bei Auswahl von "Zu Containerattribut zuordnen" werden die Ergebnisse der Berechnung dem Container als Containerattribut zugeordnet.

Optional

Attributname

Bei Aktivierung des Kontrollkästchens "Zu Containerattribut zuordnen" wird der Ergebnisname angezeigt.

Vorgehensweise zum Definieren einer Berechnung

Gehen Sie folgendermaßen vor, um eine Berechnung zu definieren:

1. Öffnen Sie die Seite Berechnung. Die Seite "Berechnung" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen der Berechnung in das Feld Berechnung ein.
4. Geben Sie einen Namen für den berechneten Wert in das Feld Ergebnis ein.
5. Wählen Sie einen Ergebnisdatentyp in der Liste aus.
6. Geben Sie die Formel oder den Ausdruck zur Berechnung des Ergebnisses in das Feld Berechnungsausdruck ein.
7. Klicken Sie auf Speichern oder verlassen Sie das Feld Berechnungsausdruck durch Drücken der Tab-Taste. Sämtliche Variablen in dem Ausdruck werden zur Tabelle "Berechnungsparameterspezifikation" hinzugefügt und ihnen wird der Standarddatentyp "Integer" zugewiesen.
8. Wählen Sie eine Zeile in der Tabelle "Berechnungsparameterspezifikation" aus, um das Feld "Datentyp" verfügbar zu machen.
9. Wählen Sie, falls nötig, einen anderen Datentyp für jede Variable aus.
10. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
11. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
