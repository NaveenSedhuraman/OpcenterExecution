Definieren von Stichprobentests

Die Seite "Stichprobentest" ermöglicht es Ihnen, Stichprobentests zu erstellen, die anschließend gruppiert werden, um Stichprobenpläne zu erstellen. Die zwei Typen von Stichprobentests sind "Gezählt" und "Gemessen".

Mit beiden Stichprobentesttypen können Sie den Container nach Stichprobengröße (wird für zerstörendes Prüfen verwendet) und nach Ablehnungszahl reduzieren, eine Abmelden-Transaktion auch dann ermöglichen, wenn der Container die Stichprobe nicht besteht, und automatisch ein Produktionsereignis erstellen, falls ein Fehler vorliegt.

Wenn ein Stichprobentest aktualisiert wird, nachdem die Stichprobenanforderungen ermittelt und in den Container oder das Los geschrieben wurden, werden diese Anforderungen nicht angewandt. Container oder Lose mit Status "In Bearbeitung" sind nicht von Revisionsaktualisierungen betroffen. Doch der nächste Container oder das nächste Los ist von diesen Änderungen betroffen.

Zum Verständnis von Stichprobentesttypen

Die zwei Typen von Stichprobentests sind "Gezählt" und "Gemessen".

Gezählte Stichprobentests

Ein gezählter Stichprobentest ermöglicht es Ihnen, die Anzahl von getesteten Stichproben und abgelehnten Stichproben aufzuzeichnen. Abgelehnte Stichproben werden aufgezeichnet, indem die Anzahl manuell neben dem entsprechenden Verlustgrund eingegeben wird. Die Liste von Verlustgründen, die bei der Sammlung von Stichprobendaten verfügbar ist, wird durch die AQL-Gruppe "Ablehnungsgrund" festgelegt. Die AQL-Gruppe "Ablehnungsgrund" wird für einen gezählten Test benötigt. Falls Sie Ablehnungen in Ausschuss umwandeln möchten, haben Sie zwei Möglichkeiten:

• Sie können die Ablehnungen in Ausschuss umwandeln, indem Sie den entsprechenden Verlustgrund, der auf der Seite "Stichprobendaten sammeln" ausgewählt wurde, verwenden. Setzen Sie hierzu ein Häkchen in das Kontrollkästchen "Gezählte Ausschussablehnungen nach Grund".
• Wandeln Sie die Ablehnungen mithilfe des Standardgrunds aus dem Feld "Standardgrund für Ausschussablehnungen" in Ausschuss um.

Gemessene Stichprobentests

Ein gemessener Stichprobentest ermöglicht es Ihnen, einen oder mehrere Datenpunktwerte einzugeben, die dann bezüglich entsprechender Grenzen ausgewertet werden. Die Stichprobe wird abgelehnt, falls einer der Werte außerhalb der festgelegten Grenzen liegt. Bei einem Stichprobentest müssen einer oder mehrere Stichprobendatenpunkte ausgewählt sein, falls es ein gemessener Stichprobentest ist. Wenn Ablehnungen in Ausschuss umgewandelt sollen, müssen Sie einen Verlustgrund aus dem Feld "Standardgrund für Ausschussablehnungen" auswählen.

Beim Definieren von Stichprobentests

Die folgenden Regeln gelten beim Modellieren des Stichprobentests:

• Das Feld "AQL-Gruppe für Ablehnungsgründe" ist erforderlich, wenn der Stichprobentyp gleich "Gezählt" ist.
• Mindestens ein Stichprobendatenpunkt ist erforderlich, wenn der Stichprobentyp gleich "Gemessen" ist. Wenn der Stichprobentyp gleich "Gezählt" ist, können Sie keine Stichprobendatenpunkte verwenden.
• Die folgenden Definitionsfelder für Produktionsereignisse müssen ausgefüllt werden, wenn bei einem fehlgeschlagenen Test ein Produktionsereignis erstellt werden soll. Weitere Informationen finden Sie unter "Definieren einer Organisation". Wenn bei einem fehlgeschlagenen Test kein Produktionsereignis erstellt werden soll, lassen Sie diese Felder leer.
• Standardfehlermodus
• Klassifikation
• Unterklassifikation
• Standardbeschreibung für Produktionsereignis

Der Stichprobentest enthält die folgenden, optionalen Modellierungsdefinitionen:

• Fehlermodus
• Klassifikation
• Unterklassifikation
• Verlustgrund
• Verlustgrundgruppe

Die Listen "Standardgrund für Ausschussablehnungen" und "Verringern um Stichprobengröße" werden beide mit dem Verlustgrund gefüllt. Die Liste "AQL-Gruppe für Ablehnungsgründe" wird hingegen mit der Verlustgrundgruppe gefüllt.

"Stichprobentest" ist ein erforderliches Feld in der Definition des Stichprobenplans.

Vorgehensweise zum Definieren eines gemessenen Stichprobentests

Gehen Sie folgendermaßen vor, um einen gemessenen Stichprobentest zu definieren:

1. Öffnen Sie die Seite Stichprobentest. Die Seite "Stichprobentest" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen des Stichprobentests in das Feld Stichprobentest ein.
4. Geben Sie die Revision dieses Stichprobentests in das Feld Revision ein.
5. Wählen Sie in der Liste Stichprobentyp den Typ Gemessen aus.
6. Erweitern Sie den Abschnitt Details.
7. Klicken Sie in der Tabelle "Stichprobendatenpunkte" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
8. Wählen Sie entweder die Standardrevision oder eine bestimmte Revision der Instanz aus.
9. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
10. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Definieren eines gezählten Stichprobentests

Gehen Sie folgendermaßen vor, um einen gezählten Stichprobentest zu definieren:

1. Öffnen Sie die Seite Stichprobentest. Die Seite "Stichprobentest" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen des Stichprobentests in das Feld Stichprobentest ein.
4. Geben Sie die Revision dieses Stichprobentests in das Feld Revision ein.
5. Wählen Sie in der Liste Stichprobentyp den Typ Gezählt aus.
6. Wählen Sie aus der Liste AQL-Gruppe für Ablehnungsgründe die Verlustgrundgruppe aus, die für diesen Stichprobentest verwendet werden soll.
7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen eines Produktionsereignisses zu einem Stichprobentest

Gehen Sie folgendermaßen vor, um einem Stichprobentest ein Produktionsereignis hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines gemessenen Stichprobentests" aus.

Oder

Führen Sie die Schritte im Verfahren "Vorgehensweise zum Definieren eines gezählten Stichprobentests" aus.

Oder

Wählen Sie eine vorhandene Stichprobentest-Instanz aus.

2. Erweitern Sie den Abschnitt Produktionsereignis.
3. Wählen Sie einen Fehlermodus aus der Liste Standardfehlermodus aus.
4. Wählen Sie eine Klassifikation aus der Liste Klassifikation aus.
5. Wählen Sie eine Unterklassifikation aus der Liste Unterklassifikation aus.
6. Geben Sie eine Beschreibung des Produktionsereignisses in das Feld Standardbeschreibung für Produktionsereignis ein.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
