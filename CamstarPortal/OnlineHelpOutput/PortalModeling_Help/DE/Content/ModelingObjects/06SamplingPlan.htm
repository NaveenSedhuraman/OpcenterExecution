Definieren von Stichprobenplänen

Die Seite "Stichprobenplan" wird verwendet, um die Stichprobenaktivitäten, die für ein Produkt durchgeführt werden sollen, zu verwalten.

Ein erstellter Stichprobenplan wird einem Produkt zugewiesen. Weitere Informationen finden Sie unter "Zuweisen von Stichprobenplänen zu Produkten".

Stichprobenrate

Eine Stichprobenrate muss entweder für den gesamten Stichprobenplan oder für jeden Stichprobentest definiert werden. Die Stichprobenrate ist eine Zahl, die die Häufigkeit der Stichproben als jeden N-ten Container angibt. Z. B. ist N=1 jeder Container, N=2 jeder zweite Container, N=3 jeder dritte Container und so weiter. Sie können die Stichprobenrate, die in einem Stichprobenplan angegeben wurde, außer Kraft setzen und die Überprüfung jedes Containers erzwingen, indem Sie das Kontrollkästchen "Alle Container überprüfen" auf der Seite "Produkt" aktivieren. Weitere Informationen finden Sie unter "Zuweisen von Stichprobenplänen zu Produkten".

Sie können die Stichprobenrate ändern, indem Sie den Stichprobenratenzähler auf der Seite "Aktualisierung des aktuellen Stichprobenstatus" bearbeiten. Sie können die Stichprobenrate für ein Stichprobenlos außer Kraft setzen, indem Sie eine Stichprobenrate auf der Seite "Stichprobenlos erstellen" definieren. Sie können die Stichprobenrate für ein Los ändern, indem Sie auf der Seite "Stichprobenlos aktualisieren" eine neue Stichprobenrate eingeben. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Portal Shop Floor-Benutzerhandbuch.

Maximale Dauer

Indem Sie die maximale Dauer definieren, stellen Sie sicher, dass nur Container, die innerhalb der maximalen Dauer des Zeitrahmens produziert wurden, berücksichtigt werden, wenn auf Grundlage der Stichprobenrate festgestellt werden soll, ob Stichproben erforderlich sind. Wenn die maximale Dauer von allen relevanten Containern überschritten wird, wird der Stichprobenzähler zurückgesetzt. So wird verhindert, dass lange, zeitliche Aussetzer die Stichprobenhäufigkeit beeinflussen.

Ein Beispiel: Es wird jeder 10. Container überprüft, aber die Produktion wird beim 5. Container gestoppt. Die Produktion wird einen Monat später wieder aufgenommen. Da die maximale Dauer auf 15 Tage festgelegt wurde, ist der Containerzähler nun veraltet und wird daher auf 1 zurückgesetzt und erneut gestartet. Der 10. Container wird anstatt des 5. untersucht, der eigentlich untersucht werden würde, wäre der Zähler nicht zurückgesetzt worden. So wird der Vergleich von Containern, die zeitlich zu weit auseinanderliegen, vermieden.

Beim Definieren von Stichprobenplänen

Die folgenden Regeln gelten bei der Modellierung des Stichprobenplans:

• Ein Wert in der Tabelle "Details" setzt den im Abschnitt "In Verarbeitung" definierten Standardwert außer Kraft.
• Die standardmäßigen Stichprobentestwerte gelten global für jeden Stichprobentest, es sei denn, sie werden durch die speziellen Werte eines bestimmten Stichprobentests in der Tabelle "Details" vorbelegt.
• Das Standardfeld im Abschnitt "In Verarbeitung" oder das gleichnamige Feld in der Tabelle "Details" kann für diese optionalen Felder ausgefüllt werden:
• Spez.
• AQL-Level
• Ressource
• Lieferantenelement
• Stichprobenrate
• Prüfebene
• Umschaltregel

AQL enthält die folgenden, optionalen Modellierungsdefinitionen:

• AQL-Level
• Prüfebene
• Ressource
• Spez.
• Umschaltregel

AQL enthält die erforderliche Modellierungsdefinition "Stichprobentest".

Wenn ein Stichprobenplan aktualisiert wird, nachdem die Stichprobenanforderungen festgelegt und in den Container geschrieben wurden, werden die aktualisierten Anforderungen nicht sofort angewandt. Anders gesagt: Container oder Lose mit Status "In Bearbeitung" sind nicht von Revisionsaktualisierungen betroffen. Doch der nächste Container oder das nächste Los ist von diesen Änderungen betroffen.

Die Stichprobenrate gilt für alle Stichprobentests, es sei denn, sie wird durch einen entsprechenden Stichprobenratenwert in der Tabelle "Stichprobenplandetails" vorbelegt. Wenn Sie dieses Feld nicht ausfüllen, wird standardmäßig der Wert 1 angegeben.

Vorgehensweise zum Definieren eines Stichprobenplans

Gehen Sie folgendermaßen vor, um einen Stichprobenplan zu definieren:

1. Klicken Sie auf Stichprobenplan. Die Seite "Stichprobenplan" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für den Produktstichprobenplan in das Feld Stichprobenplan ein.
4. Geben Sie die Revision dieses Stichprobenplans in das Feld Revision ein.
5. Führen Sie die Schritte unter "Vorgehensweise zum Definieren von Stichprobenplandetails" aus.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Definieren von Stichprobenplandetails

Gehen Sie folgendermaßen vor, um Stichprobenplandetails zu definieren:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Stichprobenplans" aus.
2. Erweitern Sie den Abschnitt Details.
3. Klicken Sie in der Tabelle "Stichprobenplandetails" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie einen Stichprobentest aus der Liste im Feld Stichprobentest aus, um ihn zu einem Teil dieses Stichprobenplans zu machen.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Wiederholen Sie die Schritte 3 bis 5, um weitere Stichprobenplandetails zu definieren.
7. Klicken Sie auf OK. Das Popup "Hinzufügen" wird geschlossen und die Anwendung zeigt das Objekt "Stichprobenplan" an.
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
