Definieren von Wiederholungsterminanforderungen

Eine Wiederholungsterminanforderung definiert den Wartungsplan einer Ressource basierend auf wiederkehrenden Kalenderdaten, an denen die Wartung stattfinden muss. Jede Wartung muss unabhängig von der Durchführung der vorherigen Wartung erfolgen.

Die Anforderungsdefinition enthält auch die Toleranz- und Warnbereiche (in Tagen) und die entsprechenden E-Mail-Verteilerlisten für E-Mail-Benachrichtigungen zu ausstehenden, fälligen und überfälligen Wartungsereignissen.

Beim Definieren von Wiederholungsterminanforderungen

Die Wiederholungsterminanforderung enthält die erforderliche Wartungsgrund-Modellierungsdefinition und die folgenden optionalen Modellierungsdefinitionen: 

• Datensammlung (parametrisch oder Benutzerdaten)
• Dokumentsatz
• E-Mail-Benachrichtigung
• Anforderung zur elektronischen Unterschrift

Wartungsanforderungen sind erforderlich, wenn Sie eine Wartungsklasse aktivieren.

Muster und Häufigkeit

Die folgenden Wiederholungsterminmuster stehen zur Verfügung:

• Stündlich – In den Zeitplänen ist eine bestimmte Anzahl von Stunden zwischen den Ausführungen festgelegt.
• Täglich – In den Zeitplänen ist eine bestimmte Anzahl von Tagen zwischen den Ausführungen festgelegt.
• Wöchentlich – In den Zeitplänen ist ein bestimmter Wochentag festgelegt.
• Monatlich – In den Zeitplänen ist ein bestimmter Tag des Monats festgelegt.
• Jährlich – In den Zeitplänen sind ein Monat und ein Tag dieses Monats festgelegt.

Allen diesen Intervallen kann eine Häufigkeit zugewiesen werden, aus der sich ein Muster dafür ergibt, wie oft die Anforderung wiederholt wird. Die Zahl 1 bedeutet, dass die Wartung bei jedem geplanten Wiederholungstermin durchgeführt wird. Die Zahl 2 steht für jeden zweiten Wiederholungstermin und so weiter.

Je nach dem, welches Wiederholungsterminmuster Sie auswählen, werden ein oder mehrere der folgenden Felder angezeigt:

Tag des Monats

Der Tag des Monats, an dem die Wartung der Ressource stattfinden soll. Dieses Feld wird angezeigt, wenn das Feld "Wiederholungsterminmuster" den Eintrag "Jährlich" oder "Monatlich" enthält.

Erforderlich

Wochentag

Der Wochentag, an dem die Wartung der Ressource stattfinden soll. Dieses Feld wird angezeigt, wenn das Feld "Wiederholungsterminmuster" den Eintrag "Wöchentlich" enthält. (Wochen gehen von Sonntag bis Samstag.)

Erforderlich

Monat des Jahres

Der Monat, in dem die Wartung an der Ressource durchgeführt werden soll. Dieses Feld wird angezeigt, wenn das Feld "Wiederholungsterminmuster" den Eintrag "Jährlich" enthält. (Zusätzlich muss ein Tag des Monats ausgewählt werden.)

Monat und Tag des Monats müssen eine gültige Kombination sein. Ein Datum wie etwa den 31. Februar führt zu einer Fehlermeldung.

Erforderlich

Bereich der Wiederholung

Erster Termin

Der Zeitpunkt (Datum und Uhrzeit), zu dem die Anforderung fällig wird. Dies setzt eine Aktivierung der Anforderung für die Ressource oder die Wartungsklasse voraus.

Wenn Sie keinen Termin eingeben, werden als Standardwert das aktuelle Datum und die aktuelle Uhrzeit verwendet.

Optional

Anzahl des Auftretens

Gibt an, wie oft die Anforderung auftritt (ein Enddatum wird automatisch berechnet).

Gültig, wenn kein Enddatum angegeben wird; hat Vorrang, wenn beides angegeben wird. (Beide Felder können leer bleiben.)

Optional

Enddatum

Das Datum, ab dem die Wartung nicht mehr dieser Anforderung unterliegt. Gültig, wenn die Anzahl des Auftretens nicht angegeben wird. (Beide Felder können leer bleiben.)

Optional

Warnung und Abweichung

Warnungsfrist

Gibt an, ab dem wievielten Tag vor dem nächsten geplanten Datum die Wartungsanforderung als ausstehend für die entsprechende Ressource gilt.

Löst die "Ausstehend"-E-Mail aus.

Die "Ausstehend"-E-Mail wird nicht versandt, wenn der Standartwert 0 für das Feld zulässig ist.

Optional

Toleranzfrist

Gibt an, ab dem wievielten Tag nach dem nächsten geplanten Datum die Wartungsanforderung als überfällig für die entsprechende Ressource gilt.

Versendet die "Überfällig"-E-Mail und die Ressource steht für die weitere Bearbeitung nicht mehr zur Verfügung.

Wenn der Standardwert 0 zulässig ist, wird die "Überfällig"- E-Mail zum ersten Termin versandt. Die Ressource ist ab sofort nicht mehr für die weitere Verarbeitung verfügbar.

Optional

Benachrichtigungen

Tabelle "Ausstehend - E-Mail-Empfänger"

Ausstehend - E-Mail-Text

Der Text der E-Mail, die versandt wird, wenn die Wartung ausstehend ist. Sie müssen einen Text eingeben, wenn Sie im Feld "Ausstehend - E-Mail-Empfänger" eine Angabe machen. Es können Textvariablen verwendet werden.

Optional

Tabelle "Fällig - E-Mail-Empfänger"

In dieser Tabelle werden die Empfänger angegeben, die eine E-Mail-Benachrichtigung erhalten, wenn die Wartung fällig ist. Siehe das Feld Erster Termin in dieser Tabelle.

Optional

Fällig - E-Mail-Text

Der Text der E-Mail, die versandt wird, wenn die Wartung fällig ist (aber noch in der Toleranzfrist liegt). Sie müssen einen Text eingeben, wenn Sie im Feld "Fällig - E-Mail-Empfänger" eine Angabe machen. Es können Textvariablen verwendet werden.

Optional

Tabelle "Überfällig - E-Mail-Empfänger"

In dieser Tabelle werden die Empfänger angegeben, die eine E-Mail-Benachrichtigung erhalten, wenn die Wartung überfällig ist. Siehe Feld Toleranzfrist in dieser Tabelle. Sie müssen einen Text eingeben, wenn Sie im Feld "Fällig - E-Mail-Empfänger" eine Angabe machen.

Optional

Überfällig - E-Mail-Text

Der Text der E-Mail, die versandt wird, wenn die Wartung überfällig ist. Sie müssen Text eingeben, wenn Sie im Feld "Überfällig - E-Mail-Empfänger" eine Angabe machen. Es können Textvariablen verwendet werden.

Vorgehensweise zum Definieren einer Wiederholungsterminanforderung

Gehen Sie folgendermaßen vor, um eine Wiederholungsterminanforderung zu definieren:

1. Öffnen Sie die Seite Wiederholungsterminanforderung. Die Seite "Wiederholungsterminanforderung" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Anforderung in das Feld Wiederholungsterminanforderung ein und den Revisionsidentifikator für diese Definition in das Feld Revision ein.
4. Erweitern Sie den Abschnitt Ablaufplanungsdetails.
5. Wählen Sie den Grund, der mit dieser Anforderung verknüpft ist, im Feld Wartungsgrund aus.
6. Wählen Sie ein Terminmuster in der Liste Wiederholungsterminmuster aus.
7. Geben Sie eine Zahl in das Feld Häufigkeit ein, um anzugeben, wie oft eine Wartung erforderlich ist.
8. Haben Sie Monatlich, Wöchentlich oder Jährlich als Wiederholungsterminmuster ausgewählt?

a. Geben Sie einen numerischen Wert für Tag des Monats ein.

Wöchentlich
a. Wählen Sie einen Wochentag aus.
b. Gehen Sie zu Schritt 10.

a. Geben Sie einen numerischen Wert für Tag des Monats ein.
b. Wählen Sie einen Monat des Jahres aus.
c. Gehen Sie zu Schritt 10.
10. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
11. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Wiederholungsterminanforderungs-Benachrichtigungen

Gehen Sie folgendermaßen vor, um Wiederholungsterminanforderungs-Benachrichtigungen hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Wiederholungsterminanforderung" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Wiederholungsterminanforderung aus.

2. Expandieren Sie den Abschnitt Benachrichtigungen.
3. Klicken Sie in der Tabelle "Ausstehend - E-Mail-Empfänger", "Fällig - E-Mail-Empfänger" oder "Überfällig - E-Mail-Empfänger" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie eine E-Mail-Benachrichtigung aus der Auswahlliste in der neu hinzugefügten Zeile aus. Die Benachrichtigung wird der Tabelle hinzugefügt.
5. Geben Sie den Text der E-Mail in das entsprechende Feld ein: Ausstehend - E-Mail-Text, Fällig - E-Mail-Text oder Überfällig - E-Mail-Text. Sie haben auch die Möglichkeit, Textvariablen zu verwenden.
6. Wiederholen Sie die Schritte 3 bis 5, um die Benachrichtigungen für weitere Empfänger einzurichten.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
