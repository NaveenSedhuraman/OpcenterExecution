Definieren eines Stammdatenkatalogs

Mit einem Stammdatenkatalog können Sie die Änderungssteuerung für alle Modellierungsobjekte oder eine Untermenge der Modellierungsobjekte erzwingen. Mithilfe des Stammdatenkatalogs gewährleistet die Anwendung, dass Modellierungsobjekt-Instanzen den Änderungsverwaltungspaketen in den jeweiligen Arbeitsplänen zugewiesen werden. Informationen zur Änderungsverwaltung und zum Erzwingen der Änderungssteuerung finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.

Beim Definieren eines Stammdatenkatalogs

Sie können nur einen Stammdatenkatalog definieren. Die Anwendung verhindert die Erstellung von mehr als einem Stammdatenkatalog, indem sie Folgendes verhindert:

• Paketzuordnung – In der Symbolleiste ist Schaltfläche "Paketzuordn." nicht für das Stammdatenkatalog-Objekt verfügbar.
• Hinzufügen von Instanzen – Der Stammdatenkatalog kann in der Tabelle "Objekte" nicht ausgewählt werden.
• Änderungsmanagement-Speicherung – Wenn die Option "Änderungsmanagement-Speicherung" beim Speichern des Stammdatenkatalogs aktiviert ist, wird eine Fehlermeldung eingeblendet.

Sie können die Modellierungsobjekte angeben, für die die Änderungssteuerung erzwungen werden soll.

Der Stammdatenkatalog bietet zwei Typen für das Erzwingen der Änderungssteuerung:

• Erzwingen für Instanzen in einem Paket für einen beliebigen Arbeitsplan – Sie können eine Modellierungsobjekt-Instanz einem Paket zuweisen, das mit einem beliebigen Änderungsmanagement-Arbeitsplan verknüpft ist.
• Erzwingen für Instanzen in einem Paket für einen Arbeitsplan mit einem Genehmigungsschritt – Sie können eine Modellierungsobjekt-Instanz einem Paket zuweisen, das mit einem Änderungsmanagement-Arbeitsplan verknüpft ist, der einen Genehmigungsschritt enthält.

Sie können nur einen Erzwingungstyp auswählen.

In der Tabelle "Objekte" werden standardmäßig alle Modellierungsobjekte angezeigt. Sie können die Tabelle "Objekte" filtern, sodass entweder nur die Modellierungsobjekte mit erzwungener Änderungssteuerung oder nur jene ohne erzwungene Änderungssteuerung angezeigt werden.

Die Anwendung zeigt eine Fehlermeldung an, wenn Sie eine Modellierungsobjekt-Instanz speichern, für die die Änderungssteuerung durchgesetzt wird, und Folgendes gilt:

• Die Instanz ist nicht mit einem Paket verknüpft. Oder
• Die Instanz ist dem falschen Arbeitsplan zugewiesen (z. B. muss die Modellierungsobjekt-Instanz einem Arbeitsplan zugewiesen werden, der einen Genehmigungsschritt enthält, Sie haben sie aber stattdessen einem Arbeitsplan zugewiesen, der keinen Genehmigungsschritt enthält).

Felddefinitionen der Seite "Stammdatenkatalog"

Diese Tabelle definiert die Felder, die nur auf der Seite "Stammdatenkatalog" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Allgemein

Objekteinstellungsfilter

Optionen, die festlegen, welche Modellierungsobjekte in der Tabelle "Objekte" angezeigt werden.

• Alle Objekte anzeigen – Zeigt alle Modellierungsobjekte in der Tabelle "Objekte" an.
• Nur Objekte mit festgelegter Änderungssteuerung anzeigen – Zeigt nur die Modellierungsobjekte an, für die die Änderungssteuerung erzwungen wird.
• Nur Objekte ohne festgelegte Änderungssteuerung anzeigen – Zeigt nur die Modellierungsobjekte an, für die die Änderungssteuerung nicht erzwungen wird.

Erzwingen (jeder Arbeitsplan)

Wenn dieses Kontrollkästchen aktiviert ist, wird die Änderungssteuerung für Instanzen des Modellierungsobjekts erzwungen. Die Modellierungsobjekt-Instanz kann jedem Änderungsmanagement-Arbeitsplan zugewiesen werden.

Zusätzliche Informationen oder Anweisungen für das Modellierungsobjekt. Sie können maximal 255 Zeichen eingeben.

Vorgehensweise zum Definieren eines Stammdatenkatalogs

Gehen Sie folgendermaßen vor, um einen Stammdatenkatalog zu definieren:

1. Öffnen Sie die Seite Stammdatenkatalog. Die Seite "Stammdatenkatalog" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie in das Feld Stammdatenkatalog einen Namen für diesen Stammdatenkatalog ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
