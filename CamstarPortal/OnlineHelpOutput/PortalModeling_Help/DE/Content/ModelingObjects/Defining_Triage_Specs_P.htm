Definieren von Einteilungsspezifikationen

Das Modellierungsobjekt "Einteilungsspez." wird verwendet, um die Standardverarbeitungswerte für die Einteilung von Ereignissen festzulegen. Es definiert die Standardwerte, die bei der manuellen Einteilung verwendet werden, und die bei der automatischen Einteilung anzuwendenden Geschäftsregeln.

Die Anwendung bestimmt die zu verwendende Einteilungsspezifikation basierend auf der Organisation, auf die sich ein Ereignis bezieht. Sie ordnen Einteilungsspezifikationen einer Organisation und Kategorie zu, wenn Sie die Organisation definieren.

Weitere Informationen zu Organisationen finden Sie unter "Definieren einer Organisation".

Beim Definieren einer Einteilungsspezifikation

"Einteilungsspez." enthält die folgenden optionalen Modellierungsdefinitionen: "Rolle", "Eigentümer", "Checklistenvorlage" und "Geschäftsregel".

Nur Geschäftsregeln mit dem Verwendungstyp "Qualitätsobjekt" sind beim Hinzufügen von Geschäftsregeln zu einer Einteilungsspezifikation verfügbar.

Hinweis: Sie müssen die Seite "Geschäftsregeln" in der Modellierung verwenden, um Geschäftsregeln hinzuzufügen oder zu ändern. Weitere Informationen finden Sie unter "Definieren von Geschäftsregeln".

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Tabelle "Einteilungselemente"

Eine Tabelle mit den Standardwerten für Elemente, für die die Einteilung manuell durchgeführt werden muss.

In Kategorie

Kategorie, in die Sie das Element eskalieren können.

Erforderlich

Rolle

Rolle, der Sie die Verantwortung für den Qualitätsdatensatz zuweisen können. Wenn Sie eine Rolle angeben, können Sie den Qualitätsdatensatz einer bestimmten Mitarbeitergruppe zuweisen. So kann jeder in dieser Gruppe die Eigentümerschaft übernehmen.

Die Auswahl in diesem Feld bestimmt die Mitarbeiter, die im Feld "Eigentümer" zur Auswahl stehen. Wenn die Sachbearbeiterrolle z. B. "Standard Qualität" ist, dann können nur Mitarbeiter, die dieser Rolle zugewiesen sind, ausgewählt werden.

Erforderlich

Eigentümer

Ein Eigentümer, dem Sie die Verantwortung für den Qualitätsdatensatz zuweisen können. Die Auswahl im Feld "Rolle" bestimmt, welche Mitarbeiter in diesem Feld zur Auswahl stehen.

Erforderlich

Checklistenvorlage

Die Checklistenvorlage, die Qualitätsdatensätzen zugewiesen wird, die mit dieser Kategorie erstellt wurden. Die Benutzer füllen diese Checkliste beim Bearbeiten des Ereignisses aus.

Hinweis: Wenn auch in den Zuordnungen von Ereignisklassifizierungsspezifikationen eine Checklistenvorlage angegeben ist, hat die hier festgelegte Checklistenvorlage Vorrang.

Optional

Tabelle "Geschäftsregeln"

Eine Tabelle, die eine Liste von Geschäftsregeln enthält, die mit dieser Einteilungsspezifikation verknüpft sind. Die Geschäftsregeln werden in der Tabelle angezeigt und von der Anwendung in der Reihenfolge ausgeführt, in der sie hinzugefügt wurden. Wenn Sie über entsprechende Berechtigungen verfügen, können Sie Geschäftsregeln hinzufügen und entfernen.

Vorgehensweise zum Definieren einer Einteilungsspezifikation

Gehen Sie folgendermaßen vor, um eine Einteilungsspezifikation zu definieren:

1. Öffnen Sie die Seite Einteilungsspez. Die Seite "Einteilungsspez." wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Aufgabenliste in das Feld Einteilungsspez. ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden manuelle Einteilungselemente einer Einteilungsspezifikation hinzugefügt

Gehen Sie folgendermaßen vor, um manuelle Einteilungselemente einer Einteilungsspezifikation hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Einteilungsspezifikation" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Einteilungsspezifikation aus.

2. Gehen Sie zur Tabelle "Einteilungselemente".
3. Klicken Sie auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie eine Eskalationskategorie aus der Liste In Kategorie aus.
5. Wählen Sie eine Rolle aus der Liste aus.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Wiederholen Sie die Schritte 3 bis 6, um zusätzliche manuelle Einteilungselemente hinzuzufügen.
8. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

So werden manuelle Einteilungselemente entfernt

Gehen Sie folgendermaßen vor, um ein manuelles Einteilungselement aus einer Einteilungsspezifikation zu entfernen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Einteilungsspezifikation" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Einteilungsspezifikation aus.

2. Gehen Sie zur Tabelle "Einteilungselemente".
3. Wählen Sie die Zeile aus, die Sie löschen möchten.
4. Klicken Sie auf Ausgewählte Zeile löschen. Die Zeile wird aus der Tabelle entfernt.
5. Wiederholen Sie die Schritte 3 bis 4, um zusätzliche Elemente zu entfernen.
6. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

So werden Geschäftsregeln einer Einteilungsspezifikation hinzugefügt

Gehen Sie folgendermaßen vor, um Geschäftsregeln für die automatische Einteilung zu einer Einteilungsspezifikation hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Einteilungsspezifikation" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Einteilungsspezifikation aus.

2. Gehen Sie zur Tabelle "Geschäftsregeln".
3. Klicken Sie auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie eine Geschäftsregel in der Liste aus.
5. Wiederholen Sie die Schritte 3 bis 4, um zusätzliche Geschäftsregeln hinzuzufügen.
6. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
