Definieren von ERP-Routen

Eine ERP-Route ist das ERP-Konzept, das einem Opcenter EX MDD- oder Opcenter EX CR-Arbeitsplan am nächsten kommt. ERP-Routen werden in Opcenter EX MDD- oder Opcenter EX CR-Arbeitsplänen und in Produktionsaufträgen referenziert. Der Arbeitsplan referenziert die ERP-Route. Somit können einzelne Arbeitsplanschritte ERP-Routenschritte referenzieren. Dies erfolgt häufig für ERP-Meilensteine oder -Lagerorte. Der Produktionsauftrag referenziert die ERP-Route, sodass Materiallistenelemente die ERP-Routenschritte referenzieren, in denen die Komponenten verbraucht werden.

Beim Definieren einer ERP-Route

ERP-Route ist ein erforderliches Objekt, um die Materiallistenelemente eines Produktionsauftrags mit dem Arbeitsplan für die Transaktion "Komponentenverbrauch" zu verknüpfen.

Die ERP-Route wird verwendet, um die Opcenter EX MDD- oder Opcenter EX CR-Transaktionen "Abmelden" und "Komponentenverbrauch" mit dem ERP-Schritt zu verknüpfen, in dem die Transaktion durchgeführt wurde. Schritte in einer ERP-Route werden normalerweise auf einer zusammengefassteren Ebene definiert, als für Schritte in einem Opcenter EX MDD- oder Opcenter EX CR-Arbeitsplan.

"ERP-Route" enthält die optionale Modellierungsdefinition "Produkt".

"ERP-Route" ist ein optionales Feld in der Definition von "Produktionsauftrag".

Beim Definieren von ERP-Routen für die Verwendung mit Opcenter Execution Core Scheduling
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Sie müssen die Verarbeitungszeit für jede ERP-Operation angeben, in der ein Produktionsauftrag oder Produkt verarbeitet wird. Sie können in der Tabelle "Routenschritt" auf der Seite "ERP-Route" folgende Durchlaufgeschwindigkeiten definieren:

• Stunden pro Einheit
• Einheiten pro Stunde
• Zeit pro Batch

Sie können nur eine Option auswählen, um die Durchlaufgeschwindigkeit zu definieren.

Sie müssen einem Arbeitsplan eine ERP-Route zuweisen. Sie können dann den Arbeitsplan dem Produkt oder der Produktfamilie zuweisen.

Beim Festlegen von Einrichtungszeiten für einen Routenschritt
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Sie können die erforderliche Zeit zum Einrichten einer Ressource für die Verarbeitung in einem Routenschritt definieren. Beispiel: Sie haben eine Arbeitsmaschine, die zwischen Durchläufen gereinigt werden muss. Die Durchführung der Reinigung dauert dreißig Minuten. Sie können diese Zeit im Feld "Einrichtungszeit" angeben.

Vorgehensweise zum Definieren einer ERP-Route

Gehen Sie folgendermaßen vor, um eine ERP-Route zu definieren:

1. Öffnen Sie die Seite ERP-Route. Die Seite "ERP-Route" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die ERP-Route in das Feld ERP-Route ein.
4. Geben Sie eine Revision in das Feld Revision ein.
5. Weitere Informationen zum Hinzufügen von Routenschritten finden Sie unter "So werden einer ERP-Route Routenschritte hinzugefügt".
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden einer ERP-Route Routenschritte hinzugefügt

Gehen Sie folgendermaßen vor, um einer ERP-Route Routenschritte hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer ERP-Route" aus.

Oder

Wählen Sie eine vorhandene Instanz einer ERP-Route aus.

2. Erweitern Sie den Abschnitt Details.
3. Klicken Sie in der Tabelle "Routenschritte" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Geben Sie den Namen dieses Routenschrittes in das Feld Routenschrittname ein.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Wiederholen Sie die Schritte 3 bis 5, um weitere Routenschritte hinzuzufügen.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
