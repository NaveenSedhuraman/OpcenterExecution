Erstellen von Benutzerdatensammlungsdefinitionen

Eine Benutzerdatensammlungsdefinition enthält einen Satz von Datensammlungsparametern. Diese Datensammlungsparameter werden Benutzern bei der Durchführung von Aktivitäten zur Verfügung gestellt.

Die Benutzerdatensammlungsdefinition wird von der Aktivitätsvorlage, die sie verwendet, referenziert.

Jeder Datensammlungsparameter enthält einen Satz von Datenpunkten. Ein Datenpunkt stellt einen Punkt in einem Geschäftsprozess dar, an dem Daten erfasst werden können oder müssen. Mehrere definierte Datenpunkte können zu einer Benutzerdatensammlungsdefinition zusammengefasst werden.

Benutzerdatensammlungsdefinitionen erlauben die Erfassung von Daten im Rahmen einer Aktivität. Dabei kann wahlweise eine Überprüfung anhand der oberen und unteren Limits, die in der Datenpunktdefinition festgelegt wurden, erfolgen.

Sie können z. B. einen Wertdatenpunkt, wie einen booleschen Wert, angeben. In diesem Fall müssen die oberen und unteren Datenpunkte identisch sein. Daten außerhalb dieses Werts würden dazu führen, dass die Datensammlungsaufgabe zu einem Fehler führt, der in der Historie aufgezeichnet wird.

Alternativ können Sie auch einen Objektdatenpunkt festlegen. Ein Objektdatenpunkt ermöglicht die Erfassung von Daten und referenziert dabei eine bestimmte Aktivität Ihres Prozessmodells.

Die Anwendung erlaubt Ihnen, Folgendes hinzuzufügen:

• Wertdatenpunkte – mit dem Popup "Wertdatenpunktdetail".
• Objektdatenpunkte – mit dem Popup "Objektdatenpunktdetail".

Beide werden von der Seite "Benutzerdatensammlungsdefinition" aus aufgerufen.

Beim Definieren einer Benutzerdatensammlung

Die Anwendung bietet zwei grundlegende Layouts für die benutzerbasierte Datensammlung an:

• Zeile und Spalte (einzelne Felder)
• Iterationstabelle

Sie können das Layout im Feld "Datenpunktlayout" auf der Seite "Benutzerdatensammlungsdefinition" festlegen.

Die Benutzerdatensammlungsdefinition enthält die optionale Modellierungsdefinition "SPC-Chartgruppe".

Zeilen- und Spalten-Layout

Das Ergebnis des Zeilen- und Spalten-Layouts ist eine standardmäßige feldbasierte Seite.

Iterationstabelle

Das Ergebnis des Iterationstabellen-Layouts ist eine Tabelle, die aus mehreren Zeilen und Spalten besteht. Sie können die Ausrichtung von Informationen, die zwischen vertikalen und horizontalen Ansichten angezeigt werden, über die Schaltfläche "Richtung" in der oberen rechten Ecke der Tabelle ändern.

Beim Hinzufügen von Objektdatenpunkten

Ein Objektdatenpunkt dient im Wesentlichen demselben Zweck wie ein Wertdatenpunkt – er bietet eine Möglichkeit, Daten an einem bestimmten Punkt im Produktionsprozess zu sammeln. Ein Objektdatenpunkt referenziert ein bestimmtes Objekt. Sie können z. B. einen Objektdatenpunkt verwenden, um einen Ursachencode zu erstellen und vordefinierte Prüfstatusergebnisse aufzuzeichnen. Diese Ergebnisse können Bedingungen wie Kontamination und Oberflächenkratzer enthalten. Der Bediener führt die Untersuchungsaufgabe in der Fertigung durch, indem er einfach das entsprechende Ergebnis aus einer Wertauswahlliste auswählt.

Es ist nicht möglich, zwei Datenpunkte mit derselben Zeilen- und Spaltenposition erstellen.

Hinweis: Für Objektdatenpunkte gelten keine Limits.

Beim Hinzufügen Wertdatenpunkte

Wenn Sie einer Benutzerdatensammlung Wertdatenpunkte hinzufügen, haben Sie folgende Möglichkeiten:

• Definieren von oberen und unteren Limits durch Eingeben statischer Werte oder eines Ausdrucks
• Angeben, ob Limitüberschreitungen erlaubt sind
• Sich für eine Zuordnung des Datenpunktwerts zu einem Containerattribut entscheiden

Sie können einen Ausdruck verwenden, um das obere Limit, das untere Limit oder beide Limits zu definieren. Die Limits werden zur Laufzeit bestimmt, wenn ein Ausdruck verwendet wird.

Verwenden Sie die Opcenter EX MDD- oder Opcenter EX CR-Syntax für einheitliche Ausdrücke (Unified Expressions), wenn Sie obere und untere Limits mit einem Ausdruck definieren. Ausdrücke können bis zu 255 Zeichen umfassen und Folgendes beinhalten:

• Feldausdrücke, z. B. Container.CurrentStatus.Spec.UserDefinedInteger
• Eine Kombination aus einen Feldausdruck und einfacher Arithmetik, z. B. (100-Container.CurrentStatus.Spec.UserDefinedInteger) * 0,2
Hinweis: Bei der Eingabe von Ausdrücken wird keine Validierung der richtigen Syntax vorgenommen. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Erlauben Limitüberschreitungen für Wertdatenpunkte

Wenn Sie der Benutzerdatensammlungsdefinition einen Wertdatenpunkt hinzufügen, können Sie einem Benutzer erlauben, Werte außerhalb der unteren und oberen Limits einzugeben. Wenn Sie das Kontrollkästchen "Limitüberschreitung erlaubt" aktivieren, färbt die Anwendung das Limitfeld orange, sobald ein Benutzer einen Wert außerhalb eines Limits eingibt. Die Anwendung ermöglicht es dem Benutzer, die Datensammlung auszuführen.

Wenn Sie das Kontrollkästchen "Limitüberschreitung erlaubt" deaktivieren, färbt die Anwendung das Limitfeld rot, wenn ein Benutzer einen Wert außerhalb eines Limits eingibt.

Hinweis: Diese Funktion ist sowohl im Zeilen- und Spalten-Layout als auch für die Iterationstabelle verfügbar.

Aktivieren der QuickInfos für die Datensammlung

Bei der Datensammlung können Sie QuickInfos verwenden. Bewegen Sie den Mauszeiger einfach über die Dateneingabefelder der Aufgabe, um die Datenpunktwerte anzuzeigen: Maßeinheit und untere und obere Limits. Diese Funktion ist sowohl im Zeilen- und Spalten-Layout als auch für die Iterationstabelle verfügbar.

Vorgehensweise zum Definieren einer Benutzerdatensammlungsdefinition

Gehen Sie folgendermaßen vor, um eine Benutzerdatensammlungsdefinition zu definieren:

1. Öffnen Sie die Seite Benutzerdatensammlungsdefinition. Die Seite "Benutzerdatensammlungsdefinition" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den eindeutigen Namen dieser Definition in das Feld Benutzerdatensammlungsdefinition ein.
4. Geben Sie den Revisionsidentifikator für diese Definition in das Feld Revision ein.
5. Aktivieren Sie das Kontrollkästchen Standardrevision, falls dies die Standardrevision ist.
6. Wählen Sie den Typ Datenpunktlayout aus der Liste aus: Iterationstabelle oder Zeilen/Spaltenposition.
7. Informationen zum Hinzufügen des ausgewählten Datenpunkttyps finden Sie in den nachfolgenden Verfahren:
• "Vorgehensweise zum Hinzufügen eines Wertdatenpunkts"
• "So wird ein Objektdatenpunkt hinzugefügt"
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen eines Wertdatenpunkts

Gehen Sie folgendermaßen vor, um der Benutzerdatensammlungsdefinition einen Wertdatenpunkt hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Benutzerdatensammlungsdefinition" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Benutzerdatensammlungsdefinition aus.

2. Erweitern Sie den Abschnitt Details.
3. Klicken Sie auf Neue Zeile hinzufügen in der Tabelle "Datenpunkte" und wählen Sie Wertdatenpunkt im Popup "Datenpunkttyp" aus. Das Popup "Wertdatenpunktdetail" wird angezeigt.
4. Geben Sie einen eindeutigen Namen für diesen Wertdatenpunkt in das Feld Datenpunktname ein.
5. Geben Sie einen numerischen Wert ein, um die Zeilenposition festzulegen, an der der Wertdatenpunkt in der Aufgabe platziert werden muss.
6. Geben Sie einen numerischen Wert ein, um die Spaltenposition festzulegen, an der der Wertdatenpunkt in der Aufgabe platziert werden muss.
7. Wählen Sie den Datentyp aus der Liste aus.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf OK. Das Popup "Wertdatenpunkt" wird geschlossen und der Wertdatenpunkt wird in der Tabelle "Datenpunkte" der Benutzerdatensammlungsseite angezeigt.

So wird ein Objektdatenpunkt hinzugefügt

Gehen Sie folgendermaßen vor, um der Benutzerdatensammlungsdefinition einen Objektdatenpunkt hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Benutzerdatensammlungsdefinition" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Benutzerdatensammlungsdefinition aus.

2. Erweitern Sie den Abschnitt Details.
3. Klicken Sie auf Neue Zeile hinzufügen in der Tabelle "Datenpunkte" und wählen Sie Objektdatenpunkt im Popup "Datenpunkttyp" aus. Das Popup "Objektdatenpunktdetail" wird angezeigt.
4. Geben Sie einen eindeutigen Namen für diesen Objektdatenpunkt in das Feld Datenpunktname ein.
5. Geben Sie einen numerischen Wert ein, um die Zeilenposition festzulegen, an der der Objektdatenpunkt in der Aufgabe platziert werden muss.
6. Geben Sie einen numerischen Wert ein, um die Spaltenposition festzulegen, an der der Objektdatenpunkt in der Aufgabe platziert werden muss.
7. Wählen Sie das entsprechende Objekt, das diesem Objektdatenpunkt zugewiesen ist, aus der Liste Objekttyp aus.
8. Wählen Sie den Werttyp, der für den Objekttyp verwendet werden soll, aus der Liste Typ für Auswahlwert aus.
9. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
10. Klicken Sie auf OK. Das Popup "Wertdatenpunkt" wird geschlossen und der Wertdatenpunkt wird in der Tabelle "Datenpunkte" der Benutzerdatensammlungsseite angezeigt.
