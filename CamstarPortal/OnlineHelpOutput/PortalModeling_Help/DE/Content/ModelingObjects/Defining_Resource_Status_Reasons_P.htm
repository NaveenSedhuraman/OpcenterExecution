Definieren von Ressourcenstatusursachen

Eine Ressourcenstatusursache liefert zusätzliche Informationen zur Ursache für den aktuellen Status einer Ressource. Zur Reparatur geschickt könnte z. B. die Ursache für den Status Reparatur einer Ressource beschreiben.

Beim Definieren einer Ressourcenstatusursache

Ähnliche Statusursachen können in einer Ressourcenstatusursachengruppe zusammengefasst werden. "Ressourcenstatusursache" ist ein optionales Feld in der Modellierungsdefinition "Ressourcenstatusursachengruppe".

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Vorgehensweise zum Definieren einer Ressourcenstatusursache

Gehen Sie folgendermaßen vor, um eine Ressourcenstatusursache zu definieren:

1. Öffnen Sie die Seite Ressourcenstatusursache. Die Seite "Ressourcenstatusursache" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Statusursache in das Feld Ressourcenstatusursache ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Unter "Gemeinsame Felder auf Modellierungsseiten" finden Sie Informationen zu den optionalen Feldern.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
