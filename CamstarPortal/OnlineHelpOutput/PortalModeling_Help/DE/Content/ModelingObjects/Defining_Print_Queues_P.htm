Definieren von Druckwarteschlangen

Die Druckwarteschlange ist das Objekt, das die Netzwerkadresse des Druckers bestimmt, auf dem das Container- oder Abweichungsetikett gedruckt wird.

Beim Definieren einer Druckwarteschlange

Die Seite "Druckwarteschlange" bietet die Möglichkeit, Vorlagen in einer Datei zu speichern, statt sie zu drucken. Wenn Sie diese Option aktivieren, werden die Vorlageninformationen in eine .txt-Datei geschrieben. Das Verzeichnis und der Basisdateiname werden dabei von Ihnen vorgegeben. Für die Textdatei selbst gilt folgende vordefinierte Namenskonvention:

<base file name>_<name of transaction>_<date stamp>_<time stamp>_<thread ID>_ <process ID>_<label count>.txt

Beispiel: Angenommen, das Feld Druckdatei enthält den Eintrag server2\myfile und bei der Starttransaktion soll ein Etikett gedruckt werden. Die resultierende, auf Server2 gespeicherte Datei könnte in diesem Fall folgendermaßen benannt werden:

myfile_Start_20061114_225850513_2332_1944_1.txt

Sie können einen Etikettenkodierungstyp angeben, wenn Sie eine Druckwarteschlange definieren. Mögliche Typen sind: ASCII, UTF-16 BE, UTF-16 LE und UTF-8. Wählen Sie den Kodierungstyp aus, der mit dem Drucker kompatibel ist, an den Druckaufträge gesendet werden.

Wenn Sie die Option "In eine Datei schreiben" auswählen, kodiert die Anwendung die Vorlage in einer Datei. Liegt die Vorlagendatei im ANSI-Format vor, kodiert die Anwendung die Vorlagendatei in dem im Feld "Etikettenkodierung" festgelegten Format, wenn die Option "In eine Datei schreiben" ausgewählt wird. Liegt die Vorlagendatei nicht im ANSI-Format vor, kodiert die Anwendung die Datei im Format UTF-16 LE.

Bitte beachten Sie bei der Definition einer Druckwarteschlange Folgendes:

• Der Objektname darf keine ungültigen Zeichen enthalten. Z. B. darf kein Apostroph verwendet werden.
Hinweis: Informationen zu gültigen Zeichen und reservierten Wörtern finden Sie im Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch und im Opcenter Execution Core Designer-Benutzerhandbuch.
• "Druckwarteschlange" ist ein optionales Feld in der Definition einer Ressource und einer Operation.

Felddefinitionen der Seite "Druckwarteschlange"

In dieser Tabelle sind die Felder definiert, die nur auf der Seite "Druckwarteschlange" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Details

Pfad für Druckwarteschlange

Der Netzwerkpfad des Druckers, auf dem Containeretiketten gedruckt werden.

Erforderlich

Etikettenkodierung

Liste der Kodierungsformate Die folgenden Optionen sind verfügbar:

• ASCII
• UTF-16 BE
• UTF-16 LE
• UTF-8

Wählen Sie den Typ aus, der mit dem Drucker kompatibel ist, an den Druckaufträge gesendet werden. Der Standardwert für den Druckprozess ist UTF-16 LE, sollten Sie keinen anderen Wert auswählen.

Optional

In eine Datei schreiben

Kontrollkästchen, das angibt, ob die Vorlagendaten in eine Datei geschrieben und nicht an einen Drucker gesendet werden sollen. Die Datei wird mit der Methode kodiert, die Sie im Etiketten-Kodierungs-Feld ausgewählt haben.

Hinweis: Die Vorlage wird im Format UTF-16 LE kodiert, wenn das Vorlagenformat nicht ANSI ist. Liegt die Vorlage im ANSI-Format vor, wird sie in dem Format kodiert, das im Feld "Etikettenkodierung" ausgewählt wurde.

Optional

Druckdatei

Die Netzwerkadresse (und optional der Basisname) der Datei, in die die Vorlagendaten geschrieben werden, wenn "In eine Datei schreiben" aktiviert ist. Der Basisname wird, falls angegeben, am Anfang des vordefinierten Dateinamens der Textdatei eingefügt.

Optional

Vorgehensweise zum Definieren einer Druckwarteschlange

Gehen Sie folgendermaßen vor, um eine Druckwarteschlange zu definieren:

1. Öffnen Sie die Seite Druckwarteschlange. Die Seite "Druckwarteschlange" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen der Druckwarteschlange in das Feld Druckwarteschlange ein.
4. Geben Sie den Pfad des Netzwerkdruckers, auf dem die Containeretiketten gedruckt werden sollen, in das Feld Pfad für Druckwarteschlange ein.
5. Sollen die Vorlagendaten nicht ausgedruckt, sondern in eine Datei geschrieben werden?

Antwort . . .

Dann . . .

Ja
a. Aktivieren Sie die Option In eine Datei schreiben.
b. Wählen Sie im Feld Etikettenkodierung ein Kodierungsformat aus, wenn Sie eine Vorlage im ANSI-Format kodieren.
c. Geben Sie den entsprechenden Netzwerkpfad in das Feld Druckdatei ein.
d. Gehen Sie zu Schritt 6.

Nein

Gehen Sie zu Schritt 6.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
