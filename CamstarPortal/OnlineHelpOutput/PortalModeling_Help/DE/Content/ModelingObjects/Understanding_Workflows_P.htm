Grundlegendes zu Arbeitsplänen

Ein Arbeitsplan ist eine Abfolge von Schritten, die verwendet werden, um ein Produkt herzustellen. Eine Produktdefinition kann auf einen Standardarbeitsplan verweisen, der die Route und die Bearbeitung definiert, die erforderlich sind, um das Produkt herzustellen. Arbeitspläne können mehrere Elemente enthalten: 

• Schritte
• Pfade
• Routen
• Unterarbeitspläne

Schritte

Ein Schritt definiert die Bearbeitung am betreffenden Punkt im Arbeitsplan. Jeder Schritt bezieht sich auf eine Reihe von Anweisungen, die eine Kombination der folgenden Elemente beinhalten können:

• Von einem Mitarbeiter auszuführende Aufgaben
• Ereignisse, die zur Bearbeitung des Containers im betreffenden Schritt erforderlich sind

Jeder Schritt referenziert eine Spezifikation oder einen anderen Arbeitsplan (Unterarbeitsplan). Eine Spezifikation ist ein revisioniertes Objekt, das die im betreffenden Schritt ausgeführten Maßnahmen definiert. Die Spezifikation enthält die Prozessanweisungen für die Ausführung der Arbeiten. Weitere Informationen zu Spezifikationen finden Sie unter "Definieren von Spezifikationen".

Ein Arbeitsplanschritt ist ein Schritt in einem Arbeitsplan oder Unterarbeitsplan, der über eine Spezifikation auf einen anderen Arbeitsplan verweist.

Ein Schritt kann null Pfade, einen Pfad oder mehrere Pfade enthalten. Wenn ein Schritt mehrere Pfade enthält, gibt es in der Regel einen Standardpfad zu einem weiteren Schritt und alternative Pfade zu anderen Schritten.

Pfade

Der Weg von einem Schritt zum nächsten heißt Pfad. Ein Arbeitsplan kann alternative Pfade von jedem beliebigen Schritt aus enthalten. Wenn alternative Pfade verfügbar sind, können Sie mithilfe von Pfadselektoren eine bestimmte Richtung angeben.

• Standard – Gekennzeichnet durch eine durchgezogene Linie. Ein Container folgt dem Standardpfad, es sei denn, eine Pfadselektoranweisung für einen alternativen Pfad ergibt wahr.
• Alternativ – Gekennzeichnet durch eine gepunktete Linie. Alternative Pfade und Routen können verwendet werden, wenn eine Bearbeitung außerhalb der Standardroute erforderlich ist. Sie können mehrere alternative Pfade von einem Schritt aus erzeugen und auch alternative Routen verschachteln. Ein Container folgt einen alternativen Pfad, wenn der Pfadselektor für diesen Pfad "wahr" ergibt.
• Nacharbeit – Gekennzeichnet durch eine rote gepunktete Linie. Nacharbeit ist ein Fertigungsprozess, bei dem die aktuellen Arbeiten zusätzliche Bearbeitungsschritte durchlaufen, weil ein Problem korrigiert werden muss, das bei der regulären Bearbeitung aufgetreten ist. Ein Schleifenpfad ist ein Typ von Nacharbeitspfad. Er führt den Container wieder an den Anfang des aktuellen Schritts zurück. Weitere Informationen finden Sie unter "Nacharbeitspfade".
• Schleife – Gekennzeichnet durch eine Linie, die zurück zu dem Schritt verweist, in dem sich der Benutzer bereits befindet. Ein Schleifenpfad führt einen Container wieder an den Anfang des aktuellen Schritts zurück, wenn ein Grund zur nochmaligen Bearbeitung des Containers in demselben Schritt vorliegt. Falls ein Container z. B. nicht den angegebenen parametrischen Datenanforderungen in einem bestimmten Schritt genügt, wird er in demselben Schritt erneut bearbeitet, bis er die Anforderungen erfüllt. Wenn die Anforderungen erfüllt sind, kann der Container seinen Weg entlang des Pfads fortsetzen.

Nacharbeitspfade

Nacharbeit ist definiert als ein Fertigungsprozess, bei dem zusätzliche Bearbeitungsschritte durchlaufen werden, um ein Problem oder unerwünschtes Ergebnis zu korrigieren, das während des regulären Prozesses auftrat.

Einige Geschäftsregeln erfordern einen Pfad von einem Schritt zurück zu sich selbst oder zu einem der vorherigen Schritte. Z. B. muss in einem bestimmten Schritt ein Metallcoil auf eine bestimmte Dicke gewalzt werden, bevor er am nächsten Schritt angemeldet werden kann. Ist die Dicke nicht korrekt, wird der Coil erneut gewalzt, bis die richtige Dicke erreicht ist. Dieser Ablauf kann mit Schleifenpfaden realisiert werden.

Im Rahmen der Fertigung kann es auch erforderlich werden, dass der Bediener eine aktuelle Arbeit erneut durch vorhergehende Schritte leitet. Falls ein Produkt z. B. nicht die Qualitätsstandards erfüllt, muss es die vorherigen Prozessschritte evtl. erneut durchlaufen. Sie erreichen dies, indem Sie Nacharbeitspfade erstellen.

Hinweis: Ein Bediener kann ein Produkt nur dann zur Nacharbeit schicken, wenn während der Modellierung vordefinierte Pfade eingerichtet wurden. Der Bediener muss außerdem die besonderen Einschränkungen beachten, denen die vordefinierten Nacharbeitspfade unterliegen. Ein Produkt kann z. B. nur dann vom Schritt "Stanzen" aus den Nacharbeitspfad "Zusätzliche Arbeit" durchlaufen, wenn diese besonderen Einschränkungen im Arbeitsplan definiert sind.

Routen

Eine Route ist eine Folge von Schritten, die über Pfade miteinander verbunden sind. Ein Arbeitsplan kann mehrere Routen mit einer Standardroute enthalten. Pfadselektoranweisungen bestimmen die Route (Standard- oder Alternativ-), der ein Container durch den Arbeitsplan folgt. Weitere Informationen finden Sie unter "Verwenden von Pfadselektoren". Beispiele für alternative Routen können Nacharbeit oder Produkte sein, die zusätzliche oder andere Bearbeitungsschritte erfordern.

Unterarbeitspläne

Ein Unterarbeitsplan ist ein unabhängiger Arbeitsplan, der in einem Arbeitsplanschritt referenziert wird. Wenn ein bestimmter Schritt kompliziert ist, muss möglicherweise ein Unterarbeitsplan referenziert werden, dem eigenen Schritte und Spezifikationen zugewiesen sind. Unterarbeitspläne sind keine alternativen Pfade, sondern eigenständige, unabhängige Arbeitspläne. Sie werden zu Unterarbeitsplänen, wenn sie in einem bestimmten Schritt eines anderen Arbeitsplans referenziert werden.

Unterschiedliche Arbeitspläne können denselben Arbeitsplan als Unterarbeitsplan referenzieren. Werden z. B. an zwei physischen Orten ähnliche Artikel hergestellt, dann verwenden die Orte eventuell verschiedene Revisionen des Hauptarbeitsplans und referenzieren dann an einem bestimmten Punkt denselben Unterarbeitsplan.

Manchmal sind die Bearbeitungsdetails in einem Schritt so kompliziert, dass sie nicht in einer Spezifikation erfasst werden können. In diesem Fall kann ein Schritt einen anderen Arbeitsplan referenzieren (wobei der referenzierte Arbeitsplan zum Unterarbeitsplan wird). Unterarbeitspläne sind unabhängige Arbeitspläne und können von mehreren Arbeitsplänen referenziert werden.

Sie können einen referenzierten Unterarbeitsplan im Arbeitsplandiagramm öffnen, indem Sie den Schritt für den Unterarbeitsplan auswählen und auf die Schaltfläche "Unterarbeitsplan öffnen" klicken. Der Unterarbeitsplan wird in einer separaten Registerkarte angezeigt, auf der Sie den Unterarbeitsplan anzeigen und ändern können.

Bewegen von Containern durch einen Arbeitsplan

Pfade und Routen helfen, die Containerbewegung in der Fertigung zu steuern. Ein Container kann nicht mit der Transaktion "Abmelden - standardmäßig" von der Standardroute bewegt werden, wenn kein alternativer Pfad definiert ist.

Sie können die Anwendung so konfigurieren, dass der richtige Pfad auf Basis der Containermerkmalen ausgewählt wird, indem Sie Pfadselektoranweisungen erstellen, die automatisch den zu benutzenden Pfad bestimmen. Weitere Informationen finden Sie unter "Verwenden von Pfadselektoren".

Arbeiten mit Schrittdetails

Zu jedem Schritt liegen Details vor, die seine Verwendung im Arbeitsplan definieren. Viele Details sind Anwendungsstandardwerte, die Sie ändern können. Änderungen, die Sie an einem Schritt innerhalb eines Arbeitsplans vornehmen, beeinflussen nur diesen Schritt. Sie können die folgenden Arten von Informationen für einen Schritt ändern:

• Allgemein – Beinhaltet Werte wie Schrittname und Beschreibung, die Reihenfolge des Schritts im Arbeitsplan, die zugehörige Spezifikation oder den Unterarbeitsplan usw. Sie können alle allgemeinen Informationen außer der Reihenfolge ändern. Beim Ändern von Spezifikations- oder Unterarbeitsplaninformationen können Sie die Spezifikation bzw. den Unterarbeitsplan als Standardrevision oder als eine bestimmte Revision festlegen.
• Pfadselektoren – Pfadselektoren definieren alternative Pfade auf Basis bestimmter Bedingungen, die den Standardpfad überschreiben. Die Anwendung prüft, ob Pfadselektoren vorliegen, bevor sie den Standardpfad verwendet. Weitere Informationen finden Sie unter "Verwenden von Pfadselektoren".
• Nacharbeitspfadselektoren – Nacharbeitspfadselektoren erfüllen den gleichen Zweck wie Pfadselektoren. Falls ein Container Nacharbeit benötigt und mehr als ein Nacharbeitsschritt möglich ist, definieren Sie Nacharbeitspfadselektoren, um die Bedingungen anzugeben, unter denen die Container dem einen oder dem anderen Nacharbeitspfad folgen.
Hinweis: Der Benutzer kann Nacharbeitspfadselektoren überschreiben, indem er manuell während der Nacharbeitstransaktion einen anderen Pfad auswählt.
• Ablaufplanungsdetails – Standardwerte aus der zugehörigen Spezifikation oder dem Unterarbeitsplan. Sie können Ablaufplanungsinformationen ändern, um schrittspezifische Anforderungen abzubilden. Die standardmäßigen Ablaufplanungsinformationen in der zugehörigen Spezifikation oder dem zugehörigen Unterarbeitsplan werden mit den Informationen, die Sie hier eingeben, überschrieben. Dies hat jedoch keine Auswirkungen auf die Spezifikation oder den Unterarbeitsplan.

Wählen Sie den Schritt im Arbeitsplan und klicken Sie auf die Schaltfläche "Element ändern", um das Popup "Schrittdetails" aufzurufen.

Arbeiten mit Pfaddetails

Ebenso wie Schritte hat auch jeder Pfad Details, die seine Verwendung im Arbeitsplan festlegen. Viele dieser Details sind Anwendungsstandardwerte, die Sie ändern können. Änderungen an einem Pfad im Arbeitsplan betreffen nur diesen Pfad.

Die Pfaddetailinformationen enthalten die "Von"- und "Zu"-Schritte des Pfads und geben an, ob der Pfad auf der Standard- oder der Alternativroute verläuft. Sie können alle Pfadinformationen mit Ausnahme der "Von"- und "Zu"-Schritte und der Standardpfadauswahl ändern.

Wählen Sie den Pfad im Arbeitsplan und klicken Sie auf die Schaltfläche "Element ändern", um das Popup "Pfaddetails" aufzurufen. Die Pfadtyp bestimmt, welches Popup angezeigt wird. Das Popup "Nacharbeitspfaddetails" wird für Nacharbeitspfade und das Popup "Pfaddetails" für alle anderen Pfadtypen angezeigt.

Erweitern des Arbeitsplandiagramms

Sie können das Arbeitsplandiagramm in einem größenveränderbaren Popup anzeigen, indem Sie in der Symbolleiste des Arbeitsplandiagramms auf die Schaltfläche "Bereich maximieren" klicken. Das Popup bietet die Möglichkeit, den Arbeitsbereich zu vergrößern, um den Arbeitsplan besser sehen zu können. Das Popup enthält die gleichen Optionen zum Erstellen und Ändern des Arbeitsplans wie die Seite "Arbeitsplan".

Klicken Sie auf "OK", um das Popup zu schließen und Arbeitsplan sowie die am Arbeitsplan vorgenommenen Änderungen im Arbeitsplandiagramm auf der Seite "Arbeitsplan" anzuzeigen. Wenn Sie auf "Abbrechen" klicken, wird ein Bestätigungsdialogfeld angezeigt. Klicken Sie im Dialogfenster auf "Ja", um das Popup zu schließen, ohne Änderungen zu speichern, die am Arbeitsplan vorgenommen wurden.
