Definieren von Geschäftsregelhandlern

Ein Geschäftsregelhandler bestimmt die Aktionen, die von einer Geschäftsregel ausgeführt werden. Geschäftsregelhandler werden einer Geschäftsregel zugewiesen. Weitere Informationen zu Geschäftsregeln finden Sie unter "Definieren von Geschäftsregeln".

Das Modellierungsobjekt "Geschäftsregelhandler" ermöglicht es Ihnen, einen der folgenden Schritte auszuführen:

• Definieren eines Skripts, um Geschäftslogik in einer Geschäftsregel zu implementieren
• Angeben eines vorhandenen Diensts, den die Anwendung aufrufen soll

Ein Skripthandler ermöglicht es Ihnen, eine Bedingung und Aktionen anzugeben, die die Anwendung ausführt, wenn die Bedingung zutrifft. Ein Servicehandler ermöglicht es Ihnen, einen vorhandenen Dienst anzugeben, den die Anwendung aufrufen soll.

Opcenter EX MDD und Opcenter EX CR stellen diese standardmäßigen Geschäftsregelhandler bereit:

• BRH_ACTIVATION: Verwaltet den Aktivierungsstatus der Modellierungsinstanzen, die einem Änderungspaket auf dem Zielsystem zugeordnet sind.
• BRH_DEPLOY: Verwaltet den Bereitstellungsstatus für das Änderungspaket, wenn ein Benutzer die Bereitstellung eines Pakets anfordert.
• BRH_DEPLOYSTATUS: Verwaltet den Status eines Änderungspakets auf dem Quellsystem zwischen dem Zeitpunkt des Exportabschlusses und dem Zeitpunkt der Paketzustellung.
• BRH_NOTIFICATIONS: Das Skript vergleicht die Fälligkeitsdatumsangaben für Genehmigungen für Änderungsverwaltungspakete und für Zusammenarbeit mit den Informationen in der Tabelle "Paketgenehmigungserinnerungen" und in der Tabelle "Inhaltszusammenarbeitserinnerungen" im Modellierungsobjekt "Änderungsmanagementeinstellungen". Es bestimmt, ob E-Mail-Erinnerungen gesendet werden müssen. Weitere Informationen finden Sie unter "Definieren von Einstellungen für die Änderungsverwaltung".

Informationen zum Exportieren von Modellierungsobjekten aus einem System und zum anschließenden Importieren dieser Objekte in ein anderes System finden Sie im Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch und im Opcenter Execution Core Change Management-Benutzerhandbuch.

Beim Definieren von Geschäftsregelskripthandlern

Ein Skript ist ein Satz von Anweisungen, der wahlweise einen oder mehrere Parameter mit Standardwerten hat, die durch Ausdrücke in den Geschäftsregeln festgelegt werden können. Die Anwendung verwendet die Standardwerte, die für Parameter angegeben sind, um zur Laufzeit Skriptparameter zu überschreiben.

Konfigurieren Sie das Skript mit der Syntax für einheitliche Ausdrücke. Diese Syntax ermöglicht es Ihnen, einfache oder komplexe Ausdrücke zu erstellen, die abhängig davon ausgewertet werden können, wo die Geschäftsregel zugeordnet ist.

Wichtig: Um Parameter in einem Geschäftsregelhandler hinzuzufügen, zu entfernen oder zu aktualisieren, müssen Sie auch jede Geschäftsregel, die auf den betreffenden Geschäftsregelhandler verweist, öffnen und speichern. Andernfalls können Fehler auftreten, wenn die zugehörigen Geschäftsregeln ausgeführt werden. Weitere Informationen finden Sie unter "Beim Anzeigen vorhandener Geschäftsregeln".
Hinweis: Weitere Informationen zur Syntax für einheitliche Ausdrücke (Unified Expressions) finden Sie im Opcenter Execution Medical Device und Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Nur diejenigen Skripthandler, für die die Syntax überprüft wurde, stehen beim Erstellen einer Geschäftsregel zur Auswahl.

Beim Definieren von Geschäftsregelservicehandlern

Wenn Sie einen Servicehandler definieren, können Sie weitere Parameter hinzufügen, die die Anwendung zur Laufzeit ausführt. Beachten Sie, dass Sie durch das Hinzufügen von Parametern nicht den tatsächlichen Service ändern. Um den Service selbst zu ändern, muss der Designer verwendet werden. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Geschäftsregelhandler ist eine optionale Modellierungsdefinition in der Geschäftsregel.

Diese Tabelle definiert die Felder, die nur auf der Seite "Geschäftsregelhandler" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Allgemein

Handlertyp

Eine Auswahl, die angibt, ob der Geschäftsregelhandler ein Skript oder ein Service sein soll. Die Auswahl in diesem Feld bestimmt die restlichen Felder, die auf der Seite angezeigt werden.

Erforderlich

Handlerdetails

Nur speichern, wenn die Syntax gültig ist

Nur verfügbar, wenn der Handlertyp gleich "Skript" ist.

Kontrollkästchen, das angibt, ob die Anwendung den Skripthandler nur dann speichert, wenn die Skriptsyntax korrekt ist. (Unabhängig von Ihrer Auswahl überprüft die Anwendung die Syntax des Skripts.)

Wenn das Kontrollkästchen aktiviert und die Syntax falsch ist, meldet die Anwendung, dass die Syntax falsch ist. Der Skripthandler wird dann nicht gespeichert. Wenn das Kontrollkästchen nicht aktiviert und die Syntax falsch ist, speichert die Anwendung den Skripthandler, obwohl die Syntax falsch ist.

Optional

Die Syntax des gespeicherten Skriptes ist

Wird nur angezeigt, wenn der Handlertyp gleich "Skript" ist.

Der Syntaxstatus ("Gültig" oder "Ungültig") des Skripts, das in der Datenbank gespeichert ist. "Gültig" bedeutet, dass die Syntax des gespeicherten Skripts korrekt ist. "Ungültig" bedeutet, dass die Syntax fehlerhaft ist. Die Anwendung aktualisiert den Syntaxstatus, wenn der Skripthandler gespeichert wird.

Die Anwendung aktualisiert dieses Feld, wenn Sie das Skript laden oder speichern.

Vorgehensweise zum Definieren eines Geschäftsregelskripthandlers

Gehen Sie folgendermaßen vor, um einen Geschäftsregelskripthandler zu definieren:

1. Öffnen Sie die Seite Geschäftsregelhandler. Die Seite "Geschäftsregelhandler" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diesen Geschäftsregelhandler in das Feld Geschäftsregelhandler ein.
4. Wählen Sie Skript aus dem Feld Handlertyp aus. Die Anwendung aktualisiert die Seite "Geschäftsregelhandler", um die benötigten Skriptfelder anzuzeigen.
5. Geben Sie das Skript, das ausgeführt werden soll, in das Feld Skript ein.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So wird ein Geschäftsregelservicehandler definiert

Gehen Sie folgendermaßen vor, um einen Geschäftsregelservicehandler zu definieren:

1. Öffnen Sie die Seite Geschäftsregelhandler. Die Seite "Geschäftsregelhandler" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diesen Geschäftsregelhandler in das Feld Geschäftsregelhandler ein.
4. Wählen Sie Service im Feld Handlertyp aus. Die Anwendung aktualisiert die Seite "Geschäftsregelhandler", um die benötigten Servicefelder anzuzeigen.
5. Wählen Sie den Service, der ausgeführt werden soll, aus dem Feld Service aus.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden Parameter zu einem Geschäftsregelhandler hinzugefügt

Gehen Sie folgendermaßen vor, um einem Geschäftsregelskripthandler Parameter hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Geschäftsregelskripthandlers" aus.

Oder

Führen Sie die Schritte unter "So wird ein Geschäftsregelservicehandler definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz eines Geschäftsregelhandlers aus.

2. Klicken Sie in der Tabelle "Parameter" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Geben Sie den Namen des Parameters aus dem Skript weiter oben in das Feld Name ein.
4. Wählen Sie den Datentyp aus der Liste aus.
5. Wählen Sie den Objekttyp aus, wenn Sie im Feld Datentyp den Typ Objekt ausgewählt haben.
6. Geben Sie den Wert für den Parameter in das Feld Standardwert ein.
7. Wiederholen Sie die Schritte 2 bis 6, um weitere Parameter hinzuzufügen.
8. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
