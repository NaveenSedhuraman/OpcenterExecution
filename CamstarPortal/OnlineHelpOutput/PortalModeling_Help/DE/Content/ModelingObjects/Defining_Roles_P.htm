Definieren von Rollen

Eine Rolle definiert eine Tätigkeit, der bestimmte Berechtigungen zugewiesen sind, die Zugang zur gesamten oder Teilen der Portal-Anwendung gewährt. Ein Portaladministrator weist den Mitarbeitern Rollen zu, um Ihnen den Zugriff auf die Teile der Anwendung zu gewähren, die sie zur Ausübung ihrer Tätigkeit benötigen. Sie könnten z. B. die Rolle des Produktions-Schichtleiters definieren und dieser Rolle die Berechtigung zum Starten eines Containers zuweisen.

Opcenter EX MDD und Opcenter EX CR stellen die folgenden Standardrollen bereit:

• Standard Export Import (Default Export Import)
• Standard Abfrage (Default Inquiry)
• Standard Produktion (Default Mfg)
• Standard Modellierung (Default Modeling)
• Standardmodellierung - Erweitert
• Standard Modellierung Schreibgeschützt (Default Modeling Read-Only)
• Standard Seiten (Default Pages)
• Standard Qualität (Default Quality)
• Anmelden (Login)
• Produktionsprotokollabfrage (Mfg Audit Trail Inquiry)
• Portalkonfiguration (Portal Configuration)
• Sicherheitsverwaltung (Security Administration)
• SPC

Informationen zu den zusätzlichen Standardrollen, die Benutzern, die die Änderungsverwaltung verwenden, zur Verfügung stehen, finden Sie in den entsprechenden produktspezifischen Handbüchern:

• Opcenter Execution Core-Anleitung zur Systemadministration
• Opcenter Execution Medical Device and Diagnostics Change Management-Benutzerhandbuch
• Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration

Grundlegendes zu den einzelnen Rollen

Eine Rolle besteht aus einem Satz von Berechtigungen und Berechtigungszugriffsmodi. Eine Berechtigung ist die Erlaubnis zum Ausführen einer Aktion, z. B. die Berechtigung, einen Container zu starten oder eine Anforderung zu erstellen. Eine Berechtigung kann auch als Transaktion oder Seite verstanden werden.

Ein Zugriffsmodus spezifiziert das Autorisierungs-Level für die zugewiesene Berechtigung. Wenn ein Benutzer z. B. eine Starttransaktion durchführen muss, so muss er eine Rolle innehaben, die eine Berechtigung für die Starttransaktion umfasst. Außerdem muss dieser Berechtigung der Zugriffsmodus "Ausführen" zugewiesen sein. Die Anwendung weist standardmäßig alle Zugriffsmodi zu, wenn Sie eine Berechtigung zuweisen.

Rollen sind nur für die Organisationen gültig, denen sie in der Manufacturing-Domäne zugewiesen wurden. Bei Objekten außerhalb des Organisationskontexts, z. B. Transaktionen, müssen Sie keine Organisation angeben, wenn Sie einem Benutzer eine Rolle zuweisen. Rollen sind außerdem nur dann für Unterorganisationen gültig, wenn der Administrator die Rollen an diese Unterorganisationen weitergegeben hat.

Hinweis: Berechtigungen für Erweiterungen der Anwendung (wie etwa neue Seiten), werden den Standard-Services nicht automatisch hinzugefügt. Sie müssen diesen Rollen manuell die Berechtigungen erteilen.

Beim Definieren von Rollen

Die Rolle umfasst die folgenden optionalen Modellierungsdefinitionen:

• Mitarbeiter
• Organisation

"Rolle" ist ein erforderliches Feld in diesen Modellierungsdefinitionen:

• Genehmigungsvorlage, wenn Sie Genehmiger hinzufügen
• Elektr. Unterschrift: Anforderung*, wenn Sie Anforderungen zur elektronischen Unterschrift hinzufügen
• Einteilungsspez., wenn Sie Einteilungselemente hinzufügen

"Rolle" ist ein optionales Feld in diesen Modellierungsdefinitionen.

• Änderungsmanagement-Spez.*
• E-Mail-Verteiler
• Mitarbeiter
• Elektr. Unterschrift: Rollengruppe*
• Organisation

* Ein Sternchen besagt, dass für das Objekt eine separat lizenziertes Modul benötigt wird.

Der Rollentyp bietet die Möglichkeit, den Typ der definierten Rolle anzugeben: "Änderungsverwaltung", "Qualität" oder "Fertigung". Sie können den Rollentyp für die Standardrolle "Package Creator" z. B. auf "Änderungsverwaltung" und den Rollentyp für "Default Quality" auf "Qualität" festlegen.

Hinweis: Sie können Abfragen in Designer aktualisieren, wenn Sie Rollenauswahllisten nach Rollentyp filtern möchten. Wenden Sie sich über das Global Technical Access Center (GTAC) an den Produkt-Support, um Informationen zu erhalten.

Beim Zuweisen von Berechtigungen

Wenn Sie einen Berechtigungstyp im Feld "Berechtigungstypfilter" auswählen, werden die Tabellen "Zugewiesene Berechtigungen" und "Verfügbare Berechtigungen" aktualisiert, sodass nur die Berechtigungen des angegebenen Berechtigungstyps angezeigt werden. Die Anwendung verwaltet alle zugewiesenen Berechtigungen der verschiedenen Berechtigungstypen in der aktuellen Instanz und zeigt diese zugewiesenen und verfügbaren Berechtigungen an, wenn Sie einen zuvor ausgewählten Berechtigungssatz anzeigen. Die Anwendung speichert die Berechtigungen der Rollendefinition nur, wenn Sie auf "Speichern" klicken.

Wenn Sie z. B. den Berechtigungstyp "Container" auswählen, wird die Tabelle "Verfügbare Berechtigungen" aktualisiert und die containerbezogenen Transaktionen werden angezeigt. Wenn Sie der Rolle einige Berechtigungen zuweisen und dann einen anderen Berechtigungstyp auswählen, aktualisiert die Anwendung die Tabelle "Verfügbare Berechtigungen" und es werden nur Berechtigungen angezeigt, die sich auf diesen Berechtigungstyp beziehen. Die Anwendung entfernt alle containerbezogenen Berechtigungen aus der Tabelle "Zugewiesene Berechtigungen", löscht sie jedoch nicht. Wenn Sie den Berechtigungstyp "Container" erneut auswählen, werden alle containerbezogenen Zuweisungen angezeigt.

Sie können eine oder mehrere verfügbare Berechtigungen auswählen, wenn nur bestimmte Berechtigungen zugewiesen werden sollen, oder aber alle verfügbaren Berechtigungen auswählen und gleichzeitig zuweisen. Wenn Sie das Kontrollkästchen in der Kopfzeile der Tabelle "Verfügbare Berechtigungen" aktivieren, werden alle angezeigten Berechtigungen automatisch ausgewählt. Die Tabelle "Zugewiesene Berechtigungen" funktioniert ähnlich. Sie ermöglicht es Ihnen, bereits zugewiesene Berechtigungen einzeln oder komplett zu entfernen.

Hinweis: In der Tabelle "Zugewiesene Berechtigungen" werden alle verfügbaren Berechtigungen unabhängig von den globalen Tabelleneinstellungen in Portal Studio ohne Seitenwechsel angezeigt. Weitere Informationen finden Sie im Opcenter Execution Core Portal Studio -Benutzerhandbuch oder im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch.

Beim Verwalten von Rollen

Sie können Rollenmitgliedschaften von zwei Modellierungsseiten aus verwalten: "Rolle" und "Mitarbeiter". Wenn Sie Änderungen an den Rollenmitgliedschaften auf der Seite "Rolle" vornehmen, werden die aktualisierten Informationen zu Rollenmitgliedschaften des betreffenden Mitarbeiters auf der Seite "Mitarbeiter" angezeigt. Ebenso werden, wenn Sie die Informationen zur Rollenmitgliedschaft auf der Seite "Mitarbeiter" aktualisieren, die aktualisierten Informationen angezeigt, wenn Sie die Rolle auf der Seite "Rolle" Rollenseite anzeigen.

Rollen und elektronische Unterschriften

Die Rolle ist ein erforderliches Modellierungsobjekt, wenn Sie elektronische Unterschriften für Fertigungs- und Qualitätsdatensatztransaktionen implementieren. Eine elektronische Unterschrift entspricht einer handschriftlichen Unterschrift und drückt die Verantwortlichkeit für die Durchführung einer bestimmten Transaktion aus. Sie können bereits aufgezeichnete elektronische Unterschriften eines Containers oder Qualitätsdatensatzes im Produktionsprotokoll anzeigen. Weitere Informationen zur Implementierung elektronischer Unterschriften finden Sie unter "Elektronische Unterschriften für die Modellierung – Einführung".

Hinweis: Die Funktionalität für elektronische Unterschriften ist ein separat lizenziertes Modul.

Vorgehensweise zum Definieren einer Rolle

Gehen Sie folgendermaßen vor, um eine Rolle und deren Berechtigungen zu definieren:

1. Öffnen Sie die Seite Rolle. Die Seite "Rolle" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen in das Feld Rolle ein.
4. Wählen Sie einen Berechtigungstyp aus dem Feld Berechtigungstypfilter aus. Die Tabelle "Verfügbare Berechtigungen" wird aktualisiert und enthält nun die Berechtigungen des angegebenen Berechtigungstyps.
5. Wählen Sie mit einer der folgenden Methoden die Berechtigungen aus, die Sie zuweisen möchten:
• Aktivieren Sie das Kontrollkästchen Berechtigungsname in der Kopfzeile der Tabelle "Verfügbare Berechtigungen", um alle Berechtigungen auszuwählen.
• Aktivieren Sie die Kontrollkästchen der einzelnen Berechtigungen.
6. Klicken Sie auf Hinzufügen, um die ausgewählten Berechtigungen zuzuweisen. Die Berechtigungen werden der Tabelle "Zugewiesene Berechtigungen" hinzugefügt und alle Zugriffsmodi werden standardmäßig ausgewählt.
7. Wiederholen Sie die Schritte 4 bis 6, um zusätzliche Berechtigungen für diese Rolle hinzuzufügen.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Zuweisen einer Rolle zu einem Mitarbeiter

Gehen Sie folgendermaßen vor, um einem Mitarbeiter eine Rolle zuzuweisen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Rolle" aus.

Oder

Wählen Sie eine vorhandene Rolleninstanz aus.

2. Klicken Sie auf Hinzufügen in der Tabelle Mitarbeiter-Rollenmitgliedschaften. Eine neue Zeile wird angezeigt.
3. Wählen Sie einen Mitarbeiter aus.
4. Wählen Sie ggf. die Organisation aus, in der die Rolle für den Mitarbeiter Gültigkeit besitzt.
5. Aktivieren Sie das Kontrollkästchen Weitergeben, wenn diese Rolle auch für alle Unterorganisationen der angegebenen Organisation gültig sein soll.
6. Wiederholen Sie die Schritte 2 bis 5, um einem anderen Mitarbeiter diese Rolle zuweisen.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
