Definieren von Arbeitsplätzen, Arbeitszellen und Gruppen

Mit den nachstehend beschriebenen Verfahren können Sie Arbeitszellen, Arbeitszellgruppen, Arbeitsplätze und Arbeitsplatzgruppen definieren. Die Definition dieser Einheiten soll im Wesentlichen eine genaue Verfolgung von Ausrüstungen ermöglichen, ohne dass Abmeldetransaktionen erforderlich sind.

Darüber hinaus gilt für Arbeitszellen, Arbeitsplätze und die entsprechenden Gruppen Folgendes:

• Sie ermöglichen das Gruppieren und Zuweisung von Aufgaben innerhalb der Produktionsumgebung.
• Sie ermöglichen die Linienfreigabe.

Es gibt viele Möglichkeiten, Arbeitszellen, Arbeitsplätze und Gruppen einzurichten. In diesen Verfahren wird eine Methode erläutert.

Definieren von Arbeitszellen, Arbeitsplätzen und Gruppen

Sie können Arbeitszellen, Arbeitsplätze und Gruppen unter Verwendung der Modellierungsobjekte "Ressource", "Ressourcengruppe" und "Spez." definieren. Je nachdem, wie Sie diese Ressourcen definieren, können Sie die Bearbeitung auf einen einzelnen Container oder eine einzelne Menge beschränken.

Hinweis: Denken Sie bei der Erstellung Ihrer Arbeitszellen daran, dass eine bestimmte Arbeitszelle immer die Eltern-Arbeitszelle der Arbeitsplätze ist, die sie umfasst.

Sie können für einen bestimmten Arbeitsbereich eine Ressourcengruppe erstellen, die alle Ressourcengruppen der Kind-Operationen des Arbeitsbereichs enthält. Dadurch können in den Feldern "Arbeitszelle" und "Arbeitsbereich" auf der Seite "Linienzuordnung" alle für den Arbeitsbereich verfügbaren Werte angezeigt werden.

Einrichten von Arbeitsplätzen, Arbeitszellen und Gruppen

Nachdem Sie Arbeitsplätze, Arbeitszellen und Gruppen definiert haben, müssen Sie die Verfügbarkeiten dieser Ressourcen und Ressourcengruppen auf der Seite "Ressourceneinrichtung" auf "Verfügbar" festlegen. Mit der Einstellung "Verfügbar" ist sichergestellt, dass die Ressource in den entsprechenden Wertelisten für die Ressourcenauswahl angezeigt wird.

Darüber hinaus können Sie auf der Seite "Ressourceneinrichtung" einen Statuscode und einen Statusgrund für die Verfolgung der Ressourcenhistorie und für Berichtszwecke angeben. Wenn z. B. eine Ressource ausfällt, können Sie die Transaktion "Ressourceneinrichtung" durchführen, um die Verfügbarkeit in "Nicht verfügbar", den Status in "Wartung" und den Grund in "Reparatur" zu ändern. Wenn die Reparatur abgeschlossen ist, können Sie die Transaktion "Ressourceneinrichtung" erneut ausführen, um die Verfügbarkeit auf "Verfügbar" und den Status auf "In Produktion" zurückzusetzen. Sie verfügen jetzt über eine Aufzeichnung, aus der hervorgeht, wann und aus welchem Grund ein Fehler auftrat und wie lange es dauerte, diesen Fehler zu beheben.

Hinweis: Ressourcenstatuscodes und Statuscodes werden in der Modellierung festgelegt. Ein Ressourcenstatuscode gibt Auskunft über die momentane Verfügbarkeit einer Ressource. Eine Ressourcenstatusursache liefert zusätzliche Informationen zur Ursache für den aktuellen Status einer Ressource.

Weitere Informationen finden Sie im Opcenter Execution Device and Diagnostics Shop Floor-Benutzerhandbuch oder im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Festlegen der Linienfreigabe für das Containerlevel

Sie können die Linienfreigabe für eine Arbeitszelle, einen Arbeitsplatz oder das Containerlevel festlegen. Die Linienfreigabe für das Containerlevel garantiert, dass nur ein Container an einer bestimmten Ressource bearbeitet wird. Sie geben die Freigabeebene in der Ressourcendefinition an.

Die Linienfreigabe für das Containerlevel wird erzwungen, wenn der Benutzer in der Fertigung eine "Abmelden"- oder "Anmelden"-Transaktion für einen Container durchführt. Der Container muss sich in einem Arbeitsplanschritt mit einer Spezifikation befinden, der ein automatischer Ablauf zugewiesen ist. Die Anwendung verhindert, dass ein zweiter Container in der Arbeitszelle oder an dem Arbeitsplatz angemeldet wird, solange der erste Container noch nicht freigegeben wurde.

Durchführen der Linienfreigabe auf der Produktionsauftragsebene

Sie können die Linienfreigabe für eine Arbeitszelle oder einen Arbeitsplatz auf Produktionsauftragsebene sicherstellen. Mit dieser Funktion können Container, die zu demselben Produktionsauftrag gehören, in der betreffenden Arbeitszelle oder an dem betreffenden Arbeitsplatz bearbeitet werden.

Die Linienfreigabe auf Produktionsauftragsebene wird erzwungen, wenn der Benutzer in der Fertigung eine "Abmelden"- oder "Anmelden"-Transaktion für einen Container durchführt. Der Container muss sich in einem Arbeitsplanschritt mit einer Spezifikation befinden, der ein automatischer Ablauf zugewiesen ist. Dies wird erzwungen, wenn der Benutzer in der Fertigung eine Startaufgabe in einer Aufgabenliste ausführt.

Der Benutzer in der Fertigung wählt aus der Ressourcenliste eine Arbeitszelle oder einen Arbeitsplatz aus, und zwar basierend auf den Auswahlwerten der Spezifikationsdefinition. Die Transaktion schlägt fehl, wenn in der Arbeitszelle oder an dem Arbeitsplatz bereits ein anderer Produktionsauftrag bearbeitet wird. Die Transaktion wird ausgeführt, wenn in der Arbeitszelle oder an dem Arbeitsplatz kein anderer Produktionsauftrag bearbeitet wird. Sobald der Benutzer in der Fertigung den ersten Container von dem Arbeitsschritt abmeldet, wird die Arbeitszelle freigegeben und ein Container, der einem anderen Produktionsauftrag zugeordnet ist, kann in der Arbeitszelle angemeldet und mittels automatischer Abläufe bearbeitet werden.

Die Linienfreigabe auf Produktionsauftragsebene kann auch erzwungen werden, wenn ein automatischer Ablauf durchgeführt wird. In diesem Fall ist weder eine Abmeldung noch eine Anmeldung erforderlich. Stattdessen wird die Freigabe mit einer Prozess starten-Aufgabe eingeleitet und mit einer Prozess beenden-Aufgabe abgeschlossen.

Beim Durchführen der Linienfreigabe

Folgende Richtlinien bestimmen die Linienfreigabe. Wenn die Freigabeeinstellung wie folgt lautet:

• Container für eine Arbeitszelle – dann kann an sämtlichen Arbeitsplätzen der Arbeitszelle jeweils nur ein Container bearbeitet werden. Ein neuer Container kann erst in der Arbeitszelle angemeldet werden, wenn alle Aktivitäten an sämtlichen Arbeitsplätzen für den aktuellen Container abgeschlossen sind.
• Container für einen Arbeitsplatz – dann kann jeweils nur ein Container an einem Arbeitsplatz bearbeitet werden.
• Produktionsauftrag für eine Arbeitszelle – dann können an allen Arbeitsplätzen der Arbeitszelle jeweils nur Container mit dem gleichen Produktionsauftrag gleichzeitig bearbeitet werden. Ein neuer Container eines anderen Produktionsauftrags kann erst in der Arbeitszelle angemeldet werden, wenn sämtliche Arbeiten an allen Arbeitsplätzen für den aktuellen Produktionsauftrag beendet sind.
• Produktionsauftrag für einen Arbeitsplatz – dann können nur Container eines bestimmten Produktionsauftrags gleichzeitig an einem Arbeitsplatz bearbeitet werden.

Weitere Informationen zu dieser Seite finden Sie unter "Felddefinitionen der Seite "Ressource"".

Seite "Ressourcengruppe"

Eine Ressourcengruppe ist eine Unterklasse einer Named Object Group, die eine Gruppe von Ressourcen darstellt. Eine Objektgruppe wird verwendet, um eine Liste von Objekten wie z. B. eine Liste von Ressourcen, Produkten oder Nutzercodes eines bestimmten Typs zur Verfügung zu stellen. Diese Listen werden in der Regel zur Überprüfung und Auswahl verwendet. Eine Operation enthält z. B. eine Referenz auf eine LossCodeGroup. Die Einträge dieser Liste definieren die zulässigen Verlust- bzw. Verlustgrundcodes für die Arbeiten, die durchgeführt werden, während sich der Container an dieser Operation befindet.

Eine Instanz einer Objektgruppe enthält eine Liste von bestimmten Objekten und eine Liste von Objektgruppen. Diese Liste von Objekten für eine bestimmte Objektgruppe ist auf einen bestimmten Typ (oder von diesem Typ abgeleitete Typen) beschränkt. Ebenso ist die Liste von Objektgruppen innerhalb der Objektgruppe auf einen bestimmten Typ (oder von diesem Typ abgeleitete Typen) beschränkt.

Vorgehensweise zum Definieren eines Arbeitsplatzes

Gehen Sie folgendermaßen vor, um einen Arbeitsplatz zu definieren:

1. Öffnen Sie die Seite Ressource. Die Seite "Ressource" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für den Arbeitsplatz in das Feld Ressource ein.
4. Wählen Sie eine Fabrik aus.
5. Wählen Sie eine Freigabeebene aus.
6. Wählen Sie eine Eltern-Ressource aus (eine der zuvor erstellten Ressourcen).
7. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu dieser Seite finden Sie unter "Felddefinitionen der Seite "Ressource"".
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So wird eine Arbeitsplatzgruppe definiert

Gehen Sie folgendermaßen vor, um eine Arbeitsplatzgruppe zu definieren:

1. Öffnen Sie die Seite Ressourcengruppe. Die Seite "Ressourcengruppe" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Arbeitsplatzgruppe in das Feld Ressourcengruppe ein.
4. Klicken Sie in der Tabelle "Einträge" auf Neue Zeile hinzufügen, um einen Arbeitsplatz zu dieser Gruppe hinzuzufügen. Eine neue Zeile wird angezeigt.
5. Wählen Sie einen Arbeitsplatz aus der Liste aus. Dies sollte einer der Arbeitsplätze sein, die Sie mit der oben beschriebenen Schrittfolge erstellt haben.
6. Wiederholen Sie die Schritte 4 und 5 für jeden Arbeitsplatz, den Sie dieser Arbeitsplatzgruppe hinzufügen möchten.
7. Geben Sie abhängig von den Anforderungen Ihres Unternehmens optionale Informationen ein. Weitere Informationen zu dieser Seite finden Sie unter "".
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
