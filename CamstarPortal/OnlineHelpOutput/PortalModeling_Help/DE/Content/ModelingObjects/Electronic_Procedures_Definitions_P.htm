Definitionen automatischer Abläufe

Diese Definitionen sollen Ihnen helfen, die Funktionalität automatischer Abläufe zu verstehen.

Automatische Abläufe

Ein automatischer Ablauf besteht aus einer oder mehreren vordefinierten Aufgabenlisten – die Schritte, die erforderlich sind, um eine Spezifikation abzuarbeiten. Wenn Sie einer Spezifikation einen automatischen Ablauf zuweisen, können Sie erst dann einen Container bei der nächsten Spezifikation des Arbeitsplans anmelden, wenn alle erforderlichen Aufgaben des automatischen Ablaufs erledigt sind. Darüber hinaus steht mit automatischen Abläufen ein elektronischer Beleg für die Erledigung einer einzelnen Aufgabe zur Verfügung.

Hinweis: Eine erforderliche Aufgabe ist eine Aufgabe, bei der der minimale Iterationswert (Min. Iterationen) größer als 0 ist. Weitere Informationen finden Sie unter "Definieren einer Aufgabenliste und zugehöriger Aufgaben".

Automatische Abläufe ermöglichen die Bewegung von Mengen durch eine Folge von Arbeitsplätzen, ohne dass eine Abmeldetransaktion erforderlich ist. Diese Funktion ermöglicht eine Art der Bearbeitung, bei der Mengen an bestimmten Arbeitsplätzen innerhalb einer ausgewählten Arbeitszelle, statt an Arbeitsplätzen außerhalb der Arbeitszelle bearbeitet werden.

Arbeitsplätze

Ein Arbeitsplatz ist die Ressource, die Sie verwenden, um eine Aufgabenliste abzuarbeiten. Sie können einem Arbeitsplatz eine Aufgabenliste zuweisen, um sicherzustellen, dass die Arbeiten mit dieser Ressource ausgeführt werden. Ein Arbeitsplatz teilt eine Arbeitszelle in verschiedene Stationen und sollte nur zu einer Arbeitszelle gehören.

Arbeitsplatzgruppen

Eine Arbeitsplatzgruppe bezeichnet mehrere Arbeitsplätze, die dieselben oder ähnliche Prozesse einer Aufgabenliste bearbeiten. Die Arbeitsplätze in einer Arbeitsplatzgruppe sollten verschiedenen Arbeitszellen zugewiesen werden.

Arbeitszellen

Eine Arbeitszelle ist ein Ort, an dem alle Aufgaben bearbeitet werden, die einem automatischen Ablauf zugewiesen wurden. Wenn die Bearbeitung eines Containers innerhalb einer bestimmten Arbeitszelle begonnen wird, müssen auch alle weiteren Aufgaben für den Container in dieser Arbeitszelle durchgeführt werden. Ein kompletter automatischer Ablauf wird also innerhalb einer Arbeitszelle bearbeitet.

Obwohl keine Arbeitszellendefinition erforderlich ist, wird eine solche Definition dringend empfohlen. Das Konzept der Arbeitszelle bietet Ihnen die Flexibilität einer zusätzlichen Grenze, die genauso wie jede andere Ressource verwendet werden kann, wenn ein Container bei einem Spezifikationsschritt angemeldet wird. Die Arbeitszelle kann auch eine Linienfreigabe erzwingen, wenn Sie diese Funktion konfiguriert haben.

Arbeitszellengruppen

Eine Arbeitszellengruppe beinhaltet mehrere Arbeitszellen, die gleichartige Verarbeitungsschritte ausführen. Daher kann der Bediener in der Fertigung wählen, an welcher Arbeitszelle ein Container bearbeitet wird. Dadurch ist auch gewährleistet, dass sämtliche Bearbeitungsschritte in der ausgewählten Arbeitszelle stattfinden.

Aufgaben und Aufgabenlisten

Eine Aufgabe ist eine bestimmte Arbeitseinheit, die an einem Container für eine Spezifikation ausgeführt werden muss. Sie können Aufgaben in logischen Aufgabenlisten zusammenfassen und eine oder mehrere dieser Aufgabenlisten einem automatischen Ablauf zuweisen. Der automatische Ablauf erzwingt die festgelegte Aufgabenreihenfolge, wenn die Aufgabenliste als "Sequenziell" definiert wurde. Ob eine Aufgabe als erfolgreich ausgeführt bewertet wird oder nicht, hängt von den erfassten Daten oder der Entscheidung des Bedieners in der Fertigung ab.

Datensammlung

Datenpunkte sind Punkte im Produktionsprozess, an denen Daten erfasst werden können oder müssen. Sie können Datenpunkte in einer Datensammlungsdefinition festlegen und diese einer Aufgabe in einer Aufgabenliste zuweisen, wenn Sie für diese Aufgabe Daten sammeln müssen. Datensammlungsdefinitionen werden in der Modellierung definiert.

Datenpunkte erlauben auch die Festlegung von oberen und unteren Limits. Als Layout für die Datenpunkte kommen entweder Zeilen- und Spaltenpositionen oder eine Iterationstabelle infrage.

Datenpunktsammlungsdefinition

Wenn der Bediener in der Fertigung Daten für eine Aufgabe sammelt, gibt es zwei Möglichkeiten:

• Datensammlung in Verbindung mit automatischen Abläufen

Datensammlungsdefinitionen erleichtern die Erfassung von parametrischen und Benutzerdaten aus der Fertigung. Wahlweise kann die Anwendung die erfassten Daten zusätzlich auf Einhaltung der oberen und unteren Limits prüfen, die in der Datensammlungsdefinition festgelegt wurden.

Sie können z. B. einen Wertdatenpunkt, wie einen booleschen Wert, angeben. In diesem Fall müssen die oberen und unteren Datenpunkte identisch sein. Daten außerhalb dieses Werts würden dazu führen, dass die Datensammlungsaufgabe zu einem Fehler führt, der in der Historie aufgezeichnet wird. Zusätzlich wird ein Historieneintrag für jeden gesammelten Datenpunkt erstellt.

Alternativ können Sie auch einen Objektdatenpunkt festlegen. Ein Objektdatenpunkt ermöglicht die Erfassung von parametrischen Daten und referenziert ein bestimmtes Objekt Ihres Fabrikmodells.

• Datensammlungsassistent

Dies ist die Standarddatensammlung, welche über Designer oder den Assistenten definiert wird. Diese Datensammlung kann für die Datensammlung zu einer Aufgabe verwendet werden. Informationen zur Einrichtung finden Sie im Data Collection-Benutzerhandbuch.

Linienfreigabe

Linienfreigabe ermöglicht eine exklusive Bearbeitung. Sie können automatische Abläufe so konfigurieren, dass diese Linienfreigabe an Arbeitsplätzen oder in Arbeitszellen für das Containerlevel oder auf Produktionsauftragsebene erzwingen. Linienfreigabe stellt sicher, dass jeweils nur ein Container oder eine bestimmte Menge des Containers an einer bestimmten Ressource bearbeitet wird. Sie geben die Freigabeebene in der Ressourcendefinition an.
