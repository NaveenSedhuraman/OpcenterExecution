Definieren von Durchsatzanforderungen

Eine Durchsatzanforderung legt die Menge fest, die eine der Anforderung zugeordnete Ressource zwischen Wartungsereignissen verarbeiten kann, bevor die nächste Wartung fällig ist. Die Anforderungsdefinition enthält auch die Toleranz- und Warnbereiche (Stückzahl) und die entsprechenden E-Mail-Verteilerlisten für E-Mail-Benachrichtigungen zu ausstehenden, fälligen und überfälligen Wartungsereignissen. Sie haben auch die Möglichkeit, einen Dokumentsatz und eine Datensammlungsdefinition mit der Anforderung zu verbinden.

Hinweis: Opcenter EX MDD und Opcenter EX CR unterstützen nur eine einzelne Maßeinheit für eine Containerhierarchie, wenn die Wartungsverwaltung auf Durchsatz basiert.

Überprüfung der Maßeinheit

Wenn Sie eine Durchsatzanforderung definieren, müssen Sie eine Maßeinheit (UOM) für die angegebene Menge eingeben. Wenn diese Anforderung für eine Ressource aktiviert wird, muss die Maßeinheit mit der Maßeinheit des Containers übereinstimmen, damit die Ressource den Container bearbeiten kann.

Allerdings kann die Überprüfung der Maßeinheit mit Designer deaktiviert werden, um die ValidateMaintenanceUOM-Methode in den entsprechenden Fertigungsunterklassen zu überschreiben, in denen keine Überprüfung stattfinden sollte. Die Methode kann überschrieben werden, wenn keine Aktion oder benutzerdefinierte Logik ausgeführt werden soll. Z. B. kann die Logik überschrieben werden, wenn eine Menge mittels einer Formel in eine andere Maßeinheit umgerechnet werden soll.

Beim Definieren von Durchsatzanforderungen

Durchsatzanforderungen beinhalten die erforderliche Modellierungsdefinition "Wartungsgrund" und folgende optionale Modellierungsdefinitionen: Dokumentsatz, Datensammlungsdefinition, E-Mail-Benachrichtigung, Elektr. Unterschrift: Anforderung und Maßeinheit.

Vorgehensweise zum Definieren einer Durchsatzanforderung

Gehen Sie folgendermaßen vor, um eine Durchsatzanforderung zu definieren:

1. Öffnen Sie die Seite Durchsatzanforderung. Die Seite "Durchsatzanforderung" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Anforderung in das Feld Durchsatzanforderung ein und geben Sie den Revisions-Identifikator für diese Definition in das Feld Revision ein.
4. Erweitern Sie den Abschnitt Ablaufplanungsdetails.
5. Wählen Sie den Grund, der mit dieser Anforderung verknüpft ist, im Feld Wartungsgrund aus.
6. Geben Sie in das Feld Menge die Stückzahl ein, die von der Ressource bearbeitet werden kann, bis eine Wartung fällig ist.
7. Wählen Sie die entsprechende Maßeinheit im Feld Maßeinheit aus.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen einer Durchsatzanforderungs-Benachrichtigung

Gehen Sie folgendermaßen vor, um eine Durchsatzanforderungs-Benachrichtigung hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Durchsatzanforderung" aus.

Oder

Wählen Sie eine bereits vorhandene Durchsatzanforderungs-Instanz aus.

2. Expandieren Sie den Abschnitt Benachrichtigungen.
3. Klicken Sie in der Tabelle "Ausstehend - E-Mail-Empfänger", "Fällig - E-Mail-Empfänger" oder "Überfällig - E-Mail-Empfänger" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
4. Wählen Sie eine E-Mail-Benachrichtigung aus der Auswahlliste in der neu hinzugefügten Zeile aus. Die Benachrichtigung wird der Tabelle hinzugefügt.
5. Geben Sie den Text der E-Mail in das entsprechende Feld ein: Ausstehend - E-Mail-Text, Fällig - E-Mail-Text oder Überfällig - E-Mail-Text. Sie haben auch die Möglichkeit, Textvariablen zu verwenden.
6. Wiederholen Sie die Schritte 3 bis 5, um die Benachrichtigungen für weitere Empfänger einzurichten.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
