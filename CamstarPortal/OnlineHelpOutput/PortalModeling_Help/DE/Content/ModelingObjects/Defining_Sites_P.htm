Definieren von Standorten

Ein Standort ist ein unabhängiges Opcenter EX MDD- oder Opcenter EX CR-Objekt innerhalb einer Produktionsorganisation. Obwohl eine Fabrik ein physisches Anlagenkonzept ist, ist ein Standort der Name einer Installation von Opcenter EX MDD oder Opcenter EX CR mit einem Anwendungsserver. Der Name wird von anderen Anwendung verwendet, um Transaktionen an diesen und von diesem Standort zu senden. Eine Fabrik kann einen oder mehrere Standorte besitzen und ein Standort kann eine oder mehrere Fabriken umfassen.

Beim Definieren eines Standorts

Eine Standortdefinition enthält die optionale Modellierungsdefinition "Datentransport". SMTP-Datentransport-Definitionen stehen in den Feldern "Standarddatentransport" und "E-Mail-Transport" zur Auswahl bereit.

"Standort" ist ein optionales Feld in den folgenden Modellierungsdefinitionen:

• Unternehmen – Sie können einen bestimmten Standort im Feld "XML-Connect Proxystandort" angeben
• Versandziel – Sie können einen bestimmten Standort im Feld "Remotestandort" angeben

Darüber hinaus ist "Standort":

• Eine erforderliche Definition beim Konfigurieren von Opcenter EX MDD oder Opcenter EX CR für Remote-E-Mails.

Beim Zuordnen eines Mitarbeiters zu einem Standort

Wenn Sie einen Mitarbeiter (Benutzer) der Standortdefinition zuordnen, kann dieser Benutzer bestimmte Remotetransaktionen durchzuführen. Der Mitarbeiter, den Sie diesem Standort zuordnen, kann z. B. die Transaktion "Remotestart" ausführen, um per Remotezugriff einen Container an dem Standort zu erstellen. Dazu muss der Mitarbeiter eine Rolle innehaben, die über Berechtigungen zur Remoteausführung dieser Transaktionen verfügt.

In dieser Tabelle werden die Remotetransaktionen beschrieben, die der zugeordnete Mitarbeiter durchführen darf.

Transaktion

Beschreibung

RemoteStart

Erzeugt einen Container oder eine mehrstufige Containerstruktur am Remotestandort.

RemoteReceive

Nachfolgender (untergeordneter) Dienst, der von der Transaktion "Versenden" (Ship) an einem Remotestandort ausgelöst wird. RemoteReceive dient zur Erstellung eines neuen Containers (über RemoteStart) bzw. zur Reaktivierung eines vorhandenen Containers (über Versenden) am Zielstandort zur weiteren Verarbeitung. Wenn der gleiche Container nicht am Zielstandort vorhanden ist, erstellt RemoteReceive einen neuen Container. Wenn der gleiche, vom Versandsystem übermittelte Container bereits am Zielstandort vorhanden ist, reaktiviert RemoteReceive den Container am Zielstandort und aktualisiert in mit den neuen Informationen.

Hinweis: RemoteReceive ist eine Verbundtransaktion, die aus "RemoteStart" oder "Versenden" und anderen gewünschten Servicetransaktionen bestehen kann.

Versenden

Sendet einen Container von einer Fabrik (einem Standort) an eine andere oder von einer Fabrik (einem Standort) an einen Kunden. Die Versandtransaktion enthält alle Überprüfungen und die "Zu-Schritt"-Logik von MoveNonStd. Der Container kann zu einem beliebigen Arbeitsplan und jedem Schritt der anderen Fabrik gesendet werden. Eine Versandtransaktion kann eine "RemoteReceive"-Verbundtransaktion auslösen, die zur weiteren Bearbeitung an die Zielfabrik gesendet wird.

InventoryTransfer (Lagertransfer)

Sendet Umlaufbestandsinformationen von einer Fabrik (einem Standort) an eine andere oder von einer Fabrik (einem Standort) an einen Kunden. "InventoryTransfer" kann als unabhängiger, eigenständiger Service verwendet werden, um Umlaufbestandsinformationen zwischen Fabriken zu versenden.

Die Transaktion "InventoryTransfer" enthält alle Verhaltensweisen und Überprüfungen der Transaktion "Versenden" mit Ausnahme der folgenden:

• "InventoryTransfer" ermöglicht es, einen geschlossenen Container erneut zu öffnen, wenn der Container vom Zielstandort zurück zum ursprünglichen Versandstandort gesendet wird.
• Am lokalen Versandstandort ist keine Anmeldung erforderlich, wenn der versendete Container zurückgesendet wird.
• Es wird keine Produktionsgutschrift überwiesen, wenn der versendete Containers zurückgesendet wird.
• "InventoryTransfer" wird in der "RemoteReceive"-Verbundtransaktion momentan verwendet, um einen Container an einen Standort zurückzuführen, an dem der gleiche Container bereits vorhanden war oder noch vorhanden ist.

In dieser Tabelle sind die Felder definiert, die nur auf der Seite "Standort" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Verbindungskonfiguration

 

 

Nutzer

Der Mitarbeiter, der berechtigt ist, diese Transaktionen durchzuführen, sofern es sich um einen Remotestandort handelt: 

• RemoteReceive
• Versenden
• InventoryTransfer (Lagertransfer)
• RemoteStart

Oder diese Transaktionen, sofern es sich um einen lokalen Standort handelt:

• Versenden

Hinweis: Der Mitarbeiter muss einer Rolle zugewiesen sein, die die Berechtigung zum Ausführen der Transaktionen hat.

Erforderlich

Passwort

Das Windows-Anmeldekennwort des Nutzers, der diesem Standort zugewiesen wurde. Es wird angenommen, dass es sich um einen Remotestandort handelt. Das Kennwort wird bei der Eingabe verschlüsselt.

Hinweis: Wenn Sie die Standortinstanz öffnen, wird das Kennwort nicht angezeigt, nachdem Sie es einmal festgelegt haben.

Erforderlich

URL

Die URL des Standortnamens. Die Anwendung verwendet diese URL bei der Weiterleitung von XML-Dokumenten für diesen Standort. URLs sollten dieses Format haben:

https://machine name/virtual directory/active server page

Erforderlich

Transport

E-Mail-Transport

Transportmethoden zum Senden von E-Mails von diesem Standort. Die Auswahlmöglichkeiten dieses Felds beinhalten alle SMTP-Datentransporte, die in der Modellierung festgelegt wurden.

Optional

Standarddatentransport

Transportmethode zum Senden ausgehender XML-Dokumente an diesen Standort. Der ausgewählte Datentransport liefert XML-Dokumente an den Routing Server. Die Auswahlmöglichkeiten dieses Felds beinhalten alle SMTP-Datentransporte, die in der Modellierung festgelegt wurden.

Vorgehensweise zum Definieren eines Standorts

Gehen Sie folgendermaßen vor, um einen Standort zu definieren:

1. Öffnen Sie die Seite Standort. Die Seite "Standort" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen in das Feld Standort ein.
4. Wählen Sie den Nutzernamen des Mitarbeiters im Feld Nutzer aus.
5. Geben Sie das Windows-Anmeldekennwort des Mitarbeiters in das Feld Passwort ein. Das Passwort wird bei der Eingabe verschlüsselt.
6. Geben Sie die URL des Standorts in das Feld URL ein.
7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
