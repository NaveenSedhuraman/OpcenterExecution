Definieren von Geschäftsregeln

Das Modellierungsobjekt "Geschäftsregel" wird verwendet, um eine zu erfüllende Bedingung sowie eine oder mehrere Aktionen zu definieren, die ausgeführt werden, wenn die Bedingung zutrifft. Sie können Geschäftsregeln an mehreren Punkten im Qualitätsmanagementprozess zuordnen, um zu steuern, wann bestimmte Aktionen automatisch ausgeführt werden. Z. B. können Sie eine Geschäftsregel zum Wechseln von Eigentümern während der Einteilung erstellen.

Opcenter EX MDD und Opcenter EX CR bieten die folgenden standardmäßigen Geschäftsregeln:

• BR_ACTIVATION: Diese Regel stellt eine Verknüpfung mit dem Standardhandler für Geschäftsregeln BRH_ACTIVATION in der Tabelle "Handler" her.
• BR_DEPLOY: Diese Regel stellt eine Verknüpfung mit dem Standardhandler für Geschäftsregeln BRH_DEPLOY in der Tabelle "Handler" her.
• BR_DEPLOYSTATUS: Diese Regel stellt eine Verknüpfung mit dem Standardhandler für Geschäftsregeln BRH_DEPLOYSTATUS in der Tabelle "Handler" her.
• BR_NOTIFICATIONS: Diese Regel stellt eine Verknüpfung mit dem Standardhandler für Geschäftsregeln BRH_NOTIFICATIONS in der Tabelle "Handler" her.
Hinweis: Weitere Informationen finden Sie unter "Definieren von Geschäftsregelhandlern".

Beim Definieren von Geschäftsregeln

Eine Geschäftsregel enthält die optionale Modellierungsdefinition "Geschäftsregelhandler".

"Geschäftsregel" ist ein optionales Feld in der Modellierungsdefinition "Einteilungsspez.". Das Feld "Verwendung" muss auf "Qualitätsobjekt" (QualityObject) festgelegt werden, bevor es als Option in der Modellierungsdefinition "Einteilungsspez." angezeigt wird.

"Geschäftsregel" ist ein optionales Feld in der Modellierungsdefinition "Spez.". Nur Geschäftsregeln, deren Verwendungstyp "Container" lautet, sind in der Tabelle "Geschäftsregel: Transaktionszuordnung" in der Spezifikation verfügbar. Weitere Informationen finden Sie unter "Hinzufügen von Geschäftsregeln zu einer Spezifikation".

Die Anwendung erzwingt die Verwendung von Geschäftsregeln, sobald diese anderen Objekten zugewiesen werden.

Konfigurieren Sie die bedingte Anweisung mit Hilfe der Syntax für einheitliche Ausdrücke. Diese Syntax ermöglicht es Ihnen, einfache oder komplexe Ausdrücke zu erstellen, die abhängig davon ausgewertet werden können, wo die Geschäftsregel zugeordnet ist.

Wenn Sie der Geschäftsregel Aktionen hinzufügen, können Sie die Parameter für einzelne Aktionen auf Ausdrücke festlegen, die die Anwendung bei der Ausführung der Geschäftsregel auswertet. So können Sie Werte im Aktionsskript oder -service überschreiben.

Planen Sie Geschäftsregeln so, dass sie zukünftig zu vorbestimmten Zeitpunkten ausgewertet werden. Sie können den Zeitplan so festlegen, dass die Regeln einmalig oder in regelmäßigen Abständen ausgewertet werden. Sie können z. B. eine Warnung an einen Verteilungsgenehmiger senden, wenn die Verteilung nicht zu einem bestimmten Zeitpunkt bestätigt wird.

Beim Anzeigen vorhandener Geschäftsregeln

Siemens empfiehlt, alle vorhandenen Geschäftsregeln, die auf Handler mit hinzugefügten, gelöschten oder aktualisierten Parametern verweisen, zu öffnen und zu speichern. Andernfalls können Fehler auftreten, wenn die zugehörigen Geschäftsregeln ausgeführt werden.

Die Anwendung synchronisiert die für die Geschäftsregel definierten Parameter mit den Parametern, die für die zugehörigen Geschäftsregelhandler definiert sind, wenn Sie eine vorhandene Geschäftsregel öffnen. Falls zwischen den Parametern keine Unterschiede bestehen, sind keine Änderungen erforderlich. Falls ein zugehöriger Handler Parameter aufweist, die nicht in der Geschäftsregel enthalten sind, fügt die Anwendung die Parameter automatisch zur Geschäftsregel hinzu. Wenn Parameter aus einem zugehörigen Handler entfernt wurden, werden die Parameter nicht in der Geschäftsregel angezeigt.

In der Anwendung werden Instanz-IDs manuell angezeigt, bevor Sie die Geschäftsregel speichern können.

Felddefinitionen der Seite "Geschäftsregel"

Diese Tabelle definiert die Felder, die nur auf der Seite "Geschäftsregel" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Details

Verwendung

Feld, das angibt, wo die Geschäftsregel zugeordnet und verwendet werden kann. Die Anwendung bestimmt anhand des Verwendungstyps, wie die Geschäftsregelparameter zur Laufzeit ausgewertet werden. Die folgenden Optionen sind verfügbar:

• Änderungspaket
• Container
• Mitarbeiter
• Ereignis
• Fabrik
• QualityObject

Erforderlich

Umfang

Art der Verarbeitung entsprechend der Rolle und den Sicherheitsrechten des ausführenden Benutzers. Der Standardwert ist "Global".

Wenn Sie "Global" auswählen, führt die Anwendung die Geschäftsregel unabhängig von der Rolle und den Sicherheitsrechten des ausführenden Benutzers aus.

Wenn Sie "Lokal" auswählen, führt die Anwendung die Geschäftsregel nur aus, wenn die Organisation, die Rolle und die Sicherheitsrechte des ausführenden Benutzers dies zulassen.

Die Anwendung erzwingt den ausgewählten Umfang, es sei denn, es liegt ein Geschäftsregelzeitplan vor. Da Benutzerrollen und Sicherheitsrechte keine Rolle spielen, wenn die Anwendung einen Geschäftsregelzeitplan ausführt, werden Geschäftsregeln mit der Annahme eines globalen Umfangs ausgeführt.

Erforderlich

Immer ausführen

Kontrollkästchen, das angibt, ob die Aktionen in der Tabelle "Handler" jedes Mal ausgeführt werden.

Obwohl dies ein optionales Feld ist, müssen Sie entweder eine Bedingung in das Feld "Bedingung" eingeben oder dieses Kontrollkästchen aktivieren, bevor die Geschäftsregel gespeichert wird. Wenn Sie dieses Kontrollkästchen aktivieren und einen bedingten Ausdruck eingeben, ignoriert die Anwendung die Bedingung und führt immer die Aktionen aus.

Optional

Bedingung

Bedingung, die von der Anwendung ausgewertet, um zu entscheiden, ob die Aktion im Feld "Handler" auszuführen ist oder nicht.

Obwohl dies ein optionales Feld ist, müssen Sie entweder eine Bedingung eingeben oder das Kontrollkästchen "Immer ausführen" aktivieren, bevor die Geschäftsregel gespeichert wird. Wenn Sie einen bedingten Ausdruck eingeben und das Kontrollkästchen "Immer ausführen" aktivieren, ignoriert die Anwendung die Bedingung und führt immer die Aktionen aus.

Optional

Aktionen

Tabelle "Handler"

Tabelle, die es Ihnen ermöglicht, einen oder mehrere Skript- und Servicehandler, die als Aktionen für die Geschäftsregel verwendet werden sollen, hinzuzufügen (oder zu löschen). (Nur Skripthandler mit gültiger Syntax können ausgewählt werden.) Die Anwendung erfordert, dass Sie für eine Geschäftsregel mindestens einen Handler auswählen.

Wenn Sie einen Handler hinzufügen, zeigt die Anwendung die Parameter in der Tabelle "Parameter" an. Wenn Sie die Geschäftsregel speichern, erstellt die Anwendung eine Kopie des Handlers und speichert sie lokal mit der Geschäftsregel.

Hinweis: Angehängte Geschäftsregelhandler, die gelöscht wurden, werden in der Tabelle durch Referenz-IDs ersetzt. Sie müssen jede Referenz-ID manuell löschen, bevor Sie die Geschäftsregel speichern können.

Wenn Sie die Geschäftsregel speichern, erstellt die Anwendung eine Kopie der Handlerparameter und speichert diese lokal mit der Geschäftsregel. Die Geschäftsregelparameterliste hat bei Ausführung zur Laufzeit Vorrang vor der Handlerparameterliste.

Vorgehensweise zum Definieren einer Geschäftsregel

Gehen Sie folgendermaßen vor, um eine Geschäftsregel zu definieren:

1. Öffnen Sie die Seite Geschäftsregel. Die Seite "Geschäftsregel" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Geschäftsregel in das Feld Geschäftsregel ein.
4. Wählen Sie das Objekt, dem die Geschäftsregel zugeordnet werden kann, aus dem Feld Verwendung aus.
5. Wählen Sie den Verarbeitungsumfang aus dem Feld Umfang aus.
6. Führen Sie einen der folgenden Schritte aus:
• Aktivieren Sie das Kontrollkästchen Immer ausführen, um anzugeben, dass die Anwendung die Aktionen im Feld Handler jedes Mal ausführen soll.
• Geben Sie die auszuwertende Bedingung gemäß der Syntax für einheitliche Ausdrücke in das Feld Bedingung ein.
7. Klicken Sie in der Tabelle "Handler" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
8. Wählen Sie einen Handler aus, der als Aktion verwendet werden soll. Die Anwendung zeigt den Handler in der Tabelle an. Zusätzlich werden jegliche Parameter, die für den Handler definiert sind, in der Tabelle "Parameter" angezeigt.
9. Wiederholen Sie die Schritte 7 bis 8, um weitere Handler hinzuzufügen.
10. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
11. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden Handlerparameter mit benutzerdefinierten Werten überschrieben

Gehen Sie folgendermaßen vor, um Handlerparameter mit benutzerdefinierten Werten zu überschreiben:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Geschäftsregel" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Geschäftsregel aus.

2. Navigieren Sie in der Tabelle "Parameter" zu dem Handlerparameter, den Sie anpassen möchten, und wählen Sie ihn aus.
3. Geben Sie einen Ausdruck in das Feld Ausdruck ein. Die Anwendung wertet den Ausdruck zur Laufzeit aus, um den Wert für diesen Parameter festzulegen.
4. Wiederholen Sie die Schritte 2 bis 3, um weitere Parameter zu ändern.
5. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
