Definieren von Zusammenfassungstabellen und Ansichten

Das Modellierungsobjekt "Zusammenfassungstabellendefinition" wird verwendet, um Ansichten und Zusammenfassungstabellen zu konfigurieren. In Ansichten und Zusammenfassungstabellen werden Daten logisch und zusammenfassend organisiert. Siemens verwendet diese Definitionen im Rahmen der BI-Berichterstattung (BI, Business Intelligence).

Zusammenfassungstabellen

Eine Zusammenfassungstabelle ist eine Zusammenstellung benachbarter Datenfelder, die permanent gespeichert und zur Laufzeit gelesen werden. Sie kann unverändert bleiben, ergänzt oder komplett aktualisiert werden.

Zusammenfassungstabellen können manuell ausgeführt oder für die automatische Ausführung konfiguriert werden. Wenn Sie festlegen, dass eine Zusammenfassungstabelle manuell auszuführen ist, deaktiviert die Anwendung alle Wiederholungsmustereinstellungen, die zuvor in der Zusammenfassungstabelle definiert wurden.

Ansichten

Eine Ansicht ist eine Möglichkeit, Informationen aus einer Datenbank darzustellen. Auch wenn eine Ansicht oft als virtuelle Tabelle angesehen wird, dient sie selbst nicht zur Speicherung von Daten, sondern verwendet Informationen aus einer oder mehreren vorhandenen Tabellen.

Obwohl eine Ansicht nicht dauerhaft ist, kann sie wiederholt aufgerufen werden, indem ihre Kriterien in einer Abfrage gespeichert werden.

Ansichten müssen manuell ausgeführt werden.

Beim Konfigurieren von Zusammenfassungstabellen und Ansichten

Wenn Sie eine Zusammenfassungstabellendefinition konfigurieren, beachten Sie bitte diese Grundsätze:

• Nur ein Datenbankadministrator kann eine Ansicht oder Zusammenfassungstabelle aus der Datenbank entfernen.
• Obwohl die Definition standardmäßig den DataStore verwendet, kann sie so neu konfiguriert werden, dass sie auf die Transaktionsdatenbank oder auf beide gleichzeitig verweist. Wenn die Definition auf die Transaktionsdatenbank oder beide Datenbanken verweist, kann dies jedoch die Leistung des Systems erheblich beeinträchtigen.
• Das Standardwiederholungsmuster der meisten Zusammenfassungstabellen sieht eine Ausführung einmal alle 24 Stunden, beginnend um Mitternacht (0 Uhr) vor. Das Standardwiederholungsmuster der Zusammenfassungstabellen csiProductionDashboard und csiYield sieht eine Ausführung alle vier Stunden, beginnend um Mitternacht (0 Uhr) vor.
• Die Festlegungen im Rahmen der Wiederholungsterminplanung addieren sich. Daher müssen Sie bei Ihrer Auswahl Vorsicht walten lassen. Wenn Sie z. B. zunächst Montag und Samstag ausgewählt haben, wird die Zusammenfassungstabelle montags und samstags ausgeführt. Wenn Sie dann zusätzlich den Ersten des Monats als "Bestimmten Tag des Monats" auswählen, wird die Zusammenfassungstabelle nur an Montagen und Samstagen, die auch der Erste des Monats sind, ausgeführt.
• Beim Testen einer SQL-Abfrage werden nur 100 Zeilen zur Prüfung zurückgegeben.
• Ansichten oder Tabellennamen, die in das Feld "Ansichts- oder Tabellenname" eingegeben werden, müssen eindeutig sein.
• Mithilfe des Modellierungsprotokolls können Sie eine Historie dieses Modellierungsobjekts anzeigen.

Beim Erstellen eine Abfrage

Wenn Sie eine Abfrage erstellen, die den Parameter LastRunDate verwendet, müssen Sie angeben, ob eine SQL-Datenbank oder eine Oracle-Datenbank für die Ausführung verwendet wird.

Datenbank

Wert

SQL

@LastRunDate

Oracle

:LastRunDate

Beispiel

Dieses Beispiel zeigt eine Abfrage auf Basis einer SQL-Datenbank.

SELECT productid,count(*) CNT

FROM container

WHERE productid IS NOT NULL

AND (originalstartdate >= @LastRunDate OR @LastRunDate IS NULL)

GROUP BY productid

Wichtig: Jedes Mal, wenn die SQL-Anweisung geändert wird, müssen Sie das Kontrollkästchen "Aktualisierung erzwingen" aktivieren und auf "Speichern und manuell ausführen" klicken, um die Änderung zu übernehmen. Dies gilt sowohl für Tabellen als auch für Ansichten.

Beim Ausführung von Zusammenfassungstabellen

Wenn Sie Zusammenfassungstabellen ausführen, beachten Sie bitte diese Grundsätze:

• Sie müssen das Kontrollkästchen "Manuell ausführen" aktivieren, wenn Sie eine bereits geplante Zusammenfassungstabelle manuell ausführen möchten. Dadurch wird der Abschnitt "Ablaufplanung" ausgeblendet. Klicken Sie anschließend auf "Speichern und manuell ausführen", um die Tabelle auszuführen.
• Deaktivieren Sie das Kontrollkästchen "Manuell ausführen", wenn Sie die Tabelle neu planen möchten. Dadurch wird der Abschnitt "Ablaufplanung" wieder mit den ursprünglichen Einstellungen angezeigt.
• Wenn Sie auf "Speichern und manuell ausführen" klicken, wird Ihre Abfrage nicht überprüft, wie etwa bei der Schaltfläche "Abfrage speichern und testen".

Vorgehensweise zum Definieren einer Zusammenfassungstabelle

Gehen Sie folgendermaßen vor, um eine Zusammenfassungstabelle zu definieren:

1. Öffnen Sie die Seite Zusammenfassungstabellendefinition. Die Seite "Zusammenfassungstabellendefinition" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen in das Feld Zusammenfassungstabellendefinition ein.
4. Geben Sie einen Abfragetext in das Feld Abfragetext ein.
5. Erweitern Sie den Abschnitt Ablaufplanung.
6. Aktivieren Sie das Kontrollkästchen Zusammenfassungstabelle. Auf der Seite werden für Zusammenfassungstabellen typische Kontrollkästchen und Wiederholungsmusterfelder angezeigt.
7. Geben Sie den Namen der Zusammenfassungstabelle in das Feld Ansichts- oder Tabellenname ein.
8. Möchten Sie die Abfrage der Zusammenfassungstabelle manuell ausführen oder sie so konfigurieren, dass sie automatisch ausgeführt wird?

Sie möchten . . .

Dann . . .

Abfrage der Zusammenfassungstabelle manuell ausführen
a. Aktivieren Sie das Kontrollkästchen Manuell ausführen. Die Wiederholungsmusterfelder sind ausgeblendet.
b. Gehen Sie zu Schritt 9.

Abfrage der Zusammenfassungstabelle für automatische Ausführung konfigurieren
a. Aktivieren Sie das Kontrollkästchen Wiederholung aktivieren.
b. Konfigurieren Sie die Laufzeiten mit den Optionen Wiederholungsmuster und Bereich der Wiederholung.
c. Gehen Sie zu Schritt 9.
9. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
10. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Definieren einer Ansicht

Gehen Sie folgendermaßen vor, um eine Ansicht zu definieren:

1. Öffnen Sie die Seite Zusammenfassungstabellendefinition. Die Seite "Zusammenfassungstabellendefinition" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen der Ansicht in das Feld Zusammenfassungstabellendefinition ein.
4. Geben Sie einen Abfragetext in das Feld Abfragetext ein.
5. Erweitern Sie den Abschnitt Ablaufplanung.
6. Wählen Sie Anzeigen aus. Die Wiederholungsmusterinformationen sind ausgeblendet.
7. Geben Sie einen Namen in das Feld Ansichts- oder Tabellenname ein.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Testen der Abfrage

Gehen Sie folgendermaßen vor, um die Abfrage einer Zusammenfassungstabellendefinition zu testen:

1. Führen Sie einen der folgenden Schritte aus:
• Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Zusammenfassungstabelle" aus.
• Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Ansicht" aus.
• Wählen Sie eine vorhandene Instanz der Zusammenfassungstabellendefinition aus.
2. Klicken Sie auf Abfrage speichern und testen. Das Popup "UserQueryMaintTest" wird mit einer Statusnachricht oder Abfrageergebnissen angezeigt.
3. Klicken Sie auf Schließen, um das Popup "UserQueryMaintTest" zu schließen.
4. Ändern und testen Sie die Abfrage ggf. erneut, bis Sie das gewünschte Ergebnis erhalten.
Wichtig: Wenn die Ausführung der Abfrage einer Zusammenfassungstabelle sehr lange dauert, kann es aufgrund eingehender Transaktionen im SQL-Server zu Fehlern aufgrund einer Sperrung der Tabelle kommen. Sie können die SQL-Anweisungen der Zusammenfassungstabelle so ändern, dass Abfragen in einem solchen Fall nicht gesperrt werden.

Vorgehensweise zum manuellen Erstellen einer Zusammenfassungstabelle oder Ansicht

Gehen Sie folgendermaßen vor, um eine Zusammenfassungstabelle oder Ansicht manuell auszuführen:

1. Führen Sie einen der folgenden Schritte aus:
• Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Zusammenfassungstabelle" aus.
• Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Ansicht" aus.
• Wählen Sie eine vorhandene Instanz der Zusammenfassungstabellendefinition aus.
2. Erweitern Sie den Abschnitt Ablaufplanung.
3. Klicken Sie auf Speichern und manuell ausführen. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
