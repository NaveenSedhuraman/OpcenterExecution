Definieren von Mitarbeitern

Ein Mitarbeiter ist ein Benutzer, der Aktivitäten im System ausführt, z. B. die Wartung von Modellierungsdefinitionen, das Durchführen von Anfragen, die Verwaltung der Sicherheitsadministration und/oder die Durchführung von Fertigungstransaktionen. Ein Systemadministrator verwendet die Sicherheitsadministrationsfunktionen, um Mitarbeitern Berechtigungen zu erteilen, damit sie auf diese Aktionen zugreifen können.

Mitarbeiter müssen ein Active Directory-Benutzerkonto haben und die Benutzernamen der Mitarbeiter müssen mit den Active Directory-Anmeldungs-IDs übereinstimmen. Wenden Sie sich an den Active Directory-Systemadministrator, um die genaue Syntax von Benutzernamen zu erfragen.

Mitarbeiter können auch ohne Benutzerkonto definiert werden. Diese Mitarbeiter können sich jedoch nicht anmelden. Sie können z. B. als der Mitarbeiter eingegeben werden, der eine Container- oder Ressourcentransaktion ausgeführt hat. Z. B. kann sich ein Vorarbeiter, der ein Active Directory-Konto (und die entsprechenden Berechtigungen) hat, bei der Anwendung anmelden und auf die Startseite zugreifen. Der Mitarbeitername, der auf der Startseite erfasst wurde, kann der des Vorarbeiters oder der eines anderen Mitarbeiters sein, je nachdem, wer den Container gestartet hat. Dieser Mitarbeiter kann ein Active Directory-Konto haben, dies ist aber nicht zwingend.

Sitzungswerte können für jeden Mitarbeiter eingestellt werden, um die Anzeige für jemanden zu beschränken, der Transaktionen ausführt. Der Mitarbeiter sieht nur, was er sehen soll, und hat eine kürzere Liste von Werten, die er durchsuchen kann. Dies verringert die Suchzeit und bietet weniger Gelegenheiten für Fehler.

Sicherheitsmodell

Das Sicherheitsmodell für Opcenter EX MDD und Opcenter EX CR basiert auf einem rollenbasierten Zugriffssteuerungsmodell. Der Systemadministrator weist einem Mitarbeiter eine Rollenmitgliedschaft zu, um dessen Zugriff auf das ganze System oder auf bestimmte Teile des Systems zu steuern. Wenn Sie eine primäre Organisation und Rollenmitgliedschaften für einen Mitarbeiter angeben, stellen Sie die Sicherheitsrechte für diesen Mitarbeiter ein. Ein Benutzer muss, um irgendeine Aktion ausführen zu können, eine Rolle in der Organisation haben, entweder direkt oder über Rollenweitergabe indirekt.

Rollen werden auf der Seite "Rolle" definiert. Sie können Rollenmitgliedschaften auf der Seite "Mitarbeiter" oder "Rolle" zuweisen. Allerdings können Sie auf der Seite "Rolle" nur Berechtigungen für diese Rolle zuweisen.

Die Sicherheitsrichtlinien für Ihr Produkt umfassen ein Kapitel zur Sicherheitsverwaltung. Darin werden die Berechtigungstypen und -modi besprochen und wird besprochen, wie der Authentifizierungs- und der Autorisierungsprozess funktioniert und wie Nutzerberechtigungen überprüft werden. Weitere Informationen finden Sie je nach Produkt in den Opcenter Execution Medical Device and Diagnostics-Sicherheitsrichtlinien und in den Opcenter Execution Core-Sicherheitsrichtlinien.

Standardmitarbeiter

Opcenter EX MDD und Opcenter EX CR bieten standardmäßig drei Mitarbeiterdefinitionen:

• Administrator
• CamstarAdmin
• InSiteAdmin

Verwenden Sie einen dieser Mitarbeiternamen für Ihre erste Anmeldung bei der Anwendung.

Die Standardmenüs von Portal werden den Administrator-, CamstarAdmin- und InSiteAdmin-Mitarbeitern in folgenden Situationen automatisch zugewiesen: bei der ersten Implementierung von Opcenter EX MDD oder Opcenter EX CR oder wenn auf eine neue Version konvertiert oder aktualisiert wird und Standardinstanzdaten aktualisiert werden. Verwenden Sie das Feld "Portalmenüdefinition" auf der Seite "Mitarbeiter", um anderen Mitarbeitern Portalmenüs zuzuweisen.

Standardrollen

Siemens bietet mehrere vordefinierte Rollen für die Standardorganisation "Corporate". Den Standardmitarbeitern werden Mitgliedschaften in allen oder einigen der Standardrollen zugewiesen. Weitere Standardrollen stehen Benutzern zur Verfügung, die über eine Lizenz für die Änderungsverwaltung verfügen.

Informationen zur Verwendung der Mitarbeiter und Rollen für die Sicherheit finden Sie in den Opcenter Execution Medical Device and Diagnostics-Sicherheitsrichtlinien und in den Opcenter Execution Core-Sicherheitsrichtlinien.

Beim Definieren eines Mitarbeiters

Mindestens einer der Standardmitarbeiter muss ein gültiger Active Directory-Benutzer sein, um sich erstmalig im Portal anmelden zu können.

Falls Sie zum ersten Mal Mitarbeiter definieren, erstellen Sie die Mitarbeiterdefinition für sich selbst als "Administrator". Verwenden Sie Ihren Windows-Benutzernamen als Ihren Mitarbeiternamen. Wenn Sie eine Definition für einen anderen Mitarbeiter erstellen, verwenden Sie den Windows-Benutzernamen dieses Mitarbeiters.

Mitarbeiterdefinitionen in der Anwendung werden wie folgt verwendet:

• Als beim Portal angemeldeter Mitarbeiter ist Ihre primäre Organisation die Standardorganisation für Sie. Nur die Organisationen, in denen Sie eine direkte Rolle haben oder über die Rollenweitergabe eine Rolle zugewiesen bekommen haben, sind auf den Portal-Seiten zur Auswahl verfügbar.
• Beim Ausführen einer Datensatzsuche in der Anwendung können Sie den Eigentümer als Teil der Suchkriterien eingeben. Der Eigentümer ist der Mitarbeiter, der mit den zu durchsuchenden Qualitätsdatensätzen verknüpft ist.

Beachten Sie Folgendes:

• "Mitarbeiter" enthält die folgenden optionalen Modellierungsdefinitionen: "Organisation", "Wörterbuch", "Elektr. Unterschrift: Rollengruppe", "Fabrik", "Trainingsplan", "Operation" und "Arbeitsbereich".
• "Mitarbeiter" ist ein optionales Feld in allen Container- und Ressourcentransaktionen.

Beim Definieren von Eskalationsempfängern

Ein Eskalationsempfänger ist diejenige Person, die eine E-Mail-Benachrichtigung erhält, wenn für einen Mitarbeiter in Kürze fällige, fällige oder überfällige Arbeit vorliegt. Sie können z. B. eine E-Mail-Benachrichtigung an den Vorgesetzten eines Mitarbeiters senden, wenn eine Arbeit fällig ist. Sie können Eskalationsempfänger über das Feld "Mitarbeiter" in der Tabelle "Eskalationsempfänger" definieren.

Mit dem Modellierungsobjekt "Änderungsmanagementeinstellungen" können Sie Erinnerungsbenachrichtigungen einrichten, die an Eskalationsempfänger gesendet werden, wenn für einen Genehmiger oder Projektmitarbeiter eine unvollständige Aufgabe vorliegt. Die Anwendung sendet die Erinnerung an den festgelegten Eskalationsempfänger für diesen Mitarbeiter, wenn Sie diesen Erinnerungsbenachrichtigungstyp auswählen. Weitere Informationen zum Festlegen der Erinnerungen für Genehmigungen und Zusammenarbeit finden Sie unter "Definieren von Einstellungen für die Änderungsverwaltung".

Die Anwendung sendet keine E-Mail-Benachrichtigungen, wenn Sie keine Eskalationsempfänger in der Tabelle "Eskalationsempfänger" auswählen. Gewährleisten Sie, dass das Feld "E-Mail-Adresse" in der Instanz des "Mitarbeiter"-Modellierungsobjekts jedes Eskalationsempfängers vollständig ist. Andernfalls erhalten die Empfänger keine E-Mail-Benachrichtigungen.

Dokumentverwaltung

Sie können einen Benutzernamen und ein Kennwort angeben, die für den Zugriff auf Dokumente verwendet werden sollen, die in einem externen Dokumentenrepository gespeichert sind. Das Feld "Authentifizierungstyp" des Modellierungsobjekts "Dokument" muss auf "Einfach" festgelegt werden, damit die Felder für Benutzername und Kennwort verwendet werden. Weitere Informationen finden Sie unter "Beim Zugreifen auf Dateien in einem externen Repository".

Beim Zuweisen von Filter-Tags für einen Mitarbeiter

Im Bereich "Filter-Tag-Zuweisung" können Sie steuern, wie der Mitarbeiter die Filter-Tag-Funktion verwenden kann, und erlauben, dass die Filter-Tag-Auswahl des Mitarbeiters sitzungsübergreifend beibehalten wird. Der Mitarbeiter hat die Möglichkeit, die ausgewählten Filter-Tags im Popup "Filter-Tags festlegen" weiter zu verfeinern. Sie können das Popup "Filter-Tags festlegen" aufrufen, indem Sie in der primären Navigationsleiste auf die Schaltfläche "Einstellungen" und im Flyout-Menü auf "Filter-Tags festlegen" klicken.

Wichtig: Als Administrator müssen Sie die eigene Filter-Tag-Zuweisung auf "Alle Instanzen" festlegen, bevor Sie anderen Mitarbeitern Filter-Tags zuweisen. Dadurch wird ein ordnungsgemäßer Zugriff auf alle Objektinstanzen mit Tags gewährleistet.

Es gibt drei Optionsfelder im Abschnitt "Filter-Tag-Zuweisung", die steuern, wie Mitarbeiter die Filter-Tag-Funktion verwenden können.

• Alle Instanzen

Wenn Sie "Alle Instanzen" auswählen, können Mitarbeiter unabhängig vom Filter-Tag-Status alle Instanzen anzeigen. Die Tabelle "Zuweisbare Filter-Tags" wird ausgeblendet und alle Filter-Tags, die zuvor dem Mitarbeiter zugewiesen wurden, werden beim Speichern der Seite entfernt. Das Popup "Filter-Tag festlegen" ist deaktiviert.

• Getaggt und nicht getaggt

Wenn Sie "Getaggt und nicht getaggt" auswählen, können Mitarbeiter Instanzen mit Filter-Tags, die ihnen zugewiesen sind, und Instanzen ohne Tags anzeigen. Dies ist die Standardauswahl für neue Mitarbeiter-Objektinstanzen.

• Nur getaggt

Wenn Sie "Nur getaggt" auswählen, können Mitarbeiter nur Instanzen mit Filter-Tags, die ihnen zugewiesen sind, anzeigen.

Wenn Sie Filter-Tags mit der Tabelle "Zuweisbare Filter-Tags" zuweisen, werden standardmäßig die Filter-Tags im Popup "Filter-Tags festlegen" ausgewählt, das in der globalen Kopfzeile aufgerufen wird. Ein Mitarbeiter kann Filter-Tags, die für die jeweilige Sitzung nicht benötigt werden, im Popup deaktivieren. Die Anwendung deaktiviert dann das entsprechende Filter-Tag in der Tabelle "Zuweisbare Filter-Tags". Dies bietet die Möglichkeit, die Auswahl des Mitarbeiters sitzungsübergreifend beizubehalten.

Wenn ein Administrator ein Filter-Tag für eine Instanz erstellt oder aktualisiert, kann ein Mitarbeiter nur dann auf diese Instanz zugreifen, wenn der neue oder aktualisierte Filter-Tag zur Tabelle "Zuweisbare Filter-Tags" des Mitarbeiters hinzugefügt wird oder der Mitarbeiter alle Instanzen anzeigen darf. Wenn darüber hinaus ein Mitarbeiter Filter-Tags einem Modellierungsobjekt zuweist, dem der Mitarbeiter nicht zugewiesen ist, geben möglicherweise alle versuchten Änderungen nicht die angezeigten Änderungen wieder. Es wird empfohlen, dass der Mitarbeiter zunächst sicherstellt, dass der Administrator ihm die Filter-Tags zugewiesen hat, bevor er versucht, sie einem Modellierungsobjekt zuzuweisen.

Hinweis: Beim Löschen eines Filter-Tags aus der Tabelle müssen Sie alle Filter-Tags deaktivieren und dann nur das Tag auswählen, das Sie löschen möchten. Nach dem Löschen des Tags können Sie erneut die Filter-Tags auswählen, die Sie dem Mitarbeiter zuweisen möchten.

So wird ein Mitarbeiter definiert

Gehen Sie folgendermaßen vor, um einen Mitarbeiter zu definieren:

1. Öffnen Sie die Seite Mitarbeiter. Die Seite "Mitarbeiter" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Windows-Benutzernamen des Mitarbeiters in das Feld Mitarbeiter ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So werden einem Mitarbeiter eine oder mehrere Rollen zugewiesen

Gehen Sie folgendermaßen vor, um einem Mitarbeiter eine oder mehrere Rollen zuzuweisen.

1. Führen Sie die Schritte unter "So wird ein Mitarbeiter definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz eines Mitarbeiters aus.

2. Klicken Sie in der Tabelle "Rollen" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Wählen Sie eine Rolle aus.
4. Wählen Sie die Organisation aus, für die die Rolle gültig ist.
5. Aktivieren Sie das Kontrollkästchen Weitergeben, damit die Rolle bei Bedarf an die Unterorganisationen der angegebenen Organisation weitergegeben wird.
6. Wiederholen Sie die Schritte 2 bis 5, um eine weitere Rolle für diesen Mitarbeiter hinzuzufügen.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

So wird einem Mitarbeiter ein Eskalationsempfänger zugewiesen

1. Führen Sie die Schritte unter "So wird ein Mitarbeiter definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz eines Mitarbeiters aus.

2. Klicken Sie in der Tabelle "Eskalationsempfänger" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
3. Wählen Sie aus der Liste Mitarbeiter den Mitarbeiter aus, der E-Mails mit Eskalationsbenachrichtigungen erhalten soll.
4. Wiederholen Sie die Schritte 2 bis 3, um einen weiteren Eskalationsempfänger hinzuzufügen.
5. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
