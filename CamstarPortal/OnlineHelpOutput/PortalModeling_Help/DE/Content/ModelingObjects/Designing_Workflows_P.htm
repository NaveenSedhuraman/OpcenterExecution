Entwerfen von Arbeitsplänen

Der Abschnitt "Arbeitsplandiagramm" auf der Seite "Arbeitsplan" ermöglicht es Ihnen, Arbeitspläne anzuzeigen, zu entwerfen und zu ändern. Sie können einzelne Schritte hinzufügen, indem Sie die Revision einer Spezifikation oder eines Arbeitsplans in das Diagramm ziehen. Sie können Pfade hinzufügen, indem Sie das Pfadsymbol in dem Schritt auswählen und es auf den nächsten Schritt in Ihrem Arbeitsplan ziehen.

Beim Definieren eines Arbeitsplans

Die Anwendung verwendet den Namen der Spezifikation als Standardnamen für den Arbeitsplanschritt und für den Pfad zum neu hinzugefügten Schritt. Wenn Sie z. B. die Versandspezifikation als Schritt hinzugefügt haben, so ist der Schrittname "Versand" und der Pfad zu diesem Schritt ist ebenfalls "Versand". Wenn Sie den Schritt und den Pfad umbenennen, ist es einfacher, den Schritt von der Spezifikation und vom Pfad zu unterscheiden. Sie könnten z. B. den Versandspezifikationsschritt in Versenden und den Pfad in Zum Versand umbenennen.

Dasselbe Prinzip gilt, wenn Sie Ihrem bereits definierten Arbeitsplan einen Arbeitsplan hinzufügen. Der hinzugefügte Arbeitsplan wird zu einem Unterarbeitsplan und wird nach dem hinzugefügten Arbeitsplan benannt.

Sie müssen die Spezifikation oder den Arbeitsplan erweitern und eine bestimmte Revision auswählen, die dem Arbeitsplandiagramm hinzugefügt werden soll.

Der Arbeitsplan enthält die erforderliche Modellierungsdefinition "Spez.". 

Der Arbeitsplan enthält die optionale Modellierungsdefinition "ERP-Route".

Darüber hinaus ist der Arbeitsplan optional in den Transaktionen "Start", "Abmelden" (Pfad & Schritt), "Abmelden - nicht standardmäßig", "Abmelden - nicht standardmäßig (Mehrere)" und "Nacharbeit (Schritt)" optional.

Beim Definieren von Arbeitsplänen für die Verwendung mit Opcenter Execution Core Scheduling
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Sie müssen einem Arbeitsplan eine ERP-Route und den entsprechenden Arbeitsplanschritten ERP-Routenschritte zuweisen.

Nicht jeder Arbeitsplanschritt muss über einen entsprechenden Routenschritt verfügen. Routenschritte können in einem bestimmten Arbeitsplan nur einmal verwendet werden.

Beim Hinzufügen von Schritten zu einem Arbeitsplan

Schrittbezeichnungen müssen innerhalb eines Arbeitsplans eindeutig sein. Die Anwendung weist den Namen der referenzierten Spezifikation bzw. des referenzierten Unterarbeitsplans standardmäßig dem Schritt zu, der zum Arbeitsplan hinzugefügt wurde. Die Anwendung weist einem Schritt eine numerische Erweiterung hinzu, wenn Sie einen zweiten Schritt hinzufügen, der dieselbe Spezifikation oder denselben Arbeitsplan referenziert. Wenn Sie z. B. eine Spezifikation namens "Visual Inspect" den Arbeitsplanergebnissen hinzufügen, so ist das Ergebnis standardmäßig ein Schritt namens Visual Inspect. Wenn Sie die "Visual Inspect"-Spezifikation erneut dem Arbeitsplan hinzufügen, so heißt dieser Visual Inspect_1.

Sie können einen Schrittnamen jederzeit ändern. Wählen Sie dazu den Schritt aus und klicken Sie auf die Schaltfläche "Element ändern", um das Popup "Schrittdetails" aufzurufen. Weitere Informationen zum Anzeigen und Ändern von Schrittinformationen finden Sie unter "Arbeiten mit Schrittdetails".

Die Symbole für die Darstellung von Spezifikationsschritten und Arbeitsplanschritten unterscheiden sich voneinander. Darüber hinaus unterscheidet sich das Symbol für den ersten Schritt von denen für die nachfolgenden Schritte, unabhängig davon, ob Ihr erster Schritt eine Spezifikation oder ein Unterarbeitsplan ist.

Durch Klicken und Ziehen können Sie Schritte in einem Arbeitsplan neu positionieren. Alle Pfade, die zu einem Schritt und von ihm wegführen, werden mit dem Schritt neu positioniert.

Wichtig: Die Konfigurationseinstellung für die maximalen zurückgegebenen Zeilen in Management Studio ist standardmäßig auf 15.000 festgelegt. Sie können die Einstellung so ändern, dass sie größer als die Anzahl der Schritte in einem Arbeitsplan ist, wenn sehr große Arbeitspläne vorhanden sind. Weitere Informationen finden Sie in der Anleitung zur Systemadministration.

Beim Hinzufügen von Pfaden zwischen Schritten

Pfadbezeichnungen müssen innerhalb eines Arbeitsplans eindeutig sein. Die Anwendung verwendet standardmäßig die Bezeichnung des Zu-Schritts als Bezeichnung des Pfads. Wenn Sie z. B. einen Pfad vom Verpackungsschritt zum Versandschritt hinzufügen, heißt der Pfadname Versand. Die Anwendung verwendet die Bezeichnung des Zu-Schritts unabhängig vom Pfadtyp: Standard, Alternativ, Nacharbeit oder Schleife.

Pfadnamen sind nur im Popup "Pfaddetails" sichtbar. Wählen Sie den Pfad aus und klicken Sie auf "Element ändern", um das Popup "Pfaddetails" anzuzeigen. Weitere Informationen zum Anzeigen und Ändern von Pfadinformation finden Sie unter "Arbeiten mit Pfaddetails".

Links des Schrittsymbols werden drei Pfadsymbole eingeblendet: Standard (primär), Alternativ und Nacharbeit. Verwenden Sie diese Symbole um den Pfadtyp von diesem Schritt zu einem anderen Schritt anzugeben. Sie können nur einen Standardpfad von einem Schritt hinzufügen, aber Sie können mehrere alternative und Nacharbeitspfade von einem Schritt zu anderen Schritten hinzufügen. Außerdem können Sie alle diese Pfade als Schleifenpfade zum aktuellen Schritt zurückführen. Klicken Sie auf ein Pfadsymbol und ziehen Sie die Maus zu einem Schritt, um einen Pfad anzulegen.

Hinweis: Sie können nur einen Schleifenpfad pro Schritt anlegen.

Die Anwendung:

• Erlaubt nur einen Pfad zwischen dem Von-Schritt und dem Zu-Schritt. Wenn Sie den Pfadtyp zwischen zwei Schritten ändern möchten, müssen Sie den Pfad entweder löschen und neu ziehen oder die Pfaddetails ändern.
• Sie müssen einen Standardpfad ziehen, bevor sie einen alternativen Pfad anlegen können. Wenn Sie einen alternativen Pfad ziehen, obwohl noch kein Standardpfad vorhanden ist, konvertiert die Anwendung den alternativen Pfad automatisch in einen Standardpfad.
• Es ist nur ein Standard(primär)pfad vom Von-Schritt erlaubt. Die Anwendung konvertiert sämtliche aufeinanderfolgenden Standardpfade automatisch in alternative Pfade. Wenn Sie den Standardpfad zu einem anderen Schritt umleiten möchten, müssen Sie entweder den vorhandenen Standardpfad löschen und ihn zu einem anderen Schritt neu ziehen oder die Pfaddetails ändern.
Hinweis: Wenn Sie auf "Zurücksetzen" klicken, setzt das nicht die Arbeitsplanzeichnung zurück. Sämtliche Schritte oder Pfade, die hinzugefügt wurden, müssen manuell gelöscht werden.

Diese Tabelle beschreibt die Pfadsymbole.

Symbol

Beschreibung

Zieht den Standardpfad zwischen Schritten oder einen Standardschleifenpfad zum aktuellen Schritt. Der resultierende Pfad ist eine durchgehende grüne Linie. Sie können von einem Schritt aus einen einzelnen Standardpfad ziehen.

Beim Löschen eines Arbeitsplans oder von Elementen eines Arbeitsplans

Es gibt zwei Möglichkeiten, einen Arbeitsplan zu löschen:

• Löschen eines Arbeitsplans und aller seiner Revisionen.
• Löschen einer Arbeitsplanrevision und Beibehalten der Arbeitsplaninstanz

Sie können die Standardrevision nicht löschen, wenn es mehr als eine Revision eines Arbeitsplans gibt. Dazu müssen Sie zuerst eine andere Revision zur Standardrevision machen.

Wenn Sie den ursprünglichen ersten Schritt löschen, müssen Sie einen anderen Schritt als ersten Schritt kennzeichnen. Die Anwendung zeigt eine Fehlermeldung an, wenn Sie den Arbeitsplan speichern, nachdem Sie den ersten Schritt gelöscht haben, ohne einen anderen Schritt als ersten Schritt zu kennzeichnen.

Mehrere Schritte lassen sich nicht auf einmal löschen, wenn einer der Schritte einen Pfadselektor enthält. Sie müssen zuerst einen Pfad löschen, auf "Speichern" klicken und anschließend den weiteren Pfad löschen. Schritte, die Pfade enthalten, lassen sich erst löschen, wenn Sie die Pfade gelöscht haben.

Wenn Sie versuchen, die Historie gelöschter Spezifikationen oder Arbeitspläne anzuzeigen, die gerade verwendet werden oder wurden, führt dies möglicherweise zu einem Fehler. Ein Container verliert seinen aktuellen Status, wenn der Schritt, an dem er sich befindet, aus dem Arbeitsplan gelöscht wird.

Das Löschen von Pfaden hat keine Auswirkungen auf die verknüpften Schritte. Allerdings wird das Standardpfadsymbol auf dem Von-Schritt angezeigt, wenn Sie einen Standardpfad löschen, sodass Sie einen neuen Standardpfad hinzufügen können.

Beim Verwenden von Arbeitsplänen mit Opcenter Execution Semiconductor (Opcenter EX SM)
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter EX SM installiert ist.

Die Anwendung führt die schrittweise Ablaufplanung für Arbeitspläne mit einer großen Anzahl an Schritten aus. Die schrittweise Ablaufplanung beinhaltet die mehrmalige Planung des Ablaufs eines Loses, wobei jedes Mal eine Untergruppe der Arbeitsplanschritte integriert wird. Die Untergruppen werden nacheinander geplant, bis alle Schritte die Umlaufbestandsverarbeitung abgeschlossen haben.

Dieses Diagramm zeigt ein Beispiel für eine schrittweise Ablaufplanung.

Beispielerläuterungen

1. Die Anwendung plant basierend auf dem im Feld "Max. zu verarbeitende Zeitplanschritte" angegebenen Wert von 20 die ersten zwanzig Arbeitsplanschritte für das Los.
2. Wenn die verbleibende Anzahl an Schritten dem für "Preactor-Schrittzähler aktualisieren" angegebenen Wert (in diesem Fall 2) entspricht, sendet die Anwendung den nächsten Schrittsatz. Dieser Satz enthält die zwei nicht beendeten Schritte sowie die nächsten 18 Schritte.
3. Der Prozess wird fortgesetzt, bis die Umlaufbestandsverarbeitung für alle im Arbeitsplan enthaltenen Schritte abgeschlossen wurde.

Vorgehensweise zum Definieren eines Arbeitsplans

Gehen Sie folgendermaßen vor, um einen Arbeitsplan zu definieren:

1. Öffnen Sie die Seite Arbeitsplan. Die Seite "Arbeitsplan" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für den Arbeitsplan in das Feld Arbeitsplan ein.
4. Geben Sie den Revisionsidentifikator für diese Definition in das Feld Revision ein.
5. Geben Sie abhängig von den Anforderungen Ihres Unternehmens optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie im Abschnitt "".
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
7. Führen Sie ggf. die Verfahren in den folgenden Abschnitten aus, um Schritte und Pfade zum Arbeitsplandiagramm hinzuzufügen:
• "Vorgehensweise zum Hinzufügen von Schritten zu einem Arbeitsplan"
• "Vorgehensweise zum Hinzufügen von Pfaden zwischen Arbeitsplanschritten"

Vorgehensweise zum Hinzufügen von Schritten zu einem Arbeitsplan

Gehen Sie folgendermaßen vor, um einem Arbeitsplan einen Schritt hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Arbeitsplans" aus.

Oder

Wählen Sie eine vorhandene Arbeitsplaninstanz aus.

2. Erweitern Sie die Spez. oder den Arbeitsplan, die/den Sie als Schritt hinzufügen möchten, um alle Revisionen anzuzeigen.
3. Wählen Sie die Revision aus, die Sie als Schritt hinzufügen möchten, und ziehen Sie sie in das Arbeitsplandiagramm. Ein Schritt, der die Spezifikation oder den Arbeitsplan referenziert, die/den Sie ausgewählt haben, wird im Arbeitsplandiagramm angezeigt. Der Schritt wird an der Position angezeigt, an der Sie ihn abgelegt haben.
4. Wiederholen Sie die Schritte 2 bis 3, um weitere Schritte hinzuzufügen.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Pfaden zwischen Arbeitsplanschritten

Gehen Sie folgendermaßen vor, um Pfade zwischen Arbeitsplanschritten hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Arbeitsplans" aus.

Oder

Wählen Sie eine vorhandene Arbeitsplaninstanz aus.

2. Möchten Sie einen Standardpfad, alternativen Pfad oder Nacharbeitspfad zwischen zwei Schritten hinzufügen?

Wenn Sie Folgendes hinzufügen möchten . . .

Dann . . .

Einen Standardpfad
a. Klicken Sie auf das Standardpfadsymbol auf dem Von-Schritt und ziehen Sie es auf den Zu-Schritt. Eine durchgehende grüne Linie wird zwischen den beiden Schritten angezeigt.
b. Wiederholen Sie Schritt a, um weitere Standardpfade vom Von-Schritt zu anderen Zu-Schritten hinzuzufügen.
c. Gehen Sie zu Schritt 3.

Einen alternativen Pfad
a. Klicken Sie auf das Symbol für alternativen Pfad auf dem Von-Schritt und ziehen Sie es auf den Zu-Schritt. Eine gestrichelte grüne Linie wird zwischen den beiden Schritten angezeigt.
b. Wiederholen Sie Schritt a, um weitere alternative Pfade vom Von-Schritt zu anderen Zu-Schritten hinzuzufügen.
c. Gehen Sie zu Schritt 3.

Einen Nacharbeitspfad
a. Klicken Sie auf das Nacharbeitspfadsymbol auf dem Von-Schritt und ziehen Sie es auf den Zu-Schritt. Eine gestrichelte rote Linie wird zwischen den beiden Schritten angezeigt und verdeckt das Nacharbeitspfadsymbol des Von-Schritts.
b. Wiederholen Sie Schritt a, um weitere Nacharbeitspfade vom Von-Schritt zu anderen Zu-Schritten hinzuzufügen.
c. Gehen Sie zu Schritt 3.
3. Führen Sie bei Bedarf die Schritte unter "Vorgehensweise zum Definieren von Pfad- und Nacharbeitspfadselektoren" in "Verwenden von Pfadselektoren" aus.
4. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

So wird einem Arbeitsplanschritt ein Schleifenpfad hinzugefügt

Gehen Sie folgendermaßen vor, um einem Arbeitsplanschritt einen Schleifenpfad hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Arbeitsplans" aus.

Oder

Wählen Sie eine vorhandene Arbeitsplaninstanz aus.

2. Klicken Sie auf das entsprechende Pfadsymbol (Standard, Alternativ oder Nacharbeit) und ziehen Sie es in die Mitte des aktuellen Schrittes. Ein Pfad wird angezeigt, der eine Schleife zurück zu demselben Schritt legt.
3. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

Vorgehensweise zum einfügen Schritte in einen vorhandenen Arbeitsplan

Gehen Sie folgendermaßen vor, um einen Schritt in einen vorhandenen Arbeitsplan einzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Arbeitsplans" aus.

Oder

Wählen Sie eine vorhandene Arbeitsplaninstanz aus.

2. Wählen Sie den Pfad zwischen den zwei Schritten aus, in dem Sie einen Schritt einfügen möchten. Der Pfad wird hervorgehoben.
3. Klicken Sie auf Element entfernen. Der Pfad wird entfernt.
4. Erweitern Sie die Spez. oder den Arbeitsplan, die/den Sie als Schritt hinzufügen möchten, um alle Revisionen anzuzeigen.
5. Wählen Sie die Revision aus, die Sie einfügen möchten, und ziehen Sie sie in das Arbeitsplandiagramm. Der Schritt wird an der Position angezeigt, an der Sie ihn abgelegt haben.
6. Wählen Sie das entsprechende Pfadsymbol auf dem Von-Schritt aus und ziehen Sie es auf den soeben eingefügten Schritt. Der Pfad wird hinzugefügt.
7. Wählen Sie das entsprechende Pfadsymbol am soeben eingefügten Schritt aus und ziehen Sie es zum nächsten, entsprechenden Schritt.
8. Wiederholen Sie Schritt 7 für alle weiteren Pfade, die Sie vom soeben eingefügten Schritt ziehen möchten.
9. Ziehen Sie ggf. entsprechende Pfade von vorhandenen Schritten zum soeben eingefügten Schritt.
10. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

So werden Schrittinformationen geändert

Gehen Sie folgendermaßen vor, um Schrittinformationen zu ändern:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Arbeitsplans" aus.

Oder

Wählen Sie eine vorhandene Arbeitsplaninstanz aus.

2. Wählen Sie den Schritt aus, den Sie ändern möchten. Die Schaltfläche "Element ändern" wird in der Symbolleiste des Arbeitsplandiagramms angezeigt.
3. Klicken Sie auf Element ändern. Das Popup "Schrittdetails" wird angezeigt.
4. Ändern Sie, falls erforderlich, die entsprechenden Felder:
• Schrittname
• Spezifikation
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu diesen Feldern finden Sie unter "Felddefinitionen des Popups "Schrittdetails"".
Hinweis: Weitere Informationen zu Pfadselektoren finden Sie unter "Verwenden von Pfadselektoren".
6. Klicken Sie auf OK. Das Popup "Schrittdetails" wird geschlossen.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

Vorgehensweise zum ändern Pfadinformationen

Gehen Sie folgendermaßen vor, um Pfadinformationen zu ändern:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Arbeitsplans" aus.

Oder

Wählen Sie eine vorhandene Arbeitsplaninstanz aus.

2. Wählen Sie den Pfad aus, den Sie ändern möchten. Die Schaltfläche "Element ändern" wird in der Symbolleiste des Arbeitsplandiagramms angezeigt.
3. Klicken Sie auf Element ändern. Das Popup "Nacharbeitsschrittdetails" für Nacharbeitspfade und das Popup "Schrittdetails" für alle anderen Pfade werden angezeigt.
4. Ändern Sie bei Bedarf den Nacharbeitspfadnamen oder den Pfadnamen.
5. Geben Sie je nach den Anforderungen Ihres Betriebs Optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in den Abschnitten "Felddefinitionen des Popups "Pfaddetails"" und "Grundlegendes zu Arbeitsplänen".
6. Klicken Sie auf OK. Das Popup wird geschlossen.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

Vorgehensweise zum löschen ein Arbeitsplanschritt oder Pfad

Gehen Sie folgendermaßen vor, um einen Arbeitsplanschritt oder Pfad zu löschen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Arbeitsplans" aus.

Oder

Wählen Sie eine vorhandene Arbeitsplaninstanz aus.

2. Wählen Sie das Element aus, das Sie löschen möchten.
3. Klicken Sie auf Element entfernen. Die Schritte und alle Pfade, die zu oder von diesem Schritt gezogen wurden, werden entfernt, wenn Sie den ausgewählten Schritt löschen. Nur der Pfad wird entfernt, wenn Sie den ausgewählten Pfad löschen.
4. Fügen Sie ggf. neue Pfade hinzu. Weitere Informationen finden Sie unter "Vorgehensweise zum Hinzufügen von Pfaden zwischen Arbeitsplanschritten".
5. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
