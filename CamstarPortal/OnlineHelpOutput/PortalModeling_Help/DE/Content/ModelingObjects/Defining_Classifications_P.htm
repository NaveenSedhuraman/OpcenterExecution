Definieren von Klassifikationen

Eine Klassifikation ist ein erforderlicher Indikator für den Typ eines Qualitätsdatensatzes. Klassifikationen werden mit Unterklassifikationen (zugehörige Modellierungsobjekte) verbunden.

Die Kombination Klassifikation/Unterklassifikation wird in der "Zuordnung von Ereignisklassifizierungsspez." des Organisationsobjekts festgelegt. Jede Kombination bestimmt, welche Portalseiten und Seitenfolgen angezeigt werden, wenn der Benutzer ein Ereignis eines bestimmten Typs erstellt.

Hinweis: Portalseiten und Seitenfolgen werden in Portal Studio definiert.

Verwenden Sie das Modellierungsobjekt "Klassifikation", um auf Ihre Geschäftspraktiken abgestimmte Klassifikationsinstanzen zu definieren.

Beim Definieren von Klassifikationen

"Klassifikation" ist ein optionales Feld im Modellierungsobjekt "Organisation", genauer sagt in der Zuordnung von Ereignisklassifizierungsspezifikationen. Obwohl "Klassifikation" ein optionales Feld ist, müssen Sie eine Klassifikation angeben, wenn Sie der Zuordnung von Ereignisklassifizierungsspezifikationen einen Eintrag hinzufügen.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Vorgehensweise zum Definieren einer Klassifikation

Gehen Sie folgendermaßen vor, um eine Klassifikation zu definieren:

1. Öffnen Sie die Seite Klassifikation. Die Seite "Klassifikation" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen der Klassifikation in das Feld Klassifikation ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Unter "Gemeinsame Felder auf Modellierungsseiten" finden Sie Informationen zu den optionalen Feldern.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
