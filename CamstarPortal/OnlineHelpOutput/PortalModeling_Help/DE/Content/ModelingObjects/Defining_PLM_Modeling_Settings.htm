Definieren von PLM-Modellierungseinstellungen

Das Modellierungsobjekt "PLM-Modellierungseinstellungen" bietet die Möglichkeit, globale Einstellungen im MOM-Arbeitsbereich zu konfigurieren, die erforderlich sind, um ein Teamcenter-Paket in Opcenter EX MDD oder Opcenter EX CR herunterzuladen. Zu diesen Einstellungen zählen:

• Produkttyp
• Durchsatzberichtsebene

Die Anwendung kann ein Teamcenter-Paket mit Objekten, für die diese Einstellungen erforderlich sind, u. U. nicht herunterladen, wenn die Einstellungen nicht konfiguriert wurden, bevor das Paket in Opcenter EX MDD oder Opcenter EX CR heruntergeladen wird.

Die Felder der Seite "PLM-Modellierungseinstellungen" werden durch Instanzen festgelegt, die in der Modellierung definiert wurden. Vor dem Konfigurieren von PLM-Modellierungseinstellungen müssen Sie in der Modellierung Instanzen für Produkttyp und Containerlevel (Durchsatzberichtsebene) definieren.

Das Feld "PLM-Modellierungseinstellungen" auf der Modellierungsseite "Fabrik" ist optional. Sie müssen PLM-Modellierungseinstellungen für die Fabrik des angemeldeten Mitarbeiters konfigurieren, damit die Einstellungen auf heruntergeladene Teamcenter-Pakete angewendet werden. Informationen zum Definieren der Modellierungsinstanzen vom Typ Fabrik oder Mitarbeiter finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Felddefinitionen der Seite "PLM-Modellierungseinstellungen"

Diese Tabelle definiert die Felder auf der Seite "PLM-Modellierungseinstellungen".

Feld

Definition

Typ

Allgemein

Produkttyp

Produkttyp-Instanzen, die in der Modellierung definiert sind. Benutzercodes für Produkttypen werden verwendet, um zwischen verschiedenen Produkttypen wie Umlaufbestand, Rohmaterial und Fertigerzeugnissen zu unterscheiden.

Hinweis: "Produkttyp" ist ein erforderliches Feld auf der Modellierungsseite "Produkt".

Optional

Durchsatzberichtsebene

Containerlevel-Instanzen, die in der Modellierung definiert sind. Die Anwendung verwendet die Durchsatzberichtsebene als Containerlevel, um den Durchsatz zu berechnen und zu melden.

Hinweis: "Durchsatzberichtsebene" ist ein erforderliches Feld auf der Modellierungsseite "Operation".

Optional

Vorgehensweise zum Definieren von PLM-Modellierungseinstellungen

Gehen Sie folgendermaßen vor, um PLM-Modellierungseinstellungen zu definieren:

1. Öffnen Sie die Seite PLM-Modellierungseinstellungen. Die Seite "PLM-Modellierungseinstellungen" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die PLM-Modellierungseinstellungen im Feld PLM-Modellierungseinstellungen ein.
4. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zur Seite "PLM-Modellierungseinstellungen" finden Sie in der Felddefinitionstabelle.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
