Definieren von Serverkatalogsystemen

Ein Serverkatalogsystem identifiziert einen Server innerhalb von Opcenter EX MDD oder Opcenter EX CR, von dem aus Sie Datenpakete senden oder empfangen können. Sie können ein Serverkatalogsystem für Ihren lokalen Server und alle verfügbaren (Ziel-)Server Ihrer Organisation definieren.

Beim Definieren von Serverkatalogsystemen

Die Seite "Serverkatalogsystem" ermöglicht es Ihnen, zwei Arten von Serverstandorten zu definieren:

• Lokales System: Der erforderliche Opcenter EX MDD- oder Opcenter EX CR-Zielserver
• Ist kein Camstar-Server: Zusätzliche Server zur Verarbeitung, die derzeit nicht Opcenter EX MDD oder Opcenter EX CR ausführen

Sie müssen gewährleisten, dass der Quellserver und die Zielserver ein gültiges SSL-Zertifikat enthalten. Bei der Definition des Feldes "IP-Adresse" müssen Sie den im Zertifikat aufgeführten vollqualifizierten Domänennamen des Servers verwenden.

Lokales System

Sie können nur einen Server als den lokalen Server festlegen. Der lokale Server ist derjenige, von dem aus alle Datenpakete an die Zielserver geschickt werden. Wenn Sie das Kontrollkästchen "Ist Lokales System" auf der Seite "Serverkatalogsystem" aktivieren, wird dieser Server zum lokalen Server. Sie müssen mindestens einen lokalen Server definieren, bei dem "Ist lokales System" ausgewählt ist.

Wenn Sie versuchen, eine Instanz als lokalen Server zu speichern, obwohl bereits eine andere Instanz als lokaler Server definiert wurde, wird eine Fehlermeldung angezeigt. Die Anwendung verlangt, dass Sie zuerst das Kontrollkästchen "Ist lokales System" für die vorhandene lokale Serverinstanz leeren, bevor Sie einen anderen Server als lokalen Server definieren können.

Anhand der Felder im Abschnitt "Serverdetails" werden die Attribute (Eigenschaften) des lokalen und des Zielservers verglichen, wenn Sie Datenpakete verschicken oder empfangen. Eine Warnmeldung wird angezeigt, wenn die Attribute nicht übereinstimmen. Sie können die Transaktion jedoch dennoch durchführen. Es wird z. B. eine Warnmeldung angezeigt, wenn für den lokalen Server die Version 6 und für den Zielserver die Version 5.8 eingegeben wurde.

Wenn Sie eine Active Directory-Domäne angeben, kann die Anwendung eine Zuordnung vom Hostserver zum Temp-Ordner auf einem Zielserver vornehmen, wenn die beiden Server in verschiedenen Domänen liegen. Die Anwendung versucht, über die Domäne des Hostservers eine Zuordnung vom Hostserver zum Zielserver herzustellen, wenn Sie keine Domäne für das Zielsystem angeben.

Ist kein Camstar-Server

Sie können zusätzliche Standorte für Verarbeitungsordner auf Servern definieren, auf denen Opcenter EX MDD oder Opcenter EX CR nicht ausgeführt wird. Server, auf denen derzeit keine Version des Produkts ausgeführt wird, stehen auf der Seite "Aktivierungswirkung" nicht zur Überprüfung zur Verfügung.

Felddefinitionen der Seite "Serverkatalogsystem"

In dieser Tabelle sind die Felder definiert, die nur auf der Seite "Serverkatalogsystem" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Serverdetails

Servertyp

Text, der den Typ oder Zweck des Servers angibt; z. B. Entwicklung, QS-System, Produktionssystem usw.

Optional

Standort

Physischer Standort des Servers, z. B. Chicago, IL, Serverraum 100 usw. Sie können maximal 255 Zeichen eingeben.

Optional

Administrator

Der Mitarbeiter, der für die Wartung des Servers verantwortlich ist.

Optional

MDB-Version

Version der Microsoft Access-Datenbank, die derzeit auf dem Server installiert ist. Dies ist die Insite.mdb-Version.

Hinweis: Dieses Feld ist deaktiviert, wenn "Ist kein Camstar-Server" ausgewählt ist.

Erforderlich

Camstar-Version

Version von Opcenter EX MDD oder Opcenter EX CR, die auf dem aktuellen Server installiert ist, z. B. 6, 7 usw.

Hinweis: Dieses Feld ist deaktiviert, wenn "Ist kein Camstar-Server" ausgewählt ist.

Erforderlich

Ist lokales System

Lokaler Server, von dem die Datenpakete stammen. Ist standardmäßig nicht aktiviert.

Hinweis: "Ist kein Camstar-Server" wird automatisch gelöscht, wenn "Ist lokales System" ausgewählt wird.

Optional

Ist kein Camstar-Server

Auf diesem Server wird weder Opcenter EX MDD noch Opcenter EX CR ausgeführt.

Hinweis: "Ist lokales System" wird automatisch gelöscht, wenn "Ist kein Camstar-Server" ausgewählt wird.

Optional

Konnektivität

Nutzername

Ein gültiger Benutzername, um auf diesen Server zuzugreifen.

Erforderlich

Passwort

Das Kennwort zum Benutzernamen, das für den Zugriff auf den Server benötigt wird.

Hinweis: Wenn Sie die Instanz öffnen, wird das Kennwort nicht angezeigt, nachdem Sie es einmal festgelegt haben.

Erforderlich

Active Directory-Domäne

Name der Active Directory-Domäne für den oben angegebenen Benutzer, der Zugriff auf diesen Server hat.

Optional

IP-Adresse

Vollqualifizierter Domänenname (FQDN) des Servers, der mit dem Namen auf dem SSL-Zertifikat übereinstimmt.

Hinweis Die Verwendung der IP-Adresse zum Herstellen einer Verbindung zum Server wird von Siemens nicht standardmäßig unterstützt.

Erforderlich

Port

Internet-Socket-Portnummer für den angegebenen Server.

Hinweis: Port ist deaktiviert, wenn "Ist kein Camstar-Server" ausgewählt ist.

Erforderlich

Verarbeitungsordner

Pfad des Verarbeitungsordners, der von Export- und Import-Services verwendet wird. Wird auch vom Camstar-Implementierungsservice verwendet, wenn der Service eine Exportdatei aufnimmt.

Hinweis: Wenn kein Pfad definiert ist, wird der Ordner C:\temp verwendet.

Vorgehensweise zum Definieren eines Serverkatalogsystems

Gehen Sie folgendermaßen vor, um ein Serverkatalogsystem zu definieren:

1. Öffnen Sie die Seite Serverkatalogsystem. Die Seite "Serverkatalogsystem" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für den Server in das Feld Serverkatalogsystem ein.
4. Geben Sie die Version der MDB, die auf dem Server installiert ist, in das Feld MDB-Version ein.
5. Geben Sie im Feld Camstar-Version die auf dem Server installierte Opcenter EX MDD- oder Opcenter EX CR-Version ein.
6. Geben Sie den Namen des Benutzers, der Zugriff auf diesen Server hat, in das Feld Nutzername ein.
7. Geben Sie das Kennwort für den Benutzernamen in das Feld Passwort ein.
8. Geben Sie den vollqualifizierten Domänennamen des Servers, der mit dem SSL-Zertifikat übereinstimmt, in das Feld IP-Adresse ein.
9. Geben Sie den Port in das Feld Port ein.
10. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
11. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
