Definieren einer Aufgabenliste und zugehöriger Aufgaben

Mithilfe einer Aufgabenliste können Sie eine Liste von Aufgaben definieren, die für eine Spezifikation ausgeführt werden sollen. Sie geben dabei an, ob die Aufgaben der Reihenfolge nach abgearbeitet werden müssen. Ein Beispiel einer Aufgabenliste ist eine Gruppe von Produktmessaufgaben, die zum automatischen Ablauf "Benutzerdatensammlung" gehören.

Grundlegendes zu Aufgaben

Eine Aufgabe ist eine Arbeitseinheit, die für einen Container bei einer Spezifikation ausgeführt werden muss. Sie können einem automatischen Ablauf, der einer Spezifikation zugewiesen werden kann, eine Reihe von Aufgaben zuordnen. Der automatische Ablauf erzwingt die Abfolge von Aufgabenlisten und eine Aufgabenliste erzwingt die jeweilige Abfolge von Aufgaben. Eine Aufgabe muss gespeichert werden, bevor sie als vorausgesetzte Aufgabe der nächsten Aufgabe festgelegt werden kann. Ob eine Aufgabe als erfolgreich ausgeführt bewertet wird oder nicht, hängt von den erfassten Daten oder der Entscheidung des Bedieners in der Fertigung ab.

Aufgabenelementtypen

Sie können den Aufgabenelementtyp auswählen, wenn Sie ein Aufgabenelement hinzufügen.

Aufgabenelementtyp

Beschreibung

Anweisung

Jede allgemeine Art von Aufgabe.

Berechnung

Berechnung, die die Anwendung auf Grundlage von Benutzerkonstanten, Parametern sowie Eingaben des (Bedieners) Operators vornimmt, wenn die Aufgabe ausgeführt wird.

Transaktion

Elementtyp, der eine Aufgabe zum Durchführen einer Transaktion definiert. Aufgabe 1 ist z. B. eine Anmelden-Transaktion und Aufgabe 2 eine Abmelden-Transaktion.

Wiegen des Verbrauchs

Aufgabe, um den Punkt im Prozess, an dem eine Komponente verbraucht wird, zu definieren und ggf. erforderliche Regeln für den Wiegevorgang zur Verfügung zu stellen.

Hinweis: Dieser Aufgabenelementtyp wird nur angezeigt, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".
Hinweis: Benutzerkonstanten und Berechnungen werden ebenfalls in der Modellierung festgelegt. Weitere Informationen finden Sie unter "Definieren von Benutzerkonstanten" und "Definieren von Berechnungen".

Aufgabentypen

Der Aufgabentyp eines Aufgabenelements ist wichtig für diejenigen Arbeitsplätze, die Linienfreigabe (Line Clearance) erzwingen. Ein Aufgabenelement kann einen der folgenden Aufgabentypen haben.

Aufgabentyp

Beschreibung

Prozess starten

Kennzeichnet den Verarbeitungsbeginn an einem Arbeitsplatz. Eine Prozess starten-Aufgabe ändert den Status des Arbeitsplatzes in "In Bearbeitung", und zwar entweder auf dem Containerlevel oder auf der Produktionsauftragsebene. Eine Prozess starten-Aufgabe ist für Arbeitsplätze, die Linienfreigabe erzwingen, erforderlich und muss die erste Aufgabe in der Aufgabenliste sein. Eine Transaktionsaufgabe kann keine Prozess starten-Aufgabe sein.

Hinweis: Der minimale Wiederholungswert für eine Prozess starten-Aufgabe muss größer als 0 sein.

Normal

Kennzeichnet Aufgaben, die zwischen einer Prozess starten-Aufgabe und einer Prozess beenden-Aufgabe für Arbeitsplätze auftreten können, die eine Linienfreigabe erzwingen. Eine Aufgabenliste kann alle normalen Aufgaben für Arbeitsplätze enthalten, die keine Linienfreigabe erzwingen. Eine normale Aufgabe muss für denselben Container oder Produktionsauftrag in derselben Aufgabenliste, im selben automatischen Ablauf und im selben Spezifikationsschritt ausgeführt werden wie die Prozess starten-Aufgabe. Andernfalls schlägt sie fehl.

Prozess beenden

Kennzeichnet das Ende eines Prozesses an einem Arbeitsplatz. Eine Prozess beenden-Aufgabe setzt den Arbeitsplatzstatus zurück. Die Prozess beenden-Aufgabe zeigt an, dass der Arbeitsplatz für die Bearbeitung des nächsten Containers oder Produktionsauftrags bereit ist. Eine Prozess beenden-Aufgabe ist für Arbeitsplätze, die Linienfreigabe erzwingen, erforderlich und muss die letzte Aufgabe in der Aufgabenliste sein.

Hinweis: Der minimale Wiederholungswert für eine Prozess beenden-Aufgabe muss größer als 0 sein.

Anweisungstypen

Eine Anweisungsaufgabe kann einer der folgenden Anweisungstypen sein.

Anweisungstyp

Beschreibung

Bestätigung

Anweisungstyp, für den eine Bestätigung der Aufgabendurchführung erforderlich ist.

Datensammlung

Aufgaben des Typs "Datensammlung", die normalerweise parametrische und Benutzerdaten aus der Fertigung erfassen. Nachfolgende Aufgaben können von den Ergebnissen abhängig sein.

Bestanden/ Durchgefallen

Anweisungstyp, für den eine Benachrichtigung erforderlich ist, ob die Aufgabe erfolgreich durchgeführt wurde oder fehlgeschlagen ist.

Grundlegendes zum Ausführungsmodus

Geben Sie die einzelnen Aufgaben in der Reihenfolge ein, in der sie durchgeführt werden müssen, wenn Sie eine Aufgabenliste erstellen. Wenn die Aufgaben nicht in einer bestimmten Reihenfolge erledigt werden müssen, können Sie diese in beliebiger Reihenfolge eingeben. Die Aufgabenliste definiert den Ausführungsmodus für die Aufgaben.

Sie geben den Ausführungsmodus für eine Aufgabenliste an, indem Sie im Feld "Ausführungsmodus" auf der Seite "Aufgabenliste" die Option "Sequenziell" oder "Nicht sequenziell" auswählen:

• Sequenziell (Standard) – Jede Aufgabenliste wird in der Reihenfolge angezeigt, die im automatischen Ablauf festgelegt wurde. Eine Aufgabenliste kann nur übersprungen werden, wenn alle Aufgaben optional sind.
• Nicht sequenziell – Die Aufgabenliste muss nicht in der Reihenfolge ausgeführt werden, die im automatischen Ablauf definiert wurde. Der automatische Ablauf definiert ebenso den Ausführungsmodus für Aufgabenlisten.

Beim Definieren einer Aufgabenliste und zugehöriger Aufgaben

Möglicherweise (je nach Ihren Anforderungen) müssen Sie die folgenden Elemente (Entitäten) definieren, bevor Sie eine Aufgabenliste definieren:

• Berechnungen
• Dokumentensätze
• Anforderungen zur elektronischen Unterschrift
• Prozesstimer
• Trainingsanforderungsgruppen
• Benutzerkonstanten
• Benutzerdatensammlungsdefinitionen

Aufgabenliste ist eine erforderliche Modellierungsdefinition in einem automatischen Ablauf.

Die Aufgabenliste verwendet folgende optionale Modellierungsdefinitionen: Ressource, Behälter, Ressourcengruppe, Behältergruppe, Anforderung für elektr. Unterschrift, Trainingsanforderungsgruppe und Benutzerdatensammlungsdefinition.

Der Objektname darf keine ungültigen Zeichen enthalten. Z. B. darf kein Apostroph verwendet werden.

Hinweis: Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Text-Editor

Siemens stellt einen Text-Editor bereit, mit dem Sie die Anweisungen in Aufgabenlisten oder zugehörigen Aufgaben formatieren können. Der Text-Editor unterstützt übliche Textformatierungen wie Schriftgröße, Schriftfarbe und Schriftart, Textausrichtung, gegliederte Listen, Markierungen, Einfügen von Bildern und so weiter.

Hinweis: Siemens empfiehlt, derzeit keine nummerierten Listen in Aufgaben- oder Aufgabenlistenanweisungen zu erstellen, da diese im Feld "Anweisungen" nicht korrekt dargestellt werden.

Siemens empfiehlt, die vorhandenen Anweisungen zu markieren und dann die entsprechende Formatierung anzuwenden. Wenn Sie keine Formatierungsattribute auswählen, wendet die Anwendung die Standardwerte für Schriftgröße, Schriftart und Farbe an. Die Anwendung zeigt dem Benutzer in der Fertigung die angewandte Formatierung an.

Sie können den Text-Editor entweder im Design- oder im HTML-Modus benutzen. Wenn Sie Text im Design-Modus kopieren, werden die verborgenen Tags nicht kopiert. Das bedeutet, dass das Format unter Umständen nicht beibehalten wird, wenn Sie kopierten Text wieder einfügen. Allerdings können Sie die verborgenen Tags kopieren, indem Sie in die HTML-Ansicht wechseln.

Es gibt eine Begrenzung von 4.000 Zeichen im Feld "Anweisung", einschließlich des eingebetteten HTML-Codes. Wenn Sie Text aus einer anderen Anwendung (z. B. Microsoft Word) kopieren und ihn in dieses Feld einfügen, wird verborgener HTML-Code zusätzlich zum Text in das Feld eingefügt. Daher kann es passieren, dass Sie über der Grenze liegen, auch wenn der Text weniger als 4.000 Zeichen enthält.

Hinweis: Google Chrome unterstützt nicht die Schaltflächen "Kopieren" und "Einfügen" des Texteditors. Sie können die Tastenkombinationen zum Kopieren und Einfügen (Strg+C und Strg+V) verwenden, wenn Sie mit Google Chrome arbeiten.

Design-Modus

Der Design-Modus ist ein WYSIWYG-Text-Editor (what you see is what you get), der die Anweisungen so anzeigt, wie der Benutzer in der Fertigung sie sieht.

HTML-Modus

Wenn Sie auf die Schaltfläche "HTML" klicken, wird der Quellcode angezeigt, der auf die Anweisungen angewandt wird. Da ein WYSIWYG-Editor Markup-Code (Tags) automatisch einfügen kann, können Sie den Code im HTML-Modus überprüfen und ggf. verändern.

Einfügen von Bildern mit dem Text-Editor

Sie können Bilder in Anweisungen einfügen, indem Sie im Text-Editor auf die Schaltfläche "Bild einfügen/bearbeiten" klicken. Geben Sie dann den Pfad eines Bildes in das Popup "Bild einfügen/bearbeiten" ein, um das Bild hochzuladen.

Der Pfad eines Bildes muss mit einer der beiden folgenden Methoden eingegeben werden:

• Relative URL, wie z. B. "Designs/MeineBilder/Bild.png"

Oder

• Eine absolute URL, z. B. "https://localhost/Themes/MyImages/Image.png"
Hinweis: Große Bilder passen möglicherweise auf der Seite "Automatischer Ablauf" nicht in den sichtbaren Bereich des Anweisungsabschnitts. Siemens empfiehlt, dass Sie die Abmessungen Ihres Bildes bearbeiten, damit es in den sichtbaren Bereich auf Ihrem bevorzugten Gerät passt.

Beim Zuweisen von Prozesstimern

Prozesstimer können in jedem der Aufgabenelementtyp-Popups ("Anweisung", "Berechnung", "Transaktion") von Aufgaben gestartet und beendet werden, jedoch nur mit den Transaktionen "Abmelden" und "Anmelden".

Hinweis: Wenn Sie versuchen, eine Aufgabenliste zu speichern, die eine Transaktionsaufgabe mit Timern und einer Transaktion enthält, die keine "Abmelden"- oder "Anmelden"-Transaktion ist, wird eine Fehlermeldung eingeblendet.

Timer können:

• Von einer Aufgabe gestartet und von einer Aufgabe beendet werden.
• Von einer Aufgabe gestartet und von einer Transaktion beendet werden (modelliert in einer Spezifikation).
• Von einer Transaktion gestartet (modelliert in einer Spezifikation) und von einer Aufgabe beendet werden.
• Von einer Transaktion gestartet und von einer Transaktion beendet werden (beide modelliert in einer Spezifikation).
Hinweis: Prozesstimer sind nur von den Transaktionen "Abmelden - standardmäßig" und "Anmelden" abhängig und berücksichtigen nicht die Anzahl an Iterationen, die für eine Aufgabe ausgeführt werden könnten. Nachdem die Aufgabe ausgeführt wurde, wird der Timer angehalten.

Es besteht keine Zuordnung zwischen Starttimern und Endtimern bei einer einzelnen Aufgabe. Z. B. können in einer Tabelle mehrere Timer definiert sein und in der anderen gar keine.

Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch oder im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Beim Definieren von Datensammlungsaufgaben für Anweisungsaufgaben

Sie müssen "Datensammlung" im Feld "Anweisungstyp" auswählen und dann eine vordefinierte Datensammlungsdefinition aus dem Feld "Datensammlungsdefinition" auswählen, wenn Sie eine Datensammlungsaufgabe definieren möchten. Wenn Sie eine Datensammlungsdefinition mit der Aufgabe verknüpfen, aber im Feld "Anweisungstyp" einen anderen Wert als "Datensammlung" auswählen, zeigt die Anwendung auf der Seite "Automatischer Ablauf" keine Datensammlungsfelder an.

Beim Definieren von Berechnungsaufgaben

Die Anwendung verlangt, dass die Datentypen des verknüpften Datenpunkts und Berechnungsparameters identisch sind, wenn Sie eine Berechnungsaufgabe festlegen. Wenn sich die Datentypen unterscheiden, wird eine Fehlermeldung angezeigt. Datenpunktdatentypen werden in der angegebenen Datensammlungsdefinition und Berechnungsparameter-Datentypen in der angegebenen Berechnungsdefinition festgelegt.

Sie müssen eine Revision des Datenpunkts auswählen. Erweitern Sie den Datenpunkt in der Liste, um eine bestimmte Revision auszuwählen.

Beim Definieren von Transaktionsaufgaben

Eine Transaktionsaufgabe kann entweder eine normale Aufgabe oder eine Prozess beenden-Aufgabe sein. Die Option "Prozess starten-Aufgabe" ist für eine Transaktionsaufgabe im Feld "Aufgabentyp" nicht verfügbar. Darüber hinaus ist das Feld "Anforderung zur elektronischen Unterschrift" für Transaktionsaufgaben nicht verfügbar.

Beim Definieren eines mittleren Batches
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Sie können einen mittleren Batch über die Felder im Abschnitt "Batchverarbeitung" definieren. Sie können einen Ausdruck eingeben, um die Menge eines mittleren Batches zu berechnen. Sie können eine Maßeinheit für den mittleren Batch festlegen.

Alle Komponenten, die zum mittleren Batch hinzugefügt werden, müssen dieselbe Maßeinheit aufweisen. Die Anwendung konvertiert die Maßeinheiten für Komponenten, damit sie mit der Maßeinheit für den Batch identisch sind, wenn die Komponenten nicht dieselbe Maßeinheit aufweisen. Die Konvertierung wird durchgeführt, bevor die Komponenten verbraucht werden. Informationen hierzu finden Sie unter "Beim Definieren von Konvertierungsinformationen für Maßeinheiten".

Beim Definieren von Verbrauchswiegeaufgaben
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Sie können eine Verbrauchswiegeaufgabe definieren, um den Punkt im Prozess, an dem eine Komponente verbraucht wird, zu definieren und ggf. erforderliche Regeln für den Wiegevorgang zur Verfügung zu stellen.

Sie können Ausdrücke als Teil der Aufgabe definieren, um die erforderliche Menge sowie die zugehörige minimal und maximal zulässige Menge zu berechnen. Diese berechneten Werte haben Vorrang vor Werten, die aus dem Produkt oder aus der Materialliste abgeleitet werden. Weitere Informationen finden Sie unter "Definieren von Ausdrücken zum Anpassen der Komponentenmengen für das Produkt", "Definieren zulässiger Komponentenmengen im Produktionsauftrag", "Definieren zulässiger Komponentenmengen in der Stückliste" und "Definieren zulässiger Komponentenmengen in der ERP-Stückliste".

Sie können über das Feld "Waagengruppe" eine Waagengruppe für die Verbrauchswiegeaufgabe zuweisen. Sie können eine Waagengruppe auch für Spezifikation, Produktfamilie und Produkt zuweisen.

Die folgenden Regeln gelten beim Zuweisen von Waagengruppen:

• Die für die Produktfamilie zugewiesene Waagengruppe hat Vorrang vor einer Waagengruppe, die der Spezifikation zugewiesen ist.
• Die für das Produkt zugewiesene Waagengruppe hat Vorrang vor der Waagengruppe, die der Produktfamilie zugewiesen ist.
• Die für die Verbrauchswiegeaufgabe zugewiesene Waagengruppe hat Vorrang vor der Waagengruppe, die allen anderen Objekten zugewiesen ist.

Weitere Informationen finden Sie unter "Zuweisen einer Waagengruppe für die Spezifikation", "Zuweisen einer Waagengruppe für die Produktfamilie" und "Zuweisen einer Waagengruppe zu einem Komponentenprodukt".

Sie können zum Wiegen der Komponente auch die folgenden Regeln definieren:

• Zulassen der manuellen Gewichtseingabe
• Anfordern einer elektronischen Unterschrift beim manuellen Wiegen
• Anfordern einer elektronischen Unterschrift, wenn Verletzungen der Wiegetoleranzen außer Kraft gesetzt werden
• Ausschließliches Zulassen einer manuellen Gewichtseingabe
• Vergrößerung der Menge des Batch-Containers um die verbrauchte Materialmenge

Eine Verbrauchswiegeaufgabe kann entweder eine normale Aufgabe oder eine "Prozess beenden-Aufgabe" sein. Die Option "Prozess starten-Aufgabe" ist für eine Verbrauchswiegeaufgabe im Feld "Aufgabentyp" nicht verfügbar.

Vorgehensweise zum Definieren einer Aufgabenliste

Gehen Sie folgendermaßen vor, um eine Aufgabenliste zu definieren:

1. Öffnen Sie die Seite Aufgabenliste. Die Seite "Aufgabenliste" wird innerhalb der Seite "Modellierung" angezeigt.

3. Geben Sie einen Namen für diese Aufgabenliste in das Feld Aufgabenliste ein.
4. Geben Sie die Revision dieser Aufgabenliste in das Feld Revision ein.
5. Wählen Sie einen Ausführungsmodus in der Liste aus.

• "Vorgehensweise zum Hinzufügen einer Berechnungsaufgabe zu einer Aufgabenliste"
• "Vorgehensweise zum Hinzufügen einer Transaktionsaufgabe zu einer Aufgabenliste"
7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
