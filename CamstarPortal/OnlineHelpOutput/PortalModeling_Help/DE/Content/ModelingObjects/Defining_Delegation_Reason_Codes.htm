Definieren von Delegierungsgründen

Delegierungsgründe geben dem Systemadministrator die Möglichkeit anzugeben, aus welchem Grund er eine Aufgabe an einen anderen Mitarbeiter delegiert.

Beim Definieren von Delegierungsgründen

Durch Konfigurieren eines Delegierungsgrunds können Sie angeben, weshalb Sie eine Aufgabe, die einem Mitarbeiter zugewiesen ist, einem anderen Mitarbeiter zuweisen. Sie können z. B. eine Zusammenarbeitsaufgabe für ein Änderungsverwaltungspaket an einen anderen Mitarbeiter delegieren, wenn der Mitarbeiter, dem die Aufgabe ursprünglich zugewiesen wurde, zum Zeitpunkt der Fälligkeit der Zusammenarbeitsaufgabe im Urlaub ist.

"Delegierungsgrund" ist ein erforderliches Feld, wenn eine Aufgabe an einen anderen Mitarbeiter delegiert wird. Informationen zum Delegieren von Aufgaben finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

So wird ein Delegierungsgrund definiert

Gehen Sie folgendermaßen vor, um einen Delegierungsgrund zu definieren:

1. Öffnen Sie die Seite Delegierungsgrund. Die Seite "Delegierungsgrund" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für den Delegierungsgrund in das Feld Delegierungsgrund ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Unter "Gemeinsame Felder auf Modellierungsseiten" finden Sie Informationen zu den optionalen Feldern.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
