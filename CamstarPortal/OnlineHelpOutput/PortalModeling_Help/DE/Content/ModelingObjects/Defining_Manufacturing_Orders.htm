Definieren von Produktionsaufträgen

Ein Produktionsauftrag ist eine Anforderung zur Herstellung eines Produkts. Produktionsaufträge werden in der Regel erstellt, um einen Verkaufsauftrag oder eine Lageranforderung zu erfüllen, und sind oft eine Verbindung zwischen einem MES- und einem ERP-System.

Ein Produktionsauftrag kann eine Liste von Produktcontainern enthalten. Die optionale Beziehung zwischen einem Container und einem Produktionsauftrag wird über den Datensatz des Containers erstellt und verwaltet. Sie können diese Informationen dazu nutzen, den Umlaufbestandsstatus nach Produktionsaufträgen und Umlaufbestandsaktualisierungen an das ERP-System weiterzugeben.

Darüber hinaus können Sie einem Produktionsauftrag benutzerdefinierte dynamische Attribute zuweisen und die Anwendung weist diese benutzerdefinierten Attribute dann zu, wenn Sie einen Container starten. Wenn Sie Informationen am Container selbst speichern, ist es nicht erforderlich, dass Sie oder andere Benutzer die Historie des Containers nach Informationen durchsuchen.

Grundlegendes zu benutzerdefinierten Attributen

Weitere Informationen zu benutzerdefinierten Attributen finden Sie unter "Grundlegendes zu benutzerdefinierten Attributen".

Weitere Informationen zum Zuweisen von benutzerdefinierten Attributen bei einem Containerstart finden Sie unter "Zuweisen von benutzerdefinierten Attributen beim Containerstart".

Beim Definieren eines Produktionsauftrags

"Produktionsauftrag" umfasst die folgenden optionalen Modellierungsdefinitionen:

• ERP-Stückliste
• Dokument
• Auftragsstatus
• Auftragstyp
• Priorität
• Produkt
• Routenschritt
• Nummerierungsregel für Container
• Maßeinheit

"Produktionsauftrag" ist ein optionales Feld in den meisten Containertransaktionen.

Ihre Eingaben in den Feldern "Priorität" und "Geplantes Startdatum" bestimmen die Sortierreihenfolge von Produktionsaufträgen in der Transaktion "Auftragserledigung".

Beim Anhängen von Dokumenten

Sie können die Modellierungsseite "Produktionsauftrag" verwenden, um Dokumente aus einem Produktionsauftrag an diesen anzuhängen, von ihm abzutrennen oder aus ihm anzuzeigen. Sie müssen die Instanz des Produktionsauftrags speichern, bevor Sie ein Dokument an die Instanz anhängen können.

Anhänge sind optional. Sie müssen beim Anhängen von Dateien jedoch bestimmte Felder in der Anwendung ausfüllen, bevor Sie den Container starten.

Die maximale Dateigröße für einen Anhang beträgt 30 MB.

Hinweis: Die Funktionalität für Dokumentanhänge, die auf der Modellierungsseite "Produktionsauftrag" verfügbar ist, kann jedem Modellierungsobjekt hinzugefügt werden. Dies erfolgt über Portal Studio, indem der Webpart "Dokumentanhang" in die Modellierungsseite aufgenommen wird. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch oder im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Beim Anzeigen von Anhängen

Wenn Sie über das Netzwerk auf die Datei zugreifen, kann es sein, dass Ihr Browser verhindert, dass Sie die Datei herunterladen können, bis Sie zusätzliche Schritte ausgeführt haben. Z. B. blockiert Internet Explorer möglicherweise das Herunterladen der Datei, bis Sie in der IE-Leiste auf "Datei herunterladen" geklickt haben.

Wenn Sie ein vorhandenes Dokument anhängen, muss eine Dokumentdatei an die ausgewählte Dokumentinstanz angehängt werden. Andernfalls wird eine Fehlermeldung angezeigt, wenn Sie im Popup auf "Dokument anzeigen" klicken.

Außerdem muss die richtige Software installiert sein, um die Datei anzuzeigen. Falls der Anhang z. B. eine Excel-Tabelle ist, müssen Sie Microsoft Excel installiert haben, um die Datei anzuzeigen.

Wenn Sie ein Dokument hinzufügen, das eine Revision erfordert, erstellt die Anwendung automatisch eine neue Dokumentinstanz.

Definieren zulässiger Komponentenmengen im Produktionsauftrag
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Sie können die erforderliche Komponentenmenge und die minimal sowie die maximal zulässige Menge in der Tabelle "Materialliste" definieren. Sie können Komponentenmengen auch über die Tabelle "Materialliste" für die Stückliste oder die ERP-Stückliste definieren. Die für den Produktionsauftrag definierte Materialliste hat Vorrang vor der Materialliste, die für die Stückliste oder die ERP-Stückliste definiert ist. Weitere Informationen finden Sie unter "Definieren zulässiger Komponentenmengen in der Stückliste" und "Definieren zulässiger Komponentenmengen in der ERP-Stückliste".

Verwenden Sie die Felder "Minimal zulässige Menge" und "Maximal zulässige Menge", um die zulässigen Komponentenmengen zu definieren. Diese Felder werden in den angezeigten Popups angezeigt, wenn Sie auf die Schaltfläche "Neue Zeile hinzufügen" klicken, um ein Material zur Tabelle "Materialliste" hinzuzufügen.

Beim Definieren von Produktionsaufträgen für die Verwendung mit Opcenter Execution Core Scheduling
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Opcenter Execution Core Scheduling verwendet die in der Produktionsauftragsinstanz angegebenen Informationen zum Planen von Aufträgen. Die dem Produktionsauftrag zugewiesene ERP-Route muss mit einem dieser Felder übereinstimmen, wenn Opcenter Execution Core Scheduling verwendet wird:

• ERP-Route, die dem Arbeitsplan des Produkts zugewiesen ist
• ERP-Route, die dem Arbeitsplan für die Produktfamilie zugewiesen ist

Im Feld "Auftragsstatus" auf der Seite "Produktionsauftrag" können Sie einem Produktionsauftrag einen Auftragsstatus zuweisen. Sie können in Opcenter Execution Core Scheduling nur offene Produktionsaufträge planen.

Beim Freigeben von Produktionsaufträgen für die Verwendung mit Opcenter Execution Core Scheduling
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Wenn Sie einen Produktionsauftrag freigeben, sendet die Schnittstelle die Auftragsdaten von Opcenter EX MDD oder Opcenter EX CR an Preactor. Ein Produktionsauftrag wird freigegeben, wenn ein neuer Auftrag gespeichert oder ein bestehender Auftrag aktualisiert wird und die folgenden Bedingungen erfüllt sind:

• Der Produktionsauftrag weist ein Produkt, eine Menge und einen Auftragsstatus auf.
• Dem Produkt oder der Produktfamilie des Produktionsauftrags ist ein Arbeitsplan zugewiesen.
• Der Auftragsstatus ist "Offen".

Produktionsaufträge werden bei der Freigabe in eine Preactor-Bereitstellungstabelle eingefügt. Bereitgestellte Aufträge können aktualisiert werden, indem der Produktionsauftrag geändert und gespeichert wird. Aufträge können aus der Bereitstellungstabelle entfernt werden, indem der Auftragsstatus in "Geschlossen" geändert und der Produktionsauftrag anschließend gespeichert wird. Aufträge werden bei einer Freigabe laufend zur Bereitstellungstabelle hinzugefügt. Beim Importieren von Aufträgen in Preactor wird die aktuelle Auftragsliste in Preactor durch die Aufträge ersetzt, die sich derzeit in der Bereitstellungstabelle befinden.

• Verbrauchscontainer (Los): Das Komponentenmaterial wird als Container in Opcenter EX MDD oder Opcenter EX CR auf der Großmengenebene verfolgt. Ein Komponentencontainer kann einen Teil oder die gesamte erforderliche Menge bis zur aktuellen Menge des Komponentencontainers bereitstellen.
• Verbrauchscontainer (seriell): Das Komponentenmaterial wird als Container in Opcenter EX MDD oder Opcenter EX CR auf serieller Ebene verfolgt. Jede erforderliche Mengeneinheit stellt einen Container dar.
• Los und Lagerort: Das Komponentenmaterial wird nicht in Opcenter EX MDD oder Opcenter EX CR verfolgt, aber Menge und Los werden aufgezeichnet. Informationen werden normalerweise mithilfe von Opcenter Connect MOM an ERP-Systeme übertragen. Im Opcenter Connect MOM-Benutzerhandbuch finden Sie weitere Informationen zu Opcenter Connect MOM.
• Nur Menge: Komponentenmaterialien werden von keinem System verfolgt. "Menge" und andere optionale Felder werden für die Kostenrechnung und sonstige Zwecke gespeichert.
• Nur Lagerort: Das Komponentenmaterial wird nicht in Opcenter EX MDD oder Opcenter EX CR verfolgt, aber Menge und Lagerort werden aufgezeichnet. Informationen werden normalerweise mithilfe von Opcenter Connect MOM an ERP-Systeme übertragen. Weitere Informationen zu Opcenter Connect MOM finden Sie im Opcenter Connect MOM-Benutzerhandbuch.

1. Öffnen Sie die Seite Produktionsauftrag. Die Seite "Produktionsauftrag" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für diese Aufgabenliste in das Feld Produktionsauftrag ein.
4. Weitere Informationen zum Hinzufügen einer Materialliste finden Sie unter "Vorgehensweise zum Hinzufügen einer Materialliste zu einem Produktionsauftrag".

6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
8. Weitere Informationen zum Anhängen von Dokumenten finden Sie in einem der folgenden Verfahren zum Anhängen eines Dokuments:
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das nicht wiederverwendet wird, an einen Produktionsauftrag"
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das wiederverwendet wird, an einen Produktionsauftrag"
• "Vorgehensweise zum Anhängen eines vorhandenen Dokuments an einen Produktionsauftrag"
9. Weitere Informationen zum Abtrennen von Dokumenten finden Sie unter "Vorgehensweise zum Abtrennen eines Dokuments von einem Produktionsauftrag".
10. Weitere Informationen zum Anzeigen von angehängten Dokumenten finden Sie unter "Vorgehensweise zum Anzeigen eines Dokuments, das an einen Produktionsauftrag angehängt ist".

Vorgehensweise zum Hinzufügen einer Materialliste zu einem Produktionsauftrag

Gehen Sie folgendermaßen vor, um einem Produktionsauftrag eine Materialliste hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren eines Produktionsauftrags" aus.

Oder

Wählen Sie eine vorhandene Produktionsauftrag-Instanz aus.

2. Expandieren Sie den Abschnitt Materialien.
3. Klicken Sie auf Neue Zeile hinzufügen in der Tabelle "Materialliste". Das Popup "Hinzufügen" wird angezeigt.
4. Wählen Sie ein Produkt in der Liste aus.
5. Wählen Sie eine Verbrauchssteuerung in der Liste aus.
6. Geben Sie einen Wert in das Feld Erforderliche Menge ein.
7. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
8. Klicken Sie auf OK. Das Popup "Hinzufügen" wird geschlossen und "Materiallistenelement" wird in der Tabelle "Materialliste" auf der Seite "Produktionsauftrag" angezeigt.
9. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
