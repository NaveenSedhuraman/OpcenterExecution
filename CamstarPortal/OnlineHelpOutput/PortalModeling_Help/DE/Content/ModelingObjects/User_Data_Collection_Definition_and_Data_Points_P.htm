Benutzerdatensammlungsdefinition und Datenpunkte

Die Benutzerdatensammlungsdefinition stellt eine Methode zum Erstellen von Datenpunkten dar. Diese können einer Aufgabenliste für einen automatischen Ablauf zugewiesen werden. Ein Datenpunkt stellt einen Punkt im Produktionsprozess dar, an dem Daten gesammelt werden müssen oder können. Mehrere definierte Datenpunkte können in einer Benutzerdatensammlungsdefinition gruppiert werden, die dann einem automatischen Ablauf zugewiesen werden kann.

Beim Definieren einer Benutzerdatensammlung

Beim Erstellen von Benutzerdatensammlungsdefinitionen können Sie sowohl Wertdatenpunkte als auch Objektdatenpunkte einschließen, und zwar entweder in einem Zeilen-Spalten-Layout oder in einem Tabellen-Layout. Die Anwendung stellt die Datenpunkte zur Laufzeit dynamisch im angegebenen Layout dar. Weitere Informationen finden Sie unter "Erstellen von Benutzerdatensammlungsdefinitionen".

Wenn Hinzufügen von Wertdatenpunkten

Wenn Sie einer Benutzerdatensammlung Wertdatenpunkte hinzufügen, haben Sie folgende Möglichkeiten:

• Definieren von oberen und unteren Limits durch Eingeben statischer Werte oder eines Ausdrucks
• Angeben, ob Limitüberschreitungen erlaubt sind
• Sich für eine Zuordnung des Datenpunktwerts zu einem Containerattribut entscheiden

Sie können einen Ausdruck verwenden, um das obere Limit, das untere Limit oder beide Limits zu definieren. Die Limits werden zur Laufzeit bestimmt, wenn ein Ausdruck verwendet wird.

Verwenden Sie Die Syntax für einheitliche Ausdrücke (Unified Expressions) einer Anwendung, wenn Sie obere und untere Limits mit einem Ausdruck definieren. Ausdrücke können bis zu 255 Zeichen umfassen und Folgendes beinhalten:

• Feldausdrücke, z. B. Container.CurrentStatus.Spec.UserDefinedInteger
• Eine Kombination aus einen Feldausdruck und einfacher Arithmetik, z. B. (100-Container.CurrentStatus.Spec.UserDefinedInteger) * 0,2
Hinweis: Bei der Eingabe von Ausdrücken wird keine Validierung der richtigen Syntax vorgenommen. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Beim Definieren von Limits für Wertdatenpunkte

Limits gelten nur für Wertdatenpunkte, sind jedoch sowohl für das Spalten-Zeilen- als auch für das Iterations-Tabellenlayout verfügbar. Das Kontrollkästchen "Limits anzeigen" auf der Seite "Benutzerdatensammlungsdefinition" bestimmt, ob die Anwendung die Limits, die für die Wertdatenpunkte definiert wurden, bei der Datensammlung anzeigt. Wenn das Kontrollkästchen aktiviert ist (die Standardeinstellung), zeigt die Anwendung die Limits für diese Datenpunkte an.

Wenn Limits definiert sind, bestimmt das Kontrollkästchen "Limitüberschreitung erlaubt" die Farbe, die dem Benutzer angezeigt wird, wenn ein eingegebener Wert außerhalb eines Limits liegt. Es bestimmt auch, ob eine Datensammlungsaufgabe eines automatischen Ablaufs als bestanden oder fehlgeschlagen bewertet wird. Unabhängig davon, ob die Aufgabe des automatischen Ablaufs als bestanden oder fehlgeschlagen bewertet wird, akzeptiert die Anwendung jeden Wert, den Sie eingeben, selbst wenn dieser außerhalb eines Limits liegt.

Beim Hinzufügen von Objektdatenpunkten

Ein Objektdatenpunkt dient im Wesentlichen demselben Zweck wie ein Wertdatenpunkt – er bietet eine Möglichkeit, Daten an einem bestimmten Punkt im Produktionsprozess zu sammeln. Ein Objektdatenpunkt referenziert ein bestimmtes Objekt. Sie können z. B. einen Objektdatenpunkt verwenden, um einen Ursachencode zu erstellen und vordefinierte Prüfstatusergebnisse aufzuzeichnen. Diese Ergebnisse können Bedingungen wie Kontamination und Oberflächenkratzer enthalten. Der Bediener in der Fertigung führt die Untersuchungsaufgabe aus, indem er einfach das entsprechende Ergebnis aus einer Wertauswahlliste auswählt.

Hinweis: Für Objektdatenpunkte gelten keine Limits.

Vorgehensweise zum Definieren einer Benutzerdatensammlung

Gehen Sie folgendermaßen vor, um eine Benutzerdatensammlung zu definieren:

1. Öffnen Sie die Seite Benutzerdatensammlungsdefinition. Die Seite "Benutzerdatensammlungsdefinition" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den eindeutigen Namen dieser Definition in das Feld Benutzerdatensammlungsdefinition ein.
4. Geben Sie den Revisionsidentifikator für diese Definition in das Feld Revision ein.
5. Aktivieren Sie das Kontrollkästchen Standardrevision, falls dies die Standardrevision ist.
6. Wählen Sie den Typ Datenpunktlayout aus der Liste aus: Iterationstabelle oder Zeilen/Spaltenposition.
7. Fügen Sie der Datenpunkttabelle mit einem der nachfolgenden Verfahren einen Datenpunkt hinzu:
• "Vorgehensweise zum Hinzufügen eines Wertdatenpunkts"
• "Vorgehensweise zum Hinzufügen eines Objektdatenpunkts"
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

Vorgehensweise zum Hinzufügen eines Wertdatenpunkts

Gehen Sie folgendermaßen vor, um einer Benutzerdatensammlung einen Wertdatenpunkt hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Benutzerdatensammlung" aus.

Oder

Wählen Sie eine vorhandene Instanz einer Benutzerdatensammlungsdefinition aus.

2. Klicken Sie in der Datenpunkttabelle auf Neue Zeile hinzufügen. Das Menü "Datenpunkttyp" wird angezeigt.
3. Wählen Sie Wertdatenpunkt aus. Das Popup "Wertdatenpunktdetail" wird angezeigt.
4. Geben Sie einen Namen in das Feld Datenpunktname ein.
5. Geben Sie einen numerischen Wert ein, um die Zeilenposition anzugeben, an der die Anwendung den Datenpunkt in der Datensammlung anzeigen soll.
6. Geben Sie einen numerischen Wert ein, um die Spaltenposition festzulegen, an der der Wertdatenpunkt in der Datensammlung platziert werden muss.
7. Wählen Sie den Datentyp aus der Liste aus.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf OK. Die Anwendung schließt das Popup und zeigt den Datenpunkt in der Tabelle "Datenpunkte" an.
10. Wiederholen Sie Schritte 2 bis 9, um weitere Wertdatenpunkte hinzuzufügen.
