Definieren von Verkaufsaufträgen

Ein Verkaufsauftrag ist eine Anforderung zur Lieferung eines Produkts. Verkaufsaufträge stammen normalerweise von einem Kunden, können aber auch von einer anderen Fabrik oder Abteilung kommen.

In einem Verkaufsauftrag ist eine Produktmenge definiert, die ein Kunde zu einem bestimmten Datum erwartet. Dies entspricht einer Position eines typischen (ERP-)Verkaufsauftrags. Ein Verkaufsauftrag kann wahlweise einem Container zugewiesen werden, wenn dieser gestartet oder aktualisiert wird. Dies bietet die Möglichkeit, Umlaufbestand (Container) Kundenaufträgen zuzuordnen.

Ein Verkaufsauftrag kann eine Liste von zugehörigen Containern enthalten. Die optionale Beziehung zwischen einem Container und einem Verkaufsauftrag wird über den Datensatz des Containers erstellt und verwaltet.

Beim Definieren eines Verkaufsauftrags

Der Verkaufsauftrag enthält die optionalen Modellierungsdefinitionen "Produkt" und "Maßeinheit".

Der Verkaufsauftrag ist ein optionales Feld in der Transaktion Zwei-Ebenen-Start, Containerwartung und Auftragsverwaltung.

Eine Liste aller Container, die mit diesem Verkaufsauftrag verknüpft sind. Die Referenz auf den Verkaufsauftrag wird erstellt, wenn ein Container erzeugt wird und das optionale Feld "Verkaufsauftrag" während der Starttransaktion dieser Container ausgefüllt wird.

Vorgehensweise zum Definieren eines Verkaufsauftrags

Gehen Sie folgendermaßen vor, um einen Verkaufsauftrag zu definieren:

1. Öffnen Sie die Seite Verkaufsauftrag. Die Seite "Verkaufsauftrag" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen in das Feld Verkaufsauftrag ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
