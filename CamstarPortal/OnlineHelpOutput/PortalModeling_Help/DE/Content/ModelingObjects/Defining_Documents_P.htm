Definieren von Dokumenten

Eine Dokumentinstanz enthält die Informationen, die erforderlich sind, um ein externes Dokument anzuzeigen. Jede Definition enthält den Dateinamen und die Revision des Dokuments.

Sie können nach jedem Dateityp suchen und jeden Dateityp hochladen und ihn dem Dokumentobjekt zuordnen. Außerdem können Sie eine beliebige Versionsnummer für die Datei eingeben. Wenn eine Datei, die der Objektinstanz zugewiesen ist, in der Datenbank gespeichert wurde, wird der Name der Datei im Feld "Name der gespeicherten Datei" angezeigt.

Sie können auch eine Datei, die ursprünglich als Teil der Objektinstanz hochgeladen wurde, herunterladen und sie in der Datenbank speichern.

Die meisten Modellierungsobjekte enthalten eine Referenz auf ein Dokument. Dokumente werden in diesem Zusammenhang dafür verwendet, erweiterte (unstrukturierte) Informationen, etwa Produktzeichnungen, gescannte Bilder, zugehörige vorhandene Produktionsspezifikationen oder Arbeitsvorgänge, zu speichern. Jede Instanz eines Dokuments (CDO) enthält einen Bezeichner (Identifikator) für das externe Dokument, z. B. einen Dateinamen.

Hinweis: Rezept ist eine Unterklasse des Dokuments. Weitere Informationen zum Modellierungsobjekt "Rezept" finden Sie unter "Definieren von Rezepten".

Anforderungen

Sie müssen für den Typ des Dokuments, das Sie öffnen möchten, die entsprechende Software auf Ihrem Computer oder Server installiert haben.

HTTP-Navigieren ist auf die virtuellen Verzeichnisse des Portal-Servers beschränkt. Daher muss sich das Dokument im Camstar Portal-Verzeichnis oder in einem seiner Unterverzeichnisse befinden. (Sie können keinen Pfad wie "C:\Meine Dokumente" verwenden, um den Speicherort des Dokuments zu definieren.)

Der Standardpfad für URL-Navigieren wird auf Grundlage der URL festgelegt, die für die Anmeldung zu dieser Sitzung verwendet wurde. Daher müssen Sie die URL für das Dokument evtl. bearbeiten. Wenn Sie sich z. B. mit "https://localhost/CamstarPortal/default.htm#/login" angemeldet haben, wird der vollständig qualifizierte Domänenname nicht im Feld "Speicherort der Datei" angezeigt, nachdem Sie einen Wert im Feld "Ansichtsmodus" ausgewählt haben. Sie müssen sich entweder mit dem vollqualifizierten Domänennamen anmelden oder die URL bearbeiten, um den vollqualifizierten Domänennamen zu benutzen und sicherzustellen, dass Benutzer von anderen Domänen auf das Dokument zugreifen können.

Die Standardeinstellungen für die Browsersicherheit verhindern, dass Dokumente auf unsicheren Websites geöffnet werden. Sie müssen Ihre Einstellungen ändern, um unsicheren Inhalt zuzulassen.

Hinweis: Wenn Sie diese Einstellung im Browser ändern, werden Sie von der Anwendung abgemeldet.

Beim Definieren eines Dokumentes

Diese Regeln gelten, wenn Sie Dokumente definieren:

• Jede Instanz eines Dokuments enthält einen Dateispeicherort im Feld "Speicherort der Datei", es sei denn, Sie definieren einen Hyperlink zu einer externen Web-Adresse.
• Der ausgewählte Ansichtsmodus bestimmt, ob die Anwendung die Datei in die Datenbank hochlädt, einen Link zu einer Datei erstellt oder einen Link zu einer externen Web-Adresse erstellt. Wählen Sie "Lokale Datei" aus und klicken Sie auf das Symbol "Datei hochladen", um zur angegebenen Datei zu navigieren und sie in die Datenbank hochzuladen.
• Das Feld "Name der gespeicherten Datei" ist schreibgeschützt und zeigt den Dateinamen der hochgeladenen, lokalen Datei an, wenn Sie auf "Speichern" geklickt haben. Das Feld "Name der gespeicherten Datei" bleibt leer, wenn Sie einen Link zu einer Datei erstellen (der Ansichtsmodus ist "HTTP-Datei").
• Die Zeit, die für das Herunterladen/Hochladen von Dateien benötigt wird, hängt von der Größe der Datei ab.
• Dokumente, die mit der Funktion für Anhangdokument als wiederverwendbare Dokumente hochgeladen wurden, werden auf der Seite "Dokument" angezeigt.

"Dokument" ist:

• Ein erforderliches Feld in der Definition eines Dokumentsatzes.
• Ein optionales Feld in der Definition einer Spezifikation (in der Rezeptdatei) und Umlaufbestandsnachricht.
Hinweis: Eine Dokumentdefinition kann zugehörige Umlaufbestandsnachrichten haben, wenn sie mit einem Feld aus einer Containerdefinition verbunden ist. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch oder im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Beim Zugreifen auf Dateien in einem externen Repository

Sie können im Portal auf Dokumente zugreifen, die in einem externen Repository gespeichert sind. Sie können für das Modellierungsobjekt "Mitarbeiter" einen Benutzernamen und ein Passwort festlegen, wenn für das externe Repository ein anderer Benutzername und ein anderes Kennwort erforderlich sind als für den Zugriff auf das Portal. Sie müssen den Authentifizierungstyp auf "Einfach" festlegen, um diese Funktion verwenden zu können.

• Der Ansichtsmodus bestimmt, ob die Anwendung die Datei in die Datenbank hochlädt, einen Link zu einer Datei erstellt oder einen Link zu einer externen Web-Adresse erstellt. Wählen Sie "HTTP-Datei" aus, um eine Verknüpfung zur angegebenen Datei im externen Repository zu erstellen.
• Die Anwendung zeigt die Auswahlliste "Authentifizierungstyp" nur an, wenn der Ansichtsmodus auf "HTTP-Datei" festgelegt ist.
• Sie müssen den Authentifizierungstyp im Modellierungsobjekt "Dokument" auf "Einfach" festlegen, damit die im Modellierungsobjekt "Mitarbeiter" festgelegten Felder für Benutzername und Kennwort verwendet werden. Bei Auswahl von "Einfach" übergibt die Anwendung den im Modellierungsobjekt "Mitarbeiter" festgelegten Benutzernamen und das Kennwort zur Authentifizierung an das externe Dokumentenrepository.
• Der im Modellierungsobjekt "Mitarbeiter" festgelegte Benutzername und das Kennwort müssen mit dem erforderlichen Benutzernamen und dem Kennwort für den Zugriff auf das Repository übereinstimmen.
• Die Anwendung übergibt die Anmeldeinformationen des Benutzers an das Repository, wenn der Authentifizierungstyp auf "Einfach" festgelegt ist, im Modellierungsobjekt "Mitarbeiter" aber kein Benutzername und kein Kennwort festgelegt wurden.
Beim Erstellen von Links zu elektronischen Arbeitsanweisungsdateien

Sie können einen Link zu EWI-Dateien erstellen, damit Benutzer mit Zugriff auf Teamcenter EWI-Informationen in einem separaten Popup anzeigen können.

• Der Ansichtsmodus bestimmt, ob die Anwendung die Datei in die Datenbank hochlädt, einen Link zu einer Datei erstellt oder einen Link zu einer externen Web-Adresse erstellt. Wählen Sie eine URL aus, um einen Link zu einer EWI-Datei zu erstellen.
• Die Anwendung zeigt das Feld "URL" nur an, wenn der Ansichtsmodus auf "URL" festgelegt ist. Bei Auswahl von "URL" für den Ansichtsmodus werden auch die Felder "Authentifizierungstyp", "Speicherort der Datei", "Name der gespeicherten Datei" und "Dateiversion" ausgeblendet.
• EWI-Dateien werden in einem separaten Popup-Fenster angezeigt. Sie werden im Popup aufgefordert, Benutzernamen und Kennwort für Teamcenter einzugeben, bevor die EWI-Datei angezeigt wird. Die Teamcenter-Sitzung bleibt geöffnet, bis Ihr Browser geschlossen wird.
• Beim Definieren eines Links zu einer EWI-Datei dürfen im Feld "URL" keine URLs eingegeben werden, die Parameter, die Arbeitsanweisungen konfigurieren, z. B. Einstellungen für Produktansicht, Endmontage, Dokumentviewer oder Anmeldeinformationen, enthalten.

So wird ein Dokument definiert

Gehen Sie folgendermaßen vor, um ein Dokument zu definieren:

1. Öffnen Sie die Seite Dokument. Die Seite "Dokument" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für das Dokument in das Feld Dokument ein.
4. Geben Sie die Revision dieses Dokuments in das Feld Revision ein.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt, aktiviert die Schaltfläche "Dokument anzeigen" und zeigt eine Bestätigungsmeldung an.

So wird eine Datei zu einer Dokumentdefinition hinzugefügt

Gehen Sie folgendermaßen vor, um eine Datei zu einer Dokumentdefinition hinzuzufügen:

1. Führen Sie die Schritte unter "So wird ein Dokument definiert" aus.
2. Möchten Sie eine Datei in die Datenbank hochladen oder einen Hyperlink zu einer Datei bzw. zu einer externen Web-Adresse oder EWI-Datei erstellen?

a. Wählen Sie Lokale Datei im Feld Ansichtsmodus aus (sofern noch nicht ausgewählt). Die Schaltfläche "Durchsuchen" wird angezeigt.
