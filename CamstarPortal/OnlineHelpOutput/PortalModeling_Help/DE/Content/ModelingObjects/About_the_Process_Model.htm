Informationen zum Prozessmodell

Prozessmodelldefinitionen können in zwei Kategorien eingeteilt werden:

• Arbeitsplandefinitionen
• Produktdefinitionen

Arbeitsplandefinitionen

Ein Arbeitsplan definiert eine Abfolge von Schritten, die verwendet werden, um ein Produkt herzustellen. Arbeitspläne sind die fundamentalen Komponenten der Modellierung und das Kernstück des Fertigungsprozesses. Die folgenden Objekte müssen in der aufgeführten Reihenfolge definiert werden, bevor ein Arbeitsplan definiert werden kann:

1. Ressourcen
2. Containerlevels
3. Operationen
4. Spezifikationen

Weitere Informationen zu Ressourcen finden Sie unter "Definieren von Ressourcen und Ressourceneinrichtung". Weitere Informationen zu Arbeitsplänen finden Sie unter "Definieren von Arbeitsplänen".

Sie können auch ERP-Routen definieren – das ERP-Konzept, das einem Opcenter EX MDD- oder Opcenter EX CR-Arbeitsplan am nächsten kommt. Das Objekt "ERP-Route" in Opcenter EX MDD oder Opcenter EX CR soll ein Spiegel der Routendefinitionen in ERP sein, wenn die in ERP gespeicherte Definition der Master ist.

Produktdefinitionen

Produkte sind die Materialien, die in einer Fabrik produziert oder von Drittlieferanten an sie geliefert werden. Ein Produkt kann ein Endprodukt, eine Baugruppe oder eine Komponente sein. Produkte können zur einfacheren Wartung in Produktfamilien gruppiert werden.

Eine Stückliste (BOM, Bill of Materials) enthält die Komponenten in den erforderlichen Mengen, um ein Produkt herzustellen (Endprodukt oder Baugruppe). Eine ERP-Stückliste verweist auf Schritte in einer ERP-Route, anstatt auf Schritte in einem Opcenter EX MDD- oder Opcenter EX CR-Arbeitsplan zu verweisen.
