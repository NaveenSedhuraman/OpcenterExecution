Definieren von Trainingseintragsstatus

Ein Trainingseintragsstatus ist eine Art Nutzercode. Ein Trainingseintragsstatus hilft dabei zu bestimmen, ob eine Person für eine Aufgabe qualifiziert ist. Die Trainingseintragsstatuscodes, die Sie hier erstellen, stehen den Trainern bei der Trainingseintragsverwaltung in den Auswahllisten zur Verfügung.

Beispiele

Sie können entsprechend Ihren geschäftlichen Anforderungen mehrere benutzerdefinierte Trainingseintragsstatuscodes erstellen. Dies sind einige Beispiele für Trainingseintragsstatuscodes:

• Im Training – Gibt an, dass der Mitarbeiter momentan ein Training für eine bestimmte Zertifizierung absolviert.
• Zertifiziert – Gibt an, dass der Mitarbeiter für eine bestimmte Aufgabe qualifiziert ist.
• Entzertifiziert – Gibt an, dass der Mitarbeiter seine Fähigkeiten wieder auf den neusten Stand bringen muss.
• Nicht zertifiziert – Gibt an, dass der Mitarbeiter das Training noch nicht begonnen hat und deshalb auch nicht für eine bestimmte Aufgabe qualifiziert oder autorisiert ist.
• Trainer – Gibt an, dass der Mitarbeiter für die Aufgabe qualifiziert ist.

Berechtigung

Im Rahmen der Definition von Trainingseintragsstatuscodes vergeben Sie auch die Befugnis, eine bestimmte Aufgabe ausführen zu dürfen oder nicht. Sie können auch festlegen, dass die Aufgabe nur ausgeführt werden kann, wenn die entsprechenden elektronischen Unterschriften eingeholt wurden.

Z. B. können Sie Ihr System so einrichten, dass ein Mitarbeiter mit dem Status "Zertifiziert" oder "Trainer" Transaktionen durchführen kann, während Mitarbeiter mit dem Status "Nicht zertifiziert" oder "Entzertifiziert" dazu nicht berechtigt sind. Außerdem könnten Sie auch einer oder mehreren Zertifizierungen, z. B. "Im Training", diese Berechtigung unter der Voraussetzung erteilen, dass die entsprechende elektronische Unterschrift erteilt wird.

Beim Definieren eines Trainingseintragsstatuscodes

Wenn Sie "Mit elektr. Unterschrift erlauben" als Berechtigung auswählen, kann ein Benutzer eine Funktion, die eine bestimmte Trainingsanforderung voraussetzt, nur mit der erforderlichen elektronischen Unterschrift für die entsprechende Transaktion durchführen. Das setzt allerdings voraus, dass die Spezifikation, an der die Trainingsanforderung überprüft wird, mit einer Anforderung zur elektronischen Unterschrift verknüpft ist.

Hinweis: Der Trainingseintragsstatus wird vom Trainingseintrag referenziert.

Vorgehensweise zum Definieren eines Trainingseintragsstatus

Gehen Sie folgendermaßen vor, um einen Trainingseintragsstatus zu definieren:

1. Öffnen Sie die Seite Trainingseintragsstatus. Die Seite "Trainingseintragsstatus" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen des Status in das Feld Trainingseintragsstatus ein.
4. Wählen Sie eine Berechtigung in der Liste aus.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
