Definieren von Textvariablen

Textvariablen werden in Nachrichten eingefügt und verwendet, um einen Wert zur Laufzeit zu ermitteln. Sie werden von Objekten wie den Umlaufbestandsnachrichten, Historienabfragen usw. verwendet.

Jede Textvariablendefinition enthält einen Namen und einen Ausdruck und beschreibt, wie vorzugehen ist, wenn der Ausdruck nicht ordnungsgemäß ausgewertet werden kann. Der Name ist ein in eine statische Zeichenkette eingebetteter Wert, dem ein Präfix vorangestellt ist, das ihn als Textvariable kennzeichnet. Sie müssen das #-Zeichen vor den Namen der Textvariableninstanz setzen (z. B. #CONTAINER), wenn Sie in einer aktivierten Zeichenkette auf eine Textvariable verweisen.

Eine Textvariable mit dem Namen CONTAINER kann z. B. in die Meldung "#CONTAINER wurde angehalten" eingefügt werden. Der Ausdruck ist eine Zeichenkette, die zur Laufzeit ausgewertet wird. Mit dem resultierenden Wert wird der Identifikator in der Laufzeitversion der Zeichenfolge ersetzt. In diesem Beispiel wird der Identifikator durch den Namen des Containers ersetzt.

Wie genau ein gültiger Ausdruck aussieht, hängt von der Art der Nachricht ab, die die Textvariable referenziert. Einige Ausdrücke (und somit Textvariablen) können von allen Nachrichtentypen verwendet werden. Mithilfe einer Textvariablen können z. B. Werte für die aktuellen SessionValues integriert werden. Andere Ausdrücke ergeben möglicherweise nur im Kontext eines bestimmten Nachrichtentyps einen Sinn. Eine Umlaufbestandsnachricht kann z. B. eine Variable für ein Produkt enthalten, das momentan auf der Maschine bearbeitet wird, wobei die Anmeldetransaktion die Umlaufbestandsnachricht anzeigt. Weitere Informationen zur Verwendung von Textvariablen mit Umlaufbestandsnachrichten finden Sie unter "Definieren von Umlaufbestandsnachrichten".

Hinweis: Textvariablen werden nur in Zeichenkettenfeldern ausgewertet, für die Designer-Einstellungen für die Ersetzung konfiguriert sind. Weitere Informationen finden Sie im Designer-Benutzerhandbuch.

Felddefinitionen der Seite "Textvariable"

In dieser Tabelle sind die Felder definiert, die nur auf der Seite "Textvariable" zu finden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Allgemein

Ausdruck

Die auszuwertende Zeichenkette, die den Wert zur Laufzeit erzeugt.

Optional

Verwendung

Definiert den Typ der Variablen. Z. B.:

• Allgemein (in jedem Text verwendbar)
• Umlaufbestandsnachricht
• Alarmmeldung
• Anweisungstext

Optional

Wert für Auflösungsfehler

Ein Wert, der verwendet wird, wenn der Ausdruck nicht erfolgreich ausgewertet werden kann. Dies kann eine leere Zeichenfolge sein (es wird ignoriert, dass der Ausdruck nicht ausgewertet werden kann) oder ein anderer Wert (z. B. "ERROR!"), um deutlich zu machen, dass der Text (der erstellten Nachricht) nicht vollständig oder nicht exakt ist.

Optional

Vorgehensweise zum Definieren einer Textvariable

Gehen Sie folgendermaßen vor, um eine Textvariable zu definieren:

1. Öffnen Sie die Seite Textvariable. Die Seite "Textvariable" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen in das Feld Textvariable ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
