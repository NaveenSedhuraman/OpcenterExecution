Definieren von Produkten

Produkte sind die Materialien, die in einer Fabrik oder von externen Zulieferern für eine Fabrik hergestellt werden. Produkte können Endprodukte, Unterbaugruppen und Komponenten sein. Jede Produktdefinition enthält zwei grundlegende Informationen: Daten, die das Produkt beschreiben, und Daten, die Standard-Verarbeitungsinformationen enthalten, die verwendet werden, wenn ein neue Produktcontainer gestartet wird.

Wenn Sie folgende Daten in die Produktdefinition eingeben (sofern in Ihren Geschäftsregeln so vorgegeben), müssen diese Daten nicht jedes Mal neu eingeben werden, wenn ein Container mit diesem Produkt gestartet wird:

• Arbeitsplan
• Startmenge
• Startmenge2
• Maßeinheit
• Sekundäre Maßeinheit
• Kunde
• Waagengruppe
Hinweis: Das Feld "Waagengruppe" wird nur angezeigt, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Diese Informationen, mit Ausnahme der Angabe zum Kunden, überschreiben alle bereits auf Ebene der Produktfamilie gemachten Angaben. (Der Kunde wird nicht auf Ebene der Produktfamilie festgelegt.)

Darüber hinaus können Sie einem Produkt benutzerdefinierte, dynamische Attribute zuweisen; die Anwendung verwendet diese dann automatisch, wenn Sie einen Container starten. Wenn Sie Informationen am Container selbst speichern, ist es nicht erforderlich, dass Sie oder andere Benutzer die Historie des Containers nach Informationen durchsuchen.

Der Portal-Benutzer kann ein Ereignis für ein Produkt erstellen.

Zuweisen eines Produkts zu einer Produktfamilie

Ein Produkt kann zu einer Produktfamilie gehören, die verwendet wird, um Produkte mit gemeinsamen Attributen, wie Arbeitsplan, Trainingsanforderungen und Startmengen, zu einer Gruppe zusammenzufassen. Produktfamilien vereinfachen die Pflege der Produktinformationen, da sie gemeinsame Daten für eine Gruppe von Produkten festlegen. Die für eine Produktfamilie definierten Attribute gelten für jedes Produkt der Produktfamilie.

Das Produkt kann die folgenden Attribute von der zugewiesenen Produktfamilie erben:

• Arbeitsplan
• Dokumentsatz
• Startmenge (und sekundäre Startmenge)
• Start-Maßeinheit (und sekundäre Startmaßeinheit)
• Standardkosten
• Aktuelle Kosten

Wenn Sie einen Wert für eines dieser Attribute auf der Seite "Produkt" auswählen, überschreiben Sie damit die Werte in der zugehörigen Produktfamiliendefinition.

Berechnen des Ablaufdatums auf der Seite "Produkt"
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Auf der Seite "Produkt" können Sie die Felder definieren, die zur automatischen Berechnung des Ablaufdatums für einen Container erforderlich sind, wenn eine vordefinierte Transaktion für die angegebene Spezifikation erfolgreich ist. Es handelt sich um folgende Felder:

• Ablaufdatumtransaktion
• Ablaufdatum-Spez.
• Maßeinheit für Auflaufperiode
• Ablaufperiode

Die Angaben zum Ablaufdatum auf Produktebene haben Vorrang vor den entsprechenden Angaben in der Spezifikation

Wenn das definierte Ablaufdatum des Containers vor dem berechneten Datum liegt, wird das Ablaufdatum nicht aktualisiert, es sei denn, das Kontrollkästchen "Ablaufdatum immer überschreiben" ist aktiviert.

Wenn als "Maßeinheit für Auflaufperiode" der Wert "Monate" festgelegt ist und die Berechnung des Ablaufdatums ein ungültiges Datum ergibt, wird das Ablaufdatum angepasst und auf den letzten Tag des Monats festgelegt. Beispiel: Wenn das aktuelle Datum der 31. Januar ist und die Ablaufperiode drei Monate beträgt, ist das berechnete Ablaufdatum der 31. April. Das Datum wird in diesem Fall auf den 30. April festgelegt.

Zuweisen eines Standardlagerorts für ein Produkt
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Sie können einem Produkt einen Standardlagerort zuweisen. Sie können auch einer Produktfamilie und einer Fabrik einen Standardlagerort zuweisen.

Die folgenden Regeln gelten beim Zuweisen von Standardlagerorten:

• Der Standardlagerort, der für das Produkt zugewiesen ist, hat Vorrang vor dem Standardlagerort, der für die Produktfamilie zugewiesen ist.
• Der Standardlagerort, der für die Produktfamilie zugewiesen ist, hat Vorrang vor dem Standardlagerort, der für die Fabrik zugewiesen ist.
• Die Anwendung weist einem Container den Standardlagerort automatisch zu, wenn der Container in eine Operation bewegt wird, die als Lagerort definiert ist, sofern für das Produkt ein Standardlagerort definiert wurde.

Weitere Informationen finden Sie unter "Zuweisen eines Standardlagerorts für eine Produktfamilie" und "Zuweisen eines Standardlagerorts für eine Fabrik".

Zuweisen einer Waagengruppe zu einem Komponentenprodukt
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Sie können im Feld "Waagengruppe" eine Waagengruppe definieren, um die zulässigen Waagen zum Wiegen von Komponentenprodukten zu definieren. Sie können eine Waagengruppe auch einer Spezifikation, Produktfamilie und Verbrauchswiegeaufgabe zuweisen.

Die folgenden Regeln gelten beim Zuweisen von Waagengruppen:

• Die für die Produktfamilie zugewiesene Waagengruppe hat Vorrang vor einer Waagengruppe, die der Spezifikation zugewiesen ist.
• Die für das Produkt zugewiesene Waagengruppe hat Vorrang vor einer Waagengruppe, die der Produktfamilie zugewiesen ist.
• Die für die Verbrauchswiegeaufgabe zugewiesene Waagengruppe hat Vorrang vor der Waagengruppe, die allen anderen Objekten zugewiesen ist.

Weitere Informationen finden Sie unter "Zuweisen einer Waagengruppe für die Spezifikation", "Zuweisen einer Waagengruppe für die Produktfamilie" und "Beim Definieren von Verbrauchswiegeaufgaben".

Definieren von Ausdrücken zum Anpassen der Komponentenmengen für das Produkt
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Sie können Komponentenmengen während der Batchverarbeitung anpassen, indem Sie Ausdrücke für das Produkt definieren. Angenommen der Betrag eines erforderlichen Komponentenprodukts für einen Batch hängt z. B. von der Wirksamkeit ab. Die Wirksamkeit des Produkts ist nicht festgelegt, sondern hängt vom jeweiligen Komponentencontainer ab. Wenn ein Komponentencontainer in Opcenter EX MDD oder Opcenter EX CR erstellt wird, wird seine Wirksamkeit basierend auf der eingehenden Prüfung aufgezeichnet. Sie können Ausdrücke definieren, die die erforderliche Menge berechnen, sowie die minimale und maximale Toleranz bei der Verarbeitung, um Wirksamkeitsabweichungen zwischen Komponentencontainern zu berücksichtigen. Sie können die Ausdrücke für das Produkt verwenden, um folgende Werte anzupassen:

• Erforderliche Komponentenmenge für den Batch
• Minimal zulässige Komponentenmenge für den Batch
• Maximal zulässige Komponentenmenge für den Batch

Sie können diese Ausdrücke auch in der Aufgabenliste für eine Verbrauchswiegeaufgabe definieren. Die Ausdrücke, die für eine Verbrauchswiegeaufgabe definiert sind, haben Vorrang vor den Ausdrücken, die für ein Produkt definiert sind. Weitere Informationen finden Sie unter "Beim Definieren von Verbrauchswiegeaufgaben".

Verwenden Sie die Syntax für einheitliche Ausdrücke (Unified Expressions), wenn Sie einen Ausdruck definieren. Ausdrücke können bis zu 4.000 Zeichen umfassen und Folgendes beinhalten:

• Variablen; z. B. (?InMenge)
• Benutzerdefinierte Konstanten; z. B. Constant::Pi

Weitere Informationen zur Syntax für einheitliche Ausdrücke (Unified Expressions) finden Sie im Opcenter Execution Medical Device und Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.

Definieren von Endprodukten als batchgesteuerte Produkte
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Das Kontrollkästchen "Batchgesteuert" bietet die Möglichkeit, ein Endprodukt als Batchprodukt zu kennzeichnen. Durch Auswahl des Kontrollkästchens "Batchgesteuert" wird angegeben, dass die Materialmenge der einzelnen Komponenten in der Stückliste des Produkts die erforderliche Gesamtmenge für einen Batch des Endprodukts ist. Die Materialmenge für Komponenten ist nicht die erforderliche Menge je Einheit.

Definieren der Maßeinheitenkonvertierungsinformationen für das Produkt
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Batch Processing installiert ist. Weitere Informationen finden Sie unter "Batch Processing".

Die Tabelle "Maßeinheitenkonvertierung" enthält Felder, mit denen Sie die Mengenkonvertierungsinformationen für Produkte definieren können. Sie können Folgendes definieren:

• Ein Faktor für die Konvertierung einer Verbrauchsmenge des Produkts von einer Maßeinheit in eine andere.
• Eine Regel für die Rundung von konvertierten Mengen.
• Die Anzahl der erlaubten Dezimalstellen für die berechnete konvertierte Menge. Der Maximalwert ist 4.

Die folgenden Regeln gelten beim Definieren von Maßeinheitsumrechnungsinformationen für Produkte:

• Damit eine Rundung stattfindet müssen die Felder "Rundungsregel" und "Dezimalstellen" ausgefüllt werden.
• Konvertierungsinformationen, die für das Produkt definiert sind, haben Vorrang vor Konvertierungsinformationen, die für die Produktfamilie definiert sind.
• Konvertierungsinformationen, die für die Produktfamilie definiert sind, haben Vorrang vor Konvertierungsinformationen, die für die Maßeinheit definiert sind.

Weitere Informationen finden Sie unter "Definieren der Maßeinheitenkonvertierungsinformationen für die Produktfamilie" und "Definieren von Maßeinheiten".
