Definieren von Objektgruppen

Objektgruppen werden verwendet, um Gruppen von Modellierungsobjekten, z. B. Nutzercodes, zu erstellen.

Es gibt zwei Arten von Objektgruppen:

• Named Object Groups
• Revisionierte Objektgruppen

Named Object Groups werden verwendet, um Nutzercodes oder andere Named Data Objects zu gruppieren. Revisionierte Objektgruppen dienen zum Gruppieren von revisionierten Objekten.

Sie können auch andere Gruppen zu einer Gruppe hinzufügen.

Diese Objektgruppen stehen anschließend in anderen Modellierungsobjekten zur Auswahl. Für das Operationsobjekt werden z. B. Komponentendefektgründe verwendet. Allerdings können Sie die von Ihnen erstellten Komponentendefektgründe erst auswählen, nachdem Sie sie der Komponentendefektgrundgruppe hinzugefügt haben. Wenn Sie die Komponentendefektgründe zu einer Gruppe hinzufügen, ist diese Gruppe in der Liste "Komponentendefektgründe" für die Operation verfügbar.

Weitere Informationen zu diesen Elementen finden Sie unter "Named Data Objects und revisionierte Objekte".

Beim Definieren einer Objektgruppe

Jede Gruppe ist eine logische Sammlung ähnlicher Einträge und Untergruppen. Wenn Sie eine bestimmte Gruppe auswählen, schränkt die Anwendung Ihre Auswahlmöglichkeit auf die Einträge und Gruppen dieses Typs ein. Sie können den betreffenden Tabellen mehrere Einträge oder Gruppen hinzufügen.

Alle Objektgruppenseiten enthalten die Schaltfläche "Aufgelöste Einträge". Durch Anklicken der Schaltfläche "Aufgelöste Einträge" wird eine Liste aller Werte angezeigt, die in der Eintragsliste für die Gruppe und alle darin enthaltenen Gruppen festgelegt sind.

Sie müssen einen Eintragstyp aus der Liste "Eintragstyp" auswählen, wenn Sie eine Instanz einer Named Object Group oder eine Instanz eines Nutzercodes auf dem Hauptlevel definieren.

Vorgehensweise zum Definieren einer Objektgruppe

Gehen Sie folgendermaßen vor, um eine Objektgruppe zu definieren:

1. Öffnen Sie eine Modellierungsseite für eine Objektgruppe. Die ausgewählte Objektgruppenseite wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Objektgruppe in das Feld <Objektgruppe> ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Definieren einer Instanz einer Named Object Group oder einer Instanz einer Nutzercodegruppe

Gehen Sie folgendermaßen vor, um eine Instanz einer Named Object Group oder eine Instanz einer Nutzercodegruppe zu definieren:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Objektgruppe" aus.

Oder

Wählen Sie eine vorhandene Objektgruppeninstanz aus.

2. Wählen Sie einen Eintragstyp aus der Liste Eintragstyp aus.
3. Fügen Sie der Gruppe wie folgt Einträge hinzu:
a. Klicken Sie unten in der Tabelle "Einträge" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
b. Klicken Sie in die neue Zeile, um eine Liste von Einträgen anzuzeigen, und wählen Sie einen Eintrag aus, den Sie der Gruppe hinzufügen möchten. Der ausgewählte Eintrag wird der Tabelle hinzugefügt.
c. Wiederholen Sie die Schritte a und b, um der Tabelle weitere Einträge hinzuzufügen.
Hinweis: Eventuell müssen Sie die Liste "Einträge" speichern, nachdem Sie Einträge hinzugefügt haben, damit Sie Gruppen zur Liste "Gruppen" hinzufügen können.
4. Fügen Sie der Gruppe wie folgt eine Untergruppe hinzu:
a. Klicken Sie unten in der Tabelle "Gruppen" auf Neue Zeile hinzufügen. Eine neue Zeile wird angezeigt.
b. Klicken Sie in die neue Zeile, um eine Liste zuvor definierter Gruppen anzuzeigen. Wählen Sie dann eine Gruppe aus, die Sie der Gruppe (als Untergruppe) hinzufügen möchten. Die Anwendung fügt die ausgewählte Gruppe der Tabelle hinzu.
c. Wiederholen Sie die Schritte a und b, um der Tabelle weitere Gruppen hinzuzufügen.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
