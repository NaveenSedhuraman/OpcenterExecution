Definieren von bedingten Prozessanweisungen

Eine bedingte Prozessanweisung (Bill of Process, BOP) ermöglicht es Ihnen, das Standardverhalten des Fertigungsprozesses gemäß Ihren Spezifikationsdefinitionen zu ändern.

Eine Prozessliste ist ein revisioniertes Objekt, das mithilfe der Opcenter EX MDD- oder Opcenter EX CR-Modellierung erstellt wurde. Nachdem sie erstellt ist, weisen Sie die bedingte Prozessanweisung einem Container, einem Produktionsauftrag oder einem Produkt zu, indem Sie die Liste "Zuweisungen von bedingten Prozessanweisungen" verwenden. Wenn eine bedingte Prozessanweisung angegeben wird, hat sie Vorrang vor der Spezifikation.

Sie können mithilfe dieser Funktion eine bedingte Prozessanweisung ändern, nachdem sie erstellt wurde, oder die Details einer bedingten Prozessanweisung anzeigen, ohne Änderungen vorzunehmen.

Beim Definieren einer bedingten Prozessanweisung

Wenn eine bedingte Prozessanweisung in mehreren Spezifikationsschritten, mehrfach in einem Arbeitsplan oder in mehreren Arbeitsplänen verwendet wird, gilt das Überschreiben für alle Schritte in der Spezifikation.

Sie können die bedingte Prozessanweisung verwenden, um die folgenden Felder in der Spezifikation zu überschreiben:

• Dokumentsatz
• Automatischer Ablauf
• Elektr. Unterschrift: Transaktionszuordnung
• Etiketten-Transaktionszuordnung
• Rezeptdatei
• Ressourcengruppe
• Einrichtung
• Trainingsanforderungsgruppe
• Transaktionszuordnung

Alle Transaktionen bis auf "Modellierung", "Export" und "Import" überprüfen auf die Überschreibungen durch eine bedingte Prozessanweisung.

Eine bedingte Prozessanweisung, die einem Eltern-Container zugewiesen ist, wird allen seinen Kind-Containern zugewiesen.

Alle Ressourcen, die Sie einer bedingten Prozessanweisung zuweisen, müssen vorhanden, bereit und verfügbar sein.

Alle Änderungen an einer bedingten Prozessanweisung werden, nachdem diese zugewiesen wurde, sofort verwendet, wenn der Container, der Produktionsauftrag oder das Produkt verarbeitet wird.

Wenn Sie eine bedingte Prozessanweisung von einem Container, einem Herstellungsauftrag oder einem Produkt löschen und an seiner Stelle eine neue bedingte Prozessanweisung zuordnen, zeichnet die Anwendung die Zuweisungen der bedingten Prozessanweisung zum Container, zum Produktionsauftrag bzw. zum Produkt in den Produktions- und Modellierungsprotokollen auf.

Zuweisen einer bedingten Prozessanweisung

Wenn Sie eine bedingte Prozessanweisung erstellen, kann diese wie folgt einem Container, einem Produktionsauftrag oder einem Produkt zugewiesen werden:

• Einem Container von den folgenden Seiten:
• Containerstart
• Containerwartung
• Starten - zwei Level
• Einem Produktionsauftrag von der Seite "Produktionsauftrag"
• Einem Produkt von der Seite "Produkt"

Bedingte Prozessanweisungen und Eltern-Kind-Beziehungen

Besondere Bedingungen gelten für Eltern- und Kind-Containern, wenn eine bedingte Prozessanweisung zugewiesen wird. Beachten Sie Folgendes:

• Wenn Sie einen Start mit zwei Leveln oder einen anderen Start mit mehreren Leveln ausführen, wird die bedingte Prozessanweisung vom Eltern-Container auf alle Kind-Container angewandt.
• Bei Verbinden- und Abtrennen-Transaktionen behalten die Container ihre individuelle bedingte Prozessanweisung.
• Bei Aufteilen- und Vereinigen-Transaktionen wird dem Aufteilungscontainer dieselbe bedingte Prozessanweisung wie dem Aufteilen-von-Container zugeordnet, und die vereinigten Container behalten die bedingte Prozessanweisung vom Vereinigen-zu-Container.
• Bei Menge vereinigen- und Container vereinigen-Transaktionen (Zuordnung eines Containers zu einem anderen Container) werden die Zuordnungen der bedingten Prozessanweisung zu den Containern nicht geändert.
• Wird eine Transaktion für einen Container ausgeführt und eine bedingte Prozessanweisung einem Container, Produktionsauftrag oder Produkt zugewiesen, werden die Überschreibungswerte angewendet. Angenommen, bei einer Container vereinigen-Transaktion ist Container C1 der Hauptcontainer und Container C2 ist mit Container C1 verknüpft. Beiden ist eine bedingte Prozessanweisung zugeordnet. Wenn Sie eine Transaktion für Container C1 ausführen, wird die bedingte Prozessanweisung angewendet, die zu Container C1 gehört. Wenn die Container aufgeteilt werden und andere Transaktionen für Container C2 ausgeführt werden, wird die bedingte Prozessanweisung angewendet, die Container C2 zugewiesen ist.

Bedingte Prozessanweisungen und automatische Abläufe

Bestimmte Regeln gelten für bedingte Prozessanweisungen, wenn automatische Abläufe ins Spiel kommen. Beachten Sie Folgendes:

• Nur Objekttypen, die von der Spezifikation referenziert werden, werden unterstützt.
• Das Überschreiben von einzelnen Aufgabenelementen (Unterobjekt) und von Aufgabenlisten (revisioniertes Objekt) wird nicht unterstützt.
• Wenn Sie eine Ressource oder Arbeitszelle und/oder einen Arbeitsplatz auf dem Bildschirm für Linienzuordnungen auswählen und eine Überschreibung für einen automatischen Ablauf einer Spezifikation angegeben ist, die momentan ausgeführt wird, aber nicht die Kombination enthält, die momentan auf dem Bildschirm ausgewählt ist, zeigt die Anwendung eine leere Aufgabenliste an. Es werden keine auszuführenden Aufgaben angezeigt.
• Wenn Sie weder einen Arbeitsplatz noch eine Arbeitszelle auswählen und eine Überschreibung für einen automatischen Ablauf definiert ist, der derzeit ausgeführt wird, gibt die Anwendung Ihnen die Möglichkeit, Arbeitsplätze auf Grundlage der Ressourcen auszuwählen, die durch die Überschreibung im automatischen Ablauf zugelassen sind. Sie können die Aufgabenliste oder die Aufgabe anzeigen, die durch die Überschreibung definiert ist.
• Wenn Sie eine Arbeitszelle zum Zeitpunkt der Anmeldung auswählen, zeigt der Aufgabenfilter der Aufgabenliste des automatischen Ablaufs nur dann Aufgabenlisten oder Aufgaben an, wenn die für die Aufgabe definierten Arbeitsplätze Teil der Arbeitszelle sind, die zum Zeitpunkt der Anmeldung ausgewählt wurde.

Automatische Abläufe, bedingte Prozessanweisungen und mehrere Container

Das folgende Szenario beschreibt den Arbeitsplan mehrerer Container für den Fall, dass mehrere bedingte Prozessanweisungen und mehrere Anforderungen für automatische Abläufe im Spiel sind:

1. Führen Sie eine Start-Transaktion für Container (C1) mit der bedingten Prozessanweisung BOP1 und einer Anforderung für automatische Abläufe aus, die optionale Aufgaben enthält.
2. Führen Sie eine Start-Transaktion für einen Multi-Level-Container (C2) mit der bedingten Prozessanweisung BOP2 und einer Anforderung für automatische Abläufe aus, die erforderliche Aufgaben enthält.
3. Vereinigen Sie die Kind-Container von C2 mit dem Container C1. BOP1 wird angewandt.

Melden Sie den neuen Container C1 (der nun die Kind-Container enthält) über die Seite "Operative Ansicht" zum nächsten Schritt an. Der Container wird erfolgreich für den nächsten Schritt angemeldet, obwohl laut automatischen Ablaufanforderungen noch Aufgaben erforderlich waren. In diesem Fall werden die erforderlichen Aufgaben, die mit den Anforderungen des automatischen Ablaufs der bedingten Prozessanweisung des Kind-Containers verknüpft sind, ignoriert, da die bedingte Prozessanweisung des primären Containers angewandt wird.

So wird eine bedingte Prozessanweisung definiert

Gehen Sie folgendermaßen vor, um eine bedingte Prozessanweisung (Bill of Process, BOP) zu definieren:

1. Öffnen Sie die Seite Bedingte Prozessanweisung. Die Seite "Bedingte Prozessanweisung" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen der bedingten Prozessanweisung in das Feld Bedingte Prozessanweisung ein.
4. Geben Sie die Revision dieser bedingten Prozessanweisung in das Feld Revision ein.
5. Aktivieren Sie das Kontrollkästchen Standardrevision, falls dies die Standardrevision ist.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
7. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

So wird eine Überschreibung für bedingte Prozessanweisung hinzugefügt

Gehen Sie folgendermaßen vor, um eine Überschreibung für bedingte Prozessanweisung hinzuzufügen:

1. Führen Sie die Schritte unter "So wird eine bedingte Prozessanweisung definiert" aus.

Oder

Wählen Sie eine vorhandene Instanz einer bedingten Prozessanweisung aus.

2. Erweitern Sie den Abschnitt Überschreiben.
3. Klicken Sie in der Tabelle "Spez.überschreibungen" auf Neue Zeile hinzufügen. Das Popup "Hinzufügen" wird angezeigt.
4. Geben Sie eine Spez. ein oder wählen Sie eine aus.
5. Geben Sie die Felder an, die in der ausgewählten Spezifikation überschrieben werden sollen.
6. Klicken Sie auf OK. Die bedingte Prozessanweisung wird aktualisiert.
7. Wiederholen Sie die Schritte 3 bis 6, um der bedingten Prozessanweisung eine weitere Überschreibung hinzuzufügen.
8. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
