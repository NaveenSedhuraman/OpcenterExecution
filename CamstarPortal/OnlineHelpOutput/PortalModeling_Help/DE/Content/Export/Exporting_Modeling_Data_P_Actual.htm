Exportieren von Modellierungsdaten

Das Portal ermöglicht es Ihnen, Informationen zu Modellierungsobjektinstanzen in eine XML-Datei zu exportieren. Sie können Exportdaten für bestimmte Modellierungsobjektinstanzen oder für alle Instanzen erstellen. Zudem können Sie sämtliche Verweise (Abhängigkeiten) der ausgesuchten Instanzen miteinbeziehen.

Vor dem Exportieren von Modellierungsdaten

Einige Modellierungsobjektinstanzen, die auf der Seite "Instanzauswahl" ausgewählt werden können, dürfen nicht exportiert und anschließend in ein Zielsystem importiert werden. Einige Modellierungsobjekte können Konflikte verursachen, wenn sie exportiert und anschließend in ein Zielsystem importiert werden.

Hinweis: In der Tabelle "Verfügbare Objekte" können nur Modellierungsobjekte ausgewählt werden, für die Instanzen definiert wurden. Die Anwendung zeigt keine Modellierungsobjekte ohne definierte Instanzen an.

Modellierungsobjekte, die nicht exportiert werden dürfen

Siemens empfiehlt, die folgenden Modellierungsobjektinstanzen nicht für den Import in ein Zielsystem zu exportieren.

Hinweis: Durch Hinzufügen von Modellierungsobjekten zur Tabelle "Auszuschließende Modellierungsobjekte" im Modellierungsobjekt "Fabrik" wird gewährleistet, dass diese Instanzen beim Export nicht angezeigt werden und nicht ausgewählt werden können. Weitere Informationen finden Sie unter "Definieren einer Fabrik".

Modellierungsobjekt

Grund für Nichtexport

Aktion

Modellierungsobjektinstanzen vom Typ "Aktion" enthalten Verweise auf Seiten. Seiten können auf verschiedenen Systemen unterschiedliche IDs aufweisen.

Datensammlungsdefinition

Für Datensammlungsdefinitionen sind Designer- und Metadatenaktualisierungen erforderlich.

Hinweis: Dies gilt nicht für Benutzerdatensammlungsdefinitionen.

Fabrik

Eine Modellierungsobjektinstanz vom Typ "Fabrik" kann Zirkelbezüge enthalten.

Produktionskalender

Das Außerkraftsetzen eines vorhandenen Kalenders kann unbeabsichtigte Folgen für die Historie haben. Eine alternative Methode zum Laden eines Kalenders aus Excel finden Sie unter "Definieren von Produktionskalendern".

Portalmenüdefinition

Instanz-IDs stimmen auf verschiedenen Systemen in der Regel nur dann überein, wenn die Datenbank geklont wurde. Möglicherweise gibt es verwaiste Menüeinträge.

Ressource

Eine Modellierungsobjektinstanz vom Typ "Ressource" kann Zirkelbezüge enthalten.

Rolle

Instanz-IDs stimmen auf verschiedenen Systemen in der Regel nur dann überein, wenn die Datenbank geklont wurde.

Modellierungsobjekte, die Konflikte verursachen können

Siemens empfiehlt, die folgenden Modellierungsobjektinstanzen nur wohlüberlegt für den Import in ein Zielsystem zu exportieren.

Modellierungsobjekt

Grund für Nichtexport

Mitarbeiter

Eine Modellierungsobjektinstanz vom Typ "Mitarbeiter" kann Zirkelbezüge enthalten. Instanzen können auch auf Rollen-IDs verweisen, die im Zielsystem nicht vorhanden sind.

Historienansicht

Die Anzeige des Produktionsprotokolls kann sich auf dem Zielsystem unerwartet ändern. Benutzerdefinierte Änderungen auf dem Zielsystem können überschrieben werden.

Anforderung für elektr. Unterschrift modellieren

Eine Instanz "Anforderung für elektr. Unterschrift modellieren" kann Zirkelbezüge enthalten. Die Benutzer, Rollen und elektronischen Unterschriften können auf Quell- und Zielsystem unterschiedlich sein.

Nummerierungsregel

Bei Nummerierungsregel-Instanzen können Konflikte auftreten, wenn die Nummer auf dem Quellsystem nicht mit der Nummer auf dem Zielsystem übereinstimmt.

Organisation

Eine Modellierungsobjektinstanz vom Typ "Organisation" kann Zirkelbezüge enthalten.

Revisionierte Objekte (ROs)

Die Anwendung legt die erste im Import von importierten Objekten enthaltene Revision als Standardrevision fest, wenn die beiden folgenden Bedingungen erfüllt sind:

• Auf dem Zielsystem gibt es keine Standardrevision für ein importiertes RO.
• Der Export enthält keine Standardrevision für das RO.

Geplante Geschäftsregel

Beim Import einer geplanten Geschäftsregel, die aus einem anderen System exportiert wurde, wird das Startdatum auf dem Zielsystem aktualisiert. Darüber hinaus müssen Sie u. U. den Camstar Deployment Server und den Camstar Notification Server neu starten, wenn die Instanz der geplanten Geschäftsregel, die auf dem Zielsystem aktualisiert oder erstellt wird, die Bereitstellung oder Benachrichtigungen beeinflusst.

Veraltete Modellierungsobjekte, die Sie exportieren und importieren können

Bestehende Kunden, die Modellierungsobjektinstanzen verwenden, die in späteren Versionen veraltet sind, können die betreffenden Modellierungsobjektinstanzen weiterhin exportieren und importieren:

• FTP-Transport
• MSMQ-Transport
• Waage
• Waagencode
• Waagengruppe
• Waagenstatusgrund
• Webservice-Transport

Beim Exportieren von Modellierungsdaten
Hinweis: Die Standardeinstellungen für die Microsoft HTTP-Klasse sind unter Umständen für Datei-Importe nicht ausreichend. Für den Import werden maximale Dateigrößen von 6 MB empfohlen. Sie können zwar auch Dateien importieren, die größer als 6 MB sind – die maximale Dateigröße beträgt standardmäßig 30 MB -, doch kann dies die Leistung beeinträchtigen. Informationen zum Importieren von Dateien, die größer als 6 MB sind, finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration und in der Opcenter Execution Core-Anleitung zur Systemadministration.
Hinweis: Überprüfen Sie die Einstellung des Anwendungspools zur Auslastung des privaten Speichers für den CamsarAppServer-Anwendungspool. Möglicherweise müssen Sie den Wert erhöhen, wenn Sie 1000 oder mehr Instanzen (einschließlich Referenzen) exportieren. Weitere Informationen zum Konfigurieren der maximalen Speichergrenze für den Anwendungspool finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Sie können sowohl Instanzen von Named Data Objects (NDO) als auch Instanzen revisionierter Objekte (RO) exportieren.

Hinweis: Einstellungen in der Designer-Anwendung bestimmen, ob Modellierungsobjektinstanzen verfügbar sind, die exportiert werden können. Nicht sichtbare, abstrakte Modellierungsobjekte sind nicht verfügbar. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch.
Hinweis: Die Anwendung importiert keine Filter-Tag-Zuordnungen. Wenn Sie z. B. eine Modellierungsobjektinstanz für ein Produkt exportieren, die eine Filter-Tag-Zuordnung enthält, ist die Filter-Tag-Zuordnung beim Import der Instanz nicht enthalten. Sie können Filter-Tag-Instanzen exportieren und importieren.

Die Beschriftungen auf den Exportseiten können Sie mit Portal Studio anpassen. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch oder im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Wenn Sie auf die Schaltfläche "Export starten" auf der Seite "Ausführungskonsole" klicken (die letzte Seite der Seitenfolge), wird der Export gestartet. Durch Klicken auf die Schaltfläche "Beenden" auf der Seite "Ausführungskonsole" wird die Export/Import-Seitenfolge geschlossen und die Seite "Ausführungsübersicht" wird angezeigt.

Die Exporte werden ausgeführt, bis der Exportprozess abgeschlossen ist. Sie können die Portal-Einstellungen in Portal Studio ändern und ein Timeout aktivieren, das den Exportprozess nach einer festgelegten Zeitspanne abbricht. Informationen zum Einrichten eines Timeouts finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch und im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Wenn Sie auf die Schaltfläche "Exportdatei herunterladen" klicken, wird ein Popup angezeigt, in dem Sie die Exportdatei anzeigen oder speichern können. Wenn Sie auf die Schaltfläche "Beenden" klicken, wird die Seite "Ausführungsübersicht" angezeigt. Hier können Sie Übersichts- und Fehlerinformationen zum Exportprozess anzeigen und drucken. Die Anwendung speichert die Exportdatei nur, wenn Sie auf "Speichern" klicken. Die Anwendung deaktiviert die Schaltfläche "Exportdatei herunterladen", nachdem Sie die Exportdatei geöffnet oder gespeichert oder die Funktionen zum Öffnen oder Speichern abgebrochen haben.

Seitenfolge für den manuellen Export von Modellierungsdaten

Die Seitenfolge für den manuellen Export von Modellierungsdaten besteht aus fünf Seiten:

• Datentransfertyp – Auswahl des manuellen Exports
• Instanzauswahl – Auswahl der Modellierungsobjektinstanzen, die Sie exportieren möchten
• Referenzen hinzufügen – Möglichkeit, alle Modellierungsobjektinstanzen, auf die in den bereits ausgewählten Modellierungsobjektinstanzen verwiesen wurde, hinzuzufügen
• Ausführungskonsole – Starten des Exports und ggf. Herunterladen der XML-Datei
• Ausführungsübersicht – Anzeigen von Übersichts- und Fehlerinformationen zu dem Export

Vorgehensweise zum manuellen Exportieren von Modellierungsdaten

Gehen Sie folgendermaßen vor, um Modellierungsdaten manuell zu exportieren:

1. Öffnen Sie die Seite Export/Import über das Menü Export/Import. Die Seite "Export/Import - Datentransfertyp" (Export/Import Data Transfer Type) wird angezeigt.
2. Füllen Sie die erforderlichen Felder der Seitenfolge wie in den folgenden Abschnitten beschrieben aus:
• "Vorgehensweise zum Auswählen des Datentransfertyps"
• "Vorgehensweise zum Auswählen der Modellierungsobjekt-Instanzen für den Export"
• "Vorgehensweise zum Hinzufügen von referenzierten Modellierungsobjekt-Instanzen"
• "Vorgehensweise zum Starten des Exports"
3. Klicken Sie auf Beenden, um den Exportprozess zu beenden. Die Anwendung schließt die Seite "Export/Import" und zeigt die Seite "Ausführungsübersicht" mit einer Übersicht über den Exportprozess und die aufgetretenen Fehler an.
