Importieren von Modellierungsdaten

Das Portal in Opcenter EX MDD und Opcenter EX CR bietet die Möglichkeit, exportierte Modellierungsobjektinstanzen von einer Installation in eine andere hochzuladen und zu importieren. Sie können einen teilweisen Import durchführen, indem Sie ausgewählte Instanzen festlegen, die Sie nicht importieren möchten, statt die gesamte Exportdatei zu importieren.

Hinweis: Die Standardeinstellungen für die Microsoft HTTP-Klasse sind unter Umständen für Datei-Importe nicht ausreichend. Für den Import werden maximale Dateigrößen von 6 MB empfohlen. Sie können zwar auch Dateien importieren, die größer als 6 MB sind, doch kann dies die Leistung beeinträchtigen. Informationen zum Importieren von Dateien, die größer als 6 MB sind, finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration und in der Opcenter Execution Core-Anleitung zur Systemadministration.
Hinweis: Überprüfen Sie die Einstellung des Anwendungspools zur Auslastung des privaten Speichers für den CamsarAppServer-Anwendungspool. Möglicherweise müssen Sie den Wert erhöhen, wenn Sie 1000 oder mehr Instanzen (einschließlich Referenzen) importieren. Weitere Informationen zum Konfigurieren der maximalen Speichergrenze für den Anwendungspool finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Beim Importieren von Modellierungsdaten

Die Anwendung importiert Modellierungsobjektinstanzen nach Namen. Beim Importprozess können vorhandene Instanzen der Anwendung überschrieben werden, wenn die Importdatei Instanzen enthält, die den gleichen Namen wie die vorhandenen Instanzen haben. Bevor Sie den Import starten, können Sie einen Überschreibmodus aktivieren, der verhindert, dass Instanzen überschrieben werden.

Die Importe werden ausgeführt, bis der Importprozess abgeschlossen ist. Sie können die Portal-Einstellungen in Portal Studio ändern und ein Timeout aktivieren, das den Importprozess abbricht, wenn ein bestimmter Zeitraum abgelaufen ist. Informationen zum Einrichten eines Timeouts finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch und im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Die Einstellungen für den Portal-Überschreibmodus auf der Seite "Ausführungskonsole" ermöglichen es Ihnen anzugeben, ob und wie Modellierungsobjektinstanzen in der Anwendung von Modellierungsobjektinstanzen in der importierten Dateien überschrieben werden. Für den Importprozess stehen drei Überschreibungsoptionen zur Wahl:

• Abbrechen – Der Importprozess wird abgebrochen, wenn während des Imports in der Anwendung eine Modellierungsobjektinstanz entdeckt wird, die denselben Namen wie eine Modellierungsobjektinstanz in der Importdatei aufweist.
• Überschreiben – Der Importprozess importiert alle Modellierungsobjektinstanzen aus der Importdatei und überschreibt alle Instanzen in der Anwendung, die denselben Namen haben.
• Überspringen – Der Importprozess importiert die Modellierungsobjektinstanzen aus der Eingabedatei, überspringt jedoch Instanzen, die denselben Namen wie eine vorhandene Instanz in der Anwendung tragen.
Hinweis: Die Anwendung verwendet standardmäßig die Methode "Überspringen", wenn Sie keine Überschreibungsmethode auswählen.

Die Anwendung zeigt alle Modellierungsobjekt-Instanzen an, die aus der angegebenen XML-Datei in die Tabelle "Objekttypname" auf der Seite "Instanzauswahl" importiert werden können. Sie können die Instanzen löschen, die Sie nicht importieren möchten. Wählen Sie die Instanzen in der Tabelle "Objekttypname" aus und klicken Sie auf "Ausgewählte Zeile löschen".

Wenn Sie Modellierungsobjekt-Definitionen importieren, ohne vorher die Modellierungsobjekt-Definitionen zu importieren, von denen sie zum Einhängen, kommt es zu einem Fehler. Ein Objekt kann z. B. von einem anderen zum Einhängen, wenn es eine Instanz definiert, die ein erforderliches Feld auf der Seite des anderen Objekts ausfüllt. Sie müssen zuerst die Modellierungsobjekte importieren, von denen die anderen Objekte abhängig sind, und danach die abhängigen Objekte separat importieren.

Wenn Sie beispielsweise Instanzen von Genehmigungsvorlage und Genehmigungsentscheidungslisten importieren, sollten zuerst die Instanzen der Genehmigungsentscheidungslisten importiert werden, da diese von den Genehmigungsvorlage beim Import benötigt werden. Die Instanzen der Genehmigungsvorlage müssen anschließend in einem separaten Prozess importiert werden.

Hinweis: Die Anwendung importiert keine Filter-Tag-Zuordnungen. Wenn Sie z. B. eine Modellierungsobjektinstanz für ein Produkt exportieren, die eine Filter-Tag-Zuordnung enthält, ist die Filter-Tag-Zuordnung beim Import der Instanz nicht enthalten. Sie können Filter-Tag-Instanzen exportieren und importieren.

Sie können die Beschriftungen, die auf den Importseiten angezeigt werden, mit Portal Studio anpassen. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch oder im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Fehler beim Import

Beendigungsnachrichten zeigen Fehler beim Import von Instanzen an. Der Import einer Instanz funktioniert nicht, wenn die Instanz in der Anwendung bereits vorhanden ist und Sie keine Überschreibung beim Import zulassen.

Seitenfolge für manuelles Importieren von Modellierungsdaten

Die Seitenfolge für den manuellen Import von Modellierungsdaten besteht aus 4 Seiten:

• Datentransfertyp – Auswahl des manuellen Imports
• Instanzauswahl – Auswahl der zu importierenden XML-Datei und der zu importierenden Modellierungsobjektinstanzen
• Ausführungskonsole – ggf. Festlegen des Überschreibmodus und Starten des Imports
• Ausführungsübersicht – Anzeigen von Übersichts- und Fehlerinformationen zu dem Import

Vorgehensweise zum manuellen Importieren von Modellierungsdaten

Gehen Sie folgendermaßen vor, um Modellierungsdaten manuell zu importieren:

1. Öffnen Sie die Seite Export/Import über das Menü Export/Import. Die Seite "Export/Import - Datentransfertyp" (Export/Import Data Transfer Type) wird angezeigt.
2. Füllen Sie die erforderlichen Felder der Seitenfolge wie in den folgenden Abschnitten beschrieben aus:
• "Vorgehensweise zum Auswählen des Datentransfertyps"
• "Vorgehensweise zum Auswählen der Modellierungsobjekt-Instanzen für den Import"
• "Vorgehensweise zum Starten des Imports"
3. Klicken Sie auf Beenden, um den Importprozess zu beenden. Die Anwendung schließt die Seite "Export/Import" und zeigt die Seite "Ausführungsübersicht" (Execution Summary) mit einer Übersicht über den Importprozess und die aufgetretenen Fehler an.
