Definieren von Materialwarteschlangen

Das Modellierungsobjekt "Materialwarteschlange" bietet die Möglichkeit, einer Ressource Container, Lose oder Produkte zuzuweisen, die während der Containerverarbeitung automatisch verbraucht werden. Das Material in der Warteschlange wird basierend auf den Materialanforderungen verbraucht, die für die Spezifikation oder für den Routenschritt des Containers definiert sind, wenn die Materialwarteschlange aktiv ist.

Beim Definieren von Materialwarteschlangen

Sie können einer Ressource mehrere Materialwarteschlangen für die Materialbereitstellung zuordnen. Für den Materialverbrauch kann aber jeweils nur eine Materialwarteschlange aktiv sein. Die Anwendung zeigt die Liste der verfügbaren Container oder Lose, die zugewiesen werden können, in der Tabelle "Materialwarteschlangendetails" an. Informationen zum Laden und Entladen von Containern, Losen und Produkten in eine bzw. aus einer Materialwarteschlange finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Wichtig: Sie können eine Instanz einer Materialwarteschlange nur kopieren, wenn die Tabelle "Materialwarteschlangendetails" leer ist.

Vorgehensweise zum Definieren einer Materialwarteschlange

Gehen Sie folgendermaßen vor, um eine Materialwarteschlange zu definieren:

1. Öffnen Sie die Seite Materialwarteschlange. Die Seite "Materialwarteschlange" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für die Materialwarteschlange in das Feld Materialwarteschlange ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
