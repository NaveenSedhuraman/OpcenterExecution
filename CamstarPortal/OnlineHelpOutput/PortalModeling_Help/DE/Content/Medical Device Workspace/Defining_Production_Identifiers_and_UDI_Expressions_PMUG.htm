Definieren von Produktionskennungen und UDI-Ausdrücken

Sie können sowohl die Produktionskennung (PI) als auch die eindeutige Gerätekennung (UDI) direkt für einen Container definieren, indem Sie die Informationen manuell eingeben, wenn Sie den Container auf der Startseite oder auf der Seite "Auftragserledigung" starten. Auf der Seite "Containerwartung" können Sie diese Felder nach dem Start des Containers aktualisieren. Informationen zum direkten Zuweisen von Produktionskennungen und der UDI zu Containern finden Sie im Opcenter Execution Medical Device and Diagnostics Shop Floor-Benutzerhandbuch und im Opcenter Execution Core Shop Floor-Benutzerhandbuch.

Ausdrücke für die UDI und die PI können mit dem Objekt für den UDI-Ausdruck erstellt werden. Diese Ausdrücke werden verwendet, um die UDI und die PI zur Laufzeit automatisch zu erstellen und dem Container zuzuweisen, sofern dies für das Produkt konfiguriert ist. In diesen Ausdrücke können alle Werte verwendet werden, die von den Transaktionen "Abmelden" und "Anmelden" referenziert werden können. Sie müssen für jeden PI- und UDI-Ausdruck, den Sie in Ihrer Implementierung verwenden, einen anderen Ausdruck erstellen.

Verwenden Sie z. B. diesen Variablenausdruck im Feld "Variablenausdruck", um eine UDI zu erstellen und zuzuweisen, die die Gerätekennung, das Produktionsdatum und den Namen des Containers enthält:

 

((Container.DeviceIdentifier!=Null? "(01)" + Container.DeviceIdentifier.Name:Null) + (DateConvert

(Container.ManufacturingDate,"yyMMdd")!=Null? "(11)" + DateConvert

(Container.ManufacturingDate,"yyMMdd"):Null) + (Container.Name!=Null? "(21)" + Container.Name:Null))

Felddefinitionen der Seite "UDI-Ausdruck"

Diese Tabelle definiert die Felder, die nur auf der Seite "UDI-Ausdruck" vorhanden sind.

Siehe auch "Gemeinsame Felder auf Modellierungsseiten". Dieser Abschnitt enthält Informationen zu den Feldern, die alle Modellierungsobjekte aufweisen.

Feld

Definition

Typ

Allgemein

Variablenausdruck

Die auszuwertende Zeichenkette, die den Wert zur Laufzeit erzeugt.

Optional

Verwendung

Definiert den Typ der Variablen. Mögliche Optionen sind:

• Alarmmeldung
• Allgemein (in jedem Text verwendbar)
• Anweisungstext
• Umlaufbestandsnachricht

Hinweis: UDI-Ausdrücke sind immer allgemeine Variablen.

Optional

Wert für Auflösungsfehler

Ein Wert, der verwendet wird, wenn der Ausdruck nicht erfolgreich ausgewertet werden kann. Dies kann eine leere Zeichenfolge sein (damit ignoriert wird, dass der Ausdruck nicht ausgewertet werden kann) oder ein anderer Wert (z. B. "FEHLER!"). um deutlich zu machen, dass der Text der erstellten Nachricht nicht vollständig oder nicht exakt ist.

Optional

Vorgehensweise zum Definieren eines UDI-Ausdrucks

Gehen Sie folgendermaßen vor, um einen UDI-Ausdruck zu definieren:

1. Öffnen Sie die Seite UDI-Ausdruck. Die Seite "UDI-Ausdruck" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie einen Namen für den UDI-Ausdruck in das Feld UDI-Ausdruck ein.
4. Geben Sie einen Variablenausdruck in das Feld Variablenausdruck ein.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
6. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.
