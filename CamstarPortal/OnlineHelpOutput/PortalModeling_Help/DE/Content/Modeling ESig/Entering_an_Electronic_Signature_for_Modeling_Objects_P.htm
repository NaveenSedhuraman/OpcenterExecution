Eingeben einer elektronischen Unterschrift für Modellierungsobjekte

Sie werden in einem Popup aufgefordert, die entsprechenden Unterschriften bereitzustellen, falls elektronische Unterschriften erforderlich sind, wenn Sie ein Modellierungsobjekt erstellen, ändern oder löschen. Das Modellierungsobjekt kann erst geändert werden, wenn diese Unterschriften geleistet und authentifiziert wurden.

Anforderungen zu Unterschriften können für einzelne oder mehrere Unterschriften verschiedener Autoritätsebenen festgelegt werden. Die Autoritätsebene und die Anzahl der erforderlichen Unterschriften hängen davon ab, wie die erforderlichen Unterschriften definiert wurden.

Beim Eingeben einer elektronischen Unterschrift

Das Feld "Mitunterzeichnungsgrund" wird nur dann ausgefüllt, wenn eine Mitunterschrift eingegeben wurde.

Wenn Sie auf die Schaltfläche "Unterschrift hinzufügen" klicken, bestätigen Sie, dass die eingegebenen Unterzeichner- und Mitunterzeichnerinformationen die Anforderungen der Transaktion zur elektronischen Unterschrift erfüllen. Die Anwendung stellt sicher, dass folgende Bedingungen erfüllt sind:

• Der Unterzeichner oder Mitunterzeichner ist ein berechtigter Mitarbeiter und hat ein gültiges Kennwort eingegeben.
• Die Rolle oder Mitunterzeichnerrolle, die in der Anforderung zur elektronischen Unterschrift definiert ist, ist gültig. Daraus folgt, dass die Rolle:
• Dem Unterzeichner oder Mitunterzeichner zugewiesen ist
• Teil der Rollengruppe für elektronische Unterschriften ist, die dem Unterzeichner oder Mitunterzeichner zugewiesen ist

Wenn die Unterschriften gültig sind, fügt die Anwendung die Informationen der ausgewählten Zeile der Kind-Tabelle hinzu, wählt die nächste Zeile automatisch aus und bewegt den Cursor in das Unterzeichnerfeld für die nächste Unterschrift.

Wenn alle erforderlichen Unterschriften für die ausgewählte Unterschriftsanforderung erfasst wurden, reagiert die Anwendung folgendermaßen:

• Sie färbt den Hintergrund der erfüllten Anforderung grün
• Sie wählt, falls zutreffend, die nächste Anforderung in der Tabelle aus
Wichtig: Wenn Sie eine Unterschrift ändern müssen, müssen Sie die Unterschrift aus der Anforderung entfernen und wieder mit den neuen Informationen hinzufügen. Durch Klicken auf die Schaltfläche "Zurücksetzen" werden alle Unterschriften aus der Tabelle entfernt. Alle Unterschriften für die einzelnen Unterschriftsanforderungen wurden hinzugefügt.

Ungültiger Informationseintrag

Wenn Sie eine falsche ID oder ein falsches Kennwort eingegeben haben, zeigt die Anwendung eine Fehlermeldung im Popup "Elektr. Unterschrift" an. Sie können die Informationen entweder korrigieren und die Unterschriften hinzufügen oder das Popup "Elektr. Unterschrift" schließen. Wenn Sie das Popup-Fenster schließen, ohne Änderungen vorzunehmen, wird die Transaktion nicht ausgeführt.

Beim Löschen von Unterschriften

Die Anwendung bietet zwei Schaltflächen, um Unterschriften aus der Tabelle "Eintrag der elektronischen Unterschrift" zu löschen: "Ausgewählte Unterschriften löschen" und "Zurücksetzen". Durch Klicken auf die Schaltfläche

• "Ausgewählte Unterschriften löschen" werden alle ausgewählten Unterschriften aus der momentan ausgewählten Unterschriftsanforderung entfernt.
• Durch Klicken auf die Schaltfläche "Zurücksetzen" werden alle Unterschriften aus der Tabelle entfernt.

Beim Abschicken elektronischer Unterschriften

Wenn Sie eine erfasste Unterschrift abschicken, führt die Anwendung eine letzte Überprüfung durch, um sicherzugehen, dass:

• Alle Unterschriften sämtlicher Unterschriftsanforderung hinzugefügt wurden.
• Dieselbe Person nicht mehrmals unterzeichnet hat. (Allerdings können Mitunterzeichner mehrere Unterschriften leisten, sofern ihnen die entsprechenden Rollen zugewiesen sind.)

Das Popup "Elektr. Unterschrift" wird geschlossen und die Transaktionsverarbeitung wird fortgesetzt, wenn die Anforderungen erfüllt wurden. Andernfalls bleibt das Popup geöffnet und eine Fehlermeldung wird angezeigt.

Popup "Elektr. Unterschrift"

Sie können Unterschriftsautorisierungen für den aktuellen Service mithilfe des Popups "Elektr. Unterschrift" eingeben, anzeigen und bearbeiten. Die Tabelle "Eintrag der elektronischen Unterschrift" enthält die Anforderungen zur elektronischen Unterschrift, die für den aktuellen Service definiert wurden.

In jeder Zeile der Unterschriftsanforderung werden folgende Informationen angezeigt:

• Bedeutung: Name oder Beschreibung, die es Ihnen erlaubt, den Grund für die Unterschrift festzustellen.
• Anzahl: Anzahl der gemäß der Bedeutung erforderlichen Unterschriften. Dieser Wert bestimmt die Anzahl der Kind-Zeilen, die für die Bedeutung angezeigt werden.
• Rolle: Benutzerrolle, die für die Unterzeichnung der Bedeutung erforderlich ist.
• Mitunterzeichnerrolle: Benutzerrolle, die eine zusätzliche Unterschrift leisten kann, sollte der Hauptunterzeichner nicht verfügbar sein oder nicht über die erforderlichen Berechtigungen verfügen, um allein zu unterzeichnen. Die Mitunterzeichnerrollen-Information wird nur angezeigt, wenn die Mitunterzeichnerrolle mit den Unterschrifts-Anforderungen im Modellierungsobjekt mit der Anforderung zur elektronischen Unterschrift verknüpft wurde.

Hinweis: Die Bedeutung und die Rolle der elektronischen Unterschrift werden in der Modellierung definiert und dem Modellierungsobjekt "Elektr. Unterschrift: Anforderung" zugewiesen. Der "Zähler" wird im Modellierungsobjekt "Elektr. Unterschrift: Anforderung" festgelegt.

Vorgehensweise zum Eingeben elektronischer Unterschriften für Modellierungsobjekte

Gehen Sie folgendermaßen vor, um eine elektronische Unterschrift für ein Modellierungsobjekt einzugeben:

1. Öffnen Sie das Modellierungsobjekt und die Instanz, die Sie aktualisieren möchten. Die Instanz und die Instanzinformationen werden angezeigt.
2. Erstellen Sie eine neue Instanz des Objekts, ändern Sie eine vorhandene Instanz oder löschen Sie eine vorhandene Instanz, für die eine Anforderung zur elektronischen Unterschrift vorliegt.
3. Klicken Sie auf Speichern. Das Popup "Elektr. Unterschrift" wird angezeigt. Die Anwendung wählt standardmäßig die erste (Kind-)Zeile der ersten nicht erfüllten Unterschriftsanforderung aus.
4. Wählen Sie ggf. eine andere Unterschriftsanforderung aus.
5. Geben Sie den Unterzeichner und das Unterzeichnerpasswort ein.
6. Geben Sie den Mitunterzeichner, das Mitunterzeichnerpasswort und den Mitunterzeichnungsgrund ein, wenn ein Mitunterzeichner zugewiesen wurde.
7. Geben Sie bei Bedarf Kommentare in das Feld Unterzeichner/Mitunterzeichner-Kommentare ein.
8. Klicken Sie auf Unterschrift hinzufügen. Die Anwendung überprüft die Unterschrifteninformationen und fügt sie der Anforderung hinzu, die in der Tabelle "Eintrag der elektronischen Unterschrift" ausgewählt wurde. Wenn zusätzliche Unterschriften benötigt werden, fügt die Anwendung die nächste Kind-Zeile automatisch hinzu und bewegt den Cursor in das Unterzeichnerfeld der nächsten Unterschrift.
9. Wiederholen Sie die Schritte 4 bis 8, um weitere Unterschriften für die ausgewählte Anforderung hinzuzufügen. Wenn alle Unterschriften für eine Anforderung erfasst wurden, wird der Hintergrund der Elternzeile grün gefärbt und eine leere Unterschriftenzeile in der letzten nicht erfüllten Unterschriftenanforderung ausgewählt.
10. Klicken Sie auf Unterschriften abschicken. Die Anwendung schließt das Popup "Elektr. Unterschrift" und zeigt eine Statusnachricht mit der Information an, dass das Modellierungsobjekt gespeichert wurde.
