Definieren von Rollen für elektronische Unterschriften

Eine Rolle (die in Opcenter EX MDD oder Opcenter EX CR erstellt wurde) definiert eine Tätigkeit, der bestimmte Berechtigungen zugewiesen sind, die Zugang zur gesamten oder Teilen der Anwendung gewährt. Sie müssen eine Rolle für jede erforderliche elektronische Unterschrift erstellen. Jedem autorisierten Mitarbeiter ist ein Satz von Rollen zugewiesen. Damit eine Unterschrift akzeptiert wird, muss dem Mitarbeiter für jede benötigte elektronische Unterschrift eine Rolle zugewiesen worden sein.

Sie können auch mehrere Unterschriften für eine einzelne Rolle fordern. Z. B. können Sie vier Unterschriften für die Herstellung eines Kartons fordern: jeweils eine für Herstellung, Füllen, Verschließen und Überprüfen des Kartons.

Ein Vorarbeiter ist eventuell verantwortlich dafür, die Kartons zu überprüfen, nachdem diese gepackt wurden. Um die Unterschrift als Karton-Überprüfer leisten zu können, muss dem Vorabeiter diese Rolle zugewiesen sein.

Vorgehensweise zum Definieren einer Rolle für eine elektronische Unterschrift

Gehen Sie folgendermaßen vor, um eine Rolle für eine elektronische Unterschrift zu definieren:

1. Öffnen Sie die Seite Rolle. Die Seite "Rolle" wird innerhalb der Seite "Modellierung" angezeigt.
2. Klicken Sie auf Neu. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Geben Sie den Namen der Rolle in das Feld Rollenname ein.
4. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
5. Klicken Sie auf Speichern. Die Anwendung speichert das Modellierungsobjekt und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Hinzufügen von Berechtigungen zu einer Rolle für eine elektronische Unterschrift

Gehen Sie folgendermaßen vor, um der Rolle für eine elektronische Unterschrift Berechtigungen hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Rolle für eine elektronische Unterschrift" aus.

Oder

Wählen Sie eine vorhandene Rolleninstanz aus.

2. Wählen Sie eine Berechtigungskategorie aus der Liste Berechtigungstypfilter als Filter aus. Die mit dem Filter verknüpften Berechtigungen werden in der Tabelle "Verfügbare Berechtigungen" angezeigt.
3. Wählen Sie jede Berechtigung (Transaktion) aus, für die eine elektronische Unterschrift benötigt wird, und klicken Sie auf Hinzufügen. Die ausgewählten Berechtigungen werden in die Tabelle "Zugewiesene Berechtigungen" verschoben.
4. Deaktivieren Sie ggf. die zugehörigen Zugriffsmodus-Kontrollkästchen, wenn der Benutzer nicht über alle angegebenen Zugriffsberechtigungen verfügen soll.
Hinweis: Wenn Sie einen anderen Berechtigungsfilter auswählen, werden die zuvor ausgewählten Berechtigungen nicht mehr in der Tabelle "Zugewiesene Berechtigungen" angezeigt. Wenn Sie die Berechtigungszuweisungen abgeschlossen haben und auf "Speichern" klicken, werden alle Berechtigungen, die Sie zugewiesen haben, gespeichert.
5. Wiederholen Sie die Schritte 2 bis 4, um dieser Rolle weitere Berechtigungen hinzuzufügen.
6. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.

Vorgehensweise zum Hinzufügen von Mitgliedschaften zu einer Rolle für eine elektronische Unterschrift

Gehen Sie folgendermaßen vor, um einer Rolle für eine elektronische Unterschrift Mitgliedschaften hinzuzufügen:

1. Führen Sie die Schritte unter "Vorgehensweise zum Definieren einer Rolle für eine elektronische Unterschrift" aus.

Oder

Wählen Sie eine vorhandene Rolleninstanz aus.

2. Klicken Sie in der Tabelle "Mitgliedschaft" auf Neue Zeile hinzufügen. Es werden leere Felder zum Definieren einer neuen Instanz angezeigt.
3. Wählen Sie einen Mitarbeiter in der Liste aus.
4. Wählen Sie aus der Liste Organisation die Organisation aus, für die die Rolle des Mitarbeiters gelten soll.
5. Aktivieren Sie das Kontrollkästchen Weitergeben, wenn die Rolle auch für alle Unterorganisationen dieser Organisation gültig sein soll.
6. Wiederholen Sie die Schritte 2 bis 5, um einem anderen Mitarbeiter diese Rolle zuweisen.
7. Klicken Sie auf Speichern. Die Anwendung zeigt eine Erfolgsmeldung an, die angibt, dass das Modellierungsobjekt aktualisiert wurde.
