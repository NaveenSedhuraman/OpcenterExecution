How Affected Containers Are Added to Events

The application adds affected containers to events when you execute any of the following transactions if your system administrator has configured them in Modeling:

• Associate
• Combine Container
• Combine Qty
• Component Issue
• Component Remove
• Split Container
• Split Qty

When Executing the Associate Transaction

The following occurs when you execute the Associate transaction:

• The parent container (selected in the Container list on the Associate page), existing child containers, and selected eligible containers are added to each active event that each selected eligible container has already been added to.
• Each selected eligible container is added to any active event of the parent container.

When Executing the Combine Container Transaction

The following occurs when you execute the Combine Container transaction:

• The parent container (selected in the Container list on the Combine Container page), existing child containers, and new child containers are added to each active event that each new child container has already been added to.
• Each new child container is added to any active event of the parent container.

When Executing the Combine Qty Transaction

The following occurs when you execute the Combine Qty transaction:

• The target (To) container (selected in the Container list on the Combine Qty page) is added to each active event that each source (From) container has already been added to.

When Executing the Component Issue Transaction

The following occurs when you execute the Component Issue transaction:

• The target (To) container (selected in the Container list on the Component Issue page) is added to each active event that the component container has already been added to.
Note: The application does not add containers to events until you click the Issue Components button on the Component Issue - Advanced page. The application queues all components to be issued at once if you click the Add to Pending button.

When Executing the Component Remove Transaction

The following occurs when you execute the Component Remove transaction:

• The component container to which the components are removed is added to each active event of the source (From) container (selected in the Container list on the Component Remove page).

When Executing the Split Container Transaction

The following occurs when you execute the Split Container transaction:

• The target (To) container and all sibling containers are added to each active event that each child container has already been added to.
