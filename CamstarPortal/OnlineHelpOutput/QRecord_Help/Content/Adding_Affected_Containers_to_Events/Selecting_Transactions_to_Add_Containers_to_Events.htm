Selecting Transactions to Add Containers to Events

A system administrator can specify whether the application adds a container to an event automatically when you execute a transaction on a related container that has already been added to the event. The transactions are specified in Modeling using the Organization modeling object.

These are the default available transactions:

• Associate
• Combine
• Component Issue
• Component Remove
• Split

The application adds the affected containers to a quality event when you execute the Combine Container or Combine Qty transactions if Combine is set on the Organization modeling object. Similarly, Split applies to the Split Container and Split Qty transactions. Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information.

Disallowed transactions take precedence over the transactions specified to automatically add containers to events. For example, the transaction fails and the application generates an error message if Associate is specified as a disallowed transaction on an event and as a transaction that automatically adds containers to an event.

Note: You cannot modify quality events that have a status of In Review. Therefore, you cannot add containers to an event that is In Review. The transaction fails and the application displays an error message.
