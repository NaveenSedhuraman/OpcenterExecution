Recording an Event

When you encounter a situation where the quality and usability of items, services, activities, or processes does not meet specifications, you should record the event that caused the situation.

This list provides a sample of data that can be captured:

• Categorization and priority
• Date (occurrence and report date)
• Discovery area
• Product information (part number, revision)
• Operation and resource
• Reporter information
• Failure information
• Affected materials, applicable quantity, applicable component, and subassemblies

Before Recording an Event

A system administrator must define these modeling objects before recording an event. These objects are required to complete the event recording process:

• Classification
• Subclassification
• Failure Type
• Failure Severity
• Failure Mode
• Failure Mode Group
• Numbering Rule
• Organization (specifically the Event Classification Spec Map, Category Processing Map, and Numbering Rule Map)
• Priority Level
• Enterprise, Factory, or Work Center
• Employee (initiator)

Additionally, you may need one or more of these optional modeling objects when recording an event:

• Operation
• Product
• Resource

Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information on defining modeling objects.

Set Up Electronic Signatures

If you want to collect an electronic signature when a user creates an event, you must add the Create Event quality service to the Electronic Signature Txn Map on the Organization modeling object. Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information on the Organization modeling object for information on setting up electronic signatures and adding quality services to the organization.

When Recording an Event

The application creates a history for a generic event when you save or submit the event. If you save an event during the recording process and then close or cancel the pageflow, you can access the event record from the Concierge to edit it or delete it.

Important: If you save an event and the organization, classification, or subclassification name used in the record changes before you submit the event, you must delete the record and then re-enter the event.

Clicking the Upload button when performing a document attachment is equivalent to performing a save. The application saves the record automatically when you attach a file.

Clicking the Cancel button before clicking the Save button when recording a new generic event or editing an existing record closes the Record Generic Event page without saving any information entered.

Clicking the Delete button deletes a saved record. The Delete button appears on all pages in the generic event recording pageflow once the event is saved. A warning message appears when you click the Delete button. Clicking Yes on the warning message deletes the record and closes the page. The application will not display deleted records in the Concierge. Refer to "Deleting a Saved Event" for information.

Clicking the Submit button saves and submits the record. The application then displays the record’s management page.

How to Record an Event

Follow these steps to record an event:

1. Select Record Generic Event from the Events menu. The Record Generic Event: Classification page appears.
2. Complete the required fields on the first two pages of the generic event recording pageflow using these procedures:
• "How to Enter Classification and Description Information"
• "How to Enter Failure Information"
3. Optionally, complete the last two pages of the pageflow using these procedures:
• "How to Add a Container to the Event"
• "How to Attach a File"
4. Click Submit on the Attachments page. The application creates the quality record and displays the record’s management page.
