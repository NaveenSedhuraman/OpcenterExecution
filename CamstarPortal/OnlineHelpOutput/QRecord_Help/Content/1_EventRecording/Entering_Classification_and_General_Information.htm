Entering Classification and General Information

The Classification page is the first page in the generic event recording pageflow. The page allows you to specify the organization for which you are recording the event and a classification and subclassification within that organization to help classify the event. Additionally, you can record general information such as:

• Brief and long descriptions of the event
• Priority level
• Date and time the event occurred
• Discovery area

The organization defaults to your primary organization (as the logged-in employee) and only those organizations in which you have a direct role or a role through propagation will be available for selection.

When Entering Classification and General Information

Classification and subclassification combinations are defined in the Event Classification Spec Maps grid in the Organization modeling object. The application defaults to a classification and subclassification when only one combination has been defined for the organization.

You can change the Classification and Subclassification before saving the event, but you will lose other data entered on this page.

Priority levels used when recording an event are created in Modeling.

The Discovery Area field provides a list of areas from which to choose such as Enterprise, Factory, and Work Center that were previously defined in Modeling. This list is provided in a hierarchical view. You can also enter a new area in this field.

The Initiator Name and Organization fields default to the information for the user logged into the application. Only those organizations in which you have a direct role or a role through propagation are available for selection.

You can save your information at this point and return later to collect more information, or you can click Next to advance to the next page in the pageflow. However, once you save the event you cannot change the Classification and Subclassification.

Text Editor

Siemens provides a text editor that enables you to format descriptions when you are recording an event. The text editor supports familiar text formatting features such as font size and color, font style, text alignment, bulleted and numbered lists, highlighting, and inserting images. Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information on inserting images using the text editor.

Siemens recommends highlighting the existing description and then applying formatting to the selected text. If you do not select any formatting attributes, the application will apply the default font size, style, and color. The application displays the applied formatting when you view the record.

You can use the text editor in either Design or HTML mode. When you copy text in Design mode, you are not copying the hidden tags. Therefore, the formatting may not be retained when you paste the text that was copied. You can copy the hidden tags, however; if you switch to HTML view.

There is a limit of 4,000 characters for the Instruction field, including embedded HTML code. When you copy text from another application (for example, Microsoft Word) and paste it into this field, hidden HTML code is pasted into the field in addition to the actual text. Therefore, you may exceed the limit even though the text contains less than 4,000 characters.

Note: Google Chrome does not support the text editor's Copy and Paste buttons. You can use the keyboard shortcuts for copying and pasting (Ctrl+C and Ctrl+V) when using Google Chrome.

Design Mode

Design mode is a WYSIWYG (what you see is what you get) text editor that displays the instructions as you will see them when viewing a record.

HTML Mode

Clicking the HTML button displays the source code applied to the instructions. Because a WYSIWYG editor may insert markup code (tags) automatically, viewing the instructions in HTML mode allows you to validate and modify that code if necessary.

How to Enter Classification and Description Information

Follow these steps to enter classification and description information:

1. Select Record Generic Event from the Events menu. The Record Generic Event: Classification page appears.
2. Select an Organization from the list or accept the default.
3. Select a Classification. The application updates the Subclassification list accordingly.
4. Select a Subclassification.
5. Enter a Brief Description and a Description for the event.
6. Enter or select a Priority Level.
7. Select the date and time the event occurred in the Occurrence Date and Time field, if it is different from the default.
8. Enter or select a Discovery Area.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on optional fields.
10. Click Next to move to the next page in the pageflow. The Failures page appears.
You can also click Save to save your changes before moving to the next page.
