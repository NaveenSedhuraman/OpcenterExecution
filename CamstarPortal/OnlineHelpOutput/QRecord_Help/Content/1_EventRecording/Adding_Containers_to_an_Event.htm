Adding Containers (Lots) to an Event

The Affected Material page allows you to add one or more containers (lots) with affected material to the event if relevant. You can also add containers that are not tracked by the manufacturing execution system (MES). The Affected Material page is the third page in the generic event recording pageflow.

When Adding Affected Materials

The Affected Material page has two subtabs:

• Search - The Search subtab provides filters you can use to search for containers you want to add to the event.
• Add Manually - The Add Manually subtab allows you to scan or enter containers that you want to add to the event. You can use this subtab to add containers or lots not tracked by the MES.

A container may be added to more than one event. Adding a parent container to an event lists the parent on the event, but not any of its children. Adding a child container lists the child on the event, but not its parent or any sibling containers.

A system administrator can specify transactions such as Associate, Combine, and Split in the Add Affected Containers to Events grid on the Organization modeling object. The application adds affected containers to an event automatically when any of the specified transactions are executed on a container listed in the Event > Affected Material grid. Refer to "Adding Affected Containers to Events" for information.

When Searching for Containers

The application returns all containers (lots) that match the criteria you entered, including those that have been added to other event records already. You can select one or more of the returned containers to add to the event.

The application provides a check box that, when selected, selects all of the returned containers on the page, allowing you to add those containers to the event simultaneously. You can select one or more containers on the page individually. Clicking the Add Selected Containers button adds the selected containers to the Event > Affected Material grid.

The application returns a maximum of 1000 containers per search; however, the number of results displayed at one time depends on how the application is configured. If your search returns more lots than the row limit configured for the grid, the application displays paging icons. Refer to the Opcenter EX MDD Portal Studio User Guide or the Opcenter EX CR Portal Studio User Guide for information on configuring the number of rows displayed in a grid.

Clearing Search Results

Use the Clear All button to clear filters (parameters) and results on the Search subtab. Clearing the Search subtab enables you to start a new search. When you click the Clear All button, the application:

• Clears all specified search filters
• Clears all containers from the search results grid
• Does not clear added containers from the Affected Materials grid
• Continues displaying the Search subtab

When Adding Containers Manually

The Add Manually subtab allows you to add containers or lots tracked by your MES and those not tracked by your MES. You can scan or enter the container name. The application may populate other fields if the container you entered is tracked by your MES and that information was entered for the container previously. Clicking the Add Container button adds the container (lot) to the Event > Affected Material grid and clears the lot fields.

Updating Containers Added Manually

When you select a row in the Event > Affected Material grid, the application populates the preceding fields with that container’s information and replaces the Add Container button with an Update Selected Container button. When you click Update Selected Container, the application saves your changes, clears the container fields, and displays the Add Container button.

If you use the Add Manually subtab to add containers that are tracked by the MES, the application does not update these containers on this subtab. Use the Container Maintenance transaction if you need to update attributes for these containers.

Affected Material Page with Search Subtab

The Affected Material page appears when you click Next on the Failures page or when you click the third step in the generic event recording pageflow. The Search subtab is the default. You can also access this page from a saved pageflow.

How to Add a Container to the Event

Follow these steps to add a container to the event:

1. Complete the "How to Enter Failure Information" procedure to access the Affected Material page.
2. Do you want to search for containers to select or add containers manually?

If you want to . . .

Then . . .

Search for containers to select
a. Enter or select one or more parameters to narrow your search.
b. Click Search. The application displays all containers matching the search criteria in the Search Results grid.
c. Select one or more containers in the Search Results grid.
d. Click Add Selected Containers. The container is added to the Event > Affected Material grid.
e. Click the Clear All button to clear the parameter fields and Search Results grid if you want to perform a new search.

Or

Go to step 3.

Add containers manually
a. Click the Add Manually subtab.
b. Enter the name of the Container/Lot. The application updates the Qty and UOM fields when you tab out of the field or place focus on a different field.
c. Update the Quantity of the container or lot, if necessary.
d. Update the UOM associated with the quantity, if necessary.
e. Enter optional information according to your business requirements. Refer to the "Affected Material Page with Add Manually Subtab Field Definitions" table for information on the optional fields.
f. Click Add Container. The container is added to the Event > Affected Material grid.
g. Repeat steps b-f to add additional containers.
h. Go to step 3.
3. Click Next. The Attachments page appears.
