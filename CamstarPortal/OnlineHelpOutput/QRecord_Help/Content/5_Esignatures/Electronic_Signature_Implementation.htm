Electronic Signature Implementation

An electronic signature is an electronically stored record that denotes accountability for the performance of a particular transaction. Its use is tracked through an audit trail. You use an electronic signature as the equivalent of a handwritten signature.

The Electronic Signatures module can be implemented for most quality record and approval-related functions. Transactions that are executed automatically through automated triage do not require an electronic signature. Functions where electronic signatures are typically employed include:

• Canceling an approval
• Closing a quality record

How It Works

Electronic signatures are configured so that a signature is entered for a specific action. When the you click the Submit button on a quality record transaction that requires an electronic signature, the ESig pop-up appears.

Depending on how the Electronic Signatures module is set up, you must enter either a user name and password or only a password. You then click the Submit button on the ESig pop-up to complete the requirement.

Once an electronic signature is submitted, it is recorded in the Audit Trail as part of the transaction.

When Configuring Electronic Signature Requirements

Typically, when specifying quality record transactions for which you want to collect electronic signatures, you need to add only that quality service to the Electronic Signature Txn Map on the Organization modeling object. For example, if you want to collect an electronic signature when a user cancels an approval for a record resolution, you would add the Cancel Approval Sheet quality service to the Electronic Signature Txn Map. The Require Entry of User ID field determines whether the Esig Pop-up requires you to enter just a password, or the signer and password.

The exception is event creation. If you want to require electronic signatures for event recording pageflows, then you must specify not only the Create Event service in the Electronic Signature Txn Map, but you must also add the Update Event service.

An operator can bypass the electronic signature requirement when creating an event, if the Update Event service is not added to the Electronic Signature Txn Map.
