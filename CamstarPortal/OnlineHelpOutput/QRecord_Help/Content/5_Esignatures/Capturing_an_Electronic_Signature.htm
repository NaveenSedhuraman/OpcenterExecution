Capturing an Electronic Signature

The ESig Popup appears when you click the Submit button on a quality record transaction that requires the signature.

When Capturing an Electronic Signature

If the Signer and Password fields are blank when the pop-up appears, then both fields are required. If the Signer field defaults to your user name when the pop-up appears, then only the Password field is required.

Note: The Require Entry of User ID field in the Organization modeling object determines whether the application requires you to enter a signer and password or password only.

ESig Popup Field Definitions

This table defines the fields on the ESig Popup.

Field

Definition

Type

Signer

User name of the person submitting the electronic signature. Settings in the Organization modeling object determine whether the application defaults this field to the logged-in user’s user name.

Required

Signer Password

Password of the user submitting the electronic signature.

Required

How to Submit an Electronic Signature

Follow these steps to enter and submit your electronic signature:

1. Submit a transaction that requires an electronic signature. The ESig Popup appears.
2. Enter the appropriate user name in the Signer field, if this field is blank.
3. Enter the appropriate password in the Password field.
4. Click Submit. The application displays a success message and returns you to the page from which you accessed the transaction.
