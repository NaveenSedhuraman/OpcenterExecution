Recording a Production Event

The Record Production Event transaction enables you to enter the details of a container-related nonconformance, which include:

• Relevant failure modes
• Containers affected by the event
• Resource affected (optional)
• Specific step in the workflow against which the failure can be charged (optional)

You can also:

• Place a hold on the affected materials
• Attach related files

When Recording a Production Event

You must enter one or more failure modes for the product in the Failure Modes grid when entering information on the Record Production Event page. You must also enter one or more containers in the Affected Containers grid. At least one failure mode and one container are required.

The application automatically populates the Affected Containers grid with the containers selected on the original page if you access the Record Production Event pop-up from another page (for example, Container Search).

The application bypasses any electronic signature requirements defined for the standard hold transaction when you select the Hold affected containers check box. It places the specified containers on hold without requiring an electronic signature.

You must assign a Primary Organization to the employee before you can record a production event. You must also create a Numbering Rule in Modeling and assign it to a nonconformance on the Organization object. In addition, the primary organization must have a Role and Owner for the logged-in user assigned to the appropriate Classification and Subclassification in the Event Classification Spec Maps grid.

The Classification and Subclassification fields in the Show/Hide additional fields section of the Record Production Event page cannot be blank. They default to the values specified for the Classification and Subclassification fields in the row of the Event Classification Spec Maps grid that is selected as the PE Default. You can change the default settings. You complete the Event Classification Spec Maps grid using the Organization modeling object. Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information.

A message appears indicating that a default classification and subclassification must be set for a production event using the Organization modeling object if there are no default values for Classification and Subclassification when the Record Production Event page loads.

You can also select to print labels manually following a production event transaction if the Production Event Label Map is not configured on the factory.Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information.

The application creates a history for that event when a production event is created.

The application prevents you from entering these characters in the Title when adding attachments: +, &, #.

The maximum size for an attachment is 30 MB.

Record Production Event Page

You can access the Record Production Event page from the Container menu. The Record Production Event page enables you to record container-related nonconformances.

Note: The Record Production Event page may also be available as a pop-up if the record production event transaction has been configured as an available action on another page (for example, Container Search).

How to Record a Production Event

Follow these steps to record a production event:

1. Select Record Production Event from the Container menu. The Record Production Event page appears.
2. Enter a Description of the event.
3. Select a Failure Mode Group. The Failures Mode grid is updated to display all failure modes in the specified failure mode group.
4. Select one or more failure modes in the Failure Modes grid.
5. Scan or enter a container in the Add Container field. The container name is added to the Affected Containers grid.
Note: If you are working on the Record Production Event pop-up, the containers you selected on the previous page are displayed in the Affected Containers grid automatically.
6. Repeat step 5 to add more containers.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on optional fields.
8. Complete the “How to Attach Files to an Event Record” procedure if you want to attach files to the event record.
9. Click Record Production Event. The application displays a status message indicating the production event record was created. The Print Production Event Label page displays if your organization purchased the Label Printing module and the Manually print labels after recording check box was selected.

How to Attach Files to an Event Record

Follow these steps to attach one or more files to an event record on the Record Production Event page:

1. Perform the "How to Record a Production Event" procedure, steps 1-8.

Or

Select an existing production event.

2. Click the Show additional fields (Attach files) control. The application displays the Attachments grid.
3. Click Add new row. The Add Attachments pop-up appears.
4. Enter the path and file name in the Original File Path field.

Or

Click Browse to navigate to the file.

5. Enter the Title of the file.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on optional fields.
7. Click Upload. The file information is added to the Attachments grid.
8. Repeat steps 3-7 to add more files.
