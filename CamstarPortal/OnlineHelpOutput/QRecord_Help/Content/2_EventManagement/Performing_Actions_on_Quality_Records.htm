Performing Actions on Quality Records

You can perform two types of actions on a quality record:

• Quality
• Shop floor

The application displays buttons grouped by action type on the right side of a record’s management page. The application enables actions only when you have permission to perform that action. Additionally, the following criteria determine whether the application enables an action button:

• Type of record you are viewing
• Current state of the record
• Tab you are viewing

Quality Actions

The application enables quality actions based on status and user roles. This table describes the quality actions, which are listed in the order they appear on the Quality Actions pane. The table also lists the criteria that must be met for the application to enable the action. You must have permission to perform these actions.

Action Button

Click this button to . . .

Triage

Display the Triage page. The application displays the default values from the triage spec in the fields if the record was triaged automatically.

The application enables the Triage button when the:

• Record has not been triaged manually
• Status of the record is NOT Pending, In Review, or Closed

Refer to "Performing Triage" for information.

Change Category

Display the Change Category page. The application enables the Change Category button when the status of the record is NOT Pending, In Review, or Closed. Refer to "Changing Categories" for information.

Reassign

Display the Reassign Owner page. The application enables the Reassign button when the status of the record is NOT Pending, In Review, or Closed. Refer to “Reassigning an Owner” for information.

Approve

Display the Sign Approval page. The application enables the Approve button when all of these conditions are met:

• The record has an associated approval sheet.
• The status of the record is In Review.
• You are a specified approver.

Refer to “Approving a Quality Record Resolution” for information.

Cancel Approval

Update the record’s management page to display the Approval Cancellation page. The application enables the Cancel Approval button when both of these conditions are met:

• The record has an associated approval sheet.
• The status of the record is In Review.

Refer to “Canceling an Approval” for information.

Resolve

Display the Quality Object Resolution page. The application enables the Resolve button when the:

• Status of the record is NOT Pending, In Review (with outstanding approvals), or Closed
• Status of the record is In Review and all approvals have been captured

Refer to “Resolving a Record” for information.

Reopen

Display the Reopen page. The application enables the Reopen button when the status of the record is Closed. Refer to “Reopening a Record” for information.

Assign Checklist

Display the Assign Checklist page. The application enables the Assign Checklist button when the status of the event record is NOT Pending, In Review, or Closed. Refer to “Assigning a Checklist” for information.

Quality Action Page Button Definitions

This table describes the buttons common to all of the quality action pages. Buttons specific to a certain action page are described in that action page’s section.

Button

Click this button to . . .

Cancel

Discard any information entered on the action page, and then return to the record’s management page.

Submit

Perform the action on the record, and then return to the record’s management page.

Note: This button appears on every quality action page except the Quality Object Resolution page.
