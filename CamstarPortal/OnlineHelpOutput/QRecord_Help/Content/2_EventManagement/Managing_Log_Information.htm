Managing Log Information

The Log tab allows you to view and enter high-level, general comments about the event. The Log tab displays all comments added by all users, and can be used as a central repository to trace and coordinate all event processing activity.

When Logging a Comment

The application requires you to select a comment type when logging a comment. Three comment types are provided by default: Containment, Disposition, and Investigation. Two comment types are provided by default: Containment and Investigation. A system administrator can define additional comment types in Modeling.

A comment may contain up to 2000 characters. When added to the Comment Log grid, the application wraps the comment and stretches the row so the entered comment is visible.

Click the Add Comment button to log your comment to the event. The application updates the Comment Log grid and displays your comment at the top.

Click the Update Comment button to save any information you add or modify on the Log tab.

Note: You cannot remove comments from the grid.

How to Log a Comment

Follow these steps to log a comment:

1. Access the quality record. The record's management page appears.
2. Click the Log tab. The Log page appears.
3. Click Set Edit Mode. The record is switched to edit mode.
4. Select a Comment Type.
5. Enter your comment in the Comment field.
6. Click Add Comment. The application displays your comment at the top of the Comment Log.
7. Click Update Comment. The application displays a message stating the event has been updated.
