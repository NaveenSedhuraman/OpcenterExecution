Performing Triage

Siemens provides the ability to triage a quality record. Triage can occur automatically or manually and is typically the first action performed on a record.

Triage enables you to perform these actions on a record:

• Change the category
• Change the role and owner
• Change the priority level
• Assign a checklist

For example, triage can change the category of a quality record from Event to Nonconformance.

Note: When changing the category of a quality record, the event type changes also. For example, when you change the category of a record from event to nonconformance, the generic event becomes a production event.

Automatic Triage

Automatic triage begins as soon as you create a quality record if a system administrator has set up business rules on a triage spec set to execute at event creation. Regardless of whether automatic triage occurs, you can manually triage the record at a later time.

The application determines the business rules to run by:

• Using the Category Map on the organization to determine the triage spec for the specified category.
• Determining whether the triage spec has one or more business rules attached. The application executes them sequentially if business rules exist.

Manual Triage

You can perform triage manually after the application performs automatic triage. The application only allows you to manually triage a record once. You also cannot manually triage a page if you change categories using the Change Category page.

You can initiate manual triage from the Message Center, the Quality Search page, and the quality record’s management page. The application displays the Triage page for that event record. The Triage page displays the default values set by the triage spec if triage was performed automatically on event creation. Users with ownership permissions can override the default values during manual triage.

Before Performing Triage

You may need to set up one or more of these modeling objects to enable automatic and manual triage before performing triage:

• Business rule
• Employee
• Organization
• Role
• Triage spec

Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information on defining these modeling objects.

When Performing Triage

The application considers triage complete when you perform one or more of these actions on the Triage page:

• Assign an owner
• Change the record’s category
Note: The application requires you to assign a role and owner before you can perform any other action on the record if the owner was not set during automatic triage. Therefore, the application considers triage complete when you assign the owner.

The application displays a warning message when you try to assign a checklist to the quality record and a checklist is assigned already. The warning message appears when you click Submit. Clicking Yes overwrites the assigned checklist and any data saved for that checklist. Clicking No allows you to remove the specified checklist template before clicking Submit again.

The application discards your triage input if you perform any of these actions during manual triage:

• Log off from the application
• Navigate away from the Triage page
• Click the Cancel button on the Triage page
• Allow the application to time out

Role and Owner

The Role selection list is not restricted to the roles associated with the quality record’s organization. However, the Owner selection list is limited to those employees who are assigned the selected Role for the quality record’s organization. The applicable employees may have been specified explicitly for the organization or may be propagated from a parent organization.

Note: The application displays an error message indicating that you do not have permission to change the record if the original record owner is not the current user or a user with the same role as the current user. The owner may be reassigned only if the event is assigned to the current user or the new owner has the same role.

When Accessing the Triage Page

You can access the Triage page from the following:

• The record’s management page
• Message Center
• Quality Search page

The Triage action is available only for those event records that have not already been triaged manually.

Clicking Submit or Cancel returns you to the page from which you accessed the Triage page.

How to Manually Perform Triage on a Quality Record

Follow these steps to perform triage manually on a quality record:

1. Select the quality record you want to triage by:
• Viewing the record’s management page
• Selecting the record in the Search Results grid on the Quality Search page
• Selecting the record in the Results grid on the Message Center page
2. Click Triage. The Triage page appears.
3. Select a Category.
4. Select a Role.
5. Select an Owner.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on optional fields.
7. Click Submit. The application displays a message indicating the record was triaged and then displays the page from which you launched the Triage page.
