Managing Cross References

The Cross References tab provides a list of event records that have been added to the record as a cross reference. Use this tab to search for and add cross references as well as remove cross references. The application updates any records added as cross references to show that they are cross referenced with this record.

When Adding Records as Cross References

You can view information such as the owner’s name, classification, subclassification, and so on, for each record. You can use SQL wildcards in the Identifier and Container fields when searching for records to add as cross references. The percent sign (%) is a SQL wildcard. Refer to the w3schools website for information on SQL wildcards: http://www.w3schools.com/sql/sql_wildcards.asp.

You can click the View Selected Event Details button to view a record returned by the search or a cross referenced record in a separate pop-up.

The application removes cross references from the record's list when you remove a cross reference.

When using the check box in the header row of the Search Results grid to select all the records returned by your search, you are selecting only those displayed—not all results.

Click the Update Cross References button after you add or remove cross references.

Cross References Tab Button Definitions

This table defines the buttons specific to the Cross References tab of the record’s management page.

Button

Click this button to . . .

Clear All

Clear the search criteria fields and the Search Results grid. Clearing the Cross References tab enables you to start a new search.

Search

Find all records meeting your entered search criteria that are eligible to be added as cross references.

Add Selected Reference

Add the records selected in the Search Results grid as cross references for the current record. You can add one or more containers simultaneously.

Note: The application enables this button for a grid when you select one or more records in that grid.

View Selected Event Details

View details for the selected record in a separate pop-up. The category of the selected record determines the pop-up displayed.

The application displays the:

• Manage G/E pop-up for generic event records.
• Manage P/E pop-up for production event (nonconformance)

This button is available for records in both the Search Results and Current References grids.

Note: The application enables this button for a grid when you select a record in that grid. If you select more than one record in the grid, the application disables the button.

How to Add a Cross Reference

Follow these steps to add a cross reference:

1. Access the quality record. The record's management page appears.
2. Click the Cross References tab. The Cross References page appears.
3. Click Set Edit Mode. The record is switched to edit mode.
4. Select the criteria to search for the records from one or more of the available fields at the top of the tab.
5. Click Search. The application displays all records meeting your search criteria that are eligible to be added as cross references in the Search Results grid.
6. Do one of the following:
• Select the check box for each event record you want to add as a cross reference.
• Select the check box in the header row to select all displayed event records.
7. Click Add Selected Reference. The application updates the record and displays the added references in the Current References grid.
8. View the successive pages of search results and repeat steps 6-7, if necessary.
9. Click Update Cross References. The application displays a message stating the event has been updated.

How to Remove a Cross Reference

Follow these steps to remove a cross reference:

1. Access the quality record. The record's management page appears.
2. Click the Cross References tab. The Cross References page appears.
3. Click Set Edit Mode. The record is switched to edit mode.
4. Select the cross reference to remove in the Current References grid.
5. Click Delete selected row in the grid. The record is removed from the grid.
6. Click Update Cross References. The application displays a message stating the event has been updated.
