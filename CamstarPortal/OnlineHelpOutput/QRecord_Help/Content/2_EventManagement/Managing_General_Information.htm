Managing General Information

The General tab allows you to view and edit information entered when a user recorded the event and to provide additional information about the event.

When Managing General Information

Click the Save button to save any information you add or modify on the General tab.

When Managing Transactions

Disallowed transactions are container transactions the application cannot execute against containers associated with events. The disallowed transactions are specified on events. The application evaluates a container when you execute a transaction to determine if the container is on an event and if the transaction is allowed. A disallowed transaction on one event prevents the application from executing that transaction on a container when that container is associated with multiple events.

The application adds disallowed transactions to an event automatically if any are specified for the organization associated with the user who created the event. Disallowed transactions appear in the Disallowed Transactions grid. You can also add disallowed transactions to the event manually. All container transactions are available to add.

Default disallowed transactions are specified in Modeling using the Organization modeling object. Refer to the Opcenter EX MDD Modeling User Guide or the Opcenter EX CR Modeling User Guide for information.

Some transactions, such as Associate, involve multiple containers. For example, if you specify Associate as a disallowed transaction and a user attempts to execute this transaction on a container, the application evaluates all containers involved in the transaction.

The application evaluates disallowed transactions only for active events. The application does not evaluate disallowed transactions for closed events. A disallowed transaction executed against a container on a closed event will succeed.

The transaction fails and the application displays an error message if you attempt to execute a disallowed transaction.

Note: Changes to the default disallowed transactions in the event creator's associated organization do not affect existing events. You must manually update the disallowed transactions for existing events.

How to Modify Information on the General Tab

Follow these steps to modify general information for an existing quality record:

1. Access the quality record. The record’s management page appears with the General tab displayed by default.
2. Click Set Edit Mode. The record is switched to edit mode.
3. Modify the information as necessary. Refer to the field definitions table for information on the fields.
4. Click Save. The tab displays the updated information.
