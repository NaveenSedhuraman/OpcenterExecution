Managing Investigation Information

The Investigation tab consists of two subtabs: Define Causes and Define Actions. The Define Causes subtab allows you to assign one or more causes to a failure mode. The Define Actions subtab allows you to create or modify actions and to assign actions to the event record.

When Assigning Cause Codes

The Define Causes subtab is the default tab displayed when you access the Investigation tab. The application populates the Event > Failure Mode > Cause Code grid with the failure modes assigned to the event. You can assign one or more cause codes to a failure mode. Cause codes and cause code groups are defined in Modeling by a system administrator.

When assigning a cause code to a failure mode, you must select a failure in the grid before clicking the Add Cause to Selected Failure(s) button. The application adds a child row containing cause code information to that failure mode, and the cause code subgrid remains expanded by default. You can also modify cause information using the Update Selected Cause button. This button appears when an assigned cause is selected in the grid.Click the Update Causes button to save any information you add or modify on the Define Causes subtab.

When Assigning Action Plans

The Define Actions subtab allows you to assign actions to cause codes assigned to failure modes. You can assign actions at the event level using the Add Action to Event button. You can assign multiple actions to cause codes and to the event itself. Failure action types and failure action type groups are defined in Modeling by a system administrator.

When assigning an action to a cause code, you must select a cause code in the grid before clicking the Add Action to Selected Cause(s) button. The application adds a child row containing action information to the cause code, and the action subgrid remains expanded by default. You can also modify action information using the Update Selected Actions button or the Update Selected Action to Event button. These buttons appear when an assigned action is selected in the grid.

How to Assign a Cause Code to a Failure Mode

Follow these steps to assign a cause code to a failure mode:

1. Access the quality record. The record's management page appears.
2. Click the Investigations tab. The application displays the Investigations tab and the Define Causes subtab. All cause modes assigned to the quality record appear in the grid.
3. Click Set Edit Mode. The record is switched to edit mode.
4. Select a Cause Code.
5. Enter optional information according to your business requirements. Refer to the Define Causes Subtab Field Definitions table for information on optional fields.
6. Select the check boxes for the failure modes to which you want to assign the cause code in the Event>Failure Mode>Cause Code grid.
7. Click Add Cause to Selected Failure(s). The application adds a child row with cause code information to each selected failure mode.
8. Repeat steps 4-7 to assign more cause codes.
9. Click Update Causes. The application displays a message stating the event has been updated.

How to Assign an Action

Follow these steps to assign an action:

1. Access the quality record. The record's management page appears.
2. Click the Investigations tab. The Investigations page appears.
3. Click Set Edit Mode. The record is switched to edit mode.
4. Click the Define Actions subtab. The action fields are displayed.
5. Select an Action Type.
6. Enter optional information according to your business requirements. Refer to the Define Actions Subtab Field Definitions table for information on optional fields.
7. Do you want to add the action to a cause code or the event?

If you want to add the action to . . .

Then . . .

A cause code
a. Expand the failure mode in the Event>Failure Mode>Cause Code>Action grid containing the cause codes to which you want to assign an action.
b. Select the check boxes for the cause codes.
c. Click Add Action to Selected Cause(s). The application adds a child row with action information to each selected cause code.
d. Go to step 8.

The event
a. Click Add Action to Event. The application adds the action to the Action Type grid.
b. Go to step 8.
8. Repeat steps 5-7 to assign more actions.

Or

Click Update Actions. The application displays a message stating the event has been updated.
