Managing a Production Event Record

The quality record management functionality provides the ability to view and to update quality records (events and nonconformances) by adding new information and modifying existing information.

When Managing Quality Records

You must be either the record owner or another user with the appropriate permissions to view and to edit a quality record.

You can access a quality record’s management page from the following:

• Your Concierge
• Message Center
• Quality Search page

The management page has two modes: view mode and edit mode. View mode is the default mode and is read-only. Edit mode allows you to make changes to the record’s details. You must be in edit mode to save any updates to a record. The application provides toggle buttons to allow you to change modes.

The application provides several tabs on a record’s management page. The application adds a Cancel button to the tabs when you switch to edit mode. A Save button is added to the General tab and an Update button specific to the transaction you are performing is added to the other tabs. All of these buttons are hidden in view mode.

Note: The application does not add any buttons to the Resolution tab because it is always read only regardless of mode.

General Button Definitions

This table describes some general buttons that apply to the record management page and tabs. Refer to “Performing Actions on Quality Records” for descriptions of the action buttons.

Button

Click this button to . . .

Set Edit Mode

Switch the record to edit mode. Switching to edit mode:

• Enables the Set View Mode button
• Replaces the Set Edit Mode button with the statement, “In Edit Mode”
• Adds the Cancel and Save buttons to the bottom of every tab

Set View Mode

Switch the record to view mode. Switching to view mode:

• Enables the Set Edit Mode button
• Replaces the Set View Mode button with the statement, “In View Mode”
• Removes the Cancel and Save buttons

Cancel

Clear any pending data entered or deleted. This button appears in edit mode only.

When Editing a Quality Record

You must switch to edit mode to modify record information. The application does not allow you to modify the following:

• The Reporter, Reporter Organization, and the Reported Date
• Closed records. (You must reopen a closed record before you can edit it.)

You must save your changes before switching to view mode. Switching to view mode without saving will clear any pending changes you made.

When saving a quality record with an assigned checklist, the application requires you to complete the required items on the checklist before saving. The application displays an error message if you try to save a record without completing the required checklist items.

Manage P/E Page

Quality record management pages appear in view mode by default when accessed. Each page consists of three parts:

• A general information pane
• Tabs for viewing and editing record details
• An actions pane for toggling the modes and performing actions on the record

General Information Pane

The General Information pane appears on the left side of every record’s management page. All fields are display-only and cannot be edited. The General Information pane displays key attributes of the record such as:

• Event (ID assigned by the application)
• Priority Level
• Category
• Organization
• Classification
• Subclassification
• Status
• Role
• Owner

Record Management Tabs

This table describes the tabs and subtabs available on a quality record’s management page. Each tab is described in detail later in this chapter.

Tab

Description

General

Displays the general information that was entered when the event was recorded and allows you to edit most of that information. Refer to "Managing General Information" for information.

Failures

Displays the failures specific to the quality record and allows you to update, add, and remove failure modes. Refer to "Managing Failure Information" for information.

Affected Material

Displays containers with affected material that have been added to the record and allows you to update, add, and remove containers. This tab consists of two subtabs:

• Search - Used to locate containers that may need to be added to the event record.
• Add Manually - Used to include a container or lot that is not tracked by the manufacturing execution software (MES).

Refer to "Managing Affected Material Information" for information.

Log

Displays high-level, general comments pertaining to any aspect of the quality record and allows you to add additional comments. Refer to "Managing Log Information" for information.

Attachments

Displays any files attached to the record and allows you to view those files. Additionally, you can upload more files to the record. Refer to "Managing Attachments Information" for information.

Investigation

Displays causes associated with the specified failure modes and actions associated with the causes or the event as a whole and allows you to update, add, and remove associations. This tab consists of two subtabs:

• Define Causes - Used to associate causes to failures associated with the event.
• Define Actions - Used to associate one or more action types to a
previously-assigned cause or directly to the event.

Refer to "Managing Investigation Information" for information.

Disposition

Displays the dispositions assigned to the containers associated with the event record and the failure modes associated with the disposition. Additionally, you can update, assign, and remove dispositions and failure mode associations. This tab consists of two subtabs:

• Assign Disposition - Used to associate dispositions with causes.
• Link Failure Modes - Used to associate failures to a disposition for reporting.

Refer to "Managing Disposition Information" for information.

Cross References

Displays links between the existing record and other previously-recorded quality records and allows you to update, add, and remove associations. Refer to "Managing Cross References" for information.

Checklist

Displays the checklist, if any, assigned to the record and allows you to complete the checklist. Refer to "Managing Checklist Information" for information.

Resolution

Displays the information entered when this record was resolved (closed). This tab is always read-only and the fields remain blank until the record has at least one resolution approval or has been resolved. Refer to "Managing Resolution Information" for information.

Record Management Actions

The quality and shop floor actions available depend on the status of the record, the tab you are viewing, and the permissions associated with your user ID and roles. Refer to “Performing Actions on Quality Records” for information on the actions you can perform from a record’s management page.
