Verwalten von Ermittlungsinformationen

Die -Registerkarte "Ermittlung" umfasst zwei Unterregisterkarten: "Ursachen definieren" und "Aktionen definieren". Die Unterregisterkarte "Ursachen definieren" bietet die Möglichkeit, Gründe zu einem Fehlermodus hinzuzufügen. Die Unterregisterkarte "Aktionen definieren" bietet die Möglichkeit, Aktionen zu erstellen oder zu ändern und dem Ereignisdatensatz Aktionen zuzuweisen.

Beim Zuweisen von Ursachencodes

Wenn Sie die Registerkarte "Ermittlung" öffnen wird standardmäßig die Unterregisterkarte "Ursachen definieren" angezeigt. Die Anwendung trägt die dem Ereignis zugeordneten Fehlermodi in die Tabelle "Ereignis > Fehlermodus > Ursachencode" ein. Sie können einem Fehlermodus einen oder mehrere Ursachencodes zuweisen. Ursachencode und Ursachencodegruppen werden von einem Systemadministrator in der Modellierung definiert.

Wenn Sie einem Fehlermodus einen Ursachencode zuweisen, müssen Sie einen Fehler in der Tabelle auswählen, bevor Sie auf die Schaltfläche "Ursache zu ausgewählten Fehlern hinzufügen" klicken. Die Anwendung fügt eine untergeordnete Zeile mit Informationen zum Ursachencode für den jeweiligen Fehlermodus hinzu. Die Untertabelle "Ursachencode" bleibt standardmäßig eingeblendet. Sie können die Ursacheninformationen auch über die Schaltfläche "Ausgewählte Ursache aktualisieren" ändern. Diese Schaltfläche wird angezeigt, wenn eine zugewiesene Ursache in der Tabelle ausgewählt ist. Klicken Sie auf die Schaltfläche "Gründe aktualisieren", um alle Informationen zu speichern, die Sie auf der Unterregisterkarte "Ursachen definieren" hinzugefügt oder geändert haben.

Beim Zuweisen von Aktionsplänen

Die Unterregisterkarte "Aktionen definieren" bietet die Möglichkeit, Aktionen mit Ursachencodes zu verknüpfen, die den Fehlermodi zugewiesen wurden. Sie können Aktionen über die Schaltfläche "Aktion zum Ereignis hinzufügen" auf der Ereignisebene zuweisen. Sie können Ursachencodes und dem eigentlichen Ereignis mehrere Aktionen zuweisen. Fehleraktionstypen und Fehleraktionstypgruppen werden von einem Systemadministrator in der Modellierung definiert.

Wenn Sie einem Ursachencode eine Aktion zuweisen, müssen Sie einen Ursachencode in der Tabelle auswählen, bevor Sie auf "Aktion zu ausgewählten Ursachen hinzufügen" klicken. Die Anwendung fügt eine untergeordnete Zeile mit Aktionsinformationen zum Ursachencode hinzu. Die Untertabelle "Aktion" bleibt standardmäßig eingeblendet. Sie können Aktionsinformationen auch über die Schaltfläche "Ausgewählte Aktion aktualisieren" oder "Ausgewählte Aktion zum Ereignis aktualisieren" ändern. Diese Schaltflächen werden angezeigt, wenn eine zugewiesene Aktion in der Tabelle ausgewählt wird.

Vorgehensweise zum Zuweisen eines Ursachencodes zu einem Fehlermodus

Führen Sie die folgenden Schritte aus, um einem Fehlermodus einen Ursachencode zuzuweisen:

1. Rufen Sie den Qualitätsdatensatz auf. Die Verwaltungsseite des Datensatzes wird angezeigt.
2. Klicken Sie auf die Registerkarte Ermittlung. Die Anwendung zeigt die Registerkarte "Ermittlung" und die Unterregisterkarte "Ursachen definieren" an. In der Tabelle werden alle Ursachenmodi angezeigt, die dem Qualitätsdatensatz zugewiesen wurden.
3. Klicken Sie auf Bearbeitungsmodus aktivieren. Der Datensatz wird in den Bearbeitungsmodus versetzt.
4. Wählen Sie einen Ursachencode aus.
5. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in der Tabelle mit den Felddefinitionen der Unterregisterkarte "Ursachen definieren".
6. Aktivieren Sie die Kontrollkästchen für die Fehlermodi, denen Sie den Ursachencode in der Tabelle "Ereignis > Fehlermodi > Ursachencode" zuweisen möchten.
7. Klicken Sie auf Ursache zu ausgewählten Fehlern hinzufügen. Die Anwendung fügt eine untergeordnete Zeile mit Ursachencodeinformationen zum jeweils ausgewählten Fehlermodus hinzu.
8. Wiederholen Sie die Schritte 4 bis 7, um weitere Ursachencodes zuzuweisen.
9. Klicken Sie auf Gründe aktualisieren. Die Anwendung zeigt eine Meldung mit dem Hinweis an, dass das Ereignis aktualisiert wurde.

Vorgehensweise zum Zuweisen einer Aktion

Führen Sie die folgenden Schritte aus, um eine Aktion zuzuweisen:

1. Rufen Sie den Qualitätsdatensatz auf. Die Verwaltungsseite des Datensatzes wird angezeigt.
2. Klicken Sie auf die Registerkarte Ermittlung. Die Seite "Ermittlung" wird angezeigt.
3. Klicken Sie auf Bearbeitungsmodus aktivieren. Der Datensatz wird in den Bearbeitungsmodus versetzt.
4. Klicken Sie auf die Unterregisterkarte Aktionen definieren. Die Aktionsfelder werden angezeigt.
5. Wählen Sie einen Aktionstyp aus.
6. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in der Tabelle mit den Felddefinitionen der Unterregisterkarte "Aktionen definieren".
7. Möchten Sie die Aktion zu einem Ursachencode oder zum Ereignis hinzuzufügen?

Wenn Sie die Aktion hinzuzufügen möchten . . .

Dann . . .

Ursachencode
a. Erweitern Sie den Fehlermodus in der Tabelle "Ereignis > Fehlermodus > Ursachencode > Aktion" mit den Ursachencodes, denen Sie eine Aktion zuweisen möchten.
b. Aktivieren Sie die Kontrollkästchen für die Ursachencodes.
c. Klicken Sie auf Aktion zu ausgewählten Ursachen hinzufügen. Die Anwendung fügt eine untergeordnete Zeile mit Aktionsinformationen zum jeweils ausgewählten Ursachencode hinzu.
d. Gehen Sie zu Schritt 8.

Ereignis
a. Klicken Sie auf Aktion zum Ereignis hinzufügen. Die Anwendung fügt die Aktion zur Tabelle "Aktionstyp" hinzu.
b. Gehen Sie zu Schritt 8.
8. Wiederholen Sie die Schritte 5 bis 7, um weitere Aktionen zuzuweisen.

Oder

Klicken Sie auf Aktionen aktualisieren. Die Anwendung zeigt eine Meldung mit dem Hinweis an, dass das Ereignis aktualisiert wurde.
