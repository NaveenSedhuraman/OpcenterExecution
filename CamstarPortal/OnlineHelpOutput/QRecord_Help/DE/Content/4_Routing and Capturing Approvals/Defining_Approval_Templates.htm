Definieren von Genehmigungsvorlagen

Sie müssen die folgenden Modellierungsobjekte definieren oder ändern, um Genehmigungsvorlagen definieren und verwenden zu können:

• Genehmigungsentscheidungsliste
• Genehmigungsvorlage (Weiterleitungsblatt)
• Organisation

Da die Anwendung fordert, eine Genehmigungsentscheidungsiste mit einer Genehmigungsorlage zu verbinden, müssen Sie zuerst eine Genehmigungsentscheidungsliste definieren. Genehmigungsentscheidungslisten enthalten die Entscheidungen, die den entsprechenden Genehmigern angezeigt werden, wenn die Anwendung eine Genehmigung für die Lösung eines Qualitätsdatensatzes verlangt. Eine Genehmigungsentscheidungsliste kann z. B. folgende Entscheidungen enthalten: "Akzeptiert" und "Abgelehnt".

Definieren Sie als nächstes eine Genehmigungsvorlage. Genehmigungsvorlagen legen die Benutzer fest, die die Qualitätsdatensatzlösung genehmigen müssen. Außerdem enthält die Genehmigungsvorlage eine Liste möglicher Entscheidungen (die zugehörige Genehmigungsentscheidungsliste) für diese Benutzer.

Wenn Sie möchten, dass die Anwendung Genehmigungsvorlagen an Qualitätsdatensätze zuweist (für Qualitätsdatensatz-Lösungen), dann müssen Sie die Genehmigungsvorlagen dem Modellierungsobjekt der Organisation zuweisen. Wenn Sie einer Organisation Genehmigungsvorlagen zuweisen, weist die Anwendung abhängig von Kategorie und Lösungsaktion des Datensatzes automatisch die entsprechende Genehmigungsvorlage einem Qualitätsdatensatz zu.

Hinweis: Wenn Sie einer Organisation keine Genehmigungsvorlagen zuweisen, kann bei der Lösung des Qualitätsdatensatzes eine Genehmigungsvorlage zugewiesen werden.

Informationen zum Definieren oder Ändern dieser Modellierungsobjekte finden Sie im Opcenter EX MDD Modeling-Benutzerhandbuch bzw. im Opcenter EX CR Modeling-Benutzerhandbuch.
