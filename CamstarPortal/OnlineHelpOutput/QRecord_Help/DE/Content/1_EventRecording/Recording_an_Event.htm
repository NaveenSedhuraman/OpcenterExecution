Aufzeichnen eines Ereignisses

Wenn die Situation auftritt, dass Qualität und Verwendbarkeit von Elementen, Diensten, Aktivitäten oder Prozessen nicht den Spezifikationen entsprechen, müssen Sie das Ereignis, das diese Situation verursacht hat, aufzeichnen.

Die folgende Liste enthält einen Auszug der Daten, die erfasst werden können:

• Kategorisierung und Priorität
• Datum (Auftreten und Meldung)
• Entdeckungsbereich
• Produktinformationen (Teilenummer, Revision)
• Operation und Ressource
• Berichterinformationen
• Fehlerinformationen
• Betroffene Materialien, anwendbare Menge, anwendbare Komponente und Unterbaugruppen

Voraussetzungen zum Aufzeichnen eines Ereignisses

Ein Systemadministrator muss die jeweiligen Modellierungsobjekte definieren, damit ein Ereignis aufgezeichnet werden kann. Die folgenden Objekte sind erforderlich, um die Ereignisaufzeichnung durchzuführen:

• Klassifikation
• Unterklassifikation
• Fehlertyp
• Fehlerschweregrad
• Fehlermodus
• Fehlermodusgruppe
• Nummerierungsregel
• Organisation (insbesondere "Zuordnung von Ereignisklassifizierungsspez.", "Kategorieverarbeitungszuordnung" und "Nummerierungsregelzuordnung")
• Prioritätsstufe
• Unternehmen, Fabrik oder Arbeitsbereich
• Mitarbeiter (Urheber)

Darüber hinaus kann bei der Aufzeichnung eines Ereignisses u. a. eines der folgenden optionalen Modellierungsobjekte erforderlich:

• Operation
• Produkt
• Ressource

Informationen zum Definieren von Modellierungsobjekten finden Sie im Opcenter EX MDD Modeling-Benutzerhandbuch bzw. im Opcenter EX CR Modeling-Benutzerhandbuch.

Einrichten von elektronischen Unterschriften

Wenn bei der Erstellung eines Ereignisses eine elektronische Unterschrift erfasst werden soll, müssen Sie den Qualitätsservice "Ereignis erstellen" zur Transaktionszuordnung für elektronische Unterschriften des Modellierungsobjekts "Organisation" hinzufügen. Informationen zum Modellierungsobjekt "Organisation", zum Einrichten von elektronischen Unterschriften und zum Hinzufügen von Qualitätsservices für die Organisation finden Sie im Opcenter EX MDD Modeling-Benutzerhandbuch bzw. im Opcenter EX CR Modeling-Benutzerhandbuch.

Beim Aufzeichnen eines Ereignisses

Die Anwendung erstellt eine Historie für ein generisches Ereignis, wenn Sie das Ereignis speichern oder übermitteln. Wenn Sie ein Ereignis während der Aufzeichnung speichern und dann die Seitenfolge schließen oder abbrechen, können Sie den Ereignisdatensatz im Concierge aufrufen, um ihn zu bearbeiten oder zu löschen.

Wichtig: Wenn Sie ein Ereignis speichern und die Namen von Organisation, Klassifikation oder Unterklassifikation, die im Datensatz verwendet werden, geändert werden, bevor Sie das Ereignis übermitteln, müssen Sie den Datensatz löschen und das Ereignis neu eingeben.

Wenn Sie ein Dokument anhängen und auf die Schaltfläche "Hochladen" klicken, ist dies gleichwertig mit einem Speichervorgang. Die Anwendung speichert den Datensatz automatisch, wenn Sie eine Datei anhängen.

Wenn Sie bei der Aufzeichnung eines neuen generischen Ereignisses oder der Bearbeitung eines bestehenden Datensatzes auf die Schaltfläche "Abbrechen" klicken, bevor Sie auf "Speichern" klicken, wird die Seite "Generisches Ereignis aufzeichnen" geschlossen, ohne die eingegebenen Informationen zu speichern.

Wenn Sie auf die Schaltfläche "Löschen" klicken, wird ein gespeicherter Datensatz gelöscht. Die Schaltfläche "Löschen" wird auf allen Seiten in der Seitenfolge für die Aufzeichnung von generischen Ereignissen angezeigt, nachdem das Ereignis gespeichert wurde. Wenn Sie auf die Schaltfläche "Löschen" klicken, wird eine Warnmeldung angezeigt. Wenn Sie in der Warnmeldung auf "Ja" klicken, wird der Datensatz gelöscht und die Seite wird geschlossen. Die Anwendung zeigt keine gelöschten Datensätze im Concierge an. Weitere Informationen finden Sie unter "Löschen eines gespeicherten Ereignisses".

Wenn Sie auf die Schaltfläche "Abschicken" klicken, wird der Datensatz gespeichert und übermittelt. Die Anwendung zeigt dann die Verwaltungsseite des Datensatzes an.

Vorgehensweise zum Aufzeichnen eines Ereignisses

Führen Sie die folgenden Schritte aus, um ein Ereignis aufzuzeichnen:

1. Wählen Sie Generisches Ereignis aufzeichnen im Menü Ereignis aus. Die Seite "Generisches Ereignis aufzeichnen: Klassifikation" wird angezeigt.
2. Führen Sie die in den folgenden Abschnitten beschriebenen Schritte aus, um die erforderlichen Felder auf den ersten beiden Seiten der Seitenfolge für die Aufzeichnung eines generischen Ereignisses auszufüllen:
• "Vorgehensweise zum Eingeben von Informationen für Klassifikation und Beschreibung"
• "Vorgehensweise zum Eingeben von Fehlerinformationen"
3. Führen Sie ggf. die nachstehend beschriebenen Schritte aus, um die letzten beiden Seiten der Seitenfolge auszufüllen:
• "Vorgehensweise zum Hinzufügen eines Container zu einem Ereignis"
• "Vorgehensweise zum Anhängen einer Datei"
4. Klicken Sie auf der Seite "Anhänge“ auf Abschicken. Die Anwendung erstellt den Qualitätsdatensatz und zeigt die Verwaltungsseite des Datensatzes an.
