Auswählen von Transaktionen zum Hinzufügen von Containern zu Ereignissen

Ein Systemadministrator kann angeben, ob die Anwendung automatisch einen Container zu einem Ereignis hinzufügt, wenn Sie eine Transaktion für einen zugehörigen Container ausführen, der bereits zum Ereignis hinzugefügt wurde. Die Transaktionen werden in der Modellierung mit dem Modellierungsobjekt "Organisation" angegeben.

Die standardmäßig verfügbaren Transaktionen sind:

• Verbinden
• Vereinigen
• Komponentenverbrauch
• Komponente entfernen
• Aufteilen

Die Anwendung fügt die betroffenen Container zu einem Qualitätsereignis hinzu, wenn Sie die Transaktion "Container vereinigen" oder "Menge vereinigen" ausführen, sofern "Vereinigen" im Modellierungsobjekt "Organisation" festgelegt ist. Gleiches gilt für "Aufteilen" und die Transaktionen "Container aufteilen" und "Menge aufteilen". Weitere Informationen finden Sie im Opcenter EX MDD Modeling-Benutzerhandbuch bzw. im Opcenter EX CR Modeling-Benutzerhandbuch.

Verweigerte Transaktionen haben Vorrang vor den angegebenen Transaktionen zum automatischen Hinzufügen von Containern zu Ereignissen. Die Transaktion schlägt z. B. fehl und die Anwendung generiert eine Fehlermeldung, wenn "Verbinden" als verweigerte Transaktion für ein Ereignis und als Transaktion, die Container automatisch zu einem Ereignis hinzufügt, angegeben ist.

Hinweis: Sie können keine Qualitätsereignisse ändern, die den Status "In Klärung" aufweisen. Daher können Sie keine Container zu einem Ereignis hinzufügen, das "In Klärung" ist. Die Transaktion schlägt fehl und die Anwendung zeigt eine Fehlermeldung an.
