Implementierung elektronischer Unterschriften

Eine elektronische Unterschrift ist ein elektronisch gespeicherter Datensatz, der die Verantwortung für eine bestimmte Transaktion kennzeichnet. Die Verwendung der elektronischen Unterschrift wird mithilfe eines Protokolls verfolgt. Sie können eine elektronische Unterschrift als Ersatz für eine handschriftliche Unterschrift verwenden.

Das Modul für elektronische Unterschriften kann für fast alle Qualitätsdatensatzfunktionen und genehmigungsbezogenen Funktionen implementiert werden. Für Transaktionen, die automatisch durch eine automatisierte Einteilung ausgeführt werden, ist keine elektronische Unterschrift erforderlich. Funktionen, bei denen normalerweise elektronische Unterschriften verwendet werden, umfassen:

• Abbrechen einer Genehmigung
• Schließen eines Qualitätsdatensatzes

Funktionsweise

Elektronische Unterschriften sind so konfiguriert, dass eine Unterschrift für eine bestimmte Aktion eingegeben wird. Wenn Sie bei einer Qualitätsdatensatztransaktion, für die eine elektronische Unterschrift erforderlich ist, auf die Schaltfläche "Abschicken" klicken, wird das Popup "Elektronische Unterschrift" geöffnet.

Je nachdem, wie das Modul für elektronische Unterschriften eingerichtet ist, müssen Sie entweder einen Benutzernamen und ein Kennwort oder nur ein Kennwort eingeben. Klicken Sie anschließend im Popup "Elektronische Unterschrift" auf die Schaltfläche "Abschicken".

Sobald eine elektronische Unterschrift abgeschickt wurde, wird dies im Protokoll als Teil der Transaktion protokolliert.

Beim Konfigurieren von Anforderungen für elektronische Unterschriften

Wenn Sie Qualitätsdatensatztransaktionen angeben, für die elektronische Unterschriften erfasst werden sollen, müssen Sie diesen Qualitätsservice im Modellierungsobjekt "Organisation" normalerweise nur zur Transaktionszuordnung für elektronische Unterschriften hinzufügen. Wenn Sie z. B. eine elektronische Unterschrift erfassen möchten, wenn ein Benutzer die Genehmigung für eine Datensatzlösung abbricht, müssen Sie den Qualitätsservice "Genehmigung abbrechen-Blatt" zur Transaktionszuordnung für elektronische Unterschriften hinzufügen. Das Feld "Benutzer-ID-Eingabe erforderlich" legt fest, ob Sie im Popup "Elektronische Unterschrift" nur ein Kennwort oder Benutzername und Kennwort eingeben müssen.

Davon ausgenommen ist die Ereigniserstellung. Wenn für Seitenfolgen zur Ereignisaufzeichnung elektronische Unterschriften vorschreiben möchten, so müssen Sie nicht nur den Service "Ereignis erstellen" in der Transaktionszuordnung für elektronische Unterschriften angeben, sondern auch den Service "Ereignis aktualisieren" hinzufügen.

Ein Operator kann bei der Erstellung eines Ereignisses die erforderliche elektronische Unterschrift umgehen, wenn der Service "Ereignis aktualisieren" nicht zur Transaktionszuordnung für elektronische Unterschriften hinzugefügt wird.
