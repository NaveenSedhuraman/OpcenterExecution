输入分类和常规信息

“分类”页是通用事件记录页面流中的第一个页面。通过该页面，可以指定要为其记录事件的组织以及该组织内的分类和子分类，以帮助对事件进行分类。此外，还可以记录常规信息，例如：

• 事件的简要和详细描述
• 优先级
• 发生事件的日期和时间
• 发现区域

组织默认为主要组织（作为登录员工），并且只有具有直接角色或通过传播角色的组织才可供选择。

输入分类和常规信息

分类和子分类组合在“组织”建模对象中的“事件分类规范映射”表格中定义。当仅为组织定义一个组合时，应用程序默认为一个分类和子分类。

可以在保存事件之前更改分类和子分类，但将丢失在此页面上输入的其他数据。

在记录建模中所创建的事件时，可使用优先级。

“发现区域”字段提供一个可供选择的区域列表，例如先前在建模中定义的“企业”、“工厂”和“工作中心”。此列表以层次结构视图的形式提供。还可以在此字段中输入一个新区域。

“发起者名称”字段和“组织”字段默认为登录到应用程序的用户的信息。只有具有直接角色或通过传播角色的组织才可供选择。

可以在此时保存信息，稍后返回以收集更多信息，也可以单击“下一步”前进到页面流中的下一页面。但是，保存事件之后，将无法更改分类和子分类。

文本编辑器

Siemens 提供文本编辑器，用于在记录事件时设置描述的格式。该文本编辑器支持常见的文本格式设置功能，例如，字体大小和颜色、字体样式、文本对齐、项目符号和编号列表、高亮显示、插入图像等。有关使用文本编辑器插入图像的信息，请参考《Opcenter EX MDD Modeling 用户指南》或《Opcenter EX CR Modeling 用户指南》。

Siemens 建议高亮显示现有描述，然后将格式设置应用于选定文本。如果未选择任何格式设置属性，则应用程序将应用默认字体大小、样式和颜色。应用程序将显示查看记录时所应用的格式设置。

可以在“设计”或 HTML 模式下使用该文本编辑器。在“设计”模式下复制文本时，不会复制隐藏的标记。因此，粘贴复制的文本时，可能不会保留格式设置。但是，如果切换到 HTML 视图，则可以复制隐藏的标记。

“指示”字段的字符数上限为 4,000，包括嵌入的 HTML 代码。从其他应用程序（例如 Microsoft Word）复制文本并将其粘贴到此字段中时，除了实际文本外，还会将隐藏的 HTML 代码粘贴到字段中。因此，即使文本包含的字符少于 4,000 个，也可能超出限制。

注释: Google Chrome 不支持文本编辑器的“复制”和“粘贴”按钮。使用 Google Chrome 时，可以使用键盘快捷方式进行复制和粘贴（Ctrl+C 和 Ctrl+V）。

设计模式

设计模式为 WYSIWYG（所见即所得）文本编辑器，该编辑器显示的指示即为将在查看记录时看到的指示。

HTML 模式

单击 HTML 按钮可显示应用于指示的源代码。由于 WYSIWYG 编辑器可能会自动插入标记代码（标记），因此如果需要，在 HTML 模式下查看指示可以验证和修改此代码。

如何输入分类和描述信息

按照以下步骤输入分类和描述信息：

1. 从事件菜单中选择记录通用事件。将显示“记录通用事件：分类”页。
2. 从列表中选择组织或接受默认值。
3. 选择分类。应用程序会相应地更新“子分类”列表。
4. 选择子分类。
5. 输入事件的简要描述和描述。
6. 输入或选择优先级。
7. 在发生日期和时间字段中选择事件发生的日期和时间（如果其与默认值不同）。
8. 输入或选择发现区域。
9. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
10. 单击下一步移动到页面流中的下一页。将显示“失败”页。
还可以单击保存以在移动到下一页面之前保存更改。
