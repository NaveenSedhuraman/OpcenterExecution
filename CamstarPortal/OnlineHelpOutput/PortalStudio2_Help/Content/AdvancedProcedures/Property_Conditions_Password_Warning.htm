Enabling Password Warning Messages

You can enable warning messages to alert Portal users of a blank password field by modifying a password control's property conditions in Portal Studio.

When Enabling Password Warning Messages

Warning messages will alert a Portal user of missing information in required fields, but will not include the password field in a warning message by default. An error message will display for a blank password field only when data has been entered in all other required fields. Warning messages can only be enabled on pages within the Modeling framework.

The StaticValue field under the RightValue properties must be left blank in order to enable password warning messages.

How to Enable Password Warning Messages

Follow these steps to enable password warning messages:

1. Access Portal Studio.
2. Click the Show/Hide Catalog button if the Components catalog is hidden. The Components catalog appears.
3. Double-click any Portal Modeling page with a password control. The page appears.
4. Click Show/Hide Properties if the Properties pane is hidden. The Properties pane appears.
5. Select the password control on the page, such as ObjectChanges_Password.
6. Click More for PropertyConditions in the Properties pane. The PropertyConditions pop-up appears.
7. Click Add. A new property condition is added to the list and its general properties appear.
8. Select Required for Property.
9. Click the Expand button for Condition under General. The properties for Condition appear.
10. Click the Expand button for LeftValue under Equivalency. The properties for LeftValue appear.
11. Select PageDataContract for Connection Type.
12. Enter SelectedInstanceRef in the PageDataContract field.
13. Click the Expand button for RightValue under Equivalency. The properties for RightValue appear.
14. Select StaticValue for Connection Type.
15. Click OK. The PropertyConditions pop-up closes.
16. Click Save. The application saves your changes and displays a success message.
