Changing the Target Host

[Remove file?]Portal Studio allows you to access and work on items hosted on a different Portal Studio compatible system.

When Changing the Target Host

If the target host is on a different domain than the current host, you must create a security trust for Silverlight to access the other domain.

This is a sample security trust:

<?xml version="1.0" encoding="utf-8"?>

<access-policy>

<cross-domain-access>

<policy>

<allow-from http-request-headers="*">

<domain uri="*"/>

</allow-from>

<grant-to>

<resource path="/" include-subpaths="true"/>

</grant-to>

</policy>

</cross-domain-access>

</access-policy>

How to Change the Target Host

Follow these steps to change the target host:

1. Access Portal Studio.
2. Click Settings and select Change Target Host. You are logged out of Portal Studio and a logon page appears.
3. Enter the User Name, Password, and Domain for the target host.
4. Click Log On. You are logged into the new target host.
