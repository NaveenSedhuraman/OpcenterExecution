Enabling Page Actions

Page actions are links and buttons that perform an action when clicked. Page actions can be enabled on any page from the Properties pane.

When Enabling Page Actions

Page actions must first be enabled on the DynamicActions pop-up before they will appear on a page.

Page actions are either disabled or hidden when page action rules are not satisfied. Disabled page actions are visible but cannot be clicked on. Hidden page actions do not appear on a page. Page actions are set to hidden by default. You can modify the DisabledDisplayOption on the DynamicActions pop-up so that actions with failed rules are disabled. Refer to "Page Properties Descriptions" for information on the fields that appear on the DynamicActions pop-up.

How to Enable Page Actions

Follow these steps to enable page actions:

1. Access Portal Studio.
2. Double-click a page in the Components catalog. The page appears in the working area.
3. Click More for Dynamic Actions in the Properties pane. The DynamicActions pop-up appears.
4. Select True from the ActionsRequired field.
5. Click Ok. The DynamicActions pop-up closes.
6. Click Save. The application saves your changes and displays a success message.

How to Set Page Actions as Disabled

Follow these steps to set page actions as disabled:

1. Access Portal Studio.
2. Double-click a page in the Components catalog. The page appears in the working area.
3. Click More for Dynamic Actions in the Properties pane. The DynamicActions pop-up appears.
4. Select Disable from the DisabledDisplayOptions field.
5. Click Ok. The DynamicActions pop-up closes.
6. Click Save. The application saves your changes and displays a success message.
