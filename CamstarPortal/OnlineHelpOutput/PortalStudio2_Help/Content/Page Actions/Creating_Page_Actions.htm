Creating Page Actions

You can create page actions in Portal Studio. A page action's properties and the function the action performs when you click on it are determined by its type of user-interface action (UIAction).

When Creating Page Actions

Page actions are configured using page action properties, and are further configured by page action rules and source pages.

Page Action Rules determine when a page action should be available on a page using customized conditions. Refer to "Creating Page Action Rules" for information on configuring page action rules.

Source Pages are a list of pages where the action will appear when the conditions of the page action rules are met. Source pages must be defined for a page action to appear on a page. Refer to "Adding Source Pages to Page Actions" for information on adding pages to source page lists.

Page actions are configured to appear as either a link or a button. Page actions are set as links by default when added to a page using the ActionsControl web part. You can modify the ActionStyle property of the ActionsControl web part so that actions appear as buttons, or add page actions directly to pages as buttons. Refer to "Configuring Page Actions" for information.

Important: You cannot change the UIAction type once you have selected the UIAction type and saved the page action.

Opening Existing Page Actions

Page actions are available from the Modeling tab of the UI Components catalog, with many already created by default. Refer to “Opening Existing Components” for information on opening existing page actions.

How to Create Page Actions

Follow these steps to create page actions:

1. Access Portal Studio.
2. Click New and select Page Action. A blank page action appears in the working area.
3. Enter a name for the page action in the Name field.  
4. Click More for UIAction. The UIAction pop-up appears.
5. Select a UI Action type from the drop-down at the top of the pop-up. The fields are reset based upon the UI Action type selected.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Ok. The UIAction pop-up closes.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Save. The application saves your changes and displays a success message.
