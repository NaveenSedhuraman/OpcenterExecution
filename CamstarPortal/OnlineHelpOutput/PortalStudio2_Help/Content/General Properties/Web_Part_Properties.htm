Web Part Properties

Web part properties allow you to view or edit the properties of web parts. Web part properties appear in the Properties pane when a web part is selected in the working area. Refer to "Properties Pane" for information on the Properties pane.

A description window displaying information about web part properties appears beneath the Properties pane when a property is selected.

Note: Some properties contain sub-properties allowing for further customization. Sub-properties appear in a separate pop-up when you click the More button for their parent property.

Web Part Properties Descriptions

This table provides the descriptions of web part properties.

Web Part Property

Property Description

Actions

Actions of the web part. Allows for definition of web part level actions and buttons.

Note: Clicking the More button displays the Actions modal. Refer to "Actions Properties Descriptions" for information on the properties that appear on this modal.

BroadcastInitiators

Used to create initiators that will trigger page level connection data contracts between web parts.

Note: Clicking the More button displays the BroadcastInitiators modal. Refer to "BroadcastInitiators Properties Descriptions" for information on the properties that appear on this modal.

ClientPropertyConditions

Allows for setting data to a property based on a condition result from an event on browser side.

CollapsibleGroups

Will be deprecated in an upcoming release.

Note: You must create collapsible sections using the ToggleDisplayMode property of the Toggle Container control. Refer to "Control Properties" for information.

Control

References a control on the page or web part.

Note: Clicking the More button displays the Control modal. The Control modal shares properties with controls. Refer to "Control Properties" for information on the properties that appear on this modal.

Created

Creation date and time of the web part.

CreatedBy

Creator of the web part.

CSSClass

Stylesheet reference for the web part.

DataContractMembers

Data Contracts used for communication between pages.

Note: Clicking the More button displays the DataContractMembers modal. Refer to "DataContractMembers Properties Descriptions" for information on the properties that appear on this modal.

DataSubmissionMode

How fields react when a page submits data to the server. Options include:

• NotSet: Mode is not set. This is the default mode.
• Force: Value is always included in submission data.
• Changed: Value is only submitted if it was changed.
• Filled: Value is only submitted if the field is not empty.
• ChangedAndFilled: Value is only submitted if the field is not empty and was changed.
• Skip: Value is not submitted or loaded.
• ViewOnly: Value is not submitted.
• Skip Request: Value is not loaded. The selection values for the control are not loaded.

Important: Siemens recommends the Changed option for editable grids on Modeling pages.

DirtyFlagTrigger

Boolean field that marks the page as dirty if any field within the web part changes value when set to True.

Note: Currently this property only affects web parts within the modeling framework.

DisplayBorder

Displays a border on the web part.

DisplayFooter

Displays a footer on the web part.

DisplayHeader

Displays a header on the web part.

EventHandlers

Allows for specification of a handler from an event.

Note: Clicking the More button displays the EventHandlers modal. Refer to "Defining Event Handlers" for information on defining event handlers.

Height

Height in pixels for the display of the web part.

Hidden

If True, the control or web part will render in the HTML but not be visible to the user. For example, display "none" in the HTML tag.

HorizontalAlignment

Horizontal alignment of web part elements.

HowLong

Time period since creation of the web part.

LastModified

Last modified date of the web part.

LastModifiedBy

User who modified the web part most recently.

Layout

Visual appearance of the web part. Siemens recommends that you do not change this property directly.

Margin

The margin of the web part.

Padding

The padding of the web part.

PrimaryServiceType

Service type for controls that do not specify a service type.

PropertyConditions

Allows definition of a web part's property value as a result of an evaluated expression (condition). The property is evaluated at Page execution run-time.

Note: Clicking the More button displays the <Component> - Property Conditions window. Refer to "Property Conditions" for information on defining property conditions.

SessionVariables

Used to configure page-level session variables.

Refer to "Defining Session Variables" for information on defining session variables.

Title

Text that appears as the title of the web part. Text entered here takes priority over the TitleLabel field when both text is entered and a label is selected.

TitleLabel

Selects a label to be used as the title of the web part. Refer to "Searching for Page Action Labels" for information on using the Label Lookup pop-up.

TypeName

Code behind the type of the web part.

VerticalAlignment

Vertical alignment of web part elements.

Width

Width in pixels for the display of the web part.
