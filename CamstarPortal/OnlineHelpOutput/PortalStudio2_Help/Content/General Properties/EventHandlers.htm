Defining Event Handlers

Event handlers allow you to configure actions to be taken based on events, and allow you to specify custom conditions.

When Defining Event Handlers

Event Handlers are object specific and are either generic or conditional. Generic Event Handlers configure actions that occur based upon predefined events. Conditional Event Handlers configure actions and allow for the configuration of additional conditions using comparisons between left and right values in addition to an event.

The Condition field and its associated fields only appear on the EventHandlers pop-up when defining conditional event handlers.

You can optionally define additional parameters and initiators from the EventHandlers pop-up. Refer to "Parameters Pop-up Field Definitions" and "Initiators Pop-up Field Definitions" for information on the fields used to define parameters and initiators.

The fields used to define left and right values in conditional event handlers are dynamic based upon the connection type selected in the ConnectionType field. The connection type determines what additional fields will appear.

How to Define a Generic Event Handler

Follow these steps to define a generic event handler:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the Components Catalog is hidden. The Components Catalog appears.
3. Double-click a page. The page appears in the working area.
4. Select a virtual object on the page. The properties for that object appear in the Properties pane.
5. Click the More button for EventHandlers in the Properties pane. The EventHandlers pop-up appears.
6. Select GenericEventHandler from the drop-down above the left pane and click Add. Blank fields appear in the right pane.
7. Select an event from the Event field.
8. Select a handler source from the HandlerSource field.
9. Select a handler from the Handler field.
10. Enter a name for the event handler in the Name field.
11. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
12. Click Save. The application saves your changes and displays a success message.

How to Define a Conditional Event Handler

Follow these steps to define a conditional event handler:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the Components Catalog is hidden. The Components Catalog appears.
3. Double-click a page. The page appears in the working area.
4. Select a virtual object on the page. The properties for that object appear in the Properties pane.
5. Click the More button for EventHandlers in the Properties pane. The EventHandlers pop-up appears.
6. Select ConditionalEventHandler from the drop-down above the left pane and click Add. Blank fields appear in the right pane.
7. Select an event from the Event field.
8. Select a handler source from the HandlerSource field.
9. Select a handler from the Handler field.
10. Enter a name for the event handler in the Name field.
11. Expand the Condition section.
12. Expand the Equivalency section. 
13. Select a comparison operator from the Comparison field.
14. Expand the LeftValue section.
15. Select a connection type from the ConnectionType field. The available LeftValue fields are refreshed based upon your selection.
16. Select a virtual object from the field that appeared based upon the selected connection type.
17. Repeat steps 14-16 for the RightValue fields.
18. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
19. Click Save. The application saves your changes and displays a success message.
