Configuring Portal Settings

Portal Studio allows you to view, change, and save the following settings:

• Default
• Studio
• Domain
• Theme
• Language
• Time Zone
• Current Culture
• Grid
• Search
• Smart Scanning
• Caching
• Intelligence
• CDO Forms
• Export Groups
• Status Message
• Template
• Appearance

When Configuring Portal Settings

Portal Studio setting are saved at the following file path:

C:\Program Files (x86)\Camstar\Camstar Portal\User\Settings.xml

Note: You can modify the default timeout by entering a new value for the Session State property in the web.config file.

DefaultTheme

Determines whether the default theme for the Portal uses Classic or Horizon stylesheets. Set to Horizon by default.

Horizon is the default theme if you are logging on to the Portal using the Default or Mobile entry mode. Classic is the default theme if you are using the Classic entry mode. You should not change the default theme after logging in to the Portal using one of these entry modes. The appropriate default theme is intended to be used with each entry mode. Refer to the Opcenter Execution Medical Device and Diagnostics Getting Started User Guide or the Opcenter Execution Core Getting Started User Guide for information.

Note: By default, the text on all shop floor pages displays in a larger font size if DefaultTheme is set to Horizon and you are viewing the pages on a computer. All Modeling, Change Management, and Export/Import pages display in the standard font size by default. The font size at the page-level is set using the CssClass page property.

DisplayDropdownIconWhenReadOnly

Display the Drop Down icon when the field is Read Only? True or False.

DisplayUserDCDTooltips

Display User Data Collection Tooltips? True or False.

EnforceCheckOutWhenEditing

Requires a page or web part to be checked out in portal studio before it can be saved. Typically used in multi-developer environments.

ESignatureRequired

Global ESignature default setting. Page setting overrides the global setting.

FullScreenMode

Displays the application's framework in full screen mode when set to True. Set to False by default.

Note: This setting expands these application framework elements to the full browser width:

• global footer
• global header
• tab row
• menu bar

You can set full screen mode on individual pages and hide page elements using page properties. Refer to "Page Properties Descriptions" for information on page properties.

HelpPage

Note: You may need to increase the values in the following settings to allow for enough time when importing or exporting larger files:

• SendTimeout and ReceiveTimeout values in the Portal web.config and WCF Services web.config files
• HttpTimeout and HttpReadWriteTimeout in the XMLClientConfig.xml file

MaxDataTransferRefreshAttempts

Specifies the maximum number of attempts for a data transfer refresh.

Current Culture Settings

The Current Culture Settings configure the date, time and number format for the Portal.

This table defines the fields on the Portal Settings pop-up with CurrentCultureSettings selected.

Field

Definition

DateTimeFormat

Allows you to set the date and time format for the Portal. This affects how the date and time are displayed on pages, in the success message, and how the transaction date is displayed on the Audit Trail.

This setting overrides any settings made in the browser language.

DateTimeFormat - General

LongTimePattern

Time pattern that overrides browser language settings when displaying time. This setting uses Microsoft’s standard date and time formats. Refer to https://msdn.microsoft.com/en-us/library/8kb3ddd4(v=vs.110).aspx?cs-save-lang=1&cs-lang=vb#code-snippet-2 for information on Microsoft date and time formats.

Entering a custom format overrides time displays across the application, such as timestamps and server messages. This setting does not modify the time format for controls that have a custom time format set for the control's Custom Format property. Refer to "Control Properties" for information on the Custom Format property.

ShortDatePattern

Date pattern that overrides browser language settings for date display. This setting uses Microsoft’s standard date and time formats. Refer to https://msdn.microsoft.com/en-us/library/8kb3ddd4(v=vs.110).aspx?cs-save-lang=1&cs-lang=vb#code-snippet-2 for information on Microsoft date and time formats.

Note: This setting does not modify the date format for controls that have a custom date format set for the control's Custom Format property. Refer to "Control Properties" for information on the Custom Format property.

NumberFormat

Allows you to set the number format for the Portal.

This setting overrides any settings made in the browser language.

NumberFormat - General

NumberDecimalSeparator

Decimal separator character that will override browser language setting.

How to Configure Portal Settings

Follow these steps to configure Portal Settings:

1. Access Portal Studio.
2. Click the Portal Settings button.The Portal Settings pop-up appears.
3. Select the setting you want to view or change. Properties for that setting are displayed.
4. Select a setting and make the necessary changes.
5. Click OK. The Portal Settings pop-up closes and saves your changes.
