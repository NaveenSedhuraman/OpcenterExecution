Defining a Parametric Data Collection

You can define a parametric data collection using a web part.

A Data Collection Definition references a parametric data definition that contains a set of data collection parameters. The data collection parameters are presented to users during transaction processing. The data collection definition and the transaction with which it is used are identified in the Specification definition (in Modeling) associated with the step where the data is collected.

When Defining a Parametric Data Collection

Data Collections must be defined in Modeling prior to using them in Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Modeling User Guide or the Opcenter Execution Core Portal Modeling User Guide and Opcenter Execution Medical Device and Diagnostics Data Collection User Guide or the Opcenter Execution Core Data Collection User Guide for information.

How to Define a Parametric Data Collection for a Web Part

Follow these steps to define a parametric data collection for a web part:

1. Access Portal Studio.
2. Click New and select Web Part. A blank web part appears in the working area.
3. Click Show/Hide Properties if the Properties pane is hidden. The Properties pane appears.
4. Enter a name for the web part in the Name field of the Properties pane.
5. Click Show/Hide Toolbox if the Toolbox is hidden. The Toolbox appears.
6. Click the Fields Directory tab. A list of services appears in the toolbox pane.
7. Select the field you want to add and drag it onto a cell on the web part.
8. Repeat step 7 for all fields in the data collection.
9. Click Save. The application saves your changes and displays a success message.
