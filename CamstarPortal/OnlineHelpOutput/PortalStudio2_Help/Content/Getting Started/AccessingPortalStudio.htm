Accessing Portal Studio

You can access Portal Studio from the Portal or as a standalone application.

If you are using the Portal, you access the Studio differently depending on the theme you used to access the Portal. Horizon is the default theme if you log in to the Portal using the Default mode. Classic is the default theme if you use the Classic entry mode.

Portal Studio 2.0 is the default version of Portal Studio for Opcenter Execution Core Version 8 and higher regardless of entry mode. Refer to the Opcenter Execution Medical Device and Diagnostics Getting Started User Guide or the Opcenter Execution Core Getting Started User Guide for information on entry modes.

You can access Portal Studio as a standalone application by entering a URL. The Log In page appears if you access Portal Studio directly. The Log In page for Portal Studio is similar to other Log In pages for Opcenter Execution Core except that it does not contain the Options icon because it is not applicable. Refer to the Opcenter Execution Medical Device and Diagnostics Getting Started User Guide or the Opcenter Execution Core Getting Started User Guide for information on logging in.

Important: Cookies must be enabled when accessing Portal Studio.

When Accessing Portal Studio

You access Portal Studio from the Portal by clicking the:

• Portal Studio icon on the Primary Navigation bar in the Portal if you are using the Horizon theme.
• Studio button in the global header of the Portal if you are using the Classic theme.

An error message appears preventing you from accessing Portal Studio if CSI is the highest active workspace. You must activate another workspace in Management Studio and then access Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on activating workspaces.

Note: The Portal does not display an error message if you are already logged into the Portal and CSI is then made the highest active workspace. An error message appears preventing you from accessing Portal Studio after you log out and then log back into the Portal.

Start Page Dashboard

A Start Page dashboard appears in the working area when accessing Portal Studio. The Start Page dashboard provides links to commonly used components and information on using Portal Studio.

The Start Page dashboard contains these sections:

• Create New, which provides links to create new components such as pages, web parts, and pageflows.
• Getting Started, which provides a link to information on using Portal Studio.
• Recent, which provides links to open the five most recently opened components.

The Start Page dashboard remembers the five most recently opened components and will update the list as you open components. An error message appears if you click the link for a component in the Recent list that has been deleted since accessing Portal Studio. Closing the error message removes the component from the list and replaces it with another recently opened component. No components will appear in the Recent list the next time you access Portal Studio if all ten recently opened components are deleted.

You can return to the Portal by clicking the Back to Portal button. Portal Studio prompts you to save your changes before returning you to the Portal.

Accessing Portal Studio 1.0

If necessary, you can access Portal Studio 1.0—the version of Portal Studio that was the default for Opcenter EX MDD and Opcenter EX CR for 8.1 and earlier. The PortalStudioVersion setting in the Portal web.config file controls the version available when accessing Portal Studio using Horizon or Classic themes. Select 1 for Portal Studio 1.0.

Note: Siemens recommends using the Microsoft Internet Explorer browser when accessing Portal Studio 1.0. Portal Studio 2.0 is the default version for the Chrome browser.

You can also access both versions of Portal Studio in separate browsers. Press the Ctrl key while clicking the:

• Portal Studio icon for the Horizon theme.
• Studio button for the Classic theme.

This opens the version of Portal Studio that is not configured in the web.config file.

Session Expiration

Portal Studio does not enforce session expiration due to timeout. Portal Studio sessions are tied to the Portal and will only expire when an instance of the Portal expires. Portal sessions expire after a default 20 minute inactivity period.

Note: You can change the default timeout value for the Portal in the web.config file located at C:\Program Files (x86)\Camstar\Camstar Portal by default.

Portal Studio does not display a session expiration warning message when session expiration occurs. Any unsaved changes will be lost and you must log back in to the Portal and access Portal Studio when session expiration occurs.

How to Access Portal Studio as a Standalone Application

Follow these steps to access Portal Studio as a standalone application:

1. Enter this URL:

https://localhost/CamstarPortal/PortalStudio.aspx The Log In page appears.

2. Enter your Username and Password.
3. Select a different Domain if necessary.
4. Click Log In. Portal Studio and the Start Page dashboard appear.

How to Return to the Portal

Follow these steps to return to the Portal:

1. Access Portal Studio.
2. Perform any necessary work.
3. Click the Back to Portal button. Portal Studio closes and you are returned to the Portal.
