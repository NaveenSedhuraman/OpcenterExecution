Portal Studio User Interface Features

Portal Studio's user interface features standard buttons and icons throughout the application's panes and pop-ups.

Standard Buttons

Standard buttons are used for consistency throughout the application's user interface to enhance your user experience and streamline your data entry tasks.

This table defines the common buttons in Portal Studio.

Searching for Components in a Pane

Panes in Portal Studio provide a search field that enables you to find specific components, fields, and properties in that catalog or directory. Enter the first few characters or the entire name in the search field to search for a specific component, field, or property.

The search field filters the list of returned components as you type. The search field assumes a wildcard on each end of the entry, enabling you to find all components that contain the phrase or partial phrase in their name.

Viewing Fields and Properties

Portal Studio displays fields and properties in three views:

• Favorites, which displays a list of fields and properties you selected as favorites.
• Sorted, which displays a list of all fields and properties in alphabetical order.
• Grouped, which displays all fields and properties organized into categories based on their function, such as, Behavior, Appearance, or Layout.

You can cycle between views at any time by selecting a view tab. Panes display fields and properties in the Sorted view by default.

Adding Fields and Properties to Favorites

You can add fields and properties to your Favorites view by pressing the Ctrl key while clicking the field or property name. Field and property names appear blue when added to your favorites list.

Note: Fields and properties persist as favorites only on the client machine where they were set.
Navigating Modals

Portal Studio uses modals to configure settings that require adding instances. Modals feature two panes:

• A left pane that lists previously created instances.
• A right pane that provides detailed information and properties for customizing the selected instance.

All modals contain an Add button that allows you to add a new instance to the list of instances in the left pane. Clicking OK saves any changes you made and closes the modal. Clicking Cancel discards your changes and closes the modal.

Boolean Values

Some fields and properties appear as drop-downs containing these Boolean values:

• Not Set
• True
• False

Many Boolean drop-downs use "Not Set" by default. A value of Not Set triggers the same functionality as setting the field or property to "False." Siemens recommends leaving fields and properties as "Not Set" unless a business need requires you to set the field to "False."
