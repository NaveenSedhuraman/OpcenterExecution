Reverting Content

The Revert To button allows you to rollback pages and web parts in the highest active workspace to their last published state or to the state of the system content found in the base Siemens PLM CEP (CSI) workspace.

When Reverting Page and Web Part Content

The Revert To button is disabled in the base Siemens PLM CEP (CSI) workspace and is only enabled on pages and web parts with changes made in the highest active workspace. Refer to "Developing in Workspaces" for information on saving and publishing changes in workspaces.

Clicking Revert To and selecting Published content reverts the page or web part back to its last published state. Clicking Revert To and selecting System content reverts the page or web part to match the component found in the base Siemens PLM CEP (CSI) workspace.

How to Revert Pages And Web Parts to the Last Published State

Follow these steps to revert pages and web parts to their last published state:

1. Access Portal Studio.
2. Open a page or web part with published changes. The page or web part appears in the working area.
3. Click Revert To and select Published content. The Revert to Last Published dialog appears.
4. Click OK. The Revert to Last Published dialog closes and the page or web part refreshes displaying the content for the last published state.

How to Revert Pages And Web Parts to the System Content

Follow these steps to revert pages and web parts to the system content:

1. Access Portal Studio.
2. Open a page or web part that has been modified. The page or web part appears in the working area.
3. Click Revert to... and select System content. The Revert to System Content dialog appears.
4. Click OK. The Revert to System Content dialog closes and the page or web part refreshes displaying the content found in the base Siemens PLM CEP (CSI) workspace.
