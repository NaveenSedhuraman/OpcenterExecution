Publishing Content

Publishing a page or web part pushes your changes to your highest active workspace and makes your changes available outside of your Developer workspace in the Portal.

When Publishing Content

The Publish button is enabled when a page or web part is saved or when no unsaved changes are present. Unpublished pages and web parts are indicated by a pencil icon on the component's icon in the Components catalog.

The Publish button is only enabled in the highest active and Developer workspaces. Saved changes published in the Developer workspace are pushed to the highest active workspace. Refer to "Developing in Workspaces" for information.

You can revert a page or web part to its last published state by clicking Revert To and selecting Published. Refer to "Reverting Content" for information.

The application displays a Publishing dialog prompting you to remove all personalization associated with the component you are publishing when you click Publish. Selecting the Remove All personalization associated with this component check box and clicking OK removes all personalization and continues the publishing process. Clicking OK without selecting the check box continues the publishing process. Clicking Cancel aborts the publishing process.

A page permission is automatically created and added to the Default Pages role when publishing a new page. A page permission is not created when publishing a page that has previously been published. You can set the AutoPagePermission property in the DefaultSettings of the Portal Settings window to False to prevent the application from adding new page permissions by default. Refer to "Configuring Portal Settings" for information.

Note: You must log out of and then log back in to the Portal before attempting to run a newly published page or view a page permission on the Default Pages role after publishing a new page. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Modeling User Guide or the Opcenter Execution Core Portal Modeling User Guide for information on roles.

How to Publish Pages and Web Parts

Follow these steps to publish pages and Web Parts:

1. Access Portal Studio.
2. Open a page or web part. The page or web part appears in the working area.
3. Make any necessary changes.
4. Click Save. The application saves your changes and displays a success message.
5. Click Publish. The Publishing dialog appears.
6. Click OK. The Publishing dialog closes, the page or web part is published to the highest active workspace, and the application displays a success message.
