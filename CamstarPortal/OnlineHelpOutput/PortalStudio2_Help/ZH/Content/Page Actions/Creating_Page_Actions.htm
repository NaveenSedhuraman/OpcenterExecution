创建页面操作

可以在 Portal Studio 中创建页面操作。页面操作的属性及用户单击该操作时将执行的功能由其用户界面操作 (UIAction) 的类型确定。

在创建页面操作时

页面操作通过页面操作属性来配置，并通过页面操作规则和源页面进一步配置。

页面操作规则使用自定义的条件来确定页面操作应何时在页面上可用。有关配置页面操作规则的信息，请参考“创建页面操作规则”。

源页面是页面的列表，这些页面中的操作将在页面操作规则条件得到满足时显示。必须为页面操作定义将显示在页面上的源页面。有关将页面添加到源页面列表的信息，请参考“向页面操作添加源页面”。

页面操作配置为以链接或按钮的形式显示。在使用 ActionsControl Web 部件将页面操作添加到页面时，它默认设为链接。可以修改 ActionsControl Web 部件的 ActionStyle 属性，使得操作显示为按钮，或者将页面操作作为按钮直接添加到页中。有关信息，请参考“配置页面操作”。

重要提示: 选择了 UIAction 类型并且保存了页面操作，就不能更改 UIAction 类型。

打开现有页面操作

页面操作可从“用户界面组件”目录的“建模”选项卡获得，默认情况下多数操作均已创建。有关打开现有页面操作的信息，请参考“打开现有组件”。

如何创建页面操作

按照以下步骤创建页面操作：

1. 访问 Portal Studio。
2. 单击新建，然后选择页面操作。随即工作区域中将显示一个空白页面操作。
3. 在名称字段中输入页面操作的名称。
4. 单击 UIAction 对应的“更多”。将显示 UIAction 弹出窗口。
5. 从弹出窗口顶部的下拉列表中选择一种用户界面操作类型。将根据选定的用户界面操作类型重置字段。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击确定。UIAction 弹出窗口将关闭。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击保存。应用程序将保存更改并显示成功消息。
