访问 Portal Studio

可以从 Portal 访问 Portal Studio，也可以作为独立应用程序访问。

如果使用 Portal，则访问 Studio 的方式因用于访问 Portal 的主题而异。如果使用默认模式登录门户，则默认主题为水平。如果使用经典进入模式，则默认主题为经典。

对于 Opcenter Execution Core 版本 8 和更高版本，无论使用哪种进入模式，Portal Studio 2.0 都是 Portal Studio 的默认版本。有关进入模式的信息，请参考《Opcenter Execution Medical Device and Diagnostics 入门用户指南》或《Opcenter Execution Core 入门用户指南》。

通过输入 URL，可以作为独立应用程序访问 Portal Studio。如果直接访问 Portal Studio，则会显示登录页面。Portal Studio 的登录页面与 Opcenter Execution Core 的其他登录页面类似，不同之处是前者不包含“选项”图标，因为该图标不适用。有关登录的信息，请参考《Opcenter Execution Medical Device and Diagnostics 入门用户指南》或《Opcenter Execution Core 入门用户指南》。

重要提示: 访问 Portal Studio 时，必须启用 Cookie。

在访问 Portal Studio 时

可以从 Portal 单击以下按钮访问 Portal Studio：

• 如果使用的是水平主题，则单击 Portal 中主导航栏上的 Portal Studio 图标。
• 如果使用的是经典主题，则单击 Portal 全局页眉中的 Studio 按钮。

如果 CSI 是最活跃的工作区，则会出现一条错误消息，阻止您访问 Portal Studio。必须在 Management Studio 中激活另一个工作区，然后再访问 Portal Studio。有关激活工作区的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

注释: 如果已经登录 Portal，CSI 然后才成为最活跃的工作区，则 Portal 不会显示错误消息。在注销并重新登录 Portal 后，将出现一条错误消息，阻止您访问 Portal Studio。

“启动页面”仪表板

当访问 Portal Studio 时，工作区域中将显示“起始页面”仪表板。“起始页面”仪表板提供指向常用组件的链接以及有关使用 Portal Studio 的信息。

“起始页面”仪表板包含以下部分：

• “新建”- 提供用于创建新组件（如页面、Web 部件和页面流等）的链接。
• “入门”- 提供有关使用 Portal Studio 的信息的链接。
• “最近打开”- 提供用于打开五个最近打开的组件的链接。

“启动页面”仪表板会记住五个最近打开的组件，并且每次打开组件时都会更新列表。如果在“最近打开”列表中单击自从访问 Portal Studio 后已删除的组件的链接，将显示一条错误消息。关闭错误消息将从列表中移除该组件，并将其替换为另一个最近打开的组件。如果十个最近打开的组件全部被删除，则下次访问 Portal Studio 时，“最近打开”列表中将不会显示任何组件。

通过单击“回到门户”按钮，可以返回到 Portal。在返回到 Portal 之前，Portal Studio 会提示您保存所做更改。

访问 Portal Studio 1.0

如果需要，可以访问 Portal Studio 1.0（Opcenter EX MDD 和 Opcenter EX CR 8.1 及更低版本的默认 Portal Studio 版本）。Portal web.config 文件中的 PortalStudioVersion 设置控制使用水平主题或经典主题访问 Portal Studio 时的可用版本。选择 1 表示 Portal Studio 1.0。

注释: Siemens 建议使用 Microsoft Internet Explorer 浏览器来访问 Portal Studio 1.0。对于 Chrome 浏览器，默认版本是 Portal Studio 2.0。

也可以在单独的窗口中访问 Portal Studio 的两个版本。按 Ctrl 键并单击以下用户界面元素：

• Portal Studio 图标（对于水平主题）。
• Studio 按钮（对于经典主题）。

这将打开未在 web.config 文件中配置的 Portal Studio 版本。

会话过期

由于超时，Portal Studio 不会强制执行会话过期。Portal Studio 会话与门户绑定，并且仅在门户的实例到期时才会到期。门户会话在默认的 20 分钟非活动期后到期。

注释: 可以在默认位于 C:\Program Files (x86)\Camstar\Camstar Portal 的 web.config 文件中更改门户的默认超时值。

在会话过期时，Portal Studio 不会显示会话过期警告消息。在会话过期时，任何未保存的更改都将丢失，用户必须重新登录到门户和访问 Portal Studio。

如何从 Portal 访问 Portal Studio

按照以下步骤访问 Portal Studio：

1. 登录 Portal。
2. 当前正在使用哪个主题？

如果使用 ...

操作步骤 ...

水平

单击 Portal 中主导航栏上的 Portal Studio 图标。将显示 Portal Studio 和“启动页面”仪表板。

经典

单击 Portal 全局页眉中的 Studio。将显示 Portal Studio 和“启动页面”仪表板。

如何作为独立应用程序访问 Portal Studio

按照以下步骤作为独立应用程序访问 Portal Studio：

1. 输入以下 URL：

https://localhost/CamstarPortal/PortalStudio.aspx 将显示登录页面。

2. 输入用户名和密码。
3. 如果需要，选择不同的域。
4. 单击登录。将显示 Portal Studio 和“启动页面”仪表板。

如何返回 Portal

按照以下步骤返回 Portal：

1. 访问 Portal Studio。
2. 执行任何必要的工作。
3. 单击返回门户按钮。Portal Studio 关闭，您将返回到 Portal。
