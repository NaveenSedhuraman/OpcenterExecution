发布内容

发布页面或 Web 部件时，应用程序会将更改推送到最高活动工作区，并使这些更改在门户中的开发者工作区外部变为可用。

在发布内容时

当页面或 Web 部件已保存或者不存在未保存的更改时，“发布”按钮处于启用状态。已取消发布的页面和 Web 部件由“组件”目录中组件图标上的铅笔图标表示。

“发布”按钮仅在最高活动和“开发人员”工作区中启用。在开发者工作区中发布的已保存更改将推送到最高活动工作区。有关信息，请参考“在工作区中进行开发”。

单击“恢复为”并选择”已发布”可以将页面或 Web 部件恢复为其上次发布的状态。有关信息，请参考“恢复内容”。

应用程序显示“发布”对话框，提示在单击“发布”时移除与要发布的组件相关联的所有个性化设置。选中“移除与此组件关联的所有个性化设置”复选框并单击“确定”将移除所有个性化设置并继续发布流程。单击“确定”而不选中该复选框将继续发布流程。单击“取消”将中止发布流程。

发布新页面时，将自动创建页面权限并将其添加到“默认页面”角色。发布先前已发布的页面时，将不会创建页面权限。可以将“门户设置”窗口中 DefaultSettings 的 AutoPagePermission 属性设为 False，以防止应用程序默认添加新页面权限。有关信息，请参考“配置门户设置”。

注释: 在尝试运行新发布的页面或在发布新页面后查看“默认页面”角色的页面权限之前，必须先注销然后重新登录门户。有关角色的信息，请参考《Opcenter Execution Medical Device and Diagnostics Portal Modeling 用户指南》或《Opcenter Execution Core Portal Modeling 用户指南》。

如何发布页面和 Web 部件

按照以下步骤发布页面和 Web 部件：

1. 访问 Portal Studio。
2. 打开一个页面或 Web 部件。随即工作区域中将显示该页面或 Web 部件。
3. 进行任何必要的更改。
4. 单击保存。应用程序将保存更改并显示成功消息。
5. 单击发布。将显示“发布”对话框。
6. 单击确定。 “发布”对话框将关闭，页面或 Web 部件将发布到最高活动工作区，应用程序将显示成功消息。
