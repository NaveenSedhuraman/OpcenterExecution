Seiteneigenschaften

Über die Seiteneigenschaften (Page Properties) können Sie die Eigenschaften von Portal Studio-Seiten anzeigen oder bearbeiten. Seiteneigenschaften werden im Bereich "Eigenschaften" (Properties) angezeigt, wenn im Arbeitsbereich eine Seite ausgewählt wird. Weitere Informationen zum Bereich "Eigenschaften" (Properties) finden Sie unter "Fensterbereich "Eigenschaften" (Properties)".

Unter dem Fensterbereich "Eigenschaften" (Properties) wird ein Beschreibungsfenster mit Informationen über Seiteneigenschaften angezeigt, wenn eine Eigenschaft ausgewählt wird.

Note: Einige Eigenschaften enthalten Untereigenschaften, die die weitere Anpassung ermöglichen. Untergeordnete Eigenschaften werden in einem separaten Popup angezeigt, wenn Sie auf die Schaltfläche "Mehr" (More) für deren übergeordnete Eigenschaft klicken.

Beschreibungen der Seiteneigenschaften

Diese Tabelle enthält die Beschreibungen der Seiteneigenschaften.

Property

Beschreibung der Eigenschaft

Wird zusammen mit dem Webpart "Aktionssteuerung" (Action Controls) verwendet, um die Gruppierung von Seitenaktionen zu steuern.

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das modale Fenster "ActionGroups" angezeigt. Weitere Informationen zu den Eigenschaften, die in diesem modalen Fenster angezeigt werden, finden Sie unter "Beschreibungen der Eigenschaften des Typs "ActionGroups"".

Aktionen der Seite. Ermöglicht die Definition von Aktionen und Schaltflächen auf Seitenebene.

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das modale Fenster "Actions" angezeigt. Weitere Informationen zu den Eigenschaften, die in diesem modalen Fenster angezeigt werden, finden Sie unter "Beschreibungen der Eigenschaften des Typs "Actions"".

AllowEditCondition

Ermöglicht das Bearbeiten der Seite als Ergebnis eines bewerteten bedingten Ausdrucks.

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das Popup "AllowEditCondition" angezeigt, das Untereigenschaften für diese Eigenschaft enthält.

Eigenschaften des Popups "AllowEditCondition"

ColumnName

Name der für den Vergleich verwendeten Spalte.

Comparison

Zur Auswertung einer Bedingung verwendeter Vergleichsoperator. Gültige Einträge sind "Equal", "NonEqual" und "Contains".

Linker Wert (LeftValue)

Durch Klicken auf die Schaltfläche "Mehr" (More) wird das Popup "Wert" (Value) angezeigt. Das Popup "Wert" (Value) enthält Eigenschaften zum Definieren von Werten auf einer bestimmten Seite der Bedingungsbewertung.

RightValue

Hinweis: Die Eigenschaften für "RightValue" sind identisch mit den Feldern für "LeftValue". Weitere Informationen zu Eigenschaftendefinitionen finden Sie unter "Eigenschaften der Popups "AllowEditCondition" – "LeftValue" oder "RightValue" – "Value"".

And

Durch Klicken auf die Schaltfläche "Mehr" (More) wird das Popup "Und" (And) angezeigt. Sie können zusätzliche Bedingungen für ein bedingtes Ereignis definieren, die mit dem Operator "Und" (And) hinzugefügt werden.

Hinweis: Die Felder im Popup "Und" (And) sind identisch mit den Eigenschaften, die angezeigt werden, wenn ein bedingter Ereignishandler definiert wird. Weitere Informationen zu Eigenschaftendefinitionen finden Sie unter "Eigenschaften des Popups "AllowEditCondition"".

Or

Durch Klicken auf die Schaltfläche "Mehr" (More) wird das Popup "Oder" (Or) angezeigt. Sie können zusätzliche Bedingungen für ein bedingtes Ereignis definieren, die mit dem Operator "Oder" (Or) hinzugefügt werden.

Hinweis: Die Felder im Popup "Und" (And) sind identisch mit den Eigenschaften, die angezeigt werden, wenn ein bedingter Ereignishandler definiert wird. Weitere Informationen zu Eigenschaftendefinitionen finden Sie unter "Eigenschaften des Popups "AllowEditCondition"".

Eigenschaften der Popups "AllowEditCondition" – "LeftValue" oder "RightValue" – "Value"

Value – ConnectionType

Gibt den Verbindungstyp auf einer bestimmten Seite der Bedingungsbewertung an.

Hinweis: Es werden je nach ausgewähltem Verbindungstyp zusätzliche Eigenschaften angezeigt.

Value – Control

Gibt ein Steuerelement an, das auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "Control" ausgewählt ist.

Value – PageDataContract

Gibt einen Seitendatenvertrag an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "PageDataContract" ausgewählt ist.

Value – SessionDataContract

Gibt einen Sitzungsdatenvertrag an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "SessionDataContract" ausgewählt ist.

Value – SessionVariable

Gibt eine Sitzungsvariable an, die auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "SessionVariable" ausgewählt ist.

Value – FieldExpression

Gibt einen Feldausdruck an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "FieldExpression" ausgewählt ist.

Value – StaticValue

Gibt einen statischen Wert (StaticValue) an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "StaticValue" ausgewählt ist.

Value – GridItem

Gibt ein Tabellenelement an, das auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "GridItem" ausgewählt ist.

Value – Property

Gibt den Steuerelementeigenschaftswert, der ausgewertet wird, an.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "Control" ausgewählt ist.

Value – RequestDataMode

Gibt die Methode an, mit der der Eigenschaftswert eines Steuerelements abgerufen wird.

Hinweis: Diese Eigenschaft wird angezeigt, wenn für die Eigenschaft "ConnectionType" die Option "Control" ausgewählt ist.

Value – AllowEmptyValue

Bestimmt, ob der Wert leer sein darf.

Value – Initiators

Das Popup "Initiatoren" (Initiators) wird angezeigt, wenn Sie auf die Schaltfläche "More" (Mehr) klicken.

DefaultButton

Gibt die Schaltfläche an, die bei Drücken der Eingabetaste automatisch angeklickt wird.

Anzeige

Passt die Seitenanzeige an, indem sie als Vollbild angezeigt wird oder indem zahlreiche Komponenten der Seite ausgeblendet werden, wie z. B. die folgenden Komponenten:

• Globale Fußzeile
• Globaler Kopf
• Registerkartenzeile
• Menüleiste

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das Popup "Display" angezeigt, das Untereigenschaften für diese Eigenschaft enthält.

Wichtig: Die auf der Seite "ModelingVP" festgelegten Anzeigeeigenschaften werden hochgestuft und steuern alle Seiten, die im Popup "CDOFormsSettings - CDO-Formulare" (CDOFormsSettings - CDO Forms) aufgeführt sind. Informationen zum Popup "CDOFormsSettings - CDO Forms" finden Sie unter "CDO-Formular-Einstellungen (CDO Forms Settings)".

Eigenschaften des Popups "Display"

FullScreen

Boolescher Wert, der die Seitenanzeige erweitert, indem die grauen Seitenleisten auf beiden Seiten der Anwendung entfernt werden, um die vollständige Browserbreite zu nutzen. Der Standardwert ist "Falsch" (False).

Hinweis: Das Layout der Felder, Schaltflächen und anderen Seitenobjekte kann sich beim Anzeigen im Vollbild ändern. Basisanwendungsseiten sind nicht für die Vollbildanzeige konzipiert. Sie können diese Funktion aktivieren, wenn Sie Ihre eigenen benutzerdefinierten Vollbildanzeigen entwerfen. Sie können auch alle Popups in einer Vollbildanzeige auf die volle Browserbreite maximieren.

Mit der Einstellung "FullScreenMode" können Sie den Vollbildmodus für die Framework-Elemente der Anwendung (globale Fußzeile, globale Kopfzeile, Menüleiste und Registerkartenzeile) aktivieren. Informationen zu den Portal-Einstellungen finden Sie unter "Standardeinstellungen (Default Settings)".

HideFooter

Boolescher Wert, der bestimmt, ob auf einer Seite die globale Fußzeile der Anwendung ausgeblendet wird. Der Standardwert ist "Falsch" (False).

HideHeader

Boolescher Wert, der bestimmt, ob auf einer Seite die globale Kopfzeile der Anwendung ausgeblendet wird. Der Standardwert ist "Falsch" (False).

HideTabs

Boolescher Wert, der bestimmt, ob auf einer Seite die Registerkartenzeile der Anwendung ausgeblendet wird. Der Standardwert ist "Falsch" (False).

Hinweis: Siemens empfiehlt, die Schaltfläche "Schließen" (Close) zu Seiten hinzuzufügen, auf denen keine Registerkartenzeile oder Menüleiste mehr angezeigt wird.

HideTopMenu

Boolescher Wert, der bestimmt, ob auf einer Seite die Menüleiste der Anwendung ausgeblendet wird. Der Standardwert ist "Falsch" (False).

Hinweis: Siemens empfiehlt, die Schaltfläche "Schließen" (Close) zu einer Seite hinzuzufügen, auf der keine Registerkartenzeile oder Menüleiste mehr angezeigt wird.

DriverComponent

Dient im Modellierungsframework zum Angeben eines Steuerelements, das einen Aufruf zum Laden von Werten erzwingt, wenn sich die Daten ändern.

DynamicActions

Wird zusammen mit dem Webpart "Aktionssteuerung" (Actions Control) verwendet, um die Anzeige und das Verhalten von dynamischen Aktionen zu steuern.

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das Popup "DynamicActions" angezeigt, das Untereigenschaften für diese Eigenschaft enthält.

Gibt an, ob diese dynamischen Seitenoptionen auf der Seite angezeigt werden. Ist standardmäßig auf "Nicht gesetzt" (Not Set) festgelegt.

Dependencies

Gibt eine Abhängigkeit für die dynamische Aktion an.

ServiceName

Service, der beim Klicken auf die Aktion ausgeführt wird.

Enthält die Liste der dynamischen Webparts, die sich derzeit auf der Seite befinden, sowie die Eigenschaften dieser Webparts.

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das modale Fenster "DynamicWebparts" angezeigt. Das modale Fenster "DynamicWebParts" verwendet dieselben Eigenschaften wie Webparts. Weitere Informationen zu den Eigenschaften, die in diesem modalen Fenster angezeigt werden, finden Sie unter "Webpart-Eigenschaften".

EProcEnabled

Gibt an, ob diese Seite im Framework automatischer Abläufe als Transaktionsseite verwendet werden kann.

EventHandlers

Ermöglicht die Angabe eines Handlers von einem Ereignis.

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das modale Fenster "EventHandler" angezeigt. Weitere Informationen zum Definieren von Ereignishandlern finden Sie unter "Definieren von Ereignishandlern".

ForceEditMode

Für Seiten, bei denen das Verhalten auf "ViewEdit" festgelegt ist, erzwingen Sie das Laden der Seite im Bearbeitungsmodus.

HelpFileUrl

URL für die Hilfedatei der Seite.

HowLong

Zeitraum seit Erstellung der Seite.

LastModified

Datum der letzten Änderung der Seite.

LastModifiedBy

Benutzer, der die letzte Änderung an der Seite vorgenommen hat.

Layout

Das optische Erscheinungsbild der Seite. Diese Eigenschaft ist schreibgeschützt.

Eigenschaften des Popups "Layout"

ColumnNumber

Anzahl der Spalten, die im Layout angezeigt werden sollen.

RowNumber

Anzahl der Zeilen, die im Layout angezeigt werden sollen.

MobileBarcodeEnabled

Ermöglicht das Scannen von Daten von einem mobilen Gerät in bearbeitbare Felder auf der Fertigungsseite. Ist auf allen Fertigungsseiten standardmäßig auf "Wahr" festgelegt. Diese Einstellung auf Seitenebene und die globale Einstellung MobileBarcodeEnabled müssen beide auf "Wahr" festgelegt sein, damit das Barcode-Scanning-Symbol in bearbeitbaren Feldern auf der Seite angezeigt wird. Wenn diese Option auf "Falsch" festgelegt ist, wird das Barcode-Scanning auf der jeweiligen Seite deaktiviert.

PrimaryServiceType

Servicetyp für Steuerelemente, die keinen Servicetyp angeben.

RenderMode

Gibt an, ob die Seite reaktionsschnelles Rendering verwendet. Die folgenden Optionen sind verfügbar:

• Nicht gesetzt (NotSet)
• Festkomma
• Reaktionsfreudig

Die Standardeinstellung für alle Fertigungsseiten ist "NotSet", was "Reaktionsfreudig" entspricht. Die Standardeinstellung für alle anderen Seiten ist "Fixed". Feste Seiten weisen eine feste Breite auf und die Elemente auf der Seite werden bei einer Änderung der Größe des Browsers nicht verschoben.

Hinweis: Sie können sich mit einem der drei Einstiegsmodi bei der Anwendung anmelden. Wenn Sie "RenderMode" auf Seitenlevel festlegen, überschreibt die eingegebene Einstellung die Standardeinstellungen für den Darstellungsmodus, die mit jedem der drei Einstiegsmodi verbunden sind. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics-Benutzerhandbuch – Erste Schritte oder im Opcenter Execution Core-Benutzerhandbuch – Erste Schritte.

Die Seiteneigenschaft für den Modus "FullScreen" ist für alle diese Seiten festgelegt, wenn das Thema "Horizont" festgelegt ist. Weitere Informationen finden Sie unter FullScreen.

SessionVariables

Wird verwendet, um Sitzungsvariablen auf Seitenebene zu konfigurieren.

Weitere Informationen zum Definieren von Sitzungsvariablen finden Sie unter "Definieren von Sitzungsvariablen".

Eine Sammlung von Tag-Namen und deren Steuerungs-ID oder Sitzungsvariable.

Hinweis: Wenn Sie auf die Schaltfläche "Mehr" (More) klicken, wird das modale Fenster "TagCollection" angezeigt. Weitere Informationen zu den Eigenschaften, die in diesem modalen Fenster angezeigt werden, finden Sie unter "Beschreibungen der Eigenschaften des Typs "TagCollection"".

Title

Text des Seitentitels. Ersetzt die Eigenschaft "Titelbeschriftung" (TitleLabel).

Name der Wörterbuchbeschriftung des Titels.

Hinweis: Das Überschreiben des Typnamens (TypeName) auf Seitenebene wird derzeit nicht unterstützt.
