Ändern des Zielhosts

[Remove file?]In Portal Studio können Sie auf Elemente zugreifen und Elemente bearbeiten, die auf einem anderen, mit Portal Studio kompatiblen System gehostet werden.

Beim Ändern des Zielhosts

Wenn sich der Zielhost in einer anderen Domäne als der aktuelle Host befindet, müssen Sie eine Vertrauensstellung erstellen, damit Silverlight auf die andere Domäne zugreifen kann.

Dies ist ein Beispiel einer Vertrauensstellung:

<?xml version="1.0" encoding="utf-8"?>

<access-policy>

<cross-domain-access>

<policy>

<allow-from http-request-headers="*">

<domain uri="*"/>

</allow-from>

<grant-to>

<resource path="/" include-subpaths="true"/>

</grant-to>

</policy>

</cross-domain-access>

</access-policy>

Vorgehensweise zum Ändern des Zielhosts

Gehen Sie folgendermaßen vor, um den Zielhost zu ändern:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Einstellungen (Settings) und wählen Sie Zielhost ändern (Change Target Host) aus. Sie werden von Portal Studio abgemeldet, worauf eine Anmeldeseite angezeigt wird.
3. Geben Sie den Benutzernamen (User Name), das Kennwort (Password) und die Domäne (Domain) für den Zielhost ein.
4. Klicken Sie auf Anmelden (Log On). Sie werden am neuen Zielhost angemeldet.
