Ausführen des Ablaufverfolgungsprotokolls

In Portal Studio können Sie zur Unterstützung der Fehlerbehebung ein Ablaufverfolgungsprotokoll (Trace Log) ausführen. Im Ablaufverfolgungsprotokoll wird der Verlauf der Änderungen erfasst, die an den Portalseiten vorgenommen werden, wenn Sie diese in Portal Studio aktualisieren und ausführen. Im Ablaufverfolgungsprotokoll können Sie außerdem Serverinformationen erfassen, wenn Sie Transaktionen auf Portalseiten ausführen.

Aktivieren des Ablaufverfolgungsprotokolls

Das Ablaufverfolgungsprotokoll ist standardmäßig deaktiviert und muss in den Standardeinstellungen von Portal Studio über die Einstellung "Ablaufverfolgung" (Trace) aktiviert werden. Anschließend müssen Sie sich erst am Portal abmelden und dann erneut anmelden, bevor Sie versuchen, das Ablaufverfolgungsprotokoll auszuführen.

Darüber hinaus müssen Sie die Berechtigungen für den in der Einstellung "TracePath" angegebenen Ordner festlegen, damit die Anwendung auf den Ordner für den angemeldeten Benutzer zugreifen kann. Der Standardspeicherort für den Ablaufverfolgungspfad ist C:\Temp.

Die Anwendung zeigt eine Fehlermeldung an, wenn das Ablaufverfolgungsprotokoll aktiviert wurde und der Benutzer, der sich anzumelden versucht, keine Zugriffsberechtigung für den Ordner im Ablaufverfolgungspfad besitzt. Sie können den Fehler beheben, indem Sie den Ablaufverfolgungspfad (TracePath) in der Benutzereinstellungendatei manuell aktualisieren. Die Benutzereinstellungendatei befindet sich standardmäßig in folgendem Pfad: 

C:\Programme (x86)\Camstar\Camstar Portal\User\Settings.xml

Beim Ausführen des Ablaufverfolgungsprotokolls

Wenn Sie auf die Schaltfläche "Ablaufverfolgungsanzeige anzeigen/ausblenden" (Show/Hide Tracer Viewer) klicken, wird das Ablaufverfolgungsprotokoll aktiviert, das unterhalb des Portal Studio-Arbeitsbereichs angezeigt wird. Ablaufverfolgungen im Portal werden im linken Bereich, Serverablaufverfolgungen im rechten Bereich angezeigt. Sie können das Ablaufverfolgungsprotokoll jederzeit ausschalten, indem Sie erneut auf die Schaltfläche "Ablaufverfolgungsanzeige anzeigen/ausblenden" klicken.

Ablaufverfolgungen im Portal werden automatisch erfasst, wenn Sie eine Seite in Portal Studio ausführen. Sie müssen mindestens eine Transaktion auf der ausgeführten Portalseite ausführen, um eine Serverablaufverfolgung zu erfassen. Eine Transaktion liegt z. B. vor, wenn Sie auf der Seite auf eine Schaltfläche klicken.

Im Ablaufverfolgungsprotokoll wird für Portal- und Serverablaufverfolgungen ein Zeitstempel angezeigt, und zwar jeweils auf der obersten Ebene einer erfassten Aktion. Der Zeitstempel enthält Jahr, Monat, Tag, Stunde, Minute, Sekunde und Millisekunde des Zeitpunkts, zu dem die Aktion erfasst wurde. Die Uhrzeitangabe im Zeitstempel hat das Format HH:MM:SS:MSS und entspricht der im Browser festgelegten Zeitzone. Das Format der Datumsangabe im Zeitstempel entspricht dem lokalen Datumsformat der Sprache, die im Browser festgelegt ist. Beispiel: Wenn als Sprache für Ihren Browser "Englisch (USA)" festgelegt ist, hat die Datumsangabe im Zeitstempel das Format MM/TT/JJJJ.

Sie können weitere Informationen für eine Serverablaufverfolgung im Fenster "Serverablaufverfolgung" (Server Trace) anzeigen. Klicken Sie dazu auf die entsprechende Schaltfläche "Serverablaufverfolgung". Das Fenster "Serverablaufverfolgung" enthält eine vereinfachte Ansicht der Serverablaufverfolgung. Über die Schaltfläche "Rohdatenansicht" (Raw View) besteht die Möglichkeit, die Ablaufverfolgung als XML-Datei anzuzeigen.

Geben Sie ein Schlüsselwort ein und klicken Sie auf "Suchen" (Search), um das Ablaufverfolgungsprotokoll nach Serviceaufrufen, Methoden und Parametern zu durchsuchen. Nachdem Sie auf "Suchen" geklickt haben, wird der erste Treffer hervorgehoben. Wenn Sie mehrmals auf "Suchen" klicken, werden die Treffer der Reihe nach hervorgehoben, bis Sie alle Treffer angezeigt haben. Wenn Sie nach dem letzten Treffer erneut auf "Suchen" klicken, wird die Nachricht angezeigt, dass keine weiteren Treffer gefunden wurden.

Im Fenster "Transaktion einreichen" (Submit Transaction) können Sie die Commit-Aktion durch Deaktivieren des Kontrollkästchens "Commit" deaktivieren, wenn Sie eine Serverablaufverfolgung erneut einreichen. Das Kontrollkästchen "Commit" ist standardmäßig aktiviert.

Vorgehensweise zum Aktivieren des Ablaufverfolgungsprotokolls

Gehen Sie folgendermaßen vor, um das Ablaufverfolgungsprotokoll zu aktivieren:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf die Schaltfläche Portaleinstellungen (Portal Settings).Das Popup "Portaleinstellungen" (Portal Settings) wird angezeigt.
3. Wählen Sie Wahr (True) für Ablaufverfolgung aktiviert (TraceEnabled) aus.
4. Klicken Sie auf OK. Das Popup "Portaleinstellungen" (Portal Settings) wird geschlossen und das Ablaufverfolgungsprotokoll ist jetzt aktiviert.
Note: Sie müssen sich am Portal abmelden und erneut anmelden, bevor Sie versuchen, das Ablaufverfolgungsprotokoll auszuführen.

Vorgehensweise zum Ausführen des Ablaufverfolgungsprotokolls

Gehen Sie folgendermaßen vor, um das Ablaufverfolgungsprotokoll auszuführen:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Ablaufverfolgungsansicht anzeigen/ausblenden (Show/Hide Trace View). Das Fenster "Ablaufverfolgung" (Tracer) wird angezeigt und die Ausführung des Ablaufverfolgungsprotokolls wird gestartet.
3. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Komponentenkatalog ausgeblendet ist. Der Komponentenkatalog wird angezeigt.
4. Doppelklicken Sie auf eine beliebige Portalseite. Die Seite wird im Arbeitsbereich angezeigt.
5. Nehmen Sie die erforderlichen Änderungen an der Seite vor.
6. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
7. Klicken Sie auf Ausführen (Run). Die Portalseite wird auf einer neuen Browserregisterkarte geöffnet.
8. Kehren Sie zu der Browserregisterkarte zurück, auf der Portal Studio ausgeführt wird, und klicken Sie im Ablaufverfolgungsfenster auf die Schaltfläche Aktualisieren (Refresh). Das Ablaufverfolgungsprotokoll wird im linken Ablaufverfolgungsfenster angezeigt.
9. Wiederholen Sie ggf. die Schritte 5 bis 8, um weitere Ablaufverfolgungsprotokoll-Daten zu erfassen.

Vorgehensweise zum Speichern des Ablaufverfolgungsprotokolls

Gehen Sie folgendermaßen vor, um das Ablaufverfolgungsprotokoll zu speichern:

1. Führen Sie die Prozedur unter "Vorgehensweise zum Ausführen des Ablaufverfolgungsprotokolls" aus.
2. Klicken Sie auf die Schaltfläche Ablaufverfolgung speichern (Save Trace) für das Ablaufverfolgungsprotokoll, das Sie speichern möchten. Das Dialogfeld "Dateidownload – Sicherheitswarnung" (File Download – Security Warning) wird angezeigt.
3. Klicken Sie auf OK. Das Dialogfeld "Dateidownload – Sicherheitswarnung" (File Download – Security Warning) wird geschlossen und das Popup "Speichern unter" (Save As) wird angezeigt.
4. Navigieren Sie zu einem Dateispeicherort und klicken Sie auf Speichern (Save). Das Popup "Speichern unter" (Save As) wird geschlossen und das Ablaufverfolgungsprotokoll wird gespeichert.

Vorgehensweise zum erneuten Einreichen eines Ablaufverfolgungsprotokolls

Gehen Sie folgendermaßen vor, um ein Ablaufverfolgungsprotokoll erneut einzureichen:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Ablaufverfolgungsansicht anzeigen/ausblenden (Show/Hide Trace View). Das Fenster "Ablaufverfolgung" (Tracer) wird angezeigt und die Ausführung des Ablaufverfolgungsprotokolls wird gestartet.
3. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Komponentenkatalog ausgeblendet ist. Der Komponentenkatalog wird angezeigt.
4. Doppelklicken Sie auf eine beliebige Portalseite. Die Seite wird im Arbeitsbereich angezeigt.
5. Nehmen Sie die erforderlichen Änderungen an der Seite vor.
6. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
7. Klicken Sie auf Ausführen (Run). Die Portalseite wird auf einer neuen Browserregisterkarte geöffnet.
8. Führen Sie auf der Portalseite eine Transaktion aus.
9. Kehren Sie zu der Browserregisterkarte zurück, auf der Portal Studio ausgeführt wird, und klicken Sie im Ablaufverfolgungsfenster auf die Schaltfläche Aktualisieren (Refresh). Das Ablaufverfolgungsprotokoll wird im rechten Ablaufverfolgungsbereich angezeigt.
10. Klicken Sie auf die Schaltfläche Serverablaufverfolgung (Server Trace) für die Serverablaufverfolgung, die Sie erneut einreichen möchten. Das Fenster "Serverablaufverfolgung" (Server Trace) wird angezeigt.
11. Klicken Sie auf Erneut einreichen (Resubmit). Das Fenster "Transaktion einreichen" (Submit Transaction) wird angezeigt.
12. Klicken Sie auf Einreichen (Submit). Die Ablaufverfolgung wird erneut eingereicht und im Fenster "Serverablaufverfolgung" (Server Trace) wird eine Erfolgsnachricht angezeigt.

Vorgehensweise zum Durchsuchen des Server-Ablaufverfolgungsprotokolls

Gehen Sie folgendermaßen vor, um das Server-Ablaufverfolgungsprotokoll zu durchsuchen:

1. Führen Sie die Prozedur unter "Vorgehensweise zum Ausführen des Ablaufverfolgungsprotokolls" aus.
2. Klicken Sie auf der Serverseite der Ablaufverfolgung auf "Serverablaufverfolgung" (Server Trace). Das Fenster "Serverablaufverfolgung" (Server Trace) wird angezeigt.
3. Geben Sie ein Schlüsselwort in das Suchfeld ein.
4. Klicken Sie auf Suchen (Search). Die Anwendung hebt den ersten Treffer hervor.
5. Wiederholen Sie Schritt 4, um weitere Treffer zu suchen.
