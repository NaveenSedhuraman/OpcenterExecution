Ein- und Auschecken von Arbeitsaufgaben

In Portal Studio können Sie Arbeitsaufgaben (Work Items) aus dem System zur Quellcodeverwaltung auschecken und wieder in das System einchecken. Ausgecheckte Arbeitsaufgaben sind durch einen roten Pfeil auf dem Symbol der Aufgabe im Katalog der Benutzeroberflächenkomponenten gekennzeichnet.

Beim Ein- und Auschecken von Arbeitsaufgaben

Wenn Sie eine Arbeitsaufgabe (Work Item) auschecken, können andere Benutzer die Aufgabe sehen. Sie können jedoch keine Änderungen an der Aufgabe vornehmen oder speichern. Für andere Portal Studio-Benutzer werden ausgecheckte Arbeitsaufgaben mit einem Vorhängeschloss angezeigt.

Arbeitsaufgaben werden als Einzelelemente ausgecheckt. Wenn eine Arbeitsaufgabe in einem Arbeitsbereich ausgecheckt ist, ist sie auch aus allen anderen Arbeitsbereichen ausgecheckt. Weitere Informationen finden Sie unter "Entwickeln in Arbeitsbereichen".

Nachdem Sie eine Arbeitsaufgabe eingecheckt haben, kann sie von allen anderen Benutzern ausgecheckt werden.

Vorgehensweise zum Ein- und Auschecken von Arbeitsaufgaben

Gehen Sie folgendermaßen vor, um Arbeitsaufgaben ein- und auszuchecken:

1. Greifen Sie auf Portal Studio zu.
2. Öffnen Sie eine Arbeitsaufgabe.
3. Klicken Sie auf Auschecken (Check Out). Die Arbeitsaufgabe wird aus dem System zur Quellcodeverwaltung ausgecheckt.
4. Führen Sie die erforderlichen Arbeitsschritte aus.
5. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
6. Klicken Sie auf Einchecken (Check In). Die Aufgabe wird in das System zur Quellcodeverwaltung eingecheckt.
