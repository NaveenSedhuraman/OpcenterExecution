Aktivieren des erzwungenen Auscheckens

In Portal Studio können Sie ein Auschecksystem erzwingen (Enforced Check Out), das verhindert, dass mehrere Entwickler gleichzeitig Änderungen an einer Seite speichern. Bei einem Auschecksystem müssen Sie eine Seite zunächst auschecken, damit Änderungen gespeichert werden können.

Beim Aktivieren des erzwungenen Auscheckens

In Portal Studio können Sie mit den Schaltflächen "Einchecken" (Check In) und "Auschecken" (Check Out) in der globalen Symbolleiste Seiten ein- und auschecken. Es kann immer nur jeweils ein Entwickler gleichzeitig eine Seite auschecken.

Das Auschecken wird standardmäßig nicht erzwungen und muss in den Standardeinstellungen von Portal Studio konfiguriert werden. Weitere Informationen zu den Felddefinitionen der Standardeinstellungen finden Sie unter "Konfigurieren von Portaleinstellungen".

Vorgehensweise zum Aktivieren des erzwungenen Auscheckens

Gehen Sie folgendermaßen vor, um erzwungenes Auschecken zu aktivieren:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf die Schaltfläche Portaleinstellungen (Portal Settings).Das Popup "Portaleinstellungen" (Portal Settings) wird angezeigt.
3. Wählen Sie Standardeinstellungen (DefaultSettings) aus der Liste der Einstellungen aus. Die Eigenschaften für Standardeinstellungen (DefaultSettings) werden angezeigt.
4. Wählen Sie Wahr (True) für Auschecken beim Bearbeiten erzwingen (EnforceCheckOutWhenEditing) aus.
5. Klicken Sie auf OK.
6. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
