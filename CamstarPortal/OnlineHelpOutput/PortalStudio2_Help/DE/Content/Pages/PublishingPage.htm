Veröffentlichen von Inhalten

Beim Veröffentlichen einer Seite oder eines Webparts werden Ihre Änderungen im höchsten aktiven Arbeitsbereich veröffentlicht und außerhalb des Entwicklerarbeitsbereichs im Portal verfügbar gemacht.

Beim Veröffentlichen von Inhalten

Die Schaltfläche "Veröffentlichen" (Publish) ist aktiviert, wenn eine Seite oder ein Webpart gespeichert wird oder wenn keine nicht gespeicherten Änderungen vorhanden sind. Nicht veröffentlichte Seiten und Webparts sind im Komponentenkatalog mit einem Bleistiftsymbol auf dem Symbol der Komponente gekennzeichnet.

Die Schaltfläche "Veröffentlichen" (Publish) ist nur im höchsten aktiven Arbeitsbereich und im Entwickler-Arbeitsbereich aktiv. Die gespeicherten Änderungen, die im Entwicklerarbeitsbereich veröffentlicht werden, werden in den höchsten aktiven Arbeitsbereich übertragen. Weitere Informationen finden Sie unter "Entwickeln in Arbeitsbereichen".

Sie können Seiten oder Webparts auf ihren zuletzt veröffentlichten Zustand zurücksetzen. Klicken Sie dazu auf "Zurücksetzen zu" (Revert To) und wählen Sie "Veröffentlicht" (Published) aus. Weitere Informationen finden Sie unter "Zurücksetzen von Inhalten".

Die Anwendung zeigt das Dialogfeld "Veröffentlichen" (Publishing) an. Wenn Sie auf "Veröffentlichen" (Publish) klicken, werden Sie aufgefordert, alle mit der zu veröffentlichenden Komponente verbundenen Personalisierungen zu entfernen. Ist das Kontrollkästchen "Alle Personalisierungen für diese Komponente entfernen" (Remove All Personalization Associated with this Component) aktiviert, werden alle Personalisierungen entfernt und der Veröffentlichungsvorgang wird fortgesetzt, wenn Sie auf "OK" klicken. Wenn Sie auf "OK" klicken, ohne das Kontrollkästchen zu aktivieren, wird der Veröffentlichungsvorgang fortgesetzt. Durch Klicken auf "Abbrechen" (Cancel) wird der Veröffentlichungsvorgang abgebrochen.

Beim Veröffentlichen einer neuen Seite wird automatisch eine Seitenberechtigung erstellt und zu der Rolle "Standardseiten" (Default Pages) hinzugefügt. Wenn Sie eine Seite veröffentlichen, die zuvor bereits veröffentlicht wurde, wird keine Seitenberechtigung erstellt. Sie haben die Möglichkeit, die Eigenschaft "Automatische Seitenberechtigung" (AutoPagePermission) im Fenster "Portaleinstellungen" (Portal Settings) in den Standardeinstellungen (DefaultSettings) auf "Falsch" (False) festzulegen. Dadurch verhindern Sie, dass die Anwendung standardmäßig neue Seitenberechtigungen hinzufügt. Weitere Informationen finden Sie unter "Konfigurieren von Portaleinstellungen".

Note: Nachdem Sie eine neue Seite erstellt haben, müssen Sie sich erst am Portal abmelden und erneut anmelden, bevor Sie versuchen, mit der Rolle "Standardseiten" (Default Pages) eine neu veröffentlichte Seite auszuführen oder eine Seitenberechtigung anzuzeigen. Informationen zu Rollen finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Modeling-Benutzerhandbuch bzw. im Opcenter Execution Core Portal Modeling-Benutzerhandbuch.

Vorgehensweise zum Veröffentlichen von Seiten und Webparts

Gehen Sie folgendermaßen vor, um Seiten und Webparts zu veröffentlichen:

1. Greifen Sie auf Portal Studio zu.
2. Öffnen Sie eine Seite oder ein Webpart. Die Seite bzw. das Webpart wird im Arbeitsbereich angezeigt.
3. Nehmen Sie die erforderlichen Änderungen vor.
4. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
5. Klicken Sie auf Veröffentlichen (Publish). Das Dialogfeld "Veröffentlichen" (Publishing) wird angezeigt.
6. Klicken Sie auf OK. Das Dialogfeld "Veröffentlichung" (Publishing) wird geschlossen, die Seite wird im höchsten aktiven Arbeitsbereich veröffentlicht und die Anwendung zeigt eine Erfolgsmeldung an.
