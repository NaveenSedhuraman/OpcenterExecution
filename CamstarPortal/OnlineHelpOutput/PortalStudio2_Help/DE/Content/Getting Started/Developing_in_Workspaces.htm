Entwickeln in Arbeitsbereichen

Siemens stellt Arbeitsbereiche für Metadaten zur Verfügung, die die Entwicklung, Bereitstellung und Aktualisierung von Partnerangeboten und Kundenanpassungen vereinfachen.

Sie können Opcenter Execution Core ändern oder anpassen, ohne dabei Änderungen anderer Benutzer zu überschreiben, indem Sie die Entwicklungen in Arbeitsbereichen in Portal Studio veröffentlichen. Mit der Veröffentlichung von Änderungen in einem Arbeitsbereich isolieren Sie diese Änderungen von Aktualisierungen, die evtl. an anderen Arbeitsbereichen vorgenommen werden. So wirken sich z. B. Änderungen, die in einem Kundenarbeitsbereich veröffentlicht wurden, nicht auf den CSI-Basisarbeitsbereich aus. Analog gilt, dass Anpassungen in anderen Arbeitsbereichen nicht überschrieben werden, wenn eine Aktualisierung des CSI-Basisarbeitsbereichs akzeptiert wird. Weitere Informationen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Beim Entwickeln in Arbeitsbereichen

Portal Studio wird standardmäßig im Entwicklerarbeitsbereich (Developer) geöffnet. Sie können die Liste der aktiven Arbeitsbereiche über die Liste "Inhalte anzeigen" (View Contents) anzeigen. Die Farbe, die für die einzelnen Arbeitsbereichsnamen angezeigt wird, entspricht der Farbe, die in Management Studio für den Arbeitsbereich festgelegt wurde. Weitere Informationen zum Verwalten von Arbeitsbereichen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

In Portal Studio sind sowohl der Entwicklerarbeitsbereich als auch der CSI-Basisarbeitsbereich standardmäßig aktiv. Der Kundenarbeitsbereich ist ebenfalls standardmäßig aktiv, wenn ein Kunde seine Metadatendatenbank nach einer Installation erhält. Andere Arbeitsbereiche werden erst im Menü "Inhalte anzeigen" (View Contents) angezeigt, nachdem sie in Management Studio aktiviert wurden. Weitere Informationen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Note: WCF-Dienste müssen bei jeder Aktivierung oder Deaktivierung von Arbeitsbereichen in Management Studio erneut generiert werden. Änderungen an verfügbaren aktiven Arbeitsbereichen werden erst in der Liste "Inhalte anzeigen" (View Contents) angezeigt, nachdem die WCF-Services erneut generiert wurden.

Änderungen können nur im Entwicklerarbeitsbereich gespeichert und veröffentlicht werden. Sie können in der Liste "Inhalte anzeigen" (View Contents) andere aktive Arbeitsbereiche auswählen, deren Inhalt und Entwicklungsänderungen angezeigt werden sollen. Die Schaltflächen "Speichern" (Save), "Alles speichern" (Save All) und "Veröffentlichen" (Publish) sind jedoch deaktiviert.

Veröffentlichte Änderungen sind arbeitsbereichsspezifisch und werden nur in dem Arbeitsbereich angezeigt, in dem sie veröffentlicht werden. Änderungen, die im Entwicklerarbeitsbereich gespeichert und veröffentlicht werden, werden im höchsten aktiven Arbeitsbereich veröffentlicht. Wenn Sie vor dem Speichern zu einem anderen Arbeitsbereich wechseln, gehen alle im aktuellen Arbeitsbereich vorgenommenen nicht gespeicherten Änderungen verloren.

Inhalte, die geändert und gespeichert wurden, werden standardmäßig mit einem roten Rahmen angezeigt. Dieser zeigt an, dass die Änderungen korrekt gespeichert wurden. Der Rahmen, der das Objekt umgibt, wird in der Farbe angezeigt, die in Management Studio für den Arbeitsbereich angegeben wurde, wenn Sie ein Steuerelement auf einer standardmäßigen virtuellen Seite im Arbeitsbereich ändern und speichern. Der farbige Rahmen gibt an, dass das Steuerelement überschrieben wurde.

Ein Arbeitsbereich kann in Management Studio aktiviert werden. Weitere Informationen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Im Entwicklerarbeitsbereich gespeicherte und veröffentlichte Änderungen werden für den CSI-Basisarbeitsbereich übernommen, wenn keine anderen Arbeitsbereiche aktiv sind. Der CSI-Basisarbeitsbereich ist immer aktiv.

Beim Verwenden des Ablaufverfolgungsprotokolls

In Portal Studio können Sie zur Unterstützung der Fehlerbehebung ein Ablaufverfolgungsprotokoll (Trace Log) ausführen. In diesem Protokoll können Sie den Verlauf der Änderungen an Portalseiten erfassen, während Sie diese in Portal Studio aktualisieren und ausführen. Weitere Informationen finden Sie unter "Ausführen des Ablaufverfolgungsprotokolls".

Vorgehensweise zum Veröffentlichen von Inhalten im aktiven Arbeitsbereich

Gehen Sie folgendermaßen vor, um Inhalte im aktiven Arbeitsbereich zu veröffentlichen:

1. Greifen Sie auf Portal Studio zu.
2. Öffnen Sie eine Seite im Komponentenkatalog. Die Seite wird im Arbeitsbereich angezeigt.
3. Nehmen Sie die erforderlichen Änderungen vor.
4. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
5. Klicken Sie auf Veröffentlichen (Publish). Das Dialogfeld "Veröffentlichen" (Publishing) wird angezeigt.
6. Klicken Sie auf OK. Das Dialogfeld "Veröffentlichen" (Publishing) wird geschlossen und die Seite wird im höchsten aktiven Arbeitsbereich veröffentlicht.
