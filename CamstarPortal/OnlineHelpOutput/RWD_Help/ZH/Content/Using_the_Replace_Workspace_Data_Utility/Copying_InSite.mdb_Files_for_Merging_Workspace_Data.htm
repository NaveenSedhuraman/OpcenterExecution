复制 InSite.mdb 文件以合并工作区数据

Siemens 建议为合并工作区数据时将使用的文件创建一个文件夹。然后，可以将当前文件和基本元数据 InSite.mdb 文件从其各自的文件夹复制到此文件夹并重命名这些文件，使其可以作为输入用于替换工作区数据实用程序。

在复制 InSite.mdb 文件以合并工作区数据时

创建用于维护 mdb 文件的文件夹（例如 C:\Users\管理员\桌面\合并），这些文件将用作替换工作区数据实用程序的输入。

必须将当前 InSite.mdb 文件和基本元数据 InSite.mdb 文件复制到此文件夹。

重要提示: 不要更新 C:\Program Files (x86)\Camstar\InSite Administration\Base Metadata 文件夹中的 InSite.mdb 文件。Opcenter EX MDD 和 Opcenter EX CR 需要将该文件用于元数据升级过程。在 Base Metadata 文件夹中使此文件保留原样，以确保 Software Update (SU) 安装程序可以使用每个 SU 成功替换该文件。

如何复制 InSite.mdb 文件以合并工作区数据

按照以下步骤复制 InSite.mdb 文件以合并工作区数据：

1. 访问安装了 Opcenter EX MDD 或 Opcenter EX CR 的服务器。
2. 为要用作替换工作区数据实用程序输入的各种 mdb 文件创建文件夹，以维护这些文件。
3. 将基本元数据 InSite.mdb 文件（默认为 C:\Program Files (x86)\Camstar\InSite Administration\Base Metadata\InSite.mdb）复制到新文件夹中。
4. 例如，将复制的文件重命名为 InSite_New.mdb。
5. 移除此文件中的只读属性：
a. 右键单击文件名。出现一个菜单。
b. 从菜单中选择属性。将显示“属性”对话框。
c. 清除只读复选框中的勾号。
d. 单击确定。 “属性”对话框将关闭，并且该文件不再标记为只读。
6. 备份刚刚复制的文件。
7. 例如，将复制的文件重命名为 InSite_Base.mdb。
8. 将当前 InSite.mdb 文件（默认为 C:\Program Files (x86)\Camstar\InSite Administration\InSite.mdb）复制到新文件夹中。
9. 例如，将副本重命名为 InSite_Current.mdb。
