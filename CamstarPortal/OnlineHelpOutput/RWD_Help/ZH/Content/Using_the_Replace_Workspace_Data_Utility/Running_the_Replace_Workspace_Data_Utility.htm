运行替换工作区数据实用程序

替换工作区数据实用程序可用于将多个位置的工作区合并到一个位置。

运行替换工作区数据实用程序时

该实用程序用于添加或更新目标 MDB 中的单个工作区。

必须按照从最低到最高的顺序替换工作区数据。例如，可以从基本 csi 工作区（顺序号 1）开始，最后为 Site 工作区（顺序号 200）运行该实用程序。可以在 Management Studio 中查看活动工作区及其顺序号。有关工作区的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

选中“完成时替换当前文件”复选框时，该实用程序会重命名在“当前元数据”字段中指定的文件并将其替换为在“目标元数据”字段中指定的文件。清除“完成时替换当前文件”复选框时，该实用程序会创建在“目标元数据”字段中指定的文件。在检查该文件以及日志文件后，必须手动重命名该文件。

注释: 如果已从 Base Metadata 文件夹中复制 InSite.mdb 文件，则无需更改活动工作区，因为此文件夹中的 InSite.mdb 文件默认位于 csi 工作区中。

进度指示器将显示替换过程的进度。进度指示器在单击“升级”按钮后显示。

如果将任何必填字段留空，该实用程序将显示“找不到文件”弹出消息。可以在弹出消息上单击“确定”以返回“替换工作区数据”窗口。

访问帮助及更多信息

如果在替换过程中发生错误，该实用程序将显示一条错误消息。可以单击“继续”以尝试继续替换操作。可以单击“退出”以关闭错误消息和该实用程序。这些错误记录在事件查看器和该实用程序所生成的日志文件中。

使用“帮助”菜单可以访问该实用程序的联机帮助。可以通过从“帮助”菜单选择“替换工作区数据帮助”来访问联机帮助。如果有已翻译版本的帮助，联机帮助会以浏览器中设置的首选语言显示。确保已为首选语言安装了最新的语言包。默认情况下，帮助以英语显示。

从“帮助”菜单选择“关于”时，会显示“关于替换工作区数据”弹出窗口。“关于替换工作区数据”弹出窗口会显示有关替换工作区数据实用程序的产品和版本信息。可以通过单击“关闭”按钮来关闭“关于替换工作区数据”弹出窗口。

单击该实用程序中某个字段旁边的信息图标，会显示包含该字段的相关详细信息的弹出窗口。也可以将鼠标悬停在带有信息图标的任何字段上方，以查看与工具提示相同的信息。

注释: 基本 MDB 位于 InSite Administration 中的 Base Metadata 文件夹中，且每次使用替换工作区数据实用程序时都必须复制该文件。

从命令提示符运行替换工作区数据实用程序

可以从命令提示符运行替换工作区数据实用程序。可以使用以下命令调用替换工作区数据实用程序：

"ReplaceWorkspaceData.exe -log:[logfilepath] -base:[baseMDBpath] -new:[newMDBpath] -current:[currentMDBpath] -target:[targetMDBpath] -replace_current_MDB:[True/False]"

每个参数都由一个空格隔开并以破折号开头。替换括号中变量的值与替换工作区数据实用程序窗口上字段的预期值相同。有关信息，请参考“替换工作区数据窗口字段定义”。

此表定义必须在命令行中将括号中变量替换为的信息。

括号

输入 ...

-log:[]

日志文件的完整路径位置。

-base:[]

基本元数据 InSite.mdb 文件副本的完整路径位置。

-new:[]

新的元数据 InSite.mdb 文件的完整路径位置。

-current:[]

当前 InSite.mdb 文件副本的完整路径位置。

-target:[]

包含合并过程结果的数据库的完整路径位置。

-replace_current_MDB:[]

“True”，表示将重命名“当前元数据”字段中指定的文件并将其替换为在“目标元数据”字段中指定的文件。

输入“False”，表示将创建目标元数据文件并根据日志文件进行审核，并在审核后手动重命名该文件。

进度指示器将在输入命令行并按 Enter 后显示。该实用程序替换工作区后，将显示一条成功消息。

如果该实用程序出现错误，将显示一条错误消息和帮助信息。还可以输入“帮助”并按 Enter 以显示帮助信息。可以查看这些信息，然后尝试重新输入命令并按 Enter 来运行该实用程序。

如何运行替换工作区数据实用程序

按照以下步骤运行替换工作区数据实用程序：

1. 单击开始 > Opcenter Execution MDD。

或

单击开始 > Opcenter Execution Core。

2. 右键单击替换工作区数据，然后选择更多 > 以管理员身份运行。将显示“替换工作区数据”窗口。
3. 在日志文件字段中输入日志文件的完整路径位置。

或

单击“浏览”按钮以导航至该位置。

4. 在基本 Camstar 工作区数据字段中输入基本元数据 InSite.mdb 文件副本的完整路径位置。

或

单击“浏览”按钮以导航至该位置。

MDB 版本将显示在“版本”字段中。

5. 在新工作区数据字段中输入新 InSite.mdb 文件的完整路径位置。

或

单击“浏览”按钮以导航至该位置。

MDB 版本显示在“版本”字段中，且当前工作区显示在“工作区”字段中。

6. 在当前元数据字段中输入当前 InSite.mdb 文件副本的完整路径位置。

或

单击浏览按钮以导航至该位置。

MDB 版本显示在“版本”字段中，且当前工作区显示在“工作区”字段中。

7. 输入数据库的完整路径位置，该数据库包含目标元数据字段中合并过程的结果。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 单击升级。进度指示器条将显示替换的进度。替换过程完成后，将显示弹出窗口。
10. 单击确定。 弹出窗口将关闭。
11. 单击关闭。“替换工作区数据”窗口将关闭。

如何从命令提示符运行替换工作区数据实用程序

按照以下步骤从命令提示符运行替换工作区数据实用程序：

1. 单击“开始 > Windows 系统 > 命令提示符”。将显示“命令提示符”窗口。
2. 输入 cd C:\Program Files (x86)\Camstar\InSite Conversion Tool\Merge Tool 并按 Enter。
3. 输入 ReplaceWorkspaceData.exe -log:[logfilepath] -base:[baseMDBpath] -new:[newMDBpath] -current:[currentMDBpath] -target:[targetMDBpath] -replace_current_MDB:[True/False]，将括号中的信息替换为 .mdb 文件的完整路径位置，并输入 True 或 False 以替换 replace_current_MDB 括号中的信息。有关在“命令提示符”窗口中输入参数的信息，请参考“从命令提示符运行替换工作区数据实用程序”。
4. 按 Enter。将显示进度指示器，显示替换的进度。替换过程完成后，将显示成功消息。
5. 单击关闭按钮以关闭“命令提示符”窗口。
