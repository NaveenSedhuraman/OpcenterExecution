Ausführen des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data)

Das Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) bietet die Möglichkeit, Arbeitsbereiche mehrerer Speicherorte in einem einzelnen Speicherort zusammenzufassen.

Beim Ausführen des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data)

Das Dienstprogramm wird verwendet, um einen einzelnen Arbeitsbereich in der Ziel-MDB hinzuzufügen oder zu aktualisieren.

Sie müssen Arbeitsbereichsdaten nacheinander ersetzen, vom niedrigsten zum höchsten. Sie beginnen z. B. mit dem CSI-Basisarbeitsbereich (Sequenznummer 1) und führen das Dienstprogramm zuletzt für den Standortarbeitsbereich (Sequenznummer 200) aus. Sie können die aktiven Arbeitsbereiche mit den zugehörigen Sequenznummern in Management Studio anzeigen. Informationen zu Arbeitsbereichen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration und in der Opcenter Execution Core-Anleitung zur Systemadministration.

Das Dienstprogramm benennt die im Feld "Aktuelle Metadaten" (Current Metadata) angegebene Datei um und ersetzt sie durch die im Feld "Zielmetadaten" (Target Metadata) angegebene Datei, wenn das Kontrollkästchen "Aktuelle nach Abschluss ersetzen" (Replace Current When Done) aktiviert ist. Das Dienstprogramm erstellt die im Feld "Zielmetadaten" (Target Metadata) angegebene Datei, wenn das Kontrollkästchen "Aktuelle nach Abschluss ersetzen" (Replace Current When Done) deaktiviert ist. Sie müssen die Datei nach der Überprüfung zusammen mit der Protokolldatei manuell umbenennen.

Hinweis: Sie müssen den aktiven Arbeitsbereich nicht ändern, wenn Sie die Datei "InSite.mdb" aus dem Ordner "Base Metadata" kopiert haben, da sich die Datei "InSite.mdb" in diesem Ordner standardmäßig im CSI-Arbeitsbereich befindet.

Der Fortschritt des Ersetzungsvorgangs wird in der Statusanzeige angezeigt. Die Statusanzeige wird angezeigt, wenn Sie auf die Schaltfläche "Aktualisieren" (Upgrade) klicken.

Das Dienstprogramm blendet die Meldung "Datei nicht gefunden" (File not found) ein, wenn eines der erforderlichen Felder nicht ausgefüllt wird. Sie können in der eingeblendeten Meldung auf "OK" klicken, um zum Fenster "Arbeitsbereichsdaten ersetzen" (Replace Workspace Data) zurückzukehren.

Aufrufen der Hilfe und weitere Informationen

Das Dienstprogramm zeigt eine Fehlermeldung an, wenn bei der Ersetzung ein Fehler auftritt. Sie können auf "Fortsetzen" (Continue) klicken, um zu versuchen, die Ersetzung fortzusetzen. Sie können auf "Beenden" (Quit) klicken, um die Fehlermeldung und das Dienstprogramm zu schließen. Fehler werden in der Ereignisanzeige und in der vom Dienstprogramm erstellten Protokolldatei protokolliert.

Das Hilfemenü bietet Zugriff auf die Onlinehilfe für das Dienstprogramm. Sie können die Onlinehilfe aufrufen, indem Sie "Hilfe zum Ersetzen von Arbeitsbereichsdaten" (Replace Workspace Data Help) im Hilfemenü auswählen. Die Onlinehilfe wird in der bevorzugten Sprache, die Sie in Ihrem Browser festgelegt haben, angezeigt, sofern übersetzte Hilfe verfügbar ist. Gewährleisten Sie, dass Sie die neuesten Sprachpakete für Ihre bevorzugte Sprache installiert haben. Standardmäßig wird die Hilfe in englischer Sprache angezeigt.

Wenn Sie im Hilfemenü "Info" (About) auswählen, wird das Popup "Hilfe zum Ersetzen von Arbeitsbereichsdaten" (About Replace Workspace Data) angezeigt. Im Popup "Hilfe zum Ersetzen von Arbeitsbereichsdaten" (About Replace Workspace Data) werden Produkt- und Versionsinformationen zum Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten angezeigt. Sie können das Popup "Hilfe zum Ersetzen von Arbeitsbereichsdaten" (About Replace Workspace Data) schließen, indem Sie auf die Schaltfläche "Schließen" (Close) klicken.

Wenn Sie im Dienstprogramm auf das Informationssymbol neben einem Feld klicken, wird ein Popup mit ausführlichen Informationen zum Feld angezeigt. Wenn Sie die Maus über ein Feld mit einem Informationssymbol bewegen, werden dieselben Informationen als QuickInfo angezeigt.

Hinweis: Die MDB-Basisdatei befindet sich im Ordner "Base Metadata" unter "InSite Administration" und muss bei jeder Verwendung des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) kopiert werden.

Ausführen des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) über die Eingabeaufforderung

Sie können das Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) über die Eingabeaufforderung ausführen. Geben Sie folgenden Befehl ein, um das Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) aufzurufen:

"ReplaceWorkspaceData.exe -log:[logfilepath] -base:[baseMDBpath] -new:[newMDBpath] -current:[currentMDBpath] -target:[targetMDBpath] -replace_current_MDB:[True/False]"

Die einzelnen Parameter werden durch ein Leerzeichen getrennt und beginnen mit einem Gedankenstrich. Die Werte, mit denen Sie die Variablen in Klammern ersetzen, sind identisch mit den Werten, die von den Feldern im Fenster des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) erwartet werden. Informationen hierzu finden Sie unter "Felddefinitionen im Fenster "Arbeitsbereichsdaten ersetzen" (Replace Workspace Data)".

Diese Tabelle definiert die Informationen, mit denen Sie die Variablen in den eckigen Klammern in der Befehlszeile ersetzen müssen.

Klammer

Geben Sie ein . . .

-log:[]

Der vollständige Dateipfad für die Protokolldatei.

-base:[]

Der vollständige Dateipfad für die Kopie der InSite.mdb-Basismetadatendatei.

-new:[]

Der vollständige Dateipfad für die neue InSite.mdb-Metadatendatei.

-current:[]

Der vollständige Dateipfad für die Kopie der aktuellen InSite.mdb-Datei.

-target:[]

Der vollständige Dateipfad für die Datenbank, in der das Ergebnis des Zusammenführungsprozesses gespeichert wird.

-replace_current_MDB:[]

"True", damit die im Feld "Aktuelle Metadaten" (Current Metadata) angegebene Datei umbenannt und durch die im Feld "Zielmetadaten" (Target Metadata) angegebene Datei ersetzt wird.

Geben Sie "False" ein, wenn die Zielmetadatendatei erstellt, zusammen mit der Protokolldatei überprüft und nach der Überprüfung manuell umbenannt werden soll.

Nachdem Sie die Befehlszeile eingegeben und die Eingabetaste gedrückt haben, wird eine Statusanzeige angezeigt. Nachdem das Dienstprogramm den Arbeitsbereich ersetzt hat, wird eine Erfolgsmeldung angezeigt.

Wenn das Dienstprogramm einen Fehler feststellt, werden eine Fehlermeldung und Hilfeinformationen angezeigt. Sie können auch "help" eingeben und die Eingabetaste drücken, um Hilfeinformationen anzuzeigen. Sie können die Informationen überprüfen und anschließend versuchen, das Dienstprogramm auszuführen, indem Sie den Befehl erneut eingeben und die Eingabetaste drücken.

Felddefinitionen im Fenster "Arbeitsbereichsdaten ersetzen" (Replace Workspace Data)

In dieser Tabelle sind die Felder im Fenster "Arbeitsbereichsdaten ersetzen" (Replace Workspace Data) definiert.

Feld

Definition

Typ

Protokolldatei (Log File)

Die Textdatei mit den Ergebnissen der Zusammenführung des neuen Arbeitsbereichs in den bestehenden Arbeitsbereich.

Erforderlich

Camstar-Basisarbeitsbereichsdaten (Base Camstar Workspace Data)

Die Datenbank mit dem CSI-Arbeitsbereich. Sie wird als Ausgangspunkt für jede Zusammenführung verwendet. Hierbei handelt es sich um die Kopie der InSite.mdb-Basismetadatendatei, die Sie im neuen Ordner für die Zusammenführung von Arbeitsbereichsdateien gespeichert haben.

Hinweis: Die InSite.mdb-Basismetadatendatei muss bei jeder Verwendung des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) kopiert werden.

Erforderlich

Version

Die MDB-Version des Exemplars der InSite.mdb-Basismetadatendatei, die Sie im Feld "Camstar-Basisarbeitsbereichsdaten" (Base Camstar Workspace Data) angegeben haben.

Nur Anzeige

Neue Arbeitsbereichsdaten (New Workspace Data)

Die Datenbank mit den aktualisierten Arbeitsbereichsdatensätzen. Diese MDB kann mehrere Arbeitsbereiche enthalten. Es wird nur der aktuelle Arbeitsbereich verarbeitet.

Hinweis: Diese Datei ersetzt die MDB-Basisdatei mit Arbeitsbereichsdaten, die im Feld "Camstar-Basisarbeitsbereichsdaten" angegeben ist, bevor die Zusammenführung beginnt, wenn "csi" der aktuelle Arbeitsbereich in dieser MDB ist.

Erforderlich

Version

Zeigt die MDB-Version der im Feld "Neue Arbeitsbereichsdaten" (New Workspace Data) angegebenen Datei an.

Nur Anzeige

Arbeitsbereich (Workspace)

Der aktuelle Arbeitsbereich (der an erster Stelle) in der Metadatendatenbank des neuen Arbeitsbereichs. Dieses Feld wird bei der Aktualisierung auf den Arbeitsbereichsnamen "csi" festgelegt.

Nur Anzeige

Aktuelle Metadaten (Current Metadata)

Die Datenbank mit der aktuellen Implementierung, die aktualisiert wird.

Erforderlich

Version

Zeigt die MDB-Version der im Feld "Aktuelle Metadaten" (Current Metadata) angegebenen Datei an.

Hinweis: Die in diesem Feld angezeigte Version wird nicht mit der neuesten MDB-Version aktualisiert, wenn Sie das Kontrollkästchen "Aktuelle nach Abschluss ersetzen" (Replace Current When Done) aktivieren. Wenn Sie den Dateipfad erneut eingeben oder auf die Schaltfläche "Durchsuchen" (Browse) klicken und zu dem Dateipfad navigieren, wird wieder die neue Version angezeigt.

Nur Anzeige

Arbeitsbereich (Workspace)

Der aktuelle Arbeitsbereich (der an erster Stelle) in der aktuellen Metadatendatenbank.

Nur Anzeige

Zielmetadaten (Target Metadata)

Die Datenbank mit dem Ergebnis der Zusammenführung. Speichern Sie die Datei im neuen Ordner für die Zusammenführung von Arbeitsbereichsdaten und weisen Sie einen eindeutigen Namen zu (z. B. "UpgradedInSite.mdb").

Erforderlich

Aktuelle nach Abschluss ersetzen (Replace Current When Done)

Kontrollkästchen, das festlegt, ob die im Feld "Aktuelle Metadaten" (Current Metadata) angegebene Datei umbenannt und durch die im Feld "Zielmetadaten" (Target Metadata) angegebene Datei ersetzt wird.

Optional

Vorgehensweise zum Ausführen des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data)

Führen Sie die folgenden Schritte aus, um das Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) auszuführen:

1. Klicken Sie auf Start > Opcenter Execution MDD.

Oder

Klicken Sie auf Start > Opcenter Execution Core.

2. Klicken Sie mit der rechten Maustaste auf Arbeitsbereichsdaten ersetzen und wählen Sie Mehr > Als Administrator ausführen aus. Das Fenster "Arbeitsbereichsdaten ersetzen" (Replace Workspace Data) wird angezeigt.
3. Geben Sie den vollständigen Dateipfad für die Protokolldatei in das Feld Protokolldatei (Log File) ein.

Oder

Klicken Sie auf die Schaltfläche Durchsuchen (Browse), um zum Speicherort zu navigieren.

4. Geben Sie den vollständigen Pfad für die Kopie der InSite.mdb-Basismetadatendatei in das Feld Camstar-Basisarbeitsbereichsdaten (Base Camstar Workspace Data) ein.

Oder

Klicken Sie auf die Schaltfläche Durchsuchen (Browse), um zum Speicherort zu navigieren.

Die MDB-Version wird im Feld "Version" angezeigt.

5. Geben Sie den vollständigen Pfad für die neue InSite.mdb-Datei in das Feld Neue Arbeitsbereichsdaten (New Workspace Data) ein.

Oder

Klicken Sie auf die Schaltfläche Durchsuchen (Browse), um zum Speicherort zu navigieren.

Die MDB-Version wird im Feld "Version" angezeigt, und der aktuelle Arbeitsbereich wird im Feld "Arbeitsbereich" angezeigt.

6. Geben Sie den vollständigen Dateipfad für die Kopie der aktuellen Datei "InSite.mdb" in das Feld Aktuelle Metadaten (Current Metadata) ein.

Oder

Klicken Sie auf die Schaltfläche Durchsuchen (Browse), um zum Speicherort zu navigieren.

Die MDB-Version wird im Feld "Version" angezeigt, und der aktuelle Arbeitsbereich wird im Feld "Arbeitsbereich" angezeigt.

7. Geben Sie den vollständigen Pfad für die Datenbank mit dem Ergebnis der Zusammenführung in das Feld Zielmetadaten (Target Metadata) ein.
8. Geben Sie abhängig von den jeweiligen Geschäftsanforderungen optionale Informationen ein. Informationen zu den optionalen Feldern finden Sie in Tabelle mit den Felddefinition.
9. Klicken Sie auf Aktualisieren (Upgrade). Der Fortschritt des Ersetzungsvorgangs wird in der Statusanzeige angezeigt. Nachdem die Ersetzung durchgeführt wurde, wird ein Dialogfeld eingeblendet.
10. Klicken Sie auf OK. Das Popup wird geschlossen.
11. Klicken Sie auf Schließen (Close). Das Fenster "Arbeitsbereichsdaten ersetzen" (Replace Workspace Data) wird geschlossen.

Vorgehensweise zum Ausführen des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) über die Eingabeaufforderung

Führen Sie die folgenden Schritte aus, um das Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) über die Eingabeaufforderung auszuführen:

1. Klicken Sie auf Start > Windows-System > Eingabeaufforderung. Das Eingabeaufforderungsfenster wird angezeigt.
2. Geben Sie cd C:\Programme (x86)\Camstar\InSite Conversion Tool\Merge Tool ein und drücken Sie die Eingabetaste.
3. Geben Sie ReplaceWorkspaceData.exe -log:[logfilepath] -base:[baseMDBpath] -new:[newMDBpath] -current:[currentMDBpath] -target:[targetMDBpath] -replace_current_MDB:[True/False] ein. Ersetzen Sie dabei die Informationen in Klammern durch die vollständigen Dateipfade für die .mdb-Dateien und geben Sie "True" oder "False" anstelle der in Klammern gesetzten replace_current_MDB-Informationen ein. Informationen zum Eingeben von Parametern im Eingabeaufforderungsfenster finden Sie unter "Ausführen des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) über die Eingabeaufforderung".
4. Drücken Sie die Eingabetaste. Der Fortschritt des Ersetzungsvorgangs wird in der Statusanzeige angezeigt. Nachdem die Ersetzung durchgeführt wurde, wird eine entsprechende Meldung angezeigt.
5. Klicken Sie auf die Schaltfläche Schließen (Close), um das Eingabeaufforderungsfenster zu schließen.
