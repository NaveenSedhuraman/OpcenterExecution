Übersicht

Die Verwendung des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) umfasst mehrere Phasen.

Wichtig: Weitere Informationen, wenn Medical Device bereits installiert war, finden Sie in "Aktualisieren einer vorhandenen Medical Device-Installation".

Die einzelnen Phasen werden nachstehend beschrieben. Die einzelnen Schritte der jeweiligen Phasen werden in den folgenden Themen erklärt.

Phase 1. Beenden der Anwendungspools und Dienste

Sie müssen die Anwendungspools und Dienste beenden, bevor Sie das Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten ausführen.

Phase 2. Kopieren von InSite.mdb-Dateien zum Zusammenführen von Arbeitsbereichsdaten

Erstellen Sie einen Ordner für die Dateien, die beim Zusammenführen von Arbeitsbereichsdaten verwendet werden. Kopieren Sie die InSite.mdb-Dateien für aktuelle Metadaten und Basismetadaten aus den jeweiligen Ordnern in diesen Ordner und benennen Sie sie um, damit sie vom Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten als Eingabedateien verwendet werden können.

Phase 3. Ausführen des Dienstprogramms zum Ersetzen von Arbeitsbereichsdaten

Mit dem Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) können Sie einen bestehenden Arbeitsbereich ersetzen oder einem bestehenden Arbeitsbereich einen Arbeitsbereich hinzufügen, um die Arbeitsbereichsdaten zusammenzuführen.

Phase 4. Umbenennen der aktualisierten InSite.mdb-Datei und Ersetzen der bestehenden Datei

Benennen Sie die aktualisierte InSite.mdb-Datei, die Sie mit dem Dienstprogramm zum Ersetzen von Arbeitsbereichsdaten (Replace Workspace Data) erstellt haben, in den Standardnamen um und kopieren Sie sie an den Standardspeicherort.

Phase 5. Aktualisieren der Transaktionsdatenbank

Aktualisieren Sie die Transaktionsdatenbank, damit die aktualisierte InSite.mdb-Datei verwendet wird.

Phase 6. Neugenerieren der WCF-Dienste nach der Aktualisierung der Datenbank

Bei Änderungen an der Datei "InSite.mdb" muss der WCF-Dienstgenerator ausgeführt werden.

Phase 7. Starten der Anwendungspools und Dienste

Sie müssen die Anwendungspools und Dienste starten.
