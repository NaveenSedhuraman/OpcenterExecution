Starting Services and Application Pools

You must start services and application pools.

Note: This topic covers starting all product-related application pools in IIS and all product-related services in Windows Administrative Tools. The Opcenter EX MDD and Opcenter EX CR servers are registered as services. You can view the status of these services (except CMSAdmin and the Siemens PLM License Server) on the Services pane in Management Studio. You can also start, stop, and restart them from the Services pane. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information.

How to Start the Siemens PLM License Server

Follow these steps to start the Siemens PLM License Server:

1. Click Start > Windows Administrative Tools > Services. The Services window appears.
2. Select Siemens PLM License Server in the list, right-click, and select Start.

How to Start Application Pools

Follow these steps to start application pools in IIS:

1. Click Start > Server Manager. The Server Manager window appears.
2. Select Internet Information Services (IIS) Manager from the Tools menu. The Internet Information Services (IIS) Manager window appears.
3. Expand the server in the Connections pane.
4. Click Application Pools. The Application Pools window appears.
5. Select each of the following application pools individually and select Start:
• CamstarSecurityServices
• CMSAdmin
• CamstarAppServer
• Camstar WCF Services
• Portal
Note: You can perform an iisreset to start all of these application pools.

How to Start Services

Follow these steps to start the other services:

1. Click Start > Windows Administrative Tools > Services. The Services window appears.
2. Select each of the following services individually in the list, right-click, and select Start:
• Camstar Deployment Server
• Camstar Notification Server
