Running the Replace Workspace Data Utility

The Replace Workspace Data utility enables you to merge workspaces from several locations into a single location.

When Running the Replace Workspace Data Utility

The utility is used to either add or update a single workspace in the Target MDB.

You must replace workspace data in sequence, working from lowest to highest. For example, you would start with the base csi workspace (sequence 1) and run the utility for the Site workspace (sequence 200) last. You can view your active workspaces and their sequences in Management Studio. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on workspaces.

The utility renames and replaces the file specified in the Current Metadata field with the file specified in the Target Metadata field when the Replace Current When Done check box is selected. The utility creates the file specified in the Target Metadata field when the Replace Current When Done check box is cleared. You must rename the file manually after reviewing it in conjunction with the Log File.

Note: You do not need to change the active workspace if you copied the InSite.mdb file from the Base Metadata folder because the InSite.mdb file in this folder is in the csi workspace by default.

The progress indicator displays the progress of the replacement process. The progress indicator appears when you click the Upgrade button.

The utility displays a File not found pop-up message if any of the required fields are left blank. You can click OK on the pop-up message to return to the Replace Workspace Data window.

Accessing Help and More Information

The utility displays an error message if an error occurs during the replacement process. You can click Continue to attempt to continue the replacement. You can click Quit to close the error message and the utility. Errors are logged in the Event Viewer and in the log file generated by the utility.

The Help menu provides access to online help for the utility. You can access online help by selecting Replace Workspace Data Help from the Help menu. Online help appears in the preferred language set in your browser if translated help is available. Ensure you have the latest language packages installed for your preferred language. Help appears in English by default.

The About Replace Workspace Data pop-up appears when you select About from the Help menu. The About Replace Workspace Data pop-up displays product and version information about the Replace Workspace Data utility. You can close the About Replace Workspace Data pop-up by clicking the Close button.

Clicking the information icon next to a field in the utility displays a pop-up containing detailed information about the field. You can also hover over any field with an information icon to view the same information as a tooltip.

Note: The Base MDB is found in the Base Metadata folder in InSite Administration and must be copied each time you use the Replace Workspace Data utility.

Running the Replace Workspace Data Utility from the Command Prompt

You can run the Replace Workspace Data utility from the command prompt. You call the Replace Workspace Data utility using the following command:

"ReplaceWorkspaceData.exe -log:[logfilepath] -base:[baseMDBpath] -new:[newMDBpath] -current:[currentMDBpath] -target:[targetMDBpath] -replace_current_MDB:[True/False]"

Each parameter is separated by a single space and starts with a dash. The values you replace the bracketed variables with are identical to the values expected by the fields on the Replace Workspace Data utility window. Refer to "Replace Workspace Data Window Field Definitions" for information.

This table defines the information you must replace the bracketed variables with in the command line.

Bracket

Enter . . .

-log:[]

The full path location for the log file.

-base:[]

The full path location for the copy of the Base Metadata InSite.mdb file.

-new:[]

The full path location for the new Metadata InSite.mdb file.

-current:[]

The full path location for the copy of the current InSite.mdb file.

-target:[]

The full path location for the database that will contain the result of the merge process.

-replace_current_MDB:[]

"True" to have the file specified in the Current Metadata field renamed and replaced with the file specified in the Target Metadata field.

Enter "False" to have the Target Metadata file created, reviewed in conjunction with the Log File, and renamed manually after the review.

A progress indicator appears after entering the command line and pressing Enter. A success message appears after the utility has replaced the workspace.

An error message and help information appear if the utility encounters an error. You can also enter "help" and press Enter to display help information. You can review the information and then attempt to run the utility by reentering the command and pressing Enter.

How to Run the Replace Workspace Data Utility

Follow these steps to run the Replace Workspace Data utility:

1. Click Start > Opcenter Execution MDD.

Or

Click Start > Opcenter Execution Core.

2. Right-click Replace Workspace Data and select More > Run as administrator. The Replace Workspace Data window appears.
3. Enter the full path location for the log file in the Log File field.

Or

Click the Browse button to navigate to the location.

4. Enter the full path location for the copy of the Base Metadata InSite.mdb file in the Base Camstar Workspace Data field.

Or

Click the Browse button to navigate to the location.

The MDB version appears in the Version field.

5. Enter the full path location for the new InSite.mdb file in the New Workspace Data field.

Or

Click the Browse button to navigate to the location.

The MDB version appears in the Version field and the current workspace appears in the Workspace field.

6. Enter the full path location for the copy of the current InSite.mdb file in the Current Metadata field.

Or

Click the Browse button to navigate to the location.

The MDB version appears in the Version field and the current workspace appears in the Workspace field.

7. Enter the full path location for the database that will contain the result of the merge process in the Target Metadata field.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Upgrade. The progress indicator bar displays the progress of the replacement. A pop-up appears when the replacement process is complete.
10. Click OK. The pop-up closes.
11. Click Close. The Replace Workspace Data window closes.

How to Run the Replace Workspace Data Utility from the Command Prompt

Follow these steps to run the Replace Workspace Data utility from the command prompt:

1. Click Start > Windows System > Command Prompt. The Command Prompt window appears.
2. Enter cd C:\Program Files (x86)\Camstar\InSite Conversion Tool\Merge Tool and press Enter.
3. Enter ReplaceWorkspaceData.exe -log:[logfilepath] -base:[baseMDBpath] -new:[newMDBpath] -current:[currentMDBpath] -target:[targetMDBpath] -replace_current_MDB:[True/False] replacing the bracketed information with the full path locations for the .mdb files and entering True or False in place of the replace_current_MDB bracketed information. Refer to "Running the Replace Workspace Data Utility from the Command Prompt" for information on entering parameters in the Command Prompt window.
4. Press Enter. A progress indicator appears showing the progress of the replacement. A success message appears when the replacement process is complete.
5. Click the Close button to close the Command Prompt window.
