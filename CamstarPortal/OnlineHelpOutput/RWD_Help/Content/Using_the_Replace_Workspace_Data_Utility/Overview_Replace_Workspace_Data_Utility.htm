Overview

Using the Replace Workspace Data utility is a multistage process.

Important: Refer to "Upgrading an Existing Medical Device Installation" for information if you previously installed Medical Device.

Descriptions of the stages follow. The individual steps that comprise the stages are explained in the topics that follow.

Stage 1. Stop Application Pools and Services

You must stop application pools and services before running the Replace Workspace Data utility.

Stage 2. Copy InSite.mdb Files for Merging Workspace Data

Create a folder for the files you will use when merging the workspace data. Copy the current and Base Metadata InSite.mdb files from their respective folders into this folder and rename them so that they can be used as input for the Replace Workspace Data utility.

Stage 3. Run the Replace Workspace Data Utility

Use the Replace Workspace Data utility to replace an existing workspace or add a workspace to an existing workspace to create merged workspace data.

Stage 4. Rename the Upgraded InSite.mdb File and Replace the Existing File

Rename the upgraded InSite.mdb file you created with the Replace Workspace Data utility to the standard name and copy it to the default location.

Stage 5. Update the Transaction Database

Update the transaction database so that it uses the upgraded InSite.mdb file.

Stage 6. Regenerate WCF Services After Updating the Database

You must run the WCF Services Generator when you make changes to the InSite.mdb file.

Stage 7. Start Application Pools and Services

You must start application pools and services.
