Disassociate Containers

The Disassociate transaction removes child containers from parent containers. Child containers are typically disassociated from a parent when separate processing is required, for example when:

• A child container requires rework processing while the remaining child and parent containers do not.
• A child container requires processing through an alternate path.
• A child container must be placed on hold at a step.
• Several child containers require expedited processing to meet customer demand.

When Disassociating Containers

If you are logged on with a Line Assignment, the parent container list is filtered by the values in the Line Assignment.

You can select one or more individual child containers or select all listed child containers to disassociate from the parent container.

When you disassociate one or more child containers, the quantity of the disassociated child containers is subtracted from the quantity of the parent container.

Disassociate Page

The Disassociate page enables you to:

• Select a parent container.
• View the list of child containers associated with the selected parent container.
• Select one or more child containers to disassociate from the parent container.

How to Disassociate Containers

Follow these steps to disassociate containers:

1. Open the Disassociate page from the Container menu. The Disassociate page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Disassociate pop-up from another shop floor page.
3. Select the containers to be disassociated using one of these methods:
• Use the Enter Container to Select field to enter a child container, either by scanning it or by typing the container name or identifier, and then tabbing out of the field. If the entered container is in the Currently Associated Containers grid, the application highlights the row listing the container, and selects the check box for that row.
• Select the check box in the grid next to each child container to be disassociated from the parent container.
• Select the top check box to select all child containers in the grid.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Disassociate All Selected or Submit on the command bar. A status message appears stating the child container or containers are disassociated from the parent container.
