Split Containers

The Split Container transaction transfers one or more child containers from a parent container to a new parent container.

Note: Use the Split Quantity transaction if you want to create one or more target containers from an existing source container and then transfer a quantity of product from the source container to the new target containers.

Examples of Splitting Quantities and Containers

These are some examples of how you use the Split transactions:

• A small quantity of a container is split so it can be processed more quickly to meet an expedited customer or inventory requirement.
• A portion of a container with quality problems is split before performing a Rework or Hold transaction. The remainder of the container is processed normally.
• Part of a container is split so it can be moved to a different workflow for processing a special order.

When Splitting a Quantity or Container

The split container transaction allows you to split parent containers only. You can split the child container to a new parent.

Splitting a container or quantity does not move either the source or target containers. Nor does it imply that any processing activity (throughput) was performed on either container.

You must specify a unique container name when creating a new container. You can either enter a container name manually or have the application generate a container name automatically. The application considers closed containers when validating a unique name.

Select the Generate Names Automatically check box to have the application generate the container name using defined numbering rules. The application clears and disables the To Container field when you select this check box.

An error message appears when Generate Names Automatically is selected but there is no auto numbering rule defined.

The application uses the numbering rules associated with the parent container from which the child container is being split to generate the To Container name. The application generates the name of the new parent container when you click the Submit button.

Numbering rules may be specified for multiple modeling object instances associated with the container. Numbering rules follow this default precedence:

1. Container Level
2. MfgOrder
3. Product
4. Product Family
5. Factory

Inheriting UDI from Split or Parent Containers
Note: This functionality is available only when Medical Device is installed. Refer to "Medical Device Workspace" for information.

A split container can inherit UDI from the container from which it was split, but a child container cannot inherit UDI from the parent.

Inheriting UDI from Split Containers

A split container inherits the device identifier, production identifier, and UDI assigned to the container from which it was split if the split occurs after the UDI is assigned.

Inheriting UDI from Parent Containers

Child containers of a parent do not automatically inherit the device identifier, production identifier, or UDI of that parent. They must have these assigned separately.

Adding Affected Containers to Events

The application can add containers to quality events automatically when you execute the Split Container transaction on a container associated with one or more quality events. For example, if an existing child container is associated with an event, the application adds the new parent container and all sibling containers to the quality event automatically. A system administrator configures the Split transaction to add containers to events in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or the Opcenter Execution Core Event Management User Guide and the Opcenter Execution Medical Device and Diagnostics Nonconformance Management User Guide or the Opcenter Execution Core Nonconformance Management User Guide for information on adding affected containers to quality events automatically.

Split Container Page

How to Split a Container

Follow these steps to split a container:

1. Open the Split Container page from the Container menu. The Split Container page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Split Container pop-up from another shop floor page.
3. Select the check box next to each child container that you want to move to another parent container.
4. Enter the name of the new parent container to which the selected child containers are to be moved in the To Container Name field.

Or

Select the Generate Names Automatically check box.

5. Click Split or Submit on the command bar. The selected child containers are moved to the specified parent container, the application clears the fields, and then displays a confirmation message.
