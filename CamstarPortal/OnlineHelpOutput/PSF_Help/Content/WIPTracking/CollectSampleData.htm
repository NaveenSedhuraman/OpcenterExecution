Collect Sampling Data

The Collect Sampling Data page is used to collect sampling data during production. The page returns all sample tests currently required for a given container based on spec, product, and optionally resource along with associated instructions. When a sample test is selected, current information is displayed including required samples, tested samples, passed samples, failed samples, AQL level, and inspection level.

Once a sample test has been selected, you can then collect the test data.

If the sample test is a:

• Measured sample type - you enter the numeric value for each data point
• Counted sample type - you record the number of rejected samples for each reason code

When a sample test is executed, the total tested samples, passed samples, and failed samples are recorded and the container quantity is decreased by the sample size or by the reject count as required.

When Collecting Sampling Data

These rules apply when collecting sampling data:

• Only containers that have sample tests and are not associated with a lot appear in the Container selection list. Line assignment values are used to filter the container list based on Operation, Workstation, Resource, and Work Center.
• You cannot move a container if sampling is required at the container's current spec until all required samples have been collected.
• You cannot move a container if sampling is in process.
• A container is not allowed to move if sampling fails unless specified during modeling.

Adding Collect Sampling Data to an Electronic Procedure

An electronic procedure is a revisionable object that enables you to assign a task list containing a collect sampling data task to a specification (Spec). You must define a task list containing a transaction task specifying the CollectSamplingDataVP form before assigning the task list to the electronic procedure. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining task lists, electronic procedures, and specs.

Collect Sampling Data Page

You can access the Collect Sampling Data page from the Container menu. The Collect Sampling Data page data collection area has two appearances depending on whether the data is measured or counted.

How to Execute a Measured Sample Data Collection

Follow these steps to execute a measured sample data collection:

1. Open the Collect Sampling Data page from the Container menu. The Collect Sampling Data page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
3. Select a measured sample test from the list in the Container Sample Data grid.
4. Enter the number of samples that you want to collect for this sample test in the Number of Samples to Test field to determine the layout of the data collection grid for measured sample tests.
5. Click the Generate button to create the number of samples designated in the Number of Samples to Test field. The application hides the Number of Samples to Test field and the Generate button, and displays the Measured grid.
6. Enter the data point value for each sample test in the measured grid, if necessary.
7. Click Submit or Submit on the command bar. The application keeps the current container selected and updates the Sample Test grid's status.

How to Execute a Counted Sample Data Collection

Follow these steps to execute a counted sample data collection:

1. Open the Collect Sampling Data page from the Container menu. The Collect Sampling Data page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container.Values appear only if they were entered for the container.
3. Select a counted sample test from the list in the Container Sample Data grid.
4. Enter the number of containers to be sampled in the Samples Tested field.
5. Enter the number of samples that failed inspection for each reject code in the Qty column of the Sample Test Counted grid.
6. Click Submit or Submit on the command bar. The application keeps the current container selected and updates the Sample Test grid's status.
