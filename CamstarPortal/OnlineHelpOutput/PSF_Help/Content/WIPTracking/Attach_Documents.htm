Attaching Documents

You can attach documents using the Attach Document page, the Manage Attachments page, and the Attach Documentcommand bar button. The Attach Document and Manage Attachments pages are available from the Attachments menu. You can also attach documents from the Start page, the Order Dispatch page, and the Mfg Order page. Refer to "Start Containers" for information on attaching documents on the Start page. Refer to "Using Order Dispatch" for information on attaching documents on the Order Dispatch page. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on attaching documents on the Mfg Order page.

You can attach documents using the Manage Attachments page with or without performing a document search.

Clicking the Attach button on the Manage Attachments page or clicking the Attach Document command bar button on a shop floor page displays the Attach Document pop-up. The Attach Document pop-up contains the same fields and functionality as the Attach Document page.

You can attach three types of documents:

• A new document that will not be reused
• A new document that will be reused
• An existing document

The application allows special characters in document names. The maximum size for an attachment is 30 MB.

Clicking Attach on the Attach Document page or the Attach Document pop-up attaches the document to the object. You can search for, detach, and view the attached document. Refer to “Managing Attachments” for information.

Attaching a New Document that Will Not Be Reused

The Attach a new document that will not be reused (no Revision needed) option attaches a new document that is unique to the object instance. This type of document is considered an attachment and cannot be assigned to other objects in the future. Examples include test results and failure mapping.

You do not need to specify a revision for this attachment type. The application assigns a revision when the document is attached.

Attaching a New Document that Will Be Reused

The Attach a new document that will be reused (Revision must be specified) option attaches a new document that is not unique to one object instance. This option creates a document instance that can be maintained in the application and can be assigned to other objects in the future.

This attachment type appears in Modeling and can be manipulated using the Document page. Examples include customer-specific instructions or research and development processing instructions.

You must provide a revision for this attachment type.

The modeling object always references the document's revision flagged as the revision of record (ROR) if the ROR check box is selected. This means that all objects flagged to use the ROR of that document are affected, if the ROR revision changes for a currently attached document. The object always references the specific revision if the ROR check box is not selected. A change to the document's revision of record does not affect this object.

A new revision for the document cannot be added and marked as the ROR from this option if a document's ROR is currently attached to an object. You must do this directly from the Document page. You can use this option to attach the ROR document once the new ROR document revision has been created through modeling. Refer to "Attaching an Existing Document" for information on attaching existing documents.

Attaching an Existing Document

The Attach an existing document (select below) option attaches a document that was created as a reusable document. Refer to "Attaching a New Document that Will Be Reused" for information on creating reusable documents.

You can attach the document's ROR or a specific revision when attaching existing documents.

The object always uses the document's revision flagged as the ROR, if the revision of record (ROR) document is selected. This means if the ROR revision changes for a document, then the object will reference the new ROR revision. The object always references the specific revision, if a specific revision is selected; therefore, changes to the document's ROR do not affect this object.

You can view an existing document before attaching it to an object. You must have the appropriate software to view the document. For example, you must have Microsoft Word to view a Word document.

How to Attach a Document

Follow these steps to attach a document:

1. What method do you want to use to attach a document?

If you want to attach a document . . .

Then . . .

Using the Attach Document page
a. Open the Attach Document page from the Attachments menu. The Attach Document page appears.
b. Select an object type in the Object Type field. The application displays the Container field if the object type selected is Container. Otherwise, the application displays the Instance Name field. The application enables the Attach button.
c. Select an instance from the list in the Instance Name field.

Or

Select a container in the Container field.

d. Go to step 2.

Using the Manage Documents page with a document search
a. Open the Manage Attachments page from the Attachments menu. The Manage Attachments page appears.
b. Make a selection in at least one field. The application activates the Search button. Refer to "Manage Attachments Page Field Definitions" for information on field selection requirements.
c. Click Search. The application populates the Attached Documents grid.
d. Select a row in the Attached Documents grid.
e. Click Attach. The Attach Document pop-up appears.
f. Go to step 2.

Using the Manage Documents page without a document search
a. Open the Manage Attachments page from the Attachments menu. The Manage Attachments page appears.
b. Click Attach. The Attach Document pop-up appears.
c. Go to step 2.

Using the Attach Document command bar button
a. Open a container transaction page. The container transaction page appears.
b. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container.Values appear only if they were entered for the container.
c. Click the Attach Document command bar button. The Attach Document pop-up appears with the Container field populated.
d. Go to step 2.
2. What type of document do you want to attach?

If you want to attach . . .

Then . . .

A new document that will not be reused
a. Select Attach a new document that will NOT be reused (no Revision needed). The Document Path, Document Name, and Document Description fields appear.
b. Click the Document Path button to select a file.
c. Enter a name for the document in the Document Name field.
d. Go to step 3.

A new document that will be reused
a. Select Attach a new document that will be reused (Revision must be specified). The Document Path, Document Name, Revision, Revision of Record, and Document Description fields appear.
b. Click the Document Path button to select a file.
c. Enter a name for the document in the Document Name field.
d. Enter the revision of this document in the Revision field.
e. Select the check box next to the revision optionally to indicate this document is the revision of record.
f. Go to step 3.

An existing document
a. Select Attach an existing document (select below). The Document Instances, Revision, Revision of Record fields, and View Document button appear.
b. Select a document in the Document Instances field. The application enables the View Document button.
c. Enter the specific revision in the Revision field or select the Revision of Record check box.
d. Click View Document optionally if you want to view the document before you attach it. The application displays a message asking you to open the document, save it, or cancel the request.
e. Go to step 3.
3. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
4. Click Attach. The application displays a message indicating the document has been attached successfully.
