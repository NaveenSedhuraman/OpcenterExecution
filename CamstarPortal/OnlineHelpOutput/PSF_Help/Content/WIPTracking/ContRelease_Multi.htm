Release Multiple Containers

The Release Multiple containers transaction enables you to release multiple held containers simultaneously so that normal processing can resume. You can perform transactions against containers that have been released.

When Releasing Multiple Held Containers

You must select a release reason to release a container. Release reasons are defined in Modeling by a system administrator. The application displays an error message if you click the Release Selected button without selecting a release reason. The application displays that reason by default if only one release reason is defined. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on release reasons.

The application validates that the specified containers are active and are held.

Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

Release (Multiple) Page

You can access the Release (Multiple) page from the Container menu. You can access the Release (Multiple) pop-up from another shop floor page (for example, Container Search) if the Release (Multiple) action has been configured for that page. Both the Release (Multiple) page and pop-up allow you to scan or enter one or more containers to release.

The page and pop-up are identical in appearance and functionality with one exception. The application populates the Release (Multiple) pop-up automatically with the containers you selected on the shop floor page from which you accessed the pop-up.

How to Release Multiple Held Containers

Follow these steps to release multiple held containers:

1. Open the Release (Multiple) page from the Container menu. The Release (Multiple) page appears.
2. Are you viewing the Release (Multiple) page or the Release (Multiple) pop-up?

If you are viewing the . . .

Then . . .

Release (Multiple) page

The Containers to Release grid is empty. Go to step 3.

Release (Multiple) pop-up

The Containers to Release grid is populated with the containers you selected on the page from which you launched the pop-up. If you want to add additional containers to release, go to step 3. Otherwise, go to step 6.
3. Scan or enter a container in the Enter Container to Select field.
4. Press Enter or tab out of the field. The container is added to the Containers to Release grid and selected automatically.
5. Repeat steps 3-4 to add additional containers.
6. Ensure the selected containers are the ones you want to release.
7. Select a Release Reason.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Release Selected or Submit on the command bar. The application displays a status icon for each container indicating whether the transaction was executed successfully.
10. Close the pop-up, if you are viewing it, and return to the previous page.
