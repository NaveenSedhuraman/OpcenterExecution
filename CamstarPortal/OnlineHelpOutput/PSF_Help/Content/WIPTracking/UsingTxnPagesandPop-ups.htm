Using Transaction Pages and Pop-ups

Container-related transactions can be executed from pages and pop-ups. Transaction pages are accessed using the Container menu and the transaction pop-ups are accessed from the Operational View and Container Search pages. You must select a container, click the Shop Floor Txn command bar button, and select an available transaction to launch a transaction pop-up.

Note: The Operational View and Container Search pages are the only pages configured for transaction pop-ups by default. Your system administrator can configure additional pages to display transaction pop-ups based on your organization's business needs.

Additionally, Siemens provides transaction pages that allow you to execute transactions for multiple containers. The Container Search page includes the ability to display a multi-container transaction pop-up when multiple containers are selected. Refer to "Multi-Container Transactions Overview" for information on multi-container transactions.

Resource-related transactions can be executed from pages only. The resource pages are accessed using the Resource menu.

Note: You can enter data into fields on shop floor pages by scanning bar codes from a mobile device if you are licensed for bar code scanning. Refer to the Opcenter Execution Medical Device and Diagnostics Getting Started User Guide or the Opcenter Execution Core Getting Started User Guide for information.

When Using a Transaction Page

When the application displays a transaction page, you must select a Container or Resource before the application will populate any predefined information and enable the command bar. For example, if you select Move from the Container menu, the application displays a blank Move page. When you select a Container, the application populates the container information section with any values that were specified previously for the container and enables the command bar.

Note: Multi-container transaction pages do not have a container information section.

The application clears the fields on the page and displays a confirmation message when the transaction is successful. The application displays an error message on the page if a transaction fails. You can either correct the values or navigate away from the page.

When Using a Transaction Pop-up

Transaction pop-ups are applicable for containers only. A transaction pop-up displays the container name and information for the container selected on the page from which you launched the pop-up. The Container field is read-only. If you want to perform the transaction on a different container, you must close the pop-up and select that container.

Note: Multi-container transaction pop-ups do not have a container information section.

When the transaction is successful, the application closes the pop-up and displays a confirmation message on the page from which the pop-up was launched. If the transaction fails, the application displays an error message on the pop-up. You can either correct the values or close the pop-up.

When Initiating a Transaction from the Operational View Page

The transactions you can initiate on the Operational View (with grids) page for a selected container depend on:

• Whether the container is In Queue or In Process
• The actions configured for those pages by default and by a system administrator
• The status of the container

For example, the Move In transaction is available only for an In Queue container, while the Move transaction is available only for an In Process container. Some transactions, such as Change Qty, may be available for both In Queue and In Process containers.

You must select a container to enable the Shop Floor Txns command bar button. Clicking an action displays the transaction pop-up for that container.

When Initiating a Transaction from the Container Search Page

You must perform a search on the Container Search page before you can select containers in the Search Results grid. The transactions you can initiate on the selected containers depend on:

• The actions configured for those pages by default and by a system administrator
• The status of the container
• Whether you have selected one or more containers in the Search Results grid

You must select at least one container to enable the Shop Floor Txns command bar button. If you have selected more than one container, the Shop Floor Txns command bar button displays actions that can be performed on the selected containers simultaneously. Clicking an action displays the transaction pop-up for the selected containers.
