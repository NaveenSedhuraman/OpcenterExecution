Multi-Container Transactions Overview

Siemens provides six Portal shop floor transactions that allow you to process one or more containers simultaneously:

• Close (Multiple)
• Hold (Multiple)
• Move Non-Std (Multiple)
• Open (Multiple)
• Record Production Event
Refer to "Recording a Production Event" or the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or the Opcenter Execution Core Event Management User Guide for information on Record Production Event.
• Release (Multiple)

System administrators can configure additional multi-container transactions based on your organization's business needs.

Note: Multiple container transactions do not support data collection or WIP messaging.

Multi-Container Transaction Pages and Pop-ups

All of the multi-container transaction pages are available from the Container menu. Additionally, the pages may be available as a pop-up if the transactions have been configured as available actions on another shop floor page (for example, Container Search). Unlike other single-container transaction pages and pop-ups, the multi-container transaction pages and pop-ups do not have a container header.

When accessing a multi-container transaction page from the Container menu, all fields are empty until you scan or enter a container. When you scan or enter a container, the application adds it to the grid on the page and selects the container automatically.

The grid name depends on the transaction you want to execute. For example, if you are on the Release (Multiple) page, the grid name is Containers to Release.

The application displays containers in the grid and selects them automatically when accessing a multi-container transaction pop-up from another shop floor page. These are the containers that were selected on the page from which you launched the transaction pop-up.

You can add more containers to the pop-up grid by scanning or entering a container. The application selects the manually added containers automatically. After executing the multi-container transaction, you must click Close to close the pop-up.

Note: Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application will process selected containers only. You can clear the check box for a container if you do not want to execute a transaction against it.

Multi-Container Transaction Modes

Siemens provides two modes for executing a transaction on multiple containers: individual and group. Individual is the default mode. The transaction mode determines how the application functions when a transaction executed against a selected container fails.

An execution failing in individual mode causes the application to:

• Continue processing all selected containers
• Display the success indicator in the status cell of the successfully processed containers
• Display the failed indicator in the status cell of the failed containers
• Display a status message for the failed containers

An execution failing in group mode causes the application to:

• Stop processing immediately
• Not commit any successful transactions
• Display the failed indicator in the status cell of all containers
• Display a status message for the failed containers

Configuring the Transaction Type Mode

The transaction mode for a multi-container transaction is defined for the page in Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information on configuring the transaction mode.

When Executing a Multi-Container Transaction

Executing a transaction on multiple containers causes the application to:

• Display the appropriate indicator for each container
• Retain all processed containers in the containers grid
• Record the transaction in each successfully processed container's history

If you click the transaction execution button again to re-execute the transaction, the application will reprocess only the failed containers. Refer to "Multi-Container Transaction Modes" for information on how the application functions in individual or group mode when you click the transaction button.
