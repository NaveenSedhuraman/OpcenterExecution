Move Multiple Containers (Non-Std)

The Move Non-Std (Multiple) transaction enables you to move multiple containers to any step in any workflow simultaneously.

When Using Move Non-Std for Multiple Containers

The application moves the container to the next step in the current workflow unless you specify a different workflow or step. Workflows are defined in Modeling by a system administrator.

The application validates that the selected containers are active, In Process, and not on Hold. Additionally, a container cannot be a child container.

Refer to "Multi-Container Transactions Overview" for information on how the application functions when executing transactions on multiple containers.

Move Non-Std (Multiple) Page

You can access the Move Non-Std (Multiple) page from the Container menu. You can access the Move Non-Std (Multiple) pop-up from another shop floor page (for example, Container Search) if the Move Non-Std (Multiple) action has been configured for that page. Both the Move Non-Std (Multiple) page and pop-up allow you to scan or enter one or more containers to move.

The page and pop-up are identical in appearance and functionality with one exception. The application populates the Move Non-Std (Multiple) pop-up automatically with the containers you selected on the shop floor page from which you accessed the pop-up.

How to Move Multiple Containers to Any Step in a Workflow

Follow these steps to move multiple containers to any step in a workflow:

1. Open the Move Non-Std (Multiple) page from the Container menu. The Move Non-Std (Multiple) page appears.
2. Are you viewing the Move Non-Std (Multiple) page or the Move Non-Std (Multiple) pop-up?

If you are viewing the . . .

Then . . .

Move Non-Std (Multiple) page

The Containers to Move grid is empty. Go to step 3.

Move Non-Std (Multiple) pop-up

The Containers to Move grid is populated with the containers you selected on the page from which you launched the pop-up. If you want to add additional containers to move, go to step 3. Otherwise, go to step 6.
3. Scan or enter a container in the Enter Container to Select field.
4. Press Enter or tab out of the field. The container is added to the Containers to Move grid and selected automatically.
5. Repeat steps 3-4 to add additional containers.
6. Ensure the selected containers are the ones you want to move.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Move Selected or Submit on the command bar. The application displays a status icon for each container indicating whether the transaction was executed successfully.
9. Close the pop-up, if you are viewing it, by clicking the Close icon to return to the previous page.
