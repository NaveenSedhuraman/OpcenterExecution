Move In Containers

Use the Move In transaction to indicate that a specific container has arrived at a step and that processing on it has begun. The Move In transaction enables the application to calculate how long the container has been in the queue before it was processed. Requiring a Move In transaction at an operation is optional. If a Move In transaction is required, it must be performed on each container before the container is moved out.

When Using Move In

All display fields are empty on the Move In page until a container is selected. Display fields are populated automatically on the Move In pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application displays a default resource for the specified container when these conditions are true:

• A resource was specified when the container was started
• The Move In transaction is the first transaction (specifying a resource) you are performing after the start

You can select a different resource, if necessary.

The application uses the resource you specify when moving in a container as the default resource at the in process step. The application displays the specified resource when you perform one of these transactions at the in process step:

• Move
• Move Non-Std
• Rework
• Transfer

You can select a different resource if necessary.

When Moving In a Container with an Open Production Event

Note: This functionality is available only as part of the separately licensed Nonconformance Management module.

A setting in the operation determines whether the move succeeds or fails when moving in a container that has an open production event (nonconformance). If the Reject Incoming Nonconformance Container check box is selected for an operation, the application prevents a user from moving in containers with open nonconformances to that operation. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining operations.

When Moving In a Container with Automatic Material Consumption Enabled

Note: This functionality is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can select the Move In transaction in the Consume Material Txns field on the Factory page to indicate material loaded on a resource is automatically consumed when a container moves through the resource. You can also select the Move In transaction in the Validate Material Queue Txns field to enable the application to verify the correct material is loaded on a resource for a container moving through the resource.

The application returns a warning message if you select the Move In transaction but do not select the Validate Material Queue Qty check box. The transaction fails if the quantity is not sufficient if you select the Move In transaction and select the Validate Material Queue Qty check box. You can select an Issue Difference Reason to prevent a failure from occurring when partial quantities are issued during automatic component issue.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining factories.

Move In Page

You can access the Move In page from the Container menu. The Move In page allows you to enter or select the container you want to move in for processing.

Note: The Move In page may be available as a pop-up also if the move in transaction has been configured as an available action on another page (for example, Container Search).

Move In Page and Pop-up Field Definitions

This table defines the fields on the Move In page and Move In pop-up.

Field

Definition

Type

Container Information

Header containing the container name and information about that container. You must select or enter a container on a transaction page, but all information is display-only on a transaction pop-up. Refer to "Container Information" for information.

Required/ Display Only

Resource

Resource that will process the container. The application defaults to this resource for transactions performed at the in process step when relevant.

The resource values available for selection depend on the resource group assigned to the spec (the step where the container is). If no resource group was assigned to the spec, all defined resources are available for selection.

Note: If the availability of the specified resource is set to Down, the application will display an error message indicating the resource is unavailable when you perform the transaction.

Optional

Location

Destination location within the factory for the container. A location may be an entire factory, an area of a factory, or as specific as a rack identifier. Locations are defined within the Factory object in Modeling.

Note: This field is hidden by default.

Optional

Clear Location

When selected, indicates you want to clear the physical location of the container once it is moved in to be processed by the resource.

Note: This check box is hidden by default.

Optional

Comments

Additional information or instructions for this transaction or container. The field accepts a maximum of 255 characters. The application truncates the data if the comments exceed 255 characters.

Optional

How to Move in a Container and Begin Processing

Follow these steps to move in a container and begin processing:

1. Open the Move In page from the Container menu. The Move In page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Move In pop-up from another shop floor page.
3. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
4. Click Move In or Submit on the command bar. A status message appears confirming that the container has been moved in.
