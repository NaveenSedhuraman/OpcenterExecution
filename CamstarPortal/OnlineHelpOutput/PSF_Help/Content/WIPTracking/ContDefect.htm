Container Defect

The Defect transaction allows you to record material defects that do not immediately result in removal or scrap. You can record the number of containers and/or the quantity of the container inspected. You can also create a list of defect codes and defect counts for the defects you observed.

Note: Use the Component Defect transaction if you want to record defects against components issued to a container.

Before Using Container Defect

You must configure container defect reasons in Modeling before you can record defects against containers. The defect reasons are available for selection under these conditions:

• A container defect reason has been defined.
• A container defect reason group containing the reason has been defined.
• The container defect reason group has been assigned to the container's current operation.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining and associating the objects necessary to record container defects.

When Using Container Defect

All display fields are empty on the Defect page until a container is selected. Display fields are populated automatically on the Container Defect pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

You can record defects against child containers. When you record defects for child containers, the application adds the defect to the child container's history only. The parent container's history does not include child container defects.

You can record defects for held containers, but you cannot record a defect for closed containers. The application does not make closed containers available for selection.

The application requires you to add at least one defect reason and enter a count for that reason before you can record a defect. The application displays an error message if you click the Record Defects button without having added a defect reason and defect count.

Clicking the Delete selected row button removes the selected defect reasons from the grid. You can select the check box in the header row to select all displayed defects automatically or you can select the check boxes for individual defects.

Defect Page

You can access the Defect page from the Container menu. The Defect page allows you to record a list of defects against the selected container.

Note: The Defect page may also be available as a pop-up if the container defect transaction has been configured as an available action on another page (for example, Container Search).

How to Record Container Defects

Follow these steps to record container defects:

1. Open the Defect page from the Container menu. The Defect page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Container Defect pop-up from another shop floor page.
3. Click Add new row in the grid. A row is activated in the grid.
4. Select a container defect reason from the Defect Reason field.
5. Enter the number of defects associated with this reason in the Defect Count field.
6. Repeat steps 3-5 to add additional container defect reasons.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Record Defects or Submit on the command bar to save the list of defects you entered against the container. The application displays a status message confirming that defects were saved.

How to Delete Defect Reasons

Follow these steps to delete defect reasons:

1. Open the Defect page from the Container menu. The Defect page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
3. Select the check box for the defect reason you want to remove from the grid.
4. Repeat step 3 for each defect reason you want to remove. If you want to remove all defect reasons currently displayed in the grid, select the check box in the header row.
5. Click Delete selected row in the grid. The selected defect reasons are removed from the grid.
