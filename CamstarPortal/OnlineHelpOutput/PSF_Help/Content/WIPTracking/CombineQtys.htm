Combine Quantities

The Combine Qty transaction transfers quantity from one or more existing containers to another container. A typical use of the Combine Qty transaction is to recombine containers that were split previously using the Split Quantity transaction.

A Combine transaction does not move either the source or target containers. Nor does it imply that any processing activity (throughput) was performed on either container. However, the Combine Qty transaction does allow you to close the containers from which you are transferring the quantity.

Combine Container and Combine Qty Eligibility Rules

The containers in a Combine transaction must meet these eligibility criteria:

• Neither of the containers being combined can be closed or on hold.
• Target and source containers must contain the same product and be at the same container level.
• Target and source containers must be at the same operation in the same factory.
• The transaction must occur before the expiration dates if expiration dates were entered for the source (From) container or the target (To) container. The Combine Qty transaction will fail if the expiration date of either container occurs before you execute the transaction. If only one container has an expiration date, the transaction must occur before that date.

When performing the Combine Container transaction, all source containers should have child containers.

Important: Combine Quantity does not permit selection of panels or other containers with children.

When Combining Quantities

All display fields are empty on the Combine Qty page until a container is selected. Display fields are populated automatically on the Combine Qty pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application displays all source containers eligible for combination with the target container in the Containers to Combine grid when you select a target container in the header. You can select the check box in the header row to select all source containers automatically, or you can select the check boxes for individual containers. Additionally, you can scan or enter the source container in the Enter Container to Select field and tab out of the field, and the application will locate and select that container in the Containers to Combine grid.

Note: The application displays an error message if you scan or enter the name or identifier of an ineligible container or a container that does not exist in the Enter Container to Select field.

You must select at least one container in the Containers to Combine grid to perform the Combine Qty transaction. The application displays an error message if you click the Combine Selected button without selecting a container in the grid. The application combines only the source containers selected in the grid.

When specifying a quantity to combine:

• You cannot combine the quantity of a container to itself.
• The amount to transfer cannot be greater than the amount in the source container.
• Selecting the Close When Empty check box closes the source container automatically when its quantity reaches zero.
• The target container is set to the earlier expiration date used by the two containers. If only the source container has an expiration date, then the expiration date of the target container is set to the expiration date of the source container

When a combine is performed, the following events take place:

• The designated quantity from one container (source) is transferred to another existing container (target).
• The quantity added from the source container assumes the attributes of the target container.

Adding Affected Containers to Events

The application can add containers to quality events automatically when you execute the Combine Qty transaction on a container associated with one or more quality events. For example, if a source container is associated with an event, the application adds the target container to the quality event automatically. A system administrator configures the Combine Qty transaction to add containers to events in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or the Opcenter Execution Core Event Management User Guide and the Opcenter Execution Medical Device and Diagnostics Nonconformance Management User Guide or the Opcenter Execution Core Nonconformance Management User Guide for information on adding affected containers to events automatically.

Combine Qty Page

You can access the Combine Qty page from the Container menu. The Combine Qty page allows you to transfer the quantities of product from one or more source containers to another existing target container.

Note: The Combine Qty page may be available as a pop-up also if the combine qty transaction has been configured as an available action on another page (for example, Container Search).

How to Combine Quantities

Follow these steps to combine quantities:

1. Open the Combine Qty page from the Container menu. The Combine Qty page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container. Additionally, the application displays all containers eligible to be combined with the target container in the Containers to Combine grid.
Note: The application populates the Container field and the container header automatically when you access the Combine Qty pop-up from another shop floor page.
3. Select the source container to combine with the target container using one of the following methods:
• Select the check box in the header row of the Combine Containers grid to select all containers.
• Select the check box for each individual container.
• Scan or enter a container in the Enter Container to Select field and tab out of the field. The application highlights the container row and selects the container's corresponding check box.
4. Click inside the Qty to Combine column for the selected container to modify the quantity if necessary. The application activates the cell when you click in the column.
5. Repeat steps 3-4 until you have selected every source container you want to combine with the target container.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Combine Selected or Submit on the command bar. The application transfers the specified quantities from the source containers to the target container and displays a success message.
