Understanding Smart Scanning

Smart scanning is available on all Portal pages. The feature allows the application to populate one or more fields when you scan a barcode. For example, scanning a barcode on a container could populate not only a Container field, but also any fields displaying information about that container. A barcode contains one or more of the following:

• A preamble, a single character indicating whether the barcode is a smart barcode. When the application detects a preamble, it identifies the barcode as smart and strips the preamble before reading the smart tag.
• A smart tag, a prefix attached to a scan string indicating entity type; for example, container. The application uses the smart tag to locate the field to populate with the entity name.
• The entity name.

A smart barcode will always contain an entity name, preamble, and smart tag. Preambles and smart tags are defined in the Portal Studio SmartScanningSettings section. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information on the settings in Portal Studio and configuring preambles and smart tags.

When Scanning a Barcode

When a shop floor operator scans a barcode, the application loads the entity name, preamble, and smart tag from the Portal Studio SmartScanningSettings. If the current page is smart scan-enabled and the scanned barcode contains a smart tag, the application populates not only the entity name but also any dependent fields for which values have been defined previously for that entity. If the barcode contains a smart tag, the barcode is considered a smart barcode.

Note: How the application populates the fields on a page depends on whether the page is Smart Scan-enabled and whether the barcode is smart. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information on enabling the smart scanning feature.

When a Page Contains Multiple Fields of the Same Type

You can define custom smart tags to distinguish between fields when a page contains multiple fields.

Cursor Placement

When a barcode is scanned with a terminator (for example, a horizontal tab, set by the scanning device), the application moves the cursor to the next logical field after populating the scanned field regardless of whether the barcode is Smart (contains a preamble) or Not Smart. Dependent fields are resolved the same as they are for values entered manually. If the barcode is scanned without a terminator, the application does not move the cursor to the next field and the barcode type (Smart or Not Smart) determines whether the application populates the field.

If the barcode is Not Smart, the application populates the field the cursor is on with the entity name and the cursor sits at the end of the last character. If the barcode is Smart, the application populates the field when the focus is removed from the target field. Regardless of barcode type, the focus may be removed automatically by the scanning device if a terminator has been configured for the device or manually by tabbing off of the target field.

Note: The terminator coming from the scanning device must match the terminator value defined in the Portal Studio SmartScanningSettings sections. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information on Smart Scanning Settings and defining terminators. Refer to the documentation supplied with the device for information on configuring a terminator for a scanning device.
