Hold Multiple Containers

The Hold (Multiple) transaction enables you to put multiple containers on hold simultaneously for a specific reason (for example, a scheduled inspection or audit). Placing containers on hold prevents users from performing transactions on the containers. The container can be released to resume normal processing when the reason for holding the container no longer exists.

When Placing Multiple Containers on Hold

You must select a hold reason to place a container on hold. Hold reasons are defined in Modeling by a system administrator. The application displays an error message if you click the Hold Selected button without selecting a hold reason. The application displays that reason by default if only one hold reason is defined.

The application validates that the specified containers are active and are not on hold already. You cannot place a closed container on hold.

Placing a container on hold does not place its children on hold. However, placing a child container on hold places the parent container on an implicit hold. You can resume processing on the parent container only after you release the held child container.

Refer to "Multi-Container Transactions Overview" for information on how the application functions when executing transactions on multiple containers.

Hold (Multiple) Page

You can access the Hold (Multiple) page from the Container menu. You can access the Hold (Multiple) pop-up from another shop floor page (for example, Container Search) if the Hold (Multiple) action has been configured for that page. Both the Hold (Multiple) page and pop-up allow you to scan or enter one or more containers and place those containers on hold.

The page and pop-up are identical in appearance and functionality with one exception. The application populates the Hold (Multiple) pop-up automatically with the containers you selected on the shop floor page from which you accessed the pop-up.

How to Place Multiple Containers on Hold

Follow these steps to place multiple containers on hold:

1. Open the Hold (Multiple) page from the Container menu. The Hold (Multiple) page appears.
2. Are you viewing the Hold (Multiple) page or the Hold (Multiple) pop-up?

If you are viewing the . . .

Then . . .

Hold (Multiple) page

The Containers to Hold grid is empty. Go to step 3.

Hold (Multiple) pop-up

The Containers to Hold grid is populated with the containers you selected on the page from which you launched the pop-up. If you want to add additional containers to hold, go to step 3. Otherwise, go to step 6.
3. Scan or enter a container in the Enter Container to Select field.
4. Press Enter or tab out of the field. The container is added to the Containers to Hold grid and selected automatically.
5. Repeat steps 3-4 to add additional containers.
6. Ensure the selected containers are the ones you want to hold.
7. Select a Hold Reason.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Hold Selected or Submit on the command bar. The application displays a status icon for each container indicating whether the transaction was executed successfully.
10. Close the pop-up, if you are viewing it, and return to the previous page.
