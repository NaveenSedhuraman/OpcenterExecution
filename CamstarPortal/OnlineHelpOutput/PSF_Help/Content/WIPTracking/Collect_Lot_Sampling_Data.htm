Collect Lot Sampling Data

You can use the Collect Lot Sampling Data page to collect sampling data for a lot. The page returns all sample tests currently required for a given container's lot along with associated instructions. When a sample test is selected, the lot's current information is displayed including required samples, tested samples, passed samples, failed samples, AQL level, and inspection level.

Once a sample test has been selected, you can then collect the sample test data for the lot's container.

If the sample test is a:

• Measured sample type - you enter the numeric value for each data point
• Counted sample type - you record the number of rejected samples for each reason code
When Collecting Lot Sampling Data

These rules apply when collecting lot sampling data:

• Only containers that have sample tests and are associated with a lot appear in the Container selection list. In Queue containers must be processed by performing a Move In transaction. Line assignment values are used to filter the container list based on Operation, Workstation, Resource, and Work Center.
• You cannot move a container if sampling is required at the container's current spec until at least one sample has been collected. The Sampling Required check box on the container list is checked for the containers that require sampling. These containers appear at the top of the list. The containers that do not require sampling are available for random sampling.
• All sample data for a lot does not have to be collected at one time. The application accumulates inspection data for a lot as sampling data for a lot container is collected for each sample test in a sampling plan.
• The Collect Lot Sampling Data page can be launched from an electronic procedure or as a standalone shop floor form.

Adding Collecting Lot Sampling Data to an Electronic Procedure

An electronic procedure is a revisionable object that enables you to assign a task list containing a collect lot sampling data task to a specification (Spec). You must define a task list containing a transaction task specifying the CollectLotSamplingData_VP form before assigning the task list to the electronic procedure. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining task lists, electronic procedures, and specs.

Collect Lot Sampling Data Page

You can access the Collect Lot Sampling Data page from the Container menu. The Collect Lot Sampling Data page data collection area has two appearances depending on whether the data is measured or counted.

How to Execute a Measured Lot Sample Data Collection

Follow these steps to execute a measured lot sample data collection:

1. Open the Collect Lot Sampling Data page from the Container menu. The Collect Lot Sampling Data page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
3. Select a measured sample test from the list in the Lot Sample Data grid.
4. Enter the number of samples that you want to collect for this sample test in the Generate Samples for the above Sample Test field to determine the layout of the data collection grid for measured sample tests.
5. Click Generate. The application hides the Generate Samples for the above Sample Test field and the Generate button, and displays the measured grid.
6. Enter the data point value for each sample test in the measured grid, if necessary.
7. Click Submit or Submit on the command bar to record the data for the sample test. The application keeps the current container selected and updates the Lot Sample Data grid's status.

How to Execute a Counted Lot Sample Data Collection

Follow these steps to execute a counted lot sample data collection:

1. Open the Collect Lot Sampling Data page from the Container menu. The Collect Lot Sampling Data page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
3. Select a counted sample test from the list in the Lot Sample Data grid.
4. Enter the number of containers to be sampled in the Samples Collected field.
5. Enter the number of samples that failed inspection for each reject code in the Count field.
6. Click Submit or Submit on the command bar. The application keeps the current container selected and updates the Lot Sample Data grid's status.
