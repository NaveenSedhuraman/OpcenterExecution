When Dispatching Containers

You can access the Order Dispatch page from the Container menu. Use the Order Dispatch page to start a container against a selected manufacturing order.

You must specify a unique container name when creating a new container. You can either enter a container name manually or have the application generate a container name automatically. The application considers closed containers when validating a unique name.

Select the Generate Names Automatically check box to have the application generate the container name using the defined numbering rules. The application clears and disables the Container field when you select this check box.

An error message appears when Generate Names Automatically is selected but there is no auto numbering rule defined.

The application generates the name of the new container when you click the Submit button.

Numbering rules may be specified for multiple modeling object instances associated with the container. Numbering rules follow this default precedence:

1. Container Level
2. MfgOrder
3. Product
4. Product Family
5. Factory

You can attach documents to a manufacturing order or to the container being started for a manufacturing order from the Order Dispatch page and then manage them from the Manage Attachments page. Refer to "Attaching Documents" for information on the options for attaching documents and "Managing Attachments" for information on managing documents. Attachments are optional. However, the application requires you to complete certain fields when attaching files before you start the container.

Note: The Order Dispatch page may be available as a pop-up also if the order dispatch transaction has been configured as an available action on another page (for example, Container Search).
