Container Release

The Release transaction enables you to release a held container so that normal processing can resume. You can perform transactions against containers that have been released.

Note: Refer to "Release Multiple Containers" if you want to release multiple held containers simultaneously.

When Releasing a Container

All display fields are empty on the Release page until a container is selected. Display fields are populated automatically on the Release pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

Only one container can be released per transaction. However, releasing a child container also releases the parent container for processing if the parent has no other child containers on hold.

You may select a release reason as part of releasing a container. Release reasons are defined in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on release reasons.

Release Page

You can access the Release page from the Container menu. The Release page allows you to enter or select a held container and release that container.

Note: The Release page may also be available as a pop-up if the container release transaction has been configured as an available action on another page (for example, Container Search).

How to Release a Container

Follow these steps to release a container:

1. Open the Release page from the Container menu. The Release page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Release pop-up from another shop floor page.
3. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
4. Click Release or Submit on the command bar. The application displays a status message confirming the container has been released.
