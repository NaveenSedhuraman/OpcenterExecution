Unique Device Identification Overview

Note: This functionality is available only when Medical Device is installed. Refer to theOpcenter Execution Medical Device and Diagnostics Medical Device User Guide or the Opcenter Execution Core Medical Device User Guide for information.

Unique Device Identification (UDI) is a requirement by the Food and Drug Administration (FDA) to uniquely identify medical devices, combination products that contain devices, and devices licensed under the Public Health Service Act. UDI helps the FDA in tracking device production and facilitating device recalls, thereby improving product quality and patient safety.

The UDI functionality provides a mechanism to define and assign UDI to a container. You can use the Spec and Product modeling pages to define UDI information that will propagate to a container. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

You can also add UDI information directly to a container from these transaction pages:

• Start
• Order Dispatch
• Container Maintenance

The Move and Move In transactions can assign UDI to a container automatically if specific information is defined on the spec and product.
