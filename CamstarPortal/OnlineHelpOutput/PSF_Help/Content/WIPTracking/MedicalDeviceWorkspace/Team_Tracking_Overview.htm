Team Tracking Overview

Note: This functionality is available only when Medical Device is configured. Refer to the Opcenter Execution Medical Device and Diagnostics Medical Device User Guide or the Opcenter Execution Core Medical Device User Guide for information.

Team Tracking functionality enables you to record more than one user for a given transaction. You must first define the transactions that require team tracking on either the Factory or Spec object and then assign employees to one of the following work areas:

• Operation
• Resource
• Spec
• Work Center
• Work Cell
• Workstation

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining team tracking functionality on the Factory and Spec modeling objects.

When a transaction that requires team tracking is performed at a work area, all employees assigned to the work area are recorded in history for that transaction.

There are two methods to assign employees to a work area – Employee Sign In and Team Management. The Employee Sign In method allows an individual user to sign into a work area. By default, a user can only sign in to one work area at a time. This can be overridden on the Employee object. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining team tracking on the Employee object. When the user is finished working at the work area, the user signs out. Each employee who must be recorded for the transaction must sign in to the work area.
