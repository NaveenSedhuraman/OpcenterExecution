Change Quantity Multi-Reason

The Change Qty Multi-Reason transaction enables you to change the quantity of product within the container using multiple reasons for a change type within a single transaction.

Note: Refer to "Change Quantity" if you only have a single reason to change the quantity in a container.

Change Quantity Types

This table describes the five types of quantity changes you can specify when changing the quantity in a container.

Adjust

An adjust indicates either an increase or decrease in quantity for an unknown reason. Use this type if, after counting a container’s quantity, the result is not the expected quantity.

Bonus

A bonus indicates an increase in a container’s quantity as a result of a production process. For example, the weight of a product increased as a result of processing.

Buy

A buy indicates an increase in a container’s quantity that is not a result of a production process. For example, certain units were returned by Engineering.

Loss

A loss indicates a decrease in a container’s quantity as a result of a production process.

Sell

A sell indicates a decrease in a container’s quantity that is not a result of a production process. For example, some containers are given up for testing or given to Sales for demo use.

The change quantity types are available for selection under these conditions:

• A reason has been defined.
• A reason group containing the reason has been defined.
• The reason group has been assigned to the container's current operation.
Note: Reasons, Reason Groups, and Operations are defined in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

When Changing a Quantity

All display fields are empty on the Change Qty Multi-Reason page until a container is selected. Display fields are populated automatically on the Change Qty Multi-Reason pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application requires a reason group to be assigned to an operation before you can execute the Change Quantity transaction. If you specify a container on the Change Qty Multi-Reason page or pop-up that is at an operation for which no reason group has been defined, the application displays a warning message accordingly. The application enables the Change Qty button only if a container has been selected and a reason group defined for the current operation.

You can change the quantity of a container using multiple reasons, but you can use only one change type per transaction. The application displays change quantity types in this order:

• Loss, the default selection
• Bonus
• Sell
• Buy
• Adjust

The list may vary depending on whether reason groups have been assigned at the container's current operation. The application condenses the list while preserving the list order if only a few reason groups were assigned to the current operation. The first change type is the default selection. For example, if no Loss Reason group is associated with the current operation, Bonus is the first change type in the list and is the default selection.

The change quantity type you select affects field label names. The names of the Loss Reason and Loss Qty fields change depending on the selection. For example, if you select Bonus, the application renames the fields Bonus Reason and Bonus Qty.

The application requires you to add at least one reason and enter an amount for that reason before you can change the quantity. The application displays an error message if you click the Change Quantity button without having added a change quantity reason and amount.

The application accepts negative numbers in the <Change Type> Qty field. Negative numbers provide the ability to reverse a change quantity transaction. For example, if you enter a loss incorrectly, you can reverse the loss by entering a negative number. Entering negative numbers for Loss and Sell reasons increases the container quantity while entering negative numbers for Bonus, Buy, and Adjust reasons decreases the container quantity.

If the sum of the quantities entered for all quantity change reasons reduces the container's quantity below its current quantity, the application displays an error message when you click the Change Qty button.

Change Qty Multi-Reason Page

You can access the Change Qty Multi-Reason page from the Container menu. The Change Qty Multi-Reason page allows you to enter or select a container, and then change the quantity within the container.

Note: The Change Qty Multi-Reason page may be available as a pop-up also if the change quantity (multiple reason) transaction has been configured as an available action on another page (for example, Container Search).

How to Change the Quantity of a Container Using Multiple Reasons

Follow these steps to change the quantity of a container using multiple reasons:

1. Open the Change Qty Multi-Reason page from the Container menu. The Change Qty Multi-Reason page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container.Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Change Qty Multi-Reason pop-up from another shop floor page.
3. Select a Change Qty Type. The field labels are updated accordingly.
4. Select the row for the desired reason in the <type> Reason grid.
5. Enter the quantity in the <type> Qty field.
6. Enter additional information for the change in the Comments field, if necessary.
7. Repeat steps 3-6 to add additional reasons and quantities to the grid.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Change Qty or Submit on the command bar to change the quantity of the container. A status message appears confirming the quantity was changed for the specified container.
