Open Containers

The Open transaction allows you to open a container that has been closed. Additionally, you can open child containers associated with the selected container.

Note: Refer to "Open Multiple Containers" if you want to open multiple closed containers simultaneously.

When Opening Containers

All display fields are empty on the Open page until a container is selected. Display fields are populated automatically on the Open pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

When selecting a container on the Open page, the application makes only those containers with a status of Closed or In Transit available for selection. If you scan or enter a container with a status other than Closed or In Transit, or enter a container that does not exist, the application displays an error message.

Optionally, you can specify a change status reason and indicate that the application open associated child containers when executing the Open transaction. Change status reasons are defined in Modeling by a system administrator. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on change status reasons.

Process timers stopped on a Close transaction do not resume when the Open transaction is executed.

Note: You cannot use the Open page transaction to start a new container. You must use the Start transaction to start a new container.

Open Page

You can access the Open page from the Container menu. The Open page allows you to enter the name of the container to open or to select the container from a list.

Note: The Open page may also be available as a pop-up if the open container transaction has been configured as an available action on another page (for example, Container Search).

How to Open a Container

Follow these steps to open a container:

1. Open the Open page from the Container menu. The Open page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Open pop-up from another shop floor page.
3. Select Include Children if the container has associated child containers and you want to open them at the same time.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Open or Submit on the command bar. The application displays a status message confirming the container has been opened.
