Executing an Electronic Procedure

If the electronic procedure is using sequential processing, you must complete the task lists in the order they appear. The order was determined when the electronic procedure was defined. Additionally, if a task list is using sequential processing, you must execute the tasks in the order they appear. Task order was determined when the task list was defined.

If line clearance is defined, the application enforces that the first task being executed is a Start Process task. The application displays an error message if the first task in a sequential task list is not a Start Process task. Additionally, if a workstation was specified for the line assignment, the Workstation field displays that value and is read-only.

When the minimum number of iterations have been executed, the application updates the status of the Count in the task tile to green if the task has passed. If the Advance to next task on Min Iterations completion check box was selected for the task when defined, the application displays the next task automatically. If the Advance to next task on Min Iterations completion check box was selected for the last task, the application clears the EProcedure page. Otherwise, focus remains on the current task.

Note: When the electronic procedure and its task lists are non-sequential, the "next" task after the current task is the task after the highest satisfied task. For example, a user completes task 3 in task list 2. Then, the user completes task 5 in task list 1. The application sets the focus on task 4 in task list 2 as the next task.

When Selecting a Workstation

The application requires you to select a workstation when the task type is a Start Process task. Otherwise, a workstation is optional for instruction and computation tasks. Workstations are not applicable for transaction tasks.

Before you can select a workstation, you must assign the workstation or a resource group containing the workstation to the task list and you must set the availability of the workstation to Up. Task lists are defined and resources are set up in Modeling. If you select a workstation that is not available, the application displays an error message indicating the resource is not available. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on task lists.

Note: The application displays a visual cue for the resource indicating that it is Up.

When Performing Instruction Tasks

When the selected task is an instruction task, the application displays fields specific to the task item type in the working pane. For example, if the selected task is a pass/fail instruction task, the application displays Pass and Fail radio buttons. Additionally, Reset and Submit buttons appear in the command bar. Data collection fields also appear in the working pane if a data collection definition was associated with the task.

The Reset button clears only the data entered in the working pane. The Submit button executes the task.

When Performing Computation Tasks

When the selected task is a process computation task, the application displays data collection fields, the computation formula, a Calculate button, and a computation result cell. The application also displays the Reset and Execute Task buttons.

Note: Although a parametric or user data collection is not required when defining a computation task, associating a data collection definition with the task is typical. The associated data collection definition allows you collect the data to use in the computation.

The Reset button clears only the data entered in the working pane. The Submit button executes the task. The Calculate button executes the computation formula and displays the result in the result cell.

Computation Results

The application displays the computation name over the result cell. Additionally, the application may display upper and lower limits to the right of the cell depending on how the computation was defined. If the calculated value falls outside of a defined limit, the application highlights the cell and the violated limit in either yellow or red. The color depends on how the computation was defined also. Computations are defined in Modeling by a system administrator.

If the application highlights the cell and violated limit in yellow, the computation has been defined to allow overrides. When you execute the task, the application accepts the calculated value and marks the task as Passed. If the application highlights the cell and violated limit in red, the computation has been defined to forbid overrides. When you execute the task, the application accepts the calculated value but marks the task as Failed.

Note: The application displays negative limits in parentheses.

When Performing Transaction Tasks

Before you can execute a transaction task, a system administrator must configure the transaction page in Portal Studio for use in an electronic procedure. Refer to "Configuring Shop Floor Pages for Electronic Procedures" for information on configuring a transaction page for electronic procedures.

When the selected task is a transaction task, the application displays the shop floor transaction page in the working pane and the Submit button appears in the command bar. Clicking Submit executes the transaction and the task simultaneously.

Note: Comments entered in the working area for a transaction task apply only to the transaction and not the task itself.

The application displays an error message if the user does not have permission to perform the transaction. If an electronic signature is required to execute the transaction, the application displays the appropriate E-Signature pop-up. Electronic signatures are assigned on the spec.

When Performing Weigh Issue Tasks
Note: This functionality is available only when Batch Processing is installed. Refer to the Opcenter Execution Medical Device and Diagnostics Batch Processing User Guide or the Opcenter Execution Core Batch Processing User Guide for information.

When the selected task is a weigh issue task, the application displays information specific to the component product being weighed and issued to a batch. This information includes the following:

• Name
• Description
• UOM
• Minimum allowed quantity
• Target quantity
• Maximum allowed quantity
• Remaining quantity
• Total quantity issued

The application also displays fields specific to the issue control type specified for the component product. All issue control types provide the functionality to weigh the issued quantity except for the Serial issue type. The Serial issue type consumes the entire container. Refer to "Issue Controls" for information on the types of issue controls.

The Reset button clears only the data entered in the working pane. The Issue Component button issues the component.

When Weighing on the EProcedure Page
Note: This functionality is available only when Batch Processing is installed. Refer to the Opcenter Execution Medical Device and Diagnostics Batch Processing User Guide or the Opcenter Execution Core Batch Processing User Guide for information.

Scales allowed to weigh the component can be specified by assigning a scale group on the following:

• Weigh issue task
• Product
• Product family
• Spec

These items are listed in the order of preference. The scale group assigned on the weigh issue task takes highest preference. You can then populate the issued quantity from the scale. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on assigning scale groups.

When Viewing Scale Quantities
Note: This functionality is available only when Batch Processing is installed. Refer to the Opcenter Execution Medical Device and Diagnostics Batch Processing User Guide or the Opcenter Execution Core Batch Processing User Guide for information.

Scale visualization functionality allows you to view the current weight of the component being issued in comparison with the minimum and maximum allowed quantities. The graphic representation of the scale automatically zooms to display smaller increments when the component amount being issued reaches 20% of the minimum allowed quantity.

When Overriding Scale Information
Note: This functionality is available only when Batch Processing is installed. Refer to the Opcenter Execution Medical Device and Diagnostics Batch Processing User Guide or the Opcenter Execution Core Batch Processing User Guide for information.

You can override information from the scale when performing weighing operations. You can define an Allow Manual Weight Entry Override task in which the application does not display scale-related fields. The user manually enters the component weight in this instance. You can collect an electronic signature for the manual override if the requirement is defined on the weigh issue task.

When Overriding Allowed Tolerances
Note: This functionality is available only when Batch Processing is installed. Refer to the Opcenter Execution Medical Device and Diagnostics Batch Processing User Guide or the Opcenter Execution Core Batch Processing User Guide for information.

The application validates if the total issued quantity is within defined minimum and maximum allowed tolerances after you enter the required information for the component issue. The application requires you to record an issue difference reason when performing the task if the total issued quantity is not within these tolerances. The task fails until the total issued quantity is within the minimum and maximum allowed tolerances. The user is unable to continue to the next task in a sequential task list in this instance.

You can enable a weigh task to pass when the total issued quantity does not fall between the minimum and maximum allowed values by selecting the Allowed Tolerance Override check box. You can require an electronic signature for the tolerance override if the requirement is defined on the weigh issue task. The task passes only if the minimum number of iterations is met.

When Collecting Data for a Task

Data collection is possible for every task type: instruction, computation, transaction, and weigh issue. Depending on how the data collection has been defined, you may have to enter a simple value or an entire grid of data to complete the task. Additionally, the application may display limits for the data collection fields. Data collections are defined in Modeling by a system administrator.

The maximum value that can be displayed is 7 characters, if the Data Collection Definition is using the Row Column format.

The application displays an error when executing a user data collection task with a limit defined by an expression when:

• The expression contains incorrect syntax, or
• The result of the expression does not match the data type.

Contact your System Administrator for assistance with limits.

Note: The application does not display an error message when the data type is string even if the syntax is incorrect. The application displays the expression as the limit on the data collection task.

Instruction Tasks

Data collection fields appear for instruction tasks when these conditions are met:

• The Instruction Type for the selected task is Data Collection.
• The task has an associated parametric or user data collection definition.

Computation Tasks

Data collection fields appear for computation tasks when the task has an associated parametric or user data collection definition.

Transaction Tasks

Data collection fields appear for transaction tasks when specified shop floor transaction performs the data collection.

EProcedure Page

You can access the EProcedure page from the Container menu.

The EProcedure page allows you to do the following:

• Manage task lists for a container
• Execute the individual tasks against a container
• View documents associated with a task

The EProcedure page consists of these parts:

• Task List pane
• Task pane
• Working pane

Task List Pane

The Task List pane appears below the container header on the EProcedure page.

If the number of task lists assigned to the electronic procedure exceeds the visible space provided in the Task List pane, the application enables a scroll bar on the pane to allow navigation.

The application displays instructions (if defined) below the task list name for the selected task list only. If the instructions exceed the space provided, the application displays an Expand button. Clicking the Expand button displays the full text of the instructions. Clicking the Expand button again collapses the instructions.

The application displays the tasks associated with the task list you select in the Task pane.

The Task List Summary button on the command bar launches the Task List Summary pop-up, which provides a vertical list of the assigned task lists.

Task Pane

The Task pane appears on the left of the EProcedure page. The application populates the Task pane with task tiles when you select a task list. The application selects the first uncompleted task automatically. If all tasks have been completed, the application selects the first task in the list.

Each tile includes task information such as sequence number, task name, status, and so on. Refer to "Task List Summary Pop-up Field Definitions" for a description of the information in the Task pane.

Viewing Attachments

The application displays a Documents icon on the task if a document set was assigned to the task. Clicking the icon displays a flyout from the command bar with a list of the attachments in the document set assigned to the task. Selecting a file opens the attachment in a separate window if you have the appropriate software installed.

Clicking Documents in the command bar enables the Document Set slide out panel. The panel contains information on documents assigned to the selected task.

Working Pane

The working pane of the EProcedure page is dynamic, changing to accommodate the instructions and fields for the selected task. A section showing the task instructions appears at the top of the working area. The instructions section is expanded by default to display all the contents. You can click the view/expand icon to collapse the instruction section to four lines.

Note: The instructions section appears only if there are instructions associated with the selected task. Additionally, you can configure the instruction section to display collapsed by default. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Exection Core Portal Studio User Guide for information.

If the content of the working pane exceeds the space provided, the application implements horizontal and vertical scrolling in the working pane as needed.

How to Execute an Electronic Procedure

Follow these steps to execute an electronic procedure:

1. Open the EProcedure from the Container menu. The EProcedure page appears.
2. Enter or select the container for which you want to perform tasks if necessary. The application populates the container header, task list pane, task pane, and the working pane with fields for the default task selected.
3. Select a different Task to execute if necessary.
Note: Your ability to select a different task list or task depends on whether the task list and electronic procedure are enforcing sequential processing.
4. Complete the required fields.
5. Are you executing an instruction, computation, or transaction task?

If you are executing . . .

Then . . .

An instruction or computation task

Click the Submit button. The application displays a success message and updates the status and execution count in the task tile.

A transaction task

Click the Submit button. The application executes the transaction and task simultaneously, displays a success message, and updates the status and execution count in the task tile.
Note: If the Advance to next task on Min Iterations completion check box was selected for the task you are performing, the application will select the next task tile in the Task pane automatically when the minimum number of iterations have been executed. If the electronic procedure and its task lists are non-sequential, the "next" task after the current task is the task after the highest satisfied task. For example, a user completes task 3 in task list 2. Then, the user completes task 5 in task list 1. The application sets the focus on task 4 in task list 2 as the next task.
6. Repeat steps 3-5 to complete the electronic procedure.

How to Weigh and Issue Components on the EProcedure Page

Follow these steps to weigh and issue components on the EProcedure page:

1. Open the EProcedure page from the Container menu. The EProcedure page appears.
2. Select a container in the Container field. The application populates the container header, task list pane, task pane, and the working pane with fields for the default task selected.
3. What type of component issue do you want to perform?

If you want to . . .

Then . . .

Perform a Lot (Bulk) component issue
a. Scan or enter a container in the Scan Container field. The application displays the container's quantity and UOM. The application validates the container's product matches the component product requirement. The application populates the minimum, maximum, target, and remaining qty fields if the Expressions Depend on From Container check box is selected on the Weigh Issue Task.
b. Go to step 4.

Perform a Lot and Stock Point component issue
a. Scan or enter a product in the Scan Product field. The application validates the product matches the component product requirement.
b. Enter the lot number in the Lot Number field.
c. Go to step 4.

Perform a Stock Point Only component issue
a. Scan or enter a product in the Scan Product field. The application validates the product matches the component product requirement..
b. Go to step 4.

Perform a Qty Only component issue
a. Go to step 4.
4. Select a scale in the Scale field. The application enables the Tare button.
5. Click Tare if you want to tare the scale and capture the tare weight.
6. Press the print button on the scale. The application displays the weight unless the Manual Reading Only or the Allow Manual Weigh Override check box is selected on the weight issue task.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Issue Component or Submit on the command bar. The task passes and the application returns a success message.

How to Perform a Serial Component Issue on the EProcedure Page

Follow these steps to perform a serial component issue on the EProcedure page:

1. Open the EProcedure page from the Container menu. The EProcedure page appears.
2. Select a container in the Container field. The application populates the container header, task list pane, task pane, and the working pane with fields for the default task selected.
3. Scan a container in the Scan Container field. The application displays the container's quantity and UOM. The application validates the container's product matches the component product requirement.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Issue Component or Submit on the command bar. The task passes and the application returns a success message.
