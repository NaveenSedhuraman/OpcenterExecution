Change Quantity

The Change Quantity transaction enables you to change the quantity of product within a container. This transaction is ideal when you are changing the quantity for a single reason.

Note: Refer to "Change Quantity Multi-Reason" if you have multiple reasons for changing the quantity in a container.

Change Quantity Types

This table describes the five types of quantity changes you can specify when changing the quantity in a container.

Adjust

An adjust indicates either an increase or decrease in quantity for an unknown reason. Use this type if, after counting a container’s quantity, the result is not the expected quantity.

Bonus

A bonus indicates an increase in a container’s quantity as a result of a production process. For example, the weight of a product increased as a result of processing.

Buy

A buy indicates an increase in a container’s quantity that is not a result of a production process. For example, certain units were returned by Engineering.

Loss

A loss indicates a decrease in a container’s quantity as a result of a production process.

Sell

A sell indicates a decrease in a container’s quantity that is not a result of a production process. For example, some containers are given up for testing or given to Sales for demo use.

The change quantity types are available for selection under these conditions:

• A reason has been defined
• A reason group containing the reason has been defined
• The reason group has been assigned to the container's current operation
Note: Reasons, Reason Groups, and Operations are defined in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

When Changing a Quantity

All display fields are empty on the Change Qty page until a container is selected. Display fields are populated automatically on the Change Quantity pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application requires a reason group to be assigned to an operation before you can execute the Change Quantity transaction. The application displays a warning message if you specify a container on the Change Qty page or pop-up that is at an operation for which no reason group has been defined. The application enables the Change Qty button only if a container has been selected and a reason group defined for the current operation.

You can change the quantity of a container using more than one type and reason, but you can use only one change type and reason per transaction. The application displays the reason automatically when you select a change type for which only one reason is defined.

The application displays change quantity types in this order:

• Loss, the default selection
• Bonus
• Sell
• Buy
• Adjust

The list may vary depending on whether reason groups have been assigned at the container's current operation. The application condenses the list while preserving the list order if only a few reason groups were assigned to the current operation. The first change type is the default selection. For example, if no Loss Reason group is associated with the current operation, Bonus is the first change type in the list and is the default selection.

The change quantity type you select affects field label names. The names of the Loss Reason, Loss Qty, and Loss Qty2 fields change depending on the selection. For example, if you select Bonus, the application renames the fields Bonus Reason, Bonus Qty, and Bonus Qty2.

In the rare event you need to change only the secondary quantity, enter a zero (0) in the primary quantity field and enter the secondary quantity to complete the transaction.

The application accepts negative numbers in the <Change Type> Qty field. Negative numbers provide the ability to reverse a change quantity transaction. For example, if you enter a loss incorrectly, you can reverse the loss by entering a negative number. Entering negative numbers for Loss and Sell reasons increases the container quantity while entering negative numbers for Bonus, Buy, and Adjust reasons decreases the container quantity.

Change Qty Page

You can access the Change Qty page from the Container menu. The Change Qty page allows you to enter or select a container, and then change the quantity within the container.

Note: The Change Qty page may be available as a pop-up also if the change quantity transaction has been configured as an available action on another page (for example, Container Search).

How to Change the Quantity of a Container

Follow these steps to change the quantity of a container:

1. Open the Change Qty page from the Container menu. The Change Qty page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Change Qty pop-up from another shop floor page.
3. Select a Change Qty Type. The field labels are updated accordingly.
4. Select a reason from the <type> Reason field.
5. Enter the quantity in the <type> Qty field.

Or

Select the User Current Qty check box to use all of the container's current quantity.

6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Change Qty or Submit on the command bar to change the quantity of the container. The application displays a status message confirming the quantity was changed for the specified container and updates the Qty value in the header and on the page or pop-up.
