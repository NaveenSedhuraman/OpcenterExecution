Component Issue

The Component Issue transaction enables you to issue raw materials and subassemblies (components) to a container at a specific operation. This transaction also tracks (and enables you to view) material issues by manufacturing step.

The components and issue quantities are defined in a material list, typically the bill of material (BOM) for a container’s product or a container-specific list of materials. Components are issued to a container in accordance with the product’s BOM and choice of issue control (tracking) types.

There are two versions of the Component Issue page. Use the:

• Component Issue - Basic page to issue one component at a time until all required materials have been satisfied.
• Component Issue - Advanced page to queue the issue commands until all required materials are satisfied and then perform a single Issue Component command.

Issue Controls

The application requires selection of an issue control for each component (product or material) added to the materials list when defining a bill of materials (BOM) in Modeling. Issue control types accommodate different ways of supplying needed materials regarding their source (container, product, or substitute product) as well as floor arrangements (stock points). Issue controls specify how the material should be dispensed to containers and how it is tracked.

Siemens supports these types of issue controls.

Issue Type

Description

Issue Container (Lot)

Component material is issued and tracked in Manufacturing at the container or lot (bulk) level. One component container can provide part or all of the required quantity, up to the current quantity of the component container.

Issue Container (Serial)

Component material is issued and tracked as a serialized container in Manufacturing. When a serial container is issued against a material requirement with a Serial issue type, the container’s entire quantity is issued.

Display Only

Component information is displayed to you but you are not required to record information on actual component usage. The Material Consumption Check is not performed for components with an Issue Type of Display Only.

Lot and Stock Point

Component material is issued from lots and locations external to Manufacturing and the material is not tracked by Manufacturing. Lot, quantity, and stock point are recorded. Information is typically transferred to ERP systems using Opcenter Connect MOM (Opcenter CN MOM).

Quantity Only

Component material is not tracked by any system. Quantity and other optional fields are recorded for cost accounting or other purposes.

Stock Point Only

Component material is issued from a location external to Manufacturing and the material is not tracked by Manufacturing. Quantity and stock point are recorded. Information is typically transferred to ERP systems using XML Connect.

When Using Component Issue

All display fields are empty on the Component Issue page (Basic and Advanced) until a container is selected. Display fields are populated automatically on the Component Issue pop-up based on the container selected on another page, such as Operational View. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

When a container is selected, the application populates the Material Requirements grid with a list of components (products or substitute products) required for the container.

Rules

The application allows you to issue substitute products to fulfill a material requirement instead of the specified product if a substitute product has been defined. Valid product substitutions are defined in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

Issue Container (Lot) and Issue Container (Serial) transactions that have expiration dates entered for the source (From) container or target (To) container must occur before the expiration dates. The Component Issue transaction fails if the expiration date of either container occurs before you execute the transaction.

The expiration date for the issue-to container is set to the earlier of the expiration dates of the issue-to and issue-from containers. If only the issue-from container has an expiration date, the issue-to container is set to that same date.

When Using the Material Requirements Grid

You can select the option to Hide satisfied requirements when viewing the Material Requirements grid on the Component Issue page or pop-up. Satisfied requirements include those that were met exactly and those that were exceeded. The application will continue to display partially met requirements and those that have not been processed.

A counter displays the number of satisfied requirements and the total number of requirements. It appears above the right corner of the grid. The counter is helpful when the number of requirements spans more than one page and if you have chosen to hide the satisfied requirements.

When the Material Requirements grid contains multiple entries for the same material, differentiated by a reference designator, you may choose to satisfy a particular requirement first. To do this, select the row, scan or enter the container or product, and continue as usual. If the scanned or entered component does not match the selected row, the application displays an error message.

The Net Qty Required field is updated after each issue. If more than one container or product is entered to satisfy a single requirement, the Net Qty Required field decrements with each successive issue.

When Entering a Component to Issue

You can scan or enter a container or product in the Scan Container/Product field. The application accepts these strings:

• Container name
• Product
• Product name and revision

Entering a value causes the application to:

• Select the first row in the Material Requirements grid containing a component product or substitute product that matches the value entered.
• Populate the read-only fields in the user data entry section.

You can also select a row in the Material Requirements grid before entering a container/product. Then, when you scan or enter the container or product, the application validates that it matches the component requirement you selected in the grid.

Scanning vs. Manual Entry

The application populates the user entry fields and validates the material automatically when you scan a container or product. You can click Issue Component if you do not need to make any changes or complete additional required fields.

When typing a container or product identifier in the Scan Container/Product field, you must press the Tab key or click the cursor outside the Scan Container/Product field to have the application populate and display the other data entry fields and validate the container or product.

Adding Affected Containers to Events

The application can add containers to quality events automatically when you execute the Component Issue transaction on a container associated with one or more quality events. For example, if the component container is associated with an event, the application adds the target container to the quality event automatically. A system administrator configures the Component Issue transaction to add containers to events in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or the Opcenter Execution Core Event Management User Guide and the Opcenter Execution Medical Device and Diagnostics Nonconformance Management User Guide or the Opcenter Execution Core Nonconformance Management User Guide for information on adding affected containers to events automatically.

Component Issue Page

There are two versions of the Component Issue page: Basic and Advanced. Both are accessed from the Container menu.

Component Issue - Basic Page

Use the Component Issue page to enter or select a container, and then issue one component at a time to that container.

Component Issue - Advanced Page

Use the Component Issue - Advanced page to enter or select a container, queue the issue commands until you have satisfied all the required materials, and then perform a single Issue Components command.

The Advanced page has subgrids for each component product, an additional check box, and an additional button. A Kitting Details section is also available to enable component issue for kitted components when Industry Solutions is installed. Also, the Issue Component button is Issue Components on the Advanced page. Otherwise, its appearance and behavior are identical to the Component Issue - Basic page.

How to Issue Components

Follow these steps to issue components:

1. Open the Component Issue page from the Container menu. The Component Issue page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Component Issue pop-up from another shop floor page. Select the first component row that contains an unsatisfied material requirement. The associated fields for the selected component's Issue Control type appear below the grid.
3. Complete the required fields using the following information. Refer to the field definitions for information on the individual fields.

If the Issue Control is . . .

Then . . .

Issue Container (Lot)
a. Verify the amount in the Issue Qty field and enter a different value if necessary.
If the value in the Issue Qty field is greater than or less than the Net Qty Required, the application displays the Issue Difference Reason field.
b. Enter a container, product, or substitute product to be dispensed to the selected container identified in the Container field.
c. Select a reason if the Issue Difference Reason field appears.
d. Go to step 4.

Issue Container (Serial)
a. Select a reason if the Issue Difference Reason field appears.
b. Enter a container, product, or substitute product to be dispensed to the selected container identified in the Container field.
c. Go to step 4.

Lot and Stock Point
a. Enter the product lot number in the Lot Number field.
b. Verify the amount in the Issue Qty field and enter a different value if necessary.
If the value in the Issue Qty field is greater than or less than the Net Qty Required, the application displays the Issue Difference Reason field.
c. Select a reason if the Issue Difference Reason field appears.
d. Go to step 4.

Stock Point Only
a. Verify the amount in the Issue Qty field and enter a different value if necessary. If the value in the Issue Qty field is greater than or less than the Net Qty Required, the application displays the Issue Difference Reason field.
b. Select a reason if the Issue Difference Reason field appears.
c. Go to step 5.

a. Verify the amount in the Issue Qty field and enter a different value if needed.
b. Go to step 5.

Basic version of the page or pop-up
a. Click Issue Component.
The application displays a status message indicating the component was issued successfully, and removes all field entries. The application also updates the Material Requirements grid as follows:
• Decrements the Net Qty Required field.
• Increments the Qty Issued field.
• Sets the Issue Status column to Satisfied.
• Changes the Qty Issued column to green to reflect that the requirement has been satisfied exactly.
b. Repeat steps 2-4 to issue another component to the container until you have finished.

Advanced version of the page or pop-up

Go to step 5.
5. Select the Auto Add to Pending when Net Qty satisfied check box if you want the application to queue each entered component automatically and each component will fully satisfy the requirement. The application adds the component to the queue. The Issue Status changes to Pending.

Or

Clear the Auto Add to Pending when Net Qty satisfied check box if you want to be able to edit the Qty Issued and Issue Difference Reason fields because one or more of the entered components will not fully satisfy the requirement. Then, click the Add to Pending button to add the component to the queue. The Issue Status changes to Pending.

6. Click Issue Components or Submit on the command bar. The application displays a status message indicating the components were issued successfully, removes all field entries, and resets the label on the Scan Container/Product field. The application also updates the Material Requirements grid as follows:
• Decrements the Net Qty Required field.
• Increments the Qty Issued field.
• Sets the Issue Status column to Satisfied.
• Changes the Qty Issued column to green to reflect that the requirement has been satisfied exactly.
