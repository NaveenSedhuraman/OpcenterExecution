Remove Components

The Component Remove transaction allows you to remove components (products and substitute products) that have been issued to a container. You also use this transaction to indicate a disposition for the removed components.

When Removing Components

All display fields are empty on the Component Remove page until a container is selected. When you select a container, the application updates the header and populates the Material Issues grid with all components issued to the container.

You can remove only one component at a time. The application updates the Material Issues grid to remove the component when the remove transaction is successful. If the transaction fails, the application displays an error message accordingly.

When you scan or enter a container, product, or substitute product in the Scan Container/Product to Select field, the application identifies and selects the container or product in the Material Issues grid. If there is not a container or product in the Material Issues grid matching the one you entered, the application displays an error message.

The application requires you to scan or enter a valid container for the issue types of Container (Lot) and Container (Serial). Additionally, if you choose to use Destination Location/Stock Point you must be associated with a factory that specifies a location. The location you specify when removing the component does not have to be the location from which it originated.

Scanning vs. Manual Entry

The application populates the user entry fields and validates the material automatically when you scan a container or product. You can click Remove if you do not need to make any changes or complete additional fields.

Note: The application populates fields and validates the material automatically upon scanning only if a terminator has been configured for the device and in Portal Studio. Otherwise, you must tab out of the Scan Container/Product to Select field before the application populates the fields and validates the material. Refer to the documentation supplied with the device for information on configuring a terminator for a scanning device. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on configuring terminators in Portal Studio.

You must leave the Scan Container/Product to Select field after manual entry before the application validates the field and displays other data entry fields and validates the container or product.

When Using the Material Issues Grid

Your selection in the Material Issues grid determines the user data entry fields displayed below the Material Issues grid and their availability. For example, if you select a component with the issue control type of Container (Serial), the application selects the Remove All Qty check box, populates the Qty Removed field with the Remaining Qty to Issue value, and disables the Qty Removed field.

Note: The application behavior is the same if you select the Remove All Qty check box manually for any other selected component.

Adding Affected Containers to Events

The application can add containers to quality events automatically when you execute the Component Remove transaction on a container associated with one or more quality events. For example, if the source container is associated with an event, the application adds the component container to which the components are removed to the quality event automatically. A system administrator configures the Component Remove transaction to add containers to events in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or Opcenter Execution Core Event Management User Guide and the Opcenter Execution Medical Device and Diagnostics Nonconformance Management User Guide or the Opcenter Execution Core Nonconformance Management User Guide for information on adding affected containers to quality events automatically.

Component Remove Page

You can access the Component Remove page from the Container menu. The Component Remove page allows you to view the components issued to a container and to remove those components from the container.

Note: The Component Remove page may be available as a pop-up also if the component remove transaction has been configured as an available action on another page (for example, Container Search).

How to Remove Issued Components

Follow these steps to remove issued components:

1. Open the Component Remove page from the Container menu. The Component Remove page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container and displays all components issued to the container.
Note: The application populates the Container field and the container header automatically when you access the Component Remove pop-up from another shop floor page.
3. Scan or enter the container, product, or substitute product you want to remove from the selected container. The application selects the component in the Material Issues grid and updates the user data entry fields accordingly.

Or

Select the component you want to remove in the Material Issues grid. The application selects the component in the Material Issues grid and updates the user data entry fields accordingly.

4. Select a Removal Reason.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Remove or Submit on the command bar. The application removes the component from the container, displays a success message, and removes the component from the Material Issues grid.
7. Repeat steps 3-6 to remove additional components.
