Container Maintenance

Use the Container Maintenance transaction to select and update an eligible container’s predefined, static container attributes, set in Designer. In the case of parent or child containers, you can change the properties of a parent container and its children at the same time.

Note: Use the Container Attribute Maintenance page to define and manage dynamic, user-defined attributes for individual containers.

When Using Container Maintenance

The only required fields on the Container Maintenance page are Level, Owner, and Start Reason.

If the value of a given attribute for the child containers is different from the parent container, the value for the child containers is not recorded in the history. However, the history of the parent container appears if the Show Full History check box is selected in the Mfg Audit Trail.

You cannot use this page to change the container's current step in the workflow (Location is display only), or to adjust the quantity of the container. You can change the UOM associated with the container.

Assigning Unique Device Identification (UDI) on the Container Maintenance Page
Note: This functionality is available only when Medical Device is installed. Refer to "Medical Device Workspace" for information.

You can use the Container Maintenance page to assign or modify a device identifier, a production identifier, or UDI to an existing container. The last value entered or automatically calculated is the one recorded for the container. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on modeling device identifiers, production identifiers, and UDI.

Container Maintenance Page

The Container Maintenance page enables you to:

• Enter or select a container.
• View information for the container.
• Access the container's history.
• Change the information in certain fields such as Level, Owner, and Start Reason.

How to Change Container Attributes

Follow these steps to change container attributes:

1. Open the Container Maintenance page from the Container menu.  The Container Maintenance page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Container Maintenance pop-up from another shop floor page.
3. Change container-related values as needed. Refer to the field definitions table for information on these fields.
4. Click Update Container or Submit on the command bar. All fields are cleared and a status message appears stating the transaction was successful. Or, if you originated this transaction from the Container Maintenance pop-up, the Operational View page appears with updated information. The Container history is updated.
