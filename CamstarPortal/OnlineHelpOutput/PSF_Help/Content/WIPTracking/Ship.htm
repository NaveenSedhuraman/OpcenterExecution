Ship Containers

The Ship page and Ship pop-up allow you to ship a container to a remote site such as a customer or distributor, or to another factory, location, or workflow step.

When Shipping Containers

All display fields are empty on the Ship page until a container is selected. Display fields are populated automatically on the Ship pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application may populate these fields when you select a Shipment Destination if the information was provided in the Shipment Destination modeling definition.

• To Customer
• To Location
• To Factory

You can change the information populated by the application, if necessary. Additionally, you can ship the container to a workflow. You must specify a step in the workflow.

Note: Locations are associated with a factory. Therefore, you must have locations defined for the factory in Modeling, or they will not be available in the To Location list. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on locations and factories.

Shipping Restrictions

Any of these conditions prevent the container from being shipped:

• The container is In Queue. A container must be in process before you can ship it.
• The status of the container is On Hold or Closed.
• The container is an associated child container. You must disassociate the child container before you can ship it separately from the parent container.
• All materials have not been consumed. If the Validate Material Consumption check is enabled for the spec, the application validates that all materials and quantities have been consumed for the current step before allowing the container to move to the next step in the workflow.
• The container is at the last step in the workflow. The container is moved to the next step in the workflow whenever you ship. Therefore, you will receive an error if you are at the last step in the workflow.

Ship Page

You can access the Ship page from the Container menu. The Ship page allows you to enter or select a container and select the customer, location, factory, or workflow step to which you want to ship the container.

How to Ship a Container

Follow these steps to ship a container:

1. Open the Ship page from the Container menu. The Ship page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Ship pop-up from another shop floor page.
3. Select a Shipment Destination for the container. The application updates the optional fields to display any information that was specified for the Shipment Destination object in Modeling.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Ship or Submit on the command bar. The application displays a status message confirming the container was shipped.
