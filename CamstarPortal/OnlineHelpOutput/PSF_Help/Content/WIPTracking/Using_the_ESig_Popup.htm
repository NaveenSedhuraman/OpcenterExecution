Using the ESignature Pop-up

The ESignature pop-up appears automatically if:

• The application has been configured to capture electronic signatures when a user executes a transaction.

Or

• A transaction attempts to stop a process timer that has an ESig Requirement and is outside of its limits.

The appearance of the pop-up depends upon the shop floor page or pop-up from which it was accessed.

If the same electronic signature requirement is required multiple times for the same transaction, they are combined and presented as a single requirement on the ESignature pop-up

If you access the ESignature pop-up from:

• A multi-container transaction page or pop-up- the application adds additional fields for adding signatures to multiple containers.
• A single transaction page or pop-up- the pop-up has fewer fields and you can add signatures only for the container specified on the transaction page or pop-up.

When Adding Signatures to the ESignature Pop-up

Clicking the Add Signature button validates that the entered signer and cosigner information meet the transaction's electronic signature requirements.

The application validates the signatures to ensure these conditions are met:

• The signer or cosigner is a valid employee and has provided a valid password
• The role or cosigner role defined in the electronic signature requirement is valid, meaning the role is:
• Associated with the signer or cosigner
• Part of the ESig role group assigned to the signer or cosigner

If the signatures are valid, the application adds the information to the row selected in the child grid, selects the next child row automatically, and places the cursor in the Signer field for the next signature.

Capturing all signatures for the selected signature meaning causes the application to:

• Change the background color of the completed meaning to green
• Select the next meaning in the grid, if applicable
Important: If you need to modify a signature, you must delete the signature from the meaning and then re-add it with the new information.

Invalid Information Entry

If you entered an incorrect ID or password, the application displays an error message on the ESignature pop-up. You can either correct the information and add the signatures or close the ESignature pop-up. If you close the pop-up without making corrections, the transaction will not be completed.

When Removing Signatures

The application provides two ways to remove signatures from the Electronic Signature Entry grid. Clicking the:

• Clear Selected Signatures button removes all selected signatures from the currently selected signature requirement.
• Reset button removes all signatures from the grid.

When Submitting Electronic Signatures

Submitting captured signatures causes the application to verify:

• All signatures for each signature requirement have been added.
• The same user did not sign multiple times for the same signature requirement. However, cosigners can fulfill multiple signatures as long as that cosigner has the appropriate roles.

If the requirements are met, the ESignature pop-up closes and transaction processing proceeds. If not, the application leaves the pop-up open and displays an error message.

ESignature Pop-up Button Definitions

This table defines the buttons on the ESignature pop-up.

Button

Click this button to . . .

Add Signature

Add the signer's and/or cosigner's signature to the child row selected in the Electronic Signature Entry grid. If you entered an incorrect ID or password, the application displays an error message immediately.

Close

Close the ESignature pop-up without submitting signatures. The application clears all signatures from the electronic signature requirement.

Clear Selected Signatures

Delete the selected signatures from the Electronic Signature Entry grid. The application removes the signatures from the selected child rows.

Reset

Delete all signatures from the Electronic Signature Entry grid. The application removes the signatures from the child rows.

Submit Signatures

Submit the signatures, close the ESignature pop-up, and complete the transaction. The application allows you to submit signatures and complete the transaction only if all signature requirements have been satisfied. If you click Submit Signatures without completing all signature requirements, the application displays an error message and leaves the ESignature pop-up displayed.

How to Capture an Electronic Signature for a Single-Container Transaction

Follow these steps to capture an electronic signature for a single-container transaction:

1. Execute a shop floor transaction that has an associated electronic signature requirement. The ESignature pop-up appears. The application selects the first signature (child) row for the first unfulfilled signature requirement by default.
2. Select a different signature requirement, if necessary.
3. Complete these fields:
• Signer
• Signer Password
4. Complete these fields if a cosigner is necessary:
• Cosigner
• Cosigner Password
• Cosign Reason
5. Have the signer or cosigner enter comments, if necessary.
6. Click Add Signature. The application validates and adds the signature information to the requirement selected in the Electronic Signature Entry grid. If additional signatures are required, the application adds the next child row automatically, and places the cursor in the Signer field for the next signature.
7. Repeat steps 2-6 to add additional signatures for the selected requirement. Once all signatures for a requirement have been captured, the application changes the background color of the parent row to green and selects a blank signature row in the next unfulfilled signature requirement.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Submit Signatures. The application closes the ESignature pop-up and displays a status message indicating the transaction was completed.

How to Capture Electronic Signatures for Multiple Meanings at Once

Follow these steps to capture electronic signatures for multiplemeanings at once:

1. Access a multi-container transaction page or pop-up that has an associated electronic signature requirement.
2. Select one or more containers.
3. Complete the required fields on the page or pop-up.
4. Click the transaction button to execute the transaction. The ESignature pop-up appears, the Perform batch signoff for all satisfied meanings (no row selection required) option is selected, and the Signer Role field is enabled.
5. Select a role in the Signer Role field.
6. Complete these fields:
• Signer
• Signer Password
7. Complete these fields if a cosigner is necessary:
• Cosigner
• Cosigner Password
• Cosign Reason
8. Have the signer or cosigner enter comments, if necessary.
9. Click Add Signature. The application validates and adds the signature information to all signature requirements that the user’s role, user name, and password satisfy. It then highlights the added signature under each parent row.
10. Repeat steps 5-9 to add additional signatures. Once all signatures for a requirement have been captured, the application changes the background color of the parent row to green and selects a blank signature row in the next unfulfilled signature requirement.
11. Click Submit Signatures. The application closes the ESignature pop-up and displays a status message indicating the transaction was completed.

How to Capture Electronic Signatures for a Single Meaning

Follow these steps to capture electronic signatures for a single meaning:

1. Access a multi-container transaction page or pop-up that has an associated electronic signature requirement.
2. Select one or more containers.
3. Complete the required fields on the page or pop-up.
4. Click the transaction button to execute the transaction. The ESignature pop-up appears, the Perform batch signoff for all satisfied meanings (no row selection required) option is selected, and the Signer Role field is enabled.
5. Select the Perform individual signoff for each meaning (row selection required) option. The application selects the first signature (child) row for the first unfulfilled signature requirement by default.
6. Select a different signature requirement, if necessary.
7. Complete these fields:
• Signer
• Signer Password
8. Complete these fields if a cosigner is necessary:
• Cosigner
• Cosigner Password
• Cosign Reason
9. Have the signer or cosigner enter comments, if necessary.
10. Click Add Signature. The application validates and adds the signature information to the requirement selected in the Electronic Signature Entry grid. If additional signatures are required, the application selects the next child row automatically, and places the cursor in the Signer field for the next signature.
11. Repeat steps 6-10 to add additional signatures. Once all signatures for a requirement have been captured, the application changes the background color of the parent row to green and selects a blank signature row in the next unfulfilled signature requirement.
12. Click Submit Signatures. The application closes the ESignature pop-up and displays a status message indicating the transaction was completed.
