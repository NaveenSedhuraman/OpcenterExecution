Using Order Dispatch

The Order Dispatch page displays a grid of manufacturing orders based on priority. Manufacturing uses the query definition associated with the dispatch rule (if one has been created) to populate the grid. The query determines the order of the items in the grid (its priority) and the columns to display.

These conditions determine how manufacturing orders are sorted, displayed, and processed.

the grid is sorted according to the earliest planned start date defined for the manufacturing order, but you can start containers for any order in the grid.

the start date is not defined in the manufacturing order, or the orders have the same planned start date

the grid is sorted based on priority but you can start containers for any order in the grid.

If you need to separate the order’s quantities into several containers, create as many containers as needed until the order’s quantity has been fulfilled. An order is removed from the grid when it is fulfilled.

Only manufacturing orders defined with a quantity appear in the Mfg Orders grid. If a manufacturing order is defined with a quantity of zero, the application assumes the order has been fulfilled and does not display the order. You must add a quantity to the order, save it, and then refresh the Mfg Orders grid to view the order on the Order Dispatch page.

The Mfg Orders grid is filtered by the employee instance's factory. Orders modeled with the employee instance's factory will appear in the Mfg Orders grid. Orders without defined factories will also appear.

Refreshing the Mfg Orders Grid

You must refresh the Mfg Orders grid to view updated details for modified manufacturing orders and to view any recently added manufacturing orders. Refreshing the grid is especially important when someone else is entering manufacturing orders from a different location.

Assigning UDI on the Order Dispatch Page
Note: The functionality described below is available only when Medical Device is installed. Refer to "Medical Device Workspace" for information.

You can assign a device identifier, production identifier, and unique device identifier directly to a container on the Order Dispatch page. This is useful when scanning UDI from an external source. The production identifier and unique device identifier can be directly entered while the device identifier is selected.

The application populates the Device Identifier field automatically if there is only one device identifier specified on the product. A list appears if there is more than one device identifier specified on the product.

When Using Opcenter Execution Core Scheduling with Order Dispatch
Note: The functionality described below is available only when Opcenter Execution Core Scheduling is installed. Refer to "Opcenter Execution Core Scheduling" for information.

You must use the Mfg Order modeling object to define a manufacturing order to appear in the Mfg Orders grid of the Order Dispatch page. Opcenter Execution Core Scheduling displays the resource information for the manufacturing order in the Resource field of the Mfg Orders grid. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining manufacturing orders.

How to Start a Container for a Selected Manufacturing Order

Follow these steps to start a container for a selected manufacturing order:

1. Open the Order Dispatch page from the Container menu. The Order Dispatch page appears with a list of current manufacturing orders displayed in the grid.
2. Select the manufacturing order to be processed. The Mfg Order, Product, and Product Description fields below the grid default to information associated with the selected manufacturing order.
3. Click the Attach Doc to this Mfg Order Instance icon to attach a document to a manufacturing order. The Attach Document pop-up appears.
4. Refer to the appropriate document attachment procedure if you want to attach a document to a manufacturing order:
• "How to Attach a New Document that Will Not Be Reused to a Manufacturing Order"
• "How to Attach a New Document that Will Be Reused to a Manufacturing Order"
• "How to Attach an Existing Document to a Manufacturing Order"
5. Enter the name of the container to start in the Container field.

Or

Click the Generate Names Automatically check box.

6. Select the Level.
7. Select the Owner.
8. Select the reason for starting this container in the Start Reason field.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click the Attach Document control to display the container attachment fields if the section is not expanded.
11. Refer to the appropriate document attachment procedure if you want to attach a document to a new container:
• "How to Attach a New Document that Will Not Be Reused to a New Container"
• "How to Attach a New Document that Will Be Reused to a New Container"
• "How to Attach an Existing Document to a New Container"
12. Click Start or Submit on the command bar. The application displays a status message confirming the container and updates the In Process Qty column with the quantities being processed. If the order's quantity has been fulfilled, the application removes the manufacturing order from the list.
13. Repeat steps 5-8 until the quantity has been fulfilled, if you need to start additional containers against a manufacturing order to fulfill the quantity. Once this is done, the manufacturing order is removed from the list.

How to Attach a New Document that Will Not Be Reused to a Manufacturing Order

Follow these steps to attach a new document that will not be reused to a manufacturing order:

1. Perform the "How to Start a Container for a Selected Manufacturing Order" procedure.
2. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
3. Select Attach a new document that will NOT be reused (no Revision needed). The Document Path, Document Name, and Document Description fields appear.
4. Click the File Upload button to select a file.
5. Enter a name for the document in the Document Name field.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Attach or Submit on the command bar. The application displays a message indicating that the document has been attached successfully.

How to Attach a New Document that Will Be Reused to a Manufacturing Order

Follow these steps to attach a new document that will be reused to a manufacturing order:

1. Perform the "How to Start a Container for a Selected Manufacturing Order" procedure.
2. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
3. Select Attach a new document that will be reused (Revision must be specified). The Document Path, Document Name, Revision, Revision of Record, and Document Description fields appear.
4. Click the File Upload button to select a file.
5. Enter a name for the document in the Document Name field.
6. Enter the revision of this document in the Revision field.
7. Select the box next to the revision optionally to indicate this document is the revision of record.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Attach or Submit on the command bar. The application displays a message indicating that the document has been attached successfully.

How to Attach an Existing Document to a Manufacturing Order

Follow these steps to attach an existing document to a manufacturing order:

1. Perform the "How to Start a Container for a Selected Manufacturing Order" procedure.
2. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
3. Select Attach an existing document (select below). The Document Instances, Revision, Revision of Record fields, and View Document button appear.
4. Select a document in the Document Instances field.
5. Enter the specific revision in the Revision field.

Or

Select the Revision of Record check box.

6. Click View Document if you want to view the document before you attach it. The application prompts you to open or save the file, or it automatically downloads the document. (Your Internet browser determines whether the application displays a pop-up, its appearance, and the wording of the Open and Save options.)
7. Click Attach or Submit on the command bar. The application displays a message indicating that the document has been attached successfully.
