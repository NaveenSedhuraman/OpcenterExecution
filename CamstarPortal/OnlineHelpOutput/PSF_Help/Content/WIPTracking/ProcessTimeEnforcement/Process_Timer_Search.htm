Process Timer Search

The Process Timer Search page enables you to view all current and historical process timers. It also enables you to stop a process timer.

When Searching for Process Timers

You can use multiple parameters to narrow your search results when searching for process timers. For example, you can enter 01/01/2016 08:00 AM in the Timer Started - Begin field and 01/01/2016 05:00 PM in the Time Started - End field to only return timers that have started between those times.

The application displays an empty Search Results grid when you initially access the Process Timer Search page. You must specify at least one parameter before it will enable the Search button.

Filtering by Completion Status can only be performed on completed timers. Current Status is automatically set to completed if Completion Status criteria is selected. Completion Status is disabled if Current Status is set to In Process.

Executing a Search causes the application to:

• Display the search results in Search Results grid.
• Retain the search parameters.

All Search Results grid column headers are sortable. The default timer sort order is:

• Time to Max - Timers with the longest time over Max Time to the furthest time under Max Time.
• Time to Min - Timers with the longest time over Min Time to the furthest time under Min Time.
• Timer Ended - Timers most recently ended.

The grid data automatically refreshes every five seconds. You can specify how often the grid data refreshes by modifying the grid in Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information.

Siemens also provides the ability to export your search results to an Excel or CSV format.

Use the Clear All button to clear parameters and results. Clearing the Process Timer Search page enables you to start a new search. Clicking the Clear All button causes the application to:

• Clear all specified search filters.
• Clear the timers from the Search Results grid.
• Clear the Stop Timer action button.
• Disable the Search button.
When Stopping Process Timers

The application displays the Stop Timer button when you select a timer in the Search Results grid with a Current Status of In Process. You must have permissions to stop timers.

Selecting an In Process timer and clicking the Stop Timer button displays the Stop Timer pop-up. Entering a comment is optional.

Clicking Stop Timer closes the pop-up and stops the selected timer. The application displays the same page of the Search Results grid after stopping a timer. A stopped timer that no longer meets the filter criteria will no longer be displayed in the Search Results grid.

Timers that are stopped using the Stop Timer pop-up can be resumed using Reverse Last Transaction. Refer to "Reverse Last Transaction" for information.

How to Search for Process Timers

Follow these steps to search for process timers:

1. Open the Process Timer Search page from the Search menu. The Process Timer Search page appears.
2. Select or enter search criteria.
3. Click Search. The application displays the process timers matching your search criteria in the Search Results grid.

How to Stop a Process Timer

Follow these steps to stop a process timer:

1. Perform the "How to Search for Process Timers" procedure. The application updates the Search Results grid with a list of process timers meeting the entered search criteria.
2. Select a timer in the Search Results grid. The application displays the Stop Timer action button for in process timers. You must have permissions to stop timers.
3. Click Stop Timer. The Stop Timer pop-up displays.
4. Enter optional comments according to your business requirements and click Stop. The application stops the selected timer, the timer End Time is set to the current system time, the Current Status is set to Complete, and the Stop Timer pop-up closes. The application updates the Search Results grid and the manufacturing audit trail.
