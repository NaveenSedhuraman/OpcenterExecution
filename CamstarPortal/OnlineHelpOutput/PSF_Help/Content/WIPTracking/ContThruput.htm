Container Throughput

The purpose of the throughput transaction is to record information about a container whose quantity, in order to be processed, needs to be distributed. The throughput transaction can be used to keep track of the processed quantity as well as how much more needs to be processed before the container is released to the next step.

The resulting throughput quantities are incremented in the resource's production status for the quantities entered.

Throughput by Shift

You can also use the Thruput transaction to record throughput by shift, recording partial completion of container processing at the end of a shift or during the shift (such as at processing break-points).

For example, a container that has 800 pounds of product requires two and a half work shifts to be processed. Using the Thruput transaction, each shift records how much quantity was processed (throughput): Shift 1 records 300 pounds and the second shift records 350. The third shift then knows to process the remaining 150 pounds of quantity and then performs a Move transaction on the container.

Information from the Thruput transaction provides the basis for most production reports.

When Recording Container Throughput

Throughput is limited to what is available in the container.

When Thruput Remaining Qty is selected, the Qty field is disabled and the entire quantity is recorded as processed when Thruput Remaining Qty is selected.

Thruput Page

You can access the Thruput page from the Container menu. The Thruput page allows you to record the quantity processed in a container.

Note: The Thruput page may be available as a pop-up also if the container thruput transaction has been configured as an available action on another page (for example, Container Search).

How to Record the Container Quantity Processed

Follow these steps to record the container quantity processed:

1. Open the Thruput page from the Container menu. The Thruput page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Thruput pop-up from another shop floor page.
3. Enter a Resource Group if you want to limit the list of resources displayed in the Resource field.
4. Select a Resource.
5. Do you want to process the entire quantity or a partial quantity?

If you want to process . . .

Then . . .

The entire quantity
a. Select the Thruput Remaining Qty check box.
b. Go to step 6.

A partial quantity
a. Enter:
• The quantity processed in the Qty field.
• The secondary quantity processed, if applicable, in   the Qty2 field.
• The units processed in the Unit Count field.
b. Go to step 6.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Thruput or Submit on the command bar. The quantity processed for the container is recorded.
