Processing WIP Messages on Portal Shop Floor Pages

Siemens provides the ability to deliver predefined WIP messages to a Portal shop floor transaction based on any of the following:

• WIP message context (Product, Operation, Owner, Start Reason, etc.)
• WIP message definition (Status, Relevant Service Type, Effectivity Dates)
• User’s current context/session values or
• Container being processed

Automatic WIP messages can be used to:

• Display a reminder or special processing instruction to the person doing the work
• Place a container on hold when it reaches a specific point
• Require an operator or supervisor to acknowledge that the special instructions provided by the WIP message have been reviewed
• Notify an engineer that a container has reached a specific location

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on configuring WIP messages.

If you receive a WIP message while performing a shop floor transaction, one of the conditions described above has been predefined and something in the process has triggered an automated message. You should review the message. The message may provide additional information or instruct you to perform a task. Additionally, the message may require acknowledgment and a password.

When Processing WIP Messages

When processing multiple WIP messages for a transaction, you must acknowledge all messages for which acknowledgment is required before you can submit the messages and execute the transaction. The application displays an error message if you click Submit without having acknowledged all required acknowledgment messages. You cannot view WIP messages after they have been submitted successfully.

When you select a message on the WIP Messages Processing pop-up, the application displays the fields configured for that message below the WIP Messages grid. The fields that appear depend on how your system administrator configured the selected WIP message in Modeling.

If electronic signatures have been configured for this transaction also, the E-Signature messages appear after all of the WIP messages are processed.

Note: WIP messaging is supported for single-container transactions only.

WIP Messages Processing Pop-up

The WIP Messages Processing pop-up may appear on a shop floor page when performing a transaction.

How to Acknowledge a WIP Message

Follow these steps to acknowledge a WIP message:

1. Select a WIP message in the grid if the WIP Messages Processing pop-up appears during a transaction. The application updates the fields displayed below the WIP Messages grid.
2. Review any documents associated with the WIP message by clicking the View Document icon. The document appears in a separate pop-up.
3. Select the Message Acknowledged check box to sign off on the message, if the check box appears.
4. Enter your password in the WIP Message Password field, if the field appears.
5. Repeat steps 1-4 to process any additional WIP messages.
6. Click Submit. The application processes the WIP messages and displays a status message.
