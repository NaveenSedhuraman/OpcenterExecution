Label Printing

Label printing functionality is available only if your organization purchased the separately licensed Label Printing module. The Label Printing module provides this functionality:

• Automatic container and production event label printing

Label printing occurs automatically when configured as part of any container or production event transaction. When executing a transaction on a tracked shop floor container, the application resolves any printing information associated with the transaction: the label to print, the printer to use, and how many copies to print. The application determines whether a container label should be printed based on the current specification of the container.

• Manual container and production event label printing

You can print a container label manually for a tracked shop floor container based on your business need. For example, you may want to print a container label when automatic label printing has not been associated with the transaction you executed. You can also print an individual production event label when doing so is the only task that needs to occur.

• Manual container and production event label reprinting

You can reprint a label based on your business need. For example, when a label has been damaged or did not print successfully due to a printer problem, you can reprint the label.

The application records information for every label printed or reprinted, automatically or manually, in the container's or production event's manufacturing audit trail. If configured, you can use the manufacturing audit trail to view details for printed and reprinted labels. You can also view label printing information for resources in the resource audit trail.
