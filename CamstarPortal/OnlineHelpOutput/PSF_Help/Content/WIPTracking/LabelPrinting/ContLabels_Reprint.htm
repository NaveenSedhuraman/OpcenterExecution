Reprint Container Labels

The Reprint Container Label transaction allows you to reprint a container label based on your need. For example, you can reprint a label if the label is damaged or did not print successfully due to printer problem.

When Reprinting Container Labels

The application displays all transactions that have been performed on the container that you select in the Transactions grid in descending order. The most recent transaction appears at the top of the list. If the number of transactions exceeds the space on the grid, the application adds paging functionality.

You can reprint a container label for only one transaction at a time. When you select the transaction, the application defaults the values in the Printer Label Definition, Print Queue, and Label Count fields to the values used the last time the label was printed. You have the option to select or enter different values for the Print Queue and Label Count fields.

When Reprinting Carrier Labels

You must add the carrier control through Portal Studio before you can reprint a label for a carrier.

The application displays the container in the Container field when you select a carrier. If you select a container that is in a carrier, the application displays the corresponding carrier in the Carrier field.

Reprint Container Label Page

You can access the Reprint Container Label page from the Container menu. The Reprint Container Label page allows you to reprint one or more labels for the specified container or carrier.

Note: The Reprint Container Label menu item is available only if your organization purchased the Label Printing module and a system administrator configured the Container menu to include the Reprint Container Label option. Contact your system administrator if you do not see the Reprint Container Label menu item.

How to Reprint a Container Label

Follow these steps to reprint a container label:

1. Open the Reprint Container Label page from the Container menu. The Reprint Container Label page appears.
2. Select or enter a container in the Container field. The application updates the container header and populates the Transaction Details grid.
3. Select a transaction in the Transaction Details grid. The application displays the printer label definition, print queue, and label count used when the label was printed previously.
4. Modify any of the defaulted values, if necessary:
• Print Queue
• Label Count
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Reprint Label or Submit on the command bar. A confirmation message appears and the container label is reprinted at the designated label printer. The application writes the label information to the text file specified for that print queue if the queue uses the Output to File option.
