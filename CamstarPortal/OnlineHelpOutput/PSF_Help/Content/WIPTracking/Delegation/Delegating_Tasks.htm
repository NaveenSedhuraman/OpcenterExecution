Delegating Tasks

You can reassign tasks from an assigned employee to a target employee. You can reassign specific tasks or you can delegate all tasks from one employee to another for a specified date range. The application provides a search feature that enables you to search for and retrieve tasks or delegation definitions that specify a date range.

This topic contains information that is common to delegation by task and delegation by date.

These topics provide information specific to either delegating by task or date:

• Delegating by Task
• Delegating by Date

Before Delegating Tasks

A system administrator must define delegation reasons in Modeling before you can delegate tasks. The system administrator can also set up delegation notifications in Modeling if you want to notify assigned and target employees when the application reassigns a task. The notification types must be set on the Change Mgt Settings instance associated with the employee's factory.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining delegation reasons and change management settings.

Using Delegation Search

You must have a role that allows access to the Delegation Search page.

The application displays two tabs when you access the Delegation Search page. The two tabs are By Task and By Date. By Task is the default tab. The tabs work independently of each other. You can search either by task or by delegation definitions that specify a date range.

The application displays multiple fields by which you can narrow your search. The application adds a filter icon to each tab to indicate that filter criteria is specified. The filter icon appears on each tab automatically because the application specifies one or more fields by default on each tab.

The application enables the Search button on each tab when you access the Delegation Search page.

The Search Results grid on the By Task tab is empty when you initially access the page. The application returns tasks that meet your criteria when you click Search. You can then select the tasks you want to delegate.

The application populates the Search Results grid on the By Date tab with active and future delegations by default.

Refer to "When Delegating by Date" for information on delegation definitions.

Clearing Search Filters and Results

Use the Clear All button to clear filters and results for the selected tab. Clearing the filters and results enables you to start a new search. The application performs the following on the selected tab when you click Clear All:

• Clears all search filters except for the Application filter on the By Task tab
• Clears the filter icon from the tab
• Clears all tasks from the Search Results grid
• Disables the Delegate Task button if the By Task tab is selected
• Continues displaying the selected tab on the Delegation Search page

About Delegated Tasks

The following applies whether you delegate by task or date:

• The assignment and due dates for a task remain the same.
• The Delegate transaction records the change from one assigned employee to another in history.
• The application permanently delegates tasks from the assigned employee to the target employee.

The application automatically delegates tasks from the assigned employee to the target employee during the specified date range if you delegate by date.

The application updates the Name field as appropriate on the following pages with the name of the target employee:

• Approvers grid on the Package Details page
• Collaborators grid on the Package Details page
• Approvers grid on the Update package page, Approvers tab
• Collaborators grid on the Update package page, Collaborators tab
Note: Delegation does not take into account the Substitute Option or role-based permissions when you reassign a task to a target employee. Ensure that you do not assign employees to tasks they cannot perform. You can set the Substitute Option on the Update Package page. Refer to the Opcenter Execution Medical Device and Diagnostics Change Management User Guide or the Opcenter Execution Core Change Management User Guide for information.

Sending Notifications for Delegated Tasks

The system administrator can set up delegation notification types on the Change Mgt Settings modeling object instance for the employee's factory. This enables the application to send e-mail notifications whether you delegate tasks by task or date.

This table shows when the application sends the e-mail notifications depending on which type of delegation you use.

And you are delegating . . .

Then the application sends an e-mail notification to the . . .

Task

An approval or collaboration task

Assigned and target employees when you delegate the approval or collaboration task on the Delegate Tasks pop-up.

Date

 

An approval task

Assigned and final target employees when you route the approval task for approval.

A collaboration task

Assigned and final target employees when you update the package.

There may be more than one target employee if you enable delegation forwarding. The final target employee is the employee to which the application ultimately reassigns the task. Refer to "Adding Delegation Definitions" for information.

The assigned and target employees receive one notification for each delegated task. For example, if you delegate three tasks (one approval task and two collaboration tasks), both employees receive three notifications: one e-mail stating there is a new approval task and two e-mails stating there is a new collaboration task.
