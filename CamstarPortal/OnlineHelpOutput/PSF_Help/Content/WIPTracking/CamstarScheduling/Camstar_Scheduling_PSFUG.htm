Opcenter Execution Core Scheduling

Note: The functionality described below is available only when Opcenter Execution Core Scheduling (Opcenter EX CR Scheduling) is installed. Refer to the Opcenter Execution Core Scheduling User Guide for information.

Opcenter Execution Core Scheduling provides a mechanism to schedule manufacturing orders using the Preactor Advance Scheduling application.

Opcenter Execution Core Scheduling enables an organization to define Enterprise Resource Planning (ERP) routing data for products and manufacturing orders on the Opcenter Execution Medical Device and Diagnostics or the Opcenter Execution Core and then use that data in the Siemens Preactor Advanced Planning and Scheduling (Preactor) application to create production schedules for manufacturing orders. Information on the scheduled orders is sent to a staging table to be transmitted back to Opcenter Execution Core when the planner releases the schedule from Preactor. You can define an Opcenter CN MOM database adapter to invoke a stored procedure to return the scheduled order information to Opcenter Execution Medical Device and Diagnostics or Opcenter Execution Core in the form of a dispatch list. Refer to the Opcenter Connect MOM User Guide for information on defining a CIO database adapter.

Preactor uses the released order information from Opcenter Execution Medical Device and Diagnostics or Opcenter Execution Core to create an optimized manufacturing schedule. The optimized schedule is returned from Preactor and is used to generate the dispatch lists for the Order Dispatch and Operational View pages. Refer to "When Using Opcenter Execution Core Scheduling with Order Dispatch" for information.

Scheduling provides "What If" Planning functionality to analyze future, or "what if," scheduling scenarios based on a planned product mix. Some or all of the manufacturing orders have yet to be created in these situations. The application sends products and their associated ERP routes from Opcenter Execution Medical Device and Diagnostics or Opcenter Execution Core to Preactor. This data enables the user to create hypothetical orders in Preactor to analyze "what if" scheduling scenarios.
