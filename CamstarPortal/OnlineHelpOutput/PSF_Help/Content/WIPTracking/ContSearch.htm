Search Containers

Siemens provides a search feature enabling you to search for and retrieve containers based on specified criteria. You can search for a feature using provided filters or user-defined queries. Siemens provides filters for searching containers based on their current status and history.

When the application returns containers meeting the specified criteria, you can select one or more containers on which you want to perform a transaction. The application displays only those transactions relevant to the selected containers.

When Searching for Containers

You can use multiple parameters to narrow your search results when searching for containers. The application provides four types of searches:

• Current Status - allows you to search for containers based on their current status
• History - allows you to search for containers based on previous processing
• Current Status and History - allows you to search for containers based on both current status and previous processing
• User Query - allows you to create custom queries to search for containers

The application displays three tabs and an empty, generic search results grid when you initially access the Container Search page. The three tabs are Current Status, History, and User Query.

The application requires you to specify at least one parameter (filter or user query) before it enables the Search button. Selecting a parameter on a search tab causes the application to:

• Add an icon to the tab, indicating a filter or user query has been selected on that tab.
• Display an information message (note) indicating the type of search results the application will return.
• Enable the Search button.
Note: You must select a user query on the User Query tab for the application to enable the Search button.

When Searching for Containers Using Current Status and History Filters

You can search for containers using both Current Status and History criteria at the same time. The application uses messages displayed on the tabs to communicate the ability to select filters on both and the type of search results the application will return. For example, when you initially access the Container Search page, the application displays a message on the Current Status tab indicating that the search results will include only containers meeting the filters specified on that tab and that you can also select History filters.

When Using Line Assignments Setting as Search Filters

You can apply line assignment settings as filters when searching for containers using current or historical criteria. You select the Apply Line Assignment check box to apply line assignment settings as filters. The Apply Line Assignment check box appears on both the Current Status and History tabs.

Line Assignment settings include:

• Resource / Work Cell
• Work Center
• Operation
• Workstation

Refer to "Line Assignment Settings" for information on line assignment settings.

Example:

Operation is available as separate filter on the Current Status and History tabs. Resource is available as a separate filter on the History tab. Search results include all containers with either filter value when both the tab filter value and the line assignment filter value are different.

When Searching for Containers Using User Queries

User queries cannot be combined with any other search filters and you must perform a user query search from the User Query tab. The application ignores any filters selected on the Current Status and History tabs when you execute a search from the User Query tab.

You must define one or more user queries in Modeling to use when searching for containers before performing a user query search. The query must contain a Container ID in the selection criteria. Otherwise, the query will not return any results when executed from the User Query tab. Additionally, you must enter values for all parameters for the search to return results.

Important: User queries are typically database specific.

Optionally, you may want to define one or more user query groups in Modeling. User query groups allow you to narrow the user queries available for selection on the User Query tab. For example, if you have user queries defined for different business needs, you can create a Container Search user query group so that you see only user queries relevant to container search.

Selecting a user query causes the application to:

• Display the parameters that you have to define to perform the search.
• Display a query description (if one was provided when the user query was defined).
• Display a note indicating the search will return results for the selected user query only.
• Add an icon to the User Query tab (if the user query is the first parameter you specified).
• Enable the Search button.

The application clears the User Query field and all the data associated with that query if you select a user query group after selecting a user query.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on user queries.

Sample Queries

These are two sample queries:

Query 1:

select ContainerId from Container c

join Product p on c.ProductId = p.ProductId

join ProductBase pb on p.ProductBaseId = pb.ProductBaseId

join CurrentStatus cs on c.CurrentStatusId = cs.CurrentStatusId

left join ResourceDef r on cs.ResourceId = r.ResourceId

Where pb.ProductName = ?ProductName -- 'Widget'

and r.ResourceName = ?ResourceName --'Workstation 1'

Order By ContainerName

Query 2:

select ContainerId from Container c

join Product p on c.ProductId = p.ProductId

join ProductBase pb on p.ProductBaseId = pb.ProductBaseId

<{?ResourceName}>?<{ join CurrentStatus cs on c.CurrentStatusId =

cs.CurrentStatusId

left join ResourceDef r on cs.ResourceId = r.ResourceId }>

Where pb.ProductName = ?ProductName

<{?ResourceName}>?<{ and r.ResourceName = ?ResourceName}>

Order By ContainerName

Executing a Search

Executing a Search causes the application to:

• Remove the information message (note) displayed below the tab fields.
• Display the search results in a grid named for the type of search performed. (For example, if you selected filters from the History tab, the grid name is History Search Results.)
• Retain the search parameters on the tabs.
Note: The application displays a message indicating no results were returned for the search if there are no containers meeting the criteria you specified.

If the search returns more results than can be displayed in the rows on the grid, the application adds paging to the grid. Additionally, the columns displayed in the grid are configurable.

The application displays transactions relevant to the selected containers when you select containers in the Search Results grid. You can select the check box in the header row to select all displayed containers automatically or you can select the check boxes for individual containers.

Siemens also provides the ability to export your search results to an Excel or CSV format.

Using Action Buttons

Action buttons allow you to execute transactions on containers selected in the search results grid. You must select one or more containers in the grid and click the Load Actions button to display the actions available for the selected containers. You must click the Load Actions button again if you change your container selections in the grid to display actions applicable to your new container selections.

The application updates the container count displayed over the Load Actions button when you select containers. The application hides the container count when you load actions for a single container.

After loading actions for selected container records, those actions will be valid until you click the Load Actions button again. The application hides the actions (and icons for single containers) and displays a new row count if you change your row selection. The application displays the actions and the relevant icons or row count if you go back to your original row selection.

When Selecting a Single Container

Selecting a single container and clicking the Load Actions button causes the application to display the action buttons for the container and three icons above the action buttons:

• Container Status- displays the status of the selected container when clicked.
• View Document- displays a list of the documents associated with the container when clicked. You can click a document to view it in a separate window.
• Mfg Audit Trail- displays the manufacturing audit trail for the selected container when clicked.

When Selecting Multiple Containers

The application displays action buttons only for transactions that can be performed on all the selected containers when you select multiple containers. Clicking an action button allows you to execute the transaction for all of the selected containers simultaneously. If the selected containers have no transactions in common, the application displays the message "No actions available for selected containers."

Additionally, you can select containers from more than one page in the search results grid. The application displays a container count above the action buttons instead of the icons it displays for a single container. The container count indicates the number of containers you selected.

When Executing an Action on Selected Containers

The application displays a pop-up enabling you to execute the transaction when you click an action button. The application populates the selected containers in the pop-up automatically.

However, if a WIP Message has been configured for the Move In transaction, the application requires you to acknowledge the WIP message before executing the Move In Immediate transaction.

Clearing Search Results

Use the Clear All button to clear parameters (filters and user queries) and results. Clearing the Container Search page enables you to start a new search. Clicking the Clear All button causes the application to:

• Clear all specified search filters or queries from all tabs.
• Clear any information messages and tab icons, if set.
• Clear the containers from the search results grid.
• Clear all action buttons except the Load Actions.
• Clear the Container Status, View Documents, and Mfg Audit Trail icons above the action buttons.
• Continue displaying the tab you are viewing.
• Disable the Search button.

Configuring Columns for the Results Grid

You can customize the search results grid by specifying the columns you want to view. Clicking the Show Columns icon displays the Select Columns pop-up, which allows you to add and remove search result grid columns. All columns are optional, but the application requires you to select at least one column to display. You must click the Show Columns icon again or click Close to hide the pop-up.

Note: Currently, your column selections apply to the current search results only.

Select Columns Pop-up

The Select columns pop-up includes two lists. Visible Columns lists the columns that appear in the search results grid. Hidden Columns lists the columns you can add to the search results grid.

Adding or removing columns from either list causes the application to:

• Add the moved columns to the bottom of the list.
• Update the page to display your changes when you click the Apply button. You do not have to refresh the page to view the changes.

You can add, remove, and re-order columns by dragging and dropping them or by using the buttons. The application adds a vertical scroll bar to accommodate the lists, if necessary.

Note: The application includes the revision value in the column on the search results grid if you choose to display a column that will show revision values. For example, if you select the Spec column, the application displays the Spec name and revision in the Spec column.

How to Configure Columns for the Results Grid

Follow these steps to configure columns for the Results grid:

1. Open the Container Search page from the Search menu. The Container Search page appears.
2. Perform a search. The search results grid appears and displays the containers matching your search criteria.
3. Click Show Columns. The application displays a pop-up containing a list of visible columns and a list of columns you can select to display.
4. Select the columns you want to appear in the grid using one of the following methods:
• Use drag and drop to move columns between Visible Columns and Hidden Columns lists and to change the order of the columns in those list.
• Click Show All to move all columns to the Visible Columns list. The application moves all columns from the Hidden Columns list to the Visible Columns list, placing them under any that were already in the Visible Columns list.
• Click Hide All to move all columns to the Hidden Columns list. The application moves all columns from the Visible Columns list to the Hidden Columns list, placing those columns under any that were already in the Hidden Columns list.
5. Click Apply to apply the column configuration and close the Show Columns pop-up. The application applies your configuration to the search results grid.

How to Search for a Container Using User Queries

Follow these steps to search for a container using user queries:

1. Open the Container Search page from the Search menu. The Container Search page appears.
2. Click the User Query tab. The User Query page appears.
3. Select a User Query Group.
4. Select a User Query. The application updates the tab with the query information, adds an icon to the tab, displays a note indicating the type of results to be returned, and enables the Search button.
5. Enter the required values in the Query Parameters grid.
6. Click Search. The application displays containers matching your search criteria in a search results grid named for the user query.

How to Execute a Transaction on Selected Containers

Follow these steps to execute a transaction on selected containers:

1. Perform a search using one of the search procedures above.
2. Select the containers for which you want to perform a transaction.
3. Click Load Actions. The application displays buttons for the actions available for the selected containers.
4. Click an action button. The application displays a transaction pop-up for the selected containers.
5. Complete the pop-up and click the transaction execution button.
6. Click the close icon to close the pop-up and return to the Container Search page.
