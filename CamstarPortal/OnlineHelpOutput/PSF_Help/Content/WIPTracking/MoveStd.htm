Move Containers

You use the Move transaction to move a container to the next step in the current workflow. Typically, you only need to specify the container and then submit the transaction. The container uses the default path unless you specify another valid path or step.

When Moving a Container

All display fields are empty on the Move page until a container is selected. Display fields are populated automatically on the Move pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application displays a default value in the Resource field for the specified container when one of the following conditions is true:

• A move in was required for the container's current step and the user moving in the container specified a resource.

Or

• A resource was specified when the container was started and Move is the first transaction (specifying a resource) that you are performing after starting the container.

You can select a different resource, if necessary.

If the Validate Material Consumption check is enabled for this spec, the application validates that all materials and quantities specified have been used for this step before allowing the container to move to the next step in the workflow.

When Moving a Container with an Open Production Event

Note: This functionality is available only as part of the separately licensed Nonconformance Management module.

When moving a container that has an open production event (nonconformance) recorded against it, a setting in the operation to which you are moving the container determines whether the move succeeds or fails. If the Reject Incoming Nonconformance Container check box is selected for an operation, the application prevents a user from moving containers with open nonconformances to that operation. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining operations.

When Moving a Container with Automatic Material Consumption Enabled

Note: This functionality is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can select the Move transaction in the Consume Material Txns field on the Factory page to indicate material loaded on a resource is automatically consumed when a container moves through the resource. You can also select the Move transaction in the Validate Material Queue Txns field to enable the application to verify the correct material is loaded on a resource for a container moving through the resource.

The application returns a warning message if you select the Move transaction but do not select the Validate Material Queue Qty check box. The transaction fails if the quantity is not sufficient if you select the Move transaction and select the Validate Material Queue Qty check box. You can select an Issue Difference Reason to prevent a failure from occurring when partial quantities are issued during automatic component issue.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining factories.

Assigning UDI on the Move Page
Note: This functionality is available only when Medical Device is installed. Refer to "Medical Device Workspace" for information.

You can assign UDI to a container automatically from the Move page by defining information on the spec and product.

The application assigns device identifier, production identifier, and UDI values to a container when the container is moved out of a spec if the Assign UDI field on the spec is checked and if the current transaction is equal to the UDI Transaction on the product.

The application sets the container's device identifier to the device identifier on the product if there is only one DI assigned to that product. The container's DI is left blank if there is more than one DI assigned to the product.

The application sets the container's production identifier to the result of the production identifier expression on the product if the product has a production identifier expression.

The application sets the container's UDI to the result of the UDI expression on the product if the product has a UDI expression.

Move Page

You can access the Move page from the Container menu. The Move page allows you to scan or enter the container you want to move to a workflow step.

Note: The Move page may be available as a pop-up also if the move transaction has been configured as an available action on another page (for example, Container Search).

Move Page and Pop-up Field Definitions

This table defines the fields on the Move page and Move pop-up.

Field

Definition

Type

Container Information

Header containing the container name and information about that container. You must select or enter a container on a transaction page, but all information is display-only on a transaction pop-up. Refer to "Container Information" for information.

Required/ Display Only

Resource

Resource that processed the container at the current step. You can select a different resource if the application displays one by default. Specifying the resource used at this step before moving the container provides traceability.

The resource values available for selection depend on the resource group assigned to the step (spec). If no resource group was assigned to the spec, all defined resources are available for selection.

Optional

Show/Hide additional fields

Optional fields hidden by default when the application displays a page or pop-up. Clicking the Show control displays the fields. Clicking the Hide control hides the fields.

  

Path

Path to the next step in the workflow. The application uses the default path If you leave this field blank. The only step available is the default step unless alternate paths were defined in the associated Workflow modeling object.

Note: This field is hidden by default.

Optional

To Resource

Resource that will process the container at the next step. The resource values available for selection depend on the resource group assigned to the next step (spec). If no resource group was assigned to the spec, all defined resources are available for selection.

Notes:

• This field is hidden by default.
• If the availability of the specified resource is set to Down, the application displays an error message indicating the resource is unavailable when you perform the transaction.

Optional

To Location

Location to which the container will go for processing at the next step. A location may be an entire factory, an area of a factory, or as specific as a rack identifier. Locations are defined within the Factory object in Modeling.

Note: This field is hidden by default.

Optional

Comments

Additional information or instructions for this transaction or container. The field accepts a maximum of 255 characters. The application truncates the data if the comments exceed 255 characters.

Optional

How to Move a Container to the Next Step

Follow these steps to move a container to the next step:

1. Open the Move page from the Container menu. The Move page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Move pop-up from another shop floor page.
3. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
4. Click Move or Submit on the command bar. A status message appears confirming that the container has been moved.
