Combine Containers

The Combine Container transaction transfers one or more child containers from one or more parent containers to an existing parent container. A Combine transaction does not move either the source or target containers. Nor does it imply that any processing activity (throughput) was performed on either container.

When Combining Containers

All display fields are empty on the Combine Container page until a parent container is selected. The application populates display fields automatically on the Combine Container pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

Selecting a target container in the header causes the application to display all group containers (parent containers with child containers) and all unit containers with a quantity of zero (0) eligible for combination with the target container in the Eligible Containers to Combine grid. Child containers appear in a subgrid below the parent container row.

Additionally, you can scan or enter the source container in the Enter Container to Select field and tab out of the field, and the application will locate and select that container in the Eligible Containers to Combine grid. The application ensures its child containers have been returned and the parent row expanded when you scan or enter a parent container. The application puts focus on the parent row and leaves the parent row expanded if it is expanded to display the child container subgrid. If the parent row is collapsed, the application expands the parent row to display the child containers.

When you scan or enter a child container, the application ensures the parent of that container and all other child containers are displayed in the grid. If all are displayed in the grid, the application selects the child container. If not, the application populates the grid with the parent and child containers and selects the scanned child container.

Note: The application displays an error message if you scan or enter the name of an ineligible container or a container that does not exist in the Enter Container to Select field.

The entire quantity of the child container (source) is transferred to the existing (target) container when a combine is performed.

The combine to container is set to the earlier expiration date used by the containers when you combine containers. However, if only one of the combine from containers has an expiration date, then the expiration date of the combine to container is set to the expiration date of the combine from container.

You have the option to have the application close the parent container when it reaches a zero quantity.

The Combine Selected button performs the Combine Containers transaction on all child containers selected in the grid. The application clears the search grid completely when the Combine is successful.

Combine Container and Combine Qty Eligibility Rules

The containers in a Combine transaction must meet these eligibility criteria:

• Neither of the containers being combined can be closed or on hold.
• Target and source containers must contain the same product and be at the same container level.
• Target and source containers must be at the same operation in the same factory.
• The transaction must occur before the expiration dates if expiration dates were entered for the source (From) container or the target (To) container. The Combine transaction will fail if the expiration date of either container occurs before you execute the transaction. If only one container has an expiration date, the transaction must occur before that date.

Additionally, when performing the Combine Container transaction, all source parent containers should have child containers.

Adding Affected Containers to Events

The application can add containers to quality events automatically when you execute the Combine Container transaction on a container associated with one or more quality events. For example, if a new child container is associated with an event, the application adds the parent container, existing child containers, and new child containers to the quality event automatically. A system administrator configures the Combine transaction to add containers to events in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or the Opcenter Execution Core Event Management User Guide and the Opcenter Execution Medical Device and Diagnostics Nonconformance Management User Guide or the Opcenter Execution Core Nonconformance Management User Guide for information on adding affected containers to events automatically.

Combine Container Page

You can access the Combine Container page from the Container menu. The Combine Container page allows you to enter or select the parent container into which you want to combine the entire quantity of one or more child containers.

Note: The Combine Container page may also be available as a pop-up if the combine container transaction has been configured as an available action on another page (for example, Container Search).

How to Combine One or More Containers

Follow these steps to combine one or more containers:

1. Open the Combine Container page from the Container menu. The Combine Container page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Combine Container pop-up from another shop floor page.
3. Select the source containers to combine with the target container using one of the following methods:
• Select the Include All Children check box in the parent row to select all child containers.
• Select the check box for each individual child container.
• Scan or enter a container in the Enter Container to Select field and tab out of the field. The application highlights the child container row and selects the child container's corresponding check box.
4. Repeat step 3 until you have selected every source container you want to combine with the target container.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Combine Selected or Submit on the command bar. The application transfers the selected child containers to the target container and displays a success message.
