Associate Containers

The Associate transaction creates parent and child container relationships. It enables you to group existing containers under a single parent. Then, an action performed once on the parent container applies to the child containers automatically.

When Associating Containers

You can associate a container (or a group of containers) with only one parent container. Therefore, if you want to associate a container that is currently associated with another parent, you must first disassociate that container from its current parent.

The parent container must have child containers already or have a quantity of zero. The displayed quantity (Qty) in the container information header is updated to reflect the child container quantities that are added to the parent container.

There are two grids on the Associate page:

• Pending Eligible Containers grid - When you specify the parent container, this grid is automatically populated with all containers eligible for association with that parent container.
• Currently Associated Containers grid - This grid provides a list of all containers currently associated with the parent container.

All display fields are empty in the container information header on the Associate page until you select a parent container. Display fields are populated automatically on the Associate pop-up.

The Reset button clears the Currently Associated Containers grid and the Comments field on the Associate page and pop-up. It also clears the container information header on the Associate page.

Valid parent levels and child levels are defined with the Container Level modeling object.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guidefor information on container levels.

Container Eligibility Rules

Containers must meet the following eligibility rules for the parent/child association to be made:

• Child and parent containers must be associated with the same factory.
• Child and parent containers must be at the same operation, workflow, and step.
• Child and parent containers must both be open and not on hold.
• Child containers must not be associated with another parent.
• Child containers must not be expired.

Adding Affected Containers to Events

The application can add containers to quality events automatically when you execute the Associate transaction on a container associated with one or more quality events. For example, if a selected eligible container is associated with an event, the application adds the parent container, existing child containers, and selected eligible containers to the quality event automatically. A system administrator configures the Associate transaction to add containers to events in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or the Opcenter Execution Core Event Management User Guide and the Opcenter Execution Medical Device and Diagnostics Nonconformance Management User Guide or the Opcenter Execution Core Nonconformance Management User Guide for information on adding affected containers to quality events automatically.

Associate Page

Use the Associate page to select a parent container, and then associate eligible child containers to the parent container. The parent container is already selected on the Associate pop-up.

How to Associate One or More Containers

Follow these steps to associate one or more containers:

1. Open the Associate page from the Container menu. The Associate page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Associate pop-up from another shop floor page.
3. Select the containers to be associated using one of these methods:
• Use the Enter Container to Select field to enter a container, either by scanning it or by typing the container name or identifier and then tabbing out of the field. The application selects the row with that container in the Pending Eligible Containers grid.
• Select the check box next to a container or containers in the grid.
• Select the check box in the grid heading to select all containers on the current grid page.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Associate All Selected or Submit on the command bar. The application clears the Container field and both grids, and displays a Status message stating the selected containers were associated to the parent container.
