Start Containers

The Start transaction creates a new container and initiates the first step in the workflow associated with the container’s product. You must start a container before processing any other transactions.

When Using Start

You must specify a unique container name when creating a new container. You can either enter a container name manually or have the application generate a container name automatically. The application considers closed containers when validating a unique name.

Select the Generate Names Automatically check box to have the application generate the container name using defined numbering rules. The application clears and disables the To Container field when you select this check box.

An error message appears when Generate Names Automatically is selected but there is no auto numbering rule defined.

The application uses the numbering rules associated with the parent container from which the child container is being split to generate the To Container name. The application generates the name of the new parent container when you click the Submit button.

Numbering rules may be specified for multiple modeling object instances associated with the container. Numbering rules follow this default precedence:

1. Container Level
2. MfgOrder
3. Product
4. Product Family
5. Factory
Note: The application uses the numbering rule associated with the relevant modeling object instances you select for this container and not from a parent or source container.

The Expiration Date determines when the container expires. This date is validated in several other transactions such as Associate, Combine, and Component Issue.

Factory is not a required field; however, a factory must either be assigned to the employee starting the container, or be specified on the Start page when you start the container. Otherwise, other transactions such as Move will not process the container.

Note: The application displays a value in the Factory field automatically if a factory was specified for the logged in employee. You can select a different factory, if necessary.

You can attach files to a container from the Start page and then manage them from the Manage Attachments page. Refer to "Attaching Documents" for information on the options for attaching documents and refer to "Managing Attachments" for information on managing documents. Attachments are optional. However, the application requires you to complete certain fields when attaching files before you start the container.

You cannot start multiple containers with the Start page. Quantities of single-level containers are defined, while quantities of multilevel containers are derived (or rolled up) from the quantities of the children containers. You must use the Start - Two Level page to start multiple containers. Refer to "Two-Level Start" for information on the Start - Two Level page.

When Starting a Container with Automatic Material Consumption Enabled

Note: This functionality is available only when the Industry Solutions workspace is installed. Refer to "Industry Solutions Workspace" for information.

You can select the Start transaction in the Consume Material Txns field on the Factory page to indicate material loaded on a resource is automatically consumed when a container moves through the resource. You can also select the Start transaction in the Validate Material Queue Txns field to enable the application to verify the correct material is loaded on a resource for a container moving through the resource.

The application returns a warning message if you select the Start transaction but do not select the Validate Material Queue Qty check box. The transaction fails if the quantity is not sufficient if you select the Start transaction and select the Validate Material Queue Qty check box. You can select an Issue Difference Reason to prevent a failure from occurring when partial quantities are issued during automatic component issue.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining factories.

Assigning UDI on the Start Page
Note: This functionality is available only when Medical Device is installed. Refer to "Medical Device Workspace" for information.

You can assign the device identifier, production identifier, and unique device identifier directly to a container on the Start page. This is useful when scanning UDI from an external source. The production identifier and unique device identifier can be directly entered while the device identifier is selected.

The application populates the Device Identifier field automatically if there is only one device identifier specified on the selected product. A list appears if there is more than one device identifier specified on the selected product.

Start Page

Use the Start page to start a single container. The lower section of the page is collapsed so that only the fields that are required to start a container appear when you first access the Start page. Click the Attach Document control to display the attachment options. Click the Show additional fields control to display the additional optional fields.

How to Start a Container

Follow these steps to start a container:

1. Open the Start page from the Container menu. The Start page appears.
2. Enter a name for the container to be started in the Container Name field.

Or

Click the Generate Names Automatically check box.

3. Select a Level. The selected level appears in the Level field.
4. Select an Owner. The selected owner appears in the Owner field.
5. Select a Start Reason. The selected reason appears in the Start Reason field.
6. Select a Workflow (name and revision). The selected workflow name and revision appear in the Workflow fields.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click the Attach Document control to display the document attachment fields if the section is not expanded.
9. Refer to the appropriate document attachment procedure if you want to attach a document.
• "How to Attach a New Document that Will Not Be Reused"
• "How to Attach a New Document that Will Be Reused"
• "How to Attach an Existing Document"
10. Click the Show additional fields control to display the additional fields if the section is not expanded.
11. Click Start or Submit on the command bar. A status message appears, confirming that the container has been started.

How to Attach a New Document that Will Not Be Reused

Follow these steps to attach a new document that will not be reused:

1. Perform the "How to Start a Container" procedure.
2. Expand the Attach Document section, if necessary.
3. Select Attach a new document that will not be reused (no Revision needed). The Document Path, Document Name, and Document Description fields appear.
4. Click the File Upload button to select a file.
5. Enter a name for the document in the Document Name field.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Start or Submit on the command bar. A status message appears, confirming that the container has been started.

How to Attach a New Document that Will Be Reused

Follow these steps to attach a new document that will be reused:

1. Perform the "How to Start a Container" procedure.
2. Expand the Attach Document section, if necessary.
3. Select Attach a new document that will be reused (Revision must be specified). The Document Path, Document Name, Revision, Revision of Record, and Document Description fields appear.
4. Click the File Upload button to select a file.
5. Enter a name for the document in the Document Name field.
6. Enter the revision of this document in the Revision field.
7. Select the box next to the revision optionally to indicate this document is the revision of record.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
9. Click Start or Submit on the command bar. A status message appears, confirming that the container has been started.

How to Attach an Existing Document

Follow these steps to attach an existing document:

1. Perform the "How to Start a Container" procedure.
2. Expand the Attach Document section, if necessary.
3. Select Attach an existing document (select below). The Document Instances, Revision, Revision of Record fields, and View Document button appear.
4. Select a document in the Document Instances field.
5. Enter the specific revision in the Revision field.

Or

Select the Revision of Record check box.

6. Click View Document if you want to view the document before you attach it. The application prompts you to open or save the file, or it automatically downloads the document. (Your Internet browser determines whether the application displays a pop-up, its appearance, and the wording of the Open and Save options.)
7. Click Start or Submit on the command bar. A status message appears, confirming that the container has been started.
