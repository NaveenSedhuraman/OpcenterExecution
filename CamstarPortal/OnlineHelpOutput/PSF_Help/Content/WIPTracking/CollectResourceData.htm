Collect Resource Data

Use the Collect Resource Data transaction to record parametric and user data for a shop floor resource. Typically, a resource is a machine where material is processed. The data to be recorded is determined by the data collection or user data collection definitions specified for the resource.

Before Collecting Data

A system administrator must define at least one data collection definition or user data collection definition in Modeling before you can collect data for a resource. Data collection definitions reference a parametric data control form created by a designer. User data collection definitions reference data points, which are points in a business process where data can or must be collected.

The requirements for parametric-based and user-based data are defined by your business and your system administrator adds specific fields based on these requirements. Additionally, a system administrator must configure the application to use parametric-based data collection definitions.

Refer to these guides for information:

• Opcenter Execution Medical Device and Diagnostics Data Collection User Guide or the Opcenter Execution Core Data Collection User Guidefor information on defining a parametric data control
• Opcenter Execution Medical Device and Diagnostics Data Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining data collection definitions and user data collection definitions
• Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information on setting up parametric data controls

When Collecting Resource Data

All available resources and carriers are displayed in the Resource list unless you select a Resource Group to limit the number of resources displayed. When you select a resource, the application updates the header to display information related to the resource, including the resource's current availability.

The Data Collection Def list contains both data and user data collection definitions. After selecting a data or user data collection definition, the application displays either a grid or data points depending on how the data collection definition was configured.

When Limits Are Defined for User-Defined Data Collections

The application may display upper and lower limits for data points, depending on how the user data collection was defined. If the Data Collection Definition is using the Row Column format, the maximum value that can be displayed for the upper and lower limit is 7 characters. If the value falls outside of a defined limit, the application highlights the cell and the violated limit in either yellow or red. The color depends on how the data collection was defined also.

If the application highlights the cell and violated limit in:

• Yellow - the data point has been defined to allow overrides
• Red - the value data point has been defined to forbid overrides

The application allows you to execute a data collection transaction regardless of the limit override settings. When the data collection is associated with an EProcedure task, the limit override settings determine whether a data collection EProcedure task passes or fails.

Note: The application displays negative limits in parentheses.

Resource Data Collection Page

You can access the Resource Data Collection page from the Resource menu. The Resource Data Collection page allows you to record resource data using data or user data collection definitions.

If the Data Collection Definition is using the Row Column format, the maximum value that can be displayed for the upper and lower limit is 7 characters.

How to Collect Resource Data

Follow these steps to collect resource data:

1. Open the Resource Data Collection page from the Resource menu. The Resource Data Collection page appears.
2. Select a Resource Group if you want to filter the list of resources to be displayed.
3. Select the Resource for which you want to collect data. The application updates the header to display information for the selected resource.
4. Select a data collection definition from the Data Collection Def field. The data collection fields appear.
5. Complete the data collection fields.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Collect Resource Data. A status message appears stating the data was collected.
