Operational View

The Operational View page is designed for container processing on a production line, such as at an assembly step. You can scan or select a container to perform the same transactions on a container based on how your production line works.

The Operational View page displays an In Process Containers grid. It lists the containers that are available to be processed at the operation.

The page can also display an In Queue Containers grid for containers that have just arrived at the operation and are waiting to be processed.

You scan or select a container in one of the grids and then select the appropriate action to process it.

Example

An assembly operator can process a container that is In Queue by performing a Move In or Move In Immediate transaction (to change the status from In Queue to In Process). The operator can then either continue processing more In Queue containers, or designate an In Process container for further processing with actions such as Move, Rework, Open, Close, and so forth.

The following occurs when you select or scan a container, or click a container in one of the grids:

• The selected container is highlighted in the grid and displayed in the Container field.
• The container status icon displays the current status of the selected container.
• The Shop Floor Txns button displays on the command bar.

When you click the Shop Floor Txns command bar button, the transactions available to be performed for that container and that have been configured to appear are displayed. Some of the available transactions have an ellipsis after the name, for example Move... The ellipsis indicates that when you click the transaction, the application does the following:

• Displays the corresponding transaction pop-up. In a few cases such as EProcedure, which does not have a corresponding pop-up, the transaction page appears in its own tab and the focus moves to that page while the Operational View page remains open in its own tab.
• Displays the selected container in the Container header on the transaction pop-up (or page).

You can then enter the required information in the pop-up and complete it.

Transactions without an ellipsis after the name, such as Move In Immediate and Move Immediate, indicate that the corresponding transaction will be completed immediately by automatically applying the default values for the transaction.

Note: If a WIP Message has been configured for either the Move In or the Move transaction, the application will require you to acknowledge the WIP message before executing the Move In Immediate or Move Immediate transactions.

The Document Set command bar button displays a list of the documents/SOPs in the Document Set if any are associated with the container. You can select a document and open it from the list.

Container Priority with Dispatch Rules

If dispatch rules are enforced and depending on how they are configured, you may be limited to moving only the top container in either of the grids. Containers are sorted by priority, due date, and container in a default Manufacturing setup.

How to Process a Container Using Grids

Follow these steps to process a container using the Operational View page:

1. Open the Operational View page from the Container menu. The Operational View page appears. The In Queue and In Process option is selected and both grids are displayed by default.
2. Optionally, click the In Queue or the In Process option to view only containers that are already in queue or in process.
3. Perform one of these to enter a container:
• Select a container in the Container field. The selected container name appears in the Container field, the row with that container is highlighted in the appropriate grid, and buttons for the available transactions appear.
• Scan a container. The scanned container name appears in the Container field, the row with that container is highlighted in the appropriate grid, and buttons for the available transactions appear.
• Click the row in the appropriate grid that lists the container to be processed. The row is highlighted, the container name appears in the Container field, and the Shop Floor Txns command bar button appears.
4. Click the Shop Floor Txns command bar button and select the appropriate transaction.If you are performing:
• The Move In Immediate transaction, a success message appears, the container is processed, the container row is cleared in the In Queue Containers grid and is displayed and selected in the In Process Containers grid. The container remains selected for further processing if needed.
• A Move Immediate transaction, a success message appears, the container is processed and is displayed and selected in the In Process Containers grid. The container remains selected for further processing if needed.
• Any other transaction, the transaction pop-up or page appears.
5. Complete the transaction on the pop-up or page. The container is processed accordingly, a success message appears, and the container remains selected for further processing if needed.
6. Repeat steps 2-5 to process another container as needed.
