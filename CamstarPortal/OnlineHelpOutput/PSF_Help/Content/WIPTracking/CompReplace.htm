Replace Components

The Component Replace transaction enables you to replace components at any step in a product's assembly.

You must first enter or select a container that contains the component that you want to replace. Entering or selecting a container populates the Product | Container | Qty | Reference Designator field with a hierarchical view of any components issued to the selected container. You can drill down to any level in the hierarchy to select the component that you want to replace.

You can also enter or scan the instance that you want to replace in the Scan Container/Product to Select field.

The Materials Requirements grid displays information for the component specified for replacement and any other materials at the same level of the product's material list.

When Replacing Component Materials

You must enter or scan the name of the replacement container, product, or substitute product in the Scan Container/Product field. Entering or scanning a container or product displays component replace fields according to the issue control type associated with the specified instance. The application populates the fields with the replacement material's information.

Substitute products can also be used when replacing a component product. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on assigning substitute products.

You can use multiple containers or products to replace component material for one component if:

• You require more quantity than is available in a replacement container or product.

Or

• You require component material from multiple containers.
Note: Containers or products assigned the Serial issue control type are completely issued during replacement regardless of quantity. Replacing a quantity less than the quantity available in a Serial container or product automatically fulfills the component's quantity requirement and you will not be able to use quantities from multiple containers.

The Scrap part being replaced checkbox enables you to designate the material removed during replacement as scrap. This field defaults to the setting on Replace Reason code. It can be overridden when executing the replace transaction.

Issue Controls

Issue controls specify how material is dispensed to containers and how it is tracked in the application. Component replace user fields are displayed and populated according to the issue control type associated with the specified replacement material. The component replace transaction supports five of the default issue control types:

• Issue Container (Lot) - Component material is issued and tracked in the application at the container or lot (bulk) level. One component container can provide part or all of the required quantity, up to the current quantity of the component material. Issue Qty is required when replacing material.
• Issue Container (Serial) - Component material is issued and tracked as a serialized container in the application. The container’s entire quantity is consumed when a serial container is used to issue material to a component.
• Lot and Stock Point - Component material is issued from lots and locations external to the application, and the material is not tracked by the application. Lot Number and Issue Qty are required when replacing material.
• Stock Point Only - Component material is issued from a location external to the application and the material is not tracked by the application. Issue Qty is required when replacing material.
• Quantity Only - Component material is not tracked by any system. Issue Qty is required when replacing material.

Refer to "Issue Controls" for information on the default supported issue control types.

Component Replace Buttons

The application adds replacement material to the Material Requirements grid when you click Add to Pending. A subgrid is displayed beneath the component being replaced containing information on the replacement material. The component's quantity values are updated to display the amount of quantity pending for replacement. You can designate replacement material for multiple components in one container using the Add to Pending button.

Important: Multiple instances of a component are displayed in the Material Requirements grid if the component's required quantity was originally issued using multiple containers. Siemens does not currently support the ability to replace multiple issues of a component using a single replacement container. You must remove and re-issue components within a container if you want to replace materials using the same replacement container or product. Refer to "Remove Components" and "Component Issue" for information.

The Component Replace button is enabled when the quantity requirement has been either satisfied or exceeded for at least one component in the grid. Clicking the Component Replace button performs the component replace transaction on any pending components added to the Material Requirements grid.

The Reset button clears any pending replacement material and resets the values on the page.

How to Replace Issued Components

Follow these steps to replace issued components:

1. Open the Component Replace page from the Container menu. The Component Replace page appears.
2. Enter or scan a container in the Container field. The application updates the container header to display the container's information and populates the Product | Container | Qty | Reference Designator field to display all materials issued to the container.
3. Scan or enter the component product, container, or reference designator of the component you want to replace in the Scan Container/Product to Select field. The application displays the specified instance in the Material Requirements grid and populates the grid with other components at the same level in the associated product's material list.

Or

Select the component product or container you want to replace in the Product | Container | Qty | Reference Designator field. The application displays the specified component in the Material Requirements grid and populates the grid with other components at the same level in the associated product's material list.

4. Enter or scan the container, product, or substitute product that contains replacement material in the Scan Container/Product field. The application displays user fields according to the issue control type associated with the specified container or product.
5. Enter any required information in the user fields according to the issue control type of the specified container or product.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
7. Click Add to Pending. The application adds the replacement material to the Material Requirements subgrid, updates the Qty values in the grid, and displays a success message.
8. Repeat steps 2-7 to add additional replacement material to the grid until all requirements are satisfied for the components you are replacing. The application adds the replacement material to the Material Requirements subgrid, updates the Qty values in the grid, enables the Component Replace button, and displays a success message.
9. Click Component Replace or Submit on the command bar. The application performs the component replace transaction for the pending instances and displays a success message.
