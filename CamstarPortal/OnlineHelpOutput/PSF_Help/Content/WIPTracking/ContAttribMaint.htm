Container Attribute Maintenance

The Container Attribute Maintenance page allows you to add and manage custom, dynamic attributes for individual containers. User-defined container attributes are a dynamic set of properties that a user can change at any time. These attributes can be unique for each container and can be managed dynamically at runtime.

The ability to assign custom attributes enables you to store information on the container that is easily accessible. For example, you may want to store lab results on the container itself to prevent having to search through the container’s history. An attribute could also be as simple as the container’s color.

Note: Use the Container Maintenance page to manage predefined, static container properties (for example, container level, owner, and so on).

When Adding Attributes to a Container

The application can assign user-defined attributes automatically when you start a container and during user data collections. You can assign attributes manually on the Container Attribute Maintenance page or pop-up.

Refer to the following for information on assigning user-defined attributes automatically:

• The Opcenter Execution Medical Device and Diagnostics Modeling User Guide or Opcenter Execution Core Modeling User Guide for information on assigning user-defined attributes on a container start.
• The topic "EProcedure" for information on assigning user-defined attributes as part of user data collection.

The Container Attribute Maintenance page or pop-up displays the attributes assigned to the selected container at that point in time. The attribute list includes attributes assigned on a container start, as part of a user data collection, or manually.

All display fields are empty on the Container Attribute Maintenance page until a container is selected. Display fields are populated automatically on the Container Attribute Maintenance pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The name of each attribute must be unique when adding an attribute to a container. The attribute value can be static text or an expression. For example, an attribute for color could be static text while an attribute for container volume might be an expression. The application requires you to provide an attribute name if you select an attribute type or enter a value, even though attributes are not required.

Double-clicking anywhere on a row activates the row so that you can update attributes. Clicking the Delete selected row button removes the selected attributes from the grid. You can select the check box in the header row to select all displayed attributes automatically or you can select the check boxes for individual attributes.

Clearing the Container field clears any attributes from the grid.

You can add or update attributes for closed containers and for held containers.

You can add a comment to a container with or without adding any attributes.

Container Attribute Maintenance Page

You can access the Container Attribute Maintenance page from the Container menu. The Container Attribute Maintenance page allows you to add attributes to a container or to update previously added attributes.

Note: The Container Attribute Maintenance page may be available as a pop-up also if the container attribute maintenance transaction has been configured as an available action on another page (for example, Container Search).

How to Add Attributes to a Container

Follow these steps to add attributes to a container:

1. Open the Container Attribute Maintenance page from the Container menu. The Container Attribute Maintenance page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Container Attribute Maintenance pop-up from another shop floor page.
3. Click the Add new row button in the User Attribute List grid. A row is activated in the grid.
4. Enter a name for the attribute in the Attribute Name field.
5. Select the attribute type from the Type list.
6. Enter a static value or an expression in the Value field.
7. Select the Is Expression check box if the value in the Value field is an expression.
8. Repeat steps 3-7 to add additional attributes.
9. Click Update Container or Submit on the command bar. The application saves your changes and displays a message indicating that your updates were successful.
