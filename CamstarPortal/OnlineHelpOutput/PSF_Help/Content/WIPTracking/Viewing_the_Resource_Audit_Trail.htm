Viewing the Resource Audit Trail

The resource audit trail enables you to view transaction history for resources. The application begins recording transactions immediately when you set up a resource.

When Searching the Resource Audit Trail

Only the Search tab is visible when the resource audit trail initially loads. You can enter information in the fields to define your search.

You must select a resource from the Resource list in the Filters section. The other fields in the Filters section are optional. Selecting an item in any of the optional fields filters the resources listed in the Resource field. Selecting an item in the optional fields does not affect the items listed in the other optional fields.

For example, if you select Microscopes from the Resource Type list, the application lists only Microscope[01-05] in the Resource list. Only Microscope01 is in the Resource Group Inspection Lab. If you then select Inspection Lab from the Resource Group list, Resource lists only Microscope01. Selecting Microscopes from the Resource Type list has no effect on the items listed in the Resource Family or Resource Group fields.

Filters in the Timeframe section enable you to narrow search results by date and time.

Filters in the Transactions section enable you to narrow search results by transaction type. The Transaction Filter field enables you to:

• Limit the transactions to resource-based transactions and their subclasses
• Example: Resource Data Collection, Resource Setup, Resource Thruput, and Maintenance Management
• Limit the transactions to container-based transactions (and their subclasses) that reference resources
• Example: MoveIn transaction that specifies a resource or an Electronic Procedure task in which you select a resource

Executing a Search

The History Mainline grid appears and additional tabs such as Attributes, Esig, Label Printing, Data Collection, and WIP Message may appear when you execute a search.

Secondary filter fields enable you to further refine your search. These fields appear on the Search tab after you perform your initial search. Selections in these fields depend on the transaction information returned by the initial search. The application narrows the results displayed in the History Mainline grid using these criteria.

When Selecting a Transaction to View

The application displays a grid containing additional transaction information below the History Mainline grid when you select a transaction in the History Mainline grid. The grid may contain one or more tabs depending on the resource type, transaction type, and the functionality configured for the transaction.

Note: System administrators can configure the information displayed on the tabs using the Audit Trail Configurator. Refer to the Opcenter Execution Medical Device and Diganostics Audit Trail User Guide or the Opcenter Execution Core Audit Trail User Guide for information.

Resource Audit Trail Page

The Resource Audit Trail page enables you to view the history for a resource. The page consists of these parts:

• Search tab - appears on page load
• Status tab - appears after initial search results are returned only if a system administrator has defined a UI preference for the resource in Modeling
Note: The panel containing the tabs is collapsible. Collapsing the tab panel expands the grids horizontally.
• Breadcrumb Navigation - appears after you select a transaction in the History Mainline grid and displays a navigable history of your selected transactions
• History Mainline grid - appears after the initial search and displays all transactions for the specified resource that meet the search criteria.
• Transaction information tabs - appear when you select a transaction in the History Mainline grid

Status Tab

The Status tab displays information about the specified resource. The application displays the resource name and the information specified in the UI Preference modeling object instance associated with that resource type or resource family. The application does not display the Status tab if a system administrator has not defined a UI Preference.

History Mainline Grid

The History Mainline grid lists all the transactions executed against the specified resource. When you select a transaction in this grid, the application displays a subgrid containing one or more tabs with additional transaction information.

You can expand the History Mainline grid horizontally by collapsing the Search tab panel. Expanding the History Mainline grid enables you to see more of the transaction history without having to scroll horizontally. The History Mainline grid expands to the width of the Resource Audit Trail page.

How to View Transaction History for a Resource

Follow these steps to view transaction history for a resource:

1. Open the Resource Audit Trail page from the Resource menu. The Resource Audit Trail page appears.
2. Select a Resource Type, Resource Family, and Resource Group to limit the resources available for selection in the Resource list, if necessary.
3. Select the resource you want to search for in the Resource list.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Search. The application displays the Status tab (if applicable) and the History Mainline grid for the specified resource.
6. Select a transaction in the History Mainline grid. The application updates the breadcrumb navigation and displays the applicable transaction grid below the History Mainline grid.
