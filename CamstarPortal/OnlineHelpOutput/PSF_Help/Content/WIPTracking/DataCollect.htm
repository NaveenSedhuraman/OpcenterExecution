Data Collection

Data collection is used for high-quality decisions in your factory including categorization, binning, failure, and engineering analysis. Data collection is an integral part of shop floor transactions, and demonstrates the application's flexibility and responsiveness to your factory run-time needs.

Siemens provides two types of container data collection methods:

• Parametric-based data collection
• User-based data collection

The data you collect, including category, bin, failure, resource, engineering analysis, and other data, is based on the current Spec for the processing step.

Before Collecting Data

A system administrator must perform these tasks before you can use the Collect Data page:

• Define at least one data collection definition or user data collection definition in Modeling.
• Assign one or more data or user data collection definitions to a spec in Modeling.

Data collection definitions reference a parametric data control form created by a designer. User data collection definitions reference data points, which are points in a business process where data can or must be collected. The requirements for parametric-based and user-based data are defined by your business and your system administrator adds specific fields based on these requirements.

Refer to the Opcenter Execution Medical Device and Diagnostics Data Collection User Guide or the Opcenter Execution Core Data Collection User Guide for information on defining a parametric data control. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining data collection definitions and user data collection definitions and assigning them to specs.

When Collecting Container Data

You can collect both parametric-based and user-based data on the Collect Data page when the container is at a spec for which data or user data collection definitions have been assigned. Additionally, you can collect data from the EProcedure page when a data collection task is part of the task list assigned to the electronic procedure. Refer to "EProcedure" for information on electronic procedures.

When viewing the Collect Data page, the application displays data collection fields by default if only one data or user data collection has been associated with the specified container's current spec. If more than one data or user data collection definition has been associated with the spec, the application displays the Data Collection Def field, allowing you to select a data collection.

When Limits Are Defined for User-Defined Data Collections

The application may display upper and lower limits for data points depending on how the user data collection was defined. If the value falls outside of a defined limit, the application highlights the cell and the violated limit in either yellow or red. The color depends on how the data collection was defined also.

If the application highlights the cell and violated limit in yellow, the data point has been defined to allow overrides. If the application highlights the cell and violated limit in red, the value data point has been defined to forbid overrides. The application allows you to execute a data collection transaction regardless of the limit override settings. The limit override settings determine whether a data collection EProcedure task passes or fails when the data collection transaction is associated with an EProcedure task.

Note: The application displays negative limits in parentheses.

Collect Data Page

You can access the Collect Data page from the Container menu. The Collect Data page allows you to collect parametric-based and user-based data for a container.

Note: The Collect Data page may be available as a pop-up also if the collect data transaction has been configured as an available action on another page (for example, Container Search).

How to Collect Data on a Container

Follow these steps to collect data on a container:

1. Open the Collect Data page from the Container menu. The Collect Data page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Collect Data pop-up from another shop floor page.
3. Select a data or user data collection definition from the Data Collection Def field, if necessary. The data collection fields appear.
4. Complete the data collection fields.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Collect Data or Submit on the command bar. A status message appears confirming that the data has been collected.
