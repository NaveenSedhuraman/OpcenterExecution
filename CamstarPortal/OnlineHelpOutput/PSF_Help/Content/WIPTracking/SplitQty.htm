Split Quantities

The Split Quantity transaction creates one or more target containers from an existing source container and then transfers a partial quantity of product from the source container to the new target containers.

Note: If you want to transfer one or more child containers from a parent container to a new parent container, use the Split Container transaction. Refer to "Split Containers" for information.

Examples of Splitting Quantities and Containers

These are some examples of how you use the Split transactions:

• A small quantity of a container is split so it can be processed more quickly to meet an expedited customer or inventory requirement.
• A portion of a container with quality problems is split before performing a Rework or Hold transaction. The remainder of the container is processed normally.
• Part of a container is split so it can be moved to a different workflow for processing a special order.

When Splitting a Quantity or Container

Splitting a container or quantity does not move either the source or target containers. Nor does it imply that any processing activity (throughput) was performed on either container.

You must specify a unique container name when creating a new container. You can either enter a container name manually or have the application generate a container name automatically. The application considers closed containers when validating a unique name.

Select the Generate Names Automatically check box to have the application generate the container name using defined numbering rules. The application clears and disables the To Container field when you select this check box.

An error message appears when Generate Names Automatically is selected but there is no auto numbering rule defined.

The application uses the numbering rules associated with the parent container from which the child container is being split to generate the To Container name. The application generates the name of the new parent container when you click the Submit button.

Numbering rules may be specified for multiple modeling object instances associated with the container. Numbering rules follow this default precedence:

1. Container Level
2. MfgOrder
3. Product
4. Product Family
5. Factory

Adding Affected Containers to Events

The application can add containers to quality events automatically when you execute the Split Qty transaction on a container associated with one or more quality events. For example, if the source container is associated with an event, the application adds any new containers split from that source container to the quality event automatically. A system administrator configures the Split transaction to add containers to events in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Event Management User Guide or the Opcenter Execution Core Event Management User Guide and the Opcenter Execution Medical Device and Diagnostics Nonconformance Management User Guide or the Opcenter Execution Core Nonconformance Management User Guide for information on adding affected containers to quality events automatically.

Split Qty Page

The Split Qty page enables you to:

• Select an eligible container.
• Define one or more new target containers to receive the quantity to be split from the source container.
• Specify the quantity to be split from the source container into the new target containers.
• Select an option to close the source container if its entire container quantity is moved to the target containers.

How to Split a Container's Quantity

Follow this procedure to use the Split Quantity page or the Split Quantity pop-up to split the entire contents of a source container into one or more new target containers.

1. Open the Split Qty page from the Container menu. The Split Qty page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Split Qty pop-up from another shop floor page.
3. Click Add new row or select a row to start a new container.
4. Enter a name for the new container in the To Container Name field in the first row of the Add Container and Quantity to Split grid.

Or

Select the Generate Names Automatically check box.

5. Enter the quantity for the new container in the associated Quantity field.
6. Repeat steps 3-5 if you want to add another new container and quantity.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Split or Submit on the command bar. The specified quantity or quantities are split into the target containers, the application clears the fields, and then displays a confirmation message.
