Resource Throughput

The Resource Thruput transaction records throughput information—actual processed quantities—for a resource. This is so production processing can be recorded in situations where the material being produced is not being tracked in a traceable manner.

It can also apply when you need to record throughput information not associated with a container. This can occur when a resource is tested by processing test material that is not part of a regular workflow. If the resource test material is outside the regular workflow, it is not practical to use container transactions for resource testing.

When Using Resource Throughput

The Resource Thruput transaction can only record throughput for a single resource per transaction. However, multiple Resource Thruput transactions can be performed on a single resource.

Resource availability must be UP for a resource to accept throughput. You can change availability for the resource in the Resource Setup page.

The Resource field displays all resources and carriers unless you select a Resource Group. Then the display of resources and carriers is filtered by that Resource Group.

The UOM field can be left blank, but cannot be set to an incorrect value.

If your site has implemented Maintenance Management, throughput quantities can be added that will exceed the quantity limit. However, throughput cannot be added once that limit has been exceeded.

Additional Methods to Add Throughput

Resource Thruput is one of the transactions used to add throughput to a resource with a throughput requirement (using Maintenance Management). In addition, you can also use these transactions to add throughput:

• Container Thruput page
• Move In transaction
• Move transaction (at an Operation with “Use Queue” checked)

Resource Thruput Page

You can access the Resource Thruput page from the Resource menu. Use the Resource Thruput page to record throughput information (actual processed quantities) for a resource. This applies when you need to track material that is not associated with a container; for example, a qualification lot that still needs to be processed by the resource.

How to Record Resource Throughput

Follow these steps to record resource throughput:

1. Open the Resource Thruput page from the Resource menu. The Resource Thruput page appears.
2. Select a Resource Group if you want to limit the list of resources displayed.
3. Select the Resource for which you want to record a processed quantity. The page updates, displaying an icon for the resource's availability, the Time at Status (time at current status), the Status Code (current Resource Status), and the Status Reason (current Resource Status Reason).
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Thruput. A message indicating success appears and the resource's remaining throughput counter is incremented by the quantity that was scheduled for processing but was not processed.
