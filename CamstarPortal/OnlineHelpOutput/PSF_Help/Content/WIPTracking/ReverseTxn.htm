Reverse Last Transaction

The Reverse Last Transaction feature allows you to reverse the last transaction performed on a container, if the transaction is reversible. For example, moving a container to the next step in a workflow is reversible, but a data collection for a container is not.

When Reversing the Last Transaction

When you access the Reverse Transaction page, scanning or entering a Container populates the header and makes selections available in the Views field. You must select a view before the application will populate the Recent Transactions grid, the Transaction Details grid, and activate the Reverse Last Txn button.

The application defaults to a view if only one view has been defined (the Default view) or if a view has been specified for the user in the Employee modeling object. The columns that appear in the Recent Transactions grid are based on the selected View, although column order may vary.

Note: Views are defined using the Audit Trail Configurator. Refer to the Opcenter Execution Medical Device and Diagnostics Audit Trail User Guide or Opcenter Execution Core Audit Trail User Guide for information on defining views.

The Recent Transactions grid contains a list of the most recent transactions in descending order. The application displays a maximum of 10 transactions in the grid and automatically selects the first (most recent) reversible transaction. If the first transaction in the list is a non-reversible transaction (for example, data collection), the application selects the second transaction. If there are more than 10 transactions, the application adds paging functionality.

The Transaction Details grid displays information about the transaction selected in the Recent Transactions grid.

A successful transaction reversal causes the application to:

• Apply a yellow background to the transaction that was reversed
• Add the reversal transaction to the grid and applies a red background
• Select the next reversible transaction, which updates the information displayed in the Transaction Details grid
• Resume any process timers that were stopped
Note: A resumed timer references its original start time to calculate remaining time.
Note: Reversing a Start transaction deletes the container. The application updates the container Status to Deleted.

Reverse Last Transaction Page

You can access the Reverse Last Transaction page from the Container menu. The Reverse Last Transaction page allows you to reverse the most recent reversible transaction, which is selected by default.

How to Reverse a Shop Floor Transaction

Follow these steps to reverse a shop floor transaction:

1. Open the Reverse Last Transaction page from the Container menu. The Reverse Last Transaction page appears.
2. Select the Container for which you want to reverse a transaction. The application updates the container header. If a view is selected by default, the application populates the Recent Transactions and Transaction Details grids, selects the first reversible transaction, and enables the Reverse Last Txn button.
3. Select Views. The application displays the Recent Transactions and Transaction Details grids, selects the first reversible transaction, and enables the Reverse Last Txn button.
4. Click Reverse Last Txn or Submit on the command bar. The application reverses the transaction, applies a yellow background to the reversed transaction, inserts a corresponding reversal transaction (indicated by a red background), and selects the next reversible transaction.
