Viewing the Manufacturing Audit Trail

The audit trail enables you to view transaction history for containers, carriers, and events. Refer to the Opcenter Execution Medical Device and Diagnostics Audit Trail User Guide or the Opcenter Execution Core Audit Trail User Guide for information on configuring the manufacturing audit trail.

When Accessing the Manufacturing Audit Trail

You can access the manufacturing audit trail by:

• Selecting Mfg Audit Trail from the Portal's Search menu. The application displays the Audit Trail page.
• Clicking the Mfg Audit Trail command bar button. The Mfg Audit Trail command bar button is available on every shop floor transaction page that has a container information web part. The application displays the manufacturing audit trail in a pop-up.

When Viewing Transaction History

The Search tab is the only visible tab when you access the Mfg Audit Trail page. You must specify a view and a name for the selected record type in the Type section of the page to perform the search. You can also enter information in the Timeframe and Transaction sections to further refine your search.

The application populates the View field automatically when one of these conditions is true:

• Only one view has been configured.
• A default history view was specified for the logged-in employee. Default history views are specified in the Employee modeling object. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.
Note: You can change the default view if more than one view has been configured.

The application populates the Record Type and Name fields automatically when you are redirected to the Mfg Audit Trail page after clicking the Mfg Audit Trail command bar button on a shop floor page or the Container Search page.

Additional tabs may appear after you execute a search, depending on the record type you selected. The Status and Attributes tabs appear for containers and carriers only. The Status tab always appears if a UI preference is defined, but the Attributes tab appears only if user-defined attribute data is available for the specified container or carrier. The Event Log tab appears only for events if event log data is available.

The application displays secondary filters on the Search tab after you perform your initial search on a container or carrier. Selections in these fields depend on the transaction information returned by the initial search. You can use the secondary filters to further refine your search.

Note: The secondary filters are visible automatically when the application redirects you to the Mfg Audit Trail page after clicking the Mfg Audit Trail command bar button on a shop floor page.

Refer to the Opcenter Execution Medical Device and Diagnostics Audit Trail User Guide or the Opcenter Execution Core Audit Trail User Guide for information on the manufacturing audit trail and transactions.

Note: The panel containing the tabs is collapsible. Collapsing the tab panel expands the grids on the page horizontally.

The application displays a breadcrumb navigation and a grid with one or more tabs below the History Mainline grid when you select a transaction in the History Mainline grid. The tabs displayed depend on the record type, transaction type, and the functionality configured for the transaction. For example, if there were attributes assigned to a container on the Start transaction, the application displays an Attributes tab.

Note: System administrators can configure the information displayed on the tabs using the Audit Trail Configurator.

Mfg Audit Trail Page

The Mfg Audit Trail page enables you to view the history for a container, carrier, or event. The page consists of these parts:

• Search tab - appears on page load
• Status tab - appears after initial search results are returned only if status data exists
• Attributes tab - appears after initial search results are returned for containers and carriers only, and only if user-defined attributes have been assigned to the carrier or container
• Event Log tab - appears after initial search results are returned for events only, and only if comments were entered on the Log tab when managing production and generic events
• Breadcrumb Navigation - appears after you select a transaction in the History Mainline grid and displays a navigable history of your selected transactions
• History Mainline grid - appears after the initial search and displays all transactions for the specified entity that meet the search criteria
• Transaction information tabs - appear when you select a transaction in the History Mainline grid

History Mainline Grid

The History Mainline grid lists all the transactions executed against the specified container, carrier, or event. The columns displayed in the grid depend on the view you selected in the Views field. When you select a transaction in this grid, the application displays a subgrid containing one or more tabs with additional transaction information.

You can expand the History Mainline grid horizontally by collapsing the Search tab panel. Expanding the History Mainline grid enables you to see more of the transaction history without having to scroll horizontally. The History Mainline grid expands to the width of the Mfg Audit Trail page.

How to View Transaction History for a Container, Carrier, or Event

Follow these steps to view transaction history for a container, carrier, or event:

1. Open the Mfg Audit Trail page from the Search menu. The Mfg Audit Trail page appears.
2. Select the type of entity you want to search for in the Record Type field.
3. Select a different view from the Views field, if necessary.
4. Enter the Name of the entity.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click Search. The application displays the Status tab (if applicable), the Attributes tab (if applicable), and the History Mainline grid for the specified entity.
7. Select a transaction in the History Mainline grid. The application populates the breadcrumb navigation and displays the applicable transaction grid below the History Mainline grid.
8. Select a transaction in the breadcrumb navigation to return to a previously selected transaction, if applicable.
9. Perform the "How to View a Document for a Container or an Event" procedure if you want to see the details of a document associated with a container or an event.
