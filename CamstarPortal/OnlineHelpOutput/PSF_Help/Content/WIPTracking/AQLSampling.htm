AQL Sampling

During a manufacturing process, sampling inspection is a statistical practice that estimates production quality by modeling and executing sampling plans for containers and container groups (also known as lots). This practice usually tests part of a lot and not the entire lot. Acceptable Quality Limits (AQL) sampling plans define the number of samples required (sample rate) and the associated acceptance and rejection limits. Sampling requirements for a container are determined during the Start or Move transactions if the container does not require a queue. The sampling requirements are determined during the Move In transaction if the container requires a queue.

There are two types of sampling available with the separately licensed AQL Sampling module: container sampling and lot sampling. You can use container sampling to evaluate an individual container against the sampling requirements. You can use lot sampling to evaluate a group of containers as one unit against the sampling requirements.

When using container sampling, the AQL Sampling module verifies all required sampling has been completed before allowing the container to move to the next step. Based on the sample’s inspection data, AQL Sampling determines if a container is accepted or rejected. A container fails sampling if the number of rejects exceeds the acceptable reject limit. Failed containers are marked as failing sampling and are not allowed to move to the next step unless overridden on the Sample Test object. Sampling data can be collected over time to prevent the need to enter all sample results at one time and to accommodate situations where sampling is done in batches.

The lot sampling process begins with the creation of a sampling lot. To group containers into a lot, you can assign containers to a sampling lot when starting a container or add them to an existing lot using the Container Maintenance page. The application uses the sampling lot's quantity to determine the number of samples required for sampling and the number of acceptable rejects. The sampling requirements can adjust based on the quantity of the lots assigned to the container if the quantity is greater than the original lot quantity. The application accumulates data for the lot as the sampling data from each container within the lot is collected for each sample test in the sampling plan. The application then uses this data to verify if the lot passes or fails inspection. If the Validate Lot Sampling check box on the spec is selected, all required sampling must be completed for the lot before any containers associated to the lot can move to the next step.
