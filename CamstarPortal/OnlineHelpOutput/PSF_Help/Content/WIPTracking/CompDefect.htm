Component Defect

The Component Defect transaction allows you to record defects against components issued to a container. These are defects that do not result in immediate removal or scrap of the components. This transaction applies to both containers and the components issued to those containers.

Note: Use the Container Defect transaction if you want to record defects against a container as a whole.

Before Using Component Defect

You must configure component defect reasons in Modeling before you can record defects against components. Defect reasons are available for selection under these conditions:

• A component defect reason has been defined.
• A component defect reason group containing the reason has been defined.
• The component defect reason group has been assigned to the container's current operation.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining and associating the objects necessary to record container defects.

When Using Component Defect

All display fields are empty on the Component Defect page until a container is selected. Display fields are populated automatically on the Component Defect pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application displays the components issued to the specified container in the Material Issues grid. The component rows are display-only. The Defects Recorded column indicates whether defects have been recorded against the component previously.

The application displays a subgrid where you can add, edit, and delete defects for the component when you select a component. Initially, the component subgrid displays only a header row. You add rows by clicking the Add new row button on the subgrid.

Note: The application displays a maximum of 20 rows in the grid. If the number of rows exceeds 20, the application adds paging functionality.

You can record defects for held containers, but you cannot record a defect for a container that is closed. The application does not make closed containers available for selection.

Multiple defects can be tracked for a container. The application requires you to add at least one defect reason and enter a count for that reason before you can record a defect. The application displays an error message if you click the Record Defects button without having added a defect reason and defect count.

Clicking the Delete selected row button on the subgrid removes the selected defect reasons from the subgrid. You can select the check box in the header row of the subgrid to select all displayed defects automatically or you can select the check boxes for individual defects.

The application clears the subgrids for each component when you click the Record Defects button and the transaction is successful. The Material List grid is not cleared of the component list.

Component Defect Page

You can access the Component Defect page from the Container menu. The Component Defect page provides a view of the components issued to the selected container, issue details for those components, and allows you to record multiple defects against one or more of the issued components.

Note: The Component Defect page may be available as a pop-up also if the component defect transaction has been configured as an available action on another page (for example, Container Search).

How to Record a Component Defect

Follow these steps to record a component defect:

1. Open the Component Defect page from the Container menu. The Component Defect page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Component Defect pop-up from another shop floor page.
3. Select the component in the Material Issues grid for which you want to record a defect. The application displays the subgrid for that component.
4. Click Add new row in the subgrid. A row is activated in the grid.
5. Select a component defect reason from the Defect Reason field.
6. Enter the number of defects associated with this reason in the Defect Count field.
7. Repeat steps 4-6 to add additional defect reasons for the component.
8. Repeat steps 3-6 to add defect reasons for different components.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Click Record Defects or Submit on the command bar. The application displays a status message confirming that defects were saved.
