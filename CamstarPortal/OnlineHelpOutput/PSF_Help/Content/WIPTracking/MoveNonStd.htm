Move Containers (Non-Std)

The Move Non-Std (non-standard) transaction allows you to move a container to any step in any workflow. This transaction provides you with more flexibility than the Move transaction, which only allows you to move a container to the next step within the same workflow. (The Move Non-Std transaction is similar to the Transfer transaction, but the Move Non-Std is subject to any Move In requirements for an operation.)

Note: Refer to "Move Multiple Containers (Non-Std)" if you want to move multiple containers simultaneously.

When Using Move Non-Std

All display fields are empty on the Move Non-Std page until a container is selected. Display fields are populated automatically on the Move Non-Std pop-up. Refer to "Using Transaction Pages and Pop-ups" for information on performing a transaction from a page or pop-up.

The application displays a default value in the Resource field when one of these conditions is true:

• A move in was required for the container's current step and the user moving in the container specified a resource.

Or

• A resource was specified when the container was started and Move Non-Std is the first transaction (specifying a resource) that you are performing after starting the container.

You can select a different resource, if necessary.

Move Non-Std Page

You can access the Move Non-Std page from the Container menu. The Move Non-Std page allows you to enter or select the container you want to move to a step in a workflow.

Note: The Move Non-Std page may be available as a pop-up also if the move in transaction has been configured as an available action on another page (for example, Container Search).

Move Non-Std Page and Pop-up Field Definitions

This table defines the fields on the Move Non-Std page and Move Non-Std pop-up.

Field

Definition

Type

Container Information

Header containing the container name and information about that container. You must select or enter a container on a transaction page, but all information is display-only on a transaction pop-up. Refer to "Container Information" for information.

Required/ Display Only

Resource

Resource that processed the container at the current step. You can select a different resource if the application displays one by default. Specifying the resource used at this step before moving the container provides traceability.

The resource values available for selection depend on the resource group assigned to the step (spec). If no resource group was assigned to the spec, all defined resources are available for selection.

Optional

To Workflow

Destination workflow and steps. (The steps appear when you expand the workflow.) The application moves the container to the next step in the current workflow unless you specify a different workflow or step here.

Optional

To Step

Step in the destination workflow. The application populates this field when you select a step (spec) from the expanded workflow in the To Workflow field.

Optional

Show/Hide additional fields

Optional fields hidden by default when the application displays a page or pop-up. Clicking the Show control displays the fields. Clicking the Hide control hides the fields.

  

To Location

Location to which the container will go for processing at the next step. A location may be an entire factory, an area of a factory, or as specific as a rack identifier. (Locations are defined within the Factory object in Modeling.)

Note: This field is hidden by default.

Optional

Comments

Additional information or instructions for this transaction or container. The field accepts a maximum of 255 characters. The application truncates the data if the comments exceed 255 characters.

Optional

How to Move a Container to Any Step in a Workflow

Follow these steps to move a container to any step in a workflow:

1. Open the Move Non-Std page from the Container menu. The Move Non-Std page appears.
2. Select or enter a container in the Container field. The application updates the container header and selection fields to display information for the specified container. Values appear only if they were entered for the container.
Note: The application populates the Container field and the container header automatically when you access the Move Non-Std pop-up from another shop floor page.
3. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
4. Click Move or Submit on the command bar. A status message appears confirming that the container has been moved.
