Print Production Event Label

Container label printing is available for customers licensed to use the Label Printing module. A label is printed data, usually including a unique identifier such as the container name in readable text and/or barcode format, with additional data such as a production description. Production event labels are used to tag containers for which a production event has been recorded while the nonconformance is being analyzed.

Labels can also be reprinted. Reprinting is used when, for example, a label has been damaged or did not print successfully due to a printer problem. A reprinted label uses data stored in the container manufacturing audit trail.

Automatic vs. Manual Label Printing

The ability to print and reprint production event labels depends on the definition of modeling objects including Printer Label Definitions and Print Queues. Each production event transaction that is identified in the Production Event Label Map on the Factory object will cause the specified number of label sets to print automatically when that transaction is performed.

If the Production Event Label Map is not implemented, then you can select to manually print labels following a production event transaction.

Accessing the Print Production Event Label Page

The Print Production Event Label page can be accessed by selecting the:

• Print Production Event Label option on another Portal page
• Manually print labels after recording check box on the Record Production Event page
• Print Production Event Label option on the Container menu

Reprinting is performed on the Print Production Event Label page; there is not a separate reprint page.

Search Options

Use the Search Criteria fields on the Print Production Event Label page to search for and locate the container or production event needed.

You can use SQL wildcards in the Container field or in the Production Event Identifier field when searching. The search results are displayed in a two-level grid below the heading, Matching Production Events.

What Happens During Transaction Processing

Labels are printed manually when selected on the Record Production Event page or automatically when set up accordingly. During transaction processing, the transaction logic:

1. Reads the Production Event Label Map on the Factory object for the transaction.
2. Reads the appropriate label template file (provided externally).
3. Replaces the template tags with information from the Printer Label Definition (identified in the Production Event Label Map on the Factory object).
4. Prints one or more copies of the label, or writes the label information to a file, based on the information associated with the entry in the Print Queue field.
5. Prints labels after the transaction logic is committed to the database to prevent printer errors causing the transaction to roll back. Any errors that occur during the label printing process are written to the event log.
Note: The reprint transaction uses the label history from a previously printed label to reprint the label.
6. Writes the relevant data to the event transaction history (the manufacturing audit trail). History is recorded for every label printed or reprinted. If configured, the Manufacturing Audit Trail transaction displays detailed information on printed and reprinted labels.

When the Printer Has an Error

When you execute the print or reprint transaction but the transaction does not complete, the application displays an error message and the printing error is logged to the event log. Depending on your organization’s business practices, you should either place the container on hold, or perform the print or reprint transaction again and specify a different print queue.

If an action such as printer repair is needed, the administrator who monitors the event log will take the appropriate action.

How to Print a Production Event Label Manually

Follow these steps to print a production event label manually:

1. Open the Print Production Event Label page from the Container menu. The Print Production Event Label page appears.
2. Select By Container or By Identifier to search for production events. The application updates the Search Criteria section to display fields for your selected search type.
3. Specify search criteria.
4. Click Search. The application displays all production events meeting your search criteria. Each top-level row lists the details of a production event record that matches the search criteria entered. Each child row lists the details of a container associated with the production event record.
5. Print one or more labels as follows:
• Select the check box in a row with a production event to print one or more label sets for all the containers related to that event.
• Select the check box in a child row with a container to print one or more label sets for that container.
6. Select the name of the Production Event Transaction that requires a printed label. This field is already populated if this page is displayed from the Record Production Event page or the Manage Production Event page. The Printer Label Definition contains the default for the transaction specified in the Production Event Transaction field.
7. Select a different Printer Label Definition, if necessary.
8. Select a different Print Queue, if necessary.
9. Enter the number of label sets to be printed in the Set Count field if different from the default.
10. Click Print Selected Labels. The designated number of labels print (or reprint) at the designated printer or the application writes the label information to a file (based on the Print Queue definition).
