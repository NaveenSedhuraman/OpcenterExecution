Anzeigen des Produktionsprotokolls

Das Protokoll bietet die Möglichkeit, die Transaktionshistorie für Container, Behälter und Ereignisse anzuzeigen. Informationen zum Konfigurieren des Produktionsprotokolls finden Sie im Opcenter Execution Medical Device and Diagnostics Audit Trail-Benutzerhandbuch und im Opcenter Execution Core Audit Trail-Benutzerhandbuch.

Beim Zugreifen auf das Produktionsprotokoll

Sie haben folgende Möglichkeiten, auf das Fertigungsprotokoll zuzugreifen:

• "Produktionsprotokoll" im Menü "Suchen" des Portals auswählen. Die Anwendung zeigt die "Protokollseite" an.
• Klicken Sie in der Befehlsleiste auf die Schaltfläche "Produktionsprotokoll". Die Schaltfläche "Produktionsprotokoll" der Befehlsleiste ist auf allen Fertigungstransaktionsseiten verfügbar, die ein Containerinformationen-Webpart aufweisen. Die Anwendung zeigt das Produktionsprotokoll in einem Popup an.

Beim Anzeigen des Transaktionsverlaufs

Die Registerkarte "Suchen" ist die einzige sichtbare Registerkarte, wenn Sie die Seite "Produktionsprotokoll" öffnen. Sie müssen eine Ansicht und einen Namen für den ausgewählten Datensatztyp im Abschnitt "Typ" der Seite angeben, um die Suche durchführen zu können. Sie können auch Information in den Abschnitten "Zeitrahmen" und "Transaktion" eingeben, um die Suche zu verfeinern.

Die Anwendung füllt das Feld "Anzeigen" automatisch aus, wenn eine der folgenden Bedingungen zutrifft:

• Es wurde nur eine Ansicht konfiguriert.
• Für den angemeldeten Mitarbeiter wurde eine Standardhistorienansicht angegeben. Standardhistorienansichten werden im Modellierungsobjekt "Mitarbeiter" angegeben. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.
Hinweis: Sie können die Standardansicht ändern, wenn mehr als eine Ansicht konfiguriert wurde.

Die Anwendung füllt die Felder "Aufzeichnungstyp" und "Name" automatisch aus, wenn die Seite "Produktionsprotokoll" aufgerufen wird, nachdem Sie auf einer Fertigungsseite oder auf der Seite "Containersuche" in der Befehlsleiste auf die Schaltfläche "Produktionsprotokoll" geklickt haben.

Abhängig vom ausgewählten Datensatztyp können nach der Ausführung einer Suche weitere Registerkarten angezeigt werden. Die Registerkarten "Status" und "Attribute" werden nur für Container und Behälter angezeigt. Die Registerkarte "Status" wird immer angezeigt, wenn eine UI-Einstellung definiert wurde. Die Registerkarte "Attribute" wird jedoch nur angezeigt, wenn für den angegebenen Container oder Behälter benutzerdefinierte Attributdaten verfügbar sind. Die Registerkarte "Ereignisprotokoll" wird nur für Ereignisse angezeigt, wenn Ereignisprotokolldaten verfügbar sind.

Die Anwendung zeigt auf der Registerkarte "Suchen" nachrangige Filter an, nachdem eine erste Suche nach einem Container oder Behälter durchgeführt wurde. Auswahlen in diesen Feldern hängen von den Transaktionsinformationen ab, die von der ersten Suche ausgegeben wurden. Sie können diese nachrangigen Filter verwenden, um die Suche weiter zu verfeinern.

Hinweis: Die nachrangigen Filter sind automatisch sichtbar, wenn die Anwendung die Seite "Produktionsprotokoll" aufruft, nachdem Sie auf einer Fertigungsseite in der Befehlsleiste auf die Schaltfläche "Produktionsprotokoll" geklickt haben.

Informationen zu dem Produktionsprotokoll und den Transaktionen finden Sie im Opcenter Execution Medical Device and Diagnostics Audit Trail-Benutzerhandbuch und im Opcenter Execution Core Audit Trail-Benutzerhandbuch.

Hinweis: Das Panel mit den Registerkarten kann zusammengeklappt werden. Durch das Zusammenklappen des Registerkartenbereichs werden die Tabellen auf der Seite horizontal erweitert.

Beim Auswählen einer anzuzeigenden Transaktion

Die Anwendung zeigt Breadcrumb-Navigation und eine Tabelle mit mindestens einer Registerkarte unter der Tabelle "Historienhauptlinie" an, wenn Sie in der Tabelle "Historienhauptlinie" eine Transaktion auswählen. Die jeweils angezeigten Registerkarten hängen vom Datensatztyp, vom Transaktionstyp und von der für die Transaktion konfigurierten Funktionalität ab. Wären beispielsweise einem Container für die "Start"-Transaktion Attribute zugewiesen, würde die Anwendung eine Registerkarte "Attribute" anzeigen.

Hinweis: Systemadministratoren können die Informationen, die auf den Registerkarten angezeigt werden, mit dem Protokoll-Konfigurationswerkzeug konfigurieren.

Seite "Produktionsprotokoll"

Auf der Seite "Produktionsprotokoll" können Sie die Historie für einen Container, einen Behälter oder ein Ereignis anzeigen. Die Seite besteht aus folgenden Teilen:

• Registerkarte "Suchen" - wird beim Laden der Seite angezeigt
• Registerkarte "Status" - wird nach Ausgabe der ersten Suchergebnisse angezeigt, aber nur, falls Statusdaten vorhanden sind
• Registerkarte "Attribute" - wird nach Ausgabe der ersten Suchergebnisse nur für Container und Behälter angezeigt und nur, falls dem Behälter oder Container benutzerdefinierte Attribute zugewiesen wurden
• Registerkarte "Ereignisprotokoll" - wird nach Ausgabe der ersten Suchergebnisse nur für Ereignisse angezeigt und nur, falls beim Verwalten von Produktions- und generischen Ereignissen Kommentare auf der Registerkarte "Protokoll" eingegeben wurden
• Breadcrumb-Navigation - wird angezeigt, nachdem Sie eine Transaktion in der Tabelle "Historienhauptlinie" ausgewählt haben, und zeigt eine Historie Ihrer ausgewählten Transaktionen mit Navigationsmöglichkeiten an.
• Tabelle "Historienhauptlinie" - wird nach Ausgabe der ersten Suchergebnisse angezeigt und zeigt alle Transaktionen für das angegebene Objekt an, die den Suchkriterien entsprechen
• Registerkarten mit Transaktionsinformationen - werden angezeigt, wenn Sie eine Transaktion in der Tabelle "Historienhauptlinie" auswählen

Wenn aktiviert, gibt die Anwendung nur die Transaktionen aus, die innerhalb des angegebenen Datumsbereichs für den angegebenen Container, Behälter oder das Ereignis ausgeführt wurden. Anfangsdatum des Datumsbereichs innerhalb dessen die Transaktionen für das angegebene Objekt ausgeführt wurden. Die Ergebnisse beinhalten nur Transaktionen, die nach diesem Datum ausgeführt wurden. Die Anwendung setzt das Anfangsdatum standardmäßig auf das Datum des Vortages und eine Zeit von 00:00 Uhr, wenn Sie die Option "Datumsbereich auswählen" wählen. Sie können ein anderes Datum und eine andere Uhrzeit im MM/TT/JJJJ hh:mm AM/PM-Format eingeben oder das Datum und die Uhrzeit auswählen, indem Sie auf das Kalender-Symbol klicken, um die Datumsauswahl-Funktion zu verwenden.

Optional

Enddatum

Enddatum des Datumsbereichs, innerhalb dessen die Transaktionen für das angegebene Objekt ausgeführt wurden. Die Ergebnisse beinhalten nur Transaktionen, die vor diesem Datum ausgeführt wurden. Die Anwendung setzt das Enddatum standardmäßig auf das heutige Datum und die aktuelle Zeit, wenn Sie die Option "Datumsbereich auswählen" auswählen. Sie können ein anderes Datum und eine andere Uhrzeit im MM/TT/JJJJ hh:mm AM/PM-Format eingeben oder das Datum und die Uhrzeit auswählen, indem Sie auf das Kalender-Symbol klicken, um die Datumsauswahl-Funktion zu verwenden.

Optional

Vorgehensweise zum Anzeigen des Transaktionsverlaufs für einen Container, Behälter oder ein Ereignis

Gehen Sie folgendermaßen vor, um den Transaktionsverlauf für einen Container, Behälter oder ein Ereignis anzuzeigen:

1. Öffnen Sie die Seite Produktionsprotokoll über das Menü Suchen. Die Seite "Produktionsprotokoll" wird angezeigt.
2. Wählen Sie im Feld Aufzeichnungstyp den Objekttyp, nach dem Sie suchen möchten.
3. Wählen Sie falls nötig eine andere Ansicht im Feld Anzeigen.
4. Geben Sie den Namen des Objekts ein.
5. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
6. Klicken Sie auf Suchen. Die Anwendung zeigt die Registerkarte "Status" (falls zutreffend), die Registerkarte "Attribut" (falls zutreffend) und die Tabelle "Historienhauptlinie" für das angegebene Objekt.
7. Wählen Sie eine Transaktion in der Tabelle "Historienhauptlinie". Die Anwendung füllt den Breadcrumb-Pfad und zeigt die entsprechende Transaktionstabelle unter der Tabelle "Historienhauptlinie" an.
8. Wählen Sie ggf. eine Transaktion in der Breadcrumb-Navigationsleiste aus, um zu einer zuvor ausgewählten Transaktion zurückzukehren.
9. Führen Sie die Schritte unter "Vorgehensweise zum Anzeigen eines Dokuments für einen Container oder ein Ereignis" aus, wenn Sie die Details eines Dokuments anzeigen möchten, das einem Container oder einem Ereignis zugeordnet ist.

Vorgehensweise zum Anzeigen eines Dokuments für einen Container oder ein Ereignis

Gehen Sie folgendermaßen vor, um ein Dokument für einen Container oder ein Ereignis anzuzeigen:

1. Führen Sie die Schritte 1-6 unter "Vorgehensweise zum Anzeigen des Transaktionsverlaufs für einen Container, Behälter oder ein Ereignis" aus.
2. Wählen Sie einen der folgenden Transaktionstypen in der Tabelle "Historienhauptlinie" aus.

Dokument anhängen

3. Klicken Sie auf Anhang anzeigen. Daraufhin wird eine Meldung angezeigt, in der Sie gefragt werden, ob Sie das Dokument öffnen oder speichern oder die Anforderung abbrechen möchten.
