Zwei-Level-Start

Die Seite "Starten – zwei Level" ermöglicht es Ihnen, zwei verschiedene Level von Containern zu starten. Die Mengen von Single-Level-Containern werden angegeben, während sich die Mengen von Multi-Level-Containern aus den Mengen der Kind-Container ableiten (oder zusammengefasst werden).

Viele Firmen verfolgen kleine (Kind-)Container innerhalb eines größeren (Eltern-)Containers. Die Gruppe von Eltern- und Kind-Containern wird gemeinsam bearbeitet und nachverfolgt. Daher erlaubt Ihnen die Seite "Starten – zwei Level", Eltern- und Kind-Container gleichzeitig zu starten.

Multi-Level-Container werden in einer Multi-Level-Struktur von zwei oder mehr hierarchischen Levels gestartet; die Container in der Struktur haben Eltern-Kind-Beziehungen.

Falls Container auf die gleiche Weise bearbeitet werden, bietet das Starten als Multi-Level-Struktur den Vorteil, nachfolgende Bewegungs-Transaktionen nur auf dem Top-Level durchzuführen, während die Transaktion für alle Kind-Container durchgeführt wird.

Multi-Level-Nachverfolgung in Manufacturing

In Manufacturing erfolgt die Bearbeitung von Eltern- und Kind-Containern über die Multi-Level-Nachverfolgung. Multi-Level-Nachverfolgung:

• Erlaubt die Nachverfolgung mehrerer Level von Containern.
• Bietet die Möglichkeit, eine Gruppe von Eltern- und Kind-Containern zu starten und bestehende Container zu gruppieren bzw. deren Gruppierung aufzuheben.
• Unterstützt die Durchführung von Manufacturing-Transaktionen an Containergruppen.

Beim Verwenden von "Zwei-Level-Start"

Containernamen müssen unabhängig vom Containerlevel eindeutig sein.

Sie können eine Hierarchie zugelassener Beziehungen zwischen Levels durchsetzen. Sie können z. B. einem Batch erlauben, Kassetten als Kinder zu haben, aber einer Kassette verbieten, Batches als Kinder zu haben.

Beim Starten einer Containergruppe erben die Kinder die auf Eltern-Level definierten Attribute. Die Attribute der Kinder können später mit der Containerwartung-Transaktion geändert werden.

Fabrik ist kein Pflichtfeld; eine Fabrik muss jedoch entweder dem Mitarbeiter, der den Container startet, zugewiesen sein oder auf der Seite "Starten – zwei Level" angegeben werden, wenn Sie den Container starten. Andernfalls bearbeiten andere Transaktionen wie z. B. "Abmelden" den Container nicht.

Hinweis: Die Anwendung zeigt automatisch einen Wert im Feld "Fabrik" an, wenn eine Fabrik für den angemeldeten Mitarbeiter angegeben wurde. Sie können eine andere Fabrik auswählen, falls erforderlich.

Alle Anzeigefelder auf der Seite "Starten – zwei Level" sind leer, bis ein Container ausgewählt wird. Anzeigefelder im Popup "Kind-Containerinfo" werden automatisch gefüllt. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Beim Anzeigen des Popups "Kind-Containerinfo" füllt die Anwendung die Felder basierend auf den aktuellen Kind-Container-Werten, die zum Zeitpunkt eingegeben wurden, als der Kind-Container zur Tabelle "Kind-Container" hinzugefügt wurde. "Containername" und "Containerlevel" können nicht geändert werden, falls Sie die automatische Nummerierung für Kind-Namen verwenden.

Die Schaltfläche "Alles löschen" löscht alle Felder auf der Seite "Starten – zwei Level". Im Popup "Kind-Containerinfo" hingegen setzt die Schaltfläche "Alles löschen" die Felder im Popup auf die Werte zurück, die beim Öffnen ursprünglich angezeigt wurden.

Klicken Sie im Popup "Kind-Containerinfo" auf "Aktualisieren", um Änderungen zu speichern, das Popup zu schließen und zur Seite "Starten – zwei Level" zurückzukehren.

Klicken Sie auf die Schaltfläche "Schließen", um das Popup "Kind-Containerinfo" zu schließen, ohne jegliche Änderungen zu speichern.

Beim Benennen von Eltern- und Kind-Containern

Sie müssen einen eindeutigen Containernamen angeben, wenn Sie neue Eltern-Container erstellen. Sie können einen Containernamen entweder manuell eingeben oder ihn automatisch von der Anwendung erstellen lassen. Es werden geschlossene Container von der Anwendung einbezogen, wenn ein eindeutiger Name geprüft wird.

Die Anwendung verwendet für die Benennung des Kind-Containers den Namen des Eltern-Containers, unabhängig von der Methode, die zur Benennung des Eltern-Containers verwendet wurde. Bei der Standardmethode wird "- 01", "- 02" usw. angehängt. an den Elternnamen für jeden Kind-Container angehängt. Es sind aber auch zusätzliche Benennungsoptionen verfügbar. Informationen finden Sie unter "Beim Erstellen von Kind-Containernamen".

Hinweis: Die Anwendung zeigt den Eltern-Container in der Erfolgsmeldung an, wenn die Transaktion erfolgreich ausgeführt wird. Die Anwendung führt keine Kind-Containernamen in der Erfolgsmeldung auf.

Beim Erstellen eines Eltern-Containernamens

Sie können die Methode zum Erstellen des Elternnamens nach dem Erstellen einer oder mehrerer Kind-Container nicht mehr ändern. Die Anwendung löscht die Tabelle "Kind-Container" und zeigt eine Warnmeldung an, wenn Sie die Methode nach dem Erstellen von Kind-Containern ändern.

Aktivieren Sie das Kontrollkästchen "Namen automatisch erstellen", um den Namen des übergeordneten Containers unter Verwendung der definierten Nummerierungsregeln von der Anwendung erstellen zu lassen. Wenn Sie dieses Kontrollkästchen aktivieren, löscht und deaktiviert die Anwendung das Feld "Container".

Die Anwendung verwendet die dem Eltern-Container zugeordneten Nummerierungsregeln, um den Containernamen zu erstellen. Nummerierungsregeln können für mehrere, dem Container zugeordnete Modellierungsobjektinstanzen festgelegt werden. Nummerierungsregeln folgen dieser Standardrangfolge:

1. Containerlevel
2. Produktionsauftrag
3. Produkt
4. Produktfamilie
5. Fabrik

Informationen zum Definieren von Nummerierungsregeln finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Erstellen von Kind-Containernamen

Wenn Sie Kind-Containernamen automatisch erstellen, müssen Sie dem Containerlevel, das Sie verwenden möchten, eine Nummerierungsregel zuordnen. Die Anwendung zeigt in folgenden Situationen eine Warnmeldung an:

• Wenn Sie ein Containerlevel auswählen, dem keine Nummerierungsregel zugeordnet wurde.
• Wenn Sie das Kontrollkästchen "Kindnamen automatisch erstellen" und auf
• "Hinzufügen" klicken.

Die Anwendung verwendet zur Erstellung von Kind-Containernamen mehrere Kriterien. Es gibt vier mögliche Szenarien:

• Wenn die Kontrollkästchen "Namen automatisch erstellen" und "Kindnamen automatisch erstellen" beide deaktiviert sind (die Standardeinstellung), weist die Anwendung jedem Kind-Container den im Feld "Container" angegebenen Namen zu und fügt "-01", "-02" usw. am Ende an.

Beispiel:

MeinContainer – 01

MeinContainer – 02

MeinContainer – 03

• Wenn das Kontrollkästchen "Namen automatisch erstellen" aktiviert und das Kontrollkästchen "Kindnamen automatisch erstellen" deaktiviert ist, weist die Anwendung jedem Kind-Containernamen den durch die zugewiesene Nummerierungsregel erstellen Eltern-Containernamen zu und fügt "-01", "-02" usw. am Ende an.

Beispiel:

c_0001 – 01

c_0001 – 02

c_0001 – 03

• Wenn das Kontrollkästchen "Namen automatisch erstellen" deaktiviert und das Kontrollkästchen "Kindnamen automatisch erstellen" aktiviert ist, weist die Anwendung jedem Kind-Container den im Feld "Container" angegebenen Namen zu und fügt die dem Kind-Containerlevel zugeordnete Nummerierungsregel am Ende an.

Beispiel:

MeineContainerCC0001

MeineContainerCC0002

MeineContainerCC0003

• Wenn die Kontrollkästchen "Namen automatisch erstellen" und "Kindnamen automatisch erstellen" beide aktiviert sind, weist die Anwendung jedem Kind-Container eine Verkettung der erstellten Eltern- und Kind-Nummerierungsregeln zu.

Beispiel:

c_0002CC0003

c_0002CC0004

c_0002CC0005

Vorgehensweise zum Starten mehrerer Container

Gehen Sie folgendermaßen vor, um mehrere Container zu starten:

1. Öffnen Sie die Seite Zwei-Level-Start im Menü Container. Die Seite "Starten – zwei Level" wird angezeigt.
2. Geben Sie einen Namen für den Eltern-Container in das Feld Containername ein.

Oder

Aktivieren Sie das Kontrollkästchen Namen automatisch erstellen.

3. Wählen Sie ein Level aus. Das ausgewählte Level wird im Feld "Level" angezeigt.
4. Wählen Sie einen Eigentümer aus. Der ausgewählte Eigentümer wird im Feld "Eigentümer" angezeigt.
5. Wählen Sie einen Startgrund aus. Der ausgewählte Grund wird im Feld "Startgrund" angezeigt.
6. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
7. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten von Kind-Containern", um neue Kind-Container zu starten.
8. Klicken Sie auf Start oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die bestätigt, dass die Container gestartet wurden.
Hinweis: Wenn die Option "Namen automatisch erstellen" ausgewählt wird, ohne vorher eine automatische Nummerierungsregel festzulegen, wird eine Warnmeldung angezeigt.

Vorgehensweise zum Starten von Kind-Containern

Gehen Sie folgendermaßen vor, um Kind-Container zu starten:

1. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten mehrerer Container".
2. Wählen Sie ein Level für die Kind-Container aus.
3. Aktivieren Sie das Kontrollkästchen Kindnamen automatisch erstellen, wenn Sie die Kind-Containernamen automatisch erstellen möchten.
4. Geben Sie eine Menge für die Kind-Container ein.
5. Geben Sie eine Maßeinheit für die Menge ein.
6. Geben Sie die Anzahl der zu erstellenden Kind-Container im Feld Kind-Container ein. Die Anwendung aktiviert die Schaltfläche "Hinzufügen".
7. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
8. Klicken Sie auf Hinzufügen. Die Kind-Container werden zur "Kind-Container"-Tabelle hinzugefügt.
9. Wiederholen Sie die Schritte 2 bis 8, um zusätzliche Kind-Container oder Schritte zu starten, oder klicken Sie auf Neue Zeile hinzufügen in der Tabelle, um das Popup "Kind-Containerinfo" anzuzeigen.
Hinweis: Wenn Sie die Option "Kind-Containernamen automatisch erstellen" für den ersten Kind-Container auswählen, müssen Sie sie auch für alle weiteren Kind-Container auswählen.
Hinweis: In der Felddefinitionstabelle finden Sie Informationen zum Popup "Kind-Containerinfo".

Vorgehensweise zum Bearbeiten von Kind-Containern

Gehen Sie folgendermaßen vor, um Kind-Container zu bearbeiten:

1. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten von Kind-Containern".
2. Wählen Sie den Kind-Container aus, den Sie bearbeiten möchten.
3. Klicken Sie in der Tabelle auf Ausgewählte Zeile bearbeiten. Das Popup "Kind-Containerinfo" wird angezeigt.
4. Ändern Sie Informationen oder geben Sie nach Bedarf Informationen ein.
Hinweis: "Containername" und "Containerlevel" können nicht geändert werden, falls Sie die automatische Nummerierung für die Kind-Namen verwenden.
5. Klicken Sie auf Aktualisieren. Die Anwendung speichert Ihre Änderungen und schließt das Popup.
