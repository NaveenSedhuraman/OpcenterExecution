Container starten

Die Transaktion "Start" erstellt einen neuen Container und initiiert den ersten Schritt im Arbeitsplan, der dem Produkt des Containers zugeordnet ist. Sie müssen einen Container starten, bevor Sie eine beliebige Transaktion ausführen.

Beim Verwenden von "Start"

Sie müssen einen eindeutigen Containernamen angeben, wenn Sie einen neuen Container erstellen. Sie können einen Containernamen entweder manuell eingeben oder ihn automatisch von der Anwendung erstellen lassen. Es werden geschlossene Container von der Anwendung einbezogen, wenn ein eindeutiger Name geprüft wird.

Aktivieren Sie das Kontrollkästchen "Namen automatisch erstellen", um den Containernamen unter Verwendung der definierten Nummerierungsregeln von der Anwendung erstellen zu lassen. Wenn Sie dieses Kontrollkästchen aktivieren, löscht und deaktiviert die Anwendung das Feld "Zielcontainer".

Wenn die Option "Namen automatisch erstellen" ausgewählt wird, ohne vorher eine automatische Nummerierungsregel festzulegen, wird eine Fehlermeldung angezeigt.

Die Anwendung verwendet zum Erstellen des Zielcontainernamens die Nummerierungsregeln, die dem Eltern-Container zugeordnet sind, von dem aus der Kind-Container aufgeteilt wird. Wenn Sie auf die Schaltfläche "Abschicken" klicken, erstellt die Anwendung den Namen des neuen Eltern-Containers.

Nummerierungsregeln können für mehrere, dem Container zugeordnete Modellierungsobjektinstanzen festgelegt werden. Nummerierungsregeln folgen dieser Standardrangfolge:

1. Containerlevel
2. Produktionsauftrag
3. Produkt
4. Produktfamilie
5. Fabrik
Hinweis: Die Anwendung verwendet die Nummerierungsregel, die den relevanten Modellierungsobjektinstanzen zugeordnet ist, die Sie für diesen Container auswählen und nicht die eines Eltern- oder Quellcontainers.

Das Ablaufdatum bestimmt, wann der Container abläuft. Dieses Datum wird an mehreren Transaktionen, wie "Verbinden", "Vereinigen" und "Komponentenverbrauch" geprüft.

Fabrik ist kein Pflichtfeld. Eine Fabrik muss jedoch entweder dem Mitarbeiter, der den Container startet, zugewiesen sein oder die Fabrik muss auf der Startseite angegeben werden, wenn Sie den Container starten. Andernfalls bearbeiten andere Transaktionen wie z. B. "Abmelden" den Container nicht.

Hinweis: Die Anwendung zeigt automatisch einen Wert im Feld "Fabrik" an, wenn eine Fabrik für den angemeldeten Mitarbeiter angegeben wurde. Sie können eine andere Fabrik auswählen, falls erforderlich.

Sie können Dateien von der Startseite aus an einen Container anhängen und sie dann auf der Seite "Anhänge verwalten" verwalten. Unter "Anhängen von Dokumenten" finden Sie Informationen zu den Optionen zum Anhängen von Dokumenten. Unter "Anhänge verwalten" finden Sie Informationen zum Verwalten von Dokumenten. Anhänge sind optional. Sie müssen beim Anhängen von Dateien jedoch bestimmte Felder in der Anwendung ausfüllen, bevor Sie den Container starten.

Sie können mit der Startseite nicht mehrere Container auf einmal starten. Die Mengen von Single-Level-Containern werden angegeben, während sich die Mengen von Multi-Level-Containern aus den Mengen der Kind-Container ableiten (oder zusammengefasst werden). Sie müssen die Seite "Starten – zwei Level" verwenden, um mehrere Container zu starten. Informationen über die Seite "Starten - zwei Level" finden Sie unter "Zwei-Level-Start".

Beim Starten eines Containers mit aktiviertem automatischen Materialverbrauch

Hinweis: Diese Funktion ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Sie können auf der Seite "Fabrik" im Feld "Material-verbrauchen-Transaktionen" die "Start"-Transaktion auswählen, um anzugeben, dass das für eine Ressource geladene Material automatisch verbraucht wird, wenn ein Container durch die Ressource bewegt wird. Sie können auch im Feld "Materialwarteschlange-prüfen-Transaktionen" die "Start"-Transaktion auswählen, um die Anwendung zu aktivieren, die überprüft, dass das korrekte Material in einer Ressource für einen Container geladen ist, der durch die Ressource bewegt wird.

Die Anwendung gibt eine Warnmeldung aus, wenn Sie die "Start"-Transaktion auswählen, aber nicht das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktivieren. Die Transaktion schlägt fehl, wenn die Menge nicht ausreichend ist, wenn Sie die "Start"-Transaktion auswählen und das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktivieren. Sie können einen "Verbrauchsabweichungsgrund" auswählen, um zu verhindern, dass ein Fehler auftritt, wenn beim automatischen Komponentenverbrauch Teilmengen verbraucht werden.

Informationen zum Definieren von Fabriken finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Zuweisen einer UDI zur Seite "Start"
Hinweis: Diese Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Sie können die Gerätekennung, die Produktionskennung und eine eindeutige Gerätekennung einem Container direkt auf der Seite "Start" zuweisen. Dies ist nützlich, wenn eine UDI von einer externen Quelle eingescannt wird. Die Produktionskennung und die eindeutige Gerätekennung können direkt eingegeben werden, während die Gerätekennung ausgewählt wird.

Die Anwendung füllt das Feld "Gerätekennung" automatisch aus, wenn für das ausgewählte Produkt nur eine Gerätekennung angegeben ist. Wenn mehr als eine Gerätekennung für das ausgewählte Produkt angegeben ist, wird eine Liste angezeigt.

Seite "Start"

Verwenden Sie die Startseite, um einen einzelnen Container zu starten. Der untere Bereich der Seite ist reduziert, sodass nur die Felder, die erforderlich sind, um einen Container zu starten, beim ersten Zugriff auf die Startseite angezeigt werden. Klicken Sie auf das Steuerelement "Dokument anhängen", um die Anlagenoptionen anzuzeigen. Klicken Sie auf die "Zusätzliche Felder anzeigen"-Steuerung, um die verborgenen Felder anzuzeigen.

Vorgehensweise zum Starten eines Containers

Gehen Sie folgendermaßen vor, um einen Container zu starten:

1. Öffnen Sie die Seite Start im Menü Container. Die "Startseite" wird angezeigt.
2. Geben Sie einen Namen für den zu startenden Container in das Feld Containername ein.

Oder

Klicken Sie auf das Kontrollkästchen Namen automatisch erstellen.

3. Wählen Sie ein Level aus. Das ausgewählte Level wird im Feld "Level" angezeigt.
4. Wählen Sie einen Eigentümer aus. Der ausgewählte Eigentümer wird im Feld "Eigentümer" angezeigt.
5. Wählen Sie einen Startgrund aus. Der ausgewählte Grund wird im Feld "Startgrund" angezeigt.
6. Wählen Sie einen Arbeitsplan (Name und Revision) aus. Der ausgewählte Arbeitsplanname und die Revision werden in den "Arbeitsplan"-Feldern angezeigt.
7. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
8. Klicken Sie auf das Steuerelement "Dokument anhängen", um die Dokumentanhangs-Felder anzuzeigen, wenn der Bereich nicht erweitert ist.
9. Ziehen Sie die entsprechenden Schritte zum Anhängen eines Dokuments hinzu, wenn Sie ein Dokument anhängen möchten.
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das nicht wiederverwendet wird"
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das wiederverwendet wird"
• "Vorgehensweise zum Anhängen eines vorhandenen Dokuments"
10. Klicken Sie auf das Steuerelement "Zusätzliche Felder anzeigen", um die zusätzlichen Felder anzuzeigen, falls der Bereich nicht erweitert ist.
11. Klicken Sie auf Start oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die bestätigt, dass der Container gestartet wurde.

Vorgehensweise zum Anhängen eines neuen Dokuments, das nicht wiederverwendet wird

Gehen Sie folgendermaßen vor, um ein neues Dokument anzuhängen, das nicht wiederverwendet wird:

1. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten eines Containers".
2. Erweitern Sie den Bereich "Dokument anhängen", falls erforderlich.
3. Wählen Sie Neues Dokument anhängen, das NICHT wiederverwendet wird (keine Revision benötigt) aus. Die Felder "Dokumentpfad", "Dokumentname" und "Dokumentbeschreibung" werden angezeigt.
4. Klicken Sie auf die Schaltfläche Datei hochladen, um eine Datei auszuwählen.
5. Geben Sie einen Namen für das Dokument in das Feld Dokumentname ein.
6. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
7. Klicken Sie auf Start oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die bestätigt, dass der Container gestartet wurde.

Vorgehensweise zum Anhängen eines neuen Dokuments, das wiederverwendet wird

Gehen Sie folgendermaßen vor, um ein neues Dokument anzuhängen, das wiederverwendet wird:

1. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten eines Containers".
2. Erweitern Sie den Bereich "Dokument anhängen", falls erforderlich.
3. Wählen Sie Neues Dokument anhängen, das wiederverwendet wird (Revision muss angegeben werden) aus. Die Felder "Dokumentpfad", "Dokumentname", "Revision", "Standardrevision" und "Dokumentbeschreibung" werden angezeigt.
4. Klicken Sie auf die Schaltfläche Datei hochladen, um eine Datei auszuwählen.
5. Geben Sie einen Namen für das Dokument im Feld Dokumentname ein.
6. Geben Sie die Revision dieses Dokuments in das Feld Revision ein.
7. Aktivieren Sie optional das Kontrollkästchen neben der Revision, um anzugeben, dass dieses Dokument die Standardrevision ist.
8. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
9. Klicken Sie auf Start oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die bestätigt, dass der Container gestartet wurde.

Vorgehensweise zum Anhängen eines vorhandenen Dokuments

Gehen Sie folgendermaßen vor, um ein existierendes Dokument anzuhängen:

1. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten eines Containers".
2. Erweitern Sie den Bereich "Dokument anhängen", falls erforderlich.
3. Wählen Sie Anhängen eines existierenden Dokumentes (unten auswählen). Die Felder "Dokumentinstanzen", "Revision", "Standardrevision" sowie die Schaltfläche "Dokument anzeigen" werden angezeigt.
4. Wählen Sie ein Dokument im Feld Dokumentinstanzen aus.
5. Geben Sie die spezifische Revision in das Feld Revision ein.

Oder

Aktivieren Sie das Kontrollkästchen Standardrevision.

6. Klicken Sie auf Dokument anzeigen, wenn Sie das Dokument anzeigen möchten, bevor Sie es anhängen. Die Anwendung fordert Sie dazu auf, die Datei zu öffnen oder zu speichern, oder sie lädt das Dokument automatisch herunter. (Ihr Internet Browser bestimmt, ob die Anwendung ein Popup öffnet, wie es aussehen soll und was in den Öffnen- und Speichern-Optionen genau steht.)
7. Klicken Sie auf Start oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die bestätigt, dass der Container gestartet wurde.
