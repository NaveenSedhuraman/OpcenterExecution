Containerdefekt

Die Transaktion "Defekt" ermöglicht es Ihnen, Materialfehler aufzuzeichnen, die nicht sofort zu einer Entfernung oder zu Ausschuss führen. Sie können die Anzahl der Container und/oder die Menge des überprüften Containers aufzeichnen. Sie können auch eine Liste der Defektcodes und Defektanzahl für die von Ihnen gesichteten Defekte erstellen.

Hinweis: Verwenden Sie die Transaktion "Komponentendefekt", wenn Sie einen Defekt aufzeichnen wollen, der mit den in einem Container verbrauchten Komponenten in Zusammenhang steht.

Vor der Verwendung von "Containerdefekt"

Sie müssen Gründe für den Containerdefekt in der Modellierung konfigurieren, bevor Sie Defekte von Containern aufzeichnen können. Die Defektgründe stehen unter diesen Bedingungen zur Auswahl:

• Ein Containerdefektgrund wurde definiert.
• Eine Containerdefektgrund-Gruppe mit dem entsprechenden Grund wurde definiert.
• Die Containerdefektgrund-Gruppe wurde der aktuellen Operation des Containers zugewiesen.

Informationen zum Definieren und Verknüpfen der Objekte, die zum Aufzeichnen von Containerdefekten erforderlich sind, finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Verwenden von "Containerdefekt"

Alle Anzeigefelder auf der "Defektseite" sind leer bis ein Container ausgewählt wurde. Anzeigefelder werden automatisch auf dem Popup "Containerdefekt" gefüllt. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Sie können Defekte an Kind-Containern aufzeichnen. Wenn Sie Defekte an Kind-Containern aufzeichnen, so schreibt die Anwendung dies nur in den Verlauf der Kind-Container. Die Historie des Eltern-Containers enthält nicht die Defekte von Kind-Containern.

Sie können Defekte für angehaltene Container aufzeichnen, jedoch nicht für geschlossene. Die Anwendung stellt keine geschlossenen Container zur Auswahl zur Verfügung.

Die Anwendung erfordert, dass Sie mindestens einen Defektgrund hinzufügen und eine Anzahl für diesen Grund eingeben, bevor Sie einen Defekt protokollieren können. Die Anwendung zeigt eine Fehlermeldung an, wenn Sie auf die Schaltfläche "Defekte protokollieren" klicken, ohne einen Defektgrund und eine Defektanzahl hinzugefügt zu haben.

Wenn Sie auf die Schaltfläche "Ausgewählte Zeile löschen" klicken, entfernen Sie die ausgewählten Defektgründe aus der Tabelle. Sie können das Kontrollkästchen in der Kopfzeile aktivieren, um alle angezeigten Defekte automatisch auszuwählen, oder Sie aktivieren die Kontrollkästchen der einzelnen Defekte.

Seite "Defekt"

Sie können auf die Defektseite vom Containermenü aus zugreifen. Die "Defektseite" ermöglicht es Ihnen, eine Liste von Defekten am ausgewählten Container aufzuzeichnen.

Hinweis: Die Seite "Defekt" ist eventuell auch als Popup verfügbar, wenn die Transaktion "Containerdefekt" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (z. B. "Containersuche").

Vorgehensweise zum Aufzeichnen eines Containerdefekts

Gehen Sie folgendermaßen vor, um Containerdefekte aufzuzeichnen:

1. Öffnen Sie im Menü Container die Seite Defekt. Die Seite "Defekt" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Containerdefekt" zugreifen.
3. Klicken Sie auf Neue Zeile hinzufügen in der Tabelle. Eine Zeile wird in der Tabelle aktiviert.
4. Wählen Sie einen Containerdefektgrund im Feld Defektgrund aus.
5. Geben Sie die Anzahl der Defekte, die diesem Grund zugeordnet sind, ins Feld Defektanzahl ein.
6. Wiederholen Sie die Schritte 3-5, um weitere Containerdefektgründe zu erstellen:
7. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
8. Klicken Sie auf Defekte protokollieren oder in der Befehlsleiste auf Abschicken, um die Liste der Defekte zu speichern, die Sie für den Container eingegeben haben. Die Anwendung zeigt eine Statusmeldung an, die bestätigt, dass Defekte gespeichert wurden.

Vorgehensweise zum Löschen von Defektgründen

Gehen Sie folgendermaßen vor:

1. Öffnen Sie auf Defekt im Container-Menü. Die Seite "Defekt" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
3. Klicken Sie auf das Kontrollkästchen für den Defektgrund, den Sie aus der Tabelle entfernen möchten.
4. Wiederholen Sie Schritt 3 für jeden Defektgrund, den Sie entfernen möchten. Wenn Sie alle Defektgründe entfernen möchten, die derzeit in der Tabelle angezeigt werden, aktivieren Sie das Kontrollkästchen in der Kopfzeile.
5. Klicken Sie auf Ausgewählte Zeile löschen in der Tabelle. Die ausgewählten Defektgründe werden aus der Tabelle gelöscht.
