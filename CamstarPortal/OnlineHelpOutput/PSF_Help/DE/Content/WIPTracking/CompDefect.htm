Komponentendefekt

Die Transaktion "Komponentendefekt" ermöglicht es Ihnen, Defekte an verbrauchten Komponenten in einem Container zu protokollieren. Dies sind Defekte, die nicht zu einer sofortigen Entnahme oder einem Verwurf der Komponenten führen. Diese Transaktion gilt sowohl für Container als auch für die verbrauchten Komponenten in diesen Containern.

Hinweis: Verwenden Sie die Transaktion "Containerdefekt", wenn Sie Defekte für einen Container als Ganzes protokollieren möchten.

Vor der Verwendung von "Komponentendefekt"

Sie müssen Komponentendefektgründe in der Modellierung angeben, bevor Sie Defekte an Komponenten protokollieren können. Defektgründe stehen unter folgenden Bedingungen zur Auswahl:

• Ein Komponentendefektgrund wurde definiert.
• Eine Komponentendefektgrundgruppe, die den Grund enthält, wurde definiert.
• Die Komponentendefektgrundgruppe wurde der aktuellen Operation des Containers zugewiesen.

Informationen zum Definieren und Verknüpfen der Objekte, die zum Aufzeichnen von Containerdefekten erforderlich sind, finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Verwenden von "Komponentendefekt"

Alle Anzeigefelder sind auf der Seite "Komponentendefekt" leer, bis ein Container ausgewählt wird. Anzeigefelder werden im "Komponentendefekt"-Popup automatisch gefüllt. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Die Anwendung zeigt die verbrauchten Komponenten im angegebenen Container in der Tabelle "Materialverbräuche" an. Die Komponentenzeilen sind schreibgeschützt. Die Spalte "Aufgezeichnete Defekte" gibt an, ob bereits zuvor Defekte an der Komponente protokolliert wurden.

Die Anwendung zeigt eine Untertabelle an, in der Sie Defekte für die Komponente hinzufügen, bearbeiten und löschen können, wenn Sie eine Komponente auswählen. Zunächst zeigt die Komponenten-Untertabelle nur eine Kopfzeile an. Sie können Zeilen hinzufügen, indem Sie auf die Schaltfläche "Neue Zeile hinzufügen" in der Untertabelle klicken.

Hinweis: Die Anwendung zeigt ein Maximum von 20 Zeilen in der Tabelle an. Übersteigt die Anzahl der Zeilen 20, fügt die Anwendung eine Umblätter-Funktionalität hinzu.

Sie können Defekte für angehaltene Container protokollieren, aber Sie können keinen Defekt für einen Container protokollieren, der geschlossen ist. Die Anwendung stellt keine geschlossenen Container zur Auswahl zur Verfügung.

Es können mehrere Defekte für einen Container nachverfolgt werden. Die Anwendung erfordert, dass Sie mindestens einen Defektgrund hinzufügen und eine Zahl für diesen Grund eingeben, bevor Sie einen Defekt protokollieren können. Die Anwendung zeigt eine Fehlermeldung an, wenn Sie auf die Schaltfläche "Defekte protokollieren" klicken, ohne einen Defektgrund und eine Defektanzahl hinzugefügt zu haben.

Durch Klicken auf die Schaltfläche "Ausgewählte Zeile löschen" in der Untertabelle entfernen Sie die ausgewählten Defektgründe aus der Untertabelle. Sie können das Kontrollkästchen in der Kopfzeile aktivieren, um alle angezeigten Defekte automatisch auszuwählen, oder Sie aktivieren die Kontrollkästchen für einzelne Defekte.

Die Anwendung löscht die Untertabellen für jede Komponente, wenn Sie auf die Schaltfläche "Defekte protokollieren" klicken und die Transaktion erfolgreich ist. Die Tabelle "Materialliste" wird nicht von der Komponentenliste gelöscht.

Seite "Komponentendefekt"

Sie können über das Container-Menü auf die Seite "Komponentendefekt" zugreifen. Die Seite "Komponentendefekt" bietet eine Ansicht der verbrauchten Komponenten im ausgewählten Container, der Verbrauchs-Details für diese Komponenten und erlaubt es Ihnen, mehrere Defekte an einer oder mehreren der verbrauchten Komponenten zu protokollieren.

Hinweis: Die Seite "Komponentendefekt" ist eventuell auch als Popup verfügbar, wenn die Transaktion "Komponentendefekt" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (z. B. "Containersuche").

Vorgehensweise zum Protokollieren eines Komponentendefekts

Gehen Sie folgendermaßen vor, um einen Komponentendefekt zu protokollieren:

1. Öffnen im Menü Container die Seite Komponentendefekt. Die Seite "Komponentendefekt" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Komponentendefekt" zugreifen.
3. Wählen Sie die Komponente in der Tabelle Materialverbräuche aus, für die Sie einen Defekt protokollieren möchten. Die Anwendung zeigt die Untertabelle für diese Komponente an.
4. Klicken Sie in der Untertabelle auf Neue Zeile hinzufügen. Eine Zeile wird in der Tabelle aktiviert.
5. Wählen Sie einen Komponentendefektgrund im Feld Defektgrund aus.
6. Geben Sie die Anzahl der Defekte, die diesem Grund zugeordnet sind, ins Feld Defektanzahl ein.
7. Wiederholen Sie Schritte 4 bis 6, um zusätzliche Defektgründe für die Komponente hinzuzufügen.
8. Wiederholen Sie Schritte 3 bis 6, um Defektgründe für andere Komponenten hinzuzufügen.
9. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
10. Klicken Sie auf Defekte protokollieren oder in der Befehlsleiste auf Abschicken. Die Anwendung zeigt eine Statusmeldung an, die bestätigt, dass Defekte gespeichert wurden.
