Multi-Container-Transaktionsübersicht

Siemens stellt sechs Portal-Fertigungstransaktionen bereit, die es Ihnen erlauben, einen oder mehrere Container gleichzeitig zu bearbeiten:

• Schließen (mehrere)
• Anhalten (mehrere)
• Abmelden - nicht standardmäßig (mehrere)
• Öffnen (mehrere)
• Produktionsereignis aufzeichnen
Informationen zum Aufzeichnen von Produktionsereignissen finden Sie unter "Produktionsereignis aufzeichnen" sowie im Opcenter Execution Medical Device and Diagnostics Event Management-Benutzerhandbuch und im Opcenter Execution Core Event Management-Benutzerhandbuch.
• Freigeben (mehrere)

Systemadministratoren können je nach den Geschäftsbedürfnissen Ihres Unternehmens weitere Multi-Container-Transaktionen konfigurieren.

Hinweis: Multi-Container-Transaktionen unterstützen keine Datensammlung oder Umlaufbestandsnachrichten.

Multi-Container-Transaktionsseiten und -Popups

Alle Multi-Container-Transaktionsseiten sind über das Container-Menü verfügbar. Darüber hinaus sind die Seiten eventuell auch als Popup verfügbar, wenn die Transaktionen als verfügbare Aktion auf einer anderen Fertigungsseite konfiguriert wurden (z. B. "Containersuche"). Im Gegensatz zu anderen Einzelcontainer-Transaktionsseiten und -Popups haben die Multi-Container-Transaktionsseiten und -Popups keinen Container-Kopf.

Wenn Sie über das Container-Menü auf eine Multi-Container-Transaktionsseite zugreifen, sind alle Felder leer, bis Sie einen Container scannen oder eingeben. Wenn Sie einen Container scannen oder eingeben, fügt die Anwendung diesen zur Tabelle auf der Seite hinzu und wählt den Container automatisch aus.

Der Tabellenname hängt von der Transaktion ab, die Sie ausführen möchten. Wenn Sie beispielsweise auf der Seite "Freigeben (mehrere)" sind, lautet der Tabellenname "Freizugebende Container".

Die Anwendung zeigt Container in der Tabelle an und wählt sie automatisch aus, wenn Sie von einer anderen Fertigungsseite auf ein Multi-Container-Transaktions-Popup zugreifen. Dabei handelt es sich um Container, die auf der Seite ausgewählt wurden, von der aus Sie das Transaktions-Popup geöffnet haben.

Sie können weitere Container zur Popup-Tabelle hinzufügen, indem Sie einen Container scannen oder eingeben. Die Anwendung wählt die manuell hinzugefügten Container automatisch aus. Nachdem Sie die Multi-Container-Transaktion ausgeführt haben, müssen Sie auf "Schließen" klicken, um das Popup zu schließen.

Hinweis: Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Die Anwendung bearbeitet nur ausgewählte Container. Sie können das Kontrollkästchen für einen Container deaktivieren, wenn Sie keine Transaktion daran durchführen möchten.

Multi-Container-Transaktionsmodi

Siemens bietet zwei Modi zum Durchführen einer Transaktion an mehreren Containern: individuell und als Gruppe. "Individuell" ist der Standardmodus. Der Transaktionsmodus bestimmt, wie die Anwendung arbeitet, wenn eine Transaktion, die an einem ausgewählten Container durchgeführt wurde, fehlschlägt.

Wenn eine Ausführung im individuellen Modus fehlschlägt, führt dies dazu, dass die Anwendung:

• Alle ausgewählten Container weiter bearbeitet
• Den Erfolgsindikator in der Statuszelle der erfolgreich bearbeiteten Container anzeigt
• Den Fehlerindikator in der Statuszelle der fehlgeschlagenen Container anzeigt
• Eine Statusmeldung für die fehlgeschlagenen Container anzeigt

Wenn eine Ausführung im Gruppenmodus fehlschlägt, führt dies dazu, dass die Anwendung:

• Die Bearbeitung sofort anhält
• Keine erfolgreichen Transaktionen bestätigt
• Den Fehlerindikator in der Statuszelle aller Container anzeigt
• Eine Statusmeldung für die fehlgeschlagenen Container anzeigt

Konfigurieren des Transaktionstyp-Modus

Der Transaktionsmodus für eine Multi-Container-Transaktion wird in Portal Studio für die Seite definiert. Informationen zum Konfigurieren des Transaktionsmodus finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch und im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Beim Ausführen einer Multi-Container-Transaktion

Das Ausführen einer Transaktion an mehreren Containern führt dazu, dass die Anwendung:

• Den entsprechenden Indikator für jeden Container anzeigt
• Alle verarbeiteten Container in der Container-Tabelle belässt
• Die Transaktion in der Historie jedes erfolgreich verarbeiteten Containers aufzeichnet

Wenn Sie erneut auf die Schaltfläche zur Ausführung der Transaktion klicken, um die Transaktion erneut durchzuführen, verarbeitet die Transaktion nur die fehlgeschlagenen Container. Informationen über das Verhalten der Anwendung im individuellen oder im Gruppenmodus, wenn Sie auf die Transaktionschaltfläche klicken, finden Sie unter "Multi-Container-Transaktionsmodi".
