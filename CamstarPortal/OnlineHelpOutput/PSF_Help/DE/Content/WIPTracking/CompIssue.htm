Komponentenverbrauch

Die Transaktion "Komponentenverbrauch" ermöglicht es Ihnen, Rohmaterialien und Unterbaugruppen (Komponenten) in einem Container in einer bestimmten Operation zu verbrauchen. Diese Transaktion verfolgt außerdem Materialverbräuche nach Fertigungsschritt (und zeigt sie Ihnen an).

Die Komponenten und Verbrauchsmengen werden in einer Materialliste definiert, in der Regel der Stückliste (BOM) für das Produkt eines Containers bzw. einer Container-spezifischen Materialliste. Komponenten werden von einem Container gemäß der Stückliste "BOM" des Produkts und der Verbrauchssteuerung (Tracking) verbraucht.

Es gibt zwei Versionen der Seite "Komponentenverbrauch". Verwenden Sie:

• Seite "Komponentenverbrauch - Einfach", um jeweils eine einzelne Komponente zu verbrauchen, bis alle erforderlichen Materialien befriedigt wurden.
• Seite "Komponentenverbrauch - Erweitert", um die Verbrauchskommandos in die Warteschlange zu setzen, bis alle benötigten Materialien befriedigt wurden. Führen Sie dann einen einzigen "Komponente verbrauchen"-Befehl aus.

Verbrauchssteuerung

Die Anwendung erfordert die Auswahl einer Verbrauchssteuerung für jede Komponente (Produkt oder Material), die zur Materialliste hinzugefügt wird, wenn eine Stückliste "BOM" in der Modellierung definiert wurde. Verbrauchssteuerungstypen bieten verschiedene Möglichkeiten der Zuführung benötigter Materialien hinsichtlich ihrer Quelle (Container, Produkt oder ein Ersatzprodukt) sowie Fertigungs-Arrangements (Lagerorte). Verbrauchssteuerungen legen fest, wie das Material an Container ausgegeben werden soll und wie es nachverfolgt wird.

Siemens unterstützt die folgenden Typen von Verbrauchssteuerungen:

Verbrauchstyp

Beschreibung

Verbrauchscontainer (Los)

Komponentenmaterial wird verbraucht und in Manufacturing auf Container- oder Los (Masse)-Ebene nachverfolgt. Ein Komponenten-Container kann einen Teil oder die gesamte erforderliche Menge liefern, max. bis zur aktuellen Menge des Komponenten-Containers.

Verbrauchscontainer (seriell)

Komponentenmaterial wird verbraucht und in Manufacturing als serialisierter Container nachverfolgt. Wenn ein serialisierter Container bei einer Materialanforderung mit einer seriellen Verbrauchs-Art verbraucht wird, wird die gesamte Menge des Containers verbraucht.

Nur Anzeige

Komponenteninformationen werden Ihnen angezeigt, aber Sie müssen keine Informationen über die tatsächliche Komponenten-Verwendung aufzeichnen. Die "Materialverbrauchsprüfung" wird nicht für Komponenten mit einer Verbrauchsart von "Nur Anzeige" ausgeführt.

Los und Lagerort

Komponentenmaterial wird von Losen und Standorten außerhalb von Manufacturing verbraucht. Das Material wird nicht durch Manufacturing nachverfolgt. Los, Menge und Lagerort werden aufgezeichnet. Informationen werden normalerweise mithilfe von Opcenter Connect MOM (Opcenter CN MOM) an ERP-Systeme übertragen.

Nur Menge

Komponenten-Material wird von keinem System nachverfolgt. Menge und andere optionale Felder werden zur Buchhaltung und für andere Zwecke gespeichert.

Nur Lagerort

Komponentenmaterial wird von einem Standort außerhalb von Manufacturing verbraucht und das Material wird nicht durch Manufacturing nachverfolgt. Menge und Lagerort werden aufgezeichnet. Informationen werden normalerweise mithilfe von XML Connect an ERP-Systeme übertragen.

Beim Verwenden von "Komponentenverbrauch"

Alle Anzeigefelder sind auf der Seite "Komponentenverbrauch" (Einfach und Erweitert) leer, bis ein Container ausgewählt wird. Anzeigefelder werden automatisch im Popup "Komponentenverbrauch" gefüllt, basierend auf dem Container, der auf einer anderen Seite, z. B. "Operative Ansicht", ausgewählt wurde. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Wenn ein Container ausgewählt ist, füllt die Anwendung die Tabelle "Materialanforderungen" mit einer Liste von Komponenten (Produkte oder Ersatzprodukte), die für den Container erforderlich sind.

Regeln

Die Anwendung ermöglicht es Ihnen, Ersatzprodukte zu verbrauchen, um eine Materialanforderung statt des angegebenen Produkts zu erfüllen, wenn ein Ersatzprodukt definiert wurde. Gültige Ersatzprodukte werden in der Modellierung definiert. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Die Transaktionen "Verbrauchscontainer (Los)" und "Verbrauchscontainer (seriell)", bei denen Ablaufdaten für den Quell (Von)-Container oder Ziel (An)-Container eingegeben wurden, müssen vor den Ablaufdaten stattfinden. Die Transaktion "Komponentenverbrauch" scheitert, wenn das Ablaufdatum für einen der Container überschritten wurde, bevor Sie die Transaktion ausführen.

Das Ablaufdatum für den Verbrauchen-in-Container wird auf das frühere der Ablaufdaten der Verbrauchen-in- und Verbrauchen-von-Container gesetzt. Wenn nur der Verbrauchen-von-Container über ein Ablaufdatum verfügt, wird der Verbrauchen-in-Container auf dasselbe Datum gesetzt.

Beim Verwenden der Tabelle "Materialanforderungen"

Sie können die Option "Erledigte Anforderungen ausblenden" auswählen, wenn die Tabelle "Materialanforderungen" auf der Seite oder dem Popup "Komponentenverbrauch" angezeigt wird. Erledigte Anforderungen beinhalten solche, die genau erledigt und solche, die überschritten wurden. Die Anwendung zeigt weiterhin teilweise erledigte Anforderungen und solche, die nicht bearbeitet wurden.

Ein Zähler zeigt die Anzahl der erledigten Anforderungen und die Gesamtzahl der Anforderungen. Er wird oberhalb der rechten Ecke der Tabelle angezeigt. Der Zähler ist hilfreich, wenn die Anzahl der Anforderungen mehr als eine Seite umfasst, und wenn Sie entschieden haben, erledigte Anforderungen auszublenden.

Wenn die Tabelle "Materialanforderungen" mehrere Einträge für das gleiche Material enthält, die durch einen Referenzbezeichner unterschieden sind, können Sie entscheiden, zuerst eine bestimmte Anforderung zu erfüllen. Wählen Sie die Zeile aus, scannen oder geben Sie den Container bzw. das Produkt ein und fahren Sie wie gewohnt fort. Wenn die gescannte bzw. eingegebene Komponente nicht mit der ausgewählten Zeile übereinstimmt, zeigt die Anwendung eine Fehlermeldung an.

Das Feld "Erford. Nettomenge" wird nach jedem Verbrauch aktualisiert. Wenn mehr als ein Container oder Produkt eingegeben wird, um eine einzelne Anforderung zu erfüllen, wird das Feld "notwendige Nettomenge" mit jedem folgenden Verbrauch verringert.

Beim Eingeben einer zu verbrauchenden Komponente

Sie können einen Container oder ein Produkt im Feld "Container/Produkt scannen" scannen oder eingeben. Die Anwendung akzeptiert folgende Zeichenketten:

• Containername
• Produkt
• Produktname und Revision

Die Eingabe eines Werts führt dazu, dass die Anwendung:

• Die erste Zeile in der Tabelle "Materialanforderungen" auswählt, die ein Komponentenprodukt oder Ersatzprodukt enthält, das mit dem eingegebenen Wert übereinstimmt.
• Die schreibgeschützten Felder im Bereich "Benutzerdateneingabe" füllt.

Sie können auch eine Zeile in der Tabelle "Materialanforderungen" auswählen, bevor Sie einen Container/ein Produkt eingeben. Wenn Sie dann den Container oder das Produkt scannen oder eingeben, überprüft die Anwendung, ob er/es der Komponenten-Anforderung entspricht, die Sie in der Tabelle ausgewählt haben.

Scannen vs. manuelle Eingabe

Die Anwendung füllt die Nutzereingabefelder und überprüft das Material automatisch, wenn Sie einen Container oder ein Produkt scannen. Sie können auf "Komponente verbrauchen" klicken, wenn Sie keine Änderungen vornehmen müssen, oder zusätzliche erforderliche Felder ausfüllen.

Wenn Sie einen Container oder eine Produkt-ID in das Feld "Container/Produkt scannen" eingeben, müssen Sie die Tab-Taste drücken oder mit dem Cursor außerhalb des Feldes "Container/Produkt scannen" klicken, damit die Anwendung die restlichen Dateneingabefelder füllt und anzeigt sowie den Container oder das Produkt überprüft.

Hinzufügen von betroffenen Containern zu Ereignissen

Die Anwendung kann Container automatisch zu Qualitätsereignissen hinzufügen, wenn Sie die Transaktion "Komponentenverbrauch" für einen Container durchführen, der mit mindestens einem Qualitätsereignis verknüpft ist. Wenn z. B. ein Komponentencontainer mit einem Ereignis verknüpft ist, fügt die Anwendung den Ziel-Container automatisch zum Qualitätsereignis hinzu. Ein Systemadministrator konfiguriert die Transaktion "Komponentenverbrauch" in der Modellierung, um Container zu Ereignissen hinzuzufügen. Informationen zum automatischen Hinzufügen von betroffenen Containern zu Ereignissen finden Sie im Opcenter Execution Medical Device and Diagnostics Event Management-Benutzerhandbuch und im Opcenter Execution Core Event Management-Benutzerhandbuch sowie im Opcenter Execution Medical Device and Diagnostics Nonconformance Management-Benutzerhandbuch und im Opcenter Execution Core Nonconformance Management-Benutzerhandbuch.

Seite "Komponentenverbrauch"

Es gibt zwei Versionen der Seite "Komponente verbrauchen": Einfach und Erweitert. Beide können aus dem Container Menü erreicht werden.

Seite "Komponente verbrauchen - Einfache"

Verwenden Sie die Seite "Komponente verbrauchen", um einen Container einzugeben oder auszuwählen, und verbrauchen Sie dann eine Komponente nach der anderen in diesem Container.

Seite "Komponente verbrauchen - Erweiterte"

Verwenden Sie die Seite "Komponente verbrauchen - Erweiterte Seite", um einen Container einzugeben oder auszuwählen, setzen Sie die Verbrauchskommandos in die Warteschlange, bis Sie alle erforderlichen Materialien erledigt haben und führen Sie dann ein einiges "Komponente verbrauchen"-Kommando aus.

Die "Erweiterte" Seite hat Untertabellen für jedes Komponentenprodukt, ein zusätzliches Ankreuzfeld und eine zusätzliche Schaltfläche. Außerdem ist ein Abschnitt für Kitverarbeitungsdetails verfügbar, um Komponentenverbrauch für in Kits einbezogene Komponenten zu aktivieren, wenn Industry Solutions installiert ist. Außerdem heißt die Schaltfläche "Komponente verbrauchen" auf der Erweiterten Seite "Komponenten verbrauchen". Ansonsten sind das Aussehen und Verhalten identisch mit der Seite "Komponente verbrauchen - Einfache Seite".
