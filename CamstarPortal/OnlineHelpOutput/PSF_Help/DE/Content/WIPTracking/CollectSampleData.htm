Stichprobendaten sammeln

Die Seite "Stichprobendaten sammeln" wird verwendet, um Stichprobendaten während der Produktion zu sammeln. Die Seite gibt neben allen Stichprobentests, die aktuell für einen bestimmten Container auf Grundlage von Spezifikation, Produkt und, Optional, Ressource erforderlich sind, auch entsprechende Anweisungen aus. Wenn ein Stichprobentest ausgewählt wird, werden aktuelle Informationen einschließlich der erforderlichen Stichproben, getesteten Stichproben, bestandenen Stichproben, durchgefallenen Stichproben, AQL-Level und Inspektionsebene angezeigt.

Sobald ein Stichprobentest ausgewählt wurde, können Sie die Testdaten sammeln.

Ist der Stichprobentest ein:

• Gemessen Stichprobentyp - geben Sie den numerischen Wert für jeden Datenpunkt ein
• Gezählt Stichprobentyp - verzeichnen Sie die Anzahl der abgelehnten Proben für jeden Grundcode

Wenn ein Stichprobentest ausgeführt wird, werden die gesamten getesteten Stichproben, bestandenen Stichproben und durchgefallenen Stichproben aufgezeichnet und die Container-Menge wird nach Bedarf um die Stichprobengröße oder die Verwurfsanzahl verringert.

Beim Sammeln von Stichprobendaten

Diese Regeln gelten beim Sammeln von Stichprobendaten:

• Nur Container mit Stichprobentests werden in der "Container"-Auswahlliste angezeigt. Linienzuordnungs-Werte dienen zum Filtern der Container-Liste anhand von Operation, Arbeitsplatz, Ressource und Arbeitsbereich.
• Container können nicht weiterbewegt werden, wenn an der aktuellen Spezifikation des Containers Sampling erforderlich ist, bis alle erforderlichen Stichproben erfasst wurden.
• Sie können einen Container nicht abmelden, wenn das Sampling in Bearbeitung ist.
• Ein Container kann nicht abgemeldet werden, wenn das Sampling fehlschlägt, es sei denn, dies wurde während der Modellierung angegeben.

Hinzufügen von Stichproben-Daten zu einem automatischen Ablauf

Ein automatischer Ablauf ist ein revisioniertes Objekt, das es Ihnen ermöglicht, einer Spezifikation (Spez.) eine Aufgabenliste mit einer "Stichprobendaten sammeln"-Aufgabe zuzuweisen. Sie müssen eine Aufgabenliste definieren, die eine Transaktions-Aufgabe enthält, die das CollectSamplingDataVP-Formular referenziert, bevor Sie die Aufgabenliste dem automatischen Ablauf zuweisen. Informationen zum Definieren von Aufgabenlisten, automatischen Abläufen und Spezifikationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Vorgehensweise zum Durchführen einer gemessenen Stichprobendatensammlung

Gehen Sie folgendermaßen vor, um eine gemessene Stichprobendatensammlung durchzuführen:

1. Öffnen Sie Stichprobendaten sammeln im Menü Container. Die Seite "Stichprobendaten sammeln" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
3. Wählen Sie einen gemessenen Stichprobentest aus der Liste in der Tabelle Container Stichprobendaten.
4. Geben Sie die Anzahl der Stichproben, die Sie für diesen Stichprobentest sammeln möchten, im Feld Anzahl zu testender Stichproben ein, um das Layout der Datensammlung-Tabelle für gemessene Stichprobentests festzulegen.
5. Klicken Sie auf die Schaltfläche Generieren, um die Anzahl der im Feld Anzahl zu testender Stichproben festgelegten Stichproben zu erzeugen. Die Anwendung verbirgt das Feld "Anzahl zu testender Stichproben" und die Schaltfläche "Generieren" und zeigt die Tabelle "Gemessen" an.
6. Geben Sie den Datenpunkt-Wert für jeden Stichprobentest in die Tabelle "Gemessen" ein, falls erforderlich.
7. Klicken Sie auf Abschicken oder in der Befehlsleiste auf Abschicken. Die Anwendung hält den aktuellen Container ausgewählt und aktualisiert den Status der Tabelle "Stichprobentest".

Vorgehensweise zum Durchführen einer gezählten Stichprobendatensammlung

Gehen Sie folgendermaßen vor, um eine gezählte Stichproben-Datensammlung durchzuführen:

1. Öffnen Sie Stichprobendaten sammeln im Menü Container. Die Seite "Stichprobendaten sammeln" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden.Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
3. Wählen Sie einen gezählten Stichprobentest aus der Liste in der Tabelle Container Stichprobendaten.
4. Geben Sie die Anzahl von Containern, die überprüft werden sollen, ins Feld Stichproben getestet ein.
5. Geben Sie für jeden Rückweiscode die Anzahl der Stichproben ins Feld Menge der Tabelle Stichprobentest gezählt ein, die Inspektion nicht bestanden haben.
6. Klicken Sie auf Abschicken oder in der Befehlsleiste auf Abschicken. Die Anwendung hält den aktuellen Container ausgewählt und aktualisiert den Status der Tabelle "Stichprobentest".
