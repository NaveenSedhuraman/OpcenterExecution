Container anmelden

Verwenden Sie die "Anmelden"-Transaktion, um anzuzeigen, dass ein bestimmter Container in einem Schritt angekommen ist und die Bearbeitung begonnen hat. Die "Anmelden"-Transaktion erlaubt der Anwendung, zu berechnen, wie lange der Container in der Warteschlange war, bevor er bearbeitet wurde. Das Verlangen einer "Anmelden"-Transaktion an einer Operation ist Optional. Falls eine "Anmelden"-Transaktion erforderlich ist, muss diese für jeden Container durchgeführt werden, bevor der Container abgemeldet wird.

Beim Verwenden von "Anmelden"

Alle Anzeigefelder sind auf der Seite "Anmelden" leer, bis ein Container ausgewählt wird. Anzeigefelder werden im "Anmelden"-Popup automatisch gefüllt. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Die Anwendung zeigt eine Standard-Ressource für den angegebenen Container an, wenn eine der folgenden Bedingungen zutrifft:

• Eine Ressource wurde angegeben, als der Container gestartet wurde
• Die "Anmelden"-Transaktion ist die erste Transaktion (mit Angabe einer Ressource), die Sie nach dem Start ausführen

Sie können, falls erforderlich, eine andere Ressource auswählen.

Die Anwendung verwendet die Ressource, die Sie angeben, wenn Sie einen Container anmelden, als Standard-Ressource am Bearbeitungsschritt. Die Anwendung zeigt die angegebene Ressource an, wenn Sie eine dieser Transaktionen im Bearbeitungsschritt ausführen:

• Abmelden
• Abmelden - nicht standardmäßig
• Nacharbeit
• Container transferieren

Sie können falls nötig eine andere Ressource auswählen.

Beim Anmelden eines Containers mit einem offenen Produktionsereignis

Hinweis: Diese Funktionalität ist nur als Teil des separat lizenzierten "Nonconformance Management"-Moduls verfügbar.

Eine Einstellung an der Operation bestimmt, ob das Anmelden erfolgreich ist oder fehlschlägt, wenn Sie einen Container anmelden, der ein offenes Produktionsereignis (Abweichung) hat. Falls das Ankreuzfeld "Einkommenden fehlerhaften Container zurückweisen" für eine Operation mit einem Häkchen versehen ist, hindert die Anwendung einen Benutzer daran, Container mit offenen Abweichungen an dieser Operation anzumelden. Informationen zum Definieren von Operationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Anmelden eines Containers mit aktiviertem automatischen Materialverbrauch

Hinweis: Diese Funktion ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Sie können auf der Seite "Fabrik" im Feld "Material-verbrauchen-Transaktionen" die "Anmelden"-Transaktion auswählen, um anzugeben, dass das für eine Ressource geladene Material automatisch verbraucht wird, wenn ein Container durch die Ressource bewegt wird. Sie können auch im Feld "Materialwarteschlange-prüfen-Transaktionen" die "Anmelden"-Transaktion auswählen, um die Anwendung zu aktivieren, die überprüft, dass das korrekte Material in einer Ressource für einen Container geladen ist, der durch die Ressource bewegt wird.

Die Anwendung gibt eine Warnmeldung aus, wenn Sie die "Anmelden"-Transaktion auswählen, aber nicht das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktivieren. Die Transaktion schlägt fehl, wenn die Menge nicht ausreichend ist, wenn Sie die "Anmelden"-Transaktion auswählen und das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktivieren. Sie können einen "Verbrauchsabweichungsgrund" auswählen, um zu verhindern, dass ein Fehler auftritt, wenn beim automatischen Komponentenverbrauch Teilmengen verbraucht werden.

Informationen zum Definieren von Fabriken finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Seite "Anmelden"

Sie können über das Container-Menü auf die Seite "Anmelden" zugreifen. Die Seite "Anmelden" ermöglicht es Ihnen, den Container einzugeben oder auszuwählen, den Sie zur Bearbeitung anmelden möchten.

Hinweis: Die "Anmelden"-Seite ist eventuell auch als Popup verfügbar, wenn die Transaktion "Anmelden" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (zum Beispiel "Containersuche").

Felddefinitionen der Seite und des Popups "Anmelden"

Diese Tabelle definiert die Felder auf der Seite und im Popup "Anmelden".

Feld

Definition

Typ

Containerinformationen

Kopf mit dem Containernamen und Informationen zu diesem Container. Sie müssen einen Container auf einer Transaktionsseite auswählen oder eingeben. Alle Informationen in einem Transaktionen-Popup sind schreibgeschützt. Weitere Informationen finden Sie unter "Containerinformationen".

Erforderlich/ Nur Anzeige

Ressource

Ressource, die den Container bearbeiten wird. Die Anwendung wählt, wo zutreffend, standardmäßig diese Ressource für Transaktionen, die im Schritt in Bearbeitung ausgeführt werden.

Die Ressourcenwerte, die zur Auswahl stehen, hängen von der Ressourcengruppe ab, die der Spezifikation (dem Schritt, an dem sich der Container befindet) zugewiesen wurde. Wenn der Spezifikation keine Ressourcengruppe zugewiesen wurde, stehen alle definierten Ressourcen zur Auswahl.

Hinweis: Wenn die Verfügbarkeit der angegebenen Ressource auf "Nicht verfügbar" gesetzt ist, zeigt die Anwendung eine Fehlermeldung an, die angibt, dass die Ressource nicht verfügbar ist, wenn Sie die Transaktion durchführen.

Optional

Standort

Zielort innerhalb der Fabrik für den Container. Ein Standort kann eine ganze Fabrik, ein Bereich einer Fabrik oder so spezifisch wie ein Regal-Identifikator sein. Standorte werden innerhalb des Fabrik-Objekts in der Modellierung definiert.

Hinweis: Dieses Feld ist standardmäßig ausgeblendet.

Optional

Standort zurücksetzen

Wenn ausgewählt, gibt es an, dass Sie den physischen Standort des Containers löschen möchten, sobald dieser zur Bearbeitung durch die Ressource angemeldet ist.

Hinweis: Dieses Ankreuzfeld ist standardmäßig versteckt.

Optional

Kommentare

Zusätzliche Informationen oder Anweisungen für diese Transaktion oder für diesen Container. Das Feld akzeptiert maximal 255 Zeichen. Wenn die Kommentare 255 Zeichen überschreiten, verkürzt die Anwendung die Daten.

Optional

Vorgehensweise zum Anmelden eines Containers und Beginnen der Verarbeitung

Gehen Sie folgendermaßen vor, um einen Container anzumelden und mit der Verarbeitung zu beginnen:

1. Öffnen Sie die Seite Anmelden im Menü Container. Die Seite "Anmelden" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Anmelden" zugreifen.
3. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
4. Klicken Sie auf Anmelden oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die bestätigt, dass der Container angemeldet wurde.
