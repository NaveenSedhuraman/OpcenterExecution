Ressourcenmaterialwarteschlange - Übersicht

Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich konfiguriert ist. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Industry Solutions-Benutzerhandbuch und im Opcenter Execution Core Industry Solutions-Benutzerhandbuch.

Die Funktionalität von Ressourcenmaterialwarteschlangen (RMQ) bietet die Möglichkeit, die Bestandsmenge an einem Lagerort zu verfolgen und den Bestand an eine Materialwarteschlange in einer Produktionslinie zu verteilen. Sie können die Materialwarteschlange einer Ressource zuordnen, damit der Bestand automatisch verbraucht wird, wenn ein Container, der der Materialwarteschlange zugeordnet ist, die Ressource durchläuft. Diese Funktionalität umfasst:

• Lagerortverwaltung
• Materialwarteschlangen
• Materialverbrauch

Lagerortverwaltung

Ein Lagerort stellt einen physischen Ort für die Lagerung von Materialien dar. Beispiele für Lagerorte sind Regale oder Lagerräume.

Materialien können in einem Lagerort als Container, Lose oder Produkte gelagert werden. Ein Los stellt in der Regel eine Lieferantenkennung dar, die in einem Arbeitsplan nicht verfolgt wird. Lose sind für Los- und Lager-Komponentenverbrauchstypen erforderlich.

Produkte werden verwendet, wenn kein Container oder Los zugeordnet ist. Produkte werden für den Komponentenverbrauchstyp "Nur Menge" verwendet.

Die Lagerortverwaltung bietet folgende Möglichkeiten:

• Verfolgung der einem Lagerort zugewiesenen Materialmenge
• Manuelle Zuweisung von Materialien zu einem Lagerort
• Automatische Zuweisung von Materialien zu einem Lagerort
• Transfer von Materialien von einem Lagerort an einen anderen
• Entfernen von Materialien aus einem Lagerort

Sie können Container automatisch in einen Lagerort stellen, indem Sie einen Standardlagerort für Fabriken, Produktfamilien und Produkte definieren. Wenn der Container in eine Operation abgemeldet wird, die als Lagerort definiert ist, wird der Container automatisch dem Lagerort zugewiesen. Informationen zum Definieren von Lagerorten finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Materialwarteschlangen - Übersicht

Eine Materialwarteschlange stellt eine Sammlung von Materialien dar, die für die Produktion eines Produkts verwendet werden. Sie können Materialien aus einem Lagerort einer Materialwarteschlange zuweisen. Dabei wird die im Lagerort verfügbare Menge entsprechend reduziert. Sie können Materialien aus einem Lagerort einer Materialwarteschlange oder mehreren Materialwarteschlangen zuweisen. Materialwarteschlangen können dann Ressourcen zugeordnet werden, damit automatisch Material verbraucht wird.

Sie können einer Ressource eine Materialwarteschlange oder mehrere Materialwarteschlangen zuordnen. Allerdings kann zu einem bestimmten Zeitpunkt nur eine Materialwarteschlange für die Ressource aktiv sein. Wenn Container von der Ressource verarbeitet werden, werden die Materialien in der aktiven Materialwarteschlange automatisch verbraucht. Die in der Materialwarteschlange verbleibende Menge wird entsprechend reduziert. Weitere Informationen zum Definieren von Materialwarteschlangen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Sie können übrig bleibende Materialien aus einer Materialwarteschlange entfernen. Materialien, die entfernt werden, werden automatisch an den Lagerort zurückgegeben, aus dem sie zugewiesen wurden.

Materialverbrauch - Übersicht

Die Materialverbrauchsfunktionalität bietet die Möglichkeit, in eine Ressource geladenes Material automatisch zu verbrauchen, wenn der Umlaufbestand (WIP) verarbeitet wird. Die Transaktion für den Materialverbrauch kann im Modellierungsobjekt "Fabrik" definiert werden. Sie können ggf. auch überprüfen, ob die Mengen der Materialwarteschlange für die Containerproduktion ausreichend sind, bevor die Verarbeitung beginnt. Informationen zum Definieren von Fabriken finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Materialwarteschlangen-Kitverarbeitung – Übersicht

Mit der Materialwarteschlangenfunktionalität können Sie Produkte oder Komponenten entsprechend dem Bedarf für Produktionsaufträge zu Materialwarteschlangen, die Kits entsprechen, hinzufügen oder aus den Warteschlangen entfernen. Materialwarteschlangen-Kits können erstellt und einem Container, einem Produktionsauftrag oder einem Arbeitsbereich zugewiesen werden. Werden die Materialien aus einem Lagerort geladen, wird die von diesem Ort verfügbare Menge verringert.

Mit der Funktionalität für Materialwarteschlangen-Kits können Sie:

• Ein Materialwarteschlangen-Kit mit Komponenten erstellen, die mit einem beliebigen Komponentenverbrauchstyp verbraucht werden.
• Anhand der Container- oder Produktionsaufträge-Materialliste überprüfen, ob die richtige Menge und das richtige Produkt verbraucht werden.
• Die Reihenfolge angeben, in der die Komponenten im Kit verwendet werden sollen.
• Die Entfernungsstrategie für Materialien angeben.
• Angeben, ob Materialien automatisch oder manuell verbraucht werden. Komponenten, die beim Verarbeiten des Containers verbraucht werden, werden überprüft und aus dem Materialwarteschlangen-Kit verbraucht.
• Angeben, welche verbrauchten Mengen an ein ERP-System (Enterprise Resource Planning) gemeldet werden, wenn ein ERP konfiguriert ist. Weitere Informationen zum Konfigurieren eines Opcenter CN MOM-Datenbankadapters finden Sie im Opcenter Connect MOM-Benutzerhandbuch.
• Elemente aus einem Materialwarteschlangen-Kit entladen. Die entladenen Elemente werden automatisch an den Lagerort zurückgegeben.
