Container abmelden

Sie verwenden die "Abmelden"-Transaktion, um einen Container zum nächsten Schritt im aktuellen Arbeitsplan abzumelden. In der Regel müssen Sie nur den Container angeben und dann die Transaktion abschicken. Wenn kein anderer gültiger Pfad oder Schritt vom Nutzer spezifiziert wurde, verwendet der Container den Standard-Pfad.

Beim Abmelden eines Containers

Alle Anzeigefelder sind auf der Seite "Abmelden" leer, bis ein Container ausgewählt wird. Anzeigefelder werden im "Abmelden"-Popup automatisch gefüllt. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Die Anwendung zeigt einen Standardwert für den angegebenen Container im Feld "Ressource" an, wenn eine der folgenden Bedingungen zutrifft:

• Für den aktuellen Schritt des Containers war ein Anmelden erforderlich, und der Benutzer, der den Container angemeldet hat, hat eine Ressource angegeben

Oder

• Eine Ressource wurde angegeben, als der Container gestartet wurde, und Abmeldung ist die erste Transaktion (mit Angabe einer Ressource), die Sie nach dem Start des Containers ausführen.

Sie können, falls erforderlich, eine andere Ressource auswählen.

Wenn das Kontrollkästchen "Materialverbrauch prüfen" für diese Spezifikation aktiviert ist, überprüft die Anwendung, ob alle Materialien und angegebenen Mengen für diesen Schritt verbraucht wurden, bevor sie den Container zum Abmelden zum nächsten Schritt im Arbeitsplan freigibt.

Beim Abmelden eines Containers mit einem offenen Produktionsereignis

Hinweis: Diese Funktionalität ist nur als Teil des separat lizenzierten Moduls "Nonconformance Management" verfügbar.

Wenn Sie einen Container abmelden, für den ein offenes Produktionsereignis (Abweichung) verzeichnet wurde, bestimmt eine Einstellung in der Operation, zu der Sie den Container abmelden, ob das Abmelden erfolgreich ist oder fehlschlägt. Falls das Kontrollkästchen "Eingehenden abweichenden Container ablehnen" für eine Operation aktiviert ist, hindert die Anwendung einen Benutzer daran, Container mit offenen Abweichungen zu dieser Operation abzumelden. Informationen zum Definieren von Operationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Abmelden eines Containers mit aktiviertem automatischen Materialverbrauch

Hinweis: Diese Funktion ist nur verfügbar, wenn der Industry Solutions-Arbeitsbereich installiert ist. Weitere Informationen finden Sie unter "Industry Solutions-Arbeitsbereich".

Sie können auf der Seite "Fabrik" im Feld "Material-verbrauchen-Transaktionen" die "Abmelden"-Transaktion auswählen, um anzugeben, dass das für eine Ressource geladene Material automatisch verbraucht wird, wenn ein Container durch die Ressource bewegt wird. Sie können auch im Feld "Materialwarteschlange-prüfen-Transaktionen" die "Abmelden"-Transaktion auswählen, um die Anwendung zu aktivieren, die überprüft, dass das korrekte Material in einer Ressource für einen Container geladen ist, der durch die Ressource bewegt wird.

Die Anwendung gibt eine Warnmeldung aus, wenn Sie die "Abmelden"-Transaktion auswählen, aber nicht das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktivieren. Die Transaktion schlägt fehl, wenn die Menge nicht ausreichend ist, wenn Sie die Abmelden-Transaktion auswählen und das Kontrollkästchen "Materialwarteschlange-prüfen-Menge" aktivieren. Sie können einen "Verbrauchsabweichungsgrund" auswählen, um zu verhindern, dass ein Fehler auftritt, wenn beim automatischen Komponentenverbrauch Teilmengen verbraucht werden.

Informationen zum Definieren von Fabriken finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Zuweisen einer UDI zur Seite "Abmelden"
Hinweis: Diese Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Über die Seite "Abmelden" können Sie einem Container automatisch eine UDI zuweisen, indem Sie Informationen für Spezifikation und Produkt definieren.

Beim Abmelden eines Container bei einer Spezifikation weist die Anwendung einem Container Werte für Gerätekennung, Produktionskennung und UDI zu, wenn das Feld zum Zuweisen einer UDI auf der Spezifikation markiert ist und wenn die aktuelle Transaktion der UDI-Transaktion des Produkts entspricht.

Die Anwendung legt die Gerätekennung des Containers auf die Gerätekennung des Produkts fest, wenn diesem Produkt nur eine Gerätekennung zugewiesen ist. Die Gerätekennung des Containers wird leer gelassen, wenn dem Produkt mehr als eine Gerätekennung zugewiesen ist.

Die Anwendung legt die Produktionskennung des Containers auf das Ergebnis des Produktionskennungs-Ausdrucks für das Produkt fest, wenn das Produkt über einen solchen Ausdruck verfügt.

Die Anwendung legt die UDI des Containers auf das Ergebnis des UDI-Ausdrucks für das Produkt fest, wenn das Produkt über einen UDI-Ausdruck verfügt.

Seite "Abmelden"

Sie können über das Container-Menü auf die Seite "Abmelden" zugreifen. Die Seite "Abmelden" ermöglicht es Ihnen, einen Container, den Sie zu einem Arbeitsplan-Schritt abmelden möchten, zu scannen oder einzugeben.

Hinweis: Die Seite "Abmelden" ist eventuell auch als Popup verfügbar, wenn die Transaktion "Abmelden" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (z. B. "Containersuche").

Felddefinitionen der Seite und des Popups "Abmelden"

Diese Tabelle definiert die Felder auf der Seite und im Popup "Abmelden".

Feld

Definition

Typ

Containerinformationen

Kopf mit dem Containernamen und Informationen zu diesem Container. Sie müssen einen Container auf einer Transaktionsseite auswählen oder eingeben. Alle Informationen in einem Transaktionen-Popup sind schreibgeschützt. Weitere Informationen finden Sie unter "Containerinformationen".

Erforderlich/ Nur Anzeige

Ressource

Ressource, die den Container im aktuellen Schritt bearbeitet hat. Sie können eine andere Ressource auswählen, wenn die Anwendung bereits standardmäßig eine angezeigt. Wenn Sie die Ressource in diesem Schritt angeben, bevor Sie den Container abmelden, sorgt das für Nachverfolgbarkeit.

Welche Ressourcenwerte zur Auswahl stehen, hängt von der Ressourcengruppe ab, die dem Schritt zugewiesen wurde (Spez.). Wenn der Spezifikation keine Ressourcengruppe zugewiesen wurde, stehen alle definierten Ressourcen zur Auswahl.

Optional

Zusätzliche Felder anzeigen/ausblenden

Optionale Felder sind standardmäßig ausgeblendet, wenn die Anwendung eine Seite oder ein Popup anzeigt. Durch Klicken auf die Steuerung "Anzeigen" werden die Felder angezeigt. Durch Klicken auf die Steuerung "Verstecken" werden die Felder ausgeblendet.

  

Pfad

Pfad zum nächsten Schritt im Arbeitsplan. Wenn Sie dieses Feld leer lassen, verwendet die Anwendung den Standard-Pfad. Der einzige verfügbare Schritt ist der Standard-Schritt, es sei denn, alternative Pfade wurden im entsprechenden Arbeitsplan-Modellierungsobjekt definiert.

Hinweis: Dieses Feld ist standardmäßig ausgeblendet.

Optional

Zur Ressource

Ressource, die den Container im nächsten Schritt bearbeiten wird. Die Ressourcenwerte, die zur Auswahl stehen, hängen von der Ressourcengruppe ab, die dem Schritt zugewiesen wurde (Spez.). Wenn der Spezifikation keine Ressourcengruppe zugewiesen wurde, stehen alle definierten Ressourcen zur Auswahl.

Hinweise:

• Dieses Feld ist standardmäßig ausgeblendet.
• Wenn die Verfügbarkeit der angegebenen Ressource auf "Nicht verfügbar" gesetzt ist, zeigt die Anwendung eine Fehlermeldung an, die angibt, dass die Ressource nicht verfügbar ist, wenn Sie die Transaktion durchführen.

Optional

Zu Standort

Standort, an den der Container im nächsten Schritt zur Bearbeitung bewegt wird. Ein Standort kann eine ganze Fabrik, ein Bereich einer Fabrik oder so spezifisch wie ein Regal-Identifikator sein. Standorte werden innerhalb des Fabrik-Objekts in der Modellierung definiert.

Hinweis: Dieses Feld ist standardmäßig ausgeblendet.

Optional

Kommentare

Zusätzliche Informationen oder Anweisungen für diese Transaktion oder für diesen Container. Das Feld akzeptiert maximal 255 Zeichen. Wenn die Kommentare 255 Zeichen überschreiten, verkürzt die Anwendung die Daten.

Optional

Vorgehensweise zum Abmelden eines Containers zum nächsten Schritt

Gehen Sie folgendermaßen vor, um einen Container zum nächsten Schritt abzumelden:

1. Öffnen Sie die Seite Abmelden im Container-Menü. Die Seite "Abmelden" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Abmelden" zugreifen.
3. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
4. Klicken Sie auf Abmelden oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die bestätigt, dass der Container abgemeldet wurde.
