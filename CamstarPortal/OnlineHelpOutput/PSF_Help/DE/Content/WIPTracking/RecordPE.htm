Produktionsereignis aufzeichnen

Die Transaktion "Produktionsereignis aufzeichnen" ermöglicht es Ihnen, die Details einer Container-bezogenen Abweichung einzugeben, darunter:

• Relevante Fehlermodi
• Container, die von dem Ereignis betroffen sind
• Betroffene Ressource (optional)
• Jeweiliger Schritt im Arbeitsplan, dem der Fehler zugeordnet werden kann (optional)

Sie können auch:

• Betroffene Materialien anhalten
• Zugehörige Dateien anhängen

Beim Aufzeichnen eines Produktionsereignisses

Sie müssen mindestens einen Fehlermodus für das Produkt in der Tabelle "Fehlermodi" eingeben, wenn Sie Informationen auf der Seite "Produktionsereignis aufzeichnen" eingeben. Sie müssen außerdem einen oder mehrere Container in der Tabelle "Betroffene Container" eingeben. Mindestens ein Fehlermodus und ein Container sind erforderlich.

Die Anwendung füllt automatisch die Tabelle "Betroffene Container" mit den Containern, die auf der ursprünglichen Seite ausgewählt wurden, wenn Sie über eine andere Seite (z. B. "Containersuche") auf das Popup-Feld "Produktionsereignis aufzeichnen" zugreifen.

Die Anwendung umgeht Anforderungen für elektronische Unterschriften, die für die Standardtransaktion "Anhalten" definiert wurden, wenn Sie das Kontrollkästchen "Betroffene Container anhalten" aktivieren. Die angegebenen Container werden angehalten, ohne dass eine elektronische Unterschrift erforderlich ist.

Sie müssen dem Mitarbeiter eine primäre Organisation zuweisen, damit Sie ein Produktionsereignis aufzeichnen können. Außerdem müssen Sie in der Modellierung eine Nummerierungsregel erstellen und diese Regel im Objekt "Organisation" einer Abweichung zuweisen. Darüber hinaus muss die primäre Organisation eine Rolle und einen Eigentümer für den angemeldeten Benutzer haben, die bzw. der einer entsprechenden Klassifikation und Unterklassifikation in der Tabelle "Zuordnungen von Ereignisklassifizierungspez." zugewiesen ist.

Die Felder "Klassifikation" und "Unterklassifikation" im Bereich "Zusätzliche Felder anzeigen/ausblenden" auf der Seite "Produktionsereignis aufzeichnen" dürfen nicht leer sein. Sie enthalten standardmäßig die Werte, die in der als PE-Standard ausgewählten Zeile der Tabelle "Zuordnungen von Ereignisklassifizierungspez." für die Felder "Klassifikation" und "Unterklassifikation" angegeben sind. Sie können die Standardeinstellungen ändern. Vervollständigen Sie die Tabelle "Zuordnungen von Ereignisklassifizierungspez." mithilfe des Modellierungsobjekts "Organisation". Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Laden der Seite "Produktionsereignis aufzeichnen" wird eine Meldung mit dem Hinweis angezeigt, dass mit dem Modellierungsobjekt "Organisation" eine Standardklassifikation und -unterklassifikation für ein Produktionsereignis festgelegt werden muss, wenn keine Standardwerte für "Klassifikation" und "Unterklassifikation" vorhanden sind.

Die Anwendung druckt automatisch Produktionsereignisetiketten, falls Ihre Organisation über eine Lizenz für das Modul "Label Printing" verfügt und ein Systemadministrator ordnungsgemäß Etikettendruckdefinitionen in der Modellierung konfiguriert hat. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch. Sie können auch auswählen, dass Etiketten manuell nach einer Produktionsereignistransaktion gedruckt werden, falls die Produktionsereignis-Etikettenzuordnung nicht werkseitig konfiguriert wurde.

Die Anwendung erstellt eine Historie für ein Produktionsereignis, wenn es erstellt wird.

Die Anwendung verhindert, dass Sie die Zeichen +, amp;, # beim Hinzufügen von Anhängen im Titel eingeben.

Die maximale Dateigröße für einen Anhang beträgt 30 MB.

Seite "Produktionsereignis aufzeichnen"

Sie können die Seite "Produktionsereignis aufzeichnen" über das Menü "Container" aufrufen. Auf der Seite "Produktionsereignis aufzeichnen" können Sie containerbezogene Abweichungen aufzeichnen.

Hinweis: Die Seite "Produktionsereignis aufzeichnen" ist u. U. auch als Popup verfügbar, wenn die Transaktion "Produktionsereignis aufzeichnen" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (z. B. Containersuche).

Vorgehensweise zum Aufzeichnen eines Produktionsereignisses

Gehen Sie folgendermaßen vor, um ein Produktionsereignis aufzuzeichnen:

1. Öffnen Sie die Seite Produktionsereignis aufzeichnen im Menü Container. Die Seite "Produktionsereignis aufzeichnen" wird angezeigt.
2. Geben Sie eine Beschreibung für das Ereignis ein.
3. Wählen Sie eine Fehlermodus-Gruppe. Die Tabelle Fehlermodus wird aktualisiert, um alle Fehlermodi in der angegebenen Fehlermodus-Gruppe anzuzeigen.
4. Wählen Sie einen oder mehrere Fehlermodi in der Tabelle Fehlermodi aus.
5. Wählen Sie einen Container im Container hinzufügen-Feld aus oder geben Sie ihn ein. Der Containername wird zur Tabelle "Betroffene Container" hinzugefügt.
Hinweis: Wenn Sie im Popup "Produktionsereignis aufzeichnen" arbeiten, werden die Container, die Sie auf der vorherigen Seite ausgewählt haben, automatisch in der Tabelle "Betroffene Container" angezeigt.
6. Wiederholen Sie Schritt 5, um weitere Container hinzuzufügen.
7. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
8. Führen Sie die Schritte unter "Vorgehensweise zum Anhängen von Dateien an einen Ereignisdatensatz" aus, wenn Sie Dateien an den Ereignisdatensatz anhängen möchten.
9. Klicken Sie auf Produktionsereignis aufzeichnen oder in der Befehlsleiste auf Abschicken. Die Anwendung zeigt eine Statusmeldung an, die bestätigt, dass die Produktionsereignis-Datensatz erstellt wurde. Wenn Ihre Organisation das "Etikettendruck"-Modul erworben hat und das Ankreuzfeld "Etikett nach der Aufzeichnung manuell ausdrucken" mit einem Häkchen versehen wurde, zeigt die Anwendung die Seite "Etiketten für Produktionsereignis drucken" an.

Vorgehensweise zum Anhängen von Dateien an einen Ereignisdatensatz

Gehen Sie folgendermaßen vor, um eine oder mehrere Dateien an eine Ereignis-Aufzeichnung auf der Seite "Produktionsereignis aufzeichnen" anzuhängen:

1. Führen Sie die Schritte 1-7 unter "Vorgehensweise zum Aufzeichnen eines Produktionsereignisses" aus.

Oder

Wählen Sie ein vorhandenes Produktionsereignis aus.

2. Klicken Sie auf das Steuerelement Zusätzliche Felder anzeigen (Dateien anhängen). Die Anwendung zeigt die Tabelle "Anhänge" an.
3. Klicken Sie auf Neue Zeile hinzufügen. Das Popup "Anhänge" wird angezeigt.
4. Geben Sie den Dateipfad im Feld Originaler Dateipfad ein.

Oder

Klicken Sie auf Durchsuchen, um zu der Datei zu navigieren.

5. Geben Sie den Titel der Datei ein.
6. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
7. Klicken Sie auf OK. Die Dateiinformationen werden zur Tabelle "Anhänge" hinzugefügt.
8. Wiederholen Sie die Schritte 3 bis 7, um weitere Dateien hinzuzufügen.
