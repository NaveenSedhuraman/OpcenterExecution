Mengenänderung: mehrere Gründe

Die Transaktion "Mengenänderung: mehrere Gründe" ermöglicht es Ihnen, die Menge des Produktes innerhalb des Containers unter Verwendung mehrerer Gründe für einen Änderungstyp innerhalb einer einzelnen Transaktion zu ändern.

Hinweis: Weitere Informationen über das Ändern der Menge in einem Container bei einem einzigen Grund finden Sie unter "Mengenänderung".

Arten von Mengenänderungen

Diese Tabelle beschreibt die fünf Arten von Mengenänderungen, die Sie angeben können, wenn Sie die Menge in einem Container ändern.

Anpassen

Anpassen gibt entweder eine Zunahme oder Abnahme der Menge aus einem unbekannten Grund an. Verwenden Sie diesen Typ, wenn das Ergebnis nach dem Zählen der Menge eines Containers nicht die erwartete Menge ist.

Bonus

Ein Bonus gibt eine Zunahme der Menge des Containers infolge eines Produktionsprozesses an. Beispielsweise hat sich das Gewicht eines Produkts infolge der Verarbeitung erhöht.

Kaufen

Ein Kaufen gibt eine Zunahme der Menge eines Containers an, die nicht das Resultat eines Produktionsprozesses ist. Beispielsweise wurden bestimmte Einheiten von der Ingenieur-Abteilung zurückgegeben.

Verlust

Ein Verlust gibt eine Abnahme der Menge eines Containers infolge eines Produktionsprozesses an.

Verkaufen

Ein Verkaufen gibt eine Abnahme der Menge eines Containers an, die nicht das Resultat eines Produktionsprozesses ist. Beispielsweise sind einige Container zu Testzwecken abgegeben oder zu Demonstrationszwecken an die Verkaufsabteilung abgegeben.

Die Arten von Mengenänderungen stehen unter diesen Bedingungen zur Wahl:

• Ein Grund wurde definiert.
• Eine Grund-Gruppe, die den Grund enthält, wurde definiert.
• Die Grundgruppe wurde der aktuellen Operation des Containers zugewiesen.
Hinweis: "Gründe", "Grundgruppen" und "Operationen" werden in der Modellierung definiert. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Ändern einer Menge

Alle Anzeigefelder sind auf der Seite "Mengenänderung: mehrere Gründe" leer, bis ein Container ausgewählt wird. Anzeigefelder werden im Popup "Mengenänderung: mehrere Gründe" automatisch gefüllt. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Die Anwendung erfordert, dass einer Operation eine Grund-Gruppe zugewiesen wird, bevor Sie die "Mengenänderungs"-Transaktion durchführen können. Wenn Sie auf der Seite oder im Popup "Mengenänderung: mehrere Gründe" einen Container angeben, der sich an einer Operation befindet, für die keine Grund-Gruppe definiert wurde, zeigt die Anwendung entsprechend eine Warnmeldung an. Die Anwendung aktiviert die Schaltfläche "Mengenänderung" nur dann, wenn ein Container ausgewählt und eine Grund-Gruppe für die aktuelle Operation definiert wurde.

Sie können die Menge eines Containers mit mehreren Gründen ändern, aber Sie können nur eine Änderungsart pro Transaktion verwenden. Die Anwendung zeigt die Arten von Mengenänderungen in dieser Reihenfolge an:

• Verlust, die Standardauswahl
• Bonus
• Verkaufen
• Kaufen
• Anpassen

Die Liste kann variieren, je nachdem, ob der aktuellen Operation des Containers Grund-Gruppen zugewiesen wurden. Wenn der aktuellen Operation nur wenige Grund-Gruppen zugewiesen wurden, verkürzt die Anwendung die Liste, während sie die Listen-Reihenfolge beibehält. Der erste Änderungstyp ist die Standardauswahl. Wenn beispielsweise der aktuellen Operation keine "Verlustgrundgruppe" zugewiesen wurde, ist Bonus der erste Änderungstyp in der Liste und damit die Standardauswahl.

Die Art der Mengenänderung, die Sie auswählen, betrifft Feld-Beschriftungsnamen. Die Namen der Felder "Verlustgrund" und "Ausfallmenge" ändern sich je nach Auswahl. Wenn Sie beispielsweise "Bonus" auswählen, benennt die Anwendung die Felder in "Bonusgrund" und "Bonusmenge" um.

Die Anwendung erfordert, dass Sie mindestens einen Grund hinzufügen und eine Menge für diesen Grund eingeben, bevor SIe die Menge ändern können. Die Anwendung zeigt eine Fehlermeldung an, wenn Sie auf die Schaltfläche "Mengenänderung" klicken, ohne dass Sie einen Mengenänderungs-Grund und eine Menge hinzugefügt haben.

Die Anwendung akzeptiert negative Zahlen im Feld "<Änderungstyp>-Menge". Negative Zahlen bieten die Möglichkeit, eine Mengenänderungs-Transaktion zurückzurollen. Wenn Sie beispielsweise fälschlicherweise einen Verlust eingeben, können Sie den Verlust zurückrollen, indem Sie eine negative Zahl eingeben. Die Eingabe negativer Zahlen für "Verlust"- und "Verkaufen"-Gründe erhöht die Container-Menge, während die Eingabe negativer Zahlen für "Bonus", "Kaufen" und "Anpassen" die Container-Menge verringert.

Wenn die Summe der Mengen, die für alle Mengenänderungen eingegeben wurden, die Menge des Containers auf einen geringeren Wert als dessen aktuelle Menge reduziert, zeigt die Anwendung eine Fehlermeldung an, wenn Sie auf die Schaltfläche "Mengenänderung" klicken.

Seite "Mengenänderung: mehrere Gründe"

Sie können über das Menü "Container" auf die Seite "Mengenänderung: mehrere Gründe" zugreifen. Die Seite "Mengenänderung: mehrere Gründe" ermöglicht es Ihnen, einen Container einzugeben oder auszuwählen und dann die Menge innerhalb des Containers zu ändern.

Hinweis: Die Seite "Mengenänderung: mehrere Gründe" ist eventuell auch als Popup verfügbar, wenn die Transaktion "Mengenänderung (mehrere Gründe)" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (z. B. "Containersuche").

Zusätzliche Felder anzeigen/ausblenden

Ressource

Kommentare

Optional

Vorgehensweise zum Ändern der Menge unter Verwendung mehrerer Gründe

Gehen Sie folgendermaßen vor, um die Menge unter Verwendung mehrerer Gründe zu ändern:

1. Wählen Sie im Container-Menü Mengenänderung: mehrere Gründe aus. Die Seite "Mengenänderung: mehrere Gründe" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden.Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Mengenänderung: mehrere Gründe" zugreifen.
3. Wählen Sie einen Mengenänderungstyp. Die Feld-Etiketten werden entsprechend aktualisiert.
4. Wählen Sie die Zeile für den gewünschten Grund in der Tabelle <Typ>-Grund aus.
5. Geben Sie die Menge in das Feld <Typ>-Menge ein.
6. Geben Sie falls nötig, zusätzliche Informationen für die Änderung in das Feld Kommentare ein.
7. Wiederholen Sie die Schritte 3 bis 6, um zusätzliche Gründe und Mengen zur Tabelle hinzuzufügen.
8. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
9. Klicken Sie auf Mengenänderung oder in der Befehlsleiste auf Abschicken, um die Menge des Containers zu ändern. Eine Statusmeldung wird angezeigt, die bestätigt, dass die Menge für den angegebenen Container geändert wurde.
