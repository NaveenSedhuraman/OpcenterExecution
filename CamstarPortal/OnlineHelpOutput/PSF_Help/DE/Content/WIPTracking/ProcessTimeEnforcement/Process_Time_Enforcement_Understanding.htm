Prozesszeiterzwingung verstehen

Mithilfe von Prozesszeiterzwingung können Sie Containern Prozesstimer zuordnen. Prozesstimer bestimmen, welche Aktionen erfolgen sollen, wenn ein Bediener oder eine Anlagenkomponente vor Ablauf eines minimalen bzw. nach Ablauf eines maximalen Zeitintervalls eine Transaktion in einem Container durchzuführen versucht.

Die Anwendung führt beim Ausführen einer Transaktion eine Validierung durch, um sicherzustellen, dass kein Timer außerhalb seiner definierten Grenzen liegt. Wenn der Timer eines Containers außerhalb der Grenzen liegt, muss beim Ausführen einer Transaktion für den betreffenden Container die im Timer definierte Aktion durchgeführt werden.

Konfigurieren von Prozesszeiterzwingung in der Modellierung

Ein Systemadministrator muss folgende Aufgaben durchführen, um Prozesszeiterzwingung einzurichten:

• Er muss je nach Ihren Geschäftsanforderungen einen oder mehrere Prozesstimer definieren.
• Er muss einer Spezifikation einen oder mehrere Prozesstimer hinzufügen. Prozesstimer können standardmäßig nur den Transaktionen "Abmelden" und "Anmelden" zugeordnet werden.
• Er muss einer Aufgabe einen oder mehrere Prozesstimer hinzufügen. Prozesstimer können standardmäßig nur den Transaktionsaufgaben "Abmelden" und "Anmelden" hinzugefügt werden.

Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Über Prozesstimer

Nur die folgenden Transaktionen können einen Timer auslösen:

• Abmelden
• Anmelden
• Aufgabe ausführen
Hinweis: Prozesstimer berücksichtigen nicht die Anzahl von Iterationen, die für eine Aufgabe definiert sind. Der Prozesstimer wird nur bei der ersten Ausführung der Aufgabe ausgeführt.

Die Transaktionen "Abmelden" und "Anmelden" lösen Timer für einen Container aus:

• An einer Spec, wenn ein oder mehrere Prozesstimer für die Spec konfiguriert wurden.

Oder

• In einem automatischen Ablauf, wenn ein oder mehrere Prozesstimer für eine Transaktionstypaufgabe konfiguriert wurden und die Transaktionen "Abmelden" und "Anmelden" für die betreffende Aufgabe angegeben sind.

Die Transaktion "Aufgabe ausführen" in automatischen Abläufen löst Timer für einen Container aus, wenn ein oder mehrere Prozesstimer für die Aufgabe konfiguriert sind. Sie können allen Aufgabentypen Prozesstimer hinzufügen: Anweisung, Berechnung und Transaktion.

Die Historie der Container- und Timerinformationen wird gespeichert und ist im Produktionsprotokoll verfügbar.

Die Anwendung sucht nach aktiven Prozesstimern, wenn die Transaktion "Schließen" zum Schließen eines Containers verwendet wird. Alle vorhandenen Timer werden angehalten und die Transaktion "Schließen" im Produktionsprotokoll für den Container zeigt die Prozesstimer-Historiendetails für alle Timer an, die angehalten werden.

Anzeigen von Prozesstimern

Es können mehrere Timer für einen Container ausgelöst werden. Der wichtigste Timer ist derjenige, der im Feld "Timer" und in der ersten Zeile des Flyouts "Aktive Timer" angezeigt wird. Timer mit einem Wert im Feld "Zeit bis Max." gelten als am wichtigsten. Falls mehr als ein Timer mit einem Wert im Feld "Zeit bis Max." vorhanden ist, gilt der Timer mit der kürzesten Restzeit als wichtigster Timer. Falls kein Timer mit einem Wert im Feld "Zeit bis Max." vorhanden ist, gilt der Timer mit der kürzesten Restzeit im Feld "Zeit bis Min" als wichtigster Timer.

Alle verfügbaren Timer werden an folgenden Stellen in der Anwendung angezeigt:

• Containerinformationen – Das Feld "Timer" wird in den Containerinformationen angezeigt. Der wichtigste Timer wird im Feld angezeigt. Wenn Sie auf das Dreieck neben der Feldbezeichnung "Timer" klicken, wird das Flyout "Aktive Timer" geöffnet und zeigt alle aktiven Timer an. Weitere Informationen finden Sie unter "Containerinformationen".
• Seite "Operative Ansicht" – Das Feld "Timer" auf der Seite "Operative Ansicht" wird in den Tabellen "Container in Warteschlange" und "Container in Bearbeitung" angezeigt. Der wichtigste Timer wird in dem Feld und den Chevrons angezeigt, die das Flyout "Aktive Timer" aktivieren, wenn sie angeklickt werden. Alle anderen aktiven Timer werden auf dem Flyout "Aktive Timer" angezeigt. Weitere Informationen finden Sie unter "Operative Ansicht".
• Seite "Prozesstimersuche" – Auf dieser Seite können Sie nach Timern suchen, die Ergebnisse sortieren, Timer ansehen und Timer stoppen. Weitere Informationen finden Sie unter "Prozesstimersuche".
Wichtig: Bei Diskrepanzen zwischen der Serveruhr und der Uhr des lokalen Clients kann es zu unerwartetem Verhalten kommen. Die Zeit wird basierend auf der Uhr des lokalen Clients angezeigt, die tatsächliche Zeit wird jedoch von der Serveruhr gesteuert. Ein Timer kann beispielsweise anzeigen, dass die minimale Zeit erreicht wurde, während der Server mehrere Sekunden hinter dieser Zeit zurückbleibt. Das Ausführen einer Transaktion kann einen Fehler erzeugen, bis die Serveruhr aufholt.

Funktionsweise von Prozesstimern

Erforderliche Timerinformationen

Der Systemadministrator sollte beim Definieren eines Timers folgende Informationen einschließen:

• Timername
• Minimale Endzeit, maximale Endzeit oder beides

Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Optionale Timerinformationen

Der Systemadministrator kann beim Einrichten des Prozesstimers in der Modellierung jeder minimalen und maximalen Endzeit Farben zuordnen. Die Farbe wird im Timerfeld überall dort angezeigt, wo der Timer sichtbar ist, wenn die minimale oder maximale Endzeit erreicht wird. Timer sind in den Containerinformationen, in den Tabellen auf der Seite "Operative Ansicht" und auf der Seite "Prozesstimersuche" sichtbar.

Prozesstimer können eine minimale oder maximale Warnzeit haben. Den Warnzeiten können in der Modellierung Farben zugeordnet werden. Dadurch wird ein Warnzeitraum für den Container festgelegt, durch den Sie darauf aufmerksam gemacht werden, dass sich die minimale oder maximale Endzeit nähert. Wenn die minimale oder maximale Warnzeit erreicht ist, ändert sich die Hintergrundfarbe des Timerfelds oder der Tabellenzeile. Daran ist zu erkennen, dass sich der Container im Warnungszeitraum befindet.

Mehrere Timer in einem Container

Sie können denselben Timer beliebig oft in demselben Container starten. Die Instanzen dieser Timer können gleichzeitig laufen. Beim Ausführen einer Timer-beenden-Transaktion für den Timer hält die Anwendung den Timer an, der am längsten gelaufen ist. Die übrigen Instanzen des Timers laufen weiter.

Die Anwendung verhindert die Ausführung doppelter Timer. Ein doppelter Timer ist ein Timer desselben Namens, der im selben Container von derselben Transaktion im selben Schritt im Arbeitsplan gestartet wird. Dieses Szenario ist bei Nacharbeiten im Arbeitsplan möglich. Die Anwendung hält den laufenden Timer an und startet eine neue Instanz dieses Timers.

Timer kleiner als Min. oder größer als Max.

Der Systemadministrator kann festlegen, wie sich die Anwendung verhalten soll, wenn eine Transaktion ausgeführt wird, obwohl der Timer die minimale Zeit noch nicht erreicht bzw. die maximale Zeit überschritten hat.

• Ausführen der Endtransaktion und aller Anordnungsaktionen nicht zulassen:
• Der Timer läuft weiter.
• Die Endtransaktion wird zugelassen, sobald die minimale Zeit erreicht wurde.
• Wenn die maximale Zeit überschritten wurde, müssen Sie den Container manuell manipulieren (eine Transaktion "Container abmelden" ausführen) und den Timer über die Schaltfläche "Timer stoppen" auf der Seite "Prozesstimersuche" stoppen.
• Ausführen der Endtransaktion und aller Anordnungsaktionen zulassen:
• Der Timer hält an.
• Die Endtransaktion und alle Anordnungsaktionen werden ausgeführt.
Hinweis: Der Systemadministrator kann einen Timer so konfigurieren, dass ein Bestätigungsdialogfeld angezeigt werden muss, bevor die Endtransaktion und Anordnungsaktionen ausgeführt werden. Sie können im Dialogfeld "Ja" auswählen, wenn die Endtransaktion und Anordnungsaktionen ausgeführt werden sollen, oder "Nein", um zum vorherigen Bildschirm zurückzukehren.
• Zulassen, dass die Endtransaktion mit einer elektronischen Unterschrift ausgeführt wird und keine Anordnungsaktionen ausgeführt werden.
• Es wird ein Elektronische Unterschriften-Popup angezeigt, bevor die Transaktion abgeschlossen wird.
• Die Endtransaktion wird ausgeführt, falls die elektronische Unterschriften-Anforderung ordnungsgemäß erfüllt ist.
• Die Endtransaktion wird nicht ausgeführt, und der Timer hält nicht an, falls die elektronische Unterschriften-Anforderung nicht ordnungsgemäß erfüllt ist.
Hinweis: Alle elektronischen Unterschriften werden in einem einzige Popup angezeigt, wenn elektronische Unterschriften für mehrere Timer in einer Transaktion erforderlich sind. Falls das Popup geschlossen wird, ohne alle elektronische Unterschriften-Anforderungen zu erfüllen, wird die Transaktion nicht ausgeführt, und der Timer hält nicht an.

Anordnungsaktionen

Der Systemadministrator kann spezifizieren, welche Anordnungsaktionen die Anwendung durchführen soll, wenn die Endtransaktion außerhalb der Grenzen ausgeführt wird. Der Systemadministrator kann seine Auswahl unter den folgenden Anordnungsaktionen treffen. Sie werden in der angegebenen Reihenfolge ausgeführt:

1. Geschäftsregel
2. Produktionsereignis
3. Nacharbeit
4. Container abmelden
5. Anhalten
Hinweis: Die Konfiguration des Arbeitsplans des Containers hat keinen Einfluss auf die Reihenfolge der Anordnungsaktionen.

Wenn der Administrator keine Anordnungsaktionen spezifiziert und der Timer so eingerichtet ist, dass er Endtransaktionen und Anordnungsaktionen zulässt, wird die Endtransaktion erfolgreich ausgeführt, und es werden keine weiteren Aktionen ausgeführt. Falls mehr als eine Anordnungsaktion spezifiziert wird und eine davon fehlschlägt, werden nachfolgende Anordnungsaktionen nicht ausgeführt. Nehmen wir beispielsweise an, dass sowohl ein Produktionsereignis als auch Nacharbeit spezifiziert werden. Wenn das Produktionsereignis fehlschlägt, wird die Nacharbeitstransaktion nicht ausgeführt. Eine erfolgreiche Anordnungsaktion wird nicht rückgängig gemacht, wenn eine nachfolgende Anordnungsaktion fehlschlägt. Falls der Timer über die Schaltfläche "Timer stoppen" auf der Seite "Prozesstimersuche" gestoppt wird, werden keine Verteilungsaktionen ausgeführt. Sie können mit einer elektronischen Unterschrift angeben, dass eine Endtransaktion ausgeführt und ein Timer außerhalb der Grenzen angehalten werden darf. Anordnungsaktionen werden mit dieser Option nicht ausgeführt.

Hinweis: "Produktionsereignis" ist eine Funktion von Quality, ein lizenziertes Feature. Eine "Produktionsereignis"-Anordnungsaktion schlägt fehl, wenn Ihre Organisation nicht für Quality lizenziert ist. Alle nachfolgenden Verteilungsaktionen werden nicht ausgeführt. Die Anwendung erstellt keine Abschluss- oder Fehlermeldungen für Verteilungsaktionen.
