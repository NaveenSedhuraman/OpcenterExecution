Container aufteilen

Die Transaktion "Container aufteilen" überträgt einen oder mehrere Kind-Container von einem Eltern-Container zu einem neuen Eltern-Container.

Hinweis: Verwenden Sie die Transaktion "Menge aufteilen", wenn Sie einen oder mehrere Zielcontainer aus einem vorhandenen Quellcontainer erstellen und dann eine Teilmenge des Produkts aus dem Quellcontainer in die neuen Zielcontainer übertragen möchten.

Beispiele für das Aufteilen von Mengen und Containern

Dies sind einige Beispiele für die Verwendung der "Aufteilen"-Transaktionen:

• Eine kleine Menge eines Containers wird aufgeteilt, damit er schneller verarbeitet werden kann, um die Kunden- oder Inventar-Anforderungen zu erfüllen.
• Ein Teil eines Containers mit Qualitätsproblemen wird geteilt, bevor eine Nacharbeits- oder Anhalten-Transaktion durchgeführt wird. Der Rest des Containers wird normal bearbeitet.
• Ein Teil des Containers wird geteilt, sodass er zu einem anderen Arbeitsplan zur Bearbeitung eines Spezial-Auftrags bewegt werden kann.

Beim Aufteilen einer Menge oder eines Containers

Die "Container aufteilen"-Transaktion ermöglicht es Ihnen lediglich, nur Eltern-Container aufzuteilen. Sie können den Kind-Container einem neuen Eltern-Container zuteilen.

Das Aufteilen eines Containers oder einer Menge bewegt weder die Quell- noch die Zielcontainer. Es impliziert auch nicht, dass eine Prozess-Aktivität (Durchsatz) an einem der Container durchgeführt wurde.

Sie müssen einen eindeutigen Containernamen angeben, wenn Sie einen neuen Container erstellen. Sie können einen Containernamen entweder manuell eingeben oder ihn automatisch von der Anwendung erstellen lassen. Es werden geschlossene Container von der Anwendung einbezogen, wenn ein eindeutiger Name geprüft wird.

Aktivieren Sie das Kontrollkästchen "Namen automatisch erstellen", um den Containernamen unter Verwendung der definierten Nummerierungsregeln von der Anwendung erstellen zu lassen. Wenn Sie dieses Kontrollkästchen aktivieren, löscht und deaktiviert die Anwendung das Feld "Zielcontainer".

Wenn die Option "Namen automatisch erstellen" ausgewählt wird, ohne vorher eine automatische Nummerierungsregel festzulegen, wird eine Fehlermeldung angezeigt.

Die Anwendung verwendet zum Erstellen des Zielcontainernamens die Nummerierungsregeln, die dem Eltern-Container zugeordnet sind, von dem aus der Kind-Container aufgeteilt wird. Wenn Sie auf die Schaltfläche "Abschicken" klicken, erstellt die Anwendung den Namen des neuen Eltern-Containers.

Nummerierungsregeln können für mehrere, dem Container zugeordnete Modellierungsobjektinstanzen festgelegt werden. Nummerierungsregeln folgen dieser Standardrangfolge:

1. Containerlevel
2. Produktionsauftrag
3. Produkt
4. Produktfamilie
5. Fabrik

Erben der UDI von geteilten oder Eltern-Containern
Hinweis: Diese Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Ein geteilter Container kann die UDI von dem Container erben, von dem er abgeteilt wurde, aber ein Kind-Container kann nicht die UDI des Eltern-Containers erben.

Erben der UDI von geteilten Containern

Ein geteilter Container erbt die Gerätekennung, die Produktionskennung und die UDI, die dem Container zugewiesen waren, von dem er abgeteilt wurde, wenn die Teilung nach Zuweisung der UDI erfolgte.

Erben der UDI von Eltern-Containern

Kind-Container eines Eltern-Containers erben nicht automatisch dessen Gerätekennung, Produktionskennung oder UDI. Sie müssen getrennt zugewiesen werden.

Hinzufügen von betroffenen Containern zu Ereignissen

Die Anwendung kann Container automatisch zu Qualitätsereignissen hinzufügen, wenn Sie die Transaktion "Container aufteilen" für einen Container durchführen, der mit mindestens einem Qualitätsereignis verknüpft ist. Wenn z. B. ein vorhandener Kind-Container mit einem Ereignis verknüpft ist, fügt die Anwendung den neuen Eltern-Container und alle gleichgeordneten Container automatisch zum Qualitätsereignis hinzu. Ein Systemadministrator konfiguriert die Transaktion "Aufteilen" in der Modellierung, um Container zu Ereignissen hinzuzufügen. Informationen zum automatischen Hinzufügen von betroffenen Containern zu Qualitätsereignissen finden Sie im Opcenter Execution Medical Device and Diagnostics Event Management-Benutzerhandbuch und im Opcenter Execution Core Event Management-Benutzerhandbuch sowie im Opcenter Execution Medical Device and Diagnostics Nonconformance Management-Benutzerhandbuch und im Opcenter Execution Core Nonconformance Management-Benutzerhandbuch.

Seite "Container aufteilen"

Die Seite "Container aufteilen" ermöglicht es Ihnen, einen Eltern-Container auszuwählen, dessen Kind-Container anzuzeigen und dann einen oder mehrere Kind-Container an einen anderen Eltern-Container zu übertragen.

Optional

Vorgehensweise zum Aufteilen eines Containers

Gehen Sie folgendermaßen vor, um einen Container aufzuteilen:

1. Öffnen Sie die Seite Container aufteilen im Menü Container. Die Seite "Container aufteilen" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Container aufteilen" zugreifen.
3. Aktivieren Sie das Kontrollkästchen neben jedem Kind-Container, den Sie in einen anderen Eltern-Container verschieben möchten.
4. Geben Sie den Namen des neuen Eltern-Containers in den die ausgewählten Kind-Container verschoben werden sollen im Feld Zu Container-Name ein.

Oder

Aktivieren Sie das Kontrollkästchen Namen automatisch erstellen.

5. Klicken Sie auf Teilen oder in der Befehlsleiste auf Abschicken. Die ausgewählten Kind-Container werden in den angegebenen Eltern-Container verschoben, die Anwendung löscht die Felder und zeigt dann eine Bestätigungsmeldung an.
