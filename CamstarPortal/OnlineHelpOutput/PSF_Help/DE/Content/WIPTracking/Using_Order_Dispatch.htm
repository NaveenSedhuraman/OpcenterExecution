Verwenden der Auftragserledigung

Die Seite "Auftragserledigung" zeigt eine Liste der Produktionsaufträge auf Grundlage ihrer Priorität an. Manufacturing verwendet die Abfragedefinition, die der Erledigungsregel zugeordnet ist (falls eine erstellt wurde), um die Tabelle auszufüllen. Die Abfrage bestimmt die Sortierreihenfolge (Priorität) der Elemente in der Tabelle und die anzuzeigenden Spalten.

Diese Bedingungen bestimmen, wie Produktionsaufträge sortiert, angezeigt und bearbeitet werden.

Wenn Sie die Mengen der Auftrags in mehrere Container unterteilen müssen, erstellen Sie so viele Container wie benötigt, bis die Menge des Auftrags erfüllt ist. Ein Auftrag wird aus der Tabelle entfernt, wenn er erfüllt wurde.

Nur Produktionsaufträge, die mit einer Menge definiert wurden, werden in der Tabelle "Produktionsaufträge" angezeigt. Die Tabelle "Produktionsaufträge" wird nach der Fabrik der Mitarbeiterinstanz gefiltert. Aufträge, die mit der Fabrik der Mitarbeiterinstanz modelliert wurden, werden in der Tabelle "Produktionsaufträge" angezeigt. Aufträge ohne definierte Fabriken werden ebenfalls angezeigt.

Aktualisieren der Tabelle "Produktionsaufträge"

Sie müssen die Tabelle "Produktionsaufträge" aktualisieren, um aktualisierte Details für geänderte Produktionsaufträge sowie kürzlich hinzugefügte Produktionsaufträge anzuzeigen. Das Aktualisieren der Tabelle ist besonders wichtig, wenn jemand anderes Produktionsaufträge von einem anderen Standort aus eingibt.

Zuweisen einer UDI zur Seite "Auftragserledigung"
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Medical Device installiert ist. Weitere Informationen finden Sie unter "Arbeitsbereich "Medical Device"".

Sie können eine Gerätekennung, eine Produktionskennung und eine eindeutige Gerätekennung einem Container direkt auf der Seite "Auftragserledigung" zuweisen. Dies ist nützlich, wenn eine UDI von einer externen Quelle eingescannt wird. Die Produktionskennung und die eindeutige Gerätekennung können direkt eingegeben werden, während die Gerätekennung ausgewählt wird.

Die Anwendung füllt das Feld "Gerätekennung" automatisch aus, wenn für das Produkt nur eine Gerätekennung angegeben ist. Wenn mehr als eine Gerätekennung für das Produkt angegeben ist, wird eine Liste angezeigt.

Beim Verwenden von Opcenter Execution Core Scheduling mit Auftragserledigung
Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling installiert ist. Weitere Informationen finden Sie unter "Opcenter Execution Core Scheduling".

Sie müssen das Modellierungsobjekt "Produktionsauftrag" verwenden, um einen Produktionsauftrag zu definieren, der in der Tabelle "Produktionsaufträge" der Seite "Auftragserledigung" angezeigt werden soll. Opcenter Execution Core Scheduling zeigt die Ressourceninformationen für den Produktionsauftrag im Feld "Ressource" der Tabelle "Produktionsaufträge" an. Informationen zum Definieren von Produktionsaufträgen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Vorgehensweise zum Starten eines Containers für einen ausgewählten Produktionsauftrag

Gehen Sie folgendermaßen vor, um einen Container für einen ausgewählten Produktionsauftrag zu starten:

1. Öffnen Sie die Seite Auftragserledigung im Menü Container. Die Seite "Auftragserledigung" wird mit einer Liste aktueller Produktionsaufträge in der Tabelle angezeigt.
2. Wählen Sie den zu bearbeitenden Produktionsauftrag. Die Felder "Produktionsauftrag", "Produkt" und "Produktbeschreibung" unterhalb der Tabelle enthalten standardmäßig die Informationen, die dem ausgewählten Produktionsauftrag zugeordnet sind.
3. Klicken Sie auf das Icon Dokument an diese Auftragsinstanz anhängen, um ein Dokument an einen Produktionsauftrag anzuhängen. Das Popup "Dokument anhängen" wird angezeigt.
4. Ziehen Sie die entsprechenden Schritte zum Anhängen eines Dokuments hinzu, wenn Sie ein Dokument an einen Produktionsauftrag anhängen möchten:
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das nicht wiederverwendet wird, an einen Produktionsauftrag"
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das wiederverwendet wird, an einen Produktionsauftrag"
• "Vorgehensweise zum Anhängen eines existierenden Dokuments an einen Produktionsauftrag"
5. Geben Sie den Namen des zu startenden Containers in das Feld Container ein.

Oder

Klicken Sie auf das Kontrollkästchen Namen automatisch erstellen.

6. Wählen Sie das Level aus.
7. Wählen Sie den Eigentümer aus.
8. Wählen Sie den Grund zum Starten dieses Containers im Feld Startgrund aus.
9. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
10. Klicken Sie auf das Steuerelement "Dokument anhängen", um die Containeranhang-Felder anzuzeigen, wenn der Bereich nicht erweitert ist.
11. Sehen Sie bei der entsprechenden Schritten zum Anhängen eines Dokuments nach, wenn Sie ein Dokument an einen neuen Container anhängen möchten:
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das nicht wiederverwendet wird, an einen neuen Container"
• "Vorgehensweise zum Anhängen eines neuen Dokuments, das wiederverwendet wird, an einen neuen Container"
• "Vorgehensweise zum Anhängen eines existierenden Dokuments an einen neuen Container"
12. Klicken Sie auf Start oder in der Befehlsleiste auf Abschicken. Die Anwendung zeigt eine Statusmeldung an, die den Container bestätigt und die Spalte "Menge in Bearbeitung" mit den Mengen in Bearbeitung aktualisiert. Wenn die Menge des Auftrags erfüllt wurde, entfernt die Anwendung den Produktionsauftrag aus der Liste.
13. Wiederholen Sie Schritte 5 bis 8, bis die Menge erfüllt wurde, falls Sie zusätzliche Container zu einem Produktionsauftrag starten müssen, um die Menge zu erfüllen. Sobald dies getan wurde, wird der Produktionsauftrag aus der Liste entfernt.

Vorgehensweise zum Anhängen eines neuen Dokuments, das nicht wiederverwendet wird, an einen Produktionsauftrag

Gehen Sie folgendermaßen vor, um ein neues Dokument, das nicht wiederverwendet wird, an einen Produktionsauftrag anzuhängen:

1. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten eines Containers für einen ausgewählten Produktionsauftrag".
2. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
3. Wählen Sie Neues Dokument anhängen, das NICHT wiederverwendet wird (keine Revision benötigt) aus. Die Felder "Dokumentpfad", "Dokumentname" und "Dokumentbeschreibung" werden angezeigt.
4. Klicken Sie auf die Schaltfläche Datei hochladen, um eine Datei auszuwählen.
5. Geben Sie einen Namen für das Dokument in das Feld Dokumentname ein.
6. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
7. Klicken Sie auf Anhängen oder in der Befehlsleiste auf Abschicken. Die Anwendung meldet, dass das Dokument erfolgreich angehängt wurde.

Vorgehensweise zum Anhängen eines neuen Dokuments, das wiederverwendet wird, an einen Produktionsauftrag

Gehen Sie folgendermaßen vor, um ein neues Dokument, das wiederverwendet wird, an einen Produktionsauftrag anzuhängen:

1. Befolgen Sie die Schritte unter "Vorgehensweise zum Starten eines Containers für einen ausgewählten Produktionsauftrag".
2. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
3. Wählen Sie Neues Dokument anhängen, das wiederverwendet wird (Revision muss angegeben werden) aus. Die Felder "Dokumentpfad", "Dokumentname", "Revision", "Standardrevision" und "Dokumentbeschreibung" werden angezeigt.
4. Klicken Sie auf die Schaltfläche Datei hochladen, um eine Datei auszuwählen.
5. Geben Sie einen Namen für das Dokument im Feld Dokumentname ein.
6. Geben Sie die Revision dieses Dokuments in das Feld Revision ein.
7. Aktivieren Sie optional das Kontrollkästchen neben der Revision, um anzugeben, dass dieses Dokument die Standardrevision ist.
8. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
9. Klicken Sie auf Anhängen oder in der Befehlsleiste auf Abschicken. Die Anwendung meldet, dass das Dokument erfolgreich angehängt wurde.
