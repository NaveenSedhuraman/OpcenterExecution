Datensammlung

Die Datensammlung wird für Qualitätsentscheidungen in Ihrer Fabrik verwendet, einschließlich Kategorisierung, Klassifikation, Fehler und technischer Analyse. Datensammlung ist ein integraler Bestandteil der Fertigungstransaktionen und zeigt die Flexibilität und Reaktionsfähigkeit der Anwendung gegenüber den Laufzeit-Anforderungen Ihrer Fabrik.

Siemens bietet zwei Arten von Container-Datensammlungen:

• Parametrische Datensammlung
• Benutzerbasierte Datensammlung

Die Daten, die Sie sammeln, einschließlich Kategorie, Klassifizierung, Fehler, Ressource, technische Analyse und anderen Daten, basiert auf der aktuellen Spez. des Prozessschrittes.

Vor dem Sammeln von Daten

Ein System-Administrator muss folgende Aufgaben ausführen, bevor Sie die Datensammlung-Seite verwenden können:

• Definieren von mindestens einer Datensammlung-Definition oder Benutzer-Datensammlung-Definition in der Modellierung.
• Zuweisen von einer oder mehreren Datensammlungs- oder Benutzerdatensammlungsdefinitionen zu einer Spezifikation in der Modellierung.

Datensammlung-Definitionen beziehen sich auf ein Formular zur parametrischen Datenkontrolle, das von einem Designer erstellt wurde. Benutzerdatensammlungs-Definitionen beziehen sich auf Datenpunkte, welche Punkte in einem Geschäftsprozess sind, an denen Daten gesammelt werden können oder müssen. Die Anforderungen für parametrische und Benutzer-basierte Daten werden von Ihrem Unternehmen definiert, und Ihr Systemadministrator fügt auf Grundlage dieser Anforderungen spezifische Felder hinzu.

Informationen zum Definieren einer parametrischen Datenkontrolle finden Sie im Opcenter Execution Medical Device and Diagnostics Data Collection-Benutzerhandbuch und im Opcenter Execution Core Data Collection-Benutzerhandbuch. Informationen zum Definieren von Datensammlungsdefinitionen und Benutzerdatensammlungsdefinitionen und deren Zuweisung zu Spezifikationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Beim Sammeln von Containerdaten

Sie können sowohl parametrische als auch Benutzerdaten auf der Seite "Daten sammeln" erheben, wenn sich der Container an einer Spezifikation befindet, der Daten- oder Benutzerdaten-Datensammlungsdefinitionen zugewiesen wurden. Darüber hinaus können Sie Daten über die Seite "Automatischer Ablauf" sammeln, wenn eine Datensammlung Teil der Aufgabenliste des automatischen Ablaufs ist. Für weitere Information über den automatischen Ablauf siehe "Automatischer Ablauf".

Wenn Sie die Seite "Daten sammeln" anzeigen, zeigt Ihnen die Anwendung standardmäßig Datensammlungsfelder an, wenn nur eine Daten- oder Benutzerdatensammlung mit der aktuellen Spez. des angegebenen Containers verbunden ist. Wenn mehr als eine Daten- oder Benutzerdatensammlung-Definition mit einer Spezifikation verknüpft wurde, zeigt die Anwendung das Datensammlung-Definitions-Feld an und ermöglicht Ihnen so, eine Datensammlung auszuwählen.

Wenn Grenzen für benutzerdefinierte Datensammlungen definiert wurden

Die Anwendung kann Ober- und Untergrenzen für Datenpunkte anzeigen, in Abhängigkeit davon, wie die Benutzerdatensammlung definiert wurde. Wenn der Wert außerhalb einer definierten Grenze liegt, markiert die Anwendung die Zelle und die verletzte Grenze entweder in gelb oder in rot. Die Farbe hängt davon ab, wie die Datensammlung definiert wurde.

Wenn die Anwendung die Zelle und den verletzten Grenzwert gelb markiert, dann wurde der Datenpunkt so eingestellt, dass er Überschreitungen erlaubt. Wenn die Anwendung die Zelle und den verletzten Grenzwert rot markiert, dann wurde der Datenpunkt so eingestellt, dass er keine Überschreitungen erlaubt. Die Anwendung ermöglicht es Ihnen, eine Datensammlung-Transaktion unabhängig von den Einstellungen zur Überschreitung von Grenzwerten auszuführen. Die Einstellungen zur Grenzwertüberschreitung bestimmen, ob eine Aufgabe zur Datensammlung in einem automatischen Ablauf bestanden wird oder nicht, sofern die Datensammlung-Transaktion mit einem automatischen Ablauf verknüpft wurde.

Hinweis: Die Anwendung zeigt negative Grenzen in Klammern an.

Seite "Daten sammeln"

Sie können über das Container-Menü auf die Seite "Datensammlung" zugreifen. Die Datensammlung-Seite ermöglicht es Ihnen, parametrische und Benutzerdaten für einen Container zu sammeln.

Hinweis: Die Seite "Datensammlung" ist eventuell auch als Popup verfügbar, wenn die Transaktion "Daten sammeln" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (z. B. "Containersuche").

Felddefinitionen der Seite und des Popups "Daten sammeln"

Diese Tabelle definiert die Felder auf der Seite und im Popup "Datensammlung".

Feld

Definition

Typ

Containerinformationen

Kopf mit dem Containernamen und Informationen zu diesem Container. Sie müssen einen Container auf einer Transaktionsseite auswählen oder eingeben. Alle Informationen in einem Transaktionen-Popup sind schreibgeschützt. Weitere Informationen finden Sie unter "Containerinformationen".

Datensammlungsdefinition

Die Daten- oder Benutzerdatensammlung für den ausgewählten Container. Die Anwendung aktualisiert die Liste, um die Datensammlung-Felder für die ausgewählte Datensammlung-Definition anzuzeigen.

Hinweis: Dieses Feld wird angezeigt, wenn mehrere Datensammlungen mit der aktuellen Spez. eines bestimmten Containers verknüpft wurden. Wenn nur eine Datensammlung verknüpft wurde, zeigt die Anwendung diese Felder standardmäßig an und zeigt das Datensammlung-Definitionsfeld nicht an.

Erforderlich

<Datensammlungsfelder>

Informationen, die in diesem Bereich angezeigt werden, hängen von der im Feld "Datensammlungsdefinition" ausgewählten Definition oder einer einzelnen Datensammlung ab, die mit der aktuellen Spezifikation eines bestimmten Containers verknüpft ist. Ob die Datensammlung-Felder erforderlich sind und ob die Grenzen angezeigt werden, hängt davon ab, wie die Datensammlung- oder Benutzerdatensammlung-Definition in der Modellierung konfiguriert wurde.

Erforderlich/ Optional

Zusätzliche Felder anzeigen/ausblenden

Optionale Felder sind standardmäßig ausgeblendet, wenn die Anwendung eine Seite oder ein Popup anzeigt. Durch Klicken auf die Steuerung "Anzeigen" werden die Felder angezeigt. Durch Klicken auf die Steuerung "Verstecken" werden die Felder ausgeblendet.

  

Ressource

Mit dem Container verknüpfte Ressource Die Anwendung zeigt ein Symbol an, das die Verfügbarkeit der ausgewählten Ressource anzeigt. Ressourcen werden in der Modellierung definiert.

Optional

Kommentare

Zusätzliche Informationen oder Anweisungen für diese Transaktion oder für diesen Container. Das Feld akzeptiert maximal 255 Zeichen. Wenn die Kommentare 255 Zeichen überschreiten, verkürzt die Anwendung die Daten.

Optional

Vorgehensweise zum Sammeln von Daten für einen Container

Gehen Sie folgendermaßen vor, um Daten über einen Container zu sammeln:

1. Öffnen Sie auf Daten sammeln im Container-Menü. Die Seite "Daten sammeln" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Daten sammeln" zugreifen.
3. Wählen Sie eine Daten- oder Benutzerdatensammlung-Definition aus dem Datensammlung-Definition-Feld aus, wenn notwendig. Die Datensammlungsfelder werden angezeigt.
4. Füllen Sie die Datensammlungs-Felder aus.
5. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
6. Klicken Sie auf Daten sammeln oder in der Befehlsleiste auf Abschicken. Eine Statusmeldung wird angezeigt, die besagt, dass die Daten erfasst wurden.
