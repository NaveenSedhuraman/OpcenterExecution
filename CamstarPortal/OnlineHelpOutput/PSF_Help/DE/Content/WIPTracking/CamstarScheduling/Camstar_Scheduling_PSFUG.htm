Opcenter Execution Core Scheduling

Hinweis: Die nachstehend beschriebene Funktionalität ist nur verfügbar, wenn Opcenter Execution Core Scheduling (Opcenter EX CR Scheduling) installiert ist. Weitere Informationen hierzu finden Sie im Opcenter Execution Core Scheduling-Benutzerhandbuch.

Opcenter Execution Core Scheduling bietet einen Mechanismus zum Planen von Produktionsaufträgen mit der Anwendung "Preactor Advance Scheduling".

Opcenter Execution Core Scheduling ermöglicht Organisationen die Definition von ERP-Routing-Daten (Enterprise Resource Planning) für Produkte und Produktionsaufträge in Opcenter Execution Medical Device and Diagnostics oder Opcenter Execution Core. Anschließend können diese Daten in der Anwendung "Siemens Preactor Advanced Planning and Scheduling (Preactor)" verwendet werden, um Produktionspläne für Produktionsaufträge zu erstellen. Die Informationen zu den geplanten Aufträgen werden an eine Zwischenspeichertabelle gesendet. Von dort aus werden sie dann zurück an Opcenter Execution Core übertragen, wenn der Planer den Zeitplan in Preactor freigibt. Sie können einen Opcenter CN MOM-Datenbankadapter so definieren, dass eine gespeicherte Prozedur aufgerufen wird, um die geplanten Auftragsinformationen an Opcenter Execution Medical Device and Diagnostics oder Opcenter Execution Core in Form einer Erledigungsliste zurückzugeben. Informationen zum Definieren eines CIO-Datenbankadapters finden Sie im Opcenter Connect MOM-Benutzerhandbuch.

Preactor verwendet die freigegebenen Auftragsinformationen von Opcenter Execution Medical Device and Diagnostics oder Opcenter Execution Core, um einen optimierten Fertigungszeitplan zu erstellen. Der optimale Zeitplan wird von Preactor zurückgegeben und verwendet, um die Erledigungslisten für die Seiten "Auftragserledigung" und "Operative Ansicht" zu erstellen. Weitere Informationen hierzu finden Sie unter "Beim Verwenden von Opcenter Execution Core Scheduling mit Auftragserledigung".

Scheduling umfasst Funktionen für eine Was-wäre-wenn-Planung, um zukünftige oder Was-wäre-wenn-Zeitplanungsszenarien basierend auf einem geplanten Produktmix zu analysieren. In diesen Situationen müssen dennoch einige oder alle Produktionsaufträge erstellt werden. Die Anwendung sendet Produkte und die zugehörigen ERP-Routen von Opcenter Execution Medical Device and Diagnostics oder Opcenter Execution Core an Preactor. Anhand dieser Daten kann der Benutzer hypothetische Aufträge in Preactor erstellen, um Was-wäre-wenn-Zeitplanungsszenarien zu analysieren.
