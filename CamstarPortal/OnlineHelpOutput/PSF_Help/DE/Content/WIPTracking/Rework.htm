Container nacharbeiten

Die Transaktion "Nacharbeit" gibt an, dass der Prozess an einem Schritt beendet ist, aber die beabsichtigten Ergebnisse nicht erreicht wurden. Die Transaktion "Nacharbeit" erlaubt es Ihnen, einen Container vom aktuellen Arbeitsplan-Schritt zu einem anderen Arbeitsplan-Schritt abzumelden, sodass bestimmte Abläufe wiederholt und zusätzliche Arbeit erledigt werden kann. Nacharbeit ist eine Art Bewegungstransaktion, die zusätzliche Funktionen hat.

Beim Verwenden von "Nacharbeit"

Alle Anzeigefelder auf der Seite "Nacharbeit" sind leer, bis ein Container ausgewählt wird. Anzeigefelder werden im Popup "Nacharbeit" automatisch gefüllt. Informationen über das Ausführen einer Transaktion über eine Seite oder ein Popup finden Sie unter "Verwenden von Transaktionsseiten und Popups".

Die Möglichkeit, einen Container nachzuarbeiten, hängt von der Operation ab. Nacharbeitsgründe müssen definiert und einer Operation zugeordnet werden (in der Modellierung), bevor sie mit der Transaktion "Nacharbeit" verwendet werden können. Ein Nacharbeitsgrund erklärt, warum ein Container nachbearbeitet wird, und kann auch darauf hinweisen, wohin der Container nach der Nacharbeit geht. Sie können die zusätzliche Arbeit, die im Zusammenhang mit einem bestimmten Problem verrichtet wurde, aus der Buchhaltungsperspektive ermitteln. Weitere Informationen zu Nacharbeitsgründen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch und im Opcenter Execution Core Modeling-Benutzerhandbuch.

Wichtig: Die Anwendung verhindert, dass Sie die Nacharbeits-Transaktion ausführen, wenn Sie in das Feld "Nacharbeitsgrund" einen Nacharbeitsgrund eingegeben haben, der nicht der aktuellen Operation des Containers zugeordnet ist. Die Anwendung zeigt eine Fehlermeldung an, wenn Sie einen ungültigen Nacharbeitsgrund angegeben haben und dann auf die Schaltfläche "Nacharbeit" klicken.

Die Transaktion "Nacharbeit" steht nur für Container an Schritten (Spezifikationen) zur Verfügung, für die ein Nacharbeitspfad im Arbeitsplan definiert wurde. Wenn dem Produkt ein Arbeitsplan zugeordnet ist, dem kein Nacharbeitspfad zugeordnet ist, werden keine Pfade im Feld "Nacharbeitspfad" angezeigt.

Beim Verwenden des Popups "Nacharbeit"

Die Anwendung zeigt einen Standardwert im Feld "Ressource" an, wenn eine dieser Bedingungen zutrifft:

• Für den aktuellen Schritt des Containers war ein Anmelden erforderlich, und der Benutzer, der den Container angemeldet hat, hat eine Ressource angegeben.

Oder

• Eine Ressource wurde angegeben, als der Container gestartet wurde, und die Nacharbeit ist die erste Transaktion (mit Angabe einer Ressource), die Sie nach dem Start des Containers ausführen.

Sie können, falls erforderlich, eine andere Ressource auswählen.

Seite "Nacharbeit"

Sie können über das Menü "Container" auf die Seite "Nacharbeit" zugreifen. Die Seite "Nacharbeit" erlaubt es Ihnen, einen Container zur Nacharbeit auszuwählen und einen Grund für die Nacharbeit zuzuweisen. Sie bietet auch die Möglichkeit, einen Nacharbeitspfad zuzuweisen.

Hinweis: Die Seite "Nacharbeit" ist eventuell auch als Popup verfügbar, wenn die Transaktion "Container anhalten" als verfügbare Aktion auf einer anderen Seite konfiguriert wurde (z. B. "Containersuche").

Felddefinitionen der Seite und des Popups "Nacharbeit"

Diese Tabelle definiert die Felder auf der Seite und im Popup "Nacharbeit".

Feld

Definition

Typ

Containerinformationen

Kopf mit dem Containernamen und Informationen zu diesem Container. Sie müssen einen Container auf einer Transaktionsseite auswählen oder eingeben. Alle Informationen in einem Transaktionen-Popup sind schreibgeschützt. Weitere Informationen finden Sie unter "Containerinformationen".

Erforderlich/ Nur Anzeige

Ressource

Ressource, die den Container im aktuellen Schritt bearbeitet hat. Sie können eine andere Ressource auswählen, wenn die Anwendung bereits standardmäßig eine angezeigt. Wenn Sie die Ressource in diesem Schritt angeben, bevor Sie den Container abmelden, sorgt das für Nachverfolgbarkeit.

Welche Ressourcenwerte zur Auswahl stehen, hängt von der Ressourcengruppe ab, die dem Schritt zugewiesen wurde (Spez.). Wenn der Spezifikation keine Ressourcengruppe zugewiesen wurde, stehen alle definierten Ressourcen zur Auswahl.

Optional

Nacharbeitsgrund

Der Grund, warum der Container nachbearbeitet werden muss. Der Grund wird mit einem Container verknüpft, sobald dieser in einen Nacharbeitsprozess kommt und mit allen Durchsatz-Transaktionen, die vom Container im Nacharbeitsprozess erzeugt werden.

Erforderlich

Zusätzliche Felder anzeigen/ausblenden

Optionale Felder sind standardmäßig ausgeblendet, wenn die Anwendung eine Seite oder ein Popup anzeigt. Durch Klicken auf die Steuerung "Anzeigen" werden die Felder angezeigt. Durch Klicken auf die Steuerung "Verstecken" werden die Felder ausgeblendet.

  

Pfad

Nacharbeitspfad zum Nacharbeitsschritt im Arbeitsplan. Wenn Sie dieses Feld leer lassen, verwendet die Anwendung den standardmäßigen Nacharbeitspfad.

Hinweis: Dieses Feld ist standardmäßig ausgeblendet.

Optional

Kommentare

Zusätzliche Informationen oder Anweisungen für diese Transaktion oder für diesen Container. Das Feld akzeptiert maximal 255 Zeichen. Wenn die Kommentare 255 Zeichen überschreiten, verkürzt die Anwendung die Daten.

Optional

Vorgehensweise zum Abmelden eines Containers zum Nacharbeitsschritt

Gehen Sie folgendermaßen vor, um einen Container zum Nacharbeitsschritt abzumelden:

1. Öffnen Sie die Seite Nacharbeit im Menü Container. Die Seite "Nacharbeit" wird angezeigt.
2. Wählen Sie einen Container im Container-Feld aus oder geben Sie ihn ein. Die Anwendung aktualisiert den Container-Kopf und die Auswahlfelder, sodass diese Informationen für den angegebenen Container angezeigt werden. Werte werden nur angezeigt, wenn sie für den Container eingegeben wurden.
Hinweis: Die Anwendung füllt das Feld "Container" und den Container-Kopf automatisch, wenn Sie von einer anderen Fertigungsseite auf das Popup "Nacharbeit" zugreifen.
3. Wählen Sie einen Nacharbeitsgrund.
4. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
5. Klicken Sie auf die Schaltfläche Nacharbeit oder in der Befehlsleiste auf Abschicken, um den Container zum angegebenen Schritt zu verschieben. Eine Statusmeldung wird angezeigt, die bestätigt, dass der Container abgemeldet wurde.
