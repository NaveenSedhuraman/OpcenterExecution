Opcenter Execution Core Scheduling

注释: 仅当安装有 Opcenter Execution Core Scheduling (Opcenter EX CR Scheduling) 时，下述功能才可用。有关信息，请参考《  Opcenter Execution Core Scheduling 用户指南》。

Opcenter Execution Core Scheduling 提供了使用 Preactor Advance Scheduling 应用程序计划制造订单的机制。

Opcenter Execution Core Scheduling 使组织能够在 Opcenter Execution Medical Device and Diagnostics 或 Opcenter Execution Core 中为产品和制造订单定义企业资源规划 (ERP) 路线数据，然后在 Siemens Preactor Advanced Planning and Scheduling (Preactor) 应用程序中使用这些数据创建制造订单的生产计划。当计划人员从 Preactor 发布计划时，有关计划订单的信息会发送到要传回 Opcenter Execution Core 的暂存表。可以定义一个 Opcenter CN MOM 数据库适配器以调用存储程序，将计划的订单信息以调度列表的形式返回到 Opcenter Execution Medical Device and Diagnostics 或 Opcenter Execution Core。有关定义 CIO 数据库适配器的信息，请参考《  Opcenter Connect MOM 用户指南》。

Preactor 使用从 Opcenter Execution Medical Device and Diagnostics 或 Opcenter Execution Core 发布的订单信息创建优化的制造计划。优化计划从 Preactor 返回，用于为“订单调度”和“作业视图”页生成调度列表。有关信息，请参考“将 Opcenter Execution Core Scheduling 与订单调度一起使用”。

Scheduling 提供“假设”计划功能以分析将来的情况，或根据计划产品组合提供“假设”计划方案。在这些情况下仍需要创建部分或全部制造订单。应用程序将把产品及其关联的 ERP 路线从 Opcenter Execution Medical Device and Diagnostics 或 Opcenter Execution Core 发送到 Preactor。通过此数据，用户可以在 Preactor 中创建假想订单以分析“假设”计划方案。
