数据收集

数据收集用于工厂高质量决策，包括分类、分选、失败及工程分析。数据收集是车间事务的一个组成部分，显示应用程序对工厂运行时需求的灵活性及响应度。

Siemens 提供了两种类型的容器数据收集方法：

• 基于参数的数据收集
• 基于用户的数据收集

收集的数据，包括分类、分选、失败、资源、工程分析及其他数据，基于处理工序的当前工艺。

收集数据之前

系统管理员必须先执行这些任务，然后您才能使用“收集数据”页：

• 在建模中至少定义一个数据收集定义或用户数据收集定义。
• 将一个或多个数据或用户数据收集定义指派给建模中的规格。

数据收集定义参考由 Designer 创建的参数数据控制形式。用户数据收集定义参考数据点，即业务流程中可以或必须收集数据的点。对基于参数和基于用户的数据的要求由业务定义，且系统管理员将根据这些要求添加特定的字段。

有关定义参数数据控制的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Data Collection 用户指南》或《  Opcenter Execution Core Data Collection 用户指南》。有关定义数据收集定义和用户数据收集定义并将其指派给规范的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Data Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。

收集容器数据

当容器位于指派了数据或用户数据收集定义的规格时，可以在“收集数据”页上收集基于参数的和基于用户的数据。此外，当数据收集任务是指派给电子程序的任务列表的一部分时，可以从“电子程序”页收集数据。有关电子程序的信息，请参考“电子流程”。

当查看“收集数据”页时，如果只有一个数据或用户数据收集与指定容器的当前规格相关联，则应用程序默认显示数据收集字段。如果多个数据或用户数据收集定义与规格关联，则应用程序将显示“数据收集定义”字段，支持选择数据收集。

为用户定义的数据收集定义限制

应用程序可能显示数据点的上限和下限，具体取决于用户数据收集的定义方式。如果值超出定义的限制范围，则应用程序将以黄色或红色突出显示对应的单元和违反的限制。颜色也取决于数据收集的定义方式。

如果应用程序以黄色突出显示单元格和违反的限制，则数据点被定义以允许替代。如果应用程序以红色突出显示单元格和违反的限制，则数据点被定义以禁止替代。无论限制替代设置如何，用户都可以在应用程序中执行数据收集事务。“限制替代”设置确定当数据收集事务与电子程序任务关联时，数据收集电子程序任务是通过还是失败。

注释: 应用程序将在括号中显示负限制。

收集数据页

可以从“容器”菜单访问“收集数据”页。“收集数据”页支持收集基于参数的和基于用户的容器数据。

注释: 如果收集数据事务已配置为另一页（如“容器搜索”）上的可用操作，则“收集数据”页也可作为弹出窗口显示。

如何收集容器上的数据

按照以下步骤收集容器上的数据：

1. 从容器菜单打开收集数据页。将显示“收集数据”页。
2. 在容器字段中选择或输入容器。 应用程序将更新容器标头和选择字段以显示指定容器的信息。 只有为容器输入了这些值，才会显示相应的值。
注释: 当从另一车间页访问“收集数据”弹出窗口时，应用程序将自动填充“容器”字段和容器标头。
3. 如果需要，从数据收集定义字段中选择数据或用户数据收集定义。将显示数据收集字段。
4. 完成数据收集字段。
5. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
6. 单击收集数据或命令栏上的提交。将显示一条状态消息，确认已经收集数据。
