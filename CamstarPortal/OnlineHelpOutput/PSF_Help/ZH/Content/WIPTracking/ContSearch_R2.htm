搜索容器

Siemens 提供了一个搜索功能，支持根据指定的准则搜索和检索容器。可以使用提供的过滤器或用户定义的查询来搜索容器。Siemens 根据容器的当前状态和历史提供过滤器用于搜索容器。

当应用程序返回符合指定准则的容器时，可以选择要执行事务的一个或多个容器。命令栏包含与所选容器相关的事务按钮。

搜索容器

搜索容器时，可以使用多个参数来缩小搜索结果。应用程序提供以下搜索：

• 当前状态 - 可根据容器的当前状态来搜索容器，包括：
• 常规
• 工作流程
• 状态
• 历史
• 用户查询 - 可创建自定义查询来搜索容器

初次访问“容器搜索”页时，应用程序将显示两个选项卡和一个空的搜索结果表格。选项卡是当前状态和用户查询。

应用程序要求在启用“搜索”按钮之前要求至少指定一个参数（过滤器或用户查询）。在搜索选项卡上选择参数会导致应用程序：

• 将图标添加到选项卡中，指示已在该选项卡上选择了过滤器或用户查询。
• 启用“搜索”按钮。
注释: 必须在应用程序的“用户查询”选项卡上选择用户查询以启用“搜索”按钮。

使用生产线指派设置作为搜索过滤器

使用当前或历史准则搜索容器时，可以将生产线指派设置作为过滤器应用。选中“应用生产线指派”复选框即可将生产线指派设置作为过滤器应用。“应用生产线指派”复选框将出现在“状态”和“历史”区域的“当前状态”选项卡上。

生产线指派设置包括：

• 资源/工作单元
• 工作中心
• 作业
• 工作站

有关生产线指派设置的信息，请参考“生产线指派设置”。

示例：

“作业”可作为“工作流程”和“历史”区域中的独立过滤器使用。资源可作为“历史”区域中的独立过滤器使用。当“当前状态”过滤器值和生产线指派过滤器值不同时，搜索结果包括所有具有其中一种过滤器值的容器。

使用用户查询搜索容器

用户查询不能与任何其他搜索过滤器结合使用，且必须在“用户查询”选项卡中执行用户查询搜索。当从“用户查询”选项卡执行搜索时，应用程序将忽略在“当前状态”选项卡上选择的任何过滤器。

在执行用户查询搜索之前，必须在建模中定义一个或多个用户查询，以便在搜索容器时使用。查询必须在选择准则中包含容器 ID。否则，当从“用户查询”选项卡执行时，查询将不会返回任何结果。此外，必须为搜索输入所有参数值才能返回结果。

重要提示: 用户查询通常是特定于数据库的。

或者，您可能希望在建模中定义一个或多个用户查询组。用户查询组支持缩小“用户查询”选项卡上可供选择的用户查询范围。例如，如果为不同的业务需求定义了用户查询，则可以创建一个“容器搜索”用户查询组，以便只查看与容器搜索相关的用户查询。

选择用户查询会导致应用程序：

• 显示必须定义以执行搜索的参数。
• 显示查询说明（如果定义用户查询时提供了查询说明）。
• 启用“搜索”按钮。

如果在选择用户查询后选择了用户查询组，应用程序将清除“用户查询”字段和与该查询关联的所有数据。

有关用户查询的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。

示例查询

这是两个示例查询：

查询 1：

select ContainerId from Container c

join Product p on c.ProductId = p.ProductId

join ProductBase pb on p.ProductBaseId = pb.ProductBaseId

join CurrentStatus cs on c.CurrentStatusId = cs.CurrentStatusId

left join ResourceDef r on cs.ResourceId = r.ResourceId

Where pb.ProductName = ?ProductName -- 'Widget'

and r.ResourceName = ?ResourceName --'Workstation 1'

Order By ContainerName

查询 2：

select ContainerId from Container c

join Product p on c.ProductId = p.ProductId

join ProductBase pb on p.ProductBaseId = pb.ProductBaseId

<{?ResourceName}>?<{ join CurrentStatus cs on c.CurrentStatusId =

cs.CurrentStatusId

left join ResourceDef r on cs.ResourceId = r.ResourceId }>

Where pb.ProductName = ?ProductName

<{?ResourceName}>?<{ and r.ResourceName = ?ResourceName}>

Order By ContainerName

使用命令栏

必须在表格中选择一个或多个容器，才能显示对所选容器可用的命令栏操作。

选择单个容器

选择单个容器会导致应用程序显示可用于单个容器的命令栏操作，包括：

• 容器状态 - 所选容器的状态和属性。
• 工作流程 - 所选容器的工作流程。
• 附加文档 - 将新文档或现有文档附加到容器。
• 文档 - 与容器相关联的文档。可以单击文档来查看它。
• 制造审核追踪 - 单击时所选容器的制造审核追踪。

选择多个容器

选择多个容器时，应用程序显示可以对所有选定容器执行的命令栏操作。单击某命令栏按钮，可以同时对所有选定容器执行相关事务。如果所选容器没有共同事务，应用程序将显示消息“没有可用于所选容器的操作”。

此外，还可以从搜索结果表格中的多个页面中选择容器。应用程序将在表格下显示选择的容器的数量。

• 从所有选项卡中清除所有指定的搜索过滤器或查询。

• 从搜索结果表格中清除容器。
• 清除所有命令栏按钮。

如何为结果表格配置列

按照以下步骤配置结果表格的列：

1. 从“搜索”菜单打开“容器搜索”页。将显示“容器搜索”页。
2. 执行搜索。将显示搜索结果表格，并显示与搜索准则匹配的容器。
3. 单击“显示列”。应用程序将显示一个弹出窗口，其中包含可见列的列表和可选择显示的列的列表。
4. 使用下列方法之一选择要在表格中显示的列：
• 使用拖放在可见列和隐藏列列表之间移动列，并更改这些列表中列的顺序。
• 单击“全部显示”将所有列移动到“可见列”列表中。应用程序将“隐藏列”列表中的所有列移动到“可见列”列表中，并将它们置于“可见列”列表中已存在的任何列下面。
• 单击“全部隐藏”将所有列移动到“隐藏列”列表中。应用程序将“可见列”列表中的所有列移动到“隐藏列”列表中，并将这些列置于“隐藏列”列表中已存在的任何列下面。
5. 单击“应用”应用列配置并关闭“显示列”弹出窗口。应用程序将配置应用于搜索结果表格。

如何使用当前状态准则来搜索容器

按照以下步骤，使用当前状态准则来搜索容器：

1. 从“搜索”菜单打开“容器搜索”页。将显示“容器搜索”页。“当前状态”选项卡处于选中状态，并且“常规”部分在过滤器面板中展开。
2. 输入或选择一个或多个参数，以缩小在“过滤器”面板上的搜索范围。单击区域标题以展开和折叠区域。使用面板滚动条滚动浏览可用的区域。应用程序将图标添加到区域标题，并启用“搜索”按钮。
3. 单击搜索。应用程序将在表格中显示与搜索准则匹配的容器。

如何使用用户查询搜索容器

按照以下步骤，使用用户查询搜索容器：

1. 从“搜索”菜单打开“容器搜索”页。将显示“容器搜索”页。
2. 单击“用户查询”选项卡。将显示“用户查询”面板。
3. 选择用户查询组。
4. 选择用户查询。应用程序使用查询信息更新选项卡，向选项卡中添加图标，显示一条注释指示要返回的结果类型，并启用“搜索”按钮。
5. 在查询参数表格中输入所需的值。
6. 单击搜索。应用程序将在“搜索结果”表格中显示与搜索准则匹配的容器。

如何对所选容器执行事务

按照以下步骤，对所选容器执行事务：

1. 使用上述搜索程序之一执行搜索。
2. 选择要执行事务的容器。应用程序将显示所选容器可用操作的命令栏按钮。
3. 单击“车间事务”命令栏按钮。应用程序将显示一个面板，其中包含所选容器可用的所有车间事务。
4. 单击事务。应用程序显示所选事务的弹出窗口。
5. 完成弹出窗口并单击“事务执行”按钮。应用程序将关闭弹出窗口并显示成功消息。
