Adding a Web Part to a Page

Web parts can be added to a page as either a local copy or as a reference.

When Adding a Web Part to a Page

Controls and fields cannot be dragged directly onto a page. Controls and fields must be placed on a web part.

Referenced web parts are read-only copies that refer to a master copy of the web part. This allows you to maintain a web part in one place instead of maintaining several individual web parts.

In general, you should add a web part as a:

• Reference when the web part will be the same on all pages and maintained in a single place. Refer to "Viewing Referenced Web Part Locations" for information on viewing a list of components referencing a web part.
• Copy when the web part is a template or when you are using one web part to make a similar web part.

When testing a page with a reference web part, the Developer content of the reference web part will always be shown.

When a page with a reference web part is published, the Published content of the reference web part is always displayed.

How to Add a Web Part to a Page

Follow these steps to add a web part to a page:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a page. The page appears in the working area.
4. Expand Web Parts in the UI Components catalog.
5. Select a web part and drag it onto a cell. The web part appears in the cell.
6. Click Save. The application saves your changes and displays a success message.
