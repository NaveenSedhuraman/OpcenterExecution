Creating a Page

Portal Studio allows you to create pages that are used in the Portal.

When Creating a Page

Once you create a new page, web parts can then be dragged onto the page. Controls and fields cannot be dragged directly onto a page; they must be placed on a web part.

You can test a page after it is saved using the Run button. Refer to "Testing a Page" for information.

WIP Messaging is set to True in the Portal Settings by default. Since page settings override the global setting, you can disable WIP Messaging for a page using the WIPMessagesRequired field in the Properties pane.

Create New Page Pop-up Field Definitions

This table defines the fields on the Create New Page pop-up.

Definition

Page Name

Required

Primary Service Type

Optional

How to Create a Page

Follow these steps to create a page:

1. Access Portal Studio.
2. Click the plus sign icon on the New button. The Create New Page pop-up appears.
3. Select a page template.
4. Enter a name for the page in the Page Name field.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
6. Click OK. The Create New Page pop-up closes and the new page appears in the working area.
7. Click Save. The application saves your changes and displays a success message.
