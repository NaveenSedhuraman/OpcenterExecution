Testing a Page

Portal Studio allows you to test a new or existing page in a Portal window.

When Testing a Page

The Run button is used to test a page and is not enabled until the page is saved.

When you test a page, the page is opened in the Portal framework in a separate browser tab, as if it were accessed from a Portal menu.

Any data you enter on a page while testing can be saved to your environment's database.

How to Test a Page

Follow these steps to test a page:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a page. The page appears in the working area.
4. Click Save. The application saves your changes and displays a success message.
5. Click Run. The page is opened in the Portal in a separate browser tab.
