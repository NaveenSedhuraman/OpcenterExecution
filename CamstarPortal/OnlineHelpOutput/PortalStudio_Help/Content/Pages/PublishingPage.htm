Publishing a Page

Publishing a page makes your changes available outside of your workspace and in the Portal. Publishing your content also makes it available for other Portal Studio users.

When Publishing a Page

Pages cannot be published until they are saved. Unpublished pages are indicated by a pencil icon on the page's icon in the UI Components catalog.

The Publish button is only enabled in the highest active and Developer workspaces. Saved changes published in the Developer workspace are published to the highest active workspace. Refer to "Developing in Workspaces" for information.

You can revert a page to its last published state by clicking Revert To and selecting Published content. Refer to "Reverting to the Last Published State" for information.

The application displays a Publishing dialog prompting you to remove all personalization associated with the page you are publishing when you click Publish. Selecting the Remove All personalization associated with this component check box and clicking OK removes all personalization and continues the publishing process. Clicking OK without selecting the check box continues the publishing process. Clicking Cancel aborts the publishing process.

A page permission is automatically created and added to the Default Pages role when publishing a new page. A page permission is not created when publishing a page that has previously been published. You can set the AutoPagePermission property in the DefaultSettings of the Portal Settings window to False to prevent the application from adding new page permissions by default. Refer to "Configuring Portal Settings" for information.

Note: You must log out of and then log back in to the Portal before attempting to run a newly published page or view a page permission on the Default Pages role after publishing a new page. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on roles.

How to Publish a Page

Follow these steps to publish a page:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a page. The page appears in the working area.
4. Make updates to the page, if necessary.
5. Click Save. The application saves your changes and displays a success message.
6. Click Publish. The Publishing dialog appears.
7. Click OK. The Publishing dialog closes, the page is published to the highest active workspace, and the application displays a success message.
Note: The application also displays a message stating that the Default Pages role has been updated if the Default Pages role exists and is assigned to the current Portal Studio user.
