Running the Trace Log

Portal Studio allows you to run a Trace Log to aid in troubleshooting. The trace log captures a history of changes to Portal pages as you update and run them in Portal Studio. The trace log also allows you to capture server information when performing transactions on Portal pages.

Enabling the Trace Log

The trace log is disabled by default and you must enable it using the TraceEnabled setting in Portal Studio's default settings. You must then log out of and back into the Portal before attempting to run the trace log.

You must also set the permissions on the folder specified with the TracePath setting to ensure the application can access the folder for the logged in user. The default location for the trace path is C:\Temp.

The application displays an error message if the trace log has been enabled and the user attempting to log on does not have permission to access the trace path folder. You can correct the error by updating the TracePath manually in the user settings file. The user settings file is located at the following path by default: 

C:\Program Files (x86)\Camstar\Camstar Portal\User\Settings.xml

When Running the Trace Log

Clicking the Show/Hide Tracer Viewer button activates the trace log which appears beneath the Portal Studio workspace. Portal traces appear in the left pane and server traces appear in the right pane. You can turn off the trace log at any time by clicking the Show/Hide Tracer Viewer button again.

Portal traces are captured automatically upon running a page in Portal Studio. You must perform at least one transaction on the running Portal page in order to capture a server trace. Clicking a button on the page is an example of a transaction.

The trace log displays a timestamp for Portal traces and server traces. Timestamps appear at the top level of each action captured. The timestamp shows the year, month, day, hour, minute, second, and millisecond of when the action was captured. The time portion of the timestamp formats as HH:MM:SS:MSS and matches the timezone set in your browser. The date portion formats to match the local date format using the language set in your browser. For example, if your browser's language is set to U.S. English, the date portion of the timestamp formats as MM/DD/YYYY.

You can view additional information for a server trace on the Server Trace window by clicking the associated Server Trace button. The Server Trace window provides a simplified view of the server trace and allows you to view the trace as an XML file by clicking the Raw View button.

You can search the trace log for service calls, methods, and parameters by entering a keyword and clicking Search. Clicking Search highlights the first matching result. Clicking Search repeatedly highlights successive matches until you have viewed all matching results. Clicking Search after the last match displays a message stating no additional matches can be found.

You can disable the commit action when resubmitting a server trace on the Submit Transaction window by deselecting the Commit check box. The Commit checkbox is selected by default.

How to Enable the Trace Log

Follow these steps to enable the trace log:

1. Access Portal Studio.
2. Click the Portal Settings button.The Portal Settings pop-up appears.
3. Select True for TraceEnabled.
4. Click OK. The Portal Settings pop-up closes and the trace log is now enabled.
Note: You must log out of and back into the Portal before attempting to run the trace log.

How to Run the Trace Log

Follow these steps to run the trace log:

1. Access Portal Studio.
2. Click Show/Hide Trace View. The tracer window appears and begins running the trace log.
3. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
4. Double-click any Portal page. The page appears in the working area.
5. Make any necessary changes to the page.
6. Click Save. The application saves your changes and displays a success message.
7. Click Run. The Portal page opens in a new browser tab.
8. Return to the browser tab running Portal Studio and click the Refresh button in the tracer window. The trace log appears in the left tracer window.
9. Repeat steps 5-8 to continue gathering trace log data as needed.

How to Save the Trace Log

Follow these steps to save the trace log:

1. Perform the "How to Run the Trace Log" procedure.
2. Click the Save Trace button for the trace log you want to save. The File Download - Security Warning dialog appears.
3. Click OK. The File Download - Security Warning dialog closes and the Save As pop-up appears.
4. Navigate to a file location and click Save. The Save As pop-up closes and the trace log is saved.

How to Resubmit a Server Trace

Follow these steps to resubmit a server trace:

1. Access Portal Studio.
2. Click Show/Hide Trace View. The tracer window appears and begins running the trace log.
3. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
4. Double-click any Portal page. The page appears in the working area.
5. Make any necessary changes to the page.
6. Click Save. The application saves your changes and displays a success message.
7. Click Run. The Portal page opens in a new browser tab.
8. Perform a transaction on the Portal page.
9. Return to the browser tab running Portal Studio and click the Refresh button in the tracer window. The server trace appears in the right tracer pane.
10. Click the Server Trace button for the server trace you want to resubmit. The Server Trace window appears.
11. Click Resubmit. The Submit Transaction window appears.
12. Click Submit. The trace is resubmitted and the Server Trace window displays a success message.

How to Search the Server Trace Log

Follow these steps to search the server trace log:

1. Perform the "How to Run the Trace Log" procedure.
2. Click Server Trace on the server side of the tracer. The Server Trace window appears.
3. Type a keyword in the search field.
4. Click Search. The application highlights the first matching result.
5. Repeat step 4 to find additional matching results.
