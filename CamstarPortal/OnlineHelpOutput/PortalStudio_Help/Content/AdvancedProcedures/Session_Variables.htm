Defining Session Variables

You can create session variables for pages or web parts in Portal Studio. You can use session variables to collect data and provide the collected data to another component.

When Defining Session Variables

Session variables are declared in the SessionVariables pop-up which is accessed from the Properties pane when a page, web part, or pop-up is selected in the working area.

Session variables are globally available within the page they were defined. Session variables defined in a web part are stored within the page containing that web part.

Session variables retain data until you navigate away from the page. Reloading the page resets session variables to the value specified in the InitialValue field. Session variables have an initial value of "null" by default.

Session Variables are set when defining conditional event handlers. Refer to "Defining Event Handlers" for information on defining conditional event handlers.

SessionVariables Pop-up Field Definitions

This table defines the fields on the SessionVariables pop-up.

Field

Definition

Type

General

ID

Unique name for the session variable.

Required

InitialValue

Determines the starting value of the session variable when the page is loaded. The session variable is reset to this value each time the page is reloaded. Session variables have an initial value of "null" by default.

Optional

IsArray

Indicates whether the session variable is in an array.

Optional

Type

Determines the expected object type, such as "NamedObjectRef" or "System.String". Session variables have a type of "System.Object" by default.

Optional

How to Define Session Variables

Follow these steps to define session variables:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a page. The page appears in the working area.
4. Click Show/Hide Properties if the Properties pane is hidden. The Properties pane appears.
5. Select a page, web part, or pop-up in the working area. The properties for the selected object appear in the properties pane.
6. Click More for SessionVariables in the Properties pane. The SessionVariables pop-up appears.
7. Click Add. A new session variable is added to the list and its general properties appear in the right pane.
8. Enter a name for the session variable in the ID field.
9. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
10. Repeat steps 7-9 to add additional session variables.
11. Click OK. The SessionVariables pop-up closes.
12. Click Save. The application saves your changes and displays a success message.
