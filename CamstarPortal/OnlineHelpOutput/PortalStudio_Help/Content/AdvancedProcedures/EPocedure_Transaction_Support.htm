Enabling EProcedure Transactions

You can enable EProcedure transactions on a page by modifying the page's EprocEnabled property in Portal Studio.

When Enabling EProcedure Transactions

EProcedure transactions are enabled on certain transaction pages by default. The Transaction Page pick list shows a list of pages that currently have Eprocedure enabled.

Newly created pages are not enabled for Eprocedure transactions by default.

You can see if a page has been enabled for Eprocedure support by viewing the EProcEnabled flag in the page's Properties pane.

This list contains the pages capable of executing an EProcedure transaction by default. Additional pages not appearing in this list may require additional programming steps based on your custom functionality.

How to Enable EProcedure Transactions

Follow these steps to enable EProcedure transactions:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a page that supports EProcedure. The page appears in the working area.
4. Click Show/Hide Properties if the Properties pane is hidden. The Properties pane appears.
5. Select True for EProcEnabled in the Properties pane.
6. Click Save. The application saves your changes and displays a success message.
