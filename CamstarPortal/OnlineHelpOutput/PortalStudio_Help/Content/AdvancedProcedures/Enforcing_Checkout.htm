Enabling Enforced Check Out

You can enforce a check out system in Portal Studio that blocks multiple developers from saving changes to a page simultaneously. The check out system requires you check out a page before changes can be saved.

When Enabling Enforced Check Out

You can check in and check out pages in Portal Studio using the Check In and Check Out buttons located in the global toolbar. A page can only be checked out by one developer at a time.

Check out is not enforced by default and must be set in Portal Studio's DefaultSettings. Refer to "Configuring Portal Settings" for information on DefaultSettings' field definitions.

How to Enable Enforced Check Out

Follow these steps to enable enforced check out:

1. Access Portal Studio.
2. Click the Portal Settings button.The Portal Settings pop-up appears.
3. Select DefaultSettings from the list of settings. The properties for DefaultSettings appear.
4. Select True for EnforceCheckOutWhenEditing.
5. Click OK.
6. Click Save. The application saves your changes and displays a success message.
