Session Expiration

Portal Studio does not enforce session expiration due to timeout. Portal Studio sessions are tied to the Portal and will only expire when an instance of the Portal expires. Portal sessions expire after a default 20 minute inactivity period.

Note: You can change the default timeout value for the Portal in the web.config file located at C:\Program Files (x86)\Camstar\Camstar Portal by default.

When Session Expiration Occurs

Portal Studio does not display a session expiration warning message when session expiration occurs. Any unsaved changes will be lost and you must log back in to the Portal and access Portal Studio, when session expiration occurs. Refer to "Accessing Portal Studio" for information.
