Developing in Workspaces

Siemens provides workspaces for metadata to make development, deployment, and upgrading of partner offerings and customer customizations easier and more manageable.

You can change or customize the Portal without overwriting changes made by others by publishing developments to workspaces in Portal Studio. Publishing changes to a workspace isolates those changes from any updates made to any other workspace. For example, changes published to a customer workspace do not affect the base Siemens PLM CEP workspace. Similarly, if an update to the base Siemens PLM CEP workspace is accepted, customizations in other workspaces are not overwritten. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information.

When Developing in Workspaces

Portal Studio opens in the Developer workspace by default. You can view the list of active workspaces from the View Contents menu. The color that appears next to each workspace name is determined by the color specified for the workspace in Management Studio. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on managing workspaces.

Portal Studio has both Developer and the base Siemens PLM CEP workspace active by default. The Customer workspace is also active by default when a customer receives their metadata database following an install of release 8.0. Other workspaces must be made active in Management Studio before they will appear in the View Contents menu. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information.

Note: WCF services must be regenerated any time you activate or deactivate workspaces in Management Studio. Changes to available active workspaces will not appear in the View Contents menu until the WCF services have been regenerated.

Changes can be saved and published only in the highest active workspace or Developer workspace. Other active workspaces can be selected from the View Contents menu to view their content and develop changes, but the Save, Save All, and Publish buttons are disabled.

Published changes are workspace specific and will only appear in the workspace they are published to. When changes are saved and published in the Developer workspace, they are published to the highest active workspace. Switching to another workspace before saving will cause any unsaved changes to be lost in the current workspace.

Any content that is changed and saved appears with a red border around it by default, indicating it has been saved successfully. The border around the object appears in the color specified for the workspace in Management Studio if you change and save a control on a default virtual page in the workspace. The colored border indicates that the control has been overridden.

A workspace can be set as the highest active workspace in Management Studio. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information.

Any changes that are saved and published in the Developer workspace are applied to the base Siemens PLM CEP workspace if no other workspaces are active. The base Siemens PLM CEP workspace is always active.

Important: Changes published to the base Siemens PLM CEP workspace will be lost after an Opcenter Execution Core update. Siemens recommends that you develop, save, and publish changes in workspaces other than the base Siemens PLM CEP workspace.

When Using the ASPX Converter

Portal Studio allows you to automatically convert ASPX files to Portal pages and continue editing them as Portal pages within a workspace. Converted pages are applied to the highest active workspace. Refer to "Converting ASPX Files to Portal Pages" for information.

When Using the Trace Log

Portal Studio allows you to run a Trace Log to aid in troubleshooting. The Trace Log allows you to capture a history of changes to Portal pages as you update and run them in Portal Studio. Refer to "Running the Trace Log" for information.

How to Publish Content to an Active Workspace

Follow these steps to publish content to an active workspace:

1. Access Portal Studio.
2. Select the highest active workspace.

Or

Leave Developer as the selected workspace.

3. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
4. Double-click a page. The page appears in the working area.
5. Make any necessary changes.
6. Click Save. The application saves your changes and displays a success message.
7. Click Publish. The Publishing dialog appears.
8. Click OK. The Publishing dialog closes and the page is published to the highest active workspace.
