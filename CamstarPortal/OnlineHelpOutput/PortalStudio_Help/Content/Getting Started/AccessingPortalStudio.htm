Accessing Portal Studio

You access Portal Studio differently depending on the theme you used to access the Portal. Horizon is the default theme if you log in to the Portal using the Default mode. Classic is the default theme if you use the Classic entry mode. Refer to the Opcenter Medical Device and Diagnostics Getting Started User Guide or the Opcenter Execution Core Getting Started User Guide for information on entry modes.

Important: Microsoft Silverlight must be installed to access Portal Studio. Google Chrome has limited or no support for Silverlight beyond version 44. Refer to "Enabling Silverlight in Google Chrome" for information on accessing Portal Studio using Chrome.

When Accessing Portal Studio

You access Portal Studio from the Portal by clicking the:

• Portal Studio button on the Primary Navigation bar in the Portal if you are using the Horizon theme.
• Studio button in the global header of the Portal if you are using the Classic theme.

An error message appears preventing you from accessing Portal Studio if CSI is the highest active workspace. You must activate another workspace in Management Studio and then access Portal Studio. Refer to the Opcenter Execution Medical Device and Diagnostics System Administration Guide or the Opcenter Execution Core System Administration Guide for information on activating workspaces.

Note: The Portal does not display an error message if you are already logged into the Portal and CSI is then made the highest active workspace. An error message appears preventing you from accessing Portal Studio after you log out and then log back into the Portal.

A Start Page dashboard appears in the working area when accessing Portal Studio. The Start Page dashboard provides links to commonly used components and information on using Portal Studio.

The Start Page dashboard contains these sections:

• Create New, which provides links to create new components, such as pages, web parts, pageflows, etc.
• What's New, which provides a link to release notes describing newly added features in Portal Studio.
• Getting Started, which provides links to information on using Portal Studio.
• Recent, which provides links to open the five most recently opened pages, web parts, and pageflows.

The Start Page dashboard remembers the ten most recently opened pages, web parts, and pageflows, and will update the list as you open components. An error message appears if you click the link for a component in the Recent list that has been deleted since accessing Portal Studio. Closing the error message removes the component from the list and replaces it with another recently opened component. No components will appear in the Recent list the next time you access Portal Studio if all ten recently opened components are deleted and no other pages, web parts, or pageflows have been opened.

Microsoft Silverlight must be installed and cookies must be enabled in your browser to access Portal Studio. You must first enable Silverlight if you are accessing Portal Studio using versions 42, 43, or 44 of Chrome. Refer to "Enabling Silverlight in Google Chrome" for information on enabling Silverlight.

You can return to the Portal at any time during your Portal Studio session by clicking the Back to Portal button. Portal Studio will prompt you to save your changes prior to returning you to the Portal.

Portal Studio can also be installed locally, allowing you to access it without going through the Portal. Refer to "Installing Portal Studio Locally" for information.

Accessing Portal Studio 1.0

Portal Studio 2.0 is the default version of Portal Studio for Opcenter EX MDD and Opcenter EX CR. You can also access Portal Studio 1.0—the original version of Portal Studio. The PortalStudioVersion setting in the Portal web.config file controls the version available when accessing Portal Studio using Horizon or Classic themes. Select 1 for Portal Studio 1.0.

Note: Siemens recommends using the Microsoft Internet Explorer browser when accessing Portal Studio 1.0. Portal Studio 2.0 is the default version for the Chrome browser.

How to Access Portal Studio

Follow these steps to access Portal Studio:

1. Log in to the Portal.
2. Which theme are you using?

If using . . .

Then . . .

Horizon

Click the Portal Studio button on the Primary Navigation bar in the Portal. Portal Studio and the Start Page dashboard appear.

Classic

Click Studio in the global header in the Portal. Portal Studio and the Start Page dashboard appear.

How to Return to the Portal

Follow these steps to return to the Portal:

1. Access Portal Studio.
2. Perform any necessary work.
3. Click the Back to Portal button. Portal Studio closes and you are returned to the Portal.
