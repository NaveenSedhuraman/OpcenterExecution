Adding a Field to a Web Part

Fields are used to enter information into a web part. For example, you add a Container field to the web part to allow a user to enter a container.

You can add a field to a new web part, an existing web part, or a web part that is already being used on a web page.

The advantage of adding a field instead of a control is that the field expression and control type are set automatically for you.

When Adding a Field to a Web Part

Fields can only be added to a web part; they cannot be added directly to a page.

Fields with the shortcut icon are recommended fields for the service.

A red pin indicates the field is already being used in the open object.

A yellow pin indicates a field in that section is being used.

How to Add a Field to a Web Part

Follow these steps to add a field to a web part:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a web part or a page containing a web part. The web part or page appears in the working area.
4. Click Show/Hide Toolbox if the Toolbox is hidden. The Toolbox appears.
5. Click the Fields Directory tab. A list of services appears in the toolbox pane.
6. Expand the service that contains the field you want to add.
7. Select a field and drag it onto a cell on the web part.
8. Repeat steps 6-7 to add additional fields to the web part.
9. Click Save. The application saves your changes and displays a success message.
