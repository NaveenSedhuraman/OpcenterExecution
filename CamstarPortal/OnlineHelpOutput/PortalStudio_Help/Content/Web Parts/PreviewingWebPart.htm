Previewing a Web Part

Portal Studio allows you to preview web parts once they have been added to a page.

When Previewing a Web Part

The web part cannot be tested by itself. It must be added to a page, and then you can preview the page. If you add the web part to a new page, you must save that page before it can be previewed.

Any data you enter on a page while testing can be saved to your environment's database.

How to Preview a Web Part

Follow these steps to preview a web part:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a page containing a web part. The page appears in the working area.
4. Click Run. The page opens in your environment, allowing you to test the web part.
