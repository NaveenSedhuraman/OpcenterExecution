Adding a Control to a Web Part

Controls incorporate the standard Web controls with Manufacturing properties added to the control. The disadvantage of adding a control instead of a field is that you must set up the control manually, whereas the field expression and control type are set automatically when adding a field.

Controls that display fields or data will generally need field expressions. Controls that are for layout purposes such as tabs, expanding sections, fly-outs, etc. will generally not need field expressions.

When Adding a Control to a Web Part

Controls can only be added to a web part. Controls cannot be added directly to a page.

Some controls, such as a tab control or flyout, do not interact with a service and therefore do not require a field expression.

When you add a control, you must add a field expression for that control. You must select a control that is compatible with the field expression you want to use.

How to Add a Control to a Web Part

Follow these steps to add a control to a web part:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a web part or a page containing a web part. The web part or page appears in the working area.
4. Click Show/Hide Toolbox if the Toolbox is hidden. The Toolbox appears.
5. Click and drag a control onto a cell on a web part. The control appears in the web part.
6. Enter a name for the control in the Name field in the Properties pane.
7. Repeat steps 5-6 to add additional controls.
8. Click Save. The application saves your changes and displays a success message.
