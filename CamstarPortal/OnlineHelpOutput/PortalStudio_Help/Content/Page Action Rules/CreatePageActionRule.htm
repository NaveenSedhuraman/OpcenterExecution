Creating Page Action Rules

Page action rules determine when an action is active on a Portal page using custom expressions.

When Creating Page Action Rules

Page action rules are created by defining one or more expressions on the Page Action Rule Builder pop-up. Expressions are statements that indicate a page action should be shown or enabled when the statement is true. Expressions are highly customizable and multiple expressions can be defined within a single page action rule using operators such as "And" and "Or" to create complex page action rules. Supported expression options include Container, Resource, Event, or Change Package.

Note: The "And" operation appears in the first drop-down on the Page Action Rule Builder pop-up by default, but will not be added to the first expression defined for a page action rule. The "And" or "Or" operations will be added to all subsequent expressions added to the page action rule.

You must add custom page action rules to page actions to customize page action behavior. Refer to "Adding Page Action Rules to Page Actions" for information.

How to Create Page Action Rules

Follow these steps to create page action rules:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Select the Modeling tab and expand Page Action Rules.
4. Click the Create New Item button in the Page Action Rules section. A blank page action rule appears in the working area.
5. Enter a name in the Name field.
6. Click More for the Expression field. The Page Action Rule Builder pop-up appears.
7. Select operations and values from the drop-downs or enter custom information manually in the text fields to define your custom expression.
8. Click Add. The expression appears in the text box above the drop-downs and in the list of expressions below the drop-downs.
9. Repeat steps 7-8 to add additional expressions.
10. Click OK. The Page Action Rule Builder pop-up closes.
11. Click Save. The page action rule is saved to Page Action Rules in the Components catalog.
