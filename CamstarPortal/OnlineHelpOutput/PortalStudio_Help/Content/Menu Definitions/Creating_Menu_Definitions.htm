Creating a Menu Definition

You can create a structure consisting of multiple menus and sub menus to support your organizational needs.

When Creating a Menu Definition

Menu definitions appear as a set of fields within a pop-up in the working area when creating a new menu definition.

Consider the following when creating a menu definition:

• The csiPortalMenuV8, csiMobileMenu,and csiPortalMenu default menus are assigned to the Administrator, CamstarAdmin, and InSite Admin users automatically in an initial implementation of CEP, or when updating to a new release and upgrading the default instance data. You can use the Employee modeling object to assign Portal menus to other employees. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.
• You should create tabs before you create your menus because the menus reference tabs. Refer to "Creating a Tab" for information.

Tabs can be referenced three different ways by menus. This table describes the ways tabs can be referenced.

Tab Reference

Description

Inherit

Inherits the tab setting from the parent. This is the default setting.

New Tab

There is no defined tab object, therefore it will always open in a new blank tab in the Portal. Selecting this option allows you to have multiple copies open at the same time.

Specified Tab

Specify a tab that was previously created. Only one version of a specified tab can be open at a time.

If you set a menu or a submenu to a specified tab, and leave all of the items below with the default values of inherit, they will all open in the same specified tab. Therefore, the latest page opened will replace the page currently open in the specified tab.

How to Create a Menu Definition

Follow these steps to create a menu definition:

1. Access Portal Studio.
2. Click New and select Menu Definition. A blank menu definition appears in the working area.
3. Enter a name for the menu definition in the Name field.
4. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
5. Click Save. The application saves your changes and displays a success message.
