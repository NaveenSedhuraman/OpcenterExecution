Adding Source Pages to Page Actions

Source pages are the list of Portal pages where the page action appears. Actions will not show up on a page that is not listed on the source pages list, regardless of whether the conditions for the page action rules are met. Actions can have multiple pages listed as source pages.

When Adding Source Pages to Page Actions

Source pages can also be removed from a page action. A page action no longer appears on a page after that page has been deleted from the source pages list.

How to Add Source Pages to Page Actions

Follow these steps to add source pages to page actions:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Select the Modeling tab.
4. Double-click a page action. The page action properties pop-up appears in the working area.
5. Click More for SourcePages. The SourcePages pop-up appears.
6. Select a page from the UIVirtualPage field.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Repeat steps 5-7 to add additional pages as necessary.
9. Click OK. The SourcePages pop-up closes.
10. Click Save. The application saves your changes and displays a success message.

How to Delete Source Pages from Page Actions

Follow these steps to delete source pages from page actions:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Select the Modeling tab.
4. Double-click a page action. The page action properties pop-up appears in the working area.
5. Click More for SourcePages. The SourcePages pop-up appears.
6. Select a page from the list in the left pane.
7. Click Delete. The page is removed from the list.
8. Repeat steps 6-7 to delete additional pages as necessary.
9. Click OK. The SourcePages pop-up closes.
10. Click Save. The application saves your changes and displays a success message.
