Creating Page Actions

You can create page actions using the page action properties pop-up in Portal Studio. A page action's properties and the function the action performs when you click on it are determined by its type of user-interface action (UIAction).

When Creating Page Actions

Page actions are available from the Modeling tab of the UI Components catalog, with many already created by default. Page actions are configured using page action properties, and are further configured by page action rules and source pages.

Important: You cannot change the UIAction type once you have selected the UIAction type and saved the page action.

Page Action Rules determine when a page action should be available on a page using customized conditions. Refer to "Page Action Rules" for information on configuring page action rules.

Source Pages are a list of pages where the action will appear when the conditions of the page action rules are met. Source pages must be defined for a page action to appear on a page. Refer to "Adding Source Pages to Page Actions" for information on adding pages to source page lists.

Page actions are configured to appear as either a link or a button. Page actions are set as links by default when added to a page using the ActionsControl web part. You can modify the ActionStyle property of the ActionsControl web part so that actions appear as buttons, or add page actions directly to pages as buttons. Refer to "Configuring Page Actions" for information.

How to Create Page Actions

Follow these steps to create page actions:

1. Access Portal Studio.
2. Click the "New" text portion on the New button and select Page Action. The page action properties pop-up appears in the working area.
3. Enter a name for the action in the Name field.  
4. Expand the UIAction section and select a UIAction type. The General properties for UIAction are reset based upon the UIAction type selected.
5. Click More for ActionRules and add action rules as needed. Refer to "Adding Page Action Rules to Page Actions" for steps on adding action rules to actions.
6. Click More for SourcePages and add source pages as needed. Refer to "Adding Source Pages to Page Actions" for steps on adding source pages to actions.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Save. The application saves your changes and displays a success message.
