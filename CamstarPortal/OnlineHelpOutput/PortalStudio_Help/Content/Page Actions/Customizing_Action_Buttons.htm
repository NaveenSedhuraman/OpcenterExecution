Configuring Page Actions

You can configure Page actions to appear as either links or buttons on their associated source pages. You can configure page actions using:

• The ActionsControl web part to directly place a page action on a page as either a link or button.
• The page action properties pop-up to set a page action to appear as a button that appears at the bottom left or right of a page.

When Configuring Page Actions

Page actions must be enabled and source pages must be added to page actions in order for actions to appear on a Portal page. Refer to "Enabling Page Actions" for information on how to enable dynamic actions. Refer to "Adding Source Pages to Page Actions" for information on adding pages to source page lists.

The placement of the ActionsControl web part determines where page actions will appear on a Portal page. The ActionsControl web part appears as a white rectangle with a gray border and is not labeled. Refer to "Adding a Web Part to a Page" for information on adding web parts to pages.

How to Configure Page Actions as Buttons

Follow these steps to configure page actions:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Select the Modeling tab on the UI Components catalog.
4. Double-click a page action. The page action properties pop-up appears in the working area.
5. Select Enhanced for Edit Mode. Additional fields appear on the page action properties pop-up.
6. Expand UIAction.
7. Select Button from the Location field.
8. Select which pane the button will appear in on the ButtonPosition field.
9. Click Save. The application saves your changes and displays a success message.

How to Configure Page Actions Using the ActionsControl Web Part

Follow these steps to configure page actions using the ActionsControl web part:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Select the Modeling tab on the UI Components catalog.
4. Double-click a page action. The page action's properties pop-up appears in the working area.
5. Select Enhanced for Edit Mode. Additional fields appear on the page action properties pop-up.
6. Expand UIAction.
7. Select ActionPanel from the Location field.
8. Click Save. The application saves your changes and displays a success message.
9. Select the Components tab on the UI Components catalog.
10. Double-click a page. The page appears in the working area.
11. Expand Web Parts.
12. Click and drag the ActionsControl web part to a cell on the page.
13. Click Show/Hide Properties if the Properties pane is hidden. The Properties pane appears.
14. Do you want the page action to appear as a link or button?

If you want the page action to appear as . . .

Then . . .

A link.
a. Enter a unique name for the ActionsControl web part in the Name field in the Properties pane.
b. Go to step 15.

A button.
a. Enter a unique name for the ActionsControl web part in the Name field in the Properties pane.
b. Select Button from the ActionsStyle field in the Properties pane.
c. Go to step 15.
15. Click Save. The application saves your changes and displays a success message.
