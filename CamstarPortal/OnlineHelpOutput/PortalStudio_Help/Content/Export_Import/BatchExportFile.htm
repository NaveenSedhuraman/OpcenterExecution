Performing Batch Exports

Batch export allows you to export multiple pages, web parts, and pageflows as a single XML file, a Zip file containing multiple XML files, and export your items to a target Portal.

When Performing Batch Exports

You can export items to a:

• Single XML file
• Zip file containing multiple XML files
• Target Portal

You can export your items to both a local file and target Portal simultaneously. At least one option must be selected when performing a batch export.

Note: Saving a batch export as a Zip file creates a Zip file containing an XML file for each page, web part, and pageflow included in the export.

You must select at least one workspace from the Personalization Type to Export list.

The Check All and Uncheck All buttons only affects items on the currently selected tab in the left pane of the Batch Export window. For example, clicking Uncheck All with the Pages tab in focus does not affect web parts already selected in the Web Parts tab.

An Export Progress... pop-up displaying two progress bars appears when you click the Start Export button. The first bar displays the progress of each selected item and the second bar displays the total progress of the export.

You must authorize access to targets when performing batch exports across domain boundaries. You can authorize access by creating and placing a clientaccesspolicy.xml file at the root of the domain where the export service is hosted. Refer to the Microsoft Developer Network for information on creating and placing this file in your environment.

Important: Batch export does not automatically find and export dependent pages. For example, when you are exporting a page that has dependent pages referenced in a page redirect action or a flyout control, the dependent pages are not found and exported. You must manually select the dependent pages as part of the export for the page to function correctly when imported.

Batch Export Window

The Batch Export window's left pane shows all pages, web parts, and pageflows available for batch export divided into separate tabs. Items selected in one tab remain selected when another tab is brought into focus.

How to Perform Batch Exports

Follow these steps to perform a batch export:

1. Access Portal Studio.
2. Click the Batch Export button. The Batch Export window appears.
3. Select items to export from the Pages, Web Parts, and Pageflows tabs.
4. Select at least one workspace from the Personalization Type to Export list.
5. Do you want to export the selected items as an XML file?

If . . .

Then . . .

Yes
a. Select the Export to File check box. The File Name field and More button are enabled.
b. Click the  More button for the File Name field. The Save As pop-up appears.
c. Enter a file name in the File name field, navigate to a save location, select a file type, and then click Save. The Save As pop-up closes and the file name appears in the File Name field.
d. Go to step 6.

No

Go to step 6.
6. Do you want to export the selected items to a target Portal?

If . . .

Then . . .

Yes
a. Select the Export to Target Portal check box. The User Name, Password, Portal Domain, and Portal URL fields, the Override existing Web Parts, Pages or Pageflows check box, and the Verify Connection button are enabled.
b. Enter the user name for the target Portal in the User Name field.
c. Enter the password for the target Portal in the Password field.
d. Enter the domain for the target Portal in the Domain field.
e. Enter the URL to the target Portal in the Portal URL field.
f. Go to step 7.

No

Go to step 7.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information on the optional fields.
8. Click Start Export. The Export Progress... pop-up appears and displays the progress of the export. The Export Progress... pop-up closes and the Batch Export window displays a success message when the export process is complete.
