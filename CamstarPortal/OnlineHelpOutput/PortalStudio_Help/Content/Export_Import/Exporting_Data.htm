Exporting an Item

Portal Studio allows you to export web parts, pages, and pageflows. This allows you to save a copy of these objects offline from the development server.

When Exporting an Item

You can export the item from the workspace currently selected in the View Contents menu or from all active workspaces.

Exported items are saved as XML files. The name of the XML file uses the name of the item being exported by default. You cannot export menus from Portal Studio.

How to Export an Item

Follow these steps to export an item:

1. Access Portal Studio.
2. Open the item you want to export.
3. Do you want to export the item data for the currently selected workspace or all workspaces?

If you want to . . .

Then . . .

Export only the currently selected workspace data
a. Click the Export button.
b. Select Selected Workspace. A File Download - Security Warning dialog appears.
c. Go to step 4.

Export the item data for all workspaces
a. Click the Export button.
b. Select All Workspaces. A File Download - Security Warning dialog appears.
c. Go to step 4.
4. Click OK. The dialog closes and a Save As pop-up appears.
5. Navigate to a save location and click Save. The pop-up closes, the item is exported as an XML file to the specified location, and Portal Studio displays a success message.
