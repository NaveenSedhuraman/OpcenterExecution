Defining Grids Using the Grid Wizard

Portal Studio provides a wizard that offers a user guided, end-to-end grid creation process for defining a new grid control.

When Defining Grids Using the Grid Wizard

The Grid Wizard appears as a pop-up containing a series of screens. The screens and fields appear dynamically in response to the type of grid data source you select on the initial screen and the values you enter or select as you define the grid.

A Grid Control Creation pop-up appears when you place a grid control on a web part. Selecting Grid Wizard displays the Grid Wizard pop-up allowing you to use the Grid Wizard to define a grid. Selecting Create a Default Grid places a default grid control allowing you to set the grid properties manually in the Grid Settings Editor. Refer to "Grid Settings Editor" for information on defining a grid manually.

You can navigate from one screen to the next in the grid wizard by clicking the Next button. You can go back to a previous screen by clicking the Back button. Clicking Cancel will close the Grid Wizard and a grid control will not be added to the web part.

A new grid control becomes visible on the web part after clicking the Finish button on the final screen of the Grid Wizard pop-up. The information entered in the Grid Wizard can be viewed or edited in the Grid Settings Editor pop-up from the Properties pane. Refer to "Grid Settings Editor" for information.

Grid Data Source

You select a grid data source on the initial screen of the Grid Wizard to determine the type of grid being defined. Screens and fields appear dynamically throughout the Grid Wizard based on the grid data source you select.

This table defines the grid data sources.

Grid Data Source

Description

Item List

Supports any typed array as the data source. Can be bound to field expressions and employs regular value loading from the server. Item List grids can also work in an unbound mode, such as without field expressions. The ItemType must be defined and data loading and submitting should be implemented in the code when in an unbound mode. Paging within the server is not allowed.

SubEntity List

Inherits data from the Item List. Supports a particular bound mode for the item type derived from the SubEntity class.

Selection Values

Works in a bound mode and loads data from the selection values request. Data built from selected rows can be submitted. Selection Values grids data source can use two field expressions, one for data loading and one for submitting.

Predefined Query

References a System or User query. Predefined Query grids allow data submission to a field using field expressions.

AdHoc Query

Stores the custom query directly on the virtual page. AdHoc Query grids allow data submission to a field using field expressions.

Column Setup in the Grid Wizard

You can create and customize the columns that appear in a grid on the Setup Columns screen in the Grid Wizard.

The Setup Columns screen appears if a field expression was set in either the Field Expressions or Request Field Expressions fields on the Field Expression Setup screen when defining Item List, SubEntity List, and Selection Values grids. This screen always appears when defining Predefined and AdHoc Query grids.

The Setup Columns screen displays the column type, ID, width of the column measured in pixels, and a check box that sets the column as hidden. Additional check boxes that set the column as editable or required also appear when defining Item List and SubEntity List grids. You can modify values for existing columns by clicking a field and entering or selecting new values.

A predefined column appears on the Setup Columns screen based on the field expressions you set on the Field Expressions or Request Field Expressions fields. Only the field expression set in the Request Field Expressions field is used to predefine a column if field expressions are set in both the Field Expressions and Request Field Expressions fields.

Note: A message pop-up will appear on the Setup Columns screen if a predefined column cannot be created based on the selected field expressions. You can click OK to close the message and define columns manually.

Predefined columns also appear based on the predefined query or custom query, and the values you enter for query parameters, when defining a Predefined or AdHoc Query grid. A predefined column does not appear based on the field expression you set when defining Predefined Query or AdHoc Query grids.

Note: A message pop-up will appear on the Setup Columns screen if columns cannot be created based on the values entered on the Define Query screen when defining Predefined Query or AdHoc Query grids. You can click OK to close the message and define columns manually.

You can add additional columns by selecting a column type from the field below the list of existing columns and then clicking the Create button. The new column appears as the last column in the list and is selected to be included in the grid by default. Newly added columns are set to a width of 200 pixels by default.

You can rearrange the order of columns on the Setup Columns screen by selecting a column and clicking the Up and Down buttons. The topmost column listed on the Setup Columns screen appears as the leftmost column on the grid within the Portal page.

How to Define Item List and SubEntity List Grids Using the Grid Wizard

Follow these steps to define Item List and SubEntity List grids using the Grid Wizard:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a page or web part. The page or web part appears in the working area.
4. Click Show/Hide Toolbox if the Toolbox is hidden. The Toolbox appears.
5. Click and drag a grid control onto a cell. The Grid Control Creation pop-up appears.
6. Click Grid Wizard. The Grid Control Creation pop-up closes and the Grid Wizard pop-up appears.
7. Select Item List or SubEntity List as the grid data source and click Next.
8. Select a grid editing mode and click Next.
9. Click the More button for Field Expressions. The Field Expressions Builder pop-up appears.
10. Select a field expression and click OK. The Field Expression Builder pop-up closes and the field expression appears in the Field Expressions field.
11. Do you want to set a request field expression?

If you . . .

Then . . .

Want to set a request field expression.
a. Click the More button for Request Field Expressions. The Field Expressions Builder pop-up appears.
b. Select a field expression and click OK. The Field Expression Builder pop-up closes and the field expression appears in the Request Field Expressions field.
c. Click Next.
d. Go to step 12.

Do not want to set a request field expression.
a. Click Next.
b. Go to step 12.
12. Select or clear the Visible and Enabled check boxes to show or hide and enable or disable grid navigator bar action buttons as needed. Then click Next.
13. Select, modify, or create columns to be added to the grid. Then click Next. Refer to "Column Setup in the Grid Wizard" for information on setting up columns.
14. Select a column from the Row ID field.
15. Select a row selection mode from the Row Selection Mode field and click Next.
16. Enter a name for the grid control in the Name field.
17. Enter the title that will appear as the grid title in the Title Text field.

Or

Click the More button for Title Label Name to browse for and select a title label that will appear as the grid title. Refer to "Searching for Page Action Labels" for information on searching for predefined labels.

18. Click Finish. The Grid Wizard pop-up closes and the grid control appears in the cell.
19. Click Save. The application saves your changes and displays a success message.

If you . . .

b. Select a field expression and click OK. The Field Expression Builder pop-up closes and the field expression appears in the Request Field Expressions field.

12. Select, modify, or create columns to be added to the grid. Then click Next. Refer to "Column Setup in the Grid Wizard" for information on setting up columns.

15. Do you want to submit data from selected rows, all of the rows, or none of the rows when data submission occurs?

If you want to submit data from . . .

Then . . .

Selected rows.
