Grid Properties

This table provides a list of grid properties.

GridDataMode

Description

DataTable

Mode for generic grid accepting any non-structural data source in DataTable or RecordSet. It is mainly used for manual programming from code behind.

ItemList

Supports any typed array as data source. Can be bound to FieldExpression and uses regular value loading from the server. It can also work in unbound mode (such as without field expression). In unbound mode the ItemType must be defined and loading and submitting data should be implemented in code. Paging with server is not allowed.

SubEntityList

Inherited from ItemList. Supports a particular bound mode for item type derived from the SubEntity class.

SelectionValues

Uses a Field Expression to resolve the grid columns and values. Inherited from DataTable. Works in a bound mode. Loads data by the selection values request. Supports submitting data built from selected rows. Can use two field expressions – one for data loading and one for submitting.

PredefinedQuery

References a Designer or User query. Inherited from DataTable. Allows dynamic query name in runtime.

AdHocQuery

Query is stored directly in the virtual page. Inherited from DataTable. Allows dynamic query text in runtime.
