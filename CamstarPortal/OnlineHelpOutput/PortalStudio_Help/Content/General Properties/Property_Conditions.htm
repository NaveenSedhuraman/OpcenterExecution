Property Conditions

Property conditions allow you to set or change a property on a control using a series of custom conditions and triggers.

When Setting Property Conditions

Property conditions are comparisons between left and right values called condition sets. Condition sets are evaluated and set a value on another element when the comparison evaluates as true.

A property condition can be as simple as a single property condition containing a single condition set or contain multiple groups of condition sets separated by And and Or operators. A property condition must contain at least one condition set and one Value If True condition. For example, a simple condition set could check to see if two controls both contain equal values on a page with a Value If True condition that sets another control to that value when the condition set evaluates as true.

Property conditions are organized in a tree structure. At the top level, each control contains property conditions which are visible from the Property Conditions pane on the <Control> - PropertyConditions pop-up.

You can define the following within each property condition:

• A condition set containing left and right values. You can define the following from this level:
• Initiators that contain events that specify when the Left or Right values within the set are evaluated.
• Condition groups separated by And and Or operators to add additional conditional sets. Condition groups can contain their own initiators and further sub-condition groups as needed.
• And the Value If True conditions that are set when the condition sets evaluate as true. You can define the following from this level:
• Initiators that contain events that specify when a Value If True condition should be set.

This structure allows you to create as many condition sets, initiator triggers, and results as needed.

<Control> - PropertyConditions Pop-up

The <Control> - PropertyConditions pop-up appears when the More... button is clicked for Property Conditions on the Properties pane while a control is selected in the working area. Refer to "Properties Pane" for information on viewing the Properties pane.

You can create new property conditions or view existing property conditions on the <Control> - PropertyConditions pop-up. The Property Conditions pane shows the property conditions currently set on the control. You can click the arrow to expand and collapse each property condition to view the associated condition set, condition groups, and For Property field. Condition groups appear indented with each sub-condition group appearing further indented for readability.

Each property condition within the Property Conditions pane has a Name and For Property field. When adding a new property condition you must select a property from the For Property field. The For Property field determines which property on the control will receive the value from the Value If True condition. You can edit the names of existing property conditions from the Property Conditions pane by selecting the property condition name and entering a new name.

The Edit Conditions... and Edit Value if True... buttons appear beneath property conditions in the Property Conditions pane. Clicking the:

• Edit Conditions... button displays the Conditions Editor pop-up. Refer to "Conditions Editor" for information.
• Edit Value If True... button displays the Value If True Editor pop-up. Refer to "Value If True Editor" for information.

You can delete a property condition by clicking the X to the right of the property condition.

You can close the <Control> - PropertyConditions pop-up by clicking the Cancel button or clicking on the X in the upper right corner.

Conditions Editor

The Conditions Editor pop-up appears when you click the Edit Conditions... button on the <Control> - PropertyConditions pop-up. You can define a property condition set and multiple condition groups and sub-condition groups from the Conditions Editor pop-up.

You create a condition set by selecting a comparison operator from the first drop-down to determine how the condition set compares the Left and Right values. You then select Left and Right values that reference an object such as a control, data contract, etc.

You can continue to define condition sets after defining an initial condition set by clicking either:

• Add AND Group, which displays an additional set of fields to define another condition set separated from the initial set by an And operator.
• Add OR Group, which displays an additional set of fields to define another condition set separated from the initial set by an Or operator.

Condition groups can contain multiple condition sets that reference the And or Or operator for that group. You can continue to create additional condition groups that can also contain multiple condition sets as needed. Condition groups and sub-condition groups appear indented for readability.

You can delete a condition set by clicking the X to the right of the condition set. You can remove a condition group by clicking the Remove AND Group or Remove OR Group buttons. You cannot delete the single starting condition set.

Note: Clicking Remove AND Group or Remove OR Group deletes all the conditions sets within the group. Click the X to the right of a condition set to delete only that set.

Value If True Editor

The Value If True Editor pop-up appears when you click Edit Value If True... button on the <Control> - PropertyConditions pop-up. The Value If True Editor pop-up allows you to configure what value is set when the associated conditions and initiators for the property condition are met. For example, you could specify that a control on the page be set to "True" when the associated condition set evaluates as true.

Value If True Editor Pop-up Field Definitions

The fields on the Value If True Editor are the same as those used to define a Left or Right value on the Conditions Editor Pop-up. Refer to "Conditions Editor Pop-up Field Definitions" for information.

Initiator Editor

The Initiator Editor pop-up is available from both the Conditions Editor and Value If True Editor pop-ups. The blue box on the right corner of the Initiators... button displays the number of initiators currently set on the value.

The Initiator Editor pop-up allows you to define triggers to customize when Portal Studio evaluates the Left or Right values in a condition set or Value If True conditions. For example, you could specify a trigger that only evaluates the associated Left value when the value of a specific control on the page has been changed. You can set multiple initiators to further customize when a condition set value or Value If True condition should be evaluated.

Initiators are optional and are not required when defining a property condition.

You can delete an initiator by clicking the X to the right of the initiator.

How to Access the <Control> - PropertyConditions Pop-up

Follow these steps to access the <Control> - PropertyConditions pop-up:

1. Access Portal Studio.
2. Click Show/Hide Catalog if the UI Components catalog is hidden. The UI Components catalog appears.
3. Double-click a web part or a page containing a web part. The web part or page appears in the working area.
4. Click Show/Hide Properties if the Properties pane is hidden. The Properties pane appears.
5. Select a control in the working area.
6. Click More for Property Conditions. The <Control> - PropertyConditions pop-up appears.

How to Define a Property Condition

Follow these steps to define a property condition:

1. Perform the "How to Access the <Control> - PropertyConditions Pop-up" procedure.
2. Enter a name for the property condition in the Enter Property Condition Name field and click Add Property Condition. The property condition appears in the Property Conditions pane.
3. Expand the property condition and select a property from the For Property field.
4. Click Edit Conditions.... The Conditions Editor pop-up appears.
5. Select a comparison operator from the first drop-down field.
6. Select a connection type from the ConnectionType field for the Left Value. The fields for the left value refresh based on the connection type you selected.
7. Enter information in the fields that appear according to your business requirements. Refer to the field definitions table for information on required and optional fields.
8. Repeat steps 5-7 for the Right Value fields.
9. Click OK. The Conditions Editor pop-up closes.
10. Click Edit Value If True.... The Value If True Editor pop-up appears.
11. Select a connection type from the ConnectionType field. The fields refresh based on the connection type you selected.
12. Enter information in the fields that appear according to your business requirements. Refer to the field definitions table for information on required and optional fields.
13. Click OK. The Value If True Editor pop-up closes.
14. Click OK. The <Control> - PropertyConditions pop-up closes and saves your changes.
