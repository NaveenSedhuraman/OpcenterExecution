Page Properties

Page properties allow you to view or edit the properties of Portal Studio pages. Page properties appear in the Properties pane when a page is selected in the working area. Refer to "Properties Pane" for information on the Properties pane.

A description window displaying information about page properties appears beneath the Properties pane when a page property is selected.

Note: Some properties contain sub-properties allowing for further customization. Sub-properties will continue to display their parent property's description in the Properties pane description window.

Page Properties Descriptions

This table provides the descriptions of page properties.

Page Property

Property Description

General

ActionGroups

Used in conjunction with the Action Controls WP to control grouping of page actions.

Actions

Actions of the page. Allows for definition of page level actions and buttons.

AllowEditCondition

Allow to edit page as a result of evaluated expression (condition).

Behavior

Behavior type of the page.

Connections

Data Contract connection between Web Parts on the page.

DataContractMembers

Data Contracts are used for communication between pages.

DefaultButton

Indicates the button that will be clicked automatically when the Enter button is pressed.

Display

Customizes the page display by making it full screen or by hiding various framework elements of the page, such as the:

• global footer
• global header
• tab row
• menu bar

Important: Display properties set on the ModelingVP page are promoted to and control all pages listed on the CDOFormsSettings - CDO Forms pop-up. Refer to "CDO Forms Settings" for information on the CDOFormsSettings - CDO Forms pop-up.

FullScreen

Boolean value that expands the page's display by removing the gray sidebars on both sides of the application to use the full browser width. Set to False by default.

Note: The layout of fields, buttons, and other page objects may shift when viewing a full screen page. Base application pages are not designed for full screen viewing. You can enable this feature when designing your own custom full screen pages. You can also maximize any pop-ups that appear on a full screen page to the full browser width.

You can enable full screen mode on these application framework elements using the FullScreenMode setting:

• global footer
• global header
• tab row
• menu bar

Refer to "Default Settings" for information on Portal settings.

HideFooter

Boolean value that determines if the page hides the application's global footer. Set to False by default.

HideHeader

Boolean value that determines if the page hides the application's global header. Set to False by default.

HideTabs

Boolean value that determines if the page hides the application's tab row. Set to False by default.

Note: Siemens recommends adding a Close button to a page if that page no longer displays a tab row or menu bar.

HideTopMenu

Boolean value that determines if the page hides the application's menu bar. Set to False by default.

Note: Siemens recommends adding a Close button to a page if that page no longer displays a tab row or menu bar.

DriverComponent

Used within the modeling framework to specify a control that will force a load values call when its data changes.

DynamicActions

Used in conjunction with the Actions Control WP to control the display and behavior of dynamic actions.

DynamicWebparts

Contains the list of dynamic web parts currently on the page, along with the properties of those web parts.

EProcEnabled

Indicates whether this page can be used within the eProcedure framework as a transaction page.

ForceEditMode

For Pages with Behavior set to 'ViewEdit', force the page to load in Edit mode.

HelpFileUrl

URL for the help file of the page.

Layout

Visual appearance of the page. Recommended not to change this property directly.

MobileBarcodeEnabled

Enables scanning of data from a mobile device into editable fields on the shop floor page. Set to True by default on each shop floor page. This page-level setting and the global setting MobileBarcodeEnabled must both be set to True for the bar code scanning icon to appear in editable fields on the page. When set to False, bar code scanning is disabled on the individual page.

Name

The name of the page.

PrimaryServiceType

Service type for controls that do not specify a service type.

RenderMode

Indicates whether the page uses responsive rendering. Options include the following:

• NotSet
• Fixed
• Responsive

The default setting for all shop floor pages is NotSet, which equates to Responsive. The default setting for all other pages is Fixed. Fixed pages contain a fixed width, and the elements on the page do not move to adjust to the resizing of the browser.

Note: You can log on to the application using one of three entry modes. If you set RenderMode at the page-level, the setting you enter overrides the default render mode settings associated with each of the three entry modes. Refer to the Opcenter Execution Medical Device and Diagnostics Getting Started User Guide or the Opcenter Execution Core Getting Started User Guide for information.

The FullScreen mode page property is set for all these pages when the Horizon theme is set. Refer to FullScreen for information.

SessionVariables

Used to configure Page level session variables.

StaticWebParts

Not implemented currently.

TagCollection

A collection of tag names and either their control ID or session variable.

Title

Text of the page title. Supercedes the TitleLabel property.

Titlelabel

Name of the dictionary label of the title.

TypeName

Code behind type of the page.

Note: Overriding the TypeName at the Page level currently not supported.

Zones

Not implemented currently.

Layout

CssClass

The stylesheet reference for the page.

HorizontalAlignment

Horizontal Alignment of Web Parts.

Margin

The margin of the page.

Padding

The padding of the page.

History

Created

Creation date and time of the page.

CreatedBy

Creator of the page.

HowLong

Time period since creation of the page.

LastModified

Last modified date of the page.

LastModifiedBy

User who modified the page most recently.
