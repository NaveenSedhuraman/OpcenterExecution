Action Properties

Action properties allow you to view or edit the properties of local actions for pages, web parts, and other objects. Action properties appear in the <Object> - Action pop-up when the More button is clicked for Actions in the Properties pane. Refer to "Properties Pane" for information on the Properties pane.

Action properties that are not required by the currently selected action type are hidden on the <Object> - Action pop-up.

This table provides the descriptions of action properties.

Action Property

Property Description

ActionCategory

Specifies an action category to associate with the action.

ButtonPosition

Determines whether the action button appears on the left or right side of a page when the Location field is set to Button.

ClearValues

Clear values when the action is submitted.

CloseButtonOnly

Deprecated.

CommandName

Used to provide information for logic in event arguments and JavaScript.

ConditionHandler

Options: DefaultConditionHandler or SpecificConditionHandler. Defines the handler method in code that is called when the action is submitted. This allows you to execute additional functionality from the app code when the action submits. 

Confirmation

Set of properties that configure the labels that appear on a confirmation message.

Confirmation - General

Cancel_LabelName

Determines the text that appears on the Cancel button of the confirmation message.

Cancel_LabelText

Determines the text that appears on the Cancel button of the confirmation message. This field supercedes text entered in the Cancel_LabelName sub-property.

Message_LabelName

Determines the text that appears as the message of the confirmation message.

Message_LabelText

Determines the text that appears as the message of the confirmation message. This field supercedes text entered in the Message_LabelName sub-property.

OK_LabelName

Determines the text that appears on the OK button of the confirmation message.

OK_LabelText

Determines the text that appears on the OK button of the confirmation message. This field supercedes text entered in the OK_LabelName sub-property.

Title_LabelName

Determines the text that appears as the title of the confirmation message.

Title_LabelText

Determines the text that appears as the title of the confirmation message. This field supercedes text entered in the Title_LabelName sub-property.

CssClass

Specifies a custom CSS class for the action.

CustomMethodHandler

Options: ExecuteMaintenanceTransaction, ExecuteNewTransaction, ShopfloorReset, TrueSubmit, VIewDocument, VIewInExcel, or WebPartCustomAction.

DataContractLinks

Data Contract members to pass to the linked page.

DataContractReturnLinks

Data Contract Members returned from the floating page.

DenyDataContract

Nullifies any Data Contracts associated with the action.

Description

Description of the action.

DisableCondition

Property Condition expression that will cause the action to be disabled if the expression evaluates to True.

DisableCondition - Description

Expression

The conditional expression that is evaluated.

DisableCondition - Equivalency

ColumnName

Name of the column used in the comparison.

Comparison

Comparison operator used to evaluate a condition. Valid entries are "Equal", "NonEqual", and "Contains".

DisableCondition - Equivalency - LeftValue - General

ConnectionType

Specifies the connection type of the event handler.

Note: Additional fields will appear based on the connection type selected.

Control

Specifies a control to be used on a given side of the condition evaluation.

Note: This field appears when Control is selected in the ConnectionType field.

PageDataContract

Specifies a page data contract to be used on a given side of the condition evaluation.

Note: This field appears when PageDataContract is selected in the ConnectionType field.

SessionDataContract

Specifies a session data contract to be used on a given side of the condition evaluation.

Note: This field appears when SessionDataContract is selected in the ConnectionType field.

SessionVariable

Specifies a session variable to be used on a given side of the condition evaluation.

Note: This field appears when SessionVariable is selected in the ConnectionType field.

FieldExpression

Specifies a field expression to be used on a given side of the condition evaluation.

Note: This field appears when FieldExpression is selected in the ConnectionType field.

StaticValue

Specifies a static value to be used on a given side of the condition evaluation.

Note: This field appears when StaticValue is selected in the ConnectionType field.

GridItem

Specifies a grid item to be used on a given side of the condition evaluation.

Note: This field appears when GridItem is selected in the ConnectionType field.

Property

Specifies the control property value that is evaluated.

Note: This field appears when Control is selected in the ConnectionType field.

RequestDataMode

Specifies the method used to retrieve the property value from a control.

Note: This field appears when Control is selected in the ConnectionType field.

AllowEmptyValue

Determines whether the value can be empty.

Initiators

The Initiators pop-up appears when you click the More button.

DisableCondition - Equivalency - RightValue

Note: The RightValue fields are identical to the LeftValue fields. Refer to the "DisableCondition - Equivalency - LeftValue - General" fields for field definitions.

DisableCondition - General

And

The And pop-up appears when you click the More button. You can define additional conditions for a conditional event handler that will be added with an "And" operator.

Note: The And pop-up fields are identical to the fields that appear when defining a conditional event handler. Refer to the "DisableCondition" fields for field definitions.

Or

The Or pop-up appears when you click the More button. You can define additional conditions for a conditional event handler that will be added with an "Or" operator.

Note: The Or pop-up fields are identical to the fields that appear when defining a conditional event handler. Refer to the "DisableCondition" fields for field definitions.

DynamicDisplayMode

Determines the display mode of the action for users with roles that are not permitted to view the action.

EndResponse

Used by floating frame actions internally to set the CallStack redirect information. A value of True follows the standard framework functionality for floating frame call stacks. A value of False exposes the FloatingPageCallStackMethod in JavaScript.

ESignatureDependencies(Collection)

Collection of Esignatures required to submit the action.

ESignatureRequired

Is an Esignature required?

ForceRedirect

Bypasses any required field validations before redirecting to the target page if set to True.

FormValidate

Indicates whether to turn on form validation before form submit (implemented in html5).

FrameLocation

Allows you to specify the dimensions of the floating page when it is loaded.

FrameLocation - General

Height

Height in pixels.

Width

Width in pixels.

GroupName

Name of the action group.

HiddenCondition

The condition that is evaluated to determine if the page action is hidden on a page.

Note: The HiddenCondition sub-properties are identical to the sub-properties for DisableCondition when defining a conditional expression. Refer to the "DisableCondition" fields for property descriptions.

HorizontalShift

Sequence of the shift: Left to Right

HorizontalShift - General

HorizontalShiftDirection

Specifies the direction to adjust the horizontal position of the action during runtime. Accepted values are "Left","Right", and "None".

HorizontalShiftValue

Number of pixels to adjust the horizontal position of the action.

Index

Integer used to order the display of the actions.

InitContractHandler

Used to assign a method that will be invoked when action processing is initializing data contracts of target page.

IsDisabled

Is the action disabled?

IsHidden

Is the action hidden?

IsPrimary

Is the action the primary action?

IsReturn

Closes the page after submittal.

LabelName

Dictionary label name for the action.

LabelText

Label text for the action. This field supercedes text entered in the LabelName property.

Location

Determines where the action will appear on a page. Selecting Button sets the action to appear as a button on the bottom of associated pages. Selecting ActionPanel sets the action to appear within the ActionsControl web part on associated pages.

MapItem

Specifies the Page Map to use when the action submits. Page Maps are maintained in the Modeling tab of Portal Studio and are used to define Source Pages, Targets and Parameters.

Name

Name of the action.

PageflowName

Name of the pageflow to be opened.

PageName

Name of the page to be opened.

Parameters

Allows for entry of a string value that can be used in the WebPartCustomAction function override in the App_Code file for a page. Parameters are typically used to determine which button was clicked in the case of multiple Custom Actions on a page.

Permissions - General

DisplayMode

Options: Disable or Hide. Display mode if the permissions are not satisfied.

PagePermission

Apply the page permissions?

ServicePermission

Apply the service permissions?

PortalTab

Existing tabs can be selected in the PortalTab field, if SpecificTab is selected for PortaTabOptions,.

PortalTabOptions

Options: CurrentTab, NewTab, or SpecificTab. Tab option for the new page to open in.

QueryStringParameters(Collection)

Allows the user to map Data Contract Members from the page to Query Parameters that will be appended to the Link URL when the link is opened.

ReloadValues

The action reloads the page values after submitting when set to True.

ServiceName

Service that is executed when the action is clicked.

ShowButtons

Displays the First, Next, Prev, and Last buttons.

SPCChild

Launches SPC in the parent frame if it is being called from a child.

SPCEnabled

Enables SPC actions.

URL

URL that is executed when the action is clicked.

WIPMessagesRequired

Specifies if a WIP message is required for the action.
