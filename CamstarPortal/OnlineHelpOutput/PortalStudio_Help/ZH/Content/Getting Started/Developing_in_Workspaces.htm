在工作区中进行开发

Siemens 为元数据提供工作区，以降低合作伙伴产品和客户自定义的开发、部署和升级的难度并提高可管理性。

要想在更改或自定义门户时不会覆盖其他人所做的更改，可将开发发布到 Portal Studio 中的工作区。将更改发布到工作区即可使这些更改独立于对其他工作区进行的任何更新。例如，发布到客户工作区的更改不会影响基础 Siemens PLM CEP 工作区。同样，如果接受对基础 Siemens PLM CEP 工作区的更新，则不会覆盖其他工作区中的自定义内容。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

在工作区中进行开发时

Portal Studio 默认在“开发者”工作区中打开。可在“查看内容”菜单中查看活动工作区列表。每个工作区名称旁显示的颜色取决于为 Management Studio 中的工作区指定的颜色。有关管理工作区的信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

“开发者”和基础 Siemens PLM CEP 工作区在 Portal Studio 中默认处于活动状态。如果客户在安装发行版 8.0 后收到其元数据数据库，则“客户”工作区默认也处于活动状态。其他工作区必须在 Management Studio 中处于活动状态，然后才会在“查看内容”菜单中显示。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

Note: 在 Management Studio 中激活或撤消工作区时，必须重新生成 WCF 服务。在重新生成 WCF 服务之前，对可用活动工作区所做的更改不会显示在“查看内容”菜单中。

只能在优先级最高的活动工作区或“开发者”工作区中保存和发布更改。要查看其他活动工作区的内容和开发更改，可从“查看内容”菜单将其选中，但是“保存”、“全部保存”和“发布”按钮均处于禁用状态。

发布的更改特定于工作区，并且仅显示在其发布至的工作区中。在“开发者”工作区中保存和发布更改时，这些更改将发布至最高层级活动工作区。在保存之前切换到其他工作区将导致当前工作区中所有未保存的更改丢失。

更改和保存的所有内容默认均带红色边框显示，表示已成功保存。在工作区中的默认虚拟页面上更改并保存控件时，对象周围的边框将以在 Management Studio 中为工作区指定的颜色显示。彩色边框表示该控件已被替代。

可将某工作区设为 Management Studio 中优先级最高的活动工作区。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《Opcenter Execution Core 系统管理指南》。

如果没有其他任何活动工作区，则在“开发者”工作区中保存和发布的所有更改都将应用于基础 Siemens PLM CEP 工作区。基础 Siemens PLM CEP 工作区始终处于活动状态。

Important: 发布到基础 Siemens PLM CEP 工作区的更改将在 Opcenter Execution Core 更新后丢失。Siemens 建议在基础 Siemens PLM CEP 工作区以外的其他工作区中开发、保存和发布更改。

在使用 ASPX 转换器时

可通过 Portal Studio 自动将 ASPX 文件转换为门户页面，然后在工作区中将其作为门户页面继续编辑。已转换的页面将应用于优先级最高的活动工作区。有关信息，请参考“将 ASPX 文件转换为门户页面”。

在使用跟踪日志时

可通过 Portal Studio 运行“跟踪日志”以帮助进行故障排除。使用“跟踪日志”，可捕获在 Portal Studio 中更新和运行门户页面时对其所做更改的历史。有关信息，请参考“运行跟踪日志”。

如何将内容发布到活动工作区

按照以下步骤将内容发布到活动工作区：

1. 访问 Portal Studio。
2. 选择优先级最高的活动工作区。

或

将“开发者”工作区保留为选定工作区。

3. 如果“用户界面组件”目录处于隐藏状态，单击显示/隐藏目录。将显示“用户界面组件”目录。
4. 双击页面。 随即工作区域中将显示该页面。
5. 进行任何必要的更改。
6. 单击保存。应用程序将保存更改并显示成功消息。
7. 单击发布。将显示“发布”对话框。
8. 单击确定。 将关闭“发布”对话框，页面随即发布至优先级最高的活动工作区。
