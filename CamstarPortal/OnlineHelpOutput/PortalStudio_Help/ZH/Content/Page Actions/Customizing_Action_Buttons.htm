配置页面操作

可以将页面操作配置为在关联源页面上显示为链接或按钮。要配置页面操作，可使用以下各项：

• ActionsControl Web 部件，用于直接将页面操作作为链接或按钮放置在页面上。
• 页面操作属性弹出窗口，用于将页面操作设为作为按钮显示在页面左下角或右下角。

在配置页面操作时

必须启用页面操作，并且必须将源页面添加至页面操作，然后这些操作才能显示在门户页面中。有关如何启用动态操作的信息，请参考“启用页面操作”。有关将页面添加至源页面列表的信息，请参考“向页面操作添加源页面”。

ActionsControl Web 部件的位置决定页面操作在门户页面上的显示位置。ActionsControl Web 部件显示为带灰色边框的白色矩形且没有标签。有关向页面添加 Web 部件的信息，请参考“向页面添加 Web 部件”。

如何将页面操作配置为按钮

按照以下步骤配置页面操作：

1. 访问 Portal Studio。
2. 如果“用户界面组件”目录处于隐藏状态，单击显示/隐藏目录。将显示“用户界面组件”目录。
3. 在用户界面组件目录中选择建模选项卡。
4. 双击页面操作。 随即工作区域中将显示页面操作属性弹出窗口。
5. 将编辑模式设为增强。随即页面操作属性弹出窗口中将显示其他字段。
6. 展开 UIAction。
7. 在位置字段中选择按钮。
8. 在 ButtonPosition 字段中选择将显示该按钮的窗格。
9. 单击保存。应用程序将保存更改并显示成功消息。

如何使用 ActionsControl Web 部件配置页面操作

按照以下步骤使用 ActionsControl Web 部件配置页面操作：

1. 访问 Portal Studio。
2. 如果“用户界面组件”目录处于隐藏状态，单击显示/隐藏目录。将显示“用户界面组件”目录。
3. 在用户界面组件目录中选择建模选项卡。
4. 双击页面操作。 随即工作区域中将显示页面操作属性弹出窗口。
5. 将编辑模式设为增强。随即页面操作属性弹出窗口中将显示其他字段。
6. 展开 UIAction。
7. 在位置字段中选择 ActionPanel。
8. 单击保存。应用程序将保存更改并显示成功消息。
9. 在用户界面组件目录中选择组件选项卡。
10. 双击页面。 随即工作区域中将显示该页面。
11. 展开 Web 部件。
12. 单击 ActionsControl Web 部件，然后将其拖至页面中的单元。
13. 如果“属性”窗格处于隐藏状态，单击显示/隐藏属性。将显示“属性”窗格。
14. 是否要使页面操作显示为链接或按钮？

要使页面操作显示为 ...

操作步骤 ...

链接。
a. 在“属性”窗格的 Name 字段中输入 ActionsControl Web 部件的唯一名称。
b. 转到步骤 15。

按钮。
a. 在“属性”窗格的 Name 字段中输入 ActionsControl Web 部件的唯一名称。
b. 在“属性”窗格的 ActionsStyle 字段中选择按钮。
c. 转到步骤 15。
15. 单击保存。应用程序将保存更改并显示成功消息。
