创建页面操作

可以在 Portal Studio 中使用页面操作属性弹出窗口来创建页面操作。页面操作的属性及用户单击该操作时将执行的功能由其用户界面操作 (UIAction) 的类型确定。

在创建页面操作时

页面操作可从“用户界面组件”目录的“建模”选项卡获得，默认情况下多数操作均已创建。页面操作通过页面操作属性来配置，并通过页面操作规则和源页面进一步配置。

Important: 选择了 UIAction 类型并且保存了页面操作，就不能更改 UIAction 类型。

页面操作规则使用自定义的条件来确定页面操作应何时在页面上可用。有关配置页面操作规则的信息，请参考“页面操作规则”。

源页面是页面的列表，这些页面中的操作将在页面操作规则条件得到满足时显示。必须为页面操作定义将显示在页面上的源页面。有关将页面添加到源页面列表的信息，请参考“向页面操作添加源页面”。

页面操作配置为以链接或按钮的形式显示。在使用 ActionsControl Web 部件将页面操作添加到页面时，它默认设为链接。可以修改 ActionsControl Web 部件的 ActionStyle 属性，使得操作显示为按钮，或者将页面操作作为按钮直接添加到页中。有关信息，请参考“配置页面操作”。

如何创建页面操作

按照以下步骤创建页面操作：

1. 访问 Portal Studio。
2. 在新建按钮上单击“新建”文本部分，然后选择页面操作。随即工作区域中将显示页面操作属性弹出窗口。
3. 在名称字段中输入操作的名称。
4. 展开 UIAction 部分并选择 UIAction 类型。UIAction 的“常规”属性将基于选定的 UIAction 类型重置。
5. 单击 ActionRules 的“更多”，然后根据需要添加操作规则。有关向操作添加操作规则的步骤，请参考“向页面操作添加页面操作规则”。
6. 单击 SourcePages 中的“更多”，然后根据需要添加源页面。有关向操作添加源页面的步骤，请参考“向页面操作添加源页面”。
7. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
8. 单击保存。应用程序将保存更改并显示成功消息。
