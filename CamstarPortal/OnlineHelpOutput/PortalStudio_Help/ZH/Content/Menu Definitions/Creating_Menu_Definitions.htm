创建菜单定义

可创建由多个菜单和子菜单组成的结构，以支持自己组织的需要。

在创建菜单定义时

新建菜单定义时，菜单定义在工作区域中显示为弹出窗口中的一组字段。

创建菜单定义时应考虑以下内容：

• 在初始执行 CEP 或者在更新到新版本并升级默认实例数据时，系统会将 csiPortalMenuV8、csiMobileMenu 和 csiPortalMenu 默认菜单自动指派给 Administrator、CamstarAdmin 和 InSite Admin 用户。可使用员工建模对象将 Portal 菜单指派给其他员工。有关信息，请参考《Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《Opcenter Execution Core Modeling 用户指南》。
• 由于菜单会引用选项卡，用户应该在创建菜单前先创建选项卡。有关信息，请参考“创建选项卡”。

菜单可通过三种方式引用选项卡。此表描述可引用选项卡的方式。

选项卡引用

描述

继承

从父项继承选项卡设置。这是默认设置。

新建选项卡

由于没有已定义的选项卡对象，因此它将始终在“门户”中的新空白选项卡中打开。通过选择此选项可以同时打开多个副本。

指定选项卡

指定之前创建的选项卡。一次仅能打开指定选项卡的一个版本。

如果将菜单或子菜单设置到指定选项卡，并保留了下面所有的项与继承的默认值，则它们都将在同一指定选项卡中打开。因此，最后打开的页面将替换指定选项卡中当前打开的页面。

如何创建菜单定义

按照以下步骤创建菜单定义：

1. 访问 Portal Studio。
2. 单击新建，然后选择菜单定义。随即工作区域中将显示一个空白菜单定义。
3. 在名称字段中输入菜单定义的名称。
4. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
5. 单击保存。应用程序将保存更改并显示成功消息。
