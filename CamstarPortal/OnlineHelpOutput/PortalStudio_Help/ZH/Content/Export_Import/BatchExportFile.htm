执行批量导出

通过批量导出，可以将多个页面、Web 部件和页面流导出为单个 XML 文件、包含多个 XML 文件的 Zip 文件，以及将项导出至目标门户。

如何执行批量导出

可以将项导出至：

• 单个 XML 文件
• 包含多个 XML 文件的 Zip 文件
• 目标门户

可以同时将项导出至本地文件和目标门户。在执行批量导出时，必须至少选择一个选项。

Note: 将批量导出另存为 Zip 文件可为导出中所含的每个页面、Web 部件和页面流创建一个包含 XML 文件的 Zip 文件。

必须至少从“要导出的个性化类型”列表中选择一个工作区。

“全部选中”和“取消全选”按钮仅影响在“批量导出”窗口的左窗格中当前选定的选项卡上的项。例如，在“页面”选项卡位于焦点时单击“取消全选”不会影响已在“Web 部件”选项卡中选择的 Web 部件。

单击“开始导出”按钮时，将显示带有两个进度条的“导出进度...”弹出窗口。第一个进度条显示每个所选项的进度，第二个进度条显示导出操作的总进度。

在跨域边界执行批量导出时，必须授权访问目标。通过在托管导出服务的域的根目录中创建和放置 clientaccesspolicy.xml 文件可授权访问。有关在环境中创建和放置此文件的信息，请参考 Microsoft Developer Network。

Important: 批量导出不会自动查找和导出从属页面。例如，如果要导出的页面在页面重定向操作或浮出控件中引用了从属页面，则不会查找和导出从属页面。必须手动选择从属页面作为页面导出的一部分，才能在导入时正常运行。

批量导出窗口

“批量导出”窗口的左窗格显示可供批量导出的所有页面、Web 部件和页面流，并将其分为单独的选项卡。在一个选项卡中选择的项会在另一个选项卡位于焦点时保持选定状态。

如何执行批量导出

按照以下步骤执行批量导出：

1. 访问 Portal Studio。
2. 单击批量导出按钮。将显示“批量导出”窗口。
3. 从“页面”、“Web 部件”和“页面流”选项卡中选择要导出的项。
4. 从要导出的个性化类型列表中选择至少一个工作区。
5. 是否要将所选项导出为 XML 文件？

如果 ...

操作步骤 ...

是
a. 选中导出到文件复选框。将启用“文件名”字段和“更多”按钮。
b. 为文件名字段单击更多按钮。将显示“另存为”弹出窗口。
c. 在文件名字段中输入文件名，导航至保存位置，选择文件类型，然后单击保存。“另存为”弹出窗口关闭，文件名会显示在“文件名”字段中。
d. 转到步骤 6。

否

转到步骤 6。
6. 是否要将所选项导出到目标门户？

如果 ...

操作步骤 ...

是
a. 选中导出到目标门户复选框。将启用“用户名”、“密码”、“门户域”和“门户 URL”字段、“替代现有 Web 部件、页面或页面流”复选框以及“验证连接”按钮。
b. 在用户名字段中输入目标门户的用户名。
c. 在密码字段中输入目标门户的密码。
d. 在域字段中输入目标门户的域。
e. 在门户 URL 字段中输入目标门户的 URL。
f. 转到步骤 7。

否

转到步骤 7。
7. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
8. 单击开始导出。将显示“导出进度...”弹出窗口，其中显示导出操作的进度。当导出流程完成时，“导出进度...”弹出窗口关闭，“批量导出”窗口会显示一条成功消息。
