定义事件处理程序

使用事件处理程序可配置要基于事件执行的操作，还可指定自定义条件。

在定义事件处理程序时

事件处理程序特定于对象，分为通用和条件两种类型。通用事件处理程序用于配置基于预定义事件的操作。条件事件处理程序用于配置操作，以及使用左侧值和右侧值之间的比较的其他条件（在事件之外）。

仅当定义条件事件处理程序时，Condition 字段及其关联字段才会显示在 EventHandlers 弹出窗口中。

可选择定义 EventHandlers 弹出窗口中的其他参数和发起程序。有关用于定义参数和发起程序的字段的信息，请参考“Parameters 弹出窗口字段定义”和“Initiators 弹出窗口字段定义”。

用于定义条件事件处理程序中左侧值和右侧值的字段均动态基于在 ConnectionType 字段中选择的连接类型。连接类型决定了将显示的其他字段。

如何定义通用事件处理程序

按照以下步骤定义通用事件处理程序：

1. 访问 Portal Studio。
2. 如果“用户界面组件”目录处于隐藏状态，单击显示/隐藏目录。将显示“用户界面组件”目录。
3. 双击页面。 随即工作区域中将显示该页面。
4. 在页面上选择虚拟对象。该对象的属性显示在“属性”窗格中。
5. 在“属性”窗格中单击 EventHandlers 的更多按钮。将显示 EventHandlers 弹出窗口。
6. 在左侧窗格上方的下拉列表中选择 GenericEventHandler，然后单击添加。空白字段显示在右侧窗格中。
7. 从 Event 字段中选择事件。
8. 从 HandlerSource 字段中选择处理程序源。
9. 从 Handler 字段中选择处理程序。
10. 在 Name 字段中输入事件处理程序的名称。
11. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
12. 单击保存。应用程序将保存更改并显示成功消息。

如何定义条件事件处理程序

按照以下步骤定义条件事件处理程序：

1. 访问 Portal Studio。
2. 如果“用户界面组件”目录处于隐藏状态，单击显示/隐藏目录。将显示“用户界面组件”目录。
3. 双击页面。 随即工作区域中将显示该页面。
4. 在页面上选择虚拟对象。该对象的属性显示在“属性”窗格中。
5. 在“属性”窗格中单击 EventHandlers 的更多按钮。将显示 EventHandlers 弹出窗口。
6. 在左侧窗格上方的下拉列表中选择 ConditionalEventHandler，然后单击添加。空白字段显示在右侧窗格中。
7. 从 Event 字段中选择事件。
8. 从 HandlerSource 字段中选择处理程序源。
9. 从 Handler 字段中选择处理程序。
10. 在 Name 字段中输入事件处理程序的名称。
11. 展开 Condition 部分。
12. 展开 Equivalency 部分。
13. 在 Comparison 字段中选择比较运算符。
14. 展开 LeftValue 部分。
15. 从 ConnectionType 字段中选择连接类型。系统将根据选择刷新可用的 LeftValue 字段。
16. 在基于所选连接类型显示的字段中选择虚拟对象。
17. 为 RightValue 字段重复步骤 14-16。
18. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
19. 单击保存。应用程序将保存更改并显示成功消息。
