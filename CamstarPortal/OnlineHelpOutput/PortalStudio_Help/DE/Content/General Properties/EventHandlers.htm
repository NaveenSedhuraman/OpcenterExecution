Definieren von Ereignishandlern

Mithilfe von Ereignishandlern (Event Handlers) können Sie die Ausführung von Aktionen auf Basis von Ereignissen konfigurieren und benutzerspezifische Bedingungen angeben.

Beim Definieren von Ereignishandlern

Ereignishandler sind objektspezifisch und entweder generisch oder bedingt. Generische Ereignishandler (Generic Event Handlers) konfigurieren Aktionen, deren Ausführung auf vordefinierten Ereignissen basiert. Bedingte Ereignishandler (Conditional Event Handlers) konfigurieren Aktionen und ermöglicht die Konfiguration von zusätzlichen Bedingungen. Dabei werden neben einem Ereignis auch Vergleiche zwischen linken und rechten Werten verwendet.

Das Bedingungsfeld und die zugehörigen Felder werden nur dann im Popup "Ereignishandler" (EventHandlers) angezeigt, wenn Sie bedingte Ereignishandler definieren.

Optional können Sie im Popup "Ereignishandler" (EventHandlers) zusätzliche Parameter und Initiatoren definieren. Weitere Informationen zu den Feldern, die zum Definieren von Parametern und Initiatoren verwendet werden, finden Sie unter "Felddefinitionen des Popups "Parameter" (Parameters)" und "Felddefinitionen des Popups "Initiatoren" (Initiators)".

Die Felder, die in bedingten Ereignishandlern zum Definieren der linken und rechten Werte verwendet werden, sind dynamische Felder und basieren auf dem im Feld "Verbindungstyp" (ConnectionType) ausgewählten Verbindungstyp. Der Verbindungstyp bestimmt, welche zusätzlichen Felder angezeigt werden.

Vorgehensweise zum Definieren eines generischen Ereignishandlers

Gehen Sie folgendermaßen vor, um einen generischen Ereignishandler zu definieren:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Katalog der Benutzeroberflächenkomponenten ausgeblendet ist. Der Katalog der Benutzeroberflächenkomponenten wird angezeigt.
3. Doppelklicken Sie auf eine Seite. Die Seite wird im Arbeitsbereich angezeigt.
4. Wählen Sie auf der Seite ein virtuelles Objekt aus. Die Eigenschaften für dieses Objekt werden im Bereich "Eigenschaften" (Properties) angezeigt.
5. Klicken Sie im Bereich "Eigenschaften" (Properties) auf Mehr (More) für Ereignishandler (EventHandlers). Das Popup "Ereignishandler" (EventHandlers) wird angezeigt.
6. Wählen Sie in der Dropdownliste über dem linken Bereich Generischer Ereignishandler (GenericEventHandler) aus und klicken Sie auf Hinzufügen (Add). Im rechten Bereich werden leere Felder angezeigt.
7. Wählen Sie im Feld Ereignis (Event) ein Ereignis aus.
8. Wählen Sie im Feld Handler-Quelle (HandlerSource) eine Handler-Quelle aus.
9. Wählen Sie im Feld Handler einen Handler aus.
10. Geben Sie im Feld Name einen Namen für den Ereignishandler ein.
11. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
12. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.

Vorgehensweise zum Definieren eines bedingten Ereignishandlers

Gehen Sie folgendermaßen vor, um einen bedingten Ereignishandler zu definieren:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Katalog der Benutzeroberflächenkomponenten ausgeblendet ist. Der Katalog der Benutzeroberflächenkomponenten wird angezeigt.
3. Doppelklicken Sie auf eine Seite. Die Seite wird im Arbeitsbereich angezeigt.
4. Wählen Sie auf der Seite ein virtuelles Objekt aus. Die Eigenschaften für dieses Objekt werden im Bereich "Eigenschaften" (Properties) angezeigt.
5. Klicken Sie im Bereich "Eigenschaften" (Properties) auf Mehr (More) für Ereignishandler (EventHandlers). Das Popup "Ereignishandler" (EventHandlers) wird angezeigt.
6. Wählen Sie in der Dropdownliste über dem linken Bereich Bedingter Ereignishandler (ConditionalEventHandler) aus und klicken Sie auf Hinzufügen (Add). Im rechten Bereich werden leere Felder angezeigt.
7. Wählen Sie im Feld Ereignis (Event) ein Ereignis aus.
8. Wählen Sie im Feld Handler-Quelle (HandlerSource) eine Handler-Quelle aus.
9. Wählen Sie im Feld Handler einen Handler aus.
10. Geben Sie im Feld Name einen Namen für den Ereignishandler ein.
11. Erweitern Sie den Abschnitt Bedingung (Condition).
12. Erweitern Sie den Abschnitt Gleichwertigkeit (Equivalency). 
13. Wählen Sie im Feld Vergleich (Comparison) einen Vergleichsoperator aus.
14. Erweitern Sie den Abschnitt Linker Wert (LeftValue).
15. Wählen Sie im Feld Verbindungstyp (ConnectionType) einen Verbindungstyp aus. Die verfügbaren Felder für "Linker Wert" (LeftValue) werden auf der Grundlage Ihrer Auswahl aktualisiert.
16. Wählen Sie ein virtuelles Objekt aus dem Feld aus, das auf der Grundlage des ausgewählten Verbindungstyps angezeigt wurde.
17. Wiederholen Sie die Schritte 14-16 für die Felder von "Rechter Wert" (RightValue).
18. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
19. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
