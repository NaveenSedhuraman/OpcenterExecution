Eigenschaftsbedingungen

Eigenschaftsbedingungen (Property Conditions) bieten Ihnen die Möglichkeit, eine Steuerelementeigenschaft mithilfe mehrerer benutzerdefinierter Bedingungen und Trigger festzulegen oder zu ändern.

Beim Festlegen der Eigenschaftsbedingungen

Eigenschaftsbedingungen sind Vergleiche zwischen linken und rechten Werten, die als Bedingungssätze bezeichnet werden. Bedingungssätze werden ausgewertet und bestimmen den Wert eines anderen Elements, wenn die Auswertung des Vergleichs "Wahr" (True) ergibt.

Eine Eigenschaftsbedingung kann eine einzelne Eigenschaftsbedingung mit nur einem Bedingungssatz sein. Sie kann jedoch auch mehrere Gruppen von Bedingungssätzen enthalten, die durch Und- und Oder-Operatoren (And/Or) getrennt sind. Eine Eigenschaftsbedingung muss mindestens einen Bedingungssatz und eine Wert-wenn-Wahr-Bedingung (Value If True) enthalten. Beispiel: Ein einfacher Bedingungssatz könnte überprüfen, ob zwei Steuerelemente gleiche Werte auf einer Seite enthalten. Dabei kann eine Wert-wenn-Wahr-Bedingung angewandt werden, die ein weiteres Steuerelement auf diesen Wert festlegt, wenn die Auswertung des Bedingungssatzes "Wahr" ergibt.

Eigenschaftsbedingungen sind in einer Baumstruktur organisiert. Auf der obersten Ebene enthält jedes Steuerelement Eigenschaftsbedingungen, die im Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions) im Bereich "Eigenschaftsbedingungen" (Property Conditions) angezeigt werden.

Innerhalb der einzelnen Eigenschaftsbedingungen können Sie Folgendes definieren:

• Ein Bedingungssatz mit linken und rechten Werten. Von dieser Ebene aus können Sie Folgendes definieren:
• Initiatoren, die Ereignisse enthalten, welche angeben, wann die linken oder rechten Werte des Bedingungssets ausgewertet werden.
• Durch Und- und Oder-Operatoren (And/Or) getrennt Bedingungsgruppen zum Hinzufügen weiterer Bedingungssets. Bedingungsgruppen können ihre eigenen Initiatoren und nach Bedarf auch weitere Unterbedingungsgruppen enthalten.
• Und die Wert-wenn-Wahr-Bedingungen, die festgelegt werden, wenn die Auswertung der Bedingungssätze "Wahr" ergibt. Von dieser Ebene aus können Sie Folgendes definieren:
• Initiatoren, die Ereignisse enthalten, welche angeben, wann eine Wert-wenn-Wahr-Bedingung festgelegt werden sollte.

Diese Struktur ermöglicht es Ihnen, so viele Bedingungssätze, Initiator-Trigger und Ergebnisse zu erstellen wie erforderlich.

Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions)

Das Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions) wird angezeigt, wenn Sie im Bereich "Eigenschaften" (Properties) auf die Schaltfläche "Mehr" (More) für "Eigenschaftsbedingungen" (PropertyConditions) klicken, während im Arbeitsbereich ein Steuerelement ausgewählt ist. Weitere Informationen zum Anzeigen des Bereichs "Eigenschaften" (Properties) finden Sie unter "Bereich "Eigenschaften" (Properties)".

Im Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions) können Sie neue Eigenschaftsbedingungen erstellen oder vorhandene Eigenschaftsbedingungen anzeigen. Im Bereich "Eigenschaftsbedingungen" (Property Conditions) werden die Eigenschaftsbedingungen angezeigt, die derzeit für das Steuerelement festgelegt sind. Sie können auf den Pfeil klicken, um die einzelnen Eigenschaftsbedingungen zu erweitern und zu reduzieren. So können Sie den Bedingungssatz, die Bedingungsgruppen und das Feld "Für Eigenschaft" (For Property) für die Eigenschaftsbedingung anzeigen. Zur besseren Lesbarkeit werden Bedingungsgruppen eingerückt und Unterbedingungsgruppen weiter eingerückt dargestellt.

Jeder Eigenschaftsbedingung im Bereich "Eigenschaftsbedingungen" (Property Conditions) ist ein Feld "Name" und ein Feld "Für Eigenschaft" (For Property) zugeordnet. Wenn Sie eine neue Eigenschaftsbedingung hinzufügen, müssen Sie im Feld "Für Eigenschaft" (For Property) eine Eigenschaft auswählen. Das Feld "Für Eigenschaft" (For Property) bestimmt, welche Eigenschaft des Steuerelements den Wert der Wert-wenn-Wahr-Bedingung erhält. Sie können die Namen vorhandener Eigenschaftsbedingungen im Bereich "Eigenschaftsbedingungen" (Property Conditions) bearbeiten. Wählen Sie dazu den Namen der Eigenschaftsbedingung aus und geben Sie einen neuen Namen ein.

Im Bereich "Eigenschaftsbedingungen" (Property Conditions) werden unter den Eigenschaftsbedingungen die Schaltflächen "Bedingungen bearbeiten..." (Edit Conditions...) und "Wert bearbeiten wenn Wahr" (Edit Value if True...) angezeigt. Wenn Sie auf die Schaltfläche

• "Bedingungen bearbeiten..." (Edit Conditions...) klicken, wird das Popup "Bedingungseditor" (Conditions Editor) angezeigt. Weitere Informationen finden Sie unter "Bedingungseditor (Conditions Editor)".
• "Wert bearbeiten wenn Wahr..." (Edit Value If True...) klicken, wird das Popup "Wert-wenn-Wahr-Editor" (Value If True Editor) angezeigt. Weitere Informationen finden Sie unter "Wert-wenn-Wahr-Editor (Value If True Editor)".

Sie können eine Eigenschaftsbedingung löschen, indem Sie auf "X" rechts neben der Eigenschaftsbedingung klicken.

Das Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions) wird geschlossen, wenn Sie auf die Schaltfläche "Abbrechen" (Cancel) oder auf "X" oben rechts in der Ecke klicken.

Bedingungseditor (Conditions Editor)

Das Popup "Bedingungseditor" (Conditions Editor) wird angezeigt, wenn Sie im Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions) auf die Schaltfläche "Bedingungen bearbeiten..." (Edit Conditions...) klicken. Im Popup "Bedingungseditor" (Conditions Editor) können Sie ein Eigenschaftsbedingungsset sowie mehrere Bedingungsgruppen und Unterbedingungsgruppen erstellen.

Wählen Sie zum Erstellen eines Bedingungssets in der ersten Dropdownliste einen Vergleichsoperator aus. Dadurch legen Sie fest, wie das Bedingungsset den linken und rechten Wert miteinander vergleicht. Wählen Sie dann linke und rechte Werte aus, die ein Objekt referenzieren, z. B. ein Steuerelement, einen Datenvertrag usw.

Sie können nach dem Definieren einer ersten Bedingungsmenge weitere Bedingungsmengen definieren. Klicken Sie hierzu auf:

• "UND-Gruppe hinzufügen" (Add AND Group). Hierdurch wird eine zusätzliche Menge von Feldern zur Definition eines weiteren Bedingungssets angezeigt, das vom ersten Set durch einen Und-Operator getrennt wird; oder auf
• "ODER-Gruppe hinzufügen" (Add OR Group). Hierdurch wird eine zusätzliche Menge von Feldern zur Definition eines weiteren Bedingungssets angezeigt, das vom ersten Set durch einen Oder-Operator getrennt wird.

Bedingungsgruppen können mehrere Bedingungssets enthalten, die den Und- bzw. Oder-Operator für die betreffende Gruppe referenzieren. Sie können weitere Bedingungsgruppen erstellen, die nach Bedarf auch mehrere Bedingungssets enthalten können. Zur besseren Lesbarkeit werden Bedingungsgruppen und Unterbedingungsgruppen eingerückt dargestellt.

Sie können ein Bedingungsset löschen, indem Sie auf "X" rechts neben dem Bedingungsset klicken. Sie können eine Bedingungsgruppe entfernen, indem Sie auf die Schaltfläche "UND-Gruppe entfernen" (Remove AND Group) oder "ODER-Gruppe entfernen" (Remove OR Group) klicken. Das einzelne erste Bedingungsset können Sie nicht löschen.

Note: Wenn Sie auf "UND-Gruppe entfernen" (Remove AND Group) oder "ODER-Gruppe entfernen" (Remove OR Group) klicken, werden alle Bedingungssets innerhalb der Gruppe gelöscht. Klicken Sie auf "X" rechts neben einem Bedingungsset, um nur dieses Set zu löschen.

Initiatoreditor (Initiator Editor)

Das Popup "Initiatoreditor" (Initiator Editor) ist sowohl über das Popup "Bedingungseditor" (Conditions Editor) als auch über das Popup "Wert-wenn-Wahr-Editor" (Value If True Editor) zugänglich. Das blaue Feld in der rechten Ecke der Schaltfläche "Initiatoren..." (Initiators...) enthält die Anzahl der derzeit für den Wert festgelegten Initiatoren.

Im Popup "Initiatoreditor" (Initiator Editor) können Sie Trigger definieren, um zu bestimmen, wann Portal Studio die linken oder rechten Werte (Left Values/Right Values) in einem Bedingungsset oder Wert-wenn-Wahr-Bedingungen auswertet. Sie können z. B. einen Trigger angeben, der den zugehörigen linken Wert (Left Value) nur auswertet, wenn der Wert eines bestimmten Steuerelements auf der Seite geändert wurde. Sie können mehrere Initiatoren festlegen, um genauer zu bestimmen, wann ein Bedingungssetwert oder eine Wert-wenn-Wahr-Bedingung ausgewertet werden soll.

Initiatoren können optional angegeben werden. Zum Definieren einer Eigenschaftsbedingung sind sie nicht erforderlich.

Sie können einen Initiator löschen, indem Sie auf "X" rechts neben dem Initiator klicken.

Vorgehensweise zum Zugreifen auf das Popup "<Steuerelement> – Eigenschaftsbedingungen" (Property Conditions)

Gehen Sie folgendermaßen vor, um das Popup "<Steuerelement> - Eigenschaftsbedingungen" (PropertyConditions) zu öffnen:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Katalog der Benutzeroberflächenkomponenten ausgeblendet ist. Der Katalog der Benutzeroberflächenkomponenten wird angezeigt.
3. Doppelklicken Sie auf ein Webpart oder auf eine Seite, die ein Webpart enthält. Das Webpart bzw. die Seite wird im Arbeitsbereich angezeigt.
4. Klicken Sie auf Eigenschaften anzeigen/ausblenden (Show/Hide Properties), wenn der Bereich "Eigenschaften" (Properties) ausgeblendet ist. Der Bereich "Eigenschaften" (Properties) wird angezeigt.
5. Wählen Sie im Arbeitsbereich ein Steuerelement aus.
6. Klicken Sie auf "Mehr" (More) für Eigenschaftsbedingungen (Property Conditions). Das Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions) wird angezeigt.

Vorgehensweise zum Definieren einer Eigenschaftsbedingung

Gehen Sie folgendermaßen vor, um eine Eigenschaftsbedingung zu definieren:

1. Führen Sie die Prozedur unter "Vorgehensweise zum Zugreifen auf das Popup "<Steuerelement> – Eigenschaftsbedingungen" (Property Conditions)" aus.
2. Geben Sie im Feld Name der Eigenschaftsbedingung eingeben (Enter Property Condition Name) einen Namen für die Eigenschaftsbedingung ein und klicken Sie auf Eigenschaftsbedingung hinzufügen (Add Property Condition). Die Eigenschaftsbedingung wird im Bereich "Eigenschaftsbedingungen" (Property Conditions) angezeigt.
3. Erweitern Sie die Eigenschaftsbedingung und wählen Sie im Feld Für Eigenschaft (For Property) eine Eigenschaft aus.
4. Klicken Sie auf Bedingungen bearbeiten... (Edit Conditions...). Das Popup "Bedingungseditor" (Conditions Editor) wird angezeigt.
5. Wählen Sie in der ersten Dropdownliste einen Vergleichsoperator aus.
6. Wählen Sie im Feld Verbindungstyp (ConnectionType) einen Verbindungstyp für den linken Wert (Left Value) aus. Die Felder für den linken Wert werden entsprechend dem ausgewählten Verbindungstyp aktualisiert.
7. Geben Sie Informationen in die Felder ein, die Ihren Geschäftsanforderungen entsprechend angezeigt werden. Informationen zu Pflichtfeldern und optionalen Feldern finden Sie in der Felddefinitionstabelle.
8. Wiederholen Sie die Schritte 5 bis 7 für die Felder für "Rechter Wert" (Right Value).
9. Klicken Sie auf OK. Das Popup "Bedingungseditor" (Conditions Editor) wird geschlossen.
10. Klicken Sie auf Wert bearbeiten wenn Wahr... (Edit Value If True...). Das Popup "Wert-wenn-Wahr-Editor" (Value If True Editor) wird angezeigt.
11. Wählen Sie im Feld Verbindungstyp (ConnectionType) einen Verbindungstyp aus. Die Felder werden entsprechend dem ausgewählten Verbindungstyp aktualisiert.
12. Geben Sie Informationen in die Felder ein, die Ihren Geschäftsanforderungen entsprechend angezeigt werden. Informationen zu Pflichtfeldern und optionalen Feldern finden Sie in der Felddefinitionstabelle.
13. Klicken Sie auf OK. Das Popup "Wert-wenn-Wahr-Editor" (Value If True Editor) wird geschlossen.
14. Klicken Sie auf OK. Das Popup "<Steuerelement> – Eigenschaftsbedingungen" (PropertyConditions) wird geschlossen und Ihre Änderungen werden gespeichert.
