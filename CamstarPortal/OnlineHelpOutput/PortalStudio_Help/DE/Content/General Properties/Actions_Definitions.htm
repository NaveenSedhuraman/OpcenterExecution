Aktionseigenschaften

Mit Aktionseigenschaften können Sie die Eigenschaften von lokalen Aktionen für Seiten, Webparts und andere Objekte anzeigen oder bearbeiten. Aktionseigenschaften werden im Popup "<Objekt> - Aktion" (Action) angezeigt, wenn Sie im Bereich "Eigenschaften" (Properties) auf die Schaltfläche "Mehr" (More) für "Aktionen" (Actions) klicken. Weitere Informationen zum Bereich "Eigenschaften" (Properties) finden Sie unter "Bereich "Eigenschaften" (Properties)".

Aktionseigenschaften, die für den derzeit ausgewählten Aktionstyp nicht benötigt werden, sind im Popup "<Objekt> - Aktion" (Action) ausgeblendet.

Diese Tabelle enthält Beschreibungen zu den Aktionseigenschaften.

Aktionseigenschaft

Beschreibung der Eigenschaft

ActionCategory

Gibt eine Aktionskategorie (ActionCategory) an, die der Aktion zugewiesen werden soll.

ButtonPosition

Bestimmt, ob die Aktionsschaltfläche links oder rechts auf einer Seite angezeigt wird, wenn das Feld "Standort" (Location) auf "Schaltfläche" (Button) festgelegt wird.

ClearValues

Löscht Werte, wenn die Aktion eingereicht wird.

CloseButtonOnly

Veraltet.

CommandName

Die Eigenschaft "Befehlsname" (CommandName) wird verwendet, um Logikinformationen für Ereignisargumente und JavaScript bereitzustellen.

ConditionHandler

Optionen: "DefaultConditionHandler" oder "SpecificConditionHandler". Definiert die Handlermethode im Code, der aufgerufen wird, wenn die Aktion eingereicht wird. Auf diese Weise können Sie zusätzliche Funktionalität aus dem App-Code ausführen, wenn die Aktion eingereicht wird. 

Confirmation

Gruppe von Eigenschaften zum Konfigurieren der Beschriftungen, die in einer Bestätigungsnachricht angezeigt werden.

Bestätigung – Allgemein (Confirmation – General)

Cancel_LabelName

Legt den Text fest, der auf der Schaltfläche "Abbrechen" (Cancel) der Bestätigungsnachricht angezeigt wird.

Cancel_LabelText

Legt den Text fest, der auf der Schaltfläche "Abbrechen" (Cancel) der Bestätigungsnachricht angezeigt wird. Dieses Feld ersetzt den für die Untereigenschaft "Beschriftungsname für Abbrechen" (Cancel_LabelName) eingegebenen Text.

Message_LabelName

Legt den Text der Bestätigungsnachricht fest.

Message_LabelText

Legt den Text der Bestätigungsnachricht fest. Dieses Feld ersetzt den für die Untereigenschaft "Beschriftungsname für Nachricht" (Message_LabelName) eingegebenen Text.

OK_LabelName

Legt den Text fest, der auf der Schaltfläche "OK" der Bestätigungsnachricht angezeigt wird.

OK_LabelText

Legt den Text fest, der auf der Schaltfläche "OK" der Bestätigungsnachricht angezeigt wird. Dieses Feld ersetzt den für die Untereigenschaft "OK_Beschriftungsname" (OK_LabelName) eingegebenen Text.

Title_LabelName

Legt den Text fest, der als Titel der Bestätigungsnachricht angezeigt wird.

Title_LabelText

Legt den Text fest, der als Titel der Bestätigungsnachricht angezeigt wird. Dieses Feld ersetzt den für die Untereigenschaft "Titel_Beschriftungsname" (Title_LabelName) eingegebenen Text.

CssClass

Gibt eine benutzerdefinierte CSS-Klasse für die Aktion an.

CustomMethodHandler

Optionen: "ExecuteMaintenanceTransaction", "ExecuteNewTransaction", "ShopfloorReset", "TrueSubmit", "VIewDocument", "VIewInExcel" oder "WebPartCustomAction".

DataContractLinks

An die verknüpfte Seite zu übergebende Datenvertragselemente.

DataContractReturnLinks

Von der unverankerten Seite zurückgegebene Vertragsdatenelemente.

DenyDataContract

Hebt alle der Aktion zugewiesenen Datenverträge auf.

Description

Beschreibung der Aktion.

DisableCondition

Eigenschaftsbedingungs-Ausdruck, der zur Deaktivierung der Aktion führt, wenn die Auswertung des Ausdrucks "Wahr" (True) ergibt.

Deaktivierungsbedingung – Beschreibung (DisableCondition – Description)

Expression

Der bedingte Ausdruck, der ausgewertet wird.

Deaktivierungsbedingung – Gleichwertigkeit (DisableCondition – Equivalency)

ColumnName

Name der für den Vergleich verwendeten Spalte.

Comparison

Zur Auswertung einer Bedingung verwendeter Vergleichsoperator. Gültige Einträge: "Gleich" (Equal), "Ungleich" (NonEqual) und "Enthält" (Contains).

Deaktivierungsbedingung – Gleichwertigkeit – Linker Wert – Allgemein (DisableCondition – Equivalency – LeftValue – General)

ConnectionType

Gibt den Verbindungstyp des Ereignishandlers an.

Anmerkung: Je nach ausgewähltem Verbindungstyp werden zusätzliche Felder angezeigt.

Steuerung

Gibt ein Steuerelement an, das auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Steuerelement" (Control) ausgewählt wird.

PageDataContract

Gibt einen Seitendatenvertrag an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Seitendatenvertrag" (PageDataContract) ausgewählt wird.

SessionDataContract

Gibt einen Sitzungsdatenvertrag an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Sitzungsdatenvertrag" (SessionDataContract) ausgewählt wird.

SessionVariable

Gibt eine Sitzungsvariable an, die auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Sitzungsvariable" (SessionVariable) ausgewählt wird.

FieldExpression

Gibt einen Feldausdruck an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Feldausdruck" (FieldExpression) ausgewählt wird.

StaticValue

Gibt einen statischen Wert (StaticValue) an, der auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Statischer Wert" (StaticValue) ausgewählt wird.

GridItem

Gibt ein Tabellenelement an, das auf einer bestimmten Seite der Bedingungsauswertung verwendet werden soll.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Tabellenelement" (GridItem) ausgewählt wird.

Property

Gibt den Steuerelementeigenschaftswert, der ausgewertet wird, an.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Steuerelement" (Control) ausgewählt wird.

RequestDataMode

Gibt die Methode an, mit der der Eigenschaftswert eines Steuerelements abgerufen wird.

Anmerkung: Dieses Feld wird angezeigt, wenn im Feld "Verbindungstyp" (ConnectionType) der Eintrag "Steuerelement" (Control) ausgewählt wird.

AllowEmptyValue

Bestimmt, ob der Wert leer sein darf.

Initiators

Das Popup "Initiatoren" (Initiators) wird angezeigt, wenn Sie auf die Schaltfläche "More" (Mehr) klicken.

Deaktivierungsbedingung – Gleichwertigkeit – Rechter Wert (DisableCondition – Equivalency – RightValue)

Hinweis: Die Felder für "Rechter Wert" (RightValue) sind identisch mit den Feldern für "Linker Wert" (LeftValue). Felddefinitionen finden Sie in der Beschreibung der Felder für "Deaktivierungsbedingung – Gleichwertigkeit – Linker Wert – Allgemein (DisableCondition – Equivalency – LeftValue – General)".

Deaktivierungsbedingung – Allgemein (DisableCondition – General)

And

Das Popup "Und" (And) wird angezeigt, wenn Sie auf die Schaltfläche "More" (Mehr) klicken. Sie können zusätzliche Bedingungen für einen bedingten Ereignishandler definieren, die mit einem "And"-Operator hinzugefügt werden.

Hinweis: Die Felder im Popup "Und" (And) sind identisch mit den Feldern, die angezeigt werden, wenn ein bedingter Ereignishandler definiert wird. Felddefinitionen finden Sie in der Beschreibung der Felder für "DisableCondition" (Deaktivierungsbedingung).

Oder

Das Popup "Oder" (Or) wird angezeigt, wenn Sie auf die Schaltfläche "More" (Mehr) klicken. Sie können zusätzliche Bedingungen für einen bedingten Ereignishandler definieren, die mit einem "Or"-Operator hinzugefügt werden.

Hinweis: Die Felder im Popup "Oder" (Or) sind identisch mit den Feldern, die angezeigt werden, wenn ein bedingter Ereignishandler definiert wird. Felddefinitionen finden Sie in der Beschreibung der Felder für "DisableCondition" (Deaktivierungsbedingung).

DynamicDisplayMode

Bestimmt den Anzeigemodus der Aktion für Benutzer mit Rollen, die nicht über die Berechtigung zum Anzeigen der Aktion verfügen.

EndResponse

Wird intern von Aktionen für unverankerte Rahmen verwendet, um die Informationen für die Aufruflistenumleitung festzulegen. Der Wert "Wahr" (True) folgt der Standard-Frameworkfunktionalität für Aufruflisten unverankerter Rahmen. Der Wert "Falsch" (False) macht die FloatingPageCallStackMethod in JavaScript verfügbar.

ESignatureDependencies (Collection)

Sammlung elektronischer Unterschriften, die zum Einreichen der Aktion erforderlich sind.

ESignatureErforderlich

Ist eine elektronische Unterschrift erforderlich?

ForceRedirect

Bei Festlegung auf "Wahr" (True) werden bei der Umleitung zur Zielseite eventuell erforderliche Feldvalidierungen umgangen.

FormValidate

Gibt an, ob vor dem Einreichen des Formulars die Formularvalidierung aktiviert werden soll (implementiert in HTML5).

FrameLocation

Ermöglicht die Festlegung der Abmessungen, die die unverankerte Seite beim Laden aufweist.

Rahmenposition – Allgemein (FrameLocation – General)

Height

Höhe in Pixel.

Width

Breite in Pixel.

GroupName

Name der Aktionsgruppe.

HiddenCondition

Die Bedingung, durch deren Auswertung ermittelt wird, ob die Seitenaktion auf einer Seite ausgeblendet wird.

Hinweis: Beim Definieren eines bedingten Ausdrucks sind die Untereigenschaften von "Ausblendebedingung" (HiddenCondition) identisch mit den Untereigenschaften von "Deaktivierungsbedingung" (DisableCondition). Beschreibungen der Eigenschaften finden Sie in der Beschreibung der Felder für "DisableCondition" (DisableCondition).

HorizontalShift

Reihenfolge der Verschiebung: von links (Left) nach rechts (Right)

Horizontale Verschiebung – Allgemein (HorizontalShift – General)

HorizontalShiftDirection

Gibt die Richtung für die Anpassung der horizontalen Position der Aktion zur Laufzeit an. Gültige Werte sind: "Links" (Left), "Rechts" (Right) und "Kein" (None).

HorizontalShiftValue

Die Anzahl der Pixel, um die die horizontale Position der Aktion angepasst wird.

Index

Ganzzahl, die verwendet wird, um die Anzeige der Aktionen anzuordnen.

InitContractHandler

Wird zum Zuweisen einer Methode verwendet, die aufgerufen wird, wenn bei der Aktionsverarbeitung Datenverträge der Zielseite initialisiert werden.

IsDisabled

Ist die Aktion deaktiviert?

IsHidden

Ist die Aktion ausgeblendet?

IsPrimary

Ist die Aktion die primäre Aktion?

IsReturn

Schließt die Seite, nachdem sie eingereicht wurde.

LabelName

Name der Wörterbuchbeschriftung für die Aktion.

LabelText

Beschriftungstext für die Aktion. Dieses Feld ersetzt den für die Eigenschaft "Beschriftungsname" (LabelName) eingegebenen Text.

Location

Bestimmt, wo auf der Seite die Aktion angezeigt wird. Bei Auswahl von "Schaltfläche" (Button) wird die Aktion unten auf den zugewiesenen Seiten als Schaltfläche angezeigt. Bei Auswahl von "Bedientafel" (ActionPanel) wird die Aktion auf zugewiesenen Seiten im Webpart "Aktionssteuerung" (ActionsControl) angezeigt.

MapItem

Gibt die Seitenzuordnung an, die beim Einreichen der Aktion zu verwenden ist. Seitenzuordnungen werden auf der Registerkarte "Modellierung" (Modeling) in Portal Studio verwaltet. Sie dienen zum Definieren von Quellseiten, Zielen und Parametern.

Name

Name der Aktion.

PageflowName

Name der zu öffnenden Seitenfolge (Pageflow).

PageName

Name der zu öffnenden Seite.

Parameters

Ermöglicht die Eingabe eines Zeichenkettenwerts, der in der WebPartCustomAction-Funktionsüberschreibung in der App_Code-Datei für eine Seite verwendet werden kann. Anhand von Parametern wird in der Regel festgestellt, welche Schaltfläche angeklickt wurde, wenn eine Seite mehrere benutzerdefinierte Aktionen enthält.

Berechtigungen – Allgemein (Permissions – General)

DisplayMode

Optionen: "Disable" oder "Hide". Verwendeter Anzeigemodus, wenn die Berechtigungen nicht vorliegen.

PagePermission

Seitenberechtigungen anwenden?

ServicePermission

Serviceberechtigungen anwenden?

PortalTab

Im Feld "Portal-Registerkarte" (PortalTab) können vorhandene Registerkarten ausgewählt werden, wenn für "Portal-Registerkarten-Optionen" (PortalTabOptions) der Eintrag "Bestimmte Registerkarte" (SpecificTab) ausgewählt wird.

PortalTabOptions

Optionen: "CurrentTab", "NewTab" oder "SpecificTab". Optionen für die Registerkarte, auf der die neue Seite geöffnet werden soll.

QueryStringParameters(Collection)

Der Benutzer kann Datenvertragselemente der Seite Abfrageparametern zuordnen, die an die Link-URL angefügt werden, wenn der Link geöffnet wird.

ReloadValues

Bei Festlegung auf "Wahr" (True) lädt die Aktion die Seitenwerte nach dem Einreichen neu.

ServiceName

Service, der beim Klicken auf die Aktion ausgeführt wird.

ShowButtons

Zeigt die Schaltflächen "Erste" (First), "Weiter" (Next), "Vorh." (Prev) und "Letzte" (Last) an.

SPCChild

Startet SPC im Eltern-Rahmen, falls der Aufruf von einem Kind erfolgt.

SPCEnabled

Aktiviert SPC-Aktionen.

URL

URL, die beim Klicken auf die Aktion ausgeführt wird.

WIPMessagesErforderlich

Gibt an, ob für die Aktion eine Umlaufbestandsnachricht erforderlich ist.
