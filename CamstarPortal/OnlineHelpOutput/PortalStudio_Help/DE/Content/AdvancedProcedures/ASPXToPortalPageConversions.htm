Konvertieren von ASPX-Dateien in Portalseiten

Portal Studio bietet Ihnen die Möglichkeit, ASPX-Dateien automatisch in Portalseiten zu konvertieren, die weiter bearbeitet und als Portalseiten verwendet werden können.

Beim Konvertieren von ASPX-Dateien in Portalseiten

Nachstehend finden Sie einige Anmerkungen zum Konvertieren von ASPX-Dateien in Portalseiten:

• Das Konvertierungsdienstprogramm konvertiert Steuerelemente auf den Web App-Seiten in möglichst genaue Entsprechungen in Portal Studio. Dabei werden Webparts für logische Abschnitte erstellt.
• Web App-Schaltflächen mit "In Rahmen anzeigen=Wahr" (DisplayInFrame=True) werden in Seitenaktionsschaltflächen konvertiert. Bei "In Rahmen anzeigen=Falsch" (DisplayInFrame=False) werden diese Schaltflächen in Schaltflächen-Steuerelemente auf der Seite konvertiert.
• Abhängigkeiten werden nach Möglichkeit beibehalten, z. B. wenn ein Ressourcenfeld auf der Web App-Seite die Eigenschaft "Abhängig von" (DependsOn) in Bezug auf das Feld "Container" aufweist.
• Der einer Seite zugrunde liegende Code wird nicht konvertiert, da das Dienstprogramm nur die ASPX-Datei konvertiert.
• Die Batchkonvertierung von ASPX-Dateien wird nicht unterstützt.

Vorgehensweise zum Konvertieren von ASPX-Dateien in Portalseiten

Gehen Sie folgendermaßen vor, um eine ASPX-Datei in eine Portalseite zu konvertieren.

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf die Schaltfläche ASPX-Seite konvertieren (Convert ASPX Page). Das Popup "Öffnen" (Open) wird angezeigt.
3. Navigieren Sie zu einer ASPX-Datei, die konvertiert werden soll, wählen Sie sie aus und klicken Sie auf die Schaltfläche Öffnen (Open). Portal Studio führt automatisch die Konvertierung durch und speichert die Seite. Eine Erfolgsmeldung mit der Angabe, dass die Datei erfolgreich konvertiert wurde, wird angezeigt.
Note: Die konvertierte Seite wird in der Liste der Seiten mit dem Präfix "ASPX_" angezeigt. Sie können die Seite im Bereich "Eigenschaften" (Properties) umbenennen.
