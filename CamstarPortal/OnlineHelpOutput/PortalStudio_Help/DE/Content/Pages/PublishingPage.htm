Veröffentlichen einer Seite

Mit der Veröffentlichung einer Seite machen Sie Ihre Änderungen außerhalb Ihres Arbeitsbereichs und im Portal verfügbar. Wenn Sie Inhalt veröffentlichen, können auch andere Portal Studio-Benutzer auf diesen Inhalt zugreifen.

Beim Veröffentlichen einer Seite

Seiten können erst veröffentlicht werden, nachdem sie gespeichert wurden. Nicht veröffentlichte Seiten sind durch ein Bleistiftsymbol auf dem Symbol der Seite im Katalog der Benutzeroberflächenkomponenten gekennzeichnet.

Die Schaltfläche "Veröffentlichen" (Publish) ist nur im höchsten aktiven Arbeitsbereich und im Entwickler-Arbeitsbereich aktiv. Die gespeicherten Änderungen, die im Entwicklerarbeitsbereich veröffentlicht werden, werden im höchsten aktiven Arbeitsbereich veröffentlicht. Weitere Informationen finden Sie unter "Entwickeln in Arbeitsbereichen".

Sie können eine Seite auf ihren zuletzt veröffentlichten Zustand zurücksetzen. Klicken Sie dazu auf "Zurücksetzen auf" (Revert to) und wählen Sie "Veröffentlichter Inhalt" (Published Content) aus. Weitere Informationen finden Sie unter "Zurücksetzen auf den zuletzt veröffentlichten Zustand".

Die Anwendung zeigt das Dialogfeld "Veröffentlichen" (Publishing) an. Wenn Sie auf "Veröffentlichen" (Publish) klicken, werden Sie aufgefordert, alle mit der zu veröffentlichenden Seite verbundenen Personalisierungen zu entfernen. Ist das Kontrollkästchen "Alle Personalisierungen für diese Komponente entfernen" (Remove All Personalization Associated with this Component) aktiviert, werden alle Personalisierungen entfernt und der Veröffentlichungsvorgang wird fortgesetzt, wenn Sie auf "OK" klicken. Wenn Sie auf "OK" klicken, ohne das Kontrollkästchen zu aktivieren, wird der Veröffentlichungsvorgang fortgesetzt. Durch Klicken auf "Abbrechen" (Cancel) wird der Veröffentlichungsvorgang abgebrochen.

Beim Veröffentlichen einer neuen Seite wird automatisch eine Seitenberechtigung erstellt und zu der Rolle "Standardseiten" (Default Pages) hinzugefügt. Wenn Sie eine Seite veröffentlichen, die zuvor bereits veröffentlicht wurde, wird keine Seitenberechtigung erstellt. Sie haben die Möglichkeit, die Eigenschaft "Automatische Seitenberechtigung" (AutoPagePermission) im Fenster "Portaleinstellungen" (Portal Settings) in den Standardeinstellungen (DefaultSettings) auf "Falsch" (False) festzulegen. Dadurch verhindern Sie, dass die Anwendung standardmäßig neue Seitenberechtigungen hinzufügt. Weitere Informationen finden Sie unter "Konfigurieren von Portaleinstellungen".

Note: Nachdem Sie eine neue Seite erstellt haben, müssen Sie sich erst am Portal abmelden und erneut anmelden, bevor Sie versuchen, mit der Rolle "Standardseiten" (Default Pages) eine neu veröffentlichte Seite auszuführen oder eine Seitenberechtigung anzuzeigen. Weitere Informationen zu Rollen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Vorgehensweise zum Veröffentlichen einer Seite

Gehen Sie folgendermaßen vor, um eine Seite zu veröffentlichen:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Katalog der Benutzeroberflächenkomponenten ausgeblendet ist. Der Katalog der Benutzeroberflächenkomponenten wird angezeigt.
3. Doppelklicken Sie auf eine Seite. Die Seite wird im Arbeitsbereich angezeigt.
4. Nehmen Sie bei Bedarf Aktualisierungen an der Seite vor.
5. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
6. Klicken Sie auf Veröffentlichen (Publish). Das Dialogfeld "Veröffentlichen" (Publishing) wird angezeigt.
7. Klicken Sie auf OK. Das Dialogfeld "Veröffentlichen" (Publishing) wird geschlossen, die Seite wird im höchsten aktiven Arbeitsbereich veröffentlicht und die Anwendung zeigt eine Erfolgsmeldung an.
Note: Eine weitere Nachricht besagt, dass die Rolle "Standardseiten" (Default Pages) aktualisiert wurde, sofern diese Rolle vorhanden und dem derzeitigen Portal Studio-Benutzer zugewiesen ist.
