Hinzufügen eines Webparts zu einer Seite

Sie können Webparts entweder als lokale Kopie oder als Referenz zu einer Seite hinzufügen.

Beim Hinzufügen eines Webparts zu einer Seite

Steuerelemente und Felder können Sie nicht direkt auf eine Seite ziehen. Steuerelemente und Felder müssen auf einem Webpart platziert werden.

Referenzierte Webparts sind schreibgeschützte Kopien, die auf eine Masterkopie des Webparts verweisen. So können Sie statt mehrerer einzelner Webparts ein Webpart an einem zentralen Standort verwalten.

Generell sollten Sie ein Webpart wie folgt hinzufügen:

• Als Referenz, wenn das Webpart auf allen Seiten identisch ist und an einem zentralen Standort verwaltet wird. Informationen zum Anzeigen einer Liste der Komponenten, die ein Webpart referenzieren, finden Sie unter "Anzeigen der Positionen referenzierter Webparts".
• Als Kopie, wenn das Webpart eine Vorlage ist oder wenn Sie ein Webpart verwenden, um ein ähnliches Webpart zu erstellen.

Beim Testen einer Seite mit einem Referenz-Webpart wird der Entwicklerinhalt des Referenz-Webparts immer angezeigt.

Wenn eine Seite mit einem Referenz-Webpart veröffentlicht wird, wird der veröffentlichte Inhalt des Referenz-Webparts immer angezeigt.

Vorgehensweise zum Hinzufügen eines Webparts zu einer Seite

Gehen Sie folgendermaßen vor, um ein Webpart zu einer Seite hinzuzufügen:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Katalog der Benutzeroberflächenkomponenten ausgeblendet ist. Der Katalog der Benutzeroberflächenkomponenten wird angezeigt.
3. Doppelklicken Sie auf eine Seite. Die Seite wird im Arbeitsbereich angezeigt.
4. Erweitern Sie im Katalog der Benutzeroberflächenkomponenten den Eintrag Webparts.
5. Wählen Sie ein Webpart aus und ziehen Sie es auf eine Zelle. Das Webpart wird in der Zelle angezeigt.
6. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
