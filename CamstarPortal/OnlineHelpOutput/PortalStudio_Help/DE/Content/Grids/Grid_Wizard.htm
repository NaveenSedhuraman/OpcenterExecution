Definieren von Tabellen mit dem Tabellenassistenten

Portal Studio umfasst einen Assistenten, der den Benutzer durch den vollständigen Tabellenerstellungsprozess zum Definieren eines neuen Tabellen-Steuerelements führt.

Beim Definieren von Tabellen mit dem Tabellenassistenten

Der Tabellenassistent (Grid Wizard) wird als Popup angezeigt und enthält eine Reihe von Anzeigen. Die Anzeigen und Felder werden dynamisch angezeigt, je nachdem, welchen Typ von Tabellendatenquelle Sie in der Anfangsanzeige ausgewählt haben und welche Werte Sie beim Definieren der Tabelle eingeben bzw. auswählen.

Das Popup "Tabellen-Steuerelement erstellen" (Grid Control Creation) wird angezeigt, wenn Sie ein Tabellen-Steuerelement auf einem Webpart platzieren. Wenn Sie den Tabellenassistenten auswählen, wird das Popup "Tabellenassistent" (Grid Wizard) angezeigt, das Sie zum Definieren einer Tabelle verwenden können. Wenn Sie "Standardtabelle erstellen" (Create a Default Grid) auswählen, wird eine Standard-Tabellen-Steuerelement platziert. Mit dieser können Sie die Tabelleneigenschaften manuell im Tabelleneinstellungseditor festlegen. Weitere Informationen zum manuellen Definieren einer Tabelle finden Sie unter "Tabelleneinstellungseditor (Grid Settings Editor)".

Mit der Schaltfläche "Weiter" (Next) können Sie zwischen den verschiedenen Anzeigen im Tabellenassistenten navigieren. Mit der Schaltfläche "Zurück" (Back) gelangen Sie zurück zu einer vorherigen Anzeige. Wenn Sie auf "Abbrechen" (Cancel) klicken, wird der Tabellenassistent geschlossen und es wird kein Tabellen-Steuerelement zum Webpart hinzugefügt.

Ein neues Tabellen-Steuerelement wird auf dem Webpart sichtbar, wenn Sie in der letzten Anzeige des Popups "Tabellenassistent" (Grid Wizard) auf die Schaltfläche "Fertigstellen" (Finish) klicken. Ihre Angaben im Tabellenassistenten können Sie im Bereich "Eigenschaften" (Properties) im Popup "Tabelleneinstellungseditor" (Grid Settings Editor) anzeigen oder bearbeiten. Weitere Informationen finden Sie unter "Tabelleneinstellungseditor (Grid Settings Editor)".

Tabellendatenquelle

In der Anfangsanzeige des Tabellenassistenten wählen Sie eine Tabellendatenquelle aus, um festzulegen, welchen Tabellentyp Sie definieren möchten. Je nachdem, welche Tabellendatenquelle Sie auswählen, werden Anzeigen und Felder dynamisch im Tabellenassistenten angezeigt.

Diese Tabelle definiert die Tabellendatenquellen.

Tabellendatenquelle

Beschreibung

Item List

Unterstützt jeden Array-Typ als Datenquelle. Kann an Feldausdrücke gebunden sein und nutzt das reguläre Laden von Werten vom Server. Elementlistentabellen können auch im ungebundenen Modus, z. B. ohne Feldausdrücke, verwendet werden. Im ungebundenen Modus müssen Sie den Elementtyp definieren; Laden und Einreichen von Daten sollten Sie im Code implementieren. Paging innerhalb des Servers ist nicht erlaubt.

SubEntity List

Erbt Daten von der Elementliste. Unterstützt einen bestimmten gebundenen Modus für den aus der Unterobjektklasse abgeleiteten Elementtyp.

Selection Values

Kann im gebundenen Modus verwendet werden und lädt Daten aus der Auswahlwertanforderung. Sie können die aus den ausgewählten Zeilen erstellten Daten einreichen. Die Tabellendatenquelle "Auswahlwerte" (Selection Values) kann zwei Feldausdrücke verwenden, einen zum Laden von Daten und einen zum Einreichen.

Predefined Query

Referenziert eine System- oder Benutzerabfrage. Tabellen vom Typ "Vordefinierte Abfrage" (Predefined Query) bieten die Möglichkeit, Daten mithilfe von Feldausdrücken für ein Feld einzureichen.

AdHoc Query

Speichert die benutzerdefinierte Abfrage direkt auf der virtuellen Seite. Tabellen vom Typ "AdHoc-Abfrage" (AdHoc Query) bieten die Möglichkeit, Daten mithilfe von Feldausdrücken für ein Feld einzureichen.

Einrichten von Spalten im Tabellenassistenten

Im Tabellenassistenten (Grid Wizard) in der Anzeige "Spalten einrichten" (Setup Columns) können Sie die in einer Tabelle angezeigten Spalten erstellen und anpassen.

Die Anzeige "Spalten einrichten" (Setup Columns) wird angezeigt, wenn Sie beim Definieren von Elementlistentabellen (Item List Grids), Unterobjektlistentabellen (SubEntity List Grids) und Auswahlwerttabellen (Selection Values Grids) in der Anzeige "Feldausdruck einrichten" (Field Expression Setup) im Feld "Feldausdrücke" (Field Expressions) oder "Feldausdrücke anfordern" (Request Field Expressions) einen Feldausdruck festgelegt haben. Diese Anzeige wird immer geöffnet, wenn Sie Tabellen vom Typ "Vordefinierte Abfrage" (Predefined Query) und "AdHoc-Abfrage" (AdHoc Query) definieren.

Die Anzeige "Spalten einrichten" (Setup Columns) enthält den Spaltentyp, die ID, die Breite der Spalte in Pixel sowie ein Kontrollkästchen zum Ausblenden der Spalte. Weitere Kontrollkästchen, mit denen die Spalte als bearbeitbar oder erforderlich definiert werden kann, werden angezeigt, wenn Sie Elementlistentabellen und Unterobjektlistentabellen definieren. Sie können die Werte für vorhandene Spalten ändern. Klicken Sie dazu auf ein Feld und geben Sie die neuen Werte ein oder wählen Sie die neuen Werte aus.

Je nachdem, welche Feldausdrücke Sie in den Feldern "Feldausdrücke" (Field Expressions) oder "Feldausdrücke anfordern" (Request Field Expressions) festlegen, wird in der Anzeige "Spalten einrichten" eine vordefinierte Spalte angezeigt. Für die Vordefinition einer Spalte wird nur der im Feld "Feldausdrücke anfordern" (Request Field Expressions) festgelegte Feldausdruck verwendet, wenn Feldausdrücke im Feld "Feldausdrücke" und im Feld "Feldausdrücke anfordern" festgelegt sind.

Note: In der Anzeige "Spalten einrichten" (Setup Columns) wird eine Popup-Nachricht angezeigt, wenn auf Basis der ausgewählten Feldausdrücke keine vordefinierte Spalte erstellt werden kann. Mit einem Klick auf "OK" können Sie die Nachricht schließen und die Spalten manuell definieren.

Die Anzeige vordefinierter Spalten basiert außerdem darauf, ob Sie beim Definieren einer Tabelle vom Typ "Vordefinierte Abfrage" oder "AdHoc-Abfrage" die vordefinierte Abfrage oder die benutzerdefinierte Abfrage ausgewählt haben und welche Werte Sie für Abfrageparameter eingegeben haben. Eine vordefinierte Spalte wird nicht aufgrund des Feldausdrucks angezeigt, den Sie beim Definieren von Tabellen vom Typ "Vordefinierte Abfrage" oder "AdHoc-Abfrage" festlegen.

Note: In der Anzeige "Spalten einrichten" wird eine Popup-Nachricht angezeigt, wenn auf Basis der Werte, die Sie beim Definieren von Tabellen des Typs "Vordefinierte Abfrage" oder "AdHoc-Abfrage" in der Anzeige "Abfrage definieren" (Define Query) eingegeben haben, keine Spalten erstellt werden können. Mit einem Klick auf "OK" können Sie die Nachricht schließen und die Spalten manuell definieren.

Sie können weitere Spalten hinzufügen. Wählen Sie hierzu aus dem Feld unter der Liste vorhandener Spalten einen Spaltentyp aus und klicken Sie dann auf die Schaltfläche "Erstellen" (Create). Die neue Spalte wird als letzte Spalte in der Liste angezeigt und ist standardmäßig für die Aufnahme in die Tabelle ausgewählt. Neu hinzugefügte Spalten haben eine standardmäßige Breiteneinstellung von 200 Pixel.

Sie können die Reihenfolge der Spalten in der Anzeige "Spalten einrichten" (Setup Columns) ändern. Wählen Sie dazu eine Spalte aus und klicken Sie auf die Schaltflächen "Nach oben" (Up) und "Nach unten" (Down). Die in der Anzeige "Spalten einrichten" zuerst aufgeführte Spalte wird in der Tabelle auf der Portalseite ganz links angezeigt.

Row Selection Mode

Legt fest, ob eine oder mehrere Zeilen ausgewählt werden können.

Auswahloptionen sind:

• CheckBox – Sie können mehrere Tabellenzeilen mithilfe Kontrollkästchen auswählen.

Submit Data Mode

Legt fest, ob Daten im Falle der Dateneinreichung aus ausgewählten Zeilen, aus allen Zeilen oder aus keiner der Zeilen eingereicht werden.

Wenn Sie "Ausgewählte Zeilen" (Selected Rows) oder "Alle Zeilen" (All Rows) auswählen, wird die Tabelle "Eingereichte Spaltendatenzuordnungen" (Submitted Column Data Mappings) angezeigt. • Selected Rows – Erlaubt die Auswahl bestimmter Zeilen, die bei der Einreichung der Daten mit eingereicht werden sollen.
• All Rows – Bei der Einreichung der Daten werden alle Zeilen eingereicht.

Hinweis: Deses Feld wird angezeigt, wenn Sie beim Definieren von Tabellen mit den Datenquellen "Auswahlwerte" (Selection Values), "Vordefinierte Abfrage" (Predefined Query) und "AdHoc-Abfrage" (AdHoc Query) in der Anzeige "Feldausdruck einrichten" (Field Expression Setup) im Feld "Feldausdrücke" (Field Expressions) oder "Feldausdrücke anfordern" (Request Field Expressions) einen Wert ausgewählt haben.

Gehen Sie folgendermaßen vor, um Elementlisten- und Unterobjektlistentabellen mit dem Tabellenassistenten zu definieren:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf Katalog anzeigen/ausblenden (Show/Hide Catalog), wenn der Katalog der Benutzeroberflächenkomponenten ausgeblendet ist. Der Katalog der Benutzeroberflächenkomponenten wird angezeigt.
3. Doppelklicken Sie auf eine Seite oder ein Webpart. Die Seite bzw. das Webpart wird im Arbeitsbereich angezeigt.
4. Klicken Sie auf Toolbox anzeigen/ausblenden (Show/Hide Toolbox), wenn die Toolbox ausgeblendet ist. Die Toolbox wird angezeigt.
5. Klicken Sie auf ein Tabellen-Steuerelement und ziehen Sie es auf eine Zelle. Das Popup "Tabellen-Steuerelement erstellen" (Grid Control Creation) wird angezeigt.
6. Klicken Sie auf Tabellenassistent (Grid Wizard). Das Popup "Tabellen-Steuerelement erstellen" (Grid Control Creation) wird geschlossen und das Popup "Tabellenassistent" (Grid Wizard) wird angezeigt.
7. Wählen Sie Elementliste (Item List) oder Unterobjektliste (SubEntity List) als Tabellendatenquelle aus und klicken Sie auf Weiter (Next).
8. Wählen Sie einen Tabellenbearbeitungsmodus aus und klicken Sie auf Weiter (Next).
9. Klicken Sie auf den Knopf Mehr (More) für Feldausdrücke (Field Expressions). Das Popup "Feldausdruckerstellung" (Field Expressions Builder) wird angezeigt.
10. Wählen Sie einen Feldausdruck (Field Expression) aus und klicken Sie auf OK. Das Popup "Feldausdruckerstellung" (Field Expression Builder) wird geschlossen und der Feldausdruck wird im Feld "Feldausdrücke" (Field Expressions) angezeigt.
