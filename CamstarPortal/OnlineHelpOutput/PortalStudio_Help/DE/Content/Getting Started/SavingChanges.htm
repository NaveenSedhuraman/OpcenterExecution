Speichern von Änderungen

In Portal Studio können Sie Änderungen an einer offenen Komponente oder an mehreren offenen Komponenten gleichzeitig speichern.

Beim Speichern von Änderungen

Im Arbeitsbereich wird im Titel der Registerkarte einer Komponente ein Sternchen als Kennzeichen für nicht gespeicherte Daten angezeigt, wenn nicht gespeicherte Änderungen vorhanden sind. Eine Warnmeldung wird angezeigt, wenn Sie versuchen, die Registerkarte zu schließen, ohne die Änderungen zu speichern. Klicken Sie zum Schließen der Komponente im Warnmeldungs-Popup auf "Ja" (Yes). Wenn Sie "Nein" auswählen, gehen alle nicht gespeicherten Änderungen verloren.

Sie können in der Symbolleiste auf die Schaltfläche "Speichern" klicken, um die ausstehenden Änderungen für die im Arbeitsbereich ausgewählte Komponente zu speichern. Sie können in der Symbolleiste auf die Schaltfläche "Alles speichern" (Save All) klicken, um alle ausstehenden Änderungen für alle geöffneten Komponenten zu speichern. Informationen zu den Schaltflächen in der globalen Symbolleiste von Portal Studio finden Sie unter "Schaltflächen der globalen Symbolleiste von Portal Studio".

Sie können keine Änderungen an Komponenten speichern, die durch einen anderen Benutzer ausgecheckt sind. Informationen hierzu finden Sie unter "Ein- und Auschecken von Arbeitsaufgaben".

Wenn Sie versuchen, eine Komponenten- oder Sammlungseigenschaft unter demselben Namen wie eine vorhandene Komponenten- oder Sammlungseigenschaft zu speichern, wird eine Fehlermeldung angezeigt. Geben Sie einen eindeutigen Namen ein und speichern Sie dann Ihre Änderungen.

Vorgehensweise zum Speichern von Änderungen

Gehen Sie folgendermaßen vor, um Änderungen zu speichern:

1. Greifen Sie auf Portal Studio zu.
2. Erstellen Sie eine neue Komponente oder öffnen Sie eine vorhandene Komponente.
3. Nehmen Sie die erforderlichen Änderungen vor.
4. Klicken Sie auf Speichern. Die Anwendung speichert Ihre Änderungen und zeigt eine Erfolgsmeldung an.
