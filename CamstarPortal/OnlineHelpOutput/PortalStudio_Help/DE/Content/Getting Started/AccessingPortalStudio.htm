Zugreifen auf Portal Studio

Abhängig vom Thema, das Sie für den Zugriff auf das Portal verwendet haben, gibt es unterschiedliche Möglichkeiten für den Zugriff auf Portal Studio. "Horizon" ist das Standardthema, wenn Sie sich beim Portal mithilfe des Standardmodus anmelden. "Klassisch" ist das Standardthema, wenn Sie den klassischen Einstiegsmodus verwenden. Weitere Informationen zu den Einstiegsmodi finden Sie im Opcenter Medical Device and Diagnostics-Benutzerhandbuch – Erste Schritte oder im Opcenter Execution Core-Benutzerhandbuch – Erste Schritte.

Important: Microsoft Silverlight muss installiert sein, damit Sie auf Portal Studio zugreifen können. Silverlight wird in Google Chrome nach Version 44 nur begrenzt oder überhaupt nicht unterstützt. Informationen zum Zugreifen auf Portal Studio mit Chrome finden Sie unter "Aktivieren von Silverlight in Google Chrome".

Beim Zugreifen auf Portal Studio

Sie greifen über das Portal auf Portal Studio zu, indem Sie auf die folgende Schaltfläche klicken:

• Die Schaltfläche "Portal Studio" in der primären Navigationsleiste im Portal, wenn Sie das Thema "Horizon" verwenden.
• Die Schaltfläche "Studio" in der globalen Kopfzeile des Portals, wenn Sie das Thema "Klassisch" verwenden.

Wenn CSI der höchste aktive Arbeitsbereich ist, wird eine Fehlermeldung angezeigt und Sie können nicht auf Portal Studio zugreifen. Sie müssen erst einen anderen Arbeitsbereich in Management Studio aktivieren. Danach können Sie auf Portal Studio zugreifen. Weitere Informationen zum Aktivieren von Arbeitsbereichen finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Note: Das Portal zeigt keine Fehlermeldung an, wenn Sie zu dem Zeitpunkt, zu dem CSI zum höchsten aktiven Arbeitsbereich gemacht wird, bereits am Portal angemeldet sind. Eine Fehlermeldung wird angezeigt und Sie können nicht auf Portal Studio zugreifen, wenn Sie sich abmelden und sich dann erneut am Portal anmelden.

Wenn Sie auf Portal Studio zugreifen, wird im Arbeitsbereich das Startseiten-Dashboard (Start Page) angezeigt. Das Startseiten-Dashboard enthält Links zu häufig verwendeten Komponenten und Informationen zur Verwendung von Portal Studio.

Das Startseiten-Dashboard enthält die folgenden Abschnitte:

• "Neu erstellen" (Create New) – enthält Links zum Erstellen neuer Komponenten, z. B. Seiten, Webparts, Seitenfolgen usw.
• "Neuigkeiten" (What's New) – enthält einen Link zu Versionshinweisen, in denen neu hinzugefügte Funktionen in Portal Studio beschrieben werden.
• "Erste Schritte" (Getting Started) – enthält Links zu Informationen zur Verwendung von Portal Studio.
• "Zuletzt verwendet" (Recent) – enthält Links zum Öffnen der fünf zuletzt geöffneten Seiten, Webparts und Seitenfolgen.

Das Startseiten-Dashboard speichert die zehn zuletzt geöffneten Seiten, Webparts und Seitenfolgen und aktualisiert diese Liste, wenn Sie neue Komponenten öffnen. Wenn Sie auf den Link zu einer Komponente in der Liste "Zuletzt verwendet" (Recent) klicken, die seit dem Zugriff auf Portal Studio gelöscht wurde, wird eine Fehlermeldung angezeigt. Wenn Sie die Fehlermeldung schließen, wird die Komponente aus der Liste entfernt und durch eine andere der zuletzt geöffneten Komponenten ersetzt. Bei Ihrem nächsten Zugriff auf Portal Studio enthält die Liste "Zuletzt verwendet" keine Komponenten, wenn alle zehn zuletzt geöffneten Komponenten gelöscht und keine weiteren Seiten, Webparts oder Seitenfolgen geöffnet werden.

Microsoft Silverlight muss installiert sein und in Ihrem Browser müssen Cookies aktiviert sein, damit Sie auf Portal Studio zugreifen können. Wenn Sie mit Chrome Version 42, 43 oder 44 auf Portal Studio zugreifen, müssen Sie zunächst Silverlight aktivieren. Weitere Informationen zum Aktivieren von Silverlight finden Sie unter "Aktivieren von Silverlight in Google Chrome".

Durch Klicken auf die Schaltfläche "Zurück zum Portal" können Sie während Ihrer Portal Studio-Sitzung zum Portal zurückkehren. Bevor Sie zum Portal zurückkehren, werden Sie von Portal Studio aufgefordert, Ihre Änderungen zu speichern.

Portal Studio kann auch lokal installiert werden. Dies ermöglicht Ihnen den Zugriff auf Portal Studio ohne Umweg über das Portal. Weitere Informationen finden Sie unter "Lokale Installation von Portal Studio".

Zugreifen auf Portal Studio 2.0

Sie können auf Portal Studio 2.0 zugreifen – die neue Version von Portal Studio für Opcenter EX MDD und Opcenter EX CR. Die Einstellung "PortalStudioVersion" in der Datei web.config bestimmt, welche Version beim Zugreifen auf Portal Studio mithilfe der Themen "Horizon" oder "Klassisch" zur Verfügung steht. Wählen Sie "2" für Portal Studio 2.0 aus.

Note: Siemens empfiehlt für den Zugriff auf Portal Studio 1.0 den Microsoft Internet Explorer-Browser zu verwenden. Portal Studio 2.0 ist die Standardversion für den Chrome-Browser.

Vorgehensweise zum Zugreifen auf Portal Studio

Gehen Sie wie folgt vor, um auf Portal Studio zuzugreifen:

1. Melden Sie sich beim Portal an.
2. Welches Thema verwenden Sie?

Bei Verwendung von . . .

Dann . . .

Horizon

Klicken Sie in der primären Navigationsleiste des Portals auf die Schaltfläche Portal Studio. Portal Studio und das Dashboard "Startseite" werden angezeigt.

Klassisch

Klicken Sie in der globalen Kopfzeile im Portal auf Studio. Portal Studio und das Dashboard "Startseite" werden angezeigt.

Vorgehensweise zum Zurückkehren zum Portal

Gehen Sie folgendermaßen vor, um zum Portal zurückzukehren:

1. Greifen Sie auf Portal Studio zu.
2. Führen Sie die erforderlichen Arbeitsschritte aus.
3. Klicken Sie auf die Schaltfläche Zurück zum Portal (Back to Portal). Portal Studio wird geschlossen und Sie kehren zum Portal zurück.
