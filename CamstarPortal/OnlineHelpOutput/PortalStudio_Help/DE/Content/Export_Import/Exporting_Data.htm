Exportieren eines Elements

Portal Studio ermöglicht es Ihnen, Webparts, Seiten und Seitenfolgen zu exportieren. Daher können Sie Kopien dieser Objekte offline außerhalb des Entwicklungsservers speichern.

Beim Exportieren eines Elements

Sie können das Element aus dem Arbeitsbereich, der derzeit im Menü "Inhalte anzeigen" (View Contents) ausgewählt ist, oder aus allen aktiven Arbeitsbereichen exportieren.

Exportierte Elemente werden als XML-Dateien gespeichert. Als Name der XML-Datei wird standardmäßig der Name des exportierten Elements verwendet. Sie können keine Menüs aus Portal Studio exportieren.

Vorgehensweise zum Exportieren eines Elements

Gehen Sie folgendermaßen vor, um ein Element zu exportieren:

1. Greifen Sie auf Portal Studio zu.
2. Öffnen Sie das Element, das Sie exportieren möchten.
3. Möchten Sie die Elementdaten für den derzeit ausgewählten Arbeitsbereich oder für alle Arbeitsbereiche exportieren?

Sie möchten . . .

Dann . . .

Nur die Daten im derzeit ausgewählten Arbeitsbereich exportieren
a. Klicken Sie auf die Schaltfläche Exportieren (Export).
b. Wählen Sie Ausgewählter Arbeitsbereich (Selected Workspace) aus. Das Dialogfeld "Dateidownload – Sicherheitswarnung" (File Download – Security Warning) wird angezeigt.
c. Gehen Sie zu Schritt 4.

Die Elementdaten für alle Arbeitsbereiche exportieren.
a. Klicken Sie auf die Schaltfläche Exportieren (Export).
b. Wählen Sie Alle Arbeitsbereiche (All Workspaces) aus. Das Dialogfeld "Dateidownload – Sicherheitswarnung" (File Download – Security Warning) wird angezeigt.
c. Gehen Sie zu Schritt 4.
4. Klicken Sie auf OK. Das Dialogfeld wird geschlossen und das Popup "Speichern unter" (Save As) wird angezeigt.
5. Navigieren Sie zu einem Speicherort und klicken Sie auf Speichern (Save). Das Popup wird geschlossen, das Element wird als XML-Datei an den angegebenen Speicherort exportiert und Portal Studio zeigt eine Erfolgsmeldung an.
