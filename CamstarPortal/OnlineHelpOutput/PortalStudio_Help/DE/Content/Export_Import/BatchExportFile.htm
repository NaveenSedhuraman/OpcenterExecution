Durchführen von Batch-Exporten

Der Batch-Export bietet die Möglichkeit, mehrere Seiten, Webparts und Seitenfolgen als eine einzelne XML-Datei oder als ZIP-Datei mit mehreren XML-Dateien zu exportieren und die Elemente in ein Zielportal zu exportieren.

Beim Durchführen von Batch-Exporten

Sie können Elemente exportieren in:

• eine einzelne XML-Datei
• eine ZIP-Datei mit mehreren XML-Dateien
• ein Zielportal

Sie können die Elemente gleichzeitig in eine lokale Datei und ein Zielportal exportieren. Bei der Ausführung eines Batch-Exports muss mindestens eine Option ausgewählt werden.

Note: Beim Speichern eines Batch-Exports als ZIP-Datei wird eine ZIP-Datei mit einer XML-Datei für jede Seite, jedes Webpart und jede Seitenfolge erstellt, die im Export enthalten ist.

Sie müssen in der Liste "Personalisierungstyp für Export" (Personalization Type to Export) mindestens einen Arbeitsbereich auswählen.

Die Schaltflächen "Alle aktivieren" (Check All) und "Alle deaktivieren" (Uncheck All) betreffen nur die Elemente auf der Registerkarte, die derzeit im linken Bereich des Fensters "Batch-Export" (Batch Export) ausgewählt ist. Wenn z. B. die Registerkarte "Seiten" (Pages) ausgewählt ist und Sie auf "Alle deaktivieren" (Uncheck All) klicken, sind Webparts, die bereits auf der Registerkarte "Webparts" (Web Parts) ausgewählt sind, hiervon nicht betroffen.

Wenn Sie auf die Schaltfläche "Export starten" (Start Export) klicken, wird das Popup-Fenster "Exportfortschritt..." (Export Progress) mit zwei Fortschrittsbalken angezeigt. Der erste Balken zeigt den Fortschritt des jeweils ausgewählten Elements an, der zweite Balken den Gesamtfortschritt des Exportvorgangs.

Beim Durchführen von Batch-Exporten über Domänengrenzen hinweg müssen Sie den Zugriff auf Ziele genehmigen. Sie können den Zugriff genehmigen, indem Sie eine Datei namens clientaccesspolicy.xml erstellen und diese im Stammverzeichnis der Domäne, in der der Exportservice gehostet wird, platzieren. Weitere Informationen zum Erstellen und Platzieren dieser Datei in Ihrer Umgebung finden Sie im Microsoft Developer Network.

Important: Beim Batch-Export werden abhängige Seiten nicht automatisch gefunden und exportiert. Wenn Sie z. B. eine Seite mit abhängigen Seiten exportieren, die in einer Seitenumleitungsaktion oder von einem Flyout-Steuerelement referenziert werden, werden die abhängigen Seiten nicht gefunden und exportiert. Sie müssen die abhängigen Seiten manuell als Bestandteil des Exports auswählen, damit die importierte Seite korrekt funktioniert.

Fenster "Batch-Export" (Batch Export)

Im linken Bereich des Fensters "Batch-Export" (Batch Export) werden alle für den Batch-Export verfügbaren Seiten, Webparts und Seitenfolgen auf separaten Registerkarten angezeigt. Die auf einer Registerkarte ausgewählten Elemente verbleiben in der Auswahl, wenn Sie den Fokus auf eine andere Registerkarte verschieben.

Felddefinitionen des Fensters "Batch-Export" (Batch Export)

Diese Tabelle definiert die Felder im Fenster "Batch-Export" (Batch Export).

Feld

Definition

Typ

Exportoptionen

Personalization Type to Export

Gibt die Arbeitsbereichsdaten an, die exportiert werden sollen. Mindestens ein Arbeitsbereich muss ausgewählt werden.

Erforderlich

Export to File

Bei Auswahl dieser Option wird der Export als Datei gespeichert und das Feld "Dateiname" (File Name) wird aktiviert.

Hinweis: Sie müssen zu einem Speicherort navigieren und einen Dateinamen für die Exportdatei eingeben, wenn das Kontrollkästchen "Export in Datei" (Export to File) aktiviert ist.

Optional

File Name

Zeigt den Namen der zu exportierenden Datei nach der Navigation zu einem Speicherort an, nachdem Dateiname und Dateityp angegeben wurden.

Durch Klicken auf die Schaltfläche "Mehr" (More) wird das Popup-Fenster "Speichern unter" (Save As) angezeigt, in dem Sie zu einem Speicherort navigieren und einen Dateityp auswählen können. Folgende Dateitypoptionen sind verfügbar:

• XML-Dateien
• ZIP-Dateien
• Alle Dateien

Hinweis: Dieses Feld wird beim Aktivieren des Kontrollkästchens "Export in Datei" (Export to File) aktiviert.

Optional

Export to Target Portal

Exportiert Daten in ein Zielportal.

Hinweis: Wenn Sie dieses Kontrollkästchen aktivieren, werden die Felder "Benutzername" (User Name), "Kennwort" (Password), "Portaldomäne" (Portal Domain) und "Portal-URL" (Portal URL), das Kontrollkästchen "Vorhandene Webparts, Seiten oder Seitenfolgen überschreiben" (Override existing Web Parts, Pages or Pageflows) und die Schaltfläche "Verbindung überprüfen" (Verify Connection) aktiviert.

Optional

Benutzername

Der Anmeldename, der zum Zugriff auf das Zielportal verwendet wird. CamstarAdmin ist standardmäßig eingegeben.

Optional

Kennwort (Password)

Das Kennwort, das zum Zugriff auf das Zielportal verwendet wird.

Optional

Domain

Die Domäne des Zielportals.

Optional

Portal URL

Die URL des Zielportals. http://targethost/CamstarPortal/ ist standardmäßig angegeben.

Optional

Vorhandene Webparts, Seiten oder Seitenfolgen überschreiben (Override existing Web Parts, Pages or Pageflows)

Wenn diese Option ausgewählt ist, werden vorhandene Elemente im Zielportal beim Export überschrieben. Diese Option ist standardmäßig aktiviert.

Hinweis: Seiten, Webparts und Seitenfolgen in der Exportdatei, die bereits im Zielportal vorhanden sind, werden nicht exportiert, wenn diese Option nicht ausgewählt ist.

Optional

Vorgehensweise zum Durchführen von Batch-Exporten

Gehen Sie folgendermaßen vor, um einen Batch-Export durchzuführen:

1. Greifen Sie auf Portal Studio zu.
2. Klicken Sie auf die Schaltfläche Batch-Export (Batch Export). Das Fenster "Batch-Export" (Batch Export) wird angezeigt.
3. Wählen Sie die zu exportierenden Elemente auf den Registerkarten "Seiten" (Pages), "Webparts" (Web Parts) und "Seitenfolgen" (Pageflows) aus.
4. Wählen Sie in der Liste Personalisierungstyp für Export (Personalization Type to Export) mindestens einen Arbeitsbereich aus.
5. Möchten Sie die ausgewählten Elemente als XML-Datei exportieren?

Antwort . . .

Dann . . .

Ja
a. Aktivieren Sie das Kontrollkästchen In Datei exportieren (Export to File). Das Feld "Dateiname" (File Name) und die Schaltfläche "Mehr" (More) werden aktiviert.
b. Klicken Sie auf die Schaltfläche Mehr (More) für das Feld Dateiname (File Name). Das Popup "Speichern unter" (Save As) wird angezeigt.
c. Geben Sie im Feld Dateiname (File Name) einen Dateinamen ein, navigieren Sie zu einem Speicherort und wählen Sie einen Dateityp aus. Klicken Sie dann auf Speichern (Save). Das Popup "Speichern unter" (Save As) wird geschlossen und im Feld "Dateiname" (File Name) wird der Dateiname angezeigt.
d. Gehen Sie zu Schritt 6.

Nein

Gehen Sie zu Schritt 6.
6. Möchten Sie die ausgewählten Elemente in ein Zielportal exportieren?

Antwort . . .

Dann . . .

Ja
a. Aktivieren Sie das Kontrollkästchen Export an Zielportal (Export to Target Portal). Die Felder "Benutzername" (User Name), "Kennwort" (Password), "Portaldomäne" (Portal Domain) und "Portal-URL" (Portal URL), das Kontrollkästchen "Vorhandene Webparts, Seiten oder Seitenfolgen überschreiben" (Override existing Web Parts, Pages or Pageflows) und die Schaltfläche "Verbindung überprüfen" (Verify Connection) werden aktiviert.
b. Geben Sie im Feld Benutzername (User Name) den Benutzernamen für das Zielportal ein.
c. Geben Sie im Feld Kennwort (Password) das Kennwort für das Zielportal ein.
d. Geben Sie im Feld Domäne (Domain) die Domäne für das Zielportal ein.
e. Geben Sie im Feld Portal-URL (Portal URL) die URL für das Zielportal ein.
f. Gehen Sie zu Schritt 7.

Nein

Gehen Sie zu Schritt 7.
7. Geben Sie entsprechend Ihren Geschäftsanforderungen optionale Informationen ein. Weitere Informationen zu den optionalen Feldern finden Sie in der Tabelle der Felddefinitionen.
8. Klicken Sie auf Export starten (Start Export). Das Popup-Fenster "Exportfortschritt..." (Export Progress) wird angezeigt und zeigt den Fortschritt des Exports an. Popup-Fenster "Exportfortschritt..." (Export Progress) wird geschlossen und im Fenster "Batch-Export" (Batch Export) wird eine Erfolgsmeldung angezeigt, wenn der Exportprozess abgeschlossen wurde.
