Package Delivery Service

Siemens provides a Delivery Service that monitors deployed change management packages and delivers them to their predefined target systems.

Package Delivery Service Overview

The delivery service can be divided into the following phases:

• Monitor
• Read
• Delivery
• Update

Monitor Phase

The Deploy Package transaction marks a package for pickup by the delivery service. Packages eligible for pickup have a status of "In Progress" in the Deploy Status Per Target grid on the Deploy Package page. The delivery service checks for eligible packages according to a scheduled business rule. The default scheduled business rule named SBR_DEPLOY runs every five minutes.

A package must be deployed from the Deploy Package page before it is eligible for delivery by the delivery service. Refer to "Deploying a Package" for information.

Read Phase

The delivery service looks into the source system database and reads the deployment details for a package to determine a package's destination target systems.

Delivery Phase

The delivery service delivers the change management package and exports the XML header files to the selected target systems. The application pulls in the XML header files on the target systems according to a scheduled business rule. The default scheduled business rule named SBR_ACTIVATION runs every five minutes. The change management package can then be activated in the target system. Refer to "Activating a Package" for information.

Update Phase

The delivery service updates the source system with the package delivery status following an attempted delivery according to a scheduled business rule. The default scheduled business rule named SBR_DEPLOYSTATUS runs every five minutes. The service updates the Deploy Status per Target grid in real-time on the Deploy Package page as either "Complete" or "Failed" for each delivery and supplies an error message for failed deliveries. A package must be redeployed for the delivery service to attempt to deliver it again if the delivery fails initially. Refer to "Deploying a Package" for information.

Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on scheduled business rules. You can use the default rules or define new rules.
