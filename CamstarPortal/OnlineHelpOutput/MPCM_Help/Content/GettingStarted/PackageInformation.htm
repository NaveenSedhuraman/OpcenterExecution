Package Information

Information about a selected package appears at the top of the change management package task pages. Typically, this information includes the core attributes of, and general information about, the package. Additionally, this area includes icons that provide additional functionality or activate flyout menus when clicked. These icons provide options to:

• View documents
• Attach documents
• View package details
• View activation impacts for the package
• View content change history

The application creates the package information when you save a new package. Refer to "Creating a Package" for information.

When Viewing Core Attributes for a Package

The application automatically populates the core attributes on the change management pages. The package selected on the Activation Search or Package Search pages determines the attributes displayed when you load the transaction page for that package.

The application populates the package information with the following:

• An icon indicating the status of the package
• Any description, owner, last update, and step information

Package Status

The package status icon indicates the status of the package. Siemens provides the following package status icons:

Package status is open meaning draft (Draft, Draft Camstar, Draft PLM), pending approval (Pending Approval Camstar, Pending Approval PLM), or pending deployment

Package has been rejected

Package deployment is complete

Package deployment is incomplete

Package is closed

Package is voided
Note: A system administrator can specify different icons when defining specs.

Clicking the package status icon displays a pop-up containing a diagram of the workflow associated with the specified package. The pop-up displays the following information:

• The name of the associated workflow in the title bar.
• A red border around the package's current step in the workflow.
• A description of the workflow.
Note: Descriptions appear for the workflow and the steps (specs) only if descriptions were provided on the modeling object instances.

You can hover your mouse over any step in the workflow to view a description of that step.

When Using Flyout Menus

Some of the icons in the package information header enable you to display flyout menus that provide additional functionality.

You can click the icons or text descriptions within these menus to open the corresponding pages:

• The page loads on a new tab after you click the icon or text description.
• The flyout menu closes when the tab opens.
