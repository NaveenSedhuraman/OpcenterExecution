Configuring Change Management

You must configure change management in Modeling before you can begin creating and deploying change management packages. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on the change management modeling object instances.

This list describes specific tasks you must perform to configure change management:

• Ensure that a valid SSL certificate is located on the source server and target servers.
• Define one or more Package Creation Reason modeling object instances. The application requires you to select a package creation reason when creating a package.
• Define a minimum of two Server Catalog System modeling object instances: one for the local host and one for the target system. You can define multiple server catalog systems for additional targets.

The delivery service uses the user name and password information from the Server Catalog System instance defined for the host server to access the host server. The user name and password must be a valid account that can log into the application.

Important: You must restart the Delivery service if you change the user name and password after the initial setup.
• Perform the following on the target servers:
• Create a folder named Temp.
• Add the user specified in the Server Catalog System instance for the host server to Computer Management on the target server.
• Share the Temp folder with the user specified in the Server Catalog System instance for the host and give the user Read/Write access.
• Start the Camstar Deployment server on the host server.
