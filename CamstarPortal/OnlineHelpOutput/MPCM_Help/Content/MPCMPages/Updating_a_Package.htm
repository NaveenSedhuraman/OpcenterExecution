Updating a Package

You can add and modify core attributes and deployment information for existing change management packages that have not been deployed. You can assign approvers to selected packages if the packages are associated with a change management workflow requiring package approval. You can specify collaborators if you want to distribute the work of assigning content to a package to other employees. You can specify whether the application updates the core package attributes displayed on other change management package pages with the changes made on the Update Package page.

Refer to "Creating a Package" for information on creating a package. Refer to "Package Information" for information on the core attributes and the pages on which they appear.

When Updating a Package

You can only update a package when the package is at a spec associated with an allowable transaction containing the permission to update a package. For example, the default DraftPermissions role includes the permission to update a package and is associated with the default Draft spec. Employees with the appropriate roles can update packages at the Draft spec.

The Update Package page displays one, two, or three tabs depending on whether you are using a change management workflow that requires approvals and whether you specify that collaborators can assign content to a package:

• General

The General tab provides the ability for you to add and modify core attributes and deployment information for the selected change management package. You can view a diagram of the change management workflow associated with the package by clicking the Information icon next to the Change Mgt Workflow field. The application displays the diagram in a pop-up. You can also specify whether you want collaborators to assign content to a package.

• Approvers

The Approvers tab appears only if you selected a change management workflow requiring approvals. This tab provides the ability for you to specify an approval template for the change management package.

• Collaborators

The Collaborators tab appears only if you selected Owner Plus Collaborators on the General tab. The Collaborators tab enables you to specify a collaborator template for the change management package, or you can add collaborators using the Collaborator pop-up. You can also send e-mail notifications to collaborators to assign content, and a notification to the package owner when the collaborators have completed assigning content.

Note: The Change Mgt Settings modeling object enables a system administrator to configure various global settings for change management including e-mail notifications sent to approvers, collaborators, and package owners when certain events occur for a change management package. Refer to "When Adding Package Approvers" and "When Adding Collaborators" for information.

When Adding Deployment Details

The application requires you to enter deployment details before you can deploy a package. Target systems selected when creating the package or when updating the package appear on the Deploy Package page.

The application displays a warning message if you select a target system that has a value configured for the MDB Version or Camstar Version fields that is different than what is configured for your source system. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining server catalog systems.

The warning message includes a warning for both the MDB Version and Camstar Version fields if both values are different on the target and source systems. This warning message only displays for the mismatched target closest to the bottom of the grid. You can close the warning message and save the package. The application does not prevent you from creating a package ifyou receive an error for mismatched values.

When Adding Activation Details

You can specify prerequisite packages to be activated prior to the activation of your package to account for dependencies on instances in other packages. Clicking Add on the Prerequisite Change Packages grid opens the Package Search pop-up. You can use the pop-up to search for and select prerequisite packages to add to your package.

When Enabling Content Collaboration

The default setting for the Content Collaborators control on the General tab is Owner Only, indicating that only the user with ownership permissions can assign content to the package. You can enable content collaboration by selecting the Owner Plus Collaborators option for the Content Collaborators control. The Collaborators tab appears when you select the Owner Plus Collaborators option. Refer to "When Adding Collaborators" for information on the Collaborators tab. Refer to "Understanding Content Collaboration" for an overview of the collaboration process.

Important: The application checks whether collaborators are specified on the Collaborators tab if you change the setting for the Content Collaborators control from Owner Plus Collaborators to Owner Only. The application no longer displays the Collaborators tab if no collaborators are specified. If collaborators are specified, a warning appears indicating that any collaborators and collaborator information on the Collaborators tab will be deleted. The application deletes the information and no longer displays the Collaborators tab if you proceed. Owner Plus Collaborators remains selected if you do not proceed.

When Adding Package Approvers

Approval templates identify the employees required to approve the change management package and the decision options they can select when approving or rejecting a package. You must specify an approval template to associate an approval decision list with the change management package. The application requires an approval decision list to be associated with the change management package before you can route the package for approval.

Note: The application saves the approval decision list and any approvers associated with the specified approval template, but not the template itself.

The application populates the Approvers grid and the Instructions for All Approvers field when you select an approval template. You can add approvers, delete approvers, modify information for each approver, and modify the instructions for all approvers.

Note: A system administrator can configure e-mail notifications for package approvers to notify them when a change management package requires their approval. Reminder and delegation notifications can also be configured. Change management notifications are configured on the Change Mgt Settings modeling object. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on configuring e-mail notifications.

Selecting a different approval template assigns a new approval decision list and overwrites the Approvers grid, including any updates to the approvers list. The original approval decision list and list of approvers are not lost until you click Update.

The application uses the Name column in the Approvers grid to build the distribution list for notifications sent to employees (approvers) when packages are routed for approval. E-mail addresses must be added for all employees designated as approvers.

Approval templates, approval decision lists, and employees are defined in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on defining approval templates and approval decision lists.

Note: You can define an approval template containing only an approval decision list if you do not want the Approvers grid populated automatically. Defining an empty approval template enables you to easily customize the approvers assigned to each package.

When Adding Collaborators

You can define a collaborator template using the Collaborator Template modeling object. The collaborator template enables you to predefine collaborators and their instructions. The application populates the Collaborators grid (including the name and specific instructions for each collaborator) and the Instructions for All Collaborators field when you select a collaborator template.

You can add collaborators manually to the grid if you do not select a template. You can add collaborators, delete collaborators, modify information for each collaborator, and modify the instructions for all collaborators whether you select a template or not. The application does not save any updates you make to the collaborators or instructions in the grid to the collaborator template.

Selecting a different collaborator template overwrites the information in the Collaborators grid. The information is not lost until you click Update.

Message to Collaborator enables you to notify collaborators that they can assign content to the package. The application sends the message when you select Notify Collaborators of Assignments on Update and click Update. A warning message appears if you select Notify Collaborators of Assignments on Update and have not specified Message to Collaborator.

Notify Collaborators of Assignments on Update is not selected by default. If you select this check box, it remains checked after you click Update. The field defaults to unselected if you close and reopen the Update Package page.

Completion Message to Owner enables you to notify the package owner when the collaborators have finished assigning content to the package. Each collaborator clicks Complete on the Assign and Review Content page. The Content Collaborator – Completed Assignment pop-up appears. The collaborator can add comments and click the Complete button in the pop-up. The application sends the message.

Note: A system administrator can configure e-mail notifications for package collaborators to notify them that they are required to assign content to a package. E-mail notifications can also be configured to notify the package owner that collaborators have finished assigning content to the package. A system administrator can also configure reminder and delegation notifications. Change management notifications are configured on the Change Mgt Settings modeling object. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on configuring e-mail notifications.

The application populates the Message to Collaborator and Completion Message to Owner lists with any e-mail messages defined with the E-mail Message modeling object. Message to Collaborator defaults to the message with the Change Mgt Collaboration Assignment notification type on the Change Mgt Settings modeling object. The Completion Message to Owner list defaults to the message with the Change Mgt Owner Collaboration Assignment Complete notification type on the Change Mgt Settings modeling object. The application does not select a default for either of these lists if you have not set the notification types with the Change Mgt Settings object.

The application sets Due Date to the Assigned Date and Status to Late on the Assign and Review Content page, Collaborator Tasks tab if you do not specify Duration and Period when adding collaborators.

Collaborator templates, employees, and change management settings are defined in Modeling. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

How to Update a Package

Follow these steps to update a package:

1. Select Package Search from the Change Management menu. The Package Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
4. Select the package you want to update. The application displays the relevant transactions for the package.
5. Click Update Package. The Update Package page appears.
6. Expand the Deployment Details section if necessary.
7. Do you want to enable content collaboration?

If . . .

Then . . .

Yes
a. Select Owner Plus Collaborators for Content Collaborators. The Collaborators tab appears.
b. Go to step 8.

No

Go to step 8.
8. Enter optional information according to your business requirements. Refer to the field definitions table for information about the optional fields.
9. Complete the "How to Add Approvers for the Package" procedure to add approvers for the package if necessary.
10. Complete the "How to Add Collaborators for the Package" procedure to add collaborators for the package if necessary.
11. Click Update. The application saves the change management package and displays a success message.

How to Add Approvers for the Package

Follow these steps to add approvers for the package:

1. Perform the "How to Update a Package" procedure.
2. Click the Approvers tab.
3. Select an Approval Template. The application updates the Approvers grid and the Instructions for All Approvers field to show the approvers and instructions associated with the selected template.
4. Do you want to accept the default list of approvers, delete approvers, modify approvers, or add approvers?

If you want to . . .

Then . . .

Accept the default list

Go to step 5.

Delete approvers
a. Select the row in the Approvers grid containing the approver.
b. Click Delete selected row. The application removes the approver from the grid.
c. Repeat steps a–b to delete additional approvers.
d. Go to step 5.

Modify approvers
a. Select the row in the Approvers grid containing the approver. The application activates the row, enabling you to modify the approver's information.
b. Modify the approver information. Refer to the field definitions table for the Approvers tab for information about the fields.
c. Repeat steps a-b to modify additional approvers.
d. Go to step 5.

Add approvers
a. Click Add new row on the Approvers grid. The application activates a row, displaying editable fields.
b. Enter the Level of the approver.
c. Select the approver's Role.
d. Select the approver's Name.
e. Enter optional information according to your business requirements. Refer to the field definitions table for information about the optional fields.
f. Repeat steps a-e to add additional approvers.
g. Go to step 5.
5. Modify the Instructions for All Approvers if necessary.
6. Click Update. The application saves the change management package and displays a success message.

How to Add Collaborators for the Package

Follow these steps to add collaborators for the package:

1. Perform the "How to Update a Package" procedure.
2. Click the Collaborators tab.
3. Do you want to use a collaborator template?

If . . .

Then . . .

Yes
a. Select a Collaborator Template. The application updates the Collaborators grid and the Instructions for All Collaborators field to show the collaborators and instructions associated with the selected template.
b. Go to step 4.

No

Go to step 4 and add collaborators.
4. Do you want to accept the default list of collaborators, delete collaborators, modify collaborators, or add collaborators?

If you want to . . .

Then . . .

Accept the default list

Go to step 5.

Delete collaborators
a. Select the row in the Collaborators grid containing the collaborator.
b. Click Delete selected row. The application removes the collaborator from the grid.
c. Repeat step a–b to delete additional collaborators.
d. Go to step 5.

Modify collaborators
a. Select the row in the Collaborators grid containing the collaborator and click Edit selected row. The Collaborator pop-up appears.
b. Modify the collaborator information. Refer to the field definitions table for the Collaborators tab for information about the fields.
c. Click OK. The application closes the Collaborator pop-up and displays the updated collaborator information in the Collaborators grid.
d. Repeat steps a-c to modify additional collaborators.
e. Go to step 5.

Add collaborators
a. Click Add new row in the Collaborators grid. The Collaborator pop-up appears.
b. Enter the Level of the collaborator.
c. Select the collaborator's Role.
d. Select the collaborator's Name.
e. Enter optional information in the pop-up according to your business requirements. Refer to the field definitions table for information about the optional fields in the pop-up.
f. Click OK. The application closes the Collaborator pop-up and displays the collaborator information in the Collaborators grid.
g. Repeat steps a-f to add additional collaborators.
h. Go to step 5.
5. Enter optional information according to your business requirements. Refer to the field definitions table for information about the optional fields.
6. Click Update. The application saves the change management package and displays a success message.
