Viewing Package Details

The Package Details page enables you to view the following for existing packages:

• Core attributes
• Current and previous approval cycle information
• Current content collaboration information
• Modeling object instances assigned to the package
• Prerequisite packages assigned to the package

When Viewing Package Details

You can navigate to the Package Details page by:

• Clicking the Package Details button from the flyout menu that appears after clicking the Content button on any page that contains the Package Information header.
• Clicking the Package Details button under Available Transactions on the Package Search or Activation Search page.
• Clicking the Package Details button under Available Transactions on the Package Search (Multiple) or Activation Search (Multiple) page if you select only one package on either of these pages.
Note: The Selected Package field is populated automatically with the package selected on the page from which you clicked the Package Details button.

Information will not appear in the Instructions for Approver or Approver Comments field until an entry has been selected in the Approvers grid.

Collaborators are sorted by Level in ascending order in the Content Collaborators grid. The Content Collaborators grid is blank if the package owner has not designated collaborators for the package.

View Document Button

You can click the View Document button to view or save the package detail information displayed on the Package Details page as a PDF. Clicking the View Document button displays a dialog in your browser containing Open, Save, and Cancel options . Click Open to view the PDF in your default PDF viewing application. You must have an application capable of viewing .pdf files, such as Adobe Acrobat Reader. Click Save to display a Save As pop-up to save the PDF to your local machine.

Note: The PDF file created by the View Document button is named Document.pdf by default.

The PDF created by the View Document button contains all the information on the Package Details page regardless of whether fields are expanded or collapsed. The approvers section of the PDF does not contain any information if you are working in the MOM workspace and routing to Teamcenter for approval. Refer to the Opcenter Execution Core Teamcenter Integration Guide for information.

Note: The dialog that appears may contain additional options such as "Open with system viewer" depending on your browser.

Change Package Content - Assigned Content Collaborators

Content Collaborators grid

Grid listing all employees specified as content collaborators for a change management package.

Display Only

Level

Level of collaboration associated with the collaborator. Level is represented by numeric values only. The grid is sorted by this column in ascending order.

Level defines the sequence in which the tasks must be completed. The application sends e-mail notifications after each task is completed if the collaboration tasks are set up sequentially. The application sends e-mail notifications when the last collaborator completes his or her task if the collaboration tasks are set up in parallel.

Serial indicates that collaborators with a lower level number must finish assigning content before collaborators with higher level numbers can begin. Parallel indicates collaborators with the same level number can assign content at the same time.

Display Only

Name

Employee assigned responsibility for the collaboration work.

Display Only

Status

Status of the collaboration work. Cell highlight color is dynamic based on status. Status settings are:

• Pending - employee has not yet completed collaboration task. Employee may or may not have started task. Cell is highlighted orange.
• Complete - employee has completed collaboration task, regardless of whether work was completed late. Work is complete when the collaborator clicks the Complete button on the Content Collaborator – Completed Assignment pop-up accessed through the Assign and Review Content page, Assign and Review Content tab. Cell is highlighted green.
• Late - employee has not completed the collaboration task and the allotted time to complete the task has expired. The task is late when the current date is later than the due date. Employee may or may not have started task. Cell is highlighted red.

Display Only

Assigned Date

Date on which the package owner assigned the collaboration work to the collaborator on the Update Package page, Collaborators tab. Date is in mm/dd/yyyy format.

Display Only

Due Date

Date on which the collaboration work is due. Due date is calculated based on the date the package owner assigned the collaboration work to the collaborator plus the Duration/Period specified on the Update Package page, Collaborators tab. Date is in mm/dd/yyyy format.

Display Only

Completion Date

Date on which the collaboration work was completed. Work is complete when the collaborator clicks the Complete button on the Content Collaborator – Completed Assignment pop-up accessed through the Assign and Review Content page, Assign and Review Content tab. Date is in mm/dd/yyyy format.

Display Only

Collaborator Comments

Text the collaborator entered in the Comments field in the Content Collaborator – Completed Assignment pop-up accessed through the Assign and Review Content page, Assign and Review Content tab.

Display Only

Collaborator Instructions

Text the package owner entered in the Instructions for Collaborator field on the Update Package page, Collaborators tab.

Display Only

How to View Package Details

Follow these steps to view package details:

1. Are you on the source or target system?

If you are on the . . .

Then . . .

Source system

Select Package Search from the Change Management menu. The Package Search page appears.

Target system

Select Activation Search from the Change Management menu. The Activation Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
4. Select the package you want to view. The application displays the relevant transactions for the package.
5. Click Package Details. The Package Details page appears.
