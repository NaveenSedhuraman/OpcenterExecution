Searching for a Package

The application provides a feature that enables you to search for and retrieve a change management package based on specified criteria. You can search by employee assignment or general criteria. The Package Search feature is available for packages at the following default steps:

• Draft (default No Approval change management workflow)
• Draft Camstar (default Camstar change management workflow)
• Draft PLM (default PLM change management workflow)
• Download in Process (default MOM PLM workflow)
Note: Refer to the "Teamcenter Change Management" appendix for information.
• Pending Enrichment (default MOM PLM workflow)
Note: Refer to the "Teamcenter Change Management" appendix for information.
• Pending Approval Camstar (default Camstar change management workflow)
• Pending Approval PLM (default PLM change management workflow and default MOM PLM workflow)
Note: Refer to the "Teamcenter Change Management" appendix for information on the MOM PLM workflow.
• Rejected
• Pending Deployment
• Deployment Complete
• Deployment Incomplete
Note: Additional steps are configurable by a system administrator.

The application displays two tabs and the Search Results grid when you access the Package Search page. The two tabs are Employee Assignment Search and General Search. Click the Search button to populate the Search Results grid. Initially, the grid lists all packages for which the current user is the package owner, package approver, or package collaborator. By default, the grid does not contain packages for which the current user has completed approvals.

Each tab contains multiple fields by which you can narrow your search. The application adds a filter icon to each tab to indicate that filter criteria is specified. The filter icon appears on the Employee Assignment Search tab automatically because several fields are selected by default on this tab. The filter icon appears on the General Search tab when you select criteria. The application displays the results of the combined search criteria from the two tabs in the Search Results grid.

When the application returns change management packages that meet your criteria, you can select the package on which you want to perform a transaction. The application enables only those transactions that are relevant to the selected package.

Note: Refer to "Searching for Multiple Packages to Close, Void, or Open" if you want to select multiple packages to close, open, or void at one time.

When Searching for a Package Using Employee Assignment Criteria

The Employee Assignment Search tab is selected by default when you initially access the Package Search page. Multiple fields enable you to filter your search.

When searching for packages using fields on this tab:

• The entry in the Employee field defaults to the current user.
• You can search by employee package assignments.
• You can search by substitution role.
• You can specify whether to hide completed assignments.
• The fields Is Owner, Is Approver, Is Collaborator, Has Owner Role, Has Approver Role and Has Collaborator Role are selected by default.
• The check boxes under Package Assignments and Substitution Role act as an "or" filter. If any of the matching check box attributes are true, the matching packages are displayed.
• The check boxes under Hide Completed Assignments act as "and" filters and can hide some of the packages returned by the other check boxes.

When Searching for a Package Using General Criteria

You can further narrow your search by using the filters on the General Search tab.

When searching for packages using fields on this tab:

• You can search by package name using SQL wildcards. For example, entering "%01%" returns packages with names containing the characters "01."
• You can search by change management workflow. You can only select one workflow at a time.
• You can search by one or more change management steps. The available steps are dependent on the change management workflow selected. You can select multiple steps. The Change Mgt Step field displays a count rather than the selected steps. For example, if you select three of the five available steps, the application displays "Selected 3 of 5" in the Change Mgt Step field.
• There are five default package statuses. Open, Deployed, and Rejected are selected by default. Voided and Closed are not selected. The Package Status field displays a count rather than the selected package statuses. The application displays "Selected 3 of 5" in the Package Status field when the page loads.
Note: Multiple selected values from the Package Status pick list act as "or" filters and add to the list of packages returned.
• The remaining General Search filters act as an "and" filter and can hide some of the packages returned.

Executing a Search

When you execute a search, the application:

• Displays the search results in a grid
• Retains the search criteria in the filter fields

The application adds paging to the grid if the search returns more results than can be displayed in the rows on the grid. The application highlights any packages at a Deployment Incomplete or Rejected step. The application enables transactions relevant to the package when you select a package in the Search Results grid. Siemens also provides the ability to export your search results to an Excel file or CSV format. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

Clearing Search Filters and Results

Use the Clear All button to clear filters and results. Clearing the Package Search page enables you to start a new search. When you click the Clear All button, the application:

• Clears all specified search filters on both tabs
• Clears the filter icon from the search tabs
• Clears all packages from the Search Results grid
• Clears all transaction buttons
• Continues displaying the Package Search page

Using the Transaction Buttons

The transaction buttons enable you to execute transactions on any selected package in the Search Results grid. The application enables only valid transactions for the package. If you select a different package in the grid, the available transactions change accordingly.

When you select a package and click a transaction button, the application displays the appropriate transaction page or pop-up depending on the type of transaction you select.

A pop-up appears if you are closing, voiding, or opening a package. You can enter relevant comments.

Note: You can open a package that was previously closed or voided on the Package Search or Package Search (Multiple) pages.

A transaction page appears if you are performing any of the other transactions. The application populates the Selected Package field (if available) on the page with the name of the package you selected in the grid. For example, if you select a package and click the Alter Step button, the Alter Package Step page appears with the Selected Package field populated.

Note: A Link to PLM button appears when you select a package downloaded from Teamcenter. Teamcenter is a Siemens application. Clicking the Link to PLM button allows you to open Teamcenter and view the engineering change notice (ECN) corresponding to the selected package. Refer to the Opcenter Execution Core Teamcenter Integration Guide for information.

How to Search for a Package Using Employee Assignment Criteria

Follow these steps to search for a package using employee assignment criteria:

1. Select Package Search from the Change Management menu. The Package Search page appears. The Employee Assignment tab is selected and a filter icon indicates that criteria is selected.
2. Select or clear criteria on the Employee Assignment tab.
3. Do you want to include General Search criteria?

If . . .

Then . . .

Yes
a. Click the General Search tab.
b. Enter or select one or more criteria on the General Search tab to narrow your search. A filter icon appears on the General Search tab.
c. Go to step 4.

No

Go to step 4.
4. Click Search. The application displays the packages matching your search criteria in the Search Results grid.

How to Search for a Package Using General Criteria

Follow these steps to search for a package using general criteria:

1. Select Package Search from the Change Management menu. The Package Search page appears. The Employee Assignment tab is selected and a filter icon indicates that criteria is selected.
2. Click the General Search tab. The General Search tab appears.
3. Enter or select one or more criteria on the General Search tab. A filter icon appears on the General Search tab.
4. Do you want to modify Employee Assignment criteria?

If . . .

Then . . .

Yes
a. Click the Employee Assignment tab.
b. Select or clear criteria on the Employee Assignment tab.
c. Go to step 5.

No

Go to step 5.
5. Click Search. The application displays the packages matching your search criteria in the Search Results grid.

How to View Target System Information for a Selected Package

Follow these steps to view target system information for a selected package:

1. Perform the "How to Search for a Package Using Employee Assignment Criteria" procedure.

Or

Perform the "How to Search for a Package Using General Criteria" procedure.
