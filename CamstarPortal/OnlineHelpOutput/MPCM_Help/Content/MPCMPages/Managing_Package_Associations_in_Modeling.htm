Managing Package Associations in Modeling

You can manage packages associated with a modeling object instance from within Modeling. You can view, remove, or add associations if you have the appropriate role and permissions.

When Managing Package Associations in Modeling

Every modeling object page contains a toolbar near the top of the page. An additional button labeled Pkg Assoc appears on the toolbar if you have licensed change management, saved at least one instance of the modeling object you want to associate with a package, and have a role with the Change Package Modeling Inquiry permission. Clicking this button enables you to manage package associations in Modeling for the selected modeling object instance. You can view packages associated with the instance, or remove or add associations if you have package ownership permissions.

The Manage Package Associations pop-up appears when you click the Pkg Assoc button. The Associated Packages tab is the default tab when you access the pop-up through the button.

Refer to "When Assigning Modeling Object Instances to a Package in Modeling" and "Manage Package Associations Pop-up Field Definitions" for information on the tabs and fields on the Manage Package Associations pop-up.

How to Manage Package Associations in Modeling

Follow these steps to manage package associations in Modeling:

1. Select Modeling from the Modeling menu. The Modeling page appears.
2. Click the All Maint arrow on the Modeling page to see a list of modeling objects under that group.

Or

Select View All (Alphabetical) from the View Objects list on the Modeling page to display a list of modeling objects in alphabetical order.A list of all modeling objects appears in alphabetical order.

3. Select a modeling object. The object's page appears within the Modeling page.
4. Create a new modeling object instance or update an existing one.
5. Click Save. The application saves the modeling object and displays a success message. The Pkg Assoc button appears on the Modeling page toolbar.
6. Click the Pkg Assoc button on the Modeling page toolbar. The Manage Package Associations pop-up appears with the Associated Packages tab selected.
7. Do you want to view, remove, or add package associations?

If you want to . . .

Then . . .

View package associations
a. View the list of packages associated with the instance.
b. Click Close. The application closes the Manage Package Associations pop-up.

Remove package associations
a. Click the Remove from Package tab.
b. Select the packages from which you want to remove the instance.

Note: You can only remove explicitly added instances, not dependencies.

c. Click Remove. The application removes the modeling object instance from the packages, displays a success message, and closes the Manage Package Associations pop-up.

Add package associations
a. Click the Add to Package tab.
b. Select the package to which you want to add the instance.
c. Click Add. The application adds the modeling object instance to the package, displays a success message, and closes the Manage Package Associations pop-up.
