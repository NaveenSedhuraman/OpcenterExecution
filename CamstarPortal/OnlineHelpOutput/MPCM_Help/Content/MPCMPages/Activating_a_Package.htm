Activating a Package

The Activate Package transaction enables you to apply a deployed change management package to a target system.

When Activating a Package

The activation process can overwrite existing instances on the target system when the change management package contains instances with the same name as the existing instances. The override mode settings on the Activate Package page enable you to specify whether and how modeling object instances on the target system will be overwritten by the modeling object instances in the change management package.

The package activation process provides three override options:

• Abort Process and Roll Back - The activation process aborts and all committed and pending package activation changes are rolled back when the process detects a modeling object instance on the target system with the same name as a modeling object instance in the package or a server error occurs.
• Overwrite Instance - The activation process overwrites any modeling object instances on the target system that have the same name as instances in the package.
• Skip Instance - The activation process loads the modeling object instances in the package to the target system, but skips loading instances with the same name as an existing instance on the target system.
Note: The application uses the Skip Instance override method by default.

Package activation begins when you click the Activate Package button. The application displays the status of the activation on this page as it loads the modeling object instances from the change management package. Status messages appear in the Activation Console. A status message will appear when attempting to activate packages with more than 2000 instances.

The application's default execution timeout setting may affect the deployment process. The default value is 110 seconds. The executionTimeout value is located in the web.config file under the httpRuntime setting of the system.web node. The Web.config file is located in C:\Program Files (x86)\Camstar\CamstarPortal by default.

Approving multiple packages with the same name, but different letter casings, will cause an error in case insensitive environments, such as Oracle. For example, you cannot activate the package "ABC" and then attempt to activate the package "abc."

Errors During Activation

The application displays error messages in the Activation Console, aborts package activation, and rolls back changes when the following occur:

• A server error.
• The application detects a modeling object instance on the target system with the same name as an instance in the change management package and the override mode specified is Abort Process and Roll Back.

The application removes the package from the target system if it encounters a server error, or it detects duplicate modeling object instances and you have selected the Abort Process and Roll Back override option. The application requires you to re-deploy the package to activate it on a target system. You can:

• Re-deploy the package and select another override option

Or

• Remove the duplicate modeling object instances, re-deploy the package, and select Abort Process and Roll Back as the override option

Activating a Package with Prerequisite Packages

The application displays the Override Confirmation pop-up when you attempt to activate a package that requires other packages to be activated first. The pop-up lists the prerequisite packages associated with the package you are trying to activate and asks if you want to override the prerequisite packages. You can enter any relevant comments in the Comments field using a maximum of 255 characters. Clicking Yes overrides any prerequisites.

Note: The application displays the first three prerequisite package names followed by an ellipsis if there are more than three.

The application performs the following when you click Yes:

• Closes the pop-up
• Overrides all prerequisite packages listed
• Records the timestamp, user name, and override comments in history
• Directs you to the Activate Package page
Note: Depending on how packages and prerequisites are configured, you may receive a failure message when attempting to override prerequisite packages.

When Accessing the Activate Package Page

The application populates the Selected Package field and displays all fields and buttons on the Activate Package page automatically when you access the page from the Activation Search page.

How to Activate a Package

Follow these steps to activate a package:

1. Log on to the Camstar Portal (on the target system).
2. Select Package Search from the Change Management menu. The Package Search page appears.
3. Enter or select one or more criteria to narrow your search.
4. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
5. Select the package you want to activate. The application displays the relevant transactions for the package.
6. Click Activate Package. The Activate Package page appears.
7. Select a different override mode from the Override Mode When Instance Exists field if necessary.
8. Click Activate Package to begin the activation process. The activation process begins and the application displays the status in the Activation Console.
