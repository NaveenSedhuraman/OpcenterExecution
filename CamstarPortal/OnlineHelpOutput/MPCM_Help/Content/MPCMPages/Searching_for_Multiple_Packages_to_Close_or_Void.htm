Searching for Multiple Packages to Close, Void, or Open

The application provides a feature that enables you to search for and retrieve multiple change management packages to close, void, or open based on specified criteria. The Package Search (Multiple) feature is available for packages at the following default steps:

• Draft (default No Approval change management workflow)
• Draft Camstar (default Camstar change management workflow)
• Draft PLM (default PLM change management workflow)
• Download in Process (default MOM PLM workflow)
Note: Refer to the "Teamcenter Change Management" appendix for information.
• Pending Enrichment (default MOM PLM workflow)
Note: Refer to the "Teamcenter Change Management" appendix for information.
• Pending Approval Camstar (default Camstar change management workflow)
• Pending Approval PLM (default PLM change management workflow)
• Rejected
• Pending Deployment
• Deployment Complete
• Deployment Incomplete
Note: Additional steps are configurable by a system administrator.

When Searching for Multiple Packages to Close, Void, or Open

When you access the Package Search (Multiple) page initially, the application displays the fields available for filtering and an empty Search Results grid. The application adds a filter icon to the tab to indicate that filter criteria is specified. Click the Search button to populate the Search Results grid.

When searching for multiple packages to close, void, or open:

• You can search by package name using SQL wildcards. For example, entering "%01%" returns packages with names containing the characters "01."
• You can search by change management workflow. You can only select one workflow at a time.
• You can search by one or more change management steps. The available steps are dependent on the change management workflow selected. You can select multiple steps. The Change Mgt Step field displays a count rather than the selected steps. For example, if you select three of the five available steps, the application displays "Selected 3 of 5" in the Change Mgt Step field.
• There are five default package statuses. Open, Deployed, and Rejected are selected by default. Voided and Closed are not selected. The Package Status field displays a count rather than the selected package statuses. The application displays "Selected 3 of 5" in the Package Status field when the page loads.
Note: You can view the package details for a package that is closed or voided if you select the Closed or Voided package statuses. Multiple selected values from the Package Status pick list act as "or" filters and add to the list of packages returned.
• The remaining General Search filters act as an "and" filter and can hide some of the packages returned.

Executing a Search

When you execute a search, the application:

• Displays the search results in a grid
• Retains the search criteria in the filter fields

The application adds paging to the grid if the search returns more results than can be displayed in the rows on the grid. The application highlights any packages at a Deployment Incomplete or Rejected step.

When the application returns packages that meet your criteria, you can select the package or packages on which you want to perform a transaction. You can select multiple packages in the grid by selecting the check box in the header row of the grid, or you can select the check boxes for individual packages.

Siemens also provides the ability to export your search results to an Excel file or CSV format. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

Clearing Search Filters and Results

Use the Clear All button to clear filters and results. Clearing the Package Search (Multiple) page enables you to start a new search. When you click the Clear All button, the application:

• Clears all specified search filters
• Clears the filter icon from the General Search tab
• Clears all packages from the Search Results grid
• Clears all transaction buttons
• Continues displaying the Package Search (Multiple) page

Using the Transaction Buttons

You must select at least one package and click the Load Actions button to enable the transaction buttons. If you select more than one package, the application displays the transactions that can be performed on the selected packages simultaneously. The only transactions you can perform on multiple packages are close, void, and open. For example, you may want to search for all packages with a status of Rejected and then close all of them simultaneously.

The Package Details transaction button appears in addition to Close Selected, Void Selected, and Open Selected if you select only one package in the Search Results grid. You can also view the package details for a package that is closed or voided if you select the Closed or Voided statuses.

When you select a package or packages and click a transaction button, the application displays the appropriate pop-up depending on whether you are closing, voiding, or opening packages. You can enter relevant comments.

The application updates the package status of each package after you close, void, or open it. Any packages you closed or voided no longer display in the Search Results grid unless you select Closed or Voided in the Package Status field.

How to Search for Multiple Packages to Close, Void, or Open

Follow these steps to search for multiple packages to close, void, or open:

1. Select Package Search (Multiple) from the Change Management menu. The Package Search (Multiple) page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.

How to Execute a Transaction on Multiple Packages

Follow these steps to execute a transaction on multiple packages:

1. Perform the "How to Search for Multiple Packages to Close, Void, or Open" procedure.
2. Select the check box in the header row in the Search Results grid to select all displayed packages.

Or

Select the check boxes for individual packages.

3. Click Load Actions.The application enables relevant transactions for the selected packages.
4. Click a transaction button. The application displays the appropriate pop-up for the transaction you selected.
5. Complete the information on the pop-up.
