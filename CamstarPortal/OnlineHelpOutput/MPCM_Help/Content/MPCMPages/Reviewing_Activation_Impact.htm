Reviewing Activation Impact

The Activation Impact page enables you to review the impact of deploying a change management package to a target system.

When Reviewing Activation Impact

You can run Activation Impact only on the source system.

The Activation Impact page lists instances that will be added or updated once the package is activated on the selected target. You must click Review Activation Impact to view the instances for the selected target. The application displays a target in the Selected Target field by default when only one target has been defined.

Use the Object Type field to filter the list of instances by modeling object. Reviewing activation impact does not activate or deploy the package. Results are sorted by Object.

The Impact field shows whether the activation process will add or update instance data.

Note: The override mode settings on the Activate Package page enable you to specify whether and how modeling object instances on the target system will be overwritten by the modeling object instances in the change management package.

Access the Activation Impact page from the Content flyout menu in the package information header or the Activation Impact transaction button.

Click the Export Data button on the status bar of the grid to view the Export Data pop-up and export Activation Impact data.

How to Review the Activation Impact for a Package

Follow these steps to view the activation impact for a package:

1. Select Package Search from the Change Management menu. The Package Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
4. Select the package you want to review. The application displays the relevant transactions for the package.
5. Click Activation Impact.
6. Select a target and click Review Activation Impact. The application populates the grid with data.
