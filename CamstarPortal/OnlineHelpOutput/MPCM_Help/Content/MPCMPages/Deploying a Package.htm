Deploying a Package

Siemens provides the ability for you to deploy change management packages containing core attributes and instance data to predefined target systems.

Before Deploying a Package

A package must be at a step that has Deploy as an allowable transaction to be eligible for deployment.

Deployment details are defined either during package creation or when updating a package before the package is deployed. Refer to "Creating a Package" and "Updating a Package" for information.

When Deploying a Package

The deployment process consists of two asynchronous steps:

• Building the package document
• Delivering the package to each target

The application begins building the package when you click Deploy on the Deploy Package tab on the Deploy Package page.

The Deploy Package page contains the Deploy Package tab and the Deployment Status tab. The Deploy Package tab comes into focus by default.

All available target systems associated with the selected package are selected in the Select Targets to Deploy field by default. You can specify which target systems the package deploys to by clearing the check box of unwanted target systems. You must select at least one target system when deploying a package. An error message appears if you attempt to deploy a package with no target systems selected.

Target systems are added to a package during package creation or when updating a package. You can update a package's deployment details to add additional targets on the Update Package page. Refer to "Updating a Package" for information.

Clicking the Deploy button on the Deploy Package tab displays the Deployment Status tab. Clicking the Reset button selects all of the available target systems in the Select Targets to Deploy field and removes any comments in the Deployment Comment field.

The Deployment Status tab enables you to monitor the progress of a package build and capture the deployment status of a package per each selected target system. The application displays the Deployment Status tab automatically when you click the Deploy button on the Deploy Package tab.

The Deployment Status tab contains the Package Build Status grid and Deploy Status Per Target grid. The Package Build Status grid only displays data for the current package build or last attempted package build associated with the selected package. The Deploy Status Per Target grid captures data for both the current deployment and maintains a history of previous deployments associated with the selected package.

Both the Package Build Status grid and Deploy Status Per Target grid update in real-time. You can also click the Refresh button to refresh the Package Build Status grid and Deploy Status Per Target grid.

The application displays one entry for each target in the Deploy Status Per Target grid. Two files for each package are deployed to the target system. The application displays "In Progress" for Deploy Status until both files complete. Deploy Status changes to "Complete" if both files complete successfully. If either file fails, processing stops and Deploy Status changes to "Failed." The application populates Deployment Errors with the error.

The Package Size field displays the total size of the change packet measured in Megabytes (MB). The size of packages is limited to 30 MB by default. Refer to the Opcenter Execution Medical Device and Diagnostics Portal Studio User Guide or the Opcenter Execution Core Portal Studio User Guide for information on increasing the maximum attachment size limit in Portal Studio's settings.

The application's default execution timeout setting may affect the deployment process. The default value is 110 seconds. The executionTimeout value is located in the web.config file under the httpRuntime setting of the system.web node. The Web.config file is located in C:\Program Files (x86)\Camstar\CamstarPortal by default.

A warning message will appear when attempting to deploy packages with more than 1000 instances.

How to Deploy a Package

Follow these steps to deploy a package:

1. Select Package Search from the Change Management menu. The Package Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
4. Select the package you want to deploy. The application displays the relevant transactions for the package.
5. Click Deploy Package. The Deploy Package page appears.
6. Enter optional information according to your business requirements. Refer to the field definitions table for information about the optional fields.
7. Click Deploy Package. The application displays the Deployment Status tab and builds the XML deployment package files and then updates the Package Build Status Grid. The application then stages the package for deployment to the specified target systems and updates the Deploy Status Per Target grid in real-time.
