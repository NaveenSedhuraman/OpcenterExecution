Change Management in Modeling

Additional functionality is available in Modeling if you have licensed change management, created at least one change management package, and have the appropriate role and permissions. You can create and update modeling object instances and assign them to existing, active packages from within Modeling.

An additional tab on each object's page enables you to view information about the number of packages associated with an instance and whether the instance is locked by a package. An additional button on the Modeling page toolbar enables you to view package associations from within Modeling. You can also add or remove package associations in Modeling if you have package ownership permissions.
