Viewing Content Change History

The Content Change History page can be accessed from the Package Search page or from the History flyout menu on the package information header. You can view the changes made to instances assigned to a change package, and display the before and after values of the modeling instances to ensure all additions or updates to a manufacturing process have been completed.

When Filtering Content Change History

The application displays all content change history by default when you click the Search button. The application also provides filters that allow you to narrow the list of instances returned when you click the Search button.

You can use the Select Date Range radio button to filter instances returned by your search to include only instances that have been changed between the Start and End Timestamp values. The Start Timestamp is defined by the package's creation date, and the End Timestamp is defined by the package's last deployment date. The End Timestamp is set to the current time and date if the package has not been deployed.

You can also filter your search by object type, instance name, or last change user. The Clear All button removes any filter that you may have selected and clears any instances returned by your search to restore the page to its default state.

When Viewing Content Change History

The results on the Content Change History page are displayed in the Content Change History by Instance grid as links to the Modeling Audit Trail pop-up. You can only view content change history one instance at a time. The content change history date range is passed to the Modeling Audit Trail pop-up. You can filter the modeling audit trail results by package name and custom date range.

View Document Button

You can click the View Document button to view or save the information displayed in the Content Change History by Instance grid as a PDF. The View Document button is enabled after searching for and displaying information in the Content Change History by Instance grid. Clicking the View Document button displays a dialog in your browser containing Open, Save, and Cancel options. Click Open to view the PDF in your default PDF viewing application. You must have an application capable of viewing .pdf files, such as Adobe Acrobat Reader. Click Save to display a Save As pop-up to save the PDF to your local machine.

Content Change History Page

You can access the Content Change History page by either:

• Clicking Content History after selecting a package from the Search Results grid on the Package Search page.

Or

• Clicking the History button on the package information header and clicking Content Change History on the flyout menu.

How to Filter Content Change History

Follow these steps to filter content change history:

1. Access the Content Change History page using one of these procedures:

If . . .

Then . . .

You are on the Package Search page

Click Content History. The Content Change History page appears.

You are already working with a package
a. Click History on the package information header. The Package History Options flyout menu appears.
b. Click Content Change History on the flyout menu. The flyout menu closes and the Content Change History page appears.
2. Enter the filter criteria and click Search. The Content Change History by Instance grid is populated.

How to View Content Change History

Follow these steps to view content change history:

1. On the Content Change History page, display the content change history using one of these procedures:
• Click Search to use the default date range and display all content change history.

Or

• Enter a date range and then click Search to filter the results.

The Content Change History by Instance grid is populated.

2. Click the link under the Instance Column for the Content Change History you want to view. The Modeling Audit Trail pop-up appears.
Note: The dates from the Content Change History page are passed to the Modeling Audit Trail pop-up. You can enter the package name and change the dates on the Modeling Audit Trail pop-up to further filter the results.
