Searching for a Package to Activate

The application provides a feature that enables you to search for and retrieve a change management package for activation on the target system based on specified criteria. The Activation Search feature is available for packages that have been deployed and have a status eligible for activation. The employee searching for a package to activate must have a role with activation permissions assigned or must have the default Package Activator role. Roles are assigned to employees in Modeling.

Note: Refer to "Searching for Multiple Packages to Close, Void, or Open on the Target System" if you want to select multiple packages to close, void, or open at one time.

When Searching for a Package to Activate

When you access the Activation Search page initially, the application displays the fields available for filtering and an empty Search Results grid. The application adds a filter icon to the tab to indicate that filter criteria is specified. Click the Search button to populate the Search Results grid.

When searching for packages:

• You can search by package name using SQL wildcards. For example, entering "%01%" returns packages with names containing the characters "01."
• You can search by Activation Status. There are three default package statuses. The application selects Activation Incomplete and Pending Activation by default. The Activated status is not selected. The Activation Status field displays a count rather than selected statuses. The application displays "Selected 2 of 3" in the Activation Status field when the page loads.
Note: Searching for a package that was recently activated may return unexpected results. The activation status is updated by default every 5 minutes. You can modify this setting in Modeling by adjusting the default scheduled business rule SBR_ACTIVATION. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.
• You can search by Package Status. There are three available statuses. The application selects Open by default. Closed and Voided are not selected. The application displays a count rather than selected statuses. The application displays "Selected 1 of 3" in the Package Status field when the page loads.
Note: You can view the package details for a package that is closed or voided if you select the Closed or Voided statuses.
• Every change management package you are authorized to access displays in the results if you do not specify any filters.
• You can narrow your search by using up to seven filters.

When the application returns change management packages that meet your criteria, you can select the package on which you want to perform a transaction. The application enables transactions that are relevant to the selected package.

Executing a Search

When you execute a search, the application:

• Displays the search results in a grid
• Retains the search criteria in the filter fields

The application adds paging to the grid if the search returns more results than can be displayed in the rows on the grid. When you select a package in the Search Results grid, the application enables transactions relevant to the package. Siemens also provides the ability to export your search results to an Excel file or CSV format. Refer to the Opcenter Execution Medical Device and Diagnostics Shop Floor User Guide or the Opcenter Execution Core Shop Floor User Guide for information.

Clearing Search Filters and Results

Use the Clear All button to clear filters and results. Clearing the Activation Search page enables you to start a new search. When you click the Clear All button, the application:

• Clears all specified search filters
• Clears the filter icon from the General Search tab
• Clears all packages from the Search Results grid
• Clears all transaction buttons
• Continues displaying the Activation Search page

Using the Transaction Buttons

The transaction buttons enable you to execute transactions on any selected package in the Search Results grid. The application enables valid transactions for the package selected.

The application enables the Close Package and Void Package transaction buttons according to the package status and activation status of the package selected.

When you select a package and click a transaction button, the application displays the appropriate transaction page or pop-up depending on the type of transaction you select.

A pop-up appears if you are closing, voiding, or opening a package or activating a package with prerequisites. You can enter relevant comments.

Note: You can open a package that was previously closed or voided on the Activation Search or Activation Search (Multiple) pages.

A transaction page appears if you are performing any of the other transactions. The application populates the Selected Package field (if available) with the name of the package you selected in the grid. For example, if you select a package and click the Activate Package button, the Activate Package page appears.

How to Search for a Package to Activate

Follow these steps to search for a package to activate:

1. Select Activation Search from the Change Management menu. The Activation Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.

How to Execute a Transaction on a Selected Package

Follow these steps to execute a transaction on a selected package:

1. Perform the "How to Search for a Package to Activate" procedure.
2. Select the package for which you want to execute a transaction. The application enables the relevant transactions for the package.
3. Click a transaction button. The application displays the appropriate transaction page or pop-up for the transaction you selected. The application populates the Selected Package field (if available) on the appropriate transaction page with the name of the package you selected in the grid.
4. Complete the information on the transaction page or pop-up.
