Performing Package Tasks

Change management enables you to perform these tasks:

• Update an existing package
• Review, assign, and/or remove the content of a change management package and its dependencies while the package is in draft mode
• Update a package's step—move the package from any step to any other step in the current change management workflow
• Review the impact of deploying a change package to a target system
• Deploy a package—the application generates the change management package (with all of its core attributes and assigned instance data) to be sent to the defined targets
• View the details of a package including its core attributes and instance data
• Search for and view the changes made to instances assigned to a change package
• Close, void, or open a single package
• Close, void, multiple packages

You must have roles with appropriate permissions assigned to use each of these options.
