Approving a Package (PLM)

You can approve packages assigned to the PLM change management workflow when they are at the Pending Approval PLM step.

When Approving a Package on the PLM Workflow

You must have the package owner role.

Approved and Rejected are the only options available for a PLM approval decision. Selecting Approved moves the change management package to the Pending Deployment step. Selecting Rejected moves the change management package to the Rejected step. Only one approval decision of Reject is necessary for the application to move the package to the Rejected step. However, all approval decisions must be received before the application moves the package to the next step.

How to Record a PLM Approval Decision

Follow these steps to record a PLM approval decision:

1. Select Package Search from the Change Management menu. The Package Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
4. Select the package you want to approve. The application displays the relevant transactions for the package.
5. Click Approve (PLM). The Approve Package (PLM) page appears.
6. Select a PLM Approval Decision.
7. Enter optional information according to your business requirements. Refer to the field definitions table for information about the optional fields.
8. Click Save Decision. The application displays a message stating you have completed the approval.
