Assigning Modeling Object Instances to a Package in Modeling

You can create and update modeling object instances and assign them to existing, active packages in Modeling. Any user with the appropriate role and permissions can select a default change management package and then assign all instances created or updated during a session to it.

When Assigning Modeling Object Instances to a Package in Modeling

The Change Mgt Save option appears on the Modeling page if you have licensed change management, have at least one package defined, and have a role with the Change Package Modeling Inquiry permission. You can assign modeling object instances to a package in Modeling when these conditions are met:

• You must have modeling permissions.
• You must have the Assign Change Pkg Content permission.
• You must have the Change Package Modeling Inquiry permission.
• You must be the package owner or have the owner role (for example, the Siemens-provided Package Owner role), or you must be a designated content collaborator for the package.
• The package must be at a step that allows content to be assigned to it; for example, a draft step.

The Change Mgt Save option is set to Off by default. You can work in Modeling as you normally do—without change management integration.

Set the Change Mgt Save option to On if you want to assign modeling object instances to a package in Modeling.

A session ends when you close the Modeling tab or exit the application. The Change Mgt Save option is set to the Off position. You must set the option to the On position the next time you load the page if you want to use this feature again.

Opening a second Modeling tab constitutes another session.

Note: You must assign modeling objects in the correct order or add references when the Change Mgt Save option is set to the On position. Refer to "Assigning and Reviewing Package Content" for information on assigning modeling object instances and references to packages. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information on the Modeling page and creating and updating modeling object instances.

You can assign each individual instance to a package when you save it if you do not select a default package.

If Setting a Default Package

Set the Change Mgt Save option to On and select a default package if you want to assign all modeling object instances created or updated during the session to the package.

You can select only one package on the Set Default Package pop-up. Select the package and then click the Set Default button to make your selection.

If Not Setting a Default Package

The Manage Package Associations pop-up appears every time you save an instance if you set the Change Mgt Save option to On, but do not select a default package. This enables you to assign each individual instance to a package.

The Manage Package Associations pop-up contains three tabs:

• Associated Packages

The Associated Packages tab lists all active packages with which the instance is associated, either explicitly or by dependency. All packages except those that are voided or closed are displayed regardless of your role.

Note: This is the default tab when you access the Manage Package Associations pop-up through the Pkg Assoc button on the Modeling page toolbar.
• Remove from Package

The Remove from Package tab lists all active packages meeting role and step criteria that are associated with the instance. You can remove the instance from multiple packages. You can only remove explicitly added instances. You cannot remove dependencies.

Note: You can use this tab to remove an instance from a package or packages even if the instance was assigned using a default package.
• Add to Package

The Add to Package tab lists all active packages meeting role and step criteria. The application does not list packages already added. This is the default tab. You can select only one package.

How to Assign a Modeling Object Instance to a Default Package in Modeling

Follow these steps to assign a modeling object instance to a default package in Modeling:

1. Select Modeling from the Modeling menu. The Modeling page appears.
2. Click Off to set the Change Mgt Save option to On. The Set Default link appears.
3. Click the Set Default link. The Set Default Package pop-up appears.
4. Select the package you want to set as the default.
5. Click Set Default. The Set Default Package pop-up closes.
6. Click the All Maint arrow on the Modeling page to see a list of modeling objects under that group.

Or

Select View All (Alphabetical) from the View Objects list on the Modeling page to display a list of modeling objects in alphabetical order. A list of all modeling objects appears in alphabetical order.

7. Select a modeling object. The object's page appears within the Modeling page.
8. Create a new modeling object instance or update an existing one.
9. Click Save. The application saves the modeling object and displays a success message. It assigns the modeling object instance to the default package.
10. Repeat steps 7-9 for any additional modeling object instances you want to assign to the default package during the session.

How to Add a Modeling Object Instance to a Package in Modeling without Setting a Default

Follow these steps to add a modeling object instance to a package in Modeling without setting a default package:

1. Select Modeling from the Modeling menu. The Modeling page appears.
2. Click Off to set the Change Mgt Save option to On. The Set Default link appears.
3. Click the All Maint arrow on the Modeling page to see a list of modeling objects under that group.

Or

Select View All (Alphabetical) from the View Objects list on the Modeling page to display a list of modeling objects in alphabetical order. A list of all modeling objects appears in alphabetical order.

4. Select a modeling object. The object's page appears within the Modeling page.
5. Create a new modeling object instance or update an existing one.
6. Click Save. The application saves the modeling object and displays a success message. The Manage Package Associations pop-up appears with the Add to Package tab selected.
7. Select the package to which you want to add the instance.
8. Click Add. The application adds the modeling object instance to the package, displays a success message, and closes the Manage Package Associations pop-up.

How to Remove an Instance from a Package in Modeling

Follow these steps to remove an instance from a package in Modeling:

1. Select Modeling from the Modeling menu. The Modeling page appears.
2. Click Off to set the Change Mgt Save option to On. The Set Default link appears.
3. Click the All Maint arrow on the Modeling page to see a list of modeling objects under that group.

Or

Select View All (Alphabetical) from the View Objects list on the Modeling page to display a list of modeling objects in alphabetical order. A list of all modeling objects appears in alphabetical order.

4. Select the modeling object that contains the instance from which you want to remove the package associations. The object's page appears within the Modeling page.
5. Select the instance from which you want to remove the package associations. Fields for the instance appear.
6. Click Save. The application displays a success message indicating the modeling object was updated. The Manage Package Associations pop-up appears with the Add to Package tab selected.
7. Click the Remove from Package tab.
8. Select the packages from which you want to remove the instance.
Note: You can only remove explicitly added instances, not dependencies.
9. Click Remove. The application removes the modeling object instance from the packages, displays a success message, and closes the Manage Package Associations pop-up.

How to View Packages Associated with an Instance in Modeling

Follow these steps to view packages associated with an instance in Modeling:

1. Select Modeling from the Modeling menu. The Modeling page appears.
2. Click Off to set the Change Mgt Save option to On. The Set Default link appears.
3. Click the All Maint arrow on the Modeling page to see a list of modeling objects under that group.

Or

Select View All (Alphabetical) from the View Objects list on the Modeling page to display a list of modeling objects in alphabetical order. A list of all modeling objects appears in alphabetical order.

4. Select the modeling object that contains the instance for which you want to view package associations. The object's page appears within the Modeling page.
5. Select the instance for which you want to view package associations. Fields for the instance appear.
6. Click Save. The application displays a success message indicating the modeling object was updated. The Manage Package Associations pop-up appears with the Add to Package tab selected.
7. Click the Associated Packages tab.
8. View the list of packages associated with the instance.
9. Click Close. The application closes the Manage Package Associations pop-up.
