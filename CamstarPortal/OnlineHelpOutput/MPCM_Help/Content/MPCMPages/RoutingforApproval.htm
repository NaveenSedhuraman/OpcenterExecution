Routing a Package for Approval

You can route a package for approval when:

• You are the package owner or a user with ownership permissions,
• The package is at a draft step, and
• The package is assigned a change management workflow with an approval step.

Before Routing a Package for Approval

Ensure approval notifications are set up before routing a package. Notifications must be configured so approvers receive an e-mail notifying them a package requires their approval. Refer to "Understanding Approval Notifications" for information on setting up notifications.

Note: Currently, only approvers for a package on the default Camstar change management workflow receive notifications.

The application requires a change management package to meet these criteria before you can route for approval:

• Instance content is assigned
• One or more target systems have been selected
• The package is assigned the default Camstar change management workflow:
• One or more approvers are assigned, and
• An approval decision list is assigned.
Note: You must select an approval template on the Update Package page to assign an approval decision list. Refer to "Updating a Package" for information.

When Routing a Package for Approval

The Route for Approval page displays the core attributes of the selected package and an Information icon you can click to view a summary of the package content.

Clicking the Information icon displays a Package Content Summary grid. The Package Content Summary grid displays this information:

• Number of modeling objects defined in the application
• Total number of those modeling object instances defined
• Number of modeling objects assigned to the package
• Total number of modeling object instances assigned to the package
• Number of modeling objects referenced by the selected modeling objects
• Total number of referenced modeling object instances assigned to the package

The application changes the package step to Pending Approval Camstar or Pending Approval PLM when you route the package for approval.

Routing a Package to Teamcenter for Approval

You route a package to Teamcenter for approval when you click the Route for Approval button if you are working in the MOM workspace. The application sends two encrypted PDF files to Teamcenter via Camstar Interoperability (CIO) to support the approval decision process. One file contains the package details and the other contains content change history. Refer to "Viewing Package Details" and "Viewing Content Change History" for information.

CIO returns a success message if it receives the two PDF files. An error appears if the file transfer does not succeed. Teamcenter sends an approval decision (approved, rejected, voided) back to CEP.

Note: This functionality is available only when the MOM workspace is installed. Refer to the Opcenter Execution Core Teamcenter Integration Guide for information.

How to Route a Package for Approval

Follow these steps to route a package for approval:

1. Select Package Search from the Change Management menu.The Package Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
4. Select the package you want to route for approval. The application displays the relevant transactions for the package.
5. Click Route for Approval. The Route for Approval page appears.
6. Click the Information icon to view a summary of the package content if necessary.
7. Click Route for Approval. The application displays a message stating the package has been routed.
