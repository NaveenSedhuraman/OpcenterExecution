Searching for Multiple Packages to Close, Void, or Open on the Target System

The application provides a feature that enables you to search for and retrieve multiple change management packages to close, void, or open on the target system based on specified criteria. You can also select a single package on the Activation Search (Multiple) page to close, void, or open. The Activation Search (Multiple) feature is available for packages that have been deployed and have a status eligible for activation. The employee searching for packages on this page must have a role with activation permissions assigned or must have the default Package Activator role. Roles are assigned to employees in Modeling.

When Searching for Multiple Packages to Close, Void, or Open on the Target System

When you access the Activation Search (Multiple) page initially, the application displays the fields available for filtering and an empty Search Results grid.

When searching for packages:

• You can search by package name using SQL wildcards. For example, entering "%01%" returns packages with names containing the characters "01."
• You can search by Activation Status. There are three default package statuses. The application selects Activation Incomplete and Pending Activation by default. The Activated status is not selected. The Activation Status field displays a count rather than selected statuses. The application displays "Selected 2 of 3" in the Activation Status field when the page loads.
Note: Searching for a package that was recently activated may return unexpected results. The activation status is updated by default every 5 minutes. You can modify this setting in Modeling by adjusting the default scheduled business rule SBR_Activation.Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.
• You can search by Package Status. There are three available statuses. The application selects Open by default. Closed and Voided are not selected. The application displays a count rather than selected statuses. The application displays "Selected 1 of 3" in the Package Status field when the page loads.
Note: You can view the package details for a package that is closed or voided if you select the Closed or Voided statuses.
• Every change management package you are authorized to access displays in the results if you do not specify any filters.
• You can narrow your search by using up to seven filters.

Executing a Search

When you execute a search, the application:

• Displays the search results in a grid
• Retains the search criteria in the filter fields

The application adds paging to the grid if the search returns more results than can be displayed in the rows on the grid. The application highlights any packages at a Deployment Incomplete or Rejected step.

When the application returns packages that meet your criteria, you can select the package or packages on which you want to perform a transaction. You can select multiple packages in the grid by selecting the check box in the header row of the grid, or you can select the check boxes for individual packages.

Siemens also provides the ability to export your search results to an Excel file or CSV format. Refer to the Opcenter Execution Medical Device and Diagnostics Modeling User Guide or the Opcenter Execution Core Modeling User Guide for information.

Clearing Search Filters and Results

Use the Clear All button to clear filters and results. Clearing the Activation Search (Multiple) page enables you to start a new search. When you click the Clear All button, the application:

• Clears all specified search filters
• Clears all packages from the Search Results grid
• Clears all transaction buttons
• Continues displaying the Activation Search (Multiple) page

Using the Transaction Buttons

You must select at least one package and click the Load Actions button to enable the transaction buttons. If you select more than one package, the application displays the transactions that can be performed on the selected packages simultaneously. The only transactions you can perform on multiple packages are close, void, and open. For example, you may want to search for all packages with a status of Activation Incomplete and then close all of them simultaneously.

The application enables the Close Package and Void Package transaction buttons according to the package status and activation status of the package or packages selected.

The Package Details transaction button appears in addition to Close Selected (Multiple), Void Selected, and Open Selected if you select only one package in the Search Results grid. You can also view the package details for a package that is closed or voided if you select the Closed or Voided statuses.

When you select a package or packages and click a transaction button, the application displays the appropriate pop-up depending on whether you are closing, voiding, or opening packages. You can enter relevant comments.

The application updates the package status of each package after you close, void, or open it. Any packages you closed or voided no longer display in the Search Results grid unless you select Closed or Voided in the Package Status field.

How to Search for Multiple Packages to Close, Void, or Open on the Target System

Follow these steps to search for multiple packages to close, void, or open on the target system:

1. Select Activation Search (Multiple) from the Change Management menu. The Activation Search (Multiple) page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.

How to Execute a Transaction on Multiple Packages on the Target System

Follow these steps to execute a transaction on multiple packages on the target system:

1. Perform the "How to Search for Multiple Packages to Close, Void, or Open on the Target System" procedure.
2. Select the check box in the header row in the Search Results grid to select all displayed packages.
