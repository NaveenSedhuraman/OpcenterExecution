Voiding a Package

You can void a single package that has a status of Open, Deployed, or Rejected on the Package Search page if you have package ownership permissions. You typically void a package before deployment occurs.

You cannot update the contents of a package or deploy it after you void it. You cannot void a package that is in a Voided or Closed status.

You can void a single package on the Activation Search page that has an Open package status, and an activation status of either Pending Activation or Activation Incomplete.

Note: Refer to "Voiding Multiple Packages" if you want to select multiple packages and void them at one time.

When Voiding a Package

The Void Package pop-up appears when you select a package in the Search Results grid on the Package Search page or Activation Search page and select Void Package. You can enter any relevant comments in the Comments field using a maximum of 255 characters and then click Void.

The application performs the following when you click Void:

• Voids the package and places it in Voided status
• Closes the Void Package pop-up
• Displays a success message indicating the following:
• Name of the package
• Date and time at which the package was voided
• Name of the employee who voided it
• Reloads the Search Results grid with the current search criteria and returns you to the same page in the grid
• Records the following information in history:
• Name of the employee who voided the package
• Date and time at which the application voided the package
• Any comments you entered in the Void Package pop-up

How to Void a Package

Follow these steps to void a package:

1. Are you voiding the package on the source system or the target system?

If . . .

Then . . .

On the source system

Select Package Search from the Change Management menu. The Package Search page appears.

On the target system

Select Activation Search from the Change Management menu. The Activation Search page appears.
2. Enter or select one or more criteria to narrow your search.
3. Click Search. The application displays the packages matching your search criteria in the Search Results grid.
4. Select the package you want to void. The application displays the relevant transactions for the package.
5. Click Void Package. The Void Package pop-up appears.
6. Enter any relevant comments in the Comments field if necessary.
7. Click Void. The application voids the package, closes the Void Package pop-up, displays a success message, reloads the Search Results grid, and records pertinent information in history.
