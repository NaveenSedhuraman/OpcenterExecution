Teamcenter Change Management Overview

Teamcenter packages are created in CEP when an ECN is downloaded in to CEP. A Teamcenter ECN contains Bill of Process (BOP) and Bill of Materials (BOM) information that is used to create corresponding modeling instances in CEP, which are then grouped in a CEP change management package. Refer to the Opcenter Execution Core Teamcenter Integration Guide for information on starting Teamcenter packages in CEP.

You can further enrich objects in the package by assigning or removing content through Camstar Change. You are able to perform most Camstar Change functionality on Teamcenter packages. Refer to the Opcenter Execution Core Teamcenter Integration Guide for information. The Teamcenter change package approval process varies slightly from default change management in the application. Refer to "Routing and Canceling Teamcenter Approvals" for information.

MOM PLM Workflow

Teamcenter packages in CEP are assigned the MOM PLM workflow if it is specified in the package creation template. The MOM PLM workflow contains seven default change management specs (steps):

• Download in Process - A Teamcenter package downloaded to CEP begins and remains at this step until successfully downloaded.
Note: You must contact the Teamcenter engineer if the package is not successfully downloaded in to CEP.
• Pending Enrichment - You can enrich objects in the package by assigning or removing content. Refer to "Assigning and Reviewing Package Content" for information on adding package content. You can route a package for approval when it is at this step.
• Pending Approval PLM - A package moves to the Pending Approval PLM step when you route a package for approval at the Pending Enrichment step. Two encrypted PDF files containing the package details and content change history are sent to Teamcenter. Refer to "Routing and Canceling Teamcenter Approvals" for information.
• Rejected - A package is moved to the Rejected step if a Teamcenter engineer rejects a package that was sent for approval.
• Pending Deployment - A package is automatically moved to the Pending Deployment step if the Teamcenter engineer approves a package. You can deploy a package to target machines when it is at this step. Refer to "Deploying a Package" for information.
• Deployment Complete - A package is moved to the Deployment Complete step when it has been successfully deployed to target machines.
• Deployment Incomplete - A package is moved to the Deployment Incomplete step when package deployment fails.

The allowable transaction role assigned to the Download in Process spec is DownloadPermissions.

Pending Enrichment

Indicates that a Teamcenter package has been successfully downloaded in to the application and users can complete package tasks such as update package attributes and assign content.

The allowable transaction role assigned to the Pending Enrichment spec is PEPermissions. Users with the appropriate permissions can perform these actions on a package at the Pending Enrichment step:

• Search for the package
• View the package details
• Edit core package attributes
• Alter the package step
• Assign content to the package
• Deploy the package

Note: Package attributes and assigned package content cannot be modified outside of this step.
