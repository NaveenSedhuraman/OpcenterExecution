Routing and Canceling Teamcenter Approvals

The Teamcenter package approval process varies slightly from the default Camstar Change approval process. The package approval process occurs outside of CEP when the approval request is sent to Teamcenter. You must contact the Teamcenter engineer to cancel a pending approval request.

When Routing a Package to Teamcenter for Approval

You can route a package to Teamcenter for approval when a package is at the Pending Enrichment step in the default MOM PLM workflow. You click the Route for Approval button on the Route for Approval page to begin the approval process. The application changes the package step to Pending Approval PLM when you route the package for approval.

The application sends two encrypted PDF files to Teamcenter via Camstar Interoperability (CIO) to support the approval decision process. One file contains the package details and the other contains content change history. The approvers section of the package details PDF does not contain any information when the approval process is completed in Teamcenter. Refer to the "Viewing Package Details" and "Viewing Content Change History" for information.

After Routing a Package to Teamcenter for Approval

CIO returns a success message if it receives the two PDF files after you route for approval. An error appears if the file transfer does not succeed.

Teamcenter uses package status to determine the approval decision. It sends one of these approval decisions back to CEP and updates the spec or package status.

Approval Decision

Spec or Status

Approved

Moves package to Pending Deployment spec.

Rejected

Moves package to Rejected spec.

Voided

Updates package status to Voided.
Note: You can manually void a package that is rejected. Refer to "Voiding a Package" for information.

When Canceling a Pending PLM Approval

A PLM Approval sent to Teamcenter can be canceled when it is at the Pending PLM Approval step (spec) in the default MOM PLM workflow. You must contact the Teamcenter engineer assigned to the package to request that the pending approval be canceled. You can search for a Teamcenter change package's information by using the Package Search page. Refer to "Searching for Teamcenter Packages" for information. You must then manually move the package back to the Pending Enrichment step by altering the package's step.

Altering the Step of a Package

You can manually alter a package's step in the MOM PLM workflow after you have received confirmation that the approval request was canceled. You must search for a Teamcenter package using the Package Search page and select the package to view the transactions available for the package. You can then use the Alter Step button to move the package back to the Pending Enrichment step of the default MOM PLM workflow. Refer to "Altering the Step of a Package" for information.

Note: Siemens recommends that you do not alter the change package's step until after you have received confirmation from the Teamcenter engineer associated with the package that the approval request has been canceled.

A Teamcenter engineer can also choose to void a package after the package is routed for approval. The application will change the package's status to Voided when it receives the approval decision. Additionally, you can manually alter a package's step to Voided if the Teamcenter engineer returns a Rejected approval status.
