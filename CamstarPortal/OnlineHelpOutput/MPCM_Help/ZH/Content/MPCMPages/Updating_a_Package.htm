更新包

可以为尚未部署的现有更改管理包添加和修改核心属性和部署信息。如果包与需要包审批的更改管理工作流程关联，则可以向所选包指派审批者。如果要将包内容指派工作分配给其他员工，则可以指定协同者。可以指定在“更新包”页上做出更改时，应用程序是否更新在其他更改管理包页上显示的核心包属性。

有关创建包的信息，请参考“创建包”。有关核心属性及其显示在哪些页上的信息，请参考“包信息”。

更新包

仅当包所处的规范关联了含包更新权限的允许事务时，才能更新该包。例如，默认 DraftPermissions 角色包含包更新权限并与默认的“起草”规范关联。因此，具有相应角色的员工可以更新处于“起草”规范的包。

根据是否正在使用需要审批的更改管理工作流程以及是否指定了协同者可以为包指派内容，“更新包”页相应显示一个、两个或三个选项卡：

• 常规

在“常规”选项卡中，可以为所选更改管理包添加和修改核心属性和部署信息。单击“更改管理工作流程”字段旁的“信息”图标，即可查看与该包关联的更改管理工作流程的图表。应用程序将在弹出窗口中显示该图表。还可以指定是否希望协同者为包指派内容。

• 审批者

仅当选择了需要审批的更改管理工作流程时，才会显示“审批者”选项卡。在该选项卡中，可以为更改管理包指定审批模板。

• 协同者

仅当在“常规”选项卡上选择了“所有者和协同者”时，才会显示“协同者”选项卡。在“协同者”选项卡中，可以为更改管理包指定协同者模板，也可以使用“协同者”弹出窗口添加协同者。此外，还可以向协同者发送电子邮件通知以指派内容，并在协同者完成内容指派时向包所有者发送通知。

Note: 通过“更改管理设置”建模对象，系统管理员可以为更改管理配置各种全局设置，包括在更改管理包发生某些事件时向审批者、协同者和包所有者发送的电子邮件通知。有关信息，请参考“添加包审批者”和“添加协同者”。

添加部署详细信息

应用程序要求先输入部署详细信息，然后才能部署包。“部署包”页上将显示创建包或更新包时选择的目标系统。

如果所选目标系统中为 MDB 版本或 Camstar 版本字段配置的值与在源系统中配置的值不同，则应用程序将显示一条警告消息。有关定义服务器目录系统的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。

如果目标系统和源系统上的 MDB 版本和 Camstar 版本字段的值不同，则系统将在警告消息中显示相关警告。此警告消息仅对最接近表格底部的不匹配目标显示。可以关闭警告消息并保存包。收到值不一致错误时，应用程序不会阻止用户创建包。

添加激活详细信息

可以指定在激活包之前激活必要条件包，以说明其他包中实例的相依项。在“必要条件更改包”表格中单击“添加”时，将打开“包搜索”弹出窗口。可以使用此弹出窗口搜索并选择要添加到包的必要条件包。

启用内容协同

“常规”选项卡上的“内容协同者”控件的默认设为“仅所有者”，表示只有具有所有权权限的用户才能为包指派内容。可以通过为“内容协同者”控件选择“所有者和协同者”选项来启用内容协同。在选择“所有者和协同者”选项时，将显示“协同者”选项卡。有关“协同者”选项卡的信息，请参考“添加协同者”。有关协同过程的概述，请参考“了解内容协同”。

Important: 如果将“内容协同者”控件的设置从“所有者和协同者”更改为“仅所有者”，应用程序将检查是否在“协同者”选项卡上指定了协同者。如果未指定协同者，应用程序将不再显示“协同者”选项卡。如果指定了协同者，则会出现一则警告，指示将删除“协同者”选项卡上的所有协同者和协同者信息。如果继续，应用程序将删除相关信息并且不再显示“协同者”选项卡。如果不继续，“所有者和协同者”将保持选中状态。

添加包审批者

审批模板标识需要审批更改管理包的员工以及他们在审批或拒绝包时可以选择的决定选项。必须指定审批模板，才能将审批决定列表与更改管理包相关联。应用程序要求将审批决定列表关联到更改管理包，然后才能路由包以进行审批。

Note: 应用程序将保存审批决定列表以及与指定审批模板关联的任何审批者，但不保存模板本身。

当选择审批模板时，应用程序将填充“审批者”表格和“对所有审批者的指示”字段。可以添加审批者、删除审批者、修改每个审核者的信息以及修改对所有审批者的指示。

Note: 系统管理员可以为包审批者配置电子邮件通知，以在更改管理包需要其审批时通知他们。此外，还可以配置提醒和委派通知。更改管理通知在“更改管理设置”建模对象上配置。有关配置电子邮件通知的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。

选择不同的审批模板时将指派新审批决定列表，并覆盖“审批者”表格，包括对审批者列表的任何更新。只有在单击“更新”后，才不会丢失原始审批决定列表和审批者列表。

在路由包以进行审批时，应用程序将使用“审批者”表格中的“名称”列构建发送给员工（审批者）的通知的通讯组列表。必须为指定为审批者的所有员工添加电子邮件地址。

审批模板、审批决定列表和员工在建模中定义。有关定义审批模板和审批决定列表的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。

Note: 如果不希望自动填充“审批者”表格，则可以定义一个仅包含审批决定列表的审批模板。通过定义空审批模板，可以轻松自定义指派给每个包的审批者。

添加协同者

可以使用“协同者模板”建模对象定义协同者模板。通过协同者模板，可以预定义协同者及其指示。在选择协同者模板时，应用程序将填充“协同者”表格（包括每个协同者的名称和具体指示）以及“对所有协同者的指示”字段。

如果不选择模板，可以手动向表格中添加协同者。无论是否选择模板，都可以添加协同者、删除协同者、修改每个协同者的信息以及修改对所有协同者的指示。应用程序不会在协同者模板中保存对表格中的协同者或指示所做的任何更新。

选择不同的协同者模板将覆盖“协同者”表格中的信息。只有在单击“更新”后，信息才不会丢失。

“给协同者的消息”用于通知协同者他们可以为包指派内容。在选择“更新时向协同者通知指派情况”并单击“更新”时，应用程序将发送该消息。如果选择“更新时向协同者通知指派情况”，但尚未指定“给协同者的消息”，则会显示警告消息。

默认情况下，“更新时向协同者通知指派情况”未选中。如果选中此复选框，单击“更新”后它将保持选中状态。如果关闭“更新包”页后再重新打开，则该字段默认处于未选中状态。

“给所有者的完成消息”用于在协同者完成包内容指派时通知包所有者。每个协同者可单击“指派和审核内容”页上的“完成”。将显示“内容协同者 – 完成的指派”弹出窗口。协同者可以添加注释，然后单击弹出窗口中的“完成”按钮。应用程序将发送消息。

Note: 系统管理员可以为包协同者配置电子邮件通知，以通知他们需要为包指派内容。还可以配置电子邮件通知以在协同者完成包内容指派时通知包所有者。系统管理员还可以配置提醒和委派通知。更改管理通知在“更改管理设置”建模对象上配置。有关配置电子邮件通知的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。

应用程序使用在“电子邮件消息”建模对象中定义的任何电子邮件消息填充“给协同者的消息”和“给所有者的完成消息”列表。“给协同者的消息”默认设为“更改管理设置”建模对象上“更改管理协同指派”通知类型的消息。“给所有者的完成消息”列表默认设为“更改管理设置”建模对象上“更改管理所有者协同指派完成”通知类型的消息。如果未使用“更改管理设置”对象设置通知类型，则应用程序不会为其中任一列表选择默认值。

如果在添加协同者时未指定“持续时间”和“期限”，则应用程序会在“指派和审核内容”页的“协同者任务”选项卡上将“到期日期”设为“指派日期”并将“状态”设为“逾期”。

协同者模板、员工和更改管理设置在建模中定义。有关信息，请参考《  Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。

协同者弹出窗口字段定义

此表定义“协同者”弹出窗口上的字段。

字段

定义

类型

层次

与协同者关联的协同者级别。只能输入数值。表格按此列的升序排序。

级别定义任务必须完成的顺序。如果协作任务是按顺序设置的, 则应用程序在每个任务完成后发送电子邮件通知。如果协作任务是并行设置的, 则应用程序在最后一个协作者完成任务时发送电子邮件通知。

串行表示级别数字较低的协同者必须在级别数字较高的协同者开始之前完成指派内容。并行表示具有相同级别数字的协同者可以同时指派内容。

有关信息，请参考“发送协同电子邮件通知”。

必需

角色

已被指派协同工作责任的角色。选定后，将填充“名称”列表。

必需

名称

已被指派协同工作责任的员工。可用名称取决于在“角色”字段中的选择。如果更改了“角色”字段中的设置，该列表将更新。

必需

替代选项

此值表示非“名称”列中指定的员工是否可以执行协同工作：

• “角色”表示具有相同角色并且在“协同者”表格中的同一级别定义的任何用户都可以执行协同工作。

注：没有默认值。如果选择“角色”，则在选择“更新时向协同者通知指派情况”并单击“更新”时，应用程序将为相同角色的所有用户生成电子邮件通知（如果已配置）。

如何更新包

按照以下步骤更新包：

1. 从更改管理菜单中选择包搜索。 将显示“包搜索”页。
2. 输入或选择一个或多个条件以缩小搜索范围。
3. 单击搜索。 应用程序将在“搜索结果”表格中显示与搜索准则匹配的包。
4. 选择要更新的包。 应用程序将显示该包的相关事务。
5. 单击更新包。将显示“更新包”页。
6. 如果需要，展开部署详细信息部分。
7. 是否要启用内容协同？

如果 ...

操作步骤 ...

是
a. 为内容协同者选择所有者和协同者。将显示“协同者”选项卡。
b. 转到步骤 8。

否

转到步骤 8。
8. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
9. 如果需要，完成“如何为包添加审批者”过程，为包添加审批者。
10. 如果需要，完成“如何为包添加协同者”过程，为包添加协同者。
11. 单击更新。应用程序将保存该更改管理包并显示成功消息。

如何为包添加审批者

按照以下步骤为包添加审批者：

1. 执行“如何更新包”过程。
2. 单击审批者选项卡。
3. 选择审批模板。应用程序将更新“审批者”表格和“对所有审批者的指示”字段，以显示与所选模板关联的审批者和指示。
4. 希望接受审批者的默认列表、删除审批者、修改审批者还是添加审批者？

如果您想完成以下操作 ...

操作步骤 ...

接受默认列表

转到步骤 5。

删除审批者
a. 在“审批者”表格中选择审批者所在的行。
b. 单击删除所选行。应用程序将从表格中移除该审批者。
c. 重复步骤 a–b 以删除其他审批者。
d. 转到步骤 5。

修改审批者
a. 在“审批者”表格中选择审批者所在的行。应用程序将激活该行，以供修改该审批者的信息。
b. 修改审批者信息。有关这些字段的信息，请参考“审批者”选项卡的字段定义表。
c. 重复步骤 a-b 以修改其他审批者。
d. 转到步骤 5。

添加审批者
a. 单击“审批者”表格上的添加新行。应用程序将激活一行，以显示可编辑字段。
b. 输入审批者的级别。
c. 选择审批者的角色。
d. 选择审批者的名称。
e. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
f. 重复步骤 a-e 以添加其他审批者。
g. 转到步骤 5。
5. 如果需要，修改对所有审批者的指示。
6. 单击更新。应用程序将保存该更改管理包并显示成功消息。
