可以选中标题行的“全选”复选框以自动选择所有显示的实例，也可以选中各个实例对应的复选框。

过滤实例列表

该应用程序提供了两种筛选实例列表中显示的实例的方法:

• 输入要查找的实例的名称。当您在 "实例筛选" 字段中输入每个字符时, 应用程序将更新实例列表。在单击 "刷新" 图标后, 应用程序只显示包含这些字符的实例。

指派大量实例和引用

如果分配给包的内容超过1000个实例 (包括引用), 应用程序将显示一条警告消息。消息警告系统性能可能受到影响, 但应用程序不会阻止您保存和稍后部署包。

Siemens 建议查看 CamsarAppServer 应用程序池的 Private Memory Usage 应用程序池设置。在使用 1000 个或更多实例时，可能需要增加该值。有关配置应用程序池内存上限的信息，请参考《  Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《  Opcenter Execution Core 系统管理指南》。

Note: 即使您继续超过包的1000实例阈值, 应用程序也只显示一次警告消息。

导出大量实例和参考时，应用程序可能超时或产生意外的结果。可以在以下位置中通过更新值来增加超时设置：

• Management Studio - 在“站点”树中选择“配置节点”，然后选择“数据库连接”选项卡。字段处于空闲状态后，将“关闭连接”中的值增加到 500 秒。
• Management Studio - 在“站点”树中选择“配置节点”，然后选择“安全”选项卡。将“空闲会话超时”字段中的值增加到 30 分钟。
Note: 有关 Management Studio 的信息，请参考《  Opcenter Execution Medical Device and Diagnostics 系统管理指南》或《  Opcenter Execution Core 系统管理指南》。
• XMLClientConfig.xml - 默认情况下，XMLClientConfig.xml 文件位于 C:\Program Files (x86)\Camstar\InSite Utility 中。将 SendTimeout 和 ReceiveTimeout 的值增加到 500000。
• Web.config - 默认情况下，Web.config 文件位于 C:\Program Files (x86)\Camstar\CamstarPortal 中。在 system.web 节点的 httpRuntime 设置中增加 executionTimeout 值。默认值为 110 秒。

应用程序使您可以添加对角色的引用, 但这可能会导致应用程序在执行包上的任务时超时。必须从包中移除对角色的引用才能完成这些任务。

完成内容协作时

您可以在 "协作任务" 选项卡上的协作网格中查看合作工作的状态。应用程序在加载页时为当前登录的协作者选择默认任务。"协作" 部分的说明中的说明包含当前选定任务的说明。如果在网格中选择了其他任务, 应用程序将更新本节中显示的说明。在网格中一次只能选择一个任务。

您在 "分配和审阅内容" 选项卡上为选定的包分配内容。如果您是协作者, 则在完成协作任务后, 在 "分配和审阅内容" 选项卡上单击 "完成"。

Note: 协作者还可以在建模过程中将内容分配给包或从中移除内容。有关信息, 请参阅 "为在建模中将建模对象实例指派给包"。

内容协作者–在单击 "完成" 时出现 "完成作业" 弹出窗口。此弹出窗口由协作网格和注释部分组成。网格列出了每个合作者的级别、名称、协作任务的状态以及协作任务分配和到期日期。

协作网格只列出当前登录的协作者有资格完成的任务。符合条件的任务包括:

• 分配给当前级别或任何以前级别的当前登录协作者的任务。
• 仅当当前登录的协作者具有使用替换选项定义的角色时, 才在当前级别或任何以前级别分配给其他协作者的任务。

应用程序为当前登录的协作者选择网格中的默认任务。您可以选择任何任务。

当您单击 "完成" 时, 您在此弹出窗口中输入的任何注释都会显示在 "协作者任务" 选项卡上 "协作者" 网格的 "合作者注释" 字段应用程序还会更新 "协作任务" 选项卡上的 "状态" 和 "完成日期" 字段。应用程序向包所有者发送电子邮件通知, 以指示协作者已完成将内容分配给包, 如果包所有者在 "更新包" 页 "协作" 选项卡上指定了 "完成" 消息给所有者。它还向下一个合作者发送电子邮件通知 (如果指定), 如果在 "更新包" 页 "协作" 选项卡上选择了 "更新工作分配通知协作者" 复选框, 则可以开始其协作工作。

如果单击 "完成多次", 应用程序将您的最新评论保存到 "协作者任务" 选项卡上的 "合作者注释" 字段中。

如果包所有者在更新包页上添加协作者时未指定工期和期间, 则应用程序将截止日期设置为 "协作者任务" 选项卡上的 "已分配日期" 和 "状态" 晚。

应用程序使协作者能够在单击 "完成" 按钮后继续将内容分配给包, 只要该包位于配置分配和审阅内容的步骤中。

当您分配内容并完成每个任务时, 应用程序会列出任何新的合格任务, 而无需在完成每个任务后关闭 "分配和审阅内容" 页。
