部署包

Siemens 提供了将包含核心属性和实例数据的更改管理包部署到预定义目标系统的功能。

在部署包之前

在包所处的工步中，“部署”必须作为允许的事务，包才符合部署条件。

在包创建期间或在部署包之前更新包时定义部署详细信息。有关信息，请参考“创建包”和“更新包”。

部署包

部署过程包含两个异步的步骤：

• 构建包文档
• 将包交付到每个目标

单击“部署包”页上的“部署包”选项卡上的“部署”时，应用程序将开始构建包。

“部署包”页包含“部署包”选项卡和“部署状态”选项卡。焦点默认位于“部署包”选项卡上。

默认情况下，在“选择要部署的目标”字段中选择了与选定包相关联的所有可用目标系统。通过清除不需要的目标系统对应的复选框，可以指定要将包部署到的目标系统。在部署包时，必须至少选择一个目标系统。如果在未选择目标系统的情况下尝试部署包，将显示一条错误消息。

目标系统在包创建期间或在更新包时添加到包。可以在“更新包”页上更新包的部署详细信息以添加其他目标。有关信息，请参考“更新包”。

单击“部署包”选项卡上的“部署”按钮时，将显示“部署状态”选项卡。单击“重置”按钮时，将在“要部署的目标”字段中选择所有可用的目标系统，并移除“部署注释”字段中的所有注释。

“部署状态”选项卡用于监视包构建的进度并捕获每个选定目标系统的包部署状态。单击“部署包”选项卡上的“部署”按钮时，应用程序将自动显示“部署状态”选项卡。

“部署状态”选项卡包含“包构建状态”表格和“每个目标的部署状态”表格。“包构建状态”表格仅显示当前包构建或与选定包相关联的上次尝试包构建的数据。“每个目标的部署状态”表格既捕获当前部署的数据，又维护与选定包相关联的先前部署的历史。

“包构建状态”表格和“每个目标的部署状态”表格都实时更新。还可以单击“刷新”按钮以刷新“包构建状态”表格和“每个目标的部署状态”表格。

应用程序将针对“每个目标的部署状态”表格中的每个目标显示一个条目。每个包的两个文件将部署到目标系统。在这两个文件都完成之前，应用程序将显示部署状态为“进行中”。如果这两个文件都成功完成，部署状态将更改为“完成”。如果任一文件失败，处理将停止且部署状态将更改为“失败”。应用程序将以相应的错误填充部署错误。

“包大小”字段显示更改包的总大小，以兆字节 (MB) 为度量单位。默认情况下，包的大小限制为 30 MB。有关在 Portal Studio 的设置中增加最大附件大小限制的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Portal Studio 用户指南》或《  Opcenter Execution Core Portal Studio 用户指南》。

应用程序的默认执行超时设置可能会影响部署过程。默认值为 110 秒。executionTimeout 值位于 system.web 节点的 httpRuntime 设置下的 web.config 文件中。默认情况下，Web.config 文件位于 C:\Program Files (x86)\Camstar\CamstarPortal 中。

如果尝试部署的包中含有超过 1000 个实例，将显示警告消息。

如何部署包

按照以下步骤部署包：

1. 从更改管理菜单中选择包搜索。 将显示“包搜索”页。
2. 输入或选择一个或多个条件以缩小搜索范围。
3. 单击搜索。 应用程序将在“搜索结果”表格中显示与搜索准则匹配的包。
4. 选择要部署的包。 应用程序将显示该包的相关事务。
5. 单击部署包。将显示“部署包”页。
6. 根据您的业务需求输入可选信息。有关可选字段的信息，请参考字段定义表。
7. 单击部署包。应用程序将显示“部署状态”选项卡并构建 XML 部署包文件，然后更新“包构建状态”表格。应用程序随后暂存包以将其部署到指定的目标系统，并实时更新“每个目标的部署状态”表格。
