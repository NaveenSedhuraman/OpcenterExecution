包交付服务

Siemens 提供了一个交付服务，用于监视已部署的更改管理包并将其交付给预定义的目标系统。

包交付服务概述

交付服务可分为以下几个阶段：

• 监视
• 读取
• 交付
• 更新

监视阶段

部署包事务标记要由交付服务提取的包。在“部署包”页上的“每个目标的部署状态”表格中，符合提取条件的包的状态为“进行中”。交付服务根据计划的业务规则查找合格的包。默认的计划的业务规则名为 SBR_DEPLOY，每五分钟运行一次。

必须先从“部署包”页部署包，之后此包才能由交付服务进行交付。有关信息，请参考“部署包”。

读取阶段

交付服务将查看源系统数据库，并读取包的部署详细信息以确定包的目标系统。

交付阶段

交付服务交付更改管理包，并将 XML 标头文件导出至所选目标系统。应用程序根据计划的业务规则调用目标系统上的 XML 标头文件。默认的计划的业务规则名为 SBR_ACTIVATION，每五分钟运行一次。然后，便可在目标系统中激活更改管理包。有关信息，请参考“激活包”。

更新阶段

交付服务根据计划的业务规则在尝试交付后，更新源系统的包交付状态。默认的计划的业务规则名为 SBR_DEPLOYSTATUS，每五分钟运行一次。该服务在“部署包”页上实时更新“每个目标的部署状态”表格，每个交付的状态分别更新为“完成”或“失败”，并为失败的交付提供错误消息。如果首次交付失败，则必须重新部署包，交付服务才能再次尝试交付。有关信息，请参考“部署包”。

有关计划的业务规则的信息，请参考《  Opcenter Execution Medical Device and Diagnostics Modeling 用户指南》或《  Opcenter Execution Core Modeling 用户指南》。可以使用默认规则，也可以定义新规则。
