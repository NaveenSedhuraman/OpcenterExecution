Zuweisen und Überprüfen des Paketinhalts

Sie können Modellierungsobjekt-Instanzen und Referenzen zu jedem vorhandenen Paket hinzufügen, das sich in einem Entwurfsschritt oder in einem Schritt befindet, der die Zuweisung und Überprüfung aufgrund seiner Konfiguration zulässt. Die Seite "Inhalt zuweisen und überprüfen" ermöglicht es Ihnen, den Inhalt eines bestehenden Pakets anzupassen und zu überprüfen.

Voraussetzungen zum Zuweisen von Paketinhalten

Einige Modellierungsobjekt-Instanzen, die auf der Seite "Inhalt zuweisen und überprüfen" ausgewählt werden können, dürfen nicht einem Paket zugewiesen werden, das auf einem Zielsystem bereitgestellt werden soll. Einige Modellierungsobjekte können Konflikte verursachen, wenn Pakete auf einem Zielsystem aktiviert werden.

Modellierungsobjekte, die keinem Paket zugewiesen werden dürfen

Siemens empfiehlt, die folgenden Modellierungsobjekt-Instanzen nicht einem Paket zuzuweisen, das auf einem Zielsystem bereitgestellt werden soll.

Darüber hinaus können diese Modellierungsobjekte durch Auswahl von "Referenzen hinzufügen" automatisch zur Tabelle "Objekttypname" hinzugefügt werden. Siemens empfiehlt, diese Instanzen manuell aus der Tabelle zu entfernen, um Konflikte zu vermeiden.

Hinweis: Durch Hinzufügen von Modellierungsobjekten zur Tabelle "Auszuschließende Modellierungsobjekte" im Modellierungsobjekt "Fabrik" wird gewährleistet, dass diese Instanzen beim Export nicht angezeigt werden und nicht ausgewählt werden können. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Instanz-IDs stimmen auf verschiedenen Systemen in der Regel nur dann überein, wenn die Datenbank geklont wurde.

Webpart

Sie sollten virtuelle Webparts mit Portal Studio exportieren und importieren. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch oder im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Modellierungsobjekte, die Konflikte verursachen können

Siemens empfiehlt, die folgenden Modellierungsobjekt-Instanzen nur wohlüberlegt einem Paket zuzuweisen, das auf einem Zielsystem bereitgestellt werden soll.

Modellierungsobjekt

Grund für Nichtzuweisung

Mitarbeiter

Eine Modellierungsobjektinstanz vom Typ "Mitarbeiter" kann Zirkelbezüge enthalten. Instanzen können auch auf Rollen-IDs verweisen, die im Zielsystem nicht vorhanden sind.

Anforderung für elektr. Unterschrift modellieren

Eine Instanz "Anforderung für elektr. Unterschrift modellieren" kann Zirkelbezüge enthalten. Die Benutzer, Rollen und elektronischen Unterschriften können auf Quell- und Zielsystem unterschiedlich sein.

Nummerierungsregel

Bei Nummerierungsregel-Instanzen können Konflikte auftreten, wenn die Nummer auf dem Quellsystem nicht mit der Nummer auf dem Zielsystem übereinstimmt.

Organisation

Eine Modellierungsobjektinstanz vom Typ "Organisation" kann Zirkelbezüge enthalten.

Historienansicht

Die Anzeige des Produktionsprotokolls kann sich auf dem Zielsystem unerwartet ändern. Benutzerdefinierte Änderungen auf dem Zielsystem können überschrieben werden.

Geplante Geschäftsregel

Beim Import einer geplanten Geschäftsregel, die aus einem anderen System exportiert wurde, wird das Startdatum auf dem Zielsystem aktualisiert. Darüber hinaus müssen Sie u. U. den Camstar Deployment Server und den Camstar Notification Server neu starten, wenn die Instanz der geplanten Geschäftsregel, die auf dem Zielsystem aktualisiert oder erstellt wird, die Bereitstellung oder Benachrichtigungen beeinflusst.

Revisionierte Objekte (ROs)

Die Anwendung legt die erste im Import von importierten Objekten enthaltene Revision als Standardrevision fest, wenn die beiden folgenden Bedingungen erfüllt sind:

• Auf dem Zielsystem gibt es keine Standardrevision für ein importiertes RO.
• Der Export enthält keine Standardrevision für das RO.

Veraltete Modellierungsobjekte, die Sie einem Paket zuweisen können

Bestehende Kunden, die Modellierungsobjekt-Instanzen verwenden, die in späteren Versionen veraltet sind, können die betreffenden Modellierungsobjekt-Instanzen weiterhin exportieren und importieren:

• MSMQ-Transport
• Waage
• Waagengruppe
• Waagencode
• Waagenstatusgrund
• Webservice-Transport

Beim Zuweisen und Überprüfen des Paketinhalts

Nur Paketeigentümer und Projektmitarbeiter können Paketen Inhalte zuweisen. Auf der Seite "Inhalt zuweisen und überprüfen" werden abhängig davon, ob die Inhaltszusammenarbeit für das Paket aktiviert wurde, eine Registerkarte oder zwei Registerkarten angezeigt:

• Inhalt zuweisen und überprüfen

Auf der Registerkarte "Inhalt zuweisen und überprüfen" können Sie Modellierungsobjekt-Instanzen und Referenzen zu einem Paket hinzufügen. Sie können außerdem eine Zusammenfassung der Informationen für das Paket anzeigen. Sie können außerdem eine Zusammenfassung der Informationen für das Paket anzeigen und alle zugeordneten Objekte und deren Referenzen in einer hierarchischen Ansicht anzeigen.

• Projektmitarbeiteraufgaben

Die Registerkarte "Projektmitarbeiteraufgaben" wird nur angezeigt, wenn die Inhaltszusammenarbeit für das Paket aktiviert wurde. Die Registerkarte umfasst die Tabelle "Projektmitarbeiter" und einen Abschnitt mit Anweisungen. In der Tabelle "Projektmitarbeiter" werden für jeden Projektmitarbeiter Angaben zu dessen Level, Name, Status der Zusammenarbeit, Zuweisungs-, Fälligkeits- und Fertigstellungsdatum der Zusammenarbeit, Projektmitarbeiterkommentare sowie Projektmitarbeiteranweisungen aufgelistet. Anhand dieser Informationen kann der Paketeigentümer den Status der Zusammenarbeit anzeigen und entscheiden, ob eine Folgemaßnahme erforderlich ist. Im Anweisungsabschnitt werden die Anweisungen des Paketeigentümers für alle Projektmitarbeiter und spezielle Anweisungen für die ausgewählte Aufgabe in der Tabelle "Projektmitarbeiter" aufgelistet. Die Anweisungen sind schreibgeschützt. Die Anweisungen werden nur angezeigt, wenn Sie ein dem Paket zugewiesener Projektmitarbeiter sind. Informationen zum Aktivieren der Inhaltszusammenarbeit finden Sie unter "Aktualisieren eines Pakets".

Hinweis: Die Anwendung zeigt keine Anweisungen an, wenn Sie keine Zusammenarbeitsaufgabe durchführen können. Dies passiert, wenn der Paketeigentümer oder ein Benutzer mit der Eigentümerrolle Inhalte zuweist und die Ersetzungsoption für alle Aufgaben auf "Kein" festgelegt ist.

Auf der Registerkarte "Inhalt zuweisen und überprüfen" können nur Modellierungsobjekte ausgewählt werden, für die Instanzen definiert wurden. Die Anwendung zeigt keine Modellierungsobjekte ohne definierte Instanzen an.

Sie können in der Liste "Objekte" nach einem spezifischen Modellierungsobjekt suchen. Geben Sie dazu die ersten Zeichen des Modellierungsobjektnamens oder den vollständigen Namen in das Filterfeld ein. Die Anwendung zeigt in der Liste "Instanzen" alle Instanzen an, die für ein ausgewähltes Modellierungsobjekt definiert sind. Sie können die Liste der Instanzen auch filtern. Informationen hierzu finden Sie unter "Filtern der Instanzliste".

Hinweis: Einstellungen in der Designer-Anwendung bestimmen, ob Modellierungsobjekt-Instanzen verfügbar sind, die zu einem Paket hinzugefügt werden können. Nicht sichtbare, abstrakte Modellierungsobjekte sind nicht verfügbar. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Designer-Benutzerhandbuch oder im Opcenter Execution Core Designer-Benutzerhandbuch

Sie können das Kontrollkästchen "Alle auswählen" in der Kopfzeile "Instanzen" aktivieren, um alle angezeigten Instanzen automatisch auszuwählen, oder die Kontrollkästchen für einzelne Instanzen aktivieren.

Durch Klicken auf die Schaltfläche "Zur Tabelle hinzufügen" werden die ausgewählten Instanzen zur Tabelle "Objekttypname" hinzugefügt und die Informationen der "Paketinhaltszusammenfassung" aktualisiert. Einträge in der Tabelle werden alphabetisch nach dem Namen des Modellierungsobjekts und dann nach dem Namen der Modellierungsobjekt-Instanz sortiert. Hinzugefügte Instanzen bleiben in der Liste "Instanzen", werden aber als ausgewählt und deaktiviert angezeigt.

Hinweis: Sie sollten alle Modellierungsobjekte und gegebenenfalls alle Modellierungsobjekt-Instanzen hinzufügen, bevor Sie Referenzen hinzufügen.

Das Hinzufügen von Referenzen ist optional, wenn Paketinhalte zugewiesen werden. Objekte werden weiterhin in der richtigen Reihenfolge bereitgestellt, wenn Sie sich entscheiden, Referenzen nicht einzufügen. Sie müssen Modellierungsobjekt-Instanzen nur dann in der richtigen Reihenfolge zuweisen oder Referenzen nur dann hinzufügen, wenn die Option "Änderungsmanagement-Speicherung" aktiviert auf "Ein" festgelegt ist. Informationen zur Option "Änderungsmanagement-Speicherung" finden Sie unter "Zuweisen von Modellierungsobjekt-Instanzen zu einem Paket in der Modellierung".

Beispiel:

Sie wählen die Batch-, Rack- und Tray-Instanzen des Containerlevel-Modellierungsobjekts aus, die Sie dem Paket zuweisen möchten. Durch Klicken auf die Schaltfläche "Zur Tabelle hinzufügen" wird der Containerlevel in der Tabelle "Objekttypname" angezeigt und die Tabelle "Paketinhaltszusammenfassung" aktualisiert, um ein Modellierungsobjekt und 3 Instanzen anzuzeigen. Darüber hinaus verbleiben "Batch", "Rack" und "Tray" in der Liste "Instanzen", aber sie sind ausgewählt und deaktiviert.

Filtern der Instanzliste

Die Anwendung bietet zwei Methoden zum Filtern von Instanzen, die in der Liste "Instanzen" angezeigt werden:

• Eingabe des Namens der Instanz, die Sie suchen möchten. Die Anwendung aktualisiert die Liste der Instanzen, während Sie die einzelnen Zeichen im Filterfeld "Instanzen" eingeben. Die Anwendung zeigt nur die Instanzen an, die diese Zeichen enthalten, nachdem Sie auf das "Aktualisieren"-Icon klicken.
• Durch Klicken auf das Filter-Icon können Sie andere Kriterien als den Instanznamen verwenden. Die Anwendung zeigt das Popup "Instanzenfilter" an, wenn Sie auf das Filter-Icon klicken. Verwenden Sie dieses Popup, um Kriterien wie "Zeitstempel", "Letzter Mitarbeiter, der die Instanz aktualisiert hat" usw. anzugeben. Klicken Sie auf "Anwenden", um das Popup "Instanzenfilter" zu schließen und alle Instanzen anzuzeigen, die diesen Kriterien entsprechen.

Die Anwendung zeigt ein Icon neben dem Filter-Icon an, das angibt, dass die Liste gefiltert wurde.

Durch Klicken auf das "Aktualisieren"-Icon werden alle Filterkriterien entfernt und die Anwendung zeigt alle Instanzen an, die für das ausgewählte Objekt definiert wurden. Die Auswahl eines anderen Modellierungsobjekts im Feld "Objekte auswählen" entfernt ebenfalls alle Filterkriterien. Durch Klicken auf das Icon neben dem Filter-Icon setzen Sie den Filter ebenfalls zurück.

Anzeigen, wo Instanzen verwendet werden

Durch Klicken auf die Schaltfläche "Wo genutzt" wird das Popup "Wo genutzt" angezeigt, wenn Folgendes zutrifft:

• Sie haben mindestens eine Instanz in der Tabelle "Objekttypname" ausgewählt und
• für die Fabrik, die dem angemeldeten Mitarbeiter zugeordnet ist, sind Einstellungen für die Änderungsverwaltung angegeben.

Im Popup "Wo genutzt" werden die Modellierungsobjekt-Instanzen angezeigt, die direkt die Instanzen referenzieren, die in der Tabelle "Objekttypname" ausgewählt sind. Zum Beispiel zeigt die Anwendung alle Spezifikationen an, in denen die von Ihnen ausgewählte Operation-Instanz referenziert wird. Die Anwendung zeigt nicht die Arbeitspläne an, in denen die Spezifikationen referenziert werden, in denen die von Ihnen ausgewählte Operation-Instanz referenziert wird.

Hinweis: Die Einstellungen für die Änderungsverwaltung bestimmen, wie viele Instanzen Sie in der Tabelle "Objekttypname" auswählen können. Weitere Informationen zum Konfigurieren der Einstellungen für die Änderungsverwaltung finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Sie können die "Wo genutzt"-Modellierungsobjekt-Instanzen zum Änderungsverwaltungspaket hinzufügen. Wählen Sie die Instanzen im Popup "Wo genutzt" aus und klicken Sie auf "Hinzufügen".

Hinweis: Die Anwendung zeigt eine Fehlermeldung an, wenn Sie auf die Schaltfläche "Wo genutzt" klicken und eine der folgenden Bedingungen zutrifft:
• Dem angemeldeten Benutzer sind keine Einstellungen für die Änderungsverwaltung zugeordnet.
• In der Tabelle "Objekttypname" sind keine Instanzen ausgewählt.
• Die Anzahl der ausgewählten Instanzen ist größer als der Wert für "Wo genutzt", der in den Einstellungen für die Änderungsverwaltung konfiguriert ist.

Beim Zuweisen einer großen Anzahl von Instanzen und Referenzen

Die Anwendung zeigt eine Warnmeldung an, wenn der Inhalt, der dem Paket zugewiesen wird, 1000 Instanzen, einschließlich Referenzen, überschreitet. Die Meldung warnt, dass die Systemleistung möglicherweise beeinträchtigt werden kann, aber die Anwendung hindert Sie nicht daran, zu speichern und das Paket später bereitzustellen.

Siemens empfiehlt, die Einstellung des Anwendungspools zur Auslastung des privaten Speichers für den CamsarAppServer-Anwendungspool zu überprüfen. Möglicherweise müssen Sie den Wert erhöhen, wenn Sie mit 1000 oder mehr Instanzen arbeiten. Weitere Informationen zum Konfigurieren der maximalen Speichergrenze für den Anwendungspool finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Hinweis: Die Anwendung zeigt die Warnmeldung nur einmal an, selbst wenn Sie weiterhin den Grenzwert von 1000 Instanzen für das Paket überschreiten.

In der Anwendung können eine Zeitüberschreitung oder unerwartete Ergebnisse auftreten, wenn eine große Anzahl von Datensätzen und Referenzen exportiert wird. Sie können die Timeout-Einstellungen erhöhen, indem Sie die Werte an den folgenden Stellen aktualisieren:

• Management Studio - Wählen Sie den Knoten "Konfiguration" (Configuration) in der Baumstruktur "Standorte" (Sites) und dann die Registerkarte "Datenbankverbindungen" (Database Connections) aus. Erhöhen Sie den Wert im Feld "Verbindungen schließen nach Leerlauf" (Close Connections after being idle) auf 500 Sekunden.
• Management Studio - Wählen Sie den Knoten "Konfiguration" (Configuration) in der Baumstruktur "Standorte" (Sites) und dann die Registerkarte "Sicherheit" (Security) aus. Erhöhen Sie den Wert im Feld "Timeout für Sitzung im Leerlauf" (Idle Session Timeout) auf 30 Minuten.
Hinweis: Weitere Informationen zu Management Studio finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.
• XMLConfigClientConfig.xml – Die Datei "XMLClientConfig.xml" befindet sich standardmäßig in "C:\Programme (x86)\Camstar\InSite Utility". Erhöhen Sie die Werte für "SendTimeout" und "ReceiveTimeout" auf "500000".
• Web.config – Die Datei "Web.config" befindet sich standardmäßig in "C:\Programme (x86)\Camstar\CamstarPortal". Erhöhen Sie den Wert "executionTimeout" in der Einstellung "httpRuntime" des Knotens "system.web". Der Standardwert ist 110 Sekunden.

Mit der Anwendung können Sie Referenzen zu Rollen hinzufügen. Beim Ausführen von Aufgaben an einem Paket kann dies jedoch zu einer Zeitüberschreitung der Anwendung führen. Sie müssen Referenzen zu Rollen aus einem Paket entfernen, um diese Aufgaben auszuführen.

Beim Fertigstellen der Inhaltszusammenarbeit

Sie können den Status der Zusammenarbeit in der Tabelle "Projektmitarbeiter" auf der Registerkarte "Projektmitarbeiteraufgaben" anzeigen. Die Anwendung wählt die Standardaufgabe für den derzeit angemeldeten Projektmitarbeiter aus, wenn die Seite geladen wird. Die Anweisungen im Abschnitt "Anweisungen für Projektmitarbeiter" enthalten Anweisungen für die derzeit ausgewählte Aufgabe. Die Anwendung aktualisiert die in diesem Abschnitt angezeigten Anweisungen, wenn Sie in der Tabelle eine andere Aufgabe auswählen. Sie können immer nur eine Aufgabe in der Tabelle auswählen.

Auf der Registerkarte "Inhalt zuweisen und überprüfen" weisen Sie dem ausgewählten Paket Inhalte zu. Als Projektmitarbeiter klicken Sie auf der Registerkarte "Inhalt zuweisen und überprüfen" auf "Beendet", wenn Sie die Ausführung Ihrer Aufgaben für die Zusammenarbeit fertigstellen möchten.

Hinweis: Der Projektmitarbeiter kann außerdem Inhalte zu einem Paket in der Modellierung zuweisen oder aus einem Paket entfernen. Informationen hierzu finden Sie unter "Zuweisen von Modellierungsobjekt-Instanzen zu einem Paket in der Modellierung".

Wenn Sie auf "Beendet" klicken, wird das Popup "Inhaltsprojektmitarbeiter – Zuweisung abgeschlossen" angezeigt. Dieses Popup umfasst die Tabelle "Projektmitarbeiter " und einen Kommentarabschnitt. In der Tabelle werden Level der einzelnen Projektmitarbeiter, Name, Status der Zusammenarbeitsaufgaben und die Daten aufgelistet, an denen die Zusammenarbeitsaufgaben zugewiesen wurden und fällig sind.

In der Tabelle "Projektmitarbeiter" werden nur die Aufgaben aufgelistet, die der derzeit angemeldete Projektmitarbeiter durchführen darf. Zulässige Aufgaben sind u. a.:

• Aufgaben, die dem derzeit angemeldeten Projektmitarbeiter auf dem aktuellen Level oder einem darunter liegenden Level zugewiesen wurden.
• Aufgaben, die einem anderen Projektmitarbeiter auf dem aktuellen Level oder einem darunter liegenden Level zugewiesen wurden, wenn der derzeit angemeldete Projektmitarbeiter die mit der Ersetzungsoption definierte Rolle hat.

Die Anwendung wählt die Standardaufgabe in der Tabelle für den derzeit angemeldeten Projektmitarbeiter aus. Sie können eine beliebige Aufgabe auswählen.

Die Kommentare, die Sie in diesem Popup eingeben, werden im Feld "Projektmitarbeiterkommentare" der Tabelle "Projektmitarbeiter" auf der Registerkarte "Projektmitarbeiteraufgaben" angezeigt, wenn Sie auf "Beendet" klicken. Die Anwendung aktualisiert außerdem die Felder "Status" und "Fertigstellungsdatum" auf der Registerkarte "Projektmitarbeiteraufgaben". Wenn der Paketeigentümer auf der Registerkarte "Projektmitarbeiter" auf der Seite "Paket aktualisieren" die Einstellung "Beendigungsnachricht an Eigentümer" angegeben hat, sendet die Anwendung eine E-Mail-Benachrichtigung an den Paketeigentümer, um diesen darauf hinzuweisen, dass der Projektmitarbeiter die Zuweisung von Inhalten zum Paket beendet hat. Wenn auf der Registerkarte "Projektmitarbeiter" der Seite "Paket aktualisieren" das Kontrollkästchen "Projektmitarbeiter über Zuweisungen bei Aktualisierung benachrichtigen" aktiviert wurde, sendet die Anwendung außerdem eine E-Mail-Benachrichtigung an den nächsten Projektmitarbeiter (sofern angegeben), um diesen darauf hinzuweisen, dass er (bzw. sie) mit den eigenen Aufgaben für die Zusammenarbeit beginnen kann.

Wenn Sie mehrmals auf "Beendet" klicken, speichert die Anwendung die Kommentare, die Sie zuletzt in das Feld "Projektmitarbeiterkommentare" auf der Registerkarte "Projektmitarbeiteraufgaben" eingegeben haben.

Falls der Paketeigentümer beim Hinzufügen von Projektmitarbeitern auf der Seite "Paket aktualisieren" keine Angaben zu Dauer und Zeitraum gemacht hat, legt die Anwendung auf der Registerkarte "Projektmitarbeiteraufgaben" als Fälligkeitsdatum das Zuweisungsdatum und als Status die Option "Spät" fest.

Nachdem auf die Schaltfläche "Beendet" geklickt wurde, können Projektmitarbeiter dem Paket in der Anwendung weiterhin Inhalte zuweisen, solange sich das Paket in einem Schritt befindet, für den die Zuweisung und Prüfung von Inhalten konfiguriert wurde.

Die Anwendung listet alle neuen zulässigen Aufgaben auf, wenn Sie Inhalte zuweisen und die einzelnen Aufgaben durchführen, ohne dass Sie die Seite "Inhalt zuweisen und überprüfen" schließen müssen, nachdem die jeweilige Aufgabe beendet wurde.

Anzeigen der Hierarchie der zugewiesenen Instanzen

Durch Klicken auf die Schaltfläche "Hierarchie" wird das schreibgeschützte Popup "Hierarchie" angezeigt. Im Popup "Hierarchie" wird eine eingerückte Strukturansicht der Modellierungsobjekt-Instanzen angezeigt, die zur Tabelle "Objekttypname" hinzugefügt wurden.

Instanzen, die der Tabelle durch Klicken auf die Schaltfläche "Zur Tabelle hinzufügen" oder über das Popup "Wo genutzt" hinzugefügt wurden, werden fett in schwarzer Schrift angezeigt. Instanzen, die der Tabelle durch Klicken auf die Schaltfläche "Referenzen hinzufügen" hinzugefügt wurden, werden in normaler schwarzer Schrift unter den fett ausgezeichneten Einträgen angezeigt.

Sie müssen mindestens eine Modellierungsobjekt-Instanz zur Tabelle "Objekttypname" hinzufügen, um die Hierarchie anzuzeigen. Die Anwendung zeigt eine Fehlermeldung an, wenn Sie dies nicht tun.

Anzeigen der Paketinhaltszusammenfassung

Durch Klicken auf das Symbol "Paketinhaltszusammenfassung" unterhalb der Tabelle "Objekttypname" wird das Popup "Paketinhaltszusammenfassung" angezeigt. Das Popup "Paketinhaltszusammenfassung" zeigt die folgenden Informationen an:

• Die Anzahl der zur Zuweisung zur Verfügung stehenden Modellierungsobjekte
• Die Gesamtzahl der zur Zuweisung zur Verfügung stehenden Modellierungsobjekt-Instanzen
• Die Anzahl der Modellierungsobjekte, die dem Paket zugewiesen sind
• Die Gesamtzahl der Modellierungsobjekt-Instanzen, die dem Paket zugewiesen sind
• Die Anzahl der Modellierungsobjekte, die von den ausgewählten Modellierungsobjekten referenziert werden
• Die Gesamtzahl der referenzierten Modellierungsobjekt-Instanzen, die dem Paket zugewiesen sind

Die Anwendung aktualisiert die Informationen zur Paketinhaltszusammenfassung, während Sie Instanzen und Referenzen zum Paket hinzufügen und davon entfernen.

Feld

Definition

Typ

Tabelle "Projektmitarbeiter"

Tabelle mit einer Liste aller Aufgaben, die der derzeit angemeldete Benutzer durchführen darf. Angezeigt werden Level, Status der Zusammenarbeit, Zuweisungsdatum und Fälligkeitsdatum.

Nur Anzeige

Level

Die mit dem Projektmitarbeiter verbundenen Zusammenarbeits-Level. Das Level wird jeweils ausschließlich durch numerische Werte dargestellt. Die Tabelle wird nach dieser Spalte in aufsteigender Reihenfolge sortiert.

Der Level definiert die Reihenfolge, in der die Aufgaben erledigt werden müssen. Die Anwendung sendet E-Mail-Benachrichtigungen, nachdem die einzelnen Aufgaben erledigt wurden, wenn die Zusammenarbeitsaufgaben sequenziell eingerichtet wurden. Die Anwendung sendet E-Mail-Benachrichtigungen, nachdem der letzte Projektmitarbeiter die zugewiesene Aufgabe erledigt hat, wenn die Zusammenarbeitsaufgaben parallel eingerichtet wurden.

Seriell bedeutet, dass Projektmitarbeiter mit einer niedrigeren Level-Nummer die Zuweisung von Inhalten erst beenden müssen, bevor Projektmitarbeiter mit höheren Level-Nummern beginnen können. Parallel bedeutet, dass Projektmitarbeiter derselben Level-Nummer gleichzeitig Inhalte zuweisen können.

Nur Anzeige

Name

Der Mitarbeiter, dem die Zuständigkeit für die Zusammenarbeit zugewiesen wurde.

Nur Anzeige

Status

Status der Zusammenarbeit. Die Zellenfarbe ist dynamisch und hängt vom Status ab. Es gibt folgende Statuseinstellungen:

• Ausstehend – Der Mitarbeiter hat die Aufgabe der Zusammenarbeit noch nicht beendet. Der Mitarbeiter hat die Aufgabe möglicherweise bereits oder noch nicht begonnen. Zelle wird orange hervorgehoben.
• Beendet – Der Mitarbeiter hat die Aufgabe der Zusammenarbeit fertiggestellt, unabhängig davon, ob die Arbeit verspätet fertiggestellt wurde. Die Arbeit ist fertiggestellt, wenn der Projektmitarbeiter auf der Registerkarte "Inhalt zuweisen und überprüfen" der Seite "Inhalt zuweisen und überprüfen" im Popup "Inhaltsprojektmitarbeiter – Zuweisung abgeschlossen" auf die Schaltfläche "Beendet" klickt. Die Zelle ist grün hervorgehoben.
• Spät – Der Mitarbeiter hat die Aufgabe der Zusammenarbeit noch nicht fertiggestellt, und die für die Erledigung der Aufgabe zugeteilte Zeit ist abgelaufen. Die Aufgabe ist verspätet, wenn das aktuelle Datum später ist als das Fälligkeitsdatum. Der Mitarbeiter hat die Aufgabe möglicherweise bereits oder noch nicht begonnen. Zelle wird rot hervorgehoben.

Nur Anzeige

Zuweisungsdatum

Das Datum, an dem der Paketeigentümer die Arbeit der Zusammenarbeit dem Projektmitarbeiter auf der Registerkarte "Projektmitarbeiter" der Seite "Paket aktualisieren" zugewiesen hat. Das Datum wird im Format MM/TT/JJJJ angegeben.

Nur Anzeige

Fälligkeitsdatum

Das Datum, an dem die Arbeit der Zusammenarbeit fällig ist. Das Fälligkeitsdatum wird anhand des Datums berechnet, an dem der Paketeigentümer die Arbeit der Zusammenarbeit dem Projektmitarbeiter auf der Registerkarte "Projektmitarbeiter" der Seite "Paket aktualisieren" zugewiesen hat; zu diesem Datum wird die auf derselben Registerkarte für "Dauer/Zeitraum" eingegebene Zeit addiert. Das Datum wird im Format MM/TT/JJJJ angegeben.

Nur Anzeige

Kommentare

Text (bis zu 255 Zeichen), der die fertiggestellte Zusammenarbeit beschreibt.

Vorgehensweise zum Anzeigen der Paketinhaltszusammenfassung

Gehen Sie folgendermaßen vor, um die Paketinhaltszusammenfassung anzuzeigen:

1. Führen Sie die Schritte 1 bis 5 unter "Vorgehensweise zum Zuweisen von Inhalten an ein Paket" durch.
2. Klicken Sie auf das Symbol Paketinhaltszusammenfassung. Die Anwendung zeigt das Popup "Paketinhaltszusammenfassung" an.
3. Klicken Sie auf Schließen, um das Popup "Paketinhaltszusammenfassung" zu schließen.

Vorgehensweise zum Hinzufügen von "Wo genutzt"-Instanzen zu einem Paket

Gehen Sie folgendermaßen vor, um "Wo genutzt"-Instanzen zu einem Paket hinzuzufügen:

1. Führen Sie die Schritte 1 bis 9 unter "Vorgehensweise zum Zuweisen von Inhalten an ein Paket" durch.
2. Wählen Sie mindestens eine Instanz aus.
3. Klicken Sie auf Wo genutzt. Die Anwendung zeigt das Popup "Wo genutzt" an.
4. Wählen Sie eine oder mehrere Instanzen aus, die Sie dem Paket hinzufügen möchten.
5. Klicken Sie auf Hinzufügen. Die Anwendung fügt die ausgewählten Instanzen zum Paket hinzu und schließt das Popup "Wo genutzt".

Vorgehensweise zum Anzeigen einer Hierarchie zugewiesener Instanzen

1. Führen Sie die Schritte 1 bis 10 unter "Vorgehensweise zum Zuweisen von Inhalten an ein Paket" durch.
2. Klicken Sie auf Hierarchie. Die Anwendung zeigt das Popup "Hierarchie" an.
3. Klicken Sie auf Schließen.

Vorgehensweise zum Anzeigen von Projektmitarbeiteraufgaben

Gehen Sie folgendermaßen vor, um Projektmitarbeiteraufgaben anzuzeigen:

1. Führen Sie die Schritte 1 bis 5 unter "Vorgehensweise zum Zuweisen von Inhalten an ein Paket" durch.
2. Klicken Sie auf die Registerkarte Projektmitarbeiteraufgaben. Die Anwendung zeigt alle Zusammenarbeitsaufgaben für das Paket und die Anweisungen für die derzeit in der Tabelle "Projektmitarbeiter" ausgewählte Aufgabe an.
