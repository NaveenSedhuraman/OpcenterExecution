Erstellen eines Pakets

Das Erstellen eines Änderungsverwaltungspakets ist der erste Schritt bei der Bereitstellung von Updates für die Modellierung von Daten und Prozessabläufen von einem Quellsystem auf einem oder mehreren Zielsystemen.

Vor dem Erstellen eines Pakets

Sie müssen die Änderungsverwaltung in der Modellierung konfigurieren, bevor Sie Änderungsverwaltungspakete erstellen und bereitstellen können. Zunächst müssen Sie die folgenden Modellierungsobjekte definieren:

• Paketerstellungsgrund – Gründe für das Erstellen eines Pakets
• Serverkatalogsystem – Quell- und Zielsysteme für ein Paket
Hinweis: Die Anwendung zeigt eine Fehlermeldung an, wenn Sie die Seite "Paket erstellen" öffnen, falls keine Serverkatalogsysteme definiert wurden.

Sie müssen Mitarbeitern Rollen mit Berechtigungen zur Änderungsverwaltung zuweisen, bevor Sie Pakete erstellen. Dem Mitarbeiter, der ein Änderungsverwaltungspaket erstellt, muss eine Rolle mit Berechtigungen zum Erstellen von Paketen zugewiesen sein. Siemens stellt eine Paketersteller-Rolle zur Verfügung, die standardmäßig über diese Berechtigungen verfügt. Rollen werden Mitarbeitern in der Modellierung zugewiesen.

Weitere Informationen zum Definieren der Modellierungsobjekte, die für die Ausführung von Änderungsverwaltungstransaktionen erforderlich sind, finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch. Weitere Informationen zu den Standardrollen für die Änderungsverwaltung finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Beim Erstellen eines Pakets

Sie müssen entweder einen standardmäßigen oder benutzerdefinierten Arbeitsplan zur Änderungsverwaltung auswählen, wenn Sie ein Paket erstellen. Das "Informationen"-Icon neben dem Feld "Änderungsmanagement-Arbeitsplan" zeigt ein Diagramm des ausgewählten Arbeitsplans in einem Popup an, wenn Sie darauf klicken. Sofern eine Beschreibung des Arbeitsplans zur Änderungsverwaltung bei der Definition der Instanz eingegeben wurde, zeigt die Anwendung diese ebenfalls an.

Hinweis: Nachdem das Paket erstellt wurde, können Sie den zugewiesenen Arbeitsplan nicht mehr aktualisieren oder ändern.

Siemens stellt standardmäßig drei Arbeitspläne für das Erstellen und Bereitstellen eines Änderungsverwaltungspakets zur Verfügung:

• Keine Genehmigung (No Approval) – Ermöglicht es Ihnen, Änderungsverwaltungspakete aus dem Quellsystem bereitzustellen, ohne dass Genehmigungen für das Paket erforderlich sind.
• Camstar – Erfordert Genehmigungen für Änderungsverwaltungspakete vor der Bereitstellung vom Quellsystem. Diese Option aktiviert die Registerkarte "Genehmiger" auf der Seite "Paket aktualisieren", wodurch Sie die Genehmigungsvorlage und die zugewiesenen Genehmiger anzeigen können.
• PLM – Erfordert eine Produktlebenszyklus-Management (PLM)-Genehmigungsentscheidung für Änderungsverwaltungspakete, bevor diese vom Quellsystem bereitgestellt werden.

Unter "Verstehen von Standard-Arbeitsplänen zur Änderungsverwaltung" finden Sie Informationen zu den Standard-Arbeitsplänen.

Sie können beim Erstellen des Pakets Informationen zur Bereitstellung angeben oder später, wenn Sie das Paket aktualisieren. Sie weisen dem Paket später Modellierungs- und Prozessablaufinformationen vor der Bereitstellung zu. Sie müssen mindestens ein Zielsystem auswählen, bevor Sie ein Paket bereitstellen.

Die Anwendung verwendet die beim Erstellen des Pakets eingegebenen Informationen, um Kernattribute des Pakets auf nachfolgenden Änderungsverwaltungspaket-Seiten anzuzeigen. Die Anwendung zeigt die Kernattribute an, die während der Paketerstellung auf der Seite "Paket aktualisieren" eingegeben wurden. Zu den Kernattributen eines Pakets gehören Informationen wie der Paketname, Schritt und Eigentümer. Unter "Paketinformationen" finden Sie Informationen zu den Kernattributen und den Seiten, auf denen sie angezeigt werden.

Beim Verwenden einer Paketerstellungsvorlage

Sie können eine Paketerstellungsvorlage mithilfe des Modellierungsobjekts "Paketerstellungsvorlage" definieren. Bei der Auswahl einer Vorlage wird die Seite "Paket erstellen" mit den vordefinierten Paket- und Bereitstellungsdetails gefüllt. Die jeweiligen vordefinierten Genehmiger- und Zusammenarbeitsvorlagen werden beim Erstellen des Pakets auf der Seite "Paket aktualisieren" angezeigt. Wenn Sie ein Paket erstellen, ist es nicht erforderlich, jedes Mal alle Informationen manuell einzugeben.

Sie können die Informationen der Vorlage auf der Seite überschreiben. Wenn Sie eine andere Paketerstellungsvorlage auswählen, werden die Informationen auf der Seite überschrieben. Die Paketinformationen werden erst gespeichert, wenn Sie auf "Erstellen" oder auf "Erstellen und zu Aktualisierung gehen" klicken. Bei der Paketerstellung vorgenommene Änderungen werden nur auf das neu erstellte Paket angewendet. Die Anwendung speichert die am Paket vorgenommenen Aktualisierungen nicht in der Paketerstellungsvorlage.

Beim Hinzufügen von Bereitstellungsdetails

Bereitstellungsdetails sind optional, wenn Sie ein Paket erstellen, aber erforderlich, bevor Sie ein Paket bereitstellen können.

Die Anwendung zeigt eine Warnmeldung an, wenn Sie ein Zielsystem auswählen, dessen MDB-Version oder Anwendungsversion, die im zugehörigen Serverkatalogsystem definiert ist, sich von der Version im Serverkatalogsystem des Quellsystems unterscheidet. Die Warnmeldung enthält sowohl eine Warnung für die MDB- als auch für die Anwendungsversion, wenn beide auf den Ziel- und Quellsystemen unterschiedlich sind. Diese Warnmeldung wird nur für das nicht übereinstimmende Ziel angezeigt, das in der Tabelle am weitesten unten steht. Sie können die Warnmeldung schließen und das Paket erstellen. Die Anwendung hindert Sie nicht daran, ein Paket zu erstellen, wenn sich die MDB- oder Anwendungsversionen zwischen Quell- und Zielsystem unterscheiden.

Beim Hinzufügen von Aktivierungsdetails

Sie können vorausgesetzte Pakete angeben, die vor einer Aktivierung Ihres Pakets aktiviert werden müssen, damit Abhängigkeiten zu Instanzen in anderen Paketen berücksichtigt werden. Durch Klicken auf "Hinzufügen" in der Tabelle "Vorausgesetzte Änderungspakete" wird das Popup "Paketsuche" geöffnet. In dem Popup können Sie nach vorausgesetzten Paketen suchen und diejenigen dieser Pakete auswählen, die Ihrem Paket hinzugefügt werden sollen.

Beim Erstellen eines Pakets

Durch Klicken auf die Schaltfläche "Erstellen und zu Aktualisierung gehen" wird das Paket gespeichert und die Seite "Paket aktualisieren" angezeigt. Sie können die Kernattribute des Pakets ändern und Genehmiger zuweisen, wenn der zugewiesene Arbeitsplan Genehmigungen erfordert. Siehe "Aktualisieren eines Pakets" für weitere Informationen.

Hinweis: Sie müssen die entsprechende Rolle haben, um ein Paket zu aktualisieren.

Durch Klicken auf die Schaltfläche "Erstellen" wird das Paket gespeichert und die Felder auf der Seite "Paket erstellen" werden gelöscht. Sie können mit der Eingabe der Informationen für ein neues Paket beginnen.

Vorgehensweise zum Erstellen eines Pakets

Gehen Sie folgendermaßen vor, um ein Paket zu erstellen:

1. Wählen Sie im Menü Änderungsverwaltung die Option Paket erstellen aus. Die Seite "Paket erstellen" wird angezeigt.
2. Möchten Sie eine Paketerstellungsvorlage verwenden?

Antwort . . .

Dann . . .

Ja
a. Wählen Sie eine Paketerstellungsvorlage aus. Die Anwendung aktualisiert die Seite "Paket erstellen" und zeigt die mit der ausgewählten Vorlage verbundenen vordefinierten Informationen an.
b. Gehen Sie zu Schritt 3.

Nein

Gehen Sie zu Schritt 3.
3. Geben Sie einen Namen für das Änderungsverwaltungs-Paket im Feld Paketname ein.
4. Wählen Sie einen Paketeigentümer aus.
5. Wählen Sie einen Paketerstellungsgrund aus.
6. Wählen Sie einen Arbeitsplan zur Änderungsverwaltung für die Paketerstellung und -bereitstellung im Feld Arbeitsplan zur Änderungsverwaltung aus.
7. Geben Sie je nach den Anforderungen Ihres Betriebs optionale Informationen ein. In der Felddefinitionstabelle finden Sie Informationen zu den optionalen Feldern.
8. Führen Sie einen der folgenden Schritte durch:
• Klicken Sie auf Erstellen. Die Anwendung erstellt das Änderungsverwaltungspaket und zeigt eine Erfolgsmeldung an.
• Klicken Sie auf Erstellen und zu Aktualisierung gehen. Die Anwendung erstellt das Änderungsverwaltungspaket und zeigt die Seite "Paket aktualisieren" mit ausgefüllten Paketinformationen an.
