Aktualisieren eines Pakets

Sie können Kernattribute und Bereitstellungsinformationen für vorhandene Änderungsverwaltungspakete hinzufügen und ändern, die noch nicht bereitgestellt wurden. Sie können ausgewählten Paketen Genehmiger zuweisen, wenn die Pakete einem Arbeitsplan zur Änderungsverwaltung zugeordnet sind, der eine Paketgenehmigung erfordert. Sie können Projektmitarbeiter angeben, wenn Sie die Arbeit der Zuweisung von Inhalten zu einem Paket auf andere Mitarbeiter verteilen möchten. Sie können angeben, ob die Anwendung Kernattribute des Pakets, die auf den Seiten anderer Änderungsverwaltungspakete angezeigt werden, mit den Änderungen, die auf der Seite "Paket aktualisieren" vorgenommen wurden, aktualisiert.

Siehe "Erstellen eines Pakets" für Informationen zum Erstellen eines Pakets. Unter "Paketinformationen" finden Sie Informationen zu den Kernattributen und den Seiten, auf denen sie angezeigt werden.

Beim Aktualisieren eines Pakets

Sie können ein Paket nur aktualisieren, wenn das Paket an einer Spez. ist, die einer erlaubten Transaktion zugeordnet ist und die Berechtigung zum Aktualisieren eines Pakets beinhaltet. Z. B. beinhaltet die standardmäßige Rolle "DraftPermissions" die Berechtigung, ein Paket zu aktualisieren und ist der standardmäßigen Spez. "Entwurf" (Draft) zugeordnet. Mitarbeiter mit den entsprechenden Rollen können Pakete an der Spez. "Entwurf" (Draft) aktualisieren.

Die Seite "Paket aktualisieren" zeigt eine, zwei oder drei Registerkarten an, je nachdem, ob Sie einen Arbeitsplan zur Änderungsverwaltung verwenden, der Genehmigungen erfordert, und ob Sie angeben, dass Projektmitarbeiter einem Paket Inhalte zuweisen können:

• Allgemein

Die Registerkarte "Allgemein" bietet Ihnen die Möglichkeit, Kernattribute und Bereitstellungsinformationen für das ausgewählte Änderungsverwaltungspaket hinzuzufügen und zu ändern. Sie können ein Diagramm des Arbeitsplans zur Änderungsverwaltung anzeigen, der dem Paket zugeordnet ist, indem Sie auf das "Informationen"-Icon neben dem Feld "Änderungsmanagement-Arbeitsplan" klicken. Die Anwendung zeigt das Diagramm in einem Popup an. Sie können auch angeben, ob Sie möchten, dass Projektmitarbeiter einem Paket Inhalte zuweisen.

• Genehmiger

Die Registerkarte "Genehmiger" wird nur angezeigt, wenn Sie einen Arbeitsplan zur Änderungsverwaltung ausgewählt haben, der Genehmigungen erfordert. Diese Registerkarte erlaubt es Ihnen, eine Genehmigungsvorlage für das Änderungsverwaltungspaket anzugeben.

• Projektmitarbeiter

Die Registerkarte "Projektmitarbeiter" wird nur angezeigt, wenn Sie auf der Registerkarte "Allgemein" die Option "Eigentümer und Projektmitarbeiter" ausgewählt haben. Auf der Registerkarte "Projektmitarbeiter" können Sie eine Projektmitarbeitervorlage für das Änderungsverwaltungspaket angeben oder Sie können Projektmitarbeiter mit dem Popup "Projektmitarbeiter" hinzufügen. Sie können außerdem E-Mail-Benachrichtigungen über die Zuweisung von Inhalten an Projektmitarbeiter senden, und Sie können eine Benachrichtigung an den Paketeigentümer senden, wenn die Projektmitarbeiter die Zuweisung von Inhalten beendet haben.

Hinweis: Mit dem Modellierungsobjekt "Änderungsmanagementeinstellungen" kann ein Systemadministrator diverse globale Einstellungen für die Änderungsverwaltung konfigurieren, z. B. E-Mail-Benachrichtigungen, die an Genehmiger, Projektmitarbeiter und Paketeigentümer gesendet werden, wenn bestimmte Ereignisse für ein Änderungsverwaltungspaket eintreten. Weitere Informationen erhalten Sie unter "Beim Hinzufügen von Paketgenehmigern" und "Beim Hinzufügen von Projektmitarbeitern".

Beim Hinzufügen von Bereitstellungsdetails

Die Anwendung erfordert, dass Sie Bereitstellungsdetails eingeben, bevor Sie ein Paket bereitstellen können. Beim Erstellen oder Aktualisieren des Pakets ausgewählte Zielsysteme werden auf der Seite "Paket bereitstellen" angezeigt.

Die Anwendung zeigt eine Warnmeldung an, wenn Sie ein Zielsystem auswählen, für das ein Wert für das Feld "MDB-Version" oder "Camstar-Version" konfiguriert wurde, der sich vom Wert unterscheidet, der für Ihr Quellsystem konfiguriert wurde. Weitere Informationen zum Definieren von Serverkatalogsystemen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Die Warnmeldung enthält eine Warnung für die Felder "MDB-Version" und "Camstar-Version", wenn sich die beiden Werte auf den Ziel- und Quellsystemen unterscheiden. Diese Warnmeldung wird nur für das nicht übereinstimmende Ziel angezeigt, das in der Tabelle am weitesten unten steht. Sie können die Warnmeldung schließen und das Paket speichern. Die Anwendung hindert Sie nicht daran, ein Paket zu erstellen, wenn eine Fehlermeldung aufgrund nicht übereinstimmender Werte angezeigt wird.

Beim Hinzufügen von Aktivierungsdetails

Sie können vorausgesetzte Pakete angeben, die vor einer Aktivierung Ihres Pakets aktiviert werden müssen, damit Abhängigkeiten zu Instanzen in anderen Paketen berücksichtigt werden. Durch Klicken auf "Hinzufügen" in der Tabelle "Vorausgesetzte Änderungspakete" wird das Popup "Paketsuche" geöffnet. In dem Popup können Sie nach vorausgesetzten Paketen suchen und diejenigen dieser Pakete auswählen, die Ihrem Paket hinzugefügt werden sollen.

Beim Aktivieren der Inhaltszusammenarbeit

Die Standardeinstellung für die Steuerung "Inhaltsprojektmitarbeiter" auf der Registerkarte "Allgemein" ist "Nur Eigentümer". Diese Einstellung gibt an, dass nur der Benutzer mit Eigentümerberechtigungen dem Paket Inhalte zuweisen kann. Sie können die Inhaltszusammenarbeit durch Auswahl der Option "Eigentümer und Projektmitarbeiter" für die Steuerung "Inhaltsprojektmitarbeiter" aktivieren. Die Registerkarte "Projektmitarbeiter" wird angezeigt, wenn Sie die Option "Eigentümer und Projektmitarbeiter" auswählen. Für weitere Informationen über die Registerkarte "Projektmitarbeiter" siehe "Beim Hinzufügen von Projektmitarbeitern". Einen Überblick über den Prozess der Zusammenarbeit erhalten Sie unter "Verstehen von Inhaltszusammenarbeit".

Wichtig: Die Anwendung prüft, ob Projektmitarbeiter auf der Registerkarte "Projektmitarbeiter" angegeben sind, wenn Sie die Einstellung für die "Inhaltsprojektmitarbeiter"-Steuerung von "Eigentümer und Projektmitarbeiter" in "Nur Eigentümer" ändern. Die Anwendung zeigt die Registerkarte "Projektmitarbeiter" nicht mehr an, wenn keine Projektmitarbeiter angegeben sind. Wenn Projektmitarbeiter angegeben sind, wird eine Warnung angezeigt, die Sie darauf hinweist, dass alle Projektmitarbeiter und Projektmitarbeiterinformationen auf der Registerkarte "Projektmitarbeiter" gelöscht werden. Wenn Sie fortfahren, löscht die Anwendung die Informationen und zeigt die Registerkarte "Projektmitarbeiter" nicht mehr an. Die Auswahl "Eigentümer und Projektmitarbeiter" wird beibehalten, wenn Sie nicht fortfahren.

Beim Hinzufügen von Paketgenehmigern

Genehmigungsvorlagen legen die Mitarbeiter fest, die das Änderungsverwaltungspaket genehmigen müssen, ebenso wie die Entscheidungsoptionen, die sie auswählen können, wenn sie ein Paket genehmigen oder ablehnen. Sie müssen eine Genehmigungsvorlage angeben, um dem Änderungsverwaltungspaket eine Genehmigungsentscheidungsliste zuzuordnen. Die Anwendung erfordert, dass dem Änderungsverwaltungspaket eine Genehmigungsentscheidungsliste zugeordnet ist, bevor Sie das Paket zur Genehmigung weiterleiten können.

Hinweis: Die Anwendung speichert die Genehmigungsentscheidungsliste und alle Genehmiger, die der angegebenen Genehmigungsvorlage zugeordnet sind, aber nicht die Genehmigungsvorlage selbst.

Die Anwendung füllt die Tabelle "Genehmiger" und das Feld "Anweisungen für alle Genehmiger", wenn Sie eine Genehmigungsvorlage auswählen. Sie können Genehmiger hinzufügen, Genehmiger löschen, Informationen für jeden Genehmiger ändern sowie die Anweisungen für alle Genehmiger ändern.

Hinweis: Ein Systemadministrator kann E-Mail-Benachrichtigungen für Paketgenehmiger konfigurieren, damit diese benachrichtigt werden, wenn ein Änderungsverwaltungspaket ihre Genehmigung erfordert. Außerdem können Erinnerungs- und Delegierungsbenachrichtigungen konfiguriert werden. Benachrichtigungen zur Änderungsverwaltung werden im Modellierungsobjekt "Änderungsmanagementeinstellungen" konfiguriert. Weitere Informationen zum Konfigurieren von E-Mail-Benachrichtigungen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Durch Auswählen einer anderen Genehmigungsvorlage wird eine neue Genehmigungsentscheidungsliste zugewiesen und die Tabelle "Genehmiger" überschrieben, einschließlich aller Aktualisierungen der Genehmigerliste. Die ursprüngliche Genehmigungsentscheidungsliste und Genehmigerliste gehen nicht verloren, bis Sie auf "Aktualisieren" klicken.

Die Anwendung verwendet die Spalte "Name" in der Tabelle "Genehmiger", um die Verteilerliste für Benachrichtigungen zu erstellen, die an Mitarbeiter (Genehmiger) gesendet werden, wenn Pakete zur Genehmigung weitergeleitet werden. Allen Mitarbeitern, die als Genehmiger angegeben werden, müssen E-Mail-Adressen hinzugefügt werden.

Genehmigungsvorlagen, Genehmigungsentscheidungslisten und Mitarbeiter werden in der Modellierung definiert. Weitere Informationen zum Definieren von Genehmigungsvorlagen und Genehmigungsentscheidungslisten finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Hinweis: Sie können eine Genehmigungsvorlage definieren, die nur eine Genehmigungsentscheidungsliste enthält, wenn Sie nicht möchten, dass die Tabelle "Genehmiger" automatisch gefüllt wird. Die Definition einer leeren Genehmigungsvorlage ermöglicht es Ihnen problemlos die Genehmiger anzupassen, die jedem Paket zugewiesen werden.

Beim Hinzufügen von Projektmitarbeitern

Sie können eine Projektmitarbeitervorlage mithilfe des Modellierungsobjekts "Projektmitarbeitervorlage" definieren. Mit der Projektmitarbeitervorlage können Sie Projektmitarbeiter und ihre Anweisungen vordefinieren. Die Anwendung füllt die Tabelle "Projektmitarbeiter" (einschließlich Namen und besondere Anweisungen für jeden Projektmitarbeiter) und das Feld "Anweisungen für alle Projektmitarbeiter", wenn Sie eine Projektmitarbeitervorlage auswählen.

Sie können Projektmitarbeiter manuell zur Tabelle hinzufügen, wenn Sie keine Vorlage auswählen. Unabhängig davon, ob Sie eine Vorlage auswählen oder nicht, können Sie Projektmitarbeiter hinzufügen oder löschen, Informationen für jeden Projektmitarbeiter ändern und die Anweisungen für alle Projektmitarbeiter ändern. Die Anwendung speichert keine Aktualisierungen, die Sie an den Projektmitarbeiter oder Anweisungen in der Tabelle vornehmen, in der Projektmitarbeitervorlage.

Wenn Sie eine andere Projektmitarbeitervorlage auswählen, werden die Informationen in der Tabelle "Projektmitarbeiter" überschrieben. Die Informationen werden erst gelöscht, wenn Sie auf "Aktualisieren" klicken.

Mit "Nachricht an Projektmitarbeiter" können Sie Projektmitarbeiter darüber benachrichtigen, dass diese dem Paket Inhalte zuweisen können. Die Anwendung sendet die Nachricht, wenn Sie "Projektmitarbeiter über Zuweisungen bei Aktualisierung benachrichtigen" auswählen und auf "Aktualisieren" klicken. Wenn Sie "Projektmitarbeiter über Zuweisungen bei Aktualisierung benachrichtigen" aktivieren und keine "Nachricht an Projektmitarbeiter" angegeben haben, wird eine Warnmeldung angezeigt.

"Projektmitarbeiter über Zuweisungen bei Aktualisierung benachrichtigen" ist standardmäßig nicht aktiviert. Wenn Sie dieses Kontrollkästchen aktivieren, bleibt es aktiviert, nachdem Sie auf "Aktualisieren" klicken. Das Feld wird standardmäßig deaktiviert, wenn Sie die Seite "Paket aktualisieren" schließen und erneut öffnen.

Mit der Option "Beendigungsnachricht an Eigentümer" können Sie den Paketeigentümer benachrichtigen, wenn die Projektmitarbeiter die Zuweisung von Inhalten zum Paket fertiggestellt haben. Jeder Projektmitarbeiter klickt auf der Seite "Inhalt zuweisen und überprüfen" auf "Beendet". Das Popup "Inhaltsprojektmitarbeiter – Zuweisung abgeschlossen" wird angezeigt. Der Projektmitarbeiter kann im Popup Kommentare hinzufügen und auf die Schaltfläche "Beendet" klicken. Die Anwendung sendet die Nachricht.

Hinweis: Ein Systemadministrator kann E-Mail-Benachrichtigungen für Paketprojektmitarbeiter konfigurieren, um diese zu benachrichtigen, dass sie einem Paket Inhalte zuweisen müssen. Außerdem können E-Mail-Benachrichtigungen konfiguriert werden, mit denen der Paketeigentümer darüber benachrichtigt wird, dass die Projektmitarbeiter die Zuweisung von Inhalten zum Paket fertiggestellt haben. Ein Systemadministrator kann auch Erinnerungs- und Delegierungsbenachrichtigungen konfigurieren. Benachrichtigungen zur Änderungsverwaltung werden im Modellierungsobjekt "Änderungsmanagementeinstellungen" konfiguriert. Weitere Informationen zum Konfigurieren von E-Mail-Benachrichtigungen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Die Anwendung füllt die Listen "Nachricht an Projektmitarbeiter" und "Beendigungsnachricht an Eigentümer" mit den E-Mail-Nachrichten, die mit dem Modellierungsobjekt "E-Mail-Nachricht" definiert wurden. Die Liste "Nachricht an Projektmitarbeiter" zeigt standardmäßig die Nachricht mit dem Benachrichtigungstyp "Änderungsmanagement: Zusammenarbeitszuweisung" im Modellierungsobjekt "Änderungsmanagementeinstellungen" an. Die Liste "Beendigungsnachricht an Eigentümer" zeigt standardmäßig die Nachricht mit dem Benachrichtigungstyp "Änderungsmanagement: Eigentümer: Zusammenarbeitszuweisung beendet" im Modellierungsobjekt "Änderungsmanagementeinstellungen" an. Die Anwendung wählt keinen Standardwert für diese Listen aus, wenn Sie die Benachrichtigungstypen nicht im Objekt "Änderungsmanagementeinstellungen" festgelegt haben.

Auf der Registerkarte "Projektmitarbeiteraufgaben" der Seite "Inhalt zuweisen und überprüfen" legt die Anwendung das Fälligkeitsdatum auf "Zuweisungsdatum" und den Status auf "Spät" fest, wenn Sie beim Hinzufügen von Projektmitarbeitern keine Angabe zu Dauer und Zeitraum machen.

Projektmitarbeitervorlagen, Mitarbeiter und die Einstellungen für die Änderungsverwaltung werden in der Modellierung definiert. Weitere Informationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Genehmigungsvorlagen, die Sie dem Änderungsverwaltungspaket zuweisen können. In Genehmigungsvorlagen sind die Benutzer angegeben, von denen die Änderungsverwaltungspakete genehmigt werden müssen. Außerdem enthält die Genehmigungsvorlage eine Liste möglicher Entscheidungen (die zugehörige Genehmigungsentscheidungsliste) für diese Benutzer.

Hinweis: Genehmigungsvorlagen werden in der Modellierung definiert.

Optional

Tabelle "Genehmiger"

Tabelle, die alle Benutzer aufführt, die als Genehmiger für das Änderungsverwaltungspaket angegeben sind.

Erforderlich

Level

Die Genehmigungs-Level, die dem Genehmiger zugewiesen wurden. Sie können nur numerische Werte eingeben.

Das Level definiert die Genehmigungs-Weiterleitungsreihenfolge. Die Anwendung kann die Genehmigung der Reihe nach, parallel oder in einer Kombination aus beiden an Genehmiger weiterleiten.

Erforderlich

Rolle

Die für die Genehmigung zuständige Rolle.

Erforderlich

Name

Die für die Genehmigung zuständige Person. Die verfügbaren Namen hängen von Ihrer Auswahl im Rollenfeld ab, wenn Sie den Namen ändern.

Hinweis: Die Anwendung verlangt von Ihnen, dass Sie einen Genehmiger festlegen, bevor Sie das Paket zur Genehmigung weiterleiten.

Erforderlich

Änderungsoption

Ein Wert, der die Art der Änderung angibt, die ein Benutzer an einer Genehmiger-Zeile während des Genehmigungsprozesses vornehmen kann.

• Rolle und Name (die Standardwerte) zeigen an, dass der Benutzer mit Eigentümerrechten die Genehmigerrolle und den Genehmigernamen ändern kann.
• Name zeigt an, dass ein Benutzer mit Eigentümerrechten nur den Genehmigernamen ändern kann.
• Kein zeigt an, dass ein Benutzer mit Eigentümerrechten weder die Genehmigerrolle noch den Genehmigernamen ändern darf. Wenn Sie "Kein" auswählen, stellt die Anwendung das Feld "Ersetzungsoption" auf "Kein" ein und das Feld wird schreibgeschützt.

Nur Anzeige

Ersetzungsoption

Ein Wert der angibt, ob ein Benutzer die Genehmigung durchführen darf, der nicht in der Spalte "Name" aufgelistet ist:

• Rolle gibt an, dass jeder Benutzer mit derselben Rolle und mit demselben Level in der Tabelle "Genehmiger" die Genehmigung durchführen darf.
• Kein gibt an, dass nur der angegebene Benutzer die Genehmigung durchführen darf.

Hinweis: Wenn Sie "Rolle" auswählen, erstellt die Anwendung E-Mail-Benachrichtigungen (falls konfiguriert) für alle Benutzer mit derselben Rolle und demselben Level wie der Eigentümer, wenn die Genehmigung weitergeleitet wird.

Nur Anzeige

Eingabe erforderlich

Wenn dies ausgewählt ist, bedeutet das, dass eine Genehmigung auf dieser Ebene notwendig ist. Der Eigentümer des Eintrags kann die Zeile für den Genehmiger ändern, aber nicht das Level des Genehmigers. Darüber hinaus kann der Eigentümer diese Zeile nicht löschen, wenn das Kontrollkästchen aktiviert ist.

Nur Anzeige

Dauer

Eine Zahl, die die Gesamtzeit angibt, die die Genehmigung in Anspruch nehmen darf.

Nur Anzeige

Zeitraum

Maßeinheit für den Zeitraum (Tage, Stunden, Wochen), in dem die Genehmigung durchgeführt werden muss.

Nur Anzeige

Anweisung

Bestimmte Anweisungen von einem Eigentümer oder einem Administrator an den Genehmiger.

Nur Anzeige

Anweisungen für alle Genehmiger

Anweisungen für alle Genehmiger. Die Anwendung zeigt die Anweisungen für die ausgewählte Genehmigungsvorlage an, falls Anweisungen eingegeben wurden, als die Genehmigungsvorlage definiert wurde. Sie können die bereitgestellten Anweisungen ändern oder Ihre eigenen hinzufügen (bis zu 255 Zeichen).

Felddefinitionen des Popups "Projektmitarbeiter"

Diese Tabelle definiert die Felder im Popup "Projektmitarbeiter".

Feld

Definition

Typ

Level

Das Zusammenarbeitslevel, das dem Genehmiger zugewiesen ist. Sie können nur numerische Werte eingeben. Die Tabelle wird nach dieser Spalte in aufsteigender Reihenfolge sortiert.

Der Level definiert die Reihenfolge, in der die Aufgaben erledigt werden müssen. Die Anwendung sendet E-Mail-Benachrichtigungen, nachdem die einzelnen Aufgaben erledigt wurden, wenn die Zusammenarbeitsaufgaben sequenziell eingerichtet wurden. Die Anwendung sendet E-Mail-Benachrichtigungen, nachdem der letzte Projektmitarbeiter die zugewiesene Aufgabe erledigt hat, wenn die Zusammenarbeitsaufgaben parallel eingerichtet wurden.

Seriell bedeutet, dass Projektmitarbeiter mit einer niedrigeren Level-Nummer die Zuweisung von Inhalten erst beenden müssen, bevor Projektmitarbeiter mit höheren Level-Nummern beginnen können. Parallel bedeutet, dass Projektmitarbeiter derselben Level-Nummer gleichzeitig Inhalte zuweisen können.

Weitere Informationen erhalten Sie unter "Senden von E-Mail-Benachrichtigungen für die Zusammenarbeit".

Rolle, der die Verantwortung für die Zusammenarbeit zugewiesen ist. Wenn diese Option gewählt ist, wird die Liste "Name" gefüllt.

Erforderlich

Ersetzungsoption
