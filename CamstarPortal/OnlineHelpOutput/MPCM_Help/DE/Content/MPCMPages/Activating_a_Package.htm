Aktivieren eines Pakets

Mit der Transaktion "Paket aktivieren" können Sie ein bereitgestelltes Änderungsverwaltungspaket auf einem Zielsystem anwenden.

Beim Aktivieren eines Pakets

Der Aktivierungsprozess kann bestehende Instanzen auf dem Zielsystem überschreiben, wenn das Änderungsverwaltungspaket Instanzen enthält, die denselben Namen haben wie vorhandene Instanzen. Die Einstellungen zur Überschreibung auf der Seite "Paket aktivieren" ermöglichen Ihnen anzugeben, ob und wie Modellierungsobjekt-Instanzen auf dem Zielsystem von Modellierungsobjekt-Instanzen im Änderungsverwaltungspaket überschrieben werden.

Der Paketaktivierungsprozess stellt drei Überschreibungsmöglichkeiten zur Verfügung:

• Prozess abbrechen und zurückrollen: Der Aktivierungsprozess wird abgebrochen und alle durchgeführten und ausstehenden Änderungen der Paketaktivierung werden zurückgerollt, wenn der Prozess eine Modellierungsobjekt-Instanz im Zielsystem mit demselben Namen wie eine Modellierungsobjekt-Instanz im Paket festgestellt hat oder ein Serverfehler auftritt.
• Instanz überschreiben: Der Aktivierungsprozess überschreibt sämtliche Modellierungsobjekt-Instanzen auf dem Zielsystem, die denselben Namen haben wie Instanzen im Paket.
• Instanz überspringen: Der Aktivierungsprozess lädt die Modellierungsobjekt-Instanzen im Paket auf das Zielsystem, aber überspringt das Laden von Instanzen mit demselben Namen wie eine bereits vorhandene Instanz auf dem Zielsystem.
Hinweis: Die Anwendung verwendet standardmäßig die Methode "Instanz überspringen".

Die Paketaktivierung beginnt, wenn Sie auf die Schaltfläche "Paket aktivieren" klicken. Die Anwendung zeigt den Status der Aktivierung auf dieser Seite an, während sie die ausgewählten Modellierungsobjekt-Instanzen aus dem Paket zur Änderungsverwaltung lädt. Statusmeldungen werden in der Aktivierungskonsole angezeigt. Eine Statusmeldung wird angezeigt, wenn Sie versuchen, Pakete mit mehr als 2.000 Instanzen zu aktivieren.

Die standardmäßige Timeout-Einstellung für die Ausführung der Anwendung kann sich auf den Bereitstellungsprozess auswirken. Der Standardwert ist 110 Sekunden. Der Wert "executionTimeout" befindet sich in der Datei "web.config" unter der Einstellung "HttpRuntime" des Knotens "system.web". Die Datei "Web.config" befindet sich standardmäßig in "C:\Programme (x86)\Camstar\CamstarPortal".

Die Genehmigung von mehreren Paketen mit demselben Namen, aber unterschiedlicher Groß-/Kleinschreibung führt zu einem Fehler in Umgebungen, die nicht zwischen Groß- und Kleinschreibung unterscheiden, z. B. in Oracle. Sie können z. B. nicht das Paket "ABC" aktivieren und dann versuchen, das Paket "abc" zu aktivieren.

Fehler bei der Aktivierung

Die Anwendung zeigt Fehlermeldungen in der Aktivierungskonsole an, bricht die Paketaktivierung ab und rollt Änderungen zurück, wenn Folgendes auftritt:

• Ein Serverfehler.
• Die Anwendung erkennt eine Modellierungsobjekt-Instanz auf dem Zielsystem mit demselben Namen wie eine Instanz im Paket zur Änderungsverwaltung und der angegebene Überschreibmodus ist "Prozess abbrechen und zurückrollen".

Die Anwendung entfernt das Paket aus dem Zielsystem, wenn sie einen Serverfehler feststellt oder doppelte Modellierungsobjekt-Instanzen erkennt und Sie den Überschreibungsmodus "Prozess abbrechen und zurückrollen" ausgewählt haben. Die Anwendung fordert Sie auf, das Paket erneut bereitzustellen, um es im Zielsystem zu aktivieren. Sie haben folgende Möglichkeiten:

• Das Paket erneut bereitstellen und einen anderen Überschreibungsmodus auswählen

Oder

• Die doppelten Modellierungsobjekt-Instanzen entfernen, das Paket erneut bereitstellen und den Überschreibungsmodus "Prozess abbrechen und zurückrollen" auswählen

Aktivieren eines Pakets mit vorausgesetzten Paketen

Die Anwendung zeigt das Popup "Bestätigung außer Kraft setzen" an, wenn Sie versuchen, ein Paket zu aktivieren, für das zunächst andere Pakete aktiviert werden müssen. Das Popup zeigt die vorausgesetzten Pakete an, die dem Paket zugewiesen sind, das Sie aktivieren möchten, und fragt, ob die vorausgesetzten Pakete außer Kraft gesetzt werden sollen. Sie können die entsprechenden Kommentare in das Feld "Kommentare" eingeben, wobei jeder Kommentar maximal 255 Zeichen umfassen darf. Wenn Sie auf "Ja" klicken, werden alle Voraussetzungen außer Kraft gesetzt.

Hinweis: Die Anwendung zeigt die Namen der ersten drei vorausgesetzten Pakete und, wenn es mehr als drei davon gibt, danach drei Punkte an.

Wenn Sie auf "Ja" klicken, führt die Anwendung folgende Aktionen durch:

• Sie schließt das Popup.
• Sie setzt alle vorausgesetzten Pakete außer Kraft, die aufgelistet sind.
• Sie zeichnet den Zeitstempel, den Benutzernamen und die Außerkraftsetzungskommentare in der Historie auf.
• Sie leitet Sie zu der Seite "Paket aktivieren".
Hinweis: Abhängig davon, wie Pakete und Voraussetzungen konfiguriert sind, kann eine Fehlermeldung angezeigt werden, wenn vorausgesetzte Pakete außer Kraft gesetzt werden sollen.

Beim Zugreifen auf die Seite "Paket aktivieren"

Die Anwendung füllt das Feld "Ausgewähltes Paket" und zeigt alle Felder und Schaltflächen auf der Seite "Paket aktivieren" automatisch an, wenn Sie über die Seite "Aktivierungssuche" auf die Seite zugreifen.

Vorgehensweise zum Aktivieren eines Pakets

Gehen Sie folgendermaßen vor, um ein Paket zu aktivieren:

1. Melden Sie sich im Camstar Portal an (auf dem Zielsystem).
2. Wählen Sie Paketsuche im Menü Änderungsverwaltung aus. Die Seite "Paketsuche" wird angezeigt.
3. Geben Sie ein oder mehrere Kriterien ein, um Ihre Suche einzugrenzen.
4. Klicken Sie auf Suchen. Die Anwendung zeigt die Pakete in der Tabelle "Suchergebnisse" an, die Ihren Suchkriterien entsprechen.
5. Wählen Sie das Paket aus, das Sie aktivieren möchten. Die Anwendung zeigt die relevanten Transaktionen für das Paket an.
6. Klicken Sie auf Paket aktivieren. Die Seite "Paket aktivieren" wird angezeigt.
7. Wählen Sie falls nötig einen anderen Überschreibmodus im Feld Überschreibmodus, wenn Instanz existiert aus.
8. Klicken Sie auf Paket aktivieren, um den Aktivierungsprozess zu starten. Der Aktivierungsprozess beginnt und die Anwendung zeigt den Status in der Aktivierungskonsole an.
