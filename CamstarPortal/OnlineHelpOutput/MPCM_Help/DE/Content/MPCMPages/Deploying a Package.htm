Bereitstellen eines Pakets

Siemens bietet Ihnen die Möglichkeit, Änderungsverwaltungspakete, die Kernattribute und Instanz-Daten enthalten, für vordefinierte Zielsysteme bereitzustellen.

Vor dem Bereitstellen eines Pakets

Damit ein Paket zur Bereitstellung zugelassen ist, muss es sich in einem Schritt befinden, der "Bereitstellen" als erlaubte Transaktion enthält.

Bereitstellungsdetails werden entweder bei der Paketerstellung oder beim Aktualisieren eines Pakets vor der Bereitstellung definiert. Siehe "Erstellen eines Pakets" und "Aktualisieren eines Pakets" für weitere Informationen.

Beim Bereitstellen eines Pakets

Der Bereitstellungsprozess besteht aus zwei asynchronen Schritten:

• Zusammenstellen des Paketdokuments
• Die Auslieferung des Paketes an jedes Ziel

Die Anwendung beginnt mit der Zusammenstellung des Pakets, wenn Sie auf der Seite "Paket bereitstellen" in der Registerkarte "Paket bereitstellen" auf "Bereitstellen" klicken.

Die Seite "Paket bereitstellen" enthält die Registerkarte "Paket bereitstellen" und die Registerkarte "Bereitstellungsstatus". Die Registerkarte "Paket bereitstellen" erhält standardmäßig den Fokus.

Alle verfügbaren Zielsysteme, die dem ausgewählten Paket zugeordnet sind, werden standardmäßig im Feld "Ziele zum Bereitstellen auswählen" ausgewählt. Sie können angeben, für welche Zielsysteme das Paket bereitgestellt werden soll, indem Sie die Kontrollkästchen von unerwünschten Zielsystemen deaktivieren. Sie müssen mindestens ein Zielsystem auswählen, wenn Sie ein Paket bereitstellen. Es wird eine Fehlermeldung angezeigt, wenn Sie versuchen, ein Paket ohne ausgewählte Zielsysteme bereitzustellen.

Zielsysteme werden einem Paket während der Paketerstellung oder beim Aktualisieren hinzugefügt. Auf der Seite "Paket aktualisieren" können Sie die Bereitstellungsdetails eines Pakets aktualisieren, um zusätzliche Ziele hinzuzufügen. Siehe "Aktualisieren eines Pakets" für weitere Informationen.

Durch Klicken auf die Schaltfläche "Bereitstellen" der Registerkarte "Paket bereitstellen" wird die Registerkarte "Bereitstellungsstatus" angezeigt. Durch einen Klick auf die Schaltfläche "Zurücksetzen" werden alle verfügbaren Zielsysteme im Feld "Ziele zum Bereitstellen auswählen" ausgewählt und die Kommentare im Feld "Bereitstellungskommentar" werden entfernt.

Die Registerkarte "Bereitstellungsstatus" ermöglicht es Ihnen, den Fortschritt eines Paket-Builds zu überwachen und den Bereitstellungsstatus eines Pakets für jedes ausgewählte Zielsystem zu erfassen. Die Anwendung zeigt die Registerkarte "Bereitstellungsstatus" automatisch an, wenn Sie auf die Schaltfläche "Bereitstellen" auf der Seite "Paket bereitstellen" klicken.

Die Registerkarte "Bereitstellungsstatus" enthält die Tabellen "Paket-Buildstatus" und "Bereitstellungsstatus pro Ziel". Die Tabelle "Paket-Buildstatus" zeigt nur Daten für den aktuellen Paket-Build oder den zuletzt versuchten Paket-Build an, der dem ausgewählten Paket zugeordnet ist. Die Tabelle "Bereitstellungsstatus pro Ziel" erfasst Daten für die aktuelle Bereitstellung und verwaltet außerdem einen Verlauf früherer Bereitstellungen, die mit dem ausgewählten Paket in Zusammenhang stehen.

Sowohl die Tabelle "Paket-Buildstatus" als auch die Tabelle "Bereitstellungsstatus pro Ziel" werden in Echtzeit aktualisiert. Sie können ebenfalls durch einen Klick auf die Schaltfläche "Aktualisieren" die Tabellen "Paket-Buildstatus" und "Bereitstellungsstatus pro Ziel" aktualisieren.

Die Anwendung zeigt einen Eintrag für jedes Ziel in der Tabelle "Bereitstellungsstatus pro Ziel" an. Für jedes Paket werden zwei Dateien auf dem Zielsystem bereitgestellt. Die Anwendung zeigt solange als Bereitstellungsstatus "In Bearbeitung" an, bis beide Dateien transferiert wurden. Der Bereitstellungsstatus ändert sich zu "Beendet", wenn beide Dateien erfolgreich bearbeitet wurden. Wenn eine Datei fehlschlägt, wird der Vorgang angehalten und der Bereitstellungsstatus ändert sich zu "Fehlgeschlagen". Die Anwendung füllt "Bereitstellungsfehler" mit dem Fehler.

Im Feld "Paketgröße" wird die Gesamtgröße des Änderungspakets in Megabyte (MB) angezeigt. Die Paketgröße ist standardmäßig auf 30 MB begrenzt. Weitere Informationen dazu, wie Sie die maximale Größenbegrenzung für die Zuordnung in den Portal Studio-Einstellungen erhöhen können, finden Sie im Opcenter Execution Medical Device and Diagnostics Portal Studio-Benutzerhandbuch oder im Opcenter Execution Core Portal Studio-Benutzerhandbuch.

Die standardmäßige Timeout-Einstellung für die Ausführung der Anwendung kann sich auf den Bereitstellungsprozess auswirken. Der Standardwert ist 110 Sekunden. Der Wert "executionTimeout" befindet sich in der Datei "web.config" unter der Einstellung "HttpRuntime" des Knotens "system.web". Die Datei "Web.config" befindet sich standardmäßig in "C:\Programme (x86)\Camstar\CamstarPortal".

Eine Warnung wird angezeigt, wenn Sie versuchen, Pakete mit mehr als 1000 Instanzen bereitzustellen.

Vorgehensweise zum Bereitstellen eines Pakets

Gehen Sie folgendermaßen vor, um ein Paket bereitzustellen:

1. Wählen Sie Paketsuche im Menü Änderungsverwaltung aus. Die Seite "Paketsuche" wird angezeigt.
2. Geben Sie ein oder mehrere Kriterien ein, um Ihre Suche einzugrenzen.
3. Klicken Sie auf Suchen. Die Anwendung zeigt die Pakete in der Tabelle "Suchergebnisse" an, die Ihren Suchkriterien entsprechen.
4. Wählen Sie das Paket aus, das Sie bereitstellen möchten. Die Anwendung zeigt die relevanten Transaktionen für das Paket an.
5. Klicken Sie auf Paket bereitstellen. Die Seite "Paket bereitstellen" wird angezeigt.
6. Geben Sie je nach den Anforderungen Ihres Betriebs optionale Informationen ein. In der Felddefinitionstabelle finden Sie Informationen zu den optionalen Feldern.
7. Klicken Sie auf Paket bereitstellen. Die Anwendung zeigt die Registerkarte "Bereitstellungsstatus" an und erstellt die XML-Bereitstellungspaketdateien und aktualisiert im Anschluss die Tabelle "Paket-Buildstatus". Die Anwendung bereitet das Paket dann für die Bereitstellung auf den angegebenen Zielsystemen vor und aktualisiert die Tabelle "Bereitstellungsstatus pro Ziel" in Echtzeit.
