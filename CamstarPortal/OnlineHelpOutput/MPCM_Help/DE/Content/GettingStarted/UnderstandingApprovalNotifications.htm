Verstehen von Genehmigungsbenachrichtigungen

Die Anwendung kann E-Mails an Mitarbeiter senden, um sie zu benachrichtigen, dass ein Änderungsverwaltungspaket zur Genehmigung bereitsteht. Genehmigungsbenachrichtigungen sind nur für Pakete zur Änderungsverwaltung auf Arbeitsplänen gültig, die Änderungsverwaltungs-Spez. enthalten, welche eine Genehmigung erfordern.

Der Paketeigentümer weist dem Änderungsverwaltungspaket auf der Seite "Paket aktualisieren" Genehmiger zu. Unter "Aktualisieren eines Pakets" finden Sie Informationen zum Zuweisen von Genehmigern an ein Paket.

Die Anwendung verwendet die Mitarbeiter, die in der Spalte "Name" der Tabelle "Genehmiger" aufgelistet sind, als E-Mail-Verteilerliste. Mitarbeiter, die als Paketgenehmiger angegeben sind, erhalten E-Mails, wenn der Paketbesitzer das Paket zur Genehmigung weiterleitet.

Konfigurieren von Genehmigungsbenachrichtigungen

Ein Systemadministrator muss einige Objekte in der Modellierung speziell für Änderungsverwaltungsgenehmigungen konfigurieren, bevor der Paketeigentümer das Paket zur Genehmigung weiterleitet. Anhand der Objekte, die konfiguriert werden müssen, kann die Anwendung E-Mails an bestimmte Mitarbeiter senden. Mit einem optionalen Objekt können Sie eine globale Einstellung für die E-Mail-Benachrichtigung verwenden.

Erforderliche Konfiguration

Ein Systemadministrator muss die folgenden Einrichtungsaufgaben in der Modellierung ausführen, um zu gewährleisten, dass Mitarbeiter Benachrichtigungen erhalten:

• Den SMTP-Transport definieren, der für E-Mail-Benachrichtigungen verwendet werden soll.
• Eine Fabrik-Modellierungsobjekt-Instanz definieren oder aktualisieren und den SMTP-Transport auswählen, den Sie für E-Mail-Benachrichtigungen definiert haben.
• Mitarbeiter-Modellierungsobjekt-Instanzen mit folgenden Informationen definieren oder aktualisieren:
• E-Mail-Adresse, unter der der Mitarbeiter die Genehmigungsbenachrichtigungen erhalten soll
• Fabrik, mit der der SMTP-Transport verbunden ist
• Mit dem Modellierungsobjekt "E-Mail-Nachricht" E-Mail-Nachrichten für Genehmiger definieren.

Weitere Informationen zum Definieren und Aktualisieren der Modellierungsobjekte "SMTP-Transport", "Mitarbeiter", "E-Mail-Nachricht" und "Fabrik" finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Optionale Konfiguration

Beim Einrichten von E-Mail-Benachrichtigungen kann ein Systemadministrator die folgenden Aufgaben ausführen:

• E-Mail-Benachrichtigungstypen mit dem Modellierungsobjekt "Änderungsmanagementeinstellungen" definieren. Die Tabelle "Genehmiger" auf der Seite "Paket aktualisieren" zeigt standardmäßig die mit dem Benachrichtigungstyp "Genehmiger der Änderungsverwaltung" verbundene E-Mail-Option im Modellierungsobjekt "Änderungsmanagementeinstellungen" an.
Hinweis: Ein Systemadministrator kann eine Paketgenehmigungsaufgabe eines Mitarbeiters einem anderen zuweisen, wenn z. B. der Mitarbeiter, dem die Aufgabe ursprünglich zugewiesen wurde, am Fälligkeitsdatum im Urlaub ist. Der Administrator kann den Benachrichtigungstyp "Änderungsmanagement: Genehmigungsdelegierung" für das Modellierungsobjekt "Änderungsmanagementeinstellungen" und die E-Mail-Nachricht auswählen, die versendet werden soll, wenn die Arbeit neu zugewiesen ist.
• Erinnerungsbenachrichtigungen zur Paketgenehmigung mit dem Modellierungsobjekt "Änderungsmanagementeinstellungen" definieren. Die Anwendung kann diese Erinnerungen an den Eigentümer, Genehmiger oder Empfänger für Genehmigereskalation senden, wenn ein Paketgenehmiger kommende, fällige oder überfällige Aufgaben für die Genehmigung hat.
• Eine Fabrik-Modellierungsobjekt-Instanz mit den globalen Einstellungen für die Änderungsverwaltung innerhalb des Objekts "Änderungsmanagementeinstellungen" definieren oder aktualisieren.

Weitere Informationen zum Definieren und Aktualisieren des Modellierungsobjekts "Änderungsmanagementeinstellungen" finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Senden von E-Mail-Benachrichtigungen über Genehmigungen

Der Genehmigungsprozess beginnt, wenn der Paketeigentümer ein Paket auswählt und auf der Seite "Paketsuche" auf "Route für Genehmigung" klickt. Die Anwendung sendet die E-Mail-Benachrichtigung über die Genehmigung an Mitarbeiter, für die in der Tabelle "Genehmiger" als Level die niedrigste Zahl angegeben ist. Die Schaltfläche "Genehmigen (Camstar)" wird für diese Genehmiger auf der Seite "Paketsuche" verfügbar. Die Anwendung sendet nur an einen Mitarbeiter eine E-Mail-Benachrichtigung, wenn nur ein Mitarbeiter zugeordnet ist.

Weitere Informationen erhalten Sie unter "Weiterleiten eines Pakets zur Genehmigung".

Die folgenden Beispiele beschreiben verschiedene Szenarien der Genehmigung.

Beispiel 1: Mehrere Genehmiger, jeder auf einem anderen Level

Folgendes geschieht:

• Die Anwendung sendet der Reihe nach Benachrichtigungen.
• Jeder Genehmiger mit einer niedrigeren Level-Nummer muss das Paket erst genehmigen, bevor Genehmiger mit höheren Level-Nummern beginnen können.
• Wenn der erste Genehmiger das Paket genehmigt, sendet die Anwendung eine E-Mail-Benachrichtigung an den Genehmiger mit der nächsthöheren angegebenen Level-Nummer.
• Dieser Prozess wird so lange wiederholt, bis alle zugeordneten Genehmiger das Paket genehmigt haben.
Hinweis: Die Anwendung versetzt das Paket in den Schritt "Abgelehnt", wenn ein Genehmiger das Paket ablehnt.

Beispiel 2: Mehrere Genehmiger auf demselben Level

Folgendes geschieht:

• Die Anwendung sendet die Benachrichtigungen parallel.
• Genehmiger können das Paket gleichzeitig genehmigen.

Beispiel 3: Mehrere Genehmiger auf mehreren Levels

Die Anwendung kann die Arbeit und Benachrichtigungen teils parallel, teils der Reihe nach senden. Wenn Sie z. B. zwei Genehmiger auf Level 1 festlegen, einen Genehmiger auf Level 2 und zwei Genehmiger auf Level 3, geschieht Folgendes:

• Die Anwendung sendet Genehmigungsbenachrichtigungen an beide Genehmiger auf Level 1 gleichzeitig.
• Die Anwendung sendet eine Genehmigungsbenachrichtigung an den Genehmiger auf Level 2, wenn beide Genehmiger auf Level 1 das Paket genehmigt haben.
• Die Anwendung sendet Benachrichtigungen an beide Genehmiger auf Level 3, wenn der Genehmiger auf Level 2 das Paket genehmigt hat.
