Verstehen von Änderungsverwaltungspaket-Berechtigungen

Die folgenden Berechtigungen bestimmen die Aktionen, die der angemeldete Benutzer an einem Änderungsverwaltungspaket ausführen kann:

• Seitenberechtigungen
• Transaktionsberechtigungen
• Berechtigungen für erlaubte Transaktionen
• Dem Paket zugewiesener Arbeitsplan zur Änderungsverwaltung

Berechtigungen für Seiten, Transaktionen und für erlaubte Transaktionen werden Rollen zugewiesen. Rollen werden Mitarbeitern und Änderungsverwaltungs-Spez. zugewiesen. Änderungsverwaltungs-Spez. werden Arbeitsplänen zur Änderungsverwaltung zugewiesen.

Weitere Informationen zum Definieren dieser Modellierungsobjekte finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

• Rollen
• Mitarbeiter
• Änderungsverwaltungs-Spez.
• Arbeitspläne zur Änderungsverwaltung

Weitere Informationen zur Sicherheitsadministration der Änderungsverwaltung finden Sie in der Opcenter Execution Medical Device and Diagnostics-Anleitung zur Systemadministration oder in der Opcenter Execution Core-Anleitung zur Systemadministration.

Seitenberechtigungen

Der angemeldete Benutzer muss zum Zugriff auf die Seite berechtigt sein, auf der die Transaktion ausgeführt werden soll. Die Seitenberechtigung muss einer der Rollen zugewiesen sein, die dem Benutzer zugewiesen ist. Z. B. muss der Benutzer, der verantwortlich für das Erstellen eines Änderungsverwaltungspakets ist, einer seiner Rollen die Berechtigung zum Zugriff auf die Seite "Paket erstellen" zugewiesen haben.

Siemens stellt eine "Standardseiten"-Rolle zur Verfügung, der standardmäßig die Portalseitenberechtigung "StartChangePkg_VP" zugewiesen ist. Sie können jeder Rolle die Seitenberechtigung zuweisen.

Transaktionsberechtigungen

Der angemeldete Benutzer muss die Berechtigung haben, auf die Seite zuzugreifen und die Transaktion auf der Seite auszuführen. Die Transaktionsberechtigung muss einer oder mehreren Rollen zugewiesen sein, die dem angemeldeten Benutzer zugewiesen sind. Z. B. muss der Benutzer, der verantwortlich für das Erstellen eines Änderungsverwaltungspakets ist, die Berechtigung zum Ausführen der Erstellen (Start)-Pakettransaktion auf der Seite "Paket erstellen" haben.

Der Standardrolle "Paketersteller" ist standardmäßig die Transaktionsberechtigung "Änderungspaket starten" zugewiesen ("Änderungspaket starten" mit den Zugriffsmodi "Ausführen" und "Erledigung überschreiben"). Sie können die Transaktionsberechtigung jeder Rolle zuweisen.

Berechtigungen für erlaubte Transaktionen

Der angemeldete Benutzer muss die Berechtigung haben, Transaktionen am Änderungsverwaltungspaket in dessen aktuellem Schritt auf dem zugewiesenen Arbeitsplan auszuführen. Die Rollen für erlaubte Transaktionen, die einer Änderungsverwaltungs-Spez. zugewiesen sind, und die Berechtigungen, die diesen Rollen zugewiesen sind, bestimmen die Aktionen, die der angemeldete Benutzer am Paket durchführen kann.

Hinweis: Rollen werden Änderungsverwaltungs-Spez. in der Tabelle "Erlaubte Transaktionen" auf der Modellierungsseite "Änderungsmanagement-Spez." zugewiesen.

Berechtigungen für erlaubte Transaktionen sind zusätzliche, zu den Berechtigungen auf Seitenebene und den Transaktionsberechtigungen auf Seitenebene des angemeldeten Benutzers gewährte Berechtigungen. Die für jede Änderungsverwaltungs-Spez. angegebenen erlaubten Transaktionen bestimmen die Seiten, auf denen der angemeldete Benutzer Transaktionen ausführen darf.

Ein Benutzer, dem z. B. die Standardrolle "Paketeigentümer" zugewiesen wurde, kann fast jede Aktion an einem Änderungsverwaltungspaket ausführen. Allerdings beschränkt die Standardrolle "DraftPermissions" den angemeldeten Benutzer auf die Durchführung folgender Änderungsverwaltungsaktionen, wenn sich ein Änderungsverwaltungspaket im standardmäßigen Entwurfsschritt befindet:

• Suchen nach einem Paket
• Aktualisieren eines Pakets
• Zuweisen von Instanzen und Abhängigkeiten zu einem Paket
• Ändern des Schritts eines Pakets
• Bereitstellen eines Pakets
• Anzeigen von Paketdetails

Die Rolle "DraftPermissions" ist standardmäßig der Standard-Spez. "Entwurf" zugewiesen.

Arbeitsplan-Berechtigungen

Der Arbeitsplan zur Änderungsverwaltung, der dem Änderungsverwaltungspaket zugewiesen wurde, bestimmt den Geschäftsprozess für das Paket. Der Schritt, an dem sich das Paket auf dem Arbeitsplan befindet, in Kombination mit den Seiten- und den Transaktionsberechtigungen, beschränken die Transaktionen, die der angemeldete Benutzer am Paket ausführen kann ebenso wie die nächsten Schritte für das Paket. Z. B. kann ein Paket im Schritt "Entwurf" (Draft) im standardmäßigen Arbeitsplan "Keine Genehmigung" (No Approval) aktualisiert und angezeigt werden, ihm können Inhalte zugewiesen werden und es kann bereitgestellt werden, wenn:

• Dem angemeldeten Benutzer eine Rolle mit Paketeigentümerberechtigungen zugewiesen wurde und
• Die Rolle, die der Änderungsverwaltungs-Spez. "Entwurf" (Draft) zugewiesen wurde, Berechtigungen zum Aktualisieren des Pakets, zum Anzeigen der Paketdetails, zum Zuweisen und Überprüfen von Inhalten und zur Bereitstellung enthält.

Ein Benutzer ohne Paketeigentümerberechtigungen wäre nicht in der Lage, ein Paket zu aktualisieren, auch wenn die Rolle, die der Änderungsverwaltungs-Spez. "Entwurf" (Draft) zugewiesen wurde, diese Transaktionen erlaubt.

Sie können Transaktionsberechtigungen jeder beliebigen Rolle zuweisen und die Rolle dann einem Mitarbeiter und jeder beliebigen Änderungsverwaltungs-Spez. zuweisen.

Beispiel zum Verstehen der Berechtigungen

Das X in der Spalte gibt an, wann Operator 1 die Seite anzeigen und darauf Transaktionen ausführen kann. Jede Ebene zusätzlicher Berechtigungen schränkt die Aktionen ein, die der angemeldete Benutzer ausführen kann:

• Operator 1 wurde die Rolle "Standardseiten" zugewiesen, die die Seiten festlegt, auf die ein Benutzer zugreifen kann. Die Rolle "Standardseiten" ermöglicht den Zugriff auf alle Portalseiten.
• Operator 1 wird die Rolle "Paketbereitsteller" zugewiesen, die die Transaktionen festlegt, die ein Benutzer auf den Änderungsverwaltungsseiten ausführen kann.
• Dem Schritt "Entwurf" (Draft) im Arbeitsplan "Keine Genehmigung" (No Approval) wird die Rolle "DraftPermissions" zugewiesen, die die Transaktionen festlegt, die der angemeldete Benutzer an einem Änderungsverwaltungspaket mit diesem Schritt ausführen kann.
Hinweis: Die Rolle "DraftPermissions", die der Spez. "Entwurf" (Draft) in diesem Szenario zugewiesen wurde, bedeutet keine weitere Einschränkung der Aktionen, die Operator 1 an diesem Paket mit diesem Schritt ausführen kann.
