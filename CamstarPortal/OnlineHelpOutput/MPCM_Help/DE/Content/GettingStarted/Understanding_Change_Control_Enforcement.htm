Verstehen der Durchsetzung der Änderungskontrolle

Siemens bietet die Möglichkeit, die Änderungskontrolle bei allen Modellierungsobjekten oder einer Teilmenge der Modellierungsobjekte in Änderungsverwaltungspaketen durchzusetzen. Es gibt zwei Arten der Durchsetzung der Änderungskontrolle:

• Durchsetzung bei Instanzen in einem Paket auf einem beliebigen Arbeitsplan – Sie können eine Modellierungsobjekt-Instanz einem Paket zuweisen, das einem Arbeitsplan zur Änderungsverwaltung zugeordnet ist.
• Durchsetzung bei Instanzen in einem Paket auf einem Arbeitsplan mit einem Genehmigungsschritt – Sie können eine Modellierungsobjekt-Instanz einem Paket zuweisen, das einem Arbeitsplan zur Änderungsverwaltung zugeordnet ist, welcher einen Genehmigungsschritt enthält.

Die Anwendung lässt nur eine Art der Durchsetzung der Änderungskontrolle pro Modellierungsobjekt zu.

Beim Einrichten der Durchsetzung der Änderungskontrolle

Damit Sie die Änderungskontrolle durchsetzen können, müssen Sie einen Masterdatenkatalog definieren. Mit dem Masterdatenkatalog können Sie Folgendes angeben:

• Die Modellierungsobjekt-Instanzen, die in ein Änderungspaket eingeschlossen werden müssen, und
• Ob die Instanzen in ein beliebiges Änderungspaket eingeschlossen werden können oder nur in Pakete mit einem Genehmigungsschritt.

Sie müssen das Kontrollkästchen "Erfordert Genehmigung" auf mindestens einer Änderungsverwaltungs-Spez. in einem Arbeitsplan zur Änderungsverwaltung mit Genehmigung aktivieren. Die Anwendung verwendet das Kontrollkästchen "Erfordert Genehmigung" mit dem Masterdatenkatalog, um die Änderungskontrolle für diejenigen Modellierungsobjekt-Instanzen durchzusetzen, die nur einem Paket auf einem Genehmigungsarbeitsplan zugeordnet werden sollen.

Die Anwendung verwendet den Masterdatenkatalog und das Kontrollkästchen "Erfordert Genehmigung", um zu gewährleisten, dass Modellierungsobjekt-Instanzen Änderungspaketen auf den geeigneten Arbeitsplänen zugewiesen werden. Weitere Informationen zum Definieren von Stammdatenkatalogen und Änderungsverwaltungsspezifikationen finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Die Anwendung zeigt eine Fehlermeldung an, wenn Sie eine Modellierungsobjekt-Instanz speichern, für die die Änderungskontrolle durchgesetzt wird, und:

• Wenn Sie die Instanz keinem Paket zugeordnet haben.

Oder

• Wenn Sie die Instanz dem falschen Arbeitsplan zugewiesen haben (z. B. wenn die Modellierungsobjekt-Instanz einem Arbeitsplan zugewiesen werden muss, der einen Genehmigungsschritt enthält, Sie sie aber stattdessen einem Arbeitsplan zugewiesen haben, der keinen Genehmigungsschritt enthält).

Beispiele für die Durchsetzung der Änderungskontrolle

Die folgenden Beispiele beschreiben verschiedene Szenarien für die Durchsetzung der Änderungskontrolle.

Beispiel 1: 

Es ist kein Änderungspaket vorhanden. Die Modellierung funktioniert wie immer.

Beispiel 2:

Ein Änderungspaket ist vorhanden, und dieses ist einem Arbeitsplan zur Änderungsverwaltung zugeordnet, aber es wurde kein Masterdatenkatalog definiert:

• Die Anwendung kann die Änderungskontrolle nicht durchsetzen.
• Sie können beliebige neue Modellierungsobjekt-Instanzen definieren, ohne dass die Anwendung von Ihnen verlangt, dass Sie diese dem Paket hinzufügen.
• Sie können dem Paket ungeachtet des zugeordneten Arbeitsplantyps (Genehmigung oder keine Genehmigung) beliebige Modellierungsobjekt-Instanzen zuweisen.

Beispiel 3:

Ein Änderungspaket ist vorhanden, und dieses ist einem Arbeitsplan zur Änderungsverwaltung zugeordnet; außerdem wurde ein Masterdatenkatalog definiert. Die Anwendung:

• Setzt die Änderungskontrolle durch.
• Verlangt von Ihnen, dass Sie Modellierungsobjekt-Instanzen einem Paket zuweisen, wenn der Masterdatenkatalog angibt, dass die Änderungskontrolle für diese Instanzen durchgesetzt werden soll.
• Verwendet die Einstellungen im Masterdatenkatalog und die Einstellung für das Kontrollkästchen "Erfordert Genehmigung" auf den Änderungsverwaltungs-Spez. innerhalb des zugeordneten Arbeitsplans, um zu gewährleisten, dass die Modellierungsobjekt-Instanz zum Paket hinzugefügt wird.

Z. B. gibt der Masterdatenkatalog die Durchsetzung der Änderungskontrolle mit Genehmigung für das Fabrik-Modellierungsobjekt an. Die Anwendung bewertet die Spez. innerhalb des Arbeitsplans, der dem Paket zugeordnet ist, das Sie zur Fabrik-Instanz hinzufügen möchten. Die Anwendung erlaubt Ihnen, die Fabrik-Instanz zum Paket hinzuzufügen, wenn das Kontrollkästchen "Erfordert Genehmigung" auf mindestens einer Spez. im zugeordneten Arbeitsplan aktiviert ist. Die Anwendung zeigt eine Fehlermeldung an, wenn das Kontrollkästchen "Erfordert Genehmigung" nicht auf mindestens einer Spez. im zugeordneten Arbeitsplan aktiviert ist.
