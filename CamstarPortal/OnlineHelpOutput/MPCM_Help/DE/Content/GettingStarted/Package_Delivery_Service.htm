Paketauslieferungsdienst

Siemens stellt einen Auslieferungsdienst zur Verfügung, der bereitgestellte Änderungsverwaltungspakete überwacht und sie an ihre vordefinierten Zielsysteme ausliefert.

Überblick über den Paketauslieferungsdienst

Der Auslieferungsdienst kann in folgende Phasen unterteilt werden:

• Überwachen
• Lesen
• Auslieferung
• Aktualisieren

Überwachungs-Phase

Die Transaktion "Paket bereitstellen" markiert ein Paket zur Abholung durch den Auslieferungsdienst. Pakete, die für die Abholung zur Verfügung stehen, haben den Status "In Bearbeitung" in der Tabelle "Bereitstellungsstatus pro Ziel" auf der Seite "Paket bereitstellen". Der Auslieferungsdienst prüft nach einer geplanten Geschäftsregel auf qualifizierte Pakete. Die standardmäßige geplante Geschäftsregel mit dem Namen SBR_DEPLOY wird alle fünf Minuten ausgeführt.

Ein Paket muss über die Seite "Paket bereitstellen" bereitgestellt werden, bevor es zur Auslieferung für den Auslieferungsdienst zur Verfügung steht. Siehe "Bereitstellen eines Pakets" für weitere Informationen.

Lesephase

Der Auslieferungsdienst überprüft die Quellsystem-Datenbank und liest die Bereitstellungsdetails für ein Paket, um die Zielsysteme eines Pakets zu bestimmen.

Auslieferungsphase

Der Auslieferungsdienst liefert das Änderungsverwaltungspaket und exportiert die XML-Kopfzeilendateien an die ausgewählten Zielsysteme. Die Anwendung holt die XML-Kopfzeilendateien nach einer geplanten Geschäftsregel auf die Zielsysteme. Die standardmäßige geplante Geschäftsregel mit dem Namen SBR_ACTIVATION wird alle fünf Minuten ausgeführt. Das Änderungsverwaltungspaket kann dann auf dem Zielsystem aktiviert werden. Unter "Aktivieren eines Pakets" finden Sie weitere Informationen.

Aktualisierungsphase

Der Auslieferungsdienst aktualisiert das Quellsystem im Anschluss an eine versuchte Auslieferung nach einer geplanten Geschäftsregel mit dem Paketauslieferungszustand. Die standardmäßige geplante Geschäftsregel mit dem Namen SBR_DEPLOYSTATUS wird alle fünf Minuten ausgeführt. Der Dienst aktualisiert die Tabelle "Bereitstellungsstatus pro Ziel" auf der Seite "Paket bereitstellen" für jede Auslieferung in Echtzeit entweder als "Beendet" oder als "Fehlgeschlagen" und sendet eine Fehlermeldung für fehlgeschlagene Auslieferungen. Wenn die Auslieferung anfangs fehlschlägt, muss ein Paket erneut bereitgestellt werden, damit der Auslieferungsdienst versucht, es erneut auszuliefern. Siehe "Bereitstellen eines Pakets" für weitere Informationen.

Weitere Informationen zu geplanten Geschäftsregeln finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch. Sie können die Standardregeln verwenden oder neue Regeln definieren.
