Konfigurieren der Änderungsverwaltung

Sie müssen die Änderungsverwaltung in der Modellierung konfigurieren, bevor Sie mit dem Erstellen und Bereitstellen von Änderungsverwaltungspaketen beginnen können. Weitere Informationen zu den Modellierungsobjektinstanzen der Änderungsverwaltung finden Sie im Opcenter Execution Medical Device and Diagnostics Modeling-Benutzerhandbuch oder im Opcenter Execution Core Modeling-Benutzerhandbuch.

Die folgende Liste beschreibt die spezifischen Aufgaben, die Sie durchführen müssen, um die Änderungsverwaltung zu konfigurieren:

• Gewährleisten Sie, dass sich ein gültiges SSL-Zertifikat auf dem Quellserver und den Zielservern befindet.
• Definieren Sie eine oder mehrere "Paketerstellungsgrund"-Modellierungsobjekt-Instanzen. Die Anwendung erfordert, dass Sie einen Paketerstellungsgrund auswählen, wenn Sie ein Paket erstellen.
• Definieren Sie mindestens zwei "Serverkatalogsystem"-Modellierungsobjekt-Instanzen: eine für den lokalen Host und eine für das Zielsystem. Sie können mehrere Serverkatalogsysteme für zusätzliche Ziele definieren.

Der Auslieferungsdienst verwendet die Informationen zu Benutzername und Passwort aus der "Serverkatalogsystem"-Instanz, die für den Hostserver definiert wurde, um auf den Hostserver zuzugreifen. Benutzername und Kennwort müssen ein gültiges Konto sein, das sich bei der Anwendung anmelden kann.

Wichtig: Sie müssen den Auslieferungsdienst neu starten, wenn Sie den Benutzernamen und das Passwort nach der Ersteinrichtung ändern.
• Führen Sie Folgendes auf den Zielservern durch:
• Erstellen Sie einen Ordner mit dem Namen "Temp".
• Fügen Sie den Benutzer, der in der "Serverkatalogsystem"-Instanz für den Hostserver angegeben wurde, zur Computerverwaltung auf dem Zielserver hinzu.
• Geben Sie den Ordner "Temp" für den Benutzer frei, der in der "Serverkatalogsystem"-Instanz für den Host angegeben wurde, und geben Sie dem Benutzer Lese-/Schreibzugriff.
• Starten Sie den Camstar Deployment Server auf dem Hostserver.
