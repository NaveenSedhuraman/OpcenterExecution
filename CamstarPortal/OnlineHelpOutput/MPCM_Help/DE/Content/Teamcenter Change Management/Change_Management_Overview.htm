Übersicht über Teamcenter Change Management

Teamcenter-Pakete werden in CEP erstellt, wenn eine ECN in CEP heruntergeladen wird. Eine Teamcenter-ECN enthält Informationen für bedingte Prozessanweisungen (BOP) und Stücklisten (BOM), die verwendet werden, um die entsprechenden Modellierungsinstanzen in CEP zu erstellen, die dann in einem CEP-Änderungsverwaltungspaket zusammengefasst werden. Weitere Informationen zum Starten von Teamcenter-Paketen in CEP finden Sie im Opcenter Execution Core Teamcenter Integration-Handbuch.

Sie können die Objekte im Paket anreichern, indem Sie über Camstar Change Inhalte zuweisen oder entfernen. Sie können die meisten Camstar Change-Funktionen für Teamcenter-Pakete ausführen. Weitere Informationen finden Sie im Opcenter Execution Core Teamcenter Integration-Handbuch. Der Genehmigungsprozess für das Teamcenter-Änderungspaket unterscheidet sich geringfügig von der Standardänderungsverwaltung in der Anwendung. Weitere Informationen finden Sie unter "Weiterleiten und Abbrechen von Teamcenter-Genehmigungen".

MOM PLM-Arbeitsplan

Teamcenter-Pakete in CEP werden dem MOM PLM-Arbeitsplan zugewiesen, wenn dies in der Paketerstellungsvorlage festgelegt ist. Der MOM PLM-Arbeitsplan enthält sieben Standardspezifikationen für die Änderungsverwaltung (Schritte):

• Download in Process (Download wird durchgeführt): Ein Teamcenter-Paket, das in CEP heruntergeladen wird, beginnt und bleibt in diesem Schritt, bis es erfolgreich heruntergeladen wurde.
Hinweis: Sie müssen sich an den Teamcenter-Ingenieur wenden, wenn das Paket nicht erfolgreich in CEP heruntergeladen wird.
• Pending Enrichment (Anreicherung ausstehend): Sie können die Objekte im Paket anreichern, indem Sie Inhalte zuweisen oder entfernen. Weitere Informationen zum Hinzufügen von Paketinhalten finden Sie unter "Zuweisen und Überprüfen des Paketinhalts". Sie können ein Paket in diesem Schritt zur Genehmigung weiterleiten.
• Pending Approval PLM (Genehmigung ausstehend PLM): Ein Paket wird in den Schritt "Pending Approval PLM" (Genehmigung ausstehend PLM) verschoben, wenn Sie ein Paket im Schritt "Pending Enrichment" (Anreicherung ausstehend) zur Genehmigung weiterleiten. An Teamcenter werden zwei verschlüsselte PDF-Dateien gesendet, die Paketdetails und Inhaltsänderungshistorie enthalten. Weitere Informationen finden Sie unter "Weiterleiten und Abbrechen von Teamcenter-Genehmigungen".
• Rejected (Abgelehnt): Ein Paket wird in den Schritt "Rejected" (Abgelehnt) verschoben, wenn ein Teamcenter-Ingenieur ein Paket ablehnt, das zur Genehmigung gesendet wurde.
• Pending Deployment (Bereitstellung ausstehend): Ein Paket wird automatisch in den Schritt "Pending Deployment" (Bereitstellung ausstehend) verschoben, wenn der Teamcenter-Ingenieur ein Paket genehmigt. Sie können ein Paket in diesem Schritt auf den Zielmaschinen bereitstellen. Siehe "Bereitstellen eines Pakets" für weitere Informationen.
• Deployment Complete (Bereitstellung beendet): Ein Paket wird in den Schritt "Deployment Complete" (Bereitstellung beendet) verschoben, wenn es erfolgreich auf den Zielmaschinen bereitgestellt wurde.
• Deployment Incomplete (Bereitstellung unvollständig): Ein Paket wird in den Schritt "Deployment Incomplete" (Bereitstellung unvollständig) verschoben, wenn die Paketbereitstellung fehlschlägt.

MOM PLM-Spezifikationen, -Rollen und -Berechtigungen

Im MOM-Arbeitsbereich werden die folgenden zusätzlichen Änderungsverwaltungsspezifikationen zur Verfügung gestellt:

Spez.

Beschreibung

Standard-Bilddatei

Download wird durchgeführt

Zeigt an, dass ein Teamcenter-Paket nach CEP übertragen wird. Ein Teamcenter-Paket bleibt in diesem Schritt, bis es erfolgreich nach CEP heruntergeladen wurde.

Die erlaubte Transaktionsrolle für die Spezifikation "Download wird durchgeführt" ist "DownloadPermissions".

Open.png

Anreicherung ausstehend

Zeigt an, dass ein Teamcenter-Paket erfolgreich in die Anwendung heruntergeladen wurde und Benutzer Paketaufgaben durchführen können, z. B. Paketattribute aktualisieren und Inhalte zuweisen.

Die erlaubte Transaktionsrolle für die Spezifikation "Anreicherung ausstehend" ist "PEPermissions". Benutzer mit den entsprechenden Berechtigungen können im Schritt "Anreicherung ausstehend" die folgenden Aktionen für Pakete ausführen:

• Nach dem Paket suchen
• Die Paketdetails anzeigen
• Kernattribute des Pakets bearbeiten
• Den Paketschritt ändern
• Dem Paket Inhalte zuweisen
• Das Paket bereitstellen

Hinweis: Paketattribute und zugewiesene Paketinhalte können nicht außerhalb dieses Schrittes geändert werden.
